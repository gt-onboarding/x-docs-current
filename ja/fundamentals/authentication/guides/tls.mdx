X API エンドポイントにアクセスするには、TLS 接続が必須です。TLS を利用して通信することで、公開インターネットを経由してユーザーと X API 間で情報が送受信される際に、その情報を保護し、ユーザーのプライバシーとセキュリティを維持できます。X API への接続には、TLS バージョン 1.2 が必要です。


<div id="verification">
  ## 検証
</div>

<div id="use-an-up-to-date-root-store">
  ### 最新のルートストアを使用する
</div>

X の証明書を検証する際、アプリケーションやライブラリが信頼でき、かつ最新のルートストアを使用していることが重要です。可能であれば、オペレーティングシステムが提供するルートストアを利用するのが最も簡単な方法となる場合があります。あるいは、[Mozilla (NSS) ルートストア](https://www.mozilla.org/en-US/about/governance/policies/security-group/certs/) は公開かつ透明性の高い形で適切に管理・維持されています。cURL も、[このストアの PEM 形式バージョン](https://curl.haxx.se/docs/caextract.html) を提供しています。

X が現在発行している証明書の大部分は [DigiCert High Assurance EV Root CA](https://www.digicert.com/digicert-root-certificates.htm) によるものですが、これは X 関連証明書の 100% に当てはまるわけではなく、将来にわたって常にそうであるとも限りません。そのため、現在使用されている DigiCert のルート証明書のみを信頼していると、将来的にアプリで問題が発生する可能性があります。

<div id="check-crls-and-the-ocsp-status">
  ### CRL と OCSP ステータスを確認する[](#check-crls-and-the-ocsp-status "Permalink to this headline")
</div>

多くのアプリケーションは、受信した証明書に対して証明書失効リスト (CRL) を確認しないか、その処理をオペレーティングシステムに依存しています。X の証明書を受け入れる前に、アプリケーションまたは TLS ライブラリが CRL および OCSP (Online Certificate Status Protocol) による検証を必須とするよう構成されていることを確認してください。

<div id="cdns">
  ### CDNs[](#cdns "Permalink to this headline")
</div>

メディアを含むポストを表示する場合は、画像を表示するための HTTPS URL として `media_url_https` 属性を使用してください。将来的には、API エンドポイントから配信されるすべての URL が HTTPS パスを提供するようになります。

<div id="provide-an-indication-of-security-status">
  ## セキュリティ状態を示す
</div>

可能であれば、アプリケーションとXとの間の現在のセキュリティ状態を表示するようにしてください。一部のWebブラウザはロックアイコンを表示することでこれを示し、他のブラウザは現在の接続状態を説明的なメッセージで示します。
