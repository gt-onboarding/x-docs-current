X API は膨大な量のデータを扱います。このデータを開発者とユーザーの双方にとって安全に保護するために用いる仕組みが認証です。認証にはいくつかの方法があり、以下で説明します。

ほとんどの開発者は、認証まわりの複雑さに対処する必要はありません。クライアントライブラリがこれらを自動的に処理してくれるためです。

利用可能なクライアントライブラリの一覧は、[ツールとライブラリ](/ja/resources/tools-and-libraries)ページで確認できます。

<div id="authentication-methods">
  ## 認証方法
</div>

<CardGroup cols={2}>
  <Card title="OAuth 1.0a User Context" href="/ja/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret">
    OAuth 1.0a を使用すると、認可された X の開発者アプリが、X アカウントの非公開のアカウント情報にアクセスしたり、そのアカウントに代わって X 上でアクションを実行したりできます。

    <br />

    [**詳細はこちら**](/ja/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret)
  </Card>

  <Card title="App only" href="/ja/resources/fundamentals/authentication/oauth-2-0/overview">
    App only アクセストークンを使用すると、X の開発者アプリが X 上で一般公開されている情報にアクセスできます。

    <br />

    [**詳細はこちら**](/ja/resources/fundamentals/authentication/oauth-2-0/overview)
  </Card>

  <Card title="Basic authentication" href="/ja/resources/fundamentals/authentication/basic-auth">
    多くの X Enterprise API では、HTTP Basic Authentication の使用が必要です。

    <br />

    [**詳細はこちら**](/ja/resources/fundamentals/authentication/basic-auth)
  </Card>

  <Card title="OAuth 2.0 Authorization Code Flow with PKCE" href="/ja/resources/fundamentals/authentication/oauth-2-0/authorization-code">
    OAuth 2.0 User Context を使用すると、アプリケーションのスコープや複数デバイス間にまたがる認可フローをより細かく制御しながら、別のアカウントに代わって認証を行うことができます。

    <br />

    [**詳細はこちら**](/ja/resources/fundamentals/authentication/oauth-2-0/authorization-code)
  </Card>
</CardGroup>

<Note>
  **注意：**
  ご利用のアプリの API Key と App only アクセストークン、およびご自身のアクセストークンと Access Token Secret は、[developer portal](/ja/resources/fundamentals/developer-portal) 内の [X developer Apps](/ja/resources/fundamentals/developer-apps) セクションから取得できます。

  **別のユーザーに代わってリクエストを行いたい場合**、[3-legged OAuth フロー](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens) を使用してそのユーザー用のアクセストークン一式を別途生成し、OAuth 1.0a User Context または OAuth 2.0 User Context のリクエストに、そのユーザーのトークンを付けて送信する必要があります。
</Note>
