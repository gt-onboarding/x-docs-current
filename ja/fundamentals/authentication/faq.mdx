<div id="general">
  ## 一般
</div>

<AccordionGroup>
  <Accordion title="OAuth とは何ですか？">
    OAuth は、ユーザーが自分のパスワードを共有することなく、自分に代わってアプリケーションに操作を許可できる認証プロトコルです。詳細は [oauth.net](http://oauth.net/) を参照してください。
  </Accordion>

  <Accordion title="アクセストークンはどのように生成しますか？">
    アクセストークンを生成するには、[X app](/ja/resources/fundamentals/developer-apps) が必要です。アクセストークンの詳細は [こちら](/ja/resources/fundamentals/authentication#oauth-1-0a-2) をご覧ください。
  </Accordion>

  <Accordion title="X app はどのように作成しますか？">
    [X app](/ja/resources/fundamentals/developer-apps) を作成するには、[開発者アカウント](/ja/resources/fundamentals/developer-portal) が必要です。サインアップは [こちら](https://developer.x.com/en/portal/petition/essential/basic-info) から行えます。
  </Accordion>

  <Accordion title="すでに X app を持っている場合、そのアプリはどのように表示・編集できますか？">
    developer.x.com 上で X アカウントにログインしている場合、[X app dashboard](https://developer.x.com/content/developer-twitter/en/apps) から自分のアプリを表示および編集できます。
  </Accordion>
</AccordionGroup>

<div id="technical">
  ## 技術情報
</div>

<AccordionGroup>
  <Accordion title="アクセストークンの有効期間はどのくらいですか？">
    アクセストークンには明示的な有効期限はありません。ユーザーが自分の X アカウント設定でアプリの連携を明示的に解除した場合、または X がアプリを凍結した場合、そのアクセストークンは無効化されます。アプリが凍結された場合は、[X app](/ja/resources/fundamentals/developer-apps) のダッシュボードに、そのアプリが凍結されている旨の注意書きが表示されます。
  </Accordion>

  <Accordion title="アクセストークンが無効になった場合はどうすればよいですか？">
    ユーザーのアクセストークンは、いつでも無効になる可能性があると想定してください。このような状況になった場合は、ユーザーにアプリを再度認可するよう促してください。この状況を適切に処理することは、良好なユーザー体験のために重要です。
  </Accordion>

  <Accordion title="アプリ登録ページで read/write アクセスについて尋ねられます。write とは何を指しますか？">
    多くのユーザーは、アプリが自分の情報を読み取ることには同意しても、ユーザープロフィール情報を変更したり、新しいポストを投稿したりすることまでは必ずしも許可しません。X API を通じて情報を更新する場合 — たとえば名前や位置情報の更新、新しいポストの投稿など — は、HTTP POST が必要です。HTTP POST を必要とする API メソッドはすべて write メソッドと見なされ、read &amp; write アクセスが必要です。
  </Accordion>
</AccordionGroup>
