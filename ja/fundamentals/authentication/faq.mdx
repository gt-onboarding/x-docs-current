<div id="general">
  ## 全般
</div>

<AccordionGroup>
  <Accordion title="OAuth とは何ですか？">
    OAuth は、ユーザーがパスワードを共有することなく、自分に代わってアプリに操作を許可できるようにする認証プロトコルです。詳細は [oauth.net](http://oauth.net/) を参照してください。
  </Accordion>

  <Accordion title="アクセストークンはどのように生成しますか？">
    アクセストークンを生成するには、[X アプリ](/ja/resources/fundamentals/developer-apps) が必要です。アクセストークンの詳細については [こちら](/ja/resources/fundamentals/authentication#oauth-1-0a-2) を参照してください。
  </Accordion>

  <Accordion title="X アプリはどのように作成しますか？">
    [X アプリ](/ja/resources/fundamentals/developer-apps) を作成するには、[開発者アカウント](/ja/resources/fundamentals/developer-portal) が必要です。サインアップは [こちら](https://developer.x.com/en/portal/petition/essential/basic-info) から行えます。
  </Accordion>

  <Accordion title="すでに X アプリを持っている場合、そのアプリを表示・編集するにはどうすればよいですか？">
    developer.x.com 上で X アカウントにログインしていれば、[X アプリダッシュボード](https://developer.x.com/content/developer-twitter/en/apps) からアプリを表示および編集できます。
  </Accordion>
</AccordionGroup>

<div id="technical">
  ## Technical
</div>

<AccordionGroup>
  <Accordion title="アクセストークンはどのくらいの期間有効ですか？">
    アクセストークンには明示的な有効期限はありません。ユーザーが自分の X アカウント設定でアプリケーションを明示的に取り消した場合、または X がアプリケーションを停止した場合、そのアクセストークンは無効化されます。アプリケーションが停止された場合は、[X app](/ja/resources/fundamentals/developer-apps) ダッシュボードに、そのアプリケーションが停止されたことを示すメッセージが表示されます。
  </Accordion>

  <Accordion title="アクセストークンが無効になった場合はどうすればよいですか？">
    ユーザーのアクセストークンは、いつでも無効になる *可能性がある* と想定してください。このような場合は、ユーザーに対してアプリケーションを再認可するよう促してください。この状況を適切に処理することは、良いユーザー体験のために重要です。
  </Accordion>

  <Accordion title="アプリケーション登録ページで read/write アクセスについて尋ねられます。write とは何を意味しますか？">
    多くのユーザーは、アプリケーションが自分の情報を読むことには許可しますが、必ずしもユーザープロフィール情報を変更したり、新しいポストを投稿したりすることまでは信頼していません。X API を介して情報を更新する操作は、名前や場所の変更、新しいポストの追加など、いずれの場合も HTTP POST を必要とします。HTTP POST を必要とする任意の API メソッドは write メソッドと見なされ、read &amp; write アクセスが必要です。
  </Accordion>
</AccordionGroup>
