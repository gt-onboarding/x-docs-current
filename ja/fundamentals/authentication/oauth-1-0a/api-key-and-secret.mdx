<div id="api-key-and-secret">
  ### API Key and Secret
</div>

API Key と Secret（Consumer Key と Secret とも呼ばれます）は、X API にアクセスするために必要となる最も基本的な認証情報です。これらの認証情報はあなたの X アプリのユーザー名とパスワードのような役割を果たし、どのアプリからのリクエストであるかを X API が識別するために使用されます。 

これらの認証情報は、[認証エンドポイント](/ja/resources/fundamentals/authentication/api-reference) で [ユーザーの Access Token と Secret](/ja/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret) や [Bearer Token](/ja/resources/fundamentals/authentication/oauth-2-0/bearer-tokens) などの追加の認証情報を生成するために使用できます。また、OAuth 1.0a User Context 認証を必要とするリクエストを[認可する](/ja/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request)には、これらの認証情報に加え、Access Token やその他の認可パラメータを組み合わせて使用する必要があります。 

<div id="how-to-acquire-an-api-key-and-secret">
  #### API Key と Secret の取得方法
</div>

X API の API Key と Secret を取得するには、次の手順に従ってください。

1. [X 開発者アカウントにサインアップする](https://developer.x.com/en/apply-for-access)
2. [developer portal](/ja/resources/fundamentals/developer-portal) 内で [X App](/ja/resources/fundamentals/developer-apps) を作成します。[X API v2](/ja/x-api/introduction) を使用したい場合は、その X App を必ず [Project](/ja/resources/fundamentals/projects) に追加してください。
    

X App を作成すると、API Key と Secret、および Bearer Token が表示されます。これらの認証情報は一度しか表示されないため、パスワードマネージャーなどの安全な場所に必ず保存してください。

[authentication best practices](/ja/resources/fundamentals/authentication/guides/authentication-best-practices) のページでは、キーとトークンの扱い方について、認証情報が漏えいした場合に取るべき対応など、より詳しい推奨事項を記載しています。
 

<div id="how-to-find-and-regenerate-your-api-key-and-secret-after-app-creation">
  #### アプリ作成後に APIキーとシークレットを見つけて再生成する方法
</div>

すでにアプリを作成済みで、APIキーとシークレットを確認または再生成する必要がある場合は、次の手順に従ってください。

1. developer portal にアクセスします
2. サイドナビで「Projects and Apps」ドロップダウンを展開します
3. 対象の APIキーとシークレットが関連付けられているアプリを開きます
4. 「Keys and tokens」タブを開きます

ここで、アプリに関連付けられているすべての認証情報を確認できます。 
 

<div id="how-to-use-your-api-key-and-secret">
  #### API Key と Secret の使い方
</div>

X Developer Platform を試しに触っている段階であれば、まずはプラットフォームで何が利用できるかを確認するために [tool or library](/ja/resources/tools-and-libraries) を使うことをおすすめします。これらのツールは認証をスムーズに処理してくれるため、多くの時間と手間を省くことができます。特に、初心者の開発者には [getting started with Postman](/ja/tutorials/postman-getting-started) または [Insomnia](https://insomnia.rest/) から始めることを推奨します。 

リクエストを一から自分で組み立てたい場合は、[authorizing an OAuth 1.0a request](/ja/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request) に関するガイドを参照してください。
