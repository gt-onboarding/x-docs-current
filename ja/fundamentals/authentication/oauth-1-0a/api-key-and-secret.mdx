<div id="api-key-and-secret">
  ### API Key and Secret
</div>

API Key と Secret（Consumer Key と Secret とも呼ばれます）は、X API にアクセスするために必要となる最も基本的な認証情報です。これらの認証情報は、X アプリに対するユーザー名とパスワードの役割を果たし、どのアプリからのリクエストかを X API が識別するために使用されます。 

これらの認証情報は、[authentication endpoints](/ja/resources/fundamentals/authentication/api-reference) によって、[user Access Tokens and Secrets](/ja/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret) や [Bearer Tokens](/ja/resources/fundamentals/authentication/oauth-2-0/bearer-tokens) などの追加の認証情報を生成するために使用されます。また、OAuth 1.0a User Context 認証を必要とするリクエストを[認可する](/ja/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request)には、アクセストークンやその他の認可パラメータとあわせて、これらの認証情報を使用する必要があります。 

<div id="how-to-acquire-an-api-key-and-secret">
  #### API Key と Secret の取得方法
</div>

X API Key と Secret を取得するには、次の手順に従ってください。

1. [X 開発者アカウントにサインアップ](https://developer.x.com/en/apply-for-access)します
2. [developer portal](/ja/resources/fundamentals/developer-portal) 内で [X App](/ja/resources/fundamentals/developer-apps) を作成します。[X API v2](/ja/x-api/introduction) を利用したい場合は、その X App を必ず [プロジェクト](/ja/resources/fundamentals/projects) に追加してください。
    

X App を作成すると、API Key と Secret に加えて、Bearer Token が表示されます。これらの認証情報は最初の 1 回しか表示されないため、パスワードマネージャーなどの安全な場所に必ず保存してください。

キーとトークンの扱い方については、認証情報が漏えいした場合に行うべき対応も含め、[authentication best practices](/ja/resources/fundamentals/authentication/guides/authentication-best-practices) ページで、より詳しい推奨事項を提供しています。
 

<div id="how-to-find-and-regenerate-your-api-key-and-secret-after-app-creation">
  #### アプリ作成後に API Key と Secret を見つけて再生成する方法
</div>

すでにアプリを作成していて、API Key と Secret を確認または再生成する必要がある場合は、次の手順に従ってください。

1. developer portal に移動します
2. サイドナビで「Projects and Apps」ドロップダウンを展開します
3. 対象の API Key と Secret が関連付けられている「App」を開きます
4. 「Keys and tokens」タブに移動します

ここで、アプリに関連付けられているすべての認証情報を確認できます。 
 

<div id="how-to-use-your-api-key-and-secret">
  #### API Key と Secret の使用方法
</div>

X Developer Platform を探索している段階であれば、まずはプラットフォーム上で利用可能な機能を把握するために、[ツールやライブラリ](/ja/resources/tools-and-libraries)の利用をおすすめします。これらのツールは認証を適切に処理してくれるため、多くの時間と手間を省くことができます。特に、初心者の開発者には [Postman で始める](/ja/tutorials/postman-getting-started) か [Insomnia](https://insomnia.rest/) の利用を推奨します。 

リクエストを一から構築したい場合は、[OAuth 1.0a リクエストの認可](/ja/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request) に関するガイドを参照してください。
