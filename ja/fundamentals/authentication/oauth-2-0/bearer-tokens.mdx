<div id="using-and-generating-an-app-only-bearer-token">
  ### アプリ専用 Bearer Token の使用と生成
</div>

Bearer Token は、開発者が X API を利用する際の、より安全なエントリーポイントを提供するものであり、OAuth 2.0 の中核機能の 1 つです。

Bearer Token を使った認証は、アプリケーション専用認証（application-only authentication）とも呼ばれます。Bearer Token は、curl コマンドのようなスクリプトを使って生成する、形式が定められていないバイト配列です。Bearer Token は、developer portal のアプリ設定内にある「キーとトークン」セクションから取得することもできます。この機能の詳細については、[OAuth の公式ドキュメント](https://oauth.net/2/bearer-tokens/)を参照してください。

<div id="when-are-they-used">
  #### どのような場合に使用しますか？
</div>

Bearer Token の使用が必要となるプロダクトは次のとおりです。

* [Engagement API](/ja/x-api/enterprise-gnip-2.0/fundamentals/engagement-api)
* [Account Activity API](/ja/x-api/enterprise-gnip-2.0/fundamentals/account-activity)
* v2 や Labs のエンドポイントなど、OAuth 2.0 Bearer Token 認証を利用するその他の API。

<div id="prerequisites">
  #### 前提条件
</div>

[開発者アカウントにサインアップ](https://developer.x.com/en/portal/petition/essential/basic-info)し、[X App](/ja/resources/fundamentals/developer-apps) を作成しておく必要があります。これらが準備できたら、さらに [developer portal](/ja/resources/fundamentals/developer-portal) で確認できる API キーを取得する必要があります。以下の手順に従ってください。

1. developer.x.com で自分の X アカウントにログインします。
2. [X App dashboard](https://developer.x.com/content/developer-twitter/en/apps) に移動し、アクセストークンを生成したい X App を開きます。
3. 「keys and tokens」ページを開きます。
4. このページに API キー、ユーザーのアクセストークン、および Bearer Token が表示されます。

<div id="how-to-generate-a-bearer-token">
  ### Bearer Token の生成方法
</div>

アプリの Bearer Token は、他の「キーとトークン」と同じ場所で確認できます。

以下の cURL リクエストを、[X App](/ja/resources/fundamentals/developer-apps) から事前に取得した次のコンシューマー API キーに置き換えたうえで、コマンドラインにコピーして実行してください。なお、このページで使用しているコンシューマー API キーはすでに無効化されており、実際のリクエストには使用できません。

* **API key** `<API key>` 例：`xvz1evFS4wEEPTGEFPHBog`
* **API secret key** `<API secret key>` 例：`L8qq9PZyRg6ieKGEKhZolGC0vJWLw8iEJ88DRdyOg`

  ```bash
  curl -u "$API_KEY:$API_SECRET_KEY" \
  --data 'grant_type=client_credentials' \
  'https://api.x.com/oauth2/token'
  ```

以下は、API キーを入力した状態での cURL リクエスト例です。

```bash
   curl -u 'xvz1evFS4wEEPTGEFPHBog:L8qq9PZyRg6ieKGEKhZolGC0vJWLw8iEJ88DRdyOg' \
  --data 'grant_type=client_credentials' \
  'https://api.x.com/oauth2/token'
```

レスポンスは次のようになります。これはすでに無効化されている Bearer Token であることに注意してください。

`{"token_type":"bearer","access_token":"AAAAAAAAAAAAAAAAAAAAAMLheAAAAAAA0%2BuSeid%2BULvsea4JtiGRiSDSJSI%3DEUifiRBkKG5E2XzMDjRfl76ZC9Ub0wnz4XsNiRVBChTYbJcE3F"}`

OAuth 2.0 を使ってリソースへアクセスする際の認証に使用していた Bearer Token は次のとおりです。

`AAAAAAAAAAAAAAAAAAAAAMLheAAAAAAA0%2BuSeid%2BULvsea4JtiGRiSDSJSI%3DEUifiRBkKG5E2XzMDjRfl76ZC9Ub0wnz4XsNiRVBChTYbJcE3F`
