<div id="introduction">
  ## はじめに
</div>

プロジェクトを使用すると、X API の利用目的に応じて作業を整理できるため、API へのアクセスを効果的に管理し、利用状況を監視できます。各プロジェクトには、アクセスレベル（このページで後述）に応じて 1 つ以上の[アプリ](/ja/resources/fundamentals/developer-apps)を含めることができます。これらのアプリを使用して、[API Keys and Secrets](/ja/resources/fundamentals/authentication#api-key-and-secret)、ユーザー[Access Tokens](/ja/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)、[App Access Token](/ja/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) などの[認証](/ja/resources/fundamentals/authentication)用資格情報を生成できます。

これらのキーとトークンは、どのアプリからのものであっても X API や X Ads API へのアクセスに使用できます（Ads API を利用するには[追加アクセスを申請](/ja/x-ads-api/getting-started/step-by-step-guide)する必要があります）。ただし、[X API v2](/ja/x-api/introduction) のエンドポイントを利用するには、プロジェクトに関連付けられたアプリのキーとトークンを使用する必要があります。

開発者アカウントをお持ちの場合は、developer portal の [Projects &amp; Apps](https://developer.x.com/en/portal/projects-and-apps) ページでプロジェクトを表示および管理できます。まだアカウントをお持ちでない場合は、[Sign up](https://developer.x.com/en/portal/petition/essential/basic-info) からアカウントを作成してください。

<div id="projects-and-x-api-access-levels">
  ### プロジェクトと X API アクセスレベル
</div>

現時点では、プロジェクト単位で適用される 4 つの異なるアクセスレベルがあります。

* Free
* Basic
* Pro
* Enterprise

それぞれのアクセスレベルで利用できる内容の詳細については、[about X API](/ja/x-api/getting-started/about-x-api) ページをご覧ください。

Free、Basic、Pro のいずれかのアクセスレベルを持つプロジェクトは、1 つだけ保有できます。

ここでは、いくつかのプロジェクト固有の違いについて説明します。

<div id="free">
  #### Free（無料）
</div>

このアクセスレベルは、開発者アカウントに[登録](https://developer.twitter.com/en/portal/petition/essential/basic-info)したすべての方が利用できます。

そのプロジェクト内のアプリ数: 1\
[ポスト消費上限](/ja/x-api/fundamentals/post-cap): 月あたり 1,500 ポスト

認証方法:

* [OAuth 2.0 with PKCE](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
* App-only（アプリのみ）

<div id="basic">
  #### Basic
</div>

このアクセスレベルは、developer portal から Basic アクセスに登録したすべてのユーザーに提供されます。

そのプロジェクト内のアプリ数: 2
[ポスト消費上限](/ja/x-api/fundamentals/post-cap): 月あたり 10,000 ポスト
認証方式:

* [OAuth 2.0 with PKCE](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
* [OAuth 1.0a](/ja/resources/fundamentals/authentication#oauth-1-0a-2)
* アプリのみ（App-only）

<div id="pro">
  #### Pro
</div>

このアクセスレベルは、developer portal を通じて Pro アクセスに登録したすべてのユーザーに提供されます。

当該プロジェクト内のアプリ数: 3\
[ポスト取得上限](/ja/x-api/fundamentals/post-cap): 月あたり 100万ポスト

認証方法:

* [OAuth 2.0 with PKCE](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
* [OAuth 1.0a](/ja/resources/fundamentals/authentication#oauth-1-0a-2)
* アプリのみ (App-only)

<div id="enterprise">
  #### Enterprise
</div>

このアクセスレベルは、Enterprise に申し込み、当社チームの承認を受け、X のアカウントマネージャー経由で月次の自動更新契約に署名した方に提供されます。[こちらから申請してください](https://docs.google.com/forms/d/e/1FAIpQLScO3bczKWO2jFHyVZJUSEGfdyfFaqt2MvmOfl_aJp0KxMqtDA/viewform)。

当該プロジェクト内のアプリ数: 3 以上\
[ポスト消費上限](/ja/x-api/fundamentals/post-cap): 月あたり 5,000 万件超のポスト

認証方法:

* [OAuth 2.0 with PKCE](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
* [OAuth 1.0a](/ja/resources/fundamentals/authentication#oauth-1-0a-2)
* アプリのみ（App-only）

<div id="tweet-caps-and-projects">
  ### ポスト上限とプロジェクト
</div>

ポスト消費の上限はプロジェクト単位で適用され、特定の X API v2 エンドポイントから 1 か月間に取得できるポスト数が制限されます。

詳しくは、[ポスト上限](/ja/x-api/fundamentals/post-cap)をご覧ください。

<div id="configuring-projects">
  ### プロジェクトの設定
</div>

<div id="creating-a-project">
  #### プロジェクトの作成
</div>

プロジェクトを作成するには、[ダッシュボード](https://developer.twitter.com/en/portal/dashboard) か developer portal 内の [Projects &amp; Apps](https://developer.twitter.com/en/portal/projects-and-apps) ページで「New Project」をクリックします。まだプロジェクトを作成していない場合にのみ、このオプションが表示されます。プロジェクト名、説明、利用目的の入力を求められます。また、新しい App を作成するか、既存のスタンドアロン App を接続するかを選択するよう求められます。

<div id="creating-or-connecting-an-app-for-your-project">
  #### プロジェクトにアプリを作成または接続する
</div>

プロジェクトにアプリが紐づいていない場合は、ダッシュボードでプロジェクト名をクリックしてアプリを追加できます。そこから、新しいアプリを作成するか、既存のスタンドアロンのアプリを選択してプロジェクトに接続できます。アプリでは、このガイドの冒頭で一覧した認証用のキーとトークンを生成できます。

<div id="editing-a-project">
  #### プロジェクトの編集
</div>

プロジェクトを編集するには、developer portal 内のダッシュボードまたは Projects &amp; Apps ページから、編集したいプロジェクト名をクリックします。次に表示されるそのプロジェクトの詳細画面で「Edit」を選択し、変更を加えます。

<div id="standalone-apps">
  ### スタンドアロンアプリ
</div>

スタンドアロンアプリは、プロジェクト構造の外に存在するアプリです。これらのスタンドアロンアプリに関連付けられた認証情報を使用すると、X の API の standard v1.1、premium v1.1、Enterprise、または X Ads API へのリクエストを正常に行えます。スタンドアロンアプリは、プロジェクトに接続しない限り、X API v2 のエンドポイントへのリクエストは失敗します。

2020 年 8 月より前にアプリを作成している場合、それらは developer portal の [Projects &amp; Apps](https://developer.twitter.com/en/portal/projects-and-apps) 配下の「Standalone Apps」セクションに表示されます。プロジェクトに接続されているアプリを含め、アプリの数は合計 10 個までに制限されます。

チームアカウントのメンバーである場合は、自分が所有しているアプリが Standalone Apps の下に表示されます。チームメイトがプロジェクトの一部として所有しているアプリについては、そのアプリ名と所有者情報は表示されますが、その設定を変更したり、そのキーとトークンにアクセスしたり、そのキーとトークンを再生成したりすることはできません。アプリに変更を加える必要がある場合は、そのアプリの所有者に連絡してください。
