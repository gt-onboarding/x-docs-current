<div id="introduction">
  ## はじめに
</div>

プロジェクトを使用すると、X API の利用目的に応じて作業内容を整理できるため、API へのアクセスを効果的に管理し、利用状況を監視できます。各プロジェクトには、アクセスレベル（このページの後半で説明します）に応じて 1 つ以上の[アプリ](/ja/resources/fundamentals/developer-apps)を含めることができます。これらのアプリを使用して、[API Keys and Secrets](/ja/resources/fundamentals/authentication#api-key-and-secret)、ユーザー[Access Tokens](/ja/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)、[App Access Token](/ja/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) などの[認証](/ja/resources/fundamentals/authentication)用の認証情報を生成します。

これらのキーとトークンは、どのアプリのものであっても X API または X Ads API へのアクセスに使用できます（Ads API を使用するには、[追加アクセスの申請](/ja/x-ads-api/getting-started/step-by-step-guide) が必要です）。ただし、[X API v2](/ja/x-api/introduction) のエンドポイントを利用するには、プロジェクトに関連付けられたアプリのキーとトークンを使用する必要があります。

開発者アカウントをお持ちの場合は、開発者ポータル内の [Projects &amp; Apps](https://developer.x.com/en/portal/projects-and-apps) ページでプロジェクトを表示および管理できます。まだお持ちでない場合は、[Sign up](https://developer.x.com/en/portal/petition/essential/basic-info) してアカウントを作成してください。

<div id="projects-and-x-api-access-levels">
  ### プロジェクトと X API のアクセスレベル
</div>

現時点では、プロジェクトレベルで適用されるアクセスレベルには、次の 4 種類のティアがあります。

* Free
* Basic
* Pro
* Enterprise

各アクセスレベルで利用できる内容の詳細については、[about X API](/ja/x-api/getting-started/about-x-api) ページをご覧ください。

Free、Basic、Pro のいずれかのアクセスレベルを持つプロジェクトは、1 つしか保持できません。

ここでは、いくつかのプロジェクト固有の違いについて説明します。

<div id="free">
  #### Free
</div>

このアクセスレベルは、[サインアップ](https://developer.twitter.com/en/portal/petition/essential/basic-info)して開発者アカウントを取得したすべての方に付与されます。

そのプロジェクト内のアプリ数: 1\
[ポスト利用上限](/ja/x-api/fundamentals/post-cap): 月あたり 1,500 件のポスト

認証方法:

* [OAuth 2.0 with PKCE](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
* アプリのみ

<div id="basic">
  #### Basic
</div>

このアクセスレベルは、Developer Portal から Basic アクセスを申し込んだすべての方に提供されます。

そのプロジェクト内のアプリ数: 2
[ポスト取得上限](/ja/x-api/fundamentals/post-cap): 月あたり 10,000 ポスト
認証方法:

* [OAuth 2.0 with PKCE](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
* [OAuth 1.0a](/ja/resources/fundamentals/authentication#oauth-1-0a-2)
* アプリのみ（App-only）

<div id="pro">
  #### Pro
</div>

このアクセスレベルは、Developer Portal から Pro アクセスを契約したすべての方に提供されます。

そのプロジェクト内のアプリ数: 3\
[ポスト消費上限](/ja/x-api/fundamentals/post-cap): 月あたり 100 万件のポスト

認証方法:

* [OAuth 2.0 with PKCE](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
* [OAuth 1.0a](/ja/resources/fundamentals/authentication#oauth-1-0a-2)
* アプリのみ（App only）

<div id="enterprise">
  #### Enterprise
</div>

このアクセスティアは、Enterprise に申請し、当社チームによる承認を受け、X のアカウントマネージャー経由で毎月自動更新契約に署名しているお客様に提供されます。[こちらから申請できます](https://docs.google.com/forms/d/e/1FAIpQLScO3bczKWO2jFHyVZJUSEGfdyfFaqt2MvmOfl_aJp0KxMqtDA/viewform)。

そのプロジェクト内のアプリ数: 3 以上\
[ポスト消費上限](/ja/x-api/fundamentals/post-cap): 月あたり 5,000 万件以上のポスト

認証方法:

* [OAuth 2.0 with PKCE](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
* [OAuth 1.0a](/ja/resources/fundamentals/authentication#oauth-1-0a-2)
* アプリのみ

<div id="tweet-caps-and-projects">
  ### ポスト上限とプロジェクト
</div>

ポスト取得上限はプロジェクト単位で適用され、特定の X API v2 エンドポイントから 1 か月間に取得できるポスト数が制限されます。

[ポスト上限](/ja/x-api/fundamentals/post-cap)について詳しくはご覧ください。

<div id="configuring-projects">
  ### プロジェクトの設定
</div>

<div id="creating-a-project">
  #### プロジェクトの作成
</div>

プロジェクトを作成するには、developer portal 内の[ダッシュボード](https://developer.twitter.com/en/portal/dashboard)または [Projects &amp; Apps](https://developer.twitter.com/en/portal/projects-and-apps) ページで「New Project」をクリックします。まだプロジェクトを作成していない場合にだけ、このオプションが表示されます。プロジェクト名、説明、ユースケースを入力するよう求められます。また、新しいアプリを作成するか、既存のスタンドアロンのアプリを接続するかを選択するよう求められます。

<div id="creating-or-connecting-an-app-for-your-project">
  #### プロジェクト用のアプリを作成または接続する
</div>

プロジェクトにアプリがまだない場合は、ダッシュボードでプロジェクト名をクリックしてアプリを追加できます。そこから、新しいアプリを作成するか、既存のスタンドアロンアプリを選択してプロジェクトに接続できます。アプリでは、このガイドの冒頭に記載されている認証用のキーとトークンを生成できます。

<div id="editing-a-project">
  #### プロジェクトの編集
</div>

プロジェクトを編集するには、dashboard または developer portal 内の Projects &amp; Apps ページで、編集したいプロジェクトの名前をクリックします。プロジェクトの詳細ページが表示されるので、変更を行うには「Edit」を選択します。

<div id="standalone-apps">
  ### スタンドアロンアプリ
</div>

スタンドアロンアプリは、プロジェクト構造の外部に存在するアプリです。これらのスタンドアロンアプリに紐づく認証情報を使用すると、X API v1.1（標準）、v1.1（プレミアム）、Enterprise、または X Ads API へのリクエストを正常に行うことができます。スタンドアロンアプリは、プロジェクトに接続しない限り、X API v2 エンドポイントへのリクエストを行おうとすると失敗します。

2020 年 8 月より前にアプリを作成している場合、それらは developer portal の [Projects &amp; Apps](https://developer.twitter.com/en/portal/projects-and-apps) 内にある「Standalone Apps」セクションに表示されます。プロジェクトに接続されているアプリを含め、アプリの合計数は最大 10 個に制限されます。

チームアカウントの一員である場合、「Standalone Apps」内に自分が所有しているアプリが表示されます。チームメイトがプロジェクトの一部であるアプリを所有している場合、そのアプリ名と所有者情報は確認できますが、その設定を変更したり、キーとトークンにアクセスしたり、キーとトークンを再生成したりすることはできません。アプリに変更を加える必要がある場合は、そのアプリの所有者に連絡してください。
