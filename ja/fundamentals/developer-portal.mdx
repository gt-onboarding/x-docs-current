  ## 概要
</div>

<div id="introduction">
  ### はじめに
</div>

X Developer Portal には、開発者が X API と X Ads API へのアクセスを管理するために利用できるセルフサービス型のツールが用意されています。

このポータルでは、次のことができます。

* X の[プロジェクト](/ja/resources/fundamentals/projects)と[アプリ](/ja/resources/fundamentals/developer-apps)（およびそれらが提供する認証用のキーとトークン）を作成および管理できます。
* X API Standard v1.1 および v2 のエンドポイントへのアクセスレベルと連携を管理できます。
* 利用可能なさまざまなエンドポイントや機能について詳しく学ぶことができます。

<div id="ready-to-get-access">
  ### アクセスする準備はできましたか？
</div>

アカウントを作成することで、X API の利用を開始できます。

追加機能やより高い[ポスト数上限](/ja/x-api/fundamentals/post-cap)が必要な場合は、開発者ポータル内で Basic または Pro のプランをご購入いただけます。Enterprise に関心がある方は、[こちら](https://docs.x.com/enterprise/forms/enterprise-api-interest)からお申し込みください。

  ## developer portal でできること
</div>

<div id="onboarding">
  ### オンボーディング
</div>

オンボーディングウィザードは、最初の[プロジェクト](/ja/resources/fundamentals/projects)と[アプリ](/ja/resources/fundamentals/developer-apps)を設定できるよう案内します。API リクエストを認証するために必要な認証情報を受け取るには、プロジェクトとアプリを作成する必要があります。開発者アカウントに初めてアクセスする場合、このウィザードが表示されます。

このプロセスを通じて、一連の認証用のキーとトークンが発行されます。詳しくは、[App overview](/ja/resources/fundamentals/developer-apps#overview) ページを参照してください。X API で認証するために必要な項目について詳しく知りたい場合は、[認証](/ja/resources/fundamentals/authentication)セクションを確認してください。

**注意:** 後から参照できるように、[キーとトークンは安全な場所に保管する](/ja/resources/fundamentals/authentication/guides/authentication-best-practices)必要があります。再生成しない限り、これらの認証情報を参照する方法はありません。

<div id="project-and-app-management">
  ### プロジェクトとアプリの管理
</div>

開発者アカウントの主な役割の 1 つは、[プロジェクトとアプリを管理](/ja/resources/fundamentals/developer-apps#app-management) できるようにすることです。開発者は、developer portal のダッシュボードから X のプロジェクトとアプリを作成および管理できます。ここでは、App ID を確認したり、アプリの設定、権限、コールバック URL を編集したり、キーとトークンを生成および失効させたりできます。

<div id="learn-more-about-whats-available-with-x-api-v2">
  ### X API v2 で利用できる機能の詳細を確認する
</div>

Developer Portal には、X API のさまざまなバージョンやアクセスレベルについて詳しく確認できる Products セクションがあります。

X API v2 の Products セクションには、Free、Basic、Pro、Enterprise の各アクセスティアに関する重要な情報が含まれています。このページには、プロジェクト単位のアプリの制限、ポスト数上限、コストの詳細に加え、エンドポイント固有のレート制限や特別な属性が記載されています。利用可能な場合は、異なるアクセスレベルを比較検討し、追加アクセスを申請することもできます。

[v2 のすべてのアクセスレベル](/ja/x-api/introduction#access-levels)を確認してください。

<div id="team-management">
  ## チーム管理
</div>

<Note>
  **Enterprise v2 アカウントでのみ利用可能です。**
</Note>

[**developer portal の「team」ページを表示します。**](https://developer.twitter.com/en/portal/teams)

<div id="why-use-team-functionality">
  ### チーム機能を使う理由
</div>

チーム機能を利用すると、X Developer Platform 上でプロジェクトやアプリをよりスムーズに共同開発できます。多くの場合、チーム内ではアクセス権限の管理や請求・支払いの担当者が分かれており、この機能を使うことで、そうしたメンバーをあなたのプロジェクトに招待して参加してもらうことができます。

<div id="inviting-team-members">
  ### チームメンバーを招待する
</div>

チームにメンバーを招待するには、管理者がそのユーザーの X ハンドルを使って招待できます。招待されたユーザーにはメールが送信され、そのメール内の招待から参加を承諾できます。承諾後、Developer Agreement &amp; Policy に同意する必要があり、その後メインアカウントのチームページにアクセスできるようになります。

**注意:** チーム管理機能では、現時点では [App](/ja/resources/fundamentals/developer-apps) の認証情報に基づいて API アクセスを付与または制限することはできません。チームメンバー間で App の管理を共有することはできません。所有者以外のユーザーは、App（キー／トークン）を編集、作成、または削除できません。

<div id="team-dashboard">
  ### チームダッシュボード
</div>

チームダッシュボードの「Members」タブでは、すべてのメンバーとその役割を確認できます。チームの管理者であれば、開発者のアクセス権を管理し、各メンバーの役割を編集できます。

管理者は、チームダッシュボードの「Pending」タブにもアクセスできます。ここでは、送信された各招待の詳細を確認し、管理することができます。

<div id="team-roles">
  ### チームのロール
</div>

**管理者ロール**

* チームのプロジェクトとアプリを管理する権限
* すべてのアプリ環境を管理する権限
* サブスクリプションを選択・アップグレードする権限
* 請求・支払い方法を更新する権限
* チームメンバーを追加・削除する権限
* チームメンバーのロールを編集する権限

**開発者ロール**

* 自分のプロジェクトとアプリを管理する権限
* チームのプロジェクトとアプリへの読み取り専用アクセス権
* チームから離脱する権限
