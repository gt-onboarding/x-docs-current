  ## 概要
</div>

<div id="introduction">
  ### はじめに
</div>

X developer portal には、開発者が X API や X Ads API へのアクセスを管理するために利用できるセルフサービス型のツールが用意されています。

developer portal では、次のことができます。

* X の [Projects](/ja/resources/fundamentals/projects) と [Apps](/ja/resources/fundamentals/developer-apps)（およびそれらが提供する認証用のキーとトークン）を作成・管理できます。
* X API standard v1.1 および v2 エンドポイントへのアクセスレベルや連携を管理できます。
* 利用可能なさまざまなエンドポイントや機能について詳しく学べます。

<div id="ready-to-get-access">
  ### アクセスする準備はできましたか？
</div>

アカウントを作成することで、X API の利用を開始できます。

追加の機能やより高い[ポスト上限](/ja/x-api/fundamentals/post-cap)が必要な場合は、developer portal 内で Basic または Pro を購入できます。Enterprise にご関心がある方は、[こちら](https://docs.x.com/enterprise/forms/enterprise-api-interest)からお申し込みください。

  ## developer portal でできること
</div>

<div id="onboarding">
  ### オンボーディング
</div>

オンボーディングウィザードは、最初の[プロジェクト](/ja/resources/fundamentals/projects)と[アプリ](/ja/resources/fundamentals/developer-apps)の設定プロセスを案内します。API リクエストを認証するために必要な認証情報を受け取るには、プロジェクトとアプリを作成する必要があります。オンボーディングウィザードは、開発者アカウントに初めてアクセスする場合に表示されます。

このプロセスを通じて、一連の認証用のキーとトークンが発行されます。詳しくは[アプリ概要ページ](/ja/resources/fundamentals/developer-apps#overview)をご覧ください。X API で認証するために必要な情報については、[認証セクション](/ja/resources/fundamentals/authentication)を参照してください。

**注意:** 後からアクセスできるように、[キーとトークンを安全な場所に保存](/ja/resources/fundamentals/authentication/guides/authentication-best-practices)しておく必要があります。これらの認証情報は、再生成しない限り再表示することはできません。

<div id="project-and-app-management">
  ### プロジェクトとアプリの管理
</div>

開発者アカウントの主な役割の1つは、[プロジェクトとアプリを管理](/ja/resources/fundamentals/developer-apps#app-management)できるようにすることです。開発者は、developer portal のダッシュボードから X プロジェクトとアプリを作成および管理できます。ここでは、App ID を確認したり、アプリの設定、権限、コールバック URL を編集したり、キーとトークンを生成および失効させたりできます。

<div id="learn-more-about-whats-available-with-x-api-v2">
  ### X API v2 で利用可能な機能について詳しく知る
</div>

developer portal には、X API のさまざまなバージョンとアクセスレベルについて詳しく確認できる products セクションがあります。

X API v2 に関する products セクションには、Free、Basic、Pro、Enterprise の各アクセスティアに関する重要な情報が含まれています。このページには、プロジェクトレベルでのアプリの制限事項、ポスト数上限、料金に加えて、エンドポイント固有のレート制限や固有の属性の詳細が記載されています。異なるアクセスレベルを比較検討したり、利用可能な場合は追加のアクセス権を申請したりすることもできます。

[v2 のすべてのアクセスレベル](/ja/x-api/introduction#access-levels)を確認してください。

<div id="team-management">
  ## チーム管理
</div>

<Note>
  **Enterprise v2 アカウントでのみ利用可能です。**
</Note>

[**developer portal の「チーム」ページを表示**](https://developer.twitter.com/en/portal/teams)

<div id="why-use-team-functionality">
  ### チーム機能を利用する理由
</div>

チーム機能を使うと、X Developer Platform 内でプロジェクトやアプリを共同で開発しやすくなります。多くの場合、チーム内ではアクセス管理や請求・支払いの担当者がそれぞれ異なりますが、この機能を使うことで、そうした担当者をプロジェクトに招待して共同作業できるようになります。

<div id="inviting-team-members">
  ### チームメンバーの招待
</div>

チームに誰かを招待するには、管理者がそのユーザーのXのハンドルを使って招待を送信します。招待されたユーザーにはメールが送信され、メールに記載された招待から参加を承諾できます。承諾後は、Developer Agreement &amp; Policy に同意する必要があり、その後メインアカウントのチームページにアクセスできるようになります。

**注意:** 現時点では、チーム管理によって [App](/ja/resources/fundamentals/developer-apps) の認証情報に基づいてAPIアクセス権を付与または制限することはできません。チーム内で App の管理を共有することはできません。オーナー以外のユーザーは、App（キー/トークン）を編集、作成、または削除することはできません。

<div id="team-dashboard">
  ### チームダッシュボード
</div>

チームダッシュボードの「Members」タブでは、すべてのメンバーとそのロールを確認できます。チームの管理者であれば、開発者アクセスを管理し、各メンバーのロールを編集できます。

管理者は、チームダッシュボードの「Pending」タブにもアクセスできます。ここでは、送信された各招待の詳細を確認し、管理できます。

<div id="team-roles">
  ### チームのロール
</div>

**管理者ロール**

* チームのプロジェクトとアプリを管理できる
* すべてのアプリ環境を管理できる
* サブスクリプションを選択／アップグレードできる
* 請求／支払い方法を更新できる
* チームメンバーを追加／削除できる
* チームメンバーのロールを編集できる

**開発者ロール**

* 自分のプロジェクトとアプリを管理できる
* チームのプロジェクトとアプリへの読み取り専用アクセス権がある
* チームから退出できる
