<div id="situation">
  ## 状況
</div>

これまでデジタルマーケターは、自身のデータを一元的に把握する手段を欠いてきました。あるシステムにはオーガニックデータがあり、別のシステムにはペイドデータがあり、会話はさらに別々の場所から発生している、といった状況です。しかしクライアントやステークホルダーから「昨日、どれくらいインプレッションを獲得しましたか？」と聞かれ、その次に「そのうちオーガニックとペイドの内訳は？」と問われるような場面では、このデータを集約して解釈するための効率的な方法の必要性は、これまでになく高まっています。

こうした問いに応えるため、X Developer Platform を活用して 10 年以上にわたり構築されてきたソーシャルアナリティクスプラットフォームである ListenFirst は、単一の信頼できる情報源からオーガニックおよびペイドの X データを収集するためのツールを作る必要がありました。これまで、このオーガニックデータとペイドデータは、異なるプラットフォーム、API、エンドポイントにまたがる別々のサイロに存在しており、集約したり、合算したり、あるいはブレンドしたりすることは、そもそも想定されていませんでした。

<div id="solution">
  ## 解決策
</div>

クライアント向けのインサイトをより効率的に導き出す方法を模索する中で、ListenFirst は X の v2 エンドポイントを採用し、X 上でユーザーが何を話しているかを自社プラットフォームに取り込むプロセスの効率化を図りました。ListenFirst のチームは、明確なゴールを掲げて作業に取り組みました。それは、よりクリーンなデータを得ることでした。

> *「私たちにとって v2 を活用することは、統合を最適化することです。自社サイト上で組み合わせるパーツが少なくなれば、それだけオペレーションが容易になり、より多くのブランド、より多くのコンテンツ、そしてより優れたインサイトへと提供範囲を拡大することができます。」*
> 
> - ListenFirst 最高プロダクト責任者（Chief Product Officer）、Jonathan Farb

ListenFirst のチームが X の v2 API、特に Tweets lookup エンドポイントの採用を優先した理由は、エンゲージメント数とインプレッション数の正確性でした。このエンドポイントを活用することで、ListenFirst は、他のネイティブな X アナリティクスで顧客が目にする数値と整合した、より信頼性の高いメトリクスを生成できるようになりました。

<div id="impact">
  ## インパクト
</div>

まだ導入初期の段階ではありますが、v2 の採用は ListenFirst の業務効率を高めています。これまで ListenFirst は複数の API セットからデータをつなぎ合わせる必要があり、その数値はネイティブのアナリティクスで確認できるものと一致していませんでした。v2 を利用することで、ListenFirst の数値は X のネイティブアナリティクスとより整合するようになり、精度と顧客からの信頼が向上しました。現在、ListenFirst は、ブランドが X 上でマーケティング戦略を完成させるのに役立つ、説得力があり洗練されたインテリジェンスを提供できるようになっています。ListenFirst は、v2 をさらに活用することで、精度と透明性が一層向上し、クライアント向けメトリクスへの信頼が高まると考えています。
