<div id="situation">
  ## 状況
</div>

これまでデジタルマーケターは、自社のデータを統合的に把握することができていませんでした。あるシステムにはオーガニックのデータがあり、別のシステムには有料のデータがあり、会話データはまた別の場所から発生するといった状況です。しかし、クライアントやステークホルダーから「昨日は何件のインプレッションを獲得しましたか？」に続いて「オーガニックと有料の内訳はどうなっていますか？」といった質問を受けるようになり、こうしたデータを集約して解釈するための効率的な方法の必要性は、これまでになく高まっています。

こうした課題を解決するため、ListenFirst（10年以上にわたり X Developer Platform 上で構築されてきたソーシャルアナリティクスプラットフォーム）は、単一の信頼できる情報源からオーガニックおよび有料の X データを収集できるツールを作る必要がありました。従来、このオーガニックと有料のデータは、さまざまなプラットフォーム、API、エンドポイントに分断された個別のサイロとして存在しており、集約したり、合計したり、あるいは組み合わせて利用したりすることは、そもそも想定されていませんでした。

<div id="solution">
  ## Solution
</div>

クライアント向けのインサイトをより効率的に導き出す方法を模索する中で、ListenFirst は X の v2 エンドポイントを採用し、X 上でユーザーが話している内容を自社プラットフォームに取り込むプロセスを効率化しました。ListenFirst のチームは、「よりクリーンなデータを作る」という明確なゴールを掲げて取り組みを開始しました。

> *「私たちにとって v2 を活用することは、インテグレーションの最適化のためです。自社サイト上で組み合わせなければならない要素が少なければ少ないほど、運用が容易になり、より多くのブランド、より多くのコンテンツ、そしてより優れたインサイトへと、私たちの提供を拡大していくことができます。」*
> 
> - Jonathan Farb, Chief Product Officer, ListenFirst

ListenFirst のチームが X の v2 API、特に Tweets lookup エンドポイントの採用を優先したのは、エンゲージメント数やインプレッション数の精度を重視したためです。このエンドポイントを活用することで、ListenFirst は、自社の顧客が他のネイティブな X アナリティクスで目にする数値と整合した、より信頼性の高いメトリクスを生成できるようになりました。

<div id="impact">
  ## インパクト
</div>

まだ初期段階ではありますが、v2 の導入により ListenFirst の効率性は向上しています。これまで ListenFirst は複数種類の API からデータをつなぎ合わせる必要があり、その数値はネイティブのアナリティクスで確認できるものと一致していませんでした。v2 を利用することで、ListenFirst の数値は X のネイティブアナリティクスにより近いものとなり、精度と顧客からの信頼が向上しました。現在、ListenFirst は、ブランドが X 上でマーケティング戦略を完成させるのに役立つ、根拠に基づいた洗練されたインテリジェンスを提供できるようになっています。ListenFirst は、v2 をさらに活用することで、精度と透明性を高め、クライアント向けメトリクスの信頼性が高まると考えています。
