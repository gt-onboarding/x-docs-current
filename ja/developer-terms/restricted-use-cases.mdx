当社の Developer Platform を利用するには、[Developer Agreement and Policy](/ja/developer-terms/agreement) および、それに関連するポリシー（[Display Requirements](https://developer.x.com/en/developer-terms/display-requirements.html) や [Automation Rules](https://help.twitter.com/en/rules-and-policies/twitter-automation) を含みます）を確認のうえ、同意していただく必要があります。これらの契約およびポリシーでは、ほかの事項とあわせて、利用が制限される複数のユースケースに関するガイドラインを示しています。以下では、これらの制限の一部について、追加情報を説明します。

<div id="sensitive-information">
  ## センシティブな情報
</div>

X データを用いて、X ユーザーについてセンシティブになり得る特性や属性を導き出したり推定して利用する際には十分注意してください。X ユーザーに関して、次のような情報を決して導出または推定したり、そのように導出または推定された情報を保存したりしてはいけません。

* 健康状態（妊娠を含む）
* 悪化した／困難な財務状況や経済状態
* 政治的な立場や信条
* 人種的または民族的な出自
* 宗教的または哲学的な立場や信条
* 性生活または性的指向
* 労働組合への所属
* 犯罪の実行（疑いを含む）に関する情報

個人データ（たとえば、ユーザー ID、ユーザー名、その他の識別子）を一切保存しない形での X コンテンツの集計分析は、その分析が適用される法律および Developer Agreement and Policy（開発者契約およびポリシー）のすべての条項に準拠している限り、許可されます。

<div id="off-x-matching">
  ## X外でのマッチング
</div>

X外でのマッチングとは、Xのコンテンツ（Xのユーザー名やユーザーIDを含む）を、人物、世帯、デバイス、ブラウザ、その他のX外の識別子と関連付けることを指します。たとえば、Xのユーザー名を企業の顧客レコードと関連付けるケースです（例：「John Doe」という顧客レコードが、X上の @johndoe と照合される場合）。

人々がX上で、実名とは別の、必要に応じて仮名のアイデンティティを安心して作成できるようにしたいと考えています。Xユーザーに関するいかなる情報であっても、それをX外の識別子と関連付けるつもりがある場合には、その関連付けを行う前に、ユーザーから明示的なオプトインの同意を取得することを求めます。たとえば、あなたのサービスへのサインアッププロセスの一環として、ユーザーが自らXのハンドルを直接あなたと共有した場合に、この同意を得ることができます。

ユーザーのX上のアイデンティティをX外の識別子にリンクすることについて、ユーザーから明示的なオプトインの同意を得ていない状況では、あなたが行うあらゆる紐付けは、その目的で利用されることをユーザーが合理的に想定し得る情報のみに基づいていなければなりません。ユーザーが、自分の提供した情報を使って自分のXアカウントがX外のアイデンティティとリンクされていると知ったときに驚くような利用であれば、そのような利用は行わないでください。さらに、本人からの明示的なオプトインの同意がない限り、あなたが保有する誰かに関する記録をX上のアイデンティティと照合しようと試みることが許されるのは、次の情報に基づく場合のみです。

* **ユーザーがあなたに直接提供した情報。** あなたとの事前の関係が一切ない個人に関する記録（第三者から取得した個人に関するデータを含む）は、この基準を満たしません。および／または
* **公開データ。** この文脈における「公開データ」とは、次のものを指します。
  * 公開されていて一般に利用可能なリソース（たとえば、ある業界団体の会員名簿など）からあなたが取得した、そのユーザーに関する情報
  * X上でそのユーザーについて公開されている情報。これには以下が含まれます。
    * ポスト
    * ユーザーの自己紹介文や公開されている所在地を含むプロフィール情報
    * 表示名およびユーザー名

<div id="redistribution-of-x-content">
  ## Xコンテンツの再配布
</div>

X API を通じて取得した X コンテンツを第三者と共有する必要がある場合、推奨される方法は、ポスト ID、ダイレクトメッセージ ID、および／またはユーザー ID を共有することです。コンテンツのエンドユーザーは、それらの ID を用いて X API を使い「リハイドレート」（すなわち、完全なポスト、ユーザー、またはダイレクトメッセージのコンテンツをリクエストして再取得）できます。これにより、X コンテンツのエンドユーザーが、常に最新の情報を当社から直接取得できるようになります。

当社は、非自動（手動）による手段での、リハイドレート済み X コンテンツの限定的な再配布を許可しています。この方法でリハイドレート済みの X コンテンツを第三者と共有する場合、受信者 1 名あたり 1 日につき、リハイドレート済みの公開 Post Object および／または User Object を最大 50,000 個までしか共有できません。また、このデータを公開の場で利用可能にしてはなりません（たとえば、ブログ記事への添付や、公開された GitHub リポジトリ内での提供など）。

X コンテンツを再配布する際に留意すべき点が、他にもいくつかあります。

* X から事前の明示的な書面による許可を受けていない限り、30 日間に単一の事業体に配布できるポスト ID の合計は 1,500,000 個までです。
* 非営利の研究目的のみにおいて学術機関を代表してポスト ID および／またはユーザー ID を再配布する個人は、ポスト ID および／またはユーザー ID を無制限に再配布することが許可されています。
* 上記のとおり X コンテンツを共有する許可をリクエストするには、API Policy サポートフォームをご利用ください。

X コンテンツを第三者に配布することが許可されている範囲においても、そのコンテンツには引き続き Developer Agreement および Developer Policy が適用されます。また、そのような第三者は、X コンテンツを受け取る前に、X の [Terms of Service](https://twitter.com/en/tos)、[Privacy Policy](https://twitter.com/en/privacy)、[Developer Agreement](/ja/developer-terms/agreement)、および [Developer Policy](/ja/developer-terms/policy) に同意しなければなりません。

<div id="multiple-applications">
  ## 複数のアプリケーション
</div>

単一のユースケース、または実質的に類似もしくは重複するユースケースに対して、複数のアプリケーションを登録することは許可されていません。これらのポリシーの詳細は[こちら](https://twittercommunity.com/t/policy-clarification-multiple-applications-for-the-same-use-case/92080)をご覧ください。

この文脈において、「ユースケース」とは、アプリケーションを通じて実行される一貫した分析、表示、またはアクションの集合を指します。同じサービスまたはアプリケーションを異なるエンドユーザーに提供する場合でも、それは 1 つのユースケースと見なされます。ただし、ツールやサービスの「ホワイトラベル」版は含まれません。アプリケーションの「ホワイトラベル」版を提供する予定がある場合は、当社に通知し、別途承認を受ける必要があります。これらのルールは、あなたが登録するアプリケーションと、あなたのツールやサービスのエンドユーザーが登録するアプリケーションの両方に適用されます。ツールやサービスを利用する目的でエンドユーザーにアプリケーションの登録を求めることにより、その結果として、あなた自身、あなたのアプリケーション、あなたの顧客、およびツールやサービスのエンドユーザーに対して、当社による措置が取られる可能性があります。

このルールの唯一の例外は、同じサービスの開発用（「dev」）、ステージング用、本番用（「prod」）インスタンスを作成することです。これらのアプリケーションは（たとえばアプリケーション名や説明などで）明確に区別してラベル付けし、本番目的で開発用またはステージング用アプリケーションを使用しないようにしてください。

<div id="automation-spam-and-auto-responses">
  ## 自動化、スパム、自動応答
</div>

スパムを生成したり、スパム的な行為を行う目的で X の API や開発者向けプロダクトを利用することは禁止されています。スパムに関する [X ルール](https://t.co/rules) を確認し、あなたのアプリケーションおよびその利用者が当社のポリシーに違反しないようにしてください。

あなたのアプリケーションが、ポストの投稿、アカウントのフォロー、ダイレクトメッセージの送信など、X サービス上で書き込み操作を行うために使用される場合は、サービスが当社のガイドラインに準拠していることを確認するために、[Automation Rules](https://support.twitter.com/articles/76915) を慎重に確認してください。特に、次の点を遵守してください。

* [自動返信や自動メッセージ](https://twittercommunity.com/t/policy-clarification-automated-replies-and-mentions/94444) を送信する前に、必ずユーザーから明示的な同意を得ること
* あなたから連絡を受けたくないというユーザーの要請には直ちに従うこと
* [フォローの一括実行](https://twittercommunity.com/t/policy-clarification-aggressive-following-and-inorganic-following-behavior/92769) を含む、大量・攻撃的・スパム的な行為を決して行わないこと
* [複数のアカウント](https://blog.twitter.com/developer/en_us/topics/tips/2018/automation-and-the-use-of-multiple-accounts.html) に対して、同一または実質的に類似したコンテンツを決して投稿しないこと

<div id="measuring-the-x-service">
  ## X サービスの測定
</div>

ベンチマークや競合分析を目的として X の可用性、パフォーマンス、機能、または利用状況を測定するために X API を使用してはなりません。たとえば、X API を使用して次のようなことを行ってはなりません。

* アクティブユーザー数やアカウント総数など、X ユーザーの集計メトリクスを算出すること
* 1 日あたりのポスト総数や、ユーザーエンゲージメント数またはアカウントエンゲージメント数など、X ポストの集計メトリクスを算出すること
* [X Rules](https://help.twitter.com/en/rules-and-policies/twitter-rules) で許可されている場合を除き、X 上のスパムやセキュリティを測定または分析すること

<div id="surveillance-privacy-and-user-protection">
  ## 監視、プライバシー、およびユーザー保護
</div>

X では、ユーザーのプライバシーを保護して守ることが当社の中核的な理念に組み込まれており、その姿勢は開発者向けプロダクトおよびデータプロダクトにも反映されています。X のデータは、世界において善のための強力な力になり得ると私たちは信じています。ジャカルタの洪水時の人命救助から、USGS による地震の追跡支援、さらには国連と協力して持続可能な開発目標の達成を目指す取り組みに至るまで、さまざまな場面で活用されています。しかし、私たちは、いかなる主体による監視目的での X データおよび X API の利用、あるいはユーザーの合理的なプライバシー上の期待と相反するその他一切の利用を禁止します。例外はありません。

当社のデータおよび開発者向けプロダクトの禁止されている利用方法については、[Developer Agreement](/ja/developer-terms/agreement) で説明しています。ここには、X ユーザーやそのコンテンツを調査・追跡する行為、およびセンシティブな出来事（抗議活動、集会、コミュニティの組織化ミーティングなど）を追跡、通知、監視する行為に対する禁止事項などが含まれます。

これらの条件の下で禁止される活動カテゴリーには、次のようなものが含まれます（これらに限定されるものではありません）。

* 組合やアクティビストグループなど、センシティブな団体や組織の調査や追跡
* 身元調査や、あらゆる形態の過度な審査（バックグラウンドチェックや厳格な適格性審査など）
* クレジットや保険リスクの分析
* 個人プロファイリングやサイコグラフィックセグメンテーション
* 顔認識

これらのポリシーは、当社 API のすべてのユーザーに適用されます。X API をこれらの目的で不正利用した場合は、アカウントの一時停止やアクセス権の解除を含む措置の対象となります。

X アカウントに関する情報を求める法執行機関向けの詳細情報については、[https://t.co/le](https://t.co/le) を参照してください。

さらに現時点では、X は [Grok](https://help.x.com/en/using-x/about-grok) を例外として、X API および／または X コンテンツを使用して基盤モデルまたはフロンティアモデルをファインチューニングまたはトレーニングする行為を一切禁止しています。
