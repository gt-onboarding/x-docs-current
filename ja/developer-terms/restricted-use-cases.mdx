当社のデベロッパープラットフォームを利用するには、[Developer Agreement and Policy](/ja/developer-terms/agreement) および、それに関連する [Display Requirements](https://developer.x.com/en/developer-terms/display-requirements.html) や [Automation Rules](https://help.twitter.com/en/rules-and-policies/twitter-automation) などの各種ポリシーを確認し、同意していただく必要があります。これらの契約およびポリシーでは、他の事項に加えて、いくつかの制限付きユースケースに関するガイダンスを提供しています。以下に、これらの制限の一部についての追加情報を記載します。

<div id="sensitive-information">
  ## センシティブな情報
</div>

X データを利用して X ユーザーについてセンシティブ（機微）な特性を導出・推測することには注意が必要です。次のような、X ユーザーに関する情報を決して導出または推測したり、そのように導出・推測された情報を保存してはいけません。

* 健康状態（妊娠を含む）
* 好ましくない財務状況や経済状態
* 政治的な所属または信条
* 人種的または民族的出自
* 宗教的または哲学的な所属または信条
* 性生活または性的指向
* 労働組合への加盟状況
* 犯罪の嫌疑または実際に犯罪を犯した事実

個人データ（たとえば、ユーザー ID、ユーザー名、その他の識別子）を一切保存しない形で行う X コンテンツの集計分析は、適用される法律および Developer Agreement and Policy のすべての条項を遵守している限り許可されます。

<div id="off-x-matching">
  ## X 外でのマッチング
</div>

X 外でのマッチングとは、X のコンテンツ（X のユーザー名やユーザー ID を含む）を、人物、世帯、デバイス、ブラウザ、その他の X 外の識別子と関連付けることを指します。1 つの例としては、X のユーザー名を事業者の顧客レコードと関連付ける場合が挙げられます（例：「あなたの顧客レコード内の “John Doe” が、X 上の @johndoe にマッチされる」など）。

人々には、X 上で別個の、そして希望すれば仮名のアイデンティティを作成できるという安心感を持ってほしいと考えています。X ユーザーに関するいかなる情報を X 外の識別子と関連付けようとする場合は、その関連付けを行う前に、ユーザーから明示的なオプトイン同意を取得する必要があります。たとえば、ユーザーがあなたのサービスへのサインアッププロセスの一環として、自らの X のハンドルをあなたに直接共有する場合には、その際に同意を得ることができます。

ユーザーの X 上のアイデンティティを X 外の識別子とリンクするための明示的なオプトイン同意を得ていない状況では、あなたが行うあらゆる関連付けは、ユーザーがその目的で利用されることを合理的に予期しうる情報のみに基づいて行わなければなりません。ユーザーが、自分が提供した情報を、X 上のアカウントと X 外のアイデンティティをリンクする目的であなたが利用していると知ったら驚くような場合には、そのような利用は行わないでください。さらに、個人からの明示的なオプトイン同意がない場合には、次の情報に基づく場合に限り、あなたが保有する誰かに関する記録を X 上のアイデンティティとマッチングしようと試みることができます。

* **ユーザーがあなたに直接提供した情報。** あなたとの間にこれまで何の関係もない個人に関する記録（第三者から取得した個人に関するデータを含む）は、この要件を満たしません。および／または
* **公開データ。** この文脈での「公開データ」とは、次のものを指します。
  * あなたが、公的に一般に利用可能なリソース（職業団体の会員名簿など）から取得した、ユーザーに関する情報
  * X 上において、ユーザーに関する情報のうち一般に公開されているもの。これには、次のものが含まれます。
    * ポスト
    * ユーザーの自己紹介文や公開された所在地を含むプロフィール情報
    * 表示名およびユーザー名

<div id="redistribution-of-x-content">
  ## Xコンテンツの再配布
</div>

X API を通じて取得した X コンテンツを第三者と共有する必要がある場合、最も適切な方法は、ポストID、ダイレクトメッセージID、および/またはユーザーIDを共有することです。コンテンツのエンドユーザーは、それらのIDを使用して X API によりリハイドレート（つまり、完全なポスト、ユーザー、またはダイレクトメッセージのコンテンツをリクエスト）できます。これにより、X コンテンツのエンドユーザーが常に最新の情報を直接当社から取得できるようになります。

当社は、自動化されていない手段によるリハイドレート済み X コンテンツの限定的な再配布を許可します。この方法でリハイドレート済み X コンテンツを第三者と共有する場合、受信者1人あたり1日につき、リハイドレート済みの公開ポストオブジェクトおよび/またはユーザーオブジェクトを合計で最大50,000件までしか共有できず、このデータを公開の場で利用可能にしてはなりません（たとえば、ブログ記事への添付や公開 GitHub リポジトリでの提供など）。

X コンテンツを再配布する際には、次の点にも留意してください。

* X から事前の明示的な書面による許可を得ていない限り、30日間に単一の事業体に配布できるポストIDの合計は1,500,000件までです。
* 非営利目的の研究のみを目的として学術機関を代表してポストIDおよび/またはユーザーIDを再配布する個人は、無制限の数のポストIDおよび/またはユーザーIDを再配布することが認められます。
* 上記のとおり X コンテンツを共有する許可をリクエストするには、API Policy サポートフォームをご利用ください。

X コンテンツを第三者に配布することが認められる範囲においても、そのコンテンツは引き続き Developer Agreement および Developer Policy の適用対象であり、その第三者は X コンテンツを受領する前に、X の[利用規約](https://twitter.com/en/tos)、[プライバシーポリシー](https://twitter.com/en/privacy)、[Developer Agreement](/ja/developer-terms/agreement)、および[Developer Policy](/ja/developer-terms/policy)に同意しなければなりません。

<div id="multiple-applications">
  ## 複数のアプリケーション
</div>

単一のユースケース、または実質的に類似・重複するユースケースについて、複数のアプリケーションを登録することは許可されていません。これらのポリシーの詳細については[こちら](https://twittercommunity.com/t/policy-clarification-multiple-applications-for-the-same-use-case/92080)をご覧ください。

本ポリシーにおける「ユースケース」とは、アプリケーションを通じて一貫して行われる分析・表示・アクションの組み合わせを指します。同じサービスまたはアプリケーションを異なるエンドユーザーに提供する場合は、1つのユースケースとして扱われます。ただし、ツールやサービスの「ホワイトラベル」版はこれに含まれません。アプリケーションの「ホワイトラベル」版を提供する予定がある場合は、当社に事前に通知し、個別の承認を受ける必要があります。これらのルールは、開発者が登録するアプリケーションと、開発者のツールやサービスのエンドユーザーが登録するアプリケーションの両方に適用されます。ツールやサービスを利用させる目的でエンドユーザーにアプリケーション登録を求める行為は、開発者本人、そのアプリケーション、顧客、および／またはツールやサービスのエンドユーザーに対する措置の対象となる可能性があります。

このルールの唯一の例外は、同一サービスに対して開発（「dev」）、ステージング、本番（「prod」）用のインスタンスを作成する場合です。これらのアプリケーションは（たとえばアプリケーション名や説明内で）明確に区別してラベル付けし、開発用またはステージング用のアプリケーションを本番目的で使用しないでください。

<div id="automation-spam-and-auto-responses">
  ## 自動化、スパム、および自動返信
</div>

スパムを作成したり、スパム的な行為に関与したりする目的で X の API および開発者向けプロダクトを使用することは禁止されています。スパムに関する [X Rules](https://t.co/rules) を確認し、アプリケーションおよびその利用者が当社のポリシーに違反しないようにしてください。

アプリケーションがポストの投稿、アカウントのフォロー、ダイレクトメッセージの送信など、X サービス上で書き込み操作を行うために使用される場合は、サービスが当社ガイドラインに準拠していることを確認するために、[Automation Rules](https://support.twitter.com/articles/76915) を慎重に確認してください。特に、次の点を遵守してください。

* [自動返信や自動メッセージ](https://twittercommunity.com/t/policy-clarification-automated-replies-and-mentions/94444) をユーザーに送信する前に、必ずそのユーザーから明示的な同意を得ること
* あなたからの連絡を望まないというユーザーからのリクエストには、直ちに従うこと
* [大量フォロー](https://twittercommunity.com/t/policy-clarification-aggressive-following-and-inorganic-following-behavior/92769) を含む、大量・攻撃的・スパム的なアクションを決して行わないこと
* [複数のアカウント](https://blog.twitter.com/developer/en_us/topics/tips/2018/automation-and-the-use-of-multiple-accounts.html) で、同一または実質的に同一のコンテンツを投稿しないこと

<div id="measuring-the-x-service">
  ## Xサービスの計測
</div>

ベンチマークや競合分析を目的として、X の可用性、パフォーマンス、機能、または利用状況を測定するために X API を使用してはなりません。たとえば、次のような目的で X API を使用してはなりません。

* アクティブユーザー数やアカウント総数など、X ユーザーに関する集計メトリクスを算出すること
* 1 日あたりのポスト総数や、ユーザーのエンゲージメント数またはアカウントのエンゲージメント数など、X 上のポストに関する集計メトリクスを算出すること
* [X Rules](https://help.twitter.com/en/rules-and-policies/twitter-rules) で認められる場合を除き、X 上のスパムやセキュリティを測定または分析すること

<div id="surveillance-privacy-and-user-protection">
  ## 監視、プライバシー、ユーザー保護
</div>

X では、ユーザーのプライバシーを保護し守ることが当社の根幹にある価値観に組み込まれており、その取り組みは開発者向け製品やデータ製品にも反映されています。X のデータは、世界に良い影響をもたらす強力な力になり得ると私たちは信じています――ジャカルタでの洪水時に人命を救うことから、USGS による地震の追跡を支援すること、国連と連携して持続可能な開発目標の達成に取り組むことまで、さまざまな場面で活用されています。一方で、いかなる組織であっても、監視目的で X データや X API を使用すること、あるいはユーザーの合理的なプライバシー期待に反するその他いかなる方法で使用することも禁止します。例外は一切認めません。

当社のデータおよび開発者向け製品の禁止される利用方法については、[Developer Agreement](/ja/developer-terms/agreement) で説明しており、そこには X ユーザーやそのコンテンツの調査や追跡の禁止、さらにデモや集会、コミュニティによる組織的な会合などといった機微なイベントの追跡、アラート、監視の禁止も含まれます。

これらの条件の下で禁止される行為のカテゴリーには、次のものなどが含まれます（ただし、これらに限定されるものではありません）。

* 労働組合やアクティビスト団体など、機微な集団や組織の調査または追跡
* 身元調査またはあらゆる形態の過度な審査・適格性評価
* クレジットまたは保険のリスク分析
* 個人プロファイリングまたは心理特性に基づくセグメンテーション
* 顔認識

これらのポリシーは、API のすべての利用者に適用されます。これらの目的で X API を不正に利用した場合は、利用停止やアクセス権の取り消しを含む是正措置の対象となります。

X アカウントに関する情報を求める法執行機関向けの追加情報については、[https://t.co/le](https://t.co/le) を参照してください。

さらに現時点では、X は [Grok](https://help.x.com/en/using-x/about-grok) を例外として、X API および/または X コンテンツを用いて、ファウンデーションモデルまたはフロンティアモデルをファインチューニングまたは学習させるあらゆる利用を禁止しています。
