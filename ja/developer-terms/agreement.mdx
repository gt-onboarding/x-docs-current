**I. 定義**

本契約において、以下の定義が適用されます。

1. 「**Developer Site**」とは、[**https://developer.x.com/**](https://developer.x.com/) にある X の開発者向けサイトを意味します。

2. 「**Direct Message**」とは、X のダイレクトメッセージ機能を使用して、1人のユーザーから1人以上の特定のユーザーに対して、X アプリケーション上で送信される非公開メッセージを意味します。

3. 「**Intellectual Property Rights**」とは、世界各国で登録済みか未登録かを問わず、すべての著作権、著作者人格権、特許権、商標権、およびその他一切の知的財産権またはこれに類する権利を意味します。

4. 「**Licensed Material**」とは、個別にまたは総称して、X API および X コンテンツを意味します。

5. 「**Paid Service(s)**」とは、本契約および該当する機能に従い、継続的な料金の支払いと引き換えにアクセスする Licensed Material の機能を意味します。本契約に別段の定めがない限り、「Licensed Material」へのすべての言及には Paid Service(s) が含まれるものとします。

6. 「**Payment Portal**」とは、Licensed Material の利用にオプトインし、Paid Services のサブスクリプションを管理し、Paid Services の支払方法を管理し、アカウント情報を更新するために、あなたに提供されるオンラインポータルを意味します。

7. 「**Services**」とは、X コンテンツを表示する、またはその他の方法で Licensed Material を使用する、あなたのサービス、ウェブサイト、アプリケーションおよびその他の提供物（リサーチを含みます）を意味します。

8. 「**Post**」とは、X アプリケーションを通じて配信される短文テキストおよびマルチメディア形式のメッセージを意味します。

9. 「**X**」とは、(a) あなたの主たる事業所が欧州連合、EFTA 加盟国および英国以外に所在する場合は X Corp.（865 FM 1209, Building 2, Bastrop, TX 78602, USA）を、または (b) あなたの主たる事業所が欧州連合、EFTA 加盟国または英国に所在する場合は X Internet Unlimited Company（One Cumberland Place, Fenian Street, Dublin 2, D02 AX07, Ireland）を意味します。

10. 「**X API**」とは、X アプリケーションプログラミングインターフェイス（それぞれ「**API**」）およびソフトウェア開発キット（それぞれ「**SDK**」）、ならびに Developer Site を通じて X により提供される関連するツール、ドキュメント、データ、テクノロジー、コードおよびその他の資料を意味します。

11. 「**X Applications**」とは、一般に「X」として知られる X のリアルタイム情報サービスを意味し、X が現在そのウェブサイトおよび X が所有、運営または管理するモバイルアプリケーション、ソーシャルプラグイン、アプリケーションプログラミングインターフェイスにおいて提供しているサービスを含みます。

12. 「**X Content**」とは、ポスト、各ポストごとに生成される固有の識別番号、X エンドユーザーのプロフィール情報、および X API を通じて、または X により承認されたその他の手段によりあなたに提供されるその他のデータおよび情報、ならびにそれらの複製物および派生物を意味します。

13. 「**X Marks**」とは、本契約に従った利用を目的として、Developer Site を含め、X があなたに提供する X の名称、商標およびロゴを意味します。

14. 「**Users**」とは、あなたの Services の訪問者、ユーザーまたは顧客を意味します。

  ### Xでの広告
</div>

**Xコンテンツの周辺で、どのように、またどこで広告を掲載できるかには制限があります。** まず、自社の広告がポストに類似したり、人々がポストと誤認するおそれがあってはなりません。その他の広告に関するルールは次のとおりです。

* **Xコンテンツと自社の広告の間には明確な区別がなければなりません。** X Ads 以外の広告を、Xのタイムライン内に配置することはできません。
* **Xは、X APIを通じて広告を配信する権利を留保します。** 当社がAPI経由でX Adsの配信を開始した後に、それらの配信を行うことを選択した場合、当社は適用される条件に従い、広告収益の一部を共有します。
* **XコンテンツやX APIから取得した情報を使用して、Xプラットフォーム外で人々を広告のターゲットとすることはできません。**

以下の追加ルールは、下記のXサービスまたは機能のいずれかを利用する場合のあらゆる利用に適用されます。

<div id="x-login">
  ### X ログイン
</div>

**OAuth プロトコルなどを利用して、X へのログインおよびログアウトの手段を、利用者が簡単に見つけられるように用意しなければなりません。** *Sign in with X* オプションは、あなたのサービス上の他のいかなるサインアップまたはサインイン機能と同等以上に目立つように表示する必要があります。X アカウントを持っていない人にも、X を通じてアカウントを作成できる機会を提供しなければなりません。

**あなたのサービス上で誰かが *Sign in with X* を通じて認証した場合、その人の X アイデンティティを明確に表示しなければなりません。** X アイデンティティには、その人の現在の X の @handle、アバター、および X ロゴが含まれます。あなたのサービス上で誰かの X のフォロワーを表示する場合、そのフォロー関係が X に関連付けられていることを明確に示さなければなりません。

<div id="x-cards">
  ### X Cards
</div>

**質の高い利用体験を確保するため、Card が表示されるすべてのプラットフォームで適切にレンダリングされるように Card を開発する必要があります。** Cards を使用する際に従わなければならない追加のルールには、次のものが含まれます。

* Card 内でそのようなメディアを表示する予定がある場合は、ポストに対してセンシティブメディアフラグを「true」に設定しなければなりません
* Card 内のすべてのアセットをホスティングする際には HTTPS を使用しなければなりません。Card によってアクティブな混在コンテンツに関するブラウザ警告が生成されてはなりません
* 音声および動画コンテンツには停止または一時停止コントロールを含め、自動再生される動画についてはデフォルトを「音声オフ」にする必要があります

**Card に対して X が課した制限（Card の想定される用途を含む）を超えたり、回避したりしてはなりません。** Cards の利用に関する追加の制限には、次のものが含まれます。

* X の承認なしに、第三者のスポンサー提供コンテンツを Cards 内に配置してはなりません
* Card 内や、Card から X 上に金銭的インセンティブ（仮想通貨を含む）を設定してはなりません
* URL やメディアなど、誤解を招く、または文脈的に関連性のないコンテンツやアクションを Card 内に含めてはなりません
* ポスト内で誰かがそのアプリを明示的に宣伝または参照している場合にのみ、ポストに App Card を添付することができます
