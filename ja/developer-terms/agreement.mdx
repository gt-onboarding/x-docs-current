4. “**Licensed Material**” とは、単独でも集合的にも、X API および X Content を意味します。

5. “**Paid Service(s)**” とは、本契約および該当する機能に応じた定期的な料金の支払いと引き換えに、Licensed Material の有償機能または機能の一部にアクセスすることを意味します。本契約に別途定めがない限り、「Licensed Material」へのすべての言及には Paid Service(s) が含まれるものとします。

8. “**Post**” とは、X Applications を通じて配信される短文テキストおよびマルチメディアベースのメッセージを意味します。

9. “**X**” とは、以下のいずれかを意味します。(a) あなたの主たる事業所が欧州連合、EFTA 諸国、および英国の外に所在する場合は X Corp.（865 FM 1209, Building 2, Bastrop, TX 78602, USA）、または (b) あなたの主たる事業所が欧州連合、EFTA 諸国、または英国に所在する場合は X Internet Unlimited Company（One Cumberland Place, Fenian Street, Dublin 2, D02 AX07, Ireland）。

10. “**X API**” とは、X の Application Programming Interfaces（それぞれを **“API”** といいます）、Software Development Kits（それぞれを **“SDK”** といいます）、および Developer Site を通じて X により提供される関連ツール、ドキュメント、データ、テクノロジー、コードその他の資料を意味します。

12. “**X Content**” とは、Posts、各 Post に対して生成される固有の識別番号、X エンドユーザーのプロフィール情報、および X API を通じて、または X により認可されたその他の手段によりあなたに提供されるその他一切のデータおよび情報、ならびにそれらの複製物および派生物を意味します。

**Xコンテンツの周囲で、どのように、どこで広告を掲載できるかには制限があります。** まず、お客様の広告はポストと同じように見えたり、人々がポストと合理的に誤認・混同したりしてはなりません。広告に関するその他のルールは次のとおりです。

* **Xコンテンツとお客様の広告との間には、明確な区別がなければなりません。** X Ads を除き、Xのタイムライン内にいかなる広告も掲載してはなりません。
* **Xは、X API を通じて広告を配信する権利を留保します。** 当社がAPI経由での広告配信を開始した後に、お客様がX Ads の配信を行うことを選択した場合、当社は関連する契約条件に従い、広告収益の一部をお客様と共有します。
* **Xコンテンツや、X API から取得した情報を利用して、Xプラットフォーム外で人々に広告をターゲティングしてはなりません。**

以下に挙げるXのサービスまたは機能を利用する場合には、次の追加ルールが適用されます。

<div id="x-login">
  ### X ログイン
</div>

**自社サービス上で、OAuth プロトコルなどを利用して、ユーザーが X にログインおよびログアウトできるオプションを、容易に見つけられる場所に用意しなければなりません。** *Sign in with X* オプションは、自社サービス上の他のいかなるサインアップまたはサインイン機能と比べても、同等以上に目立つ形で表示する必要があります。また、X アカウントを持っていないユーザーに対しては、X 経由でアカウントを作成できる機会を提供しなければなりません。

**自社サービス上でユーザーが *Sign in with X* を通じて認証した場合、そのユーザーの X 上でのアイデンティティを明確に表示しなければなりません。** X 上でのアイデンティティには、そのユーザーの現在の X の @handle、アバター、および X ロゴが含まれます。自社サービス上でそのユーザーの X 上のフォロワーを表示する場合は、そのフォロー関係が X に関連付けられていることを明確に示さなければなりません。

<div id="x-cards">
  ### X Cards
</div>

**質の高いユーザー体験を確保するため、Card が表示されるすべてのプラットフォームで正しくレンダリングされるように Card を開発しなければなりません。** Cards を使用する際に従う必要がある追加のルールは次のとおりです。

* Card 内でそのようなメディアを表示する予定がある場合は、ポストをセンシティブなメディアとして指定するために「true」とマークしなければなりません
* Card 内のすべてのアセットをホスティングする際には HTTPS を使用しなければなりません。Card によってアクティブな混在コンテンツに関するブラウザの警告が生成されることがあってはなりません
* 音声および動画コンテンツには停止または一時停止のコントロールを含める必要があり、自動再生される動画についてはデフォルトで「ミュート（音声オフ）」に設定しなければなりません

**Card に課された X の制限（Card の想定される使用方法を含む）を超えたり、それを回避したりしてはなりません。** Cards の使用に対する追加の制限は次のとおりです。

* X の承認を得ることなく、サードパーティによるスポンサー提供コンテンツを Cards 内に配置してはなりません
* Card 内、または Card から X 上に金銭的インセンティブ（仮想通貨を含む）を設定してはなりません
* URL やメディアなど、誤解を招く、またはコンテキスト上の関連性のないコンテンツやアクションを Card 内に含めてはなりません
* ポスト内でそのアプリが明示的に宣伝または言及されている場合にのみ、App Card をポストに添付することができます
