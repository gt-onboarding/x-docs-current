<div id="x-developers">
  ## X + 開発者
</div>

X は開発者を大切にしています。私たちは、X データの力を活用してこのコミュニティが生み出すツールやサービスに、常に喜びと驚きを感じています。このコミュニティへのコミットメントの一環として、開発者に対してはオープンかつ公平なかたちでデータへのアクセスを提供し、X を利用する人々にとって安全であり、X プラットフォーム全体にとって有益であることを目指しています。これらの目標をさらに推進するために、API および X コンテンツの適切な利用に関するルールと期待を理解してもらうためのガイドとして、Developer Policy を策定しました。

この Developer Policy（「**本ポリシー**」）は、X のアプリケーション、サービス、ウェブサイト、ウェブページおよびコンテンツから成るエコシステムとやり取りを行う開発者向けのルールおよびガイドラインを定めるものです。本ポリシーは、X API および X コンテンツへのアクセスおよびその利用を規定する、X とあなたとの契約（Developer Agreement または X とのその他の書面による契約の一部）を構成します。本ポリシーへの違反は、あなたの契約への違反とみなされます。本ポリシーは予告なく随時変更されることがあります。本ポリシーで使用されるが本ポリシー内で定義されていない用語のうち、頭文字が大文字で記載されたものは、[Developer Agreement](/ja/developer-terms/agreement) または Master License Agreement に定められた意味を有するものとします。

これら 3 つの重要なセクションそれぞれが何を意味するのかについて、以下でさらに詳しく説明しています。X API と X コンテンツの利用が当社のポリシーに沿ったものとなるよう、よく確認してください。

最後に、X は、X アプリケーションの改善、商用利用の検証、および承認済みユースケースと本ポリシーの順守状況を確認する目的で、あなたによる X API の利用状況を監視する場合があることに留意してください。

<div id="privacy-and-control-are-essential">
  ## プライバシーとコントロールは不可欠です
</div>

X はプライバシーを重視しており、X コンテンツおよび X API を利用するすべての方にも同様に配慮いただくことを期待しています。X 開発者プラットフォーム、X API、または X コンテンツを、人々が合理的に期待するプライバシーと一致しない方法で利用した場合、API および X コンテンツへのアクセスの一時停止や終了を含む、執行措置の対象となる可能性があります。

プライバシーとコントロールへの取り組みは、X コンテンツのあらゆる利用形態と、当社の API を使って構築するサービスのあらゆる側面にまで及ぶ必要があります。そのためには、あなたのサービスを利用する人々が、自分のデータがどのように利用されるのか、また、あなたが彼らに代わってどのように X にアクセスするのかを理解し、同意していることが必要です。これは、人々に対して明確で包括的かつ透明性のあるプライバシーポリシーを提示し、さらに、あなたのサービスを利用する各人から、その人に代わっていかなる行為を行う前に、明示的かつ十分な情報に基づく同意を得ることで実現できます。**なお、利用者があなたのサービスで認証を行うこと自体は、それだけで同意を構成するものではない点にご注意ください。**

<div id="consent-permissions">
  ## 同意と許可
</div>

**特に、以下のいずれかを行う場合は、必ず事前に本人から明示的かつ十分な説明に基づく同意を得なければなりません。**

* 本人に代わって *いかなる* 行為を行うこと。これには次のような行為（これらに限定されません）が含まれます。
  * X へのコンテンツのポスト
  * アカウントのフォロー／フォロー解除
  * プロフィールまたはアカウント情報の変更
  * ハッシュタグやその他のコンテンツをポストに追加すること

* X API またはその他の X のツール以外の手段でアクセスしたコンテンツの再公開

* 他人の X コンテンツを使用して製品やサービスを宣伝すること

* ダイレクトメッセージ（DM）などの非公開コンテンツ、その他の私的または機密情報を保存すること

* 保護されたコンテンツ、その他の私的または機密情報を共有または公開すること

**あなたのサービスがユーザーに X へコンテンツをポストできるようにしている場合、公開前に以下を行わなければなりません。**

* 何が公開されるのかを正確に表示すること
* コンテンツにどのような位置情報（ある場合）が追加されるかを、あなたのサービスの利用者に明確に示すこと

**あなたのサービスが、ユーザーにあなたのサービスと X の両方へコンテンツをポストできるようにしている場合、公開前に以下を行わなければなりません。**

* コンテンツをポストする許可を得ること
* コンテンツをどこにポストするのかを説明すること

**すべての X コンテンツにおける「保護」と「ブロック」のステータスを尊重しなければなりません。1 人の認証トークンを使って取得したコンテンツを、そのコンテンツを閲覧する権限を持たない別の人に提供してはなりません。**

* **保護アカウント:** [保護アカウント](https://help.x.com/safety-and-security/public-and-protected-posts)のコンテンツは、そのアカウントをフォローすることをアカウント所有者から承認された人のみが利用できます。そのため、保護アカウントにアクセスするサービスを運用する場合、そのコンテンツを閲覧する権限を持つ特定の人々にそのコンテンツを提供する目的に限ってアクセスすることができます。
* **ブロックされたアカウント:** X 上のユーザーは、自分の選んだあらゆる理由により、アカウントへのアクセスを[ブロック](https://help.x.com/using-x/blocking-and-unblocking-accounts)することができます。トークン（またはその他の API ベースの操作）から取得した情報を組み合わせて、この選択を回避することは認められません。

**ダイレクトメッセージ（DM）は性質上非公開であるため、DM 機能を提供するサービスは、個人のプライバシーを保護するために追加の措置を講じる必要があります。DM コンテンツを、そのコンテンツを閲覧する権限を持たない人に提供してはなりません。あなたのサービスが DM 機能を提供する場合、次のこともしなければなりません。**

* DM の既読通知イベントを送信する場合は、その旨をユーザーに通知すること。これは、サービス内で直接通知を表示するか、会話の他の参加者からの既読通知を表示することで行うことができます。
* DM で送信されるメディアを「共有」（すなわち複数の DM 間で再利用可能）として設定する前に、同意を得ること。DM 内のメディアを「共有」可能にする場合、そのメディアの URL を知っている人であれば誰でもこのコンテンツにアクセスできることを、明確な通知によって示さなければなりません。

<div id="content-compliance">
  ## コンテンツのコンプライアンス
</div>

**X コンテンツをオフラインで保存する場合、そのコンテンツの X 上での最新の状態に常に保っておく必要があります。** 具体的には、X 上で削除または変更されたコンテンツについては、あなたが保持しているコンテンツも削除または変更しなければなりません。これは、合理的に可能な限り速やかに、または X もしくは該当する X アカウントの所有者からその旨の要請を受けてから 24 時間以内、あるいは X との契約または適用法令により定められる期間内に行う必要があります。法令によりこれが禁止されている場合を除き、これを行わなければならず、そのような場合であっても、X による明示的な書面での許可がある場合にのみ例外とすることができます。

変更されるコンテンツの形態はさまざまです。これには（これらに限定されませんが）次のものが含まれます。

* 非公開にされた、または保護されたステータスを取得したコンテンツ
* プラットフォーム上で凍結されたコンテンツ
* ジオタグが削除されたコンテンツ
* X から保留または削除されたコンテンツ

<div id="off-x-matching">
  ## X 外での照合
</div>

**X 上の人物を、X 外で取得または保存した情報と照合できる状況は厳しく制限されています。** X 外での照合とは、X コンテンツ（X の @handle やユーザー ID を含みます）を、人物、世帯、デバイス、ブラウザ、その他の X 外の識別子と関連付ける行為を指します。このような照合は、その関連付けを行う前に当該人物から明示的なオプトイン同意を得ている場合、または以下に記載する場合にのみ行うことができます。

X のアイデンティティを X 外の識別子にリンクするための、その人物の明示的なオプトイン同意がない状況では、あなたが行ういかなる結び付けも、その目的で利用されることを合理的に予期できる情報のみに基づく必要があります。さらに、人物の明示的なオプトイン同意がない場合、あなたが自ら保有するある人物に関する記録を X のアイデンティティと照合しようとすることが許されるのは、次の情報に基づく場合に限られます。

* **当該人物からあなたに直接提供された情報。** あなたとの間に事前の関係がない個人に関する記録（第三者から取得した個人データなど）は、この基準を満たさない点に注意してください。および／または
* **公開データ。** この文脈における「公開データ」とは、次を指します。
  * 公開された一般に利用可能な情報源（例：職業団体の会員名簿）から取得した、人物に関する情報
  * X 上で一般に公開されている、その人物に関する情報。具体的には：
    * ポスト
    * アカウントの自己紹介文や公開されている所在地を含むプロフィール情報
    * 表示名および @handle

<div id="using-geo-data">
  ## ジオデータの利用
</div>

**ジオデータは、その情報の機微な性質により、追加の制限が適用されます。** サービスがポストに位置情報を付加する場合は、利用者に対して次の点を開示しなければなりません。

* いつ位置情報を付加するのか
* 位置情報をジオタグとして追加しているのか、アノテーション データとして追加しているのか
* 位置情報が場所として表示されているのか、地理座標として表示されているのか

アプリケーションが、利用者に位置情報付きでポストさせることを可能にしている場合、X の [Geo ガイドライン](/ja/developer-terms/geo-guidelines)を完全に順守する必要があります。

**位置データまたは地理情報を単独で利用することは禁止されています。** ポストの一部としての場合を除き、X コンテンツに含まれる位置データやその他の地理情報を保存、集約、またはキャッシュすることはできません（また、第三者に許可することもできません）。例えば、時間の経過に伴い個人がどこにいたかを示すために、ポストから位置データや地理情報だけを切り離して利用することはできません。ヒートマップや、集約されたジオアクティビティ（例：ある都市でハッシュタグを使用している人数）を表示する関連ツールは許可されています。
