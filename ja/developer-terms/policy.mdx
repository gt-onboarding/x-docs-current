<div id="x-developers">
  ## X + 開発者
</div>

X は開発者を大切にしています。私たちは、このコミュニティが X のデータの力を活用して生み出すツールやサービスに、常に喜びと驚きを感じています。このコミュニティへの取り組みの一環として、開発者に対してはオープンかつ公正であり、X を利用する人々にとって安全であり、かつ X プラットフォーム全体にとって有益となるようなデータアクセスを提供することを目指しています。これらの目標を推進するために、私たちは Developer Policy を策定し、API および X コンテンツの適切な利用に関するルールや期待を理解するためのガイドとして提供しています。

この Developer Policy（「**本ポリシー**」）は、X のアプリケーション、サービス、ウェブサイト、ウェブページおよびコンテンツから成るエコシステムを利用し、これとやり取りを行う開発者向けのルールとガイドラインを定めるものです。本ポリシーは、X API および X コンテンツへのアクセスおよびその利用を規定する、X との契約（Developer Agreement または X とのその他の書面による契約）の一部を構成しています。本ポリシーへの違反は、当該契約への違反とみなされます。本ポリシーは、予告なく随時変更されることがあります。本ポリシーで使用される用語のうち、本ポリシーで定義されていない大文字表記の用語は、[Developer Agreement](/ja/developer-terms/agreement) または Master License Agreement において定義されたそれぞれの意味を有するものとします。

</div>

このポリシーは、できるだけわかりやすく順守しやすい構成にしています。X API と X コンテンツを利用する際は、以下のポリシー各セクションの内容を常に意識してください。

1. **成功のための準備を整える** - すべての X ポリシーを順守する責任はあなたにあります。X API と X コンテンツにアクセスする前に、本ポリシーおよび本書内で参照しているポリシーを確認し、理解することが重要です。事前にポリシーの確認に時間をかけることで、後々の大きなやり直し作業を防ぐことができます。

3. **プラットフォーム利用ガイドラインに従う** - X API と X コンテンツへのアクセスが承認されることは、あくまで最初のステップにすぎません。X プラットフォームの利用方法について、計画している利用がポリシー順守に適合しているか疑問が生じた場合には、まずプラットフォーム利用ガイドラインを確認してください。

これら 3 つの重要なセクションそれぞれの趣旨について、以下でさらに詳しく説明しています。X API と X コンテンツの利用が当社のポリシーに沿ったものとなるよう、必ず注意深く確認してください。

あなたが本ポリシー（またはその他の X ポリシー）に違反していると当社が判断した場合、X API と X コンテンツへのあなたのアクセスを一時的に停止、または恒久的に取り消すことがあります。

最後に、X は X アプリケーションを改善し、商用利用の有無を確認し、承認されたユースケースおよび本ポリシーへの順守を確保する目的で、あなたによる X API の利用状況を監視する場合があることにご留意ください。

お読みいただきありがとうございます。また、X とともに構築していただき感謝します。あなたが何を生み出すのか、楽しみにしています。

  ## プライバシーとコントロールは不可欠です
</div>

X はプライバシーを重視しており、X コンテンツおよび X API を利用するすべての方にも同様の配慮を求めます。X デベロッパープラットフォーム、X API、または X コンテンツの利用が、人々が合理的に期待しうるプライバシーと一致しない形で行われた場合、API や X コンテンツへのアクセスの一時停止や打ち切りを含む、措置の対象となることがあります。

プライバシーとコントロールに対するあなたの責任は、X コンテンツのあらゆる利用と、当社の API を用いて構築するサービスのあらゆる側面にまで及ぶ必要があります。そのため、あなたのサービスの利用者は、自身のデータがどのように利用され、どのように自分に代わって X へアクセスされるのかを理解し、そのうえで同意していなければなりません。これは、利用者に対して明確で包括的かつ透明性の高いプライバシーポリシーを提示し、さらに、利用者に代わっていかなる行為を行う前にも、あなたのサービスを利用する各人から明示的かつ十分な説明に基づく同意を得ることで実現できます。**なお、利用者があなたのサービスに認証してログインすること自体は、それだけで同意を意味するものではないことに注意してください。**

<div id="consent-permissions">
  ## 同意と許可
</div>

**特に、以下のいずれかを行う前には、必ず当該の人から明示的かつ十分な説明に基づく同意を得なければなりません。**

* その人の代理で *いかなる* アクションを行うこと。これには次の内容が含まれます（これらに限りません）：
  * X にコンテンツをポストすること
  * アカウントをフォロー／フォロー解除すること
  * プロフィールまたはアカウント情報を変更すること
  * ハッシュタグやその他のコンテンツをポストに追加すること

* X API またはその他の X ツール以外の手段によってアクセスしたコンテンツを再公開すること

* 他者の X コンテンツを製品やサービスの宣伝に利用すること

* ダイレクトメッセージ（DM）などの非公開コンテンツ、その他の私的または機密情報を保存すること

* 保護されたコンテンツ、その他の私的または機密情報を共有または公開すること

**あなたのサービスが、人々に X へコンテンツをポストすることを許可する場合、公開前に次のことを行う必要があります。**

* 何が公開されるかを正確に表示すること
* あなたのサービスを利用する人々に対し、どの位置情報（ある場合）がコンテンツに追加されるかを明確に示すこと

**あなたのサービスが、人々にあなたのサービスおよび X の両方へコンテンツをポストすることを許可する場合、公開前に次のことを行う必要があります。**

* コンテンツをポストする許可を得ること
* どこにコンテンツをポストするかを説明すること

**すべての X コンテンツについて、その保護設定およびブロック設定の状態を尊重しなければなりません。ある人の認証トークンを使用して取得したコンテンツを、そのコンテンツを見る権限のない別の人に配信することはできません。**

* **保護されたアカウント：** [保護されたアカウント](https://help.x.com/safety-and-security/public-and-protected-posts) のコンテンツは、そのアカウントをフォローすることを所有者から承認された人々にのみ利用可能です。したがって、あなたが保護されたアカウントにアクセスするサービスを運営している場合、そのコンテンツを見る権限がある特定の人々に対してのみ、そのようなコンテンツを提供する目的でアクセスすることができます。
* **ブロックされたアカウント：** X 上の人々は、任意の理由で自分のアカウントへのアクセスを[ブロック](https://help.x.com/using-x/blocking-and-unblocking-accounts)することができます。トークン（またはその他の API ベースのアクション）から得られた情報を組み合わせ、この選択を回避しようとすることは許可されません。

**ダイレクトメッセージ（DM）は非公開の性質を持つため、DM 機能を提供するサービスは、個人のプライバシーを保護するために追加の措置を講じなければなりません。DM コンテンツを、そのコンテンツを見る権限のない人に配信することはできません。あなたのサービスが DM 機能を提供する場合、さらに以下も行う必要があります。**

* DM に対して既読通知イベントを送信する場合は、その旨を人々に通知すること。これは、サービス内で直接通知を提供するか、会話内の他の参加者からの既読通知を表示することで行うことができます。
* DM で送信されるメディアを「共有」（すなわち、複数の DM 間で再利用可能）として設定する前に、同意を得ること。DM 内のメディアの「共有」を許可する場合は、そのメディアの URL を持つ誰もがそのコンテンツにアクセスできることを、明確な通知によって示さなければなりません。

<div id="content-compliance">
  ## コンテンツのコンプライアンス
</div>

**X コンテンツをオフラインで保存する場合、そのコンテンツの X 上での最新の状態に合わせて更新しなければなりません。** 具体的には、X 上で削除または変更されたコンテンツについては、あなたが保有しているコンテンツも削除または変更する必要があります。これは、合理的に可能な限り速やかに、または X または該当する X アカウントの所有者からその旨の要請を受けてから 24 時間以内、もしくは X との契約または適用法令で別途定められている期限までに行わなければなりません。この義務は、適用法令によって当該削除または変更が禁止されている場合を除き適用され、そのような場合でも、X の書面による明示の許可がある場合にのみ、削除または変更を行わないことができます。

変更されたコンテンツにはさまざまな形態があります。これには（これらに限定されませんが）次のものが含まれます。

* 非公開にされた、または保護ステータスになったコンテンツ
* プラットフォーム上で利用停止となったコンテンツ
* ジオタグが削除されたコンテンツ
* X 上で保留（表示制限）または削除されたコンテンツ

<div id="off-x-matching">
  ## X外でのマッチング
</div>

**当社は、X上の人物をX外で取得または保存された情報と照合できる状況を制限しています。** X外でのマッチングとは、Xコンテンツ（Xの@handleやユーザーIDを含む）を、人物、世帯、デバイス、ブラウザ、その他のX外の識別子と関連付けることを指します。こうした関連付けは、その人物から事前に明示的なオプトイン同意を得ている場合、または以下に記載する場合にのみ行うことができます。

人物のX上のアイデンティティをX外の識別子にリンクするための、その人物からの明示的なオプトイン同意がない状況では、あなたが行うあらゆる関連付けは、その目的で使用されることを合理的に予期し得る情報のみに基づいて行う必要があります。さらに、その人物の明示的なオプトイン同意がない場合、あなたが誰かに関して保有する記録をX上のアイデンティティと照合しようと試みることができるのは、次の情報に基づく場合に限られます。

* **その人物からあなたに直接提供された情報。** なお、あなたが事前に関係を有していない個人に関する記録（第三者から取得した個人に関するデータを含む）は、この基準を満たしません。および／または
* **公開データ。** この文脈でいう「公開データ」とは、次を指します。
  * あなたが公開され一般に入手可能な情報源（たとえば、職業団体の会員名簿など）から取得した、その人物に関する情報
  * 人物についてX上で公開されている情報。これには次が含まれます。
    * ポスト
    * アカウントの自己紹介や公開されている所在地を含む、プロフィール情報
    * 表示名および@handle

<div id="using-geo-data">
  ## ジオデータの利用
</div>

**ジオデータは、その情報の性質上センシティブであるため、利用には追加の制限があります。** サービスでポストに位置情報を付加する場合、ユーザーに対して次の点を開示しなければなりません。

* いつ位置情報を追加するのか
* 位置情報をジオタグとして追加するのか、アノテーションデータとして追加するのか
* その位置情報が「場所」として表示されるのか、地理座標として表示されるのか

アプリケーションで、ユーザーが位置情報付きでポストを行えるようにしている場合は、X の[ジオガイドライン](/ja/developer-terms/geo-guidelines)を完全に順守しなければなりません。

**位置データや地理情報を、それ単独で利用することは禁止されています。** あなたは（また、他者に対しても許可してはならず）、ポストの一部としての場合を除き、X コンテンツに含まれる位置データやその他の地理情報を保存、集約、またはキャッシュしてはなりません。たとえば、個人が時間の経過とともにどこにいたかを示すために、ポストから位置データや地理情報だけを切り離してはなりません。ヒートマップや、ハッシュタグを利用している都市内の人数など、集約されたジオアクティビティを表示する関連ツールは許可されています。
