このモジュールは、X API の activity エンドポイントを操作するためのクライアントを提供します。
リアルタイムストリーミングの処理では、到着したデータを順次返すジェネレーターが返されます。
ストリーミング接続は、指数バックオフを用いた自動再試行ロジックによって管理され、堅牢に動作します。

<div id="stream">
  ### `stream`
</div>

Activity Stream（ストリーミング）
X アクティビティのストリーム
これは、利用可能になり次第リアルタイムでデータを返すストリーミングエンドポイントです。
返される各項目は、ストリームからの 1 つのデータポイントを表します。
接続は、指数バックオフを用いたリトライロジックによって自動的に管理されます。
ストリームが切断された場合でも、SDK はクライアント側での対応なしに自動的に再接続します。

<div id="parameters">
  #### パラメーター
</div>

<ParamField path="path.backfill_minutes" type="int or None" default="None">
バックフィルを要求する時間（分単位）。
</ParamField>

<ParamField path="path.start_time" type="str or None" default="None">
YYYY-MM-DDTHH:mm:ssZ。ポストに対するラベルが提供される最も早い UTC タイムスタンプ。
</ParamField>

<ParamField path="path.end_time" type="str or None" default="None">
YYYY-MM-DDTHH:mm:ssZ。ポストに対するラベルが提供される最も遅い UTC タイムスタンプ。
</ParamField>

<ParamField path="path.stream_config" type="StreamConfig">
再試行動作、タイムアウト、およびコールバックをカスタマイズするためのオプションの StreamConfig。
</ParamField>
