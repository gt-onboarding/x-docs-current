Ads API で成果を上げるためには、X Ads における各エンティティ間の関係を理解することが重要です。このチュートリアルでは、X Ads のオブジェクト階層構造の基本について説明します。

![X Ads API の階層構造](/images/adsapi-heirarchy.png.twimg.1920.png)

<div id="ad-group-line-items">
  ## 広告グループ（ラインアイテム）
</div>

広告グループは Ads API では[ラインアイテム](https://developer.x.com/content/developer-twitter/en/docs/tutorials/ads-api-hierarchy-terminology#line-items)と呼ばれ、キャンペーンに属し、特定の X ユーザー群を対象としたターゲティングおよび入札に使用されます。広告主は、ポストやメディア（例：インストリーム広告としてプロモーションされる動画）をラインアイテムに関連付けることでプロモーションします。「Ad Group」も参照してください。

<div id="account-media">
  ## アカウントメディア
</div>

[アカウントメディア](/ja/x-ads-api/creatives#account-media)として広告アカウントにアップロードされたクリエイティブをまとめたコレクションです。プロモーションに利用できる動画の[プレロールアセット](/ja/x-ads-api/campaign-management#video-views-preroll-objective)（インストリーム広告とも呼ばれます）や、[X Audience Platform](https://developer.x.com/content/developer-twitter/en/docs/ads/measurement/overview/twitter-audience-platform)上でプロモーション可能な画像などを含めることができます。

<div id="ads-accounts">
  ## 広告アカウント
</div>

X Ads の階層の最上位にあるのが広告アカウントです。1 つの広告アカウントは複数の X ユーザーによって管理できます。デモ用の家具店アカウントである @Furni は、広告アカウントを管理するユーザーの一例です。広告アカウントは、[base36](https://en.wikipedia.org/wiki/Base36) 形式の英数字文字列である account&#95;id で識別します。

各アカウントには、キャンペーン間で再利用できる複数のエンティティタイプが格納されます。たとえば、ターゲティングに使用できるテーラードオーディエンス、動画や画像で構成されるアカウントメディア、ポストに追加できる拡張クリエイティブのカードなどです。

<div id="amplify-video">
  ## Amplify Video
</div>

動画アセットをアップロードする際、アップロードする側は、その動画のカテゴリを定義できます（例: Tweet Video、Tweet Gif、Tweet Image、Amplify Video）。インストリーム広告として配信される動画をアップロードする場合には、Amplify Video カテゴリの指定が必須です。これらの動画は配信可能な形式にするために特別な処理が必要となるため、メディアカテゴリには Amplify Video を指定する必要があります。

<div id="analytics">
  ## アナリティクス
</div>

アナリティクスメトリクスは、パートナーや広告主が X 上でプロモーションしているコンテンツのパフォーマンスを把握するのに役立ちます。これには、インプレッション数、クリック数、動画再生数、支出額などの情報が含まれます。さらに、パートナーや広告主は、リーチしたオーディエンスのさまざまなセグメントごとの詳細なメトリクスも取得できます。

Ads API では、キャンペーンパフォーマンスの詳細なメトリクスを取得する方法として、同期処理と非同期処理の 2 種類をサポートしています。

[同期分析](/ja/x-ads-api/analytics#synchronous-analytics)リクエストでは、要求したメトリクスがレスポンス内に返されます。同期エンドポイントは短い期間の集計に対応しており、リアルタイムのキャンペーンパフォーマンスを照会するのに最適です。

[非同期分析](/ja/x-ads-api/analytics#asynchronous-analytics)エンドポイントでは、関連する「ジョブ」の処理が完了した後、要求したメトリクスがダウンロード可能な結果ファイルとして提供されます。非同期エンドポイントは、はるかに長い期間にわたるメトリクスのリクエストをサポートし、セグメント化されたデータの返却も可能で、より大規模なデータの取得を想定しています。レポート作成や履歴データの補完に最適です。

<div id="campaigns">
  ## キャンペーン
</div>

広告主が実施する各キャンペーンは、既存の資金提供手段に紐づけて作成する必要があります。キャンペーンでは、広告キャンペーンの名前、スケジュール、予算などを定義できます。Furni の例を使い続けると、「Furni home decor」という名前のキャンペーンを作成できます。

広告アカウントには、アクティブなキャンペーンが最大 200 件までという上限がある点に注意してください。この上限は、広告主のアカウントマネージャーによって引き上げられる場合があります。キャンペーンは、終了時刻に達するか予算を使い切るまでアクティブとみなされます。一時停止中のキャンペーンも、キャンペーン数の上限を判断する際にはアクティブなキャンペーンとして扱われます。

<div id="cards">
  ## カード
</div>

カードは、画像、動画、または UI 要素を含めることで、ポストを視覚的に拡張する手段を提供します。Ads API は、ポストで利用できる複数のカードタイプをサポートしており、それらはキャンペーンで配信できます。

<div id="creatives">
  ## クリエイティブ
</div>

広告主は、画像、GIF、動画などのクリエイティブを含むキャンペーン内でポストをプロモーション配信できます。これらのクリエイティブは、リッチな写真や動画などのメディア体験をポストに添付できる X カードに関連付けることができます。クリエイティブはポストに直接添付することも可能です。たとえば、テキストと動画（または最大 3 枚の画像）を組み合わせて、プロモーションポストを作成できます。

各ラインアイテムを配信するには、少なくとも 1 つのクリエイティブが関連付けられている必要があります。クリエイティブには、フォロワー獲得のための Promoted Accounts、TX Audience Platform 全体のラインアイテム向けに画像や動画を利用する Media Creatives、Promoted Posts など、複数のオプションがあります。

<div id="funding-instruments">
  ## 資金手段
</div>

広告アカウントには、資金手段も含まれます。これは、広告主が X Ads で利用するために設定した、さまざまな支払い手段のことです。そこには、X と締結したインサーションオーダーから、アカウントに紐づけられたクレジットカードまでが含まれます。

<div id="line-items">
  ## Line Items
</div>

キャンペーン内には 1 つ以上の広告グループ（Ads API では line item とも呼ばれます）を設定できます。line item は、キャンペーンを複数のセグメントに分割したものと考えることができ、それぞれに固有のターゲティング設定とクリエイティブがあります。複数の line item を含むキャンペーンは、「広告グループキャンペーン」と呼ばれることがよくあります。

1 つのキャンペーン内に複数の line item を設定する利点の 1 つは、ターゲティングとクリエイティブの組み合わせを、より多様なパターンで構成できることです。

「Furni home decor」キャンペーンの中で、2 つの line item を作成するとします。1 つは「Home buyers in San Francisco」、もう 1 つは「Home buyers in London」とすることができます。

1 つのキャンペーンにつき最大 100 個の line item を設定でき、1 つの広告アカウントでアクティブにできる line item は、すべてのキャンペーンを合計して最大 4,000 個までに制限されています。

<div id="mobile-app-promotion-map">
  ## モバイルアプリプロモーション (MAP)

<div id="media-library">
  ## メディアライブラリ
</div>

メディアライブラリのエンドポイントを使用すると、X Ads アカウントの画像、GIF、動画を管理できます。ライブラリ内のメディアアセットは、ポストやカードの作成に利用できます。また、複数のクリエイティブで再利用できるため、同じアセットを何度もアップロードする必要がなくなります。

<div id="promoted-accounts">
  ## プロモアカウント
</div>

プロモアカウント（フォロワーキャンペーン）は「おすすめユーザー（Who to Follow）」の一部であり、ユーザーが現在フォローしていないものの、興味を持つ可能性のあるアカウントをおすすめします。プロモアカウントは、ユーザーが楽しめるアカウントの選択肢をさらに幅広く紹介するのに役立ちます。

プロモアカウントを利用することで、広告主は自分のアカウントを宣伝し、X 上でのオーディエンスを拡大できます。フォロワーキャンペーンは、ホームタイムライン、「おすすめユーザー（Who to Follow）」、検索結果など、X プラットフォーム上の複数の場所に表示されます。また、他のおすすめアカウントと区別するために、このおすすめには「プロモーション（Promoted）」ラベルが付与されます。

<div id="promotable-users">
  ## プロモーション可能ユーザー
</div>

各 X 広告アカウントには、広告配信の対象とできる 1 つ以上のプロモーション可能ユーザーを設定できます。これは、その広告アカウントを通じてプロモーションできるユーザーのことです。Furni の @Furni ハンドルは、Furni の広告アカウントにおける FULL プロモーション可能ユーザーの一例であり、@Furni とそのポストに対してあらゆるタイプのキャンペーンを実施できることを意味します。1 つの広告アカウントに設定できる FULL プロモーション可能ユーザーは 1 つだけです。

また、広告アカウントには他の種類のプロモーション可能ユーザーを設定することもできます。たとえば、Furni 製品についての @Jack のポストをプロモーションしたい場合、彼はそのアカウント上で RETWEETS&#95;ONLY で示される、リツイート専用のプロモーション可能ユーザーになります。広告主は、リツイートユーザーを広告アカウントに追加する方法について、自身の X アカウントマネージャーに問い合わせてください。

<div id="promoted-video">
  ## プロモーション動画
</div>

プロモーション動画は、広告主が料金を支払って配信するプロモポスト内の動画コンテンツです。ホームタイムライン、X 上の検索結果の最上部、その他プラットフォーム上のさまざまな場所に表示され、「Promoted」と明確にラベル表示されます。

<div id="scheduled-tweets">
  ## 予約ポスト
</div>

予約ポストを使うと、広告主やユーザーは、後の日時に公開されるようにスケジュールされたポストを作成できます。これらのポストを作成および管理できるだけでなく、API を使用してポストをラインアイテムに関連付け、ポストが公開されたタイミングでプロモーションを行うこともできます。これにより、広告主はネイティブポストをあらかじめ用意し、主要な施策に先立ってキャンペーンクリエイティブを計画できます。たとえば、新製品発表と同時にすぐ公開されるように、ポストクリエイティブを事前に用意しておくことができます。

<div id="segmentation">
  ## セグメンテーション
</div>

セグメンテーションを使用すると、パートナーや広告主は、年齢、デバイス、言語などの特定のターゲティング値別にブレイクダウンされたメトリクスを取得できます。これは非同期アナリティクスエンドポイントからのみ利用可能です。セグメント化されたメトリクスをリクエストするには、`segmentation_type` リクエストパラメータを使用してください。セグメンテーションオプションの詳細については、「Metrics and Segmentation」を参照してください。

<div id="tailored-audience-list">
  ## テイラードオーディエンス - リスト
</div>

リストから作成するテイラードオーディエンスは、3 種類あるテイラードオーディエンスのうちの 1 つで、自社のユーザーや顧客データを含むファイルをアップロードすることで作成します。リストは X 上のユーザーと照合され、キャンペーンでそのユーザーを直接ターゲティングできるようになります。以下の制限事項を確認してください。

<div id="tailored-audience-web">
  ## テイラードオーディエンス - Web
</div>

Web 由来のテイラードオーディエンスは、3 種類あるテイラードオーディエンスのうちの 1 つで、ウェブサイトを訪れた X ユーザーをまとめたオーディエンスです。Web 由来のテイラードオーディエンスを利用するには、ウェブサイトに X ピクセルを設置するか、承認済みのタグマネージャーパートナーのいずれかを利用する必要があります。これらの制限事項を確認してください。

<div id="targeting-criteria">
  ## ターゲティング条件
</div>

前述のとおり、ターゲティング条件はラインアイテムに関連付けられます。利用できるターゲティングにはさまざまな種類があり、ロケーション、キーワード、性別といったものから、ユーザーの行動や興味関心といった、より詳細な属性まで含まれます。たとえば「Home Buyers in San Francisco」というラインアイテムでは、「home buying」という行動と「San Francisco」というロケーションを組み合わせてターゲティングすることが考えられます。

利用可能なターゲティングの種類の詳細については、開発者サイトの Targeting Overview ページをご覧ください。

<div id="twitter-object-nest-ton">
  ## Twitter Object Nest (TON)
</div>

Twitter Object Nest (TON) は、Creatives として使用するために Media Upload エンドポイントを通じてアップロードされたメディアおよび各種アセットを格納します。

<div id="creatives">
  ## クリエイティブ
</div>

各ラインアイテムは、配信を行うために少なくとも 1 つのクリエイティブが関連付けられている必要があります。クリエイティブには、フォロワー獲得のためのプロモアカウント、X Audience Platform 全体で利用できるラインアイテム向けの画像や動画などのメディアクリエイティブ、プロモポストなど、いくつかの種類があります。以下は、@Furni がプロモーションとして配信し、クリエイティブとして使用できるポストの例です。
