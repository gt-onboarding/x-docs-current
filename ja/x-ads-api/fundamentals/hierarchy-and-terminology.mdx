Ads API で成果を上げるには、X Ads における各エンティティ間の関係を理解することが重要です。このチュートリアルでは、X Ads のオブジェクト階層構造の基本について説明します。

![X Ads API の階層構造](/images/adsapi-heirarchy.png.twimg.1920.png)

<div id="ad-group-line-items">
  ## 広告グループ（ラインアイテム）
</div>

広告グループは、Ads API では[ラインアイテム](https://developer.x.com/content/developer-twitter/en/docs/tutorials/ads-api-hierarchy-terminology#line-items)と呼ばれ、キャンペーンの配下に作成され、特定の X ユーザー群に対してターゲティングや入札を行うために使用されます。広告主は、ポストやメディア（例: インストリーム広告として配信される動画）をラインアイテムに関連付けることでプロモーションを行います。「広告グループ」も参照してください。

<div id="account-media">
  ## アカウントメディア
</div>

[アカウントメディア](/ja/x-ads-api/creatives#account-media)として広告アカウントにアップロードされたクリエイティブのコレクションで、プロモーション動画の[プレロールアセット](/ja/x-ads-api/campaign-management#video-views-preroll-objective)（インストリーム広告とも呼ばれます）や、[X Audience Platform](https://developer.x.com/content/developer-twitter/en/docs/ads/measurement/overview/twitter-audience-platform)上でプロモーション可能な画像を含めることができます。

<div id="ads-accounts">
  ## 広告アカウント
</div>

X Ads の階層構造の最上位には広告アカウントがあります。1 つの広告アカウントは複数の X ユーザーが管理できます。デモ用の家具店である @Furni は、広告アカウントを管理するユーザーの一例です。広告アカウントは、[base36](https://en.wikipedia.org/wiki/Base36) の英数字文字列である account&#95;id で識別します。

各アカウントには、キャンペーン間で再利用できる複数のエンティティタイプが保持されています。たとえば、ターゲティングに利用できるテーラードオーディエンス、動画や画像であるアカウントのメディア、ポストに追加できる拡張クリエイティブであるカードなどです。

<div id="amplify-video">
  ## Amplify Video
</div>

動画アセットをアップロードする際、アップロード元のエンティティは、その動画のカテゴリ（例：Tweet Video、Tweet Gif、Tweet Image、Amplify Video）を指定できます。インストリーム広告として配信される動画をアップロードする場合、Amplify Video カテゴリを指定することが必須です。これらの動画は配信可能であることを保証するために特別な処理が必要なため、メディアカテゴリには Amplify Video を指定する必要があります。

<div id="analytics">
  ## アナリティクス
</div>

アナリティクスメトリクスは、パートナーや広告主が X 上でプロモーションしているコンテンツのパフォーマンスを把握するのに役立ちます。これには、インプレッション数、クリック数、動画再生数、支出額などの情報が含まれます。さらに、パートナーや広告主は、リーチしたオーディエンスをさまざまなセグメントに分けて、詳細なメトリクスを取得できます。

Ads API では、キャンペーンの詳細なパフォーマンスメトリクスを取得する方法として、同期と非同期の 2 通りをサポートしています。

[同期分析](/ja/x-ads-api/analytics#synchronous-analytics)リクエストでは、要求したメトリクスがレスポンス内に返されます。同期エンドポイントは短期間の指定をサポートしており、リアルタイムのキャンペーンパフォーマンスを取得するのに最適です。

[非同期分析](/ja/x-ads-api/analytics#asynchronous-analytics)エンドポイントでは、要求したメトリクスは、関連する「ジョブ」の処理が完了した後にダウンロード可能な結果ファイルとして提供されます。非同期エンドポイントは、より長期間にわたるメトリクスの取得をサポートし、セグメント別データの取得も可能で、大量のデータを取得する用途を想定しています。レポート生成や履歴データのバックフィルに最適です。

<div id="campaigns">
  ## キャンペーン
</div>

広告主が配信する各キャンペーンは、既存のファンディングインストルメント（資金手段）の配下で作成する必要があります。キャンペーンでは、広告キャンペーンの名前、スケジュール、予算などを定義できます。Furni の例を続けると、「Furni home decor」という名前のキャンペーンを作成することができます。

広告アカウントには、アクティブなキャンペーンを最大 200 個までしか保持できない点に注意してください。この上限は、広告主のアカウントマネージャーによって引き上げられる場合があります。キャンペーンは、終了時間に達するか予算を使い切るまではアクティブと見なされます。一時停止されたキャンペーンも、キャンペーン数の上限を計算する際にはアクティブなキャンペーンとして扱われます。

<div id="cards">
  ## カード
</div>

カードは、ポストに画像、動画、または UI 要素を追加して視覚的に拡張するための仕組みです。Ads API は、ポストで使用でき、その後キャンペーンでプロモーションできる複数のカードタイプをサポートしています。

<div id="creatives">
  ## クリエイティブ
</div>

広告主は、画像、GIF、動画などのクリエイティブを含むキャンペーンでポストをプロモーションすることができます。これらのクリエイティブは、ポストにリッチな写真や動画などのメディア体験を添付できるXカードと関連付けることができます。クリエイティブはポストに直接添付することもできます。たとえば、テキストと動画（または最大3枚の画像）を含めて、プロモーションポストを作成できます。

各ラインアイテムは、配信するために少なくとも1つのクリエイティブが関連付けられている必要があります。利用できるクリエイティブの種類としては、新しいフォロワー獲得のためのPromoted Accounts、TX Audience Platform全体のラインアイテム向けの画像や動画などのMedia Creatives、あるいはPromoted Postsなどがあります。

<div id="funding-instruments">
  ## 資金手段
</div>

広告アカウントの配下には、`funding instruments`（資金手段）も含まれます。これは、広告主が X Ads で利用するために設定した、さまざまな資金源を指します。ここには、広告主が X と締結したインサーションオーダーから、アカウントに紐づけられたクレジットカードに至るまでが含まれます。

<div id="line-items">
  ## ラインアイテム
</div>

キャンペーン内には 1 つ以上の広告グループ（Ads API では「ラインアイテム」と呼びます）を含めることができます。ラインアイテムは、キャンペーンを複数のセグメントに分けたものと考えることができ、それぞれが独自のターゲティング設定とクリエイティブを持ちます。複数のラインアイテムを含むキャンペーンは、「広告グループキャンペーン」と呼ばれることもあります。

1 つのキャンペーン内に複数のラインアイテムを設定する利点の 1 つは、より多様なターゲティングとクリエイティブの組み合わせを作成できることです。

「Furni home decor」キャンペーンの中で、2 つのラインアイテムを作成したいとします。1 つは「サンフランシスコの住宅購入者」、もう 1 つは「ロンドンの住宅購入者」といった形です。

1 つのキャンペーンにつき最大 100 個のラインアイテムを設定でき、1 つの広告アカウントが保有できるアクティブなラインアイテム数は、すべてのキャンペーンを合計して 4,000 個までに制限されています。

<div id="mobile-app-promotion-map">
  ## モバイルアプリプロモーション (MAP)
</div>

<div id="media-library">
  ## メディアライブラリ
</div>

メディアライブラリのエンドポイントを使用すると、X Ads アカウントの画像、GIF、動画を管理できます。ライブラリ内のメディアアセットは、ポストやカードの作成に利用できます。また、同じアセットを複数のクリエイティブで再利用できるため、同一アセットを何度もアップロードする必要がなくなります。

<div id="promoted-accounts">
  ## プロモアカウント
</div>

プロモアカウント（フォロワーキャンペーン）は「おすすめユーザー（Who to Follow）」の一部であり、現在フォローしていないものの、興味を持つ可能性のあるアカウントをおすすめします。プロモアカウントは、ユーザーが気に入るかもしれない、さらに幅広い種類のアカウントを紹介するのに役立ちます。

プロモアカウントを利用すると、広告主は自分のアカウントを宣伝し、X 上でオーディエンスを拡大できます。フォロワーキャンペーンは、ホームタイムライン、「おすすめユーザー（Who to Follow）」、検索結果など、X プラットフォーム内の複数の場所に表示されます。また、これらのおすすめアカウントには他のおすすめアカウントと区別するために「プロモーション（Promoted）」のラベルが付与されます。

<div id="promotable-users">
  ## プロモーション対象ユーザー
</div>

各 X Ads アカウントには 1 つ以上のプロモーション対象ユーザーを設定できます。プロモーション対象ユーザーとは、その広告アカウントを通じてプロモーションできるユーザーのことです。Furni の @Furni ハンドルは Furni の広告アカウントにおける FULL プロモーション対象ユーザーの一例であり、これは @Furni とそのポストに対してあらゆる種類のキャンペーンを実施できることを意味します。1 つの広告アカウントにつき、FULL プロモーション対象ユーザーは 1 つのみ設定できます。

また、広告アカウントには他の種類のプロモーション対象ユーザーを設定することもできます。たとえば、Furni 製品についての @Jack のポストをプロモーションしたい場合、彼はそのアカウント上で RETWEETS&#95;ONLY として示される、リツイート専用のプロモーション対象ユーザーになります。広告主は、リツイートユーザーを広告アカウントに追加する方法について、担当の X アカウントマネージャーに問い合わせてください。

<div id="promoted-video">
  ## プロモ動画
</div>

プロモ動画は、広告主が料金を支払って配信するプロモーションポスト内の動画です。これらはホームタイムライン、X 上の検索結果の上部、およびプラットフォーム上のその他の場所に表示され、「プロモーション」として明確に表示されます。

  ## パブリッシャーネットワーク
</div>

[X Audience Platform](/ja/x-ads-api/audiences) を参照してください。

<div id="scheduled-tweets">
  ## 予約ポスト
</div>

予約ポストを使用すると、広告主やユーザーは、後の日時に公開されるようスケジュールされたポストを作成できます。これらのポストを作成および管理できることに加えて、API を使用すると、ポストが公開された際にプロモーションできるよう、それらのポストをラインアイテムに関連付けることができます。これにより広告主は、ネイティブポストをあらかじめ作成し、主要な施策に先立ってキャンペーンクリエイティブを計画できます。たとえば、新製品発表と同時にすぐ公開されるよう、ポストのクリエイティブを事前に用意しておくといった使い方が可能です。

<div id="segmentation">
  ## セグメンテーション
</div>

セグメンテーションを利用すると、パートナーや広告主は年齢、デバイス、言語などの特定のターゲティング値別に分割したメトリクスを取得できます。この機能は非同期アナリティクスエンドポイントでのみ利用できます。セグメント化されたメトリクスをリクエストするには、`segmentation_type` リクエストパラメータを使用してください。セグメンテーションオプションの詳細については、「Metrics and Segmentation」を参照してください。

<div id="tailored-audience-list">
  ## Tailored Audience - リスト
</div>

リストから作成される Tailored Audiences（テイラードオーディエンス）は、3 種類ある Tailored Audiences のうちの 1 つであり、自社のユーザーおよび顧客データを含むファイルをアップロードして作成します。リストは X 上のユーザーと照合され、キャンペーンでそれらのユーザーを直接ターゲティングできるようになります。必ずこれらの制限事項をご確認ください。

<div id="tailored-audience-web">
  ## テイラードオーディエンス - Web
</div>

Web 由来のテイラードオーディエンスは、3 種類あるテイラードオーディエンスのタイプのうちの 1 つで、ウェブサイトを訪問した X ユーザーの集合です。Web 由来のテイラードオーディエンスを使用するには、ウェブサイトに X ピクセルを実装しているか、承認済みのタグマネージャーパートナーのいずれかを利用している必要があります。以下の制限事項を確認してください。

<div id="targeting-criteria">
  ## ターゲティング条件
</div>

前述のとおり、ターゲティング条件はラインアイテムに関連付けられます。利用できるターゲティングには、ロケーション、キーワード、性別といったものから、ユーザーの行動や興味関心など、より詳細な属性までさまざまな種類があります。「Home Buyers in San Francisco」というラインアイテムでは、「home buying」という行動ターゲティングに「San Francisco」というロケーションを組み合わせて配信することができます。

利用可能なさまざまなターゲティングの種類については、開発者サイトの Targeting Overview ページをご覧ください。

<div id="twitter-object-nest-ton">
  ## Twitter Object Nest (TON)
</div>

Twitter Object Nest (TON) には、Creatives として使用するために Media Upload エンドポイント経由でアップロードされたメディアおよび各種アセットが保存されます。

  ## クリエイティブ
</div>
