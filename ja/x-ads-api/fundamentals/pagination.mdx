より多くのキャンペーン数の上限に対応し、アカウントに関連付けられたすべてのエンティティを効率的に取得できるようにするため、Advertiser API では多くの GET エンドポイントでページネーションをサポートするようになりました。ページネーションの仕組みは使いやすく、[カーソルを使用してコレクションを操作する](https://developer.x.com/en/docs/x-api/v1/pagination) で説明されている REST API のカーソルベースのページネーションと非常によく似ています。

<div id="getting-started">
  ## はじめに
</div>

指定された GET リクエストでは、オプションのクエリパラメータとして `cursor` と `count` を受け付けるようになりました。レスポンスに含まれるエンティティ数が `count` を超える場合、これらのエンドポイントは最初の `count` 件のエンティティと、レスポンス JSON 内の `next_cursor` キーを返します。

`GET https://ads-api.x.com/5/accounts/abc1/campaigns?count=50`

```json
{
  "data": \[...\],
  "next_cursor": "c-3yvu1pzhd3i7",
  "request": {...}
}
```

次の一連のレスポンスを取得するには、`cursor` をクエリパラメーターとして追加します。

`GET https://ads-api.x.com/5/accounts/abc1/campaigns?cursor=c-3yvu1pzhd3i7&count=50`

```json
{
  "data": \[...\],
  "next_cursor": "c-3w3zdyg8ywan",
  "request": {...}
}
```


<div id="going-deeper">
  ## さらに詳しく
</div>

ほとんどのエンドポイントでは、`count` の最大値は **1,000**、最小値は **1**、デフォルト値は **200** です。

`next_cursor` によって返される値は常に文字列であり、その中身に依存しない不透明な値として扱う必要があります。実装は将来変更される可能性があります。現在の結果セットのページで返されるエンティティが `count` より少ない場合、`next_cursor` の値は `null` になります。

**注:** analytics エンドポイントでは、この方式のページネーションはサポートされていません。stats エンドポイントでのページングは、時間の期間を指定することでサポートされます。詳細については、各エンドポイントのドキュメントを参照してください。
