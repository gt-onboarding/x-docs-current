より多くのキャンペーンを扱い、アカウントに関連付けられたすべてのエンティティを効率的に取得できるようにするため、Advertiser API は多くの GET エンドポイントでページネーションをサポートするようになりました。ページングメカニズムは使いやすく、[Using cursors to navigate collections](https://developer.x.com/en/docs/x-api/v1/pagination) で説明されている REST API のカーソルベースのページネーションと非常によく似ています。

<div id="getting-started">
  ## はじめに
</div>

特定の GET リクエストでは、オプションのクエリパラメータとして `cursor` と `count` を指定できるようになりました。レスポンスに含まれるエンティティ数が `count` を超える場合、これらのエンドポイントは、先頭の `count` 件のエンティティおよびレスポンス JSON 内の `next_cursor` キーを返します。

`GET https://ads-api.x.com/5/accounts/abc1/campaigns?count=50`

```json
{
  "data": \[...\],
  "next_cursor": "c-3yvu1pzhd3i7",
  "request": {...}
}
```

次の一連のレスポンスを取得するには、クエリパラメータに `cursor` を次のように追加します:

`GET https://ads-api.x.com/5/accounts/abc1/campaigns?cursor=c-3yvu1pzhd3i7&count=50`

```json
{
  "data": \[...\],
  "next_cursor": "c-3w3zdyg8ywan",
  "request": {...}
}
```


<div id="going-deeper">
  ## さらに詳しく
</div>

ほとんどのエンドポイントでは、`count` の最大値は **1,000**、最小値は **1**、既定値は **200** です。

`next_cursor` によって提供される値は常に文字列であり、中身には依存しない不透明な値として扱ってください。実装は将来変更される可能性があります。現在の結果セットのページで返されるエンティティ数が `count` 未満の場合、`next_cursor` の値は `null` になります。

**注:** アナリティクスエンドポイントは、この形式のページネーションをサポートしていません。stats エンドポイントでのページングは、時間の区切り（期間）を指定することで行います。詳細については、それぞれのエンドポイントのドキュメントを参照してください。
