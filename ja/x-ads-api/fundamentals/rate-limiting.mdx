Advertiser API は、ここで説明されている REST API v1.1 と同様にレート制限が行われます: [REST API Rate Limiting in v1.1](https://developer.x.com/content/developer-twitter/en/docs/basics/rate-limits)。REST API v1.1 と異なり、エンドポイントごとの制限をプログラムから参照するためのインデックスは用意されていません。エンドポイントのレート制限とリセットウィンドウは、HTTP レスポンスヘッダーによって通知されます。Ads API におけるすべてのレート制限は OAuth 1.0A を利用して行われます。

<div id="user-level-and-ad-account-level-limits">
  ## ユーザーレベルおよび広告アカウントレベルの制限
</div>

レート制限には、ユーザートークンレベルと広告アカウントレベルの 2 種類があります。エンドポイントの一部は、広告アカウントレベルのレート制限を使用できるように有効化されています。ユーザートークンとは、Ads API を認証して呼び出すために使用する[OAuth アクセストークン](/ja/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)のことです。各ユーザートークンは、1 つ以上の広告アカウントにアクセスできます。

開発者は、レスポンスヘッダーで広告アカウントレベルのレート制限が返される場合はそれを利用し、広告アカウントレベルの制限が存在しない場合にのみユーザーレベルの制限を利用するようにしてください。

ユーザーレベルのレート制限は、次のヘッダーで表されます：**x-rate-limit-limit**、**x-rate-limit-remaining**、**x-rate-limit-reset**

広告アカウントレベルのレート制限が有効になっているエンドポイントでは、レート制限は次のヘッダーで表されます：**x-account-rate-limit-limit**、**x-account-rate-limit-remaining**、**x-account-rate-limit-reset**。

広告アカウントレベルのレート制限は、アプリケーションが 1 つのユーザートークンから複数の広告アカウントにアクセスして、エンティティデータ（キャンペーンやラインアイテムオブジェクトなど）を同期できるようにするため、GET エンドポイントに対してのみ提供されます。書き込み系のアクションについては、同じ広告アカウントレベルのレート制限が適用されることは保証されません。

広告アカウントレベルのレート制限があるエンドポイントでは、ユーザーレベルのレート制限は、アプリケーション全体に対するグローバルなクォータを表す高い値に設定されています。利用可能な場合は、リクエスト量を制御する際に広告アカウントレベルのレート制限を優先的に使用する必要があります。

<div id="best-practices">
  ## ベストプラクティス
</div>

1. データベースに最後に同期したタイムスタンプを保存し、データをリクエストする際に可能な場合は `sort_by=updated_at-desc` オプションを指定して、最後に同期したタイムスタンプより古いデータに到達したところで同期処理を停止できるようにします。これにより、同じデータを重複して同期することを回避できます。

2. 1 回のリクエストで複数のエンティティをリクエストする: 一部のエンドポイントでは、カンマ区切りの値のリストを指定して、同種のデータをまとめて取得できます。これにより、API 呼び出しの総数を減らし、レート制限をより効率的に活用できます。

3. リクエストでは最大の「count」を指定する: [GET accounts/:account&#95;id/targeting&#95;criteria](/ja/x-ads-api/campaign-management#targeting-criteria) のような一部のエンドポイントは、デフォルトの 200 ではなく 1000 個のオブジェクトを返すよう、count の最大値で呼び出すことを強く推奨します。

<div id="faqs">
  ## よくある質問
</div>

*特定の広告アカウントまたは自社アプリケーションのレート制限を引き上げることはできますか？*

原則として、レート制限を引き上げることはできません。レート制限は、最大規模の広告アカウントをサポートできるように設定されています。まずは本ドキュメントに記載されているベストプラクティスを実装してください。それでもなおレート制限がシステムのスケールやビジネス目標の達成に影響している場合は、ご利用のユースケースおよび関連するリクエストの詳細な情報を添えて、X Ads API の担当窓口までご連絡ください。

<div id="scopes-of-rate-limiting">
  ## レート制限の適用範囲
</div>

<div id="scopes-for-this-doc">
  ### このドキュメントにおけるスコープ
</div>

* Category: 指定されたカテゴリに属するすべてのエンドポイントは、ウィンドウごとに割り当てられた 1 つの上限値を共有してレート制限されます。
* Endpoint: 各エンドポイントにはウィンドウごとに固有の上限値が割り当てられ、その上限値に基づいてレート制限されます。
