Advertiser API には、[REST API Rate Limiting in v1.1](https://developer.x.com/content/developer-twitter/en/docs/basics/rate-limits) に記載されている REST API v1.1 と同様のレート制限が適用されます。REST API v1.1 とは異なり、エンドポイントごとの制限を示すプログラム的な一覧は提供されていません。エンドポイントのレート制限とリセットウィンドウ（リセット間隔）は、HTTP レスポンスヘッダーを通じて通知されます。Ads API におけるすべてのレート制限は OAuth 1.0A によって実装されています。

<div id="user-level-and-ad-account-level-limits">
  ## ユーザーレベルおよび広告アカウントレベルの制限
</div>

レート制限には、ユーザートークンレベルと広告アカウントレベルの 2 種類があります。エンドポイントの一部は、広告アカウントレベルのレート制限を利用できるようになっています。ユーザートークンは、Ads API の認証および呼び出しに使用する[OAuth access token](/ja/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)です。各ユーザートークンは、1 つ以上の広告アカウントにアクセスできます。

開発者は、レスポンスヘッダーで広告アカウントレベルのレート制限が返される場合はそれを利用し、広告アカウントレベルの制限が提供されない場合にのみユーザーレベルの制限を利用するようにしてください。

ユーザーレベルのレート制限は、次のヘッダーで示されます：**x-rate-limit-limit**、**x-rate-limit-remaining**、**x-rate-limit-reset**

広告アカウントレベルのレート制限が有効になっているエンドポイントでは、レート制限は次のヘッダーで示されます：**x-account-rate-limit-limit**、**x-account-rate-limit-remaining**、**x-account-rate-limit-reset**。

広告アカウントレベルのレート制限は、アプリケーションが 1 つのユーザートークンから複数の広告アカウントにアクセスしてエンティティ データ（キャンペーンやラインアイテムオブジェクトなど）を同期できるようにするため、GET エンドポイントに対してのみ提供されます。書き込み系の操作については、同じ広告アカウントレベルのレート制限が適用されることは保証されません。

広告アカウントレベルのレート制限が適用されるエンドポイントでは、ユーザーレベルのレート制限は、アプリケーション全体のグローバルクォータを表す高い値に設定されています。利用可能な場合は、リクエスト数を制御する際に広告アカウントレベルのレート制限を優先して使用する必要があります。

<div id="best-practices">
  ## ベストプラクティス
</div>

1. データベースに最後に同期したタイムスタンプを保存し、該当する場合は `sort_by=updated_at-desc` オプションを指定してデータをリクエストすることで、最後に同期したタイムスタンプより古いデータに到達した時点で同期処理を停止できるようにします。これにより、同じデータを重複して同期してしまうことを防げます。

2. 単一のリクエストで複数のエンティティをリクエストする: 一部のエンドポイントでは、カンマ区切りの値のリストを指定して、類似したデータを複数取得できます。これにより、API 呼び出し回数の総数を削減し、レート制限をより効率的に活用できます。

3. リクエストでは「count」の最大値を指定する: [GET accounts/:account&#95;id/targeting&#95;criteria](/ja/x-ads-api/campaign-management#targeting-criteria) のような一部のエンドポイントは、デフォルトの 200 件ではなく 1000 件のオブジェクトを返すよう、count パラメーターの最大値で呼び出すことが強く推奨されています。

<div id="faqs">
  ## FAQ
</div>

*特定の広告アカウントまたは自社アプリケーションのレート制限を増やすことは可能ですか？*

一般的に、レート制限を個別に引き上げることはできません。レート制限は、最大規模の広告アカウントにも対応できるように設定されています。まずは本ドキュメントに記載されているベストプラクティスを実装してください。それでもなおレート制限がスケールの拡大やビジネス目標の達成に影響している場合は、ユースケースと関係するリクエストに関する詳細な情報を添えて、X Ads API の担当窓口までお問い合わせください。

  ## レート制限の対象範囲
</div>

<div id="scopes-for-this-doc">
  ### このドキュメントにおけるスコープ
</div>

* Category: 指定されたカテゴリに属するすべてのエンドポイントは、ウィンドウごとに割り当てられたひとつの上限でレート制限されます。
* Endpoint: 各エンドポイントには、ウィンドウごとに固有の割り当て上限が設定されます。
