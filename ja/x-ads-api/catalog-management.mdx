<div id="overview">
  ## 概要
</div>

Catalog API は、広告主が商品フィードを設定し、商品をセットにグルーピングし、カタログ内の商品を一元的に管理できるようにするコマースソリューションです。Catalog API は、カタログをプログラムで管理するための中核ツールとなり、広告主がカタログの取り込み方法や更新方法をより細かく制御できるようにします。

カタログには、ユーザーの商品および商品セットに関するあらゆる情報が保持され、ユーザーのハンドルに紐づきます。現在、**1 人のユーザーが作成できるカタログは 1 つのみです**。 

Catalog API は、次の 2 つの方法で商品の取り込みをサポートします。

1.  Scheduled Feed: ユーザーはフィード URL（CSV、TSV、XML）を追加でき、定期的に取得されます（X では現在、最大 8GB までのファイルをサポートしています）

2.  Batch Products API: ユーザーはバッチ（JSON）リクエストを使って、商品属性の閲覧、作成、更新、削除を行うことができます

X のマーチャントは、フィルタールールに基づいて商品セットを作成できます。これにより、商品に追加のメタデータを付与したり、オーガニックコマース機能やダイナミック商品広告を通じてそれらの商品を訴求するための特別な商品セットを作成したりできます。Catalog API と X shopping manager は、2 種類の商品セットをサポートしています。 

1.  Manual: ユーザーは最大 50 件の商品を選択し、名前と説明を設定できます

2.  Filter: ユーザーは最大 30 個までフィルターを追加し、それらのフィルターを満たす商品によるセットを自動生成できます。フィルターの種類には、価格、Google 商品カテゴリ、商品タイプ、在庫、セール価格、カスタムフィールドなどがあります。

Catalog API には、カタログを管理するための 4 つのエンドポイントがあります。

* [Product Catalogs](https://developer.x.com/en/docs/x-ads-api/catalog-management/api-reference/product-catalogs) \- 商品情報のコンテナを作成
* [Products](https://developer.x.com/en/docs/x-ads-api/catalog-management/api-reference/products) -  カタログへの商品の同期的な追加、更新、閲覧、削除
* [Product Sets](https://developer.x.com/en/docs/x-ads-api/catalog-management/api-reference/product-sets) -  商品情報に基づいて商品をグループに分類
* [Schedule Feeds](https://developer.x.com/en/docs/twitter-ads-api/catalog-management/api-reference/scheduled-feeds) \- ファイルベースの商品データをカタログに非同期取り込みするための設定

<div id="prerequisites-for-the-catalog-api">
  ### Catalog API を利用するための前提条件
</div>

Catalog API エンドポイントは現在、アーリーアクセスでのみ利用できます。アクセスを申請するには、X の担当者までお問い合わせください。 

[X Shopping Manager](https://ads.x.com/shopping_manager) 上でサービス利用規約に同意する必要があります。

Shopping Manager および商品仕様の詳細については、[Product specifications guide](https://business.x.com/en/help/shopping-specs.html) を参照してください。
