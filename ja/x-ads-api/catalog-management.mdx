<div id="overview">
  ## 概要
</div>

Catalog API は、広告主が商品フィードを設定し、商品をセットにグループ化し、カタログ内の商品を包括的に管理できるコマースソリューションです。Catalog API は、カタログのプログラムによる管理を可能にし、広告主に対してカタログがどのように取り込まれ更新されるかについて、より高度なコントロールを提供する中核ツールとなります。

カタログには、ユーザーの商品および商品セットに関するすべての情報が保持され、ユーザーのハンドル（@ユーザー名）に紐づけられます。現在、**1人のユーザーが作成できるカタログは 1 つのみです**。 

Catalog API は、商品を取り込む方法として 2 つをサポートしています：

1.  Scheduled Feed: ユーザーはフィード URL（CSV、TSV、XML）を追加でき、指定された URL から定期的に取得されます（現在 X では、最大 8GB までのファイルをサポートしています）

2.  Batch Products API: ユーザーはバッチ（JSON）リクエストにより、商品属性の閲覧、作成、更新、削除を行うことができます

X のマーチャントは、フィルタールールに基づいて商品セットを作成できます。これにより、自身の商品に追加のメタデータを付与し、オーガニックコマース機能やダイナミック商品広告を通じてこれらの商品を表示するための特別な商品セットを作成できるようになります。Catalog API と X ショッピングマネージャーは、2 種類の商品セットをサポートしています。 

1.  Manual: ユーザーは最大 50 個の商品を選択し、名前と説明を設定できます

2.  Filter: ユーザーは最大 30 個のフィルターを追加して、これらのフィルターを満たす商品によるセットを自動生成できます。フィルターの種類には、価格、Google 商品カテゴリ、商品タイプ、在庫、セール価格、カスタムフィールドなどがあります。

Catalog API には、カタログを管理するための 4 つのエンドポイントがあります：

* [Product Catalogs](https://developer.x.com/en/docs/x-ads-api/catalog-management/api-reference/product-catalogs) \- 商品情報のコンテナを作成
* [Products](https://developer.x.com/en/docs/x-ads-api/catalog-management/api-reference/products) -  カタログから商品を同期的に追加、更新、閲覧、および削除
* [Product Sets](https://developer.x.com/en/docs/x-ads-api/catalog-management/api-reference/product-sets) -  商品情報に基づいて商品をグループに整理
* [Schedule Feeds](https://developer.x.com/en/docs/twitter-ads-api/catalog-management/api-reference/scheduled-feeds) \- ファイルベースの商品データをカタログに非同期で取り込む設定

<div id="prerequisites-for-the-catalog-api">
  ### Catalog API の前提条件
</div>

Catalog API エンドポイントは現在、早期アクセスとしてのみ利用可能です。アクセスを申請するには、X の担当者までお問い合わせください。

[X Shopping Manager](https://ads.x.com/shopping_manager) を通じて、利用規約に同意する必要があります。

Shopping Manager と商品仕様の詳細については、[Product specifications guide](https://business.x.com/en/help/shopping-specs.html) を参照してください。
