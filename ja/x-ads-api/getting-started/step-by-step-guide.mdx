<div id="how-to-get-access-to-the-ads-api">
  ## Ads API へのアクセス方法
</div>

1. [開発者アカウント](https://developer.x.com/en/apply-for-access)に登録します。
2. [開発者アプリ](https://developer.x.com/app)を作成し、トークンを安全に保管します。
3. [ads.x.com/help](https://ads.x.com/help) にアクセスし、各開発者アプリごとに Ads API アクセスのリクエストを送信します。

<Note>**注:** すでに X Developer Platform を利用しており、開発者アカウントをお持ちの場合は、手順 3 に進んでください。</Note>

<div id="step-one-signup-for-a-developer-account">
  ### ステップ1：開発者アカウントに登録する
</div>

X のいずれかの API プロダクトにリクエストを送信するには、まず開発者アカウントに登録する必要があります。開発者ポータル内でプロジェクトと開発者アプリを作成してください。これにより、API へのすべてのリクエストの認証に使用する一連の認証情報が発行されます。

<Button href="https://developer.x.com/en/apply-for-access">
  開発者アカウントに登録
</Button>

<div id="step-two-save-your-apps-key-and-tokens-and-keep-them-secure">
  ### ステップ2：アプリのキーとトークンを保存し、安全に保管する
</div>

開発者アプリ内で、API Key（Consumer Key とも呼ばれます）のセットが提供されます。また、あなたの個人の X アカウントに代わってリクエストを行うために使用できる Access Token のセットと、OAuth 2.0 Bearer Token を必要とするエンドポイントの認証に使用できる Bearer Token を生成することもできます。これらのキーとトークンは再生成しない限り有効期限が切れないため、環境変数として保存するか、安全なパスワードマネージャーを使用することをおすすめします。

あわせて、「App details」ページの URL に表示されている App ID も必ず控えておいてください。これは、次のステップで Ads API へのアクセスを申請する際に役立ちます。

<Note>
**Note:** developer portal 上でキーとトークンが表示されるのは一度きりのため、生成後すぐにパスワード管理システムにこれらの認証情報を保存することが重要です。

キーとトークンを紛失したり忘れてしまった場合は、それらを再生成する必要があります。これにより新しいキーとトークンが作成され、古いものは無効になります。つまり、以前の認証情報を使用して構成したあらゆる連携を更新しなければならないことを意味します。

[認証に関するベストプラクティス](https://developer.x.com/en/docs/authentication/guides/authentication-best-practices)について詳しくは、こちらをご覧ください。
</Note>

<div id="step-three-apply-for-access-to-the-ads-api">
  ### ステップ3: Ads API へのアクセスを申請する
</div>

この時点で、すでに [X API への Basic アクセス](https://developer.x.com/en/docs/twitter-api/v1) は取得していますが、特定の X Ads API の機能にはまだアクセスできません。次に、Ads API へのアクセスを申請し、承認を受ける必要があります。[ads.x.com/help](https://ads.x.com/help) にアクセスし、各開発者アプリごとに Ads API アクセスを申請してください。

<div id="access-tiers">
  ### アクセスレベル
</div>

申請手続きの一環として、必要なアクセスレベルを指定する必要があります。詳しくは、[アプリレベルおよび広告アカウントレベルの権限](https://developer.x.com/en/docs/twitter-ads-api/obtaining-ads-account-access#levels-of-access)をご覧ください。

<div id="conversion-only">
  #### コンバージョン専用
</div>

モバイルおよびウェブのコンバージョン用エンドポイントへの読み取りおよび書き込みアクセス。

<div id="standard-access">
  #### 標準アクセス
</div>

Analytics、Campaign Management、Creatives、Custom Audiences、Conversion 関連のエンドポイントへの読み取りおよび書き込みアクセス。

<Note>**注:** アプリが Ads API アクセス用に承認された後、正しく[認証された Ads API リクエスト](https://developer.x.com/en/docs/twitter-ads-api/making-authenticated-requests)を行うには、[すべてのユーザーアクセストークンを再生成](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)する必要があります。</Note>

<Note>**注:** 2023 年 7 月以前に Ads API へのアクセスを申請したデベロッパーは、アクセスレベルや権限が異なる場合があり、OAuth トークンが 5 個に制限されている可能性があります。追加のエンドポイントへのアクセス権を取得したり、既存アプリケーションのトークン上限を引き上げたりする方法については、[アクセス拡張](https://developer.x.com/en/docs/twitter-ads-api/increasing-access)に関するガイドを参照してください。</Note>

<div id="make-your-first-request">
  ## 最初のリクエストを送信する
</div>

Ads API へのアクセスをテストするには、[GET accounts](https://developer.x.com/en/docs/twitter-ads-api/campaign-management/api-reference/accounts) エンドポイントにリクエストを送信します。このエンドポイントは、現在認可されているユーザーがアクセスできる広告アカウントを返します。このリクエストで取得した広告アカウントの ID を、後続の API リクエストで使用して、特定の広告アカウントのデータを読み取りおよび書き込みします。コマンドラインで [Twurl](https://developer.x.com/en/docs/tutorials/using-twurl) を使用する場合、リクエストは次のようになります。

**リクエスト例**

```bash
twurl -H ads-api.x.com "/11/accounts"
```

**レスポンス例**

```json

    "params": {}
  },
```
