<div id="how-to-get-access-to-the-ads-api">
  ## Ads API へのアクセス方法
</div>

1. [開発者アカウント](https://developer.x.com/en/apply-for-access)に登録します。
2. [開発者アプリ](https://developer.x.com/app)を作成し、トークンを安全に保管します。
3. 各開発者アプリごとに Ads API へのアクセスを申請するには、[ads.x.com/help](https://ads.x.com/help) にアクセスします。

<Note>**注記：** すでに X Developer Platform を利用しており、開発者アカウントをお持ちの場合は、手順 3 から開始してください。</Note>

<div id="step-one-signup-for-a-developer-account">
  ### ステップ1：開発者アカウントに登録する
</div>

X の各種 API 製品にリクエストを行うには、まず開発者アカウントに登録する必要があります。developer portal でプロジェクトと開発者アプリを作成します。これにより、API へのすべてのリクエストを認証するために使用する一連の認証情報を取得できます。

<Button href="https://developer.x.com/en/apply-for-access">
  開発者アカウントに登録
</Button>

<div id="step-two-save-your-apps-key-and-tokens-and-keep-them-secure">
  ### ステップ2：アプリのキーとトークンを保存し、安全に管理する
</div>

開発者アプリ内で、API Keys（Consumer Keys とも呼ばれます）のセットが発行されます。あわせて、あなたの個人の X アカウントに代わってリクエストを行うために使用できる Access Tokens のセットと、OAuth 2.0 Bearer Token を要件とするエンドポイントの認証に使用できる Bearer Token を生成することもできます。これらのキーとトークンは再生成しない限り有効期限がないため、環境変数を作成するか、安全なパスワードマネージャーを使用することをおすすめします。

また、App ID も控えておいてください。これはアプリ詳細ページの URL に含まれています。次のステップで Ads API へのアクセスをリクエストする際に、これがあるとスムーズになります。

<Note>
**注意:** キーとトークンは developer portal 上で一度しか表示されません。そのため、生成したらすぐにパスワード管理システムにこれらの認証情報を保存しておくことが重要です。

キーとトークンを紛失したり忘れたりした場合は、それらを再生成する必要があり、その際に新しいキーとトークンが作成され、古いものは無効になります。これは、以前の認証情報で設定していたあらゆる連携を更新する必要があることを意味します。

[認証のベストプラクティス](https://developer.x.com/en/docs/authentication/guides/authentication-best-practices)について、詳しくはこちらをご覧ください。
</Note>

<div id="step-three-apply-for-access-to-the-ads-api">
  ### ステップ3: Ads API へのアクセスを申請する
</div>

この時点で、すでに [X API への基本アクセス](https://developer.x.com/en/docs/twitter-api/v1) はありますが、X Ads API の機能にはまだアクセスできません。次に、Ads API へのアクセスを申請し、承認を受ける必要があります。[ads.x.com/help](https://ads.x.com/help) にアクセスし、各開発者アプリごとに Ads API アクセスのリクエストを送信してください。

<div id="access-tiers">
  ### アクセスレベル
</div>

申請プロセスの一環として、必要なアクセスレベルを指定する必要があります。[アプリレベルおよび広告アカウントレベルの権限](https://developer.x.com/en/docs/twitter-ads-api/obtaining-ads-account-access#levels-of-access)については、こちらをご覧ください。

<div id="conversion-only">
  #### コンバージョン専用
</div>

Mobile および Web のコンバージョン用エンドポイントへの読み取りおよび書き込みアクセス。

<div id="standard-access">
  #### 標準アクセス
</div>

Analytics、Campaign Management、Creatives、Custom Audiences、Conversion エンドポイントへの読み取りおよび書き込みアクセス。

<Note>**Note:** アプリの Ads API へのアクセスが承認された後、適切に[認証された Ads API リクエスト](https://developer.x.com/en/docs/twitter-ads-api/making-authenticated-requests)を行うには、[ユーザーアクセストークンを再生成する](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)必要があります。</Note>

<Note>**Note:** 2023 年 7 月以前にアクセスをリクエストした Ads API 開発者は、異なるレベルのアクセス権や権限が付与されている場合があり、OAuth トークンが 5 個に制限されている可能性があります。追加のエンドポイントへのアクセスや既存アプリケーションのトークン上限の引き上げについては、[アクセスの拡大](https://developer.x.com/en/docs/twitter-ads-api/increasing-access)に関するガイドを参照してください。</Note>

<div id="make-your-first-request">
  ## 最初のリクエストを送信する
</div>

Ads API へのアクセスをテストするには、[GET accounts](https://developer.x.com/en/docs/twitter-ads-api/campaign-management/api-reference/accounts) エンドポイントにリクエストを送信します。このエンドポイントは、現在認可されているユーザーがアクセス権を持つ広告アカウントを返します。このリクエストで取得した広告アカウントの ID を、後続の API リクエストで使用して、特定の広告アカウントのデータの読み書きを行います。コマンドラインで [Twurl](https://developer.x.com/en/docs/tutorials/using-twurl) を使用すると、リクエストは次のようになります。

**リクエスト例**

```bash
twurl -H ads-api.x.com "/11/accounts"
```

**レスポンスの例**

