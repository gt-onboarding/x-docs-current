Direct Messages は、X 上で非公開の会話を行うための機能です。Direct Messages は X でもっとも人気のある機能の 1 つであり、幅広いユースケースがあります。ユースケースは、友人同士のグループチャットから、世界中のブランドのカスタマーサポートを支える用途まで多岐にわたります。Direct Messages エンドポイントの新しい v2 バージョンは段階的に導入され、この最初の段階では、Direct Messages の作成と Direct Message 会話イベントの一覧取得のための基本的なエンドポイントが含まれます。X API v2 が *グループ* 会話をサポートするのは、これが初めてです。

Manage Direct Messages の最初のリリースには、Direct Messages を作成するための 3 つの POST メソッドが含まれます。

* **POST /2/dm&#95;conversations/with/:participant&#95;id/messages** - 1 対 1 の Direct Message を作成します。このメソッドは、新しい 1 対 1 の会話を作成するか、既存の会話を取得してそこに Direct Message を追加します。:participant&#95;id パスパラメータは、メッセージを受信するアカウントのユーザー ID です。
* **POST /2/dm&#95;conversations** - 新しいグループ会話を作成し、その会話に Direct Message を追加します。これらのリクエストには、会話参加者のリストが必要です。同じ参加者リストで複数の会話を作成できる点に注意してください。これらのリクエストでは、常に新しい会話 ID が返されます。
* **POST /2/dm&#95;conversations/:dm&#95;conversation&#95;id/messages** - Direct Message を作成し、既存の会話に追加します。:dm&#95;conversation&#95;id パスパラメータは、メッセージが追加される会話の ID です。

Direct Message イベント ID は v1.1 と v2（および X アプリ）間で共通であるため、Direct Messages を非表示または削除するための v1.1 メソッドを、この新しい v2 エンドポイントと併用できます。また、Enterprise および Premium Account Activity API は v2 の 1 対 1 メッセージをサポートしていますが、グループ会話はまだサポートしていない点にも注意してください。

POST メソッドには、ユーザー単位で 15 分あたり 200 リクエストのレート制限があります。さらに、ユーザーごとに 24 時間あたり 1000 リクエストのレート制限があります。加えて、全体で 24 時間あたり 15000 リクエストのレート制限があります。これらのレート制限は、ここで説明している POST エンドポイント間で共有される点に注意してください。

Manage Direct Messages エンドポイントでユーザーに代わってリクエストを送信するため、[OAuth 1.0a User Context](https://developer-staging.x.com/resources/fundamentals/authentication) または [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) を使用して認証し、あなたのアプリを承認したユーザーに紐づくユーザーアクセストークンを使用する必要があります。OAuth 1.0a でこのユーザーアクセストークンを生成するには、[3-legged OAuth フロー](/ja/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token) を利用できます。OAuth 2.0 でユーザーアクセストークンを生成するには、[Authorization Code with PKCE グラントフロー](/ja/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) を利用できます。

<Note>
  **アカウントのセットアップ**

  これらのエンドポイントにアクセスするには、次のものが必要です。

  * 承認済みの[開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)。
  * [プロジェクト](/ja/resources/fundamentals/projects) 内にある [開発者アプリ](/ja/resources/fundamentals/developer-apps) のキーとトークンを使用した認証。

  X API v2 エンドポイントへのアクセス方法について詳しくは、[はじめにガイド](/ja/x-api/getting-started/getting-access) を参照してください。
</Note>
