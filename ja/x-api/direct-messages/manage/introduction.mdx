ダイレクトメッセージは、X 上で非公開の会話を行うための機能です。ダイレクトメッセージは X で最も人気のある機能の 1 つであり、幅広いユースケースがあります。これらのユースケースには、友人同士のグループチャットから、世界中のブランドによるカスタマーサポートまで、多様な利用方法が含まれます。ダイレクトメッセージの新しい v2 バージョンのエンドポイントは段階的に公開され、この最初のフェーズには、ダイレクトメッセージの作成およびダイレクトメッセージ会話イベントの一覧取得のための基本的なエンドポイントが含まれます。X API v2 で *初めて*、グループ会話がサポートされます。

この最初の Manage Direct Messages リリースには、ダイレクトメッセージを作成するための 3 つの POST メソッドが含まれます。

* **POST /2/dm&#95;conversations/with/:participant&#95;id/messages** - 1 対 1 のダイレクトメッセージを作成します。このメソッドは、新しい 1 対 1 会話を作成するか、既存の会話を取得してその会話にダイレクトメッセージを追加します。:participant&#95;id パスパラメータは、メッセージを受信するアカウントのユーザー ID です。
* **POST /2/dm&#95;conversations** - 新しいグループ会話を作成し、その会話にダイレクトメッセージを追加します。これらのリクエストには、会話参加者のリストが必要です。同じ参加者リストで複数の会話を作成できる点に注意してください。これらのリクエストは常に新しい会話 ID を返します。
* **POST /2/dm&#95;conversations/:dm&#95;conversation&#95;id/messages** - ダイレクトメッセージを作成し、既存の会話に追加します。:dm&#95;conversation&#95;id パスパラメータは、メッセージが追加される会話の ID です。

ダイレクトメッセージイベントの ID は v1.1 と v2（および X アプリ）全体で共通であるため、ダイレクトメッセージを非表示／削除する v1.1 メソッドは、この新しい v2 エンドポイントと併用できます。また、Enterprise および Premium Account Activity API では v2 の 1 対 1 メッセージがサポートされていますが、グループ会話はまだサポートされていない点にも注意してください。

POST メソッドには、ユーザー単位で 15 分あたり 200 リクエストというレート制限があります。さらに、ユーザーごとに 24 時間あたり 1,000 リクエストのレート制限があります。加えて、ユーザー単位とは別に、24 時間あたり 15,000 リクエストのレート制限があります。これらのレート制限は、ここで説明している POST エンドポイント間で共有される点に注意してください。

manage Posts エンドポイントでユーザーに代わってリクエストを送信するため、[OAuth 1.0a User Context](https://developer-staging.x.com/resources/fundamentals/authentication) または [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) のいずれかで認証を行い、あなたのアプリを承認したユーザーに関連付けられたユーザーアクセストークンを使用する必要があります。OAuth 1.0a でこのユーザーアクセストークンを生成するには、[3-legged OAuth フロー](/ja/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)を使用できます。OAuth 2.0 でユーザーアクセストークンを生成するには、[Authorization Code with PKCE グラントフロー](/ja/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) を使用できます。

<Note>
  **アカウント設定**

  これらのエンドポイントにアクセスするには、以下が必要です。

  * 承認済みの [開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)。
  * [プロジェクト](/ja/resources/fundamentals/projects)内にある [開発者アプリ](/ja/resources/fundamentals/developer-apps) のキーとトークンを使用して認証すること。

  X API v2 エンドポイントへのアクセス取得の詳細については、[はじめにガイド](/ja/x-api/getting-started/getting-access)をご覧ください。
</Note>
