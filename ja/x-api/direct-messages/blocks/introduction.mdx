manage DM blocks エンドポイントを使用すると、認証済みユーザーに代わって、指定したアカウントをブロックまたはブロック解除できます。これらのエンドポイントには、2つの POST メソッドが用意されています。

* **/2/users/:id/dm/block**: アカウントをブロックできます
* **/2/users/:id/dm/unblock**: アカウントのブロックを解除できます

</div>

<div id="authentication">
  ### 認証
</div>

ユーザーに代わってリクエストを行うため、これらのエンドポイントには必ず [OAuth 1.0a User Context](/ja/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) または [OAuth 2.0 Authorization Code with PKCE](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) を用いて認証を行い、リクエスト対象のユーザーに紐付いたユーザーのアクセストークンを使用する必要があります。ユーザーのアクセストークンは、[3-legged OAuth フロー](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)（OAuth 1.0a）または [Authorization Code with PKCE フロー](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)（OAuth 2.0）を使用して生成できます。

<div id="making-a-request">
  ### リクエストの送信
</div>

Block

ユーザーがアプリで認証を済ませると、以下のようにそのユーザーの代わりに Block エンドポイントを呼び出せます。

```
      curl --request POST 'https://api.x.com/2/users/:id/dm/block' --header 'Authorization: ••••••'
```

リクエストが成功すると、次のような JSON レスポンスが表示されます。

```
{
   "data": {
       "blocked": true
   }
}
```

**ブロック解除**

ユーザーがアプリで認証を完了したら、以下のようにそのユーザーに代わって Unblock エンドポイントを呼び出すことができます。

```
      curl --request POST 'https://api.x.com/2/users/:id/dm/unblock' --header 'Authorization: ••••••'
```

リクエストが成功すると、次のような JSON レスポンスが表示されます。

```
{
   "data": {
       "blocked": false
   }
}
```
