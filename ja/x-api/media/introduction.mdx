メディアオブジェクトは、単一の写真、動画、またはアニメーションGIFを表します。メディアオブジェクトは X API の多くのエンドポイントで使用され、ポスト、ダイレクトメッセージ、ユーザープロフィール、広告クリエイティブなどに含めることができます。各メディアオブジェクトには、解像度やフォーマットが異なる複数の表示用または再生用バリエーションが存在する場合があります。

<div id="media-types-size-restrictions">
  ## メディアの種類とサイズ制限
</div>

API を介してアップロードする場合のサイズ上限

* **Image**: `5 MB`
* **GIF**: `15 MB`
* **Video**: `512 MB`（`media_category=amplify_video` を使用する場合）

<div id="creation">
  ## 作成
</div>

ポスト、ダイレクトメッセージ、ユーザープロフィール画像、ホストされた広告カードなどのオブジェクトには、1 つ以上のメディアオブジェクトを含めることができます。これらのトップレベルオブジェクトは、まとめてエンティティと呼ばれます。関連するエンティティ作成用 API（例: [`POST /2/tweets`](/ja/x-api/posts/creation-of-a-post)）には、一意の `media_id` を使用して 1 つ以上のメディアオブジェクトを渡すことができます。

メディアオブジェクトを含むエンティティは、次の手順で作成できます。

1. 推奨される [chunked](/ja/x-api-/media/quickstart/media-upload-chunked) アップロード（画像/GIF/動画）または、従来の [simple](/ja/x-api/media-upload/media-upload) アップロード（画像のみ）のいずれかを使用してメディアファイルをアップロードします。
2. 手順 1 で `media_id` を取得します。エンティティが複数の `media_id` パラメータを受け取れる場合は、この手順を別のメディアで複数回繰り返すことができます。
3. `media_id` とその他の必須パラメータを含めて、適切なエンドポイントを呼び出し、エンティティを作成します。たとえば、[`POST /2/tweets`](/ja/x-api/posts/creation-of-a-post) エンドポイントを使用して、ポストに `media_id` を添付します。
