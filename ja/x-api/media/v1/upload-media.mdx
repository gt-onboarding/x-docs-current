<div id="overview">
  ## 概要
</div>

メディアオブジェクトは、1つの写真、動画、またはアニメーション GIF を表します。メディアオブジェクトは Twitter API 内の多くのエンドポイントで利用されており、ポスト、ダイレクトメッセージ、ユーザープロフィール、広告クリエイティブなどに含めることができます。各メディアオブジェクトには、解像度や形式の異なる複数の表示または再生バリエーションが存在する場合があります。

<div id="media-types-size-restrictions">
  ## メディアの種類とサイズ制限
</div>

API を経由してアップロードする場合のサイズ制限

- 画像: 5MB
- GIF: 15MB
- 動画: 512MB（media_category=amplify を使用する場合）

<div id="creation">
  ## 作成
</div>

ポスト、ダイレクトメッセージ、ユーザープロフィール画像、ホストされた広告カードなどのオブジェクトには、1 つ以上のメディアオブジェクトを含めることができます。これらのトップレベルオブジェクトはまとめてエンティティと呼ばれます。該当するエンティティ作成用の API（例: POST statuses/update）には、一意の `media_id` を使用して 1 つ以上のメディアオブジェクトを渡すことができます。

メディアオブジェクトを含むエンティティは、次の手順に従って作成できます。

1. 推奨されるチャンク（分割）アップロード（画像/GIF/動画）か、従来のシンプルアップロード（画像のみ）のいずれかを使用して、メディアファイルをアップロードします。
2. 手順 1 で `media_id` を受け取ります。エンティティが複数の `media_id` パラメータを受け付ける場合、この手順は異なるメディアに対して複数回繰り返すことができます。
3. 適切なエンドポイントを呼び出し、`media_id` とその他の必須パラメータを含めてエンティティを作成します。たとえば、POST statuses/update エンドポイントを使用して `media_id` をポストに添付します。

  ## 取得
</div>

ポストのデータ辞書にある Media Object を参照してください。
