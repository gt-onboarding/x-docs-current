<div id="getting-started-with-the-list-lookup-endpoint">
  ## List lookup エンドポイントの利用を開始する
</div>

このクイックスタートガイドでは、Postman を使用して List lookup エンドポイントへの最初のリクエストを送信する方法を説明します。

さまざまなプログラミング言語でのサンプルコードを確認したい場合は、[X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) の GitHub リポジトリにアクセスしてください。

**注:** この例では、*List lookup by ID* エンドポイントへのリクエストを行いますが、このクイックスタートで学んだ内容は、他のルックアップリクエストにも応用できます。

<Note>
  ### 前提条件

  このガイドを完了するには、リクエストを認証するための[キーとトークン](/ja/resources/fundamentals/authentication)が必要です。これらのキーとトークンは、次の手順に従って生成できます。

  * [開発者アカウントにサインアップ](https://developer.x.com/en/apply-for-access)し、承認を受けてください。
  * Developer Portal で[プロジェクト](/ja/resources/fundamentals/projects)を作成し、関連付けられた[開発者アプリ](/ja/resources/fundamentals/developer-apps)を作成します。
  * アプリの “Keys and tokens” ページに移動し、必要な認証情報を生成します。すべての認証情報は安全な場所に保存してください。
</Note>

<div id="steps-to-build-a-list-lookup-request">
  ### List ルックアップリクエストを構築する手順
</div>

**ステップ1: ツールまたはライブラリを用意する**

このエンドポイントへのリクエストを行うために使用できるツール、コード例、ライブラリはいくつかありますが、ここでは手順を簡素化するために Postman ツールを使用します。

X API v2 の Postman コレクションを環境に読み込むには、次のボタンをクリックしてください。

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2 を Postman に追加
</Button>

Postman に X API v2 コレクションを読み込んだら、「List」フォルダに移動し、その中の「List lookup」フォルダを選択し、&quot;List by ID&quot; を選択します。
 

**ステップ2: リクエストを認証する**

X API へ正しくリクエストするには、自分に権限があることを確認する必要があります。このエンドポイントでそれを行うには、[App only](/ja/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)、[OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)、または [OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) のいずれかの認証方法でリクエストを認証しなければなりません。

簡単にするため、このリクエストでは App only を使用しますが、非公開の[メトリクス](/ja/x-api/fundamentals/metrics) や List を取得したい場合は、他のいずれかの認証方法を使用する必要があります。 

App only を利用するには、Postman の右上にある “X API v2” という環境を選択し、その環境の「initial value」と「current value」フィールド（環境ドロップダウンの横にある目のアイコンをクリック）に、キーとトークン（特に [App only Access Token](/ja/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)）を追加する必要があります。

正しく設定できていれば、これらの変数はリクエストの「Authorization」タブに自動的に取り込まれます。
 

**ステップ3: 取得したい List を特定して指定する**

リクエストで、取得したい List を指定する必要があります。x.com で対象の List を開き、URL を確認することで List ID を見つけることができます。たとえば、次の URL の List ID は 84839422 です。

https://x.com/i/lists/84839422

対象 ID には、有効な任意の List ID を指定できます。Postman で &quot;Params&quot; タブを開き、id パス変数の &quot;Value&quot; 列に ID を入力します。ID の前後にスペースが入らないよう注意してください。

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| id  | 84839422 （List ID） |

**ステップ4: 取得したいフィールドを特定して指定する**

ステップ3の後に &quot;Send&quot; ボタンをクリックすると、レスポンスには既定の [List object](/ja/x-api/fundamentals/data-dictionary#list) のフィールドである id と name が含まれます。

追加のフィールドを取得したい場合は、リクエストで list.fields および/または [expansions](/ja/x-api/fundamentals/expansions) パラメータを使って、それらのフィールドを指定する必要があります。

この演習では、異なるオブジェクトから 3 つの追加のフィールドセットをリクエストします。

* プライマリの Lists オブジェクトにおける追加の created&#95;at フィールド

* expansion パラメータを使った完全な [user object](/ja/x-api/fundamentals/data-dictionary#user)

* 関連する user オブジェクトにおける追加の user.created&#95;at フィールド

Postman で &quot;Params&quot; タブに移動し、「Query Params」テーブルに次のキーと値のペアを追加します。

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| list.fields | created&#95;at | created&#95;at |
| expansions | owner&#95;id | includes.users.id,  <br />includes.users.name,  <br />includes.users.username |
| user.fields | created&#95;at | includes.users.created&#95;at |

「Send」ボタンの横に、次のような URL が表示されているはずです。

`https://api.x.com/2/lists/84839422?list.fields=owner_id&expansions=owner_id&user.fields=created_at`

**ステップ5: リクエストを送信し、レスポンスを確認する**

すべての設定が完了したら &quot;Send&quot; ボタンを押すと、次の例と同様のレスポンスを受け取ることができます。

```
{
  "data": {
    "id": "84839422",
    "name": "Official Twitter Accounts",
    "owner_id": "783214"
  },
  "includes": {
    "users": [
      {
        "name": "Twitter",
        "created_at": "2007-02-20T14:35:54.000Z",
        "username": "Twitter",
        "id": "783214"
      }
    ]
  }
}
```
