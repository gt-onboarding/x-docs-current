<div id="getting-started-with-the-list-posts-lookup-endpoint">
  ## List ポストルックアップエンドポイントの使用を開始する
</div>

このクイックスタートガイドでは、Postman を使用して List ポストルックアップエンドポイントへの最初のリクエストを送信する手順を説明します。

さまざまな言語でのサンプルコードを確認したい場合は、[X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) の GitHub リポジトリをご覧ください。

<Note>
  ### 前提条件

  このガイドを完了するには、リクエストの認証に使用する一連の [keys and tokens](/ja/resources/fundamentals/authentication) が必要です。これらのキーとトークンは、次の手順に従って生成できます。

  * [開発者アカウントに登録](https://developer.x.com/en/apply-for-access) し、承認を受けます。
  * [プロジェクト](/ja/resources/fundamentals/projects) を作成し、Developer Portal 内で関連付けられた [開発者アプリ](/ja/resources/fundamentals/developer-apps) を作成します。
  * アプリの「Keys and tokens」ページに移動して、必要な認証情報を生成します。すべての認証情報を安全な場所に保存してください。
</Note>

<div id="steps-to-build-a-list-posts-lookup-request">
  ### List Posts ルックアップリクエストの作成手順
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### ステップ1：ツールまたはライブラリから始める
</div>

このエンドポイントへのリクエストを行うために使用できるツールやコード例、ライブラリはいくつかありますが、ここでは手順を簡略化するために Postman を使用します。

X API v2 の Postman コレクションを環境に読み込むには、次のボタンをクリックしてください。

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2 を Postman に追加
</Button>

Postman に X API v2 コレクションが読み込まれたら、「List」フォルダに移動し、その中の「List Posts」フォルダを選択してから、「List Posts lookup」を選択します。
 

<div id="step-two-authenticate-your-request">
  #### ステップ 2: リクエストを認証する
</div>

X API に正しくリクエストするには、アクセス権限があることを確認する必要があります。このエンドポイントでそれを行うには、リクエストを [App only](/ja/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)、[OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)、または [OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) のいずれかの認証方式で認証する必要があります。

ここでは説明を簡単にするため、このリクエストには App only を使用しますが、非公開の [メトリクス](/ja/x-api/fundamentals/metrics) やポストをリクエストしたい場合は、他のいずれかの認証方式を使用する必要があります。 

App only を利用するには、Postman の右上にある環境ドロップダウンから「X API v2」という名前の環境を選択し、環境ドロップダウンの横にある目のアイコンをクリックして表示される「initial value」と「current value」のフィールドに、キーとトークン（具体的には [App only Access Token](/ja/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)、別名 App only Bearer Token）を追加する必要があります。

これが正しく設定されていれば、これらの変数はリクエストの Authorization タブに自動的に反映されます。
 

<div id="step-three-identify-and-specify-which-list-you-would-like-to-retrieve-posts-from">
  #### ステップ3：ポストを取得したいリストを特定して指定する
</div>

リクエスト内で、ポストを取得したいリストを指定する必要があります。リストの ID は、x.com にアクセスして任意のリストをクリックし、その後 URL を確認することで確認できます。たとえば、次の URL のリスト ID は 84839422 です。

https://x.com/i/lists/84839422

対象の ID には、有効なリスト ID であればどれでも指定できます。Postman で &quot;Params&quot; タブを開き、id パス変数の &quot;Value&quot; 列に対象の ID を入力します。どの ID についても、前後にスペースを含めないようにしてください。

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| id  | 84839422 (リスト ID) |

<div id="step-four-identify-and-specify-which-fields-you-would-like-to-retrieve">
  #### ステップ4：取得したいフィールドを特定して指定する
</div>

ステップ3の後に「Send」ボタンをクリックすると、レスポンス内にはデフォルトの [Post object](/ja/x-api/fundamentals/data-dictionary#tweet) のフィールドである id と text が返されます。

追加のフィールドを取得したい場合は、リクエスト内で tweet.fields および／または [expansion](/ja/x-api/fundamentals/expansions) パラメータを使用して、それらのフィールドを指定する必要があります。

この演習では、異なるオブジェクトから次の 3 つの追加フィールドセットをリクエストします。

* プライマリの Lists オブジェクトにおける追加の created&#95;at フィールド

* expansion パラメータを使用した完全な [user object](/ja/x-api/fundamentals/data-dictionary#user)

* 関連する user オブジェクトにおける追加の user.created&#95;at フィールド

Postman で「Params」タブに移動し、「Query Params」テーブルに次のキーと値のペアを追加します。

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **返されるフィールド** |
| tweet.fields | created&#95;at | created&#95;at |
| expansions | author&#95;id | includes.users.id,  <br />includes.users.name,  <br />includes.users.username |
| user.fields | created&#95;at | includes.users.created&#95;at |

これで、「Send」ボタンの横に次のような URL が表示されるはずです。

`https://api.x.com/2/lists/84839422/tweets?expansions=author_id&user.fields=created_at&max_results=1`

<div id="step-five-make-your-request-and-review-your-response">
  #### ステップ5：リクエストを送信してレスポンスを確認する
</div>

すべての設定が完了したら、「Send」ボタンをクリックすると、次の例のようなレスポンスが返ってきます。

```
{
  "data": [
    {
      "author_id": "4172587277",
      "id": "1458172421115101189",
      "text": "A Alemanha registrou nesta semana um recorde de novos casos de Covid-19. Segundo o governo e especialistas em Saúde, pessoas não vacinadas são responsáveis pela situação \nhttps://t.co/4POyaPwMLu"
    }
  ],
  "includes": {
    "users": [
      {
        "username": "MomentsBrasil",
        "name": "Twitter Moments Brasil",
        "created_at": "2015-11-12T16:46:02.000Z",
        "id": "4172587277"
      }
    ]
  },
  "meta": {
    "result_count": 1,
    "next_token": "7140dibdnow9c7btw3z2vwioavpvutgzrzm9icis4ndix"
  }
}
```

**ご注意ください：** このエンドポイントでは、指定したリストに対して最新のポストを最大800件まで取得できます
