<div id="manage-lists-standard-v11-compared-to-x-api-v2">
  ### リストの管理: 標準 v1.1 と X API v2 の比較
</div>

標準 v1.1 の [POST lists/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-create)、[POST lists/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-destroy)、[POST lists/update](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-update) エンドポイントを利用している場合、このガイドは、標準 v1.1 と X API v2 のリスト管理エンドポイントの類似点と相違点を理解するための手助けをすることを目的としています。

* **類似点**
  * 認証
* **相違点**
  * エンドポイント URL
  * アプリおよびプロジェクトに関する要件
  * HTTP メソッド
  * レート制限
  * リクエストパラメータ

<div id="similarities">
  #### 類似点
</div>

**認証**

両バージョンのエンドポイントは [OAuth 1.0a User Context](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication) をサポートします。そのため、以前に standard v1.1 の manage Lists エンドポイントのいずれかを使用していた場合は、X API v2 バージョンのエンドポイントへ移行しても、同じ認証方法を引き続き使用できます。

<div id="differences">
  #### 違い
</div>

**エンドポイント URL**

* Standard v1.1 エンドポイント:
  * POST https://api.x.com/1.1/lists/create.json
    (リストを作成します)
  * POST https://api.x.com/1.1/lists/destroy.json
    (リストを削除します)
  * POST https://api.x.com/1.1/lists/update.json
    (リストを更新します)
* X API v2 エンドポイント:
  * POST https://api.x.com/2/lists
    (リストを作成します)

  * DELETE https://api.x.com/2/lists/:id
    (リストを削除します)

  * PUT https://api.x.com/2/lists/:id
    (リストを更新します)

**レート制限**

| **Standard v1.1** | **X API v2** |
| :--- | :--- |
| /1.1/lists/create.json<br /><br />なし | /2/lists<br /><br />OAuth 1.0a User Context を使用した 15 分間あたり 300 リクエスト |
| /1.1/lists/destroy.json<br /><br />なし | /2/lists/:id<br /><br />OAuth 1.0a User Context を使用した 15 分間あたり 300 リクエスト |
| /1.1/lists/update.json<br /><br />なし | /2/lists/:id<br /><br />OAuth 1.0a User Context を使用した 15 分間あたり 300 リクエスト |

**アプリとプロジェクトの要件**

X API v2 エンドポイントでは、リクエストを認証する際に、[developer App](/ja/resources/fundamentals/developer-apps) に紐づけられた [Project](/ja/resources/fundamentals/projects) の認証情報を使用する必要があります。すべての X API v1.1 エンドポイントでは、スタンドアロンのアプリ、またはプロジェクトに関連付けられたアプリのいずれの認証情報も使用できます。

**リクエストパラメータ**

次の Standard v1.1 リクエストパラメータには、X API v2 での対応パラメータがあります。

**リストの作成**

| **Standard** | **X API v2** |
| :--- | :--- |
| name | name |
| mode | private |
| description | description |

**リストの削除/更新**

| **Standard** | **X API v2** |
| :--- | :--- |
| owner&#95;screen&#95;name | 対応するものはありません |
| owner&#95;id | 対応するものはありません |
| list&#95;id | id  |
| slug | 対応するものはありません |

<Note>
  **注意:** Standard v1.1 のパラメータはクエリパラメータとして渡されますが、X API v2 のパラメータは、POST エンドポイントではボディパラメータとして、DELETE および PUT エンドポイントではパスパラメータとして渡されます。
</Note>
