[X Lists](https://help.x.com/en/using-twitter/twitter-lists) を使用すると、ユーザーはタイムラインに表示されるポストをカスタマイズし、整理し、優先順位を付けることができます。Lists エンドポイントを使用すると、ユーザーの好み、関心、グループ、トピックに基づいてポストをキュレートおよび整理できるソリューションを構築できます。

manage List エンドポイントを使ってユーザーを代理してリクエストを行うため、これらのエンドポイントは必ず [OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) のいずれかで認証し、アプリを承認したユーザーに紐づくユーザーのアクセストークンを使用する必要があります。これらのアクセストークンは、[3-legged OAuth flow](/ja/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（OAuth 1.0a）または [Authorization Code with PKCE grant flow](/ja/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)（OAuth 2.0）を使用して生成できます。

<div id="manage-lists">
  ### リストを管理する
</div>

リスト管理エンドポイントを使用すると、認証済みユーザーに代わってリストを作成、削除、更新できます。これらのエンドポイントでは、POST、DELETE、PUT の 3 つのメソッドが利用可能です。POST メソッドではリストを作成でき、DELETE メソッドではリストを削除でき、PUT メソッドではリストのメタデータを更新できます。これら 3 つのエンドポイントはいずれも、ユーザー単位で 15 分あたり 300 リクエストのレート制限があります。

1 つのアカウントにつき、最大 1000 個のリストを作成できることに注意してください。

**アカウントのセットアップ**

これらのエンドポイントにアクセスするには、以下が必要です。

* 承認済みの [開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)。
* [プロジェクト](/ja/resources/fundamentals/projects) 内にある [開発者アプリ](/ja/resources/fundamentals/developer-apps) のキーとトークンを使用して認証すること。

X API v2 エンドポイントへのアクセス方法の詳細については、[はじめにガイド](/ja/x-api/getting-started/getting-access) を参照してください。
