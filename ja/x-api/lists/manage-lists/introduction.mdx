[X Lists](https://help.x.com/en/using-twitter/twitter-lists) を使用すると、ユーザーはタイムラインに表示されるポストをカスタマイズし、整理し、優先順位を付けることができます。Lists エンドポイントを使用することで、人々が自分の好み、関心、グループ、またはトピックに基づいてポストをキュレーションおよび整理できるソリューションを構築できます。

Lists の管理用エンドポイントでユーザーに代わってリクエストを行う場合、これらのエンドポイントは [OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) で認証し、あなたのアプリを承認したユーザーに紐づくユーザーのアクセストークンを使用する必要があります。これらのアクセストークンは、[3-legged OAuth フロー](/ja/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（OAuth 1.0a）または [Authorization Code with PKCE grant flow](/ja/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)（OAuth 2.0）を使用して生成できます。

<div id="manage-lists">
  ### リストを管理
</div>

リスト管理エンドポイントを使用すると、認証済みユーザーに代わってリストを作成、削除、更新できます。これらのエンドポイントでは、POST、DELETE、PUT の 3 つのメソッドが利用できます。POST メソッドではリストを作成でき、DELETE メソッドではリストを削除でき、PUT メソッドではリストのメタデータを更新できます。3 つのエンドポイントすべてに対して、ユーザー単位で 15 分あたり 300 リクエストのレート制限が適用されます。

1 つのアカウントにつき、最大 1000 個までリストを作成できます。

**アカウントのセットアップ**

これらのエンドポイントにアクセスするには、次のものが必要です。

* 承認済みの [開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)。
* [プロジェクト](/ja/resources/fundamentals/projects) 内にある [開発者アプリ](/ja/resources/fundamentals/developer-apps) のキーとトークンを使用した認証。

X API v2 エンドポイントへのアクセス方法については、[はじめにガイド](/ja/x-api/getting-started/getting-access)をご覧ください。
