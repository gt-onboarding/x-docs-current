<div id="getting-started-with-the-manage-lists-endpoint-group">
  ## List 管理エンドポイントグループの利用開始
</div>

このクイックスタートでは、[Postman](/ja/tutorials/postman-getting-started) を使用して List 管理エンドポイントへの最初のリクエストを送信する方法を説明します。

さまざまなプログラミング言語でのサンプルコードを確認したい場合は、GitHub リポジトリ [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) をご覧ください。 

**注:** この例では、*Create a List* エンドポイントへのリクエストを行いますが、このクイックスタートで学んだ内容は、他の List 管理リクエストにも適用できます。

<Note>
  **前提条件**

  このガイドを完了するには、リクエストを認証するための一連の [キーとトークン](/ja/resources/fundamentals/authentication) が必要です。これらのキーとトークンは、次の手順に従って生成できます。

  * [開発者アカウントに登録](https://developer.x.com/en/apply-for-access) し、承認を受けます。
  * 開発者ポータルで [プロジェクト](/ja/resources/fundamentals/projects) と、それに関連付けられた [開発者アプリ](/ja/resources/fundamentals/developer-apps) を作成します。
  * アプリの「Keys and tokens」ページに移動して、必要な認証情報を生成します。すべての認証情報は安全な場所に保存してください。
</Note>

<div id="steps-to-build-a-manage-list-request">
  ### リスト管理リクエストを作成する手順
</div>

**ステップ 1：ツールまたはライブラリを選択する**

このエンドポイントへのリクエストを行うために使用できるツール、コード例、ライブラリはいくつかありますが、ここでは手順を簡略化するために Postman を使用します。

X API v2 の Postman コレクションを環境に読み込むには、次のボタンをクリックしてください。

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2 を Postman に追加
</Button>

Postman に X API v2 コレクションを読み込んだら、「List」フォルダーに移動し、その中の「Manage List」フォルダーを選択し、「Create a List」を選択します。

**ステップ 2：リクエストを認証する**

X API に正しくリクエストを行うには、その権限があることを確認する必要があります。manage Lists エンドポイントでこれを行うには、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) のいずれかを使用してリクエストを認証する必要があります。

この例では、OAuth 1.0a User Context を使用します。

API Key、API Secret Key、OAuth 1.0a ユーザー Access Token、OAuth 1.0a ユーザー Access Token Secret などのキーとトークンを Postman に追加する必要があります。これを行うには、Postman の右上にある「X API v2」という名前の environment を選択し、その environment ドロップダウンの横にある目のアイコンをクリックして、「initial value」と「current value」フィールドにキーとトークンを追加します。

正しく設定されていれば、これらの変数はリクエストの Authorization タブに自動的に取り込まれます。
 
**ステップ 3：新しいリストの名前を指定する**

このエンドポイントで新しいリストを作成する場合、リスト名は必須のボディパラメーターです。任意で、説明を追加したり、リストを非公開にするかどうかを指定したりできます。

Postman で「Body」タブに移動し、name パラメーターの値としてリストの名前を入力します。さらに、リストに説明を追加したい場合は、name と同じ要領で description というラベルの新しいキーを追加し、その値としてリストの説明を入力します。リストを非公開にする場合も同様のパターンに従いますが、このパラメーターには true か false の値のみが指定できます。 

|             |                                     |                    |
| :---------- | :---------------------------------- | :----------------- |
| **Key**     | **Value**                           | **Parameter type** |
| `name`      | Name of the list (required)         | body               |
| description | Description for the list (optional) | body               |
| private     | true or false (optional)            | body               |

ここまで完了すると、「Send」ボタンの横に次のような URL が表示されます。

`https://api.x.com/2/lists`

ステップ 4：リクエストを送信し、レスポンスを確認する

すべての設定が完了したら、「Send」ボタンを押すと、次のサンプルレスポンスと同様のレスポンスが返されます。

```
{
  "data": {
    "id": "1441162269824405510",
    "name": "New list created from Postman"
  }
}
```

返されたレスポンスオブジェクトに `id` とリスト名が含まれていれば、リストは正常に作成されたことになります。 

リストを削除するには、Postman に読み込まれている X API v2 コレクションの「Lists」フォルダー内にある「Delete a List」リクエストを選択します。このエンドポイントでは、削除したいリストの ID が必要です。「Params」タブで、削除したいリストの ID を `id` 列の値として入力します。 

削除リクエストが成功すると、次の例のようなレスポンスが返されます。

```
{
  "data": {
    "deleted": true
  }
}
```
