リストをピン留めすると、X ユーザーは自分が購読しているリストや自分で作成したリストに、より簡単にアクセスできるようになります。

これらのエンドポイントに対してユーザーの代わりにリクエストを送信する場合は、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) のいずれかでそのユーザーを認証し、あなたのアプリを承認したユーザーに紐づくユーザーアクセストークンを使用する必要があります。これらのアクセストークンは、[3-legged OAuth フロー](/ja/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（OAuth 1.0a）または [Authorization Code with PKCE grant flow](/ja/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)（OAuth 2.0）を使って生成できます。

<div id="pinned-list-lookup">
  ### ピン留めされたリストのルックアップ
</div>

ピン留めされたリストのルックアップには、認証済みユーザーがピン留めしたリストを取得できるエンドポイントが 1 つあります。このエンドポイントのレート制限は、15 分あたり 15 リクエストです。
 

<div id="manage-pinned-lists">
  ### ピン留めリストを管理する
</div>

ピン留めリスト管理エンドポイントを使用すると、認証済みユーザーに代わってリストをピン留めおよびピン留め解除できます。これらのエンドポイントでは、POST と DELETE の 2 つのメソッドが利用できます。POST メソッドではリストをピン留めでき、DELETE メソッドではリストのピン留めを解除できます。どちらのエンドポイントも、ユーザーごとのレート制限は 15 分あたり 50 リクエストです。

<Note>
  **アカウントのセットアップ**

  これらのエンドポイントにアクセスするには、次が必要です。

  * 承認済みの [developer account](https://developer.x.com/en/portal/petition/essential/basic-info)。
  * [Project](/ja/resources/fundamentals/projects) 内にある [developer App](/ja/resources/fundamentals/developer-apps) のキーとトークンを使用して認証すること。 

  X API v2 エンドポイントへのアクセス方法の詳細については、[はじめに](/ja/x-api/getting-started/getting-access)をご覧ください。
</Note>
