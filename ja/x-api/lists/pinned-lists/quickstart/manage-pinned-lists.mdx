<div id="steps-to-build-a-manage-pinned-list-request">
  ### ピン留めされたリストを管理するリクエストを作成する手順
</div>

**ステップ 1: Postman で List エンドポイントコレクションを選択する**

Postman に X API v2 コレクションを読み込んだら、「List」フォルダーに移動し、その中の「Pinned Lists」フォルダーを選択してから、&quot;Pin a List&quot; を選びます。
 

**ステップ 2: ピン留めするリストを指定する**

ピン留めされたリストを管理するエンドポイントには 2 つの ID が必要です。1 つはユーザー（リストをピン留めする認証済みユーザー）の ID、もう 1 つは対象リスト（ピン留めされるリスト）の ID です。ユーザーの ID は認証しているユーザーの ID と一致している必要があり、そのため、リクエストの認証時には、そのユーザー ID に紐づくアクセストークンを渡さなければなりません。この場合は、自分のユーザーアカウントの ID を指定できます。自分の ID は次の 2 通りの方法で確認できます。

1. [users lookup](/ja/x-api/users/lookup/introduction) by username エンドポイントを使用して、ユーザー名を渡すと、`id` フィールドを受け取ることができます。
2. 自分のアクセストークンを確認すると、数値部分が自分のユーザー ID になっています。

対象リスト ID は、有効なリストであればどれでもかまいません。Postman で &quot;Params&quot; タブに移動し、id パス変数の &quot;Value&quot; 列にユーザー ID を入力します。次に &quot;Body&quot; タブに移動し、ピン留めしたいリストの ID を list&#95;id パラメーターの値として入力します。どの ID に対しても、前後にスペースを含めないように注意してください。

|             |               |                    |
| :---------- | :------------ | :----------------- |
| **Key**     | **Value**     | **Parameter type** |
| `id`        | ユーザー ID       | path               |
| list&#95;id | ピン留め対象のリスト ID | body               |

これで、&quot;Send&quot; ボタンの横に、これと似た URL が表示されるはずです。

```
      https://api.x.com/2/users/2244994945/pinned_lists
```

ステップ3: リクエストを送信し、レスポンスを確認する

すべての設定が完了したら、「Send」ボタンをクリックすると、次の例のようなレスポンスが返ってきます。

```
{
  "data": {
    "pinned": true
  }
}
```

レスポンスオブジェクトの pinned キーの値が true であれば、対象のリストが正常にピン留めされたことを意味します。 

リストのピン留めを解除するには、Postman に読み込まれている X API v2 コレクションの「Lists」フォルダー内にある「Unpin List」リクエストを選択します。このエンドポイントには、認証済みユーザーの ID と、ピン留めを解除するリストの ID が必要です。「Params」タブで、id 列の値としてユーザー ID を、list&#95;id 列の値としてピン留めを解除するリストの ID を入力します。 

削除リクエストが成功すると、次の例のようなレスポンスが返されます。

```
{
  "data": {
    "pinned": false
  }
}
```
