<div id="getting-started-with-the-pinned-list-endpoint-group">
  ## ピン留めリストエンドポイントグループの利用を開始する
</div>

この簡単な概要では、[Postman](/ja/tutorials/postman-getting-started) を使ってピン留めリストエンドポイントへの最初のリクエストを行う方法を説明します。

さまざまな言語でのサンプルコードを確認したい場合は、[X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) の GitHub リポジトリをご覧ください。

<Note>
  ### 前提条件

  このガイドを完了するには、リクエストを認証するための一連の [キーとトークン](/ja/resources/fundamentals/authentication) が必要です。これらのキーとトークンは、次の手順に従って生成できます。

  * [開発者アカウントに登録](https://developer.x.com/en/apply-for-access) し、承認を受けます。
  * Developer ポータルで [プロジェクト](/ja/resources/fundamentals/projects) と、それに関連付けられた [developer App](/ja/resources/fundamentals/developer-apps) を作成します。
  * アプリの「Keys and tokens」ページに移動して、必要な認証情報を生成します。すべての認証情報は、必ず安全な場所に保存してください。
</Note>

X API v2 Postman コレクションを読み込む

X API v2 Postman コレクションをワークスペースに読み込むには、次のボタンをクリックしてください。

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2 を Postman に追加
</Button>

 

<div id="authenticate-your-request">
  #### リクエストの認証
</div>

**lookup** エンドポイントに正常にリクエストを送信するには、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication)、[OAuth 2.0 App-Only](/ja/resources/fundamentals/authentication#oauth-2-0)、または [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) のいずれかを使用できます。ただし、**manage** エンドポイントでは、OAuth 1.0a User Context か OAuth 2.0 Authorization Code with PKCE のみで認証できます。

いずれの場合でも、Postman を使用しているときは、環境を正しく設定していれば、既定の認証用キーとトークンがリクエストに自動的に入力されます。

これを行うには、Postman 右上で “X API v2” という名前の環境を選択し、環境ドロップダウンの横にある目のアイコンをクリックして &quot;initial value&quot; および &quot;current value&quot; フィールドにキーとトークンを追加します。これらのキーには次のものが含まれます。

* API Key (Consumer Key とも呼ばれます)
* API Secret Key (Consumer Secret とも呼ばれます)
* OAuth 1.0a user Access Token
* OAuth 1.0a user Access Token Secret
* OAuth 2.0 App Access Token
* OAuth 2.0 Client Key (アプリの設定で OAuth 2.0 User Authentication の設定を行っている場合にのみ利用できます)
* OAuth 2.0 Client Secret (アプリの設定で OAuth 2.0 User Authentication の設定を行っている場合にのみ利用できます)
