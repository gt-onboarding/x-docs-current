<div id="getting-started-with-the-pinned-list-endpoint-group">
  ## ピン留めリストエンドポイントグループの概要
</div>

このクイックガイドでは、[Postman](/ja/tutorials/postman-getting-started) を使ってピン留めリストエンドポイントへの最初のリクエストを送信する手順を説明します。

さまざまな言語でのサンプルコードを確認したい場合は、[X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) の GitHub リポジトリをご覧ください。

<Note>
  ### 前提条件

  このガイドを完了するには、リクエストを認証するための一連の [キーとトークン](/ja/resources/fundamentals/authentication) が必要です。これらのキーとトークンは、次の手順に従って生成できます。

  * [開発者アカウントに登録](https://developer.x.com/en/apply-for-access) し、承認を受けてください。
  * developer portal 上で [プロジェクト](/ja/resources/fundamentals/projects) と、それに関連付けられた [開発者アプリ](/ja/resources/fundamentals/developer-apps) を作成してください。
  * アプリの「Keys and tokens」ページに移動し、必要な認証情報を生成してください。すべての認証情報は安全な場所に保存してください。
</Note>

X API v2 Postman コレクションを読み込む

X API v2 Postman コレクションをワークスペースに読み込むには、次のボタンをクリックしてください。

<div id="authenticate-your-request">
  #### リクエストを認証する
</div>

**lookup** エンドポイントに対してリクエストを正常に送信するには、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication)、[OAuth 2.0 App-Only](/ja/resources/fundamentals/authentication#oauth-2-0)、または [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) のいずれかを使用できます。一方、**manage** エンドポイントでは、OAuth 1.0a User Context か OAuth 2.0 Authorization Code with PKCE のみを使用して認証できます。

いずれの場合も、Postman を使用しているときは、環境を正しく設定していれば、デフォルトの認証用のキーとトークンが自動的にリクエストに反映されます。

これを行うには、Postman の右上にある環境ドロップダウンから「X API v2」という名前の環境を選択し、環境ドロップダウン横の目のアイコンをクリックして、「initial value」と「current value」のフィールドにキーとトークンを追加します。これらのキーには以下が含まれます。

* API Key（Consumer Key とも呼ばれます）
* API Secret Key（Consumer Secret とも呼ばれます）
* OAuth 1.0a user Access Token
* OAuth 1.0a user Access Token Secret
* OAuth 2.0 App Access Token
* OAuth 2.0 Client Key（アプリの設定で OAuth 2.0 User Authentication 設定を有効にしている場合にのみ使用可能）
* OAuth 2.0 Client Secret（アプリの設定で OAuth 2.0 User Authentication 設定を有効にしている場合にのみ使用可能）
