<div id="getting-started-with-the-list-members-endpoint-group">
  ## List メンバー endpoint グループの利用を開始する
</div>

このクイックガイドでは、[Postman](/ja/tutorials/postman-getting-started) を使用して List メンバー endpoint グループのエンドポイントに最初のリクエストを送信するまでの手順を説明します。

さまざまな言語のサンプルコードを確認したい場合は、[X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub リポジトリをご覧ください。 

<Note>
  **前提条件**

  このガイドを完了するには、リクエストを認証するための一式の [キーとトークン](/ja/resources/fundamentals/authentication) が必要です。これらのキーとトークンは、次の手順に従って生成できます。

  * [開発者アカウントにサインアップ](https://developer.x.com/en/apply-for-access) し、承認を取得します。
  * 開発者ポータルで [Project](/ja/resources/fundamentals/projects) を作成し、関連付けられた [developer App](/ja/resources/fundamentals/developer-apps) を作成します。
  * アプリの「Keys and tokens」ページに移動して、必要な認証情報を生成します。すべての認証情報は安全な場所に保存してください。
</Note>

<div id="load-the-x-api-v2-postman-collection">
  #### X API v2 の Postman コレクションを読み込む
</div>

X API v2 の Postman コレクションをワークスペースに読み込むには、以下のボタンをクリックしてください。

<div id="authenticate-your-request">
  #### リクエストを認証する
</div>

**lookup** エンドポイントへ正常にリクエストを送信するには、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication#oauth-1-0a-2)、[App only](/ja/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)、または [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) のいずれかを使用できます。ただし、**manage** エンドポイントでは、OAuth 1.0a User Context か OAuth 2.0 Authorization Code with PKCE のみを使用して認証できます。

いずれの場合も、Postman を使用している場合は、環境を正しく設定していれば、デフォルトの認証用のキーとトークンが自動的にリクエストへ反映されます。

これを行うには、Postman 画面右上にある “X API v2” という名前の環境を選択し、環境ドロップダウンの横にある目のアイコンをクリックして、&quot;initial value&quot; と &quot;current value&quot; のフィールドにキーとトークンを追加します。これらのキーには次のものが含まれます。

* API Key (Consumer Key とも呼ばれます)
* API Secret Key (Consumer Secret とも呼ばれます)
* OAuth 1.0a user Access Token
* OAuth 1.0a user Access Token Secret
* App only Access Token
* OAuth 2.0 Client Key (App の設定で OAuth 2.0 User Authentication を設定している場合にのみ利用可能)
* OAuth 2.0 Client Secret (App の設定で OAuth 2.0 User Authentication を設定している場合にのみ利用可能)
