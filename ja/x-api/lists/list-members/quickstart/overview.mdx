<div id="getting-started-with-the-list-members-endpoint-group">
  ## List メンバーエンドポイントグループの利用を開始する
</div>

このクイックガイドでは、[Postman](/ja/tutorials/postman-getting-started) を使って List メンバーエンドポイントへの最初のリクエストを送信する方法を説明します。

さまざまな言語でのサンプルコードをご覧になりたい場合は、[X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) の GitHub リポジトリをご確認ください。 

<Note>
  **前提条件**

  このガイドを完了するには、リクエストを認証するための [キーとトークン](/ja/resources/fundamentals/authentication) 一式が必要です。これらのキーとトークンは、次の手順に従って生成できます。

  * [開発者アカウントに登録](https://developer.x.com/en/apply-for-access) し、承認を受けます。
  * developer portal 上で [Project](/ja/resources/fundamentals/projects) を作成し、関連付けられた [developer App](/ja/resources/fundamentals/developer-apps) を作成します。
  * アプリの「Keys and tokens」ページに移動して、必要な認証情報を生成します。すべての認証情報は安全な場所に保存してください。
</Note>

<div id="load-the-x-api-v2-postman-collection">
  #### X API v2 Postman コレクションを読み込む
</div>

X API v2 Postman コレクションをワークスペースに読み込むには、以下のボタンをクリックしてください。

</Button>

<div id="authenticate-your-request">
  #### リクエストを認証する
</div>

**lookup** エンドポイントに対してリクエストを正常に実行するには、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication#oauth-1-0a-2)、[App only](/ja/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)、または [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) のいずれかを使用できます。ただし、**manage** エンドポイントでは、OAuth 1.0a User Context か OAuth 2.0 Authorization Code with PKCE のみを使用して認証できます。

いずれの場合でも、Postman を使用している場合は、環境を正しく設定していれば、デフォルトの認証用のキーとトークンが自動的にリクエストに設定されます。 

これを行うには、Postman の右上にある「X API v2」という名前の環境を選択し、環境ドロップダウン横の目のアイコンをクリックして、「initial value」と「current value」のフィールドにキーとトークンを追加します。これらのキーには次のものが含まれます。

* API Key（Consumer Key とも呼ばれます）
* API Secret Key（Consumer Secret とも呼ばれます）
* OAuth 1.0a user Access Token
* OAuth 1.0a user Access Token Secret
* App only Access Token
* OAuth 2.0 Client Key（App の設定で OAuth 2.0 User Authentication 設定を行っている場合にのみ利用可能）
* OAuth 2.0 Client Secret（App の設定で OAuth 2.0 User Authentication 設定を行っている場合にのみ利用可能）
