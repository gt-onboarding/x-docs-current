<div id="steps-to-build-a-manage-list-member-request">
  ### List メンバー管理用リクエストを作成する手順
</div>

<div id="step-one-choose-the-list-endpoint-collection-from-postman">
  #### ステップ1: Postman で List エンドポイントコレクションを選択する
</div>

Postman に X API v2 コレクションを読み込んだら、「List」フォルダーに移動し、その中の「List members」フォルダーを開いてから「Add a member」を選択します。
 

<div id="step-two-specify-the-user-to-add">
  #### Step two: Specify the user to add
</div>

List メンバー管理エンドポイントには 2 つの ID が必要です。1 つは List の ID、もう 1 つは追加するユーザーの ID です。[users lookup](/ja/x-api/users/lookup/introduction) を使ってユーザーの ID を取得できます。ユーザー名を指定すると、id フィールドを含むレスポンスを受け取れます。

ターゲット ID は、有効なユーザー ID であればどれでも構いません。Postman で「Params」タブに移動し、id パス変数の「Value」列に List の ID を入力します。次に「Body」タブに移動し、追加したいユーザーの ID を user&#95;id パラメーターの値として入力します。いずれの ID の前後にもスペースを入れないよう注意してください。

|             |                           |                    |
| :---------- | :------------------------ | :----------------- |
| **Key**     | **Value**                 | **Parameter type** |
| `id`        | リスト ID                    | path               |
| user&#95;id | メンバーとして追加したいターゲットユーザーの ID | body               |

これで、「Send」ボタンの横に次のような URL が表示されているはずです。

`https://api.x.com/2/lists/1441162269824405510/members`

Step three: Make your request and review your response

すべての設定が完了したら、「Send」ボタンをクリックすると、次の例と同様のレスポンスが返されます。

```
{
  "data": {
    "is_member": true
  }
}
```

返されたレスポンスオブジェクトのキー is&#95;member が true の場合、そのユーザーはリストのメンバーとして正常に追加されています。 

リストからメンバーを削除するには、Postman に読み込んだ X API v2 コレクションの「Lists」フォルダー内にある「Remove a member」リクエストを選択します。このエンドポイントには、リストの ID と、削除したいメンバーのユーザー ID が必要です。「Params」タブで、id 列の値としてリストの ID を入力し、user&#95;id 列の値として削除対象ユーザーの ID を入力します。 

DELETE リクエストが成功すると、次の例のようなレスポンスが返されます。

```
{
  "data": {
    "is_member": false
  }
}
```
