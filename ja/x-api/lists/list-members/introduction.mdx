<div id="list-members-lookup">
  ### リストメンバーの検索
</div>

メンバー検索グループには、2 つのエンドポイントがあります。指定したリストのメンバーの詳細を取得したり、特定のユーザーがメンバーになっているリストを確認したりできます。これらのエンドポイントを使用すると、メンバーシップ情報に基づいて新しいリストを作成・整理できます。

メンバーの詳細を検索する場合は 15 分あたり 900 リクエスト、ユーザーのメンバーシップを検索する場合は 15 分あたり 75 リクエストのレート制限が適用されます。

このエンドポイントへのリクエストを認証するには、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication)、[App only](https://developer.x.com/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)、または [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) を使用できます。 
 

<div id="manage-list-members">
  ### リストメンバーの管理
</div>

リストメンバーを管理するエンドポイントを使用すると、認証済みユーザーに代わってリストにメンバーを追加および削除できます。これらのエンドポイントには、POST と DELETE の 2 つのメソッドがあります。POST メソッドではリストにメンバーを追加でき、DELETE メソッドではリストからメンバーを削除できます。どちらのエンドポイントも、ユーザーごとのレート制限は 15 分あたり 300 リクエストです。

リストのメンバー数は 5,000 を超えることはできない点に注意してください。

これらのエンドポイントではユーザーに代わってリクエストを行うため、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) のいずれかでユーザーを認証し、あなたのアプリを承認したユーザーに紐づくユーザーのアクセストークンを使用する必要があります。アクセストークンは、[3-legged OAuth フロー](/ja/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（OAuth 1.0a）または [Authorization Code with PKCE grant flow](/ja/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)（OAuth 2.0）を使用して生成できます。

<Note>
  **アカウントのセットアップ**

  これらのエンドポイントにアクセスするには、以下が必要です。

  * 承認済みの[開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)。
  * [プロジェクト](/ja/resources/fundamentals/projects)内にある[開発者アプリ](/ja/resources/fundamentals/developer-apps)のキーとトークンを使用して認証すること。 

  X API v2 エンドポイントへのアクセス方法については、[はじめにガイド](/ja/x-api/getting-started/getting-access)をご覧ください。
</Note>
