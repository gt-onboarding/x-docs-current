<div id="list-members-lookup">
  ### リストメンバーのルックアップ
</div>

メンバーのルックアップグループには、利用可能なエンドポイントが 2 つあります。指定したリストのメンバーの詳細を取得したり、特定のユーザーがどのリストのメンバーであるかを確認したりできます。これらのエンドポイントを使用すると、メンバーシップ情報に基づいて、ユーザーが新しいリストを作成・整理できるようにできます。

メンバーの詳細をルックアップする場合は 15 分あたり 900 リクエスト、ユーザーのメンバーシップをルックアップする場合は 15 分あたり 75 リクエストのレート制限があります。

このエンドポイントへのリクエストを認証するには、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication)、[App only](https://developer.x.com/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)、または [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) を使用できます。 
 

<div id="manage-list-members">
  ### リストメンバーの管理
</div>

リストメンバー管理エンドポイントを使用すると、認証済みのユーザーに代わってリストにメンバーを追加および削除できます。これらのエンドポイントには、POST メソッドと DELETE メソッドの 2 種類があります。POST メソッドではリストにメンバーを追加でき、DELETE メソッドではリストからメンバーを削除できます。どちらのエンドポイントも、ユーザー単位で 15 分あたり 300 件のリクエストというレート制限があります。

なお、1 つのリストに含められるメンバーは最大 5,000 アカウントまでです。

これらのエンドポイントではユーザーに代わってリクエストを送信するため、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) のいずれかでユーザーを認証し、あなたのアプリを承認したユーザーに紐づくアクセストークンを使用する必要があります。これらのアクセストークンは、[3-legged OAuth フロー](/ja/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（OAuth 1.0a）または [Authorization Code with PKCE grant flow](/ja/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce))（OAuth 2.0）を使用して生成できます。

<Note>
  **アカウント設定**

  これらのエンドポイントにアクセスするには、次のものが必要です。

  * 承認済みの[開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)。
  * [プロジェクト](/ja/resources/fundamentals/projects)内にある[開発者アプリ](/ja/resources/fundamentals/developer-apps)のキーとトークンを使用して認証すること。

  X API v2 エンドポイントへのアクセスの取得については、[はじめにガイド](/ja/x-api/getting-started/getting-access)をご覧ください。
</Note>
