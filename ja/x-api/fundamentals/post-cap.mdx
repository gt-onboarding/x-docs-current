X API v2 には、特定のエンドポイントから 1 か月あたりに取得できるポスト数を制限する、ポスト消費上限があります。ポスト上限は[プロジェクト](/ja/resources/fundamentals/projects)単位で適用されます。つまり、そのプロジェクト内の[開発者アプリ](/ja/resources/fundamentals/developer-apps)のキーとトークンを使用して以下に記載したエンドポイントに送信されたリクエストは、すべてそのプロジェクトのポスト上限に対してカウントされます。

<div id="limited-v2-endpoints">
  ## 制限対象の v2 エンドポイント
</div>

以下のいずれかのエンドポイントから取得されたポストは、月間のポスト上限数にカウントされます。

* [Posts Lookup](/ja/x-api/posts/lookup/introduction)
* [Recent search](/ja/x-api/posts/counts/introduction)
* [Full-archive search](/ja/x-api/posts/counts/introduction)
* [Filtered stream](/ja/x-api/posts/filtered-stream)
* [Filtered Stream Webhooks](/ja/x-api/webhooks/stream/introduction)
* [User Post timeline](/ja/x-api/posts/timelines)
* [User mention timeline](/ja/x-api/posts/timelines)
* [Likes lookup - Posts liked by a user](/ja/x-api/posts/likes)
* [Bookmarks lookup - Posts bookmarked by a user](/ja/x-api/posts/bookmarks/introduction)
* [List Posts lookup](/ja/x-api/lists/list-tweets/introduction)
* [Spaces Lookup](/ja/x-api/spaces/lookup/introduction)

<Note>
  **注:** 同じポストが 1 日の間に複数のクエリから返された場合でも、そのポストが上限数にカウントされるのは 1 回だけです。ポストは日次で重複排除されます。
</Note>

<div id="post-cap-volumes">
  ## ポスト上限数
</div>

ポスト上限数は、アクセスレベルによって異なります。

| アクセスティア   | 月間ポスト上限                |
|:---------------|:---------------------------------|
| **Free**      | 月あたり100ポスト           |
| **Basic**     | 月あたり15,000ポスト          |
| **Pro**       | 月あたり1,000,000ポスト       |
| **Enterprise**| 月あたり5,000万ポスト以上     |

月間ポスト上限に対する自身の利用状況は、[developer portal](/ja/resources/fundamentals/developer-portal) の[メインダッシュボードページ](https://developer.x.com/content/developer-twitter/en/portal/dashboard)を表示して確認できます。プロジェクト名の下に、現在の月の利用状況とポスト上限との関係を示すステータスバーが表示され、取得済みポストの総数、使用率、および利用状況がリセットされる日付が確認できます。
