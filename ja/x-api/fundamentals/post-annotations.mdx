<div id="overview">
  ## 概要
</div>

すべての v2 エンドポイントのうち Post オブジェクトを返すものには、その Post オブジェクトにアノテーションが追加されています。Post アノテーションは、そのポスト自体に関するコンテキスト情報を把握する手段を提供します。ポストは 100% を対象にレビューされていますが、ポスト本文の内容により、実際にアノテーションが付与されるのは一部のみです。

1. **エンティティ・アノテーション（NER）:** エンティティには人、場所、製品、組織などが含まれ、`entity` ペイロードセクションで提供されます。これらはポスト本文内で明示的に言及されている内容（固有表現認識）に基づいて、プログラムによって自動的に割り当てられます。

2. **コンテキストアノテーション:** ポスト本文の分析に基づいて生成されるコンテキストアノテーションには、ドメインとエンティティの組み合わせが含まれ、これにより、これまで検出が難しかったトピックに関するポストを見つけやすくなります。現在、ポストを分類するために 80 以上のドメインを使用しています。利用可能なコンテキストアノテーションのエンティティの CSV ファイルは、[GitHub リポジトリ](https://github.com/xdevplatform/twitter-context-annotations) で提供しています。

<div id="post-annotation-types">
  ## ポストのアノテーションの種類
</div>

<div id="entities">
  ### エンティティ
</div>

エンティティ注釈は、`entities` フィールド内でプログラムによって定義されるエンティティであり、ペイロード内では注釈として反映されます。各注釈には信頼度スコアがあり、ポスト本文のどこでエンティティが特定されたか（`start` および `end` フィールドを使用）を示します。

エンティティ注釈の種類には次のものが含まれます。

* **Person** - 例: Barack Obama, Daniel, George W. Bush
* **Place** - 例: Detroit, Cali, San Francisco
* **Product** - 例: Mountain Dew, Mozilla Firefox
* **Organization** - 例: Chicago White Sox, IBM
* **Other** - 例: 糖尿病, 第50回スーパーボウル

<div id="context-annotations">
  ### コンテキスト注釈
</div>

以下の質問は、ポスト注釈のコンテキスト注釈要素に特有のものです。詳細については、[概要](/ja/x-api/fundamentals/post-annotations) ページを参照してください。

<AccordionGroup>
  <Accordion title="Twitter のコンテキストアノテーションはどのような仕組みですか？">
    Twitter では、ポストを意味情報に基づいて分類しています。これは、特定のトピックに関連するキーワード、ハッシュタグ、@ハンドルのリストをキュレーションしているということです。ポストに指定したテキストが含まれていれば、そのポストには適切なラベルが付与されます。これは、特定のテキスト（この場合はポスト）を分類するようにモデルを学習させ、出力／分類結果と併せて確率スコアを生成する機械学習アプローチとは異なります。
  </Accordion>

  <Accordion title="御社のデータが完全で信頼できるものだと、どうすれば分かりますか？">
    Twitterのアノテーションは、数年にわたって洗練されてきた調査およびQA（品質保証）プロセスを用いて、各分野の専門家によって作成・整備されています。このプロセスは、高い適合率と再現率を維持できる範囲でデータ追跡を最大限にスケールさせるためのカスタムツールによって支えられています。さらに、弊社のデータは社内チームによって定期的に監査されており、ここ数四半期にわたり約80%の適合率スコアを維持しています。
  </Accordion>

  <Accordion title="どのように精度を保証していますか？">
    チームメンバーは、高い適合率と再現率を維持するために、毎日エンティティのQAを行っています。さらに、当社の作業は四半期ごとに社内チームによる監査を受けており、そのチームがすべてのドメインから合計 10,000 件のポストを手作業でレビューし、適合率スコアを算出します。
  </Accordion>

  <Accordion title="何をトラッキングするかはどのように決めているのですか？">
    スポーツやテレビなど一部のドメインでは、グラフを構築するために自動取り込みに依存しています。ニュース分野では、Twitter Moments チームが公開したストーリーに関するデータを追跡しています。それ以外の分野については、チームがさまざまなリサーチ手法を用いて、プラットフォーム上で活発な議論が交わされているトピックを特定し、追跡対象としています。
  </Accordion>

  <Accordion title="Tweet Annotations では、過去データはどの範囲まで取得できますか？">
    エンティティが公開された瞬間からデータの追跡が開始されるため、特定のエンティティが追跡対象となる前に公開されたポストには注釈を付与しません。たとえば、新興ブランド／企業がタクソノミーに追加された場合、そのブランドに関するポストについて、注釈が追加される前のものを遡って注釈付けすることはありません。
  </Accordion>

  <Accordion title="Twitter は英語以外の言語で書かれたポストにもコンテキスト注釈を付与できますか。その場合、どの言語が対象で、注釈が付与されるポストのカバレッジ（割合）は変わりますか。">
    はい。言語のカバレッジは、ドメインや市場によって異なる場合があります。英語と日本語は、大部分の大規模エンティティでサポートされています。以下は、現在カバーされている言語と主な市場の一覧です。

  <Accordion title="Twitter はポストに注釈を付ける際、どのような基盤的な「セマンティクス」に基づいていますか？">
    ポスト注釈は、ポストに注釈を付ける際に用いる、次のようなセマンティクスで構成されます。

    * アカウント - 特定のハンドルから投稿されたポスト、またはそのハンドルへの言及を含むポストに注釈を付けることができます
    * ハッシュタグ
    * キーワード／フレーズ

    PowerTrack などのフィルタリング機能付きストリーミング API に慣れているお客様にとって、注釈で使用されるセマンティクスは、ポストのストリームをフィルタリングするために定義されるブール演算ルールと基本的な考え方は同じです。あるポストが基盤となるセマンティックな条件に一致した場合、そのポストには対応するタグが付与されます。
  </Accordion>

  <Accordion title="なぜ一部のポストにはエンティティが関連付けられているのに、ほかのポストには関連付けられていないのですか？">
    目標は、可能な限り多くのポストに注釈を付与することですが、いくつかの理由により注釈が付与されないポストもあります。

    * 一部のポストは意味的な情報量が十分でなく、現在の注釈ルールではラベル付けやタグ付けができない
    * 一部のポストは、特定のトピックに結びついていない
    * ポストが、当社のグラフに含まれていないごく一時的なトピックについてのもの
    * 対応していない言語／市場である
    * 言語／市場には対応しているが、すでに追跡しているトピックに関連するトピックや特定の用語／アカウント／ハッシュタグがカバー範囲に含まれていない
  </Accordion>

  <Accordion title="複数のドメイン（例：[3,30]）がある場合でも、Entity ID は同一のままですか？">
    1つのエンティティは複数のドメインに属することができます。ドメインIDは変わる可能性がありますが、エンティティのidは変わりません。Donald Glover は人物（ドメイン 10）、俳優（ドメイン 56）、音楽家（ドメイン 54）でもありますが、エンティティidは 875072662527029248 のままです。
  </Accordion>

  <Accordion title="番組や映画のトラッキング期間はあらかじめ決まっていますか？ つまり、番組／映画はリリースのどれくらい前からどれくらい後までトラッキングされますか？">
    トラッキングはリリースの1か月前から開始されます。Marvel映画のような人気の大作の場合、公開に向けたティザー展開が始まり次第、すぐにトラッキングを開始できます。
  </Accordion>

  <Accordion title="映画にも、テレビ番組の場合と同様のロケールフィルターはありますか？">
    いいえ、そのようなことはありません。
  </Accordion>
