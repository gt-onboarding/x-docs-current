<p>X API v2 とあわせて、新しいバージョニング戦略を導入しました。これにより、開発者は X のパブリック API にどのタイミングで変更が行われるのか、そしていつ新しいバージョンへの移行が必要になるのかを、より正確に把握できるようになります。 </p>
<p>開発者は、X API に対する非推奨化、提供終了、変更、追加について、<a href="https://developer.x.com/en/updates/stay-informed">コミュニケーションチャネル</a>を通じて通知を受けることができます。これにより、開発ロードマップの中で、これらの変更に対応するための計画を適切に立てることができます。API へのすべての変更は、変更履歴に記載されます。</p>
<p>現在、X API には 3 つの異なるバージョンがあります。お客様のユースケースで必要な機能がまだ v2 に提供されていない場合を除き、連携を計画する際には X API v2 をご利用いただくことを強く推奨します。 </p>
<p>各バージョンの詳細については、次のページをご覧ください。</p>

<ul>
  <li><a href="/ja/x-api/introduction">X API v2</a></li>
  <li>Gnip 2.0 (<a href="/ja/x-api/enterprise-gnip-2.0/enterprise-gnip">Enterprise</a>)</li>
</ul>

<div id="versioning-strategy">
  ## バージョニング戦略
</div>

<p>X API のバージョン管理は、エンドポイントのルートパス内に宣言されるバージョン番号によって表されます。</p>
<p>`https://api.x.com/2/tweets`</p>
<p>必要に応じて、ただし 12 か月より短い間隔では行わないことを前提に、X API のメジャーバージョンをリリースすることを目標としています。API に破壊的変更が導入される場合にメジャーバージョンをリリースします。新しいメジャーバージョンをリリースする際には、開発者が新バージョンへ移行できるよう、マイグレーションガイドを公開します。</p>
<p>破壊的変更が行われた場合、開発者はアプリ内の既存の機能を維持するためにコードを変更する必要があります。非破壊的変更は追加的な変更となり、準備が整い次第、最新バージョンに反映されます。この場合、新機能を利用したい場合を除き、開発者側での作業は必要ありません。</p>
<p>サイクルの途中で（セキュリティまたはプライバシー上の理由により）破壊的変更を適用する必要がある場合、この変更は最新バージョンに対して行われます。</p>

<div id="breaking-changes">
  ### 破壊的変更
</div>

<p>これらの変更が行われると、開発者はアプリケーションの既存の機能を維持するためにコードを変更する必要があります。</p>

<ul>
  <li>新しい必須パラメータの追加</li>
  <li>既存のエンドポイントの削除</li>
  <li>レスポンス内の任意のフィールドの削除（必須・任意を問わない）</li>
  <li>クエリパラメータの削除</li>
  <li>入力または出力フォーマットの構造の変更（たとえば、トップレベルのフィールドをサブフィールドにする、またはエラーの位置をインラインで返すように変更するなど）</li>
  <li>既存の入力パラメータまたは出力値の名前またはデータ型の変更</li>
  <li>フィールド名の変更</li>
  <li>リソース名の変更</li>
  <li>レスポンスコードの変更</li>
  <li>エラー種別の変更</li>
  <li>既存の認可スコープの変更</li>
</ul>

<div id="non-breaking-changes">
  ### 後方互換性を損なわない変更
</div>

<ul>
  <li>新しいエンドポイントの追加</li>
  <li>新しいオプションパラメータの追加</li>
  <li>新しいレスポンスフィールドの追加</li>
  <li>エラーメッセージのテキストの変更</li>
  <li>新しいスコープの追加</li>
  <li>フィールドの「null 化」（完全に削除する代わりに、プライバシー／セキュリティ上の理由で値を null に設定すること）</li>
</ul>

<div id="deprecation-and-retirement">
  ### 廃止と提供終了
</div>

<p>まず、X API における「廃止」と「提供終了」の定義を以下に示します。</p>

<ul>
  <li><b>廃止:</b> 機能はチームによるサポート対象外となります。その機能に対して新たな機能追加は行われず、プロダクトにバグや問題があっても、修正を検討する可能性は極めて低くなります。 </li>
  <li><b>提供終了:</b> 機能には今後アクセスできなくなります。</li>
</ul>

<p>ほとんどの場合、新しいバージョンがリリースされると同時に、直前のバージョンは廃止としてマークされます。バージョンは一定期間、廃止状態のまま維持された後、提供終了となります。 </p>
<p>今後の廃止および提供終了の詳細については、<a href="https://developer.x.com/en/updates/stay-informed">最新情報</a>をご確認ください。</p>
