<p>X API v2 の提供開始にあわせて、開発者が X のパブリック API にいつ変更が行われるのか、またいつ新しいバージョンへ移行する必要があるのかを把握しやすくなる、新しいバージョニング戦略を導入しました。 </p>
<p>開発者には、非推奨化・提供終了・変更・追加に関する情報を、<a href="https://developer.x.com/en/updates/stay-informed">各種コミュニケーションチャネル</a>を通じて通知します。これにより、開発ロードマップの中でこれらの変更に対応するための適切な計画を立てることができます。API へのすべての変更は、変更履歴に記載されます。</p>
<p>現在、X API には 3 つの異なるバージョンがあります。ご利用のユースケースに必要な機能がまだ v2 で提供されていない場合を除き、連携を計画する際には X API v2 を利用することを強く推奨します。 </p>
<p>各バージョンの詳細については、次のページをご覧ください。</p>

<ul>
  <li><a href="/ja/x-api/introduction">X API v2</a></li>
  <li>Gnip 2.0 (<a href="/ja/x-api/enterprise-gnip-2.0/enterprise-gnip">enterprise</a>)</li>
</ul>

<div id="versioning-strategy">
  ## バージョニング戦略
</div>

<p>X API のバージョニングは、エンドポイントのルートパス内で宣言されるバージョン番号によって表されます。</p>
<p>`https://api.x.com/2/tweets`</p>
<p>必要に応じて X API のメジャーバージョンをリリースしますが、その頻度が 12 か月未満になることはありません。メジャーバージョンは、API に破壊的変更が導入されたタイミングでリリースされます。新しいメジャーバージョンをリリースする際には、開発者が新バージョンへ移行できるようマイグレーションガイドを公開します。</p>
<p>破壊的変更が入ると、開発者はアプリ内の既存の機能を維持するためにコードを変更する必要があります。非破壊的変更は付加的な変更となり、準備ができ次第、最新バージョンにロールアウトされます。この場合、新機能を利用したい場合を除き、開発者側での作業は不要です。</p>
<p>セキュリティやプライバシー上の理由などでサイクルの途中に破壊的変更をロールアウトする必要がある場合は、その変更は最新バージョンに対して行われます。</p>

<div id="breaking-changes">
  ### 後方互換性を損なう変更
</div>

<p>これらの変更が行われた場合、開発者はアプリケーションの既存の機能を維持するためにコードを変更する必要があります。</p>

<ul>
  <li>新しい必須パラメータの追加</li>
  <li>既存のエンドポイントの削除</li>
  <li>レスポンス内のいずれかのフィールド（必須・任意を問わず）の削除</li>
  <li>クエリパラメータの削除</li>
  <li>入力または出力フォーマットの構造変更（たとえば、トップレベルフィールドをサブフィールドにする、またはエラーの位置をレスポンス本体内のインラインに変更するなど）</li>
  <li>既存の入力パラメータまたは出力値の名前またはデータ型の変更</li>
  <li>フィールド名の変更</li>
  <li>リソース名の変更</li>
  <li>レスポンスコードの変更</li>
  <li>エラー種別の変更</li>
  <li>既存の認可スコープの変更</li>
</ul>

<div id="non-breaking-changes">
  ### 後方互換な変更
</div>

<ul>
  <li>新しいエンドポイントの追加</li>
  <li>新しいオプションパラメーターの追加</li>
  <li>新しいレスポンスフィールドの追加</li>
  <li>エラーメッセージ内のテキストの変更</li>
  <li>新しいスコープの利用可能化</li>
  <li>フィールドの「null 設定」（フィールドを完全に削除する代わりに、プライバシー／セキュリティ上の理由で値を null に変更すること）</li>
</ul>

<div id="deprecation-and-retirement">
  ### 廃止と終了
</div>

<p>まず最初に、X API における「廃止」と「終了」の定義を示します。</p>

<ul>
  <li><b>廃止 (Deprecated):</b> 機能はチームによるサポート対象外となります。その機能に関する新たな機能追加は行われず、製品にバグや問題が発生した場合でも、それを修正する可能性は極めて低くなります。 </li>
  <li><b>終了 (Retired):</b> 機能は今後利用できなくなります。</li>
</ul>

<p>ほとんどの場合、新しいバージョンがリリースされるとすぐに、それまでのバージョンは廃止としてマークされます。バージョンは一定期間、廃止状態のまま維持され、その後終了されます。 </p>
<p>将来の廃止および終了についての詳細は、<a href="https://developer.x.com/en/updates/stay-informed">最新情報</a>をご確認ください。</p>
