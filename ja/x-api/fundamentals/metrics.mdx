<div id="overview">
  ## 概要
</div>

`metrics` フィールドを使用すると、開発者はポストおよびメディアオブジェクトに対する公開・非公開のエンゲージメントメトリクスにアクセスできます。公開メトリクスは開発者アカウントを持つ誰でも利用できますが、非公開メトリクスは所有している／権限を付与されたアカウント（定義は下記参照）からのみ利用できます。メトリクスには、各リクエストで指定されたポストごとのインプレッション、リツイート、引用ポスト、いいね、返信、動画再生数、動画再生の四分位、URL およびプロフィールリンクのクリック数などの合計件数が含まれます。また、ポストが [Ad](https://ads.x.com/) としてプロモーションされている場合には、オーガニックまたはプロモーションコンテキストで獲得したメトリクスの内訳を確認するオプションもあります。

公開メトリクスは [App-only Token](/ja/resources/fundamentals/authentication#oauth-2-0) 認証でリクエストできます。非公開メトリクスは所有している／権限を付与されたポストに対してのみリクエスト可能であるため、開発者は [OAuth 2.0](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) または [OAuth 1.0a](/ja/resources/fundamentals/authentication#oauth-1-0a-2) のユーザーコンテキスト認可を使用して認証する必要があります。

**非公開メトリクス、オーガニックメトリクス、およびプロモーションメトリクスは、直近 30 日以内に作成されたポストに対してのみ利用できます。**

<div id="terminology">
  ### 用語
</div>

* **認可済みアカウント**: あなたの [X 開発者アプリ](/ja/resources/fundamentals/developer-apps) に対して、そのアカウントへのアクセス権の付与を承認した X アカウント（どの [アプリの権限レベル](/ja/resources/fundamentals/developer-apps#app-permissions) でも、そのポストのメトリクスへのアクセスが可能になります）。
* **所有アカウント**: あなたの [X 開発者アプリ](/ja/resources/fundamentals/developer-apps) にリンクされている X アカウント。
* **公開メトリクス**: [X](https://x.com/home) 上で誰でもアクセスできる合計値。いいね数やリポスト数などが含まれます。
* **非公開メトリクス**: [X](https://x.com/home) 上で一般公開されない合計値。インプレッション数や動画再生の四分位数などが含まれます。[OAuth 2.0](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) または [OAuth 1.0a](/ja/resources/fundamentals/authentication#oauth-1-0a-2) によるユーザーコンテキスト認証が必要です。
* **オーガニックメトリクス**: オーガニックなコンテキスト（通常の投稿および閲覧）に起因する公開メトリクスおよび非公開メトリクスのグループです。[OAuth 2.0](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) または [OAuth 1.0a](/ja/resources/fundamentals/authentication#oauth-1-0a-2) によるユーザーコンテキスト認証が必要です。
* **プロモーションメトリクス**: プロモーションコンテキスト（広告キャンペーンの一部として投稿または閲覧された場合）に起因する公開メトリクスおよび非公開メトリクスのグループです。[OAuth 2.0](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) または [OAuth 1.0a](/ja/resources/fundamentals/authentication#oauth-1-0a-2) によるユーザーコンテキスト認証が必要であり、かつそのポストが広告キャンペーンでプロモーションされている必要があります。

<div id="available-metrics">
  ## 利用可能なメトリクス
</div>

| Metric               | API Representations                                                                                                                                                                                                                                     | Description                                                                                                                                                                                                                                                                                                                                                                                |
|:----------------------|:---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|:--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Impressions**      | `data.non_public_metrics.impression_count`, `data.organic_metrics.impression_count`, `data.promoted_metrics.impression_count`                                                                                                                           | ポストが表示された合計回数（ユーザー単位でのユニーク数ではありません）。ポストの一部でも画面上に表示されれば、ビューとしてカウントされます。OAuth 1.0a User Context 認証が必要です。                                                                                                               |
| **Retweets**         | `data.public_metrics.retweet_count`, `data.organic_metrics.retweet_count`, `data.promoted_metrics.retweet_count`                                                                                                                                       | ポストがリポスト（Retweet）された回数。引用ポスト（「コメント付きリポスト」）は含みません。X のクライアントに表示される「リポストとコメント」の合計値を取得するには、`retweet_count` と `quote_count` を合算してください。                                           |
| **Quote Tweets**     | `data.public_metrics.quote_count`                                                                                                                                                                                                                      | ポストが新しいコメント（メッセージ）付きでリポストされた回数。広告配信（有料）コンテキストからの引用ポストは存在しないため、すべての引用ポストはオーガニックです。                                                                                                                                           |
| **Likes**            | `data.public_metrics.like_count`, `data.organic_metrics.like_count`, `data.promoted_metrics.like_count`                                                                                                                                                 | ポストに「いいね」された回数。`public_metrics` フィールドは、X 上で公開されているカウントとの一貫性を保つため、オーガニックと有料コンテキストの両方からの「いいね」の合計件数を返します。                                                                                                   |
| **Replies**          | `data.public_metrics.reply_count`, `data.organic_metrics.reply_count`, `data.promoted_metrics.reply_count`                                                                                                                                              | ポストに返信された回数。`public_metrics` フィールドは、オーガニックと有料コンテキストの両方からの返信の合計件数を返します。                                                                                                                                                   |
| **URL Link Clicks**  | `data.non_public_metrics.url_link_clicks`, `data.organic_metrics.url_link_clicks`, `data.promoted_metrics.url_link_clicks`                                                                                                                             | ユーザーがポスト内の URL リンクまたは URL プレビューカードをクリックした回数。OAuth 1.0a User Context 認証が必要です。                                                                                                                                                                  |
| **User Profile Clicks** | `data.non_public_metrics.user_profile_clicks`, `data.organic_metrics.user_profile_clicks`, `data.promoted_metrics.user_profile_clicks`                                                                                                             | ユーザーがポスト内の要素（表示名、ユーザー名、プロフィール画像）をクリックした回数。OAuth 1.0a User Context 認証が必要です。                                                                                                                                               |
| **Video views**      | `includes.media.public_metrics.view_count`, `includes.media.organic_metrics.view_count`, `includes.media.promoted_metrics.view_count`                                                                                                                  | ポストに含まれる動画が再生された回数。この値は、特定の動画が含まれているすべてのポスト全体で集計された動画ビュー数です。`expansions=attachment.media_keys` のメディア expansion が必要です。                                                                |
| **Video view quartiles** | `includes.media.non_public_metrics.playback_0_count`, `includes.media.non_public_metrics.playback_25_count`, `includes.media.non_public_metrics.playback_50_count`, `includes.media.non_public_metrics.playback_75_count`, `includes.media.non_public_metrics.playback_100_count` | 動画の各四分位まで再生したユーザー数。OAuth 1.0a User Context 認証と、`expansions=attachment.media_keys` のメディア expansion が必要です。                                                                                                 |

<div id="requesting-metrics">
  ## メトリクスの取得
</div>

<div id="public-metrics">
  ### パブリックメトリクス
</div>

次のリクエストでは、ポストおよび添付されている動画について、以下のフィールドとexpansionsを指定してパブリックメトリクスを取得します。`$BEARER_TOKEN` は、必ずご自身が生成した Bearer Token に置き換えてください。

* `tweet.fields=public_metrics`
* `expansions=attachments.media_keys&media.fields=public_metrics`

<div id="sample-request">
  #### リクエスト例
</div>

```bash
curl 'https://api.x.com/2/tweets?ids=1204084171334832128&tweet.fields=public_metrics&expansions=attachments.media_keys&media.fields=public_metrics' --header 'Authorization: Bearer $BEARER_TOKEN'
```

<div id="private-metrics-non-public-organic-metrics">
  ### プライベートメトリクス（非公開メトリクス、オーガニックメトリクス）
</div>

次のリクエストでは、ポストおよび添付動画に対して、非公開メトリクスに加え、オーガニックメトリクスの追加の詳細を取得します。これらのフィールドはプライベートであり、X 上で一般公開されていないため、このリクエストには [OAuth 2.0](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) または [OAuth 1.0a](/ja/resources/fundamentals/authentication#oauth-1-0a-2) のユーザーコンテキストでの認証が必要です。以下で必要となる OAuth 1.0a シグネチャの生成方法については、こちらの[ガイド](/ja/resources/fundamentals/authentication#creating-a-signature)を参照してください。

* `tweet.fields=non_public_metrics,organic_metrics`
* `expansions=attachments.media_keys&media.fields=non_public_metrics,organic_metrics`

<div id="sample-request">
  #### リクエスト例
</div>

```bash
curl 'https://api.x.com/2/tweets/1204084171334832128?tweet.fields=non_public_metrics,organic_metrics&media.fields=non_public_metrics,organic_metrics&expansions=attachments.media_keys' --header 'authorization: OAuth oauth_consumer_key="CONSUMER_API_KEY", oauth_nonce="OAUTH_NONCE", oauth_signature="OAUTH_SIGNATURE", oauth_signature_method="HMAC-SHA1", oauth_timestamp="OAUTH_TIMESTAMP", oauth_token="ACCESS_TOKEN", oauth_version="1.0"'
```
