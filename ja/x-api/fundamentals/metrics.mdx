<div id="overview">
  ## Overview
</div>

`metrics` フィールドを使用すると、開発者はポストおよびメディアオブジェクトに対する公開・非公開のエンゲージメントメトリクスにアクセスできます。公開メトリクスは開発者アカウントを持つすべてのユーザーが利用できますが、非公開メトリクスには所有／認可済みアカウント（定義は後述）からのみアクセスできます。メトリクスには、リクエストで指定された各ポストごとのインプレッションの合計件数、リポスト、引用ポスト、いいね、返信、動画再生数、動画再生クォータイル、URL およびプロフィールリンクのクリック数が含まれます。さらに、そのポストが [Ad](https://ads.x.com/) としてプロモーションされている場合には、オーガニックまたはプロモーションのコンテキストで獲得したメトリクスの内訳を表示するオプションもあります。

公開メトリクスは [App-only Token](/ja/resources/fundamentals/authentication#oauth-2-0) 認証でリクエストできます。非公開メトリクスは所有／認可済みポストに対してのみリクエスト可能であり、そのためには [OAuth 2.0](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) または [OAuth 1.0a](/ja/resources/fundamentals/authentication#oauth-1-0a-2) のユーザーコンテキスト認可による認証が必要です。

**非公開メトリクス、オーガニックメトリクス、およびプロモーションメトリクスは、過去 30 日以内に作成されたポストに対してのみ利用可能です。**

<div id="terminology">
  ### 用語
</div>

* **Authorized account（認可済みアカウント）**: 特定のアカウントへのアクセス権限を付与することで、あなたの [X developer app](/ja/resources/fundamentals/developer-apps) を承認した X アカウント（いずれの[アプリの権限レベル](/ja/resources/fundamentals/developer-apps#app-permissions)でもポストメトリクスへのアクセスが可能です）。
* **Owned account（所有アカウント）**: あなたの [X developer app](/ja/resources/fundamentals/developer-apps) にリンクされている X アカウント。
* **Public metrics（公開メトリクス）**: いいね数やリポスト数など、[X](https://x.com/home) 上で誰でもアクセスできる合計値。
* **Non-public metrics（非公開メトリクス）**: インプレッション数や動画視聴クォータイル数など、[X](https://x.com/home) 上で一般には表示されない合計値。[OAuth 2.0](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) または [OAuth 1.0a](/ja/resources/fundamentals/authentication#oauth-1-0a-2) のユーザーコンテキスト認証が必要です。
* **Organic metrics（オーガニックメトリクス）**: オーガニックなコンテキスト（通常どおりに投稿・閲覧された場合）に起因する公開・非公開メトリクスのグループ。[OAuth 2.0](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) または [OAuth 1.0a](/ja/resources/fundamentals/authentication#oauth-1-0a-2) のユーザーコンテキスト認証が必要です。
* **Promoted metrics（プロモーションメトリクス）**: プロモーションコンテキスト（広告キャンペーンの一部として投稿または表示された場合）に起因する公開・非公開メトリクスのグループ。[OAuth 2.0](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) または [OAuth 1.0a](/ja/resources/fundamentals/authentication#oauth-1-0a-2) のユーザーコンテキスト認証に加え、そのポストが広告としてプロモーションされていることが必要です。

<div id="available-metrics">
  ## 利用可能なメトリクス
</div>

| Metric               | API Representations                                                                                                                                                                                                                                     | Description                                                                                                                                                                                                                                                                                                                                                                                |
|:----------------------|:---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|:--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Impressions**      | `data.non_public_metrics.impression_count`, `data.organic_metrics.impression_count`, `data.promoted_metrics.impression_count`                                                                                                                           | ポストが何回閲覧されたか（ユーザー単位でのユニーク数ではありません）。ポストの一部でも画面上に表示されれば 1 回の閲覧としてカウントされます。OAuth 1.0a User Context 認証が必要です。                                                                                                               |
| **Retweets**         | `data.public_metrics.retweet_count`, `data.organic_metrics.retweet_count`, `data.promoted_metrics.retweet_count`                                                                                                                                       | ポストが何回リツイートされたかを示します。Quote Tweets（「コメント付きリツイート」）は含みません。X クライアントに表示される「リツイートとコメント」の合計値を取得するには、`retweet_count` と `quote_count` を加算してください。                                           |
| **Quote Tweets**     | `data.public_metrics.quote_count`                                                                                                                                                                                                                      | ポストが新しいコメント（メッセージ）付きでリツイートされた回数です。有料コンテキストからの Quote Tweets は存在しないため、すべての Quote Tweets はオーガニックです。                                                                                                                                           |
| **Likes**            | `data.public_metrics.like_count`, `data.organic_metrics.like_count`, `data.promoted_metrics.like_count`                                                                                                                                                 | ポストが何回「いいね」されたかを示します。`public_metrics` フィールドは、X 上で公開されている件数表示との一貫性を保つため、オーガニックと有料コンテキストの両方からの「いいね」の合計件数を返します。                                                                                                   |
| **Replies**          | `data.public_metrics.reply_count`, `data.organic_metrics.reply_count`, `data.promoted_metrics.reply_count`                                                                                                                                              | ポストに何回返信が行われたかを示します。`public_metrics` フィールドは、オーガニックと有料コンテキストの両方からの返信の合計件数を返します。                                                                                                                                                   |
| **URL Link Clicks**  | `data.non_public_metrics.url_link_clicks`, `data.organic_metrics.url_link_clicks`, `data.promoted_metrics.url_link_clicks`                                                                                                                             | ユーザーがポスト内の URL リンクまたは URL プレビューカードをクリックした回数です。OAuth 1.0a User Context 認証が必要です。                                                                                                                                                                  |
| **User Profile Clicks** | `data.non_public_metrics.user_profile_clicks`, `data.organic_metrics.user_profile_clicks`, `data.promoted_metrics.user_profile_clicks`                                                                                                             | ユーザーがポスト内の各要素（表示名、ユーザー名、プロフィール画像）をクリックした合計回数です。OAuth 1.0a User Context 認証が必要です。                                                                                                                                               |
| **Video views**      | `includes.media.public_metrics.view_count`, `includes.media.organic_metrics.view_count`, `includes.media.promoted_metrics.view_count`                                                                                                                  | ポストに含まれる動画が何回再生されたかを示します。これは、該当動画が投稿されているすべてのポストを通じて集計された動画再生回数です。メディアの expansion `expansions=attachment.media_keys` が必要です。                                                                |
| **Video view quartiles** | `includes.media.non_public_metrics.playback_0_count`, `includes.media.non_public_metrics.playback_25_count`, `includes.media.non_public_metrics.playback_50_count`, `includes.media.non_public_metrics.playback_75_count`, `includes.media.non_public_metrics.playback_100_count` | 動画内の各四分位まで再生したユーザー数です。OAuth 1.0a User Context 認証およびメディアの expansion `expansions=attachment.media_keys` が必要です。                                                                                                 |

<div id="requesting-metrics">
  ## メトリクスの取得
</div>

<div id="public-metrics">
  ### 公開メトリクス
</div>

次のリクエストでは、ポストと添付されている動画に対して、以下のフィールドと expansions を指定して公開メトリクスをリクエストします。`$BEARER_TOKEN` は必ずご自身で生成した Bearer Token に置き換えてください。

* `tweet.fields=public_metrics`
* `expansions=attachments.media_keys&media.fields=public_metrics`

<div id="sample-request">
  #### リクエスト例
</div>

```bash
curl 'https://api.x.com/2/tweets?ids=1204084171334832128&tweet.fields=public_metrics&expansions=attachments.media_keys&media.fields=public_metrics' --header 'Authorization: Bearer $BEARER_TOKEN'
```

<div id="private-metrics-non-public-organic-metrics">
  ### プライベートメトリクス（非公開、オーガニックメトリクス）
</div>

次のリクエストでは、ポストおよび添付された動画に対して、オーガニックメトリクスの詳細を含む非公開メトリクスを要求します。これらのフィールドはプライベートであり、X 上で一般には表示されないため、このリクエストには [OAuth 2.0](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) または [OAuth 1.0a](/ja/resources/fundamentals/authentication#oauth-1-0a-2) のユーザーコンテキスト認証が必要です。以下で使用する OAuth 1.0a シグネチャの生成方法については、[ガイド](/ja/resources/fundamentals/authentication#creating-a-signature) を参照してください。

* `tweet.fields=non_public_metrics,organic_metrics`
* `expansions=attachments.media_keys&media.fields=non_public_metrics,organic_metrics`

<div id="sample-request">
  #### リクエスト例
</div>

```bash
curl 'https://api.x.com/2/tweets/1204084171334832128?tweet.fields=non_public_metrics,organic_metrics&media.fields=non_public_metrics,organic_metrics&expansions=attachments.media_keys' --header 'authorization: OAuth oauth_consumer_key="CONSUMER_API_KEY", oauth_nonce="OAUTH_NONCE", oauth_signature="OAUTH_SIGNATURE", oauth_signature_method="HMAC-SHA1", oauth_timestamp="OAUTH_TIMESTAMP", oauth_token="ACCESS_TOKEN", oauth_version="1.0"'
```
