<div id="conversation-threads-using-the-x-api">
  ### X API を使用した会話スレッド
</div>

X 上で会話がどのように展開していくかを見ると、1つのポストが複数の会話スレッドを生み出し、それぞれが、より多くのユーザーが参加することで長さと複雑さを増していくことがあります。ポスト同士の関係を特定し、会話スレッドを理解することは、X API v2 のペイロードおよび検索機能の一部です。あるポストに対してポストが投稿された場合（リプライとして知られています）、またはそのリプライに対してさらにリプライが投稿された場合、それぞれのリプライには定義済みの `conversation_id` が付与されており、その値は会話を開始した元のポストの Post ID と一致します。

特定のポストへのリプライと、それらのリプライへのリプライはすべて、1つの元のポストから派生した会話に含まれます。どれだけ多くのリプライスレッドが発生しても、それらはすべて、会話のきっかけとなった元のポストと共通の `conversation_id` を共有します。X API v2 を使用すると、会話スレッド全体を取得して再構築できるため、どのようなやり取りが行われているのか、そして会話やアイデアがどのように発展していくのかを把握できます。

`tweet.fields` パラメーターとしての `conversation_id` の取得
v2 エンドポイントで返されるすべてのポストについて `conversation_id` を取得するには、リクエストパラメーターに tweet.fields=conversation&#95;id フィールドを追加します。conversation&#95;id フィールドは、常にその会話の返信スレッド内にある元のポストのポスト ID です。以前の返信スレッドから派生して作成された返信スレッドを含め、同じ返信スレッド内のすべてのポストは同じ conversation&#95;id を持ちます。

<div id="request-with-conversation_id-parameter">
  ### `conversation_id` パラメータ付きのリクエスト
</div>

```
curl --request GET \
  --url 'https://api.x.com/2/tweets?ids=1225917697675886593&tweet.fields=author_id,conversation_id,created_at,in_reply_to_user_id,referenced_tweets&expansions=author_id,in_reply_to_user_id,referenced_tweets.id&user.fields=name,username' \
  --header 'Authorization: Bearer $ACCESS_TOKEN'
```

<div id="using-conversation_id-as-a-filter-operator">
  ### `conversation_id` をフィルター演算子として使用する
</div>

`conversation_id` は、recent search の検索クエリパラメータとして、または [filtered stream](/ja/x-api/posts/filtered-stream/introduction) のルール内の演算子として使用できます。これにより、filtered stream を通じてリアルタイムで、または [search Tweets](/ja/x-api/posts/search/introduction) から逆時系列で、会話スレッドに含まれるポスト全体が返されます。また、この演算子を [Posts counts](/ja/x-api/posts/counts/introduction) と組み合わせて使用することで、会話内のポスト件数を取得することもできます。

<div id="request-to-query-by-conversation_id">
  ## `conversation_id` でクエリするためのリクエスト
</div>

```
curl --request GET \
  --url 'https://api.x.com/2/tweets/search/recent?query=conversation_id:1279940000004973111&tweet.fields=in_reply_to_user_id,author_id,created_at,conversation_id' \
  --header 'Authorization: Bearer $ACCESS_TOKEN'
```
