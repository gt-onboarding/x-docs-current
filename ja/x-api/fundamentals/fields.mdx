X API v2 のエンドポイントには *フィールド* と呼ばれる一連のパラメーターが用意されており、エンドポイントのレスポンス内に含まれる各オブジェクトから、必要なデータだけを選択して取得できます。

デフォルトでは、Post オブジェクトは `id` と `text` フィールドのみを返し、さらに 2022 年 9 月 29 日以降に作成されたポストについては `edit_history_tweet_ids` フィールドも返します。ポストの作成日時や公開メトリクスが必要な場合は、それらを取得するために `tweet.fields` パラメーターを使用する必要があります。これにより、ユースケースに応じて必要なフィールドだけをリクエストでき、より柔軟にカスタマイズすることができます。たとえば、リクエストには次のクエリ文字列を含めます。

`?tweet.fields=created_at,public_metrics`

各オブジェクトには、それぞれのオブジェクトに関連付けられたフィールドを個別にリクエストするための固有のパラメーターがあります。現在利用可能なフィールド用パラメーターは次のとおりです。

* [Post](/ja/x-api/fundamentals/data-dictionary#tweet) → `tweet.fields`
* [User](/ja/x-api/fundamentals/data-dictionary#user) → `user.fields`
* [Media](/ja/x-api/fundamentals/data-dictionary#media) → `media.fields`
* [Poll](/ja/x-api/fundamentals/data-dictionary#poll) → `poll.fields`
* [Place](/ja/x-api/fundamentals/data-dictionary#place) → `place.fields`

特定のオブジェクトを主に返すエンドポイントを使用する場合は、対応するフィールドパラメーターを使用し、そのパラメーターの値として取得したいフィールドをカンマ区切りのリストで指定することで、レスポンスにそれらのフィールドを含めることができます。

<div id="example">
  ### 例
</div>

[GET /tweets/search/recent](/ja/x-api/posts/recent-search) エンドポイントを使用している場合、そのレスポンスには主に [ポスト（Tweet）オブジェクト](/ja/x-api/fundamentals/data-dictionary#tweet) が含まれます。`fields` パラメータを指定しない場合は、デフォルト値である `id` と `text` のみが返されます。レスポンスで返されるポストの公開メトリクスも取得したい場合は、リクエストに `tweet.fields` パラメータを含め、その値として `public_metrics` を指定します。

このリクエストは次のようになります。このリクエストを利用する場合は、必ず `$BEARER_TOKEN` を自身の [Bearer Token](/ja/resources/fundamentals/authentication) に置き換え、使用しているコマンドラインツールから送信してください。

```bash
curl --request GET \
  --url 'https://api.x.com/2/tweets/search/recent?query=from%3Atwitterdev&tweet.fields=public_metrics' \
  --header 'Authorization: Bearer $BEARER_TOKEN'
```

ターミナルでこのリクエストを送信すると、レスポンスとして返される各ポストには次のフィールドが含まれます。

```
{
   "data": {
       "id": "1263150595717730305",
       "public_metrics": {
           "retweet_count": 12,
           "reply_count": 14,
           "like_count": 49,
           "quote_count": 7
       },
       "text": "新しいツイート設定を👀見ましたか？\n\nAPIでこのような機能をどのように、そしてなぜ使用するのか知りたいです。詳細を確認して、ご意見をお聞かせください👇\nhttps://t.co/RtMhhfAcIB https://t.co/8wxeZ9fJER"
   }
}
```

エンドポイントから返されるプライマリオブジェクトに関連付けられたセカンダリオブジェクトから一連のフィールドを取得したい場合は、追加の `expansions` パラメータを指定する必要があります。たとえば、先ほどと同じ `GET search/tweets/recent` エンドポイントを使用していて、投稿者のプロフィール説明を取得したい場合、リクエストに `expansions=author_id` と `user.fields=description` を含める必要があります。以下は、その例です。このリクエストを試す場合は、コマンドラインツールに貼り付ける前に、`$BEARER_TOKEN` をご自身の [Bearer Token](/ja/resources/fundamentals/authentication) に置き換えてください。

```bash
  curl --request GET \
  --url 'https://api.x.com/2/tweets/search/recent?query=from%3Atwitterdev&tweet.fields=public_metrics&expansions=author_id&user.fields=description' \
  --header 'Authorization: Bearer $BEARER_TOKEN'
```

このパラメータをリクエストで指定すると、配信される各ポストには次のフィールドが含まれ、関連する[ユーザーオブジェクト](/ja/x-api/fundamentals/data-dictionary#user)のデフォルトフィールドおよび指定したフィールドが `includes` 内に返されます。ユーザーオブジェクトは、`tweet.author_id` フィールドと `users.id` フィールドを照合することで、対応するポストにマッピングできます。

```
{
  "data": [
    {
      "id": "1263150595717730305",
      "author_id": "2244994945",
      "text": "Do you 👀our new Tweet settings?\n\nWe want to know how and why you’d use a feature like this in the API. Get the details and let us know what you think👇\nhttps://t.co/RtMhhfAcIB https://t.co/8wxeZ9fJER",
      "public_metrics": {
        "retweet_count": 12,
        "reply_count": 13,
        "like_count": 51,
        "quote_count": 7
      }
    }
  ],
  "includes": {
    "users": [
      {
        "id": "2244994945",
        "username": "TwitterDev",
        "description": "#TwitterDevチームの公式アカウントです。#TwitterAPIに関する最新情報、ニュース、イベント情報をお届けします。",
        "name": "Twitter Dev"
      }
    ]
  }
}
```

特定のサブフィールド（例：`public_metrics.retweet_count`）をリクエストとして個別に指定することはできない点に留意してください。トップレベルのフィールド（`public_metrics`）を指定した場合は、すべてのサブフィールドが返されます。各エンドポイントの API リファレンスページのパラメーターテーブルに、リクエスト可能なすべてのフィールドを掲載しています。

利用可能なフィールドの全リストは [オブジェクトモデル](/ja/x-api/fundamentals/data-dictionary#tweet) に記載されています。エンドポイントの主要リソースではないオブジェクトのフィールドを展開してリクエストするには、[expansions](/ja/x-api/fundamentals/expansions) パラメータをフィールド指定と組み合わせて使用します。
