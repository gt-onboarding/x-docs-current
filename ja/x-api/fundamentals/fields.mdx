X API v2 の各エンドポイントには *フィールド* と呼ばれる一連のパラメータが用意されており、エンドポイントレスポンス内の各オブジェクトから、必要なデータだけを選択できるようになっています。

デフォルトでは、Post オブジェクトは `id` と `text` フィールド、そして 2022 年 9 月 29 日以降に作成された Post については `edit_history_tweet_ids` フィールドのみを返します。Post の作成日時や公開メトリクスが必要な場合は、それらをリクエストするために `tweet.fields` パラメータを使用する必要があります。これにより、ユースケースに応じて必要なフィールドだけをリクエストできるため、レスポンスをより柔軟にカスタマイズできます。例えば、次のクエリ文字列をリクエストに含めます。

`?tweet.fields=created_at,public_metrics`

各オブジェクトには、それぞれのオブジェクトに関連付けられたフィールドを明示的にリクエストするための専用パラメータがあります。現在利用可能なフィールドパラメータは次のとおりです。

* [Post](/ja/x-api/fundamentals/data-dictionary#tweet) → `tweet.fields`
* [User](/ja/x-api/fundamentals/data-dictionary#user) → `user.fields`
* [Media](/ja/x-api/fundamentals/data-dictionary#media) → `media.fields`
* [Poll](/ja/x-api/fundamentals/data-dictionary#poll) → `poll.fields`
* [Place](/ja/x-api/fundamentals/data-dictionary#place) → `place.fields`

特定のオブジェクトを主に返すエンドポイントを使用する場合は、そのオブジェクトに対応するフィールドパラメータを使用し、そのパラメータの値として取得したいフィールドをカンマ区切りのリストで指定するだけで、レスポンスにそれらのフィールドを含めることができます。

<div id="example">
  ### 例
</div>

[GET /tweets/search/recent](/ja/x-api/posts/recent-search) エンドポイントを使用している場合、そのレスポンスでは主に [ポストオブジェクト](/ja/x-api/fundamentals/data-dictionary#tweet) が返されます。`fields` パラメータを指定しない場合、デフォルトの `id` と `text` のみが返されます。レスポンスで返されるポストの公開メトリクスに関心がある場合は、リクエストに `tweet.fields` パラメータを含め、その値として `public_metrics` を指定します。

このリクエストは次のようになります。このリクエストを使用する場合は、`$BEARER_TOKEN` を自分の [Bearer Token](/ja/resources/fundamentals/authentication) に必ず置き換え、使用しているコマンドラインツールから送信してください。

```bash
curl --request GET \
  --url 'https://api.x.com/2/tweets/search/recent?query=from%3Atwitterdev&tweet.fields=public_metrics' \
  --header 'Authorization: Bearer $BEARER_TOKEN'
```

ターミナルからこのリクエストを送信すると、返される各ポストには以下のフィールドが含まれます。

```
{
   "data": {
       "id": "1263150595717730305",
       "public_metrics": {
           "retweet_count": 12,
           "reply_count": 14,
           "like_count": 49,
           "quote_count": 7
       },
       "text": "新しいツイート設定を👀見ましたか？\n\nAPIでこのような機能をどのように、そしてなぜ使用するのか知りたいです。詳細を確認して、ご意見をお聞かせください👇\nhttps://t.co/RtMhhfAcIB https://t.co/8wxeZ9fJER"
   }
}
```

エンドポイントによって返されるプライマリオブジェクトに関連付けられたセカンダリオブジェクトから一連のフィールドを取得したい場合は、追加で `expansions` パラメータを指定する必要があります。たとえば、先ほどと同じ `GET search/tweets/recent` エンドポイントを使用していて、ポストの作成者のプロフィール説明を取得したい場合は、リクエストに `expansions=author_id` と `user.fields=description` を指定する必要があります。以下はその例です。このリクエストを試す場合は、コマンドラインツールに貼り付ける前に、`$BEARER_TOKEN` をご自身の [Bearer Token](/ja/resources/fundamentals/authentication) に置き換えてください。

```bash
  curl --request GET \
  --url 'https://api.x.com/2/tweets/search/recent?query=from%3Atwitterdev&tweet.fields=public_metrics&expansions=author_id&user.fields=description' \
  --header 'Authorization: Bearer $BEARER_TOKEN'
```

これをリクエスト内で指定すると、配信される各ポストには次のフィールドが含まれ、関連する[ユーザーオブジェクトの](/ja/x-api/fundamentals/data-dictionary#user)デフォルトフィールドおよび指定したフィールドが `includes` 内で返されます。ユーザーオブジェクトは、`tweet.author_id` フィールドと `users.id` フィールドを照合することで、対応するポストとマッピングできます。

```
{
  "data": [
    {
      "id": "1263150595717730305",
      "author_id": "2244994945",
      "text": "Do you 👀our new Tweet settings?\n\nWe want to know how and why you’d use a feature like this in the API. Get the details and let us know what you think👇\nhttps://t.co/RtMhhfAcIB https://t.co/8wxeZ9fJER",
      "public_metrics": {
        "retweet_count": 12,
        "reply_count": 13,
        "like_count": 51,
        "quote_count": 7
      }
    }
  ],
  "includes": {
    "users": [
      {
        "id": "2244994945",
        "username": "TwitterDev",
        "description": "#TwitterDevチームの公式アカウントです。#TwitterAPIに関する最新情報、ニュース、イベント情報をお届けします。",
        "name": "Twitter Dev"
      }
    ]
  }
}
```

特定のサブフィールド（例：`public_metrics.retweet_count`）をリクエストすることはできない点に注意してください。トップレベルのフィールド（`public_metrics`）を指定すると、その配下のサブフィールドはすべて返されます。リクエスト可能なすべてのフィールドは、各エンドポイントのAPIリファレンスページにあるパラメータ表に一覧化されています。

利用可能なすべてのフィールドの一覧は、[オブジェクトモデル](/ja/x-api/fundamentals/data-dictionary#tweet) に記載されています。そのエンドポイントの主要なリソースではないオブジェクトのフィールドを展開してリクエストするには、[expansions](/ja/x-api/fundamentals/expansions) パラメータをフィールドと併用してください。
