<div id="tweet">
  ### Tweet
</div>

ポストは、Twitter におけるあらゆるものの基本的な構成要素です。Tweet オブジェクトには、`id`、`text`、`created_at` など、多くの「ルートレベル」のフィールドがあります。Tweet オブジェクトは、`user`、`media`、`poll`、`place` など複数の子オブジェクトの「親」オブジェクトでもあります。Tweet オブジェクトのこれらのルートレベルのフィールドをリクエストする場合は、フィールドパラメータ `tweet.fields` を使用してください。

Tweet オブジェクトは user リソース内で取得して展開できます。リクエストした Tweet に関連する追加の Tweet も、Tweet リソース内で取得して展開できます。オブジェクトは、user リソースでは `?expansions=pinned_tweet_id`、Tweet リソースでは `?expansions=referenced_tweets.id` を指定することで、デフォルトフィールドのみを持つオブジェクトとして expansions で取得できます。オブジェクトを補完するために追加のフィールドをリクエストする場合は、expansions とフィールドパラメータ `tweet.fields` を組み合わせて使用してください。

| :----------------------------------------------- | :----------- | :----------------------------------------------------------------------------------------------------------- | :------------------------------------------------- |
| **id (デフォルト)**                                   | string       | リクエストしたポストを一意に識別するid。                                                                                        | プログラムから特定のポストを取得するために使用します。                        |
| **text（デフォルト）**                                  | string       | ポストの実際の UTF-8 テキスト。有効な文字の詳細については [twitter-text](https://github.com/twitter/twitter-text/) を参照してください。         | キーワード抽出や感情分析・分類                                    |
| **edit&#95;history&#95;tweet&#95;ids (default)** | オブジェクト       | ポストのすべてのバージョンを示す一意の識別子です。編集されていないポストでは ID は 1 つだけです。編集履歴があるポストでは ID が複数存在します。                                | この情報を使用すると、ポストの編集履歴を特定できます。                        |
| **article**                                      | object       | このポストに含まれる Article のメタデータを格納します。                                                                             | これを使用すると、Article のテキストとエンティティを取得できます。              |
| **attachments**                                  | オブジェクト       | このポストに含まれる添付（存在する場合）の種類を指定します。                                                                               | リクエストした expansions で返されるオブジェクトを理解するために使用します。       |
| **author&#95;id**                                | string       | このポストを作成したユーザーの一意の識別子。                                                                                       | User オブジェクトの復元やピアレビュー用データセットの共有に利用できる。             |
| **card&#95;uri**                                 | string       | このポストに含まれる Card の URI。                                                                                       |                                                    |
| **community&#95;id**                             | 文字列          | このポストが属するコミュニティの一意のid。                                                                                       |                                                    |
| **context&#95;annotations**                      | 配列           | ポストに関するコンテキスト注釈を含みます。                                                                                        | エンティティの認識／抽出やトピック分析に利用できます。                        |
| **conversation&#95;id**                          | 文字列          | 会話の起点となる元のポストのポストID（その会話には直接の返信およびその返信への返信が含まれます）。                                                           | これを使用すると、ポストから会話全体を再構築できます。                        |
| **created&#95;at**                               | 日付（ISO 8601） | ポストが作成された日時。                                                                                                 | 時系列分析や、ポストがいつ作成されたかを把握するのに役立ちます。                   |
| **display&#95;text&#95;range**                   | 配列           | 表示対象のテキスト部分の開始および終了インデックスを含む配列。                                                                              | 長いポストでデフォルトで表示されるテキスト範囲を把握するのに役立ちます。               |
| **edit&#95;controls**                            | object       | ポストをあとどのくらいの期間編集できるかと、残りの編集回数を示します。                                                                          | このフィールドを使用して、ポストを編集できるかどうかを判定します。                  |
| **entities**                                     | object       | ポスト本文から解析・抽出されたエンティティ。詳細は Twitter Objects の entities フィールドを参照してください。                                         | ハッシュタグやURL、メンションなどに関する追加情報を提供します。                  |
| **geo**                                          | object       | ジオタグが付与されたポストの位置情報を示します。                                                                                     | ジオタグ付きポストの位置を特定するために使用します。                         |
| **in&#95;reply&#95;to&#95;user&#95;id**          | string       | このポストが返信である場合、このフィールドには元のポストの作成者のidが含まれます。                                                                   | ポストが別のポストへの返信として投稿されたかどうかを判定します。                   |
| **lang**                                         | 文字列          | Twitter が検出できた場合のポストの言語。                                                                                     | ポストを使用言語別に分類する。                                    |
| **non&#95;public&#95;metrics**                   | オブジェクト       | リクエスト時点におけるポストの非公開のエンゲージメントメトリクス。ユーザーコンテキストでの認証が必要です。                                                        | ポストで発生したインプレッションの総数を確認します。                         |
| **note&#95;tweet**                               | object       | 280文字を超える長文ポストの全文を格納します。                                                                                     | ポストのテキスト全文を取得します。                                  |
| **organic&#95;metrics**                          | object       | リクエスト時点のポストに対して、オーガニックコンテキストで計測されるエンゲージメントメトリクス。ユーザーコンテキストによる認証が必要です。                                        | ポストのオーガニックエンゲージメントを計測します。                          |
| **possibly&#95;sensitive**                       | boolean      | コンテンツがセンシティブとして認識される可能性があるかを示します。                                                                            | 特定の種類のコンテンツの拡散状況を分析する。                             |
| **promoted&#95;metrics**                         | object       | リクエスト時点において、そのポストがプロモーションとして配信されていた際のコンテキストで計測されたエンゲージメントメトリクス。ユーザーコンテキスト認証が必要です。                            | ポストがプロモーションされている際のエンゲージメントを測定します。                  |
| **public&#95;metrics**                           | object       | リクエスト時点のポストに対する公開エンゲージメントメトリクス。                                                                              | ポストのエンゲージメントを測定します。                                |
| **referenced&#95;tweets**                        | 配列           | このポストが参照しているポストのリストで、リポスト、引用ポスト、返信などが含まれます。                                                                  | リツイートなどを含む会話のつながりを把握する。                            |
| **reply&#95;settings**                           | 文字列          | 指定されたポストに誰が返信できるかを示します。オプションは &quot;everyone&quot;、&quot;mentioned&#95;users&quot;、&quot;followers&quot; です。 | ポストに対して誰が返信できるかの設定を判断する。                           |
| **withheld**                                     | object       | [制限対象コンテンツ](https://help.x.com/en/rules-and-policies/tweet-withheld-by-country) に関する制限情報を含みます。               |                                                    |
| **scopes**                                       | object       | ポストの公開範囲の詳細を含みます。                                                                                            | どのユーザーがこのポストを閲覧できるかを示します。プロモーションされたポストに対してのみ返されます。 |
| **media&#95;metadata**                           | 配列           | ポストに添付されたメディアのメタデータを含みます。                                                                                    | ポストに添付されたメディアの `alt_text` など、追加のメタデータを取得します。       |

**ポストオブジェクトの取得**

**リクエスト例**

次のリクエストでは、[Tweets lookup](/ja/resources/fundamentals/rate-limits)エンドポイントでポストのフィールドを指定してリクエストしています。必ず `$BEARER_TOKEN` をご自身で生成した [Bearer Token](/ja/resources/fundamentals/authentication) に置き換えてください。

```bash
curl --request GET 'https://api.x.com/2/tweets?ids=1212092628029698048&tweet.fields=attachments,author_id,context_annotations,created_at,entities,geo,id,in_reply_to_user_id,lang,possibly_sensitive,public_metrics,referenced_tweets,text,withheld&expansions=referenced_tweets.id' --header 'Authorization: Bearer $BEARER_TOKEN'
```

<div id="user">
  ### User
</div>

user オブジェクトには、参照されているユーザーを表す X ユーザーアカウントのメタデータが含まれます。user オブジェクトは、[users lookup](/ja/x-api/users/lookup/introduction) エンドポイントから返される主要なオブジェクトです。このエンドポイントで追加のユーザーフィールドを取得する場合は、`user.fields` パラメータを使用します。

user オブジェクトは、子オブジェクトとしてポストオブジェクト内にも存在し、展開することができます。このオブジェクトは、デフォルトフィールドのみを含む簡略化されたオブジェクトを取得するために、`?expansions=author_id` または `?expansions=in_reply_to_user_id` として expansions パラメータで指定できます。オブジェクトを完全な形にするために追加フィールドを取得したい場合は、expansions と併せてフィールドパラメータ `user.fields` を指定してください。

<div id="space">
  ### Space
</div>

Spaces はライブ音声会話による表現と交流を可能にします。Space データ辞書には Space に関するメタデータが含まれており、そのすべての詳細はリアルタイムで更新されます。

User オブジェクトは user リソースで取得および展開できます。これらのオブジェクトは、`expansions` クエリパラメータに `host_ids`、`creator_id`、`speaker_ids`、`mentioned_user_ids` のうち少なくとも 1 つを追加することで展開可能になります。

ポストとは異なり、Space は一時的なものであり、終了後、または作成者によってキャンセルされた場合は利用できなくなります。アプリで Spaces のデータを扱う際は、最新の情報を返す責任があり、プラットフォーム上で利用できなくなったデータは削除しなければなりません。[Spaces lookup endpoints](/ja/x-api/spaces/lookup/introduction) を使用すると、ユーザーの期待と意図を尊重できているかを確認するのに役立ちます。

<div id="list">
  ### List
</div>

List オブジェクトには、参照されている List を説明する [Twitter Lists](https://help.x.com/en/using-twitter/twitter-lists) のメタデータが含まれます。List オブジェクトは、List ルックアップエンドポイントで返される主要なオブジェクトです。このエンドポイントで追加の List フィールドをリクエストする場合は、`list.fields` パラメータを使用します。

List オブジェクトは、他のデータオブジェクトの子としては返されません。ただし、ユーザーオブジェクトは user リソースで取得して展開できます。これらのオブジェクトは、`expansions` クエリパラメータに `owner_id` を追加することで展開できます。主要な List オブジェクトを補完するために追加フィールドをリクエストする際は、この expansion を `list.fields` フィールドパラメータと併用し、展開オブジェクトを補完するには `user.fields` を使用します。

<div id="direct-message-events">
  ### ダイレクトメッセージイベント
</div>

ダイレクトメッセージ（DM）の会話はイベントで構成されます。X API v2 では現在、MessageCreate、ParticipantsJoin、ParticipantsLeave の 3 種類のイベントタイプをサポートしています。

DM イベントオブジェクトは [Direct Message lookup](/ja/x-api/direct-messages/lookup/introduction) エンドポイントから返され、[Manage Direct Messages](/ja/x-api/direct-messages/lookup/introduction) エンドポイントでダイレクトメッセージが正常に作成されると、MessageCreate イベントが生成されます。

DM イベントをリクエストする際、デフォルトではイベントオブジェクトの属性（フィールド）として id、event&#95;type、text の 3 つが含まれます。追加のイベントフィールドを受け取るには、[fields](/ja/x-api/fundamentals/fields) パラメータ dm&#95;event.fields を使用して他のフィールドを指定します。利用可能なその他のイベントフィールドには、次のものが含まれます: dm&#95;conversation&#95;id、created&#95;at、sender&#95;id、attachments、participant&#95;ids、referenced&#95;tweets。

これらのフィールドのいくつかは、ダイレクトメッセージイベントに関連する他の X オブジェクトの ID を示します:

* sender&#95;id - メッセージを送信したアカウント、またはグループ会話に参加者を招待したアカウントの ID
* partricipants&#95;ids - アカウント ID の配列。ParticipantsJoin および ParticipantsLeave イベントでは、この配列にはイベントを作成したアカウントの単一の ID が含まれます
* attachments - 送信者によって X にアップロードされたコンテンツのメディア ID を示します
* referenced&#95;tweets - text フィールド内にポストの URL が見つかった場合、そのポストの ID がレスポンスに含まれます

sender&#95;id、participant&#95;ids、referenced&#95;tweets.id、および attachments.media&#95;keys の [expansions](/ja/x-api/fundamentals/expansions) を使用して、これらの X オブジェクト ID を展開できます。

**ダイレクトメッセージイベントオブジェクトの取得**

**サンプルリクエスト**

この例では、1 対 1 の会話に関連付けられたイベントを取得するリクエストを作成します。このリクエストは、基本的なダイレクトメッセージイベントのフィールドに加えて、参照されているポストおよびその投稿者に関する追加フィールドを返します。次の内容を要求するクエリを作成します:

* 作成時刻や、どの会話 (dm&#95;conversation) に属しているかといった、基本的なイベント属性。
* ダイレクトメッセージを送信したアカウント ID と説明。
* 参照されている任意のポストのテキストと、そのポストが投稿された日時。
* 参照されている任意のポスト投稿者のアカウント ID と説明。

これらの属性を返すには、リクエストのクエリに次の内容を含めます:

`?dm_event.fields=id,sender_id,text,created_at,dm_conversation_id&expansions=sender_id,referenced_tweets.id&tweet.fields=created_at,text,author_id&user.fields=description`

```bash

    --header 'Authorization: Bearer $BEARER_TOKEN'
```

$BEARER&#95;TOKEN は、必ずご自身で生成した [Bearer Token](/ja/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token) に置き換えてください。

<div id="community">
  ### コミュニティ
</div>

コミュニティは、X のユーザーがつながり、共有し、自分が最も関心のある議論により近づくための専用の場です。

コミュニティ内のポストは、X 上の誰でも見ることができますが、そのコミュニティ内のメンバーだけがエンゲージしたり、議論に参加したりできます。

Community オブジェクトには、コミュニティに関する関連するメタデータが含まれます。

| name             | string          | コミュニティの名前。                                         |    |
| description      | string          | コミュニティの説明文（設定されている場合）。                             |    |
| access           | string          | コミュニティのアクセスレベル。<br /><br /> 次のいずれか：                |    |
|                  |                 | - `Public`                                         |    |
|                  |                 | - `Closed`                                         |    |
| join&#95;policy  | string          | コミュニティの参加ポリシー。<br /><br /> 次のいずれか：                 |    |

<div id="how-to-use-fields-and-expansions">
  ## フィールドとexpansionsの使い方
</div>

デフォルトでは、X API v2 のデータオブジェクトは、[fields](/ja/x-api/fundamentals/fields) や [expansions](/ja/x-api/fundamentals/expansions) パラメータを使用せずにリクエストした場合、ごく少数のデフォルトフィールドだけを含みます。このガイドでは、レスポンス内で追加のオブジェクトとフィールドを受け取るために、リクエストで `fields` および `expansions` クエリパラメータをどのように使用するかを説明します。

このガイドでは、次のポストのスクリーンショットを例に、複数のフィールドをリクエストしていきます。
 

![This image includes a screenshot of a Tweet posted by @X. You can see the Tweet text, username, published date and time, source, and public metrics. It also includes a video. ](https://cdn.cms-twdigitalassets.com/content/dam/developer-twitter/ScreenShotDataDictionaryGuide.png.twimg.1920.png)

スクリーンショットでわかるように、このポストには、ポストの投稿者、ポストのメトリクス、作成タイムスタンプ、動画、および動画の再生回数など、ポストに関連する複数の情報が表示されています。また、スクリーンショットには表示されていないものの、リクエスト可能なデータもいくつか存在します。 

API にリクエストを送信する際、デフォルトのレスポンスはシンプルで、デフォルトのポストフィールド（id と text）のみが含まれます。また、利用しているエンドポイントで返される主要オブジェクトだけが含まれ、それに関連しうる関連データオブジェクトは含まれません。

このシンプルな設計と、`fields` および `expansions` パラメータにより、ユースケースに応じて必要なフィールドだけをリクエストできるようになります。
