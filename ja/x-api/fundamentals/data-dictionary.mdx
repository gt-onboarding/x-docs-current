<div id="tweet">
  ### Tweet
</div>

ポストは、Twitter におけるあらゆる機能の基本単位です。Tweet オブジェクトには、`id`、`text`、`created_at` のような「ルートレベル」のフィールドが多数あります。Tweet オブジェクトは、`user`、`media`、`poll`、`place` など複数の子オブジェクトの「親」オブジェクトでもあります。これらのルートレベルのフィールドを Tweet オブジェクトでリクエストする際は、フィールドパラメータ `tweet.fields` を使用します。

Tweet オブジェクトは user リソース内で取得して展開できます。リクエストした Tweet に関連する追加のポストも、Tweet リソース内で取得して展開できます。オブジェクトは、user リソースで `?expansions=pinned_tweet_id`、または Tweet リソースで `?expansions=referenced_tweets.id` を指定することで、デフォルトフィールドのみを含むオブジェクトとして expansions で取得可能です。オブジェクトの情報を補完するために追加のフィールドをリクエストする場合は、フィールドパラメータ `tweet.fields` と expansions を併用してください。

| フィールド名                                           | 型            | 説明                                                                                                                 | 利用方法                                             |
| :----------------------------------------------- | :----------- | :----------------------------------------------------------------------------------------------------------------- | :----------------------------------------------- |
| **id (デフォルト)**                                   | string       | 指定されたポストの一意の識別子。                                                                                                   | このIDを使用すると、プログラムから特定のポストを取得できます。                 |
| **text（デフォルト）**                                  | 文字列          | ポストの UTF-8 形式の本文テキストです。使用可能な文字の詳細については [twitter-text](https://github.com/twitter/twitter-text/) を参照してください。         | キーワード抽出や感情分析・分類に使用できます。                          |
| **edit&#95;history&#95;tweet&#95;ids (default)** | オブジェクト       | ポストのすべてのバージョンを示す一意の識別子です。編集されていないポストには ID が 1 つだけ含まれます。編集履歴のあるポストには複数の ID が含まれます。                                  | この情報を使用してポストの編集履歴を特定できます。                        |
| **article**                                      | object       | このポストに含まれる Article のメタデータを含みます。                                                                                    | これを使用すると、Article のテキストとエンティティを取得できます。            |
| **attachments**                                  | オブジェクト       | このポストに添付されているオブジェクトの種類を指定します（添付がある場合）。                                                                             | リクエストした expansions によって返されるオブジェクトを理解するために使用します。  |
| **author&#95;id**                                | string       | このポストを投稿したユーザーを一意に識別するID。                                                                                          | Userオブジェクトをハイドレートしたり、ピアレビュー用データセットを共有したりするために使用。 |
| **card&#95;uri**                                 | 文字列          | このポストに含まれるカードのURI。                                                                                                 |                                                  |
| **community&#95;id**                             | string       | このポストが属しているコミュニティを一意に識別するID。                                                                                       |                                                  |
| **context&#95;annotations**                      | 配列           | ポストに関するコンテキストアノテーションを含みます。                                                                                         | エンティティ認識／抽出、トピック分析。                              |
| **conversation&#95;id**                          | 文字列          | 会話の起点となるオリジナルポストのポストID（ダイレクトリプライおよびそれらへの返信を含みます）。                                                                  | ポストから会話スレッドを再構築するために使用します。                       |
| **created&#95;at**                               | 日時（ISO 8601） | ポストの作成日時。                                                                                                          | 時系列解析やポストの作成時刻の把握に役立ちます。                         |
| **display&#95;text&#95;range**                   | 配列           | 表示されるテキスト範囲の開始インデックスと終了インデックスを含む配列。                                                                                | 長文のポストでデフォルトで表示されるテキスト範囲を把握するのに有用です。             |
| **edit&#95;controls**                            | object       | ポストをあとどのくらいの時間編集できるかと、残りの編集回数を示します。                                                                                | これを使ってポストが編集可能かどうかを判断します。                        |
| **entities**                                     | オブジェクト       | ポスト本文から解析して抽出されたエンティティ。Twitter Objects の entities を参照してください。                                                       | ハッシュタグやURL、メンションなどに関する補足情報が得られます。                |
| **geo**                                          | object       | ジオタグ付きポストの位置情報（場所）を示します。                                                                                           | これを使用して、ジオタグ付きポストの位置情報を特定します。                    |
| **in&#95;reply&#95;to&#95;user&#95;id**          | 文字列          | 対象のポストが返信である場合、このフィールドには元のポストの投稿者のidが含まれます。                                                                        | ポストが別のポストへの返信かどうかを判定するために使用します。                  |
| **lang**                                         | 文字列          | Twitter が検出した場合のポストの言語。                                                                                            | ポストを使用言語ごとに分類する。                                 |
| **non&#95;public&#95;metrics**                   | オブジェクト       | リクエスト時点におけるポストの非公開エンゲージメントメトリクス。ユーザーコンテキストでの認証が必要です。                                                               | ポストによって生成されたインプレッションの総数を把握します。                   |
| **note&#95;tweet**                               | object       | 280文字を超える長文ポストの全文を保持します。                                                                                           | ポストの全文を取得します。                                    |
| **organic&#95;metrics**                          | object       | リクエスト時点のポストにおける、オーガニックコンテキストで追跡されるエンゲージメントメトリクス。ユーザーコンテキストによる認証が必要です。                                              | ポストに対するオーガニックエンゲージメントを計測する。                      |
| **possibly&#95;sensitive**                       | boolean      | コンテンツがセンシティブな内容として認識される可能性があるかどうかを示します。                                                                            | 特定の種類のコンテンツの拡散を調査する。                             |
| **promoted&#95;metrics**                         | オブジェクト       | リクエスト時点におけるそのポストの、プロモーションコンテキストで追跡されたエンゲージメントメトリクスです。ユーザーコンテキストを伴う認証が必要です。                                         | ポストをプロモーションした際のエンゲージメントを測定します。                   |
| **public&#95;metrics**                           | オブジェクト       | リクエスト時点におけるポストの公開エンゲージメントメトリクス。                                                                                    | ポストのエンゲージメントを測定する。                               |
| **referenced&#95;tweets**                        | 配列           | このポストが参照しているポスト（リポスト、引用ポスト、返信など）のリストです。                                                                            | リツイートなどを含めた会話の関係性を把握する。                          |
| **reply&#95;settings**                           | string       | 指定されたポストに誰が返信できるかを示します。設定可能な値は &quot;everyone&quot;、&quot;mentioned&#95;users&quot;、&quot;followers&quot; のいずれかです。 | ポストに対して誰が返信できるかの設定を判別します。                        |
| **withheld**                                     | オブジェクト       | [制限されたコンテンツ](https://help.x.com/en/rules-and-policies/tweet-withheld-by-country) に関する制限の詳細を含みます。                   |                                                  |
| **scopes**                                       | object       | ポストの公開範囲の詳細を含みます。                                                                                                  | 誰がポストを閲覧できるかを示します。プロモーションされたポストに対してのみ返されます。      |
| **media&#95;metadata**                           | 配列           | ポストに添付されたメディアのメタデータを含みます。                                                                                          | ポストに添付されたメディアの `alt_text` など、追加のメタデータを取得します。     |

**ポストオブジェクトの取得**

**リクエスト例**

次のリクエストでは、[Tweets lookup](/ja/resources/fundamentals/rate-limits) エンドポイントで対象のポストについてフィールドをリクエストしています。必ず `$BEARER_TOKEN` をご自身で生成した [Bearer Token](/ja/resources/fundamentals/authentication) に置き換えてください。

```bash
curl --request GET 'https://api.x.com/2/tweets?ids=1212092628029698048&tweet.fields=attachments,author_id,context_annotations,created_at,entities,geo,id,in_reply_to_user_id,lang,possibly_sensitive,public_metrics,referenced_tweets,text,withheld&expansions=referenced_tweets.id' --header 'Authorization: Bearer $BEARER_TOKEN'
```

<div id="user">
  ### User
</div>

user オブジェクトには、参照されているユーザーを表す Twitter ユーザーアカウントのメタデータが含まれます。user オブジェクトは、[users lookup](/ja/x-api/users/lookup/introduction) エンドポイントで返される主要なオブジェクトです。このエンドポイントで追加のユーザーフィールドをリクエストする場合は、`user.fields` パラメータを使用します。

user オブジェクトは、子オブジェクトとしてポストオブジェクト内に含まれ、展開される場合もあります。オブジェクトは、`?expansions=author_id` または `?expansions=in_reply_to_user_id` を指定することで、デフォルトフィールドのみを含む省略版オブジェクトとして取得できます。オブジェクトを完全な形にするために追加のフィールドをリクエストする場合は、`user.fields` フィールドパラメータと併せて expansions を使用してください。

<div id="space">
  ### Space
</div>

Spaces は、ライブ音声会話を通じた表現と対話を可能にします。Space データ辞書には Space に関する関連メタデータが含まれており、すべての詳細はリアルタイムで更新されます。

User オブジェクトは user リソースで取得および展開できます。これらのオブジェクトは、`expansions` クエリパラメータに `host_ids`、`creator_id`、`speaker_ids`、`mentioned_user_ids` の少なくとも 1 つを追加することで、展開して取得できます。

ポストとは異なり、Space は一時的なコンテンツであり、終了後、または作成者によってキャンセルされた場合には利用できなくなります。アプリで Space データを処理する場合、常に最新の情報を返す責任があり、プラットフォーム上で利用できなくなったデータは削除しなければなりません。[Spaces lookup endpoints](/ja/x-api/spaces/lookup/introduction) は、ユーザーの期待と意図を尊重できていることを確認するのに役立ちます。

</div>

List オブジェクトには、参照されているリストを記述する [Twitter Lists](https://help.x.com/en/using-twitter/twitter-lists) のメタデータが含まれます。List オブジェクトは、List ルックアップエンドポイントで返される主要なオブジェクトです。このエンドポイントで追加の List フィールドをリクエストする場合は、`list.fields` フィールドパラメータを使用します。

List オブジェクトが他のデータオブジェクトの子として含まれることはありません。ただし、ユーザーオブジェクトは user リソース内で取得および展開できます。これらのオブジェクトは、`expansions` クエリパラメータに `owner_id` を追加することで展開オブジェクトとして利用できます。主要な List オブジェクトを補完するために追加フィールドをリクエストする場合は、この展開を `list.fields` フィールドパラメータと併用し、展開オブジェクトを補完するために `user.fields` を使用します。

| Field Value        | Type            | Description                                                                                                                    | How it can be used                                       |
| :----------------- | :-------------- | :----------------------------------------------------------------------------------------------------------------------------- | :------------------------------------------------------- |
| id (default)       | string          | このリストの一意の識別子。<br />`"id": "2244994945"`                                                                                        | 特定のリストに関する情報をプログラムから取得するために使用します。                        |
| name (default)     | string          | リストを作成するときに定義されたリスト名。<br />`"name": "Twitter Lists"`                                                                           |                                                          |
| created&#95;at     | date (ISO 8601) | リストが作成された UTC 日時。<br />`"created_at": "2013-12-14T04:35:55.000Z"`                                                              | リストがいつ作成されたか、どのくらい前のリストかを把握できます。                         |
| description        | string          | ユーザーにリストの内容を知らせるための簡潔な説明。<br />`"description": "People that are active members of the Bay area cycling community on Twitter."` |                                                          |
| follower&#95;count | integer         | このリストをフォローしているユーザー数。<br />`"follower_count": 198`                                                                              |                                                          |
| member&#95;count   | integer         | このリストに含まれているメンバー数。<br />`"member_count": 60`                                                                                   |                                                          |
| private            | boolean         | リストが非公開かどうかを示します。<br />`"private": false`                                                                                      |                                                          |
| owner&#95;id       | string          | このリストの所有者の一意の識別子。<br />`"owner_id": "1255542774432063488"`                                                                     | このユーザーが他にどのリストを所有しているかを調べたり、ユーザーオブジェクトを展開したりするために使用できます。 |

<div id="direct-message-events">
  ### ダイレクトメッセージイベント
</div>

ダイレクトメッセージ（DM）の会話は、複数のイベントで構成されます。現在、X API v2 では 3 種類のイベントタイプがサポートされています: MessageCreate、ParticipantsJoin、ParticipantsLeave です。

DM イベントオブジェクトは [Direct Message lookup](/ja/x-api/direct-messages/lookup/introduction) エンドポイントから返され、[Manage Direct Messages](/ja/x-api/direct-messages/lookup/introduction) エンドポイントでダイレクトメッセージが正常に作成されると、MessageCreate イベントが作成されます。

DM イベントをリクエストする際、デフォルトでは 3 つのイベントオブジェクト属性（またはフィールド）が含まれます: id、event&#95;type、text です。追加のイベントフィールドを受け取るには、[fields](/ja/x-api/fundamentals/fields) パラメータ dm&#95;event.fields を使用して他のフィールドを指定します。利用可能なその他のイベントフィールドには、次のものが含まれます: dm&#95;conversation&#95;id、created&#95;at、sender&#95;id、attachments、participant&#95;ids、referenced&#95;tweets。

これらのフィールドの一部には、ダイレクトメッセージイベントに関連する他の X オブジェクトの ID が含まれます:

* sender&#95;id - メッセージを送信したアカウント、またはグループ会話に参加者を招待したアカウントの ID
* partricipants&#95;ids - アカウント ID の配列。ParticipantsJoin および ParticipantsLeave イベントの場合、この配列にはイベントを作成したアカウント 1 件分の ID が含まれます
* attachments - 送信者が X にアップロードしたコンテンツのメディア ID が含まれます
* referenced&#95;tweets - text フィールド内にポストの URL が見つかった場合、そのポストの ID がレスポンスに含まれます

sender&#95;id、participant&#95;ids、referenced&#95;tweets.id、attachments.media&#95;keys の [expansions](/ja/x-api/fundamentals/expansions) を使用して、これらの X オブジェクト ID を展開できます。

|     |     |     |     |
| :--- | :--- | :--- | :--- |
| **フィールド値** | **型** | **説明** | **利用方法** |
| id (default) | string | イベントの一意の識別子。<br /><br />&quot;id&quot;: &quot;1050118621198921728&quot; | これを使用して、特定の会話イベントをプログラムから取得できます（v1.1 エンドポイントで利用可能）。 |
| event&#95;type (default) | string | イベントの種類を表します。現在、次の 3 種類がサポートされています。<br /><br />* MessageCreate<br />    <br />* ParticipantsJoin<br />    <br />* ParticipantsLeave<br />    <br /><br />&quot;event&#95;type&quot;: &quot;MessageCreate&quot; | 会話の履歴を取得する際に、メッセージが作成されたタイミングや、グループ会話では参加者が参加・退出したタイミングを把握するために使用します。すべての GET メソッドで、`event_type=` クエリパラメータを使用して特定のイベントタイプでフィルタリングできます。 |
| text (default) | string | ダイレクトメッセージの実際の UTF-8 テキスト。<br /><br />&quot;text&quot;: &quot;Hello, just you!&quot; | チャットボットでは、メッセージ内容を解析し自動応答を決定するために使用できます。会話検索機能を構築するためにも利用できます。 |
| entities          | object | ダイレクトメッセージのテキストから解析されたエンティティ。                                       | ハッシュタグ、URL、メンションなどに関する追加情報を提供します。 |
| sender&#95;id | string | イベントを作成したユーザーの ID。レスポンスでこのオブジェクトを展開するには、`sender_id` を expansion に含め、`user.fields` クエリパラメータを使用して取得したい User オブジェクト属性を指定します。<br /><br />&quot;sender&#95;id&quot;: &quot;906948460078698496&quot; | MessageCreate または ParticipantsJoin イベントを作成したユーザーの User オブジェクトを取得します。 |

**ダイレクトメッセージイベントオブジェクトの取得**

**サンプルリクエスト**

この例では、1 対 1 の会話に関連付けられたイベントを取得するリクエストを作成します。このリクエストは、基本的なダイレクトメッセージイベントフィールドに加え、参照されているポストとその投稿者に関する追加フィールドを返します。次の内容を要求するクエリを作成します。

* 作成時刻や、どの会話 (`dm_conversation`) の一部であるかといった、基本的なイベント属性。
* ダイレクトメッセージを送信したアカウントの ID と説明。
* 参照されているポストのテキストと、そのポストが投稿された時刻。
* 参照されているポストの投稿者のアカウント ID と説明。

これらの属性を返すには、リクエストクエリに次の内容を含めます。

`?dm_event.fields=id,sender_id,text,created_at,dm_conversation_id&expansions=sender_id,referenced_tweets.id&tweet.fields=created_at,text,author_id&user.fields=description`

```bash
      curl --request GET 'https://api.x.com/2/dm_conversations/with/:participant_id/dm_events?tweet.fields=created_at,text,author_id&user.fields=description&expansions=sender_id,participant_ids,referenced_tweets.id&dm_event.fields=id,sender_id,text,participant_ids,created_at,'
    --header 'Authorization: Bearer $BEARER_TOKEN'
```

必ず $BEARER&#95;TOKEN を、ご自身で生成した [Bearer Token](/ja/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token) に置き換えてください。

<div id="community">

<div id="how-to-use-fields-and-expansions">
  ## フィールドとexpansionsの使い方
</div>

デフォルトでは、[fields](/ja/x-api/fundamentals/fields) や [expansions](/ja/x-api/fundamentals/expansions) パラメータを指定せずにリクエストを行った場合、X API v2 のデータオブジェクトにはごく少数のデフォルトフィールドのみが含まれます。このガイドでは、レスポンス内で追加のオブジェクトやフィールドを受け取るために、リクエストで `fields` および `expansions` クエリパラメータをどのように使用するかを説明します。

このガイドでは、次のポストのスクリーンショットを例に、いくつかのフィールドをリクエストする方法を説明します。
 

![この画像には、@X が投稿したポストのスクリーンショットが含まれています。ポスト本文、ユーザー名、投稿日と時刻、ソース、公開メトリクスを確認できます。動画も含まれています。](https://cdn.cms-twdigitalassets.com/content/dam/developer-twitter/ScreenShotDataDictionaryGuide.png.twimg.1920.png)

スクリーンショットからわかるように、このポストには、ポストの投稿者、ポストのメトリクス、作成タイムスタンプ、動画、および動画の再生回数など、いくつかの目に見える情報があります。一方で、スクリーンショット上には表示されていないものの、リクエスト可能なデータも複数存在します。 

API にリクエストを送信するとき、デフォルトのレスポンスはシンプルで、デフォルトのポストフィールド（`id` と `text`）のみが含まれます。また、使用しているエンドポイントで返される主要オブジェクトのみを受け取り、その主要オブジェクトに関連する可能性のある関連データオブジェクトは取得されません。

このシンプルな設計と `fields` および `expansions` パラメータを組み合わせることで、ユースケースに応じて必要なフィールドだけをリクエストできるようになります。
