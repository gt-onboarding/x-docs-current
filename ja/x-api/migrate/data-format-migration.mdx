<div id="introduction">
  ## はじめに
</div>

X API v2 のリリースに伴い、新しいデータレスポンスフォーマットと、さまざまなオブジェクトやフィールドをリクエストする新しい方法を採用しました。これを便宜上、X API v2 フォーマットと呼びます。 

この「一般的な違い」セクションでは、Standard プランおよび Enterprise ユーザーに関連するいくつかの変更点について確認できます。あわせて、[Standard v1.1 の Native フォーマット](https://developer.x.com/en/docs/x-api/v1/data-dictionary/overview)、Enterprise 向けの [Native Enriched フォーマット](/ja/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#native-enriched-tweet-object)、および Enterprise 向けの [Activity Streams フォーマット](/ja/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#activity-object) それぞれに特化したガイドも用意しており、フィールドのマッピングや、新しい v2 のフィールドをリクエストする際に使用する必要があるフィールドおよび expansions について説明しています。 

* [Native フォーマットから X API v2 へ (Standard v1.1)](/ja/x-api/migrate/data-format-migration#migrating-from-standard-v1-1s-data-format-to-v2) 
* [Native Enriched から X API v2 へ (Enterprise)](/ja/x-api/migrate/data-format-migration#migrating-from-native-enriched-data-format-to-v2)
* [Activity Streams から X API v2 へ (Enterprise)](/ja/x-api/migrate/data-format-migration#migrating-from-activity-streams-data-format-to-v2)

[X API v1.1 のデータフォーマット](https://developer.x.com/en/docs/x-api/v1/data-dictionary/overview) と [X API v2 フォーマット](/ja/x-api/introduction) の違いをすばやく確認するのに役立つ、[ビジュアルデータフォーマット移行ツール](/ja/x-api/migrate/data-format-migration#visual-data-format-migration-tool) もあわせてご利用いただけます。

<div id="general-differences">
  ### 全体的な違い
</div>

<div id="requesting-objects-and-fields">
  #### オブジェクトとフィールドのリクエスト
</div>

pre-v2 エンドポイントと v2 の主な変更点の 1 つは、新しいバージョンではデフォルトでごく一部のフィールドしか返さないのに対して、Standard、Premium、Enterprise の各エンドポイントはほとんどのフィールドをデフォルトで返すことです。新しいバージョンでは、デフォルト以上の追加データを明示的にリクエストするために [fields](/ja/x-api/fundamentals/fields) と [expansions](/ja/x-api/fundamentals/expansions) というパラメータを使用します。これにより、不要なフィールドを取得することなく、必要なデータだけをリクエストできます。 

リクエストしたフィールドが主要なデータオブジェクトに関連している場合、それらはデフォルト値とともにその主要なデータオブジェクト内に返されます。ただし、expansions パラメータを使用して拡張オブジェクトをリクエストした場合、副次オブジェクトは新しい includes オブジェクト内に返されます。includes オブジェクト内の拡張オブジェクトは、両方に含まれる id フィールドを使用して主要オブジェクトと紐付けることができます。

たとえば、v2 の [Post lookup](/ja/x-api/posts/lookup/introduction) エンドポイントを使用していて、リクエストに expansions=author&#95;id パラメータを含めた場合、主要なポストオブジェクト内に author&#95;id フィールドが返されると同時に、includes オブジェクト内にポストごとに 1 つの user オブジェクトも返されます。各 user オブジェクトにはデフォルトの id フィールドが含まれており、これを使用して user オブジェクトを対応するポストオブジェクトに紐付けることができます。以下はその例です。

```
{
  "data": [
    {
      "author_id": "2244994945",
      "id": "1397568983931392004",
      "text": "The Twitter Developer Platform. Ooh la la! https://t.co/iGTdPXBfOv https://t.co/Ze8z8EODdg"
    }
  ],
  "includes": {
    "users": [
      {
        "id": "2244994945",
        "name": "Twitter Dev",
        "username": "TwitterDev"
      }
    ]
  }
}
```

<div id="updated-json-design">
  #### 更新された JSON 設計
</div>

特定のフィールドのリクエスト方法の変更に加えて、X API v2 では、API によって返されるオブジェクト、たとえば [Post](/ja/x-api/fundamentals/data-dictionary#tweet) オブジェクトや [user](/ja/x-api/fundamentals/data-dictionary#user) オブジェクトに対して、新しい JSON 設計も導入しています。

* JSON のルートレベルでは、従来の標準エンドポイントは **statuses** 配列で Post オブジェクトを返しますが、X API v2 は Post オブジェクトを含む **data** 配列を返します。
* リツイート（Retweeted）および引用（Quoted）の「statuses」を参照する代わりに、X API v2 の JSON ではリツイートおよび引用のポストを参照します。**contributors** や **user.translator&#95;type** などの多くのレガシーおよび非推奨フィールドは削除されます。
* **favorites**（Post オブジェクト内）と **favourites**（user オブジェクト内）の両方を使用する代わりに、X API v2 では like という用語を使用します。
* X では、値を持たない JSON 値（たとえば null）はペイロードに書き込まないという規約を採用しています。Post と user の属性は、null ではない値を持つ場合にのみペイロードに含まれます。 
   

<div id="new-v2-fields">
  #### 新しい v2 フィールド
</div>

また、[Post object](/ja/x-api/fundamentals/data-dictionary#tweet) に、次のような新しいフィールド群も導入しました。

* [conversation&#95;id](/ja/x-api/fundamentals/conversation-id) フィールド
* context と entities を含む、2 つの新しい [annotations](/ja/x-api/fundamentals/post-annotations) フィールド
* いくつかの新しい [metrics](/ja/x-api/fundamentals/metrics) フィールド
* 特定のポストに誰が返信できるかを示す、新しい reply&#95;setting フィールド

<div id="migrating-from-standard-v11s-data-format-to-v2">
  ### 標準 v1.1 のデータフォーマットから v2 への移行
</div>

まだ読んでいない場合は、まず [データフォーマット移行](/ja/x-api/migrate/data-format-migration) の概要に目を通すことをおすすめします。[X API v1.1 データフォーマット](https://developer.x.com/en/docs/x-api/v1/data-dictionary/overview) と [X API v2 フォーマット](/ja/x-api/fundamentals/data-dictionary) の違いをすばやく確認するには、[ビジュアルデータフォーマット移行ツール](/ja/x-api/migrate/data-format-migration#visual-data-format-migration-tool) も役立ちます。

standard v1.1 data format（ネイティブフォーマットとも呼ばれます）は、[standard v1.1](https://developer.x.com/en/docs/twitter-api/v1) エンドポイントで提供される主要なフォーマットです。

Premium 製品を利用している場合は、[ネイティブ Enriched ガイド](/ja/x-api/migrate/data-format-migration#migrating-from-native-enriched-data-format-to-v2) を参照してください。Enterprise 顧客は、Gnip コンソールでの設定方法に応じて、ネイティブ Enriched または Activity Streams を使用している可能性があります。

<div id="migrating-from-native-enriched-data-format-to-v2">
  ### Native Enriched データ形式から v2 への移行
</div>

Native Enriched データ形式は、[enterprise](/ja/x-api/enterprise-gnip-2.0/enterprise-gnip) Enterprise 製品で使用されています。

Native Enriched データ形式は、編集されたポストのメタデータを提供できるように更新されました。編集されたポストのメタデータの詳細については、[Edit Tweets fundamentals](/ja/x-api/enterprise-gnip-2.0/fundamentals/edit-tweets) ページを参照してください。

標準の v1.1 エンドポイントを使用している場合は、[standard v1.1 to v2 guide](/ja/x-api/migrate/data-format-migration#migrating-from-standard-v1-1s-data-format-to-v2) を参照してください。Activity Streams を利用する Enterprise 製品を使用している場合は、[Activity Streams to v2](/ja/x-api/migrate/data-format-migration#migrating-from-activity-streams-data-format-to-v2) ガイドも用意しています。

X API v2 では、[ポスト](/ja/x-api/fundamentals/data-dictionary#tweet) オブジェクトと [ユーザー](/ja/x-api/fundamentals/data-dictionary#user) オブジェクト向けに新しい JSON 設計が導入されています。

* JSON のルートレベルでは、Native Enriched 形式は results 配列でポストオブジェクトを返しますが、X API v2 は data 配列でポストオブジェクトを返します。 
* ポストオブジェクト内の favorites と、ユーザーオブジェクト内の favourites の両方を使用する代わりに、X API v2 では like という用語を使用します。 
* X は、値を持たない JSON 値（たとえば null）はペイロードに書き出さないという規約を採用しています。ポストおよびユーザーの属性は、値が null でない場合にのみ含まれます。 
* v2 のすべての id フィールドは文字列形式になります。
   

新しい JSON 形式への変更に加えて、ポストオブジェクトには次のような新しいフィールド群も導入されています。

* conversation&#95;id
* reply&#95;settings
* メディアの alt&#95;text
* context と entities を含む 2 つの新しい [annotations](/ja/x-api/fundamentals/post-annotations) フィールド
* 複数の新しい [metrics](/ja/x-api/fundamentals/metrics) フィールド
* 複数の新しい [polls](/ja/x-api/fundamentals/data-dictionary#poll) フィールド
   

多くのレガシーおよび非推奨フィールドが削除されています。

* contributors
* 特定の entities.media および extended&#95;entities.media フィールド
* filter&#95;level
* timestamp&#95;ms
* truncated

<div id="migrating-from-activity-streams-data-format-to-v2">
  ### Activity Streams データ形式から v2 への移行
</div>

Activity Streams データ形式は、当社の [Enterprise](/ja/x-api/enterprise-gnip-2.0/enterprise-gnip) 製品で利用できます。

Activity Streams データ形式は、編集済みポストのメタデータを提供するよう更新されています。編集機能付きポストのメタデータの詳細については、[Edit Tweets fundamentals](/ja/x-api/enterprise-gnip-2.0/fundamentals/edit-tweets) ページを参照してください。

標準の v1.1 エンドポイントを使用している場合は、[standard v1.1 to v2 guide](/ja/x-api/migrate/overview) を参照してください。premium エンドポイント、または Enterprise 向け Native Enriched 形式を使用している場合は、[Native Enriched to v2 guide](/ja/x-api/migrate/data-format-migration#migrating-from-native-enriched-data-format-to-v2) を参照してください。

X API v2 では、[Post](/ja/x-api/fundamentals/data-dictionary#tweet) オブジェクトおよび [user](/ja/x-api/fundamentals/data-dictionary#user) オブジェクトに対して新しい JSON 設計を導入しています。

* JSON のルートレベルでは、Activity Streams 形式は results 配列内に Tweet オブジェクトを返しますが、X API v2 は data 配列を返します。
* Retweeted と Quoted の「activities」を参照する代わりに、X API v2 の JSON では Retweeted と Quoted のポストを参照します。
* Tweet オブジェクトの favorites と user オブジェクトの favourites の両方を使用する代わりに、X API v2 では like という用語を使用します。
* Twitter は、値を持たない JSON 値（たとえば null）はペイロードに書き出さないという規約を採用しています。Tweet および user の属性は、非 null の値を持つ場合にのみ含まれます。
* v2 のすべての id フィールドは文字列形式になります。
   

新しい JSON 形式への変更に加えて、Tweet オブジェクトに次の新しいフィールド群も導入しました。

* conversation&#95;id
* reply&#95;settings
* メディアの alt&#95;text
* context および entities を含む、2 つの新しい [annotations](/ja/x-api/fundamentals/post-annotations) フィールド
* 複数の新しい [metrics](/ja/x-api/fundamentals/metrics) フィールド
* 複数の新しい [polls](/ja/x-api/fundamentals/data-dictionary#poll) フィールド
   

多くのレガシーおよび非推奨フィールドは削除または置き換えられます。

* display&#95;text&#95;range
* generator
* gnip
* link
* objectType
* provider
* twitter&#95;entities.symbols は data.entities.cashtags に置き換え
* 特定の twitter&#95;extended&#95;entities.media および twitter&#95;entities.media フィールド
* twitter&#95;filter&#95;level
* twitterTimeZone
* verb

<div id="visual-data-format-migration-tool">
  ### ビジュアルデータ形式移行ツール
</div>

ビジュアルデータ形式移行ツールは、特定のポストオブジェクトまたはユーザーオブジェクトについて、[X API v1.1 のデータ形式](https://developer.x.com/en/docs/x-api/v1/data-dictionary/overview)から [X API v2 形式](/ja/x-api/fundamentals/data-dictionary) へマッピングされるフィールドを表示する Web アプリケーションです。ポスト ID またはユーザー ID のいずれかをアプリに指定することで、このマッピングを確認できます。

このアプリを使用するには、Twitter アカウントでログインする必要があります。
