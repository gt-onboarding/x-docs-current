<div id="introduction">
  ## はじめに
</div>

X API v2 のリリースに伴い、新しいデータレスポンス形式と、さまざまなオブジェクトやフィールドのリクエスト方法を採用しました。本ドキュメントでは、これを単に X API v2 フォーマットと呼びます。 

「一般的な違い」セクションでは、Standard プランおよび Enterprise プランのユーザーに関連するいくつかの変更点について確認できます。さらに、[standard v1.1 Native format](https://developer.x.com/en/docs/x-api/v1/data-dictionary/overview)、Enterprise 向けの [Native Enriched format](/ja/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#native-enriched-tweet-object)、および Enterprise 向けの [Activity Streams format](/ja/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#activity-object) それぞれに関する専用ガイドも用意しており、フィールドのマッピングや、新しい v2 のフィールドをリクエストする際に使用すべきフィールドや expansions について説明しています。 

* [Native format から X API v2 への移行（standard v1.1）](/ja/x-api/migrate/data-format-migration#migrating-from-standard-v1-1s-data-format-to-v2) 
* [Native Enriched から X API v2 への移行（Enterprise）](/ja/x-api/migrate/data-format-migration#migrating-from-native-enriched-data-format-to-v2)
* [Activity Streams から X API v2 への移行（Enterprise）](/ja/x-api/migrate/data-format-migration#migrating-from-activity-streams-data-format-to-v2)

また、[visual data format migration tool](/ja/x-api/migrate/data-format-migration#visual-data-format-migration-tool) を使用すると、[X API v1.1 data format](https://developer.x.com/en/docs/x-api/v1/data-dictionary/overview) と [X API v2 format](/ja/x-api/introduction) の違いを素早く確認できますので、併せてご活用ください。

<div id="general-differences">
  ### 主な違い
</div>

<div id="requesting-objects-and-fields">
  #### オブジェクトとフィールドのリクエスト
</div>

v2 以前のエンドポイントと v2 の最大の違いの 1 つは、新しいバージョンではデフォルトで返されるフィールドがごく一部に限られているのに対し、Standard、Premium、および Enterprise エンドポイントではほとんどのフィールドがデフォルトで返される点です。新しいバージョンでは、デフォルトを超えた追加のデータを明示的にリクエストするために、[fields](/ja/x-api/fundamentals/fields) パラメータと [expansions](/ja/x-api/fundamentals/expansions) パラメータを使用します。これにより、不要なフィールドを取り込むことなく、必要なデータだけをリクエストできます。

リクエストしたフィールドがプライマリのデータオブジェクトに関連するものであれば、それらはデフォルト値とともにプライマリのデータオブジェクト内に返されます。ただし、expansions パラメータを使って拡張オブジェクトをリクエストした場合、セカンダリオブジェクトは新しい includes オブジェクト内に返されます。includes オブジェクト内の拡張オブジェクトは、両方に含まれる `id` フィールドを使用してプライマリオブジェクトと対応付けることができます。

たとえば、v2 の [Post lookup](/ja/x-api/posts/lookup/introduction) エンドポイントを使用していて、リクエストに expansions=author&#95;id パラメータを含めた場合、プライマリのポストオブジェクト内に author&#95;id フィールドが返されると同時に、includes オブジェクト内にポストごとに 1 つの user オブジェクトが返されます。それぞれの user オブジェクトには、ユーザーオブジェクトをポストオブジェクトに紐付けるために使用できるデフォルトの `id` フィールドが含まれます。以下はその例です。

```
{
  "data": [
    {
      "author_id": "2244994945",
      "id": "1397568983931392004",
      "text": "The Twitter Developer Platform. Ooh la la! https://t.co/iGTdPXBfOv https://t.co/Ze8z8EODdg"
    }
  ],
  "includes": {
    "users": [
      {
        "id": "2244994945",
        "name": "Twitter Dev",
        "username": "TwitterDev"
      }
    ]
  }
}
```

<div id="updated-json-design">
  #### 更新された JSON 設計
</div>

特定のフィールドのリクエスト方法の変更に加えて、X API v2 では、API から返されるオブジェクト（[Post](/ja/x-api/fundamentals/data-dictionary#tweet) オブジェクトや [user](/ja/x-api/fundamentals/data-dictionary#user) オブジェクトを含む）に対して新しい JSON 設計も導入しています。

* JSON のルートレベルでは、標準のエンドポイントは **statuses** 配列で Post オブジェクトを返しますが、X API v2 は **data** 配列で返します。 
* Retweeted および Quoted の「statuses」を参照する代わりに、X API v2 の JSON では Retweeted および Quoted のポストを参照します。**contributors** や **user.translator&#95;type** など、多くのレガシーおよび非推奨フィールドは削除されます。 
* **favorites**（Post オブジェクト内）と **favourites**（user オブジェクト内）の両方を使用する代わりに、X API v2 では「like」という用語を使用します。 
* X では、値を持たない JSON 値（たとえば null）はペイロードに書き出さないという規約を採用しています。Post および user の属性は、null ではない値を持つ場合にのみ含まれます。 
   

<div id="new-v2-fields">
  #### 新しい v2 フィールド
</div>

[ポストオブジェクト](/ja/x-api/fundamentals/data-dictionary#tweet) に対して新しいフィールド群も導入しました。次のものが含まれます。

* [conversation&#95;id](/ja/x-api/fundamentals/conversation-id) フィールド
* context と entities を含む 2 つの新しい [annotations](/ja/x-api/fundamentals/post-annotations) フィールド
* 複数の新しい [metrics](/ja/x-api/fundamentals/metrics) フィールド
* 指定したポストに対して誰が返信できるかを示す、新しい reply&#95;setting フィールド

<div id="migrating-from-standard-v11s-data-format-to-v2">
  ### 標準 v1.1 のデータ形式から v2 への移行
</div>

まだお読みでない場合は、まず[データ形式の移行](/ja/x-api/migrate/data-format-migration)の概要に目を通すことをおすすめします。[X API v1.1 データ形式](https://developer.x.com/en/docs/x-api/v1/data-dictionary/overview)と [X API v2 形式](/ja/x-api/fundamentals/data-dictionary)の違いをすばやく確認するには、[ビジュアルデータ形式移行ツール](/ja/x-api/migrate/data-format-migration#visual-data-format-migration-tool)も役立ちます。

標準 v1.1 データ形式はネイティブ形式とも呼ばれ、[standard v1.1](https://developer.x.com/en/docs/twitter-api/v1) エンドポイントで提供される主要な形式です。

Premium 製品を利用している場合は、[ネイティブ拡張ガイド](/ja/x-api/migrate/data-format-migration#migrating-from-native-enriched-data-format-to-v2)を参照してください。Enterprise クライアントは、Gnip コンソールでの設定方法に応じて、ネイティブ拡張またはアクティビティストリームを使用している場合があります。

<div id="migrating-from-native-enriched-data-format-to-v2">
  ### Native Enriched データフォーマットから v2 への移行
</div>

Native Enriched データフォーマットは、当社の [Enterprise](/ja/x-api/enterprise-gnip-2.0/enterprise-gnip) プロダクトで使用されています。

Native Enriched データフォーマットは、編集済みポストのメタデータを提供できるように更新されています。編集済みポストのメタデータの詳細については、[Edit Tweets fundamentals](/ja/x-api/enterprise-gnip-2.0/fundamentals/edit-tweets) ページをご覧ください。

標準の v1.1 エンドポイントを使用している場合は、[standard v1.1 to v2 guide](/ja/x-api/migrate/data-format-migration#migrating-from-standard-v1-1s-data-format-to-v2) を参照してください。Activity Streams を利用した Enterprise プロダクトを使用している場合は、[Activity Streams to v2](/ja/x-api/migrate/data-format-migration#migrating-from-activity-streams-data-format-to-v2) ガイドも用意しています。

X API v2 では、[ポスト](/ja/x-api/fundamentals/data-dictionary#tweet) オブジェクトと [user](/ja/x-api/fundamentals/data-dictionary#user) オブジェクトに対して新しい JSON 設計が導入されています。

* JSON のルートレベルでは、Native Enriched フォーマットは results 配列でポストオブジェクトを返しますが、X API v2 は data 配列で返します。 
* Tweet オブジェクトの favorites と user オブジェクトの favourites を併用するのではなく、X API v2 では like という用語を使用します。 
* X では、値が存在しない JSON 値（例: null）はペイロードに書き出さない、という規約を採用しています。ポストおよび user の属性は、値が null でない場合にのみ含まれます。 
* v2 のすべての id フィールドは文字列形式になります
   

新しい JSON フォーマットに対する変更に加えて、ポストオブジェクトには次のような新しいフィールドセットも導入されています。

* conversation&#95;id
* reply&#95;settings
* メディアの alt&#95;text
* context と entities を含む 2 つの新しい [annotations](/ja/x-api/fundamentals/post-annotations) フィールド
* 複数の新しい [metrics](/ja/x-api/fundamentals/metrics) フィールド
* 複数の新しい [polls](/ja/x-api/fundamentals/data-dictionary#poll) フィールド
   

多くのレガシーおよび非推奨フィールドは削除されます。

* contributors
* 一部の entities.media および extended&#95;entities.media フィールド
* filter&#95;level
* timestamp&#95;ms
* truncated

<div id="migrating-from-activity-streams-data-format-to-v2">
  ### Activity Streams データ形式から v2 への移行
</div>

Activity Streams データ形式は、[Enterprise](/ja/x-api/enterprise-gnip-2.0/enterprise-gnip) 製品で利用できます。

Activity Streams データ形式は、編集済みポストのメタデータを提供できるように更新されています。Edit Tweet メタデータの詳細については、[Edit Tweets fundamentals](/ja/x-api/enterprise-gnip-2.0/fundamentals/edit-tweets) ページを参照してください。

標準の v1.1 エンドポイントを使用している場合は、[standard v1.1 to v2 guide](/ja/x-api/migrate/overview) を参照してください。premium エンドポイント、または Enterprise 向けの Native Enriched 形式を使用している場合は、[Native Enriched to v2 guide](/ja/x-api/migrate/data-format-migration#migrating-from-native-enriched-data-format-to-v2) を参照してください。

X API v2 では、[Post](/ja/x-api/fundamentals/data-dictionary#tweet) および [user](/ja/x-api/fundamentals/data-dictionary#user) オブジェクト向けに新しい JSON 設計が導入されています。

* JSON のルートレベルでは、Activity Streams 形式は results 配列内にポストオブジェクトを返しますが、X API v2 は data 配列内に返します。 
* Retweeted と Quoted の「activities」を参照する代わりに、X API v2 の JSON では Retweeted および Quoted のポストを参照します。 
* ポストオブジェクトでの favorites と user オブジェクトでの favourites の両方を使用する代わりに、X API v2 では like という用語を使用します。 
* X では、値を持たない JSON 値（たとえば null）はペイロードに書き込まないという慣例を採用しています。ポストおよび user の属性は、非 null の値を持つ場合にのみ含まれます。 
* v2 のすべての id フィールドは文字列形式になります。
   

新しい JSON 形式への変更に加えて、ポストオブジェクトには次のような新しいフィールド群も導入されています。

* conversation&#95;id
* reply&#95;settings
* メディアに付与する alt&#95;text
* context と entities を含む、2 つの新しい [annotations](/ja/x-api/fundamentals/post-annotations) フィールド
* 複数の新しい [metrics](/ja/x-api/fundamentals/metrics) フィールド
* 複数の新しい [polls](/ja/x-api/fundamentals/data-dictionary#poll) フィールド
   

多くのレガシーおよび非推奨フィールドは削除または置き換えられています。

* display&#95;text&#95;range
* generator
* gnip
* link
* objectType
* provider
* twitter&#95;entities.symbols は data.entities.cashtags に置き換え
* 一部の twitter&#95;extended&#95;entities.media および twitter&#95;entities.media フィールド
* twitter&#95;filter&#95;level
* twitterTimeZone
* verb
