これら3つのタイムラインエンドポイントは、編集されたポストに対応しています。これらのエンドポイントは常に、最新の編集済みバージョンと編集履歴を返します。30分間の編集ウィンドウ経過後に取得されたポストは、その最終バージョンを表します。編集メタデータには、その履歴に含まれるすべてのポストのIDからなる配列が含まれます。編集履歴のないポストの場合、この配列には1つのIDのみが格納されます。編集されたことがあるポストの場合、この配列には複数のIDが含まれ、編集された順序を反映する昇順に並び、配列の最後の要素が最新バージョンになります。ポストの編集機能の仕組みについて詳しくは、[Edit Posts fundamentals](/ja/x-api/fundamentals/edit-posts) ページを参照してください。

<div id="reverse-chronological-home-timeline">
  ## 逆時系列のホームタイムライン
</div>

このエンドポイントを使用すると、認証済みユーザーおよびそのユーザーがフォローしているアカウントによって投稿された最新のポスト、リポスト、返信を取得できます。 

ユーザーに代わってリクエストを行うため、このエンドポイントには [OAuth 2.0 Authorization Code Flow with PKCE](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) または [OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) を使用して認証する必要があります。このエンドポイントには、ユーザーごとに 15 分間あたり 180 リクエストのレート制限があります。このエンドポイントは、過去 7 日間にタイムライン上で作成されたすべてのポストに加え、作成日時にかかわらず最新のものから最大 800 件のポストを返すことができます。

<div id="user-post-timeline">
  ## ユーザーのポストタイムライン
</div>

user Post timeline エンドポイントは、特定の X アカウントが公開したポストへのアクセスを提供します。ユーザーのポストを取得することで、ユーザーインターフェースにタイムラインを表示したり、ユーザーのポストを分析してコンテンツをより深く理解したり、そのポストを用いたエンゲージメントワークフローをプログラム的に構築したりできます。このエンドポイントでは、ユーザーのプロフィールタイムラインで確認できる内容と同様に、単一の X アカウントによる最新のポスト、リツイート、返信、および引用ツイートにアクセスできます。

以下は @XDevelopers のユーザータイムラインです:

user Post timeline エンドポイントは REST エンドポイントであり、対象のユーザー（ユーザー ID 指定）を示すための単一のパスパラメータを受け取ります。このエンドポイントは、そのユーザーによって投稿された最新のポスト、リツイート、返信、および引用ツイートを最大 3,200 件まで返すことができます。

ポストは逆時系列で返され、最も新しいものから始まります。結果は 1 ページあたり最大 100 ポストまでの[ページネーション](/ja/x-api/fundamentals/pagination)によってページングされます。大量のポストをページングするためのページネーショントークンも提供されます。指定されたページに含まれる最も新しいポストと最も古いポストの Post ID もメタデータとして提供され、最新のポストを取得するためにタイムラインをポーリングする際にも利用できます。user Post timeline では、特定の時間範囲内に作成されたポストを受け取るために、start&#95;time と end&#95;time パラメータを指定することもできます。 

user Post timeline エンドポイントは、[fields](/ja/x-api/fundamentals/fields) と [expansions](/ja/x-api/fundamentals/expansions) パラメータをサポートし、[新しい JSON データ形式](/ja/x-api/fundamentals/data-dictionary)を返します。

このエンドポイントへのリクエストを正常に行うには、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication)、[OAuth 2.0 Authorization Code with PKCE](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3)、または [OAuth 2.0 App-Only](/ja/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) 認証方式でリクエストを認証する必要があります。非公開メトリクス、プロモーションメトリクス、または保護されたユーザーのタイムラインをリクエストする場合は、OAuth 1.0a User Context もしくは OAuth 2.0 Authorization Code with PKCE を使用しなければなりません。 

user Post timeline エンドポイントは、次の 2 つの一般的な利用パターンをサポートするように設計されています。 

* 「Get a user’s historical Posts」: 関心のあるユーザーが、特定の直近期間に時系列順で作成したポストを受け取るために user Post timeline へ行うリクエストです。この期間は start&#95;time と end&#95;time で指定し、その範囲の結果をページングによってすべて取得できます。場合によっては、ユーザーのアカウントにおけるポスト数が 3,200 件以内であれば、ユーザーのポスト履歴全体を取得することも可能です。含まれるポストは、その公開状況と、リクエストに使用される認証方式に応じて決まります。

* 「Polling for new Posts」: 特定のユーザーが新たに作成したポストを取得するために、継続的に user Post timeline へ行うリクエストです。最後に受信したポストの Post ID を、前回以降の新しいリクエストのパラメータとして指定できます。

<div id="user-mention-timeline">
  ## ユーザー宛メンションタイムライン
</div>

ユーザー宛メンションタイムラインエンドポイントを使用すると、特定の X ユーザーがメンションされているポストをリクエストできます。たとえば、ある X アカウントがポスト内で @XDevelopers をメンションした場合などです。これには、指定したユーザーのポストへの返信も含まれます。ユーザーのメンションを取得することで、そのユーザーのポストに誰が返信しているか、誰がメンションしているかをすばやく把握したり、そのユーザーのポストに対するエンゲージメントワークフローをプログラムで構築したりする機能を実装できます。このエンドポイントを使用すると、単一ユーザーに対する最新のメンションと返信をリクエストでき、これは X 上のユーザーの[メンション通知](https://x.com/notifications/mentions)に表示される内容に近いものです。

ユーザー宛メンションタイムラインは REST エンドポイントであり、対象とするユーザー（ユーザー ID）を指定するための 1 つのパスパラメーターを受け取ります。このエンドポイントは、そのユーザーに対する最新 800 件のメンションを返すことができます。

ポストは新しいものから古いものへと逆時系列で返されます。結果は 1 ページあたり最大 100 件のポストで[ページネーション](/ja/x-api/fundamentals/pagination)されます。大量のポストをページングするためのページネーショントークンが提供されます。指定したページに含まれる最も新しいポストと最も古いポストの ID もメタデータとして提供され、これは最新ポストのポーリングや、v1.1 の [mentions&#95;timeline](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-mentions_timeline) エンドポイントと同様にタイムラインを移動する用途にも使用できます。また、このエンドポイントでは、特定の時間範囲内に作成されたポストを取得するために、start&#95;time および end&#95;time パラメーターを指定することもできます。 

このエンドポイントに対してリクエストを正しく行うには、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication)、[OAuth 2.0 Authorization Code with PKCE](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3)、または [OAuth 2.0 App-Only](/ja/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) 認証方式を使用してリクエストに認可を付与する必要があります。非公開メトリクス、プロモーションメトリクス、または保護されたユーザーのタイムラインをリクエストする場合は、OAuth 1.0a User Context か OAuth 2.0 Authorization Code with PKCE を使用しなければなりません。

ユーザー宛メンションタイムラインエンドポイントは、[fields](/ja/x-api/fundamentals/fields) および [expansions](/ja/x-api/fundamentals/expansions) パラメーターをサポートし、[新しい JSON データ形式](/ja/x-api/fundamentals/data-dictionary)を返します。
