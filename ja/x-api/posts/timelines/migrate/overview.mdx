<div id="comparing-x-apis-timelines-endpoints">
  ## X API のタイムラインエンドポイントの比較
</div>

v2 の逆時系列タイムライン、ユーザーのポストタイムライン、およびユーザーのメンションタイムラインの各エンドポイントは、それぞれ [v1.1 statuses/home&#95;timeine](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-home_timeline)、[v1.1 statuses/user&#95;timeline](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-user_timeline.html)、および [v1.1 statuses/mentions&#95;timeline](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-mentions_timeline.html) エンドポイントを置き換えるものです。これらの旧バージョンのエンドポイントを使用するコード、アプリ、またはツールがあり、最新の X API v2 エンドポイントへの移行を検討している場合は、本ガイドを参照してください。より詳しい移行ガイドについては、[Standard v1.1 migration to X API v2](/ja/x-api/posts/timelines/migrate/standard-to-twitter-api-v2) を参照してください。

このページには 3 つの比較表があります：

* 逆時系列ホームタイムライン
* ユーザーのポストタイムライン
* ユーザーのメンションタイムライン

<div id="reverse-chronological-home-timeline">

| ポストのJSON形式                                                                                                    | [標準 v1.1 データ形式](https://developer.x.com/en/docs/twitter-api/v1/data-dictionary/overview.html)                                                          | [X API v2](/ja/x-api/fundamentals/data-dictionary) フォーマット（fields と expansions リクエストパラメータによって決定され、v1.1 のフォーマットとの後方互換性はありません）<br /><br />Standard v1.1 フォーマットから X API v2 フォーマットへ移行する方法の詳細は、[データフォーマット移行ガイド](/ja/x-api/migrate/data-format-migration)をご覧ください。                                                                                                                                                      |
| 結果の並び順                                                                                                        | 新しい順（逆時系列）                                                                                                                                             | 新しい順（時系列の降順）                                                                                                                                                                                                                                                                                                                                                                                              |
| 結果のページネーション                                                                                                   | ページネーションは不可。ポストIDでナビゲーションする必要があります                                                                                                                     | `pagination_token` を使用して、結果を前後にページングしながら確認できます                                                                                                                                                                                                                                                                                                                                                            |
| [プロジェクト](/ja/resources/fundamentals/projects) に関連付けられた [開発者アプリ](/ja/resources/fundamentals/developer-apps) の認証情報が必要 |                                                                                                                                                        | ✔                                                                                                                                                                                                                                                                                                                                                                                                         |
