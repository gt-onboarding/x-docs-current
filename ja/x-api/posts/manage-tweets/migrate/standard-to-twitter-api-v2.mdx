<div id="standard-v11-compared-to-x-api-v2">
  ## Standard v1.1 と X API v2 の比較
</div>

Standard v1.1 の [POST statuses/update](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-update) および [POST statuses/destroy/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-destroy-id) エンドポイントを利用している場合、本ガイドの目的は、Standard v1.1 と X API v2 のポスト管理用エンドポイントの共通点と相違点を理解できるようにすることです。

* **類似点**
  * 認証
* **相違点**
  * エンドポイント URL

  * アプリとプロジェクトの要件

  * リクエストパラメータ

<div id="similarities">
  ### 類似点
</div>

**認証**

標準 v1.1 と X API v2 の両方のポストエンドポイント（[POST statuses/update](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-update) および [POST statuses/destroy/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-destroy-id)）は、[OAuth 1.0a User Context](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication) を使用します。したがって、以前に標準 v1.1 のいずれかのエンドポイントを使用していた場合、X API v2 の対応エンドポイントに移行しても、同じ認証方法を引き続き使用できます。 

<div id="differences">
  ### 違い
</div>

**エンドポイント URL**

* 標準 v1.1 エンドポイント:
  * https://api.x.com/1.1/statuses/update.json
    (ポストを作成)
  * `https://api.x.com/1.1/statuses/destroy/:id.json`
    (ポストを削除)
* X API v2 エンドポイント:
  * https://api.x.com/2/tweets
    (ポストを作成)
  * https://api.x.com/2/tweets/:id
    (指定したポストを削除)

<div id="app-and-project-requirements">
  ### アプリとプロジェクトの要件
</div>

X API v2 のエンドポイントでは、リクエストを認証する際に、[プロジェクト](/ja/resources/fundamentals/developer-apps)に関連付けられた[開発者アプリ](/ja/resources/fundamentals/developer-apps)の認証情報を使用する必要があります。X API v1.1 のすべてのエンドポイントでは、プロジェクトに関連付けられていない単体のアプリ、またはプロジェクトに関連付けられたアプリのいずれかの認証情報を使用できます。

<div id="request-parameters">
  ### リクエストパラメータ
</div>

次の standard v1.1 のリクエストパラメータは、2つのリクエストクエリパラメータ（user&#95;id または screen&#95;name）を受け付けていました。X API v2 では、DELETE エンドポイントに対して数値の Post ID のみを受け付け、この ID はエンドポイントパスの一部として渡す必要があります。

POST エンドポイントでは、追加のパラメータをリクエストの JSON 本文として渡す必要があります。利用可能なパラメータの詳細は、[API リファレンスガイド](/ja/x-api/posts/manage-tweets/introduction)を参照してください。
