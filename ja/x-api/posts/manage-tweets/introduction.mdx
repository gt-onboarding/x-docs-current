X API を使用してポストを作成および削除することは、公共の会話に参加するために不可欠です。新しい manage Posts エンドポイントを使うことで、これらに加えてさらに多くの操作が行えます。

ポストを管理するために利用できるメソッドは、POST と DELETE の 2 つです。POST メソッドを使うと、投票付きポスト、引用ポスト、返信設定付きポスト、位置情報付きポスト、メディア付きポストおよびユーザーをタグ付けしたポスト、Super Followers 向けポストなど、さまざまな機能を利用できます。同様に、DELETE メソッドを使うと特定のポストを削除できます。POST メソッドでは、リクエストをさらにカスタマイズするために必要な[パラメータ](/ja/x-api/posts/creation-of-a-post)を渡すことができます。

「delete Post」メソッドは、編集されたポストをサポートするように更新されました。ポストの編集チェーン内のいずれか 1 つのポストが削除されると、その編集チェーン内のすべてのポストも削除されます。Edit Post メタデータの詳細については、[Edit Posts fundamentals](/ja/x-api/fundamentals/edit-posts) ページを参照してください。

POST メソッドには、ユーザー単位で 15 分あたり 200 回のリクエストというレート制限があります。DELETE メソッドには、15 分あたり 50 回のリクエストというレート制限があります。さらに、manage Posts または manage Retweets のいずれかで作成されたポストを含めて、3 時間あたり 300 回のリクエストという上限があります。

manage Posts エンドポイントを使用してユーザーに代わってリクエストを送信するためには、[OAuth 1.0a User Context](https://en.docs/authentication/oauth-1-0a) または [OAuth 2.0 Authorization Code with PKCE](https://en.docs/authentication/oauth-2-0/authorization-code) のいずれかで認証を行い、あなたのアプリを承認したユーザーに関連付けられたユーザーのアクセストークンを使用する必要があります。OAuth 1.0a でこのユーザーアクセストークンを生成するには、[3-legged OAuth フロー](https://en.docs/authentication/oauth-2-0/bearer-tokens)を使用できます。OAuth 2.0 でユーザーアクセストークンを生成するには、[Authorization Code with PKCE grant フロー](https://en.docs/authentication/oauth-2-0/user-access-token)を使用できます。

<Info>
  **アカウント設定**

  これらのエンドポイントにアクセスするには、次が必要です。

  * 承認済みの[開発者アカウント](https://en/portal/petition/essential/basic-info)
  * [プロジェクト](/ja/resources/fundamentals/projects)内にある[開発者アプリ](/ja/resources/fundamentals/developer-apps)からのキーとトークンで認証すること

  X API v2 エンドポイントへのアクセス方法については、[はじめに](/ja/x-api/getting-started/getting-access)ガイドを参照してください。
</Info>
