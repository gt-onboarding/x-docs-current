X API を使用してポストを作成・削除することは、パブリックな会話に参加するために不可欠です。新しい manage Posts エンドポイントを使うことで、これらに加えてさらに多くの操作が可能になります。

Posts を管理するために利用できるメソッドは、POST と DELETE の 2 種類です。POST メソッドでは、投票付きポストの投稿、ポストの引用、返信設定付きポストの投稿、位置情報付きポストの投稿、メディア付きポストの投稿とユーザーのタグ付け、Super Followers へのポストなど、さまざまな機能を利用できます。同様に、DELETE メソッドでは特定のポストを削除できます。POST メソッドでは、リクエストをさらにカスタマイズできるように、目的に応じた[パラメータ](/ja/x-api/posts/creation-of-a-post)を指定できます。

`delete Post` メソッドは、編集済みポストをサポートするように更新されました。ポストの編集チェーン内のいずれかのポストを削除すると、その編集チェーンに含まれるすべてのポストも削除されます。Edit Post メタデータの詳細については、[Edit Posts fundamentals](/ja/x-api/fundamentals/edit-posts) ページを参照してください。

POST メソッドには、ユーザーごとに 15 分あたり 200 リクエストというレート制限があります。DELETE メソッドには、15 分あたり 50 リクエストというレート制限があります。さらに、manage Posts または manage Retweets のいずれかで作成されたポストを含めて、3 時間あたり 300 リクエストという制限があります。

manage Posts エンドポイントを使用してユーザーに代わってリクエストを送信するため、[OAuth 1.0a User Context](https://en.docs/authentication/oauth-1-0a) または [OAuth 2.0 Authorization Code with PKCE](https://en.docs/authentication/oauth-2-0/authorization-code) のいずれかで認証し、あなたのアプリを承認したユーザーに関連付けられたユーザーアクセストークンを使用する必要があります。OAuth 1.0a を使用してこのユーザーアクセストークンを生成するには、[3-legged OAuth フロー](https://en.docs/authentication/oauth-2-0/bearer-tokens)を利用できます。OAuth 2.0 を使用してユーザーアクセストークンを生成するには、[Authorization Code with PKCE グラントフロー](https://en.docs/authentication/oauth-2-0/user-access-token)を利用できます。

<Info>
  **アカウントの設定**

  これらのエンドポイントにアクセスするには、次のものが必要です。

  * 承認済みの[開発者アカウント](https://en/portal/petition/essential/basic-info)。
  * [プロジェクト](/ja/resources/fundamentals/projects)内にある[開発者アプリ](/ja/resources/fundamentals/developer-apps)から取得したキーとトークンを使用した認証。

  X API v2 エンドポイントへのアクセスの詳細については、[入門ガイド](/ja/x-api/getting-started/getting-access)をご覧ください。
</Info>
