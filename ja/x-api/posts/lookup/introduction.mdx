ポストは、X 上で主要なリソースの 1 つです。最も単純な形では、ポストには最大 280 文字を含めることができ、アカウントの設定に応じて公開または非公開で投稿できます。さらに、ポストにはメディア、場所、投票、URL など、さまざまなオブジェクトを添付できます。加えて、ほとんどのポストは作成後 30 分間は編集することができます。

ポストにはさまざまな HTTP、選択、および配信の方法がありますが、この REST エンドポイント群は、[Post ID](/ja/resources/fundamentals/x-ids) で指定された 1 件または複数のポストを返すことに特化しています。これらのエンドポイントを使用すると、ポストの最新の詳細を取得したり、ポストが利用可能であることを確認したり、その編集履歴を調べたりできます。また、[コンプライアンスイベント](/ja/x-api/enterprise-gnip-2.0/fundamentals/firehouse) を管理するうえでも重要です。

Post ルックアップエンドポイントは、編集されたポストに関するメタデータを提供します。2022 年 9 月 29 日以降に作成されたすべてのポストには、一度も編集されていない場合でも編集メタデータが含まれます。各編集によって新しい Post ID が発行され、ポストの編集履歴は、元のポストから始まる Post ID の配列として表現されます。

このエンドポイントは常に、最新の編集とその編集履歴を返します。30 分間の編集ウィンドウが過ぎた後に収集されたポストは、その最終バージョンを表します。Edit Post メタデータについて詳しくは、[Edit Posts の基本](/ja/x-api/fundamentals/edit-posts) ページを参照してください。

これらのエンドポイントは GET HTTP メソッドを使用し、ポスト本文、作成タイムスタンプ、ハッシュタグ、メンション、写真のリストとメタデータなどのフィールドを含む、1 つまたは複数の [Posts オブジェクト](/ja/x-api/fundamentals/data-dictionary) を返します。

***

<Note>
  ### アカウントのセットアップ

  これらのエンドポイントにアクセスするには、次が必要です。

  * 承認済みの [開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)。
  * [プロジェクト](/ja/resources/fundamentals/projects) 内の [開発者アプリ](/ja/resources/fundamentals/developer-apps) から取得したキーとトークンを使用した認証。

  X API v2 エンドポイントへのアクセスの取得方法については、[利用開始ガイド](/ja/x-api/getting-started/getting-access)を参照してください。
</Note>
