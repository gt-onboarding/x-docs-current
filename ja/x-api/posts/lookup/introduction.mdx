ポストは、X における主要なリソースの 1 つです。最も単純な形では、ポストには最大 280 文字を含めることができ、アカウントの設定に応じて公開または非公開として投稿できます。さらに、ポストにはメディア、場所、投票、URL など、さまざまなオブジェクトを添付することもできます。加えて、ほとんどのポストは作成後最大 30 分間編集できます。

ポストにはさまざまな HTTP、選択、および配信方法がありますが、この REST エンドポイント群は、[Post ID](/ja/resources/fundamentals/x-ids) で指定された 1 件または複数のポストを返すことに特化しています。これらのエンドポイントは、ポストに関する最新の詳細情報を取得し、ポストが利用可能かどうかを確認し、その編集履歴を調べることができます。また、[コンプライアンスイベント](/ja/x-api/enterprise-gnip-2.0/fundamentals/firehouse) を管理する上でも重要です。

Post lookup エンドポイントは、編集済みポストのメタデータを提供します。2022 年 9 月 29 日以降に作成されたすべてのポストには、一度も編集されていなくても編集メタデータが含まれます。各編集では新しい Post ID が付与され、ポストの編集履歴は、元のポストから始まる Post ID の配列として表現されます。

このエンドポイントは常に、最新の編集とその編集履歴を返します。30 分間の編集ウィンドウが経過した後に収集されたポストは、最終バージョンを表します。Edit Post メタデータの詳細については、[Edit Posts の基本](/ja/x-api/fundamentals/edit-posts) ページを参照してください。

これらのエンドポイントは GET HTTP メソッドを使用し、ポスト本文、作成タイムスタンプ、ハッシュタグ、メンション、写真のリストおよびメタデータといったフィールドを含む、1 つまたは複数の [Posts オブジェクト](/ja/x-api/fundamentals/data-dictionary) を返します。

***

<Note>
  ### アカウントのセットアップ

  これらのエンドポイントにアクセスするには、以下が必要です。

  * 承認済みの [開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)。
  * [プロジェクト](/ja/resources/fundamentals/projects) 内の [開発者アプリ](/ja/resources/fundamentals/developer-apps) から取得したキーとトークンを使用した認証。

  X API v2 エンドポイントへのアクセス方法の詳細については、[はじめに](/ja/x-api/getting-started/getting-access) ガイドを参照してください。
</Note>
