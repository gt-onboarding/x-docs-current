Reposts of Me ルックアップを使用すると、自分のポストのうち、どれが他のユーザーによって共有されているかを確認できます。

これにより、ユーザーは自分のどのポストが共有され、オーディエンスに響いているかを把握でき、より的を絞った対話やコンテンツ戦略の見直しに役立てることができます。

<div id="authentication">
  #### 認証
</div>

ユーザーに代わってリクエストを送信する場合は、これらのエンドポイントを [OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Authorization Code with PKCE](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) で認証し、リクエスト先のユーザーに紐づくユーザーアクセストークンを使用する必要があります。ユーザーアクセストークンは、[3-legged OAuth フロー](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)（OAuth 1.0a）または [Authorization Code with PKCE グラントフロー](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3)（OAuth 2.0）を使用して生成できます。

<div id="making-a-request">
  #### リクエストの送信
</div>

ユーザーがあなたのアプリで認証を完了したら、以下のように、そのユーザーを代理して Retweets of Me API を呼び出せます。

```bash
      curl 'https://api.x.com/2/users/reposts_of_me' --header 'Authorization: ••••••'
```
