「Reposts of Me」ルックアップ機能を使用すると、自分のポストのうち、どれが他のユーザーによってリポストされているかを確認できます。

これにより、ユーザーは自分のどのポストがリポストされ、オーディエンスに響いているかを把握でき、よりターゲットを絞ったインタラクションやコンテンツ戦略の調整に役立てることができます。

<div id="authentication">
  #### 認証
</div>

ユーザーに代わってリクエストを送信するため、これらのエンドポイントは必ず [OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Authorization Code with PKCE](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) のいずれかで認証し、そのユーザーに紐づくユーザー用アクセストークンを使用する必要があります。ユーザー用アクセストークンは、[3-legged OAuth フロー](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)（OAuth 1.0a）か、[Authorization Code with PKCE グラントフロー](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3)（OAuth 2.0）を使用して生成できます。

<div id="making-a-request">
  #### リクエストの送信
</div>

ユーザーがアプリで認証を完了すると、以下のようにそのユーザーに代わって Retweets of Me API を呼び出せます。

```bash
      curl 'https://api.x.com/2/users/reposts_of_me' --header 'Authorization: ••••••'
```
