<div id="retweets-lookup">
  ### リツイート検索
</div>

Retweets lookup エンドポイントを使用すると、特定のポストをリツイートしたアカウントの一覧を取得できます。このエンドポイントでは、大きな結果セットを最大100ユーザー単位のバッチでページングするためのページネーション用トークンが提供されます。

これらのエンドポイントは、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication)、[OAuth 2.0 App-Only](https://developer.x.com(/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)、または [OAuth 2.0 Authorization Code with PKCE](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) のいずれかで認証できます。 

<div id="retweets-of-me">
  ### 自分のポストのリポスト
</div>

「自分のポストのリポスト」ルックアップを使用すると、自分のポストのうちどれが他のユーザーによって共有されたかを確認できます。

これにより、ユーザーは自分のポストのうちどれが共有され、オーディエンスに響いているかについてのインサイトを得ることができ、より的を絞ったコミュニケーションやコンテンツ戦略の調整に役立ちます。

これらのエンドポイントは、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Authorization Code with PKCE](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) のいずれかで認証できます。

<div id="manage-retweets">
  #### リツイートを管理する
</div>

Manage Retweets エンドポイントでは、認証済みアカウントを代理して、指定したポストをリツイートしたり、そのリツイートを取り消したりできます。このエンドポイントグループでは、POST と DELETE の 2 つのメソッドが利用可能です。POST メソッドはポストをリツイートするために使用し、DELETE メソッドは指定したポストのリツイートを取り消すために使用します。

ユーザーの代わりにリクエストを送信するため、これらのエンドポイントは [OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) のいずれかで認証し、リクエストを行うユーザーに紐づくユーザーのアクセストークンを利用する必要があります。このユーザーのアクセストークンは、[3-legged OAuth フロー](/ja/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（OAuth 1.0a）または [Authorization Code with PKCE grant flow](/ja/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)（OAuth 2.0）を使用して生成できます。自分のアカウントからでも、認証されたユーザーのアカウントからでもポストをリツイートできます。どちらのエンドポイントでも、ユーザー単位のレート制限は、エンドポイントごとに 15 分あたり 50 リクエストです。
