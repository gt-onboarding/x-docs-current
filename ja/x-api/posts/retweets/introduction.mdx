<div id="retweets-lookup">
  ### リツイート参照
</div>

Retweets lookup エンドポイントを使用すると、特定のポストをリツイートしたアカウントの一覧を取得できます。このエンドポイントでは、多数の結果を最大 100 人ずつのバッチで取得できるよう、ページネーション用のトークンが提供されます。

これらのエンドポイントには、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication)、[OAuth 2.0 App-Only](https://developer.x.com(/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)、または [OAuth 2.0 Authorization Code with PKCE](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) のいずれかで認証できます。 

<div id="retweets-of-me">
  ### 自分のポストのリポスト
</div>

「Reposts of Me」ルックアップを使用すると、自分のどのポストが他のユーザーによってリポストされているかを確認できます。

これにより、ユーザーは自分のどのポストがリポストされてオーディエンスに響いているかを把握でき、より的確なやりとりやコンテンツ戦略の調整に役立てることができます。

これらのエンドポイントには、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Authorization Code with PKCE](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) のいずれかを使用して認証を行うことができます。

<div id="manage-retweets">
  #### リツイートの管理
</div>

Manage Retweets エンドポイントを使用すると、認証済みアカウントを代表して、指定したポストをリツイートしたり、リツイートを取り消したりできます。このエンドポイントグループでは、POST と DELETE の 2 つのメソッドが利用できます。POST メソッドはポストをリツイートするために使用し、DELETE メソッドは指定されたポストのリツイートを取り消すために使用します。

ユーザーを代表してリクエストを行うため、これらのエンドポイントは必ず [OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) で認証し、リクエスト対象となるユーザーに紐づくユーザーアクセストークンを利用する必要があります。このユーザーアクセストークンは、[3-legged OAuth フロー](/ja/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（OAuth 1.0a）か、[Authorization Code with PKCE grant flow](/ja/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)（OAuth 2.0）を使用して生成できます。自分のアカウント、または認証済みユーザーのアカウントからポストをリツイートできます。どちらのエンドポイントでも、ユーザー単位のレート制限は「各エンドポイントあたり 15 分間で 50 リクエスト」です。
