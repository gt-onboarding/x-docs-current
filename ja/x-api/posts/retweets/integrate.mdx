</div>

Postman は、エンドポイントをテストするのに役立つ優れたツールです。各 Postman リクエストには、利用可能な項目をすばやく把握できるよう、すべてのパスパラメータおよびボディパラメータが含まれています。Postman コレクションの詳細については、「[Postman を使う](/ja/tutorials/postman-getting-started)」ページをご覧ください。

<div id="authentication">
  #### 認証
</div>

すべての X API v2 エンドポイントでは、「キーとトークン」とも呼ばれる一式の認証情報を使ってリクエストを認証する必要があります。

Retweets lookup エンドポイントへのリクエストを認証するには、OAuth 1.0a User Context か OAuth 2.0 Bearer Token のどちらかを使用できます。 

Retweets を管理するエンドポイントでは OAuth 1.0a User Context の使用が必要です。つまり、リクエストを成功させるには、API キーとユーザーのアクセストークンのセットを使用する必要があります。アクセストークンは、リクエストを代行して行う対象ユーザーに紐づいていなければなりません。別のユーザー向けのアクセストークンを生成したい場合、そのユーザーは 3-legged OAuth フローを通じてあなたのアプリを認可または認証する必要があります。

OAuth 1.0a の利用は難しい場合がある点に注意してください。この認証方法に慣れていない場合は、リクエストを正しく認証するために [ライブラリ](/ja/x-api/tools-and-libraries/overview) を使用することを推奨します。

**注意**

次のフィールドをリクエストする場合は、OAuth 1.0a User Context が必要です。 

* tweet.fields.non&#95;public&#95;metrics
* tweet.fields.promoted&#95;metrics
* tweet.fields.organic&#95;metrics
* media.fields.non&#95;public&#95;metrics
* media.fields.promoted&#95;metrics
* media.fields.organic&#95;metrics

<div id="developer-portal-projects-and-developer-apps">
  #### Developer ポータル、プロジェクト、開発者アプリ
</div>

X API v2 のエンドポイントで利用できる認証情報一式を取得するには、まず [開発者アカウントにサインアップ](https://developer.x.com/en/portal/petition/essential/basic-info)し、そのアカウント内に [プロジェクト](/ja/resources/fundamentals/projects) を作成し、そのプロジェクト内に [開発者アプリ](/ja/resources/fundamentals/developer-apps) を作成する必要があります。その後、開発者アプリでキーとトークンを確認できます。 
 

<div id="rate-limits">
  #### レート制限
</div>

毎日、数万もの開発者が X API へリクエストを送信しています。これらのリクエストの膨大な数を管理するために、各エンドポイントには [レート制限](/ja/x-api/fundamentals/rate-limits) が設定されており、アプリまたは認証済みユーザーの代理として送信できるリクエスト数が制限されています。

Retweets 管理用エンドポイントは、15 分あたり 50 リクエスト（ユーザーごと）に制限されています。さらに、POST エンドポイントについては、3 時間ごとに 300 リクエスト（ユーザーごと、アプリごと）に制限されています。 

Retweets lookup エンドポイントでは、15 分あたり 75 リクエストに制限されています。さらに、このエンドポイントで返されるのは、直近にリツイートを行ったユーザーのうち最新 100 件のみです。

<div id="fields-and-expansions">
  #### フィールドとexpansions
</div>

X API v2 では、fields と expansions と呼ばれる一連の機能を使って、API から返したいデータを正確に選択できます。`expansions` パラメータを使用すると、ペイロード内で参照されているオブジェクトを展開できます。たとえば、このエンドポイントでは次の [expansions](/ja/x-api/fundamentals/expansions) を取得できます。

* attachments.poll&#95;ids
* attachments.media&#95;keys
* author&#95;id, entities.mentions.username
* geo.place&#95;id
* in&#95;reply&#95;to&#95;user&#95;id,
* referenced&#95;tweets.id,
* referenced&#95;tweets.id.author&#95;id

`fields` パラメータを使用すると、複数のデータオブジェクトのうち、どの[fields](/ja/x-api/fundamentals/fields) を受け取りたいかを正確に選択できます。これらのエンドポイントは主にポストオブジェクトを返すエンドポイントです。デフォルトでは、ポストオブジェクトは `id` と `text` フィールドを返します。`tweet.created_at` や `tweet.entities` などの追加フィールドを受け取るには、`fields` パラメータを使ってそれらを明示的にリクエストする必要があります。連携実装で使用することを検討したい重要なフィールドとしては、投票データ、メトリクス、ポストの注釈、会話 ID フィールドなどがあります。

[X API v2 データ辞書](/ja/x-api/fundamentals/data-dictionary) には、[fields と expansions を併用する方法](/ja/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) に関するガイドも追加しています。
