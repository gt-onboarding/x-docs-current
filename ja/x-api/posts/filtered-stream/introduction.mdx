filtered stream エンドポイントグループを使用すると、開発者は公開ポストのストリームをフィルタリングできます。このエンドポイントグループには、ルールの作成と管理を行い、そのルールを適用して公開ポストのみが返されるようにポストのストリームをフィルタリングできる複数のエンドポイントが含まれています。このエンドポイントグループを利用することで、特定のトピックやイベントをリッスンしたり、キャンペーンに関する会話をモニタリングしたり、トレンドがどのように形成されるかを把握したりと、さまざまなことが可能になります。

開発者は、REST の [rules エンドポイント](/ja/x-api/posts/filtered-stream#post-2-tweets-search-stream-rules) を使用して、永続的なストリーム接続を切断することなく、ルールを追加および削除できます。これらの[ルール](/ja/x-api/posts/filtered-stream#building-rules-for-filtered-stream)は、メッセージキーワード、ハッシュタグ、URL などのポスト属性にマッチするオペレーターを使って作成できます。オペレーターやルール句は、ブールロジックやかっこで組み合わせることで、フィルタのマッチング動作をより細かく制御できます。

一連のルールを追加したら、[ストリーミング接続を確立](/ja/x-api/posts/filtered-stream#get-2-tweets-search-stream)して、永続的な HTTP ストリーミング接続を通じて JSON 形式の[Post オブジェクト](/ja/x-api/fundamentals/data-dictionary#tweet)の配信を開始できます。ストリームに接続している間は、ルールにマッチしたコンテンツのみが配信されます。

また、Filtered Stream を webhook にリンクし、webhook イベントとして配信することもできます。Filtered Stream と webhook の併用について詳しくは、[Filtered Stream Webhooks の概要](/ja/x-api/webhooks/stream/introduction)をご覧ください。

filtered search エンドポイントは、編集済みポストをサポートしています。このエンドポイントは、1 つ以上のフィルターにマッチする編集済みポストと、その編集履歴（ポスト ID の配列を含む）を配信します。編集履歴のないポストでは、この配列には単一の ID が含まれます。編集が行われたポストでは、この配列には複数の ID が含まれ、編集が行われた順序を反映した昇順で並び、配列の最後の要素が最新バージョンになります。ポストの編集がどのように機能するかについては、[ポスト編集の基本](/ja/x-api/fundamentals/edit-posts)をご覧ください。

filtered stream エンドポイントの一部の機能は、[アクセスレベル](/ja/x-api/getting-started/about-x-api)によって制限されます。

**Pro アクセス**

* プロジェクトあたり 1,000 個のルール
* POST /2/tweets/search/stream/rules エンドポイントを使用してルールを追加する場合、15 分あたり 100 リクエスト
* ルールの作成時にすべてのオペレーターを使用可能
* 最大 1,024 文字までのルールを作成可能

**Enterprise アクセス**

* プロジェクトあたり 25,000 個以上のルール
* ルールの作成時にすべてのオペレーターを使用可能
* 最大 2,048 文字までのルールを作成可能
* Enterprise アクセスの申請は[こちら](https://developer.x.com/en/products/x-api/enterprise/enterprise-api-interest-form)から

filtered stream で返されるポストは、月間の[ポスト上限](/ja/x-api/fundamentals/post-cap)にカウントされます。

<Note>
  **アカウントのセットアップ**

  これらのエンドポイントにアクセスするには、次が必要です。

  * 承認済みの [開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)。
  * [プロジェクト](/ja/resources/fundamentals/projects) 内にある [開発者アプリ](/ja/resources/fundamentals/developer-apps) のキーとトークンを使用して認証すること。

  X API v2 エンドポイントへのアクセスの取得については、[はじめにガイド](/ja/x-api/getting-started/getting-access)をご覧ください。
</Note>
