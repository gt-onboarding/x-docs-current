The filtered stream エンドポイントグループは、公開ポストのストリームをフィルタリングできるようにするものです。このエンドポイントグループには、ルールを作成および管理し、それらのルールを適用して一致する公開ポストだけが返されるようにポストのストリームをフィルタリングできる、複数のエンドポイントが含まれます。このエンドポイントグループを使用すると、特定のトピックやイベントに関する投稿を受信したり、コンペティションをめぐる会話をモニタリングしたり、トレンドがどのように発展するかを把握したり、その他さまざまなことが可能になります。

開発者は REST の [rules エンドポイント](/ja/x-api/posts/filtered-stream#post-2-tweets-search-stream-rules) を使用して、接続を切断することなく永続的なストリーミング接続に対してルールを追加および削除できます。これらの[ルール](/ja/x-api/posts/filtered-stream#building-rules-for-filtered-stream)は、メッセージのキーワード、ハッシュタグ、URL などのポスト属性にマッチするオペレーターを使って作成できます。オペレーターとルール節は、ブールロジックや括弧と組み合わせることで、フィルタのマッチング動作をより細かく調整できます。

一連のルールを追加したら、[ストリーミング接続を確立](/ja/x-api/posts/filtered-stream#get-2-tweets-search-stream)できます。これにより、永続的な HTTP ストリーミング接続を通じて、JSON 形式の[ポストオブジェクト](/ja/x-api/fundamentals/data-dictionary#tweet)の配信が開始されます。ストリームに接続している間は、ルールに一致するコンテンツのみを受信します。

また、Filtered Stream を webhook にリンクし、webhook イベントとして配信することもできます。Filtered Stream で webhook を使用する方法の詳細については、[Filtered Stream Webhooks の概要](/ja/x-api/webhooks/stream/introduction)を参照してください。

filtered search エンドポイントは編集済みポストをサポートしています。このエンドポイントは、1 つ以上のフィルタに一致する編集済みポストを、その編集履歴（ポスト ID の配列を含む）とともに配信します。編集履歴のないポストの場合、この配列には 1 つの ID が格納されます。編集されたポストの場合、この配列には複数の ID が格納され、編集順に昇順で並んでおり、配列の最後の要素が最新バージョンになります。ポストの編集の仕組みについて詳しくは、[ポスト編集の基礎](/ja/x-api/fundamentals/edit-posts)ページを参照してください。

filtered stream エンドポイントのいくつかの機能は、[アクセスレベル](/ja/x-api/getting-started/about-x-api)によって制限されます。

**Pro アクセス**

* プロジェクトごとに 1000 個のルール
* POST /2/tweets/search/stream/rules エンドポイントを使用してルールを追加する場合、15 分あたり 100 リクエスト
* ルールの作成時にすべてのオペレーターを使用可能
* 最大 1024 文字までのルールを作成可能

**Enterprise アクセス**

* プロジェクトごとに 25,000 個以上のルール
* ルールの作成時にすべてのオペレーターを使用可能
* 最大 2048 文字までのルールを作成可能
* Enterprise アクセスの申請は[こちら](https://developer.x.com/en/products/x-api/enterprise/enterprise-api-interest-form)

filtered stream から返されるポストは、月間の[ポスト上限](/ja/x-api/fundamentals/post-cap)にカウントされます。

<Note>
  **アカウントの設定**

  これらのエンドポイントにアクセスするには、次が必要です。

  * 承認済みの[開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)。
  * [プロジェクト](/ja/resources/fundamentals/projects)内にある[開発者アプリ](/ja/resources/fundamentals/developer-apps)のキーとトークンを使用した認証。

  X API v2 エンドポイントへのアクセス方法の詳細については、[はじめにガイド](/ja/x-api/getting-started/getting-access)を参照してください。
</Note>
