<div id="manage-bookmarks">
  ### ブックマークの管理
</div>

ブックマークを管理するために利用できるメソッドは、POST と DELETE の 2 つです。POST メソッドではブックマークを作成できます。同様に、DELETE メソッドでは特定のブックマークを削除できます。 

POST メソッドと DELETE メソッドには、ユーザーごとに 15 分あたり 50 リクエストのレート制限があります。

manage Bookmarks エンドポイントでユーザーに代わってリクエストを実行するには、OAuth 2.0 を使用してユーザーのアクセストークンを生成し、認証する必要があります。そのためには、[Authorization Code with PKCE grant flow](/ja/resources/fundamentals/authentication/oauth-2-0/user-access-token) を使用できます。このエンドポイントを使用するには、`tweet.read`、`users.read`、`bookmark.write` の各スコープを付与する必要があります。

<div id="bookmarks-lookup">
  ### ブックマーク参照
</div>

Bookmarks lookup エンドポイントで利用可能なメソッドは、GET のみです。このメソッドを使用すると、自分のアカウント、または認証済みアカウントのブックマークを取得できます。大きな結果セットをページングするために、このエンドポイントではページネーション用のトークンが返されます。

GET メソッドには、ユーザーごとに 15 分あたり 180 リクエストのレート制限があります。このエンドポイントでは、直近のブックマーク済みポストを最大 800 件まで取得できます。

Bookmarks lookup エンドポイントでユーザーに代わってリクエストを行うため、OAuth 2.0 を用いてユーザーのアクセストークンを生成し、認証する必要があります。そのためには、[Authorization Code with PKCE grant flow](/ja/resources/fundamentals/authentication/oauth-2-0/user-access-token) を使用できます。このエンドポイントを利用するには、スコープ `tweet.read`、`users.read`、`bookmark.read` を指定する必要があります。

**アカウントのセットアップ**

これらのエンドポイントにアクセスするには、次のものが必要です。

* 承認済みの[開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)。
* [プロジェクト](/ja/resources/fundamentals/projects) 内にある[開発者アプリ](/ja/resources/fundamentals/developer-apps)のキーとトークンを使用して認証すること。

X API v2 エンドポイントへのアクセスの詳細については、[はじめにガイド](/ja/x-api/getting-started/getting-access)を参照してください。
