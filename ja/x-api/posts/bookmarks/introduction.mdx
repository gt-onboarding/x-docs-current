<div id="manage-bookmarks">
  ### ブックマークを管理する
</div>

ブックマークを管理するために利用できるメソッドは、POST と DELETE の 2 種類です。POST メソッドではブックマークを作成できます。同様に、DELETE メソッドでは特定のブックマークを削除できます。

POST および DELETE メソッドには、ユーザーごとに 15 分あたり 50 リクエストのレート制限があります。

Manage Bookmarks エンドポイントを使ってユーザーに代わってリクエストを行う場合は、OAuth 2.0 を使用してユーザーのアクセストークンを生成し、認証する必要があります。そのためには、[Authorization Code with PKCE grant flow](/ja/resources/fundamentals/authentication/oauth-2-0/user-access-token) を使用できます。このエンドポイントを利用するには、`tweet.read`、`users.read`、`bookmark.write` のスコープを指定する必要があります。

<div id="bookmarks-lookup">
  ### ブックマークのルックアップ
</div>

ブックマークのルックアップエンドポイントでは、利用可能なメソッドは GET のみです。このメソッドを使用すると、自分自身、または認証済みアカウントのブックマークを取得できます。このエンドポイントで大量の結果をページングするために、ページネーション用トークンが提供されます。

GET メソッドには、ユーザーごとに 15 分間あたり 180 回までというレート制限があります。このエンドポイントでは、最新のブックマーク済みポストを最大 800 件まで取得できます。

ブックマークのルックアップエンドポイントでユーザーの代わりにリクエストを行うため、OAuth 2.0 を使用してユーザーのアクセストークンを生成し、認証する必要があります。そのためには、[Authorization Code with PKCE grant flow](/ja/resources/fundamentals/authentication/oauth-2-0/user-access-token) を使用できます。このエンドポイントを利用するには、`tweet.read`、`users.read`、`bookmark.read` のスコープを指定して渡す必要があります。

**アカウント設定**

これらのエンドポイントにアクセスするには、以下が必要です。

* 承認済みの [developer account](https://developer.x.com/en/portal/petition/essential/basic-info)。
* [Project](/ja/resources/fundamentals/projects) 内にある [developer App](/ja/resources/fundamentals/developer-apps) のキーとトークンを使用して認証すること。 

X API v2 エンドポイントへのアクセス方法については、[はじめにガイド](/ja/x-api/getting-started/getting-access) を参照してください。
