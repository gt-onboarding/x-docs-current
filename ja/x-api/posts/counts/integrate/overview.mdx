<div id="helpful-tools">
  ### 便利なツール
</div>

いくつかの重要な概念を見ていく前に、これらのエンドポイントの機能をテストし始める際は、次のいずれかのツールまたはコードサンプルを使用することをおすすめします。

<div id="code-samples">
  #### コードサンプル
</div>

お好みのプログラミング言語でこれらのエンドポイントを使ったコードを動かしたい方のために、[GitHub ページ](https://github.com/xdevplatform/Twitter-API-v2-sample-code)で出発点として利用できるさまざまなコードサンプルを公開しています。[Python client](https://github.com/xdevplatform/search-tweets-python) も含まれています。

<div id="libraries">
  #### ライブラリ
</div>

提供されている多数の [コミュニティ製サードパーティライブラリ](/ja/x-api/tools-and-libraries/overview) を活用して、スムーズに始めましょう。v2 エンドポイントに対応したライブラリは、該当するバージョンタグを確認して探すことができます。

<div id="postman">
  #### Postman
</div>

Postman は、これらのエンドポイントをテストするために活用できる便利なツールです。各 Postman リクエストには、そのエンドポイントで使用可能なすべてのパラメータが含まれており、そこで指定・利用できる内容をすばやく把握するのに役立ちます。Postman コレクションの詳細については、[Using Postman](/ja/tutorials/postman-getting-started) ページをご覧ください。
 

<div id="key-concepts">
  ### 主要な概念
</div>

<div id="authentication">
  #### 認証
</div>

すべての X API v2 エンドポイントでは、キーとトークンとも呼ばれる一連の認証情報を使用して、リクエストを[認証](/ja/resources/fundamentals/authentication)する必要があります。このエンドポイントでは [OAuth 2.0 Bearer Token](/ja/resources/fundamentals/authentication#oauth-2-0) の使用が必須であり、そのため、リクエストを成功させるには [Bearer Token](/ja/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) を渡さなければなりません。Bearer Token は、開発者アプリ内から直接生成することも、[POST oauth2/token](/ja/resources/fundamentals/authentication#post-oauth2-token) エンドポイントを使用して生成することもできます。

<div id="developer-portal-projects-and-developer-apps">
  #### Developer portal、プロジェクト、および開発者アプリ
</div>

任意の X API v2 エンドポイントを利用するには、[開発者アカウント](/ja/resources/fundamentals/developer-portal)を取得し、そのアカウント内に[プロジェクト](/ja/resources/fundamentals/projects)を作成し、そのプロジェクト内に[開発者アプリ](/ja/resources/fundamentals/developer-apps)を作成している必要があります。その開発者アプリで発行されたキーとトークンは、最近のポスト件数を取得するエンドポイントで利用できます。full-archive ポスト件数エンドポイントを利用したい場合、または高度な演算子やより長いクエリ文字数を利用したい場合は、Enterprise アクセスの承認を受けている必要があります。

詳しくは、Enterprise アクセスに関するセクションをご覧ください。

<div id="rate-limits">
  #### レート制限
</div>

毎日、何千人もの開発者が X API にリクエストを送信しています。リクエストの量を管理するために、各エンドポイントには[レート制限](/ja/x-api/fundamentals/rate-limits)が設けられており、開発者がアプリまたは認証済みユーザーとして発行できるリクエスト数が制限されています。

このエンドポイントにはアプリレベルのレート制限が適用されます。つまり、開発者であるあなたは、使用している認証情報によって識別される任意のアプリごとに、このエンドポイントに対して一定期間内に実行できるリクエストの回数が制限されます。 

<div id="building-queries">
  #### クエリの構築
</div>

これらのエンドポイントの中心的な特徴は、単一のクエリを使ってポストをフィルタリングし、あなたに配信される件数を決定する点です。これらのクエリは、メッセージのキーワード、ハッシュタグ、URL などのポストおよびユーザー属性にマッチするオペレーターで構成されます。オペレーターは、ブーリアンロジックや括弧と組み合わせてクエリを作成でき、クエリのマッチング動作をより細かく調整するのに役立ちます。

詳しくは、[クエリの構築方法](/ja/x-api/posts/counts/integrate/build-a-query)に関するガイドを参照してください。

<div id="pagination">
  #### ページネーション
</div>

直近のポスト件数については、`next_token` は返されません。つまり、粒度の設定に関係なく、直近7日間のポスト数を1回の API コールで取得できます。

フルアーカイブのポスト件数については、直近30日分のデータを取得できます。30日より前のデータについては、`next_token` が返されるため、それを使用してページネーションを行い、追加のデータを取得できます。 

<div id="building-queries-for-post-counts">
  ### ポスト件数用のクエリを作成する
</div>

ポスト件数エンドポイントは、GET リクエストで 1 つのクエリを受け取り、そのクエリに一致する過去のポスト件数のセットを返します。クエリは、さまざまなポストの属性に一致させるために使用されるオペレーターで構成されます。
