<div id="helpful-tools">
  ### 役立つツール
</div>

いくつかの重要な概念を説明する前に、これらのエンドポイントの動作をテストするために、次のいずれかのツールまたはコードサンプルを利用することをおすすめします。

<div id="code-samples">
  #### コードサンプル
</div>

お好みのプログラミング言語のコードからこれらのエンドポイントを利用したい場合のために、出発点として使えるコードサンプルをいくつか用意しています。[GitHub ページ](https://github.com/xdevplatform/Twitter-API-v2-sample-code)で公開しており、[Python client](https://github.com/xdevplatform/search-tweets-python) などが含まれます。

<div id="libraries">
  #### ライブラリ
</div>

開発を始める際には、ぜひ多数ある[コミュニティ製のサードパーティライブラリ](/ja/x-api/tools-and-libraries/overview)のいずれかを活用してください。対応するバージョンタグを探すことで、v2 エンドポイントと連携できるライブラリを見つけることができます。

<div id="postman">
  #### Postman
</div>

Postman は、これらのエンドポイントをテストするのに便利なツールです。各 Postman リクエストには、そのエンドポイントで利用できるすべてのパラメータが含まれているため、どのような項目があるかをすばやく把握できます。Postman コレクションについて詳しくは、[Using Postman](/ja/tutorials/postman-getting-started) のページをご覧ください。
 

<div id="key-concepts">
  ### 重要な概念
</div>

<div id="authentication">
  #### 認証
</div>

すべての X API v2 エンドポイントへのリクエストは、[認証](/ja/resources/fundamentals/authentication) に使用される資格情報（キーとトークンとも呼ばれます）を用いて行う必要があります。 このエンドポイントでは [OAuth 2.0 Bearer Token](/ja/resources/fundamentals/authentication#oauth-2-0) の使用が必須のため、有効なリクエストを送信するには [Bearer Token](/ja/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) を渡す必要があります。Bearer Token は、開発者アプリ内から直接生成するか、[POST oauth2/token](/ja/resources/fundamentals/authentication#post-oauth2-token) エンドポイントを使用して生成できます。

<div id="developer-portal-projects-and-developer-apps">
  #### developer portal、プロジェクト、および開発者アプリ
</div>

任意の X API v2 エンドポイントを利用するには、[開発者アカウント](/ja/resources/fundamentals/developer-portal) を取得し、そのアカウント内に [プロジェクト](/ja/resources/fundamentals/projects) を作成し、そのプロジェクト内に [開発者アプリ](/ja/resources/fundamentals/developer-apps) を作成する必要があります。その開発者アプリ内のキーとトークンは、recent ポスト件数エンドポイントで使用できます。full-archive ポスト件数エンドポイントを利用したい場合、または高度なオペレーターやより長いクエリを使用したい場合は、Enterprise アクセスの承認を受けている必要があります。

詳しくは、Enterprise アクセスに関するセクションをご覧ください。

<div id="rate-limits">
  #### レート制限
</div>

毎日、何千もの開発者が X API にリクエストを送信しています。リクエスト量を管理するために、各エンドポイントには [レート制限](/ja/x-api/fundamentals/rate-limits) が設けられており、開発者がアプリまたは認証済みユーザーに代わって行えるリクエスト数が制限されています。

このエンドポイントにはアプリレベルのレート制限が適用されます。つまり、開発者であるあなたは、使用している認証情報によって識別される任意のアプリから、このエンドポイントに対して一定期間内に行えるリクエスト数が制限されます。 

<div id="building-queries">
  #### クエリの構築
</div>

これらのエンドポイントの中核となる機能は、単一のクエリを使ってポストをフィルタリングし、返される件数を決定することです。これらのクエリは、メッセージのキーワード、ハッシュタグ、URL などのポストおよびユーザー属性にマッチするオペレーターで構成されます。オペレーターは、ブールロジックや丸括弧と組み合わせてクエリを作成することで、クエリのマッチング動作をより細かく調整できます。

詳しくは、[クエリの構築方法](/ja/x-api/posts/counts/integrate/build-a-query)のガイドをご覧ください。

<div id="pagination">
  #### ページネーション
</div>

最近のポスト件数を取得する場合、`next_token` は返されません。これは、粒度に関係なく、直近7日間のポスト件数を1回の API 呼び出しで取得できることを意味します。

フルアーカイブのポスト件数を取得する場合、直近30日間のデータが返されます。30日より前のデータについては、`next_token` が返され、その `next_token` を使用してページネーションし、追加のデータを取得できます。 

<div id="building-queries-for-post-counts">
  ### ポスト件数のクエリを構築する
</div>

Post counts エンドポイントは、単一のクエリを指定した GET リクエストを受け取り、そのクエリに一致する過去のポスト件数のセットを返します。クエリは、さまざまなポスト属性に対してマッチさせるために使用されるオペレーターで構成されます。
