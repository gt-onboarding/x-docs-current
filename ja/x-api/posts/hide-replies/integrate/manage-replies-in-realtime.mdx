<div id="manage-replies-in-realtime">
  ### リアルタイムで返信を管理する
</div>

hide replies エンドポイントを使うことで、ユーザーが無関係である可能性が非常に高い返信を非表示にするワークフローを構築できます。

便利なアプリは、多くの場合、複数の技術を組み合わせることでユーザーにとって価値あるものになります。このページでは、[Perspective API](https://www.perspectiveapi.com/) を使って返信を非表示にする方法を説明します。この API は、Google 内の組織である [Jigsaw](https://jigsaw.google.com/) によって、有害なコメントを検出するように訓練された人工知能です。アプリケーションロジックは次のように動作します。

1.  ユーザーのポストを読み取り、その返信を非表示または再表示する許可をユーザーに求めます。
2.  [Account Activity API](https://developer.x.com/en/docs/x-api/v1/accounts-and-users/subscribe-account-activity/overview) を使用して、受信した返信を検出します。
3.  Perspective API に、「スコア」（0 から 1 の数値）を要求します。これは、そのアルゴリズムが、あるコメントが過去に見た有害なコメントにどれくらい似ているかについての確信度を示します（Perspective はサービスに送信されたテキストを保存しません）。
4.  アルゴリズムのスコアが非常に高い場合に、hide replies エンドポイントを呼び出します。

<div id="strive-for-transparency">
  #### 透明性を心がける
</div>

Perspective は実際のポストデータを用いて学習されていないため、言語のニュアンスによっては、ユーザーが非表示にしたくない返信をこのアプリが非表示にしてしまう場合があります。アプリを設計する際にどのような技術やアプローチを使用する場合でも、常に最善を尽くして、アプリが何を非表示にしているのかをユーザーが理解し、自分で変更できるようにしてください。

* 最も望ましいのは、常にユーザーを信頼し、その判断を完全にコントロールできるようにすることです。つまり、ユーザー体験には、アプリがユーザーに代わって行ったあらゆる操作を元に戻せるコントロールを含めるべきです。
* 人工知能を使用する場合、ポストを検出して非表示にする際には、非常に高い信頼度のしきい値を使用するようにしてください。
* すべての人が同じ言葉を使うわけではないため、アプリはバイアスを避けるように設計する必要があります。リクレイムされた言葉やスラングによって誤検知（false positive）が発生する可能性があることに注意してください。
* 人工知能を学習させる場合は、X でよく使われる言語をできるだけ忠実に反映したモデルを採用することを検討してください。
