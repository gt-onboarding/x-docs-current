<div id="manage-replies-in-realtime">
  ### リアルタイムでリプライを管理する
</div>

hide replies エンドポイントを使うと、ユーザーが無関係である可能性が非常に高いリプライを非表示にするためのワークフローを構築できます。

有用なアプリは、多くの場合、複数のテクノロジーを組み合わせることでユーザーに価値を提供します。このページでは、[Perspective API](https://www.perspectiveapi.com/) を使ってリプライを非表示にする方法を説明します。この API は、Google 傘下の組織である [Jigsaw](https://jigsaw.google.com/) によってトレーニングされた人工知能で、有害なコメントを検出するためのものです。アプリケーションロジックは次のように動作します。

1.  ユーザーのポストを読み取り、そのリプライを非表示または再表示するための許可をユーザーに求めます。
2.  [Account Activity API](https://developer.x.com/en/docs/x-api/v1/accounts-and-users/subscribe-account-activity/overview) を使用して、受信したリプライを検出します。
3.  Perspective API に対して、そのアルゴリズムがコメントをこれまでに見てきた有害コメントとどの程度類似していると判断しているかを示す「スコア」（0〜1 の数値）を取得します（Perspective はサービスに送信されたテキストを保存しません）。
4.  アルゴリズムのスコアが非常に高い場合に、hide replies を呼び出します。

<div id="strive-for-transparency">
  #### 透明性を重視する
</div>

Perspective は実際のポストで学習されていないため、特定の言語的なニュアンスによっては、ユーザーが非表示にしたくない返信をアプリが非表示にしてしまう場合があります。アプリを設計する際には、どのような技術やアプローチを用いる場合でも、アプリが何を非表示にしているのかをユーザーが理解し、必要に応じて変更できるよう、最大限の努力をしてください。

* 最も望ましいのは、常にユーザーを信頼し、その判断が常に優先されるようにすることです。これは、ユーザー体験に、アプリがユーザーに代わって行ったあらゆる操作を元に戻すためのコントロールを含めるべきであることを意味します。
* 人工知能を使用する場合、ポストを検出して非表示にする際には、非常に高い信頼度の閾値を使用する必要があります。
* すべての人が同じ言葉を使うわけではないため、アプリはあらゆるバイアスを避けるように設計されている必要があります。当事者によってポジティブな意味で使い直されている言葉（リクレイムされた言葉）やスラングが、誤検知につながる可能性があることに注意してください。
* 人工知能を学習させる場合は、X 上でよく使われる言語をできるだけ反映したモデルの採用を検討してください。
