  ## X API の返信非表示エンドポイントの比較
</div>

v2 の返信非表示エンドポイントは、Labs の返信非表示エンドポイントに置き換わります。Labs 版のこのエンドポイントを使用するコード、アプリ、ツールがあり、新しい X API v2 エンドポイントへの移行を検討している場合は、本ガイドがお役に立ちます。

新しい X API v2（返信非表示エンドポイントを含む）を使用するには、[新しい developer portal にオプトインする](https://developer.x.com/en/portal/opt-in)、[プロジェクト](/ja/resources/fundamentals/projects)を作成する、そのプロジェクトにアプリを追加する、という手順が必要です。その後、そのアプリに紐づく認証情報を使って返信非表示エンドポイントへリクエストを送信できます。Labs v2 返信非表示エンドポイント用に登録されているものと同じアプリを追加すると、ユーザーを引き続き認証済みの状態に保てます。

次の表は、Labs と新しい X API v2 エンドポイントの違いを比較したものです。
 

| **Description** | **Labs v2** | **X API v2** |
| :--- | :--- | :--- |
| Host domain | https://api.x.com | https://api.x.com |
| Endpoint path | /labs/2/tweets/:id/hidden | /2/tweets/:id/hidden |
| Authentication | OAuth 1.0a User context | OAuth 1.0a User context |
| HTTP methods supported | PUT | PUT |
| Default request rate limits | 10 requests per 15 minutes (共有プール内のすべての認証済みユーザーで共有) | 50 requests per 15 minutes (認証済みユーザーごと) |
| Can hide replies | ✔︎  | ✔︎  |
| Can unhide a previously hidden reply | ✔︎  | ✔︎  |
| Can hide or unhide replies multiple times | ✔︎  | ✔︎  |
| [プロジェクト](/ja/resources/fundamentals/projects) に紐づく [developer App](/ja/resources/fundamentals/developer-apps) の認証情報の使用が必須 |     | ✔   |
