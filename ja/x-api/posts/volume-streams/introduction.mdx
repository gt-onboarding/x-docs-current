* 1% サンプリングストリーム。
* 10% サンプリングストリーム。一般的に「Decahose」と呼ばれます。

これらはどちらも大量のデータを配信するため、「ボリュームストリーム」と呼ばれています。1% ストリームであっても、毎秒数十件以上のポストが配信される可能性があります。これらのストリームを使用すると、トレンドの特定・追跡、一般的な感情分析、世界的なイベントのモニタリングなど、さまざまなことが行えます。 

これらのストリーミングエンドポイントは、持続的な HTTP GET 接続を介して [Post objects](/ja/x-api/fundamentals/data-dictionary#tweet) を配信し、[OAuth 2.0 App-Only](/ja/resources/fundamentals/authentication#oauth-2-0) 認証を使用します。Essential アクセスレベルでは、一度に保持できる接続は 1 つです。すべてのアクセスレベルで、接続リクエストは 15 分間に最大 50 回まで送信できます。

これらのボリュームストリームエンドポイントは、編集済みポストをサポートします。これらのエンドポイントは、編集履歴およびポスト ID の配列とともに編集済みポストを配信します。編集履歴のないポストの場合、この配列には単一の ID が格納されます。編集されたポストの場合、この配列には複数の ID が含まれ、編集順を反映した昇順で並んでおり、配列の最後の要素が最新バージョンになります。ポスト編集の仕組みの詳細については、[Post edits fundamentals](/ja/x-api/fundamentals/edit-posts) ページを参照してください。 

*これらの API を使用するには、まず当社の Enterprise セールスチームを通じてアカウントをセットアップする必要があります。*

<Note>
  **アカウントのセットアップ**

  これらのエンドポイントにアクセスするには、次のものが必要です。

  * 承認済みの [開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)。
  * [プロジェクト](/ja/resources/fundamentals/projects) 内にある [開発者アプリ](/ja/resources/fundamentals/developer-apps) のキーとトークンを使用した認証。 

  X API v2 エンドポイントへのアクセス方法の詳細は、[はじめに](/ja/x-api/getting-started/getting-access) ガイドを参照してください。
</Note>
