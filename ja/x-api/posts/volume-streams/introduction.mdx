* 1% サンプリングストリーム。
* 10% サンプリングストリーム。一般的に「Decahose」と呼ばれます。

これらはいずれも大量のデータを配信するため、「ボリュームストリーム」と呼ばれます。1% ストリームであっても、毎秒数十件ものポストを送出する場合があります。これらのストリームを使用することで、トレンドの特定と追跡、全体的なセンチメントのモニタリング、世界的なイベントの監視など、さまざまな用途に利用できます。 

これらのストリーミングエンドポイントは、永続的な HTTP GET 接続を通じて [Post objects](/ja/x-api/fundamentals/data-dictionary#tweet) を配信し、[OAuth 2.0 App-Only](/ja/resources/fundamentals/authentication#oauth-2-0) 認証を使用します。Essential アクセスでは、一度に 1 つの接続を保持できます。すべてのアクセスレベルで、接続リクエストは 15 分間に最大 50 回まで行うことができます。

これらのボリュームストリームエンドポイントは、編集されたポストをサポートします。これらのエンドポイントは、編集履歴を含む編集済みポストを配信し、その一部としてポスト ID の配列を含みます。編集履歴のないポストの場合、この配列には 1 つの ID が格納されます。編集されたポストの場合、この配列には編集の順序を反映した昇順で複数の ID が格納されており、配列の最後の要素が最新バージョンです。ポスト編集の仕組みについて詳しくは、[Post edits fundamentals](/ja/x-api/fundamentals/edit-posts) ページをご覧ください。 

*これらの API を使用するには、まず Enterprise 営業チームを通じてアカウントを設定する必要があります。*

<Note>
  **アカウントの設定**

  これらのエンドポイントにアクセスするには、次のものが必要です。

  * 承認済みの [developer account](https://developer.x.com/en/portal/petition/essential/basic-info)。
  * [Project](/ja/resources/fundamentals/projects) 内にある [developer App](/ja/resources/fundamentals/developer-apps) のキーとトークンを使用して認証すること。 

  X API v2 エンドポイントへのアクセス方法について詳しくは、[getting started guide](/ja/x-api/getting-started/getting-access) を参照してください。
</Note>
