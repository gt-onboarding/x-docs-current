<div id="getting-started-with-the-quote-tweets-lookup-endpoint">
  ## 引用ポストルックアップエンドポイントの使用を開始する
</div>

このクイックスタートガイドでは、[Postman](/ja/tutorials/postman-getting-started) を使用して、指定した一連のフィールド付きで引用ポストルックアップエンドポイントに対して最初のリクエストを送信する方法を説明します。

さまざまなプログラミング言語向けのサンプルコードを確認したい場合は、[X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) の GitHub リポジトリをご覧ください。 

<div id="prerequisites">
  #### 前提条件
</div>

このガイドを完了するには、リクエストを認証するための [keys and tokens](/ja/resources/fundamentals/authentication)（キーとトークン）が必要です。これらのキーとトークンは、次の手順に従って生成できます。

* [開発者アカウントに登録](https://developer.x.com/en/apply-for-access)し、承認を受けます。
* developer portal で [プロジェクト](/ja/resources/fundamentals/projects) と、それに関連付けられた [開発者アプリ](/ja/resources/fundamentals/developer-apps) を作成します。
* アプリの「Keys and tokens」ページに移動して、必要な認証情報を生成します。すべての認証情報は安全な場所に保存してください。

**Quote Tweets ルックアップリクエストを構築する手順**

**ステップ 1: ツールまたはライブラリを選択する**

このエンドポイントにリクエストを送信するために使用できるツール、コード例、ライブラリはいくつかありますが、ここでは手順を簡略化するために Postman を使用します。

X API v2 の Postman コレクションを自分の環境に読み込むには、次のボタンをクリックしてください。

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2 を Postman に追加
</Button>

Postman に X API v2 コレクションが読み込まれたら、timeline フォルダに移動し、「Quote Tweets by ID」リクエストを探します。

**ステップ 2: リクエストを認証する**

X API に対して正しくリクエストを行うには、自分に権限が付与されていることを検証する必要があります。このエンドポイントでそうするには、[OAuth 2.0 App-Only](/ja/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)、[OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)、または [OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) のいずれかの認証方法でリクエストを認証する必要があります。

簡略化のため、ここではこのリクエストに OAuth 2.0 App-Only を使用しますが、非公開の [メトリクス](/ja/x-api/fundamentals/metrics) やポストをリクエストしたい場合は、他のいずれかの認証方法を使用する必要があります。 

Postman に、[App Access Token](/ja/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)（App-only Bearer Token とも呼ばれます）を含む、自分のキーとトークンを追加する必要があります。これを行うには、Postman の右上で「X API v2」という名前の Environment（環境）を選択し、「initial value」と「current value」のフィールドに（Environment ドロップダウン横の目のアイコンをクリックして）キーとトークンを追加します。

正しく設定されていれば、これらの変数はリクエストの Authorization タブに自動的に取り込まれます。

**ステップ 3: Quote Tweets を取得したいポストを特定して指定する**

Quote Tweets を取得したいポストを、リクエスト内で Post ID を渡すことで指定する必要があります。

この例では、使用する Post ID は 1409931481552543749 です。

Postman で「Params」タブに移動し、この Post ID を `id` パラメーターの「Value」列に入力します。

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 1409931481552543749 |

**ステップ 4: 取得したいフィールドを特定して指定する**

ステップ 3 の後に「Send」ボタンをクリックすると、レスポンスにはデフォルトの [Post object](/ja/x-api/fundamentals/data-dictionary#tweet) のフィールドである id、text、edit&#95;history&#95;tweet&#95;ids が含まれます。

追加のフィールドを取得したい場合は、[field](/ja/x-api/fundamentals/fields) パラメーターおよび/または [expansion](/ja/x-api/fundamentals/expansions) パラメーターを使って、それらのフィールドをリクエストで指定する必要があります。

この演習では、異なるオブジェクトから 3 つの追加フィールド セットをリクエストします。

1. メインのポストオブジェクトにおける追加の tweet.created&#95;at フィールド
2. 返されたポストに対応する著者のユーザーオブジェクトのデフォルトフィールド: id、name、username
3. 関連するユーザーオブジェクトにおける追加の user.created&#95;at フィールド
    

Postman で「Params」タブに移動し、「Query Params」テーブルに次の key:value ペアを追加します。

|     |     |     |     |
| :--- | :--- | :--- | :--- |
| **キー** | **値** | **返されるフィールド** |
| `tweet.fields` | `created_at` | `tweets.created_at` |
| `expansions` | `author_id` | `includes.users.id`, `includes.users.name`, `includes.users.username` |
| `user.fields` | `created_at` | `includes.users.created_at` |     |

これで、「Send」ボタンの横に次の URL が表示されているはずです:

`https://api.x.com/2/tweets/:id/quote_tweets?expansions=author_id&tweet.fields=created_at&user.fields=created_at`

**注意:**

Postman では、URL フィールド内のパスパラメータ :id は、`id` パラメータフィールドに入力した値に自動的には更新されません。そのため、上記の URL には 1409931481552543749 ではなく `:id` が含まれています。

<div id="step-six-paginate-through-your-results">
  #### ステップ6：結果をページネーションする
</div>

前のレスポンスの一番下には、次のフィールドを含む meta データオブジェクトがあります。

* results&#95;count
* next&#95;token

ステップ4では max&#95;results パラメータを指定しなかったため、デフォルトではポストを 10 件しか取得できません。追加のページのデータにアクセスするには、直前の結果から next&#95;token フィールドの値を取得し、その文字列を Postman の params ページで pagination&#95;token パラメータの値として設定し、それ以外の設定はすべて同じに保ちます。 

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `pagination_token` | avdjwk0udyx6 |

これらの設定が完了したら、再度「Send」をクリックすると、次のページの結果を受け取ることができます。 

この概念をさらに説明するために、[ページネーション](/ja/x-api/fundamentals/pagination) に関するガイドも用意しています。
