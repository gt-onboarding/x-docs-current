<div id="getting-started-with-the-likes-lookup-endpoint">
  ### Likes lookup エンドポイントの利用を開始する
</div>

このクイックスタートガイドでは、[Postman](/ja/tutorials/postman-getting-started) を使用して Likes lookup エンドポイントに最初のリクエストを送信する方法を説明します。

さまざまな言語で記述されたサンプルコードを確認したい場合は、[X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) の GitHub リポジトリをご覧ください。
 

<div id="prerequisites">
  #### 前提条件
</div>

このガイドを完了するには、次の手順に従って生成できる[キーとトークン](/ja/resources/fundamentals/authentication)一式が必要です。

1. [開発者アカウントを申請](https://developer.x.com/en/apply-for-access)し、承認を受けてください。

2. developer portal で[プロジェクト](/ja/resources/fundamentals/projects)と、それに関連付けられた[開発者アプリ](/ja/resources/fundamentals/developer-apps)を作成してください。

3. アプリの「Keys and tokens」ページに移動し、API Keys、Access Tokens、Bearer Token をパスワードマネージャーに保存してください。

<div id="steps-to-build-a-likes-lookup-request">
  #### Likes ルックアップリクエストを作成する手順
</div>

**ステップ 1: ツールまたはライブラリを用意する**

このエンドポイントにリクエストを送信する方法としては、いくつかのツール、コード例、ライブラリを利用できますが、ここでは手順を簡素化するために Postman ツールを使用します。

X API v2 の Postman コレクションを環境に読み込むには、次のボタンをクリックしてください。

[![](https://cdn.cms-twdigitalassets.com/content/dam/developer-twitter/m1_vnext/carat.svg)<br />Postman に X API v2 を追加](https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400)

Postman に X API v2 コレクションを読み込んだら、&quot;Likes&quot; フォルダーに移動し、&quot;Liking users&quot; を選択します。 
 

**ステップ 2: リクエストを認証する**

このエンドポイントへのリクエストを成功させるには、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) か [OAuth 2.0 Bearer Token](/ja/resources/fundamentals/authentication#oauth-2-0) 認証のいずれかを使用する必要があります。これを行うには、Postman で &quot;X API v2&quot; という名前の environment を選択し、Initial value と Current value のフィールドに次の変数を追加して、キーとトークンを設定します。

* consumer&#95;key に API Key
* consumer&#95;secret に API Key Secret
* access&#95;token に Access Token
* token&#95;secret に Access Token Secret
   

**ステップ 3: ポストを指定する**

このエンドポイントでは、いいねしたユーザーを取得したいポストの ID（Tweet ID）を指定する必要があります。ポストの ID は、X 上でそのポストに移動し、URL の末尾にある数値のコードを取得することで確認できます。たとえば、次の URL の Tweet ID は 1354143047324299264 です。

https://x.com/TwitterDev/status/1354143047324299264

Postman で &quot;Params&quot; タブに移動し、セクション下部の tweet&#95;id パス変数の &quot;Value&quot; 列にこの ID を入力します。このとき、ID の前後にスペースが入らないようにしてください。 

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | いいねしたユーザーを取得したいポストの Tweet ID |

**ステップ 4: 取得したいフィールドを特定して指定する**

ステップ 3 の後に &quot;Send&quot; ボタンをクリックすると、レスポンスにはデフォルトの [user オブジェクト](/ja/x-api/fundamentals/data-dictionary#user) のフィールドである id、name、username が含まれます。

id、name、username 以外の追加のフィールドを受け取りたい場合は、[fields](/ja/x-api/fundamentals/fields) および/または [expansions](/ja/x-api/fundamentals/expansions) パラメーターを使って、リクエスト内でそれらのフィールドを指定する必要があります。

この手順では、異なるオブジェクトから次の 3 つの追加フィールドセットをリクエストします。

1. メインの user オブジェクトにおける追加の user.created&#95;at フィールド
2. 返されたユーザーに関連付けられたピン留めされたポストオブジェクトのデフォルトフィールド: id と text
3. 関連するポストオブジェクトにおける追加の tweet.created&#95;at フィールド

Postman で &quot;Params&quot; タブに移動し、&quot;Query Params&quot; テーブルに次の key:value ペアを追加します。

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| user.fields | created&#95;at | user.created&#95;at |
| expansions | pinned&#95;tweet&#95;id | tweet.id, tweet.text |
| tweet.fields | created&#95;at | tweet.created&#95;at |

&quot;Send&quot; ボタンの横に次の URL が表示されていることを確認してください。

`https://api.x.com/2/tweets/1354143047324299264/liking_users?user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=created_at`

**ステップ6：ユーザーが「いいね」したポストを取得する**

ユーザーが「いいね」したポストを取得するリクエストも行いたい場合があるでしょう。Likes lookup エンドポイントを使用すると、ユーザーが「いいね」したポストに関する情報を取得できます。これを行うには、「Likes」フォルダに移動し、「Liked Tweets」を選択します。 

このエンドポイントでは、「いいね」したポストを取得したいユーザーの User ID を指定する必要があります。この情報を取得するには、[user lookup](/ja/x-api/users/lookup/introduction) エンドポイントを使用できます。

Postman で「Params」タブに移動し、このユーザー名を id パス変数（セクションの一番下）の「Value」列に入力し、ユーザー名の前後にスペースが含まれないようにしてください。 

|                 |                               |
| :-------------- | :---------------------------- |
| **Key**         | **Value**                     |
| id              | 「いいね」したポストを取得したいユーザーの User ID |
| max&#95;results | 5                             |

これで、「Send」ボタンの横に表示される URL が、TwitterDev ではなくあなたの ID を含む同様の URL になっているのが確認できるはずです：
 

`https://api.x.com/2/users/2244994945/liked_tweets?max_results=5`
