<div id="likes-lookup">
  ### いいねのルックアップ
</div>

Likes lookup グループのエンドポイントを使用すると、特定のポストに「いいね」したアカウントのリスト、または特定のアカウントが「いいね」したポストのリストを取得できます。これらのエンドポイントには、次のものが含まれます。

* ユーザーが「いいね」したポスト - GET /2/users/:id/liked&#95;tweets
* ポストに「いいね」したユーザー - GET /2/tweets/:id/liking&#95;users

これらのエンドポイントは、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Bearer Token](/ja/resources/fundamentals/authentication#oauth-2-0) のいずれかで認証できます。liked Posts エンドポイントでは、大量の結果セットをページングするためのページネーション用トークンが提供されます。

liking users エンドポイントでは、1 つのポストにつき、通算で liking アカウントは最大 100 件までに制限されます。さらに、liked Posts エンドポイントには、プロジェクトレベルで適用される月次の [Post cap](/ja/x-api/fundamentals/post-cap) も適用されます。
 

<div id="manage-likes">
  ### いいねを管理する
</div>

いいね管理用エンドポイントを使用すると、認証済みアカウントを代表して、指定したポストに「いいね」または「いいねの取り消し」を行うことができます。このエンドポイントグループには、POST メソッドと DELETE メソッドの 2 種類があります。POST メソッドはポストに「いいね」を付けるために使用し、DELETE メソッドはポストの「いいね」を取り消すために使用します。

ユーザーの代理でリクエストを送信するため、これらのエンドポイントは [OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) で認証し、ユーザーに紐づくアクセストークンを使用する必要があります。アクセストークンは [3-legged OAuth flow](/ja/resources/fundamentals/authentication#oauth-1-0a-2)/obtaining-user-access-tokens) を使って生成できます。ご自身のアカウントや、認証済みユーザーのアカウントのポストに対して「いいね」を付けることができます。どちらのエンドポイントにも、ユーザー単位で各エンドポイントごとに 15 分あたり 50 回というレート制限があります。 
 

これらのエンドポイントにアクセスするには、承認済みの [開発者アカウント](/ja/resources/fundamentals/developer-portal) が必要です。認証を行う際は、[プロジェクト](/ja/resources/fundamentals/projects) 内の [開発者アプリ](/ja/resources/fundamentals/developer-apps) から取得したキーとトークンを使用する必要があります。 

X API v2 のエンドポイントへのアクセス方法の詳細は、[getting started](/ja/x-api/getting-started/about-x-api) ページを参照してください。
