<div id="likes-lookup">
  ### いいねのルックアップ
</div>

Likes lookup グループのエンドポイントを使用すると、あるポストに「いいね」したアカウントのリスト、または特定のアカウントが「いいね」したポストのリストを取得できます。これらのエンドポイントには次のものが含まれます。

* ユーザーが「いいね」したポスト - GET /2/users/:id/liked&#95;tweets
* ポストに「いいね」したユーザー - GET /2/tweets/:id/liking&#95;users

これらのエンドポイントは、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Bearer Token](/ja/resources/fundamentals/authentication#oauth-2-0) のいずれかで認証できます。liked Posts エンドポイントでは、大きな結果セットをページングするためのページネーション用トークンが提供されます。

liking users エンドポイントでは、1 つのポストにつき、全期間を通して「いいね」したアカウントは合計 100 件までに制限されています。さらに、liked Posts エンドポイントには、プロジェクト単位で毎月の [Post cap](/ja/x-api/fundamentals/post-cap) が適用されます。
 

<div id="manage-likes">
  ### いいねの管理
</div>

Manage Likes エンドポイントを使用すると、認証済みアカウントを代表して、指定したポストに「いいね」したり、「いいね」を取り消したりできます。このエンドポイントグループでは、利用可能なメソッドは POST と DELETE の 2 種類です。POST メソッドではポストに「いいね」でき、DELETE メソッドではポストの「いいね」を取り消すことができます。

ユーザーを代表してリクエストを送信するため、これらのエンドポイントは [OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) で認証し、ユーザーに紐づいたアクセストークンを使用する必要があります。アクセストークンは [3-legged OAuth flow](/ja/resources/fundamentals/authentication#oauth-1-0a-2)/obtaining-user-access-tokens) を使って生成できます。自分のアカウント、または認証済みユーザーのアカウントからポストに「いいね」できます。どちらのエンドポイントも、ユーザーごとのレート制限は、各エンドポイントについて 15 分あたり 50 リクエストです。  

これらのエンドポイントにアクセスするには、承認済みの[開発者アカウント](/ja/resources/fundamentals/developer-portal)が必要です。認証を行う際は、[プロジェクト](/ja/resources/fundamentals/projects)内にある[開発者アプリ](/ja/resources/fundamentals/developer-apps)のキーとトークンを使用する必要があります。  

X API v2 エンドポイントへのアクセス方法の詳細については、[はじめに](/ja/x-api/getting-started/about-x-api)ページをご覧ください。
