このページでは、X API に含まれる Spaces エンドポイントについて説明します。Spaces 全般の詳細については、[help.x.com](https://help.x.com/en/using-twitter/spaces) を参照してください。 

Spaces は、ライブ音声による会話を通じて表現や交流を行うことを可能にします。Spaces エンドポイントは、Spaces を中心に新しい機能を構築するためのツールを提供します。これらのエンドポイントを使用して、ライブまたはスケジュール済みの Spaces を検索したり、ユーザーが興味を持ちそうな Spaces を見つけられるような発見体験を構築したりできます。

Spaces を、私たちが用意した形にとどまらず、皆さんの創造性を活かして拡張していただくことを歓迎します。これらのエンドポイントを使えば、タイトル内に含まれるキーワードに基づいてリスナーに Spaces を提案する体験を構築したり、ライブまたは今後開催予定の Spaces をホストしていて、かつユーザーにフォローされているアカウントを見つけやすくしたりできます。また、ホストが自分の Spaces のパフォーマンスをよりよく把握し、オーディエンスに関するより深い洞察を得られるよう支援することも可能です。

<div id="understanding-the-lifecycle-of-spaces">
  #### Spaces のライフサイクルを理解する
</div>

X Developer Platform の他のリソースとは異なり、Spaces には一定のライフサイクルがあります。Spaces は、開始予定日時の最大 14 日前までスケジュールでき、終了後は利用できなくなります。ホストは、開始前であれば、事前にスケジュールされた Space をいつでもキャンセルできます。

Spaces はライブの間はアクセス可能ですが、一度終了すると、Spaces エンドポイントを使用して取得することはできなくなり、この状態を示すエラーメッセージが返されます。

アプリで Spaces のデータを扱う場合、常に最新の情報を返し、プラットフォーム上で利用できなくなったデータを削除する責任があります。[Spaces lookup エンドポイント](/ja/x-api/spaces/lookup/introduction) を使用することで、ユーザーの期待や意図を尊重できているかを確認できます。
 

<div id="roles-in-spaces">
  ### スペースにおける役割
</div>

これらのエンドポイントは、X アプリにおけるスペース機能の動作を反映しています。スペースでは、X ユーザーはスペースへの関わり方やスペース内での参加の仕方に応じて、明確な役割を持つことができます。
 

<div id="creator-or-primary-host">
  ### クリエイター（またはプライマリホスト）
</div>

プライマリホストとは、Space を作成したユーザーであり、その Space 自体の所有者です。現在、Space にはホストを 1 人しか設定できないため、プライマリホストが唯一のホストとなります。[Spaces データディクショナリ](/ja/x-api/fundamentals/data-dictionary#space)では、プライマリホストの情報は `creator_id` フィールドに含まれており、[ユーザーオブジェクト](/ja/x-api/fundamentals/data-dictionary#user)として展開できます。
 

<div id="hosts">
  ### ホスト
</div>

プライマリホストは、1 人以上のユーザーを共同ホストに指定できます。Spaces のデータディクショナリでは、これらのホストは `host_ids` として表現され、ユーザーオブジェクトのリストに展開できます。Space の開催中であってもホストの指定は変更される可能性があり、これらのエンドポイントから返されるメタデータは、リクエスト時点の状態を反映します。

アプリは、`creator_id` の値を確認することでプライマリホストを特定し、`host_ids` の値を確認することで誰が共同ホストであるかを把握できます。
 

<div id="speaker">
  ### スピーカー
</div>

スピーカーは、スペース内で発言する権限を持つユーザーです。任意の時点で 0 人以上のスピーカーが存在でき、1 つのスペースにはホストを含めて最大 10 人までスピーカーが参加できます。スペースのデータ辞書では、スピーカーは `speaker_ids` リストで返され、これを展開してユーザーオブジェクトのリストとして取得できます。
 

<div id="listener">
  ### リスナー
</div>

リスナーはスペースを視聴し、あらかじめ用意されたリアクションをいつでも使用でき、（ホストがスペースの設定で許可している場合は）スピーカーになることをリクエストすることもできます。リスナーに関する情報は、ホストを含む参加者数の合計としてのみ、`participant_count` フィールドに返されます。
