X Activity API (XAA) エンドポイントグループを使用すると、開発者は X プラットフォーム上で発生するアクティビティイベントにアクセスできます。

開発者は、`ProfileBioUpdate`、`ProfilePictureUpdate` など関心のあるイベントを購読し、対象とする User ID でフィルタリングできます。その User ID に該当するイベントは、1 秒未満のレイテンシーであなたのアプリに配信されます。

<Warning>
  X Activity API はオープンベータ版として提供されています。オープンベータ期間中は、不具合、後方互換性を損なう変更、または未完成の機能が含まれる可能性があります。

  皆さまのフィードバックは非常に重要です。[developer community forums](https://devcommunity.x.com/) から問題を報告してください。
</Warning>

<div id="delivery-mechanisms">
  ## 配信方式
</div>

X Activity API では現在、イベントをアプリに送信するために、次の配信方式がサポートされています。

* 永続的な HTTP ストリーム
* [Webhook](/ja/x-api/webhooks/introduction)

<div id="supported-event-types">
  ## サポートされているイベントタイプ
</div>

現在、X Activity API では、カテゴリ別に整理された以下のイベントタイプをサポートしています。

<div id="profile-events">
  ### プロフィールイベント
</div>

プロフィールイベントは、ユーザーがプロフィール情報を変更したときにトリガーされます。

| Event Name                   | Description                                         | Filters   |
| ---------------------------- | --------------------------------------------------- | --------- |
| `profile.update.bio`           | ユーザーがプロフィールの自己紹介を更新したときに発生します         | `user_id` |
| `profile.update.profile_picture`       | ユーザーがプロフィール画像を更新したときに発生します     | `user_id` |
| `profile.update.banner_picture`        | ユーザーがプロフィールのバナー画像を更新したときに発生します      | `user_id` |
| `profile.update.geo`           | ユーザーがプロフィールの位置情報を更新したときに発生します    | `user_id` |
| `profile.update.url`           | ユーザーがプロフィールのウェブサイトのURLを更新したときに発生します | `user_id` |
| `profile.update.screenname`    | ユーザーがプロフィールの表示名を更新したときに発生します        | `user_id` |
| `profile.update.verified_badge` | ユーザーが認証バッジを更新したときに発生します      | `user_id` |

<div id="follow-events">
  ### フォローイベント
</div>

フォローイベントは、ユーザーが別のユーザーをフォローまたはフォロー解除したときにトリガーされます。

| イベント名 | 説明                                      | フィルター   |
| ---------- | ----------------------------------------- | --------- |
| `follow.follow`  | ユーザーが別のユーザーをフォローしたときに発生します | `user_id` |
| `follow.unfollow`  | ユーザーが別のユーザーのフォローを解除したときに発生します | `user_id` |

<Note>
  **Enterprise 限定:** フォローイベントは現時点では一部の Enterprise および Partner ティアのアカウントでのみ利用可能です。
</Note>

<div id="news-events">
  ### ニュースイベント
</div>

ニュースイベントは、Grok によってキュレーションされたニューストピックや見出しに関する最新情報を提供します。

| Event Name | Description                           | Filters   |
| ---------- | ------------------------------------- | --------- |
| `news.new`  | Grok によってキュレーションされた新たなニュースや見出し | `keyword` |

<Note>
  **Enterprise のみ:** 現在、ニュースイベントは一部の Enterprise および Partner ティアのアカウントでのみ利用可能です。
</Note>

今後のリリースでは、XAA はソーシャルインタラクション、コンテンツエンゲージメント、収益化機能などを含む、追加のイベントタイプをサポートする予定です。新しいイベントタイプが利用可能になり次第、このドキュメントを随時更新します。

<Note>
  **注意:** XAA はポストを配信しません。リアルタイムのポスト配信については、開発者がリアルタイムにポストをフィルタリングしてストリーミングできる [Filtered Stream](/ja/x-api/posts/filtered-stream/introduction) エンドポイントを参照してください。
</Note>

<div id="event-privacy-and-authentication">
  ## イベントのプライバシーと認証
</div>

X Activity API では、後述するように X アプリと同様に **パブリックイベント** と **プライベートイベント** を区別します。

<div id="public-events">
  ### 公開イベント
</div>

公開イベントは、公開設定のユーザーアカウントが公開で行うアクティビティであり、すべての X ユーザーに表示されます。これらのイベントは X プラットフォーム上のすべてのユーザーから閲覧可能であり、閲覧する際にユーザーによる OAuth 認証は必要ありません。

**現在の公開イベント:**

* プロフィールの更新（自己紹介、画像、バナー、位置情報、URL、ユーザー名の変更）

これらの公開イベントについては、フィルターでユーザー ID を指定してサブスクリプションを作成し、XAA 経由で受信できます。

<div id="private-events">
  ### プライベートイベント
</div>

プライベートイベントは、OAuth 認証を通じてユーザーの明示的な同意が必要となるアクティビティです。ユーザーは X で認証を行い、これらのイベントへアクセスするために開発者アプリに明示的な許可を与える必要があります。

**プライベートイベントに関する認証要件:**

* ユーザーは OAuth 2.0 を使用してあなたのアプリを認証する必要があります
* あなたのアプリは適切な OAuth スコープを取得する必要があります
* ユーザーは、これらのイベントにあなたのアプリがアクセスすることを明示的に許可する必要があります
* プライベートイベントのサブスクリプションは、あなたのアプリを承認したユーザーに対してのみ作成できます

<div id="subscription-limits">
  ## サブスクリプション数の上限
</div>

X Activity API には、アカウントのティアに応じて異なるサブスクリプション数の上限があります。

| Package Tier | Maximum Subscriptions |
| ------------ | --------------------- |
| Free         | 利用不可              |
| Basic        | 50                    |
| Pro          | 2500                  |
| Enterprise   | 50,000                |
| Partner      | 100,000               |

<Note>
  **アカウント設定**

  これらのエンドポイントにアクセスするには、次のものが必要です。

  * 承認済みの [developer account](https://developer.x.com/en/portal/petition/essential/basic-info)。
  * [プロジェクト](/ja/resources/fundamentals/projects)内にある [developer App](/ja/resources/fundamentals/developer-apps) のキーとトークンを使用して認証すること。

  X API v2 エンドポイントへのアクセス方法の詳細は、[はじめにガイド](/ja/x-api/getting-started/getting-access)をご覧ください。
</Note>
