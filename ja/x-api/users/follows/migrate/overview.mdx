<div id="comparing-x-apis-follows-endpoints">
  ## X API におけるフォロー関連エンドポイントの比較
</div>

<div id="follows-lookup">
  ### フォローのルックアップ
</div>

v2 のフォロー・ルックアップエンドポイントは、Standard v1.1 の [followers/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-followers-ids)、v1.1 [followers/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-followers-list)、v1.1 [friends/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-friends-ids)、および v1.1 [friends/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-friends-list) エンドポイントを置き換えます。

次の表は、さまざまな種類のフォロー・ルックアップエンドポイントを比較したものです。

|     |     |     |
| :--- | :--- | :--- |
| **Description** | **Standard v1.1** | **X API v2** |
| サポートされる HTTP メソッド | `GET` | `GET` |
| ホストドメイン | `https://api.x.com` | `https://api.x.com` |
| エンドポイントパス | /1.1/friends/ids.json<br /><br />/1.1/friends/list.json<br /><br />/1.1/followers/ids.json<br /><br />/1.1/followers/list.json | /2/users/:id/following<br /><br />/2/users/:id/followers |
| [Authentication](/ja/resources/fundamentals/authentication) | OAuth 1.0a User Context<br /><br />App only | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code with PKCE<br /><br />App only |
| デフォルトのリクエスト[レート制限](/ja/resources/fundamentals/rate-limits) | 15 requests per 15 min (per user)<br /><br />15 requests per 15 min (per app) | 15 requests per 15 min (per user)<br /><br />15 requests per 15 min (per app) |
| レスポンスあたりの最大ユーザー数 | GET friends/id と GET followers/id は、1 ページあたり最大 5,000 件のユーザー ID を返します。<br /><br />  <br /><br />GET friends/list と GET followers/list は、1 ページあたり最大 200 件のユーザーオブジェクトを返します。 | 1 ページあたり最大 1,000 件のユーザーオブジェクト |
| ページネーション | トークンは next&#95;cursor フィールドで返され、その値を cursor パラメータに渡すことで、次のページの結果を取得できます。 | トークンは next&#95;token フィールドで返され、その値を token パラメータに渡すことで、次のページの結果を取得できます。<br /><br />v2 のペイロードでは previous&#95;token フィールドも返され、これを pagination&#95;token パラメータとともに渡すことで、前のページの結果を取得できます。 |
| JSON フォーマット | Standard v1.1 フォーマット | [X API v2 format](/ja/x-api/fundamentals/data-dictionary)（fields および expansions リクエストパラメータによって決定され、v1.1 フォーマットとの後方互換性はありません）<br /><br />Standard v1.1 フォーマットから X API v2 フォーマットへの移行方法の詳細については、[データフォーマット移行ガイド](/ja/x-api/migrate/data-format-migration)をご覧ください。 |
| ペイロードで返す [fields](/ja/x-api/fundamentals/data-dictionary) の選択をサポート |     | ✔   |
| Post の [annotations](/ja/x-api/fundamentals/post-annotations) フィールドをサポート |     | ✔   |
| 新しい [metrics](/ja/x-api/fundamentals/metrics) フィールドのリクエストをサポート |     | ✔   |
| [conversation&#95;id](/ja/x-api/fundamentals/conversation-id) フィールドをサポート |     | ✔   |
| [project](/ja/resources/fundamentals/projects) に関連付けられた [developer App](/ja/resources/fundamentals/developer-apps) の認証情報の使用が必要 |     | ✔   |

<div id="manage-follows">
  #### フォローを管理する
</div>

フォロー管理用の v2 エンドポイントは、標準的な v1.1 の [POST friendships/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-create) および [POST friendships/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-destroy) エンドポイントを置き換えます。

次の表は、標準的な v1.1 と X API v2 のフォロー作成エンドポイントを比較したものです。

<div id="follow-a-user">
  #### ユーザーをフォローする
</div>

| 説明 | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| 対応 HTTP メソッド | POST | POST |
| ホストドメイン | https://api.x.com | https://api.x.com |
| エンドポイントのパス | /1.1/friendships/create.json | /2/users/:id/following |
| [認証](/ja/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code with PKCE |
| デフォルトのリクエスト [レート制限](/ja/resources/fundamentals/rate-limits) | 15 分あたり 50 リクエスト | 15 分あたり 50 リクエスト |
| ユーザーごとの 1 日あたりの最大操作数 | 400 | 400 |
| アプリごとの 1 日あたりの最大操作数 | 1000 | 1000 |
| [プロジェクト](/ja/resources/fundamentals/projects) に関連付けられた [開発者アプリ](/ja/resources/fundamentals/developer-apps) の認証情報の使用が必要です |     | ✔️  |

<div id="unfollow-a-user">
  #### ユーザーをフォロー解除する
</div>

次の表では、Standard v1.1 と X API v2 のフォロー解除エンドポイントを比較しています。

| 説明 | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| サポートされている HTTP メソッド | POST | DELETE |
| ホストドメイン | https://api.x.com | https://api.x.com |
| エンドポイントパス | /1.1/friendships/destroy.json | /2/users/:source&#95;user&#95;id/following/:target&#95;user&#95;id |
| [認証](/ja/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code with PKCE |
| デフォルトの[レート制限](/ja/resources/fundamentals/rate-limits)（リクエスト数） | 15 分あたり 15 件のリクエスト（ユーザーごと） | 15 分あたり 50 件のリクエスト（ユーザーごと） |
| アプリごとの 1 日あたりの最大オペレーション数 | 制限なし | 500 |
| [プロジェクト](/ja/resources/fundamentals/projects) に関連付けられた [開発者アプリ](/ja/resources/fundamentals/developer-apps) の認証情報の利用が必要 |     | ✔️  |
