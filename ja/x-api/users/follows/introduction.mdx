フォロー関係の取得、作成、削除を行うために、`follows lookup` と `manage follows` という 2 つのエンドポイントグループを提供しています。

<div id="follows-lookup">
  ### フォロー関係のルックアップ
</div>

[Enterprise access](/ja/x-api/getting-started/about-x-api) をお持ちの方のみ利用できます

フォロー関係のルックアップエンドポイントを使用すると、ユーザー間の関係を探索・分析できます。これはネットワーク分析と呼ばれることもあります。具体的には、指定したユーザーがフォローしているユーザー、または指定したユーザーをフォローしているユーザーを表す [user objects](/ja/x-api/fundamentals/data-dictionary#user) を返す 2 つの REST エンドポイントがあります。

これらのエンドポイントは、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication)、[App only](/ja/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)、または [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) のいずれかで認証できます。1 回のリクエストで最大 1,000 人のユーザーまで取得でき、大量の結果をページングするためのページネーション用トークンが提供されます。
 

<div id="manage-follows">
  ### フォローの管理
</div>

manage follows エンドポイントを使用すると、ユーザーをフォローしたりフォロー解除したりできます。

ユーザーに代わってリクエストを送信するため、これらのエンドポイントを呼び出す際は必ず [OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Authorization Code with PKCE](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) で認証し、リクエスト対象のユーザーに紐づくユーザーのアクセストークンを利用する必要があります。ユーザーのアクセストークンは、[3-legged OAuth フロー](/ja/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（OAuth 1.0a）か、[Authorization Code with PKCE grant flow](/ja/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)（OAuth 2.0）を使用して生成できます。

認証済みの各ユーザーごとに、1 日あたり最大 400 件のフォロー操作に制限され、さらに認証済みユーザー全体で、1 つのアプリあたり 1 日に 1,000 件の操作に制限されます。たとえば、5 人の認証済みユーザーがいる場合、そのうち 2 人のユーザーでそれぞれ 400 ユーザーをフォロー（ユーザーごとの上限）して合計 800 件の操作を行うことができ、残りの 200 件（アプリごとの上限）を残り 3 人のユーザーで分けて使用する必要があります。この上限はフォロー解除エンドポイントには適用されず、フォロー解除には別途 1 日あたり（アプリごと）500 件の上限があります。

<Note>
  **アカウントのセットアップ**

  これらのエンドポイントにアクセスするには、次のものが必要です。

  * 承認済みの [開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)。
  * [プロジェクト](/ja/resources/fundamentals/projects) 内にある [開発者アプリ](/ja/resources/fundamentals/developer-apps) のキーとトークンを使用して認証すること。
</Note>

X API v2 エンドポイントへのアクセス方法については、[はじめにガイド](/ja/x-api/getting-started/getting-access)を参照してください。
