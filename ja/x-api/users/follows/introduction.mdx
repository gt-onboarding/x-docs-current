フォロー関係の取得・作成・削除に利用できる 2 つのエンドポイントグループを提供しています。`follows lookup` と `manage follows` です。

<div id="follows-lookup">
  ### フォロー関係のルックアップ
</div>

[Enterprise アクセス](/ja/x-api/getting-started/about-x-api)をお持ちの方のみ利用できます。

フォロー関係のルックアップエンドポイントを使用すると、ユーザー同士の関係（ネットワーク分析と呼ばれることもあります）を調査・分析できます。具体的には、指定したユーザーがフォローしているユーザー、または指定したユーザーをフォローしているユーザーを表す[user objects](/ja/x-api/fundamentals/data-dictionary#user)を返す 2 つの REST エンドポイントがあります。

このエンドポイントには、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication)、[App only](/ja/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)、または [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) のいずれかで認証できます。1 回のリクエストで最大 1,000 人のユーザーまで取得でき、大量の結果を扱うためのページネーション用トークンが提供されます。
 

<div id="manage-follows">
  ### フォローの管理
</div>

フォロー管理エンドポイントを使用すると、ユーザーをフォローまたはフォロー解除できます。

ユーザーの代理でリクエストを送信することになるため、これらのエンドポイントは必ず [OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) で認証し、リクエストを行う対象のユーザーに紐づくユーザーのアクセストークンを利用する必要があります。このユーザーアクセストークンは、[3-legged OAuth フロー](/ja/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（OAuth 1.0a）か、[Authorization Code with PKCE grant flow](/ja/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce))（OAuth 2.0）を使用して生成できます。

各認証ユーザーについて 1 日あたり 400 回までフォローアクションが可能であり、さらに、すべての認証ユーザーを合計して 1 つのアプリにつき 1 日あたり 1,000 アクションに制限されます。たとえば、5 人の認証ユーザーがいる場合、うち 2 人のユーザーでそれぞれ 1 日あたり 400 ユーザーをフォローでき（ユーザーごとの上限）、合計 800 アクションとなり、残り 200 アクション（アプリごとの残数）を残り 3 人のユーザーで分け合って使用する必要があります。この制限はフォロー解除エンドポイントには適用されず、フォロー解除には別途 1 日あたり 500 アクション（アプリごと）の制限があります。

<Note>
  **アカウントの設定**

  これらのエンドポイントにアクセスするには、次が必要です。

  * 承認済みの [開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)。
  * [プロジェクト](/ja/resources/fundamentals/projects) 内にある [開発者アプリ](/ja/resources/fundamentals/developer-apps) のキーとトークンを使用して認証すること。 
</Note>

X API v2 エンドポイントへのアクセスの取得については、[利用開始ガイド](/ja/x-api/getting-started/getting-access)をご覧ください。
