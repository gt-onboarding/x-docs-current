<div id="blocks-lookup">
  ### ブロックのルックアップ
</div>

blocks lookup の GET エンドポイントを使用すると、認可されたユーザーに代わって、そのユーザーがブロックしているアカウントを確認できます。このエンドポイントのレート制限は、ユーザーごとに 15 分あたり 15 リクエストです。 

blocks lookup では、権限 *manage blocks* を持つユーザーに代わって非公開情報へのリクエストを行うため、これらのエンドポイントは [OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) のいずれかで認証する必要があります。また、あなたのアプリを承認したユーザーに紐づくユーザーのアクセストークンを使用する必要があり、これは [3-legged OAuth フロー](/ja/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（OAuth 1.0a）または [Authorization Code with PKCE grant flow](/ja/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)（OAuth 2.0）を使って生成できます。

<Note>
  **アカウントのセットアップ**

  これらのエンドポイントにアクセスするには、以下が必要です。

  * 承認済みの [開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)。
  * [プロジェクト](/ja/resources/fundamentals/projects) 内にある [開発者アプリ](/ja/resources/fundamentals/developer-apps) のキーとトークンを使用して認証すること。 

  X API v2 のエンドポイントへのアクセス方法については、[はじめにガイド](/ja/x-api/getting-started/getting-access)をご覧ください。
</Note>
