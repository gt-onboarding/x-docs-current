<div id="blocks-lookup">
  ### ブロックのルックアップ
</div>

blocks lookup の GET エンドポイントを使用すると、認可されたユーザーに代わって、そのユーザーがどのアカウントをブロックしているかを確認できます。このエンドポイントには、ユーザーごとに 15 分間に 15 回というレート制限があります。 

blocks lookup ではブロックに関するプライベートな情報を、manage blocks 権限を持つユーザーに代わって取得することになるため、これらのエンドポイントは必ず [OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Authorization Code with PKCE](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) を使用して認証し、あなたのアプリを承認したユーザーに紐づくユーザーの Access Token を利用する必要があります。これらのトークンは、[3-legged OAuth フロー](/ja/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（OAuth 1.0a）または [Authorization Code with PKCE grant flow](/ja/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)（OAuth 2.0）を使用して生成できます。

<Note>
  **アカウントのセットアップ**

  これらのエンドポイントにアクセスするには、次のものが必要です。

  * 承認済みの [開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)。
  * [プロジェクト](/ja/resources/fundamentals/projects) 内にある [開発者アプリ](/ja/resources/fundamentals/developer-apps) のキーとトークンを使って認証すること。 

  X API v2 エンドポイントへのアクセス方法について詳しくは、[はじめにガイド](/ja/x-api/getting-started/getting-access) を参照してください。
</Note>
