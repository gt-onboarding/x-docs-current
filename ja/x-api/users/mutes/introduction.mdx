アカウントをミュートすると、そのアカウントをフォロー解除したりブロックしたりすることなく、そのアカウントのポストを自分のタイムラインから非表示にできます。ミュートされたアカウントには、あなたがミュートしたことは通知されず、いつでもミュート解除できます。manage mutes エンドポイントを使うことで、開発者は X 上の人々にとってより安全なエクスペリエンスを構築できます。manage mutes を活用した例としては、特定のトピックについてポストする可能性のあるアカウントを、指定した期間だけミュートできるアプリケーションが挙げられます。mutes lookup エンドポイントを使用すると、あなた自身または認証済みユーザーが誰をミュートしているかを確認できます。これは、ミュートしているアカウントとの関わり方を把握するのに役立ちます。 

mute lookup では非公開情報へのリクエストを行い、また manage mutes ではユーザーに代わって操作を行うため、これらのエンドポイントは必ず [OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) か [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) のいずれかで認証し、あなたのアプリを許可したユーザーに紐づくユーザーアクセストークンを使用する必要があります。これらのアクセストークンは、[3-legged OAuth フロー](/ja/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（OAuth 1.0a）または [Authorization Code with PKCE grant flow](/ja/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)（OAuth 2.0）を使用して生成できます。

<div id="mutes-lookup">
  ### ミュートの取得
</div>

ミュートの取得エンドポイントを使用すると、認証済みユーザーがミュートしているアカウントを確認できます。このエンドポイントのレート制限は、ユーザーごとに15分あたり15リクエストです。

<div id="manage-mutes">
  ### ミュートの管理
</div>

ミュート管理用エンドポイントを使用すると、認証済みユーザーに代わって指定したアカウントをミュートまたはミュート解除できます。これらのエンドポイントでは、POST と DELETE の 2 つのメソッドが利用できます。POST メソッドではアカウントをミュートし、DELETE メソッドではアカウントのミュートを解除します。POST および DELETE の各エンドポイントには、ユーザーあたり 15 分間に 50 リクエストというレート制限があります。

**注:** ユーザーが [X](https://x.com/) 上からミュートする場合、15 分間に 200 リクエストという上限があります。

**アカウントのセットアップ**

これらのエンドポイントにアクセスするには、次が必要です。

* 承認済みの[開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)
* [プロジェクト](/ja/resources/fundamentals/projects)内にある[開発者アプリ](/ja/resources/fundamentals/developer-apps)のキーとトークンを使用した認証

X API v2 エンドポイントへのアクセス方法について詳しくは、[利用開始ガイド](/ja/x-api/getting-started/getting-access)をご覧ください。
