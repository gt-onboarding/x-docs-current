認証済みユーザーのルックアップ以外のすべてのユーザー・ルックアップエンドポイントには、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication)、[App only](/ja/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)、または [OAuth 2.0 Authorization code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) を使用してリクエストを認証できます。ただし、非公開メトリクスや非公開アカウントのデータにアクセスする場合は、OAuth 1.0a User Context か OAuth 2.0 Authorization Code with PKCE を利用し、リクエストに認証済みユーザーのアクセストークンを渡す必要があります。    

このエンドポイントは、ユーザーの最新情報を取得したり、ユーザーの存在を確認したり、コンプライアンスイベント発生後に保持しているユーザー情報を更新したりするためによく使用されます。

<div id="authenticated-user-lookup">
  ### 認証ユーザーの検索
</div>

他のユーザーの代わりに認証を行う場合、そのユーザーを代理してどのユーザーについてリクエストを送信できるかを把握しておくことが重要です。

このエンドポイントを利用するには、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) を使用する必要があります。このエンドポイントは、認可されたユーザー、つまりリクエストに付与して送信するユーザーのアクセストークンに関連付けられているユーザーに関する情報を返します。

このエンドポイントには GET メソッドでアクセスできます。このエンドポイントには、15 分あたり 75 回のユーザー単位のレート制限があります。

<Note>
  **アカウント設定**

  これらのエンドポイントにアクセスするには、以下が必要です。

  * 承認済みの [開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)。
  * [プロジェクト](/ja/resources/fundamentals/projects) 内にある [開発者アプリ](/ja/resources/fundamentals/developer-apps) のキーとトークンを使用して認証すること。 

  X API v2 エンドポイントへのアクセス方法について詳しくは、[はじめにガイド](/ja/x-api/getting-started/getting-access)をご覧ください。
</Note>
