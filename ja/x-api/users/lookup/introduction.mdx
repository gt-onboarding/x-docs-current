認証済みユーザーの lookup を除くすべてのユーザー lookup エンドポイントへのリクエストは、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication)、[App only](/ja/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)、または [OAuth 2.0 Authorization code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) を使用して認証できます。ただし、非公開メトリクスや非公開ユーザーのデータにアクセスする場合は、OAuth 1.0a User Context か OAuth 2.0 Authorization Code with PKCE を利用し、リクエストに認証済みユーザーのアクセストークンを含める必要があります。    

このエンドポイントは、ユーザーの最新情報を取得したり、ユーザーの存在を確認したり、コンプライアンスイベント後に保存しているユーザー情報を更新したりする用途で一般的に使用されます。

<div id="authenticated-user-lookup">
  ### 認証済みユーザーの検索
</div>

他のユーザーを代理して認証する場合、どのユーザーの代理でリクエストを実行できるかを把握しておくことが重要です。

このエンドポイントを利用するには、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) を使用する必要があります。このエンドポイントは、認可されたユーザー、つまりリクエストと一緒に送信するユーザーのアクセストークンに関連付けられているユーザーに関する情報を返します。

このエンドポイントには GET メソッドでアクセスできます。このエンドポイントには、ユーザー単位で 15 分あたり 75 回のレート制限が適用されます。

<Note>
  **アカウントのセットアップ**

  これらのエンドポイントにアクセスするには、次が必要です。

  * 承認済みの [開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)。
  * [プロジェクト](/ja/resources/fundamentals/projects) 内にある [開発者アプリ](/ja/resources/fundamentals/developer-apps) のキーとトークンを使用して認証すること。 

  X API v2 エンドポイントへのアクセスの取得方法について詳しくは、[はじめにガイド](/ja/x-api/getting-started/getting-access) を参照してください。
</Note>
