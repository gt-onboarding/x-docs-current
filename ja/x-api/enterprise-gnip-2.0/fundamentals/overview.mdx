## 概要

> Enterprise コンソール（console.gnip.com）は、Enterprise API への契約済みアクセス権をお持ちの Enterprise データ顧客向けに提供されています。現在 Enterprise データ顧客ではないものの、詳細を知りたい場合は、[Enterprise データ](https://developer.x.com/en/products/x-api/enterprise)についてこちらでご確認いただけます。

これは、X の Enterprise コンソール ダッシュボードの概要です。ここでは、製品の詳細、利用状況、一般的なアカウント管理に関するコンソール内の各領域に対応したセクションを説明します。お客様固有のアカウントに関してご不明な点がある場合は、担当のアカウントマネージャーにお問い合わせいただくか、[Enterprise 製品向けの技術ドキュメント](/ja/x-api/enterprise-gnip-2.0/enterprise-gnip)をご確認ください。

Enterprise トライアルおよびオンボーディング プロセスの一環として、アカウントマネージャーから、Enterprise 顧客に対し [console.gnip.com](http://console.gnip.com) へのログイン情報が提供されます。これは、Enterprise 製品へのアクセスを確認および設定できるユーザーインターフェースです。コンソールへのお客様管理者の初回アクセスは、Enterprise オンボーディング プロセスの一部として X によって設定され、その後、管理者は追加ユーザーを登録できます。Enterprise コンソールを使用すると、Enterprise 製品に関連する詳細情報を管理および参照できます。 

次の動画では、console.gnip.com ダッシュボードのさまざまな部分について概要を説明しています。 

**免責事項: 動画に表示されている一部の機能（特定のストリームエンリッチメントや X 以外のデータソース製品など）は、現在ご利用いただけない場合があります。**

<div id="products-tab">
  ## Products タブ
</div>

[console.gnip.com](http://console.gnip.com) にログインすると、ダッシュボードの Products タブが最初に表示されます。 このページには、お使いのアカウントで現在利用可能な Enterprise プロダクトの概要が含まれています。

PowerTrack や Decahose のようにストリーミング配信を利用するプロダクトについては、このページにプロダクト名とストリームラベル、現在のアクティブ接続数、各ストリームで現在アクティブなルール数（該当する場合）、および直近 24 時間に配信されたアクティビティ（例: ポスト）の生データとしての件数が表示されます。

Search API のように REST 配信を利用するプロダクトについては、このページにプロダクト名、ラベル（「streams」としても表示されます）、これらのエンドポイント経由で返されたアクティビティ（例: ポスト）、およびエンドポイントごとの複数種類のリクエスト件数が表示されます。

[Usage API](/ja/x-api/enterprise-gnip-2.0/fundamentals/usage) は、これらのデータの多くをプログラムから取得できる形で提供している点に注意してください。

各ストリームごとの詳細は、名前または設定ボタンをクリックすることで確認できます。

<div id="overview">
  ### 概要
</div>

メインダッシュボードでいずれかのストリームをクリックすると、そのストリームの概要ページに移動します。

ストリーミング配信プロダクトでは、このページに次の内容が含まれます。

1. 各ストリーム接続ごとに配信されているアクティビティ数を示すボリュームチャート

2. ストリーム上で現在アクティブな接続の詳細（接続 ID と IP アドレス）

3. ストリームの最近の接続、切断、ルール更新イベントのログ

チャートのスケールは、右上隅のリンクで調整できます。個々の接続および切断の表示・非表示は、チャート直下の凡例内の該当項目をクリックして切り替えることができます。

<div id="connections">
  ### 接続
</div>

Connections ページでは、ストリームで最近発生した接続イベントの詳細を確認できます。これには、各接続の開始時刻と終了時刻（UTC の 24 時間表記）、各接続の継続時間、接続を行ったサーバーの IP アドレス、参照用の一意の接続 ID、そして現在の接続ステータスが含まれます。ステータスには、指定された接続について直近に発生したイベントが反映され、Client Connected であるか、あるいは切断イベント（disconnect）であるかが示されます。切断イベントの場合は、その切断の種類も指定されます。接続に関するデバッグの詳細については、[disconnections explained ガイド](/ja/x-api/enterprise-gnip-2.0/powertrack-api#integrating-with-powertrack)を参照してください。

<div id="rules">
  ### ルール
</div>

Rules タブは PowerTrack ストリームで利用でき、ユーザーインターフェースからプレーンテキストのルールを手動で入力することで、すばやく使い始めるための簡便な方法を提供します。なお、このインターフェースで手動追加できるルールは最大 1000 個までであり、初期テスト用途にのみ使用することを推奨します。本番環境では、[Rules API](/ja/x-api/enterprise-gnip-2.0/powertrack-api#api-reference-index) を使用してプログラムでルールを管理することをお勧めします。

<div id="settings">
  ### 設定
</div>

「Settings」タブでは、ストリーム内のデータの出力形式を切り替えることができます。複数のフォーマットオプションがサポートされています。形式を切り替えるには、各オプションを示すラジオボタンを使用してください。変更はストリームへの再接続時に有効になります。なお、この設定の更新は次回のリクエストまたは接続から即座に反映され、新しい形式によっては既存のパーサーが新形式に対応できず、動作しなくなる可能性があります。 

**注意:** 最も多くのデータを取得するための推奨設定は「Leave data in the original format」です。この設定では、[こちら](/ja/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#post-object)で説明されている拡張済みネイティブ形式でデータが返されます。Activity Streams 形式については[こちら](/ja/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#activity-object)で説明されています。

<div id="usage-tab">
  ## Usage タブ
</div>

Usage タブでは、さまざまな期間におけるストリームの利用状況を確認できます。利用状況情報にプログラムからアクセスするには、[Usage API](/ja/x-api/enterprise-gnip-2.0/fundamentals/usage) を参照してください。

<div id="monthly">
  ### 月次
</div>

Monthly Usage ページには、プロダクトごとに分割されたストリーム利用状況が表示されます。たとえば、特定のデータソースに対するカバレッジ系プロダクト（PowerTrack、Historical PowerTrack、PowerTrack Replay など）はグループ化され、個別のデータに加えて、合計の集計件数も提供されます。件数には、当月これまでの累計、推定月末値（今月これまでの利用状況と月末までの残り時間に基づく推定）、および直近 2 か月分の件数が含まれます。

重要な点として、これらの件数はプロダクトおよびストリームごとに重複排除されています。同じ PowerTrack ストリームへの複数の接続を通じて同一のポストを受信した場合でも、この集計ではそのポストは 1 回としてカウントされます。件数は毎日、協定世界時 (UTC) 00:00 に更新されます。

<div id="daily">
  ### 日次
</div>

Daily Usage ページでは、当月について、各日ごとにプロダクト別の重複排除済み件数を表示します。件数は24時間ごとに 00:00 UTC に更新されます。

<div id="rules">
  ### ルール
</div>

ルールの上限数は、契約している PowerTrack のルールパッケージに基づいており、特定の「タイプ」に属するすべてのストリームにわたって適用されます。ルールが新規に追加または削除されるたびに、ルール数が更新されます。

<div id="account-tab">
  ## 「アカウント」タブ
</div>

<div id="my-profile">
  ### プロフィール
</div>

自分のユーザープロフィールの詳細を確認し、ここでパスワードを変更できます。また、このアカウント専用の利用量しきい値通知を設定できます。API のステータスに関する通知は、https://api.twitterstat.us のステータスページで確認できます。

<div id="account-settings">
  ### アカウント設定
</div>

ユーザーの追加・削除・編集や、各ユーザーごとのメール通知および利用量しきい値アラートを設定できます。

<Frame>
  <img src="/images/account-settings.png.twimg.1920.png" />
</Frame>

ユーザーには 3 種類があります。Account Admin、User、Email Only です。

* Account Admin は、他のユーザーの作成・削除・編集が可能であり、ユーザー名（メールアドレス）とパスワードを用いたベーシック認証で enterprise APIs に接続できます。

* User は、他のユーザーを作成または変更することはできませんが、ユーザー名（メールアドレス）とパスワードを用いたベーシック認証で enterprise APIs に接続できます。

* Email Only ユーザーはダッシュボードにアクセスできず、enterprise APIs へ接続する権限もありません。Notifications セクションで通知の受信が設定されている場合にのみ、通知メールを受信します。

<div id="usage-thresholds">
  ### 使用量のしきい値
</div>

プロダクトごとの使用量のしきい値を設定します。これらは、各プロダクトの警告しきい値および重大しきい値について、プロフィールでその通知を設定しているユーザーへのメールアラートを送信するトリガーとなります。

<Frame>
  <img src="/images/usage-thresholds.png.twimg.1920.png" />
</Frame>

これらのしきい値は 1 日に 1 回、19:30（UTC）に評価されるのみで、リアルタイムには評価されない点に注意してください。
