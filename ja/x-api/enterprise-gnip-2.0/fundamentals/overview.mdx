## 概要

> Enterprise console（console.gnip.com）は、Enterprise API への契約済みアクセス権をお持ちの Enterprise データのお客様向けに提供されています。現在 Enterprise データのお客様でない場合でも、詳しい情報については [enterprise data](https://developer.x.com/en/products/x-api/enterprise) のページをご覧ください。

これは X の Enterprise console ダッシュボードの概要です。ここで説明する各セクションは、コンソール内のそれぞれの領域に対応しており、プロダクトの詳細、利用状況、一般的なアカウント管理について説明します。お使いのアカウントに関してご不明な点がある場合は、担当のアカウントマネージャーにお問い合わせいただくか、[Enterprise プロダクト向けの技術ドキュメント](/ja/x-api/enterprise-gnip-2.0/enterprise-gnip)をご確認ください。

アカウントマネージャーとの Enterprise のトライアルおよびオンボーディング プロセスの一環として、Enterprise 顧客には [console.gnip.com](http://console.gnip.com) へのログイン情報が付与されます。ここは Enterprise プロダクトへのアクセス状況を確認および設定できるユーザーインターフェースです。コンソールへの管理者による初回アクセスは、Enterprise オンボーディング プロセスの一部として X によって設定され、その後、管理者が追加ユーザーを作成できます。Enterprise console を利用することで、Enterprise プロダクトに関連する詳細を管理・参照できます。 

次の動画では、console.gnip.com ダッシュボード内のさまざまな部分について概要を説明しています。 

**免責事項: 動画内で紹介されている一部の機能（特定のストリーム エンリッチメントや非 X データソースのプロダクトを含む）は、現在はご利用いただけない場合があります。**

<div id="products-tab">
  ## 製品タブ
</div>

[console.gnip.com](http://console.gnip.com) のアカウントにログインすると、ダッシュボードの製品タブが最初に表示されます。このページには、お使いのアカウントで現在利用可能な Enterprise 製品の概要が含まれています。

PowerTrack や Decahose のようにストリーミング配信を利用する製品については、このページに製品名とストリームラベル、現在アクティブな接続数、各ストリームで現在有効なルール数（該当する場合）、および直近 24 時間に配信されたアクティビティ（例: ポスト）の総件数が表示されます。

Search API のように REST 配信を利用する製品については、このページに製品名、ラベル（「streams」としても表示されます）、これらのエンドポイントを通じて返された現在のアクティビティ（例: ポスト）、およびエンドポイントごとのいくつかの種類のリクエスト件数が表示されます。

[Usage API](/ja/x-api/enterprise-gnip-2.0/fundamentals/usage) からは、これらのデータの多くをプログラムで取得できます。

各ストリームの詳細情報は、名前または設定ボタンをクリックすると表示されます。

<div id="overview">
  ### 概要
</div>

メインダッシュボードでいずれかのストリームをクリックすると、そのストリームの概要ページに移動します。

ストリーミング配信プロダクトの場合、このページには次の内容が含まれます。

1. 各ストリーム接続ごとに配信されているアクティビティ数のボリュームチャート

2. ストリーム上で現在アクティブな接続の詳細（接続 ID と IP アドレス）

3. ストリームに対する最近の接続、切断、およびルール更新イベントのログ

チャートのスケールは、右上隅にあるリンクで調整できます。個々の接続および切断の表示／非表示は、チャート直下の凡例内の該当項目をクリックして切り替えることができます。

<div id="connections">
  ### 接続
</div>

Connections ページでは、ストリームで最近発生した接続イベントの詳細が表示されます。これには、各接続の開始時刻と終了時刻（UTC の 24 時間表記）、各接続の継続時間、接続元サーバーの IP アドレス、参照用の一意の接続 ID、そして現在の接続ステータスが含まれます。ステータスは、指定された接続に対して直近に発生したイベントを表します。つまり、Client Connected（クライアント接続済み）か、切断イベントであり、その場合は切断の種類が指定されます。接続のデバッグの詳細については、[disconnections explained ガイド](/ja/x-api/enterprise-gnip-2.0/powertrack-api#integrating-with-powertrack)を参照してください。

<div id="rules">
  ### ルール
</div>

Rules タブは PowerTrack ストリームで利用でき、ユーザーインターフェースからプレーンテキストのルールを手動入力することで、すばやく使い始めることができます。なお、このインターフェースで手動追加できるルール数は最大 1000 個までであり、初期テストの用途にのみ使用してください。本番環境では、[Rules API](/ja/x-api/enterprise-gnip-2.0/powertrack-api#api-reference-index) を使用してルールをプログラムで管理することを推奨します。

<div id="settings">
  ### 設定
</div>

Settings タブでは、ストリーム内のデータの出力形式を切り替えることができます。複数の形式オプションがサポートされています。形式を切り替えるには、各オプションを示すラジオボタンを選択します。変更はストリームへの再接続時に有効になります。なお、この設定を更新すると、次回のリクエストまたは接続から即時に適用され、新しい形式によってはパーサーが正しく動作しなくなる可能性があります。 

**注意:** 最も多くのデータを取得するための推奨設定は「Leave data in the original format」であり、[こちら](/ja/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#post-object)で説明されている拡張されたネイティブ形式のデータが返されます。Activity Streams 形式については[こちら](/ja/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#activity-object)で説明しています。

<div id="usage-tab">
  ## Usage タブ
</div>

Usage タブでは、さまざまな期間におけるストリームの利用状況を把握できます。利用状況をプログラムから取得するには、[Usage API](/ja/x-api/enterprise-gnip-2.0/fundamentals/usage) を参照してください。

<div id="monthly">
  ### 月次
</div>

Monthly Usage ページには、プロダクト別に分解されたストリーム利用状況が表示されます。たとえば、あるデータソースに対するカバレッジ系プロダクト（PowerTrack、Historical PowerTrack、PowerTrack Replay など）はまとめてグループ化され、個別のデータに加えて、合算したロールアップの件数も提供されます。件数には、当月の月初から現在までの実績、今月これまでの利用状況と残り期間に基づく月末時点の推定値、さらに過去 2 か月分の件数が含まれます。

重要な点として、これらの件数はプロダクトおよびストリームごとに重複排除されています。同じ PowerTrack ストリームへの複数の接続を通じて同じポストを受信した場合でも、ここでの計算上はそのポストは 1 回としてカウントされます。件数は協定世界時 (UTC) の 00:00 に、24 時間ごとに更新されます。

<div id="daily">
  ### 日次
</div>

Daily Usage ページでは、当月の各日ごとの重複排除済み件数を、プロダクト別に表示します。件数は毎日 00:00 UTC に更新されます。

<div id="rules">
  ### ルール
</div>

ルールの上限数は、ご契約の PowerTrack ルールパッケージに基づいており、同じ「タイプ」に属するすべてのストリームに適用されます。新しいルールが追加または削除されるたびに、ルール数が更新されます。

<div id="account-tab">
  ## 「Account」タブ
</div>

<div id="my-profile">
  ### マイプロフィール
</div>

ここでは、個人のユーザープロフィールの詳細を表示したり、パスワードを変更したりできます。さらに、このアカウント専用の利用しきい値通知を設定できます。API ステータス通知は、https://api.twitterstat.us のステータスページで確認できます。

<div id="account-settings">
  ### アカウント設定
</div>

ユーザーの追加、削除、編集を行い、各ユーザーごとのメール通知や利用量のしきい値アラートを設定できます。

<Frame>
  <img src="/images/account-settings.png.twimg.1920.png" />
</Frame>

ユーザーには、Account Admin、User、Email Only の 3 種類があることに注意してください。  

* Account Admin は他のユーザーの作成・削除・編集が可能であり、Enterprise API に接続するために、ユーザー名（メールアドレス）とパスワードによるベーシック認証を使用できます

* User は他のユーザーの作成や変更はできませんが、Enterprise API に接続するために、ユーザー名（メールアドレス）とパスワードによるベーシック認証を使用できます

* Email Only ユーザーはダッシュボードにアクセスできず、Enterprise API に接続する権限はなく、[Notifications] セクションで受信するように設定されている場合にのみ通知を受け取ります。

<div id="usage-thresholds">
  ### 使用量しきい値
</div>

ご利用中のプロダクトごとに、使用量のしきい値を設定します。ここで設定したしきい値は、各プロダクトの警告しきい値と重大しきい値について、プロフィールで該当の通知を有効にしているユーザーへのメール通知を開始するトリガーとして機能します。

<Frame>
  <img src="/images/usage-thresholds.png.twimg.1920.png" />
</Frame>

これらのしきい値は 1 日 1 回、19:30（UTC）に評価されます。リアルタイムで評価されるわけではありません。
