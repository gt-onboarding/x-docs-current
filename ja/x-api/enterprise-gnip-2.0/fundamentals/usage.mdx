<div id="usage-api-twitter-api">
  ## 使用状況 API | Twitter API
</div>

<div id="overview">
  ### 概要
</div>

`Enterprise`

アカウント配下の各種 API の利用状況を横断的に監視・管理するための、最も重要かつ *最良の* ツールです。

**重要な免責事項:**

Usage API から返される利用件数は、トライアルやその他の請求調整の影響により、請求書に記載される件数と一致しない場合があります。すべての数値は、特定の日（UTC）に消費されたアクティビティから重複を排除して算出されています。

<div id="features">
  #### 機能
</div>

* console.gnip.com の UI で確認できる利用状況データをプログラムで取得
* ストリームレベルの利用状況データ - プロダクトレベルに加えて、ストリームレベル（例: dev と prod）での利用状況データを提供
* より細分化され、分かりやすいデータ - 検索リクエストは Full-Archive と 30-Day Search の各プロダクトごとに分類
* PowerTrack の過去の「days」と「jobs」データ 

<div id="supported-apis">
  ### 対応している API
</div>

以下は、Usage API により現在サポートされている API の一覧です。

* PowerTrack API `enterprise`
* 30-Day Search API `enterprise`
* Full-Archive Search API `enterprise`
* Historical PowerTrack `enterprise`

<div id="limitations">
  ### 制限事項
</div>

* Usage API では、2018 年 5 月 1 日以降の使用状況データにアクセスできます。2019 年 7 月 1 日以降は、**直近 13 か月分のカレンダー月単位の使用状況データ** にアクセスできます。

* `fromDate` と `toDate` で定義される **3 か月ごとの期間** の使用状況データにアクセスできます。

  ### get-usage
</div>

<div id="methods">
  #### **メソッド**
</div>

| メソッド | 説明 |
| :--- | :--- |
| [GET /metrics/usage/accounts/{ACCOUNT_NAME}.json](#GETData) | 利用状況データを取得 |

ここで:

* :account&#95;name は、console.gnip.com に表示される、ご利用のアカウントに関連付けられた（大文字と小文字を区別する）名前です。

<div id="authentication-and-rate-limit">
  #### **認証とレート制限**
</div>

<div id="authentication">
  ##### **Authentication**
</div>

Usage API へのすべてのリクエストには HTTP Basic 認証が必要です。console.gnip.com へのログインや任意の Gnip ストリームへの接続に使用できる、アカウントで有効になっている任意のメールアドレスとパスワードの組み合わせを認証情報として使用してください。

<div id="rate-limit">
  ##### **レート制限**
</div>

Usage API では、1 分あたり 2 件のリクエストというレート制限が適用されます。

<div id="best-practices-limitations">
  #### **ベストプラクティスと制限事項**
</div>

<div id="data-availability">
  ##### **データの可用性**
</div>

利用状況データは、データが処理された最後の完全な時間帯（UTC）までに消費された、重複排除済みアクティビティに基づきます。Gnip がシステムをデプロイしている場合を除き、データは通常、分単位で処理および更新されます。

* Usage API を使用すると、2018 年 5 月 1 日以降の利用状況データにアクセスできます。2019 年 7 月 1 日以降は、Usage API により、**直近 13 か月間のカレンダー月ベースの利用状況データ**にアクセスできます。 

* `fromDate` と `toDate` で定義される、**3 か月間の期間**ごとに利用状況データにアクセスできます

<div id="requesting-and-receiving-data">
  #### **データのリクエストと受信**
</div>

Usage API は、アカウントの API エンドポイントに対して HTTP BASIC-AUTH 認証情報付きの HTTP GET リクエストを送信することで動作します。

<div id="get-request">
  ##### GET Request:
</div>

ユーザー認証情報とアカウント名を使用して、次のエンドポイントに GET リクエストを送信します。

https://gnip-api.x.com/metrics/usage/accounts/:account&#95;name.json

**追加パラメーター**

|     |     |
| :--- | :--- |
| bucket | 任意。使用状況データが提供される時間の単位です。使用状況データは日単位または月単位の粒度で返すことができます。  <br />  <br />bucket を指定せずにリクエストした場合は、月単位の粒度で返されます。  <br />  <br />指定可能な値: &#39;month&#39; または &#39;day&#39; |
| fromDate (YYYYMMDDHHMM) | 任意。使用状況データは 2018 年 5 月 1 日以降のもののみ利用可能です。使用状況データが提供される最も古い UTC タイムスタンプです。タイムスタンプは日単位の粒度で、指定した日を含みます（例: 201805010000 は 05/01 日を含みます）。時と分に &#39;0000&#39; 以外の値が含まれるリクエストは、自動的に &#39;0000&#39; に変更されます。  <br />  <br />fromDate または toDate を指定しないリクエストでは、当月分の使用状況データが月単位で返され、過去 2 か月分の履歴データも含まれます。  <br />  <br />**注意:** 2019 年 6 月 1 日以降、過去 13 か月分の暦月ベースの使用状況データにアクセスできます。たとえば、10 月 10 日の場合、前年の 9 月 1 日までさかのぼって使用状況データにアクセスできます。  <br />**例:** 201810010000 を指定すると、2018 年 10 月 1 日以降（10 月 1 日を含む）のデータが返されます。 |
| toDate (YYYYMMDDHHMM) | 任意。使用状況データが提供される最新の UTC タイムスタンプです。タイムスタンプは日単位の粒度で、指定した日を含みません（例: 201703020000 は 03/02 日のデータを含みません）。toDate に当日または未来の日付を指定した場合、使用状況データは最後に完了した日（UTC）までが返されます。時と分に &#39;0000&#39; 以外の値が含まれるリクエストは、自動的に &#39;0000&#39; に変更されます。  <br />  <br />toDate を指定しないリクエストでは、次の bucket（bucket=day の場合は翌日、bucket=month の場合は翌月）がデフォルトになります。fromDate と toDate をいずれも指定しないリクエストでは、bucket=month がデフォルトとなり、当月分と直近 2 か月分のデータが表示されます。  <br />  <br />**例:** 201703050000 を指定すると、2017 年 3 月 5 日までのデータが返されますが、3 月 5 日のデータは含まれません。 |

**GET リクエスト例**

このリクエストは、2017 年 3 月 1 日から 2017 年 3 月 5 日までの月単位の粒度のデータを返しますが、2017 年 3 月 5 日のデータは含みません。

curl -u &quot;https://gnip-api.x.com/metrics/usage/accounts/:account&#95;name.json?bucket=month&amp;fromdate=201403010000&amp;toDate=201403150000&quot;
