X は、X API を利用して構築する開発者コミュニティを支援することに尽力しています。この取り組みの一環として、私たちは API を開発者に対してオープンかつ公平であり、X を利用する人々にとって安全であり、X プラットフォーム全体にとって有益なものにすることを目指しています。X のコンテンツをオフラインで保存するすべての開発者は、そのデータがユーザーの意図と X 上のコンテンツの最新の状態を反映するようにすることが極めて重要です。たとえば、X 上の誰かがポストやアカウントを削除したり、ポストを保護したり、ポストから位置情報を削除した場合、その人の期待と意図を尊重することは、X と開発者の双方にとって非常に重要です。バッチコンプライアンスエンドポイントは、X データを [X Developer Agreement and Policy](https://developer.x.com/en/developer-terms/policy) に準拠した状態に保つための簡便なツールを開発者に提供します。 

これらのバッチコンプライアンスエンドポイントを使用すると、大規模なポストまたはユーザー ID のデータセットをアップロードしてコンプライアンスステータスを取得でき、それにより、データセットを準拠させるためにどのデータに対してアクションが必要かを判断できます。バッチコンプライアンスエンドポイントの使用は、前述のユースケースに限定されており、それ以外の目的での使用は禁止されており、対処措置の対象となる場合がありますのでご注意ください。

通常、このエンドポイントを利用するには 4 つのステップがあります。

1. **コンプライアンスジョブを作成する**
   ジョブタイプを指定できます (値 tweets または users を使用して、アップロードしたいデータセットにポスト ID が含まれるのか、ユーザー ID が含まれるのかを示します)。ジョブタイプごとに、同時に実行できるジョブは 1 つのみです。
2. **データセットを upload&#95;url にアップロードする**
   次に、提供された upload&#95;url にプレーンテキストファイルとしてデータセットをアップロードします。ファイルの各行には 1 つのポスト ID またはユーザー ID を含めます。upload&#95;url は 15 分後に期限切れになります。
3. **（任意）ジョブステータスを確認する**
   コンプライアンスジョブのステータスを確認して、作成済み、in&#95;progress、failed、complete のいずれかであるかを確認できます。
4. **結果をダウンロードする**
   ジョブが完了すると、download&#95;url を使用して結果をダウンロードできます。download&#95;url は（ジョブが作成された時点から）1 週間で期限切れになります。

   この結果には、一連の JSON オブジェクト (1 行につき 1 オブジェクト) が含まれます。各オブジェクトには、ポスト ID、ポストの作成日 (日付ごとに整理されたポストを見つけるのに有用)、必要なアクション、コンプライアンスアクションの理由、およびユーザーが凍結された日付が含まれます。

結果には、次のコンプライアンスイベントタイプが含まれます。

* deleted - ポストまたはユーザーアカウントが削除されたことを示します
* deactivated - ポストまたはユーザーアカウントが無効化されたことを示します
* scrub&#95;geo - ポストまたはユーザーに関連付けられた位置情報が削除されたことを示します
* protected - ポストを作成したアカウントが非公開になったことを示します
* suspended - ポストを作成したアカウントが凍結されたことを示します

**アカウントのセットアップ**

これらのエンドポイントにアクセスするには、以下が必要です。

* 承認済みの [開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)
* [プロジェクト](/ja/resources/fundamentals/projects) 内にある [開発者アプリ](/ja/resources/fundamentals/developer-apps) のキーとトークンを使用して認証すること 

X API v2 エンドポイントへのアクセスの取得方法については、[「はじめに」ガイド](/ja/x-api/getting-started/getting-access) を参照してください。
