X は、X API を利用して開発を行う開発者コミュニティを支援することに尽力しています。この取り組みの一環として、API を開発者に対してオープンかつ公平に保ち、X を利用する人々にとって安全であり、プラットフォーム全体としても有益なものにすることを目指しています。X のコンテンツをオフラインで保存するすべての開発者は、そのデータがユーザーの意図と X 上のコンテンツの最新の状態を反映していることを確実にすることが重要です。たとえば、誰かが X 上でポストやアカウントを削除したり、ポストを保護したり、ポストから位置情報を削除した場合、その人の期待と意図を尊重することは、X と開発者の双方にとって極めて重要です。バッチコンプライアンスエンドポイントは、X データを [X Developer Agreement and Policy](https://developer.x.com/en/developer-terms/policy) に準拠した状態に維持するうえで役立つ、簡便なツールを開発者に提供します。 

これらのバッチコンプライアンスエンドポイントを使用すると、ポストまたはユーザーの ID からなる大規模なデータセットをアップロードし、それらのコンプライアンスステータスを取得して、データセットをポリシー準拠の状態にするためにどのデータに対して対応が必要かを判断できます。バッチコンプライアンスエンドポイントの利用は、前述のユースケースに限定されており、それ以外の目的での使用は禁じられており、違反した場合は措置の対象となる可能性があります。

通常、このエンドポイントを利用する際には、次の 4 つのステップが含まれます。

1. **コンプライアンスジョブの作成**
   ジョブタイプを指定できます (値として tweets または users を使用し、アップロードするデータセットにポスト ID が含まれるか、ユーザー ID が含まれるかを示します)。ジョブタイプごとに、同時に実行できるジョブは 1 件のみです。
2. **upload&#95;url へのデータセットのアップロード**
   次に、提供された upload&#95;url に対して、データセットをプレーンテキストファイルとしてアップロードします。ファイルの各行には、1 つのポスト ID またはユーザー ID を含めます。upload&#95;url は 15 分後に有効期限切れになります。
3. **(任意) ジョブステータスの確認**
   コンプライアンスジョブのステータスを確認して、created、in&#95;progress、failed、complete のいずれであるかを確認できます。
4. **結果のダウンロード**
   ジョブが完了したら、download&#95;url を使用して結果をダウンロードできます。download&#95;url は (ジョブが作成されてから) 1 週間後に有効期限切れになります。

   この結果には、一連の JSON オブジェクト (1 行につき 1 オブジェクト) が含まれます。各オブジェクトには、ポスト ID、ポストの作成日 (日付順に整理されたポストを特定するのに有用)、必要なアクション、コンプライアンスアクションの理由、およびユーザーが凍結された日付が含まれます。

結果では、次のコンプライアンスイベントタイプを受け取ります。

* deleted - ポストまたはユーザーアカウントが削除されたことを示します
* deactivated - ポストまたはユーザーアカウントが無効化されたことを示します
* scrub&#95;geo - ポストまたはユーザーに関連付けられた位置情報が削除されたことを示します
* protected - ポストを作成したアカウントが非公開 (保護) アカウントになったことを示します
* suspended - ポストを作成したアカウントが凍結されたことを示します

**アカウントのセットアップ**

これらのエンドポイントにアクセスするには、次が必要です。

* 承認済みの [開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)。
* [プロジェクト](/ja/resources/fundamentals/projects) 内にある [開発者アプリ](/ja/resources/fundamentals/developer-apps) のキーとトークンを使用した認証。 

X API v2 エンドポイントへのアクセス方法の詳細については、[はじめにガイド](/ja/x-api/getting-started/getting-access) を参照してください。
