X は、X API を利用して開発を行う開発者コミュニティを支援することに尽力しています。この取り組みの一環として、私たちは API を開発者に対してオープンかつ公平なものとし、X 上の人々にとって安全であり、X プラットフォーム全体にとって有益なものにすることを目指しています。X コンテンツをオフラインで保存する開発者は、保存しているデータがユーザーの意図と X 上のコンテンツの現在の状態を反映していることを確実にすることが極めて重要です。たとえば、誰かが X 上でポストまたはアカウントを削除した場合、ポストを保護した場合、またはポストを編集した場合には、X と開発者の双方が、その人の期待と意図を尊重することが重要です。

コンプライアンスイベントのリアルタイムストリームは、開発者が X データを [X Developer Agreement and Policy](https://developer.x.com/en/developer-terms/policy) に準拠した状態に保つためのツールを提供します。 

c&#95;ompliance event&#95; ストリームは 2 種類あり、1 つは Post&#95; compliance&#95; イベント用、もう 1 つは *User compliance* イベント用です。これらのストリームは Enterprise アクセスで利用可能であり、大量のデータを取り込むパートナーが、ポスト編集イベントのようなコンプライアンスイベントを「リッスン」できるように設計されています。

これらのストリームでは、次のイベントが提供されます。 

**Post compliance ストリーム: **

* delete - ポストが削除されたことを示します。

* tweet&#95;edit - ポストが編集され、更新されたポストの ID が提供されることを示します。 

* withheld - ポストが 1 つ以上の国で保留（表示制限）されていることを示します。 

* drop - ポストを一般公開から削除すべきであることを示します。

* undrop - ポストを再び表示でき、公開として扱ってよいことを示します。

**User compliance ストリーム: **

* **user&#95;delete** - ユーザーアカウントが削除されたことを示します

* **user&#95;undelete** - ユーザーアカウントの削除が取り消されたことを示します

* **user&#95;protect** - ユーザーアカウントが非公開（保護）になったことを示します

* **user&#95;unprotect** - ユーザーアカウントが公開になったことを示します

* **user&#95;withheld** - ユーザーアカウントが 1 つ以上の国で保留（表示制限）されていることを示します。 

* **user&#95;suspend** - ユーザーアカウントが凍結されたことを示します

* **user&#95;unsuspend** - ユーザーアカウントの凍結が解除されたことを示します

* **user&#95;profile&#95;modification** - ユーザープロフィールが更新されたことを示します。これには、説明文、名前、場所、URL の更新が含まれます。

**アカウントのセットアップ**

これらのエンドポイントにアクセスするには、次が必要です:

* 承認済みの [開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)。
* [プロジェクト](/ja/resources/fundamentals/projects) 内にある [開発者アプリ](/ja/resources/fundamentals/developer-apps) のキーとトークンを使用して認証すること。 

X API v2 エンドポイントへのアクセス取得について詳しくは、[利用開始ガイド](/ja/x-api/getting-started/getting-access) を参照してください。
