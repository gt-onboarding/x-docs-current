X は、X API を利用して開発する開発者コミュニティを大切にしています。この取り組みの一環として、私たちは API を開発者に対してオープンかつ公平に保ち、X を利用する人々にとって安全であり、X プラットフォーム全体にとって有益なものにすることを目指しています。X のコンテンツをオフラインで保存するすべての開発者は、そのデータがユーザーの意図と X 上のコンテンツの最新の状態を反映していることを確実にすることが極めて重要です。たとえば、X 上の誰かがポストまたはアカウントを削除した場合、ポストを保護した場合、あるいはポストを編集した場合、X と開発者の双方が、その人の期待と意図を尊重することが不可欠です。

コンプライアンスイベントのリアルタイムストリームは、開発者が [X Developer Agreement and Policy](https://developer.x.com/en/developer-terms/policy) に準拠した形で X データを維持するためのツールを提供します。 

2 種類の c&#95;ompliance event&#95; ストリームがあり、1 つは Post&#95; compliance&#95; イベント用、もう 1 つは *User compliance* イベント用です。これらのストリームは Enterprise アクセスで利用可能で、大量のデータを取り込むパートナーが、ポスト編集イベントなどのコンプライアンスイベントを「リッスン」できるように設計されています。

これらのストリームでは、次のイベントが提供されます。 

**Post compliance stream: **

* delete - ポストが削除されたことを示します。

* tweet&#95;edit - ポストが編集されたこと、および更新後のポストの ID を示します。 

* withheld - ポストが 1 つ以上の国で配信制限されたことを示します。 

* drop - ポストを一般公開から削除する必要があることを示します。

* undrop - ポストを再度表示し、公開コンテンツとして扱うことができることを示します。

**User compliance stream: **

* **user&#95;delete** - ユーザーアカウントが削除されたことを示します。

* **user&#95;undelete** - ユーザーアカウントの削除が解除されたことを示します。

* **user&#95;protect** - ユーザーアカウントが非公開になったことを示します。

* **user&#95;unprotect** - ユーザーアカウントが公開になったことを示します。

* **user&#95;withheld** - ユーザーアカウントが 1 つ以上の国で配信制限されたことを示します。 

* **user&#95;suspend** - ユーザーアカウントが凍結されたことを示します。

* **user&#95;unsuspend** - ユーザーアカウントの凍結が解除されたことを示します。

* **user&#95;profile&#95;modification** - ユーザーのプロフィールが更新されたことを示します。これには、説明文、名前、場所、URL の更新が含まれます。

**アカウントのセットアップ**

これらのエンドポイントにアクセスするには、次のものが必要です。

* 承認済みの [開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)。
* [プロジェクト](/ja/resources/fundamentals/projects) 内にある [開発者アプリ](/ja/resources/fundamentals/developer-apps) のキーとトークンを使用して認証すること。 

X API v2 エンドポイントへのアクセス取得については、[利用開始ガイド](/ja/x-api/getting-started/getting-access)をご覧ください。
