<div id="introduction">
  ## はじめに
</div>

ソフトウェア開発キット（SDK）は、特定のプラットフォームや API 向けに用意されたソフトウェアツールやプログラムのセットです。SDK の目的は、開発者が自分のアプリケーション内で利用できるライブラリやコードベースを提供することで、アプリケーションの機能を構築または拡張し、効率的かつ最小限のコーディングで実現できるようにすることです。これにより開発プロセスが大幅に高速化され、時間・コスト・労力を節約できます。

> X Developer Platform では現在、TypeScript/JavaScript と Java で開発する開発者向けに 2 つの公式 SDK を提供しています。これらの SDK により、X API v2 周りの複雑な処理を手作業で実装する必要がなくなり、利用可能なすべての v2 エンドポイント向けにあらかじめ用意された関数を活用できるほか、認証プロセスも簡素化されるため、開発者はより効率的に開発できます。これらは Developer Platform チームによって開発・保守されているため、今後の X API v2 のリリースにも常に追従した最新状態が保たれます。

これらの SDK は X API をラップしているため、[プロジェクト](/ja/resources/fundamentals/projects) 内にある [開発者アプリ](/ja/resources/fundamentals/developer-apps) の認証情報を用いてリクエストを認証するには、[開発者アカウント](https://developer.x.com/en/portal/dashboard) が必要です。

  #### インストール
</div>

<Tabs>
  <Tab title="Java">
    Java パッケージのインストール方法はいくつかあります（Java 1.8 以上が必要です）。

    * Maven ユーザーの場合 - 次の依存関係をプロジェクトの POM ファイルに追加します:

    ```xml
    <dependency>

      <artifactId>twitter-api-java-sdk</artifactId>
      <version>1.1.4</version>
    </dependency>
    ```

    * Gradle ユーザーの場合 - 次の依存関係をプロジェクトのビルドファイルに追加します:
      `implementation "com.twitter:twitter-api-java-sdk:1.1.4"`

    * 上記以外の場合 - まず次のコマンドを実行して JAR を生成します:
      `mvn clean package`
      その後、次の JAR を手動でインストールします:

    `target/twitter-api-java-sdk-1.1.4.jar`

    `target/lib/*.jar`
  </Tab>

  <Tab title="TypeScript">
    このパッケージは Node.js 14 以上をサポートしています。インストールするには、Node プロジェクトのディレクトリで次のコマンドを実行してください:

<div id="client-basics">
  #### Client の基本
</div>

認証およびライブラリ Client にアクセスするために、作業中のファイルの先頭でクラス（Java）およびパッケージ（TypeScript）をインポートします。ライブラリ Client のメソッドを利用するには、認証情報を渡す必要があります。これは、App-only 認証の場合は Bearer Token、OAuth 2.0 のユーザーコンテキストで認証する場合は client id/client secret のいずれかになります。

以下は、その例です。

<Tabs>
  <Tab title="Java">
    ```java
    // クラスをインポート:
    import com.twitter.clientlib.ApiClient;
    import com.twitter.clientlib.ApiException;
    import com.twitter.clientlib.Configuration;
    import com.twitter.clientlib.auth.*;
    import com.twitter.clientlib.model.*;
    import com.twitter.clientlib.TwitterCredentialsBearer;
    import com.twitter.clientlib.api.TwitterApi;

    // ライブラリ Client をインスタンス化
    TwitterApi apiInstance = new TwitterApi();

    // 認証情報をインスタンス化（App-only の例）
    TwitterCredentialsBearer credentials = new TwitterCredentialsBearer(System.getenv("APP-ONLY-ACCESS-TOKEN"));

    // 認証情報をライブラリ Client に渡す
    apiInstance.setTwitterCredentials(credentials);
    ```
  </Tab>

  <Tab title="TypeScript">
    ```typescript
    // パッケージをインポート
    import { Client, auth } from "twitter-api-sdk";

    // まず認証用 Client を初期化
    const authClient = new auth.OAuth2User({
     client_id: process.env.CLIENT_ID as string,
     client_secret: process.env.CLIENT_SECRET as string,
     callback: "YOUR-CALLBACK",
     scopes: ["tweet.read", "users.read", "offline.access"],
    });

    // 認証情報をライブラリ Client に渡す
    const twitterClient = new Client(authClient);
    ```
  </Tab>
</Tabs>

<div id="authentication-flow">
  #### 認証フロー
</div>

SDK をアプリケーション専用認証オプションで利用する場合は、トークンを指定するだけで、そのライブラリのクライアントはすぐにエンドポイントメソッドを呼び出せる状態になります。なお、アプリケーション専用トークンは、ユーザーコンテキスト認証が必要なエンドポイントでは使用できません。

OAuth 2.0 のユーザーコンテキスト認証では、認証クライアントを作成した後に、さらにいくつかの手順が必要になります。

* 認可 URL を生成する
* 認可 URL 経由でアプリケーションを認可する
* コールバック URL にリダイレクトされる（これは Developer Portal の認証設定ページで設定したコールバック URL と一致している必要があります）
* コードベリファイアを使用してアクセストークンと交換する

SDK では、これらの手順を簡略化するためのメソッドが認証クライアントに用意されています。OAuth 2.0 のユーザーコンテキストで認証してリクエストを行う一連の例については、GitHub リポジトリを参照してください。

* [TypeScript](https://github.com/xdevplatform/twitter-api-typescript-sdk/blob/main/examples/oauth2-callback_pkce_s256.ts)

* [Java](https://github.com/xdevplatform/twitter-api-java-sdk/blob/main/examples/src/main/java/com/twitter/clientlib/auth/OAuth20GetAccessToken.java)

<div id="endpoint-methods">
  #### エンドポイントメソッド
</div>

ライブラリのクライアントで提供されるメソッドは、各エンドポイントに対応するよう明確に命名されており、すべてのパラメーターは引数として渡されます。
以下に、ID でポストを取得する例を示します:

<Tabs>
  <Tab title="Java">
    ```java
    String id = "1511757922354663425"; // String | 単一のポスト ID。
    Set<String> expansions = new HashSet<>(Arrays.asList("author_id")); // Set<String> | 展開するフィールドをカンマ区切りで指定したリスト。
    Set<String> tweetFields = new HashSet<>(Arrays.asList("created_at", "lang", "context_annotations")); // Set<String> | 表示するポストのフィールドをカンマ区切りで指定したリスト。
    Set<String> userFields = new HashSet<>(Arrays.asList("created_at", "description", "name")); // Set<String> | 表示するユーザーのフィールドをカンマ区切りで指定したリスト。

    try {
     SingleTweetLookupResponse result = apiInstance.tweets().findTweetById(id, expansions, tweetFields, userFields, null, null, null);
     System.out.println(result);
    } catch (ApiException e) {
     System.err.println("Exception when calling TweetsApi#findTweetById");
     System.err.println("Status code: " + e.getCode());
     System.err.println("Reason: " + e.getResponseBody());
     System.err.println("Response headers: " + e.getResponseHeaders());
     e.printStackTrace();
    }

    ```
  </Tab>

  <Tab title="TypeScript">
    ```typescript
    const lookupTweetById = await client.tweets.findTweetById(
      // ポスト ID
      "1511757922354663425",
      {
        // 任意のパラメーター
        expansions: ["author_id"],
        "user.fields": ["created_at", "description", "name"],
      }
    );
    ```
  </Tab>
</Tabs>
