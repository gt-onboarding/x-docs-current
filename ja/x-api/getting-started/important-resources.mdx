<div id="learn-about-whats-possible">
  ## 可能なことを学ぶ
</div>

トップナビゲーションからアクセスできるユースケース、ソリューション、プロダクトページは、このプラットフォームで何が実現できるかを大まかに把握するのに最適です。

また、さまざまなユースケースに対応するために、1 つまたは複数のプラットフォームツールをどのように活用できるかを理解する助けとなる[チュートリアル](/ja/tutorials)も複数用意しています。

ドキュメントの最も低いレイヤーにはエンドポイントおよびツールのドキュメントがあり、いずれもそのツールの機能と代表的なユースケースの概要を簡潔に説明するイントロダクションコンテンツを含んでいます。たとえば、こちらが [Search Posts](/ja/x-api/posts/search/introduction) 向けの X API v2 エンドポイントセクションです。

<div id="integrate-an-endpoint-into-your-system">
  ### エンドポイントをシステムに統合する
</div>

エンドポイントとの連携をスムーズに行えるよう、さまざまなツール、ライブラリ、および統合ガイドを用意しています。最も簡単な統合方法は、[ツールとライブラリ](/ja/x-api/tools-and-libraries/overview) を利用することです。これらは、[OAuth 1.0a User Context authentication](/ja/resources/fundamentals/authentication#oauth-1-0a-2)、ページネーション、[レート制限の処理](/ja/resources/fundamentals/rate-limits) などの複雑な機能を自動的に処理します。

ゼロからソリューションを構築したい場合は、エンドポイント統合ガイドで、エンドポイントをシステムに統合する際の機能やベストプラクティスを解説しています。これらのガイドは、エンドポイントやツールごとのドキュメント内に「integrate」または単に「guides」として掲載されています。例えば、[Filter stream 用のルールを作成する方法](/ja/x-api/posts/filtered-stream/integrate/build-a-rule) に関する統合ガイドがあります。

<div id="troubleshoot-an-issue">
  ### 問題のトラブルシューティング
</div>

エラーが発生した場合や質問がある場合は、[サポートハブ](https://developer.x.com/en/support/twitter-api)をご覧ください。トラブルシューティングのヒント、よくある質問への回答、その他の役立つリソースを見つけることができます。

<div id="keep-up-to-date-with-the-latest-releases">
  ### 最新リリース情報を常に把握する
</div>

私たちは、API を通じて新しい X の機能を利用できるようにするため、プラットフォームのアップデートを頻繁に行っています。加えて、年に少なくとも 1 回はメジャーバージョンアップを行うことを目標としており、その際には後方互換性のない変更が含まれる場合があります。

最新の更新情報を受け取るには、[フォーラムの Announcements カテゴリ](https://devcommunity.x.com/c/announcements/22)を購読し、X 上で [@API](https://x.com/api) と [@XDevelopers](https://x.com/XDevelopers) をフォローして通知をオンにし、[stay informed](https://developer.x.com/en/stay-informed) ページから各種リソースをご確認ください。

これに加えて、最新バージョンの API へ移行する際の指針となるよう、新たな [migration hub](/ja/x-api/migrate/overview) も用意しています。
