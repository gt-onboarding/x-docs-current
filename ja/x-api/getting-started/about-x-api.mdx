X API は、X のデータをプログラムから取得・分析したり、X 上の会話のためのプロダクトを構築したりするために利用できます。

長年にわたり、X API は開発者向けにさまざまなアクセスレベルを追加し、パブリックな会話を強化および調査できるようにすることで進化してきました。

最近、X API v2 をリリースしました。X API v2 には、モダンな基盤、新しく高度な機能、そして [Basic access](https://developer.x.com/en/portal/products/basic) へのスムーズな導入が含まれています。

次の 3 つのタブでは、X API の各バージョンとアクセスレベル、v2 で新しくなった点、および API を使用して取得、作成、削除、調整できる X リソースについて説明します。

  <Tab title="v2 の最新情報">
    ## v2 の新機能

    X API v2 は、2012 年以降で最大の X API のアップグレードです。これにより、多数の新しく高度な機能に加え、高速かつ無料の API アクセスが提供されます。

    v2 で利用可能な機能の一部は次のとおりです。

    ### 新しいエンドポイント

    X API v2 向けに、まったく新しいエンドポイント群をリリースしました。新規のものを含む v2 エンドポイントの全リストは、X API エンドポイントマップガイドで確認できます。

    [「X API エンドポイントマップ」を見る &gt;](/ja/x-api/migrate/x-api-endpoint-map)

    ### 新しく、より詳細なデータオブジェクト

    データオブジェクトにさまざまな改善を行い、データの参照や解析をこれまでより容易に行えるように最新化しました。

    [データディクショナリを確認 &gt;](/ja/x-api/fundamentals/data-dictionary)

    ### 取得したいオブジェクトとフィールドだけを取得するための新しいパラメータ

    データエンドポイントに `fields` と `expansions` パラメータを追加しました。これにより、デフォルトで返されるフィールドを超えて、関連オブジェクトやフィールドをリクエストできます。

    [fields と expansions の使い方を学ぶ &gt;](/ja/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)

    ### 高度なメトリクス

    インプレッション数、動画再生数、ユーザープロフィール、URL クリック数などの公開メトリクスと非公開メトリクスの両方をリクエストすることで、レスポンスペイロード内で直接、ポスト、ユーザー、メディア、投票のパフォーマンスをより簡単に把握できます。これらの一部は、オーガニックとプロモーションのコンテキストに分かれています。

    [メトリクスの詳細を見る &gt;](/ja/x-api/fundamentals/metrics)

    ### 異なるトピックを含むポストをフィルタリングおよび識別

    search Posts や filtered stream を使用する際に、エンティティオペレーターおよびコンテキストオペレーターを使ってトピックでフィルタリングできるようになりました。分析を支援するため、これらのトピックはポストのペイロード内にも提供しています。

    [Post annotations の詳細を見る &gt;](/ja/x-api/fundamentals/post-annotations)

    ### 返信スレッドに属するポストをフィルタリングおよび識別

    search Posts、filtered stream、Post lookup を使用する際に、あるポストが会話スレッドの一部であるかどうかを、より簡単に特定できるようになりました。また、ポストに対して Post フィールド `reply_settings` を使い、会話の返信設定が指定されているかどうかを判定できる機能も追加しました。

    [会話トラッキングの詳細を見る &gt;](/ja/x-api/fundamentals/conversation-id)

    ### さらに多くの機能...

    * 高精度なスパムフィルタリング
    * 短縮 URL を完全に展開し、URL 分析を容易に
    * 廃止予定フィールドの削除とラベルの最新化による、シンプルな JSON レスポンスオブジェクト
    * ストリーミングエンドポイント向けのリカバリおよび冗長性機能
    * 検索クエリで一致する公開かつ利用可能なポストを 100% 返却
    * 接続を切断せずに変更できるストリーミングの「ルール」
    * filtered stream と検索のための、より表現力の高いクエリ言語
    * 新しいライブラリの構築と変更の、より透明な追跡を可能にする OpenAPI スペック
    * X を利用する開発者、研究者、企業、一般ユーザーのニーズに応えるためにプラットフォームが進化するのに合わせて、新機能およびエンドポイントへの API でのサポートをより迅速に提供
  </Tab>

  <Tab title="X API リソース">
    ## X API プラットフォームのリソース

    API 設計において、リソースとは、関連するデータや他のリソースとの関係、およびそのリソースに対して動作する一連のメソッドを持つエンティティを指します。たとえば、ポストはリソースの 1 つであり、さまざまなツールを使用して作成・削除・取得できます。これには、過去のポストを検索したり、リアルタイムで取得したりすることが含まれます。

    X API は、プラットフォーム上のさまざまな種類のリソースを作成、削除、取得、または変更するためのアクセスを提供します。主なリソースには次のものが含まれます。

    | Resource            | Description                                                                                                        |
    | :------------------ | :----------------------------------------------------------------------------------------------------------------- |
    | **Posts**           | 何百万ものポストにアクセスしてパブリックな会話を理解したり、自分のポストを作成して会話に参加したりできます。  現在の提供状況:  - X API v2  - Enterprise  - Standard v1.1        |
    | **Users**           | X ユーザーを管理または取得してネットワークを分析し、オーディエンスを理解したり、健全なオンライン関係を育んだりできます。  現在の提供状況:  - X API v2  - Enterprise  - Standard v1.1 |
    | **Spaces**          | X Spaces とその参加者を検索・取得して、人々が興味深く関連性の高い音声会話を見つけられるようにします。  現在の提供状況:  - X API v2                                      |
    | **Direct Messages** | ダイレクトメッセージを送受信して、顧客の問題に優先度を付けて対応したり、ウェルカムメッセージを送信したり、前向きな人間同士のコミュニケーションを生み出したりできます。  現在の提供状況:  - Standard v1.1     |
    | **Lists**           | アカウントのリストをキュレーションおよび管理し、業界の専門家や影響力のある声の動向を常に把握したり、フォローしているアカウントを整理したりできます。  現在の提供状況:  - X API v2  - Standard v1.1  |
    | **Trends**          | 地理的なトレンドをいち早く把握して業界動向を特定し、ホットな話題を見つけたり、最新の流行の先を行くことができます。  現在の提供状況:  - Standard v1.1                               |
    | **Media**           | メディアオブジェクトをアップロードして、クリエイティブなエネルギーを共有したり、インタラクティブな体験を作成したり、アクセシビリティツールを構築したりできます。  現在の提供状況:  - Standard v1.1        |
    | **Places**          | 場所を検索して、近隣や世界中で何が起こっているかを把握できます。  現在の提供状況:  - Standard v1.1                                                        |
  </Tab>
