このガイドでは、初めてリクエストを送信するための手順を順を追って説明します。X アカウントへのサインアップが完了した後に役立つリソースです。

サンプルコードや、より技術的なガイド、Postman のようなグラフィカルツールを使いたい場合は、次のガイドを利用して初めてのリクエストを送信することを検討してください。

* [Post エンドポイントへの初めてのリクエストを行うためのクイックスタートガイド](/ja/x-api/posts/manage-tweets/quickstart)
* [Postman を使い始める](/ja/tutorials/postman-getting-started) - 初心者向け
* [X API v2 サンプルコード](https://github.com/xdevplatform/Twitter-API-v2-sample-code)
* [Post ルックアップ API リファレンス](/ja/x-api/posts/lookup/introduction)

このガイドでは、すでに [API キーとシークレット](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret)、[ユーザーアクセストークンとシークレット](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)、[App Access Token](/ja/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) を取得し、安全な場所に保管していることを前提としています。これらの取得方法については、[X API へのアクセスの取得](/ja/x-api/getting-started/getting-access) ガイドの手順に従って学ぶことができます。

<div id="step-1-identify-which-endpoint-you-would-like-to-use">
  ### ステップ 1. 使用したいエンドポイントを選択する
</div>

X API を使用すると、X のウェブサイトやモバイルアプリケーション上で行えるさまざまな操作を、コードから実行できます。

API で利用可能なエンドポイントの完全な一覧は [API Reference Index](/ja/x-api/introduction) にまとめていますが、まずはシンプルに始めるため、次のいずれかから始めることをおすすめします。

* [Manage Posts &gt; Post a Tweet](/ja/x-api/posts/manage-tweets/introduction)
* [Search Posts](/ja/x-api/posts/search/introduction)
* [User lookup](/ja/x-api/users/lookup/introduction)

<div id="step-2-choose-a-tool-to-make-your-request">
  ### ステップ 2. リクエストに使うツールを選択する
</div>

一部のリクエストは簡単に見えても、他のリクエストは組み立てが難しい場合があります。そこで、優れた開発者コミュニティが、そうした複雑さの一部を意識せずに済むようにするツールを開発しています。

以下に、推奨されるツールとその使い方の概要を示します。

<div id="postman">
  #### Postman
</div>

Postman は、REST エンドポイントへのリクエストを送信するために使用できる GUI ツールです。X API 経由で利用できるさまざまなエンドポイントを使い始めたり調査したりする際に役立つ、Postman に関する有用な資料を用意しています。

まずは、キーとトークンの追加方法や最初のリクエストの送信方法を学ぶために、「[Getting started with Postman チュートリアル](/ja/tutorials/postman-getting-started)」を一読することをおすすめします。

また、ほとんどの X API v2 エンドポイント向けにそれぞれクイックスタートガイドを用意しており、その多くで Postman を利用しています。各エンドポイントのセクションに掲載されていますが、いくつか例を挙げます。

* クイックスタート: ポストを作成する
* クイックスタート: ポストを検索する
* クイックスタート: ユーザーを取得する

なお、Postman ではストリーミングエンドポイントに対してリクエストを送信することはできません。これらのエンドポイントの扱い方については、[filtered stream](/ja/x-api/posts/filtered-stream/quickstart) または [1% sampled stream](/ja/x-api/posts/sample-stream) のクイックスタートガイドを参照してください。

よりシンプルなグラフィカルツールを好む場合は、[Insomnia](https://insomnia.rest) の利用も検討してください。

<div id="sample-code">
  #### サンプルコード
</div>

リクエストを送信するために簡単なコードを使いたい場合は、各 X API v2 エンドポイントごとに、複数の言語で用意したサンプルコードがあります。

これらのコードサンプルは、GitHub リポジトリ [X-API-v2-sample-code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) から入手できます。このリポジトリには、リクエストを正しく送信できるように認証情報の設定方法を説明した README ファイルも含まれています。

例えば、ユーザー lookup エンドポイント用の cURL の例は次のとおりです。このリクエストを使用するには、`$ACCESS_TOKEN` と `$USERNAME` をそれぞれ [App Access Token](/ja/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) と X のハンドル名に置き換えるだけです。最後に、このコードをコマンドラインツールにコピー＆ペーストし、Return または Enter キーを押してください。

```bash
curl "https://api.x.com/2/users/by/username/$USERNAME" -H "Authorization: Bearer $ACCESS_TOKEN"
```

<div id="libraries">
  #### ライブラリ
</div>

素晴らしい X Developer コミュニティによって、さまざまなプログラミング言語向けのライブラリも多数作成されており、X API へのリクエスト送信に利用できます。

私たちは、把握しているコミュニティ製ライブラリをすべて掲載した「[ツールとライブラリ](/ja/x-api/tools-and-libraries/overview)」ページを用意しています。各ライブラリには、ローカル環境へのリポジトリのセットアップ方法と、最初のリクエストの実行方法を学べる README ファイルが含まれているはずです。

> **ご注意ください:** 問題が発生した場合は、リクエストを送信しているエンドポイントの開発者向けドキュメント、[サポートセクション](https://developer.x.com/en/support)、またはフォーラムのコミュニティをご確認・ご利用ください。必ずリクエストを成功させられるようサポートします。

<div id="step-3-review-the-response">
  ### ステップ 3. レスポンスを確認する
</div>

リクエストが成功すると、そのリクエストに関連するメタデータを含むペイロードが返されます。

GET HTTP メソッドを使用するエンドポイントを呼び出した場合、リクエストしたリソース（ポスト、ユーザー、リスト、Space など）に関連するメタデータが JSON 形式で返されます。返ってきたさまざまなフィールドを確認し、リクエストした情報が X 上のコンテンツと対応付けられるかどうか確認してみてください。

POST、PUT、DELETE HTTP メソッドを使用するエンドポイントを呼び出した場合、X 上で何らかのアクションを実行したことになります。X.com またはモバイルアプリを開き、そのアクションを見つけられるか確認してみてください。

<div id="step-4-adjust-your-request-using-parameters">
  ### ステップ 4. パラメータを使ってリクエストを調整する
</div>

各エンドポイントには、リクエストを調整するために使用できるさまざまなパラメータがあります。たとえば、`fields` と `expansions` パラメータを指定して GET エンドポイントを使用することで、追加のメタデータフィールドをリクエストできます。また、[search Posts](/ja/x-api/posts/search/introduction)、[Post counts](/ja/x-api/posts/counts/introduction)、[filtered stream](/ja/x-api/posts/filtered-stream/introduction) などのエンドポイントで、さまざまなフィルタリング手法を試し、受信するデータを自分が関心のあるポストのみに絞り込むことができます。

利用しているエンドポイントの API Reference には、リクエストパラメータとフィールドの一覧がすべて記載されています。また、インテグレーションガイドや基礎解説ページには、そのほかにも多くの有用な連携関連情報が掲載されています。

提供している各種学習資料の詳細は、[Important resources](/ja/x-api/getting-started/important-resources) ガイドで確認できます。
