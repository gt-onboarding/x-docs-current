このガイドでは、最初のリクエストを実行するために従うことができる手順を順を追って説明します。X アカウントに登録した後に活用できる便利なリソースです。

サンプルコード、より技術的なガイド、または Postman のようなグラフィカルツールを使いたい場合は、次のガイドの利用を検討してください。これらを使って最初のリクエストを実行できます。

* [Post エンドポイントへの最初のリクエストを行うためのクイックスタートガイド](/ja/x-api/posts/manage-tweets/quickstart)
* [Postman を使い始める](/ja/tutorials/postman-getting-started) - 初心者に最適
* [X API v2 コードサンプル](https://github.com/xdevplatform/Twitter-API-v2-sample-code)
* [Post lookup API リファレンス](/ja/x-api/posts/lookup/introduction)

このガイドでは、[API キーとシークレット](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret)、[ユーザー アクセストークンとシークレット](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)、[App Access Token](/ja/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) を取得し、安全な場所に保存していることを前提としています。これらの取得方法については、[X API へのアクセスを得る](/ja/x-api/getting-started/getting-access) ガイドの手順に従うことで確認できます。

<div id="step-1-identify-which-endpoint-you-would-like-to-use">
  ### ステップ 1. 使用したいエンドポイントを特定する
</div>

X API を使用すると、X のウェブサイトやモバイルアプリケーション上で行えるさまざまな操作を、コードから実行できます。

API で利用可能なエンドポイントの一覧はすべて [API Reference Index](/ja/x-api/introduction) に掲載されていますが、まずは次のいずれかに絞って利用することをおすすめします。そのほうがシンプルに始められます。

* [Manage Posts &gt; Post a Tweet](/ja/x-api/posts/manage-tweets/introduction)
* [Search Posts](/ja/x-api/posts/search/introduction)
* [User lookup](/ja/x-api/users/lookup/introduction)

<div id="step-2-choose-a-tool-to-make-your-request">
  ### ステップ 2. リクエストを送信するためのツールを選択する
</div>

一部のリクエストは比較的簡単に作成できますが、ほかのものは組み立てが難しい場合があります。そのため、優れた開発者コミュニティによって、複雑さの一部を吸収してくれるツールが開発されています。

以下に、推奨されるツールとその使い方の概要を示します。

<div id="postman">
  #### Postman
</div>

Postman は、REST エンドポイントへのリクエストを送信するために使用できるビジュアルツールです。X API 経由で利用可能なさまざまなエンドポイントを使い始めたり探索したりするのに役立つ、Postman に関する資料を用意しています。

まずは、キーとトークンの追加方法や最初のリクエストの送信方法を学ぶために、「[Getting started with Postman](/ja/tutorials/postman-getting-started)」チュートリアルをお読みいただくことをおすすめします。

また、ほとんどの X API v2 エンドポイント向けに、Postman を利用したクイックスタートガイドも作成しています。これらのガイドは、それぞれのエンドポイントセクションで参照できますが、いくつか例を挙げます。

* クイックスタート: ポストを投稿する
* クイックスタート: ポストを検索する
* クイックスタート: ユーザー情報を取得する

なお、Postman を使用してストリーミングエンドポイントへリクエストを送信することはできません。これらのエンドポイントの扱い方を学ぶには、[filtered stream](/ja/x-api/posts/filtered-stream/quickstart) または [1% sampled stream](/ja/x-api/posts/sample-stream) のクイックスタートガイドをご覧ください。

よりシンプルなグラフィカルツールをお探しの場合は、[Insomnia](https://insomnia.rest) の利用もご検討ください。

<div id="sample-code">
  #### サンプルコード
</div>

リクエストを送信する際に簡単なコードを使いたい場合は、各 X API v2 エンドポイント向けに、複数の言語でサンプルコードを用意しています。

サンプルコードは、GitHub リポジトリ [X-API-v2-sample-code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) から入手できます。このリポジトリには README ファイルも含まれており、リクエストが正しく動作するように認証情報を設定する方法を学ぶために利用できます。

たとえば、ユーザー取得エンドポイント向けの cURL の例は次のとおりです。このリクエストを使用するには、$ACCESS&#95;TOKEN と $USERNAME を、ご自身の [App Access Token](/ja/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) と X のハンドルに置き換えるだけです。あとは、このコードをコマンドラインツールにコピー＆ペーストして、「Return」または「Enter」キーを押してください。

```bash
curl "https://api.x.com/2/users/by/username/$USERNAME" -H "Authorization: Bearer $ACCESS_TOKEN"
```

<div id="libraries">
  #### ライブラリ
</div>

優れた X の開発者コミュニティによって、さまざまなプログラミング言語向けのライブラリも開発されています。これらのライブラリを使って、X API へのリクエストを送信できます。

私たちは、把握しているコミュニティ製ライブラリをすべて掲載した「[ツールとライブラリ](/ja/x-api/tools-and-libraries/overview)」ページを用意しています。各ライブラリには、ローカル環境でリポジトリをセットアップし、最初のリクエストを送信する方法を学べる README ファイルが用意されているはずです。

> **ご注意ください:** 問題が発生した場合は、リクエストを送信しているエンドポイントに関する開発者向けドキュメントや、[サポートセクション](https://developer.x.com/en/support)を確認するか、フォーラム上でコミュニティに問い合わせてサポートを受けてください。リクエストが成功するよう、私たちがサポートします。

<div id="step-3-review-the-response">
  ### ステップ 3. レスポンスを確認する
</div>

リクエストが成功すると、そのリクエストに関連するメタデータを含むペイロードを受け取ります。

GET HTTP メソッドを利用するエンドポイントを使用した場合は、リクエストを送信した対象のリソース（ポスト、ユーザー、リスト、Space など）に関連するメタデータが JSON 形式で返されます。返ってきたさまざまなフィールドを確認し、リクエストした情報を X 上のコンテンツに対応付けられるか確認してください。

POST、PUT、DELETE HTTP メソッドを利用するエンドポイントを使用した場合は、X 上でアクションを実行したことになります。X.com またはモバイルアプリにアクセスし、そのアクションを追跡できるか確認してください。

<div id="step-4-adjust-your-request-using-parameters">
  ### ステップ 4. パラメータを使用してリクエストを調整する
</div>

各エンドポイントには、リクエストを調整するために使用できるさまざまなパラメータが用意されています。たとえば、`fields` と `expansions` パラメータを指定して `GET` エンドポイントを使用することで、追加のメタデータフィールドをリクエストできます。また、[search Posts](/ja/x-api/posts/search/introduction)、[Post counts](/ja/x-api/posts/counts/introduction)、[filtered stream](/ja/x-api/posts/filtered-stream/introduction) などのエンドポイントで、さまざまなフィルタリング手法を試して、取得するデータを関心のあるポストだけに絞り込むこともできます。

利用中のエンドポイントの API リファレンスには、使用可能なリクエストパラメータとフィールドの一覧が記載されており、統合ガイドや基礎解説ページには、インテグレーションに役立つ多くの情報が掲載されています。

ご利用いただける各種学習用コンテンツについては、[Important resources](/ja/x-api/getting-started/important-resources) ガイドで詳しく確認できます。
