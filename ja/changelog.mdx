<Update label="Dec 14, 2021" description="X API v2: 認証">
  ### OAuth 2.0 認証のサポート

  本日より、すべての開発者は developer portal で認証方法として OAuth 2.0 を選択することで、OAuth 2.0 を使用して認証できるようになりました。このリリースの一環として、機密クライアントおよびパブリッククライアント、ならびに関連するすべての v2 エンドポイントでこの認証方法を利用できるようサポートを追加しました。

  OAuth 2.0 は業界標準の認可プロトコルであり、アプリケーションのスコープを開発者がより細かく制御できるようにするとともに、複数デバイス間にわたる認可フローを改善します。言い換えると、X 上の人々向けにアプリケーションを構築する開発者は、自分のアプリがユーザーからリクエストする情報を、これまで以上に細かく制御できるようになります。そのため、エンドユーザーに対して、本当に必要なデータと情報だけをリクエストすれば済むようになります。

  この最新の認可プロトコルにより、エンドユーザーがあなたのアプリを承認する際に、よりシンプルで効率的な同意フローを提示できるようになります。このフローには、あなたがリクエストした特定のスコープのみが表示されます。

  このローンチの詳細については、フォーラムのお知らせをご覧ください。
</Update>

<Update label="Nov 15, 2021" description="X API v2: 早期アクセス">
  ### X API v2 における主要なプラットフォーム更新

  **主要なプラットフォーム更新: X API v2 がプライマリバージョンに、新しい Essential および Elevated アクセス、ポリシー変更**

  本日、X API v2 が X API のプライマリバージョンになったことをお知らせします。X API v2 には、X API 上で構築されている既存のすべてのアプリの 90% のニーズを満たすのに十分な数のエンドポイントと機能が実装されました。

  X API v2 における開発者体験をさらに向上させるため、本日、以下の変更を導入します。

  * API への高速かつ無料で利用できる Essential アクセスと、利用目的が承認された開発者向けの無料の Elevated アクセス。これらのアクセス レベルについては、「About the X API」ページでさらに詳しく説明しています。

  * X のコア機能を利用した開発方法を制限し、アプリでサポートできるユーザー数を制限していた Developer Policy 上の文言を削除します。

    * 私たちは、あなたが X をより良く、より健全にし、パブリックな会話を広げるツールやプロダクトを構築できるように、プラットフォームを開放します。[私たちがぜひ構築してほしいと考えているソリューションの一覧をご覧ください](https://developer.x.com/en/docs/twitter-api/what-to-build)。

    * 特に、X のコア機能を含む X の体験の複製を制限していた条項と、多数のユーザートークンを持つために許可を要件としていた条項を削除しました。

    * X 上の人々を支援するソリューションを構築するには、多くの場合、開発者が X 上で利用できる機能の一部を構築（あるいは複製）する必要があることを、私たちは理解しています。Developer Policy のこれらの変更は、開発者エコシステムに明確性をもたらし、開発者がパブリックな会話に対して、より簡単に構築・イノベーション・貢献できるようにする、オープンな API プラットフォームを提供することを目的としています。

  このローンチおよびこの段階に至るまでの X API のローンチについては、[フォーラムの投稿](https://devcommunity.x.com/t/ushering-in-a-new-era-for-the-twitter-developer-platform-with-the-twitter-api-v2/162087)でさらに詳しく説明しています。

  本日、関心のある会話に絞り込めるようにするための新しい演算子 is:reply を導入しました。これにより、結果から返信ポストを除外できるようになりました（すでに利用可能な、reposts を除外する is:retweet や、引用ポストを除外する is:quote に加えて利用できます）。この新しい演算子は、本日より [PowerTrack](/ja/x-api/enterprise-gnip-2.0/powertrack-api)、[Historical PowerTrack](https://developer.x.com/content/developer-twitter/en/docs/tweets/batch-historical/overview)、および [Replay](/ja/x-api/enterprise-gnip-2.0/powertrack-api#replay-api) で利用できます。

<Update label="Aug 20, 2018" description="X API: Enterprise v1.1">
  ### Usage API の新バージョン

  本日、Usage API の新バージョンをリリースしました。このアップデートにより、新しいシステムに移行することで API の安定性を向上させます。

  システムのアップデートに伴い、開発者は 2018 年 5 月 1 日までのデータしか取得できないため、ユーザー側ではデータに「空白期間」が生じます。その結果、今後 1 年間（2019 年 6 月まで）は、数か月分の過去データが抜けた状態となります。

  ご不明な点がありましたら、アカウントマネージャーまでお問い合わせください。
</Update>

<Update label="May 10, 2018" description="X API: Standard v1.1">
  ### 引用ペイロードのレンダリングの変更

  本日、引用ペイロード内で URL がレンダリングされる方法に変更を加えました。

  引用されたポストを参照できるようにするために、&quot;quoted&#95;status&#95;permalink&quot; という新しいエンティティを追加し、引用されたポストの &quot;text&quot; フィールドから t.co リンクを削除します。

  新しい形式では、&quot;quoted&#95;status&quot; オブジェクト内の引用ポストの &quot;text&quot; フィールドの末尾に、もはや t.co リンクは追加されません。

  さらに、引用ポストの t.co url、expanded&#95;url、display&#95;url を、これまでのように &#39;urls&#39; エンティティには含めません。代わりに、これらは &quot;url&quot;、&quot;expanded&quot;、&quot;display&quot; の各 URL 属性を持つ新しい &quot;quoted&#95;status&#95;permalink&quot; オブジェクトに追加されます。つまり、&quot;quoted&#95;status&#95;permalink&quot; オブジェクトは引用ポストへの t.co リンク専用となり、一方で &quot;entities.urls&quot; 配列は、元のポストまたは引用内で共有されているリンクを特定するために使用できます。

  詳細については、[フォーラムの投稿](https://devcommunity.x.com/t/updating-how-urls-are-rendered-in-the-quote-tweet-payload/105473)と[更新されたドキュメント](https://devcommunity.x.com/t/upcoming-changes-to-the-developer-platform/104603)をお読みください。
</Update>

  `statuses/update` に対するオプションの `enable_dm_commands` パラメータにより、移行期間が終了する前に、アプリケーションが DM コマンドのサポートを早期に削除できるようになります。デフォルト値は *true*（つまり現在のレガシー動作）ですが、*false* に設定すると、11月1日以降の新しい動作になります。`statuses/update` に対するオプションの `fail_dm_commands` パラメータを *true* に設定すると、DM コマンドは API から HTTP 403（エラーコード 151）を返すようになります。デフォルト値は *false* です。このアップデートの詳細は[フォーラム](https://devcommunity.x.com/t/retiring-legacy-dm-commands-on-the-standard-tweet-api/86653)をご覧ください。
</Update>

<Update label="Feb 22, 2017" description="X API：Enterprise v1.1">
  ### 新しいエンリッチメントとペイロードの変更

  新しいエンリッチメント機能として、投票メタデータが Enterprise API で利用可能になりました。詳細については[ドキュメント](/ja/x-api/enterprise-gnip-2.0/fundamentals/account-activity#migration-introduction)を参照してください。

  新しいペイロードフィールド: ユーザーがポストに写真を追加する際に「Add description」オプションを選択した場合、ペイロードの entities.media[] および extended&#95;entities.media[] セクションに Image Alt-Text（decription）フィールドが含まれることがあります。これは、エンリッチ済みネイティブ形式でのみ利用可能です。

  デフォルトのペイロード形式: Gnip プロダクト上で新規に作成されるストリームは、今後は original 形式の JSON をデフォルトとします。
</Update>
