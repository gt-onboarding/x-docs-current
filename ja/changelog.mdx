  ### フィルタリングオペレーターのエイリアス追加

  検索および filtered stream エンドポイント全体（Enterprise、Premium、v2 を含む）に、フィルタリング／クエリオペレーターの「エイリアス」が追加されました。多くの場合、このオペレーターは statuses ではなくポストを参照します。たとえば、&quot;in&#95;reply&#95;to&#95;status&#95;id&quot; のエイリアスとして &quot;in&#95;reply&#95;to&#95;tweet&#95;id&quot; が提供されます。別のケースでは、新しいオペレーターエイリアスによってマッチング内容がより明確になります。たとえば、ポストには動画添付が 1 つしか付けられないため、&quot;has:videos&quot; に対する新しいエイリアスとして &quot;has:video&#95;link&quot; が提供されます。さらに多くの例については、[v2 filtered stream operator table](https://developer.x.com/en/docs/twitter-api/tweets/filtered-stream/integrate/build-a-rule#list) を参照してください。
</Update>

<Update label="Dec 14, 2021" description="X API v2：認証">
  ### OAuth 2.0 認証のサポート

  本日より、すべての開発者は developer portal で認証方法として OAuth 2.0 を選択することで、OAuth 2.0 を使用して認証できるようになりました。今回のリリースでは、機密クライアントおよびパブリッククライアントのサポートに加え、この認証方法を利用できる関連するすべての v2 エンドポイントを追加しました。

  OAuth 2.0 は業界標準の認可プロトコルであり、アプリケーションのスコープをより細かく制御できるようにし、複数デバイスにまたがる認可フローを改善します。言い換えると、X を利用する人々向けにアプリケーションを構築する開発者は、自身のアプリがユーザーからどの情報を要求するかをより細かく制御できるようになり、エンドユーザーには本当に必要なデータや情報だけをリクエストすればよくなります。

  この最新の認可プロトコルにより、エンドユーザーがあなたのアプリを認可するための同意フローを、よりシンプルな形で提示できるようになります。このフローには、あなたがリクエストした特定のスコープのみが表示されます。

  今回のリリースの詳細については、フォーラムでのアナウンスをご覧ください。
</Update>

<Update label="Nov 15, 2021" description="X API v2：早期アクセス">
  ### X API v2 に対する大規模なプラットフォームアップデート

  **大規模なプラットフォームアップデート：X API v2 が主要バージョンに、Essential および Elevated アクセスの新設、ポリシー変更**

  本日、X API v2 が X API の主要バージョンになったことをお知らせします。X API v2 には、X API 上で構築されている既存のアプリの 90% のニーズを満たすのに十分な数のエンドポイントと機能が実装されました。

  X API v2 におけるデベロッパーエクスペリエンスをさらに向上させるため、以下の変更を本日リリースします。

  * API への高速かつ無料の Essential アクセスと、利用目的が承認された開発者向けの無料の Elevated アクセスを提供します。これらのアクセスレベルの詳細については、「About the X API」ページで詳しく説明しています。

  * X のコア機能を使ってどのように構築できるかを制限し、アプリでサポートできるユーザー数を制限していた Developer Policy 上の文言を削除します。

    * 私たちはプラットフォームを開放し、X をより良く、より健全にし、パブリックな会話を拡張するツールやプロダクトをあなたが構築できるよう奨励します。[私たちがぜひ構築してほしいと考えているソリューションの一覧をご覧ください](https://developer.x.com/en/docs/twitter-api/what-to-build)。

    * 具体的には、X のコア機能を含む X のエクスペリエンスの複製を制限していた条項や、多数のユーザートークンを持つ場合に許可を必要としていた条項を削除しました。

    * X を利用する人々を支援するソリューションを構築するには、多くの場合、開発者が X 上で提供されている機能の一部を構築（あるいは複製）する必要があることを私たちは理解しています。Developer Policy のこれらの変更は、開発者エコシステムにもたらすべき点を明確にし、開発者がパブリックな会話に対して構築・イノベーション・貢献しやすくなるような、オープンな API プラットフォームを提供することを目的としています。

<Update label="May 10, 2018" description="X API: Standard v1.1">
  ### 引用ペイロードでの URL レンダリングの変更

  本日、引用ペイロード内での URL のレンダリング方法を変更しました。

  引用ポストを参照できるようにするため、新しいエンティティ &quot;quoted&#95;status&#95;permalink&quot; を追加し、引用ポストの &quot;text&quot; フィールドから t.co リンクを削除します。

  新しいフォーマットでは、t.co リンクは &quot;quoted&#95;status&quot; オブジェクト内の引用ポストの &quot;text&quot; フィールドには、もはや追加されなくなります。

  さらに、引用ポストの t.co url、expanded&#95;url、display&#95;url を ‘urls’ エンティティには含めなくなります。その代わり、これらは &quot;url&quot;、&quot;expanded&quot;、&quot;display&quot; という URL 属性を持つ新しい &quot;quoted&#95;status&#95;permalink&quot; オブジェクトに追加されます。これは、&quot;quoted&#95;status&#95;permalink&quot; オブジェクトが引用ポストへの t.co リンク専用となり、一方で &quot;entities.urls&quot; 配列は元のポストまたは引用内で共有された任意のリンクを特定するために使用できることを意味します。

  詳細については、[フォーラムの投稿](https://devcommunity.x.com/t/updating-how-urls-are-rendered-in-the-quote-tweet-payload/105473)および[更新されたドキュメント](https://devcommunity.x.com/t/upcoming-changes-to-the-developer-platform/104603)をお読みください。

  すべての HTTP ヘッダーが強制的に小文字化されました（`content-type`、`x-rate-limit-remaining`、`x-access-level` など）。このアップデートの詳細は [フォーラム](https://devcommunity.x.com/t/upcoming-http-header-changes/86715) を参照してください。
</Update>

<Update label="May 5, 2017" description="X API: Standard v1.1">
  ### DM コマンド用オプションパラメータ

  `statuses/update` のオプションパラメータ `enable_dm_commands` を利用すると、移行期間が終了する前に、アプリケーション側で DM コマンドのサポートを早期に無効化できます。デフォルト値は *true*（すなわち現在のレガシー動作）ですが、*false* に設定することで、11 月 1 日以降の新しい動作を得ることができます。`statuses/update` のオプションパラメータ `fail_dm_commands` を *true* に設定すると、DM コマンドは API から HTTP 403（エラーコード 151）を返すようになります。デフォルト値は *false* です。このアップデートの詳細は [フォーラム](https://devcommunity.x.com/t/retiring-legacy-dm-commands-on-the-standard-tweet-api/86653) を参照してください。
</Update>

<Update label="Feb 22, 2017" description="X API: Enterprise v1.1">
  ### 新しいエンリッチメントとペイロード変更

  新しいエンリッチメント：Poll Metadata が Enterprise API から利用可能になりました。詳細については [ドキュメント](/ja/x-api/enterprise-gnip-2.0/fundamentals/account-activity#migration-introduction) を参照してください。

  新しいペイロードフィールド：画像の代替テキスト（decription）フィールドは、ユーザーがポストに写真を追加する際に「Add description」オプションを選択した場合、ペイロードの `entities.media[]` および `extended_entities.media[]` セクションに含まれることがあります。これはエンリッチ済みネイティブ形式でのみ利用可能です。

  デフォルトのペイロード形式：Gnip 製品上で新たに作成されるストリームは、デフォルトでオリジナル形式の JSON になります。
</Update>
