<div id="overview">
  ## 概览
</div>

Catalog API 是一套电商解决方案，为广告主提供设置商品 Feed、将商品分组为商品集，以及全方位管理目录商品的能力。Catalog API 将成为支持程序化目录管理的核心工具，使广告主可以更好地控制其目录的导入和更新方式。

一个目录包含用户商品及商品集的所有相关信息，并与该用户的账号 handle 关联。目前，**一个用户只能创建一个目录**。 

Catalog API 支持两种商品导入方式：

1.  定时 Feed：用户可以添加一个 Feed URL（CSV、TSV 和 XML），系统会定期抓取该文件（X 目前支持的单个文件最大为 8GB）

2.  Batch Products API：用户可以通过批量（JSON）请求查看、创建、更新和删除商品属性

X 商家可以基于筛选规则创建商品集。通过这种方式，他们可以为商品附加额外的元数据，并创建特殊的商品集，通过自然电商功能或动态商品广告来展示这些商品。Catalog API 和 X shopping manager 支持两种类型的商品集。 

1.  手动：用户可以选择最多 50 个商品，并设置名称和描述

2.  筛选：用户可以添加最多 30 个筛选条件，以自动生成满足这些条件的商品集。筛选条件类型包括价格、Google 商品分类、商品类型、库存、促销价和自定义字段。

Catalog API 提供 4 个用于管理目录的端点：

* [Product Catalogs](https://developer.x.com/en/docs/x-ads-api/catalog-management/api-reference/product-catalogs) \- 创建商品信息的容器
* [Products](https://developer.x.com/en/docs/x-ads-api/catalog-management/api-reference/products) -  同步添加、更新、查看和移除目录中的商品
* [Product Sets](https://developer.x.com/en/docs/x-ads-api/catalog-management/api-reference/product-sets) -  基于商品信息将商品归类分组
* [Schedule Feeds](https://developer.x.com/en/docs/twitter-ads-api/catalog-management/api-reference/scheduled-feeds) \- 设置将基于文件的商品数据异步导入到目录中

<div id="prerequisites-for-the-catalog-api">
  ### Catalog API 的先决条件
</div>

Catalog API 端点目前仅以早期访问形式提供。若要申请访问，请联系你的 X 客户代表。 

你需要通过 [X Shopping Manager](https://ads.x.com/shopping_manager) 同意服务条款。

有关 Shopping Manager 和产品规范的详细信息，请参阅 [产品规范指南](https://business.x.com/en/help/shopping-specs.html)。

<div id="rate-limits">
  ### 速率限制
</div>

下表列出了每个端点的速率限制。

<div id="product-catalog">
  #### 产品目录
</div>

| HTTP 方法 | 速率限制 |
| :--- | :--- |
| GET /product_catalogs | 每 15 分钟 1,000 次 |
| POST /product_catalogs | 每 15 分钟 20 次 |
| PUT /product\_catalogs/:product\_catalog_id | 每 15 分钟 20 次 |
| DELETE /product\_catalogs/:product\_catalog_id | 每 15 分钟 20 次 |

| DELETE /product\_catalogs/:product\_catalog_id/products | 每 15 分钟 150 次 |
