<div id="requesting-additional-app-level-permissions">
  ### 申请额外的应用级权限
</div>

如果你的应用当前仅被授予以下任一应用级权限，并且你希望访问更多端点，请联系你的 X 客户代表申请 **Standard Access**。

在 2023 年 7 月之前申请 Ads API 访问权限的开发者，可能曾申请以下任一应用级权限，这些权限现在都已包含在 Standard Access 级别中：

- **Analytics**（只读）：用户可以以[只读](https://developer.x.com/en/docs/apps/app-permissions)权限访问 [Analytics](https://developer.x.com/en/docs/twitter-ads-api/analytics/overview) 端点。用户无法访问 Campaign Management、Creative 或 Custom Audiences 端点，但可以通过在 Analytics 端点的请求参数中传递创意实体来获取指标。若要按 media_id 或 media_name 这一粒度获取指标，用户应用需要具备 Ads API 的写入权限。

- **Campaign Management & Creative**（读写）：用户可以以[读写](https://developer.x.com/en/docs/apps/app-permissions)权限访问 [Campaign Management](https://developer.x.com/en/docs/twitter-ads-api/campaign-management/overview) 和 [Creative](https://developer.x.com/en/docs/twitter-ads-api/creatives/overview) 端点。用户同样可以访问 Analytics 端点。

- **Custom Audiences**（读写）：用户可以以[读写](https://developer.x.com/en/docs/apps/app-permissions)权限访问 [Custom Audiences](https://developer.x.com/en/docs/twitter-ads-api/audiences/overview) 端点，并且同样可以访问 Campaign Management、Creative 和 Analytics 端点。

- **Conversion**（读写）：用户可以以[读写](https://developer.x.com/en/docs/apps/app-permissions)权限访问 [Mobile](https://developer.x.com/en/docs/twitter-ads-api/measurement/mobile-conversions/overview) 和 [Web Conversion](https://developer.x.com/en/docs/twitter-ads-api/measurement/web-conversions/overview) 端点。用户可能具备，也可能不具备对其他 Ads API 端点的访问权限。 

<div id="requesting-an-increase-to-user-oauth-token-limits">
  ### 申请提高用户 OAuth 令牌上限
</div>

在 2023 年 7 月之前申请 Ads API 访问权限的开发者，其应用的用户 OAuth 令牌上限可能为 5 个。如果你需要提高开发者应用的令牌上限，请联系你的 X 业务代表。

<Note>**注意：**我们不会提高任何 Ads API 端点的[速率限制](https://developer.x.com/en/docs/twitter-ads-api/rate-limiting)。请查看我们的[最佳实践](https://developer.x.com/en/docs/twitter-ads-api/rate-limiting)，了解如何在速率限制范围内使用。</Note>
