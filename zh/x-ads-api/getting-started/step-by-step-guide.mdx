<div id="how-to-get-access-to-the-ads-api">
  ## 如何获取 Ads API 访问权限
</div>

1. 注册一个[开发者账号](https://developer.x.com/en/apply-for-access)。
2. 创建一个[开发者应用](https://developer.x.com/app)，并妥善保管你的 token。
3. 访问 [ads.x.com/help](https://ads.x.com/help)，为你的每个开发者应用申请 Ads API 访问权限。

<Note>**注意：**如果你已经在 X 开发者平台上进行开发，并且已有开发者账号，请直接跳到步骤 3。</Note>

<div id="step-one-signup-for-a-developer-account">
  ### 第一步：注册开发者账号
</div>

要向任何 X 的 API 产品发起请求，您必须先注册一个开发者账号。然后在开发者门户中创建一个项目和开发者应用。这样您就会获得一组凭证，用于对所有 API 请求进行身份验证。

<Button href="https://developer.x.com/en/apply-for-access">
  注册开发者账号
</Button>

<div id="step-two-save-your-apps-key-and-tokens-and-keep-them-secure">
  ### 步骤二：保存你的应用密钥和令牌并确保其安全
</div>

在你的开发者应用中，你将获得一组 API Keys（也称为 Consumer Keys）。你还可以生成一组可代表你的个人 X 账号发起请求的 Access Tokens，以及一个可用于对需要 OAuth 2.0 Bearer Token 的端点进行身份验证的 Bearer Token。由于这些密钥和令牌在重新生成前都不会过期，我们建议你创建环境变量，或使用安全的密码管理器进行保存。

请同时记下你的 App ID，可以在应用详情页面的 URL 中找到。这将使你在下一步请求访问 Ads API 时更加方便。

<Note>
**注意：** 你的密钥和令牌只会在开发者门户中显示一次，因此在生成后应立即将这些凭证存储到你的密码管理器中。

如果你丢失或忘记了密钥和令牌，你将需要重新生成它们，这会创建新的密钥和令牌，并使旧的失效。这意味着你必须更新所有使用旧凭证所建立的集成。

详细了解我们的[身份验证最佳实践](https://developer.x.com/en/docs/authentication/guides/authentication-best-practices)。
</Note>

<div id="step-three-apply-for-access-to-the-ads-api">
  ### 第三步：申请 Ads API 访问权限
</div>

此时你已经拥有了 [X API 的基本访问权限](https://developer.x.com/en/docs/twitter-api/v1)，但还不能使用特定的 X Ads API 功能。接下来，你需要申请并获批 Ads API 访问权限。访问 [ads.x.com/help](https://ads.x.com/help)，为你的每个开发者应用提交 Ads API 访问权限申请。

<div id="access-tiers">
  ### 访问层级
</div>

在申请过程中，你需要说明你所需的访问层级。了解更多关于[应用级别和广告账户级别权限](https://developer.x.com/en/docs/twitter-ads-api/obtaining-ads-account-access#levels-of-access)的信息。

<div id="conversion-only">
  #### 仅限转化
</div>

可读写访问移动端和网页转化端点。

<div id="standard-access">
  #### 标准访问
</div>

可访问 Analytics、Campaign Management、Creatives、Custom Audiences 和 Conversion 端点，并具有读写权限。

<Note>**注意：** 在你的应用获批 Ads API 访问权限之后，你需要[重新生成所有用户访问令牌](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)，才能正确发起[已认证的 Ads API 请求](https://developer.x.com/en/docs/twitter-ads-api/making-authenticated-requests)。</Note>

<Note>**注意：** 在 2023 年 7 月之前申请访问权限的 Ads API 开发者，可能拥有不同级别的访问权限和权限设置，并且可能被限制为最多五个 OAuth 令牌。请参阅我们的[提高访问权限](https://developer.x.com/en/docs/twitter-ads-api/increasing-access)指南，以获取更多端点的访问权限，或为现有应用解除令牌数量限制。</Note>

<div id="make-your-first-request">
  ## 发起你的第一个请求
</div>

要测试你对 Ads API 的访问权限，请向 [GET accounts](https://developer.x.com/en/docs/twitter-ads-api/campaign-management/api-reference/accounts) 端点发送请求。该端点会返回当前已授权用户可访问的广告账户。请在后续的 API 请求中使用本次请求返回的广告账户 ID，以读取和写入特定广告账户的数据。在命令行中使用 [Twurl](https://developer.x.com/en/docs/tutorials/using-twurl) 时，请求如下所示。

**示例请求**

```bash
twurl -H ads-api.x.com "/11/accounts"
```

**示例响应**

