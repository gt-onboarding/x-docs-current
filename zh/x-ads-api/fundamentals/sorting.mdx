大多数 Ads API 集合端点（返回对象列表的端点）都支持排序。根据不同的端点，可以使用不同的参数进行排序。通过这些参数返回的大多数字段（ID 和枚举类型字段除外）都可以参与排序。

| API 集合 | 可排序参数 |
| :--- | :--- |
| [Accounts](/zh/x-ads-api/campaign-management#accounts) | `created_at`, `updated_at`, `deleted`, `name` |
| [Funding Instruments](/zh/x-ads-api/campaign-management#funding-instruments) | `created_at`, `updated_at`, `deleted`, `funded_amount_local_micro`, `start_time`, `end_time` |
| [Campaigns](/zh/x-ads-api/campaign-management#campaigns) | `created_at`, `updated_at`, `deleted`, `name`, `start_time`, `end_time`, `daily_budget_amount_local_micro`, `total_budget_amount_local_micro`, `standard_delivery` |
| [Line Items](/zh/x-ads-api/campaign-management#line-items) | `created_at`, `updated_at`, `deleted`, `bid_amount_local_micro` |
| [Cards](https://developer.x.com/en/docs/ads/creatives/api-reference/image-app-download) | `created_at`, `updated_at`, `deleted`, `name` |
| [Promoted Accounts](/zh/x-ads-api/campaign-management#advertiser-api) | `created_at`, `updated_at`, `deleted`, `paused` |
| [Promoted Tweets](/zh/x-ads-api/campaign-management#promoted-tweets) | `created_at`, `updated_at`, `deleted`, `paused` |
| [App Event Tags](/zh/x-ads-api/measurement/mobile-conversions#app-event-tags) | `created_at`, `updated_at`, `deleted`, `post_view_attribution_window`, `post_engagement_attribution_window`, `assisted_conversion`, `provider_app_event_name` |

<div id="getting-started">
  ## 入门
</div>

在你的 API 请求中，只需附加 `sort_by=[attribute name]-[asc/desc]`，其中 attribute 是指在你的 GET 请求返回的 JSON 中包含的有效属性。

例如，你可以使用 `?name-asc` 按名称进行升序（字母顺序）排序，或使用 `?name-desc` 按名称进行降序（逆字母顺序）排序。
