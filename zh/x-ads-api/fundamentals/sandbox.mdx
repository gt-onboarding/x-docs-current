沙盒是一个用于测试和构建你的 Ads API 集成的环境。

通过沙盒创建的广告系列不会投放给用户，并且可以按需动态创建诸如资金工具（Funding Instrument）等对象的测试记录。

使用沙盒就像调用 ads-api-sandbox.x.com 的相应主机名一样简单，但它也包含一些只在沙盒中提供的独特功能。

首先向 [POST accounts](/zh/x-ads-api/campaign-management#post-accounts) 发起请求以创建新的广告主账户。该调用的响应将包含一个可用于后续请求的 `account_id`。这些端点使调用方可以创建多个广告主账户并针对它们进行测试。

<div id="sandbox-only-endpoint-list">
  ### 仅沙盒可用的端点列表：
</div>

* [POST accounts](/zh/x-ads-api/campaign-management#post-accounts) 在沙盒中创建账户
* [POST accounts/:account&#95;id/features](/zh/x-ads-api/campaign-management#post-accounts-account-id-features) 在沙盒中为账户添加账户功能
* [POST accounts/:account&#95;id/funding&#95;instruments](/zh/x-ads-api/campaign-management#post-accounts-account-id-funding-instruments) 在沙盒中为账户添加资金工具
* [DELETE accounts/:account&#95;id](/zh/x-ads-api/campaign-management#delete-accounts-account-id) 在沙盒中删除账户
* [DELETE accounts/:account&#95;id/features](/zh/x-ads-api/campaign-management#delete-accounts-account-id-features) 在沙盒中删除账户中的账户功能
* [DELETE accounts/:account&#95;id/funding&#95;instruments/:funding&#95;instrument&#95;id](/zh/x-ads-api/campaign-management#delete-accounts-account-id-funding-instruments-funding-instrument-id) 在沙盒中删除账户中的资金工具
