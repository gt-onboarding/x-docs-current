为了支持更高的广告活动数量上限，并高效检索某个账户关联的所有实体，Advertiser API 现在在许多 GET 端点上都支持分页。该分页机制使用简单，与 REST API 的基于游标的分页机制非常相似，详见[使用游标浏览集合](https://developer.x.com/en/docs/x-api/v1/pagination)。

<div id="getting-started">
  ## 入门
</div>

对于特定的 GET 请求，我们现在接受可选的 `cursor` 和 `count` 查询参数。如果某个响应包含的实体多于 `count`，这些端点将返回前 `count` 个实体，并在响应 JSON 中包含一个 `next_cursor` 键。

`GET https://ads-api.x.com/5/accounts/abc1/campaigns?count=50`

```json
{
  "data": \[...\],
  "next_cursor": "c-3yvu1pzhd3i7",
  "request": {...}
}
```

要获取下一批响应结果，你需要在查询参数中添加 `cursor`：

`GET https://ads-api.x.com/5/accounts/abc1/campaigns?cursor=c-3yvu1pzhd3i7&count=50`

```json
{
  "data": \[...\],
  "next_cursor": "c-3w3zdyg8ywan",
  "request": {...}
}
```


<div id="going-deeper">
  ## 深入了解
</div>

对于大多数端点，`count` 的最大值为 **1,000**，最小值为 **1**，默认值为 **200**。

`next_cursor` 提供的值始终是字符串，应视为不透明值；其实现方式可能会发生变化。如果当前结果集页面中返回的实体少于 `count`，则 `next_cursor` 的值将为 `null`。

**注意：**analytics 端点不支持这种类型的分页方式。stats 端点通过指定时间区间来实现分页。有关更多信息，请参阅这些端点各自的文档。
