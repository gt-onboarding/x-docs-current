要在 Ads API 中取得良好效果，了解 X Ads 中各实体之间的关系至关重要。本教程将介绍 X Ads 对象层级结构的基础知识。

![X Ads API 层级结构](/images/adsapi-heirarchy.png.twimg.1920.png)

<div id="ad-group-line-items">
  ## 广告组（line items）
</div>

广告组在 Ads API 中被称为[广告单元（line items）](https://developer.x.com/content/developer-twitter/en/docs/tutorials/ads-api-hierarchy-terminology#line-items)，隶属于广告系列，用于针对一组 X 用户进行定向投放和出价。广告主通过将帖子或媒体（例如作为 In-stream 广告推广的视频）与广告单元关联来进行推广。另请参阅广告组。

<div id="account-media">
  ## 账户媒体
</div>

作为[账户媒体](/zh/x-ads-api/creatives#account-media)上传到某个广告账户的一组创意素材，其中可以包括推广视频的[前贴片素材](/zh/x-ads-api/campaign-management#video-views-preroll-objective)（也称为流内广告），以及可在 [X Audience Platform](https://developer.x.com/content/developer-twitter/en/docs/ads/measurement/overview/twitter-audience-platform) 上投放推广的图片。

<div id="ads-accounts">
  ## 广告账号
</div>

在 X Ads 的层级结构中，最上层是广告账号。一个广告账号可以由多个 X 用户进行管理。我们的演示家具店账号 @Furni 就是一个管理广告账号的用户示例。我们通过 account&#95;id 来标识广告账号，它是一个 [base36](https://en.wikipedia.org/wiki/Base36) 字母数字字符串。

在每个账号下，我们会存储若干种可在不同广告系列之间复用的实体类型，例如可用于定向的自定义受众、账号媒体（视频或图片）、以及卡片（cards），即可以添加到帖子中的扩展创意素材。

<div id="amplify-video">
  ## Amplify 视频
</div>

在上传视频素材时，上传方可以为视频定义一个类别（例如，帖子视频、帖子 GIF、帖子图片和 Amplify 视频）。当你尝试上传要作为流内广告投放的视频时，必须使用 Amplify 视频类别。这类视频需要经过特殊处理以确保可以投放，因此必须将媒体类别指定为 Amplify 视频。

<div id="analytics">
  ## 分析
</div>

分析指标可帮助合作伙伴和广告主了解他们在 X 上推广内容的表现情况。这包括展示次数、点击量、视频观看次数和花费等信息。此外，合作伙伴和广告主还可以获取他们触达受众的各类细分群体的详细指标。

Ads API 支持两种获取详细广告系列表现指标的方式：同步和异步。

通过[同步分析](/zh/x-ads-api/analytics#synchronous-analytics)请求，请求的指标会直接在响应中返回。同步端点支持较短的时间范围，非常适合查询广告系列的实时表现。

通过[异步分析](/zh/x-ads-api/analytics#asynchronous-analytics)端点，请求的指标会在关联“作业”处理完成后，以可下载结果文件的形式提供。异步端点支持跨更长时间范围请求指标，允许返回细分数据，并用于获取大量数据 —— 非常适合用于生成报表或进行历史数据回填。

<div id="campaigns">
  ## Campaigns
</div>

广告主投放的每个 campaign 都必须在现有的 funding instrument 下创建。一个 campaign 用于定义广告投放计划的名称、投放排期和预算等内容。沿用 Furni 的示例，我们可以创建一个名为 “Furni home decor” 的 campaign。

需要注意的是，一个 ads account 最多只能拥有 200 个处于活动状态的 Campaign；该上限可以由广告主的客户经理提升。一个 Campaign 被视为处于活动状态，直到到达其结束时间或其预算耗尽为止。出于 campaign 数量上限的考虑，被暂停的 campaign 仍然视为活动 campaign。

<div id="cards">
  ## Cards
</div>

Cards 提供了一种将帖子以更直观的方式扩展为包含图片、视频或 UI 元素的方式。Ads API 支持多种可用于帖子中的 card 类型，这些帖子随后可以在广告活动中进行推广。

## 创意

广告主可以在包含创意（例如图片、GIF 和视频）的广告系列中推广帖子。这些创意可以与 X 卡片（X cards）关联，使帖子能够附带更丰富的照片、视频和媒体体验。创意也可以直接附加到帖子中。例如，你可以同时包含文本和视频（或最多 3 张图片），并创建一个推广帖子。

每个 line item 至少需要关联一个创意才能开始投放。可用的创意类型有多种，包括用于获取新关注者的推广账号（Promoted Accounts）、适用于 TX Audience Platform 各类 line item 的媒体创意（Media Creatives，如图片或视频），以及推广帖子（Promoted Posts）。

<div id="funding-instruments">
  ## 资金工具
</div>

在广告账户下还有资金工具，即广告主为在 X Ads 中使用而配置的不同资金来源。这些资金工具形式多样，从他们与 X 签署的广告订单（Insertion Order，IO）到绑定到账户的信用卡等。

<div id="ad-group-line-items">
  ## 广告组（line items）
</div>

在一个广告系列中，你可以设置一个或多个广告组，在 Ads API 中也称为 line items。可以将 line items 视为广告系列中的不同部分，每个部分都有自己的一套定向设置和各自的创意素材。包含多个 line items 的广告系列通常被称为“广告组广告系列”（ad group campaigns）。

在单个广告系列中使用多个 line items 的好处之一是，你可以创建更多样化的定向与创意组合。

在我们的“Furni home decor”广告系列下，我们可能希望创建两个 line items：一个可以是“Home buyers in San Francisco”，另一个可以是“Home buyers in London”。

每个广告系列最多允许 100 个 line items，而一个广告账户在所有广告系列中总共最多可以拥有 4,000 个处于启用状态的 line items。

<div id="mobile-app-promotion-map">
  ## 移动应用推广（MAP）
</div>

移动应用推广（MAP）是一套产品方案，使广告主能够通过 X 推广移动应用，其中包括应用安装、转化和再互动。

<div id="mobile-app-conversion-tracking-mact">
  ## 移动应用转化跟踪（MACT）
</div>

<div id="media-library">
  ## 媒体库
</div>

媒体库端点用于管理 X Ads 账户的图像、GIF 和视频。库中的媒体素材可用于帖子，并可用于创建卡片。它们也可以在多个创意中重复使用，从而无需反复上传同一素材。

<div id="promoted-accounts">
  ## 推广帐号
</div>

推广帐号（粉丝推广活动）是“推荐关注”的一部分，“推荐关注”会向用户推荐他们当前尚未关注但可能感兴趣的帐号。推广帐号有助于向用户展示更多他们可能会喜欢的各类帐号。

推广帐号使广告主可以推广其帐号，并在 X 上扩大其受众。粉丝推广活动会在 X 平台的多个位置展示——包括主页时间线、“推荐关注”和搜索结果——这些推荐会标记为“推广”，以便与其他推荐帐号区分开来。

<div id="promotable-users">
  ## 可推广用户
</div>

每个 X Ads 账户可以有一个或多个可推广用户，即可以通过该广告账户进行推广的用户。Furni 的 @Furni 账号就是 Furni 广告账户下一个完整（FULL）可推广用户的示例，这意味着我们可以为 @Furni 及其帖子投放所有类型的广告活动。一个广告账户只能有一个完整（FULL）可推广用户。

请注意，一个广告账户下也可以有其他可推广用户。比如，如果我们想推广 @Jack 发布的关于 Furni 产品的帖子，他将在该账户上成为仅限转帖的可推广用户，用 RETWEETS&#95;ONLY 标识。广告主应联系其 X 客户经理，了解将转帖用户添加到其广告账户的相关信息。

<div id="promoted-video">
  ## 推广视频
</div>

推广视频是指包含在推广帖子中的视频，由广告主付费投放。它们会出现在首页时间线上、X 搜索结果顶部以及平台上的其他位置，并且会被清晰地标记为「推广」。

<div id="scheduled-tweets">
  ## 定时发布的帖子
</div>

定时发布的帖子允许广告主或用户创建一个帖子，并安排在未来的某个时间上线。除了能够创建和管理这些帖子之外，API 还允许将这些帖子关联到某个广告项目（line item），以便在帖子上线后进行推广。这样，广告主就可以预先创建原生帖子，并在任何关键营销活动开始前规划他们的投放素材。例如，预先排期一个帖子的创意，使其在新产品发布公告发布的同时立即上线。

<div id="segmentation">
  ## 细分
</div>

细分使合作伙伴和广告主能够按特定定向维度（例如年龄、设备或语言）拆分并获取指标，该功能仅可通过我们的异步分析端点使用。要请求细分指标，请使用 `segmentation_type` 请求参数。有关细分选项的更多详细信息，请参阅 Metrics and Segmentation。

<div id="tailored-audience-list">
  ## 定向受众 - 列表
</div>

基于列表的定向受众是三种定向受众类型之一，通过上传包含您自有用户和客户数据的文件创建。您的列表会与 X 上的用户进行匹配，使您能够在广告活动中直接定向触达他们。请查看这些限制条件。

<div id="tailored-audience-web">
  ## 定向受众 - Web
</div>

来自 Web 的定向受众（Tailored Audience from web）是三种定向受众类型之一，由曾访问过你网站的 X 用户组成。要使用来自 Web 的定向受众，你必须在自己的网站上安装我们的 X 像素（X pixel），或使用我们认可的标签管理器合作伙伴之一。请查看相关限制条件。

<div id="targeting-criteria">
  ## 定向条件
</div>

如前所述，定向条件与广告项目（line item）相关联。你可以使用多种不同的定向方式，从地域、关键词和性别，到更细粒度的属性，例如用户行为和兴趣。对于名为 “Home Buyers in San Francisco” 的广告项目，我们可以同时根据 “home buying” 行为和 “San Francisco” 地点进行定向。

有关不同可用定向类型的详细信息，请访问我们开发者网站上的 Targeting Overview（定向概览）页面。
