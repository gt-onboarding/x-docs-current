<div id="introduction">
  ## 简介
</div>

Posting With Location 是 X API 中的地理标记功能。此功能通过为帖子添加更多情境信息，帮助为用户提供更有意义的体验。比如，“Fireworks!”（“烟花！”）如果带有位置信息就会更具关联性：“Fireworks!”（“烟花！”），来自加利福尼亚州旧金山市的 SoMa 社区。我们期待与开发者生态系统合作，围绕这一功能打造出色的使用体验！

<div id="user-privacy-and-user-control-of-their-location-information">
  ## 用户隐私以及用户对其位置信息的控制
</div>

为确保良好且安全的使用体验，开发者在处理用户位置信息时应高度重视用户隐私，并在“附带位置发布”功能的体验上保持基本的一致性。以下是一些需要牢记的要点，有助于保护用户隐私，并确保用户理解其操作所带来的影响：

<div id="general-guidelines">
  ## 一般性指南
</div>

* 用户必须主动选择使用“附带位置信息发帖”功能（将位置设为“开启”）。
* 用户必须明确授权，允许其精确位置随帖子一起显示。如果你的客户端上的位置功能是“粘性”的——即一旦位置被开启，它会始终随帖子一同发布——你可以选择仅显示地点名称（place&#95;ids），并允许用户在每条帖子基础上，或通过单独的设置选项，进一步选择是否共享精确的纬度和经度。
* 当你访问用户的位置信息来发布帖子时，除非得到用户的明确许可，你不应在帖子之外另行存储这些信息。
* 必须让用户清楚了解，其帖子会显示何种级别的位置信息（如果会显示的话）。你可以通过在帖子发布前展示一张预览该信息呈现方式的地图来实现这一点。如果在客户端上，由用户发布的信息中包含精确的纬度和经度（或仅包含这类信息），则必须让用户明确知晓其精确坐标将被发布。
* 用户在每次撰写帖子时，都应能够开启或关闭位置信息。
* 你必须遵守 X 的[开发者政策](/zh/developer-terms/policy.html)中的条款——包括以下要求：除作为帖子的一部分外，你不得汇总、缓存或存储来自 X API 的位置信息和其他地理信息；并且不得将此类位置信息或地理信息以独立形式使用。

<div id="specific-examples">
  ## 具体示例
</div>

示例 1：*不添加精确纬度和经度（这就是当前 X Web Client 的界面）*

1. 当用户撰写帖子时，显示“添加你的位置”的链接；
2. 在后台，使用 X 的 [reverse&#95;geocode](https://developer.x.com/en/docs/geo/places-near-location/api-reference/get-geo-reverse_geocode.html) API 调用，并传入纬度、经度、定位精度以及用户首选的粒度（默认是“neighborhood”）；
3. 显示默认地点的名称（X 会尝试以合理的方式对这些地点排序——排序方式使 X 客户端可以直接显示该列表，而无需再做修改）；
4. 允许用户从列表中选择其他位置（例如，从“SoMa, San Francisco”更改为“South Beach, San Francisco”）。
5. 重要：如果你将显示精确位置信息，请对用户保持透明。务必让用户知道是否会显示精确位置信息。
6. 允许用户在每次撰写帖子时选择开启或关闭位置。

示例 2：*已有的地理标记实现*

许多 X 客户端应用已经实现了最初的地理标记功能（只广播经纬度，而不提供 place&#95;ID）。API 的默认行为会模拟这种行为——也就是说，作为开发者，你不需要对现有应用进行任何更改，就可以保持当前已有的行为。实际上，如果你仅向 X 传递一个经纬度坐标，X 会自动对该纬度和经度进行逆地理编码，并在我们有可用数据的区域中，除了经纬度外，还显示 place&#95;ID。

请确保对用户清晰说明，他们的精确坐标正在被使用并在 X 上显示。针对现有应用的最佳实践是允许用户按每条帖子来选择开启或关闭其位置。你也可以让用户仅共享 place&#95;ID，而不是精确坐标，并将其设置为默认且“持续生效”的选项。

<div id="x-storage-of-user-location-data">
  ## X 对用户位置信息的存储
</div>

X 将按如下方式存储由客户端传递的数据：

* X 会保存用户选择向其关注者公开展示的所有数据。这与 X 存储帖子发布时间戳的方式类似，也就是说，如果用户在发帖时包含其精确坐标，X 将在帖子存在期间始终把该精确位置与帖子一同存储。用户可以在“设置”页面点击“清除我的位置历史记录”按钮，如[此处](https://support.twitter.com/articles/122236-how-to-tweet-with-your-location)所述，以清除其帖子中的位置信息。

<div id="summary">
  ## 总结
</div>

从用户角度来看，在帖子中添加位置信息可以显著提升内容的上下文和价值。然而，在你的客户端中实现位置功能时，应谨慎对待隐私和对用户的透明度，并将这些因素作为设计以位置为核心产品时的关键考量。

如果你对隐私、位置以及其他地理位置相关功能有任何疑问或意见，请[联系我们。](https://support.twitter.com/forms/platform)
