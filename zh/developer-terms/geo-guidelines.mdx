<div id="introduction">
  ## 简介
</div>

“Posting With Location” 是 X API 中的地理标记功能。该功能通过让帖子更具上下文信息，帮助用户获得更有意义的体验。比如，“Fireworks!” 这个帖子，如果与某个地点关联起来就更有意义：“Fireworks!” 来自加利福尼亚州旧金山市 SoMa 社区。我们期待与开发者生态系统合作，围绕这一功能打造出色的产品体验！

<div id="user-privacy-and-user-control-of-their-location-information">
  ## 用户隐私与用户对其位置信息的控制
</div>

为确保良好、安全的体验，开发者在处理用户位置信息时应重视用户隐私，并在“附带位置信息发布内容”的体验中提供基本的一致性。以下是一些需要牢记的关键事项，有助于保护用户隐私，并确保用户了解其操作所带来的影响：

<div id="general-guidelines">
  ## 一般准则
</div>

* 用户必须主动选择使用“带位置发布”功能（将位置设置为“开启”）。
* 用户必须明确同意其精确位置可以随帖子一同展示。如果你的客户端上的位置功能是“持续生效”的——即一旦打开位置，它就会始终附加在帖子上——你可以考虑仅展示地点名称（place&#95;ids），并允许用户在每条帖子基础上，或通过单独的替代设置，进一步选择是否显示精确纬度和经度。
* 当你为了发布帖子而访问用户的位置数据时，除非获得用户的明确许可，否则不应在帖子之外存储该信息。
* 必须让用户清楚了解其帖子会展示何种级别的位置信息（如果有的话）。你可以通过在帖子发布前展示一张地图，示意这些信息将如何展示，来实现这一点。如果在客户端上由用户发布的信息中包含精确纬度和经度（或其为唯一发布的信息），则必须明确告知用户其精确坐标将会被发布。
* 用户在每次撰写帖子时，都应该能够开启或关闭其位置。
* 你必须遵守 X 的[开发者政策](/zh/developer-terms/policy.html)——其中包括如下要求：除作为帖子的一部分外，你不得聚合、缓存或存储来自 X API 的位置数据和其他地理信息，也不得单独使用此类位置数据或地理信息。

<div id="specific-examples">
  ## 具体示例
</div>

示例 1：*不添加明确的纬度和经度（这是当前 X Web 客户端界面）*

1. 当用户撰写帖子时，显示“添加你的位置信息”的链接；
2. 在后台调用 X 的 [reverse&#95;geocode](https://developer.x.com/en/docs/geo/places-near-location/api-reference/get-geo-reverse_geocode.html) API，并传入纬度、经度、你定位结果的精度，以及用户偏好的粒度（默认是“街区”级别，即“neighborhood”）；
3. 显示默认地点的名称（X 会尝试以合理的方式对这些地点进行排序——排序方式使得 X 客户端可以直接显示该列表，而无需再作调整）；
4. 允许用户从列表中选择不同的位置（例如，从“SoMa, San Francisco”更改为“South Beach, San Francisco”）。
5. 重要提示：如果你将显示精确位置信息，请对用户保持透明。确保用户知晓是否会显示精确位置信息。
6. 允许用户在每次撰写帖子时选择是否开启或关闭位置。

示例 2：*现有地理标记实现*

许多 X 客户端应用已经实现了最初的地理标记功能（该功能只广播经纬度，而不提供 place&#95;ID）。API 的默认行为会模仿这种行为——也就是说，作为开发者，你无需对现有应用做任何更改即可保持当前的行为。实际上，如果你仅向 X 传入一个经纬度坐标，X 会自动对该纬度和经度进行反向地理编码，并在我们有可用数据的区域中，在经纬度之外同时显示 place&#95;ID。

请确保对你的用户保持透明，告知他们的精确坐标正在被使用并在 X 上显示。对于现有应用，一个最佳实践是允许用户按每条帖子决定是否开启或关闭位置。你也可以让用户启用一个默认的、“粘性”的设置，仅分享 place&#95;ID，而不分享精确坐标。

<div id="x-storage-of-user-location-data">
  ## X 对用户位置信息的存储
</div>

X 将按如下方式存储由客户端传入的数据：

* X 会保存用户选择向其关注者公开展示的所有数据。这与 X 存储表示帖子发布时间的时间戳的方式类似，这意味着如果用户在发帖时附带其精确坐标，X 会在该帖子存在期间始终将这一精确位置与该帖子一并存储。用户可以通过点击“设置”页面上的“清除我的位置历史”按钮来清除其帖子中的位置信息，具体说明见[此处](https://support.twitter.com/articles/122236-how-to-tweet-with-your-location))。

<div id="summary">
  ## 摘要
</div>

从用户的角度来看，在帖子中添加位置信息能够极大地丰富上下文并提升价值。然而，在你的客户端中实现位置功能时，应将隐私保护和对用户的透明度作为设计位置相关产品时的首要考量。

如果你对隐私、位置或其他地理相关功能有任何问题或意见，请[联系我们。](https://support.twitter.com/forms/platform)
