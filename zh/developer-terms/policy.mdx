<div id="x-developers">
  ## X + 开发者
</div>

X 热爱开发者。我们对这个社区利用 X 数据力量所打造的工具和服务倍感欣喜与惊叹。作为对这一社区承诺的一部分，我们的目标是为开发者提供开放且公平的数据访问，同时保障 X 上人们的安全，并让整个 X 平台受益。为进一步实现这些目标，我们制定了本开发者政策，作为指南，帮助人们理解我们关于适当使用 API 和 X 内容的规则和期望。

本开发者政策（“**政策**”）为与 X 的应用、服务、网站、网页和内容生态系统进行互动的开发者提供规则和指引。它构成您与 X 之间关于访问和使用 X API 和 X 内容的合同的一部分（无论是作为开发者协议的一部分，还是您与 X 之间的其他书面协议）。违反本政策将被视为违反您的协议。本政策可不时更新且无需另行通知。本政策中使用但未在本政策中定义的首字母大写术语，其各自含义以[开发者协议](/zh/developer-terms/agreement)或主许可协议中的定义为准。

<div id="using-this-policy">
  ## 使用本政策
</div>

我们对本政策进行了结构化设计，力求便于你查阅和遵循。请在使用 X API 和 X 内容时，始终牢记以下各政策章节中的信息：

1. **为成功打好基础** - 你有责任遵守所有 X 政策。在访问 X API 和 X 内容之前，务必审阅并理解本政策，以及我们在本文件中链接到的其他政策。花时间阅读我们的政策，可能会为你今后节省数小时的返工时间。

2. **隐私与控制至关重要** - 保护和维护在 X 上人们的隐私深深植根于我们的公司基因之中。因此，我们禁止以任何与人们对隐私的合理预期不一致的方式使用 X 数据。通过基于 X API 搭建产品或访问 X 内容，你在维护这一承诺方面承担着特殊责任，而最重要的就是尊重人们的隐私，并向他们提供关于其数据如何被使用的透明度和控制权。

3. **遵守平台使用指南** - 获批访问 X API 和 X 内容只是第一步。无论何时你对如何在计划使用 X 平台时确保政策合规定有疑问，都应首先查阅我们的平台使用指南。

我们在下文对上述三个关键章节分别作了更为详细的说明。请仔细阅读，确保你对 X API 和 X 内容的使用与我们的政策保持一致。

如果我们认为你违反了本政策（或任何其他 X 政策），我们可能会暂停或永久撤销你对 X API 和 X 内容的访问权限。

最后，请注意，X 可能会监控你对 X API 的使用情况，以改进 X 应用程序、评估任何商业用途，并确保你遵守获批用例和本政策。

感谢你的阅读，也感谢你在我们的平台上进行构建！我们期待看到你创造的成果！

<div id="set-yourself-up-for-success">
  ## 为成功打好基础
</div>

通过确保你的服务从第一天起就以正确的方式构建，你可以在使用 X API 时避免许多潜在陷阱。本《开发者政策》的本节包含所有开发者在使用 X API 或 X 内容之前必须遵守的规则。

Free、Basic 和 Pro 方案（如 [developer.x.com/en](http://developer.x.com/en) 上所述）旨在面向兴趣爱好者、商业原型开发、早期阶段的 X 产品集成，以及面向终端用户数量有限的应用。如果你对 X API 和 X 内容的使用超出了这一范围，则必须申请（或已经订阅）Enterprise 方案（如 [developer.x.com/en](http://developer.x.com/en) 上所述）。

**我们会审核所有拟在 X 开发者平台上的使用方式，以验证其是否遵守各项政策——因此，你必须披露（并在适用时更新）你计划如何使用 X API 和 X 内容，方可获得并维持访问权限。** 所有新开发者必须[申请开发者账号](https://developer.x.com/portal/petition/essential/basic-info)才能访问 X API。当前尚未获得批准开发者账号的开发者，必须在 X 要求时按指示提交申请。作为该流程的一部分，你需要向我们提供一份书面说明，描述你计划如何使用 X API 和 X 内容。

你的用例说明对你具有约束力，对其进行任何实质性偏离都可能构成对我们规则的违反，并导致执行措施。你必须在对用例进行任何实质性修改时通知我们，并在开始为该新目的使用 X 内容之前获得批准。未履行该义务可能导致你的 API 和数据访问被暂停或终止。

**在基于 X API 构建或访问 X 内容时，你必须遵守所有 X 政策。** 其中包括本[开发者政策](/zh/developer-terms/policy)、[自动化规则](https://help.x.com/rules-and-policies/x-automation)、[展示要求](https://developer.x.com/developer-terms/display-requirements.html)、[API 限制用途规则](/zh/developer-terms/restricted-use-cases)、[X 规则](https://help.x.com/rules-and-policies/x-rules)和 [X 品牌资源](https://about.x.com/who-we-are/brand-toolkit)，以及你与 X 就使用 X API 或 X 内容签订的任何其他协议（包括但不限于[开发者协议](/zh/developer-terms/agreement)或主许可协议或订单（视情况而定））。你还必须遵守这些政策的任何修订内容及 X 推出的任何新政策。你有责任监控你的服务使用情况，并在服务设计中采取措施，防止使用你服务的人违反 X 政策。未履行该义务可能导致你的 API 和 X 内容访问被暂停或终止。

**你不得为单一用例或实质上相同或重叠的用例注册多个应用。** 在此情境中，“用例”是指通过某个应用执行的一组一致的分析、展示或操作。不允许提供工具或服务的“白标”版本。如果你计划对你的应用提供“白标”版本，必须通知我们并分别获得我们的批准。

*作为对这些规则的唯一例外，* 你可以为同一服务的开发、预发布和生产实例创建并使用最多 3 个应用。这些应用必须注册在同一个账号下，并应在名称和描述中清晰标识为单一服务的 dev、staging 和 prod 实例。**你不得将开发或预发布应用用于生产目的。**

**你必须对所有 API 密钥或其他访问凭证保密。** 你不得使用，也不得鼓励或促使他人使用属于他人的 API 密钥或其他访问凭证。

**你与 X 的许可协议会限制你对 X API 和 X 内容的使用。除其他事项外，X API 设置了速率限制，以帮助确保公平的数据使用，并打击平台上的垃圾信息。** 你不得超出或规避速率限制，或本政策或你与 X 的协议中描述、列于[开发者网站](https://developer.x.com/)上、或由 X 通知你的任何其他限制或约束。

**你不得删除或更改通过 X API 收到的 X 内容上的任何专有声明或标记。** 这有助于确保人们知道 X 内容的来源以及其归属。

**出于数据完整性和平台健康的原因，你不得干扰、拦截、破坏或禁用 X API 或 X 服务的任何功能。** 换言之，请按 [developer.x.com](https://developer.x.com/) 上预期的方式并依照文档说明使用这些 API。有关可接受使用方式的更多详细信息，请参阅我们的 [HackerOne 指南](https://hackerone.com/x)。

<div id="privacy-and-control-are-essential">
  ## 隐私与控制至关重要
</div>

X 高度重视隐私，我们也期望所有使用 X 内容和 X API 的人同样重视隐私。任何以违背人们对隐私的合理预期的方式使用 X 开发者平台、X API 或 X 内容的行为，都可能会受到相应的处罚措施，包括暂停或终止对 API 和 X 内容的访问权限。

你对隐私和控制的承诺必须体现在对 X 内容的所有使用场景，以及你使用我们的 API 构建的服务的各个方面。为此，使用你服务的人必须理解并同意你如何使用他们的数据，以及你如何代表他们访问 X。你可以通过向人们提供清晰、全面且透明的隐私政策来实现这一点，并确保在代表每位使用你服务的人采取任何行动之前，获得他们明确且知情的同意。**请注意，仅仅是某个人通过你的服务完成身份验证，并不当然构成其已同意。**

<div id="consent-permissions">
  ## 同意与权限
</div>

**特别是，在执行以下任何操作之前，你必须事先获得用户的明确、知情同意：**

* 代表他们执行*任何*操作。这包括（但不限于）：
  * 在 X 上发布内容
  * 关注/取消关注账户
  * 修改个人资料或账户信息
  * 向帖子添加标签或任何其他内容

* 重新发布通过非 X API 或非其他 X 工具途径获取的内容

* 使用他人的 X 内容来推广某个产品或服务

* 存储非公开内容，例如私信（DM），或任何其他私人或机密信息

* 分享或发布受保护内容，或任何其他私人或机密信息

**如果你的服务允许人们在 X 上发布内容，你必须在发布前做到以下几点：**

* 准确展示将要发布的内容
* 明确告知使用你服务的人，将向内容添加哪些地理位置信息（如果有）

**如果你的服务允许人们同时向你的服务和 X 发布内容，你必须在发布前做到以下几点：**

* 获得发布该内容的许可
* 说明你将把内容发布到哪些平台或渠道

**你必须尊重所有 X 内容的受保护状态和被屏蔽状态。你不得将使用某个人的身份验证令牌获取的内容提供给未被授权查看该内容的其他人。**

* **受保护账户：**[受保护账户](https://help.x.com/safety-and-security/public-and-protected-posts)的内容仅对已获账户所有者批准关注该账户的人可见。因此，如果你运营的服务会访问受保护账户，你只能这样做，以便仅向被授权查看该内容的特定人群提供此类内容。
* **被屏蔽账户：**X 上的用户可以基于其自行选择的任何理由[屏蔽](https://help.x.com/using-x/blocking-and-unblocking-accounts)对其账户的访问。将通过令牌（或任何其他基于 API 的操作）获取的信息与其他信息混用以绕过这一选择是不被允许的。

**由于私信（DM）本质上是非公开的，提供 DM 功能的服务必须采取额外措施来保护个人隐私。你不得向未被授权查看该内容的人提供 DM 内容。如果你的服务提供 DM 功能，你还必须：**

* 在你为 DM 发送已读回执事件时告知用户。你可以通过在你的服务中直接提供通知，或显示会话中其他参与者的已读回执来实现这一点。
* 在将媒体配置为在 DM 中以“共享”形式发送（即可在多个 DM 中重复使用）之前获得同意。如果你确实允许 DM 中的媒体被“共享”，你必须提供清晰的通知，说明任何持有该媒体 URL 的人都可以访问此内容。

<div id="content-compliance">
  ## 内容合规
</div>

**如果你在脱机环境中存储 X 内容，必须确保其与该内容在 X 上的当前状态保持一致。** 具体来说，如果某条内容在 X 上被删除或修改，你必须删除或修改你所持有的相应内容。你必须在合理可行的情况下尽快完成此操作，或在收到 X 或相关 X 账号拥有人提出此类请求后的 24 小时内完成，或按照你与 X 的协议或适用法律的其他要求执行。除非法律禁止你这样做，且仅在获得 X 明确的书面许可的前提下，方可不履行上述义务。

已被修改的内容可以有多种形式。这包括（但不限于）：

* 已被设为私密或转为受保护状态的内容
* 已在平台上被暂停的内容
* 已被移除地理标签的内容
* 已在 X 上被限制展示或移除的内容

<div id="off-x-matching">
  ## 站外匹配
</div>

**我们会限制你在何种情况下可以将 X 上的个人与在 X 之外获取或存储的信息进行匹配。** 站外匹配是指将 X 内容（包括在 X 上的 @handle 或用户 id）与某个人、家庭、设备、浏览器或其他站外标识符进行关联。只有在你事先获得该个人明示的选择加入（opt-in）同意，或符合下文所述情形时，你才可以进行此类匹配。

在你没有获得某人明示的选择加入同意以将其 X 身份与某个站外标识符关联的情况下，我们要求你建立的任何关联必须仅基于他人合理预期会被用于此目的的信息。此外，在没有某人明示选择加入同意的前提下，你只能基于以下信息，尝试将你所持有的有关某人的记录与一个 X 身份进行匹配：

* **该人直接提供给你的信息。** 请注意，你与之没有任何既存关系的个人的记录（包括从第三方获取的个人数据）不符合这一标准；和/或
* **公共数据。** 在此语境下，“公共数据”是指：
  * 你从公开、普遍可访问的资源中获得的关于某人的信息（例如某专业协会成员名录）
  * 关于某人的、在 X 上公开可用的信息，包括：
    * 帖子
    * 个人资料信息，包括账号简介和公开声明的位置
    * 显示名称和 @handle

<div id="your-privacy-policy">
  ## 您的隐私政策
</div>

**在允许用户下载、安装或注册使用您的服务之前，您必须向他们展示您服务的隐私政策。** 该隐私政策至少应披露以下信息：

* 您从使用您服务的用户那里收集的信息
* 您如何使用和共享这些信息（包括与 X 共享）
* 用户如何就其信息相关的咨询和请求与您联系

您的隐私政策必须符合所有适用法律，并且对用户的保护程度不得低于 [X 的隐私政策](https://x.com/privacy) 和 [我们其他服务及公司关联方的隐私政策](https://help.x.com/rules-and-policies/x-services-and-corporate-affiliates)。如果您无法遵守您自身的隐私政策和/或 X 的隐私政策，您必须终止对 X API 的访问并停止使用所有 X 内容。

<div id="using-geo-data">
  ## 使用地理数据
</div>

**由于此类信息的敏感性，使用地理数据会受到额外限制。** 如果你的服务向帖子添加位置信息，你必须向用户披露：

* 你在何时添加位置信息
* 你是以地理标签（geotag）还是注释数据（annotations data）的形式添加位置信息
* 你的位置信息是以地点的形式列出，还是以地理坐标的形式列出

如果你的应用允许人们在发布内容时附带其位置信息，你必须完整遵守 X 的[地理指南](/zh/developer-terms/geo-guidelines)。

**严禁将位置信息或地理信息单独使用。** 除非作为帖子的一部分，你不得（也不得允许他人）存储、聚合或缓存 X 内容中包含的位置信息和其他地理信息。比如，你不得将位置信息或地理信息从帖子中分离出来，以展示个人在一段时间内去过哪些地方。展示聚合地理活动的热力图和相关工具是被允许的（例如，显示在某个城市中使用某个话题标签的人数）。

<div id="x-passwords">
  ## X 密码
</div>

**不得存储 X 密码，也不得要求用户直接向你提供其 X 密码、账户凭证或开发者应用信息（包括 consumer key）。** 我们建议使用 [使用 X 登录](/zh/fundamentals/authentication/guides/log-in-with-x) 作为身份验证工具，将你的服务与 X 上的用户关联起来。

<div id="platform-usage-guidelines">
  ## 平台使用指南
</div>

你是否已经认真审阅了 X 的各项政策，并以正确的方式配置了你的 API 访问权限？你的服务是否遵循了 X 的隐私和控制相关指南？如果你对这两个问题的回答都是“是”，那么你就已经准备好开始使用 X API 和 X 内容了。X 的平台使用指南提供所需的指导，帮助确保你对 X 内容的使用从第一天起直至你的服务整个生命周期内都保持合规。我们建议你定期审阅这些规则，以确保你的集成以对 X 上的人以及整个 X 平台都安全且有益的方式运行。

<div id="spam-bots-and-automation">
  ## 垃圾信息、机器人与自动化
</div>

**禁止使用 X API 和开发者产品来制造垃圾信息，或从事任何形式的平台操纵行为。** 你应当查看 X 关于[平台操纵与垃圾信息](https://help.x.com/rules-and-policies/platform-manipulation)的规则，并确保你的服务本身不会、也不会使他人能够违反我们的政策。

**执行写入操作的服务（包括发布帖子、关注帐号或发送私信）必须遵守[自动化规则](https://help.x.com/rules-and-policies/x-automation)。** 特别是，你应当：

* 在向他人发送[自动回复或私信](https://twittercommunity.com/t/policy-clarification-automated-replies-and-mentions/94444)之前，始终获得其明确同意
* 一旦对方请求不再被你联系，应立即尊重其选择
* 切勿执行批量、激进或类似垃圾信息的操作，包括[批量关注](https://twittercommunity.com/t/policy-clarification-aggressive-following-and-inorganic-following-behavior/92769)
* 切勿在[多个帐号](https://blog.twitter.com/developer/en_us/topics/tips/2018/automation-and-the-use-of-multiple-accounts.html)上发布完全相同或实质相似的内容

**如果你运营的是基于 API 的机器人帐号，你必须清楚标明该帐号的性质以及由谁负责。** 你绝不可误导或混淆他人，让他们不清楚你的帐号是否为机器人。一个好的做法是在个人简介中加入说明，表明该帐号是机器人。

<div id="x-performance-benchmarking">
  ## X 性能基准测试
</div>

**你不得为基准测试、竞争或商业目的使用 X API 来衡量 X 的可用性、性能、功能或使用情况。** 例如，你绝不能使用 X API 来：

* 计算 X 的汇总指标，例如每月活跃用户总数（MAs）或每日活跃用户总数（DAs）
* 计算 X 帖子的汇总指标，例如每天发布的帖子总数，或账号互动次数
* 衡量或分析 X 的响应速度
* 衡量或分析 X 上的垃圾信息或安全状况，但下文允许的情形除外

**我们支持有助于改善 X 上对话健康度的研究。** 未经 X 的明确书面许可，你不得公开披露任何与使用 X API 或 X 内容相关的研究或发现，也不得开发、创建或提供使用 X API 或 X 内容来衡量、分析或试图识别违反 X 政策的行为或内容的服务。

**DSA 研究人员：** 如果你需要就《数字服务法案》第 40 条项下的访问事宜联系 X，请发送邮件至 [**EU-Questions@X.com**](mailto:EU-Questions@X.com)。如果你希望申请研究人员访问权限，请提交[申请](https://forms.gle/btDwenPF7M3hgSvw7)。
