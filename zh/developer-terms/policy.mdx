<div id="x-developers">
  ## X + 开发者
</div>

X 热爱开发者。这个社区利用 X 数据的强大力量创造出的工具和服务让我们倍感欣喜与惊叹。作为我们对这个社区承诺的一部分，我们致力于为开发者提供开放且公平的数据访问，同时保障 X 上人们的安全，并使整个 X 平台整体受益。为实现这些目标，我们制定了《开发者政策》，作为指南帮助大家理解我们关于适当使用 API 和 X 内容的规则和期望。

本《开发者政策》（“**政策**”）为与 X 的应用、服务、网站、网页及内容所构成的生态系统进行交互的开发者提供规则和指引。它构成你与 X 之间合同的一部分，用于规范你对 X API 和 X 内容的访问和使用（无论是作为《开发者协议》的一部分，还是与你与 X 签订的其他书面协议的一部分）。违反本政策将被视为违反你的协议。本政策可能会不时予以修改，恕不另行通知。本政策中使用但未作定义的首字母大写术语，其各自含义应与[《开发者协议》](/zh/developer-terms/agreement)或《主许可协议》中赋予该等术语的含义相同。

<div id="using-this-policy">
  ## 使用本政策
</div>

我们对本政策进行了结构化编排，以便尽可能易于理解和遵循。在使用 X API 和 X 内容时，请始终牢记本政策中以下部分的信息：

1. **为成功打好基础** - 你有责任遵守所有 X 政策。在访问 X API 和 X 内容之前，请务必审阅并理解本政策，以及我们在本文件中链接到的其他政策。花时间审阅我们的政策，可能会为你节省后续数小时的返工时间。

2. **隐私和控制至关重要** - 保护和捍卫在 X 上人们的隐私是我们公司核心基因的一部分。因此，我们禁止以任何不符合人们对隐私的合理预期的方式使用 X 数据。当你基于 X API 构建产品或访问 X 内容时，你在履行这一承诺方面扮演着特殊角色，其中最重要的是尊重人们的隐私，并为他们提供关于其数据如何被使用的透明度和控制权。

3. **遵守平台使用指南** - 获得访问 X API 和 X 内容的批准只是第一步。每当你对如何确保你计划使用 X 平台的行为符合政策要求有疑问时，我们的平台使用指南都应当是你首要参考的资源。

下面我们对这三个关键部分各自的含义提供了更为详细的说明。请仔细阅读，以确保你对 X API 和 X 内容的使用符合我们的各项政策。

如果我们认为你违反了本政策（或任何其他 X 政策），我们可能会暂停或永久撤销你对 X API 和 X 内容的访问权限。

最后，请注意，X 可能会监控你对 X API 的使用，以改进 X 应用、审查任何商业使用情况，并确保你遵守已获批准的使用场景以及本政策。

感谢你的阅读，也感谢你与我们一起构建！我们期待看到你创造的成果！

<div id="set-yourself-up-for-success">
  ## 为成功打好基础
</div>

在使用 X API 时，通过确保你的服务从第一天起就以正确的方式构建，可以避免许多潜在的陷阱。本《开发者政策》的本节包含所有开发者在使用 X API 或 X 内容之前必须遵守的规则。

Free、Basic 和 Pro 方案（如 [developer.x.com/en](http://developer.x.com/en) 所述）专为业余爱好者、商业原型开发、早期阶段的 X 产品集成，以及面向数量有限终端用户的应用而设计。如果你对 X API 和 X 内容的使用超出了这一范围，则必须申请（或已订阅）Enterprise 方案（如 [developer.x.com/en](http://developer.x.com/en) 所述）。

**我们会审核所有拟定的 X 开发者平台使用方式，以核实其是否符合政策要求——因此，你必须披露（并在适用时更新）你计划如何使用 X API 和 X 内容，才能获得并维持访问权限。** 所有新开发者必须[申请开发者账号](https://developer.x.com/portal/petition/essential/basic-info)才能访问 X API。当前尚未获批开发者账号的现有开发者，必须在 X 要求时进行申请。作为该流程的一部分，你需要向我们提供一份书面说明，描述你打算如何使用 X API 和 X 内容。

你的用例说明对你具有约束力，对其进行任何实质性偏离都可能构成对我们规则的违反，并导致执行措施。你必须就任何对用例的实质性修改通知我们，并在开始将 X 内容用于该新的目的之前获得批准。未能做到这一点，可能导致你的 API 和数据访问被暂停或终止。

**在基于 X API 构建或访问 X 内容时，你必须遵守所有 X 政策。** 这些政策包括本[《开发者政策》](/zh/developer-terms/policy)、[《自动化规则》](https://help.x.com/rules-and-policies/x-automation)、[《展示要求》](https://developer.x.com/developer-terms/display-requirements.html)、[《API 限制用途规则》](/zh/developer-terms/restricted-use-cases)、[《X 规则》](https://help.x.com/rules-and-policies/x-rules)以及 [X 品牌资源](https://about.x.com/who-we-are/brand-toolkit)，还包括你与 X 就你对 X API 或 X 内容的使用而签署的任何其他协议（包括但不限于[《开发者协议》](/zh/developer-terms/agreement)或主许可协议或订单（视情况而定））。你还必须遵守对这些政策所作的任何修改，以及 X 推出的任何新政策。你有责任监控你的服务的使用情况，并在设计服务时防止使用你服务的人违反 X 政策。未能做到这一点，可能导致你的 API 和 X 内容访问被暂停或终止。

**你不得为单一用例或实质相似或重叠的用例注册多个应用。** 在此情境下，“用例”是指通过某一应用执行的一致的一组分析、展示或操作。不允许提供工具或服务的“白标”版本。如果你计划提供你的应用的“白标”版本，必须通知我们并单独取得我们的批准。

*作为对这些规则的唯一例外，* 你可以为同一服务的开发、预发布和生产实例最多创建并使用 3 个应用。这些应用必须注册在同一个账号下，并且应在名称和描述中清晰标明其为同一服务的 dev、staging 和 prod 实例。**你不得将开发或预发布应用用于生产目的。**

**你必须对所有 API 密钥或其他访问凭证保密。** 你不得使用，也不得鼓励或促使他人使用由他人拥有的 API 密钥或其他访问凭证。

**你与 X 之间的许可协议会限制你对 X API 和 X 内容的使用。除其他事项外，X API 设有速率限制，以帮助确保公平使用数据并打击平台上的垃圾信息。** 你不得超出或规避速率限制，或本政策、你与 X 的协议、[开发者网站](https://developer.x.com/)上所列，或 X 向你传达的任何其他限制或约束。

**你不得移除或更改通过 X API 接收的 X 内容上的任何专有声明或标记。** 这有助于确保人们了解 X 内容的来源以及其归属。

**出于数据完整性和平台健康的考虑，你不得干扰、拦截、破坏或禁用 X API 或 X 服务的任何功能。** 换句话说，请按照 [developer.x.com](https://developer.x.com/) 上所预期和文档说明的方式使用这些 API。有关可接受使用的更多详情，请参阅我们的 [HackerOne 指南](https://hackerone.com/x)。

<div id="privacy-and-control-are-essential">
  ## 隐私与控制至关重要
</div>

X 高度重视隐私，我们也期望所有使用 X 内容和 X API 的人同样如此。任何以违背人们对隐私的合理预期的方式使用 X 开发者平台、X API 或 X 内容的行为，都可能会受到相关执行措施，包括暂停或终止对 API 和 X 内容的访问。

你对隐私与控制的承诺必须体现在对所有 X 内容的使用，以及你利用我们的 API 构建的服务的各个方面。为此，使用你服务的人必须了解并同意你如何使用他们的数据，以及你如何代表他们访问 X。你可以通过向用户提供清晰、全面且透明的隐私政策来实现这一点，并确保在代表每位用户采取任何行动之前，都已获得他们的明确且知情的同意。**请注意，仅仅是某人完成身份验证并登录你的服务，本身并不构成同意。**

<div id="consent-permissions">
  ## 同意与权限
</div>

**特别是，在执行以下任何操作之前，你必须获得相关人员的明确且知情的同意：**

* 以他们的名义执行*任何*操作。这包括（但不限于）：
  * 向 X 发布内容
  * 关注/取消关注帐号
  * 修改个人资料或帐号信息
  * 在帖子中添加标签或任何其他内容

* 以非 X API 或其他 X 工具的方式获取内容并重新发布

* 使用他人的 X 内容来推广产品或服务

* 存储非公开内容，例如私信（DM），或任何其他私人或机密信息

* 分享或发布受保护的内容，或任何其他私人或机密信息

**如果你的服务允许人们向 X 发布内容，你在发布之前必须做到以下几点：**

* 准确展示将要发布的内容
* 清楚告知使用你服务的人，将向内容添加哪些地理位置信息（如果有）

**如果你的服务允许人们同时向你的服务和 X 发布内容，你在发布之前必须做到以下几点：**

* 获得发布该内容的许可
* 说明你将在哪里发布该内容

**你必须尊重所有 X 内容的受保护和被屏蔽状态。你不得将使用某人的认证令牌获取的内容提供给未被授权查看该内容的其他人。**

* **受保护帐号：**[受保护帐号](https://help.x.com/safety-and-security/public-and-protected-posts)的内容仅对已获帐号所有者批准关注该帐号的人可见。因此，如果你运营的服务会访问受保护帐号，你只能为了向有权限查看该内容的特定用户提供此类内容而访问这些帐号。
* **被屏蔽帐号：**X 上的用户可以出于其选择的任何原因[屏蔽](https://help.x.com/using-x/blocking-and-unblocking-accounts)对其帐号的访问。将通过令牌（或任何其他基于 API 的操作）获得的信息混合或合并使用，以绕过这一选择，是不被允许的。

**由于私信（DM）本质上是非公开的，提供 DM 功能的服务必须采取额外措施来保护个人隐私。你不得向未被授权查看该内容的人提供 DM 内容。如果你的服务提供 DM 功能，你还必须：**

* 如果你为 DM 发送已读回执事件，你必须通知用户。你可以通过在服务中直接提供通知，或通过显示会话中其他参与者的已读回执来实现这一点。
* 在将媒体配置为在 DM 中以“共享”（“shared”，即可在多个 DM 之间重复使用）方式发送之前，先获得同意。如果你确实允许 DM 中的媒体被“共享”，你必须明确告知该内容将对任何拥有该媒体 URL 的人可见。

<div id="content-compliance">
  ## 内容合规
</div>

**如果你在离线存储 X 内容，你必须确保其与该内容在 X 上的当前状态保持一致。** 具体而言，如果某条内容在 X 上被删除或修改，你必须相应地删除或修改你所持有的该内容。此操作应在合理可行的情况下尽快完成，或在收到 X 或相关 X 账号所有者提出的请求后 24 小时内完成，或按照你与 X 的协议或适用法律的相关要求执行。除非适用法律禁止进行此类删除或修改，并且仅在获得 X 明确的书面许可的情况下，方可不履行上述义务。

被修改的内容可以有多种形式。这包括（但不限于）：

* 已被设为私密或获得受保护状态的内容
* 已在平台上被暂停的内容
* 已被移除地理标签的内容
* 已在 X 上被限制展示或移除的内容

<div id="off-x-matching">
  ## 站外匹配（Off-X matching）
</div>

**我们会限制你在何种情况下可以将 X 上的个人与在 X 之外获取或存储的信息进行匹配。** 站外匹配是指将 X Content（包括 X 的 @handle 或 user ID）与某个人、家庭、设备、浏览器或其他站外标识符关联起来。只有在你在进行此类关联之前，已从该人处获得明示的主动选择同意（express opt-in consent），或符合下文所述情形时，你才可以进行此类匹配。

在你未获得某人明示的主动选择同意，以将其在 X 上的身份（X identity）链接到某个站外标识符的情况下，我们要求你建立的任何关联仅能基于他人可以合理预期会被用于该目的的信息。此外，在缺乏该人明示的主动选择同意时，你只能基于以下内容，尝试将你记录中关于某人的信息与其在 X 上的身份进行匹配：

* **由该人直接提供给你的信息。** 请注意，你与之不存在任何既往关系的个人的记录（包括从第三方获取的个人数据）不符合这一标准；和/或
* **公开数据。** 在此语境下，“公开数据”是指：
  * 你从公共、普遍可获取的资源中获得的关于某人的信息（例如某专业协会成员名录）
  * 关于某人的、在 X 上公开可见的信息，包括：
    * 帖子
    * 个人资料信息，包括账号简介（bio）以及公开声明的所在地
    * 显示名称和 @handle

<div id="your-privacy-policy">
  ## 你的隐私政策
</div>

**在允许用户下载、安装或注册使用你的服务之前，你必须向其提供并展示你的服务隐私政策。** 该隐私政策至少必须披露以下信息：

* 你从使用你服务的用户那里收集的信息
* 你如何使用并共享这些信息（包括与 X 共享）
* 用户如何联系你，就其信息提出咨询和请求

你的隐私政策必须符合所有适用法律，并且对个人的保护程度不得低于 [X 的隐私政策](https://x.com/privacy) 和 [我们其他服务及企业关联公司的隐私政策](https://help.x.com/rules-and-policies/x-services-and-corporate-affiliates)。如果你无法遵守你自身和/或 X 的隐私政策，你必须停止访问 X API 并停止使用所有 X 内容。

<div id="using-geo-data">
  ## 使用地理数据
</div>

**由于此类信息具有敏感性，使用地理数据会受到额外限制。** 如果你的服务向帖子添加位置信息，你必须向用户明示：

* 你在何时添加位置信息
* 你是以地理标签（geotag）还是注释数据（annotations data）的形式添加位置信息
* 你的位置信息是以地点（place）还是以地理坐标的形式列出

如果你的应用允许用户在发布内容时附带其位置，你必须完全遵守 X 的[地理指南](/zh/developer-terms/geo-guidelines)。

**严禁单独使用位置信息或地理信息。** 除非作为帖子的一部分，否则你不得（也不得允许他人）存储、聚合或缓存 X 内容中包含的位置信息和其他地理信息。例如，你不得将帖子中的位置信息或地理信息单独分离出来，用于展示个人在一段时间内出现过的地点。但允许使用显示聚合地理活动的热力图和相关工具（例如，显示在某个城市中使用特定标签的人数）。

<div id="x-passwords">
  ## X 密码
</div>

**您不得存储任何 X 密码，也不得要求用户直接向您提供其 X 密码、账户凭据或开发者应用信息（包括 consumer key）。** 我们建议使用 [使用 X 登录](/zh/fundamentals/authentication/guides/log-in-with-x) 作为身份验证工具，将您的服务与 X 上的用户关联起来。

<div id="platform-usage-guidelines">
  ## 平台使用指南
</div>

你是否已经仔细审查了 X 的各项政策，并正确设置了你的 API 访问权限？你的服务是否遵循了 X 的隐私和控制指南？如果你对这两个问题都可以回答“是”，那么你就已经准备好开始使用 X API 和 X 内容了。X 的平台使用指南将帮助你确保，从服务上线的第一天起直至其整个生命周期，你对 X 内容的使用始终保持合规。我们建议你定期查阅这些规则，以确保你的集成始终以安全且对 X 上的人们以及整个 X 平台都有益的方式运行。

<div id="spam-bots-and-automation">
  ## 垃圾信息、机器人和自动化
</div>

**禁止使用 X API 和开发者产品创建垃圾信息，或从事任何形式的平台操纵行为。** 你应当查阅关于[平台操纵和垃圾信息](https://help.x.com/rules-and-policies/platform-manipulation)的 X 规则，并确保你的服务本身不会、也不会让他人能够违反我们的政策。

**执行写入操作的服务（包括发布帖子、关注账号或发送私信）必须遵守[自动化规则](https://help.x.com/rules-and-policies/x-automation)。** 特别是，你应当：

* 在向用户发送[自动回复或私信](https://twittercommunity.com/t/policy-clarification-automated-replies-and-mentions/94444)前，始终取得其明确同意
* 在用户要求不再被你联系（选择退出）时，必须立即予以尊重并停止联系
* 切勿执行批量、激进或类似垃圾信息的操作，包括[批量关注](https://twittercommunity.com/t/policy-clarification-aggressive-following-and-inorganic-following-behavior/92769)
* 切勿在[多个账号](https://blog.twitter.com/developer/en_us/topics/tips/2018/automation-and-the-use-of-multiple-accounts.html)上发布相同或实质上相似的内容

**如果你在运营一个基于 API 的机器人账号，则必须清晰说明该账号的性质以及由谁负责。** 你绝不应误导或混淆人们，让他们无法判断你的账号是否为机器人。一种较好的做法是在个人资料简介中加入该账号为机器人的声明。

<div id="x-performance-benchmarking">
  ## X 性能基准测试
</div>

**您不得使用 X API 对 X 的可用性、性能、功能或使用情况进行衡量，以用于基准测试、竞争分析或其他商业目的。** 例如，您绝不能使用 X API 来：

* 计算聚合的 X 指标，例如每月活跃用户总数（MAs）或每日活跃用户总数（DAs）
* 计算聚合的 X 帖子指标，例如每天发布的帖子总数，或账号互动总次数
* 衡量或分析 X 的响应性
* 衡量或分析 X 上的垃圾信息或安全状况，但下文允许的情况除外

**我们支持有助于改善 X 上对话健康状况的研究。** 未经 X 的明确书面许可，您不得公开披露任何使用 X API 或 X 内容进行的、涉及衡量、分析或试图识别违反 X 政策的行为或内容的研究或研究结果，也不得基于此类研究开发、创建或提供服务。
