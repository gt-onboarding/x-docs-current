在使用我们的开发者平台之前，您必须先阅读并同意我们的[开发者协议和政策](/zh/developer-terms/agreement)，以及相关政策，包括[展示要求](https://developer.x.com/en/developer-terms/display-requirements.html)和[自动化规则](https://help.twitter.com/en/rules-and-policies/twitter-automation)。除其他内容外，这些协议和政策对若干受限使用情形提供了指导。我们在下文对其中部分限制提供了更多补充信息。

<div id="sensitive-information">
  ## 敏感信息
</div>

在使用 X 数据推导或推断 X 用户的潜在敏感特征时，你应谨慎行事。不得对任何 X 用户的以下信息进行推导或推断，也不得存储此类已推导或推断的信息：

* 健康状况（包括怀孕）
* 不良的财务状况或境况
* 政治倾向或信仰
* 种族或民族出身
* 宗教或哲学倾向或信仰
* 性生活或性取向
* 工会成员身份
* 涉嫌或实际实施犯罪的情况

在不存储任何个人数据（例如 user ID、用户名和其他标识符）的情况下，对 X 内容进行汇总分析是被允许的，前提是该分析同样遵守适用法律以及《开发者协议和政策》的所有条款。

<div id="off-x-matching">
  ## 站外匹配（Off-X matching）
</div>

站外匹配是指将 X 内容（包括 X 用户名或用户 ID）与某个人、家庭、设备、浏览器或其他站外标识符相关联的行为。比如，将某个 X 用户名与某家企业的客户记录关联起来（即，你的客户记录中的 “John Doe” 与 X 上的 @johndoe 匹配）。

我们希望人们在 X 上可以安心地创建一个独立的身份，并且在他们选择时，可以使用化名。如果你打算将有关某个 X 用户的任何信息与某个站外标识符关联，我们要求你在进行关联之前，必须获得该用户的明确、主动同意（opt-in）。例如，如果用户在注册你的服务时，作为流程的一部分，主动向你提供他们的 X 用户名（handle），你就可以据此获得该同意。

在你没有获得用户明确、主动同意将其 X 身份与站外标识符关联的情况下，我们要求你仅能基于用户在合理预期范围内会被用于该目的的信息来建立任何关联。如果用户在得知你正在使用他们提供的信息，将其 X 账号与 X 之外的身份进行关联时会感到意外，那就不要这么做。此外，在缺乏个人明确主动同意的前提下，你只能基于以下信息，尝试将你所掌握的关于某人的记录与某个 X 身份进行匹配：

* **用户直接提供给你的信息。** 关于与你没有任何既有关系的个人的记录（包括从第三方获得的个人数据）不符合这一标准；及/或
* **公共数据。** 在此语境下，“公共数据”是指：
  * 你从公开且普遍可获取的资源（例如某专业协会成员名录）中获得的关于某用户的信息
  * 关于某个用户，在 X 上公开可见的信息，包括：
    * 帖子
    * 个人资料信息，包括用户的个人简介（bio）和公开标注的位置
    * 显示名称和用户名

<div id="redistribution-of-x-content">
  ## X 内容的再分发
</div>

如果你需要将通过 X API 获取的 X 内容与其他方分享，最好的方式是分享帖子 ID、私信 ID 以及/或者用户 ID，由内容的最终用户通过 X API 对其进行再水化（即请求完整的帖子、用户或私信内容）。这有助于确保 X 内容的最终用户始终直接从我们这里获得最新的信息。

我们允许通过非自动化方式对再水化后的 X 内容进行有限再分发。如果你选择以这种方式与他方分享再水化后的 X 内容，你每天每位接收者最多只能分享 50,000 个再水化的公开 Post 对象和/或 User 对象，并且不应将这些数据向公众公开（例如，作为博客文章的附件或放在公开的 GitHub 仓库中）。

在再分发 X 内容时，还有一些其他要点需要牢记：

* 在 30 天内，你向单一实体分发的 Post ID 总量最多为 1,500,000，除非你事先获得 X 明确的书面许可。
* 代表学术机构、且仅用于非商业研究目的而再分发 Post ID 和/或 User ID 的个人，允许再分发不限数量的 Post ID 和/或 User ID。
* 若要按照上述方式申请共享 X 内容的许可，请使用 API 政策支持表单。

在你被允许向第三方分发 X 内容的范围内，请注意，这些内容仍受《开发者协议》和《开发者政策》约束，并且这些第三方在接收 X 内容之前，必须同意 X 的[服务条款](https://twitter.com/en/tos)、[隐私政策](https://twitter.com/en/privacy)、[开发者协议](/zh/developer-terms/agreement)以及[开发者政策](/zh/developer-terms/policy)。

<div id="multiple-applications">
  ## 多个应用
</div>

你不得为单一使用场景，或为在实质上相同或重叠的使用场景，注册多个应用。请在[此处](https://twittercommunity.com/t/policy-clarification-multiple-applications-for-the-same-use-case/92080)详细了解这些政策。

在本政策中，我们将“使用场景”定义为：通过某个应用执行的一组一致的分析、展示或操作。向不同终端用户提供同一项服务或应用，视为单一使用场景，但工具或服务的“白标”版本不在此列。如果你计划以“白标”形式提供应用版本，必须提前通知我们并另行获得批准。这些规则同时适用于你注册的应用，以及由你的工具或服务的终端用户所注册的应用；如果你要求终端用户为使用你的工具或服务而注册应用，则可能导致我们对你、你的应用、你的客户和/或你的工具或服务的终端用户采取执法措施。

该规则唯一的例外，是为同一服务创建开发（“dev”）、预发布（staging）和生产（“prod”）实例。请确保这些应用有清晰标注（例如在应用名称或描述中），并且不要将开发或预发布应用用于生产目的。

<div id="automation-spam-and-auto-responses">
  ## 自动化、垃圾信息和自动回复
</div>

禁止使用 X 的 API 和开发者产品来创建垃圾信息或从事垃圾信息相关行为。你应当查阅有关垃圾信息的 [X Rules](https://t.co/rules)，并确保你的应用本身不会、也不会使用户能够违反我们的政策。

如果你的应用将用于在 X 服务上执行写入操作，包括发布帖子、关注账号或发送私信，你应当仔细阅读 [Automation Rules](https://support.twitter.com/articles/76915)，以确保你的服务符合我们的准则。特别是，你应当：

* 在向用户发送[自动回复或消息](https://twittercommunity.com/t/policy-clarification-automated-replies-and-mentions/94444)之前，始终获得用户的明确同意
* 一旦用户请求不再被你联系，必须立即予以尊重并停止联系
* 切勿执行批量、过度激进或具有垃圾信息性质的操作，包括[批量关注](https://twittercommunity.com/t/policy-clarification-aggressive-following-and-inorganic-following-behavior/92769)
* 切勿在[多个账号](https://blog.twitter.com/developer/en_us/topics/tips/2018/automation-and-the-use-of-multiple-accounts.html)上发布相同或实质上相似的内容

<div id="measuring-the-x-service">
  ## 衡量 X 服务
</div>

不得出于基准测试或竞争分析的目的使用 X API 来衡量 X 的可用性、性能、功能或使用情况。例如，你绝不能使用 X API 来：

* 计算 X 用户指标的汇总数据，例如活跃用户或账号的总数
* 计算 X 帖子指标的汇总数据，例如每天的帖子总数，或用户互动次数或账号互动次数
* 衡量或分析 X 上的垃圾信息或安全状况，除非在 [X Rules](https://help.twitter.com/en/rules-and-policies/twitter-rules) 中另有允许

<div id="surveillance-privacy-and-user-protection">
  ## 监控、隐私与用户保护
</div>

在 X，保护和捍卫用户隐私已经深深融入公司的核心基因——我们的开发者产品和数据产品也充分体现了这一承诺。我们相信，X 数据可以成为推动世界向善的强大力量——从在雅加达洪灾期间挽救生命，到帮助 USGS 追踪地震，再到与联合国合作实现可持续发展目标。然而，我们禁止任何实体将 X 数据和 X API 用于监控目的，或以任何其他方式使用，只要该方式与用户对隐私的合理预期不一致，一律禁止。没有例外。

我们在[开发者协议](/zh/developer-terms/agreement)中描述了对我们数据和开发者产品的禁止用途，包括禁止调查或追踪 X 用户或其内容，以及追踪、预警或监控敏感事件（例如抗议、集会或社区组织会议）。

根据这些条款，以下其他类型的活动也被禁止（包括但不限于）：

* 调查或追踪敏感群体和组织，例如工会或维权/倡议团体
* 背景调查或任何形式的严格审查
* 信用或保险风险分析
* 个体画像或心理特征细分
* 人脸识别

这些政策适用于我们 API 的所有用户。任何将 X API 用于上述目的的滥用行为都将受到相应的执行措施，包括暂停和终止访问权限。

如执法机构需要了解 X 账号相关信息的更多说明，请访问 [https://t.co/le](https://t.co/le)。

此外，目前，除 [Grok](https://help.x.com/en/using-x/about-grok) 之外，X 禁止以任何方式使用 X API 和/或 X 内容来微调或训练基础模型或前沿模型。
