使用我们的开发者平台前，您必须阅读并同意我们的[开发者协议与政策](/zh/developer-terms/agreement)，以及相关政策，包括[展示要求](https://developer.x.com/en/developer-terms/display-requirements.html)和[自动化规则](https://help.twitter.com/en/rules-and-policies/twitter-automation)。除其他内容外，我们的协议和政策还就若干受限用例提供了指导。我们在下文就其中部分限制提供了更多补充信息。

<div id="sensitive-information">
  ## 敏感信息
</div>

在使用 X 数据推导或推断 X 用户的潜在敏感特征时，你应当格外谨慎。绝不要对任何 X 用户推导、推断，或存储基于推导或推断得出的以下信息：

* 健康状况（包括怀孕情况）
* 不良的财务状况或境况
* 政治立场或信仰
* 种族或民族背景
* 宗教或哲学立场或信仰
* 性生活或性取向
* 工会成员身份
* 被指控或实际实施犯罪的情况

在不存储任何个人数据（例如，用户 ID、用户名和其他标识符）的前提下，对 X 内容进行汇总分析是被允许的，但同时还必须遵守适用法律以及《开发者协议与政策》的所有条款。

<div id="off-x-matching">
  ## 站外 X 匹配
</div>

站外 X 匹配是指将 X 内容（包括 X 用户名或用户 id）与某个人、家庭、设备、浏览器或其他站外标识符关联起来。比如，将某个 X 用户名与某企业的客户记录关联（即你客户记录中的 “John Doe” 被匹配到 X 上的 @johndoe）。

我们希望人们在 X 上能够放心地创建一个独立的身份，并且在他们愿意的情况下可以使用化名身份。如果你打算将有关某个 X 用户的任何信息与某个站外标识符关联，我们要求你在进行关联之前，必须事先从该用户处获得其明确的、主动选择加入（opt-in）的同意。例如，如果用户在注册你的服务时，作为注册流程的一部分，主动向你提供其 X 账号（handle），你可以据此获得该同意。

在你没有获得用户明确的、主动选择加入同意、以将其 X 身份与某个站外标识符关联的情况下，我们要求你所做的任何关联只能基于用户在合理预期下会被用于该目的的信息。如果用户得知你正在使用他们提供的信息，将其 X 账号与 X 平台之外的某个身份进行关联，会感到意外，那就不要这么做。此外，在未取得个人明确的主动选择加入同意时，你只能基于以下内容，尝试将你记录中的某人与某个 X 身份进行匹配：

* **用户直接提供给你的信息。** 对于与你没有任何既往关系的个人的记录（包括从第三方获取的个人数据），不符合这一标准；和/或
* **公开数据。** 在此语境下，“公开数据”是指：
  * 你从公开的、普遍可获得的资源中获取的关于某个用户的信息（例如某专业协会会员名录）
  * 在 X 上关于某个用户且公开可见的信息，包括：
    * 帖子
    * 个人资料信息，包括用户的简介和其公开声明的位置
    * 显示名和用户名

<div id="redistribution-of-x-content">
  ## X 内容的再分发
</div>

如果你需要将通过 X API 获取的 X 内容与其他方共享，最佳方式是共享 Post ID、Direct Message ID 和/或 User ID，内容的最终用户随后可以使用 X API 对其进行“rehydrate”（即重新请求完整的 Post、User 或 Direct Message 内容）。这有助于确保 X 内容的最终用户始终直接从我们这里获取最新信息。

我们允许通过非自动化（手动）方式对已还原的 X 内容进行有限再分发。如果你选择以这种方式与其他方共享已还原的 X 内容，则你每天最多只能向每位接收方共享 50,000 个已还原的公共 Post 对象和/或 User 对象，并且不应将这些数据公开提供（例如，作为博客文章的附件或在公共 GitHub 仓库中）。

在再分发 X 内容时，还需牢记以下几点：

* 除非事先获得 X 的书面明示许可，否则在任意 30 天内，你最多只能向单一实体分发总计 1,500,000 个 Post ID。
* 代表学术机构、且仅为非商业研究目的再分发 Post ID 和/或 User ID 的个人，可不受数量限制地再分发 Post ID 和/或 User ID。
* 如需申请按上述方式共享 X 内容的许可，请使用 API 政策支持表单。

在你被允许向第三方分发 X 内容的范围内，请注意，这些内容仍受《Developer Agreement and Policy》约束，并且上述第三方在接收 X 内容之前，必须同意 X 的[服务条款](https://twitter.com/en/tos)、[隐私政策](https://twitter.com/en/privacy)、[Developer Agreement](/zh/developer-terms/agreement)和[Developer Policy](/zh/developer-terms/policy)。

<div id="multiple-applications">
  ## 多个应用程序
</div>

你不得为单一使用场景，或在实质上相同或重叠的使用场景注册多个应用程序。要了解有关这些政策的更多信息，请参见[此处](https://twittercommunity.com/t/policy-clarification-multiple-applications-for-the-same-use-case/92080)。

在这里，我们将“使用场景”定义为通过某个应用程序执行的一组持续一致的分析、展示或操作。向不同终端用户提供相同的服务或应用程序被视为一个使用场景；但工具或服务的“白标”版本不在此列。如果你计划提供应用程序的“白标”版本，必须告知我们并单独获得批准。这些规则同时适用于你所注册的应用程序，以及你的工具或服务的终端用户所注册的应用程序；如果你要求终端用户为使用你的工具或服务而注册应用程序，我们可能会对你、你的应用程序、你的客户以及和/或你的工具或服务的终端用户采取合规执法措施。

此规则的唯一例外是为同一服务创建开发（“dev”）、预发布（“staging”）和生产（“prod”）环境实例。请确保这些应用程序有清晰标注（例如在应用程序名称或描述中），并且不要将开发或预发布应用程序用于生产目的。

<div id="automation-spam-and-auto-responses">
  ## 自动化、垃圾信息和自动回复
</div>

禁止使用 X 的 API 和开发者产品创建垃圾信息，或从事垃圾式行为。你应当查看关于垃圾信息的 [X 规则](https://t.co/rules)，并确保你的应用本身不违反我们的政策，也不会使用户能够违反我们的政策。

如果你的应用将被用于在 X 服务上执行写入操作，包括发布帖子、关注账号或发送私信，你应当仔细阅读[自动化规则](https://support.twitter.com/articles/76915)，以确保你的服务符合我们的指南。具体来说，你应当：

* 在向用户发送[自动回复或消息](https://twittercommunity.com/t/policy-clarification-automated-replies-and-mentions/94444)之前，始终获得用户的明确同意
* 立即尊重并执行用户关于不再被你联系的请求
* 切勿执行批量、激进或具有垃圾信息性质的操作，包括[批量关注](https://twittercommunity.com/t/policy-clarification-aggressive-following-and-inorganic-following-behavior/92769)
* 切勿在[多个账号](https://blog.twitter.com/developer/en_us/topics/tips/2018/automation-and-the-use-of-multiple-accounts.html)上发布完全相同或实质上相似的内容

<div id="measuring-the-x-service">
  ## 衡量 X 服务
</div>

不得将 X API 用于衡量 X 的可用性、性能、功能或使用情况，以进行基准测试或竞争性比较。例如，你不得使用 X API 来：

* 计算汇总的 X 用户指标，例如活跃用户或账号的总数
* 计算汇总的 X 帖子指标，例如每天的帖子总数，或用户互动或账号互动的次数
* 衡量或分析 X 上的垃圾信息或安全问题，除非 [X Rules](https://help.twitter.com/en/rules-and-policies/twitter-rules) 另有允许

<div id="surveillance-privacy-and-user-protection">
  ## 监控、隐私和用户保护
</div>

在 X，我们将保护和捍卫用户隐私深度融入公司的核心基因之中——我们的开发者和数据产品也体现了这一承诺。我们相信，X 数据可以成为推动世界向善的强大力量——从在雅加达洪灾期间挽救生命，到帮助美国地质调查局（USGS）追踪地震，再到与联合国合作实现可持续发展目标。然而，我们禁止任何实体将 X 数据和 X API 用于监控目的，或以任何其他方式使用，从而违背用户对隐私的合理预期。没有例外。

我们在[开发者协议](/zh/developer-terms/agreement)中描述了对我们数据和开发者产品的禁止使用情形，其中包括禁止调查或追踪 X 用户或其内容，以及追踪、预警或监控敏感事件（例如抗议、集会或社区组织会议）。

在这些条款下，其他被禁止的活动类别包括但不限于：

* 调查或追踪敏感群体和组织，例如工会或激进组织
* 背景调查或任何形式的严格审查
* 信用或保险风险分析
* 个人画像或心理特征细分
* 人脸识别

这些政策适用于我们所有 API 的使用者。任何将 X API 用于上述目的的滥用行为都将受到执行措施的约束，包括暂停和终止访问权限。

如需为寻求 X 账户相关信息的执法机构提供的更多信息，请访问 [https://t.co/le](https://t.co/le)。

此外，目前，除 [Grok](https://help.x.com/en/using-x/about-grok) 之外，X 禁止以任何方式使用 X API 和/或 X 内容来微调或训练基础模型或前沿模型。
