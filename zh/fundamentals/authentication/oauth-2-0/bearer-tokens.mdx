<div id="using-and-generating-an-app-only-bearer-token">
  ### 使用和生成仅应用（app-only）Bearer Token
</div>

Bearer Token 使开发者能够以更安全的方式访问 X API，并且是 OAuth 2.0 的核心特性之一。 

使用 Bearer Token 的身份验证也称为仅应用（application-only）身份验证。Bearer Token 是一种未指定格式的字节数组，你可以通过脚本（例如 `curl` 命令）生成。你也可以在开发者门户中，进入你的应用设置的“密钥和令牌”部分获取 Bearer Token。有关此功能的更多信息，请参阅 [OAuth 的官方文档](https://oauth.net/2/bearer-tokens/)。

<div id="when-are-they-used">
  #### 在什么情况下使用？
</div>

需要使用 Bearer Token 的产品包括：

* [Engagement API](/zh/x-api/enterprise-gnip-2.0/fundamentals/engagement-api)
* [Account Activity API](/zh/x-api/enterprise-gnip-2.0/fundamentals/account-activity)
* 其他使用 OAuth 2.0 Bearer Token 认证的 API，例如 v2 和 Labs 端点。

<div id="prerequisites">
  #### 先决条件
</div>

你需要先[注册一个开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)，并创建一个 [X 应用](/zh/resources/fundamentals/developer-apps)。完成这些之后，你还需要在[开发者门户](/zh/resources/fundamentals/developer-portal)中获取 API 密钥。请按照以下步骤操作：

1. 在 developer.x.com 登录你的 X 账号。
2. 前往 [X App dashboard](https://developer.x.com/content/developer-twitter/en/apps)，并打开你希望为其生成访问令牌的 X 应用。
3. 进入 “keys and tokens” 页面。
4. 你可以在此页面找到 API 密钥、用户访问令牌（Access Token）以及 Bearer Token。

<div id="how-to-generate-a-bearer-token">
  ### 如何生成 Bearer Token
</div>

你可以在应用的“密钥和令牌”中，与其他密钥一起找到该应用的 Bearer Token。

将先前从你的 [X 应用](/zh/resources/fundamentals/developer-apps)获取的以下 consumer API 密钥替换到下面的命令中后，将该 cURL 请求复制到命令行中。请注意，本页使用的 consumer API 密钥已停用，无法用于真实请求。

* **API key** `<API key>` 例如 `xvz1evFS4wEEPTGEFPHBog`
* **API secret key** `<API secret key>` 例如 `L8qq9PZyRg6ieKGEKhZolGC0vJWLw8iEJ88DRdyOg`

  ```bash
  curl -u "$API_KEY:$API_SECRET_KEY" \
  --data 'grant_type=client_credentials' \
  'https://api.x.com/oauth2/token'
  ```

下面是填入你的 API 密钥后，cURL 请求应当呈现的示例：

```bash
   curl -u 'xvz1evFS4wEEPTGEFPHBog:L8qq9PZyRg6ieKGEKhZolGC0vJWLw8iEJ88DRdyOg' \
  --data 'grant_type=client_credentials' \
  'https://api.x.com/oauth2/token'
```

以下是响应示例。请注意，这是一个已停用的 Bearer Token：

`{"token_type":"bearer","access_token":"AAAAAAAAAAAAAAAAAAAAAMLheAAAAAAA0%2BuSeid%2BULvsea4JtiGRiSDSJSI%3DEUifiRBkKG5E2XzMDjRfl76ZC9Ub0wnz4XsNiRVBChTYbJcE3F"}`

我们用于通过 OAuth 2.0 访问资源时进行认证的 Bearer Token 将是：

`AAAAAAAAAAAAAAAAAAAAAMLheAAAAAAA0%2BuSeid%2BULvsea4JtiGRiSDSJSI%3DEUifiRBkKG5E2XzMDjRfl76ZC9Ub0wnz4XsNiRVBChTYbJcE3F`
