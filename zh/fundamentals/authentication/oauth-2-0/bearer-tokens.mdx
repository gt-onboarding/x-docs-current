<div id="using-and-generating-an-app-only-bearer-token">
  ### 使用和生成仅应用级 Bearer Token
</div>

Bearer Token 允许开发者以更安全的方式访问 X API，是 OAuth 2.0 的核心特性之一。

使用 Bearer Token 的身份验证方式也称为仅应用身份验证（application-only authentication）。Bearer Token 是一种未指定格式的字节数组，你可以通过使用类似 `curl` 命令的脚本来生成它。你也可以在开发者门户中进入你的应用设置，在“密钥和令牌”部分获取 Bearer Token。关于此功能的更多信息请参阅 [OAuth 的官方文档](https://oauth.net/2/bearer-tokens/)。

<div id="when-are-they-used">
  #### 它们在何时会被使用？
</div>

需要使用 Bearer Token 的产品包括：

* [Engagement API](/zh/x-api/enterprise-gnip-2.0/fundamentals/engagement-api)
* [Account Activity API](/zh/x-api/enterprise-gnip-2.0/fundamentals/account-activity)
* 其他使用 OAuth 2.0 Bearer Token 进行认证的 API，例如 v2 和 Labs 的端点。

<div id="prerequisites">
  #### 先决条件
</div>

你需要[注册一个开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)，并创建一个 [X App](/zh/resources/fundamentals/developer-apps)。完成这些之后，你还需要获取在[开发者门户](/zh/resources/fundamentals/developer-portal)中提供的 API 密钥。按照以下步骤操作：

1. 在 developer.x.com 上登录你的 X 账号。
2. 前往 [X App dashboard](https://developer.x.com/content/developer-twitter/en/apps)，并打开你希望为其生成访问令牌的 X App。
3. 进入 &quot;keys and tokens&quot; 页面。
4. 你可以在此页面找到 API 密钥、用户访问令牌以及 Bearer Token。

<div id="how-to-generate-a-bearer-token">
  ### 如何生成 Bearer Token
</div>

你可以在你的应用的“密钥和令牌”中找到该应用的 Bearer Token。

将下面的 cURL 请求复制到命令行中，并将其中的占位符替换为你此前从 [X 应用](/zh/resources/fundamentals/developer-apps) 获取的下列 consumer API 密钥。请注意，本页中使用的 consumer API 密钥已被停用，不能用于实际请求。

* **API key** `<API key>` 例如 `xvz1evFS4wEEPTGEFPHBog`
* **API secret key** `<API secret key>` 例如 `L8qq9PZyRg6ieKGEKhZolGC0vJWLw8iEJ88DRdyOg`

  ```bash
  curl -u "$API_KEY:$API_SECRET_KEY" \
  --data 'grant_type=client_credentials' \
  'https://api.x.com/oauth2/token'
  ```

下面是一个示例，展示在填入你的 API 密钥后，cURL 请求应呈现的形式：

```bash
   curl -u 'xvz1evFS4wEEPTGEFPHBog:L8qq9PZyRg6ieKGEKhZolGC0vJWLw8iEJ88DRdyOg' \
  --data 'grant_type=client_credentials' \
  'https://api.x.com/oauth2/token'
```

以下是响应的示例。请注意，这是一个已停用的 Bearer Token：

`{"token_type":"bearer","access_token":"AAAAAAAAAAAAAAAAAAAAAMLheAAAAAAA0%2BuSeid%2BULvsea4JtiGRiSDSJSI%3DEUifiRBkKG5E2XzMDjRfl76ZC9Ub0wnz4XsNiRVBChTYbJcE3F"}`

`AAAAAAAAAAAAAAAAAAAAAMLheAAAAAAA0%2BuSeid%2BULvsea4JtiGRiSDSJSI%3DEUifiRBkKG5E2XzMDjRfl76ZC9Ub0wnz4XsNiRVBChTYbJcE3F`
