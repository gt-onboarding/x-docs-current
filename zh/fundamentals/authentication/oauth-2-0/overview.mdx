<div id="bearer-token-also-known-as-app-only">
  ### Bearer Token（也称为仅应用模式）
</div>

OAuth 2.0 Bearer Token 用于代表你的[开发者应用](/zh/resources/fundamentals/developer-apps)对请求进行身份验证。由于这种方式仅针对应用本身，因此不涉及任何用户。该方式通常适用于只需要对公开信息进行只读访问的开发者。 

这种认证方式要求你在请求中携带一个 Bearer Token，你可以在开发者应用的“密钥和令牌”（Keys and tokens）部分生成该 Bearer Token。下面是一个带有虚构 Bearer Token 的请求示例：

```json
  curl "https://api.x.com/2/tweets?ids=1261326399320715264,1278347468690915330" \
  -H "Authorization: Bearer AAAAAAAAAAAAAAAAAAAAAFnz2wAAAAAAxTmQbp%2BIHDtAhTBbyNJon%2BA72K4%3DeIaigY0QBrv6Rp8KZQQLOTpo9ubw5Jt?WRE8avbi"
```

使用仅应用身份验证的 API 调用，会在应用级别按端点进行[速率限制](/zh/resources/fundamentals/rate-limits)。

要使用此方法，你需要一个 Bearer Token。你可以通过将 API Key 和 API Secret 传递给 [POST oauth2/token](/zh/resources/fundamentals/authentication/api-reference#post-oauth2-token) 端点来生成 Bearer Token，或者在 [developer portal](/zh/resources/fundamentals/developer-portal) 中你的 App 设置的 “keys and token” 部分生成它。

如果你想吊销 Bearer Token，可以使用 [POST oauth2/invalidate&#95;token](/zh/resources/fundamentals/authentication/api-reference#post-oauth2-invalidate-token) 端点，或者在 App 设置的 “keys and tokens” 部分中，点击 Bearer Token 旁边标为 “revoke” 的位置。

<div id="oauth-20-authorization-code-flow-with-pkce">
  ### OAuth 2.0 Authorization Code Flow with PKCE
</div>

OAuth 2.0 Authorization Code Flow with PKCE 使你可以代表另一位用户进行身份验证，同时对应用的 scopes 拥有更多控制权，并改进跨多个设备的授权流程。换句话说，为 X 上的用户构建应用的开发者将能更好地控制其应用向用户请求的信息，从而你只需向终端用户请求你真正需要的数据和信息。

这一现代授权协议将使你能够为终端用户提供更加简化的授权同意流程来授权你的应用，在该流程中只会展示你向他们请求的特定 scopes。这样不仅可以减少你的数据处理负担，还可能提升终端用户对你的信任度。
