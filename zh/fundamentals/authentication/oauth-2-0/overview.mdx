<div id="bearer-token-also-known-as-app-only">
  ### Bearer Token（也称为仅应用模式）
</div>

OAuth 2.0 Bearer Token 用于代表你的[开发者应用](/zh/resources/fundamentals/developer-apps)对请求进行认证。由于这种方式是特定于应用的，因此不涉及任何用户。此方法通常适用于只需要对公共信息进行只读访问的开发者。 

此认证方法要求你在请求中携带一个 Bearer Token，你可以在开发者应用的「密钥和令牌」部分生成该 Bearer Token。下面是一个在请求中使用虚拟 Bearer Token 的示例：

```json
  curl "https://api.x.com/2/tweets?ids=1261326399320715264,1278347468690915330" \
  -H "Authorization: Bearer AAAAAAAAAAAAAAAAAAAAAFnz2wAAAAAAxTmQbp%2BIHDtAhTBbyNJon%2BA72K4%3DeIaigY0QBrv6Rp8KZQQLOTpo9ubw5Jt?WRE8avbi"
```

使用仅应用身份验证进行的 API 调用，会在 App 级别对每个端点单独施加[速率限制](/zh/resources/fundamentals/rate-limits)。

要使用此方法，你需要一个 Bearer Token。你可以通过将 API Key 和 Secret 传给 [POST oauth2/token](/zh/resources/fundamentals/authentication/api-reference#post-oauth2-token) 端点来生成它，或者在 [developer portal](/zh/resources/fundamentals/developer-portal) 中你的 App 设置里的 “keys and token” 部分生成。

如果你想撤销 Bearer Token，可以使用 [POST oauth2/invalidate&#95;token](/zh/resources/fundamentals/authentication/api-reference#post-oauth2-invalidate-token) 端点，或者在 App 设置的 “keys and tokens” 部分，点击该 Bearer Token 旁边的 “revoke”。

<div id="oauth-20-authorization-code-flow-with-pkce">
  ### OAuth 2.0 Authorization Code Flow with PKCE
</div>

OAuth 2.0 Authorization Code Flow with PKCE 允许你代表其他用户进行认证，同时让你对应用的权限范围（scopes）拥有更多控制，并改进跨多设备的授权流程。换句话说，为 X 上的用户构建应用的开发者将能更精细地控制其应用向用户请求的信息，因此你只需要向终端用户请求你所需要的数据和信息。

这一现代授权协议允许你为终端用户呈现更流畅的授权同意流程来为你的应用授予权限，该流程只会展示你向他们请求的特定权限范围（scopes）。这样不仅可以减少你的数据负担，还可能提升终端用户对你的信任。
