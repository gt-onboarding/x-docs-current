<div id="api-key-and-secret">
  ### API Key 和 Secret
</div>

API Key 和 Secret（也称为 Consumer Key 和 Consumer Secret）是访问 X API 所需的最基本凭证。这些凭证相当于你的 X 应用的用户名和密码，X API 会通过它们来识别请求来自哪个应用。 

这些凭证可以被[认证端点](/zh/resources/fundamentals/authentication/api-reference)使用，用于生成其他凭证，例如[用户访问令牌和密钥](/zh/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret)以及[Bearer Tokens](/zh/resources/fundamentals/authentication/oauth-2-0/bearer-tokens)。在为需要 OAuth 1.0a 用户上下文认证的请求[授权](/zh/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request)时，你还需要将这些凭证与访问令牌及其他授权参数一同使用。 

<div id="how-to-acquire-an-api-key-and-secret">
  #### 如何获取 API Key 和 Secret
</div>

要获取 X API Key 和 Secret，请按照以下步骤操作：

1. [注册 X 开发者账号](https://developer.x.com/en/apply-for-access)
2. 在 [developer portal](/zh/resources/fundamentals/developer-portal) 中创建一个 [X App](/zh/resources/fundamentals/developer-apps)。请注意，如果你希望使用 [X API v2](/zh/x-api/introduction)，必须将你的 X App 添加到一个[项目](/zh/resources/fundamentals/projects)中。
    

当你创建 X App 时，你将会看到 API Key 和 Secret，以及一个 Bearer Token。请注意，这些凭据只会显示一次，因此请务必将它们保存在密码管理器或其他安全的位置。

你可以在我们的[身份验证最佳实践](/zh/resources/fundamentals/authentication/guides/authentication-best-practices)页面中找到更多关于如何处理密钥和令牌的建议，包括当你的凭据遭到泄露时应该采取的措施。
 

<div id="how-to-find-and-regenerate-your-api-key-and-secret-after-app-creation">
  #### 在创建应用后如何查找和重新生成 API Key 和 Secret
</div>

如果你已经创建了一个应用，并且需要查找或重新生成你的 API Key 和 Secret，请按照以下步骤操作：

1. 前往开发者门户
2. 展开侧边导航中的“Projects and Apps”下拉菜单
3. 打开与你想要查找或重新生成的 API Key 和 Secret 关联的应用
4. 转到“Keys and tokens”选项卡

在那里，你可以找到与你的应用关联的所有凭证。 
 

<div id="how-to-use-your-api-key-and-secret">
  #### 如何使用你的 API Key 和 Secret
</div>

如果你只是想探索 X Developer Platform，我们建议你使用[工具或库](/zh/resources/tools-and-libraries)来了解平台上都有哪些可用功能。这些工具能够优雅地处理身份验证，为你节省大量时间并减少挫败感。我们尤其推荐[使用 Postman 入门](/zh/tutorials/postman-getting-started)或 [Insomnia](https://insomnia.rest/)，适合入门级开发者使用。 

如果你有兴趣从零开始构建请求，请阅读我们的指南：[为 OAuth 1.0a 请求授权](/zh/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request)。
