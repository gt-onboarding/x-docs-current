<div id="api-key-and-secret">
  ### API Key 和 Secret
</div>

API Key 和 Secret（也称为 Consumer Key 和 Secret）是访问 X API 所需的最基本凭证。这些凭证相当于你的 X 应用的用户名和密码，X API 通过它们来识别请求来自哪个应用。 

这些凭证可以被[认证端点](/zh/resources/fundamentals/authentication/api-reference)用来生成其他凭证，例如[用户 Access Token 和 Secret](/zh/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret)以及 [Bearer Token](/zh/resources/fundamentals/authentication/oauth-2-0/bearer-tokens)。你还需要将这些凭证与 Access Token 和其他授权参数配合使用，以[为请求授权](/zh/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request)，从而满足需要 OAuth 1.0a 用户上下文认证的场景。 

<div id="how-to-acquire-an-api-key-and-secret">
  #### 如何获取 API Key 和 Secret
</div>

要获取 X API Key 和 Secret，请按照以下步骤操作：

1. [注册 X 开发者账号](https://developer.x.com/en/apply-for-access)
2. 在 [开发者门户](/zh/resources/fundamentals/developer-portal) 中创建一个 [X App](/zh/resources/fundamentals/developer-apps)。请注意，如果你想使用 [X API v2](/zh/x-api/introduction)，必须将你的 X App 添加到一个 [Project](/zh/resources/fundamentals/projects) 中。
    

当你创建 X App 时，你会看到你的 API Key 和 Secret，以及一个 Bearer Token。请注意，这些凭证只会显示一次，因此务必将它们保存在密码管理器或其他安全的位置。

在我们的[认证最佳实践](/zh/resources/fundamentals/authentication/guides/authentication-best-practices)页面中，提供了更多关于如何处理密钥和令牌的建议，包括当你的凭证遭到泄露时应采取的措施。
 

<div id="how-to-find-and-regenerate-your-api-key-and-secret-after-app-creation">
  #### 在创建应用后如何查找和重新生成 API Key 和 Secret
</div>

如果你已经创建了应用，并且需要查找或重新生成 API Key 和 Secret，请按以下步骤操作：

1. 进入开发者门户
2. 在侧边导航中展开 “Projects and Apps” 下拉菜单
3. 打开与要查找或重新生成的 API Key 和 Secret 关联的应用
4. 切换到 “Keys and tokens” 选项卡

在该页面，你可以查看与你的应用关联的所有凭证。 
 

<div id="how-to-use-your-api-key-and-secret">
  #### 如何使用你的 API Key 和 Secret
</div>

如果你只是想探索 X 开发者平台，我们建议你使用[工具或库](/zh/resources/tools-and-libraries)来了解平台提供的功能。这些工具可以优雅地处理身份验证流程，为你节省大量时间并减少挫折感。对于入门开发者，我们特别推荐[从 Postman 入门](/zh/tutorials/postman-getting-started)或 [Insomnia](https://insomnia.rest/)。

如果你有兴趣从零开始构建请求，请阅读我们的[OAuth 1.0a 请求授权指南](/zh/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request)。
