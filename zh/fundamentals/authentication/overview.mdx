X API 需要处理海量数据。我们通过身份验证机制来确保这些数据对开发者和用户都是安全的。下面列出了几种可用的身份验证方式。

大多数开发者无需处理与身份验证相关的复杂细节，因为客户端库会自动替你处理好这些问题。

你可以在我们的 [工具和库](/zh/resources/tools-and-libraries) 页面找到可用客户端库的列表。

<div id="authentication-methods">
  ## 身份验证方法
</div>

<CardGroup cols={2}>
  <Card title="OAuth 1.0a 用户上下文" href="/zh/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret">
    OAuth 1.0a 允许已获授权的 X 开发者应用访问非公开账户信息，或代表某个 X 账户在 X 上执行操作。

    <br />

    [**了解更多**](/zh/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret)
  </Card>

  <Card title="仅应用" href="/zh/resources/fundamentals/authentication/oauth-2-0/overview">
    仅应用访问令牌允许 X 开发者应用访问 X 上公开可见的信息。

    <br />

    [**了解更多**](/zh/resources/fundamentals/authentication/oauth-2-0/overview)
  </Card>

  <Card title="基本身份验证" href="/zh/resources/fundamentals/authentication/basic-auth">
    许多 X 的 Enterprise API 需要使用 HTTP Basic Authentication。

    <br />

    [**了解更多**](/zh/resources/fundamentals/authentication/basic-auth)
  </Card>

  <Card title="带 PKCE 的 OAuth 2.0 授权码流程" href="/zh/resources/fundamentals/authentication/oauth-2-0/authorization-code">
    OAuth 2.0 用户上下文允许你代表其他账户进行身份验证，对应用的权限范围进行更精细的控制，并支持跨多设备的授权流程。

    <br />

    [**了解更多**](/zh/resources/fundamentals/authentication/oauth-2-0/authorization-code)
  </Card>
</CardGroup>

<Note>
  **注意：**
  你的应用的 API Key 和仅应用访问令牌，以及你的个人访问令牌和访问令牌密钥（Access Token Secret），可以在位于 [开发者门户](/zh/resources/fundamentals/developer-portal) 中的 [X developer Apps](/zh/resources/fundamentals/developer-apps) 部分获取。

  **如果你希望代表其他用户发起请求**，则需要使用 [三方 OAuth 流程](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens) 为该用户生成单独的一组访问令牌，并在使用 OAuth 1.0a 用户上下文或 OAuth 2.0 用户上下文发起请求时，携带该用户的令牌。
</Note>
