X API 会处理海量数据。我们通过身份验证来确保这些数据在开发者和用户两端都得到安全保护。我们提供几种身份验证方式，如下所示。

多数开发者不需要直接处理与身份验证相关的复杂问题，因为客户端库会自动为你处理这些细节。

你可以在我们的 [工具和库](/zh/resources/tools-and-libraries) 页面找到可用客户端库的列表。

<div id="authentication-methods">
  ## 身份验证方法
</div>

<CardGroup cols={2}>
  <Card title="OAuth 1.0a 用户上下文" href="/zh/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret">
    OAuth 1.0a 允许已获授权的 X 开发者应用访问私有账户信息，或在 X 上代表某个账户执行操作。

    <br />

    [**了解详情**](/zh/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret)
  </Card>

  <Card title="仅应用" href="/zh/resources/fundamentals/authentication/oauth-2-0/overview">
    仅应用访问令牌允许 X 开发者应用访问 X 上公开可用的信息。

    <br />

    [**了解详情**](/zh/resources/fundamentals/authentication/oauth-2-0/overview)
  </Card>

  <Card title="基本身份验证" href="/zh/resources/fundamentals/authentication/basic-auth">
    许多 X 的 Enterprise API 需要使用 HTTP 基本身份验证。

    <br />

    [**了解详情**](/zh/resources/fundamentals/authentication/basic-auth)
  </Card>

  <Card title="带 PKCE 的 OAuth 2.0 授权码流程" href="/zh/resources/fundamentals/authentication/oauth-2-0/authorization-code">
    OAuth 2.0 用户上下文允许你代表其他账户进行身份验证，更好地控制应用的权限范围，并在多设备之间完成授权流程。

    <br />

    [**了解详情**](/zh/resources/fundamentals/authentication/oauth-2-0/authorization-code)
  </Card>
</CardGroup>

<Note>
  **注意：**
  你的应用的 API Key 和仅应用访问令牌，以及你的个人访问令牌和访问令牌密钥（Access Token Secret），可以在 [X 开发者应用](/zh/resources/fundamentals/developer-apps) 部分（位于 [开发者门户](/zh/resources/fundamentals/developer-portal) 中）获取。

  **如果你希望代表其他用户发起请求**，需要使用 [三方 OAuth 流程](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)为该用户生成一套单独的访问令牌，并在 OAuth 1.0a 用户上下文或 OAuth 2.0 用户上下文请求中携带该用户的令牌。
</Note>
