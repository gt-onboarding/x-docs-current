<div id="general">
  ## 常规
</div>

<AccordionGroup>
  <Accordion title="什么是 OAuth？">
    OAuth 是一种身份验证协议，允许用户在不共享密码的情况下授权应用代表他们执行操作。你可以在 [oauth.net](http://oauth.net/) 获取更多信息。
  </Accordion>

  <Accordion title="如何生成访问令牌？">
    你必须拥有一个 [X 应用](/zh/resources/fundamentals/developer-apps) 才能生成访问令牌。在[这里](/zh/resources/fundamentals/authentication#oauth-1-0a-2)了解更多有关访问令牌的信息。 
  </Accordion>

  <Accordion title="如何创建一个 X 应用？">
    你必须拥有一个[开发者账号](/zh/resources/fundamentals/developer-portal) 才能创建 [X 应用](/zh/resources/fundamentals/developer-apps)。你可以在[这里](https://developer.x.com/en/portal/petition/essential/basic-info)注册。
  </Accordion>

  <Accordion title="如果我已经有一个 X 应用，该如何查看和编辑这个应用？">
    如果你已在 developer.x.com 登录了你的 X 账号，你可以在 [X 应用仪表板](https://developer.x.com/content/developer-twitter/en/apps) 中查看和编辑你的应用。
  </Accordion>
</AccordionGroup>

<div id="technical">
  ## 技术
</div>

<AccordionGroup>
  <Accordion title="访问令牌的有效期是多长时间？">
    访问令牌没有被显式设置过期时间。如果用户在其 X 账户设置中明确撤销某个应用，或者 X 暂停了某个应用，则该访问令牌会失效。如果某个应用被暂停，在 [X 应用](/zh/resources/fundamentals/developer-apps) 仪表板中会有一条提示说明它已被暂停。
  </Accordion>

  <Accordion title="如果访问令牌失效怎么办？">
    要假定用户的访问令牌在任何时候都 *可能* 变得无效。如果发生这种情况，应提示用户重新授权该应用。妥善处理这种情况对于提供良好的用户体验非常重要。
  </Accordion>

  <Accordion title="应用注册页面询问读/写访问权限。什么算是写操作？">
    许多用户只信任应用读取他们的信息，但不一定信任其更改用户资料信息或发布新的帖子。通过 X API 更新信息——无论是名称、所在地还是发布新的帖子——都需要使用 HTTP POST。任何需要 HTTP POST 的 API 方法都被视为写方法，并且需要读写访问权限。
  </Accordion>
</AccordionGroup>
