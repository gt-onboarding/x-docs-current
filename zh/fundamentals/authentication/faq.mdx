<div id="general">
  ## 常规
</div>

<AccordionGroup>
  <Accordion title="什么是 OAuth？">
    OAuth 是一种身份验证协议，它允许用户在不共享密码的情况下，授权应用代表其执行操作。更多信息可在 [oauth.net](http://oauth.net/) 上查阅。
  </Accordion>

  <Accordion title="如何生成访问令牌？">
    你必须拥有一个 [X 应用](/zh/resources/fundamentals/developer-apps) 才能生成访问令牌。在[此处](/zh/resources/fundamentals/authentication#oauth-1-0a-2)了解更多关于访问令牌的信息。 
  </Accordion>

  <Accordion title="如何创建 X 应用？">
    你必须拥有一个 [开发者账号](/zh/resources/fundamentals/developer-portal) 才能创建 [X 应用](/zh/resources/fundamentals/developer-apps)。你可以在[这里](https://developer.x.com/en/portal/petition/essential/basic-info)注册一个。
  </Accordion>

  <Accordion title="如果我已经有一个 X 应用，如何查看和编辑该应用？">
    如果你已在 developer.x.com 使用你的 X 账号登录，你可以在 [X 应用控制台](https://developer.x.com/content/developer-twitter/en/apps) 中查看和编辑你的应用。
  </Accordion>
</AccordionGroup>

<div id="technical">
  ## 技术
</div>

<AccordionGroup>
  <Accordion title="访问令牌可以使用多长时间？">
    访问令牌不会被显式设定过期时间。如果用户在其 X 账号设置中明确撤销了某个应用，或者 X 暂停了某个应用，则该访问令牌会失效。如果某个应用被暂停，在 [X 应用](/zh/resources/fundamentals/developer-apps) 仪表板中会有一条说明，表明该应用已被暂停。
  </Accordion>

  <Accordion title="如果访问令牌失效怎么办？">
    应假设用户的访问令牌在任何时候都&#95;可能&#95;失效。如果出现这种情况，应提示用户重新授权该应用。确保优雅地处理这种情况，对于提供良好的用户体验非常重要。
  </Accordion>

  <Accordion title="应用注册页面会询问读/写访问权限。哪些操作被视为写入？">
    许多用户信任应用读取他们的信息，但未必信任其更改用户资料信息或发布新的帖子。通过 X API 更新信息——无论是名称、位置，还是发布一条新帖子——都需要发起 HTTP POST。任何需要 HTTP POST 的 API 方法都被视为写入方法，并且需要读写访问权限。
  </Accordion>
</AccordionGroup>
