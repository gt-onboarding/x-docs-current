要访问 X API 端点，必须使用 TLS 连接。通过 TLS 进行通信，可以在数据通过公共互联网传输时保护用户与 X API 之间的信息，从而保障用户的隐私和安全。连接到 X API 要求使用 TLS 1.2 版本。


<div id="verification">
  ## 验证
</div>

<div id="use-an-up-to-date-root-store">
  ### 使用最新的根证书存储
</div>

在验证 X 证书时，确保你的应用或库使用可信且最新的根证书存储非常重要。在可能的情况下，使用操作系统提供的根证书存储可能是最简单的做法。或者，[Mozilla (NSS) 根证书存储](https://www.mozilla.org/en-US/about/governance/policies/security-group/certs/) 以公开和透明的方式得到良好维护。cURL 也提供了 [PEM 格式版本的该证书存储](https://curl.haxx.se/docs/caextract.html)。

X 目前的大部分证书都是由 [DigiCert High Assurance EV Root CA](https://www.digicert.com/digicert-root-certificates.htm) 签发的，但这并不适用于 100% 的 X 相关证书，将来也不一定始终如此，因此，仅信任当前在用的 DigiCert 根证书，未来可能会导致你的应用出现问题。

<div id="check-crls-and-the-ocsp-status">
  ### 检查 CRL 和 OCSP 状态[](#check-crls-and-the-ocsp-status "Permalink to this headline")
</div>

许多应用不会检查返回证书的证书吊销列表（CRL），或者依赖操作系统来执行此操作。请确保你的应用或 TLS 库配置为在接受 X 的证书之前强制进行 CRL 和 OCSP（Online Certificate Status Protocol，在线证书状态协议）验证。

<div id="cdns">
  ### CDNs[](#cdns "该标题的永久链接")
</div>

在展示包含媒体的帖子时，请使用 `media_url_https` 属性提供的 HTTPS URL 来显示图片。未来，所有由 API 端点提供的 URL 都将提供 HTTPS 访问路径。

<div id="provide-an-indication-of-security-status">
  ## 提供安全状态指示
</div>

在条件允许的情况下，你应当显示你的应用与 X 之间当前连接的安全状态。一些 Web 浏览器会通过显示锁形图标来表示这一点，而其他浏览器则通过描述性消息来指示当前的连接状态。
