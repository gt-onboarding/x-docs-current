<div id="introduction">
  ## Introduction
</div>

项目允许你根据你计划如何使用 X API 来组织你的工作，从而有效管理你对 API 的访问并监控你的用量。每个项目可以包含一个或多个[应用](/zh/resources/fundamentals/developer-apps)，具体取决于你的访问级别（将在本页后文说明）。你将使用这些应用来生成[认证](/zh/resources/fundamentals/authentication)凭证，例如 [API Key 和 Secret](/zh/resources/fundamentals/authentication#api-key-and-secret)、用户[访问令牌](/zh/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)以及 [App 访问令牌](/zh/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)。

虽然你可以使用任何应用的这些密钥和令牌来访问 X API 或 X Ads API（要使用 Ads API，你必须[申请额外访问权限](/zh/x-ads-api/getting-started/step-by-step-guide)），但若要使用 [X API v2](/zh/x-api/introduction) 端点，你必须使用来自与某个项目关联的应用的密钥和令牌。

如果你拥有开发者账号，可以在开发者门户中的 [Projects &amp; Apps](https://developer.x.com/en/portal/projects-and-apps) 页面查看和管理你的项目。如果你还没有账号，请[注册](https://developer.x.com/en/portal/petition/essential/basic-info)。

<div id="projects-and-x-api-access-levels">
  ### 项目与 X API 访问级别
</div>

目前，项目可以配置 4 种不同的访问级别：

* Free
* Basic
* Pro
* Enterprise

要详细了解每个访问级别所提供的功能，请访问 [关于 X API](/zh/x-api/getting-started/about-x-api) 页面。

你最多只能拥有一个处于 Free、Basic 或 Pro 级别的项目。

下面我们将为你介绍一些特定于项目的差异：

<div id="free">
  #### 免费
</div>

此访问层级提供给任何已[注册](https://developer.twitter.com/en/portal/petition/essential/basic-info)开发者账号的用户。

该项目中的应用数量：1\
[帖子使用上限](/zh/x-api/fundamentals/post-cap)：每月 1,500 条帖子

认证方式：

* [OAuth 2.0 with PKCE](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
* 仅应用（App-only）

<div id="basic">
  #### Basic
</div>

此访问层级面向通过开发者门户订阅 Basic 访问权限的所有用户提供。

该项目中的应用数量：2\
[帖子调用上限](/zh/x-api/fundamentals/post-cap)：每月最多 10,000 条帖子\
身份验证方法：

* [OAuth 2.0 with PKCE](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
* [OAuth 1.0a](/zh/resources/fundamentals/authentication#oauth-1-0a-2)
* 仅应用（App only）

<div id="pro">
  #### Pro
</div>

此访问级别适用于通过开发者门户订阅了 Pro 访问权限的任何人。

该项目中可创建的应用数量：3\
[帖子用量上限](/zh/x-api/fundamentals/post-cap)：每月 100 万条帖子

认证方式：

* [OAuth 2.0 with PKCE](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
* [OAuth 1.0a](/zh/resources/fundamentals/authentication#oauth-1-0a-2)
* 仅应用（App only）

<div id="enterprise">
  #### Enterprise
</div>

此访问层级适用于任何已申请 Enterprise、经我们团队审批，并通过 X 客户经理签署了按月自动续约合同的用户。[在此申请](https://docs.google.com/forms/d/e/1FAIpQLScO3bczKWO2jFHyVZJUSEGfdyfFaqt2MvmOfl_aJp0KxMqtDA/viewform)。

该项目下的应用数量：3+
[帖子消耗上限](/zh/x-api/fundamentals/post-cap)：每月 5,000 万条以上帖子

认证方式：

* [OAuth 2.0 with PKCE](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
* [OAuth 1.0a](/zh/resources/fundamentals/authentication#oauth-1-0a-2)
* 仅限应用（App-only）

<div id="tweet-caps-and-projects">
  ### 帖子上限与项目
</div>

帖子获取上限适用于项目级别，它会限制你在任意给定月份内可以通过特定 X API v2 端点检索的帖子数量。

详细了解[帖子上限](/zh/x-api/fundamentals/post-cap)。

<div id="configuring-projects">
  ### 配置项目
</div>

<div id="creating-a-project">
  #### 创建项目
</div>

要创建项目，请在你的[仪表板](https://developer.twitter.com/en/portal/dashboard)或开发者门户中的 [Projects &amp; Apps](https://developer.twitter.com/en/portal/projects-and-apps) 页面上点击“New Project”。只有在你尚未创建任何项目时，才会看到此选项。系统会提示你填写项目名称、描述和用例。你还会被要求创建一个新的 App，或连接一个现有的独立 App。

<div id="creating-or-connecting-an-app-for-your-project">
  #### 为你的项目创建或关联应用
</div>

如果你的项目下还没有应用，你可以在仪表板中点击项目名称来添加一个应用。接下来，你可以创建一个新的应用，或者选择一个现有的独立应用并将其关联到你的项目。你可以在应用中生成本指南开头所列出的身份验证密钥和令牌。

<div id="editing-a-project">
  #### 编辑项目
</div>

要编辑项目，请在开发者门户的仪表盘或 “Projects &amp; Apps” 页面中点击你的项目名称。然后你会看到该项目的详细信息，并可以选择 “Edit” 来进行修改。

<div id="standalone-apps">
  ### 独立应用
</div>

独立应用是存在于项目结构之外的应用。与这些独立应用关联的认证凭据，可以成功向 X API 的标准版 v1.1、高级版 v1.1、Enterprise 或 X Ads API 发起请求。除非你将它们连接到某个项目，否则独立应用在尝试向 X API v2 端点发起请求时会失败。

如果你在 2020 年 8 月之前创建了应用，它们会显示在开发者门户中 [Projects &amp; Apps](https://developer.twitter.com/en/portal/projects-and-apps) 下的“Standalone Apps”部分。你的应用总数上限为十个，其中包括已连接到你项目的应用。

如果你是团队账户的一员，你会在 Standalone Apps 部分看到你自己拥有的应用。如果某位队友拥有一个属于某个项目的应用，你可以看到该应用的名称和所有者信息，但你无法更改其设置、访问其密钥和令牌，或重新生成其密钥和令牌。若要对该应用进行任何更改，你应联系该应用的所有者。
