  ## 概览
</div>

<div id="introduction">
  ### 介绍
</div>

X 开发者门户提供了一套自助式工具，开发者可以用它来管理自己对 X API 和 X Ads API 的访问权限。

在门户中，你可以：

* 创建并管理你的 X [Projects](/zh/resources/fundamentals/projects) 和 [Apps](/zh/resources/fundamentals/developer-apps)（以及它们所提供的认证密钥和令牌）。
* 管理你在 X API 标准版 v1.1 和 v2 端点上的访问级别以及与这些端点的集成。
* 进一步了解可用的不同端点和功能。

<div id="ready-to-get-access">
  ### 准备好获取访问权限了吗？
</div>

你可以先注册一个账号，即可开始使用 X API。

如果你需要更多功能或更高的 [帖子上限](/zh/x-api/fundamentals/post-cap)，可以在开发者门户中购买 Basic 或 Pro。对 Enterprise 感兴趣的用户，请在[此处](https://docs.x.com/enterprise/forms/enterprise-api-interest)提交申请。

  ## 开发者门户中你将会看到什么
</div>

<div id="onboarding">
  ### 入门引导
</div>

入门向导会引导你完成设置首个[项目](/zh/resources/fundamentals/projects)和[应用](/zh/resources/fundamentals/developer-apps)的流程。你需要创建一个项目和应用，以获取对 API 请求进行身份验证所需的凭证。如果这是你首次访问开发者账号，你将会看到该向导。

在此过程中，你会获得一组用于身份验证的密钥和令牌，你可以在我们的[应用概览页面](/zh/resources/fundamentals/developer-apps#overview)了解更多相关信息。若要进一步了解使用 X API 进行身份验证所需的内容，请参阅[身份验证部分](/zh/resources/fundamentals/authentication)。

**请注意：**你需要[将密钥和令牌存储在安全的位置](/zh/resources/fundamentals/authentication/guides/authentication-best-practices)，以便之后可以访问它们。如果不重新生成这些凭证，将无法再次查看它们。

<div id="project-and-app-management">
  ### 项目和应用管理
</div>

开发者账号的主要职责之一，是让你可以[管理你的项目和应用](/zh/resources/fundamentals/developer-apps#app-management)。开发者可以在开发者门户的仪表板中创建和管理 X 项目和应用。在这里，你可以找到自己的 App ID，编辑应用的设置、权限和回调 URL，并生成或撤销密钥和令牌。

<div id="learn-more-about-whats-available-with-x-api-v2">
  ### 进一步了解 X API v2 提供的功能
</div>

开发者门户中有一个产品板块，你可以在其中了解 X API 的不同版本和访问级别。

X API v2 的产品板块包含有关 Free、Basic、Pro 和 Enterprise 访问层级的重要信息。本页面说明了项目层面的应用限制、帖子上限和费用，以及特定端点的速率限制和特殊属性。你还可以对比不同的访问级别，并在可用时申请额外访问权限。

查看[所有 v2 访问级别](/zh/x-api/introduction#access-levels)。

<div id="team-management">
  ## 团队管理
</div>

<Note>
  **仅适用于 Enterprise v2 账号。**
</Note>

[**在开发者门户中查看“团队”页面。**](https://developer.twitter.com/en/portal/teams)

<div id="why-use-team-functionality">
  ### 为什么要使用团队功能？
</div>

团队功能有助于在 X 开发者平台内协作开发项目和应用。通常，团队中会有不同的人负责访问控制、账单/付款，你可以通过团队功能邀请这些成员参与到你的项目中。

<div id="inviting-team-members">
  ### 邀请团队成员
</div>

要邀请某人加入团队，管理员可以通过对方的 X 账号发送邀请。受邀人会收到一封电子邮件，并可在邮件中接受邀请。接受后，他们需要同意 Developer Agreement &amp; Policy，然后即可访问主账户的团队页面。

**请注意：** 团队管理目前不会基于 [App](/zh/resources/fundamentals/developer-apps) 凭证授予或限制 API 访问权限。无法在团队成员之间共享 App 管理权限。非所有者无法编辑、创建或删除 App（密钥/令牌）。

<div id="team-dashboard">
  ### 团队仪表板
</div>

在团队仪表板的“members”选项卡中，你可以查看所有成员及其角色。如果你是团队管理员，则可以管理开发者访问权限并编辑每位成员的角色。

管理员还可以访问团队仪表板的“pending”选项卡。在这里，管理员可以查看每个邀请的详细信息并管理每一条已发送的邀请。

<div id="team-roles">
  ### 团队角色
</div>

**管理员角色**

* 可管理团队项目和应用
* 可管理所有应用环境
* 可选择或升级订阅
* 可更新结算和付款方式
* 可添加或移除团队成员
* 可修改团队成员的角色

**开发者角色**

* 可管理自己的项目和应用
* 对团队项目和应用具有只读访问权限
* 可退出团队
