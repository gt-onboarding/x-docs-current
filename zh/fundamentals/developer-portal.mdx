  ## 概览
</div>

<div id="introduction">
  ### 简介
</div>

X 开发者门户提供了一套自助式工具，开发者可以用来管理自己对 X API 和 X Ads API 的访问。

在该门户中，你可以：

* 创建和管理你的 X [项目](/zh/resources/fundamentals/projects)和[应用](/zh/resources/fundamentals/developer-apps)（以及它们提供的认证密钥和令牌）。
* 管理你对 X API 标准版 v1.1 和 v2 端点的访问级别和集成。
* 进一步了解可用的不同端点和功能。

<div id="ready-to-get-access">
  ### 准备好获取访问权限了吗？
</div>

你可以通过注册账号开始使用 X API。

如果你需要更多功能或更高的[帖子上限](/zh/x-api/fundamentals/post-cap)，可以在开发者门户中购买 Basic 或 Pro。对 Enterprise 感兴趣的用户，请在[此处](https://docs.x.com/enterprise/forms/enterprise-api-interest)提交申请。

  ## 在开发者门户中你可以找到什么
</div>

<div id="onboarding">
  ### 入门引导
</div>

入门向导会引导你完成设置第一个[项目](/zh/resources/fundamentals/projects)和[应用](/zh/resources/fundamentals/developer-apps)的流程。你需要创建一个项目和应用，才能获取对你的 API 请求进行身份验证所需的凭证。如果你是第一次访问你的开发者账号，将会看到该向导。

在此过程中，你会收到一组用于身份验证的密钥和令牌，你可以在我们的[应用概览页面](/zh/resources/fundamentals/developer-apps#overview)中进一步了解它们。若要了解与 X API 进行身份验证所需的内容，请查看[认证章节](/zh/resources/fundamentals/authentication)。

**请注意：** 你需要[将你的密钥和令牌存储在安全的位置](/zh/resources/fundamentals/authentication/guides/authentication-best-practices)，以便之后可以访问它们。如果不重新生成，就无法再次查看这些凭证。

<div id="project-and-app-management">
  ### 项目和应用管理
</div>

开发者账号的主要职责之一是让你可以[管理你的项目和应用](/zh/resources/fundamentals/developer-apps#app-management)。开发者可以通过开发者门户中的仪表板创建和管理 X 项目和应用。在这里，你可以找到你的 App IDs，编辑应用的设置、权限和回调 URL，并生成或撤销密钥和令牌。

<div id="learn-more-about-whats-available-with-x-api-v2">
  ### 详细了解 X API v2 提供的功能
</div>

开发者门户中包含一个产品板块，你可以在其中了解 X API 不同版本和访问级别的更多信息。

X API v2 产品板块包含关于 Free、Basic、Pro 和 Enterprise 访问层级的重要信息。本页面详细说明项目级应用限制、帖子数量上限和费用，以及各端点的速率限制和特殊属性。你还可以对比不同的访问级别，并在可用时申请额外访问权限。

查看[所有 v2 访问级别](/zh/x-api/introduction#access-levels)。

<div id="team-management">
  ## 团队管理
</div>

<Note>
  **仅 Enterprise v2 账户可用。**
</Note>

[**在开发者门户中查看“Team”页面。**](https://developer.twitter.com/en/portal/teams)

<div id="why-use-team-functionality">
  ### 为什么要使用团队功能？
</div>

团队功能有助于在 X 开发者平台中协同开发项目和应用。通常，团队中会有不同的成员分别负责访问控制、计费和付款，通过该功能你可以邀请这些成员加入你的项目，共同协作。

<div id="inviting-team-members">
  ### 邀请团队成员
</div>

要邀请某人加入团队，管理员可以通过对方的 X 账号邀请他们。对方会收到一封电子邮件，并可以通过邮件中的邀请接受加入。一旦接受，他们需要同意《Developer Agreement &amp; Policy》，然后即可访问主账户的团队页面。

**请注意：** 团队管理目前不会基于 [App](/zh/resources/fundamentals/developer-apps) 凭证授予或限制 API 访问权限。无法在团队中共享 App 管理权限。非所有者无法编辑、创建或删除 App（密钥/令牌）。

<div id="team-dashboard">
  ### 团队仪表板
</div>

在团队仪表板的 &quot;members&quot; 标签中，您可以查看所有成员及其角色。如果您是某个团队的管理员，便可以管理开发者访问权限并编辑每位成员的角色。

管理员还可以访问团队仪表板中的 &quot;pending&quot; 标签。在这里，管理员可以查看并管理每个已发送邀请的详细信息。

<div id="team-roles">
  ### 团队角色
</div>

**管理员角色**

* 可管理团队项目和应用
* 可管理所有应用环境
* 可选择或升级订阅方案
* 可更新结算和支付方式
* 可添加或移除团队成员
* 可编辑团队成员的角色

**开发者角色**

* 可管理自己的项目和应用
* 对团队项目和应用仅有只读访问权限
* 可退出团队
