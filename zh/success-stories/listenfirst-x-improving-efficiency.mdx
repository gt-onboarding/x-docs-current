<div id="situation">
  ## 情况
</div>

一直以来，数字营销人员都缺乏对其数据的统一视图。一个系统里可能是自然数据，另一个系统里是付费数据，对话则来自不同的渠道。但当客户和利益相关方提出诸如“我昨天获得了多少曝光量？”以及接着问“自然与付费的占比分别是多少？”这样的问题时，对一种高效汇总和解读这些数据的方式的需求比以往任何时候都更为迫切。

为了解决这些问题，已经基于 X Developer Platform 构建超过 10 年的社交分析平台 ListenFirst 需要创建一个工具，从单一真实数据源收集自然和付费的 X 数据。过去，这些自然和付费数据分散在不同的平台、API 和接口端点之中，彼此割裂；而且最初也从未被设计成可以被汇总、统计和/或整合在一起。

<div id="solution">
  ## 解决方案
</div>

在寻找更高效的方法为客户挖掘洞察的过程中，ListenFirst 采用了 X 的 v2 端点，更顺畅地将用户在 X 上的讨论内容转化并引入其平台。ListenFirst 团队带着一个明确的目标开始推进工作：打造更干净的数据。

> *“对我们来说，使用 v2 是为了优化集成。如果我们在自己的网站上需要拼接的部分更少，运营就会更容易，也能让我们的产品扩展到更多品牌、更多内容，并产出更优质的洞察。”*
> 
> - Jonathan Farb，ListenFirst 首席产品官

驱动团队优先采用 X 的 v2 API，尤其是 Tweets lookup 端点的关键因素，是其在互动量和曝光量数据上的准确性。通过使用这一端点，ListenFirst 能够生成更可靠的指标数据，并与其客户在其他 X 原生分析产品中看到的数据保持一致。

<div id="impact">
  ## 影响
</div>

虽然仍处于起步阶段，但采用 v2 已经显著提升了 ListenFirst 的效率。过去，ListenFirst 不得不从多套 API 中拼凑数据，而且这些数字与原生分析中的结果并不一致。现在对 v2 的使用，使 ListenFirst 的数据与 X 原生分析的数据更加一致，从而提高了准确性并增强了客户信任。如今，ListenFirst 可以产出有依据且精细的情报，帮助品牌在 X 上完善其营销策略。ListenFirst 认为，进一步采用 v2 将通过提升准确性和透明度，增强客户对指标的信心。
