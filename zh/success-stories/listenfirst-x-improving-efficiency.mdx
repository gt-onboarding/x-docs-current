<div id="situation">
  ## 情况
</div>

长期以来，数字营销人员一直缺乏对其数据的统一视角。一个系统里可能是自然（非付费）数据，另一个系统里是付费数据，而用户对话则来自不同的渠道。但当客户和相关方提出诸如“我昨天获得了多少展示量？”紧接着又问“自然与付费的占比是多少？”这类问题时，如何以一种高效的方式来汇总和解读这些数据，就变得比以往任何时候都更为重要。

为了解决这些问题，已经基于 X Developer Platform 构建超过 10 年的社交分析平台 ListenFirst 需要创建一款工具，从单一可信的数据源收集自然和付费的 X 数据。过去，这些自然和付费数据分散在不同的平台、API 和端点中，彼此割裂；而且从来就不是为了被聚合、求和和/或整合在一起而设计的。

<div id="solution">
  ## 解决方案
</div>

在探索更高效的方法为客户产出洞察的过程中，ListenFirst 采用了 X 的 v2 endpoint，将用户在 X 上的讨论内容更顺畅地转化并接入到其平台中。ListenFirst 团队带着一个明确的目标开始行动：打造更干净的数据。

> *“对我们而言，使用 v2 是为了优化集成。如果我们在自己的网站上需要拼接的模块更少，运营就会更简单，也能让我们将产品扩展到更多品牌、更多内容，并提供更优质的洞察。”*
> 
> - Jonathan Farb，ListenFirst 首席产品官。

促使团队优先采用 X 的 v2 API，尤其是 Tweets lookup endpoint 的关键原因，是其在互动和曝光量指标方面的高准确性。通过使用该 endpoint，ListenFirst 得以生成更加可靠的指标，并与其客户在 X 原生分析工具中看到的数据保持一致。

<div id="impact">
  ## 影响
</div>

虽然还处于早期阶段，但采用 v2 已经显著提升了 ListenFirst 的效率。过去，ListenFirst 不得不从不同的 API 集合中拼接数据，而且这些数字与原生分析中看到的结果并不一致。现在使用 v2，使 ListenFirst 提供的数据更接近 X 原生分析，实现了更高的一致性，从而提高了准确性和客户信任度。如今，ListenFirst 能够产出有理有据且更精细的情报洞察，帮助品牌完善其在 X 上的营销策略。ListenFirst 认为，进一步采用 v2 将通过提高准确性和透明度来增强客户对指标的信心。
