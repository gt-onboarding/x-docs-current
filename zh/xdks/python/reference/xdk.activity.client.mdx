此模块提供用于与 X API 的 activity 端点交互的 ActivityClient 客户端。
实时流式操作会返回生成器，在数据到达时依次产出数据。
流式连接会自动管理，并采用指数退避的重试逻辑，以确保处理过程的健壮性。

<div id="stream">
  ### `stream`
</div>

活动流（Streaming）

X 活动的实时流
这是一个流式端点，会在数据可用时实时返回数据。
每条返回的数据表示该流中的一个独立数据点。
连接由带指数退避的重试逻辑自动管理。
如果流断开连接，SDK 会在无需客户端干预的情况下自动重新连接。

<div id="parameters">
  #### 参数
</div>

<ParamField path="path.backfill_minutes" type="int or None" default="None">
请求的回补时间长度（分钟数）。
</ParamField>

<ParamField path="path.start_time" type="str or None" default="None">
YYYY-MM-DDTHH:mm:ssZ。从该时间起开始提供帖子标签的最早 UTC 时间戳。
</ParamField>

<ParamField path="path.end_time" type="str or None" default="None">
YYYY-MM-DDTHH:mm:ssZ。截止到该时间提供帖子标签的最晚 UTC 时间戳。
</ParamField>

<ParamField path="path.stream_config" type="StreamConfig">
可选的 StreamConfig，用于自定义重试行为、超时时间和回调。
</ParamField>
