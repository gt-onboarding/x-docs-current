<div id="overview">
  ## 概览
</div>

一个媒体对象（media object）表示一张照片、一个视频或一个动态 GIF。媒体对象会被 Twitter API 中的许多端点使用，并且可以出现在帖子、私信、用户资料、广告素材等场景中。每个媒体对象可以包含多个用于展示或播放的变体，具有不同的分辨率或格式。

<div id="media-types-size-restrictions">
  ## 媒体类型与大小限制
</div>

通过 API 上传的大小限制如下：

- 图片 5 MB
- GIF 15 MB
- 视频 512 MB（使用 media_category=amplify 时）

<div id="creation">
  ## 创建
</div>

帖子、私信、用户头像、托管广告卡片等对象可以包含一个或多个媒体对象。这些顶层对象统称为实体（entity）。相关的实体创建 API（例如 `POST statuses/update`）可以通过唯一的 `media_id` 传递一个或多个媒体对象。

包含媒体对象的实体可以通过以下步骤创建：

1. 使用推荐的分块上传（图片/GIF/视频）或较早的简单上传（仅限图片）上传媒体文件。
2. 从步骤 1 中获取一个 `media_id`。如果实体允许传入多个 `media_id` 参数，则可以针对不同媒体多次重复此步骤。
3. 调用相应的 endpoint 来创建实体，并在请求中包含 `media_id` 和其他必填参数。例如，使用 `POST statuses/update` endpoint 将 `media_id` 附加到帖子。
