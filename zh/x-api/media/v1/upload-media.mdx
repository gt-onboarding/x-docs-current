<div id="overview">
  ## 概述
</div>

一个媒体对象表示一张照片、一个视频或一个动态 GIF。媒体对象会在 Twitter API 的许多端点中使用，并且可能被包含在帖子、私信、用户资料、广告创意等内容中。每个媒体对象可以具有多个用于展示或播放的变体，使用不同的分辨率或格式。

<div id="media-types-size-restrictions">
  ## 媒体类型和大小限制
</div>

通过 API 上传的媒体大小限制 

- 图片：5 MB
- GIF：15 MB
- 视频：512 MB（使用 media_category=amplify 时）

<div id="creation">
  ## 创建
</div>

帖子、私信、用户头像、托管广告卡片等对象可以包含一个或多个媒体对象。这些顶层对象统称为实体（entity）。相关的实体创建 API（例如 `POST statuses/update`）可以通过传入一个或多个唯一的 `media_id` 来关联媒体对象。

包含媒体对象的实体可以通过以下步骤创建：

1. 使用推荐的分块上传方式（图片/GIF/视频），或较旧的简单上传方式（仅图片）来上传媒体文件。
2. 从第 1 步中获取一个 `media_id`。如果该实体允许传入多个 `media_id` 参数，则可以针对不同的媒体多次重复此步骤。
3. 调用相应的端点创建实体，并包含 `media_id` 和其他必需参数。例如，使用 `POST statuses/update` 端点将 `media_id` 附加到一条帖子中。
