媒体对象表示单张照片、视频或动图 GIF。媒体对象被 X API 中的许多端点使用，并且可以包含在帖子、私信、用户资料、广告创意等内容中。每个媒体对象都可以有多个用于展示或播放的变体，具有不同的分辨率或格式。

<div id="media-types-size-restrictions">
  ## 媒体类型和大小限制
</div>

通过 API 上传时的大小限制

* **图片**：`5 MB`
* **GIF**：`15 MB`
* **视频**：`512 MB`（当使用 `media_category=amplify_video` 时）

<div id="creation">
  ## 创建
</div>

帖子、私信、用户头像、托管 Ads 卡片等对象都可以包含一个或多个媒体对象。这些顶层对象统称为实体。相关的实体创建 API（例如 [`POST /2/tweets`](/zh/x-api/posts/creation-of-a-post)）可以通过传入一个或多个具有唯一 `media_id` 的媒体对象来完成。

包含媒体对象的实体可以按照以下步骤创建：

1. 使用推荐的[分块](/zh/x-api-/media/quickstart/media-upload-chunked)上传方式（图片/GIF/视频），或较早的[简单](/zh/x-api/media-upload/media-upload)上传方式（仅图片）来上传媒体文件。
2. 从第 1 步的响应中获取一个 `media_id`。如果该实体允许传入多个 `media_id` 参数，则可以针对不同的媒体多次重复此步骤。
3. 调用相应的端点创建实体，并在请求中包含 `media_id` 和其他必需参数。例如，使用 [`POST /2/tweets`](/zh/x-api/posts/creation-of-a-post) 端点将 `media_id` 附加到帖子。
