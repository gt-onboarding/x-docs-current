媒体对象代表单张照片、一个视频或一张动图 GIF。媒体对象会被 X API 中的许多端点使用，并且可以被包含在帖子、私信、用户资料、广告创意等内容中。每个媒体对象可以具有多个用于展示或播放的版本，采用不同的分辨率或格式。

<div id="media-types-size-restrictions">
  ## 媒体类型和大小限制
</div>

通过 API 上传时的大小限制

* **图片**：`5 MB`
* **GIF**：`15 MB`
* **视频**：`512 MB`（使用 `media_category=amplify_video` 时）

<div id="creation">
  ## 创建
</div>

帖子、私信、用户头像、托管 Ads 卡片等对象可以包含一个或多个媒体对象。这些顶层对象统称为实体。相关的实体创建 API（例如 [`POST /2/tweets`](/zh/x-api/posts/creation-of-a-post)）可以通过唯一的 `media_id` 传入一个或多个媒体对象。

要创建包含媒体对象的实体，请按照以下步骤操作：

1. 使用推荐的[分片](/zh/x-api-/media/quickstart/media-upload-chunked)上传（图像/GIF/视频），或较早的[简单](/zh/x-api/media-upload/media-upload)上传（仅图像）来上传媒体文件。
2. 从步骤 1 中获取一个 `media_id`。如果该实体允许传入多个 `media_id` 参数，则可以使用不同的媒体多次重复此步骤。
3. 调用相应的 API 端点创建实体，并在请求中包含 `media_id` 及其他必需参数。例如，使用 [`POST /2/tweets`](/zh/x-api/posts/creation-of-a-post) 端点将 `media_id` 附加到帖子中。
