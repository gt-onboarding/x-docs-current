搜索帖子是一项重要功能，用于展示关于特定主题或事件的 X 对话。虽然 X 本身也提供此功能，但这些端点在筛选和获取帖子时提供了更高的灵活性和能力，从而让你更轻松地为研究找到相关数据、构建近实时的“监听”应用，或者更广泛地探索、分析以及对与某个感兴趣主题相关的帖子采取行动。 

我们提供两个用于搜索帖子的端点：最近搜索和完整存档搜索。这两个 REST 端点具有相同的设计和特性，包括它们都使用单个搜索查询来筛选围绕特定主题的帖子。这些搜索查询是通过一组运算符构建的，这些运算符会匹配帖子和用户属性，例如消息关键词、话题标签和 URL。运算符可以通过布尔逻辑和括号组合成查询，以帮助细化查询的匹配行为。 

最近搜索和完整存档搜索端点都会提供已编辑帖子的元数据。自 2022 年 9 月 29 日起创建的所有帖子对象都包含帖子编辑元数据，即便该帖子从未被编辑。每次帖子被编辑时，都会创建一个新的 Post ID。帖子的编辑历史会通过一个从原始 ID 开始的 Post ID 数组记录。

这些端点始终会返回最新的编辑版本，以及完整的编辑历史。在帖子 30 分钟编辑窗口之后收集到的任何帖子都将代表其最终版本。要进一步了解编辑帖子元数据，请查看 [Edit Posts fundamentals](/zh/x-api/fundamentals/edit-posts) 页面。

一旦你设置好查询并开始接收帖子，这些端点支持按时间和 Post ID 范围浏览结果。这样设计是为了支持两种常见用例： 

* **获取历史数据**：请求针对某个感兴趣的时间段，对数据的实时性没有要求。会发出单个请求，并根据需要通过分页返回所有匹配的数据。这是 Search Posts 端点的默认模式。
* **轮询** 或 **监听**：以“自上次请求以来有没有新的帖子？”的方式发出请求。请求会持续发出，通常用于围绕感兴趣的帖子进行近实时“监听”的用例。

许多运算符和查询限制仅适用于 [Enterprise access](/zh/x-api/getting-started/about-x-api)，这意味着你必须使用一个位于具有 Enterprise access 的[项目](/zh/resources/fundamentals/projects)中的应用的密钥和令牌，才能使用这些附加功能。你可以在下面的端点部分中进一步了解这方面内容。

最近搜索和完整存档搜索端点返回的帖子都会计入每月的 [Post cap](/zh/x-api/fundamentals/post-cap)。

**账号设置**

要访问这些端点，你需要：

* 一个已获批准的[开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)。
* 使用位于某个[项目](/zh/resources/fundamentals/projects)中的[开发者应用](/zh/resources/fundamentals/developer-apps)的密钥和令牌进行认证。 

在我们的[入门指南](/zh/x-api/getting-started/getting-access)中了解有关获取 X API v2 端点访问权限的更多信息。

<div id="recent-search">
  ## 最近搜索
</div>

最近搜索端点允许你以编程方式访问过去一周内发布的、经过过滤的公开帖子。所有拥有开发者账号，并在某个[应用](/zh/resources/fundamentals/developer-apps)所属的[项目](/zh/resources/fundamentals/projects)中使用密钥和令牌的开发者都可以使用该端点。

你可以使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication)、[OAuth 2.0 应用仅模式](/zh/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) 或 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 来对请求进行身份验证。不过，如果你希望在帖子结果中接收私有指标，或获取自然指标和推广指标的细分数据，你必须使用 OAuth 1.0a 用户上下文或 OAuth 2.0 Authorization Code with PKCE，并传递与发布该内容的用户关联的用户访问令牌。

该端点每次请求最多可按时间倒序返回 100 条帖子，并提供[分页](/zh/x-api/posts/search/integrate/paginate)令牌，用于浏览大规模匹配帖子的结果集。

当使用具有常规访问权限的项目时，你可以使用基础[运算符](/zh/x-api/posts/search/integrate/build-a-query)集合，并可发起最长为 512 个字符的查询。当使用具有 Enterprise 访问权限的项目时，你可以使用额外的运算符。具有 Enterprise 访问权限的项目可发起最长为 4096 个字符的查询。

详细了解[访问级别](/zh/x-api/getting-started/about-x-api)。

<div id="full-archive-search">
  ## 全量存档搜索
</div>

v2 全量存档搜索端点仅对具有 [Pro](/zh/x-api/getting-started/about-x-api "Pro") 访问权限和 [Enterprise](/zh/x-api/getting-started/about-x-api) 访问权限的项目开放。该端点允许你根据搜索查询，以编程方式访问自 2006 年 3 月第一条帖子以来的完整存档中的公开帖子。

你可以使用 [OAuth 2.0 App-Only](/zh/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) 为对此端点的请求进行身份验证，并且 [App Access Token](/zh/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token) 必须来自某个位于具有 Pro 或 Enterprise 访问权限的项目下的 App。由于你不能使用此端点代表其他用户（OAuth 1.0a 用户上下文或 OAuth 2.0 Authorization Code with PKCE）发起请求，因此你将无法获取私有[指标](/zh/x-api/fundamentals/metrics)。

此端点每次请求最多可按时间倒序返回 500 条帖子，并提供[分页](/zh/x-api/posts/search/integrate/paginate)令牌，以便在大量匹配帖子中进行翻页。

**注意：** 如果通过 tweet.fields 参数请求 [annotations](/zh/x-api/fundamentals/post-annotations)，max&#95;results 参数目前的最大取值限制为 100。此限制未来可能会变更，但请注意当前的这一限制。

由于此端点仅对已获批 Pro 和 Enterprise 访问权限的用户开放，你可以使用完整的搜索[操作符](/zh/x-api/posts/search/integrate/build-a-query)集合，并且可以构造最长 1024 个字符的查询。
