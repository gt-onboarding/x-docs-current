这三个时间线端点支持已编辑的帖子。这些端点始终会返回最新的编辑版本以及完整的编辑历史。在 30 分钟编辑窗口结束后获取的任何帖子都将代表其最终版本。编辑元数据包含一个数组，其中存放其历史中所有帖子的 ID。对于没有编辑历史的帖子，此数组中只包含一个 ID。对于已经编辑过的帖子，此数组包含多个 ID，按升序排列，反映各次编辑的顺序，数组中的最后一个元素为最新版本。要进一步了解帖子编辑的工作机制，请参阅 [Edit Posts fundamentals](/zh/x-api/fundamentals/edit-posts) 页面。

<div id="reverse-chronological-home-timeline">
  ## 按时间倒序排列的主页时间线
</div>

此 endpoint 允许你获取经过身份验证的用户及其所关注账号最近发布的帖子、转发和回复。

由于你是在代表某个用户发起请求，必须使用 [OAuth 2.0 Authorization Code Flow with PKCE](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) 或 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 对这些 endpoint 进行身份验证。此 endpoint 具有按用户计的速率限制：每个用户在每个 15 分钟窗口内最多可发起 180 次请求。此 endpoint 可以返回时间线上最近 7 天内创建的所有帖子，以及不受创建日期限制的最近 800 条帖子。

<div id="user-post-timeline">
  ## 用户帖子时间线
</div>

用户帖子时间线端点提供对特定 X 账户已发布帖子（Posts）的访问能力。获取某个用户的帖子可以让你构建诸如在用户界面中展示时间线、分析用户的帖子以更好地理解其内容，或以编程方式基于其帖子创建互动工作流等体验。该端点使你能够访问单个 X 账户最近的帖子、转发（Retweet）、回复和引用转发（Quote Tweet），类似于在用户个人资料时间线中所看到的内容。

用户帖子时间线端点是一个 REST 端点，它接收一个路径参数来指明目标用户（按用户 ID）。该端点可以返回该用户最近的 3,200 条帖子、转发（Retweet）、回复和引用转发（Quote Tweet）。

帖子按照时间倒序返回，从最新的开始。[分页](/zh/x-api/fundamentals/pagination)后每页最多 100 条帖子。会提供分页令牌，便于在大量帖子中进行翻页。还会将当前页面中最新和最旧帖子的帖子 ID 作为元数据提供，这些 ID 也可用于轮询时间线以获取最新帖子。用户帖子时间线还支持指定 `start_time` 和 `end_time` 参数，以接收在特定时间窗口内创建的帖子。 

用户帖子时间线端点支持 [fields](/zh/x-api/fundamentals/fields) 和 [expansions](/zh/x-api/fundamentals/expansions) 参数，并返回[新的 JSON 数据格式](/zh/x-api/fundamentals/data-dictionary)。

要成功向该端点发起请求，你需要使用 [OAuth 1.0a User Context](/zh/resources/fundamentals/authentication)、[OAuth 2.0 Authorization Code with PKCE](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) 或 [OAuth 2.0 App-Only](/zh/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) 认证方式对请求进行授权。当请求非公开指标、推广指标或受保护用户的时间线时，你必须使用 OAuth 1.0a User Context 或 OAuth 2.0 Authorization Code with PKCE。 

用户帖子时间线端点旨在支持两种常见的使用模式： 

* “获取用户的历史帖子”：向用户帖子时间线发起请求，以在特定的近期时间范围内按时间顺序接收目标用户创作的帖子。可以通过设置 `start_time` 和 `end_time` 并对完整结果进行分页来指定时间范围。在某些情况下，如果用户在其账户中创作的帖子不超过 3,200 条，则可以检索该用户完整的发帖历史。所包含的帖子将取决于其公开状态以及你对请求所使用的认证方式。

* “轮询新帖子”：持续地向用户帖子时间线发起请求，以检索某个特定用户新创作的帖子。你可以将上次收到的最后一个帖子 ID 作为参数，用于请求自上次帖子以来的新帖子。

<div id="user-mention-timeline">
  ## 用户提及时间线
</div>

用户提及时间线端点允许你检索包含特定 X 用户提及的帖子，例如，当某个 X 账号在一条帖子中提到了 @XDevelopers 时。结果还会包含对该用户帖子所作的回复。获取某个用户的提及信息，可以让你构建诸如快速发现是谁在回复某个用户的帖子、谁在提及他们，或以编程方式构建与其帖子互动的工作流等体验。该端点允许你请求单个用户最近的提及和回复，类似于在 X 上用户的[提及通知](https://x.com/notifications/mentions)中所见的内容。

用户提及时间线是一个 REST 端点，它接受一个路径参数，用于指定目标用户（通过用户 id）。该端点最多可以返回该用户最近的 800 条提及。

帖子按时间倒序返回，从最新的开始。结果会[分页](/zh/x-api/fundamentals/pagination)，每页最多 100 条帖子。会提供分页令牌，用于在大量帖子中分页浏览。给定页面中包含的最新和最旧帖子的帖子 ID 也会作为元数据提供，你可以利用这些信息来轮询时间线中的最新帖子，或像在 v1.1 [mentions&#95;timeline](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-mentions_timeline) 端点中那样在时间线中导航。该端点还支持指定 start&#95;time 和 end&#95;time 参数，以便接收在特定时间窗口内创建的帖子。 

要成功向该端点发出请求，你需要使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication)、[OAuth 2.0 Authorization Code with PKCE](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) 或 [OAuth 2.0 应用仅模式](/zh/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) 这几种认证方式之一来为你的请求授权。当你请求非公开指标、推广指标或受保护用户的时间线时，必须使用 OAuth 1.0a 用户上下文或 OAuth 2.0 Authorization Code with PKCE。

用户提及时间线端点支持 [fields](/zh/x-api/fundamentals/fields) 和 [expansions](/zh/x-api/fundamentals/expansions) 参数，并返回[新的 JSON 数据格式](/zh/x-api/fundamentals/data-dictionary)。
