<div id="retweets-lookup">
  ### 转推查询
</div>

使用 Retweets 查询端点，你可以获取转发了某条帖子的账号列表。对于此端点，会提供分页令牌，以便将大型结果集按每批最多 100 个用户进行分页浏览。 

你可以使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication)、[OAuth 2.0 应用仅模式](https://developer.x.com(/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) 或 [OAuth 2.0 Authorization Code with PKCE](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) 对这些端点进行身份验证。 

<div id="retweets-of-me">
  ### 转发我的帖子
</div>

“转发我的帖子”查询可以让你查看哪些你自己的帖子被其他人分享。

这可以帮助用户了解哪些帖子正在被分享并引起受众共鸣，从而便于开展更有针对性的互动并调整内容策略。

你可以使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 或 [OAuth 2.0 Authorization Code with PKCE](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 来对这些端点进行身份验证。

<div id="manage-retweets">
  #### 管理转推
</div>

管理转推端点可以让你代表已认证账号对指定的帖子进行转推或撤销转推。对于这一端点组，有两种可用的 HTTP 方法：POST 和 DELETE。POST 方法用于转推帖子，DELETE 方法用于撤销对指定帖子的转推。

由于你是代表某个用户发起请求，因此必须使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 或 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 对这些端点进行认证，并使用与该被代表用户关联的用户访问令牌。你可以通过 [三方 OAuth 流程](/zh/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（OAuth 1.0a）或 [Authorization Code with PKCE grant flow](/zh/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)（OAuth 2.0）生成该用户访问令牌。你可以从自己的账号或其他已认证用户的账号转推一条帖子。对于这两个端点，用户级速率限制为每个端点每 15 分钟 50 个请求。
