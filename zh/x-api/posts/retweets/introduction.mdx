<div id="retweets-lookup">
  ### 转帖查询
</div>

通过 Retweets lookup 端点，你可以检索对某个帖子执行转帖操作的账号列表。对于此端点，将提供分页令牌，用于以最多 100 个用户为一批，对大型结果集进行分页检索。 

你可以使用 [OAuth 1.0a User Context](/zh/resources/fundamentals/authentication)、[OAuth 2.0 App-Only](https://developer.x.com(/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) 或 [OAuth 2.0 Authorization Code with PKCE](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) 对该端点进行身份验证。 

<div id="retweets-of-me">
  ### 转发我的帖子
</div>

“转发我的帖子”查询功能可以让你查看有哪些自己的帖子被其他人分享。

这可以帮助用户了解哪些帖子正在被分享并引起受众共鸣，从而促进更有针对性的互动和内容策略调整。

你可以使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 或 [OAuth 2.0 Authorization Code with PKCE](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 来对这些端点进行身份验证。

<div id="manage-retweets">
  #### 管理转发
</div>

管理转发端点允许你代表已认证的账户转发指定的帖子，或撤销对该帖子的转发。对于该端点组，有两种可用的方法：POST 和 DELETE。POST 方法允许你转发一条帖子，而 DELETE 方法则允许你撤销对指定帖子的转发。

由于你是代表某个用户发起请求，你必须使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 或 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 对这些端点进行身份验证，并使用与你代表其发起请求的用户关联的用户访问令牌。你可以使用 [三方 OAuth 流程](/zh/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（OAuth 1.0a），或使用 [Authorization Code with PKCE grant flow](/zh/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)（OAuth 2.0）来生成该用户访问令牌。你可以从你自己的账户或某个已认证用户的账户转发一条帖子。对于这两个端点，用户速率限制均为每个端点每 15 分钟 50 次请求。
