<div id="comparing-x-apis-retweets-endpoints">
  ## 比较 X API 的转推（Retweet）端点
</div>

**转推查询**

v2 转推查询端点将取代标准 [v1.1 GET statuses/retweets/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-retweets-id) 和 [v1.1 GET statuses/retweets/:ids](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-retweeters-ids) 端点。

下表对比了标准 v1.1 与 X API v2 的转推端点：

| 说明 | 标准 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 支持的 HTTP 方法 | GET | GET |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /1.1/retweeters/id.json<br /><br />`/1.1/retweets/ids.json` | /2/users/:id/retweeted&#95;by |
| [认证](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文 | OAuth 2.0 Bearer Token<br /><br />OAuth 1.0a 用户上下文 |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 每 15 分钟 75 次请求 | 每 15 分钟 75 次请求（每个应用）<br /><br />每 15 分钟 75 次请求（每个用户） |
| 数据格式 | 标准 v1.1 格式 | [X API v2 格式](/zh/x-api/fundamentals/data-dictionary)（由 fields 和 expansions 请求参数决定，与 v1.1 格式不向后兼容）<br /><br />如需详细了解如何从标准 v1.1 格式迁移到 X API v2 格式，请访问我们的[数据格式迁移指南](/zh/x-api/migrate/data-format-migration)。 |
| 是否需要使用与某个 [项目](/zh/resources/fundamentals/projects) 关联的 [开发者应用](/zh/resources/fundamentals/developer-apps) 的凭证 |     | ✔️  |

<div id="manage-retweets">
  ### 管理转发
</div>

下表比较了标准 v1.1 和 X API v2 中用于撤销 Retweet 的端点：

**转发一条帖子**

| 描述 | 标准 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 支持的 HTTP 方法 | POST | POST |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /1.1/statuses/retweet/:id.json | /2/users/:id/retweets |
| [身份验证](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文 | OAuth 1.0a 用户上下文 |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 无<br /><br />每 3 小时内 300 次请求（每个用户、每个应用）。与 POST Tweet 端点共享 | 每 15 分钟 50 次请求（每个用户）<br /><br />每 3 小时内 300 次请求（每个用户、每个应用）。与用于管理帖子 的 POST Tweet 端点共享。 |
| 需要使用与某个[项目](/zh/resources/fundamentals/projects)关联的[开发者应用](/zh/resources/fundamentals/developer-apps)的凭证 |     | ✔️  |

<div id="undo-a-retweet">
  #### 撤销转发
</div>

下表比较了 Standard v1.1 与 X API v2 的撤销转发端点：

| 描述 | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| 支持的 HTTP 方法 | POST | DELETE |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /1.1/statuses/unretweet/:id.json | /2/users/:id/retweets/:source&#95;tweet&#95;id |
| [认证](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文 | OAuth 1.0a 用户上下文 |
| 默认请求的[速率限制](/zh/resources/fundamentals/rate-limits) | 无 | 每位用户每 15 分钟 50 次请求 |
| 需要使用与某个[项目](/zh/resources/fundamentals/projects)关联的[开发者应用](/zh/resources/fundamentals/developer-apps)凭证 |     | ✔️  |
