<div id="comparing-x-apis-retweets-endpoints">
  ## 比较 X API 的转发（Retweets）端点
</div>

**转发查询**

v2 转发查询端点将取代标准版 [v1.1 GET statuses/retweets/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-retweets-id) 和 [v1.1 GET statuses/retweets/:ids](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-retweeters-ids) 端点。

下表比较了标准版 v1.1 与 X API v2 的转发端点：

| 描述 | 标准版 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 支持的 HTTP 方法 | GET | GET |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /1.1/retweeters/id.json<br /><br />`/1.1/retweets/ids.json` | /2/users/:id/retweeted&#95;by |
| [认证](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文 | OAuth 2.0 Bearer Token<br /><br />OAuth 1.0a 用户上下文 |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 每 15 分钟 75 次请求 | 每 15 分钟 75 次请求（每个应用）<br /><br />每 15 分钟 75 次请求（每个用户） |
| 数据格式 | 标准版 v1.1 格式 | [X API v2 格式](/zh/x-api/fundamentals/data-dictionary)（由 `fields` 和 `expansions` 请求参数决定，与 v1.1 格式不向后兼容）<br /><br />要了解如何从标准版 v1.1 格式迁移到 X API v2 格式，请参阅我们的[数据格式迁移指南](/zh/x-api/migrate/data-format-migration)。 |
| 是否需要使用某个与 [Project](/zh/resources/fundamentals/projects) 关联的 [developer App](/zh/resources/fundamentals/developer-apps) 的凭证 |     | ✔️  |

<div id="manage-retweets">
  ### 管理转发
</div>

下表对比了标准版 v1.1 与 X API v2 的撤销转发端点：

**转发帖子**

| 描述 | 标准版 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 支持的 HTTP 方法 | POST | POST |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /1.1/statuses/retweet/:id.json | /2/users/:id/retweets |
| [Authentication](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文 | OAuth 1.0a 用户上下文 |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 无<br /><br />每 3 小时窗口内 300 次请求（每个用户、每个应用）。这与 POST Tweet 端点共享配额 | 每 15 分钟 50 次请求（每个用户）<br /><br />每 3 小时窗口内 300 次请求（每个用户、每个应用）。这与用于管理帖子的 POST Tweet 端点共享配额。 |
| 需要使用与某个[开发者应用](/zh/resources/fundamentals/developer-apps)关联的[项目](/zh/resources/fundamentals/projects)中的凭证 |     | ✔️  |

<div id="undo-a-retweet">
  #### 撤销转帖
</div>

下表比较了标准版 v1.1 与 X API v2 撤销转帖端点：

| 描述 | 标准版 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 支持的 HTTP 方法 | POST | DELETE |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /1.1/statuses/unretweet/:id.json | /2/users/:id/retweets/:source&#95;tweet&#95;id |
| [身份验证](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文 | OAuth 1.0a 用户上下文 |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 无 | 每 15 分钟 50 次请求（每个用户） |
| 需要使用与某个[项目](/zh/resources/fundamentals/projects)关联的[开发者应用](/zh/resources/fundamentals/developer-apps)的凭证 |     | ✔️  |
