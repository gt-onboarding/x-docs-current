<div id="manage-retweets-standard-v11-compared-to-x-api-v2">
  ### 管理转推：标准 v1.1 与 X API v2 的比较
</div>

如果你一直在使用标准 v1.1 的 [POST statuses/retweet/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-retweet-id) 和 [POST statuses/unretweet/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-unretweet-id) 端点，本指南旨在帮助你了解标准端点与 X API v2 转推端点之间的相似点和差异。

* **相似点**
  * 身份验证
* **差异**
  * 端点 URL 和 HTTP 方法
  * 请求限制
  * 应用和项目要求
  * 请求参数

<div id="similarities">
  #### 相似之处
</div>

**身份验证**

标准 v1.1 和 X API v2 中用于管理转推（[POST statuses/retweet/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-retweet-id) 和 [POST statuses/unretweet/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-unretweet-id)）的端点都使用 [OAuth 1.0a 用户上下文](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication)。因此，如果你之前使用的是标准 v1.1 的某个转推查询端点，那么在迁移到 X API v2 版本时，可以继续使用相同的身份验证方法。 

<div id="differences">
  #### 差异
</div>

**端点 URL 和 HTTP 方法**

* 标准 v1.1 端点：
  * https://api.x.com/1.1/statuses/retweet/:id.json
    （转发一条帖子。返回包含转发详情的原始帖子）
  * https://api.x.com/1.1/statuses/unretweet/:id.json
    （撤销一次转发。返回包含转发详情的原始帖子）
* X API v2 端点：
  * https://api.x.com/2/tweets/:id/retweets
    （转发指定帖子）
  * https://api.x.com/2/users/:id/retweets/:source&#95;tweet&#95;id
    （撤销对指定帖子的转发） 

**应用和项目要求**

在对请求进行身份验证时，X API v2 端点要求你使用与某个[项目](/zh/resources/fundamentals/projects)关联的[开发者应用](/zh/resources/fundamentals/developer-apps)的凭证。所有 X API v1.1 端点都可以使用来自独立应用或与项目关联的应用的凭证。

**请求参数**

以下标准 v1.1 端点在请求中可以接受两个查询参数之一（user&#95;id 或 screen&#95;name）。X API v2 只接受数值型用户 ID，并且必须作为端点路径的一部分传递。

| 标准 v1.1 | X API v2 |
| :--- | :--- |
| **id** | **id** |
| **includes&#95;entities** | 无对应参数 |

请注意，标准 v1.1 参数作为查询参数传递，而 X API v2 参数在 POST 端点中作为请求体参数传递，在 DELETE 端点中作为路径参数传递。

另外，在使用标准 v1.1 端点时，不需要提供转发帖子的用户 id，因为与 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication)一起传递的[访问令牌](/zh/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)即可推断出是哪个用户在发起或撤销转发。
