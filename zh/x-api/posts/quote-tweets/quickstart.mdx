<div id="getting-started-with-the-quote-tweets-lookup-endpoint">
  ## 开始使用引用帖子查询端点
</div>

本快速入门指南将帮助你使用一组指定的字段，通过 [Postman](/zh/tutorials/postman-getting-started) 向引用帖子查询端点发出你的第一个请求。

如果你想查看不同语言的示例代码，请访问我们在 GitHub 上的 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) 仓库。 

<div id="prerequisites">
  #### 先决条件
</div>

要完成本指南，你需要一组用于对请求进行认证的[密钥和令牌](/zh/resources/fundamentals/authentication)。你可以通过以下步骤生成这些密钥和令牌：

* [注册开发者账号](https://developer.x.com/en/apply-for-access)并通过审批。
* 在开发者门户中创建一个[项目](/zh/resources/fundamentals/projects)以及一个关联的[开发者应用](/zh/resources/fundamentals/developer-apps)。
* 前往你的 App 的 “Keys and tokens” 页面生成所需的凭证。请确保将所有凭证保存在安全的位置。

**构建 Quote Tweets 查询请求的步骤**

**第一步：选择一个工具或库**

有多种不同的工具、代码示例和库可用于向此端点发起请求，但为了简化流程，我们将在这里使用 Postman 工具。

要将 X API v2 的 Postman 集合加载到你的环境中，请点击以下按钮：

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  将 X API v2 添加到 Postman
</Button>

在你将 X API v2 集合加载到 Postman 之后，导航到 Timeline 文件夹并找到 “Quote Tweets by ID” 请求。

**第二步：为请求进行认证**

要正确向 X API 发起请求，你需要确认自己拥有相应的权限。对于该端点，你必须使用 [OAuth 2.0 App-Only](/zh/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)、[OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 或 [OAuth 1.0a User Context](/zh/resources/fundamentals/authentication) 认证方法之一为请求进行认证。

为简单起见，我们在本示例中将使用 OAuth 2.0 App-Only，但如果你希望请求私有[指标](/zh/x-api/fundamentals/metrics)或帖子，则需要使用其他认证方法之一。 

你必须将你的密钥和令牌，具体来说是 [App Access Token](/zh/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)（也称为 App-only Bearer Token）添加到 Postman 中。你可以在 Postman 右上角选择名为 “X API v2” 的环境，然后通过点击环境下拉菜单旁的眼睛图标，将你的密钥和令牌添加到 &quot;initial value&quot; 和 &quot;current value&quot; 字段中。

如果设置正确，此变量会自动被注入到该请求的 Authorization 选项卡中。

**第三步：确定并指定你想要检索其 Quote Tweets 的帖子**

你必须在请求中通过传入帖子 ID 来指定你想要检索其 Quote Tweets 的帖子。

在本示例中，我们将使用的帖子 ID 为 1409931481552543749。

在 Postman 中，导航到 &quot;Params&quot; 选项卡，并将此帖子 ID 输入到 id 参数 &quot;Value&quot; 列中。

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 1409931481552543749 |

**第四步：确定并指定你想要检索的字段**

如果你在第三步之后点击 &quot;Send&quot; 按钮，你将在响应中收到默认的 [Post object](/zh/x-api/fundamentals/data-dictionary#tweet) 字段：id、text 和 edit&#95;history&#95;tweet&#95;ids。

如果你希望接收更多字段，则需要在请求中通过 [field](/zh/x-api/fundamentals/fields) 和/或 [expansion](/zh/x-api/fundamentals/expansions) 参数指定这些字段。

在本练习中，我们将从不同对象中请求三组额外字段：

1. 主帖子对象中的额外字段 tweet.created&#95;at。
2. 返回帖子所关联作者的用户对象默认字段：id、name 和 username。
3. 关联用户对象中的额外字段 user.created&#95;at。
    

在 Postman 中，导航到 &quot;Params&quot; 选项卡，并将以下键值对添加到 &quot;Query Params&quot; 表中：

|     |     |     |     |
| :--- | :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| `tweet.fields` | `created_at` | `tweets.created_at` |
| `expansions` | `author_id` | `includes.users.id`, `includes.users.name`, `includes.users.username` |
| `user.fields` | `created_at` | `includes.users.created_at` |     |

现在，你应该会在 “Send” 按钮旁边看到如下 URL：

`https://api.x.com/2/tweets/:id/quote_tweets?expansions=author_id&tweet.fields=created_at&user.fields=created_at`

**请注意：**

在 Postman 中，URL 字段中的路径参数 :id **不会** 自动更新为你在 `id` 参数字段中输入的值，因此上面的 URL 中显示的是 `:id`，而不是 1409931481552543749。

<div id="step-six-paginate-through-your-results">
  #### 步骤六：对结果进行分页
</div>

在上一个响应的底部，你会看到一个名为 `meta` 的数据对象，其中包括以下字段：

* results&#95;count
* next&#95;token

在步骤四中，因为我们没有指定 `max_results` 参数，默认情况下我们只会获取 10 条帖子。要访问后续的数据页面，我们将从上一页结果中获取 `next_token` 字段的值，并在 Postman 的参数页面中将该字符串作为 `pagination_token` 参数的值，同时保持其他内容不变。 

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `pagination_token` | avdjwk0udyx6 |

完成所有设置后，你可以再次点击“Send”按钮，此时应会收到下一页结果。 

我们整理了一篇关于[分页](/zh/x-api/fundamentals/pagination)的指南，以进一步解释这一概念。
