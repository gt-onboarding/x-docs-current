Postman 是一个非常适合测试这些端点的工具。每个 Postman 请求都包含该端点的所有参数，帮助你快速了解有哪些可用选项。如需进一步了解我们的 Postman 集合，请访问[使用 Postman](/zh/tutorials/postman-getting-started)页面。
 

<div id="key-concepts">
  ### 关键概念
</div>

<div id="authentication">
  #### 身份验证
</div>

所有 X API v2 端点都要求使用一组凭证（也称为密钥和令牌）对请求进行[身份验证](/zh/resources/fundamentals/authentication)。该端点要求使用 [OAuth 2.0 Bearer Token](/zh/resources/fundamentals/authentication#oauth-2-0)，也就是说，你必须在请求中携带一个 [Bearer Token](/zh/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) 才能请求成功。你可以直接在开发者应用中生成一个 Bearer Token，或者使用 [POST oauth2/token](/zh/resources/fundamentals/authentication#post-oauth2-token) 端点生成一个。

<div id="developer-portal-projects-and-developer-apps">
  #### 开发者门户、项目和开发者应用
</div>

要使用任何 X API v2 端点，你必须拥有一个[开发者账号](/zh/resources/fundamentals/developer-portal)，在该账号下设置一个[项目](/zh/resources/fundamentals/projects)，并在该项目中创建一个[开发者应用](/zh/resources/fundamentals/developer-apps)。该开发者应用中的密钥和令牌可用于近期帖子计数端点。如果你想使用全量归档帖子计数端点，或使用高级操作符和更长的查询字符串，则需要事先通过 Enterprise 级访问权限的审核。

请参阅我们关于 Enterprise 访问的章节以了解更多信息。

<div id="rate-limits">
  #### 速率限制
</div>

每天都有成千上万的开发者向 X API 发起请求。为便于管理请求量，我们在每个端点上设置了[速率限制](/zh/x-api/fundamentals/rate-limits)，以限制每位开发者代表某个应用或代表已通过身份验证的用户所能发起的请求次数。

此端点在应用级别（App 级别）实施速率限制，这意味着你作为开发者，在任意给定的时间段内，从任一应用（由你所使用的凭证所对应）向该端点只能发起一定数量的请求。 

<div id="building-queries">
  #### 构建查询
</div>

这些端点的核心特性是使用单个查询，将帖子筛选汇总为返回给你的计数结果。查询由一组运算符构成，这些运算符基于帖子和用户属性进行匹配，比如消息关键词、话题标签（hashtag）和 URL。你可以使用布尔逻辑和括号将运算符组合成查询，从而进一步细化查询的匹配行为。

你可以参考我们的指南[如何构建查询](/zh/x-api/posts/counts/integrate/build-a-query)以了解更多信息。

<div id="pagination">
  #### 分页
</div>

对于近期的帖子计数，不会返回 next&#95;token，这意味着无论粒度如何，你都可以通过一次 API 调用获取过去 7 天的帖子数量。

对于完整存档的帖子计数，你将获取过去 30 天的数据。对于早于最近 30 天的数据，你会得到一个 next&#95;token，然后可以使用它进行分页以获取额外的数据。 

<div id="building-queries-for-post-counts">
  ### 为帖子计数构建查询
</div>

Post 计数端点接受通过 GET 请求发送的单个查询，并返回一组与该查询匹配的历史帖子计数数据。查询由运算符组成，这些运算符用于匹配各种帖子属性。
