<div id="helpful-tools">
  ### 实用工具
</div>

在开始了解一些关键概念之前，我们建议你先使用下列任一工具或代码示例来测试这些端点的功能。

<div id="code-samples">
  #### 代码示例
</div>

想用你常用的编程语言，通过一些代码来集成这些端点吗？我们在 [GitHub 页面](https://github.com/xdevplatform/Twitter-API-v2-sample-code)上提供了多种可作为起点的代码示例，其中包括一个 [Python 客户端](https://github.com/xdevplatform/search-tweets-python)。

<div id="libraries">
  #### 库
</div>

你可以利用我们提供的众多[社区第三方库](/zh/x-api/tools-and-libraries/overview)来帮助你上手。通过查找相应的版本标签，你可以找到可与 v2 端点配合使用的库。

<div id="postman">
  #### Postman
</div>

Postman 是一个非常适合用来测试这些端点的工具。每个 Postman 请求都包含相应端点的所有参数，帮助你快速了解该端点为你提供的内容。要进一步了解我们的 Postman 集合，请参阅我们的 [使用 Postman](/zh/tutorials/postman-getting-started) 页面。
 

<div id="key-concepts">
  ### 核心概念
</div>

<div id="authentication">
  #### 认证
</div>

所有 X API v2 端点都要求使用一组凭证（也称为密钥和令牌）对请求进行[认证](/zh/resources/fundamentals/authentication)。此特定端点要求使用 [OAuth 2.0 Bearer Token](/zh/resources/fundamentals/authentication#oauth-2-0)，也就是说，你必须在请求中携带一个 [Bearer Token](/zh/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) 才能成功发出请求。你可以直接在开发者应用中生成 Bearer Token，或者使用 [POST oauth2/token](/zh/resources/fundamentals/authentication#post-oauth2-token) 端点生成一个。

<div id="developer-portal-projects-and-developer-apps">
  #### 开发者门户、项目和开发者应用
</div>

要使用任意 X API v2 端点，你必须拥有一个[开发者账号](/zh/resources/fundamentals/developer-portal)，在该账号下设置一个[项目](/zh/resources/fundamentals/projects)，并在该项目中创建一个[开发者应用](/zh/resources/fundamentals/developer-apps)。该开发者应用中的密钥和令牌可用于最近帖子计数端点。如果你希望使用完整存档帖子计数端点，或使用高级运算符和更长的查询语句，则需要事先获得 Enterprise 访问权限的批准。

请访问我们的 Enterprise 访问权限相关章节以了解更多信息。

<div id="rate-limits">
  #### 速率限制
</div>

每天都有成千上万的开发者向 X API 发起请求。为了管理请求量，我们在每个端点上都设置了[速率限制](/zh/x-api/fundamentals/rate-limits)，用于限制每位开发者代表某个应用或某个已认证用户所能发起的请求次数。

此端点采用应用级别的速率限制，这意味着你作为开发者，在给定时间段内，从任一应用（由你所使用的凭证所对应的应用）向该端点发起的请求次数是有限的。 

<div id="building-queries">
  #### 构建查询
</div>

这些端点的核心特性在于通过单个查询，将帖子筛选并汇总为返回给你的计数结果。查询由一组运算符组成，这些运算符会根据帖子和用户的属性进行匹配，例如文本关键词、话题标签和 URL。你可以使用布尔逻辑和括号将多个运算符组合成一个查询，以便更精细地控制查询的匹配行为。

你可以参考我们的[如何构建查询](/zh/x-api/posts/counts/integrate/build-a-query)指南以了解更多信息。

<div id="pagination">
  #### 分页
</div>

对于最近的帖子计数，不会返回 next&#95;token，这意味着无论粒度如何，你都可以在一次 API 调用中获取过去 7 天的帖子数量。

对于全量归档的帖子计数，你可以获得过去 30 天的数据。对于早于最近 30 天的数据，你将会收到一个 next&#95;token，然后可以使用它进行分页以获取更多数据。 

<div id="building-queries-for-post-counts">
  ### 为帖子计数构建查询
</div>

帖子计数端点接受在 GET 请求中携带的单个查询，并返回与该查询匹配的一组历史帖子计数。查询由一组运算符组成，这些运算符用于匹配帖子上的各种属性。
