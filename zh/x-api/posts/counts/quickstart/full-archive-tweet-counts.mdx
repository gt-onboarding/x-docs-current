<div id="getting-started-with-the-full-archive-post-counts-endpoint">
  ### 开始使用全量存档帖子计数端点
</div>

本快速入门指南将帮助你使用 Postman（一个用于发出 HTTP 请求的图形界面工具）向全量存档帖子计数端点发送你的第一个请求。

如果你想查看使用不同编程语言编写的示例代码，请访问我们的 [X API v2 示例代码](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 代码库。 

<Note>
  ### 前提条件

  全量存档帖子计数端点目前仅向具有 Pro 或 Enterprise 访问权限的用户开放。要使用此端点，你必须申请 Pro 或 [Enterprise 访问权限](https://docs.google.com/forms/d/e/1FAIpQLScO3bczKWO2jFHyVZJUSEGfdyfFaqt2MvmOfl_aJp0KxMqtDA/viewform)。 

  除获得访问权限外，你还需要一组 [keys and tokens](/zh/resources/fundamentals/authentication) 来对你的请求进行身份验证。你可以按照以下步骤生成这些密钥和令牌：

  * 在开发者门户中，进入具有 Enterprise 访问权限的 [Project](/zh/resources/fundamentals/projects)，并确保该 Project 下关联了一个 [developer App](/zh/resources/fundamentals/developer-apps)。

  * 前往你的 App 的“Keys and tokens”页面以生成所需的凭证。请务必将所有凭证保存在安全的位置。
</Note>

<div id="steps-to-build-a-full-archive-post-counts-request">
  #### 构建全档案帖子计数请求的步骤
</div>

**第一步：从工具或库开始**

有多种不同的工具、代码示例和库可用于向此端点发出请求，但在这里我们将使用 Postman 工具来简化这一过程。

要将 X API v2 Postman 集合加载到你的环境中，请点击以下按钮：

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  将 X API v2 添加到 Postman
</Button>

在你将 X API v2 集合加载到 Postman 之后，导航到 Post counts &gt; Full-archive Post counts 请求。

**第二步：为你的请求进行身份验证**

要正确地向 X API 发出请求，你需要验证你是否具备相应权限。对于该端点，你必须使用 [OAuth 2.0 App-Only](/zh/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) 身份验证方法对你的请求进行身份验证。

你必须将你的密钥和令牌，特别是 [App Access Token](/zh/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)（也称为 App-only Bearer Token）添加到 Postman。你可以通过在 Postman 右上角选择名为 “X API v2” 的环境，并将你的密钥和令牌添加到 &quot;initial value&quot; 和 &quot;current value&quot; 字段中（点击环境下拉菜单旁边的眼睛图标）来完成此操作。

如果你已正确完成此操作，该变量将自动被注入到请求的 authorization 选项卡中。
 

**第三步：创建查询**

每个全档案帖子计数请求都需要一个 [query](/zh/x-api/posts/counts/integrate/build-a-query)。在本示例中，我们将使用一个匹配 @XDevelopers 账号发布的帖子的查询。对于此查询，我们使用 from 运算符并将其设置为 XDevelopers（不区分大小写）：

`from:XDevelopers`

在 Postman 中，导航到 &quot;Params&quot; 选项卡，并将此 ID，或以逗号分隔的一串帖子 ID，输入到 `ids` 参数的 &quot;Value&quot; 列中。
 

| Key | Value | Description |
| :--- | :--- | :--- |
| `query` | `from:XDevelopers` | 提交到全档案帖子计数端点的查询 |

**Step four (optional): Specify the granularity and time period**

如果在第三步之后点击 “Send” 按钮，你将获得默认的全档案帖子计数结果：过去 30 天内按小时聚合。如果你想按天获取全档案帖子计数，你需要添加 granularity 参数并将其值设置为 day。如果你想获取 30 天之前的帖子计数，你需要为 start&#95;time 和 end&#95;time 参数指定所需的值。 

在 Postman 中，导航到 &quot;Params&quot; 选项卡，并在 &quot;Query Params&quot; 表中添加以下键值对：

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Description** |
| granularity | day | 帖子计数结果的粒度。可用值为 day、hour 或 minute |
| start&#95;time | 2021-05-01T00:00:00Z | 返回帖子所依据的最早 UTC 时间戳 |
| end&#95;time | 2021-06-01T00:00:00Z | 返回帖子所依据的最早 UTC 时间戳。 |

你现在应当能在 &quot;Send&quot; 按钮旁边看到以下 URL：

`https://api.x.com/2/tweets/counts/all?query=from%3AXDevelopers&start_time=2021-05-01T00:00:00Z&end_time=2021-06-01T00:00:00Z&granularity=day`

**第六步：分页获取结果**

如果响应中的 `meta` 对象也包含 `next_token`，你可以将其值传入 `next_token` 查询参数。

| 键 | 值 | 说明 |
| :--- | :--- | :--- |
| next&#95;token | 在这里填写你从上一次请求的 `meta` 对象中获取到的 `next_token`。 | 如果你最近一次请求尚未返回全部结果，你会在 `meta` 对象中收到一个 `next_token`。你需要提取该字段的值，并在下一次请求中将其作为 `next_token` 参数的值，同时保持其他请求参数不变。 |

当你为 `next_token` 设置了正确的值后，点击“Send”按钮，你就会收到下一页的结果。
