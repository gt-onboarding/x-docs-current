* 1% 采样流。
* 10% 采样流，通常称为 “Decahose”。

这些被称为“大流量流（volume streams）”，因为它们都会传输大量数据。即使是 1% 采样流，每秒也可能产生数十条帖子。通过这些流，你可以识别和跟踪趋势、监控整体情绪、监测全球事件等。

这些流式端点通过持久的 HTTP GET 连接传递 [Post 对象](/zh/x-api/fundamentals/data-dictionary#tweet)，并使用 [OAuth 2.0 App-Only](/zh/resources/fundamentals/authentication#oauth-2-0) 认证。使用 Essential 访问级别时，你一次只能有一个连接。在所有访问级别下，在每个 15 分钟时间窗口内最多可发起 50 次连接请求。

这些大流量流端点支持已编辑的帖子。这些端点会传递已编辑的帖子及其编辑历史记录，其中包含一个帖子 ID 数组。对于没有编辑历史的帖子，该数组只包含一个 ID。对于已编辑的帖子，该数组包含多个 ID，按升序排列，反映编辑顺序，数组最后一个位置是最新版本。要进一步了解帖子编辑功能的工作方式，请参阅[帖子编辑基础](/zh/x-api/fundamentals/edit-posts)页面。

*要使用这些 API，你必须先通过我们的 Enterprise 销售团队完成账号开通。*

<Note>

  * 使用位于某个[项目](/zh/resources/fundamentals/projects)中的[开发者应用](/zh/resources/fundamentals/developer-apps)的密钥和令牌进行认证。
