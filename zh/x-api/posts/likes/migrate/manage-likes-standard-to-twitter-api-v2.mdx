<div id="manage-likes-standard-v11-compared-to-x-api-v2">
  ### 管理点赞：标准 v1.1 与 X API v2 的比较
</div>

如果你一直在使用标准 v1.1 的 [POST favorites/create](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-create) 和 [POST favorites/destroy](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-destroy) 端点，本指南旨在帮助你了解标准 v1.1 与 X API v2 管理点赞端点之间的相似之处和差异。

* **相似之处**
  * OAuth 1.0a 用户上下文
* **差异**
  * 端点 URL 和 HTTP 方法
  * 应用和项目要求
  * 请求参数

<div id="similarities">
  #### 相似点
</div>

**OAuth 1.0a 用户上下文身份验证方法**

这两个版本的端点都支持 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication#oauth-1-0a-2)。因此，如果你之前使用的是 v1.1 中用于管理点赞的某个标准端点，那么在迁移到 X API v2 版本后，可以继续使用相同的身份验证方法。

<div id="differences">
  #### 差异
</div>

**端点 URL 和 HTTP 方法**

* Standard v1.1 endpoints：
  * POST https://api.x.com/1.1/favorites/create.json
    （点赞一个帖子）
  * POST https://api.x.com/1.1/favorites/destroy.json
    （取消点赞一个帖子）
* X API v2 endpoint：
  * POST https://api.x.com/2/tweets/:id/likes
    （点赞一个帖子）
  * DELETE https://api.x.com/2/tweets/:id/likes/:tweet&#95;id
    （取消点赞一个帖子） 

**应用和项目要求**

在对请求进行认证时，X API v2 端点要求你使用与某个[开发者应用](/zh/resources/fundamentals/developer-apps)及其所属[项目](/zh/resources/fundamentals/projects)关联的凭证。所有 X API v1.1 端点则可以使用独立应用或与项目关联的应用中的凭证。

**请求参数**

以下 Standard v1.1 请求参数在 X API v2 中有对应项：

| Standard v1.1 | X API v2 |
| :--- | :--- |
| id  | id  |
| includes&#95;entities | 无对应项 |

请注意，Standard v1.1 参数作为查询参数传递，而 X API v2 参数则作为 POST 端点的请求体参数或 DELETE 端点的路径参数传递。

另外，在使用 Standard v1.1 端点时，不需要提供点赞用户的 id，因为使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 传递的[访问令牌](/zh/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)可以推断出是哪个用户在发起点赞或取消点赞操作。
