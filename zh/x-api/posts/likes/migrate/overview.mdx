<div id="comparing-x-apis-likes-endpoints">
  ## 比较 X API 的点赞端点
</div>

这些指南将重点介绍以下几个方面：

* **API 请求参数** - X API v2 端点引入了一套新的请求参数。虽然其中一些参数对已经集成 Labs 的开发者来说会比较熟悉，但也存在许多重要差异，例如新增了 [fields](/zh/x-api/fundamentals/fields) 和 [expansions](/zh/x-api/fundamentals/expansions) 参数。

* **App 和 Project 要求** - 要访问 X API v2，您需要使用来自某个 [developer App](/zh/resources/fundamentals/developer-apps)（该 App 隶属于某个 [Project](/zh/resources/fundamentals/projects)）的凭证。

<div id="likes-lookup">
  #### 点赞查询
</div>

**点赞某个帖子的用户**

`liked users` 端点是 v2 中的新功能，允许你获取某个帖子点赞用户的信息。

| 描述 | X API v2 |
| :--- | :--- |
| 支持的 HTTP 方法 | GET |
| 主机域名 | https://api.x.com |
| 端点路径 | /2/tweets/:id/liking&#95;users |
| [认证](/zh/resources/fundamentals/authentication) | OAuth 2.0 Bearer Token<br /><br />OAuth 1.0a 用户上下文 |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 每 15 分钟 75 次请求（每个应用）<br /><br />每 15 分钟 75 次请求（每个用户） |

**某个用户点赞的帖子**

下表对比了标准 v1.1 [GET favorites/list](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-favorites-list) 端点与 X API v2 中 liked Posts 端点：

| 描述 | 标准 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 支持的 HTTP 方法 | GET | GET |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /1.1/favorites/list.json | /2/users/:id/liked&#95;tweets |
| [认证](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文 | OAuth 2.0 Bearer Token<br /><br />OAuth 1.0a 用户上下文 |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 每 15 分钟 75 次请求 | 每 15 分钟 75 次请求（每个应用）<br /><br />每 15 分钟 75 次请求（每个用户） |
| 数据格式 | 标准 v1.1 格式 | [X API v2 格式](/zh/x-api/fundamentals/data-dictionary)（由 fields 和 expansions 请求参数决定，与 v1.1 格式不向后兼容）<br /><br />要了解如何从标准 v1.1 格式迁移到 X API v2 格式，请参阅我们的[数据格式迁移指南](/zh/x-api/migrate/data-format-migration)。 |

<div id="manage-likes">
  #### 管理点赞
</div>

v2 中用于管理点赞的端点取代了 v1.1 的 [POST favorites/create](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-create) 和 [POST favorites/destroy](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-destroy) 端点。

下表比较了标准 v1.1 与 X API v2 的点赞管理端点：

<div id="like-a-post">
  #### 为帖子点赞
</div>

| Description | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| HTTP methods supported | POST | POST |
| Host domain | https://api.x.com | https://api.x.com |
| Endpoint path | /1.1/favorites/create.json | /2/users/:id/likes |
| [Authentication](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文 | OAuth 1.0a 用户上下文 |
| Default request [rate limits](/zh/resources/fundamentals/rate-limits) | 每 24 小时 1000 次请求（每位用户）<br /><br />每 24 小时 1000 次请求（每个应用） | 每 15 分钟 50 次请求（每位用户）<br /><br />每 24 小时 1000 次请求（每位用户，与 DELETE 共享） |

<div id="unlike-a-post">
  #### 取消对帖子的点赞
</div>

| 描述 | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| 支持的 HTTP 方法 | POST | DELETE |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /1.1/favorites/destroy.json | /2/users/:id/likes/:tweet&#95;id |
| [身份验证](/zh/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 每个用户 24 小时内 1000 次请求<br /><br />每个应用 24 小时内 1000 次请求 | 每个用户每 15 分钟 50 次请求<br /><br />每个用户 24 小时内 1000 次请求（与 POST 共享） |
