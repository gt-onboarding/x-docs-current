<div id="likes-lookup">
  ### 点赞查询
</div>

使用 Likes 查找分组中的端点，你可以检索点赞某个帖子的一组账号，或某个账号点过赞的一组帖子。这些端点包括：

* 某用户点赞的帖子 - GET /2/users/:id/liked&#95;tweets
* 点赞某帖子的用户 - GET /2/tweets/:id/liking&#95;users

你可以使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 或 [OAuth 2.0 Bearer Token](/zh/resources/fundamentals/authentication#oauth-2-0) 对这些端点进行身份验证。对于“被点赞的帖子”相关端点，会提供分页 token，用于在大型结果集中进行分页浏览。

对于 liking users 端点，在整个时间范围内，每个帖子最多只能返回 100 个点赞账号。此外，“被点赞的帖子”端点也受限于应用在项目级别的每月 [帖子上限](/zh/x-api/fundamentals/post-cap)。
 

<div id="manage-likes">
  ### 管理点赞
</div>

“管理点赞”端点允许你代表已认证账号对指定帖子进行点赞或取消点赞。对于这一端点组，可用的 HTTP 方法有两种：POST 和 DELETE。POST 方法允许你给帖子点赞，而 DELETE 方法允许你取消对帖子的点赞。

由于你是代表用户发起请求，因此必须使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 对这些端点进行认证，并使用与该用户关联的访问令牌，这些令牌可通过 [三方 OAuth 流程](/zh/resources/fundamentals/authentication#oauth-1-0a-2)/obtaining-user-access-tokens) 生成。你可以从自己的账号，或某个已认证用户的账号，为帖子点赞。对于这两个端点，每个端点对每个用户的速率限制为每 15 分钟 50 个请求。 
 

要访问这些端点，你必须拥有已获批准的 [开发者账号](/zh/resources/fundamentals/developer-portal)。在进行认证时，你必须使用位于某个 [项目](/zh/resources/fundamentals/projects) 中的 [开发者应用](/zh/resources/fundamentals/developer-apps) 的密钥和令牌。 

在我们的 [入门](/zh/x-api/getting-started/about-x-api) 页面了解更多关于如何获取 X API v2 端点访问权限的信息。
