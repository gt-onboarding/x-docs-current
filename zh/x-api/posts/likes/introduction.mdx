<div id="likes-lookup">
  ### 点赞查找
</div>

使用 Likes 查找分组中的端点，你可以检索点赞某条帖子的一组账号，或某个账号点赞过的一组帖子。这些端点包括：

* 某用户点赞过的帖子 - GET /2/users/:id/liked&#95;tweets
* 点赞了某条帖子的用户 - GET /2/tweets/:id/liking&#95;users

你可以使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 或 [OAuth 2.0 Bearer Token](/zh/resources/fundamentals/authentication#oauth-2-0) 来对这些端点进行认证。对于已点赞帖子相关端点，会提供分页令牌，方便你在大量结果中进行分页。

liking users 端点在整个历史期间，每条帖子最多只返回 100 个点赞账号。此外，liked Posts 端点同样受限于应用在项目级别的每月[帖子上限](/zh/x-api/fundamentals/post-cap)。
 

<div id="manage-likes">
  ### 管理点赞
</div>

管理点赞端点可让你代表经过认证的账号对指定帖子进行点赞或取消点赞。对于这一端点组，有两种可用的方法：POST 和 DELETE。POST 方法允许你为帖子点赞，DELETE 方法则允许你取消对帖子的点赞。

由于你是代表用户发起请求，你必须使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 对这些端点进行身份验证，并使用与该用户关联的访问令牌，这些令牌可以通过 [三方 OAuth 流程](/zh/resources/fundamentals/authentication#oauth-1-0a-2)/obtaining-user-access-tokens) 生成。你可以为你自己的账号，或任一经过认证用户的账号点赞。对于这两个端点，每个端点的用户速率限制均为每 15 分钟 50 次请求。 
 

要访问这些端点，你必须拥有一个已通过审批的[开发者账号](/zh/resources/fundamentals/developer-portal)。在进行身份验证时，你必须使用位于某个[项目](/zh/resources/fundamentals/projects)中的[开发者应用](/zh/resources/fundamentals/developer-apps)的密钥和令牌。 

在我们的[入门](/zh/x-api/getting-started/about-x-api)页面了解更多关于如何获取 X API v2 端点访问权限的信息。
