过滤流端点组使开发者能够过滤公共帖子流。该端点组的功能包括多个端点，可用于创建和管理规则，并应用这些规则来过滤帖子流，从而返回与之匹配的公共帖子。通过该端点组，用户可以监听特定话题和事件、监控围绕竞赛的讨论、了解趋势如何发展等更多用例。

开发者可以使用 REST [rules 端点](/zh/x-api/posts/filtered-stream#post-2-tweets-search-stream-rules)，在不中断持久流连接的情况下添加和删除规则。这些[规则](/zh/x-api/posts/filtered-stream#building-rules-for-filtered-stream)可以使用运算符来匹配帖子属性，例如消息关键词、主题标签和 URL。运算符和规则子句可以与布尔逻辑和括号组合，以帮助细化过滤器的匹配行为。

添加完一组规则后，你可以[建立一个流式连接](/zh/x-api/posts/filtered-stream#get-2-tweets-search-stream)，它会通过持久的 HTTP 流式连接，以 JSON 格式开始传送[Post 对象](/zh/x-api/fundamentals/data-dictionary#tweet)。在连接到该流期间，你只会接收与规则匹配的内容。

或者，你可以将 Filtered Stream 连接到一个 webhook，以 webhook 事件的形式进行投递。有关将 webhook 与 Filtered Stream 搭配使用的更多详情，请参阅 [Filtered Stream Webhooks 介绍](/zh/x-api/webhooks/stream/introduction)。

过滤搜索端点支持已编辑的帖子。该端点会投递匹配你一个或多个过滤条件的已编辑帖子，以及其编辑历史，其中包括一个帖子 ID 数组。对于没有编辑历史的帖子，该数组只包含一个 ID。对于已被编辑的帖子，该数组包含多个 ID，并按升序排列以反映编辑顺序，数组最后一个元素为最新版本。要了解帖子编辑的工作方式，请参阅[帖子编辑基础](/zh/x-api/fundamentals/edit-posts)页面。

过滤流端点的某些方面会受[访问级别](/zh/x-api/getting-started/about-x-api)限制：

**Pro access**

**Pro 访问权限**

* 每个项目最多 1000 条规则
* 使用 POST /2/tweets/search/stream/rules 端点添加规则时，每 15 分钟最多 100 个请求
* 在构建规则时可以使用所有运算符
* 规则长度最多为 1024 个字符

**Enterprise access**

**Enterprise 访问权限**

* 每个项目 25,000+ 条规则
* 在构建规则时可以使用所有运算符
* 规则长度最多为 2048 个字符
* 在[此处](https://developer.x.com/en/products/x-api/enterprise/enterprise-api-interest-form)申请 Enterprise 访问

从过滤流返回的帖子会计入每月的[帖子上限](/zh/x-api/fundamentals/post-cap)。

<Note>

  * 使用位于某个[项目](/zh/resources/fundamentals/projects)中的[开发者应用](/zh/resources/fundamentals/developer-apps)的密钥和令牌进行身份验证。
