<div id="getting-started-with-the-filtered-stream-endpoints">
  ## filtered stream 端点入门
</div>

本快速入门指南将帮助你使用 cURL 向 filtered stream 端点组发送你的第一个请求。cURL 是一个命令行工具，可让你以最少的配置发出请求。

如果你想查看不同语言的示例代码，请访问我们的 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 仓库。

<Note>
  **前提条件**

  要完成本指南，你需要一组用于对请求进行身份验证的 [密钥和令牌](/zh/resources/fundamentals/authentication)。你可以按照以下步骤生成这些密钥和令牌：

  * [注册开发者账号](https://developer.x.com/en/apply-for-access) 并通过审核。
  * 在开发者门户中创建一个 [项目](/zh/resources/fundamentals/projects) 以及关联的 [开发者应用](/zh/resources/fundamentals/developer-apps)。
  * 进入你的应用的 “Keys and tokens” 页面以生成所需的凭证。请确保将所有凭证保存在安全的位置。
</Note>

<div id="steps-to-build-a-filtered-stream-request-using-curl">
  ### 使用 cURL 构建过滤流请求的步骤
</div>

**步骤一：创建规则**

规则由一个或多个不同的[运算符](/zh/x-api/posts/filtered-stream#building-rules-for-filtered-stream)组成，通过布尔逻辑和括号组合起来，用于定义哪些帖子会传递到你的流中。在本指南中，我们将过滤流，以查找同时包含关键词 “cat” 和图片的帖子。下面是我们的规则：

```cat has:images
```

**步骤二：为你的规则添加标签**

你可以向流中添加多条同时生效的规则。当你打开流式连接时，匹配任意这些规则的帖子都会通过同一个流式连接。为确保你知道每条帖子匹配的是哪条规则，你可以在创建规则的请求中一并传递一个标签。每条匹配该规则的帖子随后都会在帖子负载中包含一个 tag 字段，用于标明它匹配了哪条规则。对于这条规则，我们将分配以下标签：

```cats with images
```

**步骤三：将规则添加到流中**

此 endpoint 要求你在请求中携带一个 `application/json` 的请求体，其中包含你的规则及其标签。你还会注意到，我们把规则的值和标签放在一个 `add` 对象中，因为我们要将此规则添加到流中。这个 JSON 请求体如下所示：

```json
{
  "add": [
    {
      "value": "cat has:images",
      "tag": "带图片的猫"
    }
  ]
}
```

现在你已经完整设置好了这个 JSON 请求体，可以将它添加到一个 cURL 请求中，请求会大致如下所示。这个请求现在还不能发送，请先不要提交，等到后面的步骤再执行。

```json
curl -X POST 'https://api.x.com/2/tweets/search/stream/rules' \
-H "Content-type: application/json" \
-H "Authorization: Bearer $APP_ACCESS_TOKEN" -d \
'{
  "add": [
    {"value": "cat has:images", "tag": "cats with images"}
  ]
}'`
```

**步骤四：为请求进行身份验证**

由于 filtered stream 规则端点需要使用 [OAuth 2.0 应用仅模式身份验证](/zh/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)，你需要将第三步 cURL 命令中的 $APP&#95;ACCESS&#95;TOKEN 替换为你在前置条件中生成的 [App Access Token](/zh/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)。 

**步骤五：将规则添加到流中**

下一步是执行你的 cURL 请求，这会将规则添加到你的流中。将该 cURL 命令复制并粘贴到命令行界面中，然后按下 “return” 键。 

如果你的 cURL 请求成功，你将收到一个响应，其中包含关于你的 value、tag 和 id（作为该规则唯一标识符）的数据。该响应将类似如下所示：

```json
{
  "data": [
    {
      "value": "cat has:images",
      "tag": "带图片的猫咪",
      "id": "1273026480692322304"
    }
  ],
  "meta": {
    "sent": "2020-06-16T22:55:39.356Z",
    "summary": {
      "created": 1,
      "not_created": 0,
      "valid": 1,
      "invalid": 0
    }
  }
}
```

你可以通过向 rules 端点发送以下 GET 请求来确认你的规则是否已成功添加，并再次确保将 $APP&#95;ACCESS&#95;TOKEN 替换为你的访问令牌。该请求会返回已添加到你的流中的所有规则的完整列表。

```bash
curl -X GET 'https://api.x.com/2/tweets/search/stream/rules' \
-H "Authorization: Bearer $APP_ACCESS_TOKEN"
```

如果你的 cURL 请求成功，你将会收到如下内容：

```json
{
	"data": [{
		"id": "1273028376882589696",
		"value": "cat has:images",
		"tag": "cats with images"
	}],
	"meta": {
		"sent": "2020-06-16T23:14:06.498Z"
	}
}
```

**步骤六：确定并指定你想要获取的字段**

如果你在完成第 5 步后再连接到流，你将在响应中收到默认的 [Post 对象](/zh/x-api/fundamentals/data-dictionary#tweet) 字段：id、text 和 edit&#95;history&#95;tweet&#95;ids。 如果你希望接收除这些以外的更多字段，你需要在请求中通过 [field](/zh/x-api/fundamentals/fields) 和/或 [expansion](/zh/x-api/fundamentals/expansions) 参数来指定这些字段。

在本练习中，我们将从不同对象中请求三组不同的字段：

1. 主 Post 对象中的额外 tweet.created&#95;at 字段。
2. 返回帖子对应作者的 [user 对象](/zh/x-api/fundamentals/data-dictionary#user) 的默认字段：id、name 和 username
3. 关联的 user 对象中的额外 user.created&#95;at 字段。
    

要请求这些字段，你需要在请求中传入以下内容：

|              |                |                                                                 |
| :----------- | :------------- | :-------------------------------------------------------------- |
| Key          | Value          | Returned fields（返回字段）                                           |
| tweet.fields | created&#95;at | tweets.created&#95;at                                           |
| expansions   | author&#95;id  | includes.users.id, includes.users.name, includes.users.username |
| user.fields  | created&#95;at | includes.users.created&#95;at                                   |

现在你已经了解这些信息，可以将它们组合成用于连接到流的请求 URL，其形式如下所示：

```bash
https://api.x.com/2/tweets/search/stream?tweet.fields=created_at&expansions=author_id&user.fields=created_at
```

**步骤七：连接到流并查看响应**

现在你的规则已经生效，并且你也指定了希望返回的字段，可以连接到流了。该流会返回与你提交的规则匹配的 Post 对象。下面是在请求中加入第六步中的 URL 之后，cURL 命令的形式：

```bash
curl -X GET -H "Authorization: Bearer $APP_ACCESS_TOKEN" "https://api.x.com/2/tweets/search/stream?tweet.fields=created_at&expansions=author_id&user.fields=created_at"
```

再次提醒，此请求必须使用 OAuth 2.0 应用仅模式进行身份验证，因此在将其复制并粘贴到命令行工具之前，请务必将 $APP&#95;ACCESS&#95;TOKEN 替换为你自己的凭证。

一旦你连接到 filtered stream，你就会开始以如下 JSON 格式接收与你的规则匹配的帖子：

