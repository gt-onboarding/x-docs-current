Postman 是一个非常实用的工具，可用于测试 API 端点。每个 Postman 请求都会包含所有路径和请求体参数，帮助你快速了解可用的参数和选项。要进一步了解我们的 Postman 集合，请访问我们的[《使用 Postman》](/zh/tutorials/postman-getting-started)页面。

  ### 关键概念

<div id="authentication">
  #### 身份验证
</div>

所有 X API v2 端点都要求你使用一组凭证（亦称为密钥和令牌）对请求进行身份验证。 

这些特定的端点要求使用 [OAuth 2.0 带 PKCE 的授权码流程](/zh/resources/fundamentals/authentication/oauth-2-0/authorization-code)，这意味着你必须使用一组密钥和用户访问令牌才能成功发出请求。访问令牌必须与你代其发出请求的用户相关联。如果你想为另一位用户生成一组访问令牌，他们必须通过 [Authorize URL](/zh/resources/fundamentals/authentication#authorize-url) 授权或对你的应用进行认证。

请注意，OAuth 2.0 在使用上可能比较复杂。如果你不熟悉这种身份验证方式，建议使用某个[库](/zh/x-api/tools-and-libraries/overview)或 Postman 等工具来正确完成请求的身份验证。

<div id="developer-portal-projects-and-developer-apps">
  #### 开发者门户、项目和开发者应用
</div>

要获取一组可用于 X API v2 端点的身份验证凭证，你必须拥有一个[开发者账号](/zh/resources/fundamentals/developer-portal)，在该账号下创建一个[项目](/zh/resources/fundamentals/projects)，并在该项目中创建一个[开发者应用](/zh/resources/fundamentals/developer-apps)。然后你可以在该开发者应用中找到你的密钥和令牌。 

<div id="rate-limits">
  #### 速率限制
</div>

每天都有成千上万的开发者向 X API 发起请求。为了管理庞大的请求量，我们在每个端点上都设置了[速率限制](/zh/resources/fundamentals/rate-limits)，用以限制你代表你的应用或已认证用户所能发起的请求次数。 

这些端点按用户级别进行速率限制，这意味着你代表其发起请求的已认证用户，在任何开发者应用中调用该端点的次数都有上限。对于 GET 方法，用户级别的速率限制是每个 15 分钟窗口内 180 次请求。使用 Bookmarks 查询端点的 GET 方法时，你将会获得最近添加书签的最多 800 条帖子。此外，对于 POST 和 DELETE 方法，用户级别的速率限制是每 15 分钟 50 次请求。
