<div id="manage-bookmarks">
  ### 管理书签
</div>

我们提供两种用于管理书签的方法：POST 和 DELETE。POST 方法可用于创建书签，同样，DELETE 方法允许你删除特定书签。 

对于 POST 和 DELETE 方法，每位用户在每 15 分钟内的请求速率上限为 50 次。

由于你是在代表用户调用用于管理书签的端点，因此必须通过 OAuth 2.0 生成用户访问令牌进行认证。你可以使用[带 PKCE 的授权码授权流程](/zh/resources/fundamentals/authentication/oauth-2-0/user-access-token)来完成此操作。要使用此端点，你必须传入 `tweet.read`、`users.read` 和 `bookmark.write` 这几个 scope。

<div id="bookmarks-lookup">
  ### 书签查询
</div>

Bookmarks 查询端点仅提供一种方法：GET。使用此方法，你可以获取你自己或已认证账号的书签。对于该端点的大量结果集，会提供分页令牌，以便对结果进行分页浏览。

GET 方法对每位用户的请求频率限制为每 15 分钟窗口 180 个请求。通过此端点，你最多可以取回最近的 800 条你已添加书签的帖子。

由于在使用 Bookmarks 查询端点时，你是代表某个用户发起请求，因此必须通过 OAuth 2.0 生成用户访问令牌进行认证。你可以使用[带 PKCE 的授权码授予流程](/zh/resources/fundamentals/authentication/oauth-2-0/user-access-token)来完成此操作。要使用该端点，你必须在请求中包含 `tweet.read`、`users.read` 和 `bookmark.read` 这些 scope。

**账号设置**

要访问这些端点，你需要：

* 一个已获批准的[开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)。
* 使用位于某个[项目](/zh/resources/fundamentals/projects)中的[开发者应用](/zh/resources/fundamentals/developer-apps)提供的密钥和令牌进行认证。

在我们的[入门指南](/zh/x-api/getting-started/getting-access)中了解更多关于获取 X API v2 端点访问权限的信息。
