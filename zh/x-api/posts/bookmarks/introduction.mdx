<div id="manage-bookmarks">
  ### 管理书签
</div>

我们提供两种用于管理书签的方法：POST 和 DELETE。POST 方法用于创建书签，同样，DELETE 方法用于删除特定书签。 

对于 POST 和 DELETE 方法，每位用户每 15 分钟的请求上限为 50 次。

由于你是代表某个用户调用管理书签的端点，因此必须通过 OAuth 2.0 生成用户访问令牌进行认证。你可以使用[带 PKCE 的授权码授权流程](/zh/resources/fundamentals/authentication/oauth-2-0/user-access-token)来完成认证。要使用此端点，你必须传入 `tweet.read`、`users.read` 和 `bookmark.write` 这几个 scope。

<div id="bookmarks-lookup">
  ### 书签查找
</div>

Bookmarks 查找端点仅提供一种方法：GET。此方法允许你从你自己的账号或已认证账号中获取书签内容。对于该端点的大规模结果集，将提供分页令牌，便于按页检索结果。

对于 GET 方法，每个用户在 15 分钟时间窗口内的速率限制为 180 次请求。通过此端点，你最多可以获取你最近添加书签的 800 条帖子。

由于你是通过 Bookmarks 查找端点代表用户发起请求，因此必须使用 OAuth 2.0 生成用户访问令牌进行认证。你可以使用 [Authorization Code with PKCE grant flow](/zh/resources/fundamentals/authentication/oauth-2-0/user-access-token) 来完成这一操作。要使用此端点，你必须传入 `tweet.read`、`users.read` 和 `bookmark.read` 这些权限范围（scopes）。

**账号设置**

要访问这些端点，你需要：

* 一个已获批准的[开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)。
* 使用位于某个[项目](/zh/resources/fundamentals/projects)中的[开发者应用](/zh/resources/fundamentals/developer-apps)的密钥和令牌进行认证。 

在我们的[入门指南](/zh/x-api/getting-started/getting-access)中了解更多关于获取 X API v2 端点访问权限的信息。
