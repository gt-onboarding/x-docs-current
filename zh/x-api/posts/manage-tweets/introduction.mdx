使用 X API 创建和删除帖子是参与公共对话的关键。新的 manage Posts 端点不仅能帮你完成这些操作，还提供更多功能。

我们提供两种用于管理帖子的 HTTP 方法：POST 和 DELETE。POST 方法允许你发布投票、引用帖子、设置回复权限发帖、附带地理信息发帖、附带媒体并标记用户发帖，以及向 Super Followers 发帖等。相应地，DELETE 方法允许你删除特定帖子。对于 POST 方法，你可以传入[所需的参数](/zh/x-api/posts/creation-of-a-post)，以进一步自定义你的请求。

`delete Post` 方法已经更新，以支持已编辑的帖子。当某个帖子编辑链中的任意一条被删除时，该编辑链中的所有帖子都会被删除。要了解更多 Edit Post 元数据的信息，请参阅 [Edit Posts 基础知识](/zh/x-api/fundamentals/edit-posts)页面。

POST 方法在用户级别有每 15 分钟 200 次请求的频率限制。DELETE 方法的频率限制为每 15 分钟 50 次请求。此外，还有每 3 小时 300 次请求的上限，其中包括通过 manage Posts 或 manage Retweets 创建帖子的请求。

由于你是通过 manage Posts 端点代表某个用户发起请求，因此必须使用 [OAuth 1.0a User Context](https://en.docs/authentication/oauth-1-0a) 或 [OAuth 2.0 Authorization Code with PKCE](https://en.docs/authentication/oauth-2-0/authorization-code) 进行身份验证，并使用已授权你的应用的用户所关联的用户访问令牌。要使用 OAuth 1.0a 生成该用户访问令牌，你可以使用 [3-legged OAuth flow](https://en.docs/authentication/oauth-2-0/bearer-tokens)。要使用 OAuth 2.0 生成用户访问令牌，你可以使用 [Authorization Code with PKCE grant flow](https://en.docs/authentication/oauth-2-0/user-access-token)。

<Info>
  **账号设置**

  要访问这些端点，你需要：

  * 一个已经获批的[开发者账号](https://en/portal/petition/essential/basic-info)。
  * 使用位于某个[项目](/zh/resources/fundamentals/projects)中的[开发者应用](/zh/resources/fundamentals/developer-apps)所提供的密钥和令牌进行身份验证。

  在我们的[入门指南](/zh/x-api/getting-started/getting-access)中进一步了解如何获取 X API v2 端点的访问权限。
</Info>
