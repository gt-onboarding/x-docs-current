使用 X API 创建和删除帖子是参与公共对话的关键。新的 manage Posts 端点不仅可以实现这些操作，还支持更多功能。

我们提供两种用于管理帖子的 HTTP 方法：POST 和 DELETE。POST 方法允许你发布投票、引用帖子、设置回复权限的帖子、带地理位置信息的帖子、带媒体并标记用户的帖子，以及向 Super Followers 发布帖子，此外还有其他功能。同样，DELETE 方法允许你删除特定的帖子。对于 POST 方法，你可以传入所需的[参数](/zh/x-api/posts/creation-of-a-post)，以进一步自定义你的请求。

“删除帖子”方法已更新，以支持已编辑的帖子。当帖子编辑链中的任意一个帖子被删除时，该编辑链中的所有帖子都会被一并删除。要进一步了解 Edit Post 元数据，请查看 [Edit Posts 基础知识](/zh/x-api/fundamentals/edit-posts) 页面。

POST 方法的用户级速率限制为每 15 分钟 200 个请求。DELETE 方法的速率限制为每 15 分钟 50 个请求。此外，还有每 3 小时 300 个请求的限制，其中包括通过 manage Posts 或 manage Retweets 创建的帖子。

由于你是通过 manage Posts 端点代表用户发起请求，因此必须使用 [OAuth 1.0a User Context](https://en.docs/authentication/oauth-1-0a) 或 [OAuth 2.0 Authorization Code with PKCE](https://en.docs/authentication/oauth-2-0/authorization-code) 进行身份验证，并使用与已授权你的应用的用户关联的用户访问令牌。要使用 OAuth 1.0a 生成此用户访问令牌，你可以使用 [三方 OAuth 流程](https://en.docs/authentication/oauth-2-0/bearer-tokens)。要使用 OAuth 2.0 生成用户访问令牌，你可以使用 [Authorization Code with PKCE 授权流程](https://en.docs/authentication/oauth-2-0/user-access-token)。

  * 使用位于某个[项目](/zh/resources/fundamentals/projects)中的[开发者应用](/zh/resources/fundamentals/developer-apps)所提供的密钥和令牌进行身份验证。

  在我们的[入门指南](/zh/x-api/getting-started/getting-access)中了解更多关于如何获取 X API v2 端点访问权限的信息。
