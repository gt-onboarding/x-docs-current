<div id="standard-v11-compared-to-x-api-v2">
  ## 标准 v1.1 与 X API v2 的比较
</div>

如果你一直在使用标准 v1.1 的 [POST statuses/update](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-update) 和 [POST statuses/destroy/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-destroy-id) 端点，本指南旨在帮助你了解标准端点与 X API v2 管理帖子端点之间的相似之处和差异。

* **相似点**
  * 认证
* **差异**
  * 端点 URL

  * 应用和项目的要求

  * 请求参数

<div id="similarities">
  ### 相似之处
</div>

**身份验证**

标准 v1.1 和 X API v2 中用于管理帖子（[POST statuses/update](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-update) 和 [POST statuses/destroy/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-destroy-id)）的端点都使用 [OAuth 1.0a 用户上下文](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication)。因此，如果你之前在使用某个标准 v1.1 端点，那么在迁移到对应的 X API v2 版本时，可以继续使用相同的身份验证方式。 

<div id="differences">
  ### 差异
</div>

**端点 URL**

* 标准 v1.1 端点：
  * https://api.x.com/1.1/statuses/update.json
    （创建一个帖子）
  * `https://api.x.com/1.1/statuses/destroy/:id.json`
    （删除一个帖子）
* X API v2 端点：
  * https://api.x.com/2/tweets
    （创建一个帖子）
  * https://api.x.com/2/tweets/:id
    （删除指定的帖子）

<div id="app-and-project-requirements">
  ### 应用和项目要求
</div>

在对请求进行认证时，X API v2 端点要求你使用与某个[项目](/zh/resources/fundamentals/developer-apps)关联的[开发者应用](/zh/resources/fundamentals/developer-apps)的凭证。所有 X API v1.1 端点都可以使用来自独立应用或与项目关联的应用的凭证。

<div id="request-parameters">
  ### 请求参数
</div>

以下 v1.1 标准请求参数只接受两个查询参数（user&#95;id 或 screen&#95;name）。X API v2 的 DELETE 端点仅接受数值形式的帖子 ID，并且必须作为端点路径的一部分传入。

对于 POST 端点，需要通过请求的 JSON 正文传入额外参数。你可以在 [API 参考指南](/zh/x-api/posts/manage-tweets/introduction) 中了解可用参数的详细信息。
