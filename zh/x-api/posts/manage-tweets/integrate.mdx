<div id="postman">

</div>

Postman 是一款非常适合用来测试端点的工具。每个 Postman 请求都会包含所有路径和请求体参数，帮助你快速了解你可以使用的内容。要进一步了解我们的 Postman 集合，请访问我们的「[使用 Postman](/zh/tutorials/postman-getting-started)」页面。

<div id="authentication">
  ### 身份验证
</div>

所有 X API v2 端点都要求你使用一组凭证（也称为密钥和令牌）对请求进行身份验证。 

这些特定的端点要求使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication)，这意味着你必须使用一组 API 密钥和用户访问令牌才能成功发出请求。访问令牌必须与你代表其发出请求的用户相关联。如果你想为其他用户生成一组访问令牌，他们必须通过使用 [三方 OAuth 流程](/zh/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) 来授权或验证你的应用。

请注意，OAuth 1.0a 使用起来可能比较复杂。如果你不熟悉这种身份验证方法，我们建议你使用一个[库](/zh/x-api/tools-and-libraries/overview)、使用 Postman 之类的工具，或者使用 OAuth 2.0 来对请求进行身份验证。

[OAuth 2.0 Authorization Code with PKCE](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 允许对应用的权限范围（scope）以及跨多个设备的授权流程进行更精细的控制。OAuth 2.0 允许你选择特定的细粒度权限范围（scope），从而代表用户获得特定权限。 

要在你的应用中启用 OAuth 2.0，你必须在开发者门户中该应用的 “App settings” 部分找到其身份验证设置，并在其中启用 OAuth 2.0。

<div id="developer-portal-projects-and-developer-apps">
  ### 开发者门户、项目和开发者应用
</div>

要获取可用于 X API v2 端点的一组身份验证凭据，你必须先拥有一个 [开发者账号](/zh/resources/fundamentals/developer-portal)，在该账号下设置一个 [项目](/zh/resources/fundamentals/projects)，并在该项目中创建一个 [开发者应用](/zh/resources/fundamentals/developer-apps)。然后，你可以在该开发者应用中找到你的密钥和令牌。 
 

<div id="rate-limits">
  ### 速率限制
</div>

每天都有成千上万的开发者向 X API 发起请求。为了帮助管理庞大的请求量，每个端点都会设置[速率限制](https://developer.x.com/x-api/fundamentals/rate-limits)，从而限制你代表你的应用或代表已认证用户所能发起的请求次数。

这些端点在用户级别施加速率限制，这意味着，无论通过任何开发者应用，你所代表的已认证用户调用该端点的次数都被限定在一定范围内。对于 POST 方法，用户级别的速率限制是每 15 分钟 200 次请求。对于 DELETE 方法，速率限制是每 15 分钟 50 次请求。此外，还存在一个[每 3 小时 300 次请求](https://blog.x.com/developer/en_us/topics/tools/2018/new-developer-requirements-to-protect-our-platform)的限制，其中包括通过 manage Posts 或 manage Retweets 创建的帖子。
 

<div id="source-labels">
  ### 源标签
</div>

您的应用名称和网站 URL 会作为[源标签](https://help.x.com/en/using-twitter/how-to-tweet#source-labels)显示在由您的应用通过编程方式创建的任何帖子元数据中。如果您更改了某个 X App 的用例，请务必在这些设置中更新该用例，以确保您遵守[开发者条款](https://developer.x.com/content/developer-twitter/en/developer-terms/agreement-and-policy)。
 

<div id="profile-settings">
  ### 个人资料设置
</div>

只有在个人资料设置中启用了地理位置功能（geo）时，你才能在帖子中添加位置。如果你没有启用地理位置，你仍然可以在请求体中添加 location 参数，但该位置不会附加到你的帖子上。对在图片中标记用户也是同样的道理。如果你要标记的用户没有启用照片标记功能，他们的名字就不会出现在被标记用户列表中，即便帖子已经成功创建。 
 

<div id="adding-media-to-a-post">
  ### 向帖子添加媒体
</div>

目前，X API 的 v2 尚不支持从头到尾完成媒体上传。不过，你可以将之前已上传的媒体附加到帖子中。你可以使用已经[通过 v1 媒体上传端点上传](https://developer.x.com/en/docs/twitter-api/v1/media/upload-media/api-reference/post-media-upload)或[通过 X Media Studio 上传](https://media.x.com/en/articles/products/2018/media-studio)的 media id。这些 media id 必须属于你自己或已通过身份验证的用户。
