Postman 是一款非常实用的 API 端点测试工具。每个 Postman 请求都会包含所有路径参数和请求体参数，帮助你快速了解可用的内容。要进一步了解我们的 Postman 集合，请访问我们的「[使用 Postman](/zh/tutorials/postman-getting-started)」页面。

<div id="authentication">
  ### 身份验证
</div>

所有 X API v2 端点都要求你使用一组凭证（也称为密钥和令牌）对请求进行身份验证。 

这些端点需要使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication)，这意味着你必须使用一组 API 密钥和用户访问令牌才能成功发出请求。访问令牌必须与你代表其发出请求的用户关联。如果你想为其他用户生成一组访问令牌，他们必须通过使用 [三方 OAuth 流程](/zh/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) 来授权或验证你的应用。

请注意，OAuth 1.0a 可能较难使用。如果你不熟悉此身份验证方法，我们建议你使用一个[库](/zh/x-api/tools-and-libraries/overview)、使用 Postman 之类的工具，或使用 OAuth 2.0 对请求进行身份验证。

[OAuth 2.0 Authorization Code with PKCE](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 允许你对应用的权限范围（scope）以及跨多设备的授权流程进行更精细的控制。OAuth 2.0 允许你选择特定的细粒度权限范围（scope），以便代表用户获得相应的权限。 

要在你的应用中启用 OAuth 2.0，必须在开发者门户中该应用的身份验证设置里将其启用，该设置位于 App 设置部分。

<div id="developer-portal-projects-and-developer-apps">
  ### 开发者门户、项目和开发者应用
</div>

要获取可用于 X API v2 端点的一组身份验证凭证，你必须拥有一个 [开发者账号](/zh/resources/fundamentals/developer-portal)，在该账号下设置一个 [项目](/zh/resources/fundamentals/projects)，并在该项目中创建一个 [开发者应用](/zh/resources/fundamentals/developer-apps)。然后你就可以在开发者应用中找到相应的密钥和令牌。 
 

<div id="rate-limits">
  ### 速率限制
</div>

每天都有成千上万的开发者向 X API 发起请求。为了帮助管理如此庞大的请求量，我们在每个端点上都设置了[速率限制](https://developer.x.com/x-api/fundamentals/rate-limits)，从而限制你代表你的应用或代表已认证用户所能发起的请求次数。

这些端点采用用户级别的速率限制，这意味着，你代表其发起请求的已认证用户，无论通过哪一个开发者应用调用该端点，调用次数都是有限的。对于 POST 方法，用户级别的速率限制为每 15 分钟 200 次请求。对于 DELETE 方法，速率限制为每 15 分钟 50 次请求。此外，还有一个[每 3 小时 300 次请求](https://blog.x.com/developer/en_us/topics/tools/2018/new-developer-requirements-to-protect-our-platform)的限制，其中包括通过 manage Posts 或 manage Retweets 创建的帖子。
 

<div id="source-labels">
  ### 来源标签
</div>

通过你的应用程序以编程方式创建的任何帖子，其元数据中都会将你的应用名称和网站 URL 显示为[来源标签](https://help.x.com/en/using-twitter/how-to-tweet#source-labels)。如果你更改了某个 X 应用的使用场景，请务必在这些设置中同步更新使用场景，以确保你符合[开发者条款](https://developer.x.com/content/developer-twitter/en/developer-terms/agreement-and-policy)的要求。
 

<div id="profile-settings">
  ### 个人资料设置
</div>

只有在你在个人资料设置中启用了地理位置功能时，你才能在帖子中添加位置。如果你没有启用地理位置功能，仍然可以在请求体中包含位置参数，但它不会附加到你的帖子中。为图片中的用户添加标记也是同样的道理。如果你要标记的用户没有启用照片标记功能，即使帖子已成功创建，他们的名字也不会出现在已标记用户列表中。 
 

<div id="adding-media-to-a-post">
  ### 向帖子添加媒体
</div>

目前，尚无法通过 X API v2 直接上传媒体。不过，你可以将之前已上传的媒体附加到帖子中。你可以使用已[通过 v1 媒体端点上传](https://developer.x.com/en/docs/twitter-api/v1/media/upload-media/api-reference/post-media-upload)或 [X Media Studio](https://media.x.com/en/articles/products/2018/media-studio) 上传的 media id。这些 media id 必须属于你自己，或属于已通过身份验证的用户。
