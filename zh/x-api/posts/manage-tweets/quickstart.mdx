## 开始使用管理帖子端点**

本快速入门指南将帮助你使用 [Postman](/zh/tutorials/postman-getting-started) 向管理帖子端点发送你的第一个请求。

如果你想查看不同语言的示例代码，请访问我们的 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 仓库。

<Info>
  **先决条件**

  要完成本指南，你需要一组 [密钥和令牌](/zh/resources/fundamentals/authentication)，用于对请求进行身份验证。你可以按照以下步骤生成这些密钥和令牌：

  * [注册开发者账号](https://developer.x.com/en/apply-for-access) 并通过审核。
  * 在开发者门户中创建一个 [项目](/zh/resources/fundamentals/projects) 以及关联的 [开发者应用](/zh/resources/fundamentals/developer-apps)。
  * 前往你的应用的“Keys and tokens”页面以生成所需凭证。请确保将所有凭证保存在安全的位置。
</Info>

<div id="steps-to-building-manage-posts-requests">
  ### 构建 manage Posts 请求的步骤
</div>

**步骤一：从工具或库开始**

有多种不同的工具、代码示例和库可用于向此端点发起请求，但为了简化流程，我们将在这里使用 Postman 工具。

要将 X API v2 Postman 集合加载到你的环境中，请点击以下按钮：

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  将 X API v2 添加到 Postman
</Button>

在你将 X API v2 集合加载到 Postman 之后，导航到 “Manage Posts” 文件夹，然后选择 “Create a Post”。
 

**步骤二：对请求进行身份验证**

要正确调用 X API，你需要验证自己是否有权限这样做。对于 manage Posts 端点，你必须使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 或 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 对请求进行身份验证。

在本示例中，我们将使用 OAuth 1.0a 用户上下文。

你必须将你的密钥和令牌（特别是 API Key、API Secret Key、OAuth 1.0a 用户 Access Token 和 OAuth 1.0a 用户 Access Token Secret）添加到 Postman 中。你可以通过选择名为 “X API v2” 的环境（位于 Postman 右上角），并将你的密钥和令牌添加到 &quot;initial value&quot; 和 &quot;current value&quot; 字段中（点击环境下拉菜单旁边的眼睛图标）来完成此操作。

如果你已正确完成这些操作，这些变量会自动被填入请求的 authorization 选项卡中。

**步骤三：指定帖子的文本内容**

使用此端点创建新帖子时，帖子的文本或媒体是必需的请求体参数。

在 Postman 中，导航到 “Body” 选项卡，并将帖子的文本作为 `text` 参数的值输入。此外，如果你希望添加诸如投票、回复某个帖子 ID 或添加回复设置等参数，也可以在此处进行。你可以在我们的 [API 参考指南](/zh/x-api/posts/manage-tweets/introduction) 中了解更多可用内容。

|         |              |                    |
| :------ | :----------- | :----------------- |
| **Key** | **Value**    | **Parameter type** |
| `text`  | Hello world! | body               |

**步骤四：确定并指定你希望检索的字段**

完成所有设置后，点击 &quot;Send&quot; 按钮，你将收到与以下示例响应类似的响应：

```
{
  "data": {
    "id": "1445880548472328192",
    "text": "Hello world!"
  }
}
```

如果返回的响应对象中包含你的帖子对应的 `id` 和 `text` 字段，则说明你已成功创建一个帖子。
 

**步骤五：删除你的帖子**

要删除帖子，请在 Postman 中已加载的 X API v2 collection 的 “Manage Posts” 文件夹中选择 “Delete a Post” 请求。该端点需要你想要删除的帖子的 ID。然后，在 “Params” 选项卡中，将你想要删除的帖子的 ID 作为 `id` 列的值进行填写。 

当删除请求成功时，你会收到类似于以下示例的响应：

```
{
   "data": {
       "deleted" : true
   }
}
```
