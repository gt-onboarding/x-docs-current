  ## 比较 X API 的隐藏回复端点
</div>

v2 隐藏回复端点将取代 Labs 隐藏回复端点。如果你已有使用 Labs 版本该端点的代码、应用或工具，并且正在考虑迁移到较新的 X API v2 端点，那么本指南适合你。

要使用新的 X API v2（包括隐藏回复端点），你需要先[选择加入新的开发者门户](https://developer.x.com/en/portal/opt-in)，创建一个[项目](/zh/resources/fundamentals/projects)，并在该项目中添加一个应用。然后你可以使用与该应用关联的凭证来向隐藏回复端点发出请求。将已注册 Labs v2 隐藏回复端点的同一个应用添加到该项目中，可以保持你的用户授权状态不变。

下表比较了 Labs 与较新的 X API v2 端点之间的差异：
 

| **说明** | **Labs v2** | **X API v2** |
| :--- | :--- | :--- |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /labs/2/tweets/:id/hidden | /2/tweets/:id/hidden |
| 认证方式 | OAuth 1.0a 用户上下文 | OAuth 1.0a 用户上下文 |
| 支持的 HTTP 方法 | PUT | PUT |
| 默认请求速率限制 | 每 15 分钟 10 次请求（在所有已认证用户之间共享） | 每位已认证用户每 15 分钟 50 次请求 |
| 是否可以隐藏回复 | ✔︎  | ✔︎  |
| 是否可以取消隐藏先前已隐藏的回复 | ✔︎  | ✔︎  |
| 是否可以多次隐藏或取消隐藏回复 | ✔︎  | ✔︎  |
| 是否需要使用来自[开发者应用](/zh/resources/fundamentals/developer-apps)（该应用隶属于某个[项目](/zh/resources/fundamentals/projects)）的凭证 |     | ✔   |
