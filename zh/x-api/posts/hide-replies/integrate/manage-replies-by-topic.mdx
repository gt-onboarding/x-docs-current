<div id="manage-replies-by-topic">
  ### 按主题管理回复
</div>

通过 hide replies 端点，你可以构建集成方案，帮助个人和品牌让他们的对话始终围绕指定主题展开。此页面展示了如何使用 hide replies 和 [recent search](/zh/x-api/posts/recent-search) 端点来管理一段对话。

Recent search 具备获取一段对话及其回复的功能，而 Post 负载会返回 [Post annotations](/zh/x-api/fundamentals/post-annotations)，帮助你理解每条 Post 的上下文和主题，而不受语言影响。

应用的流程将包含用于展示和管理对话的控件：

1. 它会请求用户授权读取其帖子并管理其回复。

2. 它会通过某个 Post URL 获取一段最近的对话，并检查该对话是否来自进行身份验证的用户。

3. 它会调用 recent search 端点来展示对话中的每条 Post。请求将包含以会话 ID 为条件的搜索查询以及注解扩展，用于根据 X 对该 Post 的理解来确定该 Post 是否与体育相关。

4. 当用户选择隐藏某条回复时，它会调用 hide replies 来隐藏该回复。同时，它还会提供撤销此操作的方式，以确保用户始终掌控全局。 

5. 对于较长的对话，它会提供控件以便[对搜索结果进行分页](/zh/x-api/posts/search/integrate/paginate)。
    

<div id="optimize-for-the-user-and-for-usage">
  #### 以用户为中心进行优化（同时兼顾使用效率）
</div>

你可以设计一个流程，让用户始终掌控他们打算执行的任何操作。牢记这一原则还能帮助你构建一个能优化帖子消费体验的集成方案。

1. 由于已认证用户只能管理他们自己发起的会话，当情况并非如此时，你的流程应尽早终止。
   * 先发起一次初始帖子查找（lookup）请求。如果帖子 URL 无效，或者会话并非由进行身份验证的用户发起，则应尽早终止流程。
   * 这样一来，如果该会话无法由已认证用户进行管理，你的应用就不必再发起 Recent search 请求。
2. 在同一个请求中请求 [user 和 Post fields](/zh/x-api/fundamentals/fields)，以避免发起单独的请求。这种做法还能提升应用性能。
3. 尽量避免发起不必要的请求。此应用会在用户的浏览器中缓存某条回复的隐藏状态。这对于较大的会话非常有用，因为用户可能希望在后续阶段继续他们的内容管理工作，同时这也有助于你的应用优化用于隐藏或取消隐藏回复的请求次数。
