<div id="manage-replies-by-topic">
  ### 按话题管理回复
</div>

通过 hide replies 端点，你可以构建集成，帮助用户和品牌让他们的对话保持围绕正确的话题展开。本页展示了如何使用 hide replies 和 [recent search](/zh/x-api/posts/recent-search) 端点来管理一段对话。

Recent search 具备检索一段对话及其回复的功能，而 Post 有效负载会返回 [Post annotations](/zh/x-api/fundamentals/post-annotations)，帮助你理解每条 Post 的上下文和话题，而不受语言限制。

应用流程将包括用于显示和管理对话的控件：

1. 它会请求用户授权读取他们的帖子并管理他们的回复。

2. 它会从一个 Post URL 中提取最近的一段对话，并检查该对话是否来自当前进行身份验证的用户。

3. 它会调用 recent search 端点来展示对话中的每条 Post。请求将包含对话 ID 的搜索查询以及注释扩展，以根据 X 对该 Post 的理解判断该 Post 是否与体育相关。

4. 当用户选择隐藏某条回复时，它会调用 hide replies 来隐藏该回复。同时，它还会提供一种方式来撤销此操作，以确保用户始终保持控制权。 

5. 对于较长的对话，它会提供控件以[对搜索结果进行分页](/zh/x-api/posts/search/integrate/paginate)。
    

<div id="optimize-for-the-user-and-for-usage">
  #### 为用户（以及使用效率）进行优化
</div>

你可以通过设计流程，让用户始终掌控自己打算执行的任何操作。牢记这一原则还能帮助你构建一个能够针对帖子内容消费进行优化的集成方案。

1. 由于经身份验证的用户只能管理由自己发起的会话，当不满足这一条件时，你的流程应尽早终止。
   * 先发起一次初始的帖子查询请求。如果帖子 URL 无效，或该会话不是由经身份验证的用户发起，则应尽早终止流程。
   * 这样一来，如果该会话无法由经身份验证的用户进行管理，你的应用就不必再发起近期搜索请求。
2. 在同一个请求中同时请求[用户和帖子字段](/zh/x-api/fundamentals/fields)，以避免发起多个独立请求。此举还可以提升你的应用性能。
3. 避免在不必要时发起请求。该应用会在用户的浏览器中缓存回复的隐藏状态。这在较大的会话中很有用，因为用户可能希望稍后再继续他们的审核工作，同时这也有助于你的应用优化用于隐藏或取消隐藏回复的请求次数。
