<div id="manage-replies-in-realtime">
  ### 实时管理回复
</div>

通过使用 hide replies 端点，你可以构建一个工作流程，帮助用户隐藏那些高度可能与主题无关的回复。

实用的应用通常会结合多种技术，从而为用户提供价值。本页展示了如何使用 [Perspective API](https://www.perspectiveapi.com/) 来隐藏回复。该 API 是由 Google 内部的一个部门 [Jigsaw](https://jigsaw.google.com/) 训练的人工智能，用于检测有害评论。应用逻辑的工作方式如下：

1.  它会请求用户授权读取他们的帖子，并隐藏或取消隐藏他们的回复。
2.  它使用 [Account Activity API](https://developer.x.com/en/docs/x-api/v1/accounts-and-users/subscribe-account-activity/overview) 来检测收到的回复。
3.  它请求 Perspective API 给出一个“评分”（介于 0 到 1 之间的数字），以表明其算法对某条评论与其过去见过的有害评论相似的置信程度。（Perspective 不会存储发送到该服务的文本。）
4.  如果算法的评分非常高，它就会调用 hide replies 端点。

<div id="strive-for-transparency">
  #### 尽量做到透明
</div>

由于 Perspective 并未在真实的帖子数据上进行训练，某些细微的语言差异可能会导致此应用隐藏用户本希望保持可见的回复。无论你在设计应用时使用何种技术或方法，都应尽最大努力确保用户明白你的应用隐藏了哪些内容，并且能够进行调整。

* 最理想的做法是始终信任用户，并将决定权完全交给他们。这意味着在你的用户体验中，应提供选项或控件，让用户可以撤销应用代表他们所执行的任何操作。
* 当使用人工智能时，你的应用在检测和隐藏帖子时应采用非常高的置信度阈值。
* 并不是所有人都会使用相同的词语，你的应用在设计时应避免任何偏见。请注意那些被重新赋予积极含义的词语和俚语，它们可能会导致误报。
* 如果你在训练人工智能，考虑采用一个能紧密贴合 X 上常用语言的模型。
