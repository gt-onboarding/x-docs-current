<div id="getting-started-with-the-hide-replies-endpoint">
  ## 开始使用隐藏回复端点
</div>

本快速入门指南将帮助你使用 [Postman](/zh/tutorials/postman-getting-started) 向隐藏回复端点发送第一个请求。

如果你希望查看不同编程语言的代码片段，请访问 [隐藏回复 API 参考页面](/zh/x-api/posts/hide-replies#put-2-tweets-id-hidden)。 

<Note>
  ### 前提条件

  要完成本指南，你需要准备一组用于对请求进行认证的 [密钥和令牌](/zh/resources/fundamentals/authentication)。你可以通过以下步骤生成这些密钥和令牌：

  * [注册开发者账号](https://developer.x.com/en/apply-for-access) 并通过审核。
  * 在开发者门户中创建一个 [项目](/zh/resources/fundamentals/projects) 以及关联的 [开发者应用](/zh/resources/fundamentals/developer-apps)。
  * 前往你的应用的 “Keys and tokens” 页面以生成所需的凭证。请确保将所有凭证保存在安全的位置。
</Note>

<div id="steps-to-build-a-put-tweetsidhidden-request">
  ### 构建 PUT /tweets/:id/hidden 请求的步骤
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### 步骤一：从工具或库入手
</div>

你可以使用多种不同的工具、代码示例和库来向此端点发起请求，但为了简化流程，我们将在这里使用 Postman 工具。

要将 X API v2 Postman 集合加载到你的环境中，请点击下面的按钮：

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  将 X API v2 添加到 Postman
</Button>

在你将 X API v2 集合加载到 Postman 之后，导航到隐藏回复端点（hide replies endpoint）。

<div id="step-two-authenticate-your-request">
  #### 步骤二：为请求进行身份验证
</div>

要正确向 X API 发出请求，需要先验证你是否拥有相应的权限。对于此端点，你必须使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 或 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 对请求进行身份验证。

在本示例中，我们将使用 OAuth 1.0a 用户上下文。

你必须将你的密钥和令牌——具体来说是 API Key、API Secret Key、OAuth 1.0a 用户 Access Token，以及 OAuth 1.0a 用户 Access Token Secret——添加到 Postman 中。你可以在 Postman 右上角选择名为“X API v2”的环境（environment），然后在 &quot;initial value&quot; 和 &quot;current value&quot; 字段中添加你的密钥和令牌（点击环境下拉菜单旁边的眼睛图标）来完成此操作。

如果上述操作配置正确，这些变量会自动填充到该请求的 authorization 选项卡中。
 

<div id="step-three-find-a-post-id-to-hide">
  #### 步骤三：找到要隐藏的帖子 ID
</div>

hide replies 接口可以代表已授权用户隐藏或取消隐藏回复。因为在本示例中我们使用的是与你的用户个人资料关联的访问令牌，你将能够隐藏由你发起的对话中其他用户的回复。同样，如果你使用的是属于另一位已授权你的应用的用户的访问令牌，你就可以管理该账号发起的任何对话中的回复。

请朋友回复一条帖子（告诉 TA 你在测试隐藏回复功能），或者使用测试账号回复你自己的任意帖子。点击该回复，然后复制其 URL 中的数字部分。这就是我们要隐藏的帖子 ID。

在本例中，我们将查看以下帖子，它的 ID 为 `1232720193182412800`：

`https://x.com/TwitterDev/status/1232720193182412800`

<div id="step-four-hide-the-post">
  #### 步骤四：隐藏帖子
</div>

在 Postman 中，打开 Hide replies 文件夹并选择 Hide a reply。在 Params 选项卡中，在 `id` 字段旁边粘贴帖子 ID（无需替换 URL 中的 `:id`）。点击“Send”后，你会看到一个成功的响应。

`{"hidden":true}`

<div id="step-five-unhide-the-post">
  #### 步骤五：取消隐藏帖子
</div>

已隐藏的帖子会被移动到 X 应用中的单独标签页。要在 Postman 中取消隐藏帖子，打开 Hide replies 文件夹并选择 Unhide a reply。在 Params 选项卡中，将上一步使用的同一个帖子 ID 粘贴到 `id` 字段中。点击“Send”，你将看到一个成功的响应。

`{"hidden":false}`

`hidden` 字段表示帖子的隐藏状态。隐藏状态为 `true` 表示帖子已被隐藏，同样，`false` 表示帖子未被隐藏。
