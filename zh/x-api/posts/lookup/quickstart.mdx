<div id="getting-started-with-the-posts-lookup-endpoints">
  ## 开始使用 Posts 查找端点
</div>

本快速入门指南将帮助你使用 Postman，向 Posts 查找端点发送第一个包含指定字段的请求。

如果你希望查看不同语言的示例代码，请访问我们的 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 仓库。

<Note>
  ### 前提条件

  要完成本指南，你需要一组用于对请求进行身份验证的[密钥和令牌](/zh/resources/fundamentals/authentication)。你可以按照以下步骤生成这些密钥和令牌：

  * [注册开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)并通过审批。
  * 在开发者门户中创建一个 [Project](/zh/resources/fundamentals/projects) 以及关联的 [developer App](/zh/resources/fundamentals/developer-apps)。
  * 前往你的 App 的“Keys and tokens”页面生成所需的凭证。请务必将所有凭证保存在安全的位置。
</Note>

<div id="steps-to-build-a-get-tweets-request">
  ### 构建 GET /tweets 请求的步骤
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### 第一步：先选用工具或库
</div>

你可以使用多种工具、代码示例和库来向该端点发起请求，但在这里我们将使用 Postman 工具来简化流程。

要将 X API v2 Postman 集合加载到你的环境中，请点击下面的按钮：

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  将 X API v2 添加到 Postman
</Button>

在你将 X API v2 集合加载到 Postman 之后，导航到 &quot;Post Lookup &gt; Multiple Posts&quot; 请求。

<div id="step-two-authenticate-your-request">
  #### 步骤二：对请求进行认证
</div>

要正确地向 X API 发起请求，你需要确认自己具有相应的权限。你可以通过以下方式对请求进行认证：

* [OAuth 2.0 App-Only](resources/fundamentals/authentication)
* [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication)
* [OAuth 1.0a User Context](resources/fundamentals/authentication)

为简单起见，我们将在本次请求中使用 OAuth 2.0 App-Only，但如果你想请求私有[指标](/zh/x-api/fundamentals/metrics)或帖子，则需要使用其他认证方法之一。

要使用 OAuth 2.0 App-Only，你必须将你的密钥和令牌（特别是 App Access Token，也称为 App-only Bearer Token）添加到 Postman 中：在 Postman 右上角选择名为 “X API v2” 的环境，点击环境下拉菜单旁边的眼睛图标，然后将你的密钥和令牌添加到 “initial value” 和 “current value” 字段中。

如果配置正确，这些变量会自动被填充到该请求的 Authorization 选项卡中。

<div id="step-three-identify-and-specify-which-posts-you-would-like-to-retrieve">
  #### 步骤三：确定并指定你想要检索的帖子
</div>

你必须在请求中指定一条或一组你想要接收的帖子。你可以通过访问 X.com，点击某条帖子并查看其 URL 来找到该帖子的 ID。例如，下面这个 URL 中帖子的 ID 是 `1228393702244134912`。

`https://x.com/TwitterDev/status/1228393702244134912`

在 Postman 中，切换到 &quot;Params&quot; 选项卡，并在 ids 参数的 &quot;Value&quot; 列中输入这个 ID，或输入用逗号分隔的一串帖子 ID。

| Key  | Value |
| :---- | :----- |
| `ids` | `1228393702244134912,1227640996038684673,1199786642791452673` |

***

<div id="step-four-identify-and-specify-which-fields-you-would-like-to-retrieve">
  #### 步骤四：确定并指定你想要获取的字段
</div>

如果你在完成步骤三后点击 &quot;Send&quot; 按钮，你将在响应中收到默认的 [Post 对象字段](/zh/x-api/fundamentals/data-dictionary)：`id`、`text` 和 `edit_history_tweet_ids`。

在本练习中，我们将从不同对象中额外请求三组不同的字段：

* 主 Post 对象中额外的 `tweet.created_at` 字段。
* 返回帖子所关联作者的用户对象的默认字段：`id`、`name` 和 `username`。
* 关联用户对象中额外的 `user.created_at` 字段。

在 Postman 中，导航到 &quot;Params&quot; 选项卡，并在 &quot;Query Params&quot; 表中添加以下 `key:value` 键值对：

| Key            | Value        | Returned fields                                                       |
| :------------- | :----------- | :-------------------------------------------------------------------- |
| `tweet.fields` | `created_at` | `tweets.created_at`                                                   |
| `expansions`   | `author_id`  | `includes.users.id`, `includes.users.name`, `includes.users.username` |
| `user.fields`  | `created_at` | `includes.users.created_at`                                           |

你应该会在 &quot;Send&quot; 按钮旁边看到这个 URL：

```
https://api.x.com/2/tweets?ids=1228393702244134912,1227640996038684673,1199786642791452673&tweet.fields=created_at&expansions=author_id&user.fields=created_at
```
