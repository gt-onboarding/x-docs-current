帖子是 X 上的主要资源之一。在最简单的形式下，单条帖子最多可包含 280 个字符，并且可以根据账号设置选择公开或私密发布。此外，还可以在帖子中附加多种不同的对象，包括媒体、地点、投票和 URL。此外，大多数帖子在创建后最多可以在 30 分钟内进行编辑。

虽然有多种针对帖子的 HTTP、筛选和传输方式，但这一组 REST 端点专注于返回由 [Post ID](/zh/resources/fundamentals/x-ids) 指定的单条帖子或一组帖子。这些端点可以获取帖子的最新详细信息、验证帖子是否可用，并检查其编辑历史。它们对于管理[合规事件](/zh/x-api/enterprise-gnip-2.0/fundamentals/firehouse)也至关重要。

Post 查询端点会提供已编辑帖子的元数据。自 2022 年 9 月 29 日起创建的所有帖子都包含编辑元数据，即使它们从未被编辑过。每次编辑都会生成一个新的 Post ID，某条帖子的编辑历史通过一个 Post ID 数组来表示，并以最初的版本开头。

此端点始终会返回最新编辑版本及其编辑历史。在其 30 分钟编辑窗口过期后再被收集到的任何帖子，都将表示其最终版本。要进一步了解编辑帖子的元数据，请查看[编辑帖子基础知识](/zh/x-api/fundamentals/edit-posts)页面。

这些端点使用 GET HTTP 方法，并返回一个或多个 [Post 对象](/zh/x-api/fundamentals/data-dictionary)，其中包括帖子文本、创建时间戳，以及话题标签、提及和照片的列表和元数据等字段。

  ### 账号设置

  要访问这些端点，你需要：

  * 一个已获批准的[开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)。
  * 使用某个 [Project](/zh/resources/fundamentals/projects) 下 [developer App](/zh/resources/fundamentals/developer-apps) 提供的密钥和令牌进行认证。

  在我们的[入门指南](/zh/x-api/getting-started/getting-access)中了解更多关于如何获取 X API v2 端点访问权限的信息。
</Note>
