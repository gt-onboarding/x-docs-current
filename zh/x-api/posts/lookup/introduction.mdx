Post（帖子）是 X 上的主要资源之一。在最简单的情况下，一个 Post 最多可以包含 280 个字符，并且可以根据账号设置选择公开或私密发布。此外，还可以为 Post 附加多种不同的对象，包括媒体、地点、投票和 URL。另外，大多数 Post 在创建后的 30 分钟内都可以进行编辑。

虽然有多种 HTTP、选取和投递方式可用于处理 Post，但这一组 REST 端点侧重于返回由 [Post ID](/zh/resources/fundamentals/x-ids) 指定的单个 Post 或一组 Post。这些端点可以获取某个 Post 的最新详细信息、验证该 Post 是否可用，并检查其编辑历史。它们对于管理 [合规事件](/zh/x-api/enterprise-gnip-2.0/fundamentals/firehouse) 也至关重要。

Post 查询端点会提供与编辑相关的 Post 元数据。自 2022 年 9 月 29 日起创建的所有 Post 都包含编辑元数据，即使它们从未被编辑过。每次编辑都会生成一个新的 Post ID，而某个 Post 的编辑历史则由一个 Post ID 数组表示，并以原始 Post 开始。

此端点始终会返回最近一次编辑及其编辑历史。任何在 30 分钟编辑窗口过期后收集到的 Post 都将表示其最终版本。要详细了解 Edit Post 元数据，请查看 [Edit Posts 基础知识](/zh/x-api/fundamentals/edit-posts) 页面。

这些端点使用 GET HTTP 方法，并返回一个或多个 [Post 对象](/zh/x-api/fundamentals/data-dictionary)，其中包含诸如 Post 文本、创建时间戳，以及话题标签、提及和照片的列表及其元数据等字段。

<Note>
  ### 账号设置

  要访问这些端点，你需要：

  * 一个已获批准的[开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)。
  * 使用来自某个 [Project](/zh/resources/fundamentals/projects) 中的 [developer App](/zh/resources/fundamentals/developer-apps) 的密钥和令牌进行认证。

  在我们的[入门指南](/zh/x-api/getting-started/getting-access)中了解更多关于如何获取 X API v2 端点访问权限的信息。
</Note>
