X Activity API（XAA）端点组允许开发者获取在 X 平台上发生的 Activity 事件。

开发者可以订阅他们感兴趣的事件，例如 `ProfileBioUpdate`、`ProfilePictureUpdate` 等，并按所需的用户 ID 进行过滤。与该用户 ID 匹配的事件将以亚秒级延迟发送到你的应用。

<Warning>
  X Activity API 目前处于公开测试阶段。在公开测试期间，请预期可能会存在缺陷、破坏性变更或尚未完成的功能。

  你的反馈非常重要——请通过我们的[开发者社区论坛](https://devcommunity.x.com/)报告问题。
</Warning>

<div id="delivery-mechanisms">
  ## 投递机制
</div>

X Activity API 目前支持以下事件投递机制，将事件发送到你的应用：

* 持久 HTTP 流
* [Webhook](/zh/x-api/webhooks/introduction)

<div id="supported-event-types">
  ## 支持的事件类型
</div>

目前，X Activity API 支持以下按类别划分的事件类型：

<div id="profile-events">
  ### 个人资料事件
</div>

当用户更改其个人资料信息时，就会触发相应的个人资料事件。

| Event Name                   | Description                                         | Filters   |
| ---------------------------- | --------------------------------------------------- | --------- |
| `profile.update.bio`           | 当用户更新其个人简介时触发                           | `user_id` |
| `profile.update.profile_picture`       | 当用户更新其头像时触发                               | `user_id` |
| `profile.update.banner_picture`        | 当用户更新其个人资料横幅图片时触发                   | `user_id` |
| `profile.update.geo`           | 当用户更新其个人资料位置时触发                       | `user_id` |
| `profile.update.url`           | 当用户更新其个人资料中的网站 URL 时触发              | `user_id` |
| `profile.update.screenname`    | 当用户更新其展示名称时触发                           | `user_id` |
| `profile.update.verified_badge` | 当用户更新其认证徽章时触发                           | `user_id` |

<div id="follow-events">
  ### 关注事件
</div>

当用户关注或取消关注另一位用户时，会触发关注事件。

| Event Name | Description                           | Filters   |
| ---------- | ------------------------------------- | --------- |
| `follow.follow`   | 当用户关注另一位用户时触发         | `user_id` |
| `follow.unfollow` | 当用户取消关注另一位用户时触发     | `user_id` |

<Note>
  **仅限 Enterprise：** 关注事件目前仅向部分 Enterprise 和合作伙伴层级账户开放。
</Note>

<div id="news-events">
  ### 新闻事件
</div>

新闻事件提供由 Grok 精选的新闻主题和头条更新。

| Event Name | Description                    | Filters     |
| ---------- | ------------------------------ | ----------- |
| `news.new` | Grok 新精选的新闻和头条        | `keyword`   |

<Note>
  **仅限 Enterprise：** 新闻事件目前仅向部分 Enterprise 和 Partner 等级的账户开放。
</Note>

在未来的版本中，XAA 将扩展支持更多事件类型，包括社交互动、内容互动、变现功能等。我们会在有新的事件类型可用时持续更新文档。

<Note>
  **注意：** XAA 不会投递帖子。若需要实时帖子投递，请参阅我们的 [Filtered Stream](/zh/x-api/posts/filtered-stream/introduction) 端点，它允许开发者筛选并实时流式传输帖子。
</Note>

<div id="event-privacy-and-authentication">
  ## 事件隐私和身份验证
</div>

X Activity API 区分**公开事件**和**私有事件**，其行为与 X 应用保持一致，如下所述。

<div id="public-events">
  ### 公共事件
</div>

公共事件是指公开的用户账号以公开方式执行、且对所有 X 用户可见的活动。这些事件对 X 平台上的所有用户可见，并且在查看时不需要用户通过 OAuth 进行身份验证。

**当前的公共事件：**

* 个人资料更新（简介、头像、横幅、位置、URL、用户名变更）

针对这些公共事件，你可以在过滤器中指定用户 id 来创建订阅，并通过 XAA 接收这些事件。

<div id="private-events">
  ### 私有事件
</div>

私有事件是指需要用户通过 OAuth 认证明确授权的活动。用户必须通过 X 完成认证，并向开发者应用显式授予访问这些事件的权限。

**私有事件的认证要求：**

* 用户必须通过 OAuth 2.0 授权你的应用
* 你的应用必须获取相应的 OAuth 范围（scopes）
* 用户必须明确授予你的应用访问这些事件的权限
* 只能为已授权你的应用的用户创建私有事件订阅

<div id="subscription-limits">
  ## 订阅限制
</div>

X Activity API 会根据你的账号套餐等级设定不同的订阅上限：

| 套餐等级     | 最大订阅数             |
| ------------ | --------------------- |
| Free         | 不可用                |
| Basic        | 50                    |
| Pro          | 2500                  |
| Enterprise   | 50,000                |
| Partner      | 100,000               |

<Note>
  **账号设置**

  要访问这些端点，你需要：

  * 一个已获批准的[开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)。
  * 使用位于某个[项目](/zh/resources/fundamentals/projects)中的[开发者应用](/zh/resources/fundamentals/developer-apps)提供的密钥和令牌进行认证。

  请参阅我们的[入门指南](/zh/x-api/getting-started/getting-access)，了解如何获取对 X API v2 端点的访问权限。
</Note>
