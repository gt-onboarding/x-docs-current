将某个账户静音，可以在不取消关注或屏蔽该账户的情况下，把该账户的帖子从你的时间线上移除。被静音的账户不会知道你已将其静音，你可以随时取消静音。借助管理静音相关的端点，开发者可以为 X 上的用户打造更安全的使用体验。一个使用管理静音功能的示例是：某个应用允许你在指定时间内，将可能会就特定话题发布帖子的账户静音。通过 mutes lookup 端点，你可以查看你或某个已认证用户静音了哪些账户。这有助于你了解自己是如何与这些被静音的账户进行互动的。

由于在使用 mute lookup 时你会请求私人信息，并在使用 manage mutes 时代表用户行事，因此必须使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 或 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 对这些端点进行认证，并使用与已授权你应用的用户关联的用户访问令牌。访问令牌可以通过 [三方 OAuth 流程](/zh/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（OAuth 1.0a）或 [Authorization Code with PKCE grant flow](/zh/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)（OAuth 2.0）生成。

<div id="mutes-lookup">
  ### 静音查询
</div>

静音查询端点用于查看当前经过身份验证的用户已静音的账号。此端点的速率限制为每个用户每 15 分钟最多 15 次请求。

<div id="manage-mutes">
  ### 管理静音
</div>

管理静音端点使你能够代表已认证用户对指定账号执行静音或取消静音操作。对于这些端点，有两种可用的 HTTP 方法：POST 和 DELETE。POST 方法允许你将账号设为静音，DELETE 方法允许你取消对账号的静音。这些 POST 和 DELETE 端点对每位用户的速率限制都是每 15 分钟 50 个请求。

**请注意：**如果用户在 [X](https://x.com/) 上执行静音操作，则每 15 分钟有 200 个请求的限制。

**账号设置**

要访问这些端点，你需要：

* 一个已获批准的[开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)。
* 使用位于某个[项目](/zh/resources/fundamentals/projects)中的[开发者应用](/zh/resources/fundamentals/developer-apps)所提供的密钥和令牌进行认证。

在我们的[入门指南](/zh/x-api/getting-started/getting-access)中了解更多关于如何获取 X API v2 端点访问权限的信息。
