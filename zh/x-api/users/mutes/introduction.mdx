将某个账号静音，可以让你在不取消关注或拉黑该账号的情况下，将该账号的帖子从你的时间线上隐藏。被静音的账号不会知道你已将其静音，并且你可以随时取消静音。借助用于管理静音的端点，开发者可以为 X 上的用户创建更安全的使用体验。例如，你可以构建这样一个应用：允许你在指定的一段时间内，将可能发布特定话题帖子的账号静音。通过静音查询端点，你可以查看你自己或经身份验证的用户已静音了哪些账号，这有助于了解你与这些被静音账号之间的互动情况。 

由于在使用静音查询端点时会请求私密信息，并且在使用管理静音端点时会代表用户行事，因此你必须使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 或 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 对这些端点进行认证，并使用与已授权你应用的用户关联的用户访问令牌。你可以通过 [三方 OAuth 流程](/zh/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（OAuth 1.0a）或 [Authorization Code with PKCE 授权流程](/zh/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)（OAuth 2.0）生成这些令牌。

<div id="mutes-lookup">
  ### 静音查询
</div>

静音查询端点允许你查看经过身份验证的用户静音了哪些账号。此端点的速率限制为每位用户每 15 分钟 15 个请求。

<div id="manage-mutes">
  ### 管理静音
</div>

管理静音的端点使你能够代表已通过认证的用户，对指定账号执行静音或取消静音操作。对于这些端点，有两种可用的 HTTP 方法：POST 和 DELETE。POST 方法用于将账号设为静音，DELETE 方法用于取消对账号的静音。对于 POST 和 DELETE 端点，每个用户的速率限制为每 15 分钟最多 50 次请求。

**请注意：**如果用户在 [X](https://x.com/) 上执行静音操作，则限制为每 15 分钟 200 个请求。

**账号设置**

要访问这些端点，你需要：

* 一个已获批准的[开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)。
* 使用位于某个[项目](/zh/resources/fundamentals/projects)中的[开发者应用](/zh/resources/fundamentals/developer-apps)的密钥和令牌进行认证。

在我们的[入门指南](/zh/x-api/getting-started/getting-access)中了解更多有关获取 X API v2 端点访问权限的信息。
