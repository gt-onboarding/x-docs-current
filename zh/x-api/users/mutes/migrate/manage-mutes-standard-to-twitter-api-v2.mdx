<div id="manage-mutes-standard-v11-compared-to-x-api-v2">
  ### 管理静音：标准 v1.1 与 X API v2 的比较
</div>

如果你一直在使用标准 v1.1 中的 [POST mutes/users/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-create) 和 [POST mutes/users/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-destroy) 端点，本指南旨在帮助你了解标准 v1.1 与 X API v2 中管理静音端点之间的相似点和差异。

* **相似点**
  * OAuth 1.0a 用户上下文
* **差异**
  * 端点 URL
  * 应用和项目要求
  * HTTP 方法
  * 请求参数
     

<div id="similarities">
  #### 相似之处
</div>

**OAuth 1.0a 用户上下文认证方法**

这两个版本的端点均支持 [OAuth 1.0a 用户上下文](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication)。因此，如果你之前使用的是某个标准 v1.1 管理静音的端点，在迁移到 X API v2 后，可以继续使用相同的身份验证方法。

<div id="differences">
  #### 差异
</div>

**端点 URL**

* Standard v1.1 端点：
  * POST https://api.x.com/1.1/mutes/users/create.json
    （将某个用户静音）
  * POST https://api.x.com/1.1/mutes/users/destroy.json
    （取消对某个用户的静音）
* X API v2 端点：
  * POST https://api.x.com/2/users/:id/muting
    （将某个用户静音）
  * DELETE https://api.x.com/2/users/:source&#95;user&#95;id/muting/:target&#95;user&#95;id
    （取消对某个用户的静音） 

**应用和项目要求**

在对请求进行身份验证时，X API v2 端点要求你使用来自与某个 [Project](/zh/resources/fundamentals/projects) 关联的 [developer App](/zh/resources/fundamentals/developer-apps) 的凭证。所有 X API v1.1 端点都可以使用独立应用的凭证，或者使用与某个项目关联的应用的凭证。

**请求参数**

以下 Standard v1.1 请求参数在 X API v2 中有对应项：

| Standard v1.1 | X API v2 |
| :--- | :--- |
| user&#95;id | target&#95;user&#95;id |
| screen&#95;name | 无对应参数 |

请注意，Standard v1.1 参数作为查询参数传递，而 X API v2 参数则作为请求体参数（用于 POST 端点）或路径参数（用于 DELETE 端点）传递。

另外，在使用 Standard v1.1 端点时，不需要提供执行对目标用户静音操作的用户的 id，因为通过 [OAuth 1.0a User Context](/zh/resources/fundamentals/authentication) 传递的访问令牌可以推断出是哪个用户发起了静音或取消静音操作。
