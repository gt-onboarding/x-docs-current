<div id="manage-mutes-standard-v11-compared-to-x-api-v2">
  ### 管理静音：标准 v1.1 与 X API v2 的比较
</div>

如果你一直在使用标准 v1.1 的 [POST mutes/users/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-create) 和 [POST mutes/users/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-destroy) 端点，本指南旨在帮助你理解标准 v1.1 与 X API v2 管理静音端点之间的相似性和差异。

* **相似性**
  * OAuth 1.0a 用户上下文
* **差异**
  * 端点 URL
  * 应用和项目要求
  * HTTP 方法
  * 请求参数
     

<div id="similarities">
  #### 相似之处
</div>

**OAuth 1.0a 用户上下文认证方式**

这两个版本的 endpoint 都支持 [OAuth 1.0a 用户上下文](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication)。因此，如果你之前使用的是标准 v1.1 的某个管理静音（manage mutes）endpoint，那么在迁移到 X API v2 版本时，可以继续使用相同的认证方式。

<div id="differences">
  #### 差异
</div>

**端点 URL**

* Standard v1.1 端点：
  * POST https://api.x.com/1.1/mutes/users/create.json
    （静音某个用户）
  * POST https://api.x.com/1.1/mutes/users/destroy.json
    （取消静音某个用户）
* X API v2 端点：
  * POST https://api.x.com/2/users/:id/muting
    （静音某个用户）
  * DELETE https://api.x.com/2/users/:source&#95;user&#95;id/muting/:target&#95;user&#95;id
    （取消静音某个用户） 

**应用和项目要求**

在对请求进行认证时，X API v2 端点要求你使用来自某个与[项目](/zh/resources/fundamentals/projects)关联的[开发者应用](/zh/resources/fundamentals/developer-apps)的凭证。所有 X API v1.1 端点则可以使用来自独立应用或与项目关联的应用的凭证。

**请求参数**

以下 Standard v1.1 请求参数在 X API v2 中有对应项：

| Standard v1.1 | X API v2 |
| :--- | :--- |
| user&#95;id | target&#95;user&#95;id |
| screen&#95;name | 无对应项 |

请注意，Standard v1.1 参数通过查询参数传递，而 X API v2 参数则通过请求体参数（针对 POST 端点）或路径参数（针对 DELETE 端点）传递。

另外，在使用 Standard v1.1 端点时，不需要提供对目标用户执行静音操作的用户的 id，因为通过 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 传递的访问令牌可以推断出是哪个用户在发起静音或取消静音。
