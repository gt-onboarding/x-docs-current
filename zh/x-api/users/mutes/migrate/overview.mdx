  ## 比较 X API 的静音端点
</div>

<div id="mutes-lookup">
  #### 静音查询
</div>

v2 静音查询端点将取代标准的 [v1.1 GET mutes/users/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/get-mutes-users-ids) 和 [GET mutes/users/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/get-mutes-users-list) 端点。

下表比较了标准 v1.1 和 X API v2 静音端点：
 

| 描述 | 标准 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 支持的 HTTP 方法 | ******GET****** | ******GET****** |
| 主机域名 | ******https://api.x.com****** | ******https://api.x.com****** |
| 端点路径 | ******/1.1/mutes/users/ids.json******<br /><br />/1.1/mutes/users/list.json | ******/2/users/:id/muting****** |
| [身份验证](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文 | OAuth 1.0a 用户上下文<br /><br />OAuth 2.0 Authorization Code with PKCE |
| 默认请求 [速率限制](/zh/resources/fundamentals/rate-limits) | 每 15 分钟 15 次请求（每个用户） | 每 15 分钟 15 次请求（每个用户） |
| 数据格式 | 标准 v1.1 格式 | [X API v2 格式](/zh/x-api/fundamentals/data-dictionary)（由 fields 和 expansions 请求参数确定，与 v1.1 格式不向后兼容）<br /><br />要了解如何从标准 v1.1 格式迁移到 X API v2 格式，请参阅我们的[数据格式迁移指南](/zh/x-api/migrate/data-format-migration)。 |
| 是否需要使用与某个 [Project](/zh/resources/fundamentals/projects) 关联的 [developer App](/zh/resources/fundamentals/developer-apps) 的凭证 |     | ✔️  |

<div id="manage-mutes">
  #### 管理静音
</div>

v2 中的管理静音端点将取代标准 v1.1 的 [POST mutes/users/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-create) 和 [POST mutes/users/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-destroy) 端点。

下表比较了标准 v1.1 和 X API v2 的静音端点：

**将用户静音**

| 描述 | 标准 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 支持的 HTTP 方法 | POST | POST |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /1.1/mutes/users/create.json | /2/users/:id/muting |
| [Authentication](/zh/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code with PKCE |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 每 15 分钟 50 次请求 | 每 15 分钟 50 次请求 |
| 需要使用与某个[项目](/zh/resources/fundamentals/projects)关联的[开发者应用](/zh/resources/fundamentals/developer-apps)的凭证 |     | ✔️  |

**取消对用户的静音**

下表比较了标准 v1.1 和 X API v2 的取消静音端点：

| 描述 | 标准 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 支持的 HTTP 方法 | POST | DELETE |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /1.1/mutes/users/destroy.json | /2/users/:source&#95;user&#95;id/muting/:target&#95;user&#95;id |
| [Authentication](/zh/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code with PKCE |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 每 15 分钟 50 次请求 | 每 15 分钟 50 次请求 |
| 需要使用与某个[项目](/zh/resources/fundamentals/projects)关联的[开发者应用](/zh/resources/fundamentals/developer-apps)的凭证 |     | ✔️  |
