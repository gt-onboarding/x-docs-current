<div id="authentication">
  #### 身份验证
</div>

所有 X API v2 端点都要求你使用一组凭证（也称为密钥和令牌）来对请求进行身份验证。你可以使用 OAuth 1.0a 用户上下文，或 OAuth 2.0 Authorization Code with PKCE 来为这些端点的请求进行身份验证。 

[OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 要求你使用 API Keys、用户访问令牌，以及少量其他参数来[创建授权头](/zh/resources/fundamentals/authentication#authorizing-a-request)，然后随请求一起发送。访问令牌必须与你代表其发起请求的用户关联。如果你想为另一位用户生成一组访问令牌，该用户必须通过[三方 OAuth 流程](https://developer-staging.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)授权你的 App。 

请注意，OAuth 1.0a 使用起来可能比较困难。如果你不熟悉这种身份验证方法，我们建议你使用一个[库](/zh/x-api/tools-and-libraries/overview)、使用类似 Postman 的工具，或者使用 OAuth 2.0 来为你的请求进行身份验证。

[OAuth 2.0 Authorization Code with PKCE](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 允许你对应用的权限范围（scope）和跨多设备的授权流程进行更精细的控制。OAuth 2.0 允许你选择特定的细粒度权限范围（scope），从而代表用户获取特定权限。 

要在你的 App 中启用 OAuth 2.0，你必须在开发者门户中该 App 的 App settings 部分里的 Authentication settings 中启用它。

<div id="developer-portal-projects-and-developer-apps">
  #### 开发者门户、项目和开发者应用
</div>

要获取可用于 X API v2 端点的一组身份验证凭证，你必须先[注册开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)，在该账号下创建一个[项目](/zh/resources/fundamentals/projects)，并在该项目内创建一个[开发者应用](/zh/resources/fundamentals/developer-apps)。然后，你可以在你的开发者应用中找到你的密钥和令牌。
 

<div id="rate-limits">
  #### 速率限制
</div>

每天都有数以千计的开发者向 X API 发起请求。为了管理大量的请求，我们在每个端点上都设置了[速率限制](/zh/x-api/fundamentals/rate-limits)，用以限制你代表自己的应用或代表已通过身份验证的用户所能发起的请求次数。 

这些端点在用户级别上受到速率限制，这意味着你代表其发起请求的已通过身份验证的用户，在任意开发者应用中调用该端点的次数都是有限的。

对于同时支持 POST 和 DELETE 方法的每个端点，用户速率限制为每 15 分钟 50 个请求。而对于 GET 方法，速率限制仅为每 15 分钟 15 个请求。
 

<div id="fields-and-expansions">
  #### 字段和扩展字段
</div>

X API v2 的 GET 端点允许用户通过一组称为字段和扩展字段的工具，精确选择希望从 API 返回的具体数据。`expansions` 参数允许你展开在负载中被引用的对象。比如，该端点允许你获取以下[扩展字段](/zh/x-api/fundamentals/expansions)：

* pinned&#95;tweet&#95;id

**fields** 参数允许你在不同的数据对象中精确选择你希望接收的[字段](/zh/x-api/fundamentals/fields)。该端点主要返回 User 对象。默认情况下，User 对象返回 **id**、**name** 和 **username** 字段。要接收额外字段，比如 **user.created&#95;at** 或 **user.entities**，你必须通过 **fields** 参数显式请求这些字段。 

我们在 [X API v2 数据字典](/zh/x-api/fundamentals/data-dictionary)中新增了一篇关于[结合使用字段和扩展字段](/zh/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)的指南。

<div id="pagination">
  #### 分页
</div>

静音查询可能会返回大量数据。为了确保在任意时间点都能返回一致且高性能的结果，我们使用分页。分页是 X API v2 端点中的一项功能，当返回结果多于单次响应所能包含的数量时，用于返回更多结果。此时，数据会以一系列“页面”的形式返回。了解如何[对结果进行分页](/zh/x-api/fundamentals/pagination)。

<Note>
  **请注意：** 如果用户在 [X](https://x.com/) 上执行静音操作，每 15 分钟最多只能发送 200 个请求。
</Note>
