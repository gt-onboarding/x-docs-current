  ## 比较 X API 的关注关系端点
</div>

<div id="follows-lookup">
  ### 关注关系查找
</div>

v2 的关注关系查找端点将取代标准版 v1.1 的 [followers/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-followers-ids)、v1.1 [followers/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-followers-list)、v1.1 [friends/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-friends-ids) 和 v1.1 [friends/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-friends-list) 端点。

下表对比了多种类型的关注关系查找端点：

|     |     |     |
| :--- | :--- | :--- |
| **说明** | **Standard v1.1** | **X API v2** |
| 支持的 HTTP 方法 | `GET` | `GET` |
| 主机域名 | `https://api.x.com` | `https://api.x.com` |
| 端点路径 | /1.1/friends/ids.json<br /><br />/1.1/friends/list.json<br /><br />/1.1/followers/ids.json<br /><br />/1.1/followers/list.json | /2/users/:id/following<br /><br />/2/users/:id/followers |
| [身份验证](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文<br /><br />仅应用（App only） | OAuth 1.0a 用户上下文<br /><br />OAuth 2.0 Authorization Code with PKCE<br /><br />仅应用（App only） |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 每 15 分钟 15 次请求（每个用户）<br /><br />每 15 分钟 15 次请求（每个应用） | 每 15 分钟 15 次请求（每个用户）<br /><br />每 15 分钟 15 次请求（每个应用） |
| 每个响应的最大用户数量 | GET friends/id 和 GET followers/id 每页最多返回 5000 个用户 ID。<br /><br />  <br /><br />GET friends/list 和 GET followers/list 每页最多返回 200 个用户对象。 | 每页最多 1000 个用户对象 |
| 分页 | 令牌会在 next&#95;cursor 字段中返回，然后可以将其作为 cursor 参数的值传入，以返回下一页结果。 | 令牌会在 next&#95;token 字段中返回，然后可以将其作为 token 参数的值传入，以返回下一页结果。<br /><br />v2 响应负载还会提供 previous&#95;token 字段，同样可以与 pagination&#95;token 参数一起传入，以返回上一页结果。 |
| JSON 格式 | 标准版 v1.1 格式 | [X API v2 格式](/zh/x-api/fundamentals/data-dictionary)（由 fields 和 expansions 请求参数决定，与 v1.1 格式不向后兼容）<br /><br />要详细了解如何从标准版 v1.1 格式迁移到 X API v2 格式，请参阅我们的[数据格式迁移指南](/zh/x-api/migrate/data-format-migration)。 |
| 支持选择在负载中返回哪些[字段](/zh/x-api/fundamentals/data-dictionary) |     | ✔   |
| 支持帖子 [annotations](/zh/x-api/fundamentals/post-annotations) 字段 |     | ✔   |
| 支持请求新的[指标](/zh/x-api/fundamentals/metrics) 字段 |     | ✔   |
| 支持 [conversation&#95;id](/zh/x-api/fundamentals/conversation-id) 字段 |     | ✔   |
| 需要使用与某个[项目](/zh/resources/fundamentals/projects) 关联的[开发者应用](/zh/resources/fundamentals/developer-apps)的凭证 |     | ✔   |

<div id="manage-follows">
  #### 管理关注
</div>

用于管理关注的 v2 端点将取代标准 v1.1 的 [POST friendships/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-create) 和 [POST friendships/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-destroy) 端点。

下表比较了标准 v1.1 和 X API v2 中用于创建关注的端点：

<div id="follow-a-user">
  #### 关注用户
</div>

| 描述 | 标准 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 支持的 HTTP 方法 | POST | POST |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /1.1/friendships/create.json | /2/users/:id/following |
| [身份验证](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文 | OAuth 1.0a 用户上下文<br /><br />OAuth 2.0 授权码模式（PKCE） |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 每 15 分钟 50 次请求 | 每 15 分钟 50 次请求 |
| 每个用户每天的最大操作次数 | 400 | 400 |
| 每个应用每天的最大操作次数 | 1000 | 1000 |
| 需要使用与某个[项目](/zh/resources/fundamentals/projects)关联的[开发者应用](/zh/resources/fundamentals/developer-apps)的凭证 |     | ✔️  |

<div id="unfollow-a-user">
  #### 取消关注用户
</div>

下表对比了标准版 v1.1 和 X API v2 中用于取消关注的端点：

| 描述 | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| 支持的 HTTP 方法 | POST | DELETE |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /1.1/friendships/destroy.json | /2/users/:source&#95;user&#95;id/following/:target&#95;user&#95;id |
| [Authentication](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文 | OAuth 1.0a 用户上下文<br /><br />OAuth 2.0 授权码模式（配合 PKCE） |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 每位用户每 15 分钟 15 个请求 | 每位用户每 15 分钟 50 个请求 |
| 每个应用的每日最大操作数 | 无 | 500 |
| 是否必须使用与某个[项目](/zh/resources/fundamentals/projects)关联的[开发者应用](/zh/resources/fundamentals/developer-apps)的凭据 |     | ✔️  |
