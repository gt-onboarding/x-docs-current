<div id="manage-follows-standard-v11-compared-to-x-api-v2">
  #### 管理关注：标准 v1.1 与 X API v2 对比
</div>

如果您一直在使用标准 v1.1 的 [POST friendships/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-create) 和 [POST friendships/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-destroy) 端点，本指南旨在帮助您了解标准版与 X API v2 关注管理端点之间的相似点和差异。

* **相似点**
  * OAuth 1.0a 用户上下文
* **差异**
  * 端点 URL
  * 应用和项目要求
  * HTTP 方法
  * 请求参数

     

<div id="similarities">
  #### 相似点
</div>

**OAuth 1.0a 用户上下文身份验证方法**

这两个版本的端点都支持 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication#oauth-1-0a-2)。因此，如果你之前使用的是某个标准的 v1.1 管理关注端点，在迁移到 X API v2 版本时，可以继续使用相同的身份验证方法。 

<div id="differences">
  #### 差异
</div>

**端点 URL**

* 标准 v1.1 端点：
  * POST https://api.x.com/1.1/friendships/create.json
    （关注用户）
  * POST https://api.x.com/1.1/friendships/destroy.json
    （取消关注用户）
* X API v2 端点：
  * POST https://api.x.com/2/users/:id/following
    （关注用户）
  * DELETE https://api.x.com/2/users/:source&#95;user&#95;id/following/:target&#95;user&#95;id
    （取消关注用户） 

**应用和项目要求**

在对请求进行认证时，X API v2 端点要求你使用某个与 [项目](/zh/resources/fundamentals/projects) 关联的 [开发者应用](/zh/resources/fundamentals/developer-apps) 的凭证。所有 X API v1.1 端点都可以使用来自独立应用或与项目关联应用的凭证。

**请求参数**

以下标准 v1.1 请求参数在 X API v2 中有对应项：

| 标准 v1.1 | X API v2 |
| :--- | :--- |
| 无对应项 | id (POST), source&#95;user&#95;id (DELETE) |
| user&#95;id | target&#95;user&#95;id |
| screen&#95;name | 无对应项 |

请注意，标准 v1.1 参数通过查询参数传递，而 X API v2 参数则通过请求体参数（用于 POST 端点）或路径参数（用于 DELETE 端点）传递。

此外，在使用标准 v1.1 端点时，不需要 v2 的 id 和 source&#95;user&#95;id 参数，因为通过 OAuth 1.0a 用户上下文传递的访问令牌可以用来推断是哪位用户发起了关注/取消关注操作。
