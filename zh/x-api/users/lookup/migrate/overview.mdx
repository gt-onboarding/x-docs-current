  ## 比较 X API 的用户查找端点
</div>

v2 用户查找端点将取代标准 v1.1 的 [GET users/lookup](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-users-lookup.html) 和 [GET users/show](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-users-show.html) 端点。如果你有使用这些用户查找端点版本的代码、应用或工具，并且正在考虑迁移到更新的 X API v2 端点，那么这一系列指南就是为你准备的。 
 

下表比较了各种类型的用户查找端点： 

|     |     |     |
| :--- | :--- | :--- |
| **说明** | **标准 v1.1** | **X API v2** |
| 支持的 HTTP 方法 | `GET` | `GET` |
| 主机域名 | `https://api.x.com` | `https://api.x.com` |
| 端点路径 | `/1.1/users/show.json` `/1.1/users/lookup.json` | `/2/users`<br /><br />`/2/users/:id`<br /><br />`/2/users/by`<br /><br />`/2/users/by/:username` |
| [身份验证](/zh/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />仅应用（App only）<br /><br />OAuth 2.0 Authorization Code with PKCE |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 每 15 分钟 900 次请求（每用户）<br /><br />/show - 每 15 分钟 900 次请求（每应用）  <br />/lookup - 每 15 分钟 300 次请求（每应用） | 每 15 分钟 900 次请求（每用户）<br /><br />每 15 分钟 300 次请求（每应用） |
| 每个响应中的最大用户数 | /show - 1<br /><br />/lookup - 100 | 100 |
| JSON 响应对象格式 | 标准 v1.1 格式 | [X API v2 格式](/zh/x-api/fundamentals/data-dictionary)（由 fields 和 expansions 请求参数决定，与 v1.1 格式不向后兼容）<br /><br />要了解如何从标准 v1.1 格式迁移到 X API v2 格式，请访问我们的[数据格式迁移指南](/zh/x-api/migrate/data-format-migration)。 |
| 是否支持选择要在有效负载中返回的哪些[字段](/zh/x-api/fundamentals/data-dictionary) |     | ✔   |
| 是否支持 [annotations](/zh/x-api/fundamentals/post-annotations) 字段（用于置顶帖子） |     | ✔   |
| 是否支持请求新的[指标](/zh/x-api/fundamentals/metrics) 字段（用于置顶帖子） |     | ✔   |
| 是否支持 [conversation&#95;id](/zh/x-api/fundamentals/conversation-id) 字段（用于置顶帖子） |     | ✔   |
| 是否要求使用与某个[项目](/zh/resources/fundamentals/projects)关联的[开发者应用](/zh/resources/fundamentals/developer-apps)的凭证 |     | ✔   |
