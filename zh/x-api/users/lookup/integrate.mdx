<div id="authentication">
  #### 身份验证
</div>

所有 X API v2 端点都要求使用一组凭证（也称为密钥和令牌）对请求进行[身份验证](/zh/resources/fundamentals/authentication)。你可以使用 OAuth 1.0a 用户上下文、App only，或 OAuth 2.0 Authorization Code with PKCE 来对这些端点的请求进行身份验证。 

[OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication#oauth-1-0a-2)要求你使用 API Key、用户访问令牌（Access Token）以及少量其他参数来[创建授权头](/zh/resources/fundamentals/authentication#authorizing-a-request)，然后将该授权头随请求一起发送。访问令牌必须与你代表其发起请求的用户关联。如果你想为另一位用户生成一组访问令牌，该用户必须使用[三方 OAuth 流程](https://developer-staging.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)授权你的 App。 

请注意，OAuth 1.0a 使用起来可能比较复杂。如果你不熟悉这一身份验证方法，我们建议你使用[库](/zh/x-api/tools-and-libraries/overview)、Postman 之类的工具，或者使用 OAuth 2.0 来对请求进行身份验证。如果你想从这些端点请求某条帖子或私有指标数据，你需要使用 OAuth 1.0a 用户上下文或 OAuth 2.0 Authorization Code with PKCE，以确保你拥有内容所属用户授予的适当权限。
 

[App only](/zh/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) 只要求你在请求中携带一个 [App only Access Token](/zh/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)。你可以直接在开发者应用中生成 App only Access Token，或者使用 [POST oauth2/token](/zh/resources/fundamentals/authentication#post-oauth2-token) 端点生成。 

[OAuth 2.0 Authorization Code with PKCE](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 允许对应用的 scope（权限范围）以及跨多个设备的授权流程进行更精细的控制。OAuth 2.0 允许你选择特定的细粒度 scope，从而代表用户获取相应的具体权限。 

要在你的 App 中启用 OAuth 2.0，必须在开发者门户的 App 设置部分中，在该 App 的身份验证设置里将其启用。

**请注意**

如果你要请求以下字段，则必须使用 OAuth 1.0a 用户上下文或 OAuth 2.0 Authorization Code： 

* tweet.fields.non&#95;public&#95;metrics
* tweet.fields.promoted&#95;metrics
* tweet.fields.organic&#95;metrics

<div id="developer-portal-projects-and-developer-apps">
  #### 开发者门户、项目和开发者应用
</div>

要获取可用于 X API v2 端点的一组身份验证凭证，你必须先[注册开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)，在该账号下创建一个[项目](/zh/resources/fundamentals/projects)，并在该项目中创建一个[开发者应用](/zh/resources/fundamentals/developer-apps)。然后，你可以在该开发者应用中找到你的密钥和令牌。 
 

<div id="rate-limits">
  #### 速率限制
</div>

每天都有成千上万的开发者向 X API 发起请求。为了帮助管理如此庞大的请求量，我们在每个端点上设置了[速率限制](/zh/x-api/fundamentals/rate-limits)，用来限制你代表你的应用或代表已认证用户所能发起的请求数量。 

用户查找端点在 App 级和用户级都会受到速率限制。不过，已认证用户查找端点仅在用户级受到速率限制。

App 级速率限制意味着你作为开发者，在给定时间段内，从任一特定 App（由你正在使用的密钥和令牌所标识）向该端点只能发起一定数量的请求。用户级速率限制意味着，你代表其发起请求的已认证用户，在所有开发者应用中只允许执行一定次数的调用。

下表展示了各个端点对应的速率限制。

|     |     |     |
| :--- | :--- | :--- |
| **端点** | **HTTP 方法** | **速率限制 / 级别** |
| /2/users | GET | 每 15 分钟 900 次请求 / App 和用户 |
| /2/users/:id | GET | 每 15 分钟 900 次请求 / App 和用户 |
| /2/users/by | GET | 每 15 分钟 900 次请求 / App 和用户 |
| /2/users/by/username/:username | GET | 每 15 分钟 900 次请求 / App 和用户 |
| /2/users/me | GET | 每 15 分钟 75 次请求 / 用户 |

<div id="fields-and-expansions">
  #### 字段和扩展字段
</div>

X API v2 允许用户通过一组称为字段和扩展字段的工具，精确选择希望从 API 返回的具体数据。`expansions` 参数允许你展开响应载荷中被引用的对象。例如，此端点允许你使用 `pinned_tweet_id` 扩展字段。

`fields` 参数允许你选择希望在不同数据对象中接收的精确[字段](/zh/x-api/fundamentals/fields)。这些端点主要返回用户对象。默认情况下，用户对象会返回 `id`、`name` 和 `username` 字段。要接收其他字段（例如 `user.created_at` 或 `user.location`），你必须通过 `fields` 参数显式请求这些字段。你在集成中可能希望考虑使用的一些重要字段包括帖子投票数据、指标、注释以及会话 ID 字段。

我们在 [X API v2 数据字典](/zh/x-api/fundamentals/data-dictionary)中添加了一篇关于[如何使用字段和扩展字段](/zh/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)的指南。
