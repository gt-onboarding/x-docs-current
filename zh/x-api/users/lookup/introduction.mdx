除了已认证用户查询端点之外，你可以使用 [OAuth 1.0a User Context](/zh/resources/fundamentals/authentication)、[仅应用（App only）](/zh/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) 或 [OAuth 2.0 Authorization code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 来对所有用户查询端点的请求进行认证。不过，如果你想访问私密用户的私有指标或数据，你需要使用 OAuth 1.0a User Context 或 OAuth 2.0 Authorization Code with PKCE，并在请求中传递已认证用户的访问令牌（Access Token）。    

此端点通常用于获取用户的最新详细信息、验证用户是否存在，或者在合规事件发生后更新你存储的用户详细信息。

<div id="authenticated-user-lookup">
  ### 已认证用户查找
</div>

如果你要代表其他用户进行认证，那么明确你可以代表哪些用户发起请求至关重要。

此端点要求你使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 或 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)。它返回已授权用户的信息，即与你在请求中传递的用户访问令牌关联的用户。

你可以通过 GET 方法调用此端点。此端点的按用户速率限制为每 15 分钟 75 次请求。

<Note>
  **账号设置**

  要访问这些端点，你需要：

  * 一个已获批准的 [开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)。
  * 使用位于某个[项目](/zh/resources/fundamentals/projects)中的[开发者应用](/zh/resources/fundamentals/developer-apps)所提供的密钥和令牌进行认证。

  在我们的[入门指南](/zh/x-api/getting-started/getting-access)中了解更多关于获取 X API v2 端点访问权限的信息。
</Note>
