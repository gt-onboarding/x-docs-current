<div id="getting-started-with-the-users-lookup-endpoints">
  ## 开始使用用户查找端点
</div>

本快速入门指南将帮助你使用一组指定字段，通过 [Postman](/zh/tutorials/postman-getting-started) 向用户查找端点发起你的第一个请求。

如果你想查看不同语言的示例代码，请访问我们的 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 仓库。 

<Note>
  ### 前提条件

  要完成本指南，你需要准备一组用于对请求进行认证的 [密钥和令牌](/zh/resources/fundamentals/authentication)。你可以按照以下步骤生成这些密钥和令牌：

  * [注册开发者账号](https://developer.x.com/en/apply-for-access) 并获得批准。
  * 在开发者门户中创建一个 [项目](/zh/resources/fundamentals/projects) 以及关联的 [开发者应用](/zh/resources/fundamentals/developer-apps)。
  * 前往你的 App 的“Keys and tokens”页面生成所需的凭证。请确保将所有凭证保存在安全的位置。
</Note>

<div id="steps-to-build-a-users-lookup-request">
  ### 构建用户查找请求的步骤
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### 第一步：先选择一个工具或库
</div>

有多种不同的工具、代码示例和库可用于向此端点发起请求，但为了简化流程，我们将在这里使用 Postman 工具。

要将 X API v2 的 Postman 集合导入到你的环境中，请点击以下按钮：

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  将 X API v2 添加到 Postman
</Button>

在你将 X API v2 集合加载到 Postman 之后，转到 GET /users/by 端点。
 

<div id="step-two-authenticate-your-request">
  #### 第二步：对请求进行身份验证
</div>

要正确向 X API 发起请求，你需要验证自己是否拥有访问权限。为此，此端点要求你使用 [App only](/zh/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)、[OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 或 [OAuth 1.0a User Context](/zh/resources/fundamentals/authentication) 等身份验证方式对请求进行认证。

为简单起见，我们在本示例中将使用 App only，但如果你希望请求私有的 [指标](/zh/x-api/fundamentals/metrics) 或用户数据，则需要使用其他身份验证方式之一。 

要使用 App only，你必须在 Postman 中添加你的密钥和令牌，尤其是 [App Access Token](/zh/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)（也称为 App only Bearer Token）。你可以在 Postman 右上角选择名为 “X API v2” 的环境，然后将你的密钥和令牌添加到该环境的 &quot;initial value&quot; 和 &quot;current value&quot; 字段中（通过单击环境下拉菜单旁的眼睛图标完成）。

如果你已正确完成这些操作，这些变量将会自动填充到该请求的 Authorization 选项卡中。
 

<div id="step-three-identify-and-specify-which-users-you-would-like-to-retrieve">
  #### 步骤三：确定并指定你希望检索的用户
</div>

你必须在请求中指定一个或一组要检索的用户。根据你使用的用户端点不同，你可以传递 user ID 或用户名。在本示例中，我们将使用 [GET /users/by 端点](/zh/x-api/users/user-lookup-by-usernames)，它允许你在单个请求中传递多个用户名（而不是使用 [单个 ID](/zh/x-api/users/user-lookup-by-id)、[多个 ID](/zh/x-api/users/user-lookup-by-ids) 和 [单个用户名](/zh/x-api/users/user-lookup-by-username) 端点），并通过 `usernames` 查询参数传递一组用户名。

用户名就是你可以在账号个人资料 URL 中找到的账号 handle。例如，下面这个账号的用户名是 XDevelopers。

`https://x.com/XDevelopers`

在 Postman 中，导航到 &quot;Params&quot; 选项卡，在 `username` 参数的 &quot;Value&quot; 列中输入该用户名，或输入用逗号分隔的一串用户名，并确保在用户名和逗号之间不要包含任何空格。 

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `username` | XDeveloeprs,API,adsapi |

<div id="step-four-identify-and-specify-which-fields-you-would-like-to-retrieve">
  #### 步骤四：确定并指定你想要检索的字段
</div>

如果你在步骤三之后点击 “Send” 按钮，你将在响应中收到默认的 [user 对象](/zh/x-api/fundamentals/data-dictionary#user) 字段：id、name 和 username。

如果你想在 id、name 和 username 之外接收更多字段，你需要在请求中使用 [field](/zh/x-api/fundamentals/fields) 和/或 [expansion](/zh/x-api/fundamentals/expansions) 参数来指定这些字段。

在本练习中，我们将从不同对象中请求三组额外的字段：

1. 主 user 对象中的额外 user.created&#95;at 字段。
2. 返回用户所关联的置顶帖子对象的默认字段：id 和 text。
3. 关联帖子（Post）对象中的额外 tweet.created&#95;at 字段。
    

在 Postman 中，导航到 “Params” 选项卡，并在 “Query Params” 表格中添加以下 key:value 键值对：

|                |              |                             |
| :------------- | :----------- | :-------------------------- |
| **Key**        | **Value**    | **Returned fields**         |
| `user.fields`  | `created_at` | `user.created_at`           |
| `expansions`   | `author_id`  | tweet.id, tweet.text        |
| `tweet.fields` | `created_at` | `includes.users.created_at` |

现在你应该会在 “Send” 按钮旁边看到如下 URL：

```
      https://api.x.com/2/users/by?usernames=twitterdev,twitterapi,adsapi&user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=author_id,created_at
```
