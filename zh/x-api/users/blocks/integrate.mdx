<div id="authentication">
  #### 身份验证
</div>

所有 X API v2 端点都要求你使用一组凭证（也称为密钥和令牌）对请求进行身份验证。你可以使用 OAuth 1.0a 用户上下文或 OAuth 2.0 Authorization Code with PKCE 来对发往这些端点的请求进行身份验证。 

[OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 要求你使用你的 API 密钥、用户访问令牌，以及其他一些参数来[创建授权请求头](/zh/resources/fundamentals/authentication#authorizing-a-request)，然后将其随请求一起发送。访问令牌必须与由你代为发起请求的用户相关联。如果你想为另一位用户生成一组访问令牌，他们必须使用 [三方 OAuth 流程](https://developer-staging.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens) 授权你的 App。 

请注意，OAuth 1.0a 相对较难使用。如果你对这种身份验证方法不熟悉，我们建议你使用某个[库](/zh/x-api/tools-and-libraries/overview)、使用 Postman 这类工具，或者使用 OAuth 2.0 来对请求进行身份验证。

[OAuth 2.0 Authorization Code with PKCE](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) 允许你对应用的权限范围（scope）以及跨多设备的授权流程进行更精细的控制。OAuth 2.0 允许你选择特定的细粒度权限范围（scope），以在代表用户时获得特定的权限。 

要在你的 App 中启用 OAuth 2.0，你必须在开发者门户中进入该 App 的设置，在其中的身份验证设置里将其启用。

<div id="developer-portal-projects-and-developer-apps">
  #### 开发者门户、项目和开发者应用
</div>

要获取可用于 X API v2 端点的一套身份验证凭据，你必须先[注册开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)，在该账号下设置一个[项目](/zh/resources/fundamentals/projects)，并在该项目中创建一个[开发者应用](/zh/resources/fundamentals/developer-apps)。然后你可以在该开发者应用中找到你的密钥和令牌。
 

<div id="rate-limits">
  #### 速率限制
</div>

每天都有成千上万的开发者向 X API 发起请求。为了帮助管理如此海量的请求，我们在每个端点上都设置了[速率限制](/zh/x-api/fundamentals/rate-limits)，用来限制你代表你的应用或某个已认证用户所能发起的请求次数。 

这些端点采用用户级速率限制，这意味着你代表其发起请求的已认证用户，在任意开发者应用中对该端点的调用次数都是有限的。对每个端点，使用 POST 和 DELETE 方法时的用户速率限制为每 15 分钟 50 个请求。然而，当使用 GET 方法时，速率限制仅为每 15 分钟 15 个请求。
 

<div id="fields-and-expansions">
  #### 字段和扩展字段
</div>

X API v2 的 GET 端点允许你使用称为字段（fields）和扩展字段（expansions）的一组工具，精确选择希望从 API 返回的哪些数据。`expansions` 参数允许你展开在响应负载中被引用的对象。例如，此端点允许你获取以下[扩展字段](/zh/x-api/fundamentals/expansions)：

* pinned&#95;tweet&#95;id

**fields** 参数允许你精确选择希望接收的不同数据对象中的哪些[字段](/zh/x-api/fundamentals/fields)。这些端点主要返回 Post 对象。默认情况下，Post 对象只返回 **id** 和 **text** 字段。要接收其他字段，例如 **tweet.created&#95;at** 或 **tweet.entities**，你必须通过 **fields** 参数显式请求这些字段。你在集成中可能希望考虑使用的一些重要字段包括投票数据、指标、Post 标注以及会话 ID 字段。

我们在 [X API v2 数据字典](/zh/x-api/fundamentals/data-dictionary)中新增了一篇关于如何[同时使用字段和扩展字段](/zh/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)的指南。
