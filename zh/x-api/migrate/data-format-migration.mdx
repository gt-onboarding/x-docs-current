<div id="introduction">
  ## 介绍
</div>

随着 X API v2 的推出，我们采用了一种新的数据响应格式以及请求不同对象和字段的新方式，我们将其简称为 X API v2 格式。 

在“常规差异”部分，你可以了解一些与标准用户和 Enterprise 用户相关的变更。另外，我们还专门整理了针对 [standard v1.1 Native 格式](https://developer.x.com/en/docs/x-api/v1/data-dictionary/overview)、Enterprise [Native Enriched 格式](/zh/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#native-enriched-tweet-object)以及 Enterprise [Activity Streams 格式](/zh/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#activity-object)的指南，帮助你进行字段映射，并说明在请求新的 v2 字段时必须使用哪些字段和扩展字段。 

* [从 Native 格式到 X API v2（standard v1.1）](/zh/x-api/migrate/data-format-migration#migrating-from-standard-v1-1s-data-format-to-v2) 
* [从 Native Enriched 到 X API v2（Enterprise）](/zh/x-api/migrate/data-format-migration#migrating-from-native-enriched-data-format-to-v2)
* [从 Activity Streams 到 X API v2（Enterprise）](/zh/x-api/migrate/data-format-migration#migrating-from-activity-streams-data-format-to-v2)

你也可能会对我们的[可视化数据格式迁移工具](/zh/x-api/migrate/data-format-migration#visual-data-format-migration-tool)感兴趣，它可以帮助你快速了解 [X API v1.1 数据格式](https://developer.x.com/en/docs/x-api/v1/data-dictionary/overview)与 [X API v2 格式](/zh/x-api/introduction)之间的差异。

<div id="general-differences">
  ### 总体差异
</div>

<div id="requesting-objects-and-fields">
  #### 请求对象和字段
</div>

pre-v2 端点与 v2 端点之间的一个重大变化是：新版本默认只返回少量字段，而 standard、premium 和 Enterprise 端点默认会返回大部分字段。新版本通过名为 [fields](/zh/x-api/fundamentals/fields) 和 [expansions](/zh/x-api/fundamentals/expansions) 的参数来显式请求超出默认值的额外数据，这意味着你可以只请求所需的数据，而不必接收和处理对你无关紧要的字段。 

你请求的、与主数据对象相关的任何字段，都会连同默认值一起返回在该主数据对象中。不过，如果你使用 expansions 参数请求任何展开的对象，这些次级对象会返回在一个名为 includes 的新对象中。你可以使用同时在两者中返回的 id 字段，将 includes 对象中的展开对象与主对象进行关联。

例如，如果你在使用 v2 的 [Post lookup](/zh/x-api/posts/lookup/introduction) 端点，并在请求中包含 expansions=author&#95;id 参数，你会在主 Post 对象中收到 author&#95;id 字段，同时在 includes 对象中针对每条 Post 收到一个 user 对象。每个 user 对象都会包含默认的 id 字段，可用于将该 user 对象与对应的 Post 对象进行匹配。下面是一个示例展示其结构：

```
{
  "data": [
    {
      "author_id": "2244994945",
      "id": "1397568983931392004",
      "text": "The Twitter Developer Platform. Ooh la la! https://t.co/iGTdPXBfOv https://t.co/Ze8z8EODdg"
    }
  ],
  "includes": {
    "users": [
      {
        "id": "2244994945",
        "name": "Twitter Dev",
        "username": "TwitterDev"
      }
    ]
  }
}
```

<div id="updated-json-design">
  #### 更新后的 JSON 设计
</div>

除了请求某些字段的方式发生变化之外，X API v2 还为 API 返回的对象引入了新的 JSON 设计，包括 [Post](/zh/x-api/fundamentals/data-dictionary#tweet) 和 [user](/zh/x-api/fundamentals/data-dictionary#user) 对象。

* 在 JSON 根层级，标准端点会在 **statuses** 数组中返回 Post 对象，而 X API v2 会返回一个 **data** 数组。
* X API v2 的 JSON 不再使用 “statuses” 来指代被转发和被引用的内容，而是直接指代被转发和被引用的帖子。许多传统和已弃用的字段（例如 **contributors** 和 **user.translator&#95;type**）正在被移除。
* X API v2 不再在 Post 对象中使用 **favorites**，在 user 对象中使用 **favourites**，而是统一使用术语 `like`。
* X 采用这样一种约定：取值为空（例如 null）的 JSON 字段不会写入负载。只有在具有非 null 值时，Post 和 user 属性才会被包含在内。 
   

<div id="new-v2-fields">
  #### 新的 v2 字段
</div>

我们还为 [Post 对象](/zh/x-api/fundamentals/data-dictionary#tweet) 引入了一组新的字段，包括：

* 一个 [conversation&#95;id](/zh/x-api/fundamentals/conversation-id) 字段
* 两个新的 [annotations](/zh/x-api/fundamentals/post-annotations) 字段：context 和 entities
* 多个新的 [metrics](/zh/x-api/fundamentals/metrics) 字段
* 一个新的 reply&#95;setting 字段，用于显示谁可以回复某个帖子

<div id="migrating-from-standard-v11s-data-format-to-v2">
  ### 从标准 v1.1 的数据格式迁移到 v2
</div>

如果你还没有阅读，我们建议你先通读[数据格式迁移](/zh/x-api/migrate/data-format-migration)介绍。你可能也会对我们的[可视化数据格式迁移工具](/zh/x-api/migrate/data-format-migration#visual-data-format-migration-tool)感兴趣，它可以帮助你快速查看 [X API v1.1 数据格式](https://developer.x.com/en/docs/x-api/v1/data-dictionary/overview)与 [X API v2 格式](/zh/x-api/fundamentals/data-dictionary)之间的差异。

标准 v1.1 数据格式（也称为原生格式）是[标准 v1.1](https://developer.x.com/en/docs/twitter-api/v1)端点默认返回的主要数据格式。

如果你使用的是高级版产品，请参阅[原生富化数据指南](/zh/x-api/migrate/data-format-migration#migrating-from-native-enriched-data-format-to-v2)。Enterprise 客户可能会使用原生富化数据或活动流，具体取决于你在 Gnip 控制台中的配置方式。

<div id="migrating-from-native-enriched-data-format-to-v2">
  ### 从 Native Enriched 数据格式迁移到 v2
</div>

Native Enriched 数据格式用于我们的 [enterprise](/zh/x-api/enterprise-gnip-2.0/enterprise-gnip) 产品中。

Native Enriched 数据格式已经更新，以提供已编辑帖子（Tweet）的元数据。要进一步了解编辑帖子元数据，请参阅 [Edit Tweets 基础指南](/zh/x-api/enterprise-gnip-2.0/fundamentals/edit-tweets) 页面。

如果你在使用标准 v1.1 端点，请参考[从标准 v1.1 数据格式迁移到 v2 指南](/zh/x-api/migrate/data-format-migration#migrating-from-standard-v1-1s-data-format-to-v2)。如果你在使用带有 Activity Streams 的 enterprise 产品，我们也为你提供了[从 Activity Streams 数据格式迁移到 v2](/zh/x-api/migrate/data-format-migration#migrating-from-activity-streams-data-format-to-v2) 指南。

X API v2 为 [Tweet](/zh/x-api/fundamentals/data-dictionary#tweet) 和 [user](/zh/x-api/fundamentals/data-dictionary#user) 对象引入了新的 JSON 设计。

* 在 JSON 根级别，Native Enriched 格式在 results 数组中返回帖子对象，而 X API v2 在 data 数组中返回。
* X API v2 不再同时使用 favorites（在 Tweet 对象中）和 favourites（在 user 对象中），而是统一使用术语 like。
* X 采用的约定是：值为空的 JSON 字段（例如 null）不会写入载荷。只有在具有非空值时，才会包含 Tweet 和 user 属性。
* v2 中所有 id 字段都将采用字符串格式。
   

除了对新的 JSON 格式所做的这些更改之外，我们还在 Tweet 对象中引入了一组新的字段，包括：

* conversation&#95;id
* reply&#95;settings
* 媒体上的 alt&#95;text
* 两个新的 [annotations](/zh/x-api/fundamentals/post-annotations) 字段，包括 context 和 entities
* 若干新的 [metrics](/zh/x-api/fundamentals/metrics) 字段
* 若干新的 [polls](/zh/x-api/fundamentals/data-dictionary#poll) 字段
   

许多旧版和已弃用字段将被移除：

* contributors
* 某些 entities.media 和 extended&#95;entities.media 字段
* filter&#95;level
* timestamp&#95;ms
* truncated

<div id="migrating-from-activity-streams-data-format-to-v2">
  ### 从 Activity Streams 数据格式迁移到 v2
</div>

Activity Streams 数据格式可在我们的 [Enterprise](/zh/x-api/enterprise-gnip-2.0/enterprise-gnip) 产品中使用。

Activity Streams 数据格式已更新，以提供编辑后的帖子元数据。若要了解有关编辑帖子元数据的更多信息，请查看 [Edit Tweets fundamentals](/zh/x-api/enterprise-gnip-2.0/fundamentals/edit-tweets) 页面。

如果你正在使用标准版 v1.1 端点，请参阅 [standard v1.1 to v2 guide](/zh/x-api/migrate/overview)。如果你正在使用高级端点，或 Enterprise 的 Native Enriched 格式，请参阅 [Native Enriched to v2 guide](/zh/x-api/migrate/data-format-migration#migrating-from-native-enriched-data-format-to-v2)。

X API v2 为 [帖子](/zh/x-api/fundamentals/data-dictionary#tweet) 和 [用户](/zh/x-api/fundamentals/data-dictionary#user) 对象引入了新的 JSON 设计。

* 在 JSON 根级别，Activity Streams 格式会在 results 数组中返回 Tweet 对象，而 X API v2 会在 data 数组中返回对象。 
* 在 JSON 中，X API v2 不再将转发和引用称为“activities”，而是直接引用为被转发和被引用的帖子。 
* X API v2 不再在 Tweet 对象中使用 favorites、在 user 对象中使用 favourites，而是统一使用 like 这一术语。 
* Twitter 采用了这样一种约定：没有值的 JSON 字段（例如 null）不会写入载荷。Tweet 和 user 属性仅在其值非 null 时才会包含。 
* v2 中的所有 id 字段都将采用字符串格式。
   

除了对新的 JSON 格式进行的更改外，我们还在 Tweet 对象中引入了一组新字段，包括以下内容：

* conversation&#95;id
* reply&#95;settings
* 媒体上的 alt&#95;text
* 两个新的 [annotations](/zh/x-api/fundamentals/post-annotations) 字段，包括 context 和 entities
* 若干新的 [metrics](/zh/x-api/fundamentals/metrics) 字段
* 若干新的 [polls](/zh/x-api/fundamentals/data-dictionary#poll) 字段
   

许多旧版和已弃用字段已被移除或替换：

* display&#95;text&#95;range
* generator
* gnip
* link
* objectType
* provider
* twitter&#95;entities.symbols 被 data.entities.cashtags 所取代
* 某些 twitter&#95;extended&#95;entities.media 和 twitter&#95;entities.media 字段
* twitter&#95;filter&#95;level
* twitterTimeZone
* verb
