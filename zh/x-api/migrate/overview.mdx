<div id="what-is-the-x-api-v2">
  ## 什么是 X API v2？
</div>

X API v2 现在是主要的 X API，也是我们集中进行产品投入和创新的核心所在。我们与开发者合作，共同打造新一代的 X API，更好地服务多元化的开发者社区。基于开发者反馈，我们重新构建了该 API，以更好地满足更广泛的需求场景，引入新的功能和端点，并提升整体开发者体验。

X API v2 现在是主要的 X API，也是我们集中进行产品投入和创新的核心所在。在过去几年中，我们与开发者合作，重新构建了该 API，以更好地满足更广泛的需求，引入新的功能和端点，并优化开发者体验。我们将持续致力于构建一个开放的开发者平台，并期待看到你使用 X API v2 构建的一切成果。

<div id="why-migrate">
  ## 为什么要迁移？
</div>

X API v2 构建在更现代且更具可持续性的基础之上，既为标准 v1.1 和 Enterprise 产品提供了改进后的替代端点，也引入了全新的功能。我们强烈建议使用旧版 API（v1.1 和 Enterprise）的客户尽快开始迁移到 v2，因为我们确实计划最终弃用这些旧版 API。使用 X API 可以监测和分析公开对话、在 X 上与人互动并进行创新。

在本节中，我们将介绍这些端点及其功能。

<div id="discover-new-and-updated-response-objects">
  ### 了解新的和已更新的响应对象
</div>

以下六种数据对象可通过 v2 端点获取：

| Object | Description |
|:--------|:-------------|
| [Tweet](/zh/x-api/fundamentals/data-dictionary#tweet) | Tweet 对象拥有大量根级字段，例如 `id`、`text` 和 `created_at`。Tweet 对象也是多个子对象（包括 `user`、`media`、`poll` 和 `place`）的父对象。 |
| [User](/zh/x-api/fundamentals/data-dictionary#user) | user 对象包含描述所引用用户的 X 用户帐户元数据。 |
| [Spaces](/zh/x-api/fundamentals/data-dictionary#space) | Space 对象由 `state`、`host_id`、`is_ticketed` 以及 `lang` 等字段构成。 |
| [Lists](/zh/x-api/fundamentals/data-dictionary#list) | List 对象包含所请求列表的基本信息，包括 `description`、`member_count` 和 `owner_id`。 |
| [Media](/zh/x-api/fundamentals/data-dictionary#media) | 如果某条帖子包含媒体（例如图像），则可以使用 `media.fields` 参数请求 media 对象，其中包括 `media_key`、`type`、`url`、`preview_image_url` 等字段。 |
| [Poll](/zh/x-api/fundamentals/data-dictionary#poll) | 包含在 Tweet 中的投票不是任何端点中的主对象，但可以在 Tweet 对象中找到并进行扩展。 |
| [Place](/zh/x-api/fundamentals/data-dictionary#place) | place 对象由 `place_id`、`geo` 对象、`country_code` 等字段构成。这些信息可用于按位置识别和研究帖子。 |

详细了解[如何使用字段和扩展字段](/zh/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)。

<div id="flexibility-to-choose-which-objects-and-fields-you-receive">
  ### 灵活选择要接收的对象和字段
</div>

当你向 GET 端点发出请求时，你会收到与该端点相关的主要数据对象，其中会包含一组默认字段。例如，Tweet（帖子）对象默认返回 `id` 和 `text` 字段。

如果你希望在请求中获取更多字段，则需要使用 [fields](/zh/x-api/fundamentals/fields) 和 [expansions](/zh/x-api/fundamentals/expansions) 参数。`expansions` 参数可以让你检索关联的数据对象，例如用户置顶的帖子或媒体对象，而字段相关参数（field operators）可以让你在返回对象中请求默认字段之外的特定字段。

下面是你可以在不同 X API v2 端点中请求的全部扩展字段（expansions）列表：

| Object / Resource | Available Expansions |
|:-------------------|:----------------------|
| Tweets            | `author_id`, `edit_history_tweet_ids`, `entities.mentions.username`, `in_reply_to_user_id`, `referenced_tweets.id`, `referenced_tweets.id.author_id`, `attachments.poll_ids`, `attachments.media_keys`, `geo.place_id` |
| Users             | `pinned_tweet_id` |
| Spaces            | `invited_user_ids`, `speaker_ids`, `creator_id`, `host_ids`, `topic_ids` |

在[如何使用 fields 和 expansions](/zh/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)中了解更多信息。

<div id="edit-tweets">
  ## 编辑帖子
</div>

X API v2 端点提供与已编辑帖子（Tweet）相关的元数据。*编辑帖子（Edit Tweet）* 功能最初于 2022 年 9 月 1 日在 X 员工中开始测试。从该日期起，符合条件的帖子在发布后 30 分钟内可编辑，且最多可编辑 5 次。了解更多关于[编辑帖子](/zh/x-api/fundamentals/edit-posts)的内容。
使用 X API v2，开发者可以了解：

* 某条帖子在创建时是否具备可编辑资格。某些帖子（例如包含投票或已预定发布时间的帖子）无法编辑。
* 帖子在发布后 30 分钟内可编辑，且最多可编辑 5 次。对于可编辑的帖子，你可以查看是否仍有剩余编辑时间，以及还可以编辑多少次。
* 你当前查看的是否是该帖子的已编辑版本（在大多数情况下，除非你通过帖子 ID 请求特定的历史版本，否则 API 会返回帖子的最新版本）。
* 该帖子的完整编辑历史。
* 归因于该帖子每个版本的互动数据。

<div id="track-threaded-conversations">
  ## 跟踪会话线程
</div>

一个新的帖子字段可帮助识别某条帖子属于哪个会话线程。会话 ID 是发起该会话的那条帖子的 id。详细了解[会话跟踪](/zh/x-api/fundamentals/conversation-id)。

<div id="ready-to-migrate">
  ## 准备迁移
</div>

要使用 v2 端点，你需要具备以下内容：

* [一个开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)
* 在某个[项目](resources/fundamentals/projects)内创建的[开发者应用](https://developer.x.com/en/apps)
* 来自该项目的开发者应用的[密钥和令牌](resources/fundamentals/authentication)

请务必使用项目内某个应用的密钥和令牌。如果你使用的是项目之外应用的密钥和令牌，将无法向 v2 端点发起请求。

一旦你拥有开发者账号，就可以在[开发者门户](https://developer.x.com/en/portal/petition/essential/basic-info)中完成以上所有设置。

<div id="authentication">
  ### 身份验证
</div>

在新的 Twitter API 中，你将使用两种不同的身份验证方式：OAuth 1.0a 用户上下文和 OAuth 2.0 Bearer Token 来访问不同的端点。在向端点发起请求时，它们各自有不同的用途：

当你代表某个 Twitter 用户发起请求时，需要使用 OAuth 1.0a 用户上下文\
当你代表自己的开发者应用发起请求时，需要使用 OAuth 2.0 Bearer Token

为帮助你进一步了解这些变化，我们正在编写一系列指南，帮助你将 v2 之前的数据格式字段映射到 v2 中的新字段，并说明如何请求这些新字段。
