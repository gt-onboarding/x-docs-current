X 致力于支持使用 X API 构建产品的开发者社区。作为这一承诺的一部分，我们的目标是让我们的 API 对开发者开放且公平，对 X 上的用户安全，并整体上有利于 X 平台。对于任何将 X 内容进行离线存储的开发者而言，确保这些数据能够反映用户意图以及 X 上内容的当前状态至关重要。例如，当某位 X 用户删除了一条帖子或其账号、将其帖子设置为受保护，或编辑了一条帖子时，X 和我们的开发者都必须尊重该用户的期望和意图。

实时的合规事件流为开发者提供了工具，以便在遵守 [X Developer Agreement and Policy](https://developer.x.com/en/developer-terms/policy) 的前提下维护 X 数据。 

有两个 c&#95;ompliance event&#95; 流，一个用于 Post&#95; compliance&#95; 事件，另一个用于 *User compliance* 事件。这些流可通过 Enterprise 访问权限获取，旨在帮助摄取大规模数据的合作伙伴“监听”诸如帖子编辑事件等合规事件。

这些流会提供以下事件： 

**Post compliance 流： **

* delete - 表示该帖子已被删除。

* tweet&#95;edit - 表示一条帖子已被编辑，并提供更新后帖子的 ID。 

* withheld - 表示该帖子在一个或多个国家被屏蔽。 

* drop - 表示该帖子应从公开视图中移除。

* undrop - 表示该帖子可以再次展示并视为公开内容。

**User compliance 流： **

* **user&#95;delete** - 表示该用户账号已被删除

* **user&#95;undelete** - 表示该用户账号已被恢复

* **user&#95;protect** - 表示该用户账号变为私密账号

* **user&#95;unprotect** - 表示该用户账号变为公开账号

* **user&#95;withheld** - 表示该用户账号在一个或多个国家被屏蔽。 

* **user&#95;suspend** - 表示该用户账号已被暂停使用

* **user&#95;unsuspend** - 表示该用户账号已被解除暂停

* **user&#95;profile&#95;modification** - 表示该用户资料已被更新。这包括更新后的个人简介、名称、所在地和 URL。

**账户设置**

要访问这些端点，你需要：

* 已获批准的 [developer account](https://developer.x.com/en/portal/petition/essential/basic-info)。
* 使用位于某个 [Project](/zh/resources/fundamentals/projects) 中的 [developer App](/zh/resources/fundamentals/developer-apps) 的密钥和令牌进行认证。 

在我们的 [getting started guide](/zh/x-api/getting-started/getting-access) 中了解更多有关获取 X API v2 端点访问权限的信息。
