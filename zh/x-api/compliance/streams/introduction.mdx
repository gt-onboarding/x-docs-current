X 致力于支持使用 X API 构建产品的开发者社区。作为这一承诺的一部分，我们的目标是让我们的 API 对开发者开放且公平，对 X 上的用户安全，并整体上有利于 X 平台。任何在离线环境中存储 X 内容的开发者，都必须确保这些数据能够反映用户意图以及 X 上内容的当前状态。例如，当 X 上的某位用户删除一条帖子或其账号、将其帖子设为受保护状态，或编辑一条帖子时，X 和我们的开发者都必须切实尊重该用户的期望和意图。

实时合规事件流为开发者提供工具，以确保 X 数据符合 [X Developer Agreement and Policy](https://developer.x.com/en/developer-terms/policy) 的要求。 

有两个 c&#95;ompliance event&#95; 流，一个用于 Post&#95; compliance&#95; 事件，另一个用于 *User compliance* 事件。这些流在 Enterprise 访问权限下可用，旨在帮助摄取大量数据的合作伙伴“监听”合规事件，例如帖子编辑事件。

这些流提供以下事件： 

**Post compliance 流： **

* delete - 表示该帖子已被删除。

* tweet&#95;edit - 表示一条帖子已被编辑，并提供更新后帖子的 ID。 

* withheld - 表示该帖子已在一个或多个国家/地区被屏蔽。 

* drop - 表示该帖子应从公开视图中移除。

* undrop - 表示该帖子可以再次展示，并被视为公开内容。

**User compliance 流： **

* **user&#95;delete** - 表示该用户账号已被删除

* **user&#95;undelete** - 表示该用户账号已被恢复

* **user&#95;protect** - 表示该用户账号变为私密

* **user&#95;unprotect** - 表示该用户账号变为公开

* **user&#95;withheld** - 表示该用户账号已在一个或多个国家/地区被屏蔽。 

* **user&#95;suspend** - 表示该用户账号已被暂停使用

* **user&#95;unsuspend** - 表示该用户账号已被解除暂停

* **user&#95;profile&#95;modification** - 表示该用户资料已被更新。包括更新的简介、名称、所在地和 URL。

**账号设置**

要访问这些端点，你需要：

* 已获审批的[开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)。
* 使用位于某个[项目](/zh/resources/fundamentals/projects)中的[开发者应用](/zh/resources/fundamentals/developer-apps)的密钥和令牌进行认证。 

在我们的[入门指南](/zh/x-api/getting-started/getting-access)中了解更多关于如何获取 X API v2 端点访问权限的信息。
