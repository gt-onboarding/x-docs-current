X 致力于支持使用 X API 进行构建的开发者社区。作为这一承诺的一部分，我们的目标是让我们的 API 对开发者来说开放且公平，对 X 上的用户而言安全可靠，并让整个 X 平台受益。对于任何将 X 内容离线存储的开发者来说，确保数据能够反映用户意愿以及 X 上内容的当前状态至关重要。例如，当 X 上的某人删除了某条帖子或其账号、将其帖子设为受保护，或从其帖子中清除地理位置信息时，X 和我们的开发者都必须尊重该用户的预期和意图。批量合规端点为开发者提供了一个简单的工具，帮助你根据 [X Developer Agreement and Policy](https://developer.x.com/en/developer-terms/policy) 保持 X 数据合规。 

这些批量合规端点允许你上传包含帖子或用户 ID 的大型数据集，以获取它们的合规状态，从而确定哪些数据需要采取行动，使你的数据集符合合规要求。请注意，批量合规端点的使用仅限于前述使用场景，任何其他用途均被禁止，并可能导致平台执法措施。

通常，使用此端点需要经过 4 个步骤：

1. **创建合规任务**
   你可以指定任务类型（值为 tweets 或 users），以指明你要上传的数据集包含的是帖子 ID 还是用户 ID。你在任意时间每种任务类型最多只能有一个并发任务。
2. **将数据集上传到 upload&#95;url**
   接下来，你将数据集作为纯文本文件上传到提供的 upload&#95;url，文件的每一行包含一个帖子 ID 或用户 ID。该 upload&#95;url 在 15 分钟后过期。
3. **（可选）检查任务状态**
   你可以检查合规任务的状态，以查看其是已创建、进行中（in&#95;progress）、失败还是已完成。
4. **下载结果**
   当你的任务完成后，你可以使用 download&#95;url 下载结果。该 download&#95;url 在一周后过期（从任务创建时开始计时）。

   结果中将包含一组 JSON 对象（每行一个对象）。每个对象会包含一个帖子 ID、该帖子的创建日期（有助于按日期定位帖子）、所需的操作、采取合规操作的原因，以及该用户被暂停的日期。

在结果中，你将会收到以下合规事件类型：

* deleted - 表示帖子或用户账号已被删除
* deactivated - 表示帖子或用户账号已被停用
* scrub&#95;geo - 表示与该帖子或用户关联的地理位置信息已被移除
* protected - 表示发布该帖子的账号变为私密账号
* suspended - 表示发布该帖子的账号已被暂停使用

**账号设置**

要访问这些端点，你需要：

* 一个已获批准的 [开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)。
* 使用位于某个 [项目](/zh/resources/fundamentals/projects) 中的 [开发者应用](/zh/resources/fundamentals/developer-apps) 所提供的密钥和令牌进行认证。 

在我们的 [入门指南](/zh/x-api/getting-started/getting-access) 中了解更多关于获取 X API v2 端点访问权限的信息。
