<div id="step-one-sign-up-for-a-developer-account">
  ### 第一步：注册开发者账号
</div>

注册开发者账号既快捷又简单！只需点击下方按钮，回答几个问题，就可以通过 [Basic access](https://developer.x.com/en/portal/products/basic) 开始探索并基于 X API v2 开发。

接下来，你将在入门流程中创建一个项目以及一个关联的开发者应用，这会为你提供一组凭证，用来对你发往 API 的所有请求进行身份验证。

[注册](https://developer.x.com/en/portal/products/basic)

<div id="step-two-save-your-apps-key-and-tokens-and-keep-them-secure">
  ### 步骤二：保存你的应用密钥和令牌并确保它们的安全
</div>

在你获得访问权限并创建了项目和应用之后，你就可以在你的开发者应用中找到或生成以下凭证：

* **[API Key and Secret](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret)：** 本质上相当于你的应用的用户名和密码。你将使用它们来对需要 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 的请求进行身份验证，或生成其他令牌，例如用户访问令牌或应用访问令牌。

* **[Access Token and Secret](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)：** 一般来说，Access Token 代表了你所代为发起请求的用户。你可以通过开发者门户生成的这些令牌代表的是拥有该应用的用户。你将使用它们来对需要 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 的请求进行身份验证。如果你希望代表其他用户发起请求，你需要使用三方 OAuth 流程，让该用户授权你。

* **[Client ID and Client Secret](/zh/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)：** 这些凭证用于通过 OAuth 2.0 认证流程获取用户访问令牌。与 OAuth 1.0a 类似，用户访问令牌用于对提供私有用户账户信息或代表其他账户执行操作的请求进行身份验证，但具备更细粒度的 scope，以便更好地控制客户端应用对用户所拥有的访问权限。

* **[App only Access Token](/zh/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)：** 当你向仅返回 X 上公开可见信息的端点发起请求时，需要使用此令牌。

由于这些密钥和令牌除非被重新生成否则不会过期，我们建议你在收到凭证后，将它们保存在安全的位置，例如密码管理器中。

> **请注意：你的密钥和令牌只会在 [开发者门户](/zh/resources/fundamentals/developer-portal) 中显示一次，因此在你生成这些凭证后，务必立即将它们存储到你的密码管理系统中。**
>
> 如果你丢失或忘记了密钥和令牌，你将需要重新生成它们，这会创建新的凭证并使旧凭证失效。这意味着你必须更新所有使用旧凭证所配置的集成。
>
> 详细了解我们的[认证最佳实践](/zh/resources/fundamentals/authentication#authentication-best-practices)。
