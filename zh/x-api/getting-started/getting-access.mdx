<div id="step-one-sign-up-for-a-developer-account">
  ### 第一步：注册开发者账号
</div>

注册开发者账号既快速又简单！只需点击下面的按钮，回答几个问题，就可以开始使用 [Basic access](https://developer.x.com/en/portal/products/basic) 探索并基于 X API v2 进行开发。

接下来，你将在入门流程中创建一个项目以及一个关联的开发者应用，这将为你提供一组凭证，用于对你发送到 API 的所有请求进行身份验证。

[注册](https://developer.x.com/en/portal/products/basic)

<div id="step-two-save-your-apps-key-and-tokens-and-keep-them-secure">
  ### 第二步：保存你的应用的密钥和令牌并确保其安全
</div>

在你获得访问并创建项目和应用之后，你就可以在你的开发者应用中找到或生成以下凭证：

* **[API Key and Secret](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret)：** 本质上是你的应用的用户名和密码。你将使用这些来对需要 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 的请求进行身份验证，或生成其他令牌，例如用户访问令牌或应用访问令牌。

* **[Access Token and Secret](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)：** 一般来说，访问令牌表示你代表其发出请求的用户身份。你可以通过开发者门户生成的访问令牌代表的是拥有该应用的用户。你将使用这些令牌来对需要 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 的请求进行身份验证。如果你想代表其他用户发出请求，你需要使用三方 OAuth 流程，让他们授权你。

* **[Client ID and Client Secret](/zh/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)：** 这些凭证用于通过 OAuth 2.0 认证获取用户访问令牌。与 OAuth 1.0a 类似，用户访问令牌用于对那些访问私有用户账户信息或代表其他账户执行操作的请求进行身份验证，但其具备细粒度的作用域，从而可以更好地控制客户端应用对用户所拥有的访问权限。

* **[App only Access Token](/zh/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)：** 当你向会返回 X 上公开可见信息的端点发出请求时，将使用此令牌。

由于这些密钥和令牌在未重新生成的情况下不会过期，我们建议在你收到这些凭证后，将其保存在安全的位置，例如密码管理器中。

> **请注意：你的密钥和令牌只会在 [开发者门户](/zh/resources/fundamentals/developer-portal) 中显示一次，因此在你生成这些凭证后，务必立即将它们存储到你的密码管理器中。**
>
> 如果你丢失或忘记了这些密钥和令牌，则需要重新生成密钥和令牌，这会创建新的凭证并使旧的凭证失效。这意味着你必须更新所有使用旧凭证所设置的集成。
>
> 详细了解我们的[身份验证最佳实践](/zh/resources/fundamentals/authentication#authentication-best-practices)。
