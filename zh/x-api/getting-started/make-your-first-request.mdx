本指南将带你逐步完成一些步骤，帮助你发出你的第一个请求。在你注册了 X 帐号之后，这会是一个非常有用的参考资源。

如果你想使用代码示例、更技术向的指南，或像 Postman 这样的图形化工具来发出第一个请求，可以参考以下指南：

* [快速开始指南：向帖子端点发出你的第一个请求](/zh/x-api/posts/manage-tweets/quickstart)
* [使用 Postman 入门](/zh/tutorials/postman-getting-started) - 最适合初学者
* [X API v2 代码示例](https://github.com/xdevplatform/Twitter-API-v2-sample-code)
* [帖子查询 API 参考](/zh/x-api/posts/lookup/introduction)

本指南假设你已经获取了你的 [API Key 和 Secret](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret)、[用户访问令牌和 Secret](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)、[应用访问令牌](/zh/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)，并已将它们安全地存储起来。你可以按照[获取 X API 访问权限](/zh/x-api/getting-started/getting-access)指南中的步骤来了解如何完成这些操作。

<div id="step-1-identify-which-endpoint-you-would-like-to-use">
  ### 步骤 1. 确定你想要使用的端点
</div>

X API 允许你通过代码执行多种操作，这些操作通常也可以在 X 网站或移动应用上完成。

我们在 [API Reference Index](/zh/x-api/introduction) 中列出了所有可通过 API 使用的端点，但为了简单起见，我们建议你先从以下几项中选择一个：

* [管理帖子 &gt; 发布一条帖子](/zh/x-api/posts/manage-tweets/introduction)
* [搜索帖子](/zh/x-api/posts/search/introduction)
* [用户查询](/zh/x-api/users/lookup/introduction)

<div id="step-2-choose-a-tool-to-make-your-request">
  ### 步骤 2. 选择一个用于发起请求的工具
</div>

有些请求可能相对简单，但也有一些会比较难构建。因此，优秀的开发者社区开发了各种工具，帮你屏蔽掉部分复杂性。

下面是一些推荐的工具以及如何使用它们的说明：

<div id="postman">
  #### Postman
</div>

Postman 是一款可视化工具，你可以使用它向 REST 端点发起请求。我们围绕 Postman 制作了一些优质资料，帮助你入门并探索通过 X API 提供的各类端点。

我们建议你先阅读我们的[《Postman 入门》教程](/zh/tutorials/postman-getting-started)，了解如何添加你的密钥和令牌，并发出你的第一个请求。

我们还为 X API v2 的每个端点制作了快速入门指南，其中大多数都使用 Postman。你可以在各自的端点章节中找到这些指南，这里列出其中的一些链接：

* 快速入门：发布一条帖子
* 快速入门：搜索帖子
* 快速入门：查找用户

请注意，你不能使用 Postman 向流式端点发起请求。请访问 [filtered stream](/zh/x-api/posts/filtered-stream/quickstart) 或 [1% sampled stream](/zh/x-api/posts/sample-stream) 快速入门指南，了解如何使用这些端点。

如果你更喜欢更简单的图形界面工具，你也可以考虑使用 [Insomnia](https://insomnia.rest)。

<div id="sample-code">
  #### 示例代码
</div>

如果你希望使用一些简单的代码来发出请求，我们已经为每个 X API v2 端点准备了多种语言的示例代码。

你可以在我们的 GitHub 仓库 [X-API-v2-sample-code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) 中找到这些代码示例。该仓库还包含一个 README 文件，你可以通过它了解如何设置凭证，以便正确配合这些请求使用。

例如，下面是一个用于用户查询端点的 cURL 示例。要使用此请求，你只需将 $ACCESS&#95;TOKEN 和 $USERNAME 分别替换为你的[应用访问令牌](/zh/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)和 X 账号。然后，将这段代码复制粘贴到命令行工具中，并按下 “Return” 或 “Enter” 键即可。

```bash
curl "https://api.x.com/2/users/by/username/$USERNAME" -H "Authorization: Bearer $ACCESS_TOKEN"
```

  #### 库
</div>

出色的 X 开发者社区还构建了多种编程语言的客户端库，可用于向 X API 发起请求。

我们制作了一个[「工具和库」页面](/zh/x-api/tools-and-libraries/overview)，列出了我们所知的所有社区库。每个库通常都会包含一个 README 文件，你可以通过它了解如何在本地机器上设置该代码仓库，并发出你的第一个请求。

> **请注意：** 如果你遇到任何问题，请查看你要调用的端点对应的开发者文档、我们的[支持页面](https://developer.x.com/en/support)，或者在论坛上向社区寻求帮助。我们会帮助你顺利完成请求！

<div id="step-3-review-the-response">
  ### 步骤 3. 查看响应
</div>

一旦你成功发送请求，你会收到一个包含与该请求相关元数据的有效负载。

如果你使用的是使用 GET HTTP 方法的端点，你会以 JSON 格式收到与所请求资源（帖子、用户、列表、Space 等）相关的元数据。查看返回的各个字段，并尝试将你请求的信息对应到 X 上的具体内容。

如果你使用的是使用 POST、PUT 或 DELETE HTTP 方法的端点，则表示你在 X 上执行了一个操作。前往 X.com 或移动应用，查看能否找到你执行的那个操作。

<div id="step-4-adjust-your-request-using-parameters">
  ### 步骤 4：使用参数调整请求
</div>

每个端点都有一组不同的参数，可用于修改你的请求。例如，在使用带有 `fields` 和 `expansions` 参数的 GET 端点时，你可以请求额外的元数据字段。你还可以在诸如 [搜索帖子](/zh/x-api/posts/search/introduction)、[帖子计数](/zh/x-api/posts/counts/introduction) 和 [filtered stream](/zh/x-api/posts/filtered-stream/introduction) 等端点上尝试多种不同的过滤方式，帮助你将接收到的数据范围缩小到你感兴趣的那些帖子。

你可以在所使用端点的 API 参考文档中找到请求参数和字段的完整列表，还可以在我们的集成指南和基础入门页面中找到大量其他实用的集成相关信息。

你可以通过我们的[重要资源](/zh/x-api/getting-started/important-resources)指南，进一步了解我们为你提供的所有学习材料。
