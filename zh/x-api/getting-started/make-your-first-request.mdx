本指南将通过几个步骤引导您发出第一个请求。在您注册 X 账号之后，它会对您非常有帮助。

如果您有兴趣使用代码示例、更技术性的指南，或像 Postman 这样的图形化工具，请考虑使用以下指南来完成您的第一个请求：

* [向帖子端点发出第一个请求的快速入门指南](/zh/x-api/posts/manage-tweets/quickstart)
* [Postman 入门](/zh/tutorials/postman-getting-started) - 最适合初学者
* [X API v2 代码示例](https://github.com/xdevplatform/Twitter-API-v2-sample-code)
* [帖子查询 API 参考](/zh/x-api/posts/lookup/introduction)

本指南假设您已经获取了 [API key 和 secret](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret)、[用户访问令牌（Access Token）和密钥（Secret）](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)、[应用访问令牌（App Access Token）](/zh/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)，并已将它们安全存储。您可以按照[获取 X API 访问权限](/zh/x-api/getting-started/getting-access) 指南中的步骤了解如何完成这些操作。

<div id="step-1-identify-which-endpoint-you-would-like-to-use">
  ### 步骤 1. 确定要使用的端点
</div>

X API 允许你通过代码执行多种操作，这些操作通常也可以在 X 网站或移动应用上完成。

我们在 [API 参考索引](/zh/x-api/introduction) 中列出了通过 API 可用的完整端点列表，不过为简化起见，我们建议先从以下端点之一开始：

* [管理帖子 &gt; 发布帖子](/zh/x-api/posts/manage-tweets/introduction)
* [搜索帖子](/zh/x-api/posts/search/introduction)
* [用户查询](/zh/x-api/users/lookup/introduction)

<div id="step-2-choose-a-tool-to-make-your-request">
  ### 步骤 2：选择一个用于发起请求的工具
</div>

虽然有些请求相对简单，但另一些可能比较难以构建。这就是为什么优秀的开发者社区开发了各种工具，来帮你屏蔽掉部分复杂性。

下面是一些推荐使用的工具以及如何使用它们的说明：

<div id="postman">
  #### Postman
</div>

Postman 是一款可视化工具，你可以用它向 REST 端点发起请求。我们围绕 Postman 提供了一些优质资源，帮助你上手并探索通过 X API 提供的各个端点。

我们建议你阅读我们的 [《Postman 入门教程》](/zh/tutorials/postman-getting-started)，以了解如何添加你的密钥和令牌，并发送你的第一个请求。

我们还为每个 X API v2 端点制作了快速入门指南，其中大多数都使用 Postman。你可以在各自的端点章节中找到这些指南，下面是其中的一些链接：

* 快速入门：发布一条帖子
* 快速入门：搜索帖子
* 快速入门：查询用户

请注意，你无法使用 Postman 向流式端点发起请求。请访问 [过滤流](/zh/x-api/posts/filtered-stream/quickstart) 或 [1% 抽样流](/zh/x-api/posts/sample-stream) 的快速入门指南，以了解如何使用这些端点。

如果你更喜欢界面更简单的图形化工具，也可以考虑使用 [Insomnia](https://insomnia.rest)。

<div id="sample-code">
  #### 示例代码
</div>

如果你想使用一些简单的代码来发送请求，我们为每个 X API v2 端点准备了多种语言的示例代码。

你可以在我们的 GitHub 代码仓库 [X-API-v2-sample-code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) 中找到这些代码示例，其中还包含一个 README 文件，你可以通过它学习如何设置你的凭证，以便正确地发送这些请求。

例如，下面是一个用于用户查询端点的 cURL 示例。你只需要将其中的 $ACCESS&#95;TOKEN 和 $USERNAME 替换为你的 [App Access Token](/zh/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) 和 X 账号。然后，将这段代码复制并粘贴到你的命令行工具中，并按下 “Return” 或 “Enter” 键即可。

```bash
curl "https://api.x.com/2/users/by/username/$USERNAME" -H "Authorization: Bearer $ACCESS_TOKEN"
```

<div id="libraries">
  #### 库
</div>

活跃的 X 开发者社区还使用多种编程语言构建了大量库，你可以借助这些库向 X API 发起请求。

我们创建了一个[“工具和库”页面](/zh/x-api/tools-and-libraries/overview)，列出了我们所知的所有社区库。每个库通常都会包含一个 README 文件，你可以通过它了解如何在本地环境中设置该仓库，并发出你的第一个请求。

> **请注意：** 如果你遇到任何问题，请先查阅你要调用的该 endpoint 的开发者文档、我们的[支持页面](https://developer.x.com/en/support)，或者在我们的论坛向社区寻求帮助。我们会帮助你顺利完成请求！

<div id="step-3-review-the-response">
  ### 步骤 3. 查看响应
</div>

成功发出请求后，你会收到一个包含与该请求相关元数据的响应负载。

如果你使用的是使用 GET HTTP 方法的端点，你会以 JSON 格式收到与你请求的资源（帖子、用户、列表、Space 等）相关的元数据。查看返回的各个字段，并尝试将你请求的信息与 X 上的内容一一对应。

如果你使用的是使用 POST、PUT 或 DELETE HTTP 方法的端点，那么你是在 X 上执行了某个操作。前往 X.com 或移动应用，看看能否找到你执行的那个操作。

<div id="step-4-adjust-your-request-using-parameters">
  ### 步骤 4：使用参数调整你的请求
</div>

每个 endpoint 都有一组不同的参数，你可以使用它们来修改请求。例如，在使用带有 `fields` 和 `expansions` 参数的 GET endpoint 时，你可以请求额外的元数据字段。你还可以在诸如 [搜索帖子](/zh/x-api/posts/search/introduction)、[帖子计数](/zh/x-api/posts/counts/introduction) 和 [filtered stream](/zh/x-api/posts/filtered-stream/introduction) 等 endpoint 上尝试多种不同的过滤工具，帮助你将接收到的 数据 范围缩小到只包含你感兴趣的那些 帖子。

你可以在所使用 endpoint 的 API Reference 中找到该 endpoint 的完整请求参数和 字段 列表，还可以在我们的集成指南和基础知识页面中找到大量其他有用的集成信息。

你可以通过我们的 [重要资源](/zh/x-api/getting-started/important-resources) 指南，进一步了解我们为你提供的所有教学资料。
