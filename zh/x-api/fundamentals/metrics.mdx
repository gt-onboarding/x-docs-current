<div id="overview">
  ## 概览
</div>

`metrics` 字段允许开发者访问帖子和媒体对象的公开和非公开互动指标。公开指标对任何拥有开发者账号的人都可访问，而非公开指标仅可从自有/已授权账号（定义见下文）获取。`metrics` 包括每个请求中指定帖子获得的总曝光次数、转发（Retweets）、引用帖子（Quote Tweets）、点赞、回复、视频观看次数、视频观看四分位数，以及 URL 和个人资料链接点击次数。如果帖子作为[广告](https://ads.x.com/)进行投放，还可以选择查看在自然（organic）或推广（promoted）投放场景下获得的指标拆分。

公开指标可以通过 [App-only Token](/zh/resources/fundamentals/authentication#oauth-2-0) 认证来请求。非公开指标只能针对自有/已授权的帖子请求，这意味着开发者需要使用 [OAuth 2.0](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) 或 [OAuth 1.0a](/zh/resources/fundamentals/authentication#oauth-1-0a-2) 用户上下文授权进行认证。

**非公开、自然和推广指标仅适用于最近 30 天内创建的帖子。**

<div id="terminology">
  ### 术语
</div>

* **已授权账户（Authorized account）**：通过向你的 [X 开发者应用](/zh/resources/fundamentals/developer-apps)授予对自身账户的访问权限而完成授权的 X 账户（任何 [应用权限级别](/zh/resources/fundamentals/developer-apps#app-permissions) 都允许访问帖子指标）。
* **已关联账户（Owned account）**：与你的 [X 开发者应用](/zh/resources/fundamentals/developer-apps)关联的 X 账户。
* **公开指标（Public metrics）**：在 [X](https://x.com/home) 上任何人都可以访问的汇总数据，例如点赞数和转发数。
* **非公开指标（Non-public metrics）**：在 [X](https://x.com/home) 上不对公众展示的汇总数据，例如展示次数和视频观看四分位数。需要使用 [OAuth 2.0](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) 或 [OAuth 1.0a](/zh/resources/fundamentals/authentication#oauth-1-0a-2) 的用户上下文进行身份验证。
* **自然流量指标（Organic metrics）**：归因于自然场景（以常规方式发布和浏览）的公开和非公开指标的集合。需要使用 [OAuth 2.0](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) 或 [OAuth 1.0a](/zh/resources/fundamentals/authentication#oauth-1-0a-2) 的用户上下文进行身份验证。
* **推广指标（Promoted metrics）**：归因于推广场景（作为广告活动的一部分被发布或浏览）的公开和非公开指标的集合。需要使用 [OAuth 2.0](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) 或 [OAuth 1.0a](/zh/resources/fundamentals/authentication#oauth-1-0a-2) 的用户上下文进行身份验证，并且相关帖子已在广告中投放推广。

<div id="available-metrics">
  ## 可用指标
</div>

| 指标                 | API 表示形式                                                                                                                                                                                                                                            | 说明                                                                                                                                                                                                                                                                                                                                                                                       |
|:----------------------|:---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|:--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Impressions**      | `data.non_public_metrics.impression_count`, `data.organic_metrics.impression_count`, `data.promoted_metrics.impression_count`                                                                                                                           | 帖子被展示的总次数（不按用户去重）。只要帖子有任意部分在屏幕上可见，即计为一次展示。需要使用 OAuth 1.0a 用户上下文认证。                                                                                                                                                                                                                           |
| **Retweets**         | `data.public_metrics.retweet_count`, `data.organic_metrics.retweet_count`, `data.promoted_metrics.retweet_count`                                                                                                                                       | 帖子被转发的总次数。不包括 Quote Tweets（“带评论的转发”）。要获得在 X 客户端中显示的“转发和评论”总数，请将 `retweet_count` 与 `quote_count` 相加。                                                                                                                                                           |
| **Quote Tweets**     | `data.public_metrics.quote_count`                                                                                                                                                                                                                      | 帖子被带新评论（消息）转发的总次数。不存在来自付费场景的 Quote Tweet，因此所有 Quote Tweet 都是自然行为。                                                                                                                                                                                                  |
| **Likes**            | `data.public_metrics.like_count`, `data.organic_metrics.like_count`, `data.promoted_metrics.like_count`                                                                                                                                                 | 帖子被点赞的总次数。`public_metrics` 字段返回来自自然（非付费）和付费两种场景的点赞总数，以保持与 X 上公开显示的计数一致。                                                                                                                                                                                  |
| **Replies**          | `data.public_metrics.reply_count`, `data.organic_metrics.reply_count`, `data.promoted_metrics.reply_count`                                                                                                                                              | 帖子被回复的总次数。`public_metrics` 字段返回来自自然（非付费）和付费两种场景的回复总数。                                                                                                                                                                                                                   |
| **URL Link Clicks**  | `data.non_public_metrics.url_link_clicks`, `data.organic_metrics.url_link_clicks`, `data.promoted_metrics.url_link_clicks`                                                                                                                             | 用户在帖子中点击 URL 链接或 URL 预览卡片的总次数。需要使用 OAuth 1.0a 用户上下文认证。                                                                                                                                                                                                                      |
| **User Profile Clicks** | `data.non_public_metrics.user_profile_clicks`, `data.organic_metrics.user_profile_clicks`, `data.promoted_metrics.user_profile_clicks`                                                                                                             | 用户点击帖子中以下部分的总次数：显示名称、用户名、头像。需要使用 OAuth 1.0a 用户上下文认证。                                                                                                                                                                                                                |
| **Video views**      | `includes.media.public_metrics.view_count`, `includes.media.organic_metrics.view_count`, `includes.media.promoted_metrics.view_count`                                                                                                                  | 帖子中包含的视频被观看的总次数。此值为该视频在所有包含该视频的帖子中的观看次数总和。需要使用媒体扩展字段 `expansions=attachment.media_keys`。                                                                                                                                                                 |
| **Video view quartiles** | `includes.media.non_public_metrics.playback_0_count`, `includes.media.non_public_metrics.playback_25_count`, `includes.media.non_public_metrics.playback_50_count`, `includes.media.non_public_metrics.playback_75_count`, `includes.media.non_public_metrics.playback_100_count` | 在视频中播放到各个四分位节点的用户数量。需要使用 OAuth 1.0a 用户上下文认证以及媒体扩展字段 `expansions=attachment.media_keys`。                                                                                                                                                                              |

<div id="requesting-metrics">
  ## 请求指标
</div>

<div id="public-metrics">
  ### 公开指标
</div>

在下面的请求中，我们会为帖子本身以及其所附视频请求公开指标，使用如下字段和扩展字段。请务必将 `$BEARER_TOKEN` 替换为你自己生成的 Bearer Token。

* `tweet.fields=public_metrics`
* `expansions=attachments.media_keys&media.fields=public_metrics`

<div id="sample-request">
  #### 示例请求
</div>

```bash
curl 'https://api.x.com/2/tweets?ids=1204084171334832128&tweet.fields=public_metrics&expansions=attachments.media_keys&media.fields=public_metrics' --header 'Authorization: Bearer $BEARER_TOKEN'
```

<div id="private-metrics-non-public-organic-metrics">
  ### 私有指标（非公开，自然指标）
</div>

以下请求会为帖子及其附加视频获取非公开指标，并包含更详细的自然指标信息。由于这些字段是私有的，且在 X 上无法公开查看，因此该请求需要使用 [OAuth 2.0](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) 或 [OAuth 1.0a](/zh/resources/fundamentals/authentication#oauth-1-0a-2) 的用户上下文认证。有关生成下方所需 OAuth 1.0a 签名的方法，请参阅我们的[指南](/zh/resources/fundamentals/authentication#creating-a-signature)。

* `tweet.fields=non_public_metrics,organic_metrics`
* `expansions=attachments.media_keys&media.fields=non_public_metrics,organic_metrics`

#### 请求示例

```bash
curl 'https://api.x.com/2/tweets/1204084171334832128?tweet.fields=non_public_metrics,organic_metrics&media.fields=non_public_metrics,organic_metrics&expansions=attachments.media_keys' --header 'authorization: OAuth oauth_consumer_key="CONSUMER_API_KEY", oauth_nonce="OAUTH_NONCE", oauth_signature="OAUTH_SIGNATURE", oauth_signature_method="HMAC-SHA1", oauth_timestamp="OAUTH_TIMESTAMP", oauth_token="ACCESS_TOKEN", oauth_version="1.0"'
```
