<div id="tweet">
  ### 帖子
</div>

帖子是构成 Twitter 上一切内容的基本单元。Tweet 对象包含一长串顶层字段，例如 `id`、`text` 和 `created_at`。Tweet 对象也是多个子对象的“父”对象，包括 `user`、`media`、`poll` 和 `place`。在请求 Tweet 对象上的这些顶层字段时，请使用字段参数 `tweet.fields`。

可以在 user 资源中找到并展开 Tweet 对象。与所请求 Tweet 相关的其他 Tweet 也可以在 Tweet 资源中找到并展开。该对象可以在 user 资源中通过 `?expansions=pinned_tweet_id`，或在 Tweet 资源中通过 `?expansions=referenced_tweets.id` 进行扩展，以仅使用默认字段获取该对象。在请求更多字段以完善该对象时，请将此扩展与字段参数 `tweet.fields` 结合使用。

| 字段                                               | 数据类型            | 说明                                                                                              | 使用方式                           |
| :----------------------------------------------- | :-------------- | :---------------------------------------------------------------------------------------------- | :----------------------------- |
| **id（默认）**                                       | 字符串             | 所请求的帖子的唯一标识符。                                                                                   | 可用于以编程方式检索特定帖子。                |
| **text（默认）**                                     | 字符串             | 帖子的实际 UTF-8 文本。有关有效字符的详细信息，请参见 [twitter-text](https://github.com/twitter/twitter-text/)。        | 关键词提取和情感分析/分类。                 |
| **edit&#95;history&#95;tweet&#95;ids (default)** | 对象              | 用于标识帖子所有版本的唯一标识符。对于没有编辑记录的帖子，将只有一个 ID；对于有编辑历史的帖子，将会有多个 ID。                                      | 使用这些信息查找帖子的编辑历史记录。             |
| **article**                                      | 对象              | 包含此帖子中关联 Article 的元数据。                                                                          | 可用于获取 Article 的文本和实体。          |
| **attachments**                                  | 对象              | 指定此帖子所包含的附件类型（如有）。                                                                              | 用于了解通过所请求扩展字段返回的对象。            |
| **author&#95;id**                                | 字符串             | 发布此帖子的用户的唯一标识符。                                                                                 | 用于补全 User 对象，以及在同行评审时共享数据集。    |
| **card&#95;uri**                                 | 字符串             | 此帖子中所包含 Card 的 URI。                                                                             |                                |
| **community&#95;id**                             | 字符串             | 该帖子所属 Community 的唯一标识符。                                                                         |                                |
| **context&#95;annotations**                      | 数组              | 包含与该帖子相关的上下文标注。                                                                                 | 实体识别/提取、主题分析。                  |
| **conversation&#95;id**                          | 字符串             | 会话中原始帖子的帖子 ID（包括对该帖子的直接回复及其后续回复）。                                                               | 使用该字段可以根据任意一条帖子还原整段会话。         |
| **created&#95;at**                               | 日期（ISO 8601 格式） | 帖子创建时间。                                                                                         | 可用于时间序列分析，并了解帖子创建时间。           |
| **display&#95;text&#95;range**                   | 数组              | 一个数组，包含要显示文本片段的起始和结束索引。                                                                         | 对于较长帖子，可用于确定默认会显示文本的哪一部分。      |
| **edit&#95;controls**                            | 对象              | 表示该帖子距离编辑截止还剩多长时间，以及剩余的可编辑次数。                                                                   | 用于判断帖子是否符合编辑条件。                |
| **entities**                                     | 对象              | 从帖子文本中解析出的实体。请参阅 Twitter 对象中的 entities。                                                         | 提供有关话题标签、URL、提及等的更多信息。         |
| **geo**                                          | 对象              | 表示带地理标记帖子的位置。                                                                                   | 使用此字段获取带地理标签帖子的位置信息。           |
| **in&#95;reply&#95;to&#95;user&#95;id**          | 字符串             | 如果当前帖子是回复，则此字段将包含原始帖子的作者 ID。                                                                    | 用于判断该帖子是否为对另一条帖子的回复。           |
| **lang**                                         | 字符串             | 帖子所用的语言（如果已被 Twitter 识别）。                                                                       | 按帖子所用语言进行分类。                   |
| **non&#95;public&#95;metrics**                   | 对象              | 在发送请求时该帖子产生的非公开互动指标。需要用户上下文身份验证。                                                                | 用于确定该帖子产生的总展示次数。               |
| **note&#95;tweet**                               | object          | 包含长篇帖子（&gt;280 个字符）的完整文本。                                                                       | 获取帖子全文。                        |
| **organic&#95;metrics**                          | 对象              | 在请求发出时，该帖子在自然（非付费）环境中所追踪到的互动指标。需要用户上下文身份验证。                                                     | 用于衡量该帖子的自然互动情况。                |
| **possibly&#95;sensitive**                       | 布尔值             | 指示该内容是否可能被识别为敏感内容。                                                                              | 研究某类内容的传播情况。                   |
| **promoted&#95;metrics**                         | 对象              | 在请求发出时，在推广场景下跟踪的该帖子的互动指标。需要用户上下文身份验证。                                                           | 衡量帖子在推广期间的互动情况。                |
| **public&#95;metrics**                           | 对象              | 请求发出时该帖子的公开互动指标。                                                                                | 衡量帖子的互动表现。                     |
| **referenced&#95;tweets**                        | 数组              | 此帖子所涉及的其他帖子列表，例如其转发、引用或回复的帖子。                                                                   | 用于理解转发等帖子的对话关系。                |
| **reply&#95;settings**                           | 字符串             | 显示谁可以回复给定的帖子。可选值为 &quot;everyone&quot;、&quot;mentioned&#95;users&quot; 和 &quot;followers&quot;。 | 用于确定该帖子的会话回复设置。                |
| **withheld**                                     | 对象              | 包含有关[受限制内容](https://help.x.com/en/rules-and-policies/tweet-withheld-by-country)的屏蔽详情。           |                                |
| **scopes**                                       | 对象              | 包含该帖子的可见性范围详细信息。                                                                                | 表明谁可以查看该帖子。仅对推广帖子返回。           |
| **media&#95;metadata**                           | 数组              | 包含帖子所附媒体的元数据。                                                                                   | 获取更多元数据，例如帖子中媒体附件的 `alt_text`。 |

**检索帖子对象**

**示例请求**

在下面的请求中，我们在 [Tweets lookup](/zh/resources/fundamentals/rate-limits) 端点上请求该帖子的字段。请务必将 `$BEARER_TOKEN` 替换为你自己生成的 [Bearer Token](/zh/resources/fundamentals/authentication)。

```bash
curl --request GET 'https://api.x.com/2/tweets?ids=1212092628029698048&tweet.fields=attachments,author_id,context_annotations,created_at,entities,geo,id,in_reply_to_user_id,lang,possibly_sensitive,public_metrics,referenced_tweets,text,withheld&expansions=referenced_tweets.id' --header 'Authorization: Bearer $BEARER_TOKEN'
```

<div id="user">
  ### User
</div>

user 对象包含 Twitter 用户账户元数据，用于描述被引用的用户。user 对象是在 [users lookup](/zh/x-api/users/lookup/introduction) 端点中返回的主要对象。在调用该端点时，如需请求额外的用户字段，只需使用字段参数 `user.fields`。

user 对象也可以作为子对象出现在帖子对象中，并在其中被展开。该对象可通过 `?expansions=author_id` 或 `?expansions=in_reply_to_user_id` 进行扩展，以获取仅包含默认字段的精简版对象。若要请求额外字段以完善该对象，请在使用扩展的同时配合字段参数 `user.fields`。

<div id="direct-message-events">
  ### 私信事件
</div>

私信（Direct Message，DM）会话是由一系列事件构成的。X API v2 目前支持三种事件类型：MessageCreate、ParticipantsJoin 和 ParticipantsLeave。

DM 事件对象由 [私信查找](/zh/x-api/direct-messages/lookup/introduction) 端点返回；当通过 [管理私信](/zh/x-api/direct-messages/lookup/introduction) 端点成功创建私信时，会生成一个 MessageCreate 事件。

在请求 DM 事件时，默认会包含三个事件对象属性（字段）：id、event&#95;type 和 text。要获取更多事件字段，请使用 [fields](/zh/x-api/fundamentals/fields) 参数 dm&#95;event.fields 来选择其他字段。其他可用的事件字段包括：dm&#95;conversation&#95;id、created&#95;at、sender&#95;id、attachments、participant&#95;ids 和 referenced&#95;tweets。

其中有若干字段会提供与该私信事件相关的其他 X 对象的 ID：

* sender&#95;id - 发送消息的账号 ID，或在群组会话中邀请参与者的账号 ID
* participant&#95;ids - 账号 ID 的数组。对于 ParticipantsJoin 和 ParticipantsLeave 事件，此数组将只包含一个创建该事件的账号 ID
* attachments - 提供发送方上传到 X 的内容的媒体 ID
* referenced&#95;tweets - 如果在 text 字段中发现帖子（Tweet）URL，则响应中会包含该帖子的 ID

可以使用 sender&#95;id、participant&#95;ids、referenced&#95;tweets.id 和 attachments.media&#95;keys 的 [expansions](/zh/x-api/fundamentals/expansions) 扩展字段来展开这些 X 对象 ID。

|     |     |     |     |
| :--- | :--- | :--- | :--- |
| **Field value** | **Type** | **Description** | **How it can be used** |
| id (default) | string | 事件的唯一标识符。<br /><br />&quot;id&quot;: &quot;1050118621198921728&quot; | 用于以编程方式检索特定会话事件（可通过 v1.1 端点获取）。 |
| event&#95;type (default) | string | 描述事件类型。目前支持三种类型： <br /><br />* MessageCreate<br />    <br />* ParticipantsJoin<br />    <br />* ParticipantsLeave<br />    <br /><br />&quot;event&#95;type&quot;: &quot;MessageCreate&quot; | 在检索会话历史时，用于了解消息是在何时创建的；对于群组会话，用于了解参与者在何时加入和离开。所有 GET 方法都支持使用 event&#95;type= 查询参数对特定事件类型进行过滤。 |
| text (default) | string | 私信的实际 UTF-8 文本。 <br /><br />&quot;text&quot;: &quot;Hello, just you!&quot; | 在聊天机器人场景中，可用于分析消息内容并确定自动回复。也可用于构建会话搜索功能。 |

<div id="community">
  ### Community（社区）
</div>

Community 是 X 用户用于交流、分享，并更贴近其最关心讨论话题的专属空间。

Community 中的帖子在 X 上任何人都可以看到，但只有该 Community 内的成员才能参与互动和讨论。

Community 对象包含与该 Community 相关的元数据。

| Field value      | Type            | Description                                        |    |
| :--------------- | :-------------- | :------------------------------------------------- | :- |

<div id="how-to-use-fields-and-expansions">
  ## 如何使用 fields 和 expansions
</div>

默认情况下，当在请求中未使用 [fields](/zh/x-api/fundamentals/fields) 或 [expansions](/zh/x-api/fundamentals/expansions) 参数时，X API v2 数据对象仅包含少量默认字段。本文将向你展示如何在请求中使用 `fields` 和 `expansions` 查询参数，从而在响应中获取更多对象和字段。

在本指南中，我们将以下面这张帖子的截图为例，请求其中的多个字段。
 

![该图片是一张由 @X 发布的帖子的截图。你可以看到帖子文本、用户名、发布时间（日期和时间）、来源以及公开指标。截图中还包含一个视频。](https://cdn.cms-twdigitalassets.com/content/dam/developer-twitter/ScreenShotDataDictionaryGuide.png.twimg.1920.png)

如你在截图中所见，与该帖子相关的可见信息有多项内容，包括帖子作者、帖子指标、创建时间戳、视频以及视频观看次数。此外，还有若干在截图中不可见、但仍可通过请求获取的数据。

当你向 API 发出请求时，默认的响应非常简单，只包含默认的帖子字段（id 和 text）。你还只会收到所使用端点返回的主对象，而不会收到与该主对象相关联的其他数据对象。

这种简洁性，再加上 fields 和 expansions 参数，使你能够根据具体用例，只请求你所需要的字段。
