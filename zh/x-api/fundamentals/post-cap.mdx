X API v2 具有帖子消耗上限，用于限制每月可从特定端点检索的帖子数量。帖子上限在[项目](/zh/resources/fundamentals/projects)级别生效，这意味着在该项目中使用[开发者应用](/zh/resources/fundamentals/developer-apps)的密钥和令牌对下方列出的端点发起的任何请求，都会计入该项目的帖子上限额度。

<div id="limited-v2-endpoints">
  ## 受限的 v2 端点
</div>

从以下任一端点接收的帖子都会计入当月的帖子上限：

* [帖子查询](/zh/x-api/posts/lookup/introduction)
* [最近搜索](/zh/x-api/posts/counts/introduction)
* [完整归档搜索](/zh/x-api/posts/counts/introduction)
* [Filtered stream](/zh/x-api/posts/filtered-stream)
* [Filtered Stream Webhooks](/zh/x-api/webhooks/stream/introduction)
* [用户帖子时间线](/zh/x-api/posts/timelines)
* [用户提及时间线](/zh/x-api/posts/timelines)
* [点赞查询 - 某用户点赞的帖子](/zh/x-api/posts/likes)
* [书签查询 - 某用户加入书签的帖子](/zh/x-api/posts/bookmarks/introduction)
* [List 帖子查询](/zh/x-api/lists/list-tweets/introduction)
* [Spaces 查询](/zh/x-api/spaces/lookup/introduction)

<Note>
  **注意：** 如果同一条帖子在一天内通过多个查询被多次返回，它在帖子上限中只计为一次；帖子会按天进行去重。
</Note>

<div id="post-cap-volumes">
  ## 帖子上限配额
</div>

帖子上限配额会根据你的访问层级而变化：

| 访问层级   | 每月帖子上限                |
|:---------------|:---------------------------------|
| **Free**      | 每月 100 条帖子           |
| **Basic**     | 每月 15,000 条帖子          |
| **Pro**       | 每月 1,000,000 条帖子       |
| **Enterprise**| 每月超过 5,000 万条帖子     |

你可以在[开发者门户](/zh/resources/fundamentals/developer-portal)中打开[主仪表板页面](https://developer.x.com/content/developer-twitter/en/portal/dashboard)，查看相对于每月帖子上限的使用情况。在你的项目名称下方，你会看到一条进度条，用于显示当月的使用量相对于帖子上限的情况，包括已获取的帖子总数、已使用帖子百分比以及你的使用量重置日期。
