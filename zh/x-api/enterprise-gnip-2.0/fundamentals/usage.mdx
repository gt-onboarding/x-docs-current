<div id="usage-api-twitter-api">
  ## 用量 API | Twitter API
</div>

<div id="overview">
  ### 概述
</div>

`Enterprise`

这是用于帮助你监控和管理账号下各个 API 使用情况的最重要、*最佳* 的工具。

**重要免责声明：**

由于试用和其他账单调整，Usage API 返回的使用统计数据可能与账单发票上的不一致。所有数值均基于在特定日期（以 UTC 计）内消耗的去重后活动量。

<div id="features">
  #### 功能
</div>

* 以编程方式检索在 console.gnip.com 界面中可用的用量数据
* 流级别用量数据 —— 除了产品级别之外，还提供流级别（例如 dev 和 prod）的用量数据
* 细粒度且具有描述性的用量数据 —— 搜索 “requests” 会按 Full-Archive 和 30-Day Search 产品进行拆分
* 历史 PowerTrack “days” 和 “jobs” 

<div id="supported-apis">
  ### 支持的 API
</div>

以下是 Usage API 当前支持的 API 列表：

* PowerTrack API `enterprise`
* 30-Day Search API `enterprise`
* Full-Archive Search API `enterprise`
* Historical PowerTrack `enterprise`

<div id="limitations">
  ### 限制
</div>

* Usage API 允许你访问自 2018 年 5 月 1 日以来的使用数据。自 2019 年 7 月 1 日起，Usage API 允许你访问**最近 13 个日历月**的使用数据

* 你可以通过 `fromDate` 和 `toDate` 定义的**三个月时间区间**来访问使用数据

请参见下方的请求和响应示例：

```bash
curl -u<username>:<password> \
"https://gnip-api.x.com/metrics/usage/accounts/<account-name>.json?bucket=month"
```

<div id="authentication-and-rate-limit">
  #### **身份验证与速率限制**
</div>

<div id="authentication">
  ##### **身份验证**
</div>

所有对 Usage API 的请求都必须使用 HTTP 基本身份验证。你可以使用你帐户中已启用的任意电子邮件/密码凭据（即用于登录 console.gnip.com 帐户或连接任意 Gnip 流的凭据）。

<div id="rate-limit">
  ##### **速率限制**
</div>

Usage API 的调用频率限制为每分钟最多 2 次请求。

<div id="best-practices-limitations">
  #### **最佳实践和限制**
</div>

<div id="data-availability">
  ##### **数据可用性**
</div>

使用量数据基于已去重的活动，这些活动来自最近一个已完成处理的完整时间周期（UTC）。除非 Gnip 正在部署系统，否则数据通常会被处理并实时更新到分钟级。

* Usage API 允许你访问自 2018 年 5 月 1 日起的使用量数据。从 2019 年 7 月 1 日起，Usage API 允许你访问**最近 13 个日历月**的使用量数据。 

* 你可以在由 fromDate 和 toDate 定义的**每三个月区间**内访问使用量数据

<div id="requesting-and-receiving-data">
  #### **请求和接收数据**
</div>

Usage API 的工作方式是，通过向与你的账户对应的 API 端点发出携带 HTTP BASIC-AUTH 凭证的 HTTP GET 请求来实现的。

<div id="get-request">
  ##### GET 请求：
</div>

使用你的用户凭证和账户名，向以下端点发起 GET 请求：

https://gnip-api.x.com/metrics/usage/accounts/:account&#95;name.json

**附加参数**

|     |     |
| :--- | :--- |
| bucket | 可选。提供使用情况数据所对应的时间单位。使用情况数据可以按天或按月粒度返回。  <br />  <br />未指定 bucket 的请求将返回按月粒度的数据。  <br />  <br />可选值：&#39;month&#39; 或 &#39;day&#39; |
| fromDate (YYYYMMDDHHMM) | 可选。使用情况数据仅从 2018 年 5 月 1 日起可用。此参数为提供使用情况数据的最早 UTC 时间戳。时间戳以天为粒度，且为包含关系（即 201805010000 包含 05 月 01 日当天）。在小时和分钟位置填写除 &#39;0000&#39; 以外数值的请求会被重置为 &#39;0000&#39;。  <br />  <br />未指定 fromDate 或 toDate 的请求将按月返回本月的使用情况数据，并包含前两个月的历史数据。  <br />  <br />**请注意：** 自 2019 年 6 月 1 日起，你可以访问过去 13 个日历月的使用情况数据。例如，如果当前日期是 10 月 10 日，你可以访问到上一年 9 月 1 日起的使用情况数据。  <br />**示例：** 201810010000 将返回从 2018 年 10 月 1 日起（含 10 月 1 日本日）的数据。 |
| toDate (YYYYMMDDHHMM) | 可选。此参数为提供使用情况数据的最新 UTC 时间戳。时间戳以天为粒度，且为不包含关系（即 201703020000 不包含 03 月 02 日当天的数据）。当将 toDate 指定为当前日期或将来日期时，使用情况数据将返回到最后一个完整日（UTC）。在小时和分钟位置填写除 &#39;0000&#39; 以外数值的请求会被重置为 &#39;0000&#39;。  <br />  <br />未指定 toDate 的请求将默认使用下一个 bucket（当 bucket=day 时为明天，当 bucket=month 时为下个月）。未指定 fromDate 和 toDate 的请求将默认 bucket=month，并返回当前月加上前两个月的数据。  <br />  <br />**示例：** 201703050000 将返回截至 2017 年 3 月 5 日的数据，但不包含任何来自 3 月 5 日当日的数据。 |

**GET 请求示例**

此请求将按月粒度返回从 2017 年 3 月 1 日到 2017 年 3 月 5 日的数据，但不包含 2017 年 3 月 5 日的任何数据。

curl -u &quot;https://gnip-api.x.com/metrics/usage/accounts/:account&#95;name.json?bucket=month&amp;fromdate=201403010000&amp;toDate=201403150000&quot;

<div id="data-format">
  #### **数据格式**
</div>

下表描述了 Usage API 返回的使用情况数据的根级数据结构。对于具有多级子字段的字段，请点击提供的链接以查看子字段的详细信息。

如果你想查看一份完整的 Usage API 负载示例，请访问[此页面](/zh/x-api/usage/introduction)。

|     |     |
| :--- | :--- |
| **account** | 表示已请求其使用情况数据的账号对象。 |
| **bucket** | 提供使用情况数据的时间单位，可以是“day”或“month”。 |
| **fromDate** | 你希望获取使用情况数据的最早 UTC 时间戳（包含该时间）。 |
| **toDate** | 你希望获取使用情况数据的最晚 UTC 时间戳（不包含该时间）。 |
| **publishers** | 包含三个主要对象：Used、Projected 和 Products。 |
