## 概览

> 企业控制台（位于 console.gnip.com）适用于已签订 Enterprise API 访问合同的企业数据客户。如果你目前还不是企业数据客户但希望了解更多信息，可以在此了解更多关于 [Enterprise 数据](https://developer.x.com/en/products/x-api/enterprise) 的内容。

本节概述了 X 的 Enterprise 控制台仪表板。以下各部分分别对应控制台中的具体区域，涵盖产品详情、使用情况以及常规账户管理。如果你对自己的账户有任何问题，请联系你指定的客户经理，或查阅我们的 [Enterprise 产品技术文档](/zh/x-api/enterprise-gnip-2.0/enterprise-gnip)。

在与你的客户经理进行 Enterprise 试用和入门流程的过程中，Enterprise 客户会获得 [console.gnip.com](http://console.gnip.com) 的登录信息，该站点是用于查看和配置 Enterprise 产品访问权限的用户界面。控制台中客户管理员的初始访问权限由 X 在 Enterprise 入门流程中进行配置，之后管理员可以添加更多用户。Enterprise 控制台允许你管理并访问与你的 Enterprise 产品相关的更多详细信息。

<div id="products-tab">
  ## 产品标签页
</div>

在 [console.gnip.com](http://console.gnip.com) 登录你的帐号后，你会进入控制面板的“产品”标签页。此页面包含当前在你帐号中可用的 Enterprise 产品概览。

对于使用流式传输（streaming delivery）的产品（例如 PowerTrack 或 Decahose），此页面会列出产品名称和流标签（stream label）、当前活动连接数、每个流当前启用的规则数量（如适用），以及最近 24 小时内传输的活动原始计数（例如帖子数量）。

对于采用 REST 传输的产品（例如 Search API），此页面会列出产品名称、标签（也显示为“streams”）、通过这些端点返回的当前活动数量（例如帖子数量），以及每个端点的若干不同类型的请求计数。

请注意，[Usage API](/zh/x-api/enterprise-gnip-2.0/fundamentals/usage) 可以以编程方式提供其中大部分数据。

点击名称或设置按钮可查看每个流的具体详情。

<div id="overview">
  ### 概览
</div>

在主仪表板中点击某个流，会进入该流的概览页面。

对于流式传输产品，此页面包括以下内容：

1. 显示通过每个特定流连接传输给你的活动数量的流量图表

2. 流上当前活动连接的详细信息（连接 ID 和 IP 地址）

3. 你的流最近发生的连接、断开连接以及规则更新事件的日志

请注意，可以使用右上角的链接来调整图表的刻度。你可以通过点击图表正下方图例中的相应项来切换单个连接和断开连接的可见性。

<div id="connections">
  ### 连接
</div>

Connections 页面提供你的流中最近连接事件的详细信息。这包括每次连接的开始和结束时间（24 小时制 UTC）、每次连接的持续时间、发起该连接的服务器 IP 地址、用于参考的唯一连接 ID，以及当前连接状态。状态对应于该连接最近一次发生的事件——例如 `Client Connected`，或一次断开连接事件，并标明断开连接的类型。有关连接调试的更多详细信息，请参阅 [断开连接详解指南](/zh/x-api/enterprise-gnip-2.0/powertrack-api#integrating-with-powertrack)。

<div id="rules">
  ### 规则
</div>

“规则”选项卡适用于 PowerTrack 流，并通过用户界面提供了一种快速入门方式，让你可以手动输入文本规则。请注意，该界面通过此手动方式最多只支持添加 1000 条规则，因此仅建议用于初始测试。我们建议在任何生产环境中通过 [Rules API](/zh/x-api/enterprise-gnip-2.0/powertrack-api#api-reference-index) 以编程方式管理你的规则。

<div id="settings">
  ### 设置
</div>

“Settings” 选项卡允许你切换流中数据的输出格式，并支持多种格式选项。要切换格式，只需使用对应不同选项的单选按钮。更改会在你重新连接到流时生效。请注意，此设置的更新会在下一次请求或连接时立即生效，并且可能因为新的格式而导致你现有的解析器无法正常工作。 

**请注意：** 若想获取尽可能多的数据，推荐的设置是 “Leave data in the original format”（保留数据的原始格式），该设置会以增强后的原生格式返回数据，详见[此处](/zh/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#post-object)。Activity Streams 格式说明见[此处](/zh/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#activity-object)。

<div id="usage-tab">
  ## Usage 选项卡
</div>

Usage 选项卡可帮助你了解在不同时间段内你的流的使用情况。要以编程方式访问用量信息，请参阅 [Usage API](/zh/x-api/enterprise-gnip-2.0/fundamentals/usage)。

<div id="monthly">
  ### 每月
</div>

“每月使用量”页面按产品细分展示你的流使用情况。比如，针对某个给定的数据源，覆盖类产品（例如 PowerTrack、Historical PowerTrack 和 PowerTrack Replay）会被归为一组，既提供各自的数据，也提供合并汇总的计数。这些计数包括本月至今的使用量、预估月末值（基于本月目前的使用情况以及本月剩余时间）以及前两个月的计数。

需要注意的是，这些计数在每个产品和每条流内都会进行去重。如果你通过多个连接从同一条 PowerTrack 流接收到了相同的帖子，那么在这些统计中该帖子只会被计数一次。计数每 24 小时在 00:00 UTC 更新一次。

<div id="daily">
  ### 每日
</div>

“每日使用量”页面按产品分类，提供当月每一天的去重计数。计数每 24 小时于 00:00（UTC）更新一次。

<div id="rules">
  ### 规则
</div>

规则数量上限基于你所签约的 PowerTrack 规则套餐，并适用于给定「类型」的所有流。每当添加或删除规则时，规则数量都会更新。

<div id="account-tab">
  ## 账号选项卡
</div>

<div id="my-profile">
  ### 我的个人资料
</div>

在这里查看你的个人资料详细信息，并更改密码。此外，你还可以为自己的账号设置专属的用量阈值通知。API 状态通知可在状态页面 https://api.twitterstat.us 上查看。

<div id="account-settings">
  ### 账户设置
</div>

您可以添加、删除和编辑用户，并为各个用户以及用量阈值警报配置电子邮件通知。

<Frame>
  <img src="/images/account-settings.png.twimg.1920.png" />
</Frame>

请注意，有三种类型的用户 —— Account Admin、User 和 Email Only。  

* Account Admin 可以创建、删除和编辑其他用户，并且可以使用用户名（电子邮件）和密码进行基本身份验证，以连接到 Enterprise API

* User 无法创建或修改其他用户，但可以使用用户名（电子邮件）和密码进行基本身份验证，以连接到 Enterprise API

* Email Only 用户无法访问仪表板，无权连接到 Enterprise API，并且只有在在 Notifications 部分配置为接收通知时才会收到通知。

<div id="usage-thresholds">
  ### 用量阈值
</div>

为你的产品配置用量阈值。对于在个人资料中配置了相关通知的用户，当每个产品的用量达到对应的预警阈值或严重阈值时，将触发电子邮件提醒。

<Frame>
  <img src="/images/usage-thresholds.png.twimg.1920.png" />
</Frame>

请注意，这些阈值每天在 19:30（UTC）统一评估一次，而不是实时评估。
