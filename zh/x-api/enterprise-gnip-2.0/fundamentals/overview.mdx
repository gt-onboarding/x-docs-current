## 概览

> 企业控制台（位于 console.gnip.com）适用于已签订 Enterprise API 访问合同的企业级数据客户。如果您目前还不是企业级数据客户但希望了解更多信息，可以在此处了解更多关于[Enterprise 数据](https://developer.x.com/en/products/x-api/enterprise)的内容。

本文概述了 X 的 Enterprise 控制台仪表板。以下各部分分别对应控制台中与产品详情、使用情况以及通用账号管理相关的具体区域。如果您对自己的账号有任何疑问，请联系您指定的客户经理，或查阅我们的[Enterprise 产品技术文档](/zh/x-api/enterprise-gnip-2.0/enterprise-gnip)。

作为您与客户经理一起进行的 Enterprise 试用和接入流程的一部分，Enterprise 客户将获得 [console.gnip.com](http://console.gnip.com) 的登录账号，该站点是用于查看和配置 Enterprise 产品访问权限的用户界面。客户管理员的初始控制台访问权限由 X 在 Enterprise 接入流程中完成设置，之后管理员可以添加其他用户。Enterprise 控制台使您能够管理并查看与您的 Enterprise 产品相关的更多详细信息。

<div id="products-tab">
  ## 产品选项卡
</div>

在 [console.gnip.com](http://console.gnip.com) 登录你的账户后，你将进入仪表板中的“产品”选项卡。此页面包含你账户当前可用 Enterprise 产品的概览。

对于使用流式传输（streaming delivery）的产品，例如 PowerTrack 或 Decahose，此页面会列出产品名称和流标签（stream label）、当前活动连接数量、每个流当前生效规则的数量（如适用），以及在最近 24 小时内交付的活动（例如帖子）的原始数量。

对于使用 REST 方式交付的产品，例如 Search API，此页面会列出产品名称、标签（也显示为“streams”）、通过这些端点返回的当前活动（例如帖子），以及每个端点若干不同的请求计数。

请注意，[Usage API](/zh/x-api/enterprise-gnip-2.0/fundamentals/usage) 可以以编程方式提供其中大部分数据。

通过点击名称或设置按钮，可以查看每个流的具体详情。

<div id="overview">
  ### 概览
</div>

在主仪表盘中点击某个流，会进入该流的概览页面。

对于流式投递产品，此页面包括以下内容：

1. 显示通过每个特定流连接传递给你的活动数量的图表

2. 当前在该流上处于活动状态的连接的详细信息（connection id 和 IP 地址）

3. 该流的近期连接、断开连接以及规则更新事件日志

请注意，你可以使用右上角的链接调整图表的显示范围。你也可以通过点击图表正下方图例中的相应条目来控制单个连接和断开事件的显示或隐藏。

<div id="connections">
  ### 连接
</div>

“Connections” 页面提供你的流中最近连接事件的详细信息。这包括每个连接的开始和结束时间（UTC 24 小时制）、每个连接的持续时间、发起连接的服务器 IP、供参考的唯一连接 ID，以及当前连接状态。状态对应于该连接最近一次事件——即 Client Connected，或者一次断开连接事件，并指明断开连接的类型。有关连接调试的更多详情，请参阅[“断开连接详解”指南](/zh/x-api/enterprise-gnip-2.0/powertrack-api#integrating-with-powertrack)。

<div id="rules">
  ### 规则
</div>

Rules 选项卡适用于 PowerTrack 流，并通过用户界面提供了一种快速入门方式，让你可以手动输入纯文本规则。请注意，该界面在这种手动方式下最多只支持添加 1000 条规则，因此仅应在初始测试时使用。我们建议在任何生产环境中，通过 [Rules API](/zh/x-api/enterprise-gnip-2.0/powertrack-api#api-reference-index) 以编程方式管理你的规则。

<div id="settings">
  ### 设置
</div>

“Settings” 选项卡允许你切换流中数据的输出格式，支持多种格式选项。要切换格式，只需使用不同选项对应的单选按钮。更改会在你重新连接到流时生效。请注意，更新此设置会在下一次请求或连接时立即生效，并且由于使用了新格式，可能会导致你的解析器出错。 

**请注意：** 若要获取尽可能多的数据，推荐的设置是 “Leave data in the original format”，该设置会以增强的原生格式返回数据，详见[此处](/zh/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#post-object)。Activity Streams 格式的说明见[此处](/zh/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#activity-object)。

<div id="usage-tab">
  ## Usage 选项卡
</div>

Usage 选项卡可帮助你了解在不同时间段内对数据流的使用情况。若要以编程方式访问用量信息，请参阅 [Usage API](/zh/x-api/enterprise-gnip-2.0/fundamentals/usage)。

<div id="monthly">
  ### 每月
</div>

“每月使用情况”页面按产品展示你的流使用情况。比如，对于某个给定数据源的覆盖类产品（例如 PowerTrack、Historical PowerTrack 和 PowerTrack Replay），会被归为一组，既提供各自独立的数据，又提供合并汇总的计数。计数包括：当前月份截至目前的用量、本月预计月末用量（基于本月迄今的使用情况以及本月剩余时间），以及前两个月的计数。

需要特别注意的是，这些计数会在每个产品和每条流内分别去重。如果你通过多个与同一 PowerTrack 流的连接收到了相同的帖子，则在此处该帖子只会计数一次。这些计数会每 24 小时在 00:00 UTC 更新一次。

<div id="daily">
  ### 每日
</div>

“每日使用量”页面提供当月每一天的去重计数，并按产品进行细分。计数会在每天 00:00（UTC）更新一次。

<div id="rules">
  ### 规则
</div>

规则数量限制基于您所签约的 PowerTrack 规则套餐，并适用于给定“类型”的所有流。每当添加或删除规则时，该数量都会更新。

<div id="account-tab">
  ## 账户标签页
</div>

<div id="my-profile">
  ### 我的个人资料
</div>

在这里查看你的个人资料详细信息，并修改密码。此外，你还可以为你的账户配置特定的账户使用量阈值通知。API 状态通知可通过状态页面 https://api.twitterstat.us 获取。

<div id="account-settings">
  ### 账号设置
</div>

可以添加、移除和编辑用户，并为单个用户以及用量阈值警报配置电子邮件通知。

<Frame>
  <img src="/images/account-settings.png.twimg.1920.png" />
</Frame>

请注意，有三种类型的用户——Account Admin、User 和 Email Only。

* Account Admin 可以创建/删除/编辑其他用户，并且可以使用带用户名（电子邮件）和密码的基本身份验证连接到 Enterprise APIs。

* User 无法创建或修改其他用户，但可以使用带用户名（电子邮件）和密码的基本身份验证连接到 Enterprise APIs。

* Email Only 用户无法访问控制面板，无权连接到 Enterprise APIs，仅在在 Notifications 部分中被配置为接收通知时才会接收通知。

<div id="usage-thresholds">
  ### 用量阈值
</div>

为你的产品配置用量阈值。对于在个人资料中启用了相关通知的用户，当每个产品达到预警阈值或关键阈值时，系统都会触发电子邮件提醒。

<Frame>
  <img src="/images/usage-thresholds.png.twimg.1920.png" />
</Frame>

请注意，这些阈值每天在 19:30（UTC）评估一次，不会进行实时评估。
