<div id="getting-started-with-enterprise-rules-and-queries">
  ## 入门：Enterprise 规则和查询
</div>

使用 Enterprise 运算符的产品会基于你设置的过滤规则向你提供社交数据。规则由一个或多个“子句”组成，其中子句可以是关键词、精确短语，或众多 Enterprise 运算符之一。在开始使用 Enterprise 运算符构建规则之前，请务必先查看下文所述的语法，浏览可用运算符列表，并理解构建规则时的各项限制。你还需要了解规则在逻辑评估方式上的细微差别，可参见“[运算顺序](#orderofoperations)”一节。

多个子句可以同时使用 &quot;and&quot; 和 &quot;or&quot; 逻辑进行组合。

**请注意：** “and” 逻辑通过在子句之间使用空格来指定，而 or 逻辑通过使用大写的 `OR` 来指定。

每条规则最多可包含 2,048 个字符，对正向子句（你希望匹配或用于过滤的内容）和负向子句（你希望排除且不匹配的内容）的数量不设上限。
 

<div id="building-rules-and-queries">
  ### 构建规则和查询
</div>

**关键词匹配**

关键词匹配类似于搜索界面中的查询。例如，下面的 Enterprise 运算符规则会匹配正文中包含术语 &quot;social&quot; 的活动。

`social`

**用空格实现 AND 连接**

添加另一个关键词，相当于为匹配增加一个额外条件。例如，这条规则只会匹配同时包含 &quot;social&quot; 和 &quot;media&quot; 的活动，顺序不限 —— *在词项之间使用空格的行为等同于布尔 AND 逻辑*。如果你在规则中显式包含 AND，它会被 rules 端点拒绝。

`social media`

**使用大写 OR 实现 OR 连接**

不过，很多情况下实际上需要布尔 OR 逻辑。这也很容易实现。*请注意，OR 运算符必须为大写，使用小写的 ‘or’ 将被视为普通关键词。*

`social OR data`

**取反（排除）词项**

还有一些场景需要排除带有某些关键词的结果（布尔 NOT 逻辑）。例如，包含 ‘happy’ 的活动，但排除正文中包含 ‘birthday’ 的任何活动。

`social -personality`

**使用括号进行分组**

这些类型的逻辑可以通过使用括号进行分组来组合，并扩展为更加复杂的查询。

`(social OR data) (academic OR research) -personality -information -university`

不过这只是开始 —— 上面的示例仅依赖于对关键词进行分词匹配，而 Enterprise 产品还提供运算符，用于在文本上执行不同类型的匹配。

**精确匹配**

`"social media research"`

**子串匹配**

`contains:info`

**近邻匹配**

`"social media research"~3`

此外，其他运算符允许你在文本之外，基于社交数据的一些独特属性进行过滤。 

**发布帖子的用户**

`from:XDeveloeprs`

**距离美国科罗拉多州博尔德市 Pearl St. 10 英里范围内的带地理标签的帖子**

`point_radius:[-105.27346517 40.01924738 10.0mi]`

**综合运用**

这些可以按照上文描述的相同类型逻辑，与文本过滤组合使用。

`(social OR data) (academic OR research OR "social media research") point_radius:[-105.27346517 40.01924738 10.0mi] lang:en -personality -information -university`

<div id="boolean-syntax">
  ### 布尔语法
</div>

前一节中的示例使用了多种类型的布尔逻辑和分组。请参阅下表，了解每种语法和要求的更多详细信息。

|     |     |     |
| :--- | :--- | :--- |
| **逻辑类型** | **运算符语法** | 说明 |
| AND | social data | 两个运算符之间的空格表示它们之间为 AND 逻辑关系。  <br />  <br />匹配同时包含两个关键词（&quot;social&quot;、&quot;data&quot;）的活动。  <br />  <br />**不要在规则中显式使用 AND。只使用空格。显式写出的 AND 会被当作普通关键词处理。** |
| OR  | social OR data | 要将两个运算符进行 OR 运算，请在它们之间插入用空格分隔的全大写 OR。  <br />  <br />匹配包含任一关键词（&quot;social&quot; 或 &quot;data&quot;）的活动。  <br />  <br />请注意，如果你在单个规则中同时使用 OR 和 AND 逻辑，应理解我们“[运算顺序](#OrderOfOperations)”部分所描述的运算顺序，并考虑按如下所述使用括号将未取反的运算符分组在一起，以确保规则按预期运行。  <br />  <br />你必须在规则中使用大写的 &quot;OR&quot;。小写的 &quot;or&quot; 会被当作普通关键词处理。 |
| NOT | social data  <br />-apple -android -phone | 在运算符或运算符组前面紧接着插入一个 `-` 字符。  <br />  <br />上述示例规则匹配包含关键词 &quot;social&quot; 的活动，但排除包含关键词 &quot;data&quot; 的活动。  <br />  <br />不允许使用那种会请求“firehose 中除了取反部分之外的所有内容”的被取反 OR 表达式。例如，`apple OR -ipad` 是无效的，因为它将匹配所有不包含 &quot;ipad&quot; 的活动。 |
| 分组 | (social OR data) -XDeveloeprs -api | 将多个运算符置于括号内会创建一个功能性“分组”。<br /><br />分组可以像单个子句一样，通过空格（AND）或 OR 与其他子句连接。但请注意，最佳实践是不通过在整个分组前添加取反符号 - 来对一组条件整体取反。相反，你应当对每个单独的运算符进行取反，并通过空格（AND）将它们串联起来。<br /><br />例如，不要使用 -(iphone OR imac OR macbook)，而应使用以下形式：-iphone -imac -macbook。  <br />  <br />在单个规则中同时组合 AND 和 OR 逻辑时，分组尤为重要，因为评估规则时会使用特定的运算顺序。详见下文。 |

**请注意：** 运算符可以是正向的，也可以是负向的。

**正向运算符** 定义你希望在结果中**包含**的内容。例如，`has:hashtags` 运算符表示“我要包含话题标签（hashtags）的活动”。

**负向运算符** 定义你希望从结果中**排除**的内容，并通过使用上文所述的布尔 NOT 逻辑来创建。例如，`-has:hashtags` 表示“排除任何包含话题标签的活动，即使它们在其他方面符合我的规则”。

Premium 运算符产品在正向和负向子句的数量上没有限制，但规则的最大长度为 2,048 个字符。
 

<div id="order-of-operations">
  #### 运算顺序
</div>

当在单个规则中组合 AND 和 OR 逻辑时，将按照以下运算顺序来解析你的规则：

1. 先组合由 AND 逻辑连接的运算符
2. 然后再应用由 OR 逻辑连接的运算符

**示例：**

* `apple OR iphone ipad` 会被解析为 `apple OR (iphone ipad)`
* `ipad iphone OR android` 会被解析为 `(iphone ipad) OR android`

为了消除不确定性，并确保规则按照预期进行解析，请在合适的地方使用括号将词项分组。例如：

* `(apple OR iphone) ipad`
* `iphone (ipad OR android)`

<div id="punctuation-diacritics-and-case-sensitivity">
  #### 标点符号、变音符号和大小写敏感性
</div>

如果你在 Enterprise 运算符中指定带有重音或变音符号的关键词或话题标签规则，它会仅匹配帖子文本中同样带有这些变音符号的内容（话题标签或关键词）。例如，包含关键词 `Diacr**í**tica` 或话题标签 `#cumplea**ñ**os` 的规则，会匹配 &quot;Diacr**í**tica&quot; 或 &quot;#cumplea**ñ**os&quot;，但不会匹配不含波浪符 **í** 或 **eñe** 的 &quot;Diacritica&quot; 或 &quot;#cumpleanos&quot;。

带有重音或变音符号的字符与普通字符一视同仁，不会被视作词边界。例如，规则 cumplea**ñ**os 只会匹配包含单词 cumpleaños 的活动数据，而不会匹配包含 cumplea、cumplean 或 os 的活动数据。

所有运算符在评估时都不区分大小写。例如，规则 `Cat` 会匹配以下所有内容：&quot;cat&quot;、&quot;CAT&quot;、&quot;Cat&quot;。

<div id="powertrack-rule-tags">
  #### PowerTrack 规则标签
</div>

如我们“[Matching rules](/zh/x-api/enterprise-gnip-2.0/fundamentals/data-enrichments#matching-rules)”页面所述，每条规则在创建时都可以附带一个标签。这些标签本身不会影响过滤结果，但可用于在你的应用中对规则进行逻辑分组。每条规则最多只能有一个标签，且标签长度上限为 255 个字符。在通过 API 创建规则时，标签会作为 JSON 格式规则的一部分一同提交，具体如我们的“Matching rules”页面所述。

<div id="putting-rules-in-json-format">
  #### 将规则表示为 JSON 格式
</div>

要通过 API 向流中添加或删除规则，规则必须使用 JSON 格式表示。基本上，这意味着需要将每条规则写成以下结构：

`{"value":"insert_rule_here"}`

**包含双引号的规则**

如果规则中包含用于精确匹配或其他运算符的双引号字符（`"`），则必须使用反斜杠进行转义，以便将它们与 JSON 格式本身的结构区分开来。

`"social data" @XDevelopers`

对应的 JSON 格式规则如下：

`{"value":"\"social data\" @XDevelopers"}`

**包含双引号字符串字面量的规则**

如果要在精确匹配中将双引号字符作为字符串字面量包含在内，则必须进行双重转义。比如，对于匹配精确短语 &quot;Toys “R” Us&quot; 的规则（包括围绕 &quot;R&quot; 的双引号），其纯文本表示如下：

`"Toys \"R\" Us"`

将其转换为 JSON 格式时，应使用以下结构：

`{"value":"\"Toys \\\"R\\\" Us\""}`

**带有标签的规则**

要在规则中包含一个可选标签（如上所述），只需在规则值旁边添加一个 `tag` 字段即可。

`{"value":"\"social data\" @XDevelopers","tag":"RULE-TAG-01"}`

**API 请求的格式**

通过 API 向流中添加或删除规则时，多个 JSON 格式的规则应使用逗号分隔，并放在一个 JSON `rules` 数组中，如下所示：

`{"rules":[{"value":"from:XDevelopers"},{"value":"\social data\" @XDevelopers","tag":"RULE-TAG-01"}]}`

<div id="operators-that-match-quote-tweets">
  #### 匹配引用帖子的运算符
</div>

使用 [PowerTrack API](/zh/x-api/enterprise-gnip-2.0/powertrack-api) 和 Historical PowerTrack API 时，以下运算符会同时匹配被引用的原始帖子内容和新的引用帖子内容。

但是，如果你使用的是 [Search API](/zh/x-api/enterprise-gnip-2.0/fundamentals/search-api)，这些运算符只会匹配引用帖子的内容，而不会匹配任何被引用的原始帖子内容。

* `Keywords`
* `Phrases`
* `Proximity`
* `#hashtags`
* `@mentions`
* `$cashtags`
* `url:`
* `url_contains:`
* `has:links`
* `has:mentions`
* `has:hashtags`
* `has:media`
* `has:symbols`
* `is:quote`
* `is:reply`

<div id="enterprise-operators">
  ## Enterprise 运算符
</div>

以下是 PowerTrack 和 Historical PowerTrack 可用的运算符。其中一部分运算符也可用于 30 天和 Full-Archive 搜索 API。有关各产品可用运算符的列表，请参阅[此表](/zh/x-api/enterprise-gnip-2.0/fundamentals/rules-filtering#operators-by-product)。 

| 运算符                                    | 说明                                                                                                                                                                                                                                                                               |
| :------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| keyword                                | 匹配帖子正文或 URL 中的关键词。关键词必须以数字 (0-9) 或任何非标点字符开头。<br />关键词匹配是基于分词的，这意味着会将关键词与帖子正文分词后的文本进行匹配。<br />对于包含标点符号的字符串（例如 &quot;coca-cola&quot;），请使用带引号的 &quot;精确短语匹配（exact phrase match）&quot;。<br />*示例*：`(social OR pizza OR wildfire) -planet`                                            |
| emoji                                  | 在帖子正文中匹配表情符号，使用基于标点符号、符号/表情符号和分隔符字符的分词匹配。<br />如果某个表情符号存在变体，请使用引号来实现精确匹配。<br />*示例*: `(🍕 OR 💜 OR 🐢) -🤖`                                                                                                                                                                      |
| &quot;精确短语匹配&quot;                     | 匹配帖子正文中的精确短语。标点符号会被视为空白字符。<br />*示例*：`("social media" OR "developer.x.com" OR "wildfire911" OR "coca-cola") -"planet earth"`                                                                                                                                                     |

| @                                      | 匹配任何提及指定用户名的帖子。<br />*示例*：`(@XDevelopers OR @api OR @twittereng) -@jack`                                                                                                                                                                                                         |

| followers&#95;count:                   | 当作者的粉丝数在指定范围内时匹配帖子。<br />如果只指定一个数字，则匹配所有大于或等于该数字的情况。<br />可以指定一个范围以匹配该范围内的任意数字（例如，`followers_count:1000..10000`）。<br />*注意：* 仅适用于 PowerTrack 和 Historical PowerTrack。                                                                                                            |
| friends&#95;count:                     | 当作者的好友计数（其关注的用户数量）在给定范围内时，匹配该作者的帖子。<br />如果指定的是单个数字，则会匹配任何大于或等于该数字的值。<br />可以指定一个范围来匹配该范围内的任意数字（例如：`friends_count:1000..10000`）。<br />*注意：* 仅在 PowerTrack 和 Historical PowerTrack 中可用。                                                                                           |
