[X 列表](https://help.x.com/en/using-twitter/twitter-lists) 允许用户自定义、整理和优先查看时间线中的帖子。通过 Lists 端点，你可以构建解决方案，帮助人们根据偏好、兴趣、群组或主题来策划和组织帖子。

由于你是在代表某个用户调用 manage List 端点，你必须使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 或 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 对这些端点进行认证，并使用与已授权你应用的用户关联的用户访问令牌。这些访问令牌可以通过 [三方 OAuth 流程](/zh/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（OAuth 1.0a）或 [Authorization Code with PKCE 授权流程](/zh/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)（OAuth 2.0）生成。

<div id="manage-lists">
  ### 管理列表
</div>

列表管理端点允许你代表已通过认证的用户创建、删除和更新列表。对于这些端点，有三种可用的 HTTP 方法：POST、DELETE 和 PUT。POST 方法用于创建列表，DELETE 方法用于删除列表，PUT 方法用于更新列表的元数据。对于这三个端点，每个用户在每 15 分钟内的速率限制为 300 个请求。

请注意，每个账号最多可以创建 1000 个列表。

**账号设置**

要访问这些端点，你需要：

* 一个已获批准的[开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)。
* 使用位于某个[项目](/zh/resources/fundamentals/projects)中的[开发者应用](/zh/resources/fundamentals/developer-apps)的密钥和令牌进行认证。 

在我们的[入门指南](/zh/x-api/getting-started/getting-access)中了解更多关于获取 X API v2 端点访问权限的信息。
