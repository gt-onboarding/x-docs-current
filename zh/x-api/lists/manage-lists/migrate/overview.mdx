<div id="comparing-x-apis-lists-endpoints">
  ## 比较 X API 的列表端点
</div>

v2 的列表管理端点最终会取代 [POST lists/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-create)、[POST lists/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-destroy) 和 [POST lists/update](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-update)。如果你已有使用这些旧版端点的代码、应用或工具，并且正在考虑迁移到更新的 X API v2，那么本指南适合你。 

下表比较了标准 v1.1 和 X API v2 的列表端点：

**创建列表**

| 描述 | 标准 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 支持的 HTTP 方法 | POST | POST |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /1.1/lists/create.json | /2/lists |
| [Authentication](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文 | OAuth 1.0a 用户上下文 |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 无 | 每 15 分钟 300 个请求（每个用户） |

**删除列表**

| 描述 | 标准 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 支持的 HTTP 方法 | POST | DELETE |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /1.1/lists/destroy.json | /2/lists/:id |
| [Authentication](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文 | OAuth 1.0a 用户上下文 |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 无 | 每 15 分钟 300 个请求（每个用户） |

**更新列表**

| 描述 | 标准 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 支持的 HTTP 方法 | POST | PUT |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /1.1/lists/update.json | /2/lists/:id |
| [Authentication](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文 | OAuth 1.0a 用户上下文 |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 无 | 每 15 分钟 300 个请求（每个用户） |

要访问 X API v2 端点，你必须先[注册开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)。进行认证时，你必须使用某个[项目](/zh/resources/fundamentals/projects)中的[开发者应用](/zh/resources/fundamentals/developer-apps)所提供的密钥和令牌。 

在我们的[入门](/zh/x-api/getting-started/getting-access)页面了解更多关于获取 X API v2 端点访问权限的信息。
