<div id="getting-started-with-the-manage-lists-endpoint-group">
  ## 开始使用管理 List 端点组
</div>

本快速概览将帮助你使用 [Postman](/zh/tutorials/postman-getting-started) 向管理 List 的端点发出你的第一个请求。

如果你希望查看不同语言的示例代码，请访问我们的 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 仓库。

**注意：**在本示例中，我们将向 *Create a List* 端点发出请求，但你也可以将本快速入门中的方法应用到其他管理 List 的请求中。

<Note>
  **先决条件**

  要完成本指南，你需要一组 [keys and tokens](/zh/resources/fundamentals/authentication)，用于对请求进行认证。你可以通过以下步骤生成这些密钥和令牌：

  * [注册开发者账号](https://developer.x.com/en/apply-for-access) 并获得批准。
  * 在开发者门户中创建一个 [Project](/zh/resources/fundamentals/projects) 以及关联的 [developer App](/zh/resources/fundamentals/developer-apps)。
  * 前往你的应用的 “Keys and tokens” 页面生成所需凭证。请务必将所有凭证保存在安全的位置。
</Note>

<div id="steps-to-build-a-manage-list-request">
  ### 构建管理列表请求的步骤
</div>

**第一步：从工具或库开始**

有多种不同的工具、代码示例和库可以用来向此端点发起请求，但在这里我们将使用 Postman 工具来简化流程。

要将 X API v2 的 Postman 集合加载到你的环境中，请点击以下按钮：

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  将 X API v2 添加到 Postman
</Button>

在将 X API v2 集合加载到 Postman 之后，导航到 “List” 文件夹，选择其中的 “Manage List” 文件夹，然后选择 &quot;Create a List&quot;。

**第二步：对你的请求进行身份验证**

要正确地向 X API 发起请求，你需要确认自己是否拥有相应的权限。对于 manage Posts 端点，你必须使用 [OAuth 1.0a User Context](/zh/resources/fundamentals/authentication) 或 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 对请求进行身份验证。

在本示例中，我们将使用 OAuth 1.0a User Context。

你必须将你的密钥和令牌（具体来说是 API Key、API Secret Key、OAuth 1.0a 用户访问令牌（Access Token）和 OAuth 1.0a 用户访问令牌密钥（Access Token Secret））添加到 Postman 中。你可以通过选择名为 “X API v2” 的环境（位于 Postman 右上角），然后将你的密钥和令牌添加到 &quot;initial value&quot; 和 &quot;current value&quot; 字段中（点击环境下拉菜单旁边的眼睛图标）来完成此操作。

如果你已正确完成这些操作，这些变量将会自动被填入请求的 Authorization（授权）选项卡中。
 
**第三步：为新列表指定名称**

使用此端点创建新列表时，列表名称是一个必填的请求体参数。你还可以选择性地提供描述，并指定该列表是否为私有。

在 Postman 中，导航到 “Body” 选项卡，并将列表的名称作为 name 参数的值输入。此外，如果你希望为列表添加描述，只需以与 name 相同的方式添加一个名为 description 的新键，并将列表的描述作为其值。将列表设置为私有遵循相同的模式，但此参数只接受 true 或 false 值。 

|             |                                     |                    |
| :---------- | :---------------------------------- | :----------------- |
| **Key**     | **Value**                           | **Parameter type** |
| `name`      | Name of the list (required)         | body               |
| description | Description for the list (optional) | body               |
| private     | true or false (optional)            | body               |

此时，你应该会在 &quot;Send&quot; 按钮旁边看到类似如下的 URL：

`https://api.x.com/2/lists`

Step four: Make your request and review your response

一切准备就绪后，点击 &quot;Send&quot; 按钮，你将会收到类似于以下示例响应的结果：

```
{
  "data": {
    "id": "1441162269824405510",
    "name": "New list created from Postman"
  }
}
```

如果返回的响应对象中同时包含一个 id 和你的列表名称，则表示你已成功创建该列表。 

若要删除列表，请在已加载到 Postman 中的 X API v2 集合的“Lists”文件夹中选择“Delete a List”请求。此端点需要你想要删除的列表 ID。在“Params”选项卡中，将你想要删除的列表 ID 填入 id 列的值。 

当删除请求成功时，你会收到类似于以下示例的响应：

```
{
  "data": {
    "deleted": true
  }
}
```
