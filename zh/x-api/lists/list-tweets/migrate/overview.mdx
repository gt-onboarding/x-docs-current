  ## 比较 X API 的 List 帖子查询端点
</div>

v2 List 帖子查询端点将取代标准 v1.1 的 [GET lists/statuses](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-statuses)。如果你有使用此版本端点的代码、应用或工具，并且正在考虑迁移到更新的 X API v2 端点，那么本指南集适合你。

下表比较了标准 v1.1 和 X API v2 的 List 端点：

**按 ID 查询 List**

|     |     |     |
| :--- | :--- | :--- |
| 描述 | 标准 v1.1 | X API v2 |
| 支持的 HTTP 方法 | `GET` | `GET` |
| 主机域名 | `https://api.x.com` | `https://api.x.com` |
| 端点路径 | `/1.1/lists/statuses.json` | `/2/lists/:id/tweets` |
| [Authentication](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文<br /><br />仅应用 | OAuth 1.0a 用户上下文<br /><br />OAuth 2.0 Authorization Code with PKCE<br /><br />仅应用 |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 使用 OAuth 1.0a 时每 15 分钟 900 次请求<br /><br />仅应用时每 15 分钟 900 次请求 | 使用 OAuth 1.0a 时每 15 分钟 900 次请求<br /><br />使用 OAuth 2.0 时每 15 分钟 900 次请求<br /><br />仅应用时每 15 分钟 900 次请求 |

要访问 X API v2 端点，你必须先[注册开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)。进行认证时，你必须使用位于某个[项目](/zh/resources/fundamentals/projects)中的[开发者应用](/zh/resources/fundamentals/developer-apps)的密钥和令牌。

请参阅我们的[入门](/zh/x-api/getting-started/getting-access)页面，了解更多关于获取 X API v2 端点访问权限的信息。
