<div id="list-members-lookup">
  ### 列表成员查找
</div>

列表成员查找接口组提供两个可用的端点。你可以获取指定列表成员的详细信息，以及查看某个用户是哪些列表的成员。这些端点可用于基于成员信息帮助人们策划和整理新的列表。

在查找成员详细信息时，每 15 分钟最多可发出 900 次请求；在查找用户所属列表时，每 15 分钟最多可发出 75 次请求。

你可以使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication)、[仅应用（App only）](https://developer.x.com/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)、或 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 来对发送到此端点的请求进行身份验证。 
 

<div id="manage-list-members">
  ### 管理列表成员
</div>

管理列表成员的端点允许你代表经过身份验证的用户向某个列表添加和移除成员。对于这些端点，有两种可用的方法：POST 和 DELETE。POST 方法允许你向列表添加成员，DELETE 方法允许你从列表中移除成员。对于这两个端点，单个用户的速率限制为每 15 分钟最多 300 个请求。

请注意，列表的成员数量不能超过 5,000 个。

由于你是使用这些端点代表某个用户发起请求，因此必须使用 [OAuth 1.0a User Context](/zh/resources/fundamentals/authentication) 或 [OAuth 2.0 Authorization Code with PKCE](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 对其进行身份验证，并使用与已授权你的应用的用户关联的访问令牌。你可以通过 [3-legged OAuth flow](/zh/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（三方 OAuth 流程，适用于 OAuth 1.0a）或 [Authorization Code with PKCE grant flow](/zh/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)（OAuth 2.0）生成这些访问令牌。

<Note>
  **账号设置**

  要访问这些端点，你需要：

  * 已获批准的[开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)。
  * 使用位于某个[项目](/zh/resources/fundamentals/projects)中的[开发者应用](/zh/resources/fundamentals/developer-apps)的密钥和令牌进行身份验证。

  在我们的[入门指南](/zh/x-api/getting-started/getting-access)中了解更多关于如何获取 X API v2 端点访问权限的信息。
</Note>
