<div id="comparing-x-apis-list-members-endpoints">
  ## 比较 X API 的 List 成员端点
</div>

v2 List 成员端点组将取代 v1.1 中标准的 [GET lists/members](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-members)、[GET lists/memberships](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-memberships)、[POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create) 和 [POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy) 端点。如果你已有使用上述任一 List 成员端点版本的代码、应用或工具，并且正在考虑迁移到较新的 X API v2 端点，那么本系列指南将适合你。 

<div id="list-members-lookup">
  ### 列表成员查找
</div>

v2 列表成员查找端点将取代标准的 [GET lists/members](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-members)、[GET lists/memberships](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-memberships) 端点。

下表比较了标准 v1.1 和 X API v2 列表端点：

**列表成员查找**

|     |     |     |
| :--- | :--- | :--- |
| 描述 | 标准 v1.1 | X API v2 |
| 支持的 HTTP 方法 | `GET` | `GET` |
| 主机域名 | `https://api.x.com` | `https://api.x.com` |
| 端点路径 | `/1.1/lists/members.json` | `/2/lists/:id/members` |
| [Authentication](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文<br /><br />仅应用 | OAuth 1.0a 用户上下文<br /><br />OAuth 2.0 Authorization Code with PKCE<br /><br />仅应用 |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 使用 OAuth 1.0a 时每 15 分钟 900 次请求<br /><br />仅应用时每 15 分钟 75 次请求 | 使用 OAuth 1.0a 时每 15 分钟 900 次请求<br /><br />使用 OAuth 2.0 时每 15 分钟 900 次请求<br /><br />仅应用时每 15 分钟 900 次请求 |

**列表成员关系查找**

|     |     |     |
| :--- | :--- | :--- |
| 描述 | 标准 v1.1 | X API v2 |
| 支持的 HTTP 方法 | `GET` | `GET` |
| 主机域名 | `https://api.x.com` | `https://api.x.com` |
| 端点路径 | `/1.1/lists/memberships.json` | `/2/users/:id/list_memberships` |
| [Authentication](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文<br /><br />仅应用 | OAuth 1.0a 用户上下文<br /><br />OAuth 2.0 Authorization Code with PKCE<br /><br />仅应用 |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 使用 OAuth 1.0a 时每 15 分钟 75 次请求<br /><br />仅应用时每 15 分钟 75 次请求 | 使用 OAuth 1.0a 时每 15 分钟 75 次请求<br /><br />使用 OAuth 2.0 时每 15 分钟 75 次请求<br /><br />仅应用时每 15 分钟 75 次请求 |

<div id="manage-list-members">
  ### 管理 List 成员
</div>

用于在 v2 中管理 List 成员的端点将取代标准的 [POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create)、[POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy) 端点。

下表比较了标准 v1.1 和 X API v2 的 List 端点：

**添加成员**

| 说明 | 标准 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 支持的 HTTP 方法 | POST | POST |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /1.1/lists/members/create.json | /2/lists/:id/members |
| [Authentication](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文 | OAuth 1.0a 用户上下文<br /><br />OAuth 2.0 Authorization Code with PKCE |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 无 | 每 15 分钟 300 个请求（每个用户） |

**移除成员**

| 说明 | 标准 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 支持的 HTTP 方法 | POST | DELETE |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /1.1/lists/members/destroy.json | /2/lists/:id/:user&#95;id |
| [Authentication](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文 | OAuth 1.0a 用户上下文<br /><br />OAuth 2.0 Authorization Code with PKCE |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 无 | 每 15 分钟 300 个请求（每个用户） |

要访问 X API v2 端点，你必须[注册开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)。进行身份验证时，你必须使用位于某个[项目](/zh/resources/fundamentals/projects)中的[开发者应用](/zh/resources/fundamentals/developer-apps)的密钥和令牌。

在我们的[入门](/zh/x-api/getting-started/getting-access)页面了解有关获取 X API v2 端点访问权限的更多信息。
