<div id="manage-list-members-quick-start-guide">
  ### 管理列表成员快速入门指南
</div>

<Note>
  **请注意：** 本指南假设你已经完成了[快速入门概览](/zh/x-api/lists/list-members#getting-started-with-the-list-members-endpoint-group)中的前提条件。
</Note>

<div id="steps-to-build-a-manage-list-member-request">
  ### 构建用于管理 List 成员的请求的步骤
</div>

<div id="step-one-choose-the-list-endpoint-collection-from-postman">
  #### 第一步：在 Postman 中选择 List 相关的端点集合
</div>

将 X API v2 集合加载到 Postman 后，导航到“List”文件夹，选择其中的“List members”子文件夹，然后选择“Add a member”。
 

<div id="step-two-specify-the-user-to-add">
  #### 步骤二：指定要添加的用户
</div>

用于管理列表成员的端点需要两个 ID：列表的 ID 以及要添加的用户的 ID。你可以使用 [users lookup](/zh/x-api/users/lookup/introduction)，传入用户名来查找该用户的 ID，从而获取返回对象中的 `id` 字段。

目标 ID 可以是任意有效的用户 ID。在 Postman 中，导航到 “Params” 选项卡，并在 id 路径变量的 “Value” 列中输入你的 ID。然后导航到 “Body” 选项卡，将你希望添加的用户的 ID 作为 user&#95;id 参数的值。请确保在任何 ID 前后都不要包含空格。

|             |                  |                    |
| :---------- | :--------------- | :----------------- |
| **Key**     | **Value**        | **Parameter type** |
| `id`        | 列表 ID            | path               |
| user&#95;id | 你希望添加为成员的目标用户 ID | body               |

现在你应该会在 “Send” 按钮旁边看到类似如下的 URL：

`https://api.x.com/2/lists/1441162269824405510/members`

步骤三：发出请求并查看响应

在完成所有设置后，点击 “Send” 按钮，你将收到与以下示例响应类似的返回结果：

```
{
  "data": {
    "is_member": true
  }
}
```

如果在返回的响应对象中，键 is&#95;member 的值为 true，则表示你已成功将该用户添加为列表成员。 

要从列表中移除成员，请在已加载到 Postman 中的 X API v2 集合的 “Lists” 文件夹中选择 “Remove a member” 请求。此端点需要提供该列表的 ID，以及你希望移除成员的用户 ID。在 “Params” 选项卡中，将列表的 ID 作为 id 列的值输入，并将要移除的用户的 ID 作为 user&#95;id 列的值输入。 

当删除请求成功后，你将收到类似于以下示例的响应：

```
{
  "data": {
    "is_member": false
  }
}
```
