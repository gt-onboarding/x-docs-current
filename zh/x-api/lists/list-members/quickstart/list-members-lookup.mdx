<div id="steps-to-build-a-list-members-lookup-request">
  ### 构建 List 成员查询请求的步骤
</div>

**步骤一：在 Postman 中选择 List 相关的端点集合**

在你将 X API v2 集合加载到 Postman 之后，导航到 “List” 文件夹，选择其中的 “List members” 文件夹，然后选择 &quot;Members lookup&quot;。
 

**步骤二：确定并指定你希望检索成员的 List**

你必须指定一个你希望获取成员的 List。你可以通过访问 x.com，点击某个 List，然后查看浏览器地址栏中的 URL 来找到该 List 的 ID。例如，下面这个 URL 的 List ID 是 84839422。

https://x.com/i/lists/84839422

目标 ID 可以是任意有效的 List ID。在 Postman 中，导航到 &quot;Params&quot; 选项卡，并在 id 路径变量的 &quot;Value&quot; 列中输入你的 ID。务必确保在任何 ID 前后都不要包含空格。

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 84839422 (List ID) |

**步骤三：确定并指定你希望检索的字段**

如果你在完成步骤三后点击 &quot;Send&quot; 按钮，你将在响应中收到默认的 [user object](/zh/x-api/fundamentals/data-dictionary#user) 字段：id、name 和 username。

如果你希望在 id、name 和 username 之外再获取更多字段，就必须在请求中通过 [字段](/zh/x-api/fundamentals/fields) 和/或 [扩展字段](/zh/x-api/fundamentals/expansions) 参数来指定这些字段。

在本练习中，我们将从不同对象请求三组额外字段：

* 主 user 对象中的额外 user.created&#95;at 字段。
* 使用扩展字段参数请求完整的 [Post object](/zh/x-api/fundamentals/data-dictionary#tweet)。
* 关联 Post 对象中的额外 tweet.created&#95;at 字段。

在 Postman 中，导航到 &quot;Params&quot; 选项卡，并在 &quot;Query Params&quot; 表中添加以下键值对：

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| user.fields | created&#95;at | `created_at` |
| expansions | pinned&#95;tweet&#95;id | `includes.tweets.id,  <br/>includes.tweets.text  <br/>` |
| tweet.fields | created&#95;at | `includes.tweets.created_at` |

此时你应该会在 “Send” 按钮旁边看到一个类似的 URL：

`https://api.x.com/2/lists/84839422/members?user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=created_at`
