<div id="list-lookup-standard-v11-compared-to-x-api-v2">
  ### 列表查询：标准 v1.1 与 X API v2 的对比
</div>

如果你一直在使用标准 v1.1 的 [GET lists/show](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-show) 和 [GET lists/ownerships](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-ownerships) 端点，本指南旨在帮助你了解标准 v1.1 与 X API v2 列表查询端点之间的相似点和差异。

* **相似之处**
  * 身份验证方式
  * 速率限制
* **差异**
  * 端点 URL
  * 应用和项目要求
  * 每个请求可返回的数据对象数量限制
  * 响应数据格式
  * 请求参数

<div id="similarities">
  #### 相似点
</div>

**身份验证**

这两个版本的端点都支持 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication#oauth-1-0a-2) 和 [仅应用（App only）](/zh/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) 两种方式。因此，如果你之前使用的是标准 v1.1 的某个 List 查找端点，在迁移到 X API v2 版本时，可以继续使用相同的身份验证方式。

根据你所选用的身份验证库或包，仅应用（App only）身份验证通常是最容易上手的方式，并且只需通过一个简单的请求头就能完成设置。要了解如何生成仅应用访问令牌，请参阅[此仅应用指南](/zh/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)。

**速率限制**

|     |     |
| :--- | :--- |
| **Standard v1.1** | **X API v2** |
| /1.1/lists/show.json<br /><br />使用 OAuth 1.0a 用户上下文时，每个 15 分钟窗口最多 75 次请求<br /><br />仅应用时，每个 15 分钟窗口最多 75 次请求 | /2/lists/:id<br /><br />使用 OAuth 1.0a 用户上下文时，每个 15 分钟窗口最多 75 次请求<br /><br />使用 OAuth 2.0 Authorization Code with PKCE 授权方式时，每个 15 分钟窗口最多 75 次请求 |
| /1.1/lists/ownerships.json<br /><br />使用 OAuth 1.0a 用户上下文时，每个 15 分钟窗口最多 15 次请求<br /><br />仅应用时，每个 15 分钟窗口最多 15 次请求 | /2/users/:id/owned&#95;lists<br /><br />使用 OAuth 1.0a 用户上下文时，每个 15 分钟窗口最多 15 次请求<br /><br />使用 OAuth 2.0 Authorization Code with PKCE 授权方式时，每个 15 分钟窗口最多 15 次请求<br /><br />仅应用时，每个 15 分钟窗口最多 15 次请求 |

<div id="differences">
  #### 差异
</div>

**Endpoint URL**

* 标准 v1.1 端点：
  * GET https://api.x.com/1.1/lists/show.json
    （查询指定的列表）
  * GET https://api.x.com/1.1/lists/ownerships.json
    （查询指定用户拥有的列表）
* X API v2 端点：
  * GET https://api.x.com/2/lists/:id
    （查询指定的列表）

  * GET https://api.x.com/2/users/:id/owned&#95;lists
    （查询指定用户拥有的列表）

**应用和项目要求**

在为请求进行身份验证时，X API v2 端点要求你使用与某个[开发者应用](/zh/resources/fundamentals/developer-apps)和[项目](/zh/resources/fundamentals/projects)关联的凭证。所有 X API v1.1 端点都可以使用来自独立应用或与项目关联的应用的凭证。

**每个请求可返回的数据对象数量限制**

标准 v1.1 /lists/ownerships 端点允许你每个请求最多返回 1000 个列表。新的 v2 端点允许你每个请求最多返回 100 个列表。默认情况下，会返回 100 个用户对象；如果要更改结果数量，你需要传入查询参数 max&#95;results=，其值为 1-100 之间的数字；然后，你可以将响应载荷中返回的 next&#95;token 传给下一次请求中的 pagination&#95;token 查询参数，以继续分页。

**响应数据格式**

标准 v1.1 与 X API v2 版本的端点之间的最大差异之一，是你如何选择在载荷中返回哪些字段。

对于标准端点，你默认会收到许多响应字段，然后可以选择使用参数来指定哪些额外字段或字段集合应在载荷中返回。

X API v2 版本默认仅返回列表的 id 和 name 字段。如果要请求任何额外字段或对象，你需要使用 [fields](/zh/x-api/fundamentals/fields) 和 [expansions](/zh/x-api/fundamentals/expansions) 参数。通过此端点请求的任何列表字段都会在主列表对象中返回。任何扩展得到的帖子或用户对象及其字段都会出现在响应中的 includes 对象内。然后，你可以通过匹配用户对象和扩展帖子对象中的 ID，将这些扩展对象与列表对象关联起来。 

下面是一些可能的列表字段和扩展字段示例：

* created&#95;at

* follower&#95;count

* member&#95;count

* owner&#95;id

* description

* private

|     |     |
| :--- | :--- |
| **Endpoint** | **Expansion（扩展字段）** |
| /2/lists/:id | owner&#95;id |
| /2/users/:id/owned&#95;lists | owner&#95;id |

我们建议你在各自的指南中进一步了解这些新参数，或者阅读我们的[如何使用 fields 和 expansions](/zh/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)指南。 

我们还整理了一份[数据格式迁移指南](/zh/x-api/migrate/data-format-migration)，帮助你将标准 v1.1 字段映射到新的 v2 字段。该指南还会为你提供在 v2 请求中需要传入哪些特定的扩展字段参数和字段参数，以返回指定字段。 

除了你请求某些字段的方式发生变化之外，X API v2 还为 API 返回的对象（包括 [Post](/zh/x-api/fundamentals/data-dictionary#tweet) 和 [user](/zh/x-api/fundamentals/data-dictionary#user) 对象）引入了新的 JSON 设计。

* 在 JSON 根级别，标准端点在 statuses 数组中返回帖子对象，而 X API v2 则返回 data 数组。 

* X API v2 JSON 不再引用转发和引用的“statuses”，而是引用转发和引用的帖子。许多传统和已弃用的字段（例如 contributors 和 user.translator&#95;type）都将被移除。 

* X API v2 不再同时使用 favorites（Post 对象中）和 favourites（user 对象中），而统一使用 like 这个术语。 

* X 采用的约定是：在 JSON 中为空（例如为 null）的字段不会写入载荷。只有当帖子和用户属性具有非 null 值时，才会包含在载荷中。

**请求参数**

以下标准 v1.1 请求参数在 X API v2 中有对应项：

**通过 ID 查询列表**

|     |     |
| :--- | :--- |
| **标准 v1.1** | **X API v2** |
| list&#95;id | id  |
| slug | 无对应项 |
| owner&#95;screen&#95;name | 无对应项 |
| owner&#95;id | 通过 expansions/fields 参数请求 |

**查询用户拥有的列表**

|     |     |
| :--- | :--- |
| **标准版 v1.1** | **X API v2** |
| user&#95;id | id  |
| screen&#95;name | 无对应字段 |
| count | max&#95;results |
| cursor | pagination&#95;token |
