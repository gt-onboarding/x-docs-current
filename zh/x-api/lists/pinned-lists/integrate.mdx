<div id="authentication">
  #### 身份验证
</div>

所有 X API v2 端点都要求你使用一组凭证（也称为密钥和令牌）对请求进行身份验证。你可以使用 OAuth 1.0a 用户上下文来对发往此端点的请求进行身份验证。 

使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication)，这意味着你必须使用一组 API Key 和用户访问令牌才能成功发出请求。访问令牌必须与你代表其发出请求的用户关联。如果你想为另一位用户生成一组访问令牌，则他们必须通过 [三方 OAuth 流程](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)授权你的应用。

请注意，OAuth 1.0a 可能比较难用。如果你对这种身份验证方法不熟悉，我们建议你使用某个[库](/zh/x-api/tools-and-libraries/overview)，或使用 Postman 等工具。

<div id="developer-portal-projects-and-developer-apps">
  #### 开发者门户、项目和开发者应用
</div>

要获取一组可用于 X API v2 端点的身份验证凭据，你需要先[注册一个开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)，在该账号下设置一个[项目](/zh/resources/fundamentals/projects)，并在该项目中创建一个[开发者应用](/zh/resources/fundamentals/developer-apps)。然后，你可以在自己的开发者应用中找到密钥和令牌。
 

<div id="rate-limits">
  #### 速率限制
</div>

每天都有成千上万的开发者向 X API 发起请求。为了帮助管理如此庞大的请求量，我们在每个端点上都设置了[速率限制](/zh/x-api/fundamentals/rate-limits)，以限制你代表你的应用或代表经过身份验证的用户所能发起的请求次数。 

这些端点在用户级别上实施速率限制，这意味着你所代表的已认证用户，在跨任何开发者应用调用该端点时，都只能在一定次数以内。 

下表显示了每个端点对应的速率限制。

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **HTTP method** | **Rate limit** |
| /2/users/:id/pinned&#95;lists | POST | 每 15 分钟 50 次请求 |
| /2/users/:id/pinned&#95;lists/:list&#95;id | DELETE | 每 15 分钟 50 次请求 |
| /2/users/:id/pinned&#95;lists | GET | 每 15 分钟 15 次请求 |

<div id="fields-and-expansions">
  #### 字段和扩展字段
</div>

X API v2 的 GET 端点允许用户使用一组称为 `fields` 和 `expansions` 的工具，精确指定希望由 API 返回的具体数据。`expansions` 参数允许你展开在请求载荷中被引用的对象。例如，查询置顶列表时，可以使用以下[扩展字段](/zh/x-api/fundamentals/expansions)：

* `owner_id`

`fields` 参数允许你在不同的数据对象中，精确选择希望接收的[字段](/zh/x-api/fundamentals/fields)。此端点主要返回用户对象。默认情况下，List 对象会返回 `id` 和 `name` 字段。要接收其他字段，例如 `list.created_at` 或 `list.description`，你需要通过 fields 参数显式请求这些字段。 

我们在 [X API v2 数据字典](/zh/x-api/fundamentals/data-dictionary)中新增了关于如何同时使用[字段和扩展字段](/zh/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)的指南。

下表展示了该查找端点可用的字段和扩展字段：

|     |     |     |
| :--- | :--- | :--- |
| **端点** | **字段** | **扩展字段** |
| /2/users/:id/pinned&#95;lists | `list.fields`<br /><br />`user.fields` | `owner_id` |
