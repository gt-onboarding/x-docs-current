<div id="steps-to-build-a-pinned-list-lookup-request">
  ### 构建置顶列表查询请求的步骤
</div>

**步骤一：在 Postman 中选择 List 相关的端点集合**

在你已经将 X API v2 集合加载到 Postman 之后，导航到 “List” 文件夹，选择其中的 “Pinned Lists” 文件夹，然后选择 &quot;User&#39;s pinned Lists&quot;。
 

**步骤二：确定并指定用户**

要检索某个用户的置顶列表，你必须在请求中指定该用户的 user ID。该 user ID 必须与正在进行认证的用户 ID 对应，这意味着在对请求进行认证时，你必须传入与该 user ID 关联的访问令牌。在此示例中，你可以指定属于你自己账号的 ID。你可以通过以下两种方式找到你的 ID：

1. 使用按用户名查询的 [users lookup](/zh/x-api/users/lookup/introduction) 端点，你可以传入一个用户名并获得 `id` 字段。
2. 查看你的访问令牌，你会发现其中的数字部分就是你的 user ID。

在 Postman 中，导航到 &quot;Params&quot; 选项卡，在 `id` 参数的 &quot;Value&quot; 列中输入该 user ID。

|         |                      |
| :------ | :------------------- |
| **Key** | **Value**            |
| `id`    | 2244994945 (user ID) |

**步骤三：确定并指定你希望检索的字段**

如果你在完成步骤三后点击 &quot;Send&quot; 按钮，你将在响应中收到默认的 [List object](/zh/x-api/fundamentals/data-dictionary#list) 字段：`id` 和 `name`。

如果你希望接收除 `id` 和 `name` 之外的其他字段，你需要在请求中通过 [`field`](/zh/x-api/fundamentals/fields) 和/或 [`expansion`](/zh/x-api/fundamentals/expansions) 参数来指定这些字段。

在本练习中，我们将从不同对象中请求三组额外字段：

* 主 List 对象中的额外字段 `follower_count`。
* 使用 expansion 参数获取完整的 [user object](/zh/x-api/fundamentals/data-dictionary#user)。
* 关联 user 对象中的额外字段 `tweet.created_at`。

在 Postman 中，导航到 &quot;Params&quot; 选项卡，在 &quot;Query Params&quot; 表中添加以下 key:value 对：

|             |                    |                                                                               |
| :---------- | :----------------- | :---------------------------------------------------------------------------- |
| Key         | Value              | Returned fields                                                               |
| list.fields | follower&#95;count | `follower_count`                                                              |
| expansions  | owner&#95;id       | `includes.users.id,  <br/>includes.users.name,  <br/>includes.users.username` |
| user.fields | created&#95;at     | `includes.users.created_at`                                                   |

现在你应该会在 “Send” 按钮旁看到类似如下的 URL：

`https://api.x.com/2/users/2244994945/pinned_lists?expansions=owner_id&list.fields=follower_count&user.fields=created_at`

**步骤四：发出请求并检查响应**

当你完成所有设置后，点击 &quot;Send&quot; 按钮，你将收到与下面示例响应类似的结果：

```
{
  "data": [
    {
      "follower_count": 0,
      "id": "1454155907651158017",
      "name": "test list",
      "owner_id": "2244994945"
    }
  ],
  "includes": {
    "users": [
      {
        "username": "TwitterDev",
        "id": "2244994945",
        "created_at": "2013-12-14T04:35:55.000Z",
        "name": "Twitter Dev"
      }
    ]
  },
  "meta": {
    "result_count": 1
  }
}
```
