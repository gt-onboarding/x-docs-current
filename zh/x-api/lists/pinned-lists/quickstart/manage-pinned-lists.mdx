<div id="steps-to-build-a-manage-pinned-list-request">
  ### 构建管理固定列表请求的步骤
</div>

**第一步：在 Postman 中选择 List 相关的端点集合**

在你已经在 Postman 中加载了 X API v2 集合之后，导航到 “List” 文件夹，选择其中的 “Pinned Lists” 文件夹，然后选择 &quot;Pin a List&quot;。
 

**第二步：指定要固定的 List**

管理固定 List 的端点需要两个 ID：一个是用户 ID（将 List 设为固定的已认证用户），另一个是目标 List（要被固定的 List）。用户的 ID 必须与进行认证的用户 ID 相对应，这意味着在对请求进行认证时，你必须传递与该用户 ID 关联的访问令牌（Access Token）。在这个例子中，你可以指定属于你自己的用户 ID。你可以通过以下两种方式找到你的 ID：

1. 使用按用户名查询用户的 [users lookup](/zh/x-api/users/lookup/introduction) 端点，你可以传递一个用户名并获取响应中的 `id` 字段。
2. 查看你的访问令牌，你会发现其中的数字部分就是你的用户 ID。

目标 List ID 可以是任何有效的 List。在 Postman 中，导航到 &quot;Params&quot; 选项卡，在 id 路径变量的 &quot;Value&quot; 列中输入用户 ID。然后导航到 “Body” 选项卡，将你希望固定的 List 的 ID 作为 list&#95;id 参数的值输入。请务必不要在任何 ID 前后包含空格。

|             |                  |                    |
| :---------- | :--------------- | :----------------- |
| **Key**     | **Value**        | **Parameter type** |
| `id`        | 用户 ID            | path               |
| list&#95;id | 要固定的目标 List 的 ID | body               |

现在你应该会在 &quot;Send&quot; 按钮旁边看到类似的 URL：

```
      https://api.x.com/2/users/2244994945/pinned_lists
```

步骤三：发起请求并查看响应

在完成所有设置后，点击“Send”按钮，你将会收到一份类似于以下示例的响应：

```
{
  "data": {
    "pinned": true
  }
}
```

如果返回的响应对象中 `pinned` 键的值为 true，则表示你已成功固定目标列表。

要取消固定某个列表，请在 Postman 中已加载的 X API v2 集合的 “Lists” 文件夹中选择 “Unpin List” 请求。此端点需要已认证用户的 ID，以及要取消固定的列表的 ID。在 “Params” 选项卡中，将用户 ID 填写为 `id` 列的值，将要取消固定的列表 ID 填写为 `list_id` 列的值。

删除请求成功后，你会收到类似于以下示例的响应：

```
{
  "data": {
    "pinned": false
  }
}
```
