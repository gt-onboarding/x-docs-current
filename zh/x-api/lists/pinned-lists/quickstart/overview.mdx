<div id="getting-started-with-the-pinned-list-endpoint-group">
  ## 开始使用置顶 List 端点组
</div>

本简要概览将帮助你使用 [Postman](/zh/tutorials/postman-getting-started) 向置顶 List 端点发出第一个请求。

如果你想查看不同语言的示例代码，请访问我们的 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 代码库。

<Note>
  ### 前提条件

  要完成本指南，你需要一组 [密钥和令牌](/zh/resources/fundamentals/authentication) 来对请求进行身份验证。你可以按照以下步骤生成这些密钥和令牌：

  * [注册开发者账号](https://developer.x.com/en/apply-for-access) 并通过审批。
  * 在开发者门户中创建一个 [项目](/zh/resources/fundamentals/projects) 以及关联的 [开发者应用](/zh/resources/fundamentals/developer-apps)。
  * 前往你的 App 的“Keys and tokens”页面生成所需凭证。请务必将所有凭证保存在安全的位置。
</Note>

加载 X API v2 Postman 集合

要将 X API v2 Postman 集合加载到你的工作区中，请点击以下按钮：

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  将 X API v2 添加到 Postman
</Button>

 

<div id="authenticate-your-request">
  #### 对请求进行身份验证
</div>

要向 **lookup** 端点发出成功请求，你可以使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication)、[OAuth 2.0 应用仅模式](/zh/resources/fundamentals/authentication#oauth-2-0) 或 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)。但是，对于 **manage** 端点，你只能使用 OAuth 1.0a 用户上下文或 OAuth 2.0 Authorization Code with PKCE 进行身份验证。

无论使用哪种方式，在使用 Postman 时，只要你正确配置了环境，请求中都会自动填充默认的身份验证密钥和令牌。

你可以通过在 Postman 右上角选择名为“X API v2”的环境，并将你的密钥和令牌添加到“initial value”和“current value”字段中（点击环境下拉菜单旁边的眼睛图标）来完成配置。这些密钥包括：

* API Key（也称为 Consumer Key）
* API Secret Key（也称为 Consumer Secret）
* OAuth 1.0a 用户访问令牌（Access Token）
* OAuth 1.0a 用户访问令牌密钥（Access Token Secret）
* OAuth 2.0 应用访问令牌（App Access Token）
* OAuth 2.0 Client Key（仅当你在应用的设置中配置了 OAuth 2.0 用户身份验证设置时可用）
* OAuth 2.0 Client Secret（仅当你在应用的设置中配置了 OAuth 2.0 用户身份验证设置时可用）
