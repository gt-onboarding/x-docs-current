将列表固定可以让 X 用户更轻松地访问他们订阅或自己创建的列表。

由于你在这些端点上是代表某个用户发起请求，因此必须使用 [OAuth 1.0a User Context](/zh/resources/fundamentals/authentication) 或 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 对其进行身份验证，并使用与已授权你应用的用户关联的用户访问令牌；这些令牌可以通过 [三方 OAuth 流程](/zh/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（OAuth 1.0a）或 [Authorization Code with PKCE 授权流程](/zh/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce))（OAuth 2.0）生成。

<div id="pinned-list-lookup">
  ### 置顶列表查询
</div>

置顶列表查询提供一个端点，可用于检索已通过身份验证的用户的置顶列表。该端点的速率限制为每 15 分钟 15 次请求。
 

<div id="manage-pinned-lists">
  ### 管理置顶列表
</div>

管理置顶列表端点允许你代表已认证的用户置顶和取消置顶列表。对于这些端点，有两种可用方法：POST 和 DELETE。POST 方法允许你置顶一个列表，而 DELETE 方法允许你取消置顶一个列表。两个端点的用户速率限制都是每 15 分钟最多 50 次请求。

<Note>
  **账号设置**

  要访问这些端点，你需要：

  * 一个已获批准的 [开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info))。
  * 使用位于某个 [项目](/zh/resources/fundamentals/projects) 中的 [开发者应用](/zh/resources/fundamentals/developer-apps) 所提供的密钥和令牌进行认证。 

  在我们的 [入门指南](/zh/x-api/getting-started/getting-access) 中了解更多关于获取 X API v2 端点访问权限的信息。
</Note>
