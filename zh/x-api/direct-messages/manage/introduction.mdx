私信（Direct Messages）可在 X 上实现私密对话。私信是 X 上最受欢迎的功能之一，拥有多种使用场景，从朋友之间的群聊，到为全球品牌提供客户支持。新的 v2 版本私信端点将分阶段推出，本次第一阶段包含用于创建私信以及列出私信会话事件的基础端点。X API v2 首次支持*群组*会话。

本次 Manage Direct Messages 的初始发布包含三个用于创建私信的 POST 方法：

* **POST /2/dm&#95;conversations/with/:participant&#95;id/messages** - 创建一对一私信。此方法要么创建一个新的一对一会话，要么检索当前会话并向其中添加这条私信。:participant&#95;id 路径参数是接收消息账号的用户 ID。 
* **POST /2/dm&#95;conversations** - 创建一个新的群组会话并向其中添加一条私信。这些请求需要提供会话参与者列表。请注意，你可以使用相同的参与者列表创建多个会话。这类请求始终会返回一个新的会话 ID。 
* **POST /2/dm&#95;conversations/:dm&#95;conversation&#95;id/messages** - 创建一条私信并将其添加到现有会话中。:dm&#95;conversation&#95;id 路径参数是要添加消息的会话 ID。 

请注意，私信事件 ID 在 v1.1 和 v2（以及 X 应用）之间是通用的，因此可以将 v1.1 中用于隐藏/删除私信的方法与这个新的 v2 端点配合使用。还要注意，Enterprise 和 Premium Account Activity API 支持 v2 一对一消息，但暂不支持群组会话。   

POST 方法对每位用户有每 15 分钟 200 个请求的速率限制。每位用户还存在每 24 小时 1000 个请求的速率限制。此外，还有每 24 小时 15000 个请求的速率限制。请注意，这些速率限制在这些 POST 端点之间是共享的。

由于你在使用 manage Posts 端点代表用户发起请求，因此必须使用 [OAuth 1.0a User Context](https://developer-staging.x.com/resources/fundamentals/authentication) 或 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 进行身份验证，并使用与已授权你的应用的用户关联的用户访问令牌。若要使用 OAuth 1.0a 生成该用户访问令牌，你可以使用 [3-legged OAuth flow](/zh/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)。若要使用 OAuth 2.0 生成用户访问令牌，你可以使用 [Authorization Code with PKCE 授权流程](/zh/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce))。

<Note>
  **账号设置**

  要访问这些端点，你需要：

  * 一个已获批准的[开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)。
  * 使用位于某个[项目](/zh/resources/fundamentals/projects)中的[开发者应用](/zh/resources/fundamentals/developer-apps)的密钥和令牌进行身份验证。 

  你可以在我们的[入门指南](/zh/x-api/getting-started/getting-access)中进一步了解如何获取对 X API v2 端点的访问权限。
</Note>
