用于管理 DM 屏蔽的端点允许你代表已认证用户屏蔽或取消屏蔽指定账号。对于这些端点，有两种可用的 POST 方法：

* **/2/users/:id/dm/block**: 允许你屏蔽一个账号
* **/2/users/:id/dm/unblock**: 允许你取消屏蔽一个账号

<div id="getting-started">
  ### 快速入门
</div>

<div id="authentication">
  ### 身份验证
</div>

由于你代表某个用户发起请求，必须使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) 或 [OAuth 2.0 Authorization Code with PKCE](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 对这些端点进行身份验证，并使用与你所代表的该用户关联的用户访问令牌。你可以通过 [三方 OAuth 流程](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)（OAuth 1.0a）或通过 [Authorization Code with PKCE 授权流](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)（OAuth 2.0）生成该用户访问令牌。

<div id="making-a-request">
  ### 发起请求
</div>

Block

一旦用户已经通过你的应用完成身份验证，你就可以如下面所示，代表该用户调用 Block 端点：

```
      curl --request POST 'https://api.x.com/2/users/:id/dm/block' --header 'Authorization: ••••••'
```

如果请求成功，你将会看到如下所示的 JSON 响应：

```
{
   "data": {
       "blocked": true
   }
}
```

**取消屏蔽**

一旦用户在你的应用中完成认证，你就可以代表该用户调用 Unblock 端点，如下所示：

```
      curl --request POST 'https://api.x.com/2/users/:id/dm/unblock' --header 'Authorization: ••••••'
```

如果请求成功，你会看到如下所示的 JSON 响应：

```
{
   "data": {
       "blocked": false
   }
}
```
