  ## 比较 v1.1 和 v2 Direct Message 事件查询端点
</div>

Direct Messages 端点的 v1.1 和 v2 版本都提供了查询 Direct Message 事件的方法。本指南旨在帮助你理解它们之间的差异，并为迁移到 v2 提供相关信息。 

两者之间的一个主要区别是，v1.1 仅支持一对一会话，而 v2 引入了对群组会话的支持。其中一个体现是，v1.1 只支持“消息创建”事件，而 v2 还支持与参与者加入和离开会话相关的事件。事实上，v2 的一个重要更新是将 dm&#95;conversations 设定为核心 API 对象。   

在 v1.1 中，有两个用于检索 Direct Messages 的端点（同样，v1.1 仅支持“新消息”这一种事件类型）：

* GET direct&#95;messages/events/show - 通过 ID 检索单个事件。 

* GET direct&#95;messages/events/list - 检索已认证用户在一对一 Direct Messages 会话中发送和接收的、最多 30 天内的消息。请注意，此方法无法检索群组会话中的消息。 

在此次 v2 发布中，有三个 GET 方法用于检索 Direct Message 会话事件： 

* **GET /2/dm&#95;conversations/with/:participant&#95;id/dm&#95;events** - 检索与某个一对一会话相关的 Direct Message 事件。:participant&#95;id 路径参数是与发起此请求的已认证用户进行会话的账号的用户 ID。 

* **GET /2/dm&#95;conversations/:dm&#95;conversation&#95;id/dm&#95;events** - 检索与特定会话 ID 相关联的 Direct Message 事件，该会话 ID 由 :dm&#95;conversation&#95;id 路径参数指明。此方法同时支持一对一和群组会话。 

* **GET /2/dm&#95;events** - 检索与某个用户相关联的 Direct Message 事件，包括一对一和群组会话。可获取最多追溯至 30 天前的事件。  

一个重要细节是，会话和事件 ID 在 X 平台的 v1.1 和 v2 版本之间是共享的。这意味着可以将两个版本结合使用。例如，Direct Messages v1.1 端点提供了返回单个事件和删除事件的方法，而这些方法目前在 v2 中尚不可用。由于 ID 在 v1.1 和 v2 之间是通用的，你可以基于 v2 提供的 ID 发起 v1.1 请求，或者通过引用 X 应用中会话 URL 中显示的会话 ID 来发起请求。

下表比较了 v1.1 和 v2 Direct Message 事件查询端点的一些基本方面。此处所述的 X API v2 特性适用于所有 Direct Message 查询端点。

| :--- | :--- | :--- |
| **说明** | **标准 v1.1** | **X API v2** |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点根路径 | [/1.1/direct&#95;messages](https://api.x.com/1.1/direct_messages) | [/2/dm&#95;conversations](https://api.x.com/2/users/:id/dm_conversations)<br /><br />Direct Messages 会话被作为一个基础 API 对象引入。   <br /><br />这些端点会检索 MessageCreate、ParticipantsJoin 和 ParticipantLeave 事件。 |
| 支持的 HTTP 方法 | GET | GET |
| 是否支持群组 Direct Messages |     | ✔   |
| 支持的事件类型 | message&#95;create | MessageCreate, ParticipantsJoin, ParticipantsLeave |
| [Authentication](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文 | OAuth 1.0a 用户上下文<br /><br />OAuth 2 用户上下文（权限范围：dm.read, tweet.read, user.read） |
