  过滤/查询运算符“别名”（aliases）已添加到 Enterprise、premium 和 v2 的 search 与 filtered stream 端点。在许多情况下，该运算符针对的是帖子而不是状态（statuses），例如，将 &quot;in&#95;reply&#95;to&#95;tweet&#95;id&quot; 作为 &quot;in&#95;reply&#95;to&#95;status&#95;id&quot; 的别名。在其他情况下，新的运算符别名使所执行的匹配更加清晰。例如，由于帖子只能包含一个视频附件，因此提供了新的 &quot;has:video&#95;link&quot; 别名，作为 &quot;has:videos&quot; 的替代。请查看 [v2 filtered stream operator table](https://developer.x.com/en/docs/twitter-api/tweets/filtered-stream/integrate/build-a-rule#list) 以查看更多示例。

<Update label="2021年12月14日" description="X API v2：身份验证">
  ### 支持 OAuth 2.0 身份验证

  从今天开始，所有开发者都可以在开发者门户中选择 OAuth 2.0 作为认证方式。我们在此次发布中新增了对机密客户端（confidential client）和公共客户端（public client）的支持，以及所有相关 v2 端点对这种身份验证方式的支持。

  OAuth 2.0 是一种行业标准的授权协议，它为开发者在应用的权限范围（scope）上提供了更多控制权，并改进了跨多设备的授权流程。换句话说，为 X 上的用户构建应用的开发者将对其应用向用户请求的信息拥有更多控制权，因此你只需要向终端用户请求你所需的数据和信息。

  这一现代授权协议将允许你为终端用户提供更简化的授权同意流程来授权你的应用，只展示你向他们请求的特定权限范围（scope）。

  若要了解此次发布的更多信息，请访问我们的论坛公告。
</Update>

<Update label="2021年11月15日" description="X API v2：早期访问">
  ### X API v2 的主要平台更新

  **主要平台更新：X API v2 现在是主要版本，全新的 Essential 和 Elevated 访问级别，以及政策更新**

  今天，我们宣布 X API v2 现在是 X API 的主要版本。我们在 X API v2 中已经发布了足够多的端点和功能，可以满足 90% 基于 X API 构建的现有应用的需求。

  我们今天推出以下变更，以进一步改善 X API v2 的开发者体验：

  * 提供快速且免费的 Essential 级别 API 访问，以及对使用场景已获批准开发者的免费 Elevated 级别访问。关于这些访问级别，我们在 “About the X API” 页面上提供了更多详细信息。

  * 我们正在移除《开发者政策》中限制你如何使用 X 核心功能进行构建、以及限制你的应用可以支持的用户数量的相关表述。

    * 我们正在开放平台，以鼓励你构建使 X 更好、更健康、并扩展公共对话的工具和产品。[查看我们希望你构建的解决方案列表](https://developer.x.com/en/docs/twitter-api/what-to-build)。

    * 具体来说，我们已移除了限制复制 X 体验（包括 X 的核心功能）以及要求获得许可才能拥有大量用户 token 的条款。

    * 我们知道，构建帮助 X 上人们的解决方案，往往意味着开发者必须构建（或复制）X 上已经提供的一些内容。我们对《开发者政策》所做的这些更改，旨在为开发者生态系统带来更清晰的指引，并提供一个开放的 API 平台，使开发者更容易进行构建、创新，并对公共对话产生影响。

  我们对这次发布以及促成此时刻的 X API 发布做了更详细的说明，请参阅我们的[论坛帖子](https://devcommunity.x.com/t/ushering-in-a-new-era-for-the-twitter-developer-platform-with-the-twitter-api-v2/162087)。

  除了查看之前的更新日志条目之外，我们还整理了一些关键资源，帮助你了解截至目前为止 X API v2 已经发布的全部内容：

  * [X API 端点映射](/zh/x-api/migrate/x-api-endpoint-map)

  * [About the X API](/zh/x-api/getting-started/about-x-api)
</Update>

<Update label="Jun 29, 2021" description="v2 filtered stream 和 sampled stream 现已提供冗余连接和回填功能">
  ### 流式端点的可靠性功能

  今天，我们为 X API v2 的 filtered stream 和 sampled stream 推出了两个提升流式可靠性的功能：回填（backfill）和冗余连接（redundant connections），目前已在 Academic Research 产品线中提供。

  冗余连接功能使你可以针对给定流建立两个连接，这有助于在其中一个流发生断开时，确保尽可能提高流式连接的在线时间。

  你可以在流式请求中添加 `backfill_minutes` 参数，并设置介于一到五之间的整数来使用回填功能，从而请求由于流中断而可能错过的数据显示。

  若要进一步了解此次更新，请访问我们的[论坛公告](https://devcommunity.x.com/t/new-reliability-and-recovery-functionality-available-for-twitter-api-v2-streaming-endpoints/156244/2)。

<Update label="Dec 18, 2020" description="适用于 X API v2 的会话回复设置帖子字段">
  ### 新增回复设置字段

  今天，我们在帖子对象中推出了一个名为 `reply_settings` 的新字段。该字段在所有返回帖子对象的 X API v2 端点中均可用，包括帖子查找、隐藏回复、recent search、sampled stream、filtered stream、关注查找、用户帖子时间线和用户提及时间线端点，或者通过在任意端点中[扩展](/zh/x-api/fundamentals/expansions)帖子对象来获取。

  此字段指示帖子作者允许哪些人回复其帖子：是所有人、仅在帖子中被提及的人，还是作者所关注的人。`reply_settings` 字段返回的取值包括 `everyone`、`mentionedUsers` 和 `following`。如果该帖子未设置会话控制，该字段仍会出现在帖子对象中，其取值为 `everyone`。
</Update>

  * 无论认证方式（用户上下文或 Bearer Token）如何，限速均在应用级别进行控制

  通过我们的[论坛公告](https://devcommunity.x.com/t/twitter-developer-labs-is-open-to-all-developers/126717)了解更多关于此次发布的信息。

  ### 引用负载渲染方式的变更

  今天，我们对引用负载中 URL 的渲染方式进行了更改。

  我们添加了一个名为 &quot;quoted&#95;status&#95;permalink&quot; 的新实体，以确保可以引用被引用的帖子，并且我们会从被引用帖子 &quot;text&quot; 字段中移除 t.co 链接。

  有关更多详情，请阅读我们的[论坛帖子](https://devcommunity.x.com/t/updating-how-urls-are-rendered-in-the-quote-tweet-payload/105473)和[更新后的文档](https://devcommunity.x.com/t/upcoming-changes-to-the-developer-platform/104603)。
