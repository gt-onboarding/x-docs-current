  过滤/查询运算符“别名”已添加到 Enterprise、Premium 和 v2 的搜索以及 filtered stream 端点中。在许多情况下，运算符现在指代帖子而不是状态，例如，将 &quot;in&#95;reply&#95;to&#95;tweet&#95;id&quot; 作为 &quot;in&#95;reply&#95;to&#95;status&#95;id&quot; 的别名。在其他情况下，新的运算符别名为匹配行为提供了更清晰的含义。例如，由于帖子只能有一个视频附件，因此为 &quot;has:videos&quot; 新增了一个 &quot;has:video&#95;link&quot; 别名。请查看 [v2 filtered stream 运算符表](https://developer.x.com/en/docs/twitter-api/tweets/filtered-stream/integrate/build-a-rule#list)了解更多示例。

<Update label="2021 年 12 月 14 日" description="X API v2：身份验证">
  ### 对 OAuth 2.0 身份验证的支持

  从今天起，所有开发者都可以在开发者门户中选择 OAuth 2.0 作为身份验证方法，从而使用 OAuth 2.0 进行身份验证。我们在本次发布中为机密客户端和公共客户端以及所有相关的 v2 端点新增了对这种身份验证方法的支持。

  OAuth 2.0 是一种行业标准的授权协议，它为开发者在应用作用域控制方面提供了更多能力，并改进了跨多设备的授权流程。换句话说，为 X 用户构建应用的开发者将能更好地控制其应用向用户请求的信息，这样你只需要向终端用户请求你所需要的数据和信息即可。

  这一现代授权协议将使你能够向终端用户呈现更精简的授权同意流程来为你的应用授予权限，并且只会展示你向他们请求的特定作用域。

  若想进一步了解此次发布，请访问我们的论坛公告。
</Update>

<Update label="2021 年 11 月 15 日" description="X API v2：早期访问">
  ### X API v2 的重大平台更新

  **重大平台更新：X API v2 现已成为主要版本，新的 Essential 和 Elevated 访问级别，以及政策变更**

  今天，我们宣布 X API v2 现已成为 X API 的主要版本。我们已经在 X API v2 中发布了足够多的端点和功能，可以满足 90% 基于 X API 构建的现有应用的需求。

  我们今天将推出以下变更，以进一步提升 X API v2 的开发者体验：

  * 提供快速、免费的 Essential API 访问，以及向使用场景已获审批的开发者提供免费的 Elevated 访问。关于这些访问级别，我们在“关于 X API”页面中提供了更多详细信息。

  * 我们正在移除开发者政策中，限制你如何基于 X 的核心功能进行构建，以及限制你通过应用所能支持用户数量的相关表述。

    * 我们正在开放我们的平台，以鼓励你构建能够让 X 变得更好、更健康，并拓展公共对话的工具和产品。[查看我们期望你构建的解决方案列表](https://developer.x.com/en/docs/twitter-api/what-to-build)。

    * 具体来说，我们已经移除了限制复制 X 体验的条款，其中包括对 X 核心功能的限制，以及要求获得许可才能拥有大量用户 token 的条款。

    * 我们知道，构建帮助 X 用户的解决方案往往意味着开发者必须构建（或复制）一些 X 上已经提供的功能或体验。这些对开发者政策的更改旨在为开发者生态带来更清晰的指引，并提供一个开放的 API 平台，使开发者能够更轻松地进行构建、创新，并对公共对话产生影响。

  关于此次发布以及此前促成这一时刻的各项 X API 发布内容，我们在[论坛帖子](https://devcommunity.x.com/t/ushering-in-a-new-era-for-the-twitter-developer-platform-with-the-twitter-api-v2/162087)中提供了更多详细信息。

  除了查看更早的更新日志条目之外，我们还整理了一些关键资源，供你查阅，以了解迄今为止 X API v2 已经发布的全部内容：

  * [X API 端点映射](/zh/x-api/migrate/x-api-endpoint-map)

  * [关于 X API](/zh/x-api/getting-started/about-x-api)
</Update>

<Update label="2021 年 6 月 29 日" description="在 v2 filtered stream 和 sampled stream 中现已提供冗余连接和回填功能">
  ### 流式端点的可靠性功能

  今天，我们为 X API v2 的 filtered stream 和 sampled stream 推出了两项流式传输可靠性功能，称为回填（backfill）和冗余连接（redundant connections），现已在 Academic Research 产品路径中提供。

  冗余连接功能使你能够与指定流建立两条连接，这有助于在其中一条流连接中断的情况下，最大化你的流式连接在线时长。

  你可以在流式请求中添加 `backfill_minutes` 参数，并提供介于一到五之间的整数来使用回填功能，从而请求因流断开而可能错过的数据。

  如需了解本次发布的更多信息，请访问我们的[论坛公告](https://devcommunity.x.com/t/new-reliability-and-recovery-functionality-available-for-twitter-api-v2-streaming-endpoints/156244/2)。

  ### Academic Research 学术研究产品通道简介

  今天，我们为 [X API v2](/zh/x-api/introduction) 推出全新的 [Academic Research 学术研究产品通道](/zh/resources/fundamentals/projects)。此次更新为学术研究人员引入了新的申请流程，获得批准的申请人将会获得一个 Academic Research [Project](/zh/resources/fundamentals/projects)，从而解锁更高的数据访问权限和高级功能。

  通过全新的 Academic Research 学术研究产品通道，你将能够使用以下功能：

  * 全新的 X API v2 [全量存档搜索端点](/zh/x-api/posts/full-archive-search)，目前仅对 Academic Research 学术研究产品通道开放。

  * 更高的 [帖子上限](/zh/x-api/fundamentals/post-cap)：1,000 万条，高于 [Standard 产品通道](/zh/resources/fundamentals/projects) 在 Basic [访问级别](https://developer.x.com/en/products/x-api/early-access/guide#na_2) 下可用的 50 万条。

  * 在 [recent search](/zh/x-api/posts/recent-search)、[full-archive search](/zh/x-api/posts/full-archive-search) 和 [filtered stream](/zh/x-api/posts/filtered-stream) 中可用的新过滤运算符，包括 `$`（又称 cashtag）、`bio`（仅可通过 filtered stream 使用）、`bio_name`（仅可通过 filtered stream 使用）、`bio_location`（仅可通过 filtered stream 使用）、`place`、`place_country`、`point_radius`、`bounding_box`、`-is:nullcast`、`has:cashtags` 和 `has:geo`。

  * filtered stream 的并发规则数上限提高至 1,000 条，高于 Standard 产品通道在 Basic 访问级别下可用的 25 条。

  * 查询和规则的字符上限提高到 1,024 个字符，高于 Standard 产品通道在 Basic 访问级别下可用的 512 个字符限制。

<Update label="2020 年 12 月 18 日" description="X API v2 中帖子会话回复设置字段">
  ### 新增回复设置字段

  今天，我们在 post 对象中推出了一个名为 `reply_settings` 的新字段。该字段适用于所有返回帖子数据的 X API v2 端点，包括 post lookup、hide replies、recent search、sampled stream、filtered stream、follows lookup、user post timeline 和 user mention timeline 端点，或通过在任意端点中[扩展](/zh/x-api/fundamentals/expansions) post 对象获取。

  此字段表示帖子作者允许哪些人回复其帖子：可以是所有人、只限于他们在帖子中提到的用户，或仅限他们关注的用户。`reply_settings` 字段返回的字段值包括 `everyone`、`mentionedUsers` 和 `following`。如果帖子没有设置会话控制，该字段仍会出现在 post 对象中，且字段值为 `everyone`。
</Update>

  今天，我们将帖子和用户、最近搜索以及隐藏回复端点引入 Labs 的 v2。我们正在弃用 Labs v1 端点，并将在此次初始发布后 90 天彻底停用它们。在此期间，我们也会继续提供 v1 文档。你可以通过我们的[反馈渠道](https://twitterdevfeedback.uservoice.com/)就这一版本策略向我们提供反馈。

  ### 引用负载呈现方式的变更

  今天，我们对引用负载中 URL 的呈现方式进行了调整。

  此外，我们将不再把被引用帖子对应的 t.co URL、expanded&#95;url 和 display&#95;url 放入 &quot;urls&quot; 实体中。取而代之的是，这些将被添加到一个新的 &quot;quoted&#95;status&#95;permalink&quot; 对象中，该对象包含 &quot;url&quot;、&quot;expanded&quot; 和 &quot;display&quot; URL 属性。这意味着，&quot;quoted&#95;status&#95;permalink&quot; 对象将专门用于指向被引用帖子的 t.co 链接，而 &quot;entities.urls&quot; 数组可以用来识别原始帖子或引用中分享的任何链接。

  有关更多详细信息，请阅读我们的[论坛帖子](https://devcommunity.x.com/t/updating-how-urls-are-rendered-in-the-quote-tweet-payload/105473)和[更新后的文档](https://devcommunity.x.com/t/upcoming-changes-to-the-developer-platform/104603)。
