<div id="how-to-get-access-to-the-ads-api">
  ## Ads API에 액세스하는 방법
</div>

1. [개발자 계정](https://developer.x.com/en/apply-for-access)에 가입합니다.
2. [개발자 앱](https://developer.x.com/app)을 생성하고 토큰을 안전하게 관리합니다.
3. 각 개발자 앱마다 Ads API 액세스를 요청하려면 [ads.x.com/help](https://ads.x.com/help)를 방문해 요청을 제출합니다.

<Note>**참고:** 이미 X Developer Platform을 사용 중이고 개발자 계정이 있다면 3단계로 건너뛰십시오.</Note>

<div id="step-one-signup-for-a-developer-account">
  ### 1단계: 개발자 계정에 가입하기
</div>

X의 어떤 API 제품에든 요청을 보내려면 먼저 개발자 계정에 가입해야 합니다. 개발자 포털에서 프로젝트와 개발자 앱을 생성하세요. 그러면 API에 대한 모든 요청을 인증하는 데 사용할 자격 증명 정보를 받게 됩니다.

<Button href="https://developer.x.com/en/apply-for-access">
  개발자 계정에 가입하기
</Button>

<div id="step-two-save-your-apps-key-and-tokens-and-keep-them-secure">
  ### 2단계: 앱의 키와 토큰을 저장하고 안전하게 보관하기
</div>

개발자 앱에서는 API 키(Consumer Key라고도 함) 세트를 제공받게 됩니다. 또한 개인 X 계정을 대신해 요청을 보내는 데 사용할 수 있는 액세스 토큰 세트와, OAuth 2.0 Bearer Token이 필요한 엔드포인트를 인증하는 데 사용할 수 있는 Bearer Token을 생성할 수도 있습니다. 이러한 키와 토큰은 재생성하지 않는 한 만료되지 않으므로, 환경 변수를 생성하거나 안전한 비밀번호 관리자를 사용하는 것을 권장합니다.

또한 App ID를 반드시 기록해 두십시오. App ID는 앱 세부정보 페이지의 URL에서 확인할 수 있습니다. 이렇게 해 두면 다음 단계에서 Ads API 액세스를 요청하는 과정이 더 수월해집니다.

<Note>
**참고:** 키와 토큰은 개발자 포털에서 한 번만 표시되므로, 생성 즉시 이러한 자격 증명을 비밀번호 관리 시스템에 저장하는 것이 중요합니다.

키와 토큰을 분실하거나 잊어버린 경우, 새 키와 토큰을 재생성해야 하며, 이때 기존 키와 토큰은 더 이상 유효하지 않게 됩니다. 이는 이전 자격 증명을 사용해 설정해 둔 모든 통합을 업데이트해야 함을 의미합니다.

[인증 모범 사례](https://developer.x.com/en/docs/authentication/guides/authentication-best-practices)에 대해 자세히 알아보세요.
</Note>

<div id="step-three-apply-for-access-to-the-ads-api">
  ### 3단계: Ads API 액세스 신청
</div>

이 시점에서는 [X API에 대한 기본 액세스](https://developer.x.com/en/docs/twitter-api/v1)는 갖게 되지만, 특정 X Ads API 기능에는 아직 액세스할 수 없습니다. 다음으로, Ads API 액세스를 요청하고 승인을 받아야 합니다. 각 개발자 앱에 대해 Ads API 액세스를 요청하려면 [ads.x.com/help](https://ads.x.com/help)를 방문해 요청을 제출하세요.

<div id="access-tiers">
  ### 액세스 티어
</div>

신청 절차에서 필요한 액세스 수준을 지정해야 합니다. [앱 수준 및 광고 계정 수준 권한](https://developer.x.com/en/docs/twitter-ads-api/obtaining-ads-account-access#levels-of-access)에 대해 자세히 알아보세요.

<div id="conversion-only">
  #### 전환 전용
</div>

모바일 및 웹 전환 엔드포인트에 대한 읽기 및 쓰기 권한.

<div id="standard-access">
  #### Standard Access
</div>

읽기 및 쓰기 권한으로 Analytics, Campaign Management, Creatives, Custom Audiences, Conversion 엔드포인트에 액세스할 수 있습니다.

<Note>**참고:** 앱이 Ads API 사용 승인을 받은 후에는 [인증된 Ads API 요청](https://developer.x.com/en/docs/twitter-ads-api/making-authenticated-requests)을 올바르게 수행하기 위해 [기존 사용자 액세스 토큰을 재발급](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)받아야 합니다.</Note>

<Note>**참고:** 2023년 7월 이전에 액세스를 요청한 Ads API 개발자는 액세스 수준과 권한이 다를 수 있으며, OAuth 토큰이 최대 다섯 개로 제한될 수 있습니다. 추가 엔드포인트에 대한 액세스 권한을 얻거나 기존 애플리케이션의 토큰 한도를 해제하려면 [액세스 확대](https://developer.x.com/en/docs/twitter-ads-api/increasing-access) 가이드를 참조하세요.</Note>

<div id="make-your-first-request">
  ## 첫 번째 요청 보내기
</div>

Ads API에 대한 접근 권한을 테스트하려면 [GET accounts](https://developer.x.com/en/docs/twitter-ads-api/campaign-management/api-reference/accounts) 엔드포인트로 요청을 보내세요. 이 엔드포인트는 현재 권한이 부여된 사용자가 액세스할 수 있는 광고 계정을 반환합니다. 이후 API 요청에서 이 요청으로부터 받은 광고 계정 ID를 사용하여 특정 광고 계정의 데이터를 읽고 쓸 수 있습니다. 명령줄에서 [Twurl](https://developer.x.com/en/docs/tutorials/using-twurl)을 사용할 경우 요청은 다음과 같은 형태가 됩니다.

**예제 요청**

```bash
twurl -H ads-api.x.com "/11/accounts"
```
