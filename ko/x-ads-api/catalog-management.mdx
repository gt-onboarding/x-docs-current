<div id="overview">
  ## 개요
</div>

Catalog API는 광고주가 상품 피드를 설정하고, 상품을 세트로 그룹화하며, 카탈로그 상품을 총괄적으로 관리할 수 있도록 하는 커머스 솔루션입니다. Catalog API는 프로그램적으로 카탈로그를 관리할 수 있게 해 주는 핵심 도구로, 광고주가 카탈로그 데이터가 수집·업데이트되는 방식에 대해 더 많은 제어 권한을 가질 수 있도록 합니다.

카탈로그는 사용자의 상품 및 상품 세트와 관련된 모든 정보를 보유하며, 사용자의 핸들(handle)에 연결됩니다. 현재는 **한 사용자가 하나의 카탈로그만 생성할 수 있습니다**. 

Catalog API는 상품을 카탈로그로 가져오는 두 가지 방식을 지원합니다:

1.  Scheduled Feed: 사용자가 피드 URL(CSV, TSV, XML)을 추가할 수 있으며, 해당 파일을 주기적으로 가져옵니다(X는 현재 최대 8GB의 파일을 지원합니다)

2.  Batch Products API: 사용자는 배치(JSON) 요청을 통해 상품 속성을 조회, 생성, 업데이트 및 삭제할 수 있습니다

X 판매자는 필터 규칙을 기반으로 상품 세트를 생성할 수 있습니다. 이렇게 하면 상품에 추가 메타데이터를 연결하고, 오가닉 커머스 기능 또는 동적 상품 광고를 통해 이러한 상품을 노출하기 위한 특별한 상품 세트를 만들 수 있습니다. Catalog API와 X 쇼핑 매니저는 두 가지 유형의 상품 세트를 지원합니다. 

1.  Manual: 사용자는 최대 50개의 상품을 선택하고, 이름과 설명을 설정할 수 있습니다

2.  Filter: 사용자는 최대 30개의 필터를 추가하여, 해당 필터를 충족하는 상품으로 구성된 세트를 자동으로 생성할 수 있습니다. 필터 유형에는 가격, Google 상품 카테고리, 상품 유형, 재고, 할인 가격, 사용자 지정 필드 등이 포함됩니다.

Catalog API에는 카탈로그를 관리하기 위한 4개의 엔드포인트가 포함되어 있습니다:

* [Product Catalogs](https://developer.x.com/en/docs/x-ads-api/catalog-management/api-reference/product-catalogs) \- 상품 정보를 위한 컨테이너 생성
* [Products](https://developer.x.com/en/docs/x-ads-api/catalog-management/api-reference/products) -  카탈로그에서 상품을 동기적으로 추가, 업데이트, 조회 및 제거
* [Product Sets](https://developer.x.com/en/docs/x-ads-api/catalog-management/api-reference/product-sets) -  상품 정보를 기반으로 상품을 그룹으로 분류
* [Schedule Feeds](https://developer.x.com/en/docs/twitter-ads-api/catalog-management/api-reference/scheduled-feeds) \- 파일 기반 상품 데이터를 카탈로그로 비동기 수집하도록 설정

<div id="prerequisites-for-the-catalog-api">
  ### Catalog API 사용을 위한 사전 요구사항
</div>

Catalog API 엔드포인트는 현재 조기 액세스(early access) 프로그램을 통해서만 제공됩니다. 액세스를 신청하려면 X 담당자에게 문의하세요. 

[X Shopping Manager](https://ads.x.com/shopping_manager)를 통해 서비스 약관에 동의해야 합니다.

Shopping Manager 및 상품 사양에 대한 자세한 내용은 [Product specifications guide](https://business.x.com/en/help/shopping-specs.html)를 참고하세요.

  ### 요청 한도
</div>

다음 표는 각 엔드포인트별 요청 한도를 보여줍니다.

<div id="product-catalog">
  #### Product Catalog
</div>

| HTTP 메서드 | 요청 제한 |
| :--- | :--- |
| GET /product_catalogs | 15분당 1,000회 |
| POST /product_catalogs | 15분당 20회 |
| PUT /product\_catalogs/:product\_catalog_id | 15분당 20회 |
| DELETE /product\_catalogs/:product\_catalog_id | 15분당 20회 |

  #### Batch Products API

| DELETE /product\_catalogs/:product\_catalog_id/products | 15분당 150회 |
