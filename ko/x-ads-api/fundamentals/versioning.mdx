<div id="overview">
  ## 개요
</div>

매달 X Ads API에 변경 사항을 적용하고 여러 가지 새로운 기능을 출시합니다. 이러한 변경 사항은 거의 항상 하위 호환성을 유지하지만, 매년 일부 비호환적인 변경 사항이 발생하기도 합니다. Ads API의 빠른 변경 주기가 새로운 기능 구현, 사용 중단(deprecation) 처리, 변경 사항 테스트 등 개발자들의 개발 주기에 부담을 준다는 피드백을 받아 왔습니다. 우리는 Ads 플랫폼을 사용하는 개발자 경험을 개선하고자 하며, 그 일환으로 엔드포인트 버저닝 개념을 도입했습니다.

아래는 문서 전반에서 사용하는 몇 가지 개념 정의입니다.

**Version**: Ads API 요청의 URL 경로에서 찾을 수 있는 버전 번호를 의미합니다. 예: GET /{version_number}/accounts. 이러한 방식의 버저닝은 URI 버저닝이라고 합니다.

**Breaking Changes**: Breaking changes는 기존 기능을 유지하기 위해 개발자 리소스 투입이 필요한 모든 변경 사항을 의미합니다. 여기에는 필요한 변경 사항을 조사하는 데 드는 리소스, 사용 중단되는 기능/엔드포인트를 파악하는 데 드는 리소스, 그리고 이 모든 변경 사항을 최종적으로 구현하는 데 드는 리소스가 포함됩니다. Breaking changes의 예시는 다음과 같습니다.

* API 요청/응답에서 파라미터를 제거하는 경우

* 어떤 파라미터나 엔드포인트의 이름을 변경하는 경우

* 값 표현 방식 변경 (preview&#95;url → card&#95;uri)

* 엔드포인트 동작 방식 변경 (예: 비동기 vs 동기 통계)

* 선택/필수 파라미터 추가 또는 변경 (예: 요청에서 name 필드를 필수로 만드는 경우)

**Deprecation**: 사용 중단(deprecated)된 버전이나 제품은 더 이상 지원되지 않으며, 개발자는 해당 API 사용을 중단할 것이 권장됩니다.

**Sunset**: 제품이나 API가 sunset되면, 해당 제품과 연관된 엔드포인트 집합은 더 이상 API를 통해 접근할 수 없습니다.

<div id="versioning-strategy">
  ## 버전 관리 전략
</div>

이 전략의 주요 원칙은 다음과 같습니다.

1. 모든 하위 호환성을 깨는 변경 사항은 새 버전에 묶어서 배포됩니다.

2. 새 버전이 발표될 때 기존 버전에 대한 사용 중단(deprecation) 유예 기간은 6개월입니다.

3. 언제든지 API는 동시에 두 개 버전에서의 요청을 허용하지만, 이 두 버전 중 오래된 버전은 지원되지 않습니다.

4. 신규 제품을 더 빠르게 도입할 수 있도록, 이러한 제품은 버전 릴리스 주기와는 별도로 수시로 출시됩니다.

5. 모든 API 응답에는 현재 API 버전을 나타내는 `x-current-api-version` 헤더가 포함되며, 사용 중단된 API endpoint를 호출할 때에는 여기에 더해 `x-api-warn` 헤더도 포함됩니다.

여러 연령 버킷 타기팅 사용 중단과 같이 API에 하위 호환성을 깨는 변경이 필요할 정도의 근본적인 제품 요구 사항 변경이 발생하는 경우, 해당 변경 사항을 알리기 위해 최소 90일 전에 사전 공지를 발송하며, 공지 후 최소 90일이 지난 이후에 이 하위 호환성을 깨는 변경을 배포합니다.

<div id="v9">
  ### v9
</div>

오늘인 2021년 3월 3일, X Ads API 버전 9(v9)이 출시되었습니다. 이번 릴리스는 기능 차이를 줄이고 캠페인 생성을 단순화하며, Cards 및 Mobile App Promotion 엔드포인트에 대한 주요 업데이트를 도입하도록 설계되었습니다.

이전 버전들과 마찬가지로 v9로 마이그레이션하기 위한 6개월 전환 기간이 주어집니다. 2021년 8월 31일부로 기존 Ads API 버전 8(v8)은 더 이상 사용할 수 없습니다. 서비스 중단을 피하기 위해 모든 개발자에게 가능한 한 빨리 Ads API의 최신 버전으로 마이그레이션할 것을 권장합니다.

<Note>
  **참고:** 이번 릴리스를 기준으로 Ads API 버전 7(v7)은 지원이 종료되어 더 이상 사용할 수 없습니다.
</Note>

자세한 내용은 [개발자 포럼의 공지](https://devcommunity.x.com/t/ads-api-version-9/150316)를 참조하세요.

</div>

이전 버전과 마찬가지로 v8로 마이그레이션할 수 있도록 6개월의 전환 기간이 제공됩니다. 2021-03-02부로 Ads API 버전 7은 더 이상 제공되지 않습니다. 서비스 중단을 방지하기 위해 모든 개발자에게 가능한 한 빨리 최신 API 버전으로 마이그레이션할 것을 권장합니다.

자세한 내용은 [Developer Forum의 공지](https://devcommunity.x.com/t/ads-api-version-8/141914)를 참조하세요.

자세한 내용은 [개발자 포럼 공지](https://devcommunity.x.com/t/ads-api-version-7/135093)를 참조하세요.

<div id="v6">
  ### v6
</div>

2019년 8월 28일, X는 일관성 강화와 개발자 경험 개선에 초점을 맞춘 업데이트를 포함한 Ads API v6를 출시합니다.

이번 릴리스에는 트윗을 조회하기 위한 새로운 endpoint, Promoted Accounts에 대한 통계, 이름으로 엔터티를 검색하는 기능, 현재 처리 중인 비동기 분석 작업 개수에 대한 정보가 포함되어 있습니다. 또한 미디어를 사용하는 endpoint와 타게팅 기준 endpoint에 일관성을 부여하기 위한 업데이트를 진행했습니다. 마지막으로 일부 parameter 이름과 response attribute를 소폭 수정하고, Scoped Timeline endpoint는 사용 중단(deprecate)합니다.

자세한 내용은 [developer forum의 공지](https://devcommunity.x.com/t/ads-api-version-6/129060)를 참고하세요.

<div id="v5">
  ### v5
</div>

2019년 2월 28일, X는 확장성과 효율성 향상에 초점을 맞춘 업데이트를 포함하는 Ads API v5를 출시했습니다.

이번 릴리스에는 지정된 기간 동안 어떤 엔티티가 활성 상태였는지 파악하기 위한 새로운 endpoint, Media Creatives(예: X Audience Platform의 인스트림 동영상 및 이미지)에 대한 통계, 카드 URI로 *여러* 카드를 가져오는 기능, 타게팅 기준 및 기타 엔티티를 더 유연하게 조회할 수 있는 기능이 포함되어 있습니다. 추가로, 일부 버그를 수정하고 parameter 이름과 response attribute를 업데이트했습니다. 마지막으로, non-media app 카드와 POST `accounts/:account_id/account_media` endpoint는 사용 중단(deprecated) 처리되었습니다.

이전 버전과 마찬가지로, v5로 마이그레이션하기 위한 6개월 전환 기간이 제공됩니다. **2019-08-28** 이후 Ads API 버전 4는 더 이상 제공되지 않습니다. 모든 파트너가 서비스 중단을 방지하기 위해 가능한 한 빨리 최신 버전의 API로 마이그레이션할 것을 권장합니다. *Ads API 버전 3은 수명이 종료되어 더 이상 제공되지 않습니다.*

#### 신규

**어떤 엔티티가 활성 상태였는지 판단하기**

[Active Entities](/ko/x-ads-api/analytics#active-entities-2) 엔드포인트는 광고 엔티티에 대한 애널리틱스 메트릭이 변경되었는지를 나타냅니다. 애널리틱스 엔드포인트와 함께 사용하도록 설계되었으며, Active Entities는 엔티티 타입과 날짜 범위(최대 90일)를 지정하면 플랫폼에서 애널리틱스를 요청해야 하는 엔티티 ID 배열을 반환합니다. 반환되지 않은 ID는 이후 애널리틱스 요청에서 조회하지 않아야 합니다.

이 엔드포인트는 다음 엔티티 타입을 지원합니다: `CAMPAIGN`, `FUNDING_INSTRUMENT`, `LINE_ITEM`, `MEDIA_CREATIVE`, `PROMOTED_TWEET`.

**`MEDIA_CREATIVE` 통계**

Ads API의 애널리틱스 엔드포인트는 이제 Media Creative 엔티티에 대한 메트릭을 제공합니다. Media Creative는 X Audience Platform에서 인스트림 광고 또는 이미지를 프로모션하는 방식입니다. X Ads UI는 Media Creative 메트릭을 “In-stream videos” 및 “Display creatives” 탭에 표시합니다. [동기식](/ko/x-ads-api/analytics#get-stats-accounts-account-id)과 [비동기식](/ko/x-ads-api/analytics#asynchronous-analytics) 애널리틱스 엔드포인트 모두 이제 `MEDIA_CREATIVE` 엔티티 enum을 지원합니다.

**여러 카드 가져오기**

카드 URI 값으로 단일 카드를 가져오도록 설계된 엔드포인트의 [v3 릴리스](https://devcommunity.x.com/t/ads-api-version-3/100732)를 개선하여, 이제 [GET accounts/:account&#95;id/cards/all](/ko/x-ads-api/creatives#cards-fetch) 엔드포인트를 사용해 *여러* 카드를 가져올 수 있게 되었습니다. 이제 각 카드마다 요청을 보내는 대신, 단일 요청으로 최대 200개의 카드를 가져올 수 있습니다.

다음 두 가지를 유의하세요.

1. URL 경로는 이제 `accounts/:account_id/cards/all`입니다. (이전 경로는 더 이상 사용할 수 없습니다.) 이는 ID로 카드를 가져오도록 설계된 [엔드포인트](/ko/x-ads-api/creatives#cards-fetch)와 일관성을 유지하기 위한 것입니다.
2. 필수 요청 파라미터 이름은 이제 card&#95;uris(복수형)입니다.

**조회 유연성 향상**

[GET accounts/:account&#95;id/targeting&#95;criteria](/ko/x-ads-api/campaign-management#get-accounts-account-id-targeting-criteria) 엔드포인트는 이제 여러 개의 라인 아이템 ID를 지원합니다. 최대 200개의 ID를 받는 line&#95;item&#95;ids 파라미터가 필수입니다. 이전에는 단일 라인 아이템만 허용되어 동기화가 어려웠습니다. 이 변경으로 더 많은 타게팅 정보를 더 짧은 시간 안에 조회할 수 있게 되었습니다.

다음 엔드포인트들 역시 이제 여러 라인 아이템 ID를 지원하며, 이들에 대해서는 line&#95;item&#95;ids 파라미터가 선택 사항입니다.

* [GET accounts/:account&#95;id/line&#95;item&#95;apps](https://developer.x.com/en/docs/x-ads-api/campaign-management/api-reference/line-item-apps#get-accounts-account-id-line-item-apps)
* [GET accounts/:account&#95;id/media&#95;creatives](/ko/x-ads-api/campaign-management#get-accounts-account-id-media-creatives)
* [GET accounts/:account&#95;id/promoted&#95;accounts](/ko/x-ads-api/campaign-management#get-accounts-account-id-promoted-accounts)
* [GET accounts/:account&#95;id/preroll&#95;call&#95;to&#95;actions](/ko/x-ads-api/creatives#preroll-call-to-actions)

<div id="changed">
  #### 변경 사항
</div>

**초안 캠페인 및 라인 아이템 조회**

초안 캠페인 및 라인 아이템을 조회하는 방식이 업데이트되었습니다. 이제 with&#95;draft(boolean) 파라미터를 true로 설정하면 초안 엔티티와 비초안 엔티티를 *모두* 반환합니다. 이는 삭제된 엔티티를 조회하는 방식(`with_deleted` 사용)과 일관됩니다. 이전에는 초안과 비초안 엔티티를 모두 가져오기 위해 최소 두 번의 요청이 필요했습니다. 이제 단일 API 호출로 이를 처리할 수 있습니다.

| **v4** | **v5** |
| :--- | :--- | :--- |
| `draft_only` | `with_draft` |     |

**Network Activation Duration 타기팅**

Ads API에서 Network Activation Duration 타기팅을 추가하면, 응답의 타기팅 타입에 &#95;IN&#95;SEC 접미사가 포함되던 표시 문제가 해결되었습니다. Network Activation Duration은 항상 개월 단위로 표현되므로, 초(second)에 대한 참조가 포함되는 것은 혼란을 줄 수 있었습니다. 이번 수정으로 표현 방식이 일관되게 정리되어 혼란이 줄어듭니다.

| **v4** | **v5** |
| :--- | :--- | :--- |
| `NETWORK_ACTIVATION_DURATION_IN_SEC` | `NETWORK_ACTIVATION_DURATION` |     |

**총 개수 및 커서**

v5에서는 with&#95;total&#95;count와 cursor는 상호 배타적입니다. 요청에서 둘 다를 지정하면 EXCLUSIVE&#95;PARAMETERS 오류 코드를 반환합니다. v5 이전에는 cursor가 지정된 경우 with&#95;total&#95;count는 무시되었습니다. 이 변경 사항을 통해 두 파라미터 간의 관계가 명시적으로 정의되었습니다.

<div id="removed">
  #### 제거된 항목
</div>

다음 세 개의 필드가 Ads API 응답에서 제거됩니다: preview&#95;url, account&#95;id, parent&#95;ids. 이 세 필드를 제거하는 데 필요한 개발 작업은 최소 수준입니다.

* v4에서 카드에 대한 preview&#95;url 응답 매개변수가 항상 null임이 공지되었습니다. 이 마이그레이션의 마지막 단계로, 모든 카드 응답에서 preview&#95;url을 제거합니다.
* 다음 리소스에 대해 account&#95;id 응답 속성이 제거됩니다. 이는 광고 계정 ID가 이미 URL과 request.params 둘 다에 포함되어 있기 때문입니다. (펀딩 인스트루먼트가 이 목록에서 의도적으로 제외된 이유는, 가능한 경우 parent ID는 응답 객체에 포함되어야 하고, 계정 ID가 펀딩 인스트루먼트의 상위 엔터티이기 때문입니다.)
  * Account media
  * App event providers
  * App event tags
  * Campaigns
  * Cards
  * Line items
  * Promotable users
  * Targeting criteria
* [GET accounts/:account&#95;id/targeting&#95;criteria](/ko/x-ads-api/campaign-management#get-accounts-account-id-targeting-criteria) 요청의 경우, parent&#95;ids 필드는 항상 빈 배열이었기 때문에 더 이상 반환되지 않습니다.

**Non-media 앱 카드**

v5에서는 non-media 앱 카드를 더 이상 지원하지 않습니다. [이전에](https://devcommunity.x.com/t/deprecation-announcement-app-download-cards/87807) non-media 앱 카드를 생성하거나 수정하는 기능이 제거된 바 있습니다. 이제 이 리소스에 대한 남아 있는 엔드포인트들이 사용 중단(deprecated)됩니다.

* 참고: 이는 이미지 및 동영상 앱 다운로드 카드에는 **영향을 주지 않습니다.**

**Account media 생성**

POST accounts/:account&#95;id/account&#95;media 엔드포인트는 v5에서 더 이상 사용할 수 없습니다. 이 리소스의 다른 엔드포인트는 영향을 받지 *않습니다*. 이 변경의 이유는 [Media Library](/ko/x-ads-api/creatives#media-library)에 미디어를 추가할 때, 일부 경우 해당 애셋이 *자동으로* Account Media 엔터티로 추가되며, 이미 존재하는 애셋을 Account Media 리소스에 다시 추가하려고 하면 오류가 발생하기 때문입니다. 이는 다음과 같은 경우에 발생합니다.

* Media Library에 추가된 `AMPLIFY_VIDEO` 애셋은 `PREROLL` 크리에이티브 타입의 Account Media 애셋으로 자동 추가됩니다.
* 특정 크기의 이미지를 Media Library에 추가하면 자동으로 Account Media 애셋으로 추가됩니다. 크리에이티브 타입(예: `INTERSTITIAL`)은 이미지 크기에 따라 달라집니다. (크기에 대한 내용은 [Enumerations](/ko/x-ads-api/introduction) 페이지를 참조하세요.)

<div id="v4">
  ### v4
</div>

Ads API 버전 4가 오늘인 2018년 8월 28일에 출시됩니다.

이번 릴리스에는 [Audiences](/ko/x-ads-api/audiences) 제품 개선 사항이 포함되어 있으며, 더 견고한 오디언스 처리 백엔드를 기반으로 하는 새로운 API 인터페이스가 도입되었습니다. 버전 4에는 사용자, 계정, 세금 설정을 관리하기 위한 일련의 엔드포인트도 포함되어 있습니다. 추가로, accounts/:account&#95;id/videos 엔드포인트는 사용 중단될 예정입니다. 또한 이번 릴리스에는 몇 가지 작은 파라미터 및 응답 이름 변경도 포함되어 있습니다.

버전 3과 마찬가지로 6개월 전환 기간을 제공합니다. **2019-02-28** 이후에는 Ads API 버전 3을 더 이상 사용할 수 없습니다. 모든 파트너가 서비스 중단을 방지하기 위해 가능한 한 빨리 최신 버전의 API로 마이그레이션할 것을 권장합니다. 버전 관리 전략에 대한 자세한 내용은 [Versions](/ko/x-ads-api/introduction) 페이지를 참고하세요.

<div id="new">
  #### New
</div>

***Audience API***

새로운 Audiences API는 향상된 안정성과 신뢰성을 제공하는 새로운 오디언스 처리 백엔드를 기반으로 구축되었습니다. 이 새로운 엔드포인트를 통해 파트너는 단일 사용자에 대해 여러 사용자 식별자 유형을 제공할 수 있으며, 이는 매칭을 위해 추가적인 신호를 활용할 수 있음을 의미합니다. 새로운 Audience 엔드포인트에 대한 참고 문서는 [여기](/ko/x-ads-api/audiences)에서 확인할 수 있습니다. 올해 남은 기간 동안 이 제품에 대한 업데이트와 개선 사항을 지속적으로 제공할 예정입니다.

다음 엔드포인트는 중복된 기능 때문에 v4에서는 더 이상 제공되지 않습니다. (v3에서는 계속 동작하며, v3가 더 이상 제공되지 않을 때 완전히 **중단**됩니다.)

* TON Upload:
  * GET accounts/:account&#95;id/tailored&#95;audience&#95;changes
  * GET accounts/:account&#95;id/tailored&#95;audience&#95;changes/:tailored&#95;audience&#95;change&#95;id
  * POST accounts/:account&#95;id/tailored&#95;audience&#95;changes
  * PUT accounts/:accounti&#95;d/tailored&#95;audiences/global&#95;opt&#95;out
* Real Time Audiences:
  * POST tailored&#95;audience&#95;memberships

마지막으로, 버전 4에서는 *모든* [Tailored Audiences 엔드포인트](/ko/x-ads-api/audiences)의 요청 및 응답에서 `list_type` 파라미터가 제거됩니다.

***Settings Endpoints***

이제 계정 관리자는 사용자, 계정, 세금 설정을 구성하고 업데이트할 수 있습니다. [User settings](/ko/x-ads-api/campaign-management#get-accounts-account-id-user-settings-user-id)는 특정 광고 계정에 대한 사용자별 연락 선호 설정에 해당합니다. [PUT accounts/:account&#95;id](/ko/x-ads-api/campaign-management#put-accounts-account-id) 엔드포인트를 사용하면 광고주는 계정 이름과 업종 유형을 업데이트할 수 있습니다. 마지막으로, [tax settings](/ko/x-ads-api/campaign-management#get-accounts-account-id-tax-settings) 엔드포인트를 통해 부가가치세(VAT)가 부과되는 국가의 광고주는 회사명, 주소, VAT ID, 계정이 광고주 소유인지 또는 광고주를 대신해 광고하는 에이전시 소유인지와 같은 정보를 업데이트할 수 있습니다.

**모든 곳에서 `country_code` 사용**

Ads API 전반의 [일관성](https://devcommunity.x.com/t/feedback-ads-api-consistency/109145)을 높이기 위한 작업의 일환으로, 다음 엔드포인트에서 파라미터 이름을 `app_country_code`에서 `country_code`로 변경했습니다.

* [POST accounts/:account&#95;id/cards/image&#95;app&#95;download](https://devcommunity.x.com/t/ads-api-version-11/168814)
* [PUT accounts/:account&#95;id/cards/image&#95;app&#95;download/:card&#95;id](https://developer.x.com/content/developer-twitterhttps://developer.x.com/en/docs/ads/creatives/api-reference/image-app-download#put-accounts-account-id-cards-image-app-download-card-id)
* [POST accounts/:account&#95;id/cards/video&#95;app&#95;download](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/api-reference/video-app-download#post-accounts-account-id-cards-video-app-download)
* [PUT accounts/:account&#95;id/cards/video&#95;app&#95;download/:card&#95;id](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/api-reference/video-app-download#put-accounts-account-id-cards-video-app-download-card-id)

이 변경은 해당 파라미터의 동작이나 허용되는 값에는 영향을 주지 않으며, 단순한 이름 변경입니다.

**`preview_url`은 항상 null**

v3 공지에서 약속한 대로, 이제 모든 기존 카드에는 `card_uri`가 있습니다. 그 결과 `preview_url` 값은 항상 `null`입니다.

참고로, 카드의 `card_uri` 값을 사용하여 카드를 트윗과 연결할 수 있습니다. 아래 예시 요청을 참조하세요.

$ twurl -X POST -H ads-api.x.com &quot;/4/accounts/18ce54d4x5t/tweet?text=Version 4&amp;card&#95;uri=card://958225772740714496&quot;

#### 제거됨

**동영상 엔드포인트**

accounts/:account&#95;id/videos 엔드포인트는 v4에서 더 이상 제공되지 않습니다. 이 엔드포인트는 [Media Library](/ko/x-ads-api/creatives#get-accounts-account-id-media-library) 엔드포인트의 도입으로 대체되었습니다. 아래의 사용 예를 비교해 보세요.

* v3 동영상 엔드포인트: `twurl -H ads-api.x.com "/3/accounts/18ce54d4x5t/videos"`

* v4 동영상용 Media Library 엔드포인트: `twurl -H ads-api.x.com "/4/accounts/18ce54d4x5t/media_library?media_type=VIDEO"`

Media Library 엔드포인트는 videos 엔드포인트와 기능 면에서 완전히 동일하며, 이미지 및 GIF 처리와 같은 추가 기능도 지원합니다. 파트너는 모든 미디어 관리에 대해 Media Library만 사용해 주시기 바랍니다.

**트윗 보기에서의 `as_user_id`**

[GET accounts/:account&#95;id/tweet/preview/:tweet&#95;id](/ko/x-ads-api/creatives#get-accounts-account-id-tweets) 엔드포인트에서 사용 가능하던 `as_user_id` 파라미터는 더 이상 허용되지 않습니다. 미리보기는 항상 트윗 작성자 기준으로 표시됩니다.

<div id="v3">
  ### v3
</div>

Ads API 버전 3는 2018년 2월 1일에 [출시](https://devcommunity.x.com/t/ads-api-version-3/100732)되었습니다. Ads API 버전 2는 2018년 8월 1일에 수명 종료(EOL)를 맞이합니다.

이번 릴리스에는 새로운 Audience Intelligence 제품, Media Library 접근, 그리고 개선된 카드 워크플로가 포함됩니다. 또한 [PUT accounts/:account&#95;id/targeting&#95;criteria](/ko/x-ads-api/campaign-management#get-accounts-account-id-targeting-criteria) 엔드포인트 사용 중단을 발표합니다. 마지막으로, 버전 3에는 일부 소규모 파라미터 및 응답 변경 사항과 더 낮은 배치 크기 한도가 포함되어 있습니다.

[버전 2](https://devcommunity.x.com/t/ads-api-version-2/90360)와 마찬가지로 파트너에게 전환을 위한 **6개월**을 제공합니다. **2018-08-01**에 Ads API v2는 중단됩니다. 모든 파트너와 개발자에게 가능한 한 빨리 v3로 마이그레이션할 것을 권장합니다.

**Audience Intelligence**

Audience Intelligence는 특정 X 오디언스와 가장 관련성이 높은 상위 해시태그, @핸들, 이벤트에 대한 실시간 인사이트를 제공합니다. 예를 들어 미국의 남성 18-34세를 입력하면, 이 오디언스에서 #nintendoswitch, #cardinal, @ricegum이 트렌드로 나타나는 것을 볼 수 있습니다.

Audience Intelligence [엔드포인트](/ko/x-ads-api/audiences)는 다음 기능을 제공합니다.

* 입력 오디언스를 기준으로 상위 관련 해시태그, @핸들, 이벤트를 조회합니다.
* 입력 오디언스를 기준으로 핵심 인구통계 정보(예: 연령, 성별, 가구 소득)를 조회합니다.
* 키워드를 기준으로 트윗 볼륨 시계열을 조회합니다.

**Media Library**

[Media Library](/ko/x-ads-api/creatives#get-accounts-account-id-media-library)는 광고 계정의 이미지, GIF, 비디오를 관리할 수 있는 기능을 제공합니다. 이러한 미디어 오브젝트는 트윗과 카드 생성에 사용할 수 있습니다. 또한 여러 크리에이티브에서 재사용할 수 있으므로, 동일한 에셋을 여러 번 업로드할 필요가 없습니다.

라이브러리의 오브젝트는 media&#95;key로 식별됩니다. media&#95;key는 예를 들어 13&#95;875943225764098048과 같은 형식의 문자열 값입니다. Ads API에서는 모든 미디어에 대해 media&#95;key를 사용하는 방향으로 전환하고 있습니다.

**개선된 카드 워크플로**

모든 카드 엔드포인트가 이제 media&#95;key를 지원합니다. 이를 통해 Media Library의 오브젝트를 사용하여 카드를 생성하거나 업데이트할 수 있습니다.

또한 [카드 세부 정보 조회](/ko/x-ads-api/creatives#fetching-cards)를 위한 두 개의 새로운 엔드포인트를 도입했습니다. 이 엔드포인트는 트윗 또는 예약된 트윗에 사용된 카드를 조회하는 데 사용할 수 있으며, 예를 들어 card&#95;uri 또는 id를 지정해 조회할 수 있습니다. 이전에는 이러한 작업이 불가능했습니다.

<div id="other-changes">
  #### 기타 변경 사항
</div>

이러한 새 기능 외에도 버전 3에는 다음과 같은 변경 사항이 포함됩니다.

**신규**

* [GET insights/keywords/search](/ko/x-ads-api/audiences) 엔드포인트 응답에 이제 입력 키워드와 관련된 30개의 용어를 포함하는 related&#95;keywords 속성이 추가되었습니다.

**변경 사항**

* 타겟팅 기준 배치의 최대 크기는 이제 500입니다.
* card&#95;uri 및 preview&#95;url 응답 속성은 이제 서로 배타적입니다. 카드에 card&#95;uri가 있는 경우 preview&#95;url은 null이 됩니다. 카드에 card&#95;uri가 없는 경우 preview&#95;url만 반환됩니다.
  * 2018-01-29 이후 생성된 모든 카드는 card&#95;uri를 갖게 됩니다.
  * 버전 4까지는 기존의 모든 카드에 card&#95;uri가 추가됩니다.
* 5:2 이미지를 사용하는 카드를 *생성* 하는 것은 더 이상 불가능합니다. *기존* 5:2 이미지 기반 카드는 계속 동작하지만, 파트너가 성과가 더 좋은 1.91:1 또는 1:1 가로세로 비율(지원되는 경우)로 전환하는 것을 권장합니다.

**제거됨**

* [PUT accounts/:account&#95;id/targeting&#95;criteria](/ko/x-ads-api/campaign-management#get-accounts-account-id-targeting-criteria) 엔드포인트는 더 이상 사용할 수 없습니다. 이 엔드포인트의 replace 동작이 광고주에게 혼란을 주었고, 한 번에 단일 리소스를 업데이트하는 다른 PUT 엔드포인트와도 일관되지 않았기 때문에 이러한 변경을 결정했습니다. 대신, 파트너는 [POST batch/accounts/:account&#95;id/targeting&#95;criteria](/ko/x-ads-api/campaign-management#post-batch-accounts-account-id-targeting-criteria) 엔드포인트를 사용해야 하며, 이는 단일 요청에서 타겟팅을 추가 및 제거할 수 있는 기능 등 더 큰 유연성을 제공합니다.
* paused 응답 속성은 더 이상 자금 조달 수단(funding instruments)에 대해 반환되지 않습니다. 대신, 자금 조달 수단이 일시 중지되었는지 여부를 판단하려면 entity&#95;status 응답 속성을 확인해야 합니다. 추가로, paused와 cancelled가 동일한 값을 나타내기 때문에 cancelled 역시 응답에서 더 이상 반환되지 않습니다.
* [GET accounts/:account&#95;id/tweet/preview](/ko/x-ads-api/creatives#get-accounts-account-id-tweets) 엔드포인트에서 card&#95;id 매개변수를 제거했습니다.
* 삭제된 예약 트윗(Scheduled Tweets)을 조회하는 것은 불가능하므로, with&#95;deleted 매개변수는 더 이상 지원되지 않습니다.
* 다음 엔드포인트에서는 해당 엔티티가 초안 상태가 될 수 없기 때문에 draft&#95;only 매개변수가 제거되었습니다:
  * [GET accounts/:account&#95;id/targeting&#95;criteria](/ko/x-ads-api/campaign-management#get-accounts-account-id-targeting-criteria)
  * [GET accounts/:account&#95;id/promoted&#95;tweets](/ko/x-ads-api/campaign-management#get-accounts-account-id-promoted-tweets)
  * [GET accounts/:account&#95;id/promoted&#95;accounts](/ko/x-ads-api/campaign-management#get-accounts-account-id-promoted-accounts)
  * [GET accounts/:account&#95;id/media&#95;creatives](/ko/x-ads-api/campaign-management#get-accounts-account-id-media-creatives)
  * [GET accounts/:account&#95;id/preroll&#95;call&#95;to&#95;actions](/ko/x-ads-api/creatives#get-accounts-account-id-preroll-call-to-actions)

**참고**

Video Website Cards와 예약 트윗(Scheduled Tweets)은 이제 베타 단계를 벗어났습니다. 출시 이후 예약 트윗에 대해 우리가 수행한 변경 사항은 [이 스레드](https://devcommunity.x.com/t/announcement-update-to-scheduled-tweets/94869)를 참조하십시오. 여기에는 예약 트윗에 대해 HTML [미리보기](https://devcommunity.x.com/t/announcement-tweet-preview-improvements/96577)를 생성할 수 있는 기능이 포함됩니다.

**하위 호환성 중단 변경/지원 중단 사항[¶](#breaking-changes-deprecations "Permalink to this headline")**

* `total_count`는 이제 선택적 응답 속성입니다. `with_total_count`가 `true`로 설정된 경우에만 제공됩니다.

* `line_items` 및 `campaigns` 요청 및 응답 객체의 `paused` 및 `draft_only` 필드는 단일 `entity_status` 파라미터로 대체되었습니다.

* [POST accounts/:account&#95;id/tweet](/ko/x-ads-api/creatives) 및 [GET accounts/:account&#95;id/tweet/preview](/ko/x-ads-api/creatives#get-accounts-account-id-tweets) 엔드포인트에서 `status` 파라미터 이름이 `text`로 변경되었습니다.

* [GET targeting&#95;criteria/locations](/ko/x-ads-api/campaign-management#get-targeting-criteria-locations) 엔드포인트의 `location_type` enum 값은 이제 복수형입니다. `COUNTRY`는 `COUNTRIES`, `REGION`은 `REGIONS` 등이 되었습니다. 한 가지 예외는 v2에서 `CITY`가 이제 `METROS`가 되었다는 점으로, 이는 위치 유형이 Designated Marker Areas(DMA) 또는 “metros”를 가리킨다는 사실을 정확히 반영하기 위한 것입니다.

* PUT accounts/:account&#95;id/promoted&#95;tweets 엔드포인트의 `display_properties` 속성이 제거되었습니다. 이 값은 응답의 일부로도 더 이상 반환되지 않습니다.

* 이전 항목의 결과로, 더 이상 promoted&#95;tweets 엔티티를 업데이트(PUT)할 수 없습니다.

* [GET accounts/:account&#95;id/promoted&#95;tweets](/ko/x-ads-api/campaign-management#get-accounts-account-id-promoted-tweets) 엔드포인트의 `line_item_id` 파라미터가 제거되었습니다.

* v2 엔드포인트에서는 더 이상 5:2 Website Card를 생성할 수 없습니다.

* `data_type` 응답 속성은 더 이상 반환되지 않습니다.

**Notes[¶](#notes "Permalink to this headline")**

* 초안 라인 아이템 또는 캠페인은 `entity_status`가 `DRAFT`인 상태에서만 `PAUSED` 또는 `ACTIVE`로 전환할 수 있습니다.
* 여러 라인 아이템을 포함한 전체 캠페인을 활성화하려면, 캠페인 아래의 각 라인 아이템과 캠페인 자체 모두의 `entity_status`를 `ACTIVE`로 설정해야 합니다.
* 캠페인 또는 라인 아이템의 `entity_status`를 변경하려면, 해당하는 PUT 엔드포인트를 사용하십시오.

* 도달 범위 추정의 정확성을 높이기 위해 이제 이 endpoint는 예산을 고려하도록 변경되었습니다. 다음 매개변수가 이제 필수입니다:
  * [신규] `campaign_daily_budget_amount_local_micro`
  * `currency`
  * `bid`
  * `objective`
* 응답 객체 형식이 변경되어 이제 응답 값에 대한 범위를 반환합니다.
* `infinite_count`는 용도를 더 명확히 하기 위해 `infinite_bid_count`로 이름이 변경되었습니다
* `count` 및 `infinite_bid_count` 외에도 이제 다음과 같은 새로운 데이터 포인트가 반환됩니다:`
  * `impressions`
  * `engagements`
  * `estimated_daily_spend_local_micro`
* Tailored Audiences 데이터 유형 변경
* Tailored Audiences에 대한 `data_type` 값이 모든 응답에서 `tailored_audiences`에서 `tailored_audience`로 변경되었습니다.
* 공유 맞춤 오디언스(Shared Tailored Audiences)는 이제 *API 전용* 베타로 제공됩니다. 공유 맞춤 오디언스를 사용하면 하나의 오디언스를 여러 광고 계정에서 사용할 수 있습니다. 여러 광고 계정에서 공유하려는 맞춤 오디언스의 권한을 관리하려면 [POST accounts/:account&#95;id/tailored&#95;audiences/:tailored&#95;audience&#95;id/permissions](/ko/x-ads-api/audiences) 및 관련 엔드포인트를 사용하세요.
* 광고주 계정의 성과 [analytics](/ko/x-ads-api/analytics)를 수집하는 방식이 크게 개선되었습니다:

* 응답이 **대폭 단순화**되었으며, 이제 메트릭이 UI에 표시되는 방식과 더욱 가깝게 일치합니다.
  * 이전에는 각 게재 위치(검색의 Promoted Tweets, 타임라인의 Promoted Tweets, 프로필 및 트윗 상세의 Promoted Tweets, X Audience Platform)에 대해 각각 별도의 메트릭을 제공했습니다. 이제는(`promoted_tweet_timeline_impressions`, `promoted_tweet_search_impressions`, `promoted_tweets_profile_impressions`, `promoted_tweets_tpn_impressions` 대신) 각 위치에 대해 표준화된 메트릭 세트를 반환하며, 아래 카테고리 중 하나로 요청했을 때 단일 메트릭인 `impressions`로 제공됩니다(모든 메트릭에 동일하게 적용됩니다).
  * `ALL_ON_TWITTER`
  * `PUBLISHER_NETWORK`
  * 요청을 보내면 UI의 값과 쉽게 일치시킬 수 있도록 단일 `impressions` 메트릭만 받게 됩니다.
  * `ALL_ON_TWITTER`와 `PUBLISHER_NETWORK` 데이터를 모두 얻으려면, 이 둘은 결합할 수 없기 때문에 두 번의 쿼리를 수행해야 합니다.

* v0 메트릭과 v1 메트릭 간 매핑이 포함된 analytics v1 마이그레이션 가이드를 참고하세요
