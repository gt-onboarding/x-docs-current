정렬은 대부분의 Ads API 컬렉션 엔드포인트(객체 목록을 반환하는 엔드포인트)에서 사용할 수 있습니다. 엔드포인트에 따라 정렬에 사용할 수 있는 다양한 매개변수가 제공됩니다. 이 매개변수를 통해 반환되는 대부분의 필드(`id` 및 ENUM 제외)는 정렬할 수 있습니다.

| API 컬렉션 | 정렬 가능한 매개변수 |
| :--- | :--- |
| [Accounts](/ko/x-ads-api/campaign-management#accounts) | `created_at`, `updated_at`, `deleted`, `name` |
| [Funding Instruments](/ko/x-ads-api/campaign-management#funding-instruments) | `created_at`, `updated_at`, `deleted`, `funded_amount_local_micro`, `start_time`, `end_time` |
| [Campaigns](/ko/x-ads-api/campaign-management#campaigns) | `created_at`, `updated_at`, `deleted`, `name`, `start_time`, `end_time`, `daily_budget_amount_local_micro`, `total_budget_amount_local_micro`, `standard_delivery` |
| [Line Items](/ko/x-ads-api/campaign-management#line-items) | `created_at`, `updated_at`, `deleted`, `bid_amount_local_micro` |
| [Cards](https://developer.x.com/en/docs/ads/creatives/api-reference/image-app-download) | `created_at`, `updated_at`, `deleted`, `name` |
| [Promoted Accounts](/ko/x-ads-api/campaign-management#advertiser-api) | `created_at`, `updated_at`, `deleted`, `paused` |
| [Promoted Tweets](/ko/x-ads-api/campaign-management#promoted-tweets) | `created_at`, `updated_at`, `deleted`, `paused` |
| [App Event Tags](/ko/x-ads-api/measurement/mobile-conversions#app-event-tags) | `created_at`, `updated_at`, `deleted`, `post_view_attribution_window`, `post_engagement_attribution_window`, `assisted_conversion`, `provider_app_event_name` |

<div id="getting-started">
  ## 시작하기
</div>

API 요청에 `sort_by=[attribute name]-[asc/desc]`를 추가하기만 하면 됩니다. 여기서 attribute는 GET 요청의 JSON 응답에 포함되어 반환되는 유효한 속성 이름이어야 합니다.

예를 들어, 이름을 알파벳 순으로 정렬하려면 `?name-asc`를, 역순으로 정렬하려면 `?name-desc`를 사용할 수 있습니다.
