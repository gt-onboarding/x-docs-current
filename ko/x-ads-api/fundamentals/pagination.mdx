계정과 연관된 캠페인의 최대 허용 개수를 늘리고 계정에 속한 모든 엔티티를 효율적으로 조회할 수 있도록, Advertiser API는 이제 여러 GET 엔드포인트에서 페이지네이션을 지원합니다. 이 페이징 메커니즘은 사용하기 쉽고, [커서를 사용한 컬렉션 탐색](https://developer.x.com/en/docs/x-api/v1/pagination)에 설명된 REST API의 커서 기반 페이지네이션과 매우 유사합니다.

<div id="getting-started">
  ## 시작하기
</div>

특정 GET 요청의 경우 이제 선택적인 `cursor` 및 `count` 쿼리 매개변수를 지원합니다. 응답에 `count` 값보다 많은 엔티티가 포함된 경우, 해당 엔드포인트는 처음 `count`개의 엔티티와 함께 응답 JSON에 `next_cursor` 키를 반환합니다.

`GET https://ads-api.x.com/5/accounts/abc1/campaigns?count=50`

```json
{
  "data": \[...\],
  "next_cursor": "c-3yvu1pzhd3i7",
  "request": {...}
}
```

다음 일련의 응답을 가져오려면 쿼리 파라미터에 `cursor`를 추가합니다:

`GET https://ads-api.x.com/5/accounts/abc1/campaigns?cursor=c-3yvu1pzhd3i7&count=50`

```json
{
  "data": \[...\],
  "next_cursor": "c-3w3zdyg8ywan",
  "request": {...}
}
```


<div id="going-deeper">
  ## 더 깊이 알아보기
</div>

대부분의 엔드포인트에서 최대 `count` 값은 **1,000**, 최소값은 **1**, 기본값은 **200**입니다.

`next_cursor`에서 제공되는 값은 항상 문자열이며, 내부 구조를 알 수 없는 불투명한 값으로 간주해야 합니다. 구현 방식은 변경될 수 있습니다. 현재 결과 집합의 페이지에서 반환되는 엔티티 수가 `count`보다 적으면 `next_cursor` 값은 `null`이 됩니다.

**참고:** analytics 엔드포인트는 이러한 방식의 페이지네이션을 지원하지 않습니다. stats 엔드포인트에서는 기간을 지정하여 페이지네이션을 수행합니다. 자세한 내용은 해당 개별 엔드포인트 문서를 참조하세요.
