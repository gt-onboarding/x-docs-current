<div id="timezones-accounts-and-billing">
  ## 시간대, 계정 및 결제
</div>

Datetime 값은 항상 UTC 시간으로 반환됩니다(값 끝에 붙은 Z 표기로 구분됩니다). Datetime 값은 POST 또는 PUT 명령에서 ISO 8601 표준 시간대 형식을 사용해 어떤 시간대이든 지정할 수 있습니다. 시간은 [ISO-8601](http://en.wikipedia.org/wiki/ISO_8601) 표준의 일부 형식을 사용해 표현됩니다. 보다 구체적으로, 당사 날짜 형식에 대한 strptime 문자열은 `%Y-%m-%dT%l:%M:%S%z`입니다. 광고주의 계정 시간대가 공식 결제 수치가 확정되는 실제 시간을 결정합니다.

계정 수준에서 API를 조회할 때([GET accounts](/ko/x-ads-api/campaign-management#accounts)), 다음과 같은 시간대 정보를 확인할 수 있습니다:

```json
    {
      "request": {
        "params": {
          "account_id": "18ce54d4x5t"
        }
      },
      "data": {
        "name": "API McTestface",
        "business_name": null,
        "timezone": "America/Los_Angeles",
        "timezone_switch_at": "2016-07-21T07:00:00Z",
        "id": "18ce54d4x5t",
        "created_at": "2016-07-21T22:42:09Z",
        "salt": "54cb7b5a34183f77d82dd6d0f4329691",
        "updated_at": "2017-09-09T06:42:14Z",
        "business_id": null,
        "approval_status": "ACCEPTED",
        "deleted": false
      }
    }
```

여기에는 두 개의 시간대 값이 포함됩니다: `timezone`(개요는 [위키피디아의 timezones](https://en.wikipedia.org/wiki/Tz_database#Names_of_time_zones)를 참고)와 `timezone_switch_at`입니다. `timezone_switch_at`은 UTC 시간대(+00:00) 기준으로 제공되지만, 언제나 지정된 시간대의 자정(00:00)을 나타낸다는 점에 유의하십시오.

광고주의 시간대는 API를 통해 수정할 수 없습니다. 이 속성은 X의 광고주 계정 담당자가 계약/청구 단계에서 설정합니다.

계정이 America/Los&#95;Angeles 시간대에서 새 로컬 시간대로 전환되는 날에는 데이터 공백이 발생하므로, 보고서를 생성하고 분석용 엔드포인트를 조회할 때 `timezone_switch_at` 값을 반드시 고려해야 합니다.

<div id="specifying-datetime-values-with-timezone">
  ### 타임존을 포함한 Datetime 값 지정
</div>

Datetime 값은 항상 UTC 시간으로 반환됩니다(이는 datetime 값 끝의 `Z`로 표시됩니다). Datetime 값은 POST 또는 PUT 명령에서 타임존을 나타내는 [ISO 8601 표준](http://en.wikipedia.org/wiki/ISO_8601) 형식을 사용하여 어느 타임존이든 지정할 수 있습니다. 예를 들어, `2017-07-10T08:00:00-0800`은 허용되는 입력 값이며, 자동으로 UTC 값인 `2017-07-10T16:00:00Z`로 변환됩니다.

analytics 엔드포인트에서 `granularity`가 `DAY` 또는 `TOTAL`인 경우, `start_time` 값은 계정 보유자의 로컬 타임존 기준으로 원하는 날짜의 자정으로 지정해야 합니다. 사용되는 타임존 오프셋은 해당 날짜의 오프셋이 아니라 현재 날짜의 오프셋입니다. 예를 들어, America/Los&#95;Angeles에 있는 ads 계정이 태평양 일광 절약 시간제(Pacific Daylight Savings Time)에 속해 있는 경우 UTC 오프셋은 -0700입니다. 따라서 analytics 요청에서는 시간을 `start_time=2017-05-21T07:00:00Z` 또는 `start_time=2017-05-21T00:00:00-0700`로 지정해야 합니다. ads 계정이 Asia/Tokyo에 있고 이 지역의 오프셋이 항상 +09:00인 경우 값은 `start_time=2017-05-20T15:00:00Z` 또는 `start_time=2017-05-21T00:00:00+0900`로 지정해야 합니다.
