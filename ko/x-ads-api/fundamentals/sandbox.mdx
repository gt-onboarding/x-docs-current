샌드박스는 Ads API 구현을 테스트하고 구축할 수 있는 환경입니다.

샌드박스를 통해 생성된 캠페인은 사용자에게 게재되지 않으며, Funding Instruments와 같은 객체에 대한 테스트 레코드를 동적으로 생성할 수 있습니다.

샌드박스를 사용하는 것은 적절한 호스트 이름인 ads-api-sandbox.x.com을 호출하는 것만큼 간단하지만, 샌드박스에서만 제공되는 고유한 기능도 포함하고 있습니다.

새 광고주 계정을 생성하려면 먼저 [POST accounts](/ko/x-ads-api/campaign-management#post-accounts)에 요청을 보내세요. 이 요청에 대한 응답에는 이후 요청에 사용할 수 있는 account&#95;id가 포함됩니다. 이러한 엔드포인트를 사용하면 여러 광고주 계정을 생성하고 각각에 대해 테스트할 수 있습니다.

<div id="sandbox-only-endpoint-list">
  ### 샌드박스 전용 엔드포인트 목록:
</div>

* [POST accounts](/ko/x-ads-api/campaign-management#post-accounts) 샌드박스에서 계정을 생성합니다
* [POST accounts/:account&#95;id/features](/ko/x-ads-api/campaign-management#post-accounts-account-id-features) 샌드박스 계정에 계정 기능을 추가합니다
* [POST accounts/:account&#95;id/funding&#95;instruments](/ko/x-ads-api/campaign-management#post-accounts-account-id-funding-instruments) 샌드박스 계정에 결제 수단을 추가합니다
* [DELETE accounts/:account&#95;id](/ko/x-ads-api/campaign-management#delete-accounts-account-id) 샌드박스에서 계정을 삭제합니다
* [DELETE accounts/:account&#95;id/features](/ko/x-ads-api/campaign-management#delete-accounts-account-id-features) 샌드박스 계정에서 계정 기능을 삭제합니다
* [DELETE accounts/:account&#95;id/funding&#95;instruments/:funding&#95;instrument&#95;id](/ko/x-ads-api/campaign-management#delete-accounts-account-id-funding-instruments-funding-instrument-id) 샌드박스 계정에서 결제 수단을 삭제합니다
