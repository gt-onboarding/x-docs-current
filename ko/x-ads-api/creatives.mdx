<div id="overview">
  ## 개요
</div>

크리에이티브는 캠페인에서 프로모션 대상으로 사용할 수 있는 모든 개체를 의미합니다. 게시물에는 텍스트, 이미지, GIF, 동영상 또는 카드가 포함될 수 있습니다. 카드는 이미지 또는 동영상을 포함할 수 있습니다.

이미지, GIF 또는 동영상 크리에이티브는 [POST media/upload](/ko/x-api/media/upload-media#post-media-upload) — 이미지만 지원하는 간단한 업로드 엔드포인트 — 또는 POST media/upload (chunked) 엔드포인트를 사용하여 업로드합니다. 이렇게 업로드한 크리에이티브는 다음 항목에 추가할 수 있습니다.

cards:

* POST accounts/:account&#95;id/cards
  Tweets:
* POST accounts/:account&#95;id/tweets - 트윗에 카드를 추가하려면 card&#95;uri 파라미터를 사용하세요.
  Scheduled Tweets:
* POST accounts/:account&#95;id/scheduled&#95;tweets

카드에 대한 자세한 내용은 Cards 페이지를 참조하세요. Promoted Video 페이지에서는 카드 또는 트윗과 동영상을 연결하는 방법을 설명합니다.

<div id="cards">
  ### 카드
</div>

Ads API는 캠페인에서 프로모션할 트윗에 사용할 수 있는 여러 카드 유형을 지원합니다. **참고**: 트윗이 게시되면 카드 상세 정보는 공개되며, 여기에는 카드를 소유한 사용자의 정보가 포함될 수 있습니다.

<div id="image">
  #### 이미지
</div>

다음 이미지 사양은 [Cards](/ko/x-ads-api/creatives#cards-2)에 사용되는 에셋에 적용됩니다. 이미지는 3MB 이하여야 하며, 가로가 최소 800px 이상이어야 합니다. 또한 다음과 같은 가로:세로 비율을 지원합니다.

* Website: 1:1 및 1.91:1
* Image App Download: 1:1 및 1.91:1
* Poll: 1.91:1
* Image Conversation: 1.91:1
* Image Direct Message: 1.91:1

다음 이미지 형식을 지원합니다: .bmp, .jpeg 및 .png.

<div id="video">
  #### 비디오
</div>

다음 비디오 사양은 [Cards](/ko/x-ads-api/creatives#cards-2)에 사용되는 소재에 적용됩니다. 다음과 같은 가로:세로 비율을 지원합니다.

* Video Website: 16:9 및 1:1
* Video App Download: 16:9 및 1:1
* Poll: 16:9
* Video Conversation: 16:9
* Video Direct Message: 16:9

<div id="promoted-video">
  ### 프로모션 비디오
</div>

이 문서는 Ads API를 통해 비디오를 업로드하고 프로모션하는 절차에 대한 간단한 개요를 제공합니다.

Ads API는 [트윗](/ko/x-ads-api/creatives#tweets-2)과 다음 카드에서 프로모션 비디오를 지원합니다:

* [Video Website](https://devcommunity.x.com/t/ads-api-version-11/168814)
* [Video App Download](https://devcommunity.x.com/t/ads-api-version-11/168814)
* [Video Conversation](/ko/x-ads-api/creatives#video-conversation-cards)

먼저 [POST media/upload (chunked)](https://developer.x.com/en/docs/media/upload-media/api-reference/post-media-upload-init.html) 엔드포인트를 사용하여 비디오를 업로드합니다. 그런 다음 `media_id`를 사용해 POST accounts/:account&#95;id/videos 엔드포인트로 광고 계정에 비디오를 연결합니다. 비디오의 `id`는 `media_key`라고도 하며, 이후 요청에서 사용됩니다. 이는 정수로 시작하고, 그 뒤에 밑줄이 오며, 마지막에 long 값이 붙는 문자열입니다. 예를 들면 다음과 같습니다: `13_875943225764098048`.

<div id="promoted-video-in-tweets">
  #### 트윗 내 Promoted Video
</div>

트윗을 생성하려면 비디오의 `id`와 함께 [POST accounts/:account&#95;id/tweet](/ko/x-ads-api/creatives#get-accounts-account-id-tweets) 엔드포인트를 사용합니다. 이 단계에서 비디오 제목, 설명, 콜투액션(CTA)도 제공할 수 있습니다. 이 값들은 사용자에게 표시됩니다.

<div id="promoted-video-in-cards">
  #### 카드 내 프로모션 동영상
</div>

Video App Download 카드와 Video Conversation 카드는 포스터 이미지를 추가하는 기능을 지원합니다. 이러한 카드에서 사용할 이미지를 [POST media/upload](https://developer.x.com/en/docs/media/upload-media/api-reference/post-media-upload.html) 엔드포인트를 사용해 업로드합니다.

다음 엔드포인트 중 하나를 사용해 카드를 생성합니다:

* [POST accounts/:account&#95;id/cards/video&#95;website](https://devcommunity.x.com/t/ads-api-version-11/168814)
* [POST accounts/:account&#95;id/cards/video&#95;app&#95;download](https://devcommunity.x.com/t/ads-api-version-11/168814)
* [POST accounts/:account&#95;id/cards/video&#95;conversation](https://devcommunity.x.com/t/ads-api-version-11/168814)

동영상의 `id`와, 선택적으로 이미지의 `media_id`(포스터 이미지용)를 사용합니다.

마지막으로, POST accounts/:account&#95;id/tweet 엔드포인트를 사용해 트윗을 생성합니다. 카드는 `card_uri` 매개변수를 사용해 트윗에 첨부됩니다.

<div id="general-information">
  #### 일반 정보
</div>

API를 통한 비디오 업로드에 대한 자세한 안내는 [비디오 업로드 가이드](https://developer.x.com/en/docs/media/upload-media/uploading-media.html)를 참고하세요.

비디오는 프리롤 소재로도 프로모션할 수 있습니다. 자세한 설명은 [Video Views Pre-roll Objective Guide](/ko/x-ads-api/campaign-management#video-views-preroll-objective)를 참고하세요.

* (2015-10-22 기준) 프로모션용 콘텐츠에 사용할 비디오를 업로드할 때는 [POST media/upload (chunked)](https://developer.x.com/en/docs/media/upload-media/api-reference/post-media-upload-init.html) 엔드포인트에 대한 모든 `INIT` 명령 요청에서 `media_category` 파라미터 값을 `amplify_video`로 설정해야 합니다. 이 파라미터를 사용하면 비디오가 비동기적으로 사전 처리되어 프로모션 콘텐츠에서 사용할 수 있도록 준비됩니다. 비디오 업로드 후 비동기 처리 완료 여부는 `STATUS` 명령으로 확인할 수 있습니다.
* 현재 허용되는 프로모션 비디오의 최대 길이는 10분이며, 파일 크기는 500MB 이하여야 합니다.
* 업로드하는 비디오는 mp4 또는 mov 형식이어야 합니다.
* 업로드된 비디오는 일반적으로 빠르게 처리되지만, 처리 시간은 비디오 길이와 파일 크기에 따라 달라질 수 있습니다.
* 업로드하는 포스터 이미지는 png 또는 jpg 형식이어야 합니다. 종횡비나 크기 요구 사항은 없지만, 포스터 이미지는 비디오 플레이어에 맞게 조정됩니다.

### 예약 트윗

<div id="introduction">
  #### 소개
</div>

예약 트윗 기능을 사용하면 광고주나 사용자가 나중에 게시되도록 트윗을 미리 작성해 예약할 수 있습니다. 이러한 트윗을 생성하고 관리할 수 있을 뿐만 아니라, API를 사용하면 해당 트윗을 라인 아이템에 연결하여 트윗이 게시되었을 때 프로모션할 수 있습니다. 이를 통해 광고주는 네이티브 트윗을 미리 만들어 두고, 주요 이니셔티브 이전에 캠페인 크리에이티브를 미리 계획할 수 있습니다. 예를 들어, 신제품 발표와 동시에 바로 게시되도록 트윗 크리에이티브를 사전에 준비해 둘 수 있습니다.

Scheduled Tweets API 엔드포인트에서 제공하는 전체 기능은 다음과 같습니다:

* 새로 예약할 트윗 생성, 수정 및 조회
* 예약 트윗을 라인 아이템과 연결
* 기존 예약 트윗 조회 및 관리
* 예약 트윗이 게시된 후 게시된 트윗의 `id` 조회

예약 트윗은 “라이브” 트윗과는 별도의 리소스이기 때문에, 이 트윗들을 생성하거나 수정할 때는 두 가지 서로 다른 검증 절차가 실행됩니다. 첫 번째 검증 규칙 세트는 예약 트윗을 생성하는 단계에서, 구체적으로 다음과 같이 실행됩니다:

<div id="scheduled-tweet-create">
  #### 예약 트윗 생성:
</div>

* 인증된 사용자가 지정된 @handle에 대해 오가닉(organic) 트윗을 생성할 수 있는 권한이 있는지 검증합니다. Promoted-Only 트윗 생성 권한을 갖추려면, 인증된 사용자는 [Tweet composer permissions](https://developer.x.com/content/developer-twitter/en/docs/ads/campaign-management/api-reference/authenticated-user-access#get-accounts-account-id-authenticated-user-access)을 가진 계정 사용자여야 합니다.
* `scheduled_at` 시간 기준, 이후 15분 이내에 생성되도록 예약된 트윗이 30개를 초과하지 않는지 검증합니다. `SCHEDULED_TWEET_LIMIT_EXCEEDED` 오류 메시지는 동일한 향후 15분 시간대에 너무 많은 예약 트윗이 잡혀 있음을 나타냅니다. 광고주는 기존 예약 트윗을 하나 제거하거나 `scheduled_at` 시간을 더 이르거나 늦은 시점으로 조정해야 합니다.

<div id="scheduled-tweet-goes-live">
  #### Scheduled Tweet가 게시되는 시점:
</div>

* 이러한 유효성 검사 규칙은 scheduled&#95;at 시간에 실행되며, API에서 일반 트윗을 생성할 때 적용되는 규칙과 동일합니다. 예를 들어 Scheduled Tweet에 이미지와 GIF가 모두 포함되어 있는 경우 해당 Scheduled Tweet은 게시되지 않으며, scheduled&#95;status가 FAILED로 설정됩니다.

<div id="workflow">
  #### Workflow
</div>

**새 예약 트윗 만들기**

새 예약 트윗은 [POST accounts/:account&#95;id/scheduled&#95;tweets](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/api-reference/scheduled-tweets#post-accounts-account-id-scheduled-tweets) 엔드포인트를 사용해 생성할 수 있습니다. 이 엔드포인트에는 예약 시간인 `scheduled_at` 파라미터와, 트윗에 미디어 엔티티가 포함되지 않은 경우 트윗 `text` 파라미터가 필수입니다. 추가로, 이 엔드포인트는 `as_user_id` 파라미터를 통해 다른 @handle을 대신해 예약 트윗을 생성할 수 있는 옵션과, 카드(`card_uri`) 및 미디어(`media_ids`)를 추가할 수 있는 기능을 제공합니다. 트윗에는 동일한 유형의 엔티티만 포함될 수 있다는 점에 유의하십시오. 즉, Video, Gif 또는 Image 중 한 종류만 포함할 수 있습니다. `nullcast` 파라미터는 해당 트윗이 “Promoted-Only” 트윗인지 여부를 제어합니다. 새로 생성된 모든 예약 트윗은 기본적으로 &quot;Promoted-Only&quot;(`nullcast=true`)입니다. `nullcast=false`인 경우 오가닉(Organic) 예약 트윗이 생성됩니다.

예약 트윗이 성공적으로 생성되면, 응답에는 예약 트윗 자체의 고유 식별자를 나타내는 `id` 필드가 포함됩니다. 이 필드 외에 `tweet_id`라는 또 다른 필드도 반환됩니다. 이 필드는 처음에는 `null`이지만, 트윗이 실제로 게시되면 “live” 트윗의 ID로 채워집니다.

```
twurl -H 'ads-api.x.com' -X POST "/6/accounts/:account_id/scheduled_tweets" -d 'scheduled_at=2017-12-24T23:59:00Z&text=Happy Holidays!'
```

이 작업으로 다음과 같은 예약 트윗이 생성됩니다:

```json
{
  "request": {
    "params": {
      "text": "Happy Holidays!",
      "scheduled_at": "2017-12-24T23:59:00Z"
    }
  },
  "data": {
    "completed_at": null,
    "id_str": "917507899668099072",
    "text": "Happy Holidays!",
    "user_id": "3271358660",
    "scheduled_status": "SCHEDULED",
    "id": 917507899668099100,
    "nullcast": true,
    "created_at": "2017-10-09T21:51:44Z",
    "scheduled_at": "2017-12-24T23:59:00Z",
    "card_uri": null,
    "updated_at": "2017-10-09T21:51:44Z",
    "tweet_id": null,
    "media_keys": []
  }
}
```

이 예약 트윗이 라이브 상태가 되면 `tweet_id` 필드가 &quot;라이브&quot; 트윗의 ID로 채워집니다.

**예약 트윗 보기**

[GET accounts/:account&#95;id/tweet&#95;previews](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/api-reference/tweet-previews#get-accounts-account-id-tweet-previews) 엔드포인트는 이전 단계에서 가져온 예약 트윗 `id`와 함께 사용하여 트윗의 미리보기를 생성할 수 있습니다. API 응답에는 예약 트윗의 미리보기를 렌더링하는 데 바로 사용할 수 있는 iframe URL이 포함됩니다. 관련 CSS 및 이미지는 X를 통해 직접 제공됩니다.

```
twurl -H ads-api.x.com -X GET "/6/accounts/18ce54bgxky/tweet_previews?tweet_type=SCHEDULED&tweet_ids=917507899668099072
```

```json
{
  "request": {
    "params": {
```

**예약 트윗을 라인 아이템에 연결하기**

예약 트윗은 오가닉 트윗(Organic Tweet)을 생성하는 데 사용할 수 있을 뿐만 아니라, 파트너가 “Promoted-Only”(`nullcast=true`) 트윗을 생성할 수도 있으며, 이 둘 중 어느 쪽이든 라인 아이템에 연결할 수 있습니다. 이를 지원하기 위해 [POST accounts/:account&#95;id/scheduled&#95;promoted&#95;tweets](https://developer.x.com/content/developer-twitter/en/docs/ads/campaign-management/api-reference/scheduled-promoted-tweets#post-accounts-account-id-scheduled-promoted-tweets) 엔드포인트도 제공합니다. 이 엔드포인트는 단일 API 호출당 하나의 예약 프로모션 트윗만 라인 아이템에 연결할 수 있습니다. 동일한 라인 아이템에 여러 예약 트윗을 연결하려면 여러 번의 API 호출이 필요합니다.

기존에 생성된 예약 프로모션 트윗은 수정할 수 없다는 점에 유의하십시오.

```
twurl -H 'ads-api.x.com' -X POST "/6/accounts/:account_id/scheduled_promoted_tweets" -d 'line_item_id=a44qc&scheduled_tweet_id=917507899668099072'
```

```json
{
  "data": {
    "line_item_id": "a44qc",
    "id": "26576",
    "created_at": "2017-10-09T22:24:16Z",
    "updated_at": "2017-10-09T22:24:16Z",
    "scheduled_tweet_id": "917507899668099072",
    "tweet_id": null,
    "deleted": false
  },
  "request": {
    "params": {
      "line_item_id": "a44qc",
      "scheduled_tweet_id": 917507899668099100,
      "account_id": "aaaaa"
    }
  }
}
```

이 엔드포인트는 지정된 예약된 트윗과 라인 아이템 사이의 연결만 생성합니다. 캠페인/라인 아이템 플라이트 날짜가 도래하면, 해당 라인 아이템은 자동으로 해당 “라이브” 트윗 노출을 시작합니다. 이 단계에서는 예약된 트윗이 `SCHEDULED` 상태인지, 그리고 주어진 목적(objective)에 대해 해당 예약된 트윗이 유효한지만 검증하며, 그 외의 검증은 수행되지 않습니다. 라인 아이템과 예약된 트윗에 적용되는 나머지 검증 규칙은 트윗이 “라이브” 상태가 될 때 실행됩니다.

캠페인 노출에 문제가 발생하지 않도록, 예약된 트윗의 `scheduled_at` 시간을 캠페인/라인 아이템 플라이트 날짜보다 이른 시점으로 설정할 것을 권장합니다.

예를 들어, 예약된 트윗이 캠페인 시작일 이후에 라이브 되도록 설정되어 있고(또 단일 라인 아이템에 단 하나의 트윗만 연결되어 있다고 가정하면), 캠페인은 `ACTIVE` 상태가 되지만 예약된 트윗이 아직 라이브가 아니므로, 노출 가능한 크리에이티브가 존재하지 않게 됩니다.

**예약된 트윗 관리**

나머지 엔드포인트 세트는 API 사용자가 모든 예약된 트윗과 예약된 프로모션 트윗을 관리할 수 있도록 합니다. 이 API들은 모든 예약된 트윗 목록을, 선택적으로 특정 상태로 필터링하여 반환하거나, 특정 예약된 트윗을 해당 `id`로 조회하는 데 사용할 수 있습니다.

<div id="what-happens-when-a-scheduled-tweet-goes-live">
  #### 예약 트윗이 라이브 상태가 되면 어떻게 되나요?
</div>

특정 예약 트윗이 라이브 상태가 되려는 시점, 즉 `scheduled_at` 시간에 다음과 같은 업데이트가 이루어집니다:

* “라이브” 트윗이 생성되지만, 최대 1초 정도의 지연이 발생할 수 있습니다
* `tweet_id`가 다음 엔티티에 추가됩니다:
* 예약 트윗
* 프로모션용 예약 트윗
* 새로운 프로모션 트윗 엔티티가 생성됩니다

<div id="best-practices">
  #### 모범 사례
</div>

예약 트윗을 생성하거나 프로모션할 때는 다음 모범 사례를 따를 것을 권장합니다.

* 예약 트윗을 생성할 때 트윗이 유효한지 확인합니다. (예: 하나의 트윗에는 이미지, 동영상, GIF 중 하나만 포함할 수 있으며 세 가지를 조합해서는 안 됩니다.)
* 캠페인 플라이트 기간(즉, `start_time` 및 `end_time`)이 예약 트윗의 `scheduled_at` 시간과 일치하는지 확인합니다.
* 예약 트윗은 현재 시점으로부터 1년(365일)을 초과하는 시점으로는 예약하지 않아야 합니다.
* 예약 트윗에 대해서는 트윗 미리보기 기능이 현재 지원되지 않습니다. (이는 예약 트윗을 생성하기 전에 미리 보는 기능을 의미합니다.)

### 미디어 라이브러리

<div id="introduction">
  #### 소개
</div>

Media Library 엔드포인트를 사용하면 X Ads 계정에서 이미지, GIF 및 동영상을 관리할 수 있습니다. 라이브러리에 있는 미디어 애셋은 트윗과 카드 생성에 사용할 수 있습니다. 또한 여러 [creatives](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/overview)에서 재사용할 수 있어, 동일한 애셋을 여러 번 업로드할 필요가 없습니다.

#### API 엔드포인트

* [POST media/upload](https://developer.x.com/content/developer-twitter/en/docs/media/upload-media/api-reference/post-media-upload#post-media-upload) 또는 [POST media/upload (chunked)](https://developer.x.com/content/developer-twitter/en/docs/media/upload-media/api-reference/post-media-upload-init#post-media-upload-init) (미디어 업로드)
* [POST accounts/:account&#95;id/media&#95;library](https://developer.x.com/x-ads-api/creatives#get-accounts-account-id-media-library#post-accounts-account-id-media-library) (미디어 라이브러리에 미디어 추가)

<div id="adding-to-the-library">
  #### 라이브러리에 추가하기
</div>

라이브러리에 미디어를 추가하는 과정은 두 단계로 진행됩니다. 먼저, [POST media/upload](https://developer.x.com/content/developer-twitter/en/docs/media/upload-media/api-reference/post-media-upload#post-media-upload) 엔드포인트나 [POST media/upload (chunked)](https://developer.x.com/content/developer-twitter/en/docs/media/upload-media/api-reference/post-media-upload-init#post-media-upload-init) 엔드포인트 집합 중 하나를 사용하여 에셋을 업로드합니다. (멀티파트 업로드 프로세스에 대한 자세한 내용은 [Chunked media upload](https://developer.x.com/content/developer-twitter/en/docs/media/upload-media/uploading-media/chunked-media-upload) 가이드를 참조하세요.)

```
twurl -X POST -H upload.x.com "/1.1/media/upload.json?additional_owners=756201191646691328" --file latte.jpeg --file-field "media"
```

```json
{
   "media_id":966947208837742592,
   "media_id_string":"966947208837742592",
   "size":74194,
   "expires_after_secs":86400,
   "image":{
      "image_type":"image/jpeg",
      "w":800,
      "h":418
   }
}
```

다음으로, 미디어 ID를 사용해 [POST accounts/:account&#95;id/media&#95;library](https://developer.x.com/x-ads-api/creatives#get-accounts-account-id-media-library#post-accounts-account-id-media-library) 엔드포인트를 호출하여 해당 미디어를 광고 계정의 라이브러리에 추가합니다.

```
twurl -X POST -H ads-api.x.com "/9/accounts/18ce54d4x5t/media_library?file_name=latte.jpeg&media_category=TWEET_IMAGE&media_key=966947208837742592&name=Latte"
```

<div id="request-parameters">
  #### 요청 매개변수
</div>

모든 Media Library POST 요청에는 미디어 식별자가 필요합니다. 이 값은 업로드 단계에서 반환됩니다. 위 예시처럼 `media_id`를 사용할 때는 `media_category`도 함께 지정해야 합니다. 가능한 카테고리 값은 네 가지이며, 각각 AMPLIFY&#95;VIDEO, TWEET&#95;GIF, TWEET&#95;IMAGE, TWEET&#95;VIDEO입니다.

선택적으로, Media Library에 있는 객체에 `name` 및 `file_name` 값을 설정할 수 있습니다. 이러한 속성은 라이브러리 내에서 서로 다른 미디어 버전을 사용자가 구분하는 데 도움을 줍니다.

비디오의 경우, 제목과 설명을 추가로 설정할 수 있습니다. 이 값들은 [POST accounts/:account&#95;id/tweet](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/api-reference/tweets#post-accounts-account-id-tweet) 엔드포인트에서 `video_title` 및 `video_description` 요청 매개변수로 전달되도록 되어 있습니다. 트윗에서는 이 텍스트가 비디오 아래에 표시됩니다.

<div id="attributes">
  #### 속성
</div>

Media Library는 media&#95;key 개념을 정식으로 도입합니다. 이는 라이브러리 내 객체의 고유 식별자입니다. 미디어 키는 다음 형식의 문자열 값입니다: 13&#95;875943225764098048. 이러한 미디어 키는 모든 카드 엔드포인트에서 전면적으로 지원됩니다.

또한 Media Library 응답에는 문자열로 표현되는 media&#95;id가 포함됩니다. 이는 현재 media key를 지원하지 않는 리소스에 포함됩니다: [트윗](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/api-reference/tweets#post-accounts-account-id-tweet)*, [트윗 미리보기](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/api-reference/tweets#get-accounts-account-id-tweet-preview)*, 그리고 [예약된 트윗](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/api-reference/scheduled-tweets#post-accounts-account-id-scheduled-tweets). 우리는 모든 곳에서 media key를 지원할 수 있도록 작업 중입니다.

aspect&#95;ratio 속성은 GIF와 동영상에 대해 반환됩니다. 이는 특정 가로세로 비율만 허용하는 카드에서 사용할 미디어를 필터링하는 데 사용할 수 있습니다.

*이 엔드포인트들은 media key인 video&#95;id 매개변수를 지원합니다.

<div id="identifying-cards">
  ### 카드 식별
</div>

<div id="introduction">
  #### 소개
</div>

카드는 미디어를 사용하는 사용자 정의형 광고 포맷으로, 웹사이트, 앱 또는 다이렉트 메시지 시작과 같은 특정 사용자 참여를 유도하는 행동 유도(Call to Action)와 연동할 수 있습니다. 카드는 트윗, 예약된 트윗, 초안 트윗에 추가할 수 있습니다.

카드는 트윗 객체에서 두 가지 방식 중 하나로 참조될 수 있습니다. 카드의 `card_uri`로 참조되거나, `preview_url`로 참조될 수 있습니다. 각 값의 예시는 아래와 같습니다.

| card&#95;uri | preview&#95;url |
| :--- | :--- |
| card://1043282691834048513 | https://cards.x.com/cards/18ce54d4x5t/68w3s |

**참고**: Ads API 버전 3부터는 새로 생성된 카드에 대해 cards 응답에서 `card_uri`만 생성되고 반환됩니다.

**참고**: Ads API 버전 5부터는 cards 응답에서 `preview_url`이 더 이상 반환되지 않습니다.

트윗 객체 응답에서 어떤 방식으로 참조되는지는 트윗이 생성된 방식에 따라 달라집니다. 다시 말해, `card_uri` 요청 매개변수를 사용하여 트윗을 생성한 경우 응답에 card URI 값이 나타납니다. 반면 트윗 텍스트의 일부로 `preview_url`이 포함된 경우에는 응답에 preview URL이 나타납니다.

트윗 텍스트에 preview URL을 포함해 트윗을 만든 경우, URL은 entities[&quot;urls&quot;][i][&quot;expanded&#95;url&quot;]에서 찾을 수 있습니다(이때 text 필드에는 단축된 t.co URL이 포함됩니다). 여기서 i는 배열 인덱스이며, 하나의 트윗에는 여러 개의 URL이 포함될 수 있습니다.

예약 또는 임시 저장된 트윗 객체의 경우, preview URL은 항상 text 필드에 나타납니다.

  #### 카드 가져오기
</div>

특정 카드에 대한 추가 정보를 조회하려면 다음 두 개의 엔드포인트를 사용할 수 있습니다: [GET accounts/:account&#95;id/cards/all](https://developer.x.com/en/docs/ads/creatives/api-reference/cards-fetch.html#get-accounts-account-id-cards-all) 및 [GET accounts/:account&#95;id/cards/all/:card&#95;id](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/api-reference/cards-fetch#get-accounts-account-id-cards-all-card-id). 첫 번째 엔드포인트는 card&#95;uri로 카드를 가져오고, 두 번째 엔드포인트는 카드의 ID로 카드를 가져옵니다. 카드의 ID는 preview&#95;url의 끝부분에서 확인할 수 있습니다. 위 예시에서 ID는 68w3s입니다.

<div id="identifying-media">
  ### 미디어 식별하기
</div>

<div id="introduction">
  #### 소개
</div>

이미지, GIF, 동영상과 같은 미디어는 트윗과 카드에 추가할 수 있습니다. 또한 동영상은 프리롤용 소재로 사용할 수 있고, 이미지는 [X Audience Platform](https://developer.x.com/en/docs/ads/measurement/overview/twitter-audience-platform)에서 프로모션할 수 있습니다. 이 섹션에서는 이러한 엔터티에서 미디어 참조를 찾는 방법을 설명합니다.

미디어 식별자에는 두 가지 유형이 있습니다. ID와 키입니다. 각 유형의 예시 값은 아래에 제시되어 있습니다.

| **Media ID** | **Media key** |
| :--- | :--- |
| 1029825579531807971 | 13&#95;1029825579531807971 |

Media key는 ID 앞에 숫자 접두사와 밑줄이 붙은 형태입니다.

<div id="images">
  #### Images
</div>

다음 표는 각 이미지 관련 리소스의 응답에서 현재 사용 가능한 식별자 유형과 해당 속성 이름을 보여줍니다.

| **Resource** | **Identifier** | **Attribute(s)** |
| :--- | :--- | :--- |
| Image cards | 없음 |  |
| Tweet | 둘 다 | `entities["media"]["id_str"]`  `entities["media"]["media_key"]` |
| Scheduled Tweet | 둘 다 | `media_ids` and `media_keys` |
| Draft Tweet | 둘 다 | `media_ids` and `media_keys` |
| Account Media | 없음 |  |
| Media Library | 둘 다 | `media_id` and `media_key` |

Image cards와 Account Media 이미지에는 어떠한 미디어 식별자에 대한 참조도 포함되지 않습니다. 트윗은 미디어 ID만 포함합니다. Scheduled Tweet 및 Draft Tweet은 미디어 ID와 미디어 key 둘 다를 포함합니다. Media Library 또한 둘 다를 반환합니다.

트윗의 경우, entities[&quot;media&quot;] 배열 안에 있는 객체의 id 및 id&#95;str 필드는 미디어 ID에 해당합니다. 트윗에 여러 이미지가 포함되어 있는 경우, 각 미디어 엔티티에 대한 참조는 extended&#95;entities[&quot;media&quot;]에서만 찾을 수 있습니다.

식별자에 대한 참조 외에도, 이미지 URL에 접근할 수 있는 것 또한 중요한 경우가 많습니다.

| **Resource** | **Attribute(s)** | **Format** |
| :--- | :--- | :--- |
| Image cards | `image` | .jpg |
| Tweet* | `entities["media"][0]["media_url"]` or `extended_entities["media"][i]["media_url"]` | .jpg |
| Scheduled Tweet | 없음 |  |
| Draft Tweet | 없음 |  |
| Account Media | `media_url` | .jpg |
| Media Library | `media_url` | .jpg |

* 이 URL의 위치는 트윗에 단일 이미지가 포함되어 있는지, 여러 이미지가 포함되어 있는지에 따라 달라집니다.

모든 Image cards에는 X 이미지 URL을 포함하는 image라는 응답 속성이 있습니다. (image app download cards의 경우 이름은 wide&#95;app&#95;image입니다.)

트윗의 경우, 미디어 URL 위치는 미디어 유형과 사용 중인 엔드포인트 둘 다에 따라 달라집니다. 단일 이미지를 포함하는 트윗의 경우, URL은 entities[&quot;media&quot;][0][&quot;media&#95;url&quot;]에서 찾을 수 있습니다. 이는 Ads API와 Standard API 모두에 해당합니다. 그러나 트윗에 여러 이미지가 포함되어 있을 때는 URL이 extended&#95;entities[&quot;media&quot;][i][&quot;media&#95;url&quot;]에서만 찾을 수 있습니다. 이는 Standard API에서만 제공됩니다.

<div id="videos">
  #### 동영상
</div>

다음 표는 각 동영상 관련 리소스의 응답에 현재 사용 가능한 식별자 유형과 해당하는 속성 이름을 보여줍니다.

| **리소스** | **식별자** | **속성** |
| :--- | :--- | :--- |
| 동영상 카드 | 둘 다일 수 있음 | `video_content_id` |
| 동영상 투표 카드 | 없음 |  |
| 트윗 | 둘 다 | `entities["media"]["id_str"]`  `entities["media"]["media_key"]` |
| 예약 트윗 | 둘 다 | `media_ids` 및 `media_keys` |
| 임시 트윗 | 둘 다 | `media_ids` 및 `media_keys` |
| 계정 미디어 | 미디어 키 | `video_id` |
| 미디어 라이브러리 | 둘 다 | `media_id` 및 `media_key` |

동영상 카드(동영상이 포함된 투표 카드는 제외)의 경우 응답 속성으로 `video_content_id`가 포함되지만, 반환되는 값의 유형에 일관성이 없습니다. 어떤 경우에는 미디어 ID이고, 다른 경우에는 미디어 키입니다.

동영상 URL에 액세스하는 방법은 아래 표와 같습니다.

| **리소스** | **속성** | **형식** |
| :--- | :--- | :--- |
| 동영상 카드 | `video_url` 및 `video_hls_url` | .vmap  .m3u8 |
| 동영상이 포함된 트윗 | `extended_entities["media"][i]["video_info"]["variants"][j]["url"]` | .mp4 |
| 예약 트윗 | 없음 |  |
| 임시 트윗 | 없음 |  |
| 계정 미디어 | 없음 |  |
| 미디어 라이브러리 | `media_url` | .mp4 |

동영상 카드는 `video_url` 및 `video_hls_url` 응답 속성을 포함하며, 각각 .vmap 및 .m3u8 형식의 URL을 제공합니다.

<div id="media-library">
  #### 미디어 라이브러리
</div>

때때로 미디어 에셋에 대한 추가 정보를 가져와야 할 때가 있습니다. 예를 들어 비디오 카드의 경우 vmap URL 대신 mp4 URL을 가져와야 하는 경우가 있습니다. 이는 미디어 라이브러리에서 확인할 수 있습니다. 사용 가능한 정보에 대한 자세한 내용은 [Media Library Guide](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/guides/media-library)를 참조하세요. 광고 계정의 FULL promotable user에 속한 대부분의 에셋은 라이브러리에서 찾을 수 있습니다. 다만, 몇 가지 예외가 있습니다.

**미디어 가져오기**

위에서 설명했듯이, 이미지 카드는 media ID나 media key에 대한 참조를 포함하지 않습니다. 따라서 미디어 라이브러리를 통해 해당 에셋을 가져올 수 없습니다. 이는 [Account Media](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/api-reference/account-media#account-media) 이미지에도 동일하게 적용됩니다.

비디오 카드는 생성 이전에 비디오 에셋이 미디어 라이브러리(또는 그 이전의 Videos 리소스)의 일부여야 합니다. 그 결과, 이러한 에셋은 항상 미디어 라이브러리에서 조회할 수 있습니다. 마찬가지로 Account Media PREROLL 에셋도 항상 조회할 수 있습니다.

마지막으로, 트윗에 포함된 미디어는 항상 미디어 라이브러리에 존재하는 것이 보장됩니다.

다음 표는 조회에 사용할 식별자가 리소스 응답에 포함되는지를 고려하여, 어떤 에셋이 미디어 라이브러리에서 조회 가능한지 요약한 것입니다.

| **리소스** | **미디어 라이브러리에 존재 여부** |
| :--- | :--- |
| 이미지 카드 | 아니요 |
| 비디오 카드 | 예* |
| 트윗(모든 미디어)** | 예 |
| 예약된 트윗 | 예 |
| 초안 트윗 | 예 |
| Account media 이미지 | 아니요 |
| Account media 비디오 (`PREROLL`) | 예 |

* `video_content_id`가 media key인 카드의 경우입니다. 값이 media ID인 경우에도 에셋은 여전히 미디어 라이브러리에 존재하지만, 조회하려면 해당 값 앞에 숫자 접두사와 밑줄을 추가해야 합니다.\
** 트윗은 media ID만 반환합니다. 에셋이 미디어 라이브러리에 존재하는 것은 보장되지만, 가져오려면 ID 앞에 숫자 접두사와 밑줄을 추가해야 합니다.

**Account Media와의 상호 작용**

라이브러리에 추가된 미디어 에셋이 자동으로 Account Media 리소스에도 추가되는 경우는 두 가지가 있습니다.

* AMPLIFY&#95;VIDEO 에셋이 미디어 라이브러리에 추가되면, 자동으로 PREROLL 크리에이티브 타입의 Account Media 에셋으로 추가됩니다.
* 특정 크기(당사의 [enumerations 페이지](https://developer.x.com/content/developer-twitter/en/docs/ads/general/overview/enums)의 &quot;Creative Types&quot; 참조)를 가진 이미지가 미디어 라이브러리에 추가되면, 자동으로 Account Media 에셋으로 추가됩니다. 크리에이티브 타입(예: INTERSTITIAL)은 이미지 크기에 따라 결정됩니다.

<div id="tweets">
  ### 트윗
</div>

<div id="introduction">
  #### 소개
</div>

X Ads API는 게시된 트윗, 예약 트윗, 초안 트윗 등 세 가지 유형의 트윗을 지원합니다.

<div id="nullcasted-tweets">
  #### 널캐스트된 트윗
</div>

트윗은 널캐스트(nullcasted, &quot;Promoted-only&quot;) 또는 오가닉(organic)일 수 있습니다. 널캐스트된 트윗은 게시되더라도 사용자의 공개 타임라인에는 표시되지 않지만, 공개 상태입니다. 반면 오가닉 트윗은 사용자의 팔로워에게 노출되며 사용자의 공개 타임라인에도 표시됩니다.

**트윗 생성**

세 개의 트윗 생성 엔드포인트 각각은 Boolean 형식의 `nullcast` 매개변수를 지원하며, 이를 통해 API 사용자는 널캐스트된 트윗 또는 오가닉 트윗을 생성할 수 있습니다. 널캐스트된 트윗은 사용자 본인 또는 사용자 대신 트윗을 생성할 권한이 있는 누구나 생성할 수 있습니다. 오가닉 트윗은 [full promotable user](https://developer.x.com/content/developer-twitter/en/docs/tutorials/ads-api-hierarchy-terminology#promotable-users)만 생성할 수 있습니다.

**트윗 업데이트**

예약된 트윗과 초안 트윗에 대해서는 `nullcast` 속성을 업데이트할 수 있습니다. 예약된 트윗의 경우, 트윗의 `scheduled_at` 시간까지 수정이 가능합니다. 초안 트윗은 무기한으로 수정할 수 있습니다. 그러나 한 번 게시된 후에는 트윗을 널캐스트에서 오가닉으로, 또는 그 반대로 변경하는 것은 불가능합니다.

<div id="promoting-tweets">
  #### 트윗 프로모션
</div>

게시된 트윗과 예약된 트윗만 프로모션할 수 있습니다. 이 트윗들은 널캐스트(nullcast)이거나 오가닉(organic)일 수 있으며, 이에 대한 제한은 없습니다. 광고주는 트윗을 프로모션할 수 있는 권한만 확보했다면 자신의 트윗뿐만 아니라 다른 사용자의 트윗도 프로모션할 수 있습니다. (자세한 내용은 [다른 사용자의 트윗 프로모션](https://developer.x.com/content/developer-twitter/en/docs/tutorials/promoting-another-users-content)을 참조하세요.)

하나의 캠페인에서 여러 트윗을 프로모션할 수 있습니다. 마찬가지로 하나의 트윗을 하나 이상의 캠페인에서 프로모션할 수 있습니다.

게시된 트윗을 프로모션하려면 [POST accounts/:account&#95;id/promoted&#95;tweets endpoint](https://developer.x.com/content/developer-twitter/en/docs/ads/campaign-management/api-reference/promoted-tweets#post-accounts-account-id-promoted-tweets)를 사용합니다. 이 엔드포인트는 게시된 트윗을 라인 아이템과 연결합니다. 예약된 트윗을 프로모션하려면 [POST accounts/:account&#95;id/scheduled&#95;promoted&#95;tweets](https://developer.x.com/content/developer-twitter/en/docs/ads/campaign-management/api-reference/scheduled-promoted-tweets#post-accounts-account-id-scheduled-promoted-tweets) endpoint를 사용합니다.

<div id="tweet-ids">
  #### 트윗 ID
</div>

게시된, 예약된, 초안 트윗의 ID는 [숫자형](https://developer.x.com/content/developer-twitter/en/docs/basics/twitter-ids)이며, 64비트 부호 없는 정수입니다. 예를 들어, 다음에 나오는 게시된 트윗의 ID는 1166476031668015104입니다.

게시되었거나 예약된 트윗을 프로모션하면, 이에 대응하는 프로모션 트윗 엔티티가 생성됩니다. 이 엔티티에는 각각 고유한 ID가 있으며, 영숫자 조합이고 base-36으로 인코딩된 값으로 표현됩니다. 예를 들어, 위에 나온 게시된 트윗을 프로모션하여 line item 6c62d와 연결하면, 다음과 같은 API 응답이 반환됩니다.

```

      "account_id": "18ce54d4x5t"
    }
  },
  "data": [
    {
      "line_item_id": "6c62d",
      "id": "3qwlq6",
      "entity_status": "ACTIVE",
      "created_at": "2019-09-12T21:39:10Z",
      "updated_at": "2019-09-12T21:39:10Z",
```

<div id="carousels">
  ### 캐러셀
</div>

<div id="introduction">
  #### 소개
</div>

X Ads API는 비디오 캐러셀과 이미지 캐러셀의 생성 및 조회를 지원합니다. 캐러셀은 2개에서 6개 사이의 미디어 자산을 포함할 수 있는 카드 유형입니다. 캐러셀 카드는 사용자를 웹사이트로 이동시키거나 모바일 앱 설치를 유도할 수 있습니다. 캐러셀의 개요, 장점, 모범 사례, FAQ 등에 대한 자세한 내용은 [Carousel Ads on X](https://business.x.com/en/advertising/carousels.html) 페이지를 참고하세요.

캐러셀은 다른 카드 유형과 마찬가지로 트윗에 사용할 수 있으며, 이렇게 만든 트윗은 프로모션할 수 있습니다. 전체 워크플로는 이미 익숙한 것과 동일합니다:

1. 미디어 업로드
2. 카드 생성
3. 트윗 생성
4. 트윗 프로모션

유일한 차이점은 카드 생성 방식입니다. 다른 카드 생성 요청은 쿼리 매개변수를 허용하는 반면, 캐러셀 카드 생성 요청은 JSON POST 요청 본문만 허용합니다.

<div id="endpoints">
  #### 엔드포인트
</div>

Ads API에서는 캐러셀을 생성하고 조회할 수 있습니다.

어떤 종류의 캐러셀이든 생성하려면 [POST accounts/:account&#95;id/cards](https://developer.x.com/en/docs/twitter-ads-api/creatives/api-reference/cards#post-accounts-account-id-cards) 엔드포인트를 사용하십시오. 캐러셀을 조회하려면 [GET accounts/:account&#95;id/cards](https://developer.x.com/en/docs/twitter-ads-api/creatives/api-reference/cards#get-accounts-account-id-cards) 엔드포인트를 사용하십시오.

<div id="json-post-body">
  #### JSON POST Body
</div>

캐러셀은 두 개의 컴포넌트를 사용하여 생성됩니다. 첫 번째는 사용할 미디어 에셋을 지정하고, 두 번째는 웹사이트 또는 앱에 대한 정보를 지정합니다.

구체적으로, 캐러셀 카드는 아래 순서대로 다음 컴포넌트를 사용하여 생성합니다:

* 미디어 키 배열을 받는 `SWIPEABLE_MEDIA` 컴포넌트 1개
* *다음 중 하나*:
* 웹사이트 정보를 지정하기 위한 `DETAILS` 컴포넌트
* 앱 정보를 지정하기 위한 `BUTTON` 컴포넌트

`SWIPEABLE_MEDIA` 컴포넌트에는 2개에서 6개의 이미지 또는 동영상을 지정할 수 있는 `media_keys` 배열이 포함되어야 합니다. 전달되는 미디어 키의 순서에 따라 렌더링되는 순서가 결정됩니다.

```
{
  "type": "SWIPEABLE_MEDIA",
  "media_keys": [
    "13_1089771253848666112",
    "13_1191948012077092867"
  ]
}
```

다시 한번 말씀드리면, [GET accounts/:account&#95;id/media&#95;library](https://developer.x.com/en/docs/twitter-ads-api/creatives/api-reference/media-library#get-accounts-account-id-media-library) 엔드포인트에 요청을 보내 `media key`를 획득할 수 있습니다.

두 번째 컴포넌트 객체의 구성은 사용자를 웹사이트로 이동시키려는지, 아니면 앱 설치를 유도하려는지에 따라 달라집니다. 아래 표는 이 두 가지 옵션을 요약한 것입니다. (**참고**: 나열된 키는 모두 필수입니다.)

|            | **Website**                                                        | **App**                                           |
| :--------- | :----------------------------------------------------------------- | :------------------------------------------------ |
| 컴포넌트 유형 지정 | `"type": "DETAILS"`                                                | `"type": "BUTTON"`                                |
| 제목/라벨      | `"title": "X"`                                                     | `"label": { "type": "ENUM", "value": "INSTALL" }` |
| 목적지        | `"destination": { "type": "WEBSITE", "url": "https://www.x.com" }` | `"destination": { "type": "APP", ... }`           |

이를 종합하면, 아래와 같은 웹사이트용 캐러셀 JSON POST 본문 예시가 만들어집니다.

```
{
  "name": "website carousel",
  "components": [
    {
      "type": "SWIPEABLE_MEDIA",
      "media_keys": [
        "13_1089771253848666112",
        "13_1191948012077092867"
      ]
    },
    {
      "type": "DETAILS",
      "title": "X",
      "destination": {
        "type": "WEBSITE",
        "url": "https://www.x.com"
      }
    }
  ]
}
```

`BUTTON` 컴포넌트 내의 앱 대상 객체에는 국가 코드와 적어도 하나의 앱 식별자가 필요합니다. 옵션으로 딥 링크를 지정할 수도 있습니다. 이러한 필드에 대한 설명은 [참조 문서](https://developer.x.com/en/docs/twitter-ads-api/creatives/api-reference/cards#app-destination)를 참고하세요.

이를 종합하면, 아래는 앱 캐러셀의 예시 JSON POST 본문입니다.

```
{
  "name": "app carousel",
  "components": [
    {
      "type": "SWIPEABLE_MEDIA",
      "media_keys": [
        "13_1089771253848666112",
        "13_1191948012077092867"
      ]
    },
    {
      "type": "BUTTON",
      "label": {
        "type": "ENUM",
        "value": "INSTALL"
      },
      "destination": {
        "type": "APP",
        "country_code": "US",
        "googleplay_app_id": "com.twitter.android",
        "iphone_app_id": "333903271"
      }
    }
  ]
}
```

<div id="creative-metadata-tagging">
  ### 크리에이티브 메타데이터 태그 지정
</div>

<div id="introduction">
  #### 소개
</div>

이 가이드는 크리에이티브 파트너, 에이전시, 크리에이티브 개발자가 X 캠페인에 사용되는 에셋에 태그를 추가하여 개별 에셋의 가치와 성과를 더 잘 파악할 수 있도록 하기 위한 것입니다.

**참고:** 미디어 에셋은 해당 미디어 에셋을 생성한 파트너 또는 개발자만 태그해야 합니다. 미디어 에셋의 사용자가 그 미디어 에셋을 생성하지 않았다면, 메타데이터 태깅을 적용하지 마십시오.

Creative Metadata Tagging은 크리에이티브 파트너가 제작한 이미지와 동영상에 대해, 에셋이 어디에서 X에 업로드되든, 또는 어떤 주체가 에셋을 업로드하든 관계없이 귀속 정보를 제공합니다. 크리에이티브 에셋과 크리에이티브 파트너 간의 연결을 만들기 위해 [XMP](https://exiftool.org/TagNames/XMP.html) 표준을 사용합니다.

<div id="tagging-creative-assets">
  #### 크리에이티브 에셋 태깅
</div>

아래 표는 각 이미지 관련 리소스의 응답에서 현재 사용 가능한 식별자 유형과 해당 속성 이름을 보여 줍니다. 크리에이티브 에셋을 태깅하려면 태깅 도구가 필요합니다. [ExifTool](https://exiftool.org/)은 메타 정보를 읽고, 쓰고, 편집하기 위한 플랫폼 독립적인 Perl 라이브러리와 [명령줄 애플리케이션](https://exiftool.org/exiftool_pod.html)으로 구성되어 있으며, 사용을 권장합니다. 지원되는 모든 [파일 유형](https://exiftool.org/#supported)을 확인하세요.

제공된 [ExifTool 설치 안내](https://exiftool.org/install.html)를 따라 설치하세요. macOS 및 Linux에서 [exiftool 설치 명령](https://formulae.brew.sh/formula/exiftool)을 제공해 설치를 더욱 간소화하는 [Homebrew](https://formulae.brew.sh/) 소프트웨어 패키지도 있습니다. 명령줄에서 `exiftool -ver` 를 입력해 도구의 버전 번호가 반환되는지 확인하여, 도구가 올바르게 설치되었는지 확인하세요. ExifTool 명령 매개변수에 대해서는 [ExifTool 문서](https://exiftool.org/)에서 자세히 알아볼 수 있습니다.

크리에이티브 파트너는 새 크리에이티브 에셋이든 기존 크리에이티브 에셋이든, 자신의 X app&#95;id 를 contributor XMP 태그와 date 태그의 메타데이터로 지정할 수 있습니다. 크리에이티브 에셋은 [Uploading Media](https://developer.x.com/en/docs/twitter-api/v1/media/upload-media/overview) 시 적용되는 기존 크기 제한을 그대로 따릅니다. 

**참고:** X가 contributor XMP 태그를 사용하는 것은 메타데이터가 X에서 진행되는 캠페인에 대해서만 값을 기록하도록 보장하기 위한 것입니다.

`exiftool -contributor="<YOUR APP ID>" -creative_file.jpg`

`exiftool -date="<date>" -creative_file.jpg`

app&#95;id 는 [Developer Portal](https://developer.x.com/en/portal/dashboard)의 Projects &amp; Apps 섹션에서 확인할 수 있습니다. 예: 16489123

다음 예시는 이미지 파일에 app&#95;id 를 contributor 태그로, date 를 date 태그로 추가합니다:

```
app_id:858382169
```

```
date:2022-03-13
```

```
creative_file: eiffel_tower.jpg
```

```
exiftool -contributor=858382169 eiffel_tower.jpg
```

```
   1 image files updated
```

```
exiftool -date=2022-03-13 eiffel_tower.jpg
```

```
  1 image files updated
```

이미지에 태그가 올바르게 지정되었는지 확인하세요: 

`exiftool -xmp:all -G1 <filename>`

예:

`exiftool -xmp:all -G1 eiffel_tower.jpg`

```
[XMP]        XMP Toolkit            : Image::ExifTool 12.30
```

```
[XMP]        Contributor            : 858382169
```

```
[XMP]        Date                   : 2022:03:13
```

<div id="components">
  #### 구성요소
</div>

모든 구성요소에는 객체의 스키마를 결정하는 `type` 필드가 포함되어야 합니다. Ads API는 구성요소를 미디어 기반과 설명 기반 구성요소로 구분하여 다음과 같은 타입을 지원합니다.

* 미디어:
* `MEDIA`: 단일 동영상 또는 이미지
* `SWIPEABLE_MEDIA`: 2~6개의 동영상 또는 이미지
* 설명:
* `DETAILS`
* `BUTTON`

각 구성요소에는 (`type` 키 외에) 필수 필드 집합이 있으며, 이는 아래 표에 나와 있습니다.

| 구성요소 `type`       | 필드                      | 값 유형             |
| :---------------- | :---------------------- | :--------------- |
| `MEDIA`           | `media_key`             | string           |
| `SWIPEABLE_MEDIA` | `media_keys`            | array of strings |
| `DETAILS`         | `title`   `destination` | string   object  |
| `BUTTON`          | `label`   `destination` | object   object  |

다음은 `components` 배열 맥락에서의 `BUTTON` 구성요소 예시입니다 (`name` 키는 의도적으로 생략되어 있습니다). (줄임표는 추가 정보를 지정해야 하는 위치를 나타냅니다.)

```
{
  "components": [
    {
      "type": "BUTTON",
      "label": {
        ...
      },
      "destination": {
        ...
      }
    }
  ]
}
```

구성 요소 객체가 지정되는 순서는 카드가 위에서 아래로 렌더링되는 순서를 결정합니다. 카드는 하나의 미디어 기반 구성 요소와 `DETAILS` 또는 `BUTTON` 구성 요소 중 하나를 함께 사용하여 생성해야 합니다. 설명 기반 구성 요소는 미디어 아래에 렌더링되며, URL 또는 모바일 앱을 목적지로 갖습니다.

**Label**

Label은 버튼에 표시되는 텍스트를 정의하므로 `BUTTON` 구성 요소에만 적용됩니다. Label 객체에는 `type`과 `value`라는 두 개의 필수 키가 있습니다. `type`은 `ENUM`으로 설정해야 하며, `value`는 `BOOK`, `CONNECT`, `INSTALL`, `OPEN`, `ORDER`, `PLAY`, `SHOP` 중 하나가 될 수 있습니다.

이전 예제를 바탕으로, 다음은 `BUTTON` 구성 요소 내의 `label` 객체를 보여줍니다.

```
{
  "components": [
    {
      "type": "BUTTON",
      "label": {
        "type": "ENUM",
        "value": "INSTALL"
      },
      "destination": {
        ...
      }
    }
  ]
}
```

  #### POST accounts/:account&#95;id/tweet
</div>

계정의 전체 프로모션 대상 사용자(기본값) 또는 `as_user_id` 매개변수에 지정된 사용자에 대해 트윗을 생성합니다. 널캐스트(nullcasted, 기본값) 트윗과 오가닉 트윗 생성을 모두 지원합니다. 널캐스트 트윗은 공개 타임라인에 표시되지 않으며 팔로워에게도 노출되지 않습니다. 두 유형 모두 캠페인에서 사용할 수 있습니다.

인증된 사용자가 이 계정의 `FULL` 프로모션 대상 사용자가 아닌 경우, [GET accounts/:account&#95;id/authenticated&#95;user&#95;access](https://developer.x.com/en/docs/ads/campaign-management/api-reference/authenticated-user-access#get-accounts-account-id-authenticated-user-access) 엔드포인트에 요청을 보내 이 사용자를 대신하여 트윗을 작성할 권한이 있는지 확인해야 합니다. 권한이 `TWEET_COMPOSER`인 경우, 사용자는 이 엔드포인트를 사용해 `FULL` [promotable user](https://developer.x.com/en/docs/ads/campaign-management/api-reference/promotable-users#get-accounts-account-id-promotable-users)를 대신하여 널캐스트 트윗을 생성할 수 있음을 의미합니다.

미디어에 대해 [upload.x.com 엔드포인트](https://developer.x.com/rest/reference/post/media/upload)를 사용할 때는, 이 엔드포인트에 전달하는 `as_user_id` 값과 동일한 `user_id` 값을 `additional_owners` 매개변수에 전달해야 합니다.
