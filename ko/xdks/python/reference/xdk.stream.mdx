<div id="labels_compliance">
  ### `labels_compliance`
</div>

Post 라벨 스트리밍
게시물에 적용된 모든 라벨링 이벤트를 스트리밍합니다.
이 엔드포인트는 사용 가능해지는 즉시 실시간으로 데이터를 제공하는 스트리밍 엔드포인트입니다.
각 반환 항목은 스트림에서 나온 단일 데이터 포인트를 나타냅니다.
연결은 지수 백오프 재시도 로직으로 자동 관리됩니다.
스트림이 끊기면 SDK가 클라이언트 개입 없이 자동으로 재연결합니다.

#### 매개변수

<ParamField path="path.backfill_minutes" type="int or None" default="None">
  요청한 백필(backfill) 기간을 분 단위로 지정합니다.
</ParamField>

<ParamField path="path.start_time" type="str or None" default="None">
  YYYY-MM-DDTHH:mm:ssZ. 게시물 레이블이 제공될 가장 이른 UTC 타임스탬프입니다.
</ParamField>

<ParamField path="path.end_time" type="str or None" default="None">
  YYYY-MM-DDTHH:mm:ssZ. 게시물 레이블이 제공될 가장 늦은 UTC 타임스탬프입니다.
</ParamField>

<ParamField path="path.stream_config" type="StreamConfig">
  재시도 동작, 타임아웃, 콜백을 사용자 지정하기 위한 선택적 StreamConfig입니다.
</ParamField>

Likes 규정 준수 데이터 스트림 (Streaming)

이 엔드포인트는 데이터가 생성되는 즉시 실시간으로 전달하는 스트리밍 엔드포인트입니다.
스트림에서 수신되는 각 항목은 하나의 데이터 포인트를 나타냅니다.
연결은 지수적 백오프 재시도 로직을 통해 자동으로 관리됩니다.
스트림이 끊기면 SDK가 클라이언트의 개입 없이 자동으로 다시 연결합니다.

</ParamField>

<ParamField path="path.end_time" type="str or None" default="None">

  ### `users_compliance`

Users 컴플라이언스 데이터를 스트리밍합니다 (Streaming)
Users와 관련된 모든 컴플라이언스 데이터를 스트리밍합니다.
이것은 데이터가 생성되는 대로 실시간으로 전달하는 스트리밍 엔드포인트입니다.

연결은 지수 백오프를 사용하는 재시도 로직으로 자동 관리됩니다.
스트림 연결이 끊어지면 SDK가 클라이언트 개입 없이 자동으로 재연결합니다.
