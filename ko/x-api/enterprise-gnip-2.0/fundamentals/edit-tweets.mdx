Enterprise 엔드포인트가 편집된 게시물 메타데이터를 제공하도록 업데이트되었습니다. &#95;Edit Posts&#95; 기능은 2022년 9월 1일 X 직원들을 대상으로 테스트용으로 처음 도입되었습니다. 해당 날짜부터, 대상이 되는 게시물은 30분 동안 최대 5회까지 수정할 수 있었습니다. *2022년 9월 29일 이후에 생성된 모든 게시물 객체*에는, 게시물이 실제로 수정되지 않았더라도 게시물 편집 메타데이터가 포함됩니다. 게시물이 수정될 때마다 새로운 게시물 ID가 생성됩니다. 게시물의 편집 기록은 원래 ID에서 시작해 이 ID들을 체인처럼 연결함으로써 설명할 수 있습니다. 또한 편집 체인에 있는 어떤 게시물이든 삭제되면, 그 체인에 속한 모든 게시물도 함께 삭제됩니다. 

이러한 메타데이터 세부 정보는 자동으로 포함됩니다. 게시물 객체에 사용 가능한 편집 기록을 포함하기 위해 별도의 요청 매개변수가 필요하지 않습니다. 

이 새로운 메타데이터를 통해 개발자는 다음을 확인할 수 있습니다:

* 게시물이 생성 시점에 편집 가능 대상이었는지 여부. 투표가 포함된 게시물이나 예약 게시물과 같이, 편집할 수 없는 게시물도 있습니다.
* 게시물은 30분 동안 편집할 수 있으며, 최대 5회까지 수정할 수 있습니다. 편집 가능한 게시물에 대해서는 남은 편집 가능 시간과 추가로 몇 번 더 수정할 수 있는지를 확인할 수 있습니다.
* 현재 보고 있는 게시물이 편집된 버전인지 여부. 대부분의 경우, API는 특정 과거 버전이 게시물 ID로 요청되지 않는 한 게시물의 최신 버전을 반환합니다.

루트 레벨에 세 가지 새로운 게시물 속성이 추가되었습니다:

* **edit&#95;history**  - 게시물의 편집 기록과 관련된 모든 게시물 ID를 제공합니다. &quot;initial&#95;tweet&#95;id&quot; 속성은 원본 게시물을 나타내며, &quot;edit&#95;tweet&#95;ids&quot; 속성은 해당 게시물의 편집 기록과 관련된 모든 ID를 제공하는 배열입니다. 게시물이 수정되지 않은 경우 이 배열에는 하나의 ID만 포함됩니다.

```
"edit_history": {
    "initial\_tweet\_id": "1283764123"
    "edit\_tweet\_ids": \["1283764123"\]
  }
```

* **edit&#95;controls** - 30분 편집 가능 시간이 언제 끝나는지와 남은 편집 가능 횟수를 나타내는 속성을 제공합니다.

```
"edit_controls": {  
     "editable\_until\_ms": 1660155761384
     "edits_remaining": 3   
  }
```

* **editable** - 게시물이 생성될 당시 수정 가능한지 여부를 나타냅니다. 

&quot;editable&quot;: true

대부분의 게시물은 수정할 수 있습니다. 그러나 다음 유형의 게시물은 수정이 불가능합니다: 

* 프로모션된 게시물
* 투표가 포함된 게시물
* 본인 스레드가 아닌 답글 게시물
* 리트윗(인용 트윗은 수정 가능함)
* nullcast된 게시물
* 커뮤니티 게시물
* 슈퍼팔로우 게시물
* 공동 작성 게시물

**수정되지 않은 게시물의 예시 속성**

아래 JSON은 게시물 수정 기능이 추가된 이후에 게시된 게시물에 포함되는 수정 메타데이터를 강조해서 보여줍니다. 이 예시는 수정 이력이 없는 게시물에 대한 것입니다. 

`"edit_tweet_ids"` 배열에 단일 ID만 있다는 점에 주의하십시오.

```
{
  "created_at": "Wed Aug 16 18:29:02 +0000 2022",
  "id": 1557433858676740098,
  "id_str": "1557433858676740098",
  "text": "I wonder if I will every use teh edit button",
  "edit_history": {
    "initial_tweet_id": "1557433858676740098",
    "edit_tweet_ids": ["1557433858676740098"]
  },
  "edit_controls": {
    "editable_until_ms": 1660155761384,
    "edits_remaining": 5
  },
  "editable": true
}
```

**편집된 게시물의 예시 속성**

아래 JSON은 편집 Post 기능이 추가된 이후에 게시된 게시물에 포함되는 편집 메타데이터를 강조해서 보여줍니다. 이 예시는 한 번만 편집된 게시물에 대한 것입니다. 

`"edit_tweet_ids"` 배열에는 원본 게시물의 ID 하나와 편집된 업데이트의 ID 하나, 이렇게 두 개의 ID가 있다는 점에 유의하세요.

```
{
  "created_at": "Wed Aug 16 18:35:42 +0000 2022",
  "id": 1557445923210514432,
  "id_str": "1557445923210514432",
  "text": "I wonder if I will ever use the edit button",
  "edit_history": {
    "initial_tweet_id": "1557433858676740098",
    "edit_tweet_ids": ["1557433858676740098", "1557445923210514432"]
  },
  "edit_controls": {
    "editable_until_ms": 1660155761384,
    "edits_remaining": 4
  },
  "editable": true
}
```

**컴플라이언스 지원**

[Compliance Firehose](/ko/x-api/enterprise-gnip-2.0/fundamentals/firehouse)와 v2 [배치 컴플라이언스 엔드포인트](/ko/x-api/compliance/batch-compliance/introduction)가 모두 게시물 편집 기능을 지원하도록 업데이트되었습니다. 

새로운 &quot;tweet&#95;edit&quot; 이벤트 유형이 Compliance Firehose에 추가되었습니다.

```
{
  "tweet_edit": {
    "id": <tweetId>,
    "initial_tweet_id": <tweetId>,
    "edit_tweet_ids": [<tweetId1>, <tweetId2>, <tweetId3> ...],
    "timestamp_ms": "<timestampMsStr>"
  }
}
```
