## 개요

> Enterprise 콘솔(console.gnip.com)은 계약된 Enterprise API 액세스 권한을 보유한 Enterprise 데이터 고객에게 제공됩니다. 현재 Enterprise 데이터 고객이 아니지만 더 많은 정보를 원하시면, [Enterprise 데이터](https://developer.x.com/en/products/x-api/enterprise)에 대해 여기에서 자세히 확인하실 수 있습니다.

이 문서는 X의 Enterprise 콘솔 대시보드에 대한 개요입니다. 여기에서 다루는 섹션은 제품 상세 정보, 사용량, 일반 계정 관리 등 콘솔의 각 개별 영역에 해당합니다. 개별 계정과 관련해 궁금한 사항이 있으면 배정된 계정 담당자에게 문의하시거나, [Enterprise 제품용 기술 문서](/ko/x-api/enterprise-gnip-2.0/enterprise-gnip)를 검토해 주십시오.

계정 담당자와 진행하는 Enterprise 체험 및 온보딩 프로세스의 일환으로, Enterprise 고객은 [console.gnip.com](http://console.gnip.com)에 로그인할 수 있는 계정을 받게 됩니다. 여기는 Enterprise 제품 액세스를 검토하고 구성할 수 있는 사용자 인터페이스입니다. 고객 관리자용 초기 콘솔 액세스는 Enterprise 온보딩 프로세스의 일부로 X에서 설정하며, 이후 관리자는 추가 사용자를 직접 추가할 수 있습니다. Enterprise 콘솔을 사용하면 Enterprise 제품과 관련된 더 많은 세부 정보를 관리하고 확인할 수 있습니다. 

아래 영상은 console.gnip.com 대시보드의 여러 구성 요소에 대한 개요를 제공합니다. 

**면책조항: 동영상에 표시된 일부 기능(특정 스트림 보강 기능 및 비-X 데이터 소스 제품 포함)은 더 이상 제공되지 않을 수 있습니다.**

<div id="products-tab">
  ## Products 탭
</div>

[console.gnip.com](http://console.gnip.com)에 계정으로 로그인하면 대시보드의 Products 탭이 처음 표시됩니다. 이 페이지에는 현재 계정에서 사용 가능한 Enterprise 제품에 대한 개요가 포함되어 있습니다.

PowerTrack 또는 Decahose처럼 스트리밍 방식으로 데이터를 전달하는 제품의 경우, 이 페이지에는 제품 이름, 스트림 레이블, 현재 활성 연결 수, 각 스트림에 대해 현재 활성화된 규칙 수(해당되는 경우), 그리고 가장 최근 24시간 동안 전달된 활동(예: 게시물)의 총 개수가 표시됩니다.

Search API처럼 REST 방식으로 데이터를 전달하는 제품의 경우, 이 페이지에는 제품 이름, 레이블(“streams”로도 표시됨), 이러한 엔드포인트를 통해 반환된 현재 활동(예: 게시물) 수, 그리고 엔드포인트별 여러 유형의 요청 개수가 표시됩니다.

[Usage API](/ko/x-api/enterprise-gnip-2.0/fundamentals/usage)를 사용하면 이 데이터의 상당 부분을 프로그래밍 방식으로 확인할 수 있습니다.

스트림별 세부 정보는 이름 또는 설정 버튼을 클릭하면 확인할 수 있습니다.

<div id="overview">
  ### 개요
</div>

메인 대시보드에서 스트림 중 하나를 클릭하면 해당 스트림의 개요 페이지로 이동합니다.

스트리밍 전송 제품의 경우, 이 페이지에는 다음 정보가 포함됩니다.

1. 각 스트림 연결을 통해 전달되는 활동 수를 보여주는 볼륨 차트

2. 스트림에서 현재 활성 상태인 연결에 대한 세부 정보(연결 id 및 IP 주소)

3. 스트림의 최근 연결, 연결 해제, 규칙 업데이트 이벤트 로그

차트의 스케일은 오른쪽 상단의 링크를 사용하여 조정할 수 있습니다. 각 연결 및 연결 해제 이벤트의 표시 여부는 차트 바로 아래 범례에서 해당 항목을 클릭해 전환할 수 있습니다.

<div id="connections">
  ### Connections
</div>

Connections 페이지에서는 스트림에서 발생한 최근 연결 이벤트에 대한 세부 정보를 제공합니다. 여기에는 각 연결의 시작 및 종료 시각(UTC 기준 24시간 형식), 각 연결의 지속 시간, 연결을 수행한 서버의 IP 주소, 참조용 고유 연결 ID, 그리고 현재 연결 상태가 포함됩니다. 상태 값은 지정된 연결에 대해 발생한 가장 최근 이벤트를 반영합니다. 예를 들어 Client Connected 또는 연결 해제 이벤트이며, 이때 연결 해제 유형이 함께 표시됩니다. 연결 디버깅에 대한 더 자세한 내용은 [disconnections explained guide](/ko/x-api/enterprise-gnip-2.0/powertrack-api#integrating-with-powertrack)를 참조하세요.

<div id="rules">
  ### Rules
</div>

Rules 탭은 PowerTrack 스트림에서 제공되며, 사용자 인터페이스를 통해 플레인 텍스트 규칙을 수동으로 입력해 손쉽게 시작할 수 있습니다. 이 인터페이스에서는 이와 같은 수동 방식으로 최대 1000개의 규칙만 추가할 수 있으므로, 초기 테스트용으로만 사용해야 합니다. 실제 운영 환경에서는 [Rules API](/ko/x-api/enterprise-gnip-2.0/powertrack-api#api-reference-index)를 통해 규칙을 프로그래밍 방식으로 관리할 것을 권장합니다.

<div id="settings">
  ### Settings
</div>

Settings 탭에서는 스트림 데이터의 출력 형식을 변경할 수 있으며, 여러 가지 형식 옵션이 지원됩니다. 형식을 변경하려면 각 옵션을 나타내는 라디오 버튼을 선택하면 됩니다. 변경 사항은 스트림에 다시 연결할 때 적용됩니다. 이 설정을 업데이트하면 다음 요청 또는 연결부터 즉시 반영되며, 새로운 형식으로 인해 기존 파서가 동작하지 않을 수도 있습니다. 

**참고:** 가장 많은 데이터를 받기 위한 권장 설정은 &quot;Leave data in the original format&quot;이며, 이 설정은 [여기](/ko/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#post-object)에 설명된 것처럼, 데이터가 enriched 네이티브 형식으로 반환되도록 합니다. Activity streams 형식은 [여기](/ko/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#activity-object)에 설명되어 있습니다.

<div id="usage-tab">
  ## Usage 탭
</div>

Usage 탭에서는 다양한 기간에 걸친 스트림 사용량을 확인할 수 있습니다. Usage 정보에 프로그램 방식으로 접근하려면 [Usage API](/ko/x-api/enterprise-gnip-2.0/fundamentals/usage)를 참고하세요.

<div id="monthly">
  ### 월간
</div>

월간 사용량 페이지에서는 제품별로 세분화된 스트림 사용량이 표시됩니다. 예를 들어, 특정 데이터 소스에 대한 커버리지 제품(예: PowerTrack, Historical PowerTrack, PowerTrack Replay)은 개별 데이터뿐 아니라 통합 집계 개수를 제공하기 위해 함께 그룹화됩니다. 개수에는 이번 달 현재까지의 사용량, 이달의 남은 기간과 지금까지의 사용량을 기반으로 한 이달 말 기준 예상 사용량, 그리고 이전 두 달의 사용량 개수가 포함됩니다.

특히, 이 개수는 각 제품 및 스트림 단위로 중복 제거되어 계산됩니다. 동일한 PowerTrack 스트림에 여러 연결을 통해 같은 게시물을 받은 경우, 이 집계에서는 그 게시물은 한 번만 계산됩니다. 개수는 매일 00:00 UTC에 업데이트됩니다.

<div id="daily">
  ### 일별
</div>

Daily Usage 페이지에서는 현재 달의 각 날짜에 대해 제품별로 구분된 일별 중복 제거 개수를 제공합니다. 이 개수는 매 24시간마다 00:00 UTC에 업데이트됩니다.

### 규칙

규칙 한도는 계약된 PowerTrack 규칙 패키지를 기준으로 결정되며, 지정된 &quot;유형&quot;의 모든 스트림에 적용됩니다. 새로운 규칙이 추가되거나 삭제될 때마다 규칙 개수가 업데이트됩니다.

  ### 내 프로필

개별 사용자 프로필 정보를 확인하고 이곳에서 비밀번호를 변경할 수 있습니다. 또한 계정별로 계정 사용 한도 알림을 설정할 수 있습니다. API 상태 알림은 https://api.twitterstat.us 에 있는 상태 페이지에서 확인할 수 있습니다.

<div id="account-settings">
  ### 계정 설정
</div>

사용자를 추가, 제거 및 수정하고, 개별 사용자와 사용 임계값 알림에 대한 이메일 알림을 구성할 수 있습니다.

<Frame>
  <img src="/images/account-settings.png.twimg.1920.png" />
</Frame>

사용자 유형은 세 가지가 있습니다 – Account Admin, User, Email Only입니다.  

* Account Admin은 다른 사용자를 생성/삭제/수정할 수 있으며, 사용자 이름(이메일)과 비밀번호를 사용하는 기본 인증을 통해 Enterprise API에 연결할 수 있습니다.

* User는 다른 사용자를 생성하거나 수정할 수 없지만, 사용자 이름(이메일)과 비밀번호를 사용하는 기본 인증을 통해 Enterprise API에 연결할 수 있습니다.

* Email Only 사용자는 대시보드에 접속할 수 없으며, Enterprise API에 연결할 권한이 없고, 알림 섹션에서 수신하도록 설정된 경우에만 알림을 받습니다.
