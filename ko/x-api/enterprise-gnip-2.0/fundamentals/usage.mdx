<div id="usage-api-twitter-api">
  ## Usage API | Twitter API
</div>

<div id="overview">
  ### 개요
</div>

`Enterprise`

이는 계정에서 사용하는 다양한 API 전반의 사용량을 모니터링하고 관리하는 데 도움이 되는 가장 핵심적이고 *가장 효과적인* 도구입니다.

**중요 고지 사항:**

Usage API에서 반환되는 사용량 개수는 체험 기간(trial) 및 기타 요금 조정으로 인해 청구서에 표시되는 사용량과 일치하지 않을 수 있습니다. 모든 수치는 특정 일자(UTC 기준) 동안 소비된 활동을 중복 제거한 결과를 기준으로 합니다.

<div id="features">
  #### 기능
</div>

* console.gnip.com UI에서 확인할 수 있는 사용량 데이터를 프로그래밍 방식으로 조회
* 스트림 수준 사용량 데이터 - 제품 수준뿐 아니라 스트림 수준(예: dev 및 prod)의 사용량 데이터를 제공
* 세분화되고 설명적인 데이터 - 검색 &quot;requests&quot;가 Full-Archive 및 30-Day Search 제품별로 구분되어 제공됨
* Historical PowerTrack &quot;days&quot; 및 &quot;jobs&quot; 데이터 

<div id="supported-apis">
  ### 지원되는 API
</div>

다음은 현재 Usage API에서 지원하는 API 목록입니다:

* PowerTrack API `enterprise`
* 30-Day Search API `enterprise`
* Full-Archive Search API `enterprise`
* Historical PowerTrack `enterprise`

<div id="limitations">
  ### 제한 사항
</div>

* Usage API를 사용하면 2018년 5월 1일부터의 사용량 데이터에 접근할 수 있습니다. 2019년 7월 1일 이후에는 Usage API를 통해 **직전 13개월(캘린더 기준)**의 사용량 데이터에 접근할 수 있습니다.

* `fromDate`와 `toDate`로 정의된 **3개월 단위 간격**으로 사용량 데이터에 접근할 수 있습니다.

<div id="methods">
  #### **Methods**
</div>

| Method | Description |
| :--- | :--- |
| [GET /metrics/usage/accounts/{ACCOUNT_NAME}.json](#GETData) | 사용량 데이터를 조회합니다 |

여기서:

* :account&#95;name은 console.gnip.com에 표시되는, 계정에 연결된 이름으로 대소문자를 구분합니다.

<div id="authentication-and-rate-limit">
  #### **인증 및 요청 한도**
</div>

<div id="authentication">
  ##### **Authentication**
</div>

모든 Usage API 요청에는 HTTP Basic Authentication이 필요하며, console.gnip.com에 로그인하거나 어떤 Gnip 스트림에 연결할 때 사용하는 계정의 활성화된 이메일/비밀번호 자격 증명을 사용해야 합니다.

<div id="rate-limit">
  ##### **요청 한도**
</div>

Usage API에는 분당 최대 2회의 요청까지 허용되는 요청 한도가 적용됩니다.

<div id="best-practices-limitations">
  #### **모범 사례 및 제한 사항**
</div>

<div id="data-availability">
  ##### **데이터 가용성**
</div>

Usage 데이터는 데이터가 처리된 마지막 전체 시간 구간(UTC)까지의, 중복 제거된 activity 사용량을 기준으로 합니다. 일반적으로 데이터는 Gnip이 시스템을 배포하는 경우를 제외하고는 매 분 단위로 처리 및 업데이트됩니다.

* Usage API를 사용하면 2018년 5월 1일부터의 Usage 데이터를 조회할 수 있습니다. 2019년 7월 1일 이후에는 Usage API를 사용하여 **직전 13개 달력 월**에 대한 Usage 데이터에 액세스할 수 있습니다. 

* fromDate와 toDate로 정의되는 **3개월 단위 구간**으로 Usage 데이터에 액세스할 수 있습니다.

<div id="requesting-and-receiving-data">
  #### **데이터 요청 및 수신**
</div>

Usage API는 사용자 계정의 API 엔드포인트에 HTTP BASIC-AUTH 자격 증명을 포함한 HTTP GET 요청을 보내는 방식으로 동작합니다.

<div id="get-request">
  ##### GET Request:
</div>

다음 endpoint에 사용자 자격 증명과 account 이름을 사용해 GET 요청을 보냅니다:

https://gnip-api.x.com/metrics/usage/accounts/:account&#95;name.json

**Additional Parameters**

|     |     |
| :--- | :--- |
| bucket | 선택 사항. 사용량 데이터가 제공될 시간 단위입니다. 사용량 데이터는 일 또는 월 단위로 반환될 수 있습니다.  <br />  <br />bucket을 지정하지 않고 요청하면 월 단위로 반환됩니다.  <br />  <br />옵션: &#39;month&#39; 또는 &#39;day&#39; |
| fromDate (YYYYMMDDHHMM) | 선택 사항. 사용량 데이터는 2018년 5월 1일부터만 조회할 수 있습니다. 사용량 데이터가 제공되는 가장 이른 UTC 타임스탬프입니다. 타임스탬프는 일 단위이며 포함 범위입니다(예: 201805010000은 05월 01일을 포함합니다). 시(hour)와 분(minute)에 &#39;0000&#39; 이외의 값이 포함된 요청은 &#39;0000&#39; 으로 기본값 처리됩니다.  <br />  <br />fromDate 또는 toDate 없이 요청하면, 현재 월의 사용량 데이터를 월 단위로 반환하며, 직전 두 달에 대한 이력 데이터도 함께 포함합니다.  <br />  <br />**참고:** 2019년 6월 1일부터는 직전 13개월치 달력 기준 사용량 데이터를 조회할 수 있습니다. 예를 들어 10월 10일이라면, 전년도 9월 1일까지의 사용량 데이터를 조회할 수 있습니다.  <br />**예:** 201810010000은 2018년 10월 1일부터(10월 1일 포함) 시작하는 데이터를 반환합니다. |
| toDate (YYYYMMDDHHMM) | 선택 사항. 사용량 데이터가 제공되는 가장 최근 UTC 타임스탬프입니다. 타임스탬프는 일 단위이며 포함 범위가 아닙니다(예: 201703020000은 03월 02일에 대한 데이터를 포함하지 않습니다). 현재 날짜 또는 미래 날짜를 toDate로 지정하면, 사용량 데이터는 마지막으로 완료된 하루(UTC)까지 반환됩니다. 시(hour)와 분(minute)에 &#39;0000&#39; 이외의 값이 포함된 요청은 &#39;0000&#39; 으로 기본값 처리됩니다.  <br />  <br />toDate 없이 요청하면, 다음 bucket이 기본값이 됩니다(bucket=day 인 경우 내일, bucket=month 인 경우 다음 달). fromDate와 toDate 모두 없이 요청하면 bucket=month가 기본값이 되며, 현재 달과 직전 두 달에 대한 데이터를 반환합니다.  <br />  <br />**예:** 201703050000은 2017년 3월 5일까지의 데이터를 반환하되, 3월 5일의 데이터는 포함하지 않습니다. |

**Example GET Request**

이 요청은 2017년 3월 1일부터 2017년 3월 5일까지(3월 5일 데이터는 제외) 월 단위로 데이터를 반환합니다.

curl -u &quot;https://gnip-api.x.com/metrics/usage/accounts/:account&#95;name.json?bucket=month&amp;fromdate=201403010000&amp;toDate=201403150000&quot;
