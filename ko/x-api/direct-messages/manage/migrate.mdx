  ## v1.1 및 v2 다이렉트 메시지 관리 엔드포인트 비교
</div>

v1.1 및 v2 버전의 다이렉트 메시지 엔드포인트는 모두 다이렉트 메시지를 생성하는 메서드를 제공합니다. 이 가이드는 두 버전 간의 차이를 이해하고 v2로 마이그레이션하는 데 필요한 정보를 제공하기 위한 것입니다. 

두 버전 간의 주요 차이점 중 하나는 v1.1은 일대일 대화만 지원하는 반면, v2는 그룹 대화를 지원한다는 점입니다. 이로 인해 나타나는 차이 중 하나는 v1.1은 &quot;message created&quot; 이벤트만 지원하는 반면, v2는 참가자가 대화에 참여하거나 떠나는 것과 관련된 이벤트도 지원한다는 것입니다. 또한 v2의 근본적인 업데이트 중 하나는 dm&#95;conversations를 핵심 API 객체로 도입한 것입니다.   

v1.1에서는 다이렉트 메시지를 관리하기 위한 두 개의 엔드포인트가 있습니다:

* POST direct&#95;messages/events/new - 일대일 다이렉트 메시지를 생성합니다. 이 v1.1 엔드포인트는 일대일 메시지만 생성할 수 있으며, 그룹 메시지는 지원하지 않습니다.  

* DELETE direct&#95;messages/events/destroy - 인증된 사용자의 보기에서 일대일 메시지를 삭제합니다. 

이 v2 릴리스에서는 다이렉트 메시지를 생성하기 위한 POST 메서드가 세 개 있습니다: 

* **POST /2/dm&#95;conversations/with/:participant&#95;id/messages** - 일대일 다이렉트 메시지를 생성합니다. 이 메서드는 기존 일대일 대화에 메시지를 추가하거나 새 대화를 생성합니다. :participant&#95;id 경로 매개변수는 메시지를 수신하는 계정의 사용자 ID입니다. 

* **POST /2/dm&#95;conversations** - 새 그룹 대화를 생성하고 여기에 다이렉트 메시지를 추가합니다. 이러한 요청에는 대화 참가자 목록이 필요합니다. 동일한 참가자 목록으로 여러 대화를 생성할 수 있다는 점에 유의하십시오. 이러한 요청은 항상 새로운 대화 ID를 반환합니다. 

* **POST /2/dm&#95;conversations/:dm&#95;conversation&#95;id/messages** - 다이렉트 메시지를 생성하고 기존 대화에 추가합니다. :dm&#95;conversation&#95;id 경로 매개변수는 메시지가 추가될 대화의 ID입니다. 

중요한 세부사항으로, 대화 및 이벤트 ID는 X 플랫폼의 v1.1 및 v2 버전에서 공통으로 사용됩니다. 이는 두 버전을 함께 사용할 수 있음을 의미합니다. 예를 들어, 다이렉트 메시지 v1.1 엔드포인트는 단일 이벤트를 반환하고 이벤트를 삭제하는 메서드를 제공하는데, 이러한 메서드는 아직 v2에서 사용할 수 없습니다. ID가 v1.1과 v2 전체에서 공통이므로, v2에서 제공된 ID를 기준으로 v1.1 요청을 하거나, X 애플리케이션의 대화 URL에 표시되는 대화 ID를 참조하여 요청할 수 있습니다.

다음 표는 v1.1 및 v2 다이렉트 메시지 관리 엔드포인트의 근본적인 측면을 비교합니다. 여기서 공유하는 X API v2 특성은 모든 다이렉트 메시지 조회 엔드포인트에 공통적으로 적용됩니다.
