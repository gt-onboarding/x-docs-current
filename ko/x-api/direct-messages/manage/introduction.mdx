Direct Messages는 X에서 비공개 대화를 할 수 있게 해 줍니다. Direct Messages는 X에서 가장 인기 있는 기능 중 하나로, 다양한 사용 사례에 활용됩니다. 이러한 사용 사례는 친구들 간의 그룹 채팅부터 전 세계 브랜드의 고객 지원까지 폭넓게 걸쳐 있습니다. 새로운 v2 Direct Messages 엔드포인트는 단계적으로 도입되며, 이 첫 단계에는 Direct Messages를 생성하고 Direct Message 대화 이벤트를 나열하기 위한 기본 엔드포인트가 포함됩니다. X API v2는 처음으로 *그룹* 대화를 지원합니다.

Manage Direct Messages의 이 초기 릴리스에는 Direct Messages를 생성하기 위한 세 가지 POST 메서드가 포함됩니다:

* **POST /2/dm&#95;conversations/with/:participant&#95;id/messages** - 1:1 Direct Message를 생성합니다. 이 메서드는 새로운 1:1 대화를 만들거나, 기존 1:1 대화를 가져와 해당 대화에 Direct Message를 추가합니다. :participant&#95;id 경로 매개변수는 메시지를 받는 계정의 User ID입니다. 
* **POST /2/dm&#95;conversations** - 새로운 그룹 대화를 생성하고 Direct Message를 해당 대화에 추가합니다. 이 요청에는 대화 참여자 목록이 필요합니다. 동일한 참여자 목록으로 여러 개의 대화를 생성할 수 있다는 점에 유의하세요. 이러한 요청은 항상 새로운 대화 ID를 반환합니다. 
* **POST /2/dm&#95;conversations/:dm&#95;conversation&#95;id/messages** - Direct Message를 생성하고 기존 대화에 추가합니다. :dm&#95;conversation&#95;id 경로 매개변수는 메시지가 추가될 대화의 ID입니다. 

Direct Message 이벤트 ID는 v1.1, v2, 그리고 X 앱 전반에서 공통이므로, Direct Messages를 숨기거나 삭제하기 위한 v1.1 메서드를 이 새로운 v2 엔드포인트와 함께 사용할 수 있습니다. 또한 Enterprise 및 Premium Account Activity API는 v2 1:1 메시지를 지원하지만, 아직 그룹 대화는 지원하지 않는다는 점에 유의하세요.   

POST 메서드는 사용자당 15분 동안 200개의 요청 한도가 있습니다. 또한 사용자당 24시간 동안 1000개의 요청 한도가 있습니다. 추가로, 24시간 동안 15000개의 요청 한도가 있습니다. 이러한 요청 한도는 이 POST 엔드포인트들 간에 공유됩니다.

manage Posts 엔드포인트를 사용해 사용자를 대신하여 요청을 보내고 있으므로, [OAuth 1.0a User Context](https://developer-staging.x.com/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)로 인증해야 하며, 귀하의 앱을 승인한 사용자와 연결된 사용자 액세스 토큰을 사용해야 합니다. OAuth 1.0a로 이 사용자 액세스 토큰을 생성하려면 [3-legged OAuth flow](/ko/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)를 사용할 수 있습니다. OAuth 2.0으로 사용자 액세스 토큰을 생성하려면 [Authorization Code with PKCE grant flow](/ko/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)를 사용할 수 있습니다.

<Note>
  **계정 설정**

  이 엔드포인트들에 접근하려면 다음이 필요합니다:

  * 승인된 [developer account](https://developer.x.com/en/portal/petition/essential/basic-info)
  * [Project](/ko/resources/fundamentals/projects) 내에 위치한 [developer App](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용해 인증

  X API v2 엔드포인트에 대한 액세스 권한을 얻는 방법은 [getting started guide](/ko/x-api/getting-started/getting-access)에서 자세히 알아보세요.
</Note>
