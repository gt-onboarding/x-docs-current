  ## v1.1과 v2 Direct Message 이벤트 조회 엔드포인트 비교
</div>

v1.1과 v2 버전의 Direct Messages 엔드포인트는 모두 Direct Message 이벤트를 조회하는 메서드를 제공합니다. 이 가이드는 두 버전 간의 차이를 이해하고 v2로 마이그레이션하는 데 필요한 정보를 제공하기 위한 것입니다. 

두 버전 간 주요 차이점 중 하나는 v1.1은 1:1 대화만 지원하는 반면, v2는 그룹 대화 지원을 도입했다는 점입니다. 이로 인한 결과 중 하나는 v1.1은 &quot;message created&quot; 이벤트만 지원하는 반면, v2는 대화에 참가자가 참여하거나 나가는 것과 관련된 이벤트도 지원한다는 점입니다. 실제로 v2의 근본적인 업데이트 중 하나는 dm&#95;conversations를 핵심 API 객체로 확립한 것입니다.   

v1.1에서는 Direct Messages를 조회하기 위한 엔드포인트가 두 개 있습니다(v1.1에서 지원되는 이벤트 유형은 새 메시지뿐입니다).

* GET direct&#95;messages/events/show - ID로 단일 이벤트를 조회합니다. 

* GET direct&#95;messages/events/list - 인증된 사용자가 주고받은 1:1 Direct Messages를 최대 30일 치까지 조회합니다. 이 메서드는 그룹 대화의 메시지는 조회할 수 없다는 점에 유의하세요. 

이번 v2 릴리스에는 Direct Message 대화 이벤트를 조회하기 위한 GET 메서드가 세 가지 있습니다. 

* **GET /2/dm&#95;conversations/with/:participant&#95;id/dm&#95;events** - 1:1 대화와 연관된 Direct Message 이벤트를 조회합니다. :participant&#95;id 경로 매개변수는 이 요청을 수행하는 인증된 사용자와 대화를 나누고 있는 계정의 User ID입니다. 

* **GET /2/dm&#95;conversations/:dm&#95;conversation&#95;id/dm&#95;events** - :dm&#95;conversation&#95;id 경로 매개변수로 지정된 특정 대화 ID와 연관된 Direct Message 이벤트를 조회합니다. 이 메서드는 1:1 대화와 그룹 대화 모두를 지원합니다. 

* **GET /2/dm&#95;events** - 사용자와 연관된 Direct Message 이벤트를 조회하며, 여기에는 1:1 대화와 그룹 대화가 모두 포함됩니다. 최대 30일 전의 이벤트까지 조회할 수 있습니다.  

중요한 점은 대화와 이벤트 ID가 X 플랫폼의 v1.1 및 v2 버전 간에 공유된다는 것입니다. 이는 두 버전을 함께 사용할 수 있음을 의미합니다. 예를 들어, Direct Messages v1.1 엔드포인트는 단일 이벤트를 반환하고 이벤트를 삭제하는 메서드를 제공하는데, 이러한 메서드는 아직 v2에서 제공되지 않습니다. ID가 v1.1과 v2에서 공통이므로, v2에서 제공되는 ID를 기반으로 v1.1 요청을 수행하거나, X 애플리케이션의 대화 URL에 표시된 대화 ID를 참고해 요청을 수행할 수 있습니다.
