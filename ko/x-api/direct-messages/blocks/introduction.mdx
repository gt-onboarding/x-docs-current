DM 차단 관리 엔드포인트를 사용하면 인증된 사용자를 대신해 지정한 계정을 차단하거나 차단 해제할 수 있습니다. 이 엔드포인트에는 두 가지 POST 메서드가 제공됩니다:

* **/2/users/:id/dm/block**: 계정을 차단합니다
* **/2/users/:id/dm/unblock**: 계정 차단을 해제합니다

  ### 시작하기
</div>

<div id="authentication">
  ### 인증
</div>

사용자를 대신해 요청을 보내므로, 해당 엔드포인트는 반드시 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) 또는 [OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용해 인증해야 하며, 요청을 수행하는 대상 사용자와 연결된 사용자 액세스 토큰을 사용해야 합니다. 이 사용자 액세스 토큰은 [3-legged OAuth 플로우](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens) (OAuth 1.0a) 또는 [Authorization Code with PKCE grant 플로우](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) (OAuth 2.0)를 사용하여 발급받을 수 있습니다.

<div id="making-a-request">
  ### 요청 보내기
</div>

Block

사용자가 앱을 통해 인증을 완료하면, 아래와 같이 그 사용자를 대신해 Block 엔드포인트를 호출할 수 있습니다:

```
      curl --request POST 'https://api.x.com/2/users/:id/dm/block' --header 'Authorization: ••••••'
```

요청이 성공적으로 완료되면 아래와 같은 JSON 응답을 확인할 수 있습니다.

```
{
   "data": {
       "blocked": true
   }
}
```

**차단 해제**

사용자가 앱에서 인증을 완료하면, 아래와 같이 해당 사용자의 권한으로 Unblock 엔드포인트를 호출할 수 있습니다.

```
      curl --request POST 'https://api.x.com/2/users/:id/dm/unblock' --header 'Authorization: ••••••'
```

요청이 성공적으로 처리되면 아래와 같은 JSON 응답이 표시됩니다:

```
{
   "data": {
       "blocked": false
   }
}
```
