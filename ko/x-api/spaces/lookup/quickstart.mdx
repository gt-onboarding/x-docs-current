<div id="getting-started-with-the-spaces-lookup-endpoints">
  ## Spaces 조회 엔드포인트 사용 시작하기
</div>

이 퀵스타트 가이드는 Postman을 사용해 지정한 필드 세트와 함께 Spaces 조회 엔드포인트 중 하나에 첫 요청을 보내는 방법을 안내합니다.

여러 프로그래밍 언어로 작성된 샘플 코드를 보고 싶다면, [X API v2 sample code GitHub](https://github.com/xdevplatform/Twitter-API-v2-sample-code) 저장소를 방문하세요.

<Note>
  ### 사전 준비 사항

  이 가이드를 완료하려면 요청을 인증하기 위한 [keys and tokens](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따르면 이러한 키와 토큰을 생성할 수 있습니다:

  * [개발자 계정에 가입](https://developer.x.com/en/apply-for-access)하고 승인을 받습니다.
  * developer portal에서 [Project](/ko/resources/fundamentals/projects)와 연결된 [developer App](/ko/resources/fundamentals/developer-apps)을 생성합니다.
  * App의 “Keys and tokens” 페이지로 이동해 필요한 자격 증명을 생성합니다. 생성한 자격 증명은 모두 안전한 위치에 저장해 두세요.
</Note>

<div id="steps-to-build-a-spaces-lookup-request">
  #### Spaces 조회 요청을 구성하는 단계
</div>

*이 예시에서는 creator ID로 사용자 Spaces를 조회하는 엔드포인트에 요청을 보내 보겠지만, 이 빠른 시작 가이드에서 배우는 내용은 다른 조회 요청에도 동일하게 적용할 수 있습니다.* 

**1단계: 도구나 라이브러리 선택**

이 엔드포인트에 요청을 보내는 데 사용할 수 있는 도구, 코드 예제, 라이브러리가 여러 가지 있지만, 여기서는 과정을 단순화하기 위해 Postman 도구를 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 로드하려면 아래 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2를 Postman에 추가
</Button>

Postman에 X API v2 컬렉션을 로드한 후 Spaces 폴더로 이동해서 &quot;Lookup Spaces created by one or more users&quot; 요청을 찾습니다.
 

**2단계: 요청 인증하기**

X API에 올바르게 요청을 보내려면 권한이 있는지 검증해야 합니다. 이를 위해 이 엔드포인트는 [OAuth 2.0 App-Only](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) 또는 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 인증 방법 중 하나로 요청을 인증할 것을 요구합니다.

단순화를 위해 이 요청에서는 OAuth 2.0 App-Only를 사용하겠지만, 비공개 [metrics](/ko/x-api/fundamentals/metrics) 이나 비공개 사용자 계정의 Spaces를 요청하려면 다른 인증 방법 중 하나를 사용해야 합니다. 

OAuth 2.0 App-Only를 사용하려면, 특히 [App Access Token](/ko/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token) (App-only Bearer Token이라고도 함)을 포함한 키와 토큰을 Postman에 추가해야 합니다. 이렇게 하려면 Postman 우측 상단에서 “X API v2”라는 이름의 환경을 선택한 뒤, 환경 드롭다운 옆의 눈 아이콘을 클릭하여 &quot;initial value&quot; 및 &quot;current value&quot; 필드에 키와 토큰을 추가합니다.

이 작업을 올바르게 완료했다면, 이 변수들은 요청의 authorization 탭으로 자동으로 가져와집니다.
 

**3단계: Spaces를 조회하려는 사용자를 식별하고 지정하기**

요청에서 실시간 또는 예정된 Spaces를 조회하려는 사용자를 반드시 지정해야 합니다. 이 예시에서는 단일 사용자 ID를 전달합니다.

사용자 ID는 계정 프로필 URL에서 찾을 수 있는 계정 핸들을 나타내는 숫자 값입니다. 예를 들어, 다음 계정의 username은 XDevelopers입니다.

https://x.com/XDevelopers

이 username을 사용자 ID로 변환하려면, username을 사용해 user lookup 엔드포인트에 요청을 보내고 응답 payload에서 숫자 형태의 사용자 ID를 찾아야 합니다. @XDevelopers의 경우, 사용자 ID는 2244994945입니다.

Postman에서 &quot;Params&quot; 탭으로 이동한 후 id 파라미터의 &quot;Value&quot; 열에 이 사용자 ID를 입력합니다.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| id  | 2244994945 |

**4단계: 조회하려는 필드를 식별하고 지정하기**

3단계까지 완료한 후 &quot;Send&quot; 버튼을 클릭하면, 기본적으로 응답에 반환되는 유일한 [Space object](/ko/x-api/fundamentals/data-dictionary#space) 필드인 id를 받게 됩니다.

추가 필드를 받고 싶다면, 요청에서 space.fields 또는 expansions 파라미터를 사용해 해당 필드를 지정해야 합니다.

이 연습에서는 서로 다른 오브젝트에서 세 가지 추가 필드 세트를 요청합니다.

* 기본 Spaces 오브젝트의 추가 title 필드
* 지정한 creator ID에 해당하는 전체 [user object](/ko/x-api/fundamentals/data-dictionary#user)
* 관련 user 오브젝트의 추가 user.created&#95;at 필드

Postman에서 “Params” 탭으로 이동한 후, “Query Params” 테이블에 다음 key:value 쌍을 추가합니다:

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| space.fields | title | creator&#95;id |
| expansions | creator&#95;id | includes.users.id, includes.users.name, includes.users.username |
| user.fields | created&#95;at | includes.users.created&#95;at |

이제 “Send” 버튼 옆에 다음 URL이 표시되는 것을 볼 수 있어야 합니다:

[https://api.x.com/2/spaces/by/creator&#95;ids?user&#95;ids=2244994945&amp;space.fields=creator&#95;id&amp;expansions=creator&#95;id&amp;user.fields=created&#95;at](https://api.x.com/2/spaces/by/creator\\_ids?user\\_ids=2244994945\&space.fields=creator\\_id\&expansions=creator\\_id\&user.fields=created_at)

**5단계: 요청을 보내고 응답을 검토합니다**

모든 설정을 완료했다면 &quot;Send&quot; 버튼을 눌러 요청을 보내고, 다음과 같은 응답을 받습니다:

```
{
   "data": [
    {
        "creator_id": "2244994945",
        "id": "1zqKVXPQhvZJB",
        "title": "Hello world 👋",
        "state": "Running"
   },
   "includes": {
       "users": [
           {
               "created_at": "2013-12-14T04:35:55.000Z",
               "name": "Twitter Dev",
               "id": "2244994945",
               "username": "TwitterDev"
           }
       ]
   }
]
}
```
