<div id="overview">
  ## Overview
</div>

필터링 스트림 endpoint 그룹은 개발자가 공개 게시물 스트림을 필터링할 수 있도록 해줍니다. 이 endpoint 그룹에는 규칙을 생성 및 관리하고, 해당 규칙을 적용해 일치하는 공개 게시물만 반환하도록 게시물 스트림을 필터링할 수 있게 해주는 여러 endpoint가 포함되어 있습니다. 이 endpoint 그룹을 사용하면 특정 주제와 이벤트를 수신하고, 경쟁과 관련된 대화를 모니터링하며, 트렌드가 어떻게 형성되는지 파악하는 등 다양한 작업을 수행할 수 있습니다.

개발자는 REST [rules endpoint](/ko/x-api/posts/filtered-stream#post-2-tweets-search-stream-rules)를 사용하여, 정의된 기준과 일치하는 게시물을 필터링하기 위한 규칙을 추가하고 제거할 수 있습니다. 이러한 [rules](/ko/x-api/posts/filtered-stream#building-rules-for-filtered-stream)는 메시지 키워드, 해시태그, URL과 같은 게시물 속성과 일치하는 operator를 사용해 생성할 수 있습니다. operator와 규칙 절(rule clause)은 부울 논리와 괄호를 조합하여 필터의 매칭 동작을 더 정교하게 조정할 수 있습니다. 

규칙 집합을 추가한 후에는 webhook을 등록할 수 있으며, 그러면 X가 JSON 형식의 [Post objects](/ko/x-api/fundamentals/data-dictionary#tweet)를 전송하기 시작합니다. webhook에는 규칙과 일치하는 콘텐츠만 전달됩니다.

이 endpoint는 수정된 게시물을 지원합니다. webhook은 하나 이상의 필터와 일치하는 수정된 게시물과 함께, 게시물의 수정 이력(게시물 ID 배열 포함)을 수신하게 됩니다. 수정 이력이 없는 게시물의 경우 이 배열에는 단일 ID가 포함됩니다. 수정된 게시물의 경우 이 배열에는 여러 개의 ID가 포함되며, 수정 순서를 반영하는 오름차순으로 정렬되어 있고, 배열의 마지막 위치에 가장 최신 버전이 위치합니다. 게시물 수정 방식에 대한 자세한 내용은 [Posts edits fundamentals](/ko/x-api/fundamentals/edit-posts) 페이지를 참고하세요. 

**참고:** 이 endpoint 그룹은 규칙을 설정하는 방식에서 [v2 filtered stream endpoint](/ko/x-api/posts/filtered-stream/introduction)와 유사합니다. 유일한 차이점은 규칙과 일치하는 게시물이 전달되는 방식입니다.

* v2 filtered stream endpoint의 경우, 지속 연결을 설정하고 규칙과 일치하는 게시물을 수신 대기해야 합니다.
* 이 filtered stream webhook endpoint의 경우, webhook을 등록하면 X가 규칙과 일치하는 게시물을 webhook으로 전달합니다.

이를 통해 클라이언트 연결 끊김이나, 서비스의 일반적인 재시작으로 인해 발생하는 서버 측 연결 종료에서 비롯되는 연결 해제를 따로 처리할 필요가 없어집니다. 초당 수백 개의 이벤트를 전달할 수 있으므로, webhook 서버는 이러한 동시 연결 시도를 처리할 수 있어야 합니다.

이 endpoint는 현재 Enterprise 개발자에게 제공되며, 다음과 같은 구성을 지원합니다:

* 프로젝트당 25,000개 이상의 규칙
* 규칙을 작성할 때 모든 operator 사용 가능
* 최대 2048자 길이의 규칙 작성 가능
* Enterprise 액세스를 위해 [여기](https://developer.x.com/en/products/x-api/enterprise/enterprise-api-interest-form)에서 신청

필터링 스트림에서 반환된 게시물은 월별 [Post cap](/ko/x-api/fundamentals/post-cap)에 포함됩니다.
