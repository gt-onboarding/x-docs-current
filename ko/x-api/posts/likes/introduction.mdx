<div id="likes-lookup">
  ### 좋아요 조회
</div>

Likes lookup 그룹의 엔드포인트를 사용하면 특정 게시물을 좋아요한 계정 목록이나 특정 계정이 좋아요한 게시물 목록을 조회할 수 있습니다. 이 엔드포인트에는 다음이 포함됩니다:

* 사용자가 좋아요한 게시물 - GET /2/users/:id/liked&#95;tweets
* 게시물을 좋아요한 사용자들 - GET /2/tweets/:id/liking&#95;users

이러한 엔드포인트는 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Bearer Token](/ko/resources/fundamentals/authentication#oauth-2-0)을 사용해 인증할 수 있습니다. 좋아요한 게시물 관련 엔드포인트의 경우, 많은 양의 결과를 페이지 단위로 조회할 수 있도록 페이지네이션 토큰이 제공됩니다.

liking users 엔드포인트는 전체 기간 동안 각 게시물당 최대 100개의 좋아요한 계정만 조회할 수 있습니다. 또한 liked Posts 엔드포인트 역시 프로젝트 수준에 적용되는 월별 [Post cap](/ko/x-api/fundamentals/post-cap)의 적용을 받습니다.
 

<div id="manage-likes">
  ### 좋아요 관리
</div>

좋아요 관리 엔드포인트를 사용하면 인증된 계정을 대신하여 특정 게시물에 좋아요를 추가하거나 취소할 수 있습니다. 이 엔드포인트 그룹에는 POST와 DELETE 두 가지 메서드가 있습니다. POST 메서드는 게시물에 좋아요를 추가할 때 사용하고, DELETE 메서드는 게시물의 좋아요를 취소할 때 사용합니다.

사용자를 대신해 요청을 보내는 것이므로, 이 엔드포인트는 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication)로 인증해야 하며, [3-legged OAuth 플로우](/ko/resources/fundamentals/authentication#oauth-1-0a-2)/obtaining-user-access-tokens)를 사용해 생성한 사용자 연관 액세스 토큰을 사용해야 합니다. 자신의 계정이나 인증된 사용자의 계정으로 게시물에 좋아요를 추가할 수 있습니다. 두 엔드포인트 모두에 대해 사용자 단위 레이트 리밋은 엔드포인트별로 15분당 50개 요청입니다. 
 

이 엔드포인트에 액세스하려면 승인된 [개발자 계정](/ko/resources/fundamentals/developer-portal)이 있어야 합니다. 인증 시에는 [프로젝트](/ko/resources/fundamentals/projects) 내에 있는 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용해야 합니다. 

X API v2 엔드포인트에 액세스하는 방법은 [시작하기](/ko/x-api/getting-started/about-x-api) 페이지에서 자세히 확인할 수 있습니다.
