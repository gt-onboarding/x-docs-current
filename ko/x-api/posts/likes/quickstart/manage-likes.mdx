<div id="getting-started-with-the-manage-likes-endpoints">
  ### Likes 관리 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 [Postman](/ko/tutorials/postman-getting-started)을 사용해 Likes 관리 엔드포인트에 첫 요청을 보내는 데 도움이 됩니다.

다른 언어로 된 샘플 코드를 보고 싶다면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요. 

<div id="prerequisites">
  #### 사전 준비 사항
</div>

이 가이드를 완료하려면 [키와 토큰](/ko/resources/fundamentals/authentication) 세트가 필요하며, 다음 단계를 따라 생성할 수 있습니다.

1. [개발자 계정 신청](https://developer.x.com/en/apply-for-access)을 완료하고 승인을 받습니다.

2. 개발자 포털에서 [프로젝트](/ko/resources/fundamentals/projects)를 만들고 연결된 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성합니다.

3. 앱의 &quot;Keys and tokens&quot; 페이지로 이동하여 API 키, 액세스 토큰, Bearer Token을 비밀번호 관리자에 저장합니다.

<div id="steps-to-build-a-manage-likes-request">
  #### 좋아요 관리 요청을 구성하는 단계
</div>

**1단계: 도구 또는 라이브러리 선택**

이 엔드포인트에 요청을 보내는 데 사용할 수 있는 도구, 코드 예제, 라이브러리가 여러 가지 있지만, 여기서는 과정을 단순화하기 위해 Postman 도구를 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 로드하려면, 아래 버튼을 클릭하세요:

[![](https://cdn.cms-twdigitalassets.com/content/dam/developer-twitter/m1_vnext/carat.svg)<br />X API v2를 Postman에 추가](https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400)

Postman에 X API v2 컬렉션을 로드한 후, &quot;Likes&quot; 폴더로 이동해서 &quot;Like a Post&quot;를 선택합니다.

**2단계: 요청 인증**

이 엔드포인트에 성공적으로 요청하려면 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication)를 사용해야 합니다. 이를 위해 &quot;X API v2&quot;라는 이름의 환경을 선택한 뒤, 다음 키와 토큰을 초기 값 및 현재 값 필드에 변수로 추가해야 합니다:

* consumer&#95;key: API Key
* consumer&#95;secret: API Key Secret
* access&#95;token: Access Token
* token&#95;secret: Access Token Secret
   

**3단계: 좋아요를 누를 게시물 지정**

좋아요 관리 엔드포인트에는 두 개의 ID가 필요합니다. 하나는 사용자(게시물에 좋아요를 누르려는 사용자)의 ID이고, 다른 하나는 사용자가 좋아요 또는 좋아요 취소를 시도하는 게시물의 Post ID입니다. 

사용자의 ID는 인증하는 사용자의 ID와 일치해야 합니다. 즉, 요청을 인증할 때 해당 사용자 ID와 연관된 [Access Tokens](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)를 전달해야 합니다. 이 경우, 본인 사용자 계정의 ID를 지정할 수 있습니다. ID는 다음 두 가지 방법으로 찾을 수 있습니다:

1. [user lookup by username](/ko/x-api/users/user-lookup-by-id) 엔드포인트를 사용해 사용자 이름을 전달하면 id 필드를 받을 수 있습니다. 
2. Access Token 값을 보면, 숫자 부분이 본인의 사용자 ID입니다.
    

또한 좋아요를 누르려는 게시물도 지정해야 합니다. x.com에 접속해서 게시물을 클릭한 다음, URL을 보면 Post ID를 확인할 수 있습니다. 예를 들어, 아래 URL의 Post ID는 1228393702244134912입니다.

[https://x.com/TwitterDev/status/1228393702244134912](https://x.com/TwitterDev/status/1228393702244134912)

Postman에서 &quot;Params&quot; 탭으로 이동한 뒤, id 경로 변수의 &quot;Value&quot; 열에 본인의 ID를 입력합니다. 그런 다음 &quot;Body&quot; 탭으로 이동하여 좋아요를 누르려는 게시물의 ID를 tweet&#95;id 파라미터의 값으로 입력합니다. 어떤 ID 앞뒤에도 공백이 들어가지 않도록 주의하세요.

|              |                     |
| :----------- | :------------------ |
| **Key**      | **Value**           |
| `id`         | (본인의 사용자 ID)        |
| tweet&#95;id | (좋아요를 누르려는 게시물의 ID) |

&quot;Send&quot; 버튼을 클릭하면 관계 상태를 포함한 응답 객체를 받게 됩니다:

* &quot;liked&quot;: true를 받으면, id가 tweet&#95;id에 성공적으로 좋아요를 누른 것입니다.
   

**4단계: 요청 전송 및 응답 검토**

모든 설정을 마쳤다면, &quot;Send&quot; 버튼을 눌러 다음과 같은 응답을 받게 됩니다:

```
{
    "data": {
        "liked": true
    }
}
```

동일한 게시물의 좋아요를 취소하려면 Postman의 X API v2 컬렉션 내 “Likes” 폴더에 있는 “Unlike a Post” 요청을 사용할 수 있습니다. `id`와 `tweet_id`는 이전 예시와 동일한 방식으로 사용해야 합니다. 게시물의 좋아요를 취소할 때는 이를 JSON 본문으로 추가할 필요가 없으므로, `id`와 `tweet_id`에 필요한 쿼리 매개변수를 꼭 추가했는지 확인해야 합니다.
