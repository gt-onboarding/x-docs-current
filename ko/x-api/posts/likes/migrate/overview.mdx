<div id="comparing-x-apis-likes-endpoints">
  ## X API의 좋아요(Likes) 엔드포인트 비교
</div>

이 가이드에서는 다음 영역에 중점을 둡니다:

* **API 요청 매개변수** - X API v2 엔드포인트는 새로운 요청 매개변수 세트를 도입합니다. 일부 매개변수는, 특히 Labs를 사용해 통합해 본 분들께는 익숙하겠지만, [fields](/ko/x-api/fundamentals/fields) 및 [expansions](/ko/x-api/fundamentals/expansions) 매개변수의 도입과 같은 여러 중요한 차이점이 있습니다.

* **앱 및 프로젝트 요구 사항** - X API v2에 액세스하려면 [프로젝트](/ko/resources/fundamentals/projects)와 연결된 [developer App](/ko/resources/fundamentals/developer-apps)의 자격 증명을 사용해야 합니다.

<div id="likes-lookup">
  #### 좋아요 조회
</div>

**게시물에 좋아요를 누른 사용자**

좋아요를 누른 사용자를 조회하는 엔드포인트는 X API v2에서 새로 제공되는 기능으로, 특정 게시물을 좋아요한 사용자에 대한 정보를 가져올 수 있습니다.

| 설명 | X API v2 |
| :--- | :--- |
| 지원 HTTP 메서드 | GET |
| 호스트 도메인 | https://api.x.com |
| 엔드포인트 경로 | /2/tweets/:id/liking&#95;users |
| [인증](/ko/resources/fundamentals/authentication) | OAuth 2.0 Bearer Token<br /><br />OAuth 1.0a User Context |
| 기본 요청 [요청 한도](/ko/resources/fundamentals/rate-limits) | 15분당 75회 요청 (앱당)<br /><br />15분당 75회 요청 (사용자당) |

**사용자가 좋아요한 게시물**

다음 표는 표준 v1.1 [GET favorites/list](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-favorites-list) 엔드포인트와 X API v2 좋아요한 게시물 관련 엔드포인트를 비교합니다:

| 설명 | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| 지원 HTTP 메서드 | GET | GET |
| 호스트 도메인 | https://api.x.com | https://api.x.com |
| 엔드포인트 경로 | /1.1/favorites/list.json | /2/users/:id/liked&#95;tweets |
| [인증](/ko/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 2.0 Bearer Token<br /><br />OAuth 1.0a User Context |
| 기본 요청 [요청 한도](/ko/resources/fundamentals/rate-limits) | 15분당 75회 요청 | 15분당 75회 요청 (앱당)<br /><br />15분당 75회 요청 (사용자당) |
| 데이터 형식 | 표준 v1.1 형식 | [X API v2 형식](/ko/x-api/fundamentals/data-dictionary) (요청 매개변수인 fields 및 expansions에 의해 결정되며, v1.1 형식과 하위 호환되지 않음)<br /><br />표준 v1.1 형식에서 X API v2 형식으로 마이그레이션하는 방법을 자세히 알아보려면 [데이터 형식 마이그레이션 가이드](/ko/x-api/migrate/data-format-migration)를 방문하세요. |

<div id="manage-likes">
  #### 좋아요 관리
</div>

v2 좋아요 관리 엔드포인트는 v1.1의 [POST favorites/create](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-create) 및 [POST favorites/destroy](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-destroy) 엔드포인트를 대체합니다.

아래 표는 표준 v1.1과 X API v2의 좋아요 관리 엔드포인트를 비교합니다.

<div id="like-a-post">
  #### 게시물에 좋아요 표시
</div>

| 설명 | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| 지원되는 HTTP 메서드 | POST | POST |
| 호스트 도메인 | https://api.x.com | https://api.x.com |
| 엔드포인트 경로 | /1.1/favorites/create.json | /2/users/:id/likes |
| [Authentication](/ko/resources/fundamentals/authentication) | OAuth 1.0a 사용자 컨텍스트 | OAuth 1.0a 사용자 컨텍스트 |
| 기본 [요청 한도](/ko/resources/fundamentals/rate-limits) | 24시간당 1000회 요청(사용자당)<br /><br />24시간당 1000회 요청(앱당) | 15분당 50회 요청(사용자당)<br /><br />24시간당 1000회 요청(사용자당, DELETE와 공유) |

<div id="unlike-a-post">
  #### 게시물 좋아요 취소
</div>

| 설명 | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| 지원되는 HTTP 메서드 | POST | DELETE |
| 호스트 도메인 | https://api.x.com | https://api.x.com |
| 엔드포인트 경로 | /1.1/favorites/destroy.json | /2/users/:id/likes/:tweet&#95;id |
| [인증](/ko/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context |
| 기본 요청 [요청 한도](/ko/resources/fundamentals/rate-limits) | 24시간 기준 1000회 요청 (사용자당)<br /><br />24시간 기준 1000회 요청 (앱당) | 15분 기준 50회 요청 (사용자당)<br /><br />24시간 기준 1000회 요청 (사용자당, POST와 공유) |
