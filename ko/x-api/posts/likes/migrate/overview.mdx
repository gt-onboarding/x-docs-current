<div id="comparing-x-apis-likes-endpoints">
  ## X API의 좋아요(Like) 엔드포인트 비교
</div>

이 가이드에서는 다음 영역에 중점을 둡니다:

* **API 요청 파라미터** - X API v2 엔드포인트에서는 새로운 요청 파라미터 집합을 제공합니다. 특히 Labs를 이미 사용해 본 경우 익숙한 파라미터도 있지만, [fields](/ko/x-api/fundamentals/fields) 및 [expansions](/ko/x-api/fundamentals/expansions) 파라미터 도입처럼 여러 가지 중요한 차이점이 있습니다.

* **앱 및 프로젝트 요구 사항** - X API v2에 액세스하려면 [Project](/ko/resources/fundamentals/projects)와 연결된 [developer App](/ko/resources/fundamentals/developer-apps)의 자격 증명을 사용해야 합니다.

<div id="likes-lookup">
  #### 좋아요 조회
</div>

**게시물에 좋아요를 누른 사용자**

liked&#95;users 엔드포인트는 v2의 새로운 기능으로, 게시물에 좋아요를 누른 사용자에 대한 정보를 가져올 수 있습니다.

| 설명 | X API v2 |
| :--- | :--- |
| 지원되는 HTTP 메서드 | GET |
| 호스트 도메인 | https://api.x.com |
| 엔드포인트 경로 | /2/tweets/:id/liking&#95;users |
| [인증](/ko/resources/fundamentals/authentication) | OAuth 2.0 Bearer Token<br /><br />OAuth 1.0a User Context |
| 기본 [요청 한도](/ko/resources/fundamentals/rate-limits) | 15분당 75회 요청 (앱당)<br /><br />15분당 75회 요청 (사용자당) |

**사용자가 좋아요한 게시물**

아래 표는 표준 v1.1 [GET favorites/list](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-favorites-list) 엔드포인트와 X API v2 liked Posts 엔드포인트를 비교한 것입니다:

| 설명 | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| 지원되는 HTTP 메서드 | GET | GET |
| 호스트 도메인 | https://api.x.com | https://api.x.com |
| 엔드포인트 경로 | /1.1/favorites/list.json | /2/users/:id/liked&#95;tweets |
| [인증](/ko/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 2.0 Bearer Token<br /><br />OAuth 1.0a User Context |
| 기본 [요청 한도](/ko/resources/fundamentals/rate-limits) | 15분당 75회 요청 | 15분당 75회 요청 (앱당)<br /><br />15분당 75회 요청 (사용자당) |
| 데이터 형식 | Standard v1.1 형식 | [X API v2 형식](/ko/x-api/fundamentals/data-dictionary) (fields 및 expansions 요청 파라미터에 따라 결정되며, v1.1 형식과는 하위 호환되지 않음)<br /><br />Standard v1.1 형식에서 X API v2 형식으로 마이그레이션하는 방법에 대해 자세히 알아보려면 [데이터 형식 마이그레이션 가이드](/ko/x-api/migrate/data-format-migration)를 참조하세요. |

<div id="manage-likes">
  #### 좋아요 관리
</div>

X API v2 좋아요 관리 엔드포인트는 v1.1의 [POST favorites/create](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-create) 및 [POST favorites/destroy](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-destroy) 엔드포인트를 대체합니다.

다음 표는 표준 v1.1과 X API v2의 좋아요 관리 엔드포인트를 비교한 것입니다.

<div id="like-a-post">
  #### 게시물 좋아요
</div>

| Description | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| HTTP methods supported | POST | POST |
| Host domain | https://api.x.com | https://api.x.com |
| Endpoint path | /1.1/favorites/create.json | /2/users/:id/likes |
| [Authentication](/ko/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context |
| Default request [rate limits](/ko/resources/fundamentals/rate-limits) | 24시간당 사용자당 1,000회 요청<br /><br />24시간당 앱당 1,000회 요청 | 15분당 사용자당 50회 요청<br /><br />24시간당 사용자당 1,000회 요청(DELETE 요청과 공유) |

<div id="unlike-a-post">
  #### 게시물 좋아요 취소
</div>

| Description | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| HTTP methods supported | POST | DELETE |
| Host domain | https://api.x.com | https://api.x.com |
| Endpoint path | /1.1/favorites/destroy.json | /2/users/:id/likes/:tweet&#95;id |
| [Authentication](/ko/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context |
| Default request [rate limits](/ko/resources/fundamentals/rate-limits) | 24시간당 1000개 요청(사용자당)<br /><br />24시간당 1000개 요청(앱당) | 15분당 50개 요청(사용자당)<br /><br />24시간당 1000개 요청(사용자당, POST와 공유) |
