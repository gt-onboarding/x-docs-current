<div id="retweets-lookup-standard-v11-compared-to-x-api-v2">
  ### 리트윗 조회: 표준 v1.1과 X API v2 비교
</div>

표준 v1.1의 [v1.1 GET statuses/retweets/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-retweets-id), [v1.1 GET statuses/retweets/:ids](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-retweeters-ids)를 사용해 왔다면, 이 가이드는 표준 v1.1과 X API v2 리트윗 조회 엔드포인트 간의 공통점과 차이점을 이해하는 데 도움이 되도록 작성되었습니다.

* **유사점**
  * 인증
  * 요청당 사용자 수 제한
* **차이점**
  * 엔드포인트 URL

  * 요청 제한

  * 앱 및 프로젝트 요구 사항

  * 응답 데이터 형식

  * 요청 파라미터

<div id="similarities">
  #### 유사점
</div>

**인증**

표준 v1.1 및 X API v2 Retweets 조회 엔드포인트([v1.1 GET statuses/retweets/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-retweets-id) 및 [v1.1 GET statuses/retweeters/:ids](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-retweeters-ids))는 모두 [OAuth 1.0a 사용자 컨텍스트](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication) 또는 OAuth 2.0 Bearer Token을 사용합니다.

**요청당 사용자 수 한도**

v1.1 및 v2 GET 엔드포인트 모두에서 Retweets 조회 엔드포인트가 반환하는 최대 사용자 수는 100명입니다. v2 Retweets 조회 엔드포인트의 경우 페이지네이션 토큰은 전달되지 않으며, 기본적으로 100명의 사용자를 반환하며 이것이 반환 가능한 최대치입니다.

<div id="differences">
  #### Differences
</div>

**Endpoint URLs**

* Standard v1.1 endpoints:
  * https://api.x.com/1.1/statuses/retweets/:id.json
    (`id` 파라미터로 지정된 게시물에 대한 가장 최근 리트윗 100개의 컬렉션을 반환합니다)
  * `https://api.x.com/1.1/statuses/retweeters/ids.json`
    (`id` 파라미터로 지정된 게시물을 리트윗한 사용자들의 최대 100개 사용자 ID 컬렉션을 반환합니다)
* X API v2 endpoint:
  * https://api.x.com/2/tweets/:id/retweeted&#95;by
    (특정 게시물을 리트윗한 계정 목록을 반환합니다)

**App and Project requirements**

X API v2 엔드포인트에서는 요청을 인증할 때 [developer App](/ko/resources/fundamentals/developer-apps)과 [Project](/ko/resources/fundamentals/developer-apps)에 연결된 자격 증명을 사용해야 합니다. 모든 X API v1.1 엔드포인트는 프로젝트에 연결되지 않은 독립형 앱이든 프로젝트에 연결된 앱이든, 어떤 앱의 자격 증명도 사용할 수 있습니다.

**Response data format**

표준 v1.1과 X API v2 엔드포인트 버전 간의 가장 큰 차이점 중 하나는 어떤 필드를 페이로드에 반환할지 선택하는 방식입니다.

표준 엔드포인트에서는 응답 필드의 상당수를 기본값으로 수신하며, 이후 파라미터를 사용해 어떤 필드 또는 필드 집합을 페이로드에 반환할지 선택할 수 있습니다.

X API v2에서는 기본적으로 사용자 id, name, username 필드만 반환합니다. 추가 필드나 객체를 요청하려면 [fields](/ko/x-api/fundamentals/fields) 및 [expansions](/ko/x-api/fundamentals/expansions) 파라미터를 사용해야 합니다. 이 엔드포인트에서 요청한 모든 사용자 필드는 기본 user 객체에 포함되어 반환됩니다. 확장된 게시물 객체 및 필드는 응답의 includes 객체에 포함되어 반환됩니다. 그런 다음 user 객체와 확장된 게시물 객체 양쪽에 있는 ID를 매칭하여 확장된 객체를 user 객체에 다시 연결할 수 있습니다. 

이러한 새로운 파라미터에 대해서는 각 가이드를 참고하거나, [fields 및 expansions 사용 방법](/ko/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)에 대한 가이드를 읽어 보시기를 권장합니다. 

또한 표준 v1.1 필드를 새로운 v2 필드에 매핑하는 데 도움이 되는 [data format migration guide](/ko/x-api/migrate/data-format-migration#migrating-from-standard-v1-1s-data-format-to-v2)를 제공합니다. 이 가이드는 특정 필드를 반환하기 위해 v2 요청과 함께 전달해야 하는 expansions 및 field 파라미터도 구체적으로 안내합니다. 
 

특정 필드를 요청하는 방식의 변화에 더해, X API v2는 API가 반환하는 객체(예: [Post](/ko/x-api/fundamentals/data-dictionary#tweet) 및 [user](/ko/x-api/fundamentals/data-dictionary#user) 객체)에 대해 새로운 JSON 설계를 도입하고 있습니다.

* JSON 루트 레벨에서, 표준 엔드포인트는 Post 객체를 statuses 배열로 반환하는 반면, X API v2는 data 배열로 반환합니다. 
* 리트윗 및 인용된 &quot;statuses&quot;를 참조하는 대신, X API v2 JSON은 리트윗 및 인용된 트윗을 참조합니다. contributors 및 user.translator&#95;type과 같이 레거시이거나 사용 중단된 필드 다수가 제거되고 있습니다. 
* Post 객체의 favorites 및 user 객체의 favourites를 둘 다 사용하는 대신, X API v2는 like라는 용어를 사용합니다. 
* X는 값이 없는 JSON 값(예: null)은 페이로드에 기록하지 않는 관례를 채택하고 있습니다. Post 및 user 속성은 null이 아닌 값을 가진 경우에만 포함됩니다. 
   

또한 [Post object](/ko/x-api/fundamentals/data-dictionary#tweet)에 다음과 같은 새로운 필드 집합을 도입했습니다.

* [conversation&#95;id](/ko/x-api/fundamentals/conversation-id) 필드
* context 및 entities를 포함하는 두 개의 새로운 [annotations](/ko/x-api/fundamentals/post-annotations) 필드
* 여러 새로운 [metrics](/ko/x-api/fundamentals/metrics) 필드 
* 특정 게시물에 누가 답글을 달 수 있는지 표시하는 새로운 reply&#95;setting 필드

**Request parameters**

다음 표준 v1.1 요청 파라미터는 두 개의 요청 쿼리 파라미터(user&#95;id 또는 screen&#95;name)를 허용했습니다. X API v2는 숫자형 사용자 ID만 허용하며, 반드시 엔드포인트 경로의 일부로 전달해야 합니다.
