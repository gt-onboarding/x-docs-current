<div id="authentication">
  #### 인증
</div>

모든 X API v2 엔드포인트는 키와 토큰이라고도 불리는 자격 증명 세트를 사용해 요청을 인증해야 합니다.

Retweets 조회 엔드포인트에 대한 요청은 OAuth 1.0a 사용자 컨텍스트 또는 OAuth 2.0 Bearer Token을 사용해 인증할 수 있습니다. 

Retweets 관리 엔드포인트는 OAuth 1.0a 사용자 컨텍스트 사용이 필요합니다. 이는 성공적인 요청을 위해 API 키와 사용자 액세스 토큰 세트를 사용해야 함을 의미합니다. 액세스 토큰은 요청을 대신 보내려는 대상 사용자 계정과 연결되어 있어야 합니다. 다른 사용자를 위한 액세스 토큰 세트를 생성하려면, 해당 사용자가 3-legged OAuth 플로우를 사용해 귀하의 앱을 승인하거나 인증해야 합니다.

OAuth 1.0a는 사용하기 까다로울 수 있습니다. 이 인증 방식에 익숙하지 않다면, 요청을 올바르게 인증하기 위해 [라이브러리](/ko/x-api/tools-and-libraries/overview)를 사용할 것을 권장합니다.

**참고**

다음 필드를 요청하는 경우 OAuth 1.0a 사용자 컨텍스트가 필요합니다. 

* tweet.fields.non&#95;public&#95;metrics
* tweet.fields.promoted&#95;metrics
* tweet.fields.organic&#95;metrics
* media.fields.non&#95;public&#95;metrics
* media.fields.promoted&#95;metrics
* media.fields.organic&#95;metrics

  #### developer portal, 프로젝트, 그리고 개발자 앱

X API v2 엔드포인트에서 사용할 수 있는 인증에 필요한 자격 증명 세트를 발급받으려면, [개발자 계정에 가입](https://developer.x.com/en/portal/petition/essential/basic-info)하고 해당 계정 내에 [프로젝트](/ko/resources/fundamentals/projects)를 설정한 다음, 그 프로젝트 내에 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성해야 합니다. 그런 다음 개발자 앱 내에서 키와 토큰을 확인할 수 있습니다. 
 

<div id="rate-limits">
  #### 요청 한도
</div>

매일 수천 명이 넘는 개발자들이 X API에 요청을 보냅니다. 이러한 대량의 요청을 관리하기 위해, [요청 한도](/ko/x-api/fundamentals/rate-limits)가 각 엔드포인트에 적용되어 있으며, 이는 앱 또는 인증된 사용자를 대신해 보낼 수 있는 요청 수를 제한합니다.

리트윗 관리 엔드포인트는 사용자당 15분에 50개의 요청으로 제한됩니다. 추가로, POST 엔드포인트의 경우 사용자별, 앱별 3시간 단위 윈도우당 300개의 요청으로 제한됩니다. 

리트윗 조회 엔드포인트의 경우, 15분 단위 윈도우당 75개의 요청으로 제한됩니다. 또한, 이 엔드포인트는 가장 최근에 리트윗한 사용자 100명만 반환합니다.

<div id="fields-and-expansions">
  #### 필드와 expansions
</div>

X API v2를 사용하면 개발자가 fields 및 expansions라는 도구 집합을 통해 API에서 어떤 데이터를 반환받을지 정확히 선택할 수 있습니다. `expansions` 파라미터를 사용하면 페이로드 내에서 참조된 객체를 확장해 가져올 수 있습니다. 예를 들어, 이 엔드포인트에서는 다음과 같은 [expansions](/ko/x-api/fundamentals/expansions)을 가져올 수 있습니다:

* attachments.poll&#95;ids
* attachments.media&#95;keys
* author&#95;id, entities.mentions.username
* geo.place&#95;id
* in&#95;reply&#95;to&#95;user&#95;id,
* referenced&#95;tweets.id,
* referenced&#95;tweets.id.author&#95;id

`fields` 파라미터를 사용하면 서로 다른 데이터 객체 내에서 어떤 [fields](/ko/x-api/fundamentals/fields)를 수신할지 정확히 선택할 수 있습니다. 이 엔드포인트는 주로 Post(게시물) 객체를 반환합니다. 기본적으로 Post 객체는 `id`와 `text` 필드를 반환합니다. `tweet.created_at` 또는 `tweet.entities`와 같은 추가 필드를 받으려면 `fields` 파라미터를 사용해 해당 필드를 명시적으로 요청해야 합니다. 통합 시 고려해 볼 만한 중요한 필드로는 투표(poll) 데이터, 메트릭, 게시물 주석(annotations) 정보, 그리고 conversation&#95;id(대화 ID) 필드 등이 있습니다.

[X API v2 데이터 사전](/ko/x-api/fundamentals/data-dictionary)에 [fields와 expansions 사용 방법](/ko/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)을 함께 설명하는 가이드를 제공하고 있습니다.
