<div id="authentication">
  #### 인증
</div>

모든 X API v2 엔드포인트에서는 키와 토큰이라고 부르는 자격 증명 세트를 사용해 요청을 인증해야 합니다.

리트윗 조회 엔드포인트에 대한 요청을 인증할 때는 OAuth 1.0a 사용자 컨텍스트 또는 OAuth 2.0 Bearer Token 중 하나를 사용할 수 있습니다. 

리트윗 관리 엔드포인트는 OAuth 1.0a 사용자 컨텍스트 사용을 요구합니다. 이는 곧, 요청을 성공적으로 수행하려면 API 키 세트와 사용자 액세스 토큰을 사용해야 함을 의미합니다. 액세스 토큰은 요청을 대신 수행하려는 사용자와 연결되어 있어야 합니다. 다른 사용자를 위한 액세스 토큰 세트를 생성하려는 경우, 해당 사용자는 3-legged OAuth 플로우를 사용하여 앱을 승인하거나 인증해야 합니다.

OAuth 1.0a는 사용하기 까다로울 수 있습니다. 이 인증 방식에 익숙하지 않다면, 요청을 올바르게 인증하기 위해 [라이브러리](/ko/x-api/tools-and-libraries/overview)를 사용할 것을 권장합니다.

**주의 사항**

다음 필드를 요청하는 경우, OAuth 1.0a 사용자 컨텍스트가 필요합니다: 

* tweet.fields.non&#95;public&#95;metrics
* tweet.fields.promoted&#95;metrics
* tweet.fields.organic&#95;metrics
* media.fields.non&#95;public&#95;metrics
* media.fields.promoted&#95;metrics
* media.fields.organic&#95;metrics

<div id="rate-limits">
  #### 요청 한도
</div>

매일 수천 명이 넘는 개발자가 X API에 요청을 보냅니다. 이러한 요청의 방대한 양을 관리하기 위해 각 엔드포인트에는 [요청 한도](/ko/x-api/fundamentals/rate-limits)가 설정되어 있으며, 이를 통해 앱 또는 인증된 사용자를 대신해 보낼 수 있는 요청 수가 제한됩니다.

리트윗 관리 엔드포인트는 15분당 사용자당 50건으로 제한됩니다. 또한 POST 엔드포인트의 경우 3시간 단위 창에서 사용자당, 앱당 300건으로 제한됩니다. 

리트윗 조회 엔드포인트의 경우 15분당 75건의 요청으로 제한됩니다. 또한 이 엔드포인트는 가장 최근에 리트윗한 사용자 100명만 반환합니다.

<div id="fields-and-expansions">
  #### 필드와 expansions
</div>

X API v2를 사용하면 `fields`와 `expansions`라는 도구 집합을 통해 API에서 어떤 데이터를 반환받을지 정확하게 선택할 수 있습니다. `expansions` 파라미터를 사용하면 페이로드 내에서 참조된 객체를 확장해 가져올 수 있습니다. 예를 들어, 이 엔드포인트에서는 다음과 같은 [expansions](/ko/x-api/fundamentals/expansions)을 가져올 수 있습니다:

* attachments.poll&#95;ids
* attachments.media&#95;keys
* author&#95;id, entities.mentions.username
* geo.place&#95;id
* in&#95;reply&#95;to&#95;user&#95;id,
* referenced&#95;tweets.id,
* referenced&#95;tweets.id.author&#95;id

`fields` 파라미터를 사용하면 서로 다른 데이터 객체 내에서 받고자 하는 정확한 [fields](/ko/x-api/fundamentals/fields)를 선택할 수 있습니다. 이 엔드포인트들은 주로 게시물(Post) 객체를 반환합니다. 기본적으로 Post 객체는 `id`와 `text` 필드를 반환합니다. `tweet.created_at` 또는 `tweet.entities`와 같은 추가 필드를 받으려면, `fields` 파라미터를 사용해 해당 필드를 명시적으로 요청해야 합니다. 통합 시 고려해볼 만한 중요한 필드로는 설문조사(poll) 데이터, 메트릭, 게시물 주석(annotations), 그리고 대화 ID 필드 등이 있습니다.

[X API v2 데이터 사전](/ko/x-api/fundamentals/data-dictionary)에는 [fields와 expansions를 함께 사용하는 방법](/ko/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)에 대한 가이드를 제공하고 있습니다.
