<div id="retweets-lookup">
  ### 리트윗 조회
</div>

Retweets lookup 엔드포인트를 사용하면 특정 게시물을 리트윗한 계정 목록을 조회할 수 있습니다. 이 엔드포인트에서는 한 번에 최대 100명까지의 사용자 단위로 큰 결과 집합을 페이지 단위로 조회할 수 있도록 페이지네이션 토큰이 제공됩니다. 

이 엔드포인트는 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication), [OAuth 2.0 앱 전용(App-Only)](https://developer.x.com(/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), 또는 [OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3)로 인증할 수 있습니다. 

<div id="retweets-of-me">
  ### 나에 대한 리포스트
</div>

Reposts of Me 조회를 사용하면 다른 사용자가 공유한 자신의 게시물이 무엇인지 확인할 수 있습니다.

이 기능은 내 게시물 중 어떤 것들이 공유되어 오디언스에게 공감을 얻고 있는지 파악할 수 있게 해 주어, 더 정교한 상호 작용과 콘텐츠 전략 수정을 하는 데 도움이 됩니다.

이 엔드포인트들은 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용하여 인증할 수 있습니다.

<div id="manage-retweets">
  #### 리트윗 관리
</div>

Retweet 관리 엔드포인트를 사용하면 인증된 계정을 대신해 지정된 게시물을 리트윗하거나, 해당 게시물에 대한 리트윗을 취소할 수 있습니다. 이 엔드포인트 그룹에는 POST와 DELETE 두 가지 메서드가 있습니다. POST 메서드는 게시물을 리트윗할 수 있게 해 주며, DELETE 메서드는 특정 게시물에 대한 리트윗을 취소할 수 있게 해 줍니다.

사용자를 대신해 요청을 보내는 것이므로, 이 엔드포인트는 반드시 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용해 인증해야 하며, 요청을 수행하려는 대상 사용자와 연결된 사용자 액세스 토큰을 사용해야 합니다. 이 사용자 액세스 토큰은 [3-legged OAuth flow](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) 또는 [Authorization Code with PKCE grant flow](/ko/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0)를 사용해 생성할 수 있습니다. 자신의 계정에서든 인증된 사용자의 계정에서든 게시물을 리트윗할 수 있습니다. 두 엔드포인트 모두, 엔드포인트별로 사용자 기준 15분당 50회의 요청이라는 레이트 리밋이 적용됩니다.
