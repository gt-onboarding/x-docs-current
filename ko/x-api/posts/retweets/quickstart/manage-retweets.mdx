<div id="getting-started-with-the-manage-retweets-endpoints">
  ### 리트윗 관리 엔드포인트 사용 시작하기
</div>

이 빠른 시작 가이드는 Postman을 사용하여 리트윗 관리 엔드포인트에 첫 요청을 보내는 방법을 안내합니다.

여러 언어로 작성된 샘플 코드를 보고 싶다면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 저장소를 방문하세요. 

<div id="prerequisites">
  #### 사전 준비 사항
</div>

이 가이드를 완료하려면 요청을 인증하는 데 사용할 [키와 토큰](/ko/resources/fundamentals/authentication) 묶음이 필요합니다. 다음 단계를 따라 이러한 키와 토큰을 생성하세요.

* [개발자 계정에 가입](https://developer.x.com/en/apply-for-access)하고 승인을 받으세요.
* developer 포털에서 [프로젝트](/ko/resources/fundamentals/projects)를 생성하고 연결된 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 만드세요.
* 앱의 “Keys and tokens” 페이지로 이동하여 필요한 자격 증명을 발급받으세요. 모든 자격 증명을 안전한 위치에 보관하세요.

<div id="steps-to-build-a-manage-retweets-request">
  #### 리트윗 관리 요청을 구성하는 단계
</div>

**1단계: 도구 또는 라이브러리 선택**

이 endpoint에 요청을 보내기 위해 사용할 수 있는 여러 가지 도구, 코드 예제, 라이브러리가 있지만, 여기에서는 과정을 단순화하기 위해 Postman 도구를 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 불러오려면 다음 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2를 Postman에 추가
</Button>

Postman에 X API v2 컬렉션을 불러온 후, “Retweets” 폴더로 이동하여 “Retweet a Post”를 선택하세요.
 

**2단계: 요청 인증**

X API에 정상적으로 요청을 보내려면, 권한이 있는지 확인해야 합니다. 이 endpoint에서는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용해 요청을 인증해야 합니다.

이 예시에서는 OAuth 1.0a User Context를 사용하겠습니다.

Postman에 키와 토큰, 특히 API Key, API Secret Key, OAuth 1.0a 사용자 액세스 토큰, OAuth 1.0a 사용자 액세스 토큰 시크릿을 추가해야 합니다. Postman의 오른쪽 상단에서 “X API v2”라는 환경을 선택한 다음, 해당 환경 드롭다운 옆의 눈 아이콘을 클릭해 &quot;initial value&quot;와 &quot;current value&quot; 필드에 키와 토큰을 추가하면 됩니다.

올바르게 설정했다면, 이 변수들은 요청의 authorization 탭에 자동으로 채워집니다.

**3단계: 리트윗할 게시물 지정**

리트윗 관리 endpoint는 두 개의 ID를 필요로 합니다. 하나는 리트윗 또는 리트윗 취소를 하려는 사용자의 ID이고, 다른 하나는 게시물의 id입니다.

사용자의 ID는 인증에 사용되는 사용자 ID와 일치해야 합니다. 즉, 요청을 인증할 때 해당 사용자 ID와 연결된 [Access Tokens](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)를 전달해야 합니다. 이 경우, 자신의 사용자 ID를 지정하면 됩니다. ID는 다음 두 가지 방법으로 찾을 수 있습니다:

1. [user lookup by username](/ko/x-api/users/user-lookup-by-username) endpoint를 사용해 사용자 이름을 전달하면 id 필드를 받을 수 있습니다. 
2. 액세스 토큰을 보면, 숫자 부분이 자신의 사용자 ID입니다.
    

또한 리트윗하려는 게시물도 지정해야 합니다. 게시물 ID는 X.com에서 게시물을 클릭한 후 URL을 확인하여 찾을 수 있습니다. 예를 들어, 다음 URL의 게시물 ID는 1228393702244134912입니다.

[https://x.com/TwitterDev/status/1228393702244134912](https://x.com/TwitterDev/status/1228393702244134912)

Postman에서 &quot;Params&quot; 탭으로 이동하여, id path 변수의 &quot;Value&quot; 열에 자신의 ID를 입력하고, 리트윗하려는 게시물의 ID를 tweet&#95;id 파라미터 값으로 입력합니다. 어떤 ID든 앞뒤에 공백이 포함되지 않도록 주의하세요.

|              |                 |
| :----------- | :-------------- |
| **Key**      | **Value**       |
| `id`         | (your user ID)  |
| tweet&#95;id | (리트윗하려는 트윗의 ID) |

&quot;Send&quot; 버튼을 클릭하면, 관계 상태를 포함하는 response 객체를 받게 됩니다:

* &quot;retweeted&quot;: true를 받으면, 해당 id가 tweet&#95;id를 성공적으로 리트윗한 것입니다.
   

**4단계: 요청 보내기 및 응답 검토**

모든 설정이 완료되면 &quot;Send&quot; 버튼을 눌러 요청을 보내고, 다음과 같은 응답을 받게 됩니다:

```
{
    "data": {
        "retweeted": true
    }
}
```

`"retweeted": true` 값을 받았다면 해당 `tweet_id`가 성공적으로 리트윗된 것입니다.

동일한 사용자의 리트윗을 취소하려면, Postman에 로드된 X API v2 컬렉션의 “Retweets” 폴더에 있는 “Undo a Retweet” 요청을 사용할 수 있습니다. `id`는 자신의 사용자 ID여야 하고, `source_tweet_id`는 리트윗을 취소하려는 게시물 ID여야 합니다. 이를 JSON body로 추가할 필요는 없으므로, `id`와 `source_tweet_id`에 필요한 쿼리 매개변수를 반드시 추가해야 합니다. 

리트윗을 성공적으로 취소하면, 다음 예와 유사한 응답을 받게 됩니다:

```
{
  "data": {
    "retweeted": false
  }
}
```
