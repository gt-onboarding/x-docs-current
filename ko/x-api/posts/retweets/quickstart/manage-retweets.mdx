<div id="getting-started-with-the-manage-retweets-endpoints">
  ### 리트윗 관리 엔드포인트 사용 시작하기
</div>

이 빠른 시작 가이드는 Postman을 사용하여 리트윗 관리 엔드포인트에 첫 번째 요청을 보내는 방법을 안내합니다.

다른 언어로 작성된 샘플 코드를 보고 싶다면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 저장소를 방문하세요. 

<div id="prerequisites">
  #### 사전 준비 사항
</div>

이 가이드를 완료하려면 요청을 인증하기 위한 [키와 토큰](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따라 이러한 키와 토큰을 생성할 수 있습니다:

* [개발자 계정에 가입](https://developer.x.com/en/apply-for-access)하여 승인을 받습니다.
* developer portal에서 [프로젝트](/ko/resources/fundamentals/projects)와 연결된 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성합니다.
* 앱의 &quot;Keys and tokens&quot; 페이지로 이동해 필요한 자격 증명을 생성합니다. 모든 자격 증명은 안전한 위치에 보관해야 합니다.

<div id="steps-to-build-a-manage-retweets-request">
  #### Steps to build a manage Retweets request
</div>

**1단계: 도구 또는 라이브러리 선택하기**

이 엔드포인트에 요청을 보내는 데 사용할 수 있는 다양한 도구, 코드 예제, 라이브러리가 있지만, 여기서는 과정을 단순화하기 위해 Postman 도구를 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 로드하려면, 아래 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2를 Postman에 추가
</Button>

Postman에 X API v2 컬렉션을 로드한 후, “Retweets” 폴더로 이동하여 “Retweet a Post”를 선택합니다.
 

2단계: 요청 인증하기

X API에 올바르게 요청하려면, 권한이 있는지 확인해야 합니다. 이 엔드포인트에서 이를 수행하려면, [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용하여 요청을 인증해야 합니다.

이 예제에서는 OAuth 1.0a User Context를 사용하겠습니다.

Postman에 키와 토큰, 즉 API Key, API Secret Key, OAuth 1.0a 사용자 액세스 토큰(Access Token), OAuth 1.0a 사용자 액세스 토큰 시크릿(Access Token Secret)을 추가해야 합니다. 이를 위해 Postman 우측 상단에서 “X API v2”라는 이름의 환경(environment)을 선택한 뒤, 환경 드롭다운 옆의 눈 모양 아이콘을 클릭하여 &quot;initial value&quot;와 &quot;current value&quot; 필드에 키와 토큰을 추가합니다.

올바르게 설정했다면, 이 변수들은 요청의 Authorization 탭에 자동으로 반영됩니다.

**3단계: 리트윗할 게시물 지정하기**

Manage Retweets 엔드포인트에는 두 개의 ID가 필요합니다. 리트윗하거나 리트윗을 취소하려는 사용자의 ID와 게시물의 id입니다.

사용자의 ID는 인증하는 사용자의 ID와 일치해야 합니다. 즉, 요청을 인증할 때 해당 사용자 ID와 연결된 [Access Tokens](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)를 전달해야 합니다. 이 경우, 자신의 사용자 ID를 지정할 수 있습니다. ID를 확인하는 방법은 두 가지가 있습니다.

1. [user lookup by username](/ko/x-api/users/user-lookup-by-username) 엔드포인트를 사용하여 username을 전달하면 id 필드를 받을 수 있습니다. 
2. Access Token을 보면, 숫자 부분이 자신의 사용자 ID입니다.
    

또한 리트윗하려는 게시물도 지정해야 합니다. X.com에서 게시물을 클릭해 URL을 보면 Post ID를 확인할 수 있습니다. 예를 들어, 다음 URL의 게시물 ID는 1228393702244134912입니다.

[https://x.com/TwitterDev/status/1228393702244134912](https://x.com/TwitterDev/status/1228393702244134912)

Postman에서 &quot;Params&quot; 탭으로 이동한 다음, id 경로 변수의 &quot;Value&quot; 열에 자신의 ID를 입력하고, 리트윗할 게시물의 ID를 tweet&#95;id 파라미터 값으로 입력합니다. 어떤 ID든 앞뒤에 공백이 포함되지 않도록 주의하세요.

|              |                 |
| :----------- | :-------------- |
| **Key**      | **Value**       |
| `id`         | (your user ID)  |
| tweet&#95;id | (리트윗하려는 트윗의 ID) |

&quot;Send&quot; 버튼을 클릭하면 관계 상태를 나타내는 응답 객체를 받게 됩니다.

* &quot;retweeted&quot;: true를 받았다면, 해당 id가 tweet&#95;id를 성공적으로 리트윗한 것입니다.
   

**4단계: 요청 보내기 및 응답 검토하기**

모든 설정이 완료되면 &quot;Send&quot; 버튼을 클릭하여 다음과 같은 응답을 받게 됩니다:

```
{
    "data": {
        "retweeted": true
    }
}
```

`"retweeted": true` 응답을 받았다면, 해당 `tweet_id` 트윗이 성공적으로 리트윗된 것입니다.

동일한 사용자의 리트윗을 취소하려면, Postman에 로드된 X API v2 컬렉션의 “Retweets” 폴더에 있는 “Undo a Retweet” 요청을 사용할 수 있습니다. `id`는 자신의 사용자 ID여야 하며, `source_tweet_id`는 리트윗을 취소하려는 게시물의 ID여야 합니다. 이 값을 JSON 본문으로 추가할 필요는 없으므로, `id`와 `source_tweet_id`에 필요한 쿼리 파라미터를 반드시 추가해야 합니다. 

리트윗을 성공적으로 취소하면, 아래 예시와 유사한 응답을 받게 됩니다:

```
{
  "data": {
    "retweeted": false
  }
}
```
