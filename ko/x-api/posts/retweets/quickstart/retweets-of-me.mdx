Reposts of Me 조회 기능을 사용하면 다른 사용자가 공유한 자신의 게시물이 무엇인지 확인할 수 있습니다.

이를 통해 사용자는 어떤 게시물이 공유되고 자신의 오디언스에게 반향을 얻고 있는지 파악할 수 있으며, 보다 타기팅된 상호작용과 콘텐츠 전략 조정에 도움을 받을 수 있습니다.

<div id="authentication">
  #### 인증
</div>

사용자를 대신해 요청을 보내는 경우, 반드시 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 중 하나로 해당 엔드포인트에 인증을 수행하고, 요청을 수행하는 대상 사용자의 액세스 토큰을 사용해야 합니다. 이 사용자 액세스 토큰은 [3-legged OAuth 플로우](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens) (OAuth 1.0a) 또는 [Authorization Code with PKCE grant flow](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) (OAuth 2.0)를 사용해 발급할 수 있습니다.

<div id="making-a-request">
  #### 요청 보내기
</div>

사용자가 앱에서 인증을 완료하면, 다음과 같이 해당 사용자를 대신해 Retweets of Me API를 호출할 수 있습니다.

```bash
      curl 'https://api.x.com/2/users/reposts_of_me' --header 'Authorization: ••••••'
```
