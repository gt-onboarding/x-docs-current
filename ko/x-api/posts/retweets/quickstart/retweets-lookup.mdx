<div id="getting-started-with-the-retweets-lookup-endpoint">
  ### Retweets 조회 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 [Postman](/ko/tutorials/postman-getting-started)을 사용하여 Retweets 조회 엔드포인트에 첫 번째 요청을 보내는 방법을 안내합니다. 다양한 언어의 샘플 코드를 보고 싶다면 GitHub의 [X API v2 sample code](https://github.com/xdevplatformTwitter-API-v2-sample-code) 저장소를 방문하세요.

<div id="prerequisites">
  #### 사전 준비 사항
</div>

이 가이드를 완료하려면 요청을 인증하기 위한 [keys and tokens](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따라 필요한 키와 토큰을 생성하세요.

* [개발자 계정에 가입](https://developer.x.com/en/apply-for-access)하고 승인을 받으세요.
* developer portal에서 [프로젝트](/ko/resources/fundamentals/projects)와 연결된 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성하세요.
* 앱의 “Keys and tokens” 페이지로 이동하여 필요한 자격 증명을 생성하세요. 모든 자격 증명은 안전한 위치에 보관하세요.

<div id="steps-to-build-a-retweets-lookup-request">
  #### 리트윗 조회 요청 구성 단계
</div>

**1단계: 도구 또는 라이브러리 선택**

이 엔드포인트에 요청을 보내는 데 사용할 수 있는 여러 도구, 코드 예제, 라이브러리가 있지만, 여기서는 과정을 간소화하기 위해 Postman 도구를 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 로드하려면 다음 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Postman에 X API v2를 추가하기
</Button>

Postman에 X API v2 컬렉션을 로드한 후 &quot;Retweets&quot; 폴더로 이동하여 &quot;Retweeted by&quot;를 선택합니다.

**2단계: 요청 인증**

X API에 요청을 수행하려면 권한이 있는지 확인해야 합니다. 이 엔드포인트를 사용하려면 [OAuth 2.0 앱 전용(App-Only)](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), 또는 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication) 인증 방법 중 하나로 요청을 인증해야 합니다.

간단하게 설명하기 위해 이 요청에는 OAuth 2.0 앱 전용(App-Only)을 사용하겠지만, 비공개 [메트릭](/ko/x-api/fundamentals/metrics)이나 리트윗을 요청하려면 다른 인증 방법 중 하나를 사용해야 합니다.

OAuth 2.0 앱 전용(App-Only)을 사용하려면 키와 토큰, 특히 [앱 액세스 토큰](/ko/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)(앱 전용 Bearer Token이라고도 함)을 Postman에 추가해야 합니다. Postman 오른쪽 상단에서 &quot;X API v2&quot; 환경을 선택한 다음, 환경 드롭다운 옆의 눈 아이콘을 클릭하여 &quot;initial value&quot; 및 &quot;current value&quot; 필드에 키와 토큰을 추가하면 됩니다.

올바르게 설정한 경우 이러한 변수가 요청의 인증 탭으로 자동으로 가져와집니다.

**3단계: 게시물 지정하기**

이 엔드포인트를 사용할 때는 리트윗한 사용자를 조회할 게시물 ID를 지정해야 합니다. 게시물 ID는 X에서 해당 게시물로 이동한 후 URL 끝에 있는 숫자 코드를 확인하면 찾을 수 있습니다. 예를 들어 다음 URL의 게시물 ID는 1354143047324299264입니다.

[https://x.com/TwitterDev/status/1354143047324299264](https://x.com/TwitterDev/status/1354143047324299264)

Postman에서 &quot;Params&quot; 탭으로 이동한 후, id 경로 변수의 &quot;Value&quot; 열(섹션 하단)에 해당 사용자 이름을 입력합니다. 이때 사용자 이름 앞뒤에 공백이 없도록 주의하세요.

|       |                        |
| :---- | :--------------------- |
| **키** | **값**                  |
| `id`  | 리트윗한 사용자를 조회하려는 게시물 ID |

**4단계: 조회하려는 필드 식별 및 지정**

3단계를 완료한 후 &quot;Send&quot; 버튼을 클릭하면 응답에서 기본 [사용자 객체](/ko/x-api/fundamentals/data-dictionary#user) 필드(id, name, username)를 받게 됩니다.

id, name, username 이외의 추가 필드를 받으려면 요청 시 [fields](/ko/x-api/fundamentals/fields) 및/또는 [expansions](/ko/x-api/fundamentals/expansions) 매개변수로 해당 필드를 지정해야 합니다.

이 실습에서는 서로 다른 객체의 필드 세 가지 추가 세트를 요청합니다:

1. 주요 user 객체에 추가된 user.created&#95;at 필드입니다.
2. 반환된 사용자에 대해 연관된 상단 고정 게시물 객체의 기본 필드는 id와 text입니다.
3. 연결된 게시물 객체에 포함된 추가 tweet.created&#95;at 필드.

Postman에서 &quot;Params&quot; 탭으로 이동한 후 &quot;Query Params&quot; 테이블에 다음 키:값 쌍을 추가하세요:

|              |                         |                      |
| :----------- | :---------------------- | :------------------- |
| **키**        | **값**                   | **반환되는 필드**          |
| user.fields  | created&#95;at          | user.created&#95;at  |
| expansions   | pinned&#95;tweet&#95;id | tweet.id, tweet.text |
| tweet.fields | created&#95;at          | tweet.created&#95;at |

이제 &quot;Send&quot; 버튼 옆에 다음 URL이 표시되어야 합니다:

`https://api.x.com/2/tweets/1354143047324299264/retweeted_by?user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=created_at`

**5단계: 요청을 전송하고 응답을 검토합니다**
