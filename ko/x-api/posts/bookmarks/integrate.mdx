  ### 유용한 도구

이 엔드포인트를 통합하는 데 도움이 될 몇 가지 핵심 개념을 살펴보기 전에 먼저 다음 도구에 익숙해지시기를 권장합니다:

<div id="postman">
  #### Postman
</div>

Postman은 엔드포인트를 시험해 볼 때 사용할 수 있는 매우 유용한 도구입니다. 각 Postman 요청에는 어떤 기능을 사용할 수 있는지 빠르게 파악하는 데 도움이 되도록 모든 경로(path) 및 본문(body) 매개변수가 포함되어 있습니다. Postman 컬렉션에 대해 더 알아보려면 [&quot;Postman 사용하기&quot;](/ko/tutorials/postman-getting-started) 페이지를 방문하세요.

<div id="authentication">
  #### 인증
</div>

모든 X API v2 엔드포인트에 요청을 보내려면, 키와 토큰이라고도 부르는 자격 증명 세트로 요청을 인증해야 합니다. 

이 엔드포인트들은 [OAuth 2.0 Authorization Code Flow with PKCE](/ko/resources/fundamentals/authentication/oauth-2-0/authorization-code)를 사용해야 하며, 이는 요청을 성공적으로 보내기 위해 키 세트와 사용자 액세스 토큰을 함께 사용해야 함을 의미합니다. 액세스 토큰은 대신 요청을 보내려는 사용자와 연결되어 있어야 합니다. 다른 사용자를 위한 액세스 토큰 세트를 생성하려면, 해당 사용자가 [Authorize URL](/ko/resources/fundamentals/authentication#authorize-url)을 사용하여 귀하의 앱을 승인해야 합니다.

OAuth 2.0은 사용하기 까다로울 수 있습니다. 이 인증 방식에 익숙하지 않다면, [라이브러리](/ko/x-api/tools-and-libraries/overview)나 Postman과 같은 도구를 사용해 요청을 올바르게 인증할 것을 권장합니다.

<div id="developer-portal-projects-and-developer-apps">
  #### 개발자 포털, 프로젝트, 그리고 개발자 앱
</div>

X API v2 엔드포인트와 함께 사용할 수 있는 인증용 자격 증명 세트를 발급받으려면, 먼저 [개발자 계정](/ko/resources/fundamentals/developer-portal)을 보유하고 그 계정 내에 [프로젝트](/ko/resources/fundamentals/projects)를 설정한 뒤, 해당 프로젝트 안에 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성해야 합니다. 그런 다음 개발자 앱에서 키와 토큰을 확인할 수 있습니다. 

<div id="rate-limits">
  #### Rate limits
</div>

매일 수천 명이 넘는 개발자들이 X API에 요청을 보냅니다. 이러한 요청의 방대한 양을 관리하기 위해, [rate limits](/ko/resources/fundamentals/rate-limits)가 각 엔드포인트에 설정되어 있어 앱 또는 인증된 사용자를 대신해 보낼 수 있는 요청 수를 제한합니다. 

이 엔드포인트들은 사용자 수준에서 요청 한도가 적용되며, 이는 여러분이 요청을 보내는 대상이 되는 인증된 사용자가 어떤 개발자 앱을 사용하든 해당 엔드포인트를 호출할 수 있는 횟수가 제한된다는 의미입니다. GET 메서드의 경우, 15분 시간 창마다 사용자 요청 한도는 180건입니다. Bookmarks lookup 엔드포인트의 GET 메서드로는 가장 최근에 북마크한 게시물 800개가 반환됩니다. 또한 POST 및 DELETE 메서드의 경우 15분마다 사용자 요청 한도가 50건입니다.
