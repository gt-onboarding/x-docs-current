<div id="postman">
  #### Postman
</div>

Postman은 엔드포인트를 시험해 볼 때 사용할 수 있는 매우 유용한 도구입니다. 각 Postman 요청에는 모든 path 및 body 파라미터가 포함되어 있어, 어떤 기능을 사용할 수 있는지 빠르게 파악하는 데 도움이 됩니다. Postman 컬렉션에 대해 더 알아보려면 [&quot;Postman 사용하기&quot;](/ko/tutorials/postman-getting-started) 페이지를 방문하세요.

<div id="authentication">
  #### 인증
</div>

모든 X API v2 엔드포인트에 요청을 보내려면, 키와 토큰이라고도 불리는 자격 증명 세트를 사용해 요청을 인증해야 합니다. 

이 엔드포인트들은 [OAuth 2.0 Authorization Code Flow with PKCE](/ko/resources/fundamentals/authentication/oauth-2-0/authorization-code)를 사용해야 하며, 이는 성공적인 요청을 위해 키와 사용자 액세스 토큰 세트를 사용해야 함을 의미합니다. 액세스 토큰은 요청을 보내려는 대상 사용자와 연결되어 있어야 합니다. 다른 사용자를 위한 액세스 토큰 세트를 생성하려면, 그 사용자가 [Authorize URL](/ko/resources/fundamentals/authentication#authorize-url)을 사용하여 귀하의 앱을 승인하거나 인증해야 합니다.

OAuth 2.0은 사용하기 까다로울 수 있습니다. 이 인증 방식에 익숙하지 않다면, [라이브러리](/ko/x-api/tools-and-libraries/overview)나 Postman 같은 도구를 사용하여 요청을 올바르게 인증할 것을 권장합니다.

<div id="developer-portal-projects-and-developer-apps">
  #### developer portal, 프로젝트, 그리고 개발자 앱
</div>

X API v2 엔드포인트에서 사용할 수 있는 인증 자격 증명 세트를 발급받으려면 [개발자 계정](/ko/resources/fundamentals/developer-portal)을 보유하고, 해당 계정 내에 [프로젝트](/ko/resources/fundamentals/projects)를 생성한 뒤, 그 프로젝트 내에 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 만들어야 합니다. 그런 다음 해당 개발자 앱에서 키와 토큰을 확인할 수 있습니다. 

<div id="rate-limits">
  #### 요청 한도
</div>

매일 수만 명의 개발자가 X API에 요청을 보냅니다. 이러한 요청의 막대한 양을 관리하기 위해 각 엔드포인트에는 [요청 한도](/ko/resources/fundamentals/rate-limits)가 설정되어 있으며, 이는 앱 또는 인증된 사용자를 대신해 보낼 수 있는 요청 수를 제한합니다. 

이 엔드포인트에는 사용자 단위의 요청 한도가 적용됩니다. 즉, 요청을 대신 보내는 인증된 사용자는 어떤 개발자 앱에서든 해당 엔드포인트를 호출할 수 있는 횟수가 일정하게 제한됩니다. GET 메서드의 경우 15분당 180회의 사용자 요청 한도가 있습니다. Bookmarks 조회 엔드포인트의 GET 메서드를 사용하면 가장 최근에 북마크한 게시물 최대 800개를 반환받게 됩니다. 추가로 POST 및 DELETE 메서드에는 15분당 50회의 사용자 요청 한도가 있습니다.
