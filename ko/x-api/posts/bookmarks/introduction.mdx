<div id="manage-bookmarks">
  ### 북마크 관리
</div>

북마크를 관리하기 위해 사용할 수 있는 메서드는 POST와 DELETE 두 가지입니다. POST 메서드는 북마크를 생성할 때 사용합니다. 마찬가지로 DELETE 메서드는 특정 북마크를 삭제할 때 사용합니다. 

POST와 DELETE 메서드에는 사용자당 15분에 50회 요청이라는 rate limit(요청 한도)이 있습니다.

manage Bookmarks 엔드포인트를 사용하여 사용자를 대신해 요청을 보내는 것이므로, OAuth 2.0을 사용해 사용자 액세스 토큰을 생성하여 인증해야 합니다. 이를 위해 [Authorization Code with PKCE grant flow](/ko/resources/fundamentals/authentication/oauth-2-0/user-access-token)를 사용할 수 있습니다. 이 엔드포인트를 사용하려면 `tweet.read`, `users.read`, `bookmark.write` 스코프를 포함해야 합니다.

<div id="bookmarks-lookup">
  ### 북마크 조회
</div>

Bookmarks 조회 엔드포인트에서는 사용할 수 있는 메서드가 하나 있으며, 바로 GET입니다. 이 메서드를 사용하면 본인 계정 또는 인증된 계정의 Bookmarks를 가져올 수 있습니다. 이 엔드포인트에서 대량의 결과를 페이지 단위로 탐색할 수 있도록 페이지네이션 토큰이 제공됩니다.

GET 메서드에는 사용자당 15분 동안 180회의 요청 제한이 있습니다. 이 엔드포인트를 사용하면 가장 최근에 북마크한 게시물 800개를 가져올 수 있습니다.

Bookmarks 조회 엔드포인트를 사용하여 사용자를 대신해 요청을 보내는 것이므로, OAuth 2.0을 사용해 사용자 액세스 토큰을 생성하여 인증해야 합니다. 이를 위해 [Authorization Code with PKCE grant flow](/ko/resources/fundamentals/authentication/oauth-2-0/user-access-token)를 사용할 수 있습니다. 이 엔드포인트를 사용하려면 `tweet.read`, `users.read`, `bookmark.read` 스코프를 전달해야 합니다.

**계정 설정**

이 엔드포인트에 액세스하려면 다음이 필요합니다.

* 승인된 [개발자 계정](https://developer.x.com/en/portal/petition/essential/basic-info)
* [Project](/ko/resources/fundamentals/projects) 내에 있는 [developer App](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용한 인증

X API v2 엔드포인트에 대한 액세스 권한을 얻는 방법에 대해 더 알아보려면 [시작하기 가이드](/ko/x-api/getting-started/getting-access)를 참고하세요.
