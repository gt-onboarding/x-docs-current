<div id="manage-bookmarks">
  ### 북마크 관리
</div>

북마크를 관리하기 위해 사용할 수 있는 메서드는 POST와 DELETE 두 가지입니다. POST 메서드를 사용하면 북마크를 생성할 수 있습니다. 마찬가지로 DELETE 메서드를 사용하면 특정 북마크를 삭제할 수 있습니다. 

POST 및 DELETE 메서드에는 사용자별로 15분당 50개 요청의 레이트 리밋이 적용됩니다.

Manage Bookmarks 엔드포인트를 사용해 사용자를 대신하여 요청을 보내게 되므로, OAuth 2.0을 사용해 사용자 액세스 토큰을 생성하여 인증해야 합니다. 이를 위해 [Authorization Code with PKCE grant flow](/ko/resources/fundamentals/authentication/oauth-2-0/user-access-token)를 사용할 수 있습니다. 이 엔드포인트를 사용하려면 `tweet.read`, `users.read`, `bookmark.write` 스코프를 포함해야 합니다.

<div id="bookmarks-lookup">
  ### 북마크 조회
</div>

Bookmarks lookup 엔드포인트에는 사용 가능한 메서드가 하나 있으며, `GET`입니다. 이 메서드를 사용하면 본인 또는 인증된 계정의 북마크를 조회할 수 있습니다. 이 엔드포인트에서는 대량의 결과를 페이지 단위로 조회할 수 있도록 페이지네이션 토큰이 제공됩니다.

`GET` 메서드에는 사용자당 15분당 180개의 요청 한도가 적용됩니다. 이 엔드포인트를 사용하면 가장 최근에 북마크한 게시물 최대 800개를 조회할 수 있습니다.

북마크 조회 엔드포인트를 사용해 사용자를 대신하여 요청을 보내는 경우, OAuth 2.0으로 사용자 액세스 토큰을 생성해 인증해야 합니다. 이를 위해 [Authorization Code with PKCE grant flow](/ko/resources/fundamentals/authentication/oauth-2-0/user-access-token)를 사용할 수 있습니다. 이 엔드포인트를 사용하려면 `tweet.read`, `users.read`, `bookmark.read` 스코프를 요청에 포함해야 합니다.

**계정 설정**

이 엔드포인트에 접근하려면 다음이 필요합니다.

* 승인된 [개발자 계정](https://developer.x.com/en/portal/petition/essential/basic-info)
* [프로젝트](/ko/resources/fundamentals/projects) 내에 있는 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용한 인증

X API v2 엔드포인트에 대한 접근 권한을 얻는 방법은 [시작하기 가이드](/ko/x-api/getting-started/getting-access)에서 자세히 알아보세요.
