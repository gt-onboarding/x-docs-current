<div id="getting-started-with-the-bookmarks-lookup-endpoint">
  ### Bookmarks 조회 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 [Postman](/ko/tutorials/postman-getting-started)을 사용해 Bookmarks 조회 엔드포인트에 첫 요청을 보내는 방법을 안내합니다.

다양한 언어로 작성된 샘플 코드를 보고 싶다면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요.

<div id="prerequisites">
  #### 사전 준비 사항
</div>

이 가이드를 완료하려면 요청을 인증하기 위한 [키와 토큰](/ko/resources/fundamentals/authentication) 일련이 필요합니다. 다음 단계를 따라 이러한 키와 토큰을 생성할 수 있습니다:

* [개발자 계정에 가입](https://developer.x.com/en/apply-for-access)하고 승인을 받으세요.
* Developer Portal에서 [프로젝트](/ko/resources/fundamentals/projects)와 연결된 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성하세요.
* App의 “Keys and tokens” 페이지로 이동하여 필요한 자격 증명을 생성하세요. 모든 자격 증명은 안전한 위치에 저장해야 합니다.

<div id="steps-to-build-a-bookmarks-lookup-request">
  #### Bookmarks 조회 요청을 구성하는 단계
</div>

**1단계: 도구 또는 라이브러리 선택**

이 엔드포인트에 요청을 보내기 위해 사용할 수 있는 도구, 코드 예제, 라이브러리가 여러 가지 있지만, 여기서는 과정을 단순화하기 위해 Postman 도구를 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 로드하려면 아래 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2를 Postman에 추가
</Button>

Postman에 X API v2 컬렉션을 로드한 후 “Bookmarks” 폴더로 이동하여 “Bookmarks lookup”을 선택합니다.

**2단계: 요청 인증**

이 엔드포인트에 대한 요청을 성공적으로 수행하려면 [OAuth 2.0 Authorization Code Flow with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3)를 사용해야 합니다. Postman 내에서 액세스 토큰을 생성할 수 있습니다. 

“Authorization” 탭으로 이동한 뒤 “OAuth 2.0”을 선택합니다.

이 탭에서 아래 단계를 따라 주세요:

1. 토큰 이름 지정

2. Grant Type을 Authorization Code (with PKCE)로 선택

3. 다음과 같이 파라미터를 업데이트:

   **Callback URL** - [https://www.example.com](https://www.example.com/)

   이 값은 Developer Portal의 인증 설정 페이지에서 설정한 callback URL과 일치해야 합니다.

   **Auth URL** - [https://x.com/i/oauth2/authorize](https://x.com/i/oauth2/authorize)

   **Access Token URL** - [https://api.x.com/2/oauth2/token](https://api.x.com/2/oauth2/token)

   **Client ID** - Developer Portal에서 OAuth 2.0 client ID를 복사해 붙여넣기

   **Client Secret** - Developer Portal에서 OAuth 2.0 client secret을 복사해 붙여넣기. 기밀 client 유형의 앱을 사용하는 경우에만 필요합니다.

4. scopes를 다음 값들로 업데이트: post.read users.read bookmark.read

5. state 필드를 “State”로 채우기

6. “Generate Token”이라고 표시된 곳을 클릭

7. 저장 아이콘을 눌러 폴더 변경 내용을 저장

X에 로그인되어 있지 않다는 메시지가 표시될 수 있습니다. 이 오류가 발생하면, Postman 안에서 요청을 대신 보내려는 X 계정으로 로그인해야 합니다.

**3단계: 사용자 지정**

이 엔드포인트에서는 응답에서 Bookmarks 정보를 받고자 하는 사용자 ID를 지정해야 합니다. 예를 들어, @XDevelopers의 사용자 ID는 2244994945입니다. Postman에서 “Params” 탭으로 이동한 후, 자신 또는 인증된 사용자의 ID를 id 파라미터 값으로 입력합니다.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 2244994945 |

**4단계: 요청 보내기 및 응답 검토**
