<div id="getting-started-with-the-manage-bookmarks-endpoints">
  ## Bookmarks 관리 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 [Postman](/ko/tutorials/postman-getting-started)을 사용하여 Bookmarks 관리 엔드포인트에 첫 번째 요청을 보내는 방법을 설명합니다.

다양한 언어로 된 샘플 코드를 보고 싶다면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요. 

<Note>
  **사전 준비 사항**

  이 가이드를 완료하려면 요청을 인증하는 데 사용할 [keys and tokens](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따르면 이러한 키와 토큰을 생성할 수 있습니다.

  * [개발자 계정에 등록](https://developer.x.com/en/apply-for-access)하고 승인을 받습니다.
  * 개발자 포털에서 [프로젝트](/ko/resources/fundamentals/projects)와 연결된 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성합니다.
  * 앱의 “Keys and tokens” 페이지로 이동하여 필요한 자격 증명을 생성합니다. 모든 자격 증명은 안전한 위치에 보관하세요.
</Note>

<div id="steps-to-build-a-manage-bookmarks-request">
  ### Manage Bookmarks 요청을 구성하는 단계
</div>

**1단계: 도구 또는 라이브러리 선택**

이 엔드포인트에 요청을 보내기 위해 사용할 수 있는 다양한 도구, 코드 예제, 라이브러리가 있지만, 여기서는 프로세스를 단순화하기 위해 Postman 도구를 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 불러오려면 아래 버튼을 클릭하세요.

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2를 Postman에 추가
</Button>

Postman에 X API v2 컬렉션을 불러온 후 “Bookmarks” 폴더로 이동하여 “Create a Bookmark”를 선택합니다.

**2단계: 요청 인증**

이 엔드포인트에 성공적으로 요청하려면 [OAuth 2.0 Authorization Code Flow with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3)를 사용해야 합니다. Postman 내에서 액세스 토큰을 생성할 수 있습니다. 

“Authorization” 탭으로 이동하여 “OAuth 2.0”을 선택합니다.

이 탭에서 다음 단계를 반드시 따르세요.

1. 토큰 이름을 지정합니다.

2. Grant Type을 Authorization Code (with PKCE)로 선택합니다.

3. 다음과 같이 파라미터를 업데이트합니다.

   **Callback URL** - [https://www.example.com](https://www.example.com/)

   이 URL은 Developer Portal의 인증 설정 페이지에서 설정한 callback URL과 일치해야 합니다.

   **Auth URL** - [https://x.com/i/oauth2/authorize](https://x.com/i/oauth2/authorize)

   **Access Token URL** - [https://api.x.com/2/oauth2/token](https://api.x.com/2/oauth2/token)

   **Client ID** - Developer Portal에서 OAuth 2.0 client ID를 복사해 붙여넣습니다.

   **Client Secret **- Developer Portal에서 OAuth 2.0 client ID를 복사해 붙여넣습니다. 기밀 client App 유형을 사용하는 경우에만 필요합니다.

4. scope를 다음 값으로 업데이트합니다: post.read users.read bookmark.write

5. state 필드를 “State”로 채웁니다.

6. “Generate Token”이라고 쓰인 곳을 클릭합니다.

7. 저장 아이콘을 눌러 폴더 변경 사항을 저장합니다.

X에 로그인되어 있지 않다는 메시지가 표시될 수 있습니다. 이 오류가 표시되면 Postman 내에서 대신 게시하려는 X 계정에 로그인해야 합니다.

**3단계: 사용자 지정**

이 엔드포인트를 사용할 때는 응답에서 팔로워 정보를 받고자 하는 사용자 ID를 지정해야 합니다. 예를 들어 @XDevelopers의 사용자 ID는 `2244994945`입니다.

Postman에서 “Params” 탭으로 이동하여 id 파라미터 값으로 본인의 ID 또는 인증된 사용자의 ID를 입력합니다.

|    |     |              |
| :- | :-- | :----------- |
|    | Key | Value        |
|    | id  | `2244994945` |

**4단계: 북마크할 게시물 지정**

“Body” 탭으로 이동하여 북마크에 저장하려는 게시물의 ID(Post ID)가 원하는 값으로 설정되어 있는지 확인합니다. JSON 페이로드는 아래 예시와 비슷한 형태여야 합니다.

```
      {"tweet_id": "1460323737035677698"}
```

**5단계: 요청을 보내고 응답을 검토합니다**

모든 설정이 완료되었다면 &quot;Send&quot; 버튼을 클릭해 아래 예시와 비슷한 응답을 확인해 보세요:

```
{
   "data": {
       "bookmarked": true
   }
}
```

게시물을 삭제하려면 Postman에 로드된 X API v2 컬렉션의 “Bookmarks” 폴더에 있는 “Remove a Bookmark” 요청을 선택합니다. 먼저, 요청을 보내려는 사용자의 사용자 ID를 “`id`” 열의 값으로 지정해야 합니다. 이 엔드포인트에는 또한 삭제하려는 게시물의 ID가 필요합니다. 그런 다음 “Params” 탭에서 삭제하려는 게시물의 ID를 “`tweet_id`” 열의 값으로 입력합니다. 

삭제 요청이 성공하면 다음 예시와 유사한 응답을 받게 됩니다.

```
{
   "data": {
       "bookmarked": false
   }
}
```
