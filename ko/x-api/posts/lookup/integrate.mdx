v2 엔드포인트와 호환되는 커뮤니티 제작 [서드파티 라이브러리](/ko/resources/tools-and-libraries)를 활용하세요.

<div id="authentication">
  ### 인증
</div>

모든 X API v2 엔드포인트는 인증된 요청이 필요합니다. 다음 중 하나를 사용하여 인증할 수 있습니다:

* API 키, 액세스 토큰, 추가 매개변수를 사용해 [인증 헤더를 생성](/ko/resources/fundamentals/authentication#authorizing-a-request)하는 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication)
* 요청에 [앱 액세스 토큰](/ko/resources/fundamentals/authentication)을 함께 전송하는 [OAuth 2.0 앱 전용(App-Only)](/ko/resources/fundamentals/authentication#oauth-2-0)
* 앱 범위와 여러 기기 인증을 보다 세밀하게 제어하기 위한 [OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication)

<Note>
  OAuth 1.0a는 구현이 까다로울 수 있습니다. 익숙하지 않다면, 요청 시 [라이브러리](/ko/resources/tools-and-libraries)나 OAuth 2.0 사용을 고려하세요. 비공개 메트릭 또는 게시물을 요청하는 경우에는 OAuth 1.0a 사용자 컨텍스트 또는 OAuth 2.0 Authorization Code with PKCE를 사용해야 합니다.
</Note>

<Tip>
  **참고해 주세요**

  > 다음 필드를 요청하는 경우, OAuth 1.0a 사용자 컨텍스트 또는 OAuth 2.0 Authorization Code가 필요합니다:
  >
  > * `tweet.fields.non_public_metrics`
  > * `tweet.fields.promoted_metrics`
  > * `tweet.fields.organic_metrics`
  > * `media.fields.non_public_metrics`
  > * `media.fields.promoted_metrics`
  > * `media.fields.organic_metrics`
</Tip>

<div id="developer-portal-projects-and-apps">
  ### Developer portal, 프로젝트, 앱
</div>

X API v2용 자격 증명을 얻으려면 다음이 필요합니다:

1. 승인된 [개발자 계정](/ko/resources/fundamentals/developer-portal).
2. 해당 개발자 계정 내의 [프로젝트](/ko/resources/fundamentals/projects).
3. 키와 토큰을 확인할 수 있는 해당 프로젝트 내의 [개발자 앱](/ko/resources/fundamentals/developer-apps).

<div id="rate-limits">
  ### 요청 한도
</div>

X API 요청은 요청량을 관리하기 위해 [요청 한도](/ko/resources/fundamentals/rate-limits)가 적용됩니다. 한도는 앱 수준과 사용자 수준 모두에 적용됩니다:

* **앱 수준**: 어떤 앱이든 기간당 보낼 수 있는 요청 수를 제한합니다.
* **사용자 수준**: 인증된 사용자가 여러 개발자 앱에서 게시물 조회를 수행할 수 있는 빈도를 제한합니다.

<div id="fields-and-expansions">
  ### 필드와 expansions
</div>

X API v2를 사용하면 `fields` 및 `expansions`를 통해 특정 데이터 필드를 선택해 요청할 수 있습니다.

* **Expansions**: 추가로 관련된 객체를 조회할 수 있도록 합니다. 지원되는 expansions는 다음과 같습니다.
  * `edit_history_tweet_ids`
  * `attachments.poll_ids`
  * `attachments.media_keys`
  * `author_id`
  * `entities.mentions.username`
  * `geo.place_id`
  * `in_reply_to_user_id`
  * `referenced_tweets.id`
  * `referenced_tweets.id.author_id`

* **Fields**: 객체 내에서 반환할 데이터 필드를 지정해 추가 데이터를 요청합니다. Post 객체는 기본적으로 `id`, `text`, `edit_history_tweet_ids`를 포함합니다. `tweet.created_at` 및 `tweet.entities`와 같은 다른 옵션은 명시적으로 요청해야 합니다.

자세한 내용은 [X API v2 데이터 사전](/ko/x-api/fundamentals/data-dictionary)의 fields 및 expansions 가이드를 참고하세요.

<div id="post-edits">
  ### 게시물 편집
</div>

자격 요건을 충족하는 게시물은 게시 후 30분 이내에 최대 다섯 번까지 편집할 수 있습니다. Posts 조회 엔드포인트는 항상 게시물의 최신 버전을 제공합니다. 준 실시간(near-real-time) 사용 사례의 경우 이 시간 범위를 유의하세요. 자세한 내용은 [게시물 편집 기본 사항](/ko/x-api/fundamentals/edit-posts)을 참조하세요.

<div id="edge-cases">
  ### 엣지 케이스
</div>

* **프로모션 메트릭**: 프로모션되지 않은 게시물에 대해 프로모션 메트릭을 요청하면 빈 응답이 반환됩니다.
* **잘린 텍스트**: 리포스트의 경우 게시물 텍스트가 잘립니다. 전체 텍스트를 가져오려면 참조된 게시물을 확장하세요.
