게시물은 X에서 주요 리소스 중 하나입니다. 가장 단순한 형태로는 게시물에 최대 280자까지 작성할 수 있으며, 계정 설정에 따라 공개 또는 비공개로 게시할 수 있습니다. 또한 게시물에는 미디어, 장소, 설문조사, URL 등 다양한 객체를 첨부할 수 있습니다. 대부분의 게시물은 생성된 후 최대 30분 동안 수정할 수 있습니다.

게시물에 대한 HTTP, 선택, 전송 방식은 다양하지만, 이 REST 엔드포인트 그룹은 [Post ID](/ko/resources/fundamentals/x-ids)로 지정된 하나의 게시물 또는 게시물 그룹을 반환하는 데 초점을 맞춥니다. 이 엔드포인트들은 게시물에 대한 최신 세부 정보를 가져오고, 게시물이 사용 가능한지 확인하며, 수정 이력을 조회할 수 있습니다. 또한 [compliance events](/ko/x-api/enterprise-gnip-2.0/fundamentals/firehouse)를 관리하는 데에도 중요합니다.

Post 조회 엔드포인트는 수정된 게시물 메타데이터를 제공합니다. 2022년 9월 29일 이후에 생성된 모든 게시물에는 실제로 수정되지 않았더라도 수정 메타데이터가 포함됩니다. 각 수정은 새로운 Post ID를 생성하며, 게시물의 수정 이력은 원본으로 시작하는 Post ID 배열로 표현됩니다.

이 엔드포인트는 항상 가장 최근 수정본과 그 수정 이력을 함께 반환합니다. 30분 수정 가능 시간이 지난 후 수집된 게시물은 최종 버전을 나타냅니다. 게시물 수정 메타데이터에 대해 더 알아보려면 [Edit Posts fundamentals](/ko/x-api/fundamentals/edit-posts) 페이지를 확인하세요.

이 엔드포인트들은 GET HTTP 메서드를 사용하며, 게시물 텍스트, 생성 타임스탬프, 해시태그, 멘션, 사진의 목록 및 메타데이터와 같은 필드를 포함하는 하나 이상의 [Posts objects](/ko/x-api/fundamentals/data-dictionary)를 반환합니다.

  이 엔드포인트들에 액세스하려면 다음이 필요합니다.

  * [프로젝트](/ko/resources/fundamentals/projects) 내 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용한 인증

  X API v2 엔드포인트에 대한 액세스 권한을 얻는 방법은 [getting started guide](/ko/x-api/getting-started/getting-access)에서 자세히 알아볼 수 있습니다.
</Note>
