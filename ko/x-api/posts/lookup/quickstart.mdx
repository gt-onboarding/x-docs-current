<div id="getting-started-with-the-posts-lookup-endpoints">
  ## 게시물 조회 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 Postman을 사용해 지정한 필드 세트를 포함하여 게시물 조회 엔드포인트로 첫 요청을 보내는 방법을 안내합니다.

다른 언어로 된 샘플 코드를 보고 싶다면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요.

<Note>
  ### 사전 준비 사항

  이 가이드를 완료하려면 요청을 인증할 수 있는 [keys and tokens](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따라 이러한 키와 토큰을 생성할 수 있습니다:

  * [개발자 계정에 가입](https://developer.x.com/en/portal/petition/essential/basic-info)하고 승인을 받습니다.
  * developer portal에서 [Project](/ko/resources/fundamentals/projects)와 연결된 [developer App](/ko/resources/fundamentals/developer-apps)을 생성합니다.
  * 앱의 “Keys and tokens” 페이지로 이동하여 필요한 자격 증명을 생성합니다. 모든 자격 증명은 안전한 위치에 보관해야 합니다.
</Note>

<div id="steps-to-build-a-get-tweets-request">
  ### GET /tweets 요청을 만드는 단계
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### 1단계: 도구 또는 라이브러리로 시작하기
</div>

이 엔드포인트에 요청을 보내는 데 사용할 수 있는 다양한 도구, 코드 예제, 라이브러리가 있지만, 여기서는 과정을 단순화하기 위해 Postman 도구를 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 불러오려면 아래 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2를 Postman에 추가
</Button>

Postman에서 X API v2 컬렉션을 불러온 후, &quot;Post Lookup &gt; Multiple Posts&quot; 요청으로 이동하세요.

<div id="step-two-authenticate-your-request">
  #### 2단계: 요청을 인증하세요
</div>

X API에 정상적으로 요청을 보내려면 먼저 요청 권한이 있는지 확인해야 합니다. 다음 방법으로 요청을 인증할 수 있습니다:

* [OAuth 2.0 App-Only](resources/fundamentals/authentication)
* [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication)
* [OAuth 1.0a User Context](resources/fundamentals/authentication)

설명을 간단히 하기 위해 이 예제 요청에서는 OAuth 2.0 App-Only를 사용하겠습니다. 다만 비공개 [메트릭](/ko/x-api/fundamentals/metrics)이나 비공개 게시물을 요청하려면 다른 인증 방법 중 하나를 사용해야 합니다.

OAuth 2.0 App-Only를 사용하려면 Postman에서 오른쪽 상단에 있는 “X API v2”라는 이름의 환경을 선택한 다음, 환경 드롭다운 옆의 눈 모양 아이콘을 클릭해 &quot;initial value&quot; 및 &quot;current value&quot; 필드에 키와 토큰(특히 App Access Token, 즉 App-only Bearer Token)을 추가해야 합니다.

이 작업을 올바르게 완료했다면, 이 변수들이 요청의 Authorization 탭에 자동으로 채워집니다.

<div id="step-three-identify-and-specify-which-posts-you-would-like-to-retrieve">
  #### 3단계: 조회하려는 게시물을 식별하고 지정하기
</div>

요청에서 수신하려는 하나 이상의 게시물을 반드시 지정해야 합니다. 게시물의 ID는 X.com에서 해당 게시물을 클릭한 뒤 URL에서 확인할 수 있습니다. 예를 들어, 다음 URL의 게시물 ID는 `1228393702244134912`입니다.

`https://x.com/TwitterDev/status/1228393702244134912`

Postman에서 &quot;Params&quot; 탭으로 이동한 후, 이 ID 또는 쉼표로 구분된 여러 게시물 ID를 `ids` 파라미터의 &quot;Value&quot; 열에 입력합니다.

| Key  | Value |
| :---- | :----- |
| `ids` | `1228393702244134912,1227640996038684673,1199786642791452673` |

***

<div id="step-four-identify-and-specify-which-fields-you-would-like-to-retrieve">
  #### 4단계: 조회할 필드를 선택하고 지정하기
</div>

3단계를 마친 후 &quot;Send&quot; 버튼을 클릭하면, 응답에 기본 [Post object 필드](/ko/x-api/fundamentals/data-dictionary)인 `id`, `text`, `edit_history_tweet_ids`가 반환됩니다.

이 연습에서는 서로 다른 객체에서 세 가지 추가 필드 집합을 요청해 보겠습니다:

* 기본 게시물(Post) 객체에서 추가 `tweet.created_at` 필드
* 반환된 게시물의 작성자 user 객체에 대한 기본 필드: `id`, `name`, `username`
* 관련된 user 객체에서 추가 `user.created_at` 필드

Postman에서 &quot;Params&quot; 탭으로 이동한 뒤, &quot;Query Params&quot; 표에 다음 `key:value` 쌍을 추가합니다:

| Key            | Value        | Returned fields                                                       |
| :------------- | :----------- | :-------------------------------------------------------------------- |
| `tweet.fields` | `created_at` | `tweets.created_at`                                                   |
| `expansions`   | `author_id`  | `includes.users.id`, `includes.users.name`, `includes.users.username` |
| `user.fields`  | `created_at` | `includes.users.created_at`                                           |

&quot;Send&quot; 버튼 옆에 다음과 같은 URL이 표시되어야 합니다:

```
https://api.x.com/2/tweets?ids=1228393702244134912,1227640996038684673,1199786642791452673&tweet.fields=created_at&expansions=author_id&user.fields=created_at
```
