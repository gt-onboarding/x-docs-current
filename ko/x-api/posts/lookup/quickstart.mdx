<div id="getting-started-with-the-posts-lookup-endpoints">
  ## Posts 조회 엔드포인트 사용 시작하기
</div>

이 빠른 시작 가이드는 Postman을 사용해 지정한 필드 집합과 함께 Posts 조회 엔드포인트에 첫 요청을 보내는 방법을 안내합니다.

다양한 언어의 샘플 코드를 보고 싶다면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요.

<Note>
  ### 사전 준비 사항

  이 가이드를 완료하려면 요청을 인증하기 위한 [keys and tokens](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따르면 이러한 키와 토큰을 생성할 수 있습니다:

  * [개발자 계정](https://developer.x.com/en/portal/petition/essential/basic-info)에 가입하고 승인을 받습니다.
  * 개발자 포털에서 [Project](/ko/resources/fundamentals/projects)를 생성하고, 연결된 [developer App](/ko/resources/fundamentals/developer-apps)을 만듭니다.
  * App의 “Keys and tokens” 페이지로 이동하여 필요한 자격 증명을 생성합니다. 생성한 모든 자격 증명은 반드시 안전한 위치에 보관하세요.
</Note>

<div id="steps-to-build-a-get-tweets-request">
  ### GET /tweets 요청을 구성하는 단계
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### 1단계: 도구 또는 라이브러리 선택하기
</div>

이 엔드포인트에 요청을 보내기 위해 사용할 수 있는 도구, 코드 예제, 라이브러리가 여러 가지 있지만, 여기서는 과정을 단순화하기 위해 Postman 도구를 사용할 것입니다.

X API v2 Postman 컬렉션을 환경에 로드하려면 아래 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2를 Postman에 추가
</Button>

Postman에 X API v2 컬렉션을 로드한 후, &quot;Post Lookup &gt; Multiple Posts&quot; 요청으로 이동하세요.

<div id="step-two-authenticate-your-request">
  #### 2단계: 요청 인증하기
</div>

X API에 정상적으로 요청을 보내려면, 권한이 있는지 먼저 확인해야 합니다. 요청은 다음 방법으로 인증할 수 있습니다:

* [OAuth 2.0 App-Only](resources/fundamentals/authentication)
* [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication)
* [OAuth 1.0a User Context](resources/fundamentals/authentication)

간단히 하기 위해 이 예시 요청에서는 OAuth 2.0 App-Only를 사용할 것이지만, 비공개 [메트릭](/ko/x-api/fundamentals/metrics)이나 게시물을 요청하려면 다른 인증 방법 중 하나를 사용해야 합니다.

OAuth 2.0 App-Only를 사용하려면, Postman에서 오른쪽 상단에 있는 “X API v2” 환경을 선택한 뒤, 키와 토큰(특히 App 전용 Bearer Token이라고도 하는 App Access Token)을 &quot;initial value&quot; 및 &quot;current value&quot; 필드에 추가해야 합니다(환경 드롭다운 옆의 눈 모양 아이콘을 클릭하여 입력).

이 작업을 올바르게 완료했다면, 이 변수 값은 요청의 Authorization 탭에 자동으로 채워집니다.

<div id="step-three-identify-and-specify-which-posts-you-would-like-to-retrieve">
  #### 3단계: 조회하려는 게시물을 식별하고 지정하기
</div>

요청에 가져오려는 게시물 또는 게시물 집합을 지정해야 합니다. 게시물 ID는 X.com에서 해당 게시물을 클릭한 후 URL에서 확인할 수 있습니다. 예를 들어, 다음 URL의 게시물 ID는 `1228393702244134912`입니다.

`https://x.com/TwitterDev/status/1228393702244134912`

Postman에서 &quot;Params&quot; 탭으로 이동한 다음, 이 ID 또는 쉼표로 구분된 여러 게시물 ID 문자열을 ids 매개변수의 &quot;Value&quot; 열에 입력합니다.

| Key  | Value |
| :---- | :----- |
| `ids` | `1228393702244134912,1227640996038684673,1199786642791452673` |

***

<div id="step-four-identify-and-specify-which-fields-you-would-like-to-retrieve">
  #### 4단계: 조회하려는 필드를 식별하고 지정하기
</div>

3단계까지 완료한 뒤 &quot;Send&quot; 버튼을 클릭하면, 응답에 기본 [Post 객체 필드](/ko/x-api/fundamentals/data-dictionary)인 `id`, `text`, `edit_history_tweet_ids`가 포함됩니다.

이번 예제에서는 서로 다른 객체에서 세 가지의 추가 필드 세트를 요청해 보겠습니다.

* 기본 Post 객체에 추가로 `tweet.created_at` 필드
* 반환된 게시물에 대한 작성자(user) 객체의 기본 필드: `id`, `name`, `username`
* 연관된 user 객체에 추가로 `user.created_at` 필드

Postman에서 &quot;Params&quot; 탭으로 이동한 후 &quot;Query Params&quot; 표에 다음과 같은 `key:value` 쌍을 추가합니다:

| Key            | Value        | Returned fields                                                       |
| :------------- | :----------- | :-------------------------------------------------------------------- |
| `tweet.fields` | `created_at` | `tweets.created_at`                                                   |
| `expansions`   | `author_id`  | `includes.users.id`, `includes.users.name`, `includes.users.username` |
| `user.fields`  | `created_at` | `includes.users.created_at`                                           |

이제 &quot;Send&quot; 버튼 옆에 다음과 같은 URL이 표시되어야 합니다:

```
https://api.x.com/2/tweets?ids=1228393702244134912,1227640996038684673,1199786642791452673&tweet.fields=created_at&expansions=author_id&user.fields=created_at
```
