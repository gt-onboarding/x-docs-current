<div id="authentication">
  ### 인증
</div>

모든 X API v2 엔드포인트는 자격 증명 세트(키와 토큰)를 사용해 요청을 인증해야 합니다. 

이 엔드포인트들은 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication)를 사용해야 하며, 이는 요청을 성공적으로 보내기 위해 API 키와 사용자 액세스 토큰 세트를 사용해야 함을 의미합니다. 액세스 토큰은 요청을 대신 수행하려는 사용자와 연결되어 있어야 합니다. 다른 사용자를 위한 액세스 토큰 세트를 생성하려면, 해당 사용자가 [3-legged OAuth 플로우](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)를 사용하여 앱을 승인하거나 인증해야 합니다.

OAuth 1.0a는 사용하기 어려울 수 있습니다. 이 인증 방식을 잘 알지 못한다면, [라이브러리](/ko/x-api/tools-and-libraries/overview)를 사용하거나 Postman과 같은 도구를 사용하거나, OAuth 2.0을 사용하여 요청을 인증할 것을 권장합니다.

[OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)는 애플리케이션의 스코프와 여러 디바이스에 걸친 인가 플로우를 더 세밀하게 제어할 수 있게 해줍니다. OAuth 2.0을 사용하면 사용자 대신 수행할 수 있는 구체적인 권한을 부여하는 세밀한 스코프를 선택할 수 있습니다. 

앱에서 OAuth 2.0을 활성화하려면, developer portal의 App 설정 섹션에 있는 App 인증 설정에서 OAuth 2.0을 활성화해야 합니다.

  ### developer portal, 프로젝트, 및 개발자 앱

X API v2 엔드포인트와 함께 사용할 인증 자격 증명 세트를 얻으려면 [개발자 계정](/ko/resources/fundamentals/developer-portal)을 보유하고 있어야 하며, 해당 계정 내에 [프로젝트](/ko/resources/fundamentals/projects)를 설정하고 그 프로젝트 내에 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성해야 합니다. 그런 다음 개발자 앱에서 키와 토큰을 확인할 수 있습니다. 
 

<div id="rate-limits">
  ### 요청 한도
</div>

매일 수천 명이 넘는 개발자들이 X API에 요청을 보냅니다. 이러한 요청의 방대한 양을 관리하기 위해, 각 엔드포인트마다 [요청 한도](https://developer.x.com/x-api/fundamentals/rate-limits)가 설정되어 있으며, 이를 통해 앱 또는 인증된 사용자를 대신해 보낼 수 있는 요청 수가 제한됩니다.

이러한 엔드포인트에는 사용자 단위 요청 한도가 적용됩니다. 즉, 요청을 보낼 때 사용되는 인증된 사용자는 어떤 개발자 앱을 사용하더라도 해당 엔드포인트를 호출할 수 있는 횟수가 정해져 있습니다. POST 메서드에는 15분당 200개의 요청이라는 사용자 요청 한도가 있습니다. DELETE 메서드는 15분당 50개의 요청 한도가 있습니다. 추가로, manage Posts 또는 manage Retweets 권한을 통해 생성된 게시물을 포함해 [3시간당 300개의 요청](https://blog.x.com/developer/en_us/topics/tools/2018/new-developer-requirements-to-protect-our-platform)이라는 한도가 있습니다.
 

<div id="source-labels">
  ### 소스 라벨
</div>

앱 이름과 웹사이트 URL은 앱에서 프로그래밍 방식으로 생성한 모든 게시물의 메타데이터에 [소스 라벨](https://help.x.com/en/using-twitter/how-to-tweet#source-labels)로 표시됩니다. X 앱의 사용 사례가 변경되면, [Developer Terms](https://developer.x.com/content/developer-twitter/en/developer-terms/agreement-and-policy)을 준수하기 위해 이 설정에서 해당 사용 사례를 반드시 업데이트해야 합니다.
 

<div id="profile-settings">
  ### 프로필 설정
</div>

프로필 설정에서 위치 정보(geo)를 활성화한 경우에만 게시물에 위치를 추가할 수 있습니다. 위치 정보가 활성화되어 있지 않더라도 요청 본문에 위치 파라미터를 포함할 수는 있지만, 해당 위치는 게시물에 연결되지 않습니다. 이미지에 사용자를 태그하는 경우에도 동일하게 적용됩니다. 태그하려는 사용자가 사진 태그 기능을 켜 두지 않았다면, 게시물이 성공적으로 생성되더라도 태그된 사용자 목록에 해당 사용자의 이름이 표시되지 않습니다. 
 

<div id="adding-media-to-a-post">
  ### 게시물에 미디어 추가하기
</div>

현재는 X API v2만으로 미디어를 전부 업로드하는 방법은 없습니다. 그러나 이미 업로드된 미디어를 게시물에 첨부할 수는 있습니다. 이미 [v1 media 엔드포인트를 사용해 업로드한](https://developer.x.com/en/docs/twitter-api/v1/media/upload-media/api-reference/post-media-upload) 미디어이거나, [X Media Studio](https://media.x.com/en/articles/products/2018/media-studio)를 통해 업로드한 media id를 사용할 수 있습니다. 이러한 media id는 귀하 또는 인증된 사용자 계정의 media id여야 합니다.
