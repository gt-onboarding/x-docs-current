<div id="standard-v11-compared-to-x-api-v2">
  ## Standard v1.1과 X API v2 비교
</div>

기존에 Standard v1.1 [POST statuses/update](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-update) 및 [POST statuses/destroy/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-destroy-id) 엔드포인트를 사용해 오셨다면, 이 가이드의 목적은 Standard와 X API v2 게시물 관리 엔드포인트 간의 공통점과 차이점을 이해하는 데 도움을 드리는 것입니다.

* **유사점**
  * 인증
* **차이점**
  * 엔드포인트 URL

  * 앱 및 프로젝트 요구 사항

  * 요청 파라미터

<div id="similarities">
  ### 유사점
</div>

**인증**

표준 v1.1과 X API v2에서 게시물을 관리하는([POST statuses/update](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-update) 및 [POST statuses/destroy/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-destroy-id)) 엔드포인트는 모두 [OAuth 1.0a 사용자 컨텍스트](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication)를 사용합니다. 따라서 이전에 표준 v1.1 엔드포인트 중 하나를 사용하고 있었다면, X API v2로 마이그레이션하더라도 동일한 인증 방식을 계속 사용할 수 있습니다. 

<div id="differences">
  ### 차이점
</div>

**엔드포인트 URL**

* Standard v1.1 엔드포인트:
  * https://api.x.com/1.1/statuses/update.json
    (게시물 생성)
  * `https://api.x.com/1.1/statuses/destroy/:id.json`
    (게시물 삭제)
* X API v2 엔드포인트:
  * https://api.x.com/2/tweets
    (게시물 생성)
  * https://api.x.com/2/tweets/:id
    (지정된 게시물 삭제)

<div id="app-and-project-requirements">
  ### 앱 및 프로젝트 요구 사항
</div>

X API v2 엔드포인트를 사용하려면, 요청을 인증할 때 [developer App](/ko/resources/fundamentals/developer-apps)과 연결된 [Project](/ko/resources/fundamentals/developer-apps)에서 발급된 자격 증명을 사용해야 합니다. 모든 X API v1.1 엔드포인트는 프로젝트에 연결되지 않은 독립 실행형 앱이나 프로젝트와 연결된 앱에서 발급된 자격 증명을 모두 사용할 수 있습니다.

<div id="request-parameters">
  ### 요청 매개변수
</div>

다음의 표준 v1.1 엔드포인트는 두 가지 요청 쿼리 매개변수(`user_id` 또는 `screen_name`)를 허용했습니다. X API v2에서는 DELETE 엔드포인트에 대해 숫자형 게시물 ID만 허용하며, 이 값은 엔드포인트 경로의 일부로 전달되어야 합니다.

POST 엔드포인트의 경우, 추가 매개변수는 요청의 JSON 본문을 통해 전달해야 합니다. 사용 가능한 매개변수에 대해서는 [API reference guide](/ko/x-api/posts/manage-tweets/introduction)에서 더 자세히 확인할 수 있습니다.
