<div id="standard-v11-compared-to-x-api-v2">
  ## 표준 v1.1과 X API v2 비교
</div>

표준 v1.1 [POST statuses/update](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-update) 및 [POST statuses/destroy/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-destroy-id) 엔드포인트를 사용해 왔다면, 이 가이드는 표준 엔드포인트와 X API v2 게시물 관리용 엔드포인트 간의 공통점과 차이점을 이해하는 데 도움이 되도록 작성되었습니다.

* **유사점**
  * 인증
* **차이점**
  * 엔드포인트 URL

  * 앱 및 프로젝트 요구 사항

  * 요청 매개변수

<div id="similarities">
  ### 유사점
</div>

**인증**

표준 v1.1과 X API v2의 게시물([POST statuses/update](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-update) 및 [POST statuses/destroy/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-destroy-id)) 엔드포인트는 모두 [OAuth 1.0a 사용자 컨텍스트](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication)를 사용합니다. 따라서 이전에 표준 v1.1 엔드포인트 중 하나를 사용해 왔다면, X API v2 버전으로 마이그레이션하더라도 동일한 인증 방식을 계속 사용할 수 있습니다. 

<div id="differences">
  ### 차이점
</div>

**엔드포인트 URL**

* 표준 v1.1 엔드포인트:
  * https://api.x.com/1.1/statuses/update.json
    (게시물을 생성합니다)
  * `https://api.x.com/1.1/statuses/destroy/:id.json`
    (게시물을 삭제합니다)
* X API v2 엔드포인트:
  * https://api.x.com/2/tweets
    (게시물을 생성합니다)
  * https://api.x.com/2/tweets/:id
    (지정된 게시물을 삭제합니다)

<div id="app-and-project-requirements">
  ### 앱 및 프로젝트 요구 사항
</div>

X API v2 엔드포인트는 요청을 인증할 때 [프로젝트](/ko/resources/fundamentals/developer-apps)에 속한 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 자격 증명을 사용해야 합니다. 모든 X API v1.1 엔드포인트는 독립 실행형 앱 또는 프로젝트에 속한 앱의 자격 증명을 사용할 수 있습니다.

<div id="request-parameters">
  ### 요청 매개변수
</div>

다음 표준 v1.1 요청 매개변수는 두 개의 요청 쿼리 매개변수(`user_id` 또는 `screen_name`)를 허용했습니다. X API v2에서는 DELETE 엔드포인트의 경우 숫자형 게시물 ID만 허용하며, 엔드포인트 경로의 일부로 전달해야 합니다.

POST 엔드포인트의 경우 추가 매개변수는 JSON 요청 본문을 통해 전달해야 합니다. 어떤 매개변수를 사용할 수 있는지는 [API reference guide](/ko/x-api/posts/manage-tweets/introduction)에서 자세히 확인할 수 있습니다.
