<div id="comparing-x-apis-manage-posts-endpoints">
  ## X API의 게시물 관리 엔드포인트 비교
</div>

v2 게시물 관리용 엔드포인트는 기존 v1.1의 표준 엔드포인트인 [POST statuses/update](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-update) 및 [POST statuses/destroy/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-destroy-id) 엔드포인트를 대체하게 됩니다. v1.1 버전의 게시물 관리 엔드포인트를 사용하는 코드, 앱 또는 도구가 있고 이를 최신 X API v2 엔드포인트로 마이그레이션하는 것을 고려하고 있다면, 이 가이드 모음이 여러분을 위한 것입니다.

다음 표에서는 v1.1 표준 엔드포인트와 X API v2 게시물 관리 엔드포인트를 비교합니다.

<div id="create-a-post">
  ### 게시물 생성
</div>

| 설명 | 표준 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 지원되는 HTTP 메서드 | POST | POST |
| 호스트 도메인 | https://api.x.com | https://api.x.com |
| 엔드포인트 경로 | /1.1/statuses/update.json | /2/tweets |
| [인증](/ko/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code with PKCE |
| 기본 [요청 한도](/ko/x-api/fundamentals/rate-limits) | 없음<br /><br />앱당 사용자 1명 기준 3시간 동안 300회 요청. v1.1 POST Retweets 엔드포인트와 공유됩니다. | 사용자 1명당 15분 동안 200회 요청<br /><br />앱당 사용자 1명 기준 3시간 동안 300회 요청. v2 POST Retweets 엔드포인트와 공유됩니다. |
| [프로젝트](/ko/resources/fundamentals/projects)와 연결된 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 자격 증명 사용 필요 |     | ✔   |

<div id="delete-a-post">
  ### 게시물 삭제
</div>

| 설명 | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| 지원되는 HTTP 메서드 | POST | DELETE |
| 호스트 도메인 | https://api.x.com | https://api.x.com |
| 엔드포인트 경로 | /1.1/statuses/destroy/:id.json | /2/tweets/:id |
| [인증](/ko/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code with PKCE |
| 기본 [요청 한도](/ko/x-api/fundamentals/rate-limits) | 없음 | 사용자당 15분에 50회 요청 |
| [프로젝트](/ko/resources/fundamentals/projects)와 연결된 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 자격 증명을 사용해야 함 |     | ✔   |
