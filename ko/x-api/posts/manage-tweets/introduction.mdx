X API를 사용해 게시물을 생성하고 삭제하는 것은 공개 대화에 참여하는 데 필수적입니다. 새로 제공되는 manage Posts 엔드포인트를 사용하면 이러한 작업은 물론 그보다 훨씬 더 많은 작업을 수행할 수 있습니다.

게시물을 관리하는 데 사용할 수 있는 메서드는 POST와 DELETE 두 가지입니다. POST 메서드를 사용하면 투표 게시, 트윗 인용 게시, 답글 설정을 지정한 게시, 위치 정보(geo)가 포함된 게시, 미디어를 포함하고 사용자를 태그한 게시, Super Followers 대상 게시 등 다양한 기능을 사용할 수 있습니다. 마찬가지로 DELETE 메서드를 사용하면 특정 게시물을 삭제할 수 있습니다. POST 메서드의 경우, 요청을 더 세밀하게 사용자 정의할 수 있도록 원하는 [매개변수](/ko/x-api/posts/creation-of-a-post)를 전달할 수 있습니다.

&#39;delete Post&#39; 메서드는 수정된 게시물을 지원하도록 업데이트되었습니다. 편집 체인에 있는 게시물 중 하나가 삭제되면 해당 편집 체인에 속한 모든 게시물도 함께 삭제됩니다. Edit Post 메타데이터에 대해 더 알아보려면 [Edit Posts fundamentals](/ko/x-api/fundamentals/edit-posts) 페이지를 확인하세요.

POST 메서드에는 사용자 기준으로 15분당 200개의 요청 제한이 있습니다. DELETE 메서드에는 15분당 50개의 요청 제한이 있습니다. 추가로, manage Posts 또는 manage Retweets를 통해 생성된 게시물을 포함하여 3시간당 300개의 요청 제한이 있습니다.

manage Posts 엔드포인트를 사용해 사용자를 대신해 요청을 보내는 것이므로, [OAuth 1.0a User Context](https://en.docs/authentication/oauth-1-0a) 또는 [OAuth 2.0 Authorization Code with PKCE](https://en.docs/authentication/oauth-2-0/authorization-code) 중 하나로 인증해야 하며, 앱을 승인한 사용자와 연결된 사용자 액세스 토큰을 사용해야 합니다. OAuth 1.0a로 이 사용자 액세스 토큰을 생성하려면 [3-legged OAuth 플로우](https://en.docs/authentication/oauth-2-0/bearer-tokens)를 사용할 수 있습니다. OAuth 2.0으로 사용자 액세스 토큰을 생성하려면 [Authorization Code with PKCE grant flow](https://en.docs/authentication/oauth-2-0/user-access-token)를 사용할 수 있습니다.
