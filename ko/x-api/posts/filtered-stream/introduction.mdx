필터링 스트림 엔드포인트 그룹을 사용하면 개발자가 공개 게시물 스트림을 필터링할 수 있습니다. 이 엔드포인트 그룹에는 규칙을 생성·관리하고, 해당 규칙을 적용해 일치하는 공개 게시물만 반환하는 게시물 스트림을 필터링할 수 있게 해주는 여러 엔드포인트가 포함됩니다. 이를 통해 사용자는 특정 주제와 이벤트를 모니터링하고, 경쟁 상황과 관련된 대화를 추적하며, 트렌드가 어떻게 형성되는지 파악하는 등 다양한 작업을 수행할 수 있습니다.

개발자는 REST [rules 엔드포인트](/ko/x-api/posts/filtered-stream#post-2-tweets-search-stream-rules)를 사용해 연결을 끊지 않고도 지속적인 스트림 연결에 규칙을 추가하거나 제거할 수 있습니다. 이러한 [규칙](/ko/x-api/posts/filtered-stream#building-rules-for-filtered-stream)은 메시지 키워드, 해시태그, URL과 같은 게시물 속성과 일치하도록 연산자를 사용해 생성할 수 있습니다. 연산자와 규칙 절은 불리언 로직과 괄호를 조합해 사용하여 필터의 매칭 동작을 더 세밀하게 조정할 수 있습니다.

규칙 세트를 추가한 후에는 [스트리밍 연결을 설정](/ko/x-api/posts/filtered-stream#get-2-tweets-search-stream)하여, 지속적인 HTTP 스트리밍 연결을 통해 JSON 형식의 [Post 객체](/ko/x-api/fundamentals/data-dictionary#tweet)를 전달받을 수 있습니다. 스트림에 연결되어 있는 동안에는 규칙과 일치하는 콘텐츠만 수신하게 됩니다.

또한 Filtered Stream을 웹훅에 연결하여 웹훅 이벤트로 전달받을 수도 있습니다. Filtered Stream과 함께 웹훅을 사용하는 방법에 대한 자세한 내용은 [Filtered Stream Webhooks 소개](/ko/x-api/webhooks/stream/introduction)를 참고하세요.

필터링 검색 엔드포인트는 수정된 게시물을 지원합니다. 이 엔드포인트는 하나 이상의 필터와 일치하는 수정된 게시물과 함께, 게시물 id 배열을 포함한 수정 이력도 함께 전달합니다. 수정 이력이 없는 게시물의 경우 이 배열에는 단일 id만 포함됩니다. 수정된 게시물의 경우 이 배열에는 여러 개의 id가 포함되며, 수정된 순서를 반영해 오름차순으로 정렬되며, 배열의 마지막 요소가 가장 최근 버전입니다. 게시물 수정 동작에 대한 자세한 내용은 [게시물 수정 기본 개념](/ko/x-api/fundamentals/edit-posts) 페이지를 참고하세요.

필터링 스트림 엔드포인트의 일부 기능은 [액세스 레벨](/ko/x-api/getting-started/about-x-api)에 따라 제한됩니다:

**Pro 액세스**

* 프로젝트당 1000개의 규칙
* POST /2/tweets/search/stream/rules 엔드포인트를 사용해 규칙을 추가할 때 15분당 100개의 요청
* 규칙을 작성할 때 모든 연산자 사용 가능
* 최대 1024자 길이의 규칙 작성 가능

**Enterprise 액세스**

* 프로젝트당 25,000개 이상의 규칙
* 규칙을 작성할 때 모든 연산자 사용 가능
* 최대 2048자 길이의 규칙 작성 가능
* Enterprise 액세스를 받으려면 [여기](https://developer.x.com/en/products/x-api/enterprise/enterprise-api-interest-form)에서 신청하세요

필터링 스트림에서 반환되는 게시물은 월간 [게시물 상한](/ko/x-api/fundamentals/post-cap)에 포함됩니다.

<Note>
  **계정 설정**

  이 엔드포인트들에 액세스하려면 다음이 필요합니다.

  * 승인된 [개발자 계정](https://developer.x.com/en/portal/petition/essential/basic-info)
  * [프로젝트](/ko/resources/fundamentals/projects) 내에 위치한 [개발자 App](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용한 인증

  X API v2 엔드포인트에 대한 액세스 권한을 얻는 방법은 [시작하기 가이드](/ko/x-api/getting-started/getting-access)를 참고하세요.
</Note>
