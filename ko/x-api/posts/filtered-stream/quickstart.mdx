<div id="getting-started-with-the-filtered-stream-endpoints">
  ## filtered stream 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 cURL 요청을 사용하여 filtered stream 엔드포인트 그룹에 처음으로 요청을 보내는 방법을 설명합니다. cURL은 최소한의 설정만으로 요청을 보낼 수 있게 해주는 명령줄 도구입니다.

다른 언어의 샘플 코드를 보고 싶다면, [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요. 

<Note>
  **사전 준비 사항**

  이 가이드를 완료하려면 요청을 인증하기 위한 [키와 토큰](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따라 이러한 키와 토큰을 생성할 수 있습니다:

  * [개발자 계정에 가입](https://developer.x.com/en/apply-for-access)하여 승인을 받습니다.
  * 개발자 포털(developer.x.com)에서 [프로젝트](/ko/resources/fundamentals/projects)를 생성하고 연결된 [developer App](/ko/resources/fundamentals/developer-apps)을 만듭니다.
  * 앱의 &#39;Keys and tokens&#39; 페이지로 이동하여 필요한 자격 증명을 생성합니다. 모든 자격 증명은 안전한 위치에 저장해야 합니다.
</Note>

<div id="steps-to-build-a-filtered-stream-request-using-curl">
  ### cURL을 사용해 필터링 스트림 요청을 만드는 단계
</div>

**1단계: 규칙 만들기**

규칙은 스트림으로 전송할 게시물을 정의하기 위해 불리언 논리와 괄호를 사용해 결합된 하나 이상의 서로 다른 [연산자](/ko/x-api/posts/filtered-stream#building-rules-for-filtered-stream)로 구성됩니다. 이 가이드에서는 스트림을 필터링해 “cat”이라는 키워드와 이미지를 모두 포함하는 게시물을 찾습니다. 사용할 규칙은 다음과 같습니다.

```cat has:images
```

**2단계: 규칙에 태그 추가하기**

스트림에는 여러 개의 규칙을 동시에 추가할 수 있습니다. 스트리밍 연결을 열면, 이러한 규칙 중 하나라도 일치하는 게시물은 모두 동일한 스트리밍 연결을 통해 전송됩니다. 어떤 게시물이 어떤 규칙과 일치했는지 알 수 있도록, 규칙 생성 요청과 함께 태그를 전달할 수 있습니다. 해당 규칙과 일치하는 각 게시물은 이후 게시물 페이로드 내에 어떤 규칙과 일치했는지를 나타내는 `tag` 필드를 포함하게 됩니다. 이 규칙에는 다음과 같은 태그를 지정하겠습니다:

```cats with images
```

**3단계: 규칙을 스트림에 추가하기**

이 엔드포인트를 호출할 때는 규칙과 해당 태그가 포함된 application/json 본문을 함께 전달해야 합니다. 또한 이 규칙을 스트림에 추가하려고 하기 때문에, 규칙 값과 태그를 `add` 객체 안에 넣은 것을 볼 수 있습니다. 이 JSON 본문은 다음과 같은 형태입니다:

```json
{
  "add": [
    {
      "value": "cat has:images",
      "tag": "이미지가 포함된 고양이"
    }
  ]
}
```

이제 이 JSON 본문의 설정을 모두 마쳤으므로, 이를 cURL 요청에 추가할 수 있습니다. 이 요청은 다음과 같은 형태가 됩니다. 이 요청은 아직 완성된 것이 아니므로, 이후 단계까지 실행하지 말고 기다려 주세요.

```json
curl -X POST 'https://api.x.com/2/tweets/search/stream/rules' \
-H "Content-type: application/json" \
-H "Authorization: Bearer $APP_ACCESS_TOKEN" -d \
'{
  "add": [
    {"value": "cat has:images", "tag": "cats with images"}
  ]
}'`
```

**4단계: 요청 인증하기**

filtered stream 규칙 엔드포인트는 [OAuth 2.0 앱 전용(App-Only) 인증](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)이 필요하므로, 3단계에서 사용한 cURL 명령어의 $APP&#95;ACCESS&#95;TOKEN을 사전 준비 단계에서 생성한 [App 액세스 토큰](/ko/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)으로 교체해야 합니다. 

**5단계: 규칙을 스트림에 추가하기**

이제 cURL 요청을 실행하여 규칙을 스트림에 추가합니다. cURL 명령어를 복사해 명령줄 인터페이스에 붙여넣고 &quot;return&quot; 키를 누르십시오. 

cURL 요청이 성공하면 value, tag, 그리고 id(규칙의 고유 식별자 역할을 함)에 대한 데이터를 포함한 응답을 받게 됩니다. 이 응답은 다음과 비슷한 형태입니다:

```json
{
  "data": [
    {
      "value": "cat has:images",
      "tag": "이미지가 포함된 고양이",
      "id": "1273026480692322304"
    }
  ],
  "meta": {
    "sent": "2020-06-16T22:55:39.356Z",
    "summary": {
      "created": 1,
      "not_created": 0,
      "valid": 1,
      "invalid": 0
    }
  }
}
```

규칙이 성공적으로 추가되었는지 검증하려면, 다시 한 번 $APP&#95;ACCESS&#95;TOKEN을 자신의 토큰으로 바꾼 뒤, 다음 GET 요청을 rules 엔드포인트로 보내면 됩니다. 이 요청은 스트림에 추가된 모든 규칙의 전체 목록을 반환합니다.

```bash
curl -X GET 'https://api.x.com/2/tweets/search/stream/rules' \
-H "Authorization: Bearer $APP_ACCESS_TOKEN"
```

cURL 요청이 성공적으로 완료되었다면, 다음과 같은 응답을 받게 됩니다:

```json
{
	"data": [{
		"id": "1273028376882589696",
		"value": "cat has:images",
		"tag": "cats with images"
	}],
	"meta": {
		"sent": "2020-06-16T23:14:06.498Z"
	}
}
```

**6단계: 가져올 필드를 선택하고 지정하기**

5단계를 마친 후 스트림에 연결하면, 응답에서 기본 [Post object](/ko/x-api/fundamentals/data-dictionary#tweet) 필드인 id, text, edit&#95;history&#95;tweet&#95;ids를 받게 됩니다. 이들 외의 추가 필드를 받고 싶다면, 요청에 [field](/ko/x-api/fundamentals/fields) 및/또는 [expansion](/ko/x-api/fundamentals/expansions) 매개변수를 사용해 해당 필드를 명시해야 합니다.

이 연습에서는 서로 다른 객체에서 세 가지 서로 다른 필드 집합을 요청해 보겠습니다.

1. 기본 Post 객체에서 추가 tweet.created&#95;at 필드
2. 반환된 게시물에 대한 작성자의 [user object](/ko/x-api/fundamentals/data-dictionary#user) 기본 필드: id, name, username
3. 연관된 user 객체에서 추가 user.created&#95;at 필드
    

이 필드를 요청하려면, 요청에 다음 값을 전달해야 합니다.

|              |                |                                                                 |
| :----------- | :------------- | :-------------------------------------------------------------- |
| Key          | Value          | Returned fields                                                 |
| tweet.fields | created&#95;at | tweets.created&#95;at                                           |
| expansions   | author&#95;id  | includes.users.id, includes.users.name, includes.users.username |
| user.fields  | created&#95;at | includes.users.created&#95;at                                   |

이제 이것을 알았으니, 스트림에 연결하기 위한 요청 URL을 다음과 같이 구성할 수 있습니다.

```bash
https://api.x.com/2/tweets/search/stream?tweet.fields=created_at&expansions=author_id&user.fields=created_at
```

**7단계: 스트림에 연결하고 응답을 검토하기**

이제 규칙을 설정했고 반환받고자 하는 필드를 지정했으므로, 제출한 규칙과 일치하는 Post 객체를 전달받기 위해 스트림에 연결할 준비가 되었습니다. 다음은 6단계에서 만든 URL을 요청에 추가한 후의 cURL 명령 예시입니다:

```bash
curl -X GET -H "Authorization: Bearer $APP_ACCESS_TOKEN" "https://api.x.com/2/tweets/search/stream?tweet.fields=created_at&expansions=author_id&user.fields=created_at"
```

다시 한 번 말씀드리지만, 이 요청은 OAuth 2.0 앱 전용(App-Only)으로 인증해야 하므로, 명령줄 도구에서 실행하기 전에 $APP&#95;ACCESS&#95;TOKEN을 자신의 자격 증명으로 반드시 교체해야 합니다.

filtered stream에 연결되면, 설정한 규칙과 일치하는 게시물을 다음 JSON 형식으로 받게 됩니다:
