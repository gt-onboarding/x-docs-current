<div id="getting-started-with-the-quote-tweets-lookup-endpoint">
  ## 인용 트윗 조회 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 [Postman](/ko/tutorials/postman-getting-started)을 사용하여, 지정한 필드 집합과 함께 인용 트윗 조회 엔드포인트에 처음으로 요청을 보내는 방법을 안내합니다.

다양한 언어의 샘플 코드를 보고 싶다면, [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 저장소를 방문하세요. 

<div id="prerequisites">
  #### 사전 준비 사항
</div>

이 가이드를 완료하려면 요청을 인증할 수 있는 [keys and tokens](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따라 이러한 키와 토큰을 생성할 수 있습니다:

* [개발자 계정에 가입](https://developer.x.com/en/apply-for-access)하고 승인을 받습니다.
* developer 포털에서 [Project](/ko/resources/fundamentals/projects)와 연결된 [developer App](/ko/resources/fundamentals/developer-apps)을 생성합니다.
* App의 “Keys and tokens” 페이지로 이동해 필요한 자격 증명을 생성합니다. 모든 자격 증명은 안전한 위치에 저장해야 합니다.

**Quote Tweets 조회 요청을 작성하는 단계**

**1단계: 도구 또는 라이브러리 선택**

이 엔드포인트에 요청을 보내는 데 사용할 수 있는 다양한 도구, 코드 예제, 라이브러리가 있지만, 여기서는 과정을 단순화하기 위해 Postman 도구를 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 로드하려면 아래 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2를 Postman에 추가
</Button>

Postman에 X API v2 컬렉션을 로드한 후, timeline 폴더로 이동하여 &quot;Quote Tweets by ID&quot; 요청을 찾습니다.

**2단계: 요청 인증**

X API에 올바르게 요청하려면, 권한이 있는지 확인해야 합니다. 이 엔드포인트에서는 [OAuth 2.0 App-Only](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), 또는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 인증 방식을 사용해 요청을 인증해야 합니다.

단순화를 위해 이 예제에서는 OAuth 2.0 App-Only를 사용하지만, 비공개 [metrics](/ko/x-api/fundamentals/metrics)나 비공개 게시물을 요청하려면 다른 인증 방식 중 하나를 사용해야 합니다. 

Postman에 키와 토큰, 특히 [App Access Token](/ko/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)(App-only Bearer Token이라고도 함)을 추가해야 합니다. 이를 위해 Postman 우측 상단에서 “X API v2”라는 이름의 environment를 선택한 다음, 환경 드롭다운 옆의 눈 모양 아이콘을 클릭해 &quot;initial value&quot; 및 &quot;current value&quot; 필드에 키와 토큰을 추가합니다.

이 변수가 올바르게 설정되었다면, 요청의 &quot;Authorization&quot; 탭에 자동으로 반영됩니다.

**3단계: Quote Tweets를 조회하려는 게시물을 식별하고 지정**

Quote Tweets를 조회하려는 게시물을 지정하기 위해, 요청에 Post ID를 전달해야 합니다.

이 예제에서 사용할 Post ID는 1409931481552543749입니다.

Postman에서 &quot;Params&quot; 탭으로 이동한 뒤, id 파라미터의 &quot;Value&quot; 열에 이 Post ID를 입력합니다.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 1409931481552543749 |

**4단계: 조회하려는 필드를 식별하고 지정**

3단계 이후 &quot;Send&quot; 버튼을 클릭하면, 응답에서 기본 [Post object](/ko/x-api/fundamentals/data-dictionary#tweet) 필드인 id, text, edit&#95;history&#95;tweet&#95;ids를 받게 됩니다.

추가 필드를 받고 싶다면, [field](/ko/x-api/fundamentals/fields) 및/또는 [expansion](/ko/x-api/fundamentals/expansions) 파라미터를 사용해 요청에 해당 필드를 지정해야 합니다.

이 연습에서는 서로 다른 객체에서 세 가지 추가 필드 세트를 요청하겠습니다:

1. 기본 user 객체에서 추가 tweet.created&#95;at 필드
2. 반환된 게시물에 대한 작성자의 user 객체 기본 필드: id, name, username
3. 관련 user 객체에서 추가 user.created&#95;at 필드
    

Postman에서 &quot;Params&quot; 탭으로 이동한 뒤, &quot;Query Params&quot; 테이블에 다음 key:value 쌍을 추가합니다:

|     |     |     |     |
| :--- | :--- | :--- | :--- |
| **Key** | **Value** | **반환되는 필드** |
| `tweet.fields` | `created_at` | `tweets.created_at` |
| `expansions` | `author_id` | `includes.users.id`, `includes.users.name`, `includes.users.username` |
| `user.fields` | `created_at` | `includes.users.created_at` |     |

이제 &quot;Send&quot; 버튼 옆에 다음 URL이 표시되어야 합니다:

`https://api.x.com/2/tweets/:id/quote_tweets?expansions=author_id&tweet.fields=created_at&user.fields=created_at`

**참고:**

Postman에서는 URL 필드의 경로 파라미터 :id가 `id` params 필드에 입력한 값으로 자동으로 업데이트되지 않습니다. 따라서 위의 URL에는 1409931481552543749 대신 `:id`가 포함되어 있습니다.

<div id="step-six-paginate-through-your-results">
  #### 6단계: 결과를 페이지 단위로 조회하기
</div>

이전 응답의 하단에서 다음 필드를 포함하는 meta 데이터 객체를 확인할 수 있습니다.

* results&#95;count
* next&#95;token

4단계에서 max&#95;results 파라미터를 지정하지 않았기 때문에 기본적으로 게시물 10개만 받게 됩니다. 추가 데이터 페이지에 접근하려면, 마지막 결과에서 next&#95;token 필드의 값을 가져와 그 문자열을 Postman의 Params 페이지에서 pagination&#95;token 파라미터 값으로 설정하고, 나머지 값들은 그대로 둡니다. 

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `pagination_token` | avdjwk0udyx6 |

모든 설정을 마쳤다면 &quot;Send&quot;를 다시 클릭하여 다음 페이지의 결과를 받을 수 있습니다. 

이 개념을 더 자세히 설명하기 위해 [페이지네이션](/ko/x-api/fundamentals/pagination)에 대한 가이드를 준비해 두었습니다.
