게시물을 검색하는 것은 특정 주제나 이벤트와 관련된 X 대화를 노출하는 데 사용되는 중요한 기능입니다. 이 기능은 X 내 기본 기능으로도 제공되지만, 이 엔드포인트들은 게시물을 필터링하고 수집할 때 더 큰 유연성과 강력한 기능을 제공하여, 연구에 관련된 데이터를 더 쉽게 찾고, 준실시간 ‘리스닝(listening)’ 애플리케이션을 구축하거나, 관심 있는 주제와 관련된 게시물을 전반적으로 탐색·분석·활용할 수 있도록 합니다. 

게시물 검색을 위해 사용할 수 있는 두 개의 엔드포인트를 제공합니다: Recent search와 full-archive search입니다. 이 두 REST 엔드포인트는 특정 주제와 관련된 게시물을 필터링하기 위해 단일 검색 쿼리를 사용하는 등 공통된 설계와 기능을 공유합니다. 이 검색 쿼리는 메시지 키워드, 해시태그, URL과 같은 게시물 및 사용자 속성에 매칭되는 연산자 집합으로 구성됩니다. 연산자는 불리언 로직과 괄호를 사용해 조합할 수 있어 쿼리의 매칭 동작을 세밀하게 조정할 수 있습니다. 

Recent search와 full-archive search 엔드포인트 모두 편집된 게시물 메타데이터를 제공합니다. 2022년 9월 29일 이후에 생성된 게시물의 모든 객체에는, 게시물이 한 번도 편집되지 않았더라도, 게시물 편집 메타데이터가 포함됩니다. 게시물이 편집될 때마다 새로운 게시물 ID가 생성됩니다. 게시물의 편집 기록은 원래 ID부터 시작하는 게시물 ID 배열로 기록됩니다.

이 엔드포인트들은 항상 가장 최근에 편집된 버전과 편집 이력을 함께 반환합니다. 게시물의 30분 편집 가능 시간 이후에 수집된 게시물은 최종 버전을 나타냅니다. Edit Post 메타데이터에 대해 더 알아보려면 [Edit Posts fundamentals](/ko/x-api/fundamentals/edit-posts) 페이지를 확인하세요.

쿼리를 설정하고 게시물을 수신하기 시작하면, 이 엔드포인트들은 시간 및 게시물 ID 범위를 기준으로 결과를 탐색하는 기능을 지원합니다. 이는 두 가지 일반적인 사용 사례를 지원하도록 설계되었습니다: 

* **Get historical**: 요청은 데이터의 실시간 특성보다는 관심 있는 기간에 초점을 맞춥니다. 하나의 요청이 이루어지며, 필요한 경우 페이지네이션을 사용해 일치하는 모든 데이터가 전달됩니다. 이것이 Search Posts의 기본 모드입니다.
* **Polling** 또는 **listening**: 요청은 “마지막 요청 이후 새 게시물이 있나요?” 모드로 이루어집니다. 요청은 지속적으로 수행되며, 일반적으로 관심 있는 게시물을 준실시간으로 ‘리스닝(listening)’하는 사용 사례에 초점이 맞춰져 있습니다.

많은 연산자와 쿼리 한도는 [Enterprise access](/ko/x-api/getting-started/about-x-api)에 한정되어 있으며, 추가 기능을 활용하려면 Enterprise access가 있는 [Project](/ko/resources/fundamentals/projects) 내 앱의 키와 토큰을 사용해야 합니다. 이에 대해서는 아래 엔드포인트 섹션에서 더 자세히 알아볼 수 있습니다.

Recent search와 full-archive search 엔드포인트에서 반환된 게시물은 모두 월간 [Post cap](/ko/x-api/fundamentals/post-cap)에 포함됩니다.

**Account setup**

이 엔드포인트에 액세스하려면 다음이 필요합니다:

* 승인된 [개발자 계정](https://developer.x.com/en/portal/petition/essential/basic-info)
* [Project](/ko/resources/fundamentals/projects) 내에 위치한 [developer App](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용한 인증 

X API v2 엔드포인트에 대한 액세스를 얻는 방법에 대해 더 알아보려면 [getting started guide](/ko/x-api/getting-started/getting-access)를 참고하세요.

<div id="recent-search">
  ## 최근 검색
</div>

recent search 엔드포인트를 사용하면 지난 일주일 동안 게시된 공개 게시물에 대해 필터링된 결과에 프로그래밍 방식으로 접근할 수 있습니다. 이 엔드포인트는 개발자 계정을 보유하고, 하나의 [App](/ko/resources/fundamentals/developer-apps)이 속한 [Project](/ko/resources/fundamentals/projects)의 키와 토큰을 사용하는 모든 개발자가 이용할 수 있습니다.

요청은 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication), [OAuth 2.0 App-Only](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), 또는 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용해 인증할 수 있습니다. 다만 비공개 메트릭을 받거나, 검색 결과에 포함된 게시물에 대해 organic 및 promoted 메트릭의 세부 내역을 받고 싶은 경우에는 OAuth 1.0a User Context 또는 OAuth 2.0 Authorization Code with PKCE를 사용해야 하며, 해당 콘텐츠를 게시한 사용자와 연결된 사용자 액세스 토큰을 전달해야 합니다. 

이 엔드포인트는 요청당 최대 100개의 게시물을 최신순(역순 연대기)으로 반환할 수 있으며, 일치하는 게시물이 많을 경우 이를 페이지 단위로 조회할 수 있도록 [페이지네이션](/ko/x-api/posts/search/integrate/paginate) 토큰을 제공합니다. 

일반 액세스 권한이 있는 프로젝트를 사용하는 경우 기본 [연산자(operator)](/ko/x-api/posts/search/integrate/build-a-query) 집합을 사용할 수 있으며, 최대 512자 길이의 쿼리를 만들 수 있습니다. Enterprise 액세스 권한이 있는 프로젝트를 사용하는 경우에는 추가 연산자에 접근할 수 있습니다. Enterprise 액세스를 가진 프로젝트는 최대 4096자 길이의 쿼리를 만들 수 있습니다.

[액세스 수준](/ko/x-api/getting-started/about-x-api)에 대해 더 알아보세요.

<div id="full-archive-search">
  ## 전체 아카이브 검색
</div>

v2 전체 아카이브 검색 엔드포인트는 [Pro](/ko/x-api/getting-started/about-x-api "Pro") 액세스와 [Enterprise](/ko/x-api/getting-started/about-x-api) 액세스가 있는 프로젝트에서만 사용할 수 있습니다. 이 엔드포인트를 사용하면 검색 쿼리에 따라 2006년 3월의 첫 번째 게시물까지 거슬러 올라가는 전체 아카이브에서 공개 게시물을 프로그래밍 방식으로 조회할 수 있습니다.

이 엔드포인트에 대한 요청은 [OAuth 2.0 App-Only](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)를 사용하여 인증할 수 있으며, [App Access Token](/ko/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)은 Pro 또는 Enterprise 액세스가 있는 프로젝트 내 App에서 발급된 것이어야 합니다. 이 엔드포인트로는 다른 사용자를 대신하여 요청할 수 없기 때문에(OAuth 1.0a 사용자 컨텍스트 또는 OAuth 2.0 Authorization Code with PKCE 사용 불가), 비공개 [메트릭](/ko/x-api/fundamentals/metrics)을 가져올 수 없습니다. 

이 엔드포인트는 요청당 최대 500개의 게시물을 최신 순(역순 시계열)으로 반환할 수 있으며, 일치하는 게시물이 많은 경우 이를 페이지 단위로 탐색할 수 있도록 [페이지네이션](/ko/x-api/posts/search/integrate/paginate) 토큰을 제공합니다.

**참고:** tweet.fields 매개변수를 통해 [주석(annotations)](/ko/x-api/fundamentals/post-annotations)을 요청하는 경우, max&#95;results 매개변수의 최대값은 현재 100으로 제한됩니다. 이는 향후 변경될 수 있으나, 이 제한 사항을 염두에 두고 사용해 주시기 바랍니다.

이 엔드포인트는 Pro 및 Enterprise 액세스 승인을 받은 사용자에게만 제공되므로, 전체 검색 [연산자](/ko/x-api/posts/search/integrate/build-a-query) 세트를 사용할 수 있으며 최대 1024자 길이의 쿼리를 작성할 수 있습니다.
