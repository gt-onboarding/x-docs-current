이 세 가지 타임라인 endpoint는 수정된 게시물을 지원합니다. 이러한 endpoint는 항상 최신 수정본과 함께 수정 이력을 반환합니다. 게시물이 30분 편집 가능 시간 이후에 수집된 경우, 해당 게시물은 최종 버전을 나타냅니다. 수정 메타데이터에는 해당 게시물 이력에 있는 모든 게시물의 ID 배열이 포함됩니다. 수정 이력이 없는 게시물의 경우 이 배열에는 하나의 ID만 포함됩니다. 수정된 게시물의 경우 이 배열에는 여러 개의 ID가 포함되며, 수정 순서를 반영하도록 오름차순으로 정렬되어 배열의 마지막 위치에 가장 최신 버전이 놓입니다. 게시물 수정 방식에 대해 더 알아보려면 [Edit Posts fundamentals](/ko/x-api/fundamentals/edit-posts) 페이지를 참조하세요.

<div id="reverse-chronological-home-timeline">
  ## 최신순 Home 타임라인
</div>

이 엔드포인트를 사용하면 인증된 사용자와 해당 사용자가 팔로우하는 계정이 작성한 최신 게시물, 리포스트, 답글을 조회할 수 있습니다. 

특정 사용자를 대신해 요청을 보내는 것이므로, 이 엔드포인트는 [PKCE를 사용하는 OAuth 2.0 Authorization Code Flow](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) 또는 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication)를 사용해 인증해야 합니다. 이 엔드포인트의 사용자별 레이트 리밋은 15분당 180건의 요청입니다. 이 엔드포인트는 타임라인에서 최근 7일 동안 생성된 모든 게시물과, 생성 날짜와 관계없이 가장 최근 게시물 800개까지 반환할 수 있습니다.

<div id="user-post-timeline">
  ## 사용자 게시물 타임라인
</div>

사용자 게시물 타임라인 엔드포인트는 특정 X 계정이 게시한 게시물에 접근할 수 있도록 해줍니다. 사용자의 게시물을 가져오면 사용자 인터페이스에 타임라인을 표시하거나, 사용자의 게시물을 분석해 해당 사용자의 콘텐츠를 더 잘 이해하거나, 게시물을 프로그래밍 방식으로 활용하는 참여(engagement) 워크플로를 구성하는 등의 경험을 구현할 수 있습니다. 이 엔드포인트는 사용자의 프로필 타임라인에서 볼 수 있는 것과 유사하게, 단일 X 계정의 최신 게시물, 리트윗, 답글, 인용 트윗에 대한 액세스를 제공합니다.

다음은 @XDevelopers 계정의 사용자 타임라인 예시입니다.

사용자 게시물 타임라인 엔드포인트는 원하는 사용자를 지정하기 위해 단일 경로 매개변수(사용자 ID 기준)를 받는 REST 엔드포인트입니다. 이 엔드포인트는 해당 사용자가 게시한 가장 최근의 게시물, 리트윗, 답글, 인용 트윗을 최대 3,200개까지 반환할 수 있습니다.

게시물은 가장 최근 게시물부터 시작하는 역순(내림차순) 시간순으로 제공됩니다. 결과는 한 페이지당 최대 100개의 게시물까지 [페이지네이션](/ko/x-api/fundamentals/pagination)됩니다. 많은 수의 게시물을 페이지 단위로 조회할 수 있도록 페이지네이션 토큰이 제공됩니다. 주어진 페이지에 포함된 가장 최신 게시물과 가장 오래된 게시물의 게시물 ID도 메타데이터로 제공되며, 이는 최신 게시물에 대한 타임라인을 폴링하는 데에도 사용할 수 있습니다. 사용자 게시물 타임라인은 또한 start&#95;time과 end&#95;time 매개변수를 지정하여 특정 시간 범위 내에 생성된 게시물만 받을 수 있도록 지원합니다. 

사용자 게시물 타임라인 엔드포인트는 [fields](/ko/x-api/fundamentals/fields) 및 [expansions](/ko/x-api/fundamentals/expansions) 매개변수를 지원하며, [새 JSON 데이터 형식](/ko/x-api/fundamentals/data-dictionary)을 반환합니다.

이 엔드포인트에 요청을 성공적으로 보내려면 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication), [OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3), 또는 [OAuth 2.0 App-Only](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) 인증 방식을 사용해 요청을 인가해야 합니다. 비공개 메트릭, 프로모션 메트릭, 또는 보호된 사용자의 타임라인을 요청하는 경우에는 반드시 OAuth 1.0a User Context 또는 OAuth 2.0 Authorization Code with PKCE를 사용해야 합니다. 

사용자 게시물 타임라인 엔드포인트는 두 가지 일반적인 사용 패턴을 지원하도록 설계되었습니다. 

* &quot;사용자의 과거 게시물 가져오기&quot;: 관심 있는 사용자가 특정 최신 기간 동안 작성한 게시물을 시간순(과거 → 최신)으로 받기 위해 사용자 게시물 타임라인에 요청을 보내는 패턴입니다. 이 기간은 start&#95;time과 end&#95;time을 사용하고 전체 결과를 페이지네이션하여 설정할 수 있습니다. 일부 경우, 해당 사용자가 계정에서 작성한 게시물이 3,200개 이하인 경우에는 사용자의 전체 게시 이력을 모두 가져올 수 있습니다. 포함되는 게시물은 공개 여부와 요청에 사용된 인증 방식에 따라 달라집니다.

* &quot;새 게시물 폴링&quot;: 특정 사용자가 새로 작성한 게시물을 가져오기 위해 사용자 게시물 타임라인에 지속적으로 요청을 보내는 패턴입니다. 마지막으로 수신한 게시물 ID를 이후 요청에서 파라미터로 설정하여, 마지막 게시물 이후의 새 게시물만 가져올 수 있습니다.

<div id="user-mention-timeline">
  ## 사용자 멘션 타임라인
</div>

user mention timeline 엔드포인트를 사용하면 특정 X 사용자를 멘션한 게시물을 요청할 수 있습니다. 예를 들어, 어떤 X 계정이 게시물에서 @XDevelopers를 멘션한 경우입니다. 여기에는 요청한 사용자의 게시물에 대한 답글도 포함됩니다. 사용자의 멘션을 조회하면 누가 해당 사용자의 게시물에 답글을 남기고 있는지, 누가 멘션하고 있는지 등을 빠르게 파악하거나, 해당 사용자의 게시물을 대상으로 한 참여(engagement) 워크플로를 프로그램 방식으로 구성하는 등의 기능을 구현할 수 있습니다. 이 엔드포인트를 사용하면 단일 사용자의 가장 최근 멘션 및 답글을 요청할 수 있으며, 이는 X에서 사용자의 [멘션 알림](https://x.com/notifications/mentions)에 표시되는 내용과 유사합니다.

user mention timeline은 원하는 사용자(사용자 ID 기준)를 지정하기 위해 단일 경로 매개변수를 받는 REST 엔드포인트입니다. 이 엔드포인트는 해당 사용자의 가장 최근 멘션 800개까지 반환할 수 있습니다.

게시물은 가장 최근 것부터 시작하여 내림차순(최신 순)으로 전달됩니다. 결과는 페이지당 최대 100개의 게시물까지 [페이지네이션](/ko/x-api/fundamentals/pagination)됩니다. 많은 수의 게시물을 페이지 단위로 조회할 수 있도록 페이지네이션 토큰이 제공됩니다. 주어진 페이지에 포함된 게시물 중 가장 최신과 가장 오래된 게시물의 게시물 id는 메타데이터로 함께 제공되며, 이를 사용하여 최신 게시물을 폴링하거나 v1.1의 [mentions&#95;timeline](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-mentions_timeline) 엔드포인트와 유사한 방식으로 타임라인을 탐색할 수 있습니다. 또한 이 엔드포인트는 start&#95;time 및 end&#95;time 매개변수를 지정하여 특정 시간 범위 내에 생성된 게시물을 수신하는 기능도 지원합니다. 

이 엔드포인트에 대한 요청을 성공적으로 수행하려면 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication), [OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3), 또는 [OAuth 2.0 App-Only](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) 인증 방법을 사용하여 요청을 승인해야 합니다. 비공개 메트릭, 광고 메트릭(promoted metrics) 또는 보호된 사용자의 타임라인을 요청하는 경우에는 반드시 OAuth 1.0a User Context 또는 OAuth 2.0 Authorization Code with PKCE를 사용해야 합니다.

user mention timeline 엔드포인트는 [fields](/ko/x-api/fundamentals/fields) 및 [expansions](/ko/x-api/fundamentals/expansions) 매개변수를 지원하며, [새 JSON 데이터 형식](/ko/x-api/fundamentals/data-dictionary)을 반환합니다.
