<div id="standard-v11-timelines-to-x-api-v2-timelines">
  ## 표준 v1.1 타임라인에서 X API v2 타임라인으로
</div>

v1.1 타임라인 엔드포인트(statuses/user&#95;timeline 및 statuses/mentions&#95;timeline)를 사용해 왔다면, 이 가이드의 목표는 표준 타임라인 엔드포인트와 X API v2 타임라인 엔드포인트 간의 공통점과 차이점을 이해하여, 현재 통합을 새 버전으로 마이그레이션할 수 있도록 돕는 것입니다.

* **유사점:**
  * 인증:
    * OAuth 1.0a 사용자 컨텍스트(역순 시간순 홈 타임라인, 사용자 게시물 타임라인 및 사용자 멘션 타임라인)
    * OAuth 2.0 앱 전용(App-Only)(사용자 게시물 타임라인)
  * 과거 데이터 접근 한도: 사용자 타임라인(사용자 게시물 타임라인)은 최신 3,200개의 게시물에 대한 접근을 제공하며, 멘션 타임라인(사용자 멘션 타임라인)은 최신 800개의 멘션에 대한 접근을 제공합니다.
  * 게시물 편집 기록 및 메타데이터 지원
  * 요청 한도(사용자 게시물 타임라인)
  * 새로 고침 폴링: since&#95;id 이후의 새로운 결과를 가져오는 기능
  * 게시물 ID를 통한 타임라인 탐색
  * 결과 사양:
    * 결과 순서: 결과는 최신순(역순 시간순)으로 반환됨
    * 답글 제외 기능(사용자 게시물 타임라인만 해당)
    * 리트윗 제외 기능(사용자 게시물 타임라인만 해당)
* **차이점**
  * 새로운 인증 기능: 
    * OAuth 2.0 앱 전용(App-Only)(사용자 멘션 타임라인)
    * OAuth 2.0 Authorization Code Flow with PKCE(역순 시간순 홈 타임라인, 사용자 게시물 타임라인 및 사용자 멘션 타임라인)
  * 액세스 요구 사항: X API v2 앱 및 프로젝트 요구 사항
  * 요청 한도(사용자 멘션 타임라인 및 역순 시간순 홈 타임라인)
  * 요청 한도 및 처리량 한도의 차이
  * 추가 페이지네이션 방식
    * 응답당 서로 다른 max&#95;results(count) 값
  * 응답 데이터 형식
  * 요청 파라미터
    * v2 필드 및 expansions를 포함하여 요청 파라미터에 따라 맞춤형 데이터 형식 제공
    * 추가로 제공되는 데이터: 메트릭, 게시물 주석, 투표

<div id="similarities">
  ### 유사점
</div>

**인증**

v1.1 `statuses/user_timeline`과 X API v2 사용자 게시물 타임라인 엔드포인트는 모두 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication)와 [OAuth 2.0 앱 전용(App-Only)](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)를 지원합니다. 따라서 X API v2 버전으로 마이그레이션하더라도 동일한 인증 방식과 권한 부여 토큰을 계속 사용할 수 있습니다.

**과거 데이터 접근**

v1.1 `statuses/user_timeline`과 X API v2 사용자 게시물 타임라인 엔드포인트는 모두 리트윗을 포함하여 가장 최근 3,200개의 게시물을 반환합니다.

v1.1 `statuses/mentions_timeline`과 X API v2 사용자 멘션 타임라인 엔드포인트는 가장 최근 800개의 게시물을 반환할 수 있습니다.

**게시물 수정 이력 및 메타데이터 지원**

두 버전 모두 수정 이력을 설명하는 메타데이터를 제공합니다. 자세한 내용은 [filtered stream API 레퍼런스](/ko/x-api/posts/filtered-stream#api-reference-index)와 [게시물 수정(Edit Posts) 기본 사항 페이지](/ko/x-api/fundamentals/edit-posts)를 참고하세요. 

**요청 한도**

|     |     |
| :--- | :--- |
| **Standard v1.1** | **X API v2** |
| user&#95;timeline:<br /><br />OAuth 1.0a 사용자 컨텍스트로 15분당 900회 요청<br /><br />OAuth 2.0 앱 전용(App-Only)으로 15분당 1,500회 요청 | 사용자 게시물 타임라인:<br /><br />OAuth 1.0a 사용자 컨텍스트로 15분당 900회 요청<br /><br />OAuth 2.0 앱 전용(App-Only)으로 15분당 1,500회 요청 |

**since&#95;id를 사용한 폴링 갱신**

두 버전 모두 `since_id`를 사용하여 최신 결과를 폴링할 수 있습니다.

**게시물 ID를 사용한 타임라인 탐색**

두 엔드포인트 모두 게시물 ID가 구성되는 방식에 기반해 게시물 ID ‘타임스탬프’를 사용하여 타임라인을 탐색할 수 있습니다. 이 기능은 일반적으로 동일하지만 다음과 같은 차이점이 있습니다:

|     |     |
| :--- | :--- |
| **Standard timelines v1.1** | **timelines v2** |
| since&#95;id(제외)<br /><br />max&#95;id(포함) | since&#95;id(제외)<br /><br />until&#95;id(포함이었던 max&#95;id와 달리 이 값도 제외) |

**응답 필터링 매개변수**

|     |     |
| :--- | :--- |
| **Standard timelines v1.1** | **timelines v2** |
| 응답 필터링 매개변수:<br /><br />* include&#95;rts<br />* exclude&#95;replies | 응답 필터링 매개변수:<br /><br />* exclude=retweets,replies |
| 예시 <br /><br />https://api.x.com/1.1/statuses/user&#95;timeline.json?user&#95;id=2244994945&amp;include&#95;rts=0&amp;&amp;exclude&#95;replies=1 | 예시:<br /><br />https://api.x.com/2/users/2244994945/tweets?max&#95;results=100&amp;exclude=retweets,replies |
| 참고:<br /><br />user&#95;timeline의 경우:<br /><br />* `include_rts=0`을 사용해도 가장 최근 3,200개까지 가능한 과거 게시물 한도는 변경되지 않습니다 | 참고:<br /><br />사용자 게시물 타임라인의 경우:<br /><br />* `exclude=retweets`를 사용해도 가장 최근 3,200개까지 가능한 과거 게시물 한도는 변경되지 않습니다<br />* `exclude=replies`를 사용하면 가능한 과거 게시물 한도가 가장 최근 800개의 답글로 줄어듭니다 |

<div id="differences">
  ### 차이점
</div>

**인증**

**v1.1 `statuses/mentions_timeline` 엔드포인트는 [OAuth 1.0a User Context](https://developer-staging.x.com/resources/fundamentals/authentication)만 지원합니다. X API v2 user mention timeline 엔드포인트는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication), [OAuth 2.0 App-Only](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only), 그리고 [OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authenticationoauth-2-0/authorization-code "이 메서드는 권한이 부여된 앱이 사용자를 대신하여, 사용자로서 동작할 수 있도록 허용합니다. 일반적으로 특정 사용자의 공개 정보를 조회하거나 게시할 때 사용되며, 이 엔드포인트가 반환하는 데이터와 사용자 간의 관계를 앱이 인지해야 할 때 유용합니다. OAuth 2.0 Authorization Code with PKCE로 인증하는 방법을 알아보려면 클릭하세요.")를 모두 지원합니다.**

X API v2 user Post timeline 엔드포인트를 사용해 비공개 또는 프로모션 메트릭에 액세스하려면 OAuth 1.0a User Context 또는 OAuth 2.0 Authorization Code with PKCE를 사용하고, 메트릭에 액세스하려는 게시물을 작성한 사용자와 연관된 사용자 액세스 토큰을 전달해야 합니다.
 

**엔드포인트 URL**

X API v2 timelines 엔드포인트는 사용자 ID에 해당하는 경로 매개변수인 :id가 필요합니다.

* 표준 v1.1 엔드포인트:
  * https://api.x.com/1.1/statuses/home&#95;timeline
  * https://api.x.com/1.1/statuses/user&#95;timeline
  * https://api.x.com/1.1/statuses/mention&#95;timeline
* X API v2 엔드포인트:
  * https://api.x.com/2/users/:id/timelines/reverse&#95;chronological
  * https://api.x.com/2/users/:id/tweets
  * https://api.x.com/2/users/:id/mentions

**앱 및 프로젝트 요구 사항**

X API v2 엔드포인트를 사용하려면, 요청을 인증할 때 [developer App](/ko/resources/fundamentals/developer-apps)의 자격 증명을 사용해야 하며, 이 App은 [Project](/ko/resources/fundamentals/projects)에 연결되어 있어야 합니다. 모든 X API v1.1 엔드포인트는 프로젝트에 연결되지 않은 독립형 App 또는 프로젝트에 연결된 App의 자격 증명을 사용할 수 있습니다. 
 

**요청 한도**

|     |     |
| :--- | :--- |
| **mentions&#95;timeline:**<br /><br />OAuth 1.0a User Context로 15분당 75회 요청 | **user mention timeline:**<br /><br />OAuth 1.0a User Context로 15분당 180회 요청  <br />OAuth 2.0 Bearer Token으로 15분당 450회 요청 |
| **home&#95;timelime:**<br /><br />15분당 15회 요청  <br /><br />홈 타임라인에서 최대 800개의 게시물을 조회할 수 있음 | **reverse chronological home timeline:**<br /><br />15분당 180회 요청<br /><br />지난 7일 동안 타임라인에 생성된 모든 게시물과, 생성일과 관계없이 가장 최근 800개의 게시물을 모두 반환할 수 있습니다. |

**요청 제한**

|     |     |
| :--- | :--- |
| **Standard timelines v1.1** | **timelines v2** |
| 일일 요청 한도<br /><br />* 24시간 동안 100,000건 요청 상한. | 별도의 일일 요청 한도는 없으며, 소비량에 의해서만 제한됩니다. |

**소비량 제한**

|     |     |
| :--- | :--- |
| **Standard timelines v1.1** | **timelines v2** |
| 요청 한도에 의해서만 소비량이 제한됨<br /><br />* 24시간 동안 100,000건 요청 상한. | 소비량은 [access level](/ko/x-api/getting-started/about-x-api)에 따라 (여러 v2 엔드포인트 전반에 걸친) 프로젝트 단위 월별 [Post cap](/ko/x-api/fundamentals/post-cap)에 의해서만 제한됩니다.<br /><br />* Essential 액세스로 월 500,000개 게시물<br />* Elevated 액세스로 월 200만 개 게시물<br />* Academic Research 액세스로 월 1,000만 개 게시물 <br /><br />Reverse chronological home timeline에는 월별 Post cap이 적용되지 않습니다. |

**요청 매개변수**

|     |     |
| :--- | :--- |
| **Standard timelines v1.1** | **timelines v2** |
| 필수: user&#95;id 또는 screen&#95;name | 필수: 특정 사용자 ID를 path 파라미터에 지정해야 합니다 |
| 선택 사항:<br /><br />count - 요청당 반환되는 최대 결과 개수 설정<br /><br />exclude&#95;replies - 결과에서 답글 제거<br /><br />Include&#95;rts - 0으로 설정하면 결과에서 리트윗 제거<br /><br />trim&#95;user - 결과에서 재하이드레이션된 사용자 객체 제거<br /><br />tweet&#95;mode - 결과에 대해 반환되는 데이터 형식을 설정하며, 140자를 초과하는 게시물의 경우 extended로 설정<br /><br />since&#95;id - 결과에서 가장 이른 게시물 ID를 설정(제외)<br /><br />max&#95;id - 결과에서 가장 늦은 게시물 ID를 설정(포함) | 선택 사항:<br /><br />max&#95;results - 요청당 반환되는 최대 결과 개수 설정<br /><br />exclude=retweets,replies - 결과에서 리트윗 또는 답글 제거<br /><br />tweet.fields - 반환할 게시물(Post) 객체 필드 설정<br /><br />user.fields - 반환할 사용자(User) 객체 필드 설정<br /><br />place.fields - 반환할 장소(place) 객체 필드 설정<br /><br />media.fields - 반환할 미디어(media) 객체 필드 설정<br /><br />poll.fields - 반환할 투표(poll) 객체 필드 설정<br /><br />expansions - 확장할 필드와 반환할 데이터 설정<br /><br />start&#95;time - 결과에 대해 가장 이른 created&#95;at 시간 설정<br /><br />end&#95;time - 결과에 대해 가장 늦은 created&#95;at 시간 설정<br /><br />since&#95;id - 결과에 대해 가장 이른 게시물 ID 설정(제외)<br /><br />until&#95;id - 결과에서 가장 늦은 게시물 ID 설정(제외) |

**X API v2 JSON 형식**

X API v2는 API가 반환하는 객체에 대해 새로운 JSON 설계를 도입하고 있으며, 여기에는 [게시물(Post)](/ko/x-api/fundamentals/data-dictionary#tweet) 및 [사용자(user)](/ko/x-api/fundamentals/data-dictionary#user) 객체가 포함됩니다. X API v2 형식과 필드 및 expansions 사용 방법에 대해서는 [가이드](/ko/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)와 보다 포괄적인 [data dictionary](/ko/x-api/fundamentals/data-dictionary)를 통해 자세히 알아볼 수 있습니다.

* JSON 루트 레벨에서, standard 엔드포인트는 statuses 배열 안에 게시물(Post) 객체를 반환하는 반면, X API v2는 data 배열을 반환합니다. 
* 리트윗(Retweeted) 및 인용(Quoted) &quot;statuses&quot;를 참조하는 대신, X API v2 JSON은 리트윗된(Retweeted) 및 인용된(Quoted) 트윗을 참조합니다. contributors, user.translator&#95;type과 같은 많은 레거시 및 사용 중단 예정 필드는 제거되고 있습니다. 
* 게시물(Post) 객체의 favorites와 사용자(user) 객체의 favorites를 모두 사용하는 대신, X API v2는 like라는 용어만 사용합니다. 
* X는 값이 없는 JSON 값(예: null)은 페이로드에 기록하지 않는 규칙을 채택하고 있습니다. 게시물(Post) 및 사용자(user) 속성은 null이 아닌 값을 가진 경우에만 포함됩니다.
   

standard v1.1 버전과 X API v2 엔드포인트 버전 사이의 가장 큰 차이점 중 하나는 페이로드에 어떤 필드를 반환할지 선택하는 방식입니다. standard 엔드포인트에서는 페이로드에 어떤 필드 또는 필드 세트를 반환할지 지정하기 위해 여러 파라미터를 사용할 수 있었던 반면, X API v2 버전에서는 이러한 다양한 파라미터를 [fields](/ko/x-api/fundamentals/fields) 및 [expansions](/ko/x-api/fundamentals/expansions)로 단순화했습니다. 

* fields: X API v2 엔드포인트를 사용하면 페이로드에 어떤 필드를 포함할지 선택할 수 있습니다. 예를 들어, 게시물, 사용자, 미디어, 장소, 투표 객체는 모두 반환할 (또는 반환하지 않을) 수 있는 필드 목록을 가지고 있습니다. 

* expansions: 게시물 JSON 페이로드에서 참조되는 관련 객체를 확장하는 데 사용됩니다. 예를 들어, 모든 리트윗과 답글은 다른 게시물을 참조합니다. expansions=referenced&#95;tweets.id 를 설정하면 이러한 다른 게시물 객체는 tweet.fields 설정에 따라 확장됩니다. 사용자, 투표, 미디어와 같은 다른 객체들도 확장할 수 있습니다. 

* conversation&#95;id

* context와 entities를 포함하는 두 개의 새로운 [annotations](/ko/x-api/fundamentals/post-annotations) 필드

* 여러 새로운 [metrics](/ko/x-api/fundamentals/metrics) 필드 
   

Standard v1.1 필드를 최신 v2 필드에 매핑하는 데 도움이 되는 [데이터 형식 마이그레이션 가이드](/ko/x-api/migrate/data-format-migration#migrating-from-standard-v1-1s-data-format-to-v2)를 준비해 두었습니다. 이 가이드는 또한 특정 필드를 반환하기 위해 v2 요청과 함께 전달해야 하는 expansions 및 필드 매개변수를 구체적으로 안내합니다.
