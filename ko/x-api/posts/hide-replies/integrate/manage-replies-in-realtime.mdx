<div id="manage-replies-in-realtime">
  ### 실시간으로 답글 관리하기
</div>

hide replies 엔드포인트를 사용하면, 관련성이 매우 낮을 가능성이 큰 답글을 사용자가 숨길 수 있도록 해 주는 워크플로를 구축할 수 있습니다.

유용한 앱은 종종 여러 기술을 결합해 사용자에게 더 큰 가치를 제공합니다. 이 페이지에서는 [Perspective API](https://www.perspectiveapi.com/)를 사용해 답글을 숨기는 방법을 설명합니다. 이 API는 Google 산하 조직인 [Jigsaw](https://jigsaw.google.com/)에서 학습시킨 인공지능으로, 악성 댓글을 탐지합니다. 애플리케이션 로직은 다음과 같이 동작합니다.

1. 사용자의 게시물을 읽고, 그 게시물에 대한 답글을 숨기거나 다시 표시할 수 있도록 사용자에게 권한을 요청합니다.
2. [Account Activity API](https://developer.x.com/en/docs/x-api/v1/accounts-and-users/subscribe-account-activity/overview)를 사용해 새로 들어오는 답글을 감지합니다.
3. Perspective API에 0과 1 사이의 숫자인 “점수”를 요청해, 해당 댓글이 과거에 봤던 악성 댓글과 얼마나 유사한지에 대한 알고리즘의 신뢰도를 나타내게 합니다. (Perspective는 서비스로 전송된 텍스트를 저장하지 않습니다.)
4. 알고리즘의 점수가 매우 높으면 hide replies를 호출합니다.

<div id="strive-for-transparency">
  #### 투명성을 추구하세요
</div>

Perspective는 실제 게시물에 대해 학습된 것이 아니기 때문에, 특정 언어적 뉘앙스로 인해 사용자가 숨기고 싶지 않은 답글이 이 앱에 의해 숨겨질 수 있습니다. 앱을 설계할 때 사용하는 기술이나 접근 방식과 상관없이, 항상 사용자가 앱이 무엇을 숨겼는지 이해하고 스스로 변경할 수 있도록 최선을 다해야 합니다.

* 가장 바람직한 방법은 항상 사용자를 신뢰하고, 사용자가 내린 결정에 대한 완전한 통제권을 제공하는 것입니다. 이는 사용자 경험에, 사용자를 대신해 앱이 수행한 모든 동작을 되돌릴 수 있는 제어 기능을 포함해야 함을 의미합니다.
* 인공지능을 사용할 때는, 앱이 게시물을 감지하고 숨기는 데 매우 높은 신뢰도 임계값을 사용해야 합니다.
* 모든 사람이 동일한 단어를 사용하는 것은 아니며, 앱은 편견을 피할 수 있도록 설계되어야 합니다. 오탐으로 이어질 수 있는 재전유된(reclaimed) 단어와 속어에 유의하세요.
* 인공지능을 학습시키는 경우, X에서 자주 사용되는 언어 사용을 잘 반영하는 모델을 채택하는 것을 고려하세요.
