<div id="manage-replies-in-realtime">
  ### 실시간으로 답글 관리하기
</div>

`hide replies` 엔드포인트를 사용하면, 관련성이 매우 낮을 것으로 예상되는 답글을 사용자가 숨길 수 있도록 도와주는 워크플로를 구축할 수 있습니다.

유용한 앱은 종종 여러 기술을 결합해 사용자에게 더 큰 가치를 제공합니다. 이 페이지에서는 [Perspective API](https://www.perspectiveapi.com/)를 사용하여 답글을 숨기는 방법을 설명합니다. 이 API는 Google 내 조직인 [Jigsaw](https://jigsaw.google.com/)에서 학습시킨 인공지능으로, 유해한 댓글을 탐지하도록 설계되었습니다. 애플리케이션 로직은 다음과 같이 동작합니다:

1. 사용자의 게시물을 읽고 해당 게시물의 답글을 숨기거나 다시 표시하기 위한 권한을 요청합니다.
2. [Account Activity API](https://developer.x.com/en/docs/x-api/v1/accounts-and-users/subscribe-account-activity/overview)를 사용하여 새로 들어오는 답글을 감지합니다.
3. Perspective API에 “점수”(0과 1 사이의 숫자)를 요청해, 특정 댓글이 과거에 본 유해 댓글과 얼마나 유사한지에 대한 알고리즘의 확신 정도를 나타내게 합니다. (Perspective는 서비스에 전송된 텍스트를 저장하지 않습니다.)
4. 알고리즘 점수가 매우 높으면 `hide replies`를 호출합니다.

<div id="strive-for-transparency">
  #### 투명성을 추구하세요
</div>

Perspective는 실제 게시물 데이터를 기반으로 학습된 모델이 아니므로, 특정 언어적 뉘앙스로 인해 사용자가 숨기고 싶지 않은 답글을 이 앱이 숨겨 버릴 수 있습니다. 앱을 설계할 때 사용하는 기술이나 접근 방식과 관계없이, 항상 사용자가 앱이 무엇을 숨겼는지 이해하고 필요할 경우 직접 변경할 수 있도록 최선을 다해야 합니다.

* 가장 바람직한 방법은 항상 사용자를 신뢰하고, 결정에 대한 전적인 통제권을 사용자에게 부여하는 것입니다. 이는 사용자 경험에 앱이 사용자를 대신해 수행한 모든 작업을 되돌릴 수 있는 제어 기능이 포함되어야 함을 의미합니다.
* 인공지능을 사용할 때에는 게시물을 탐지하고 숨기기 위해 매우 높은 신뢰 임계값을 사용해야 합니다.
* 모든 사람이 같은 단어를 사용하는 것은 아니며, 앱은 편향을 피할 수 있도록 설계되어야 합니다. 원래의 의미를 되찾아 사용되는(reclaimed) 표현과 은어가 오탐지(false positive)로 이어지지 않도록 주의하세요.
* 인공지능 모델을 직접 학습시키는 경우, X에서 자주 사용되는 언어 사용을 잘 반영하는 모델을 채택하는 것을 고려하세요.
