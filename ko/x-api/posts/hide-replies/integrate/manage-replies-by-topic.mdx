<div id="manage-replies-by-topic">
  ### 주제별 답글 관리
</div>

hide replies 엔드포인트를 통해 사람과 브랜드가 대화를 본래 주제에 맞게 유지하도록 돕는 통합 기능을 구축할 수 있습니다. 이 페이지에서는 hide replies와 [recent search](/ko/x-api/posts/recent-search) 엔드포인트를 사용해 대화를 관리하는 방법을 보여줍니다.

recent search는 하나의 대화와 그 답글을 조회할 수 있는 기능을 제공하며, Post 페이로드는 각 Post의 맥락과 주제를 언어와 상관없이 이해할 수 있도록 [Post annotations](/ko/x-api/fundamentals/post-annotations)을 반환합니다.

앱의 플로우에는 대화를 표시하고 관리하기 위한 제어 기능이 포함됩니다:

1. 사용자의 게시물을 읽고 답글을 관리할 수 있도록 사용자에게 권한을 요청합니다.

2. Post URL에서 최근 대화를 불러오고, 해당 대화가 인증된 사용자의 것인지 확인합니다.

3. recent search 엔드포인트를 호출하여 대화에 포함된 각 Post를 표시합니다. 요청에는 대화 ID 검색 쿼리와 annotation 확장을 포함하여, X가 해당 Post를 어떻게 해석하는지에 따라 그 Post가 스포츠 관련인지 여부를 판별합니다.

4. 사용자가 선택할 때 답글을 숨기기 위해 hide replies를 호출합니다. 또한 사용자가 항상 제어권을 가질 수 있도록, 필요한 경우 이 작업을 되돌릴 수 있는 방법도 제공합니다. 

5. 대화가 긴 경우, [검색 결과를 페이지네이션](/ko/x-api/posts/search/integrate/paginate)할 수 있는 제어 기능을 제공합니다.
    

<div id="optimize-for-the-user-and-for-usage">
  #### 사용자(와 사용량)을 위한 최적화
</div>

사용자가 수행하려는 모든 작업에 대해 사용자에게 제어권이 돌아가도록 흐름을 설계할 수 있습니다. 이 원칙을 염두에 두면 게시물 소비에 최적화된 통합을 구축하는 데에도 도움이 됩니다.

1. 인증된 사용자는 자신이 시작한 대화만 관리할 수 있으므로, 그렇지 않은 경우에는 흐름을 초기에 종료해야 합니다.
   * 초기 게시물 조회 요청을 수행합니다. 게시물 URL이 유효하지 않거나 대화를 시작한 사용자가 인증된 사용자가 아니라면 흐름을 초기에 종료합니다.
   * 이렇게 하면 인증된 사용자가 관리할 수 없는 대화에 대해 Recent search 요청을 보낼 필요가 없습니다.
2. 별도의 요청을 보내지 않도록 동일한 요청에서 [user 및 Post fields](/ko/x-api/fundamentals/fields)를 함께 요청하세요. 이 접근 방식은 앱의 성능을 향상시키는 데에도 도움이 될 수 있습니다.
3. 불필요한 요청은 피하세요. 이 앱은 사용자의 브라우저에 답글의 숨김 상태를 캐시합니다. 이는 사용자가 나중에 중재 작업을 이어서 진행하고 싶어 할 수 있는 대화가 큰 경우에 유용하며, 앱이 답글을 숨기거나 다시 표시하는 요청을 최적화하는 데 도움이 됩니다.
