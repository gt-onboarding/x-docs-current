<div id="getting-started-with-the-recent-post-counts-endpoint">
  ### 최근 게시물 개수 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 HTTP 요청을 보낼 수 있는 그래픽 도구인 Postman을 사용하여, 최근 게시물 개수 엔드포인트에 첫 번째 요청을 보내는 방법을 설명합니다.

여러 프로그래밍 언어로 된 샘플 코드를 보고 싶다면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요.

<Note>
  ### 사전 준비 사항

  이 가이드를 완료하려면 요청을 인증하기 위한 [keys and tokens](/ko/resources/fundamentals/authentication) 묶음이 필요합니다. 다음 단계를 따라 이러한 키와 토큰을 생성할 수 있습니다:

  * [개발자 계정 가입](https://developer.x.com/en/apply-for-access)을 완료하고 승인을 받습니다.
  * developer portal에서 [프로젝트](/ko/resources/fundamentals/projects)와 관련된 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성합니다.
  * App의 &quot;Keys and tokens&quot; 페이지로 이동하여 필요한 자격 증명을 생성합니다. 모든 자격 증명을 안전한 위치에 저장해야 합니다.
</Note>

<div id="steps-to-build-a-recent-post-counts-request">
  #### 최근 게시물 개수 요청을 구성하는 단계
</div>

**1단계: 도구 또는 라이브러리 선택**

이 엔드포인트에 요청을 보내는 데 사용할 수 있는 도구, 코드 예제, 라이브러리가 여러 가지 있지만, 여기서는 과정을 단순화하기 위해 Postman 도구를 사용하겠습니다.

환경에 X API v2 Postman 컬렉션을 불러오려면 아래 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2를 Postman에 추가
</Button>

Postman에 X API v2 컬렉션을 불러온 후, Post counts &gt; Recent Post counts 요청으로 이동합니다.

**2단계: 요청 인증**

X API에 요청을 올바르게 보내려면, 권한이 있는지 확인해야 합니다. 이 엔드포인트의 경우 [OAuth 2.0 App-Only](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) 인증 방식을 사용해 요청을 인증해야 합니다.

특히 [App Access Token](/ko/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token) (App-only Bearer Token이라고도 함)을 포함한 키와 토큰을 Postman에 추가해야 합니다. 이렇게 하려면 Postman 우측 상단에서 “X API v2”라는 이름의 environment를 선택한 후, 환경 드롭다운 옆의 눈 아이콘을 클릭해 &quot;initial value&quot; 및 &quot;current value&quot; 필드에 키와 토큰을 추가합니다.

이 변수가 올바르게 설정되었다면, 요청의 authorization 탭에 자동으로 반영됩니다.
 

**3단계: 쿼리 생성**

각 최근 게시물 개수 요청에는 하나의 [query](/ko/x-api/posts/counts/integrate/build-a-query)가 필요합니다. 이 예제에서는 @XDevelopers 계정이 게시한 게시물과 일치하는 쿼리를 사용하겠습니다. 이 쿼리에서는 from: 연산자를 사용하고 값을 XDevelopers(대소문자 구분 없음)로 설정합니다:

`from:XDevelopers`

Postman에서 &quot;Params&quot; 탭으로 이동한 후, 이 ID 또는 쉼표로 구분된 게시물 ID 문자열을 `ids` 파라미터의 &quot;Value&quot; 열에 입력합니다.

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Description** |
| `query` | from:XDevelopers | recent Post counts 엔드포인트에 전송할 쿼리 |

**4단계(선택): 요청의 세분성 지정**

3단계 이후에 ‘Send’ 버튼을 클릭하면, 기본 최근 게시물 개수(지난 7일 동안의 시간 단위 집계)를 받게 됩니다. 최근 게시물 개수를 일 단위로 받고 싶다면, granularity 파라미터를 추가하고 값을 day로 설정해야 합니다.

Postman에서 &quot;Params&quot; 탭으로 이동한 후, granularity 파라미터의 &quot;Value&quot; 열에 day를 입력합니다.

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Description** |
| granularity | day | 게시물 개수 결과의 세분성. 가능한 값은 day, hour 또는 minute입니다 |

이제 &quot;Send&quot; 버튼 옆에 다음 URL이 표시되어야 합니다:

`https://api.x.com/2/tweets/counts/recent?query=from%3AXDevelopers&granularity=day`
