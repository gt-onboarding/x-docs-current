<div id="getting-started-with-the-full-archive-post-counts-endpoint">
  ### 전체 아카이브 게시물 개수     엔드포인트 시작하기
</div>

이 퀵스타트 가이드는 HTTP 요청을 보낼 수 있는 그래픽 인터페이스 도구인 Postman을 사용해 전체 아카이브 게시물 개수 엔드포인트에 처음으로 요청을 보내는 방법을 안내합니다.

여러 프로그래밍 언어로 된 샘플 코드를 보고 싶다면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요. 

<Note>
  ### 사전 준비 사항

  전체 아카이브 게시물 개수 엔드포인트는 현재 Pro 또는 Enterprise 액세스 권한이 있는 경우에만 사용할 수 있습니다. 이 엔드포인트를 사용하려면 Pro 또는 [Enterprise access](https://docs.google.com/forms/d/e/1FAIpQLScO3bczKWO2jFHyVZJUSEGfdyfFaqt2MvmOfl_aJp0KxMqtDA/viewform)을(를) 신청해야 합니다. 

  액세스 승인을 받는 것 외에도, 요청을 인증하려면 [keys and tokens](/ko/resources/fundamentals/authentication) 일련이 필요합니다. 다음 단계를 따라 이러한 키와 토큰을 생성할 수 있습니다:

  * developer portal에서 Enterprise 액세스 권한이 있는 [Project](/ko/resources/fundamentals/projects)로 이동하고, 해당 Project에 연관된 [developer App](/ko/resources/fundamentals/developer-apps)이 있는지 확인합니다.

  * App의 &quot;Keys and tokens&quot; 페이지로 이동하여 필요한 자격 증명을 생성합니다. 모든 자격 증명을 안전한 위치에 보관하세요.
</Note>

<div id="steps-to-build-a-full-archive-post-counts-request">
  #### 전체 보관함 게시물 개수 요청을 작성하는 단계
</div>

**1단계: 도구 또는 라이브러리 선택**

이 엔드포인트에 요청을 보내기 위해 사용할 수 있는 도구, 코드 예제, 라이브러리가 여러 가지 있지만, 여기서는 프로세스를 단순화하기 위해 Postman 도구를 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 불러오려면 아래 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2를 Postman에 추가
</Button>

Postman에 X API v2 컬렉션을 불러온 후, Post counts &gt; Full-archive Post counts 요청으로 이동합니다.

**2단계: 요청 인증**

X API에 올바르게 요청을 보내려면, 권한이 있는지 확인해야 합니다. 이 엔드포인트에서는 [OAuth 2.0 App-Only](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) 인증 방법으로 요청을 인증해야 합니다.

특히 [App Access Token](/ko/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token) (App-only Bearer Token이라고도 함)을 포함한 키와 토큰을 Postman에 추가해야 합니다. 이를 위해 Postman 우측 상단에서 “X API v2”라는 이름의 환경을 선택한 후, 환경 드롭다운 옆의 눈 아이콘을 클릭해 &quot;initial value&quot;와 &quot;current value&quot; 필드에 키와 토큰을 추가합니다.

이 변수가 올바르게 설정되었다면 요청의 Authorization 탭에 자동으로 반영됩니다.
 

**3단계: 쿼리 생성**

각 전체 보관함 게시물 개수 요청에는 하나의 [query](/ko/x-api/posts/counts/integrate/build-a-query)가 필요합니다. 이 예제에서는 @XDevelopers 계정이 게시한 게시물에 매칭되는 쿼리를 사용하겠습니다. 이 쿼리에서는 from 연산자를 사용하고, 값을 XDevelopers(대소문자 구분 없음)로 설정합니다:

`from:XDevelopers`

Postman에서 &quot;Params&quot; 탭으로 이동한 뒤, 이 ID 또는 쉼표로 구분된 게시물 ID 문자열을 `ids` 파라미터의 &quot;Value&quot; 열에 입력합니다.
 

| Key | Value | Description |
| :--- | :--- | :--- |
| `query` | `from:XDevelopers` | 전체 보관함 게시물 개수 엔드포인트에 제출할 쿼리 |

4단계(선택 사항): 세분성(granularity) 및 기간 지정

3단계 이후에 ‘Send’ 버튼을 클릭하면, 기본 전체 보관함 게시물 개수 결과(지난 30일 동안의 시간별 집계)를 받게 됩니다. 전체 보관함 게시물 개수를 일 단위로 받고 싶다면, granularity 파라미터를 값 day로 추가해야 합니다. 30일 이전 기간의 게시물 개수를 조회하려면, 원하는 값으로 start&#95;time과 end&#95;time 파라미터를 지정해야 합니다. 

Postman에서 &quot;Params&quot; 탭으로 이동한 다음, &quot;Query Params&quot; 테이블에 아래 key:value 쌍을 추가합니다:

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Description** |
| granularity | day | 게시물 개수 결과의 세분성. 가능한 값은 day, hour, minute 입니다 |
| start&#95;time | 2021-05-01T00:00:00Z | 게시물이 제공될 가장 오래된 UTC 타임스탬프 |
| end&#95;time | 2021-06-01T00:00:00Z | 게시물이 제공될 가장 오래된 UTC 타임스탬프입니다. |

이제 &quot;Send&quot; 버튼 옆에 다음 URL이 표시되는 것을 볼 수 있습니다:

`https://api.x.com/2/tweets/counts/all?query=from%3AXDevelopers&start_time=2021-05-01T00:00:00Z&end_time=2021-06-01T00:00:00Z&granularity=day`

**6단계: 결과 페이지네이션 처리**

응답의 `meta` 객체에 `next_token`도 포함되어 있다면, 해당 값을 `next_token` 쿼리 파라미터로 전달할 수 있습니다.

| Key | Value | Description |
| :--- | :--- | :--- |
| next&#95;token | 이전 요청의 `meta` 객체에서 가져온 `next_token` 값을 여기 Value로 입력합니다. | 최신 요청이 나머지 결과를 모두 반환하지 않은 경우, `meta` 객체에 `next_token`이 포함되어 반환됩니다. 해당 필드의 값을 가져와 다음 요청에서 `next_token` 파라미터의 값으로 사용하고, 나머지 요청 파라미터는 동일하게 유지합니다. |

적절한 `next_token` 값을 설정했으면 &quot;Send&quot; 버튼을 눌러 다음 페이지의 결과를 받습니다.
