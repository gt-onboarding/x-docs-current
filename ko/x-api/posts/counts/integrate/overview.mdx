<div id="helpful-tools">
  ### 유용한 도구
</div>

몇 가지 핵심 개념을 살펴보기 전에 이러한 엔드포인트의 기능을 테스트해 보기 위해 다음 도구나 코드 샘플 중 하나를 사용하실 것을 권장합니다.

<div id="code-samples">
  #### 코드 샘플
</div>

선호하는 프로그래밍 언어로 이 엔드포인트를 사용하는 코드를 작성해 보고 싶으신가요? 시작점으로 활용할 수 있는 다양한 코드 샘플을 [GitHub 페이지](https://github.com/xdevplatform/Twitter-API-v2-sample-code)에서 제공하며, 여기에는 [Python 클라이언트](https://github.com/xdevplatform/search-tweets-python)도 포함되어 있습니다.

<div id="libraries">
  #### 라이브러리
</div>

시작할 때 도움이 되도록 다양한 [커뮤니티 서드 파티 라이브러리](/ko/x-api/tools-and-libraries/overview)를 활용하세요. 해당 버전 태그를 확인해 v2 엔드포인트와 호환되는 라이브러리를 찾을 수 있습니다.

<div id="postman">
  #### Postman
</div>

Postman은 이러한 엔드포인트를 테스트하는 데 사용할 수 있는 매우 유용한 도구입니다. 각 Postman 요청에는 해당 엔드포인트의 모든 파라미터가 포함되어 있어, 어떤 것들을 사용할 수 있는지 빠르게 파악하는 데 도움이 됩니다. Postman 컬렉션에 대해 더 알아보려면 [Postman 사용하기](/ko/tutorials/postman-getting-started) 페이지를 방문하세요.
 

<div id="key-concepts">
  ### 핵심 개념
</div>

<div id="authentication">
  #### 인증
</div>

모든 X API v2 엔드포인트에 대한 요청은 키와 토큰이라고도 부르는 자격 증명 세트를 사용해 [인증](/ko/resources/fundamentals/authentication)되어야 합니다. 이 엔드포인트는 [OAuth 2.0 Bearer Token](/ko/resources/fundamentals/authentication#oauth-2-0)의 사용을 요구하며, 이는 요청을 성공적으로 보내려면 [Bearer Token](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)을 포함해야 함을 의미합니다. Bearer Token은 개발자 앱에서 직접 생성하거나, [POST oauth2/token](/ko/resources/fundamentals/authentication#post-oauth2-token) 엔드포인트를 사용해 생성할 수 있습니다.

<div id="developer-portal-projects-and-developer-apps">
  #### Developer portal, 프로젝트, 그리고 개발자 앱
</div>

어떤 X API v2 엔드포인트와도 연동하려면 [개발자 계정](/ko/resources/fundamentals/developer-portal)을 보유하고, 해당 계정 내에 [프로젝트](/ko/resources/fundamentals/projects)를 설정한 뒤, 그 프로젝트 안에 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성해야 합니다. 해당 개발자 앱의 키와 토큰은 최근 게시물 개수 엔드포인트에서 사용할 수 있습니다. 전체 아카이브 게시물 개수 엔드포인트를 사용하거나 고급 연산자와 더 긴 쿼리 길이를 활용하려면 Enterprise 액세스 승인을 받아야 합니다.

자세한 내용은 Enterprise 액세스 섹션을 참조하세요.

<div id="rate-limits">
  #### 요청 한도
</div>

매일 수천 명이 넘는 개발자들이 X API에 요청을 보냅니다. 이 트래픽 규모를 관리하기 위해 각 엔드포인트에는 [요청 한도](/ko/x-api/fundamentals/rate-limits)가 설정되어 있으며, 이를 통해 각 개발자가 앱 또는 인증된 사용자를 대신해 보낼 수 있는 요청 수가 제한됩니다.

이 엔드포인트에는 앱 수준(App-level)의 요청 한도가 적용됩니다. 이는 여러분이 사용하는 자격 증명으로 식별되는 어떤 앱에서든, 일정 기간 동안 이 엔드포인트에 보낼 수 있는 요청 수에 제한이 있다는 의미입니다. 

<div id="building-queries">
  #### 쿼리 구성
</div>

이 엔드포인트들의 핵심 기능은 단일 쿼리를 사용해 게시물을 필터링하여, 여러분에게 전달되는 개수 결과를 만들어내는 것입니다. 이 쿼리는 메시지 키워드, 해시태그, URL과 같은 게시물 및 사용자 속성과 일치하는 연산자들로 구성됩니다. 연산자는 불리언 논리와 괄호를 사용해 하나의 쿼리로 결합할 수 있으며, 이를 통해 쿼리의 매칭 동작을 세밀하게 조정할 수 있습니다.

[쿼리 작성 방법](/ko/x-api/posts/counts/integrate/build-a-query)에 대한 가이드를 참고해 더 자세한 내용을 확인하세요.

<div id="pagination">
  #### 페이지네이션
</div>

최근 게시물 개수의 경우, `next_token`이 반환되지 않습니다. 이는 세분성에 상관없이 하나의 API 호출로 지난 7일 동안의 게시물 개수를 모두 받게 된다는 의미입니다.

전체 아카이브 게시물 개수의 경우, 최근 30일치 데이터가 제공됩니다. 30일을 초과하는 기간의 데이터에 대해서는 `next_token`이 반환되며, 이를 사용해 페이지네이션을 통해 추가 데이터를 가져올 수 있습니다. 

<div id="building-queries-for-post-counts">
  ### 게시물 개수에 대한 쿼리 구성
</div>

게시물 개수 엔드포인트는 단일 쿼리를 포함하는 GET 요청을 받고, 해당 쿼리와 일치하는 과거 게시물 개수 데이터 집합을 반환합니다. 쿼리는 다양한 게시물 속성에 대한 일치 여부를 판단하는 연산자들로 구성됩니다.
