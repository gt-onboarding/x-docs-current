<div id="comparing-x-apis-post-counts-endpoints">
  ## X API의 게시물 개수 엔드포인트 비교
</div>

v2 게시물 개수 엔드포인트는 향후 [Enterprise Search API의 개수 엔드포인트](/ko/x-api/enterprise-gnip-2.0/fundamentals/search-api#counts-requests-post-count)를 대체할 예정입니다. 기존 버전의 게시물 개수 엔드포인트를 사용하는 코드, 앱 또는 도구를 사용 중이며 이를 새로운 X API v2 엔드포인트로 마이그레이션하려는 경우, 이 가이드를 참고하시기 바랍니다.

이 페이지에는 다음과 같은 비교 내용이 포함되어 있습니다:

* [최근 게시물 개수](#recent)
* [전체 아카이브 게시물 개수](#full-archive)
* [필터링 연산자 비교](#operator)

<div id="recent-post-counts-comparison">
  ### 최근 게시물 개수 비교
</div>

Enterprise 버전의 게시물 개수 엔드포인트는 최근 30일 또는 전체 아카이브에 대한 개수를 가져올 수 있습니다. 따라서 7일 기간을 조회하는 v2 최근 게시물 개수 엔드포인트는 앞에서 언급한 어느 엔드포인트의 직접적인 대체 엔드포인트가 아닙니다.

그러나 비교에 도움이 되도록 v2 최근 게시물 개수 엔드포인트가 Enterprise 30일 엔드포인트와 어떻게 비교되는지 살펴보겠습니다.

다음 표는 다양한 유형의 최근 게시물 개수 엔드포인트를 비교합니다:

| **Description** | **Enterprise** | **X API v2** |
| :--- | :--- | :--- |
| Host domain | https://gnip-api.x.com | https://api.x.com |
| Endpoint path | /search/30day/accounts/:account&#95;name/:label/counts.json | /2/tweets/counts/recent |
| [Authentication](/ko/resources/fundamentals/authentication) | Basic 인증 | OAuth 2.0 Bearer Token |
| Timestamp format | YYYYMMDDhhmm | YYYY-MM-DDTHH:mm:ssZ  <br />[ISO 8601 / RFC 3339](https://tools.ietf.org/html/rfc3339#section-5.6) |
| 반환되는 게시물이 가질 수 있는 최대 경과 시간 | 31일 | 7일 |
| 지원 HTTP 메서드 | GET | GET |
| 기본 요청 한도 | 1초당 20개 요청(검색 데이터와 개수 요청 전체에 대해 합산)  <br />분당 요청 한도는 계약서에 명시된 대로 파트너별로 달라집니다. | 사용자 기준 15분당 180개 요청  <br />앱 기준 15분당 450개 요청 |
| [annotations](/ko/x-api/fundamentals/post-annotations)을 사용한 필터링 지원 |     | ✔   |
| [conversation&#95;id](/ko/x-api/fundamentals/conversation-id)를 사용한 필터링 지원 |     | ✔   |
| 게시물 데이터 배열에 대한 JSON key 이름 | results | data |
| 시간 단위(해상도) | 일, 시, 또는 분 | 일, 시, 또는 분 |
| Timezone | UTC | UTC |
| 기간 선택을 위한 요청 매개변수 | fromDate  <br />toDate | start&#95;time  <br />end&#95;time |
| 게시물 ID 기준으로 탐색하기 위한 요청 매개변수 |     | since&#95;id  <br />until&#95;id |
| [project](/ko/resources/fundamentals/projects)에 연결된 [developer App](/ko/resources/fundamentals/developer-apps)의 자격 증명 사용 필요 |     | ✔   |

다음 표에서는 다양한 전체 아카이브 검색 엔드포인트를 비교합니다:

| 설명 | Enterprise | X API v2 |
| :--- | :--- | :--- |
| 호스트 도메인 | https://gnip-api.x.com | https://api.x.com |
| 엔드포인트 경로 | /search/fullarchive/accounts/:account&#95;name/:label/counts | /2/tweets/counts/all |
| [Authentication](/ko/resources/fundamentals/authentication) | 기본 인증 | OAuth 2.0 Bearer Token |
| 타임스탬프 형식 | YYYYMMDDHHMM | YYYY-MM-DDTHH:mm:ssZ  <br />[ISO 8601 / RFC 3339](https://tools.ietf.org/html/rfc3339#section-5.6) |
| 반환되는 게시물 개수에 포함되는 가장 오래된 시점 | 2006년 3월부터의 전체 아카이브 | 2006년 3월부터의 전체 아카이브 |
| 지원되는 HTTP 메서드 | GET  <br />POST | GET |
| 기본 요청 한도 | 분당 요청 한도는 계약서에 명시된 대로 파트너별로 달라집니다.  <br />초당 20회 요청 | 앱당 15분당 300회 요청  <br />앱당 초당 1회 요청 |
| 세분성 | 일, 시, 분 | 일, 시, 분 |
| [annotations](/ko/x-api/fundamentals/post-annotations)를 사용한 필터링 지원 |     | ✔   |
| [conversation&#95;id](/ko/x-api/fundamentals/conversation-id)를 사용한 필터링 지원 |     | ✔   |
| 게시물 데이터 배열에 대한 JSON 키 이름 | results | data |
| 기간 선택을 위한 요청 파라미터 | fromDate  <br />toDate | start&#95;time  <br />end&#95;time |
| 게시물 ID로 탐색하기 위한 요청 파라미터 |     | since&#95;id  <br />until&#95;id |
| 페이지네이션을 위한 JSON 키 이름 | next | meta.next&#95;token |
| 페이지네이션을 위한 요청 파라미터 | next&#95;token | next&#95;token 또는 pagination&#95;token |
| 타임존 | UTC | UTC |
| [Academic Research access](/ko/x-api/getting-started/about-x-api)가 부여된 [Project](/ko/resources/fundamentals/projects)에 연결된 [developer App](/ko/resources/fundamentals/developer-apps)의 자격 증명 필요 여부 |     | ✔   |

<div id="filtering-operator-comparison">
  ### 필터링 연산자 비교
</div>

두 가지 서로 다른 버전(Enterprise와 v2)의 게시물 개수 엔드포인트는 사용 가능한 연산자가 서로 다르며, 각 버전 내에서도 연산자 가용 수준이 서로 다릅니다. 이에 대해서는 아래에서 설명합니다.

Enterprise

* Enterprise 연산자에는 하위 티어가 없습니다. 모든 Enterprise 연산자는 모든 Enterprise 사용자가 사용할 수 있습니다.

X API v2

* **코어(Core):** 이 연산자는 모든 X API v2 사용자가 사용할 수 있습니다.
* **고급(Advanced):** 이 연산자는 Academic Research 액세스가 승인된 사용자만 사용할 수 있습니다.

각 연산자 집합에 대한 더 자세한 내용은 다음 가이드를 참고하세요.

* [Enterprise operators](/ko/x-api/enterprise-gnip-2.0/fundamentals/search-api#building-search-queries)
* [X API v2 operators](/ko/x-api/posts/search/integrate/build-a-query)
