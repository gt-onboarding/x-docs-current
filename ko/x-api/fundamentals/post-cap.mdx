X API v2에는 특정 엔드포인트에서 월 단위로 조회할 수 있는 게시물 수를 제한하는 게시물 사용 한도(Post consumption cap)가 있습니다. 게시물 상한은 [프로젝트](/ko/resources/fundamentals/projects) 단위로 적용되며, 해당 프로젝트에 속한 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용해 아래에 나열된 엔드포인트에 요청을 보내면, 그 모든 요청이 해당 프로젝트의 게시물 상한에 포함됩니다.

<div id="limited-v2-endpoints">
  ## 제한된 v2 엔드포인트
</div>

아래 엔드포인트에서 수신한 게시물은 모두 월간 게시물 한도에 포함됩니다:

* [Posts Lookup](/ko/x-api/posts/lookup/introduction)
* [Recent search](/ko/x-api/posts/counts/introduction)
* [Full-archive search](/ko/x-api/posts/counts/introduction)
* [Filtered stream](/ko/x-api/posts/filtered-stream)
* [Filtered Stream Webhooks](/ko/x-api/webhooks/stream/introduction)
* [User Post timeline](/ko/x-api/posts/timelines)
* [User mention timeline](/ko/x-api/posts/timelines)
* [Likes lookup - Posts liked by a user](/ko/x-api/posts/likes)
* [Bookmarks lookup - Posts bookmarked by a user](/ko/x-api/posts/bookmarks/introduction)
* [List Posts lookup](/ko/x-api/lists/list-tweets/introduction)
* [Spaces Lookup](/ko/x-api/spaces/lookup/introduction)

<Note>
  **참고:** 동일한 게시물이 하루 동안 여러 쿼리에서 반환되더라도 게시물 한도에는 하루에 한 번만 계산됩니다. 게시물은 일 단위로 중복이 제거됩니다.
</Note>

<div id="post-cap-volumes">
  ## 게시물 상한량
</div>

게시물 상한량은 액세스 티어에 따라 달라집니다:

| Access Tier   | Monthly Post Cap         |
|:---------------|:--------------------------|
| **Free**      | 월 100개 게시물           |
| **Basic**     | 월 15,000개 게시물        |
| **Pro**       | 월 1,000,000개 게시물     |
| **Enterprise**| 월 5,000만 개 이상 게시물 |

[developer portal](/ko/resources/fundamentals/developer-portal)의 [main dashboard page](https://developer.x.com/content/developer-twitter/en/portal/dashboard)에서 월별 게시물 상한 대비 사용량을 확인할 수 있습니다. 프로젝트 이름 아래의 상태 표시줄에서 현재 월의 사용량이 게시물 상한과의 비율로 표시되며, 여기에는 가져온 게시물 총수, 사용 비율(%) 및 사용량이 초기화되는 날짜가 포함됩니다.
