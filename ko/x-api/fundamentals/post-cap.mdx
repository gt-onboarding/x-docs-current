X API v2에는 특정 엔드포인트에서 월별로 조회할 수 있는 게시물 수를 제한하는 게시물 소비 상한(Post consumption cap)이 있습니다. 게시물 상한은 [Project](/ko/resources/fundamentals/projects) 수준에서 적용되며, 해당 Project에 속한 [developer Apps](/ko/resources/fundamentals/developer-apps)에서 발급된 키와 토큰을 사용해 아래에 나열된 엔드포인트로 보내는 모든 요청은 Project 게시물 상한에서 차감됩니다.

<div id="limited-v2-endpoints">
  ## 제한된 v2 엔드포인트
</div>

* [전체 아카이브 검색](/ko/x-api/posts/counts/introduction)
* [Filtered stream](/ko/x-api/posts/filtered-stream)
* [Filtered Stream Webhooks](/ko/x-api/webhooks/stream/introduction)
* [사용자 게시물 타임라인](/ko/x-api/posts/timelines)
* [사용자 멘션 타임라인](/ko/x-api/posts/timelines)
* [좋아요 조회 - 사용자가 좋아한 게시물](/ko/x-api/posts/likes)
* [북마크 조회 - 사용자가 북마크한 게시물](/ko/x-api/posts/bookmarks/introduction)
* [리스트 게시물 조회](/ko/x-api/lists/list-tweets/introduction)

<Note>
  **참고:** 동일한 게시물이 하루 동안 여러 쿼리에서 반환되더라도 게시물 상한에는 하루에 한 번만 반영됩니다. 게시물은 매일 중복이 제거됩니다.
</Note>

<div id="post-cap-volumes">
  ## 게시물 상한량
</div>

게시물 상한량은 액세스 수준에 따라 달라집니다:

| 액세스 티어   | 월별 게시물 상한량                |
|:---------------|:---------------------------------|
| **Free**      | 월 100개 게시물           |
| **Basic**     | 월 15,000개 게시물          |
| **Pro**       | 월 1,000,000개 게시물       |
| **Enterprise**| 월 5,000만 개 이상 게시물     |

월별 게시물 상한 대비 사용량은 [developer portal](/ko/resources/fundamentals/developer-portal)의 [메인 대시보드 페이지](https://developer.x.com/content/developer-twitter/en/portal/dashboard)에서 확인할 수 있습니다. 프로젝트 이름 아래에 현재 월 사용량과 게시물 상한의 관계를 보여주는 상태 표시줄이 표시되며, 여기에는 불러온 게시물 총량, 사용된 게시물 비율, 사용량이 초기화되는 날짜가 포함됩니다.
