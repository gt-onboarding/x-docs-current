<div id="tweet">
  ### 트윗
</div>

트윗은 Twitter의 모든 기능을 구성하는 기본 단위입니다. Tweet 오브젝트에는 `id`, `text`, `created_at`과 같은 ‘루트 레벨’ 필드가 다수 포함되어 있습니다. Tweet 오브젝트는 `user`, `media`, `poll`, `place` 등의 여러 하위 오브젝트를 포함하는 ‘부모’ 오브젝트이기도 합니다. Tweet 오브젝트의 이러한 루트 레벨 필드를 요청할 때는 필드 파라미터 `tweet.fields`를 사용하세요.

Tweet 오브젝트는 user 리소스에서 조회하고 expansions 파라미터를 사용해 확장할 수 있습니다. 요청한 트윗과 관련된 추가 트윗은 Tweet 리소스에서도 조회 및 확장할 수 있습니다. 이 오브젝트는 user 리소스에서 `?expansions=pinned_tweet_id`, Tweet 리소스에서 `?expansions=referenced_tweets.id`를 사용해 기본 필드만 포함된 상태로 확장에 사용할 수 있습니다. 오브젝트를 완전하게 가져오기 위해 추가 필드를 요청하려면, 필드 파라미터 `tweet.fields`와 함께 expansions를 사용하세요.

| 필드 값                                             | 유형           | 설명                                                                                                                    | 활용 방법                                                 |
| :----------------------------------------------- | :----------- | :-------------------------------------------------------------------------------------------------------------------- | :---------------------------------------------------- |
| **id (기본)**                                      | string       | 요청된 트윗을 나타내는 고유 식별자입니다.                                                                                               | 특정 트윗을 프로그램을 통해 조회할 때 사용합니다.                          |
| **text (기본값)**                                   | string       | 트윗의 실제 UTF-8 텍스트입니다. 사용 가능한 문자에 대한 자세한 내용은 [twitter-text](https://github.com/twitter/twitter-text/)를 참조하세요.           | 키워드 추출 및 감성 분석/분류.                                    |
| **edit&#95;history&#95;tweet&#95;ids (default)** | object       | 트윗의 모든 버전을 가리키는 고유 ID입니다. 수정 이력이 없는 트윗에는 ID가 하나만 있고, 수정 이력이 있는 트윗에는 여러 개의 ID가 있습니다.                                   | 이 정보를 사용해 트윗의 편집 이력을 조회할 수 있습니다.                      |
| **article**                                      | object       | 이 트윗에 포함된 Article의 메타데이터를 담고 있습니다.                                                                                    | 이를 사용하여 Article의 텍스트와 엔터티를 가져올 수 있습니다.                |
| **attachments**                                  | 객체           | 이 트윗에 포함된 첨부 파일의 유형(있는 경우)을 지정합니다.                                                                                    | 요청한 expansions에 대해 반환된 객체를 이해하는 데 사용합니다.              |
| **author&#95;id**                                | 문자열          | 해당 트윗을 게시한 User의 고유 식별자입니다.                                                                                           | User 객체를 하이드레이션할 때, 동료 검토용으로 데이터셋을 공유할 때 사용.          |
| **card&#95;uri**                                 | 문자열          | 이 트윗에 포함된 Card의 URI입니다.                                                                                               |                                                       |
| **community&#95;id**                             | string       | 이 게시물이 속한 커뮤니티의 고유 식별자입니다.                                                                                            |                                                       |
| **context&#95;annotations**                      | 배열           | 트윗의 컨텍스트 주석을 포함합니다.                                                                                                   | 개체 인식/추출, 토픽 분석.                                      |
| **conversation&#95;id**                          | 문자열          | 직접 답글과 답글의 답글까지 포함하는 대화에서 원본 트윗의 트윗 ID.                                                                               | 이 값을 사용하여 해당 트윗으로부터 대화 스레드를 재구성할 수 있습니다.              |
| **created&#95;at**                               | 날짜(ISO 8601) | 트윗이 생성된 시각입니다.                                                                                                        | 시계열 분석을 수행하고 트윗이 언제 생성되었는지 파악하는 데 유용합니다.              |
| **display&#95;text&#95;range**                   | 배열           | 표시되는 텍스트 구간의 시작 인덱스와 끝 인덱스를 포함하는 배열입니다.                                                                               | 긴 게시물에서 기본으로 표시되는 텍스트 범위를 파악하는 데 유용합니다.               |
| **edit&#95;controls**                            | object       | 트윗을 앞으로 얼마나 더 수정할 수 있는지와 남은 수정 횟수를 나타냅니다.                                                                             | 이를 사용하여 트윗이 편집 가능한지 여부를 판단합니다.                        |
| **entities**                                     | 객체           | 트윗 텍스트에서 추출된 엔티티입니다. 자세한 내용은 Twitter Objects의 entities 항목을 참고하세요.                                                     | 해시태그, URL, 멘션 등에 대한 부가 정보를 제공합니다.                     |
| **geo**                                          | 객체           | 지오태그가 지정된 트윗의 위치 또는 장소를 나타냅니다.                                                                                        | 이 필드를 사용하여 지오태그된 트윗의 위치를 확인합니다.                       |
| **in&#95;reply&#95;to&#95;user&#95;id**          | string       | 해당 트윗이 답글인 경우, 이 필드에는 원본 트윗 작성자의 id가 포함됩니다.                                                                           | 트윗이 다른 트윗에 대한 답글인지 확인합니다.                             |
| **lang**                                         | 문자열          | Twitter에서 감지한 경우 제공되는 트윗의 언어입니다.                                                                                      | 트윗을 사용 언어별로 분류합니다.                                    |
| **non&#95;public&#95;metrics**                   | object       | 요청 시점 기준 트윗의 비공개 참여 메트릭입니다. 사용자 컨텍스트 인증이 필요합니다.                                                                       | 트윗에 대해 발생한 총 노출 수를 확인합니다.                             |
| **note&#95;tweet**                               | 객체           | 장문 게시물(280자 초과)의 전체 텍스트를 포함합니다.                                                                                       | 게시물 전체 텍스트를 가져옵니다.                                    |
| **organic&#95;metrics**                          | object       | 요청 시점의 트윗에 대해 오가닉 컨텍스트에서 추적된 참여 메트릭입니다. 사용자 컨텍스트 인증이 필요합니다.                                                           | 트윗의 오가닉 참여도를 측정합니다.                                   |
| **possibly&#95;sensitive**                       | boolean      | 콘텐츠가 민감한 콘텐츠로 인식될 수 있는지 여부를 나타냅니다.                                                                                    | 특정 유형의 콘텐츠 확산을 분석합니다.                                 |
| **promoted&#95;metrics**                         | object       | 요청 시점 기준 트윗에 대해 프로모션(광고) 환경에서 추적되는 참여 메트릭입니다. 사용자 컨텍스트를 사용하는 인증이 필요합니다.                                               | 트윗이 프로모션된 기간의 참여도를 측정합니다.                             |
| **public&#95;metrics**                           | 객체           | 요청 시점 기준 트윗의 공개 참여 메트릭입니다.                                                                                            | 트윗 참여를 측정합니다.                                         |
| **referenced&#95;tweets**                        | array        | 이 트윗이 참조하는 트윗들의 목록으로, 예를 들어 리트윗, 인용 트윗 또는 답글 등이 이에 해당합니다.                                                             | 리트윗 등과 관련된 대화 맥락을 파악합니다.                              |
| **reply&#95;settings**                           | 문자열          | 특정 트윗에 누가 답글을 남길 수 있는지 보여줍니다. 가능한 값은 &quot;everyone&quot;, &quot;mentioned&#95;users&quot;, &quot;followers&quot;입니다. | 해당 트윗에 대한 답글 허용 대상을 파악합니다.                            |
| **withheld**                                     | object       | [보류된 콘텐츠](https://help.x.com/en/rules-and-policies/tweet-withheld-by-country)에 대한 보류 관련 세부 정보를 포함합니다.                 |                                                       |
| **scopes**                                       | 객체           | 트윗의 공개 범위 정보가 포함됩니다.                                                                                                  | 어떤 사용자가 이 게시물을 볼 수 있는지 나타냅니다. 프로모션 게시물에 대해서만 반환됩니다.   |
| **media&#95;metadata**                           | 배열           | 트윗에 첨부된 미디어의 메타데이터를 포함합니다.                                                                                            | 트윗의 미디어 첨부파일에 대한 `alt_text`와 같은 추가 메타데이터를 가져올 수 있습니다. |

**트윗 객체 가져오기**

**샘플 요청**

다음 요청에서는 [Tweets lookup](/ko/resources/fundamentals/rate-limits) 엔드포인트에서 해당 트윗의 필드를 요청합니다. `$BEARER_TOKEN`을(를) 반드시 직접 생성한 [Bearer Token](/ko/resources/fundamentals/authentication)으로 교체하세요.

```bash
curl --request GET 'https://api.x.com/2/tweets?ids=1212092628029698048&tweet.fields=attachments,author_id,context_annotations,created_at,entities,geo,id,in_reply_to_user_id,lang,possibly_sensitive,public_metrics,referenced_tweets,text,withheld&expansions=referenced_tweets.id' --header 'Authorization: Bearer $BEARER_TOKEN'
```

  ### Space
</div>

Space는 라이브 오디오 대화를 통해 사용자가 표현하고 상호작용할 수 있게 합니다. Space 데이터 사전에는 Space와 관련된 메타데이터가 포함되며, 모든 세부 정보는 실시간으로 업데이트됩니다.

User 오브젝트는 user 리소스에서 조회하고 확장(expansions)할 수 있습니다. 이 오브젝트들은 `expansions` 쿼리 매개변수에 `host_ids`, `creator_id`, `speaker_ids`, `mentioned_user_ids` 중 최소 하나를 추가하여 확장 대상으로 사용할 수 있습니다.

트윗과 달리 Space는 일시적인 객체로, 종료되거나 생성자가 취소하면 더 이상 사용할 수 없습니다. 앱에서 Space 데이터를 처리할 때에는 항상 최신 정보를 반환해야 하며, 플랫폼에서 더 이상 제공되지 않는 데이터는 반드시 제거해야 합니다. [Spaces lookup endpoints](/ko/x-api/spaces/lookup/introduction)를 사용하면 이용자의 기대와 의도를 준수하고 있는지 확인하는 데 도움이 됩니다.

  ### 리스트
</div>

List 객체에는 참조된 리스트를 설명하는 [Twitter Lists](https://help.x.com/en/using-twitter/twitter-lists) 메타데이터가 포함됩니다. List 객체는 List lookup 엔드포인트에서 반환되는 기본 객체입니다. 이 엔드포인트에서 추가 List 필드를 요청하려면 `list.fields` 필드 파라미터를 사용하면 됩니다.

List 객체는 다른 데이터 객체의 하위로 포함되지 않습니다. 그러나 User 객체는 user 리소스에서 찾을 수 있으며 확장할 수 있습니다. 이 객체들은 `expansions` 쿼리 파라미터에 `owner_id`를 추가하여 확장할 수 있습니다. 기본 List 객체를 완성하기 위해 추가 필드를 요청할 때는 `list.fields` 필드 파라미터를, 확장 객체를 완성하기 위해서는 `user.fields`를 사용하십시오.

| Field Value        | Type            | Description                                                                                                                            | How it can be used                                       |
| :----------------- | :-------------- | :------------------------------------------------------------------------------------------------------------------------------------- | :------------------------------------------------------- |
| id (default)       | string          | 이 리스트의 고유 식별자입니다.<br />`"id": "2244994945"`                                                                                            | 이를 사용해 특정 리스트에 대한 정보를 프로그래매틱하게 조회할 수 있습니다.               |
| name (default)     | string          | 리스트를 생성할 때 정의한 리스트의 이름입니다.<br />`"name": "Twitter Lists"`                                                                              |                                                          |
| created&#95;at     | date (ISO 8601) | 리스트가 생성된 UTC 날짜 및 시간입니다.<br />`"created_at": "2013-12-14T04:35:55.000Z"`                                                               | 리스트가 Twitter에 생성된 지 얼마나 되었는지 파악할 수 있습니다.                 |
| description        | string          | 사용자에게 리스트에 대해 알려 주기 위한 간단한 설명입니다.<br />`"description": "People that are active members of the Bay area cycling community on Twitter."` |                                                          |
| follower&#95;count | integer         | 이 리스트를 팔로우하는 사용자의 수를 표시합니다.<br />`"follower_count": 198`                                                                               |                                                          |

| owner&#95;id       | string          | 이 리스트 소유자의 고유 식별자입니다.<br />`"owner_id": "1255542774432063488"`                                                                         | 이 사용자가 다른 리스트도 소유하고 있는지 확인하고 User 객체를 확장하는 데 사용할 수 있습니다. |

트윗에 포함된 투표는 어떤 엔드포인트에서도 기본(primary) 객체로 취급되지는 않지만, 트윗 객체 안에서 찾을 수 있으며 expansions를 통해 확장할 수 있습니다. 이 객체는 기본 필드만 포함한 요약 객체를 가져오기 위해 `?expansions=attachments.poll_ids`로 확장할 수 있습니다. 객체를 완성하기 위해 추가 필드를 요청하려면 필드 매개변수 `poll.fields`와 함께 expansions를 사용하세요.

| Field value          | Type            | Description                                                                                                                       |
| :------------------- | :-------------- | :-------------------------------------------------------------------------------------------------------------------------------- |
| id (default)         | string          | 확장된 투표의 고유 식별자입니다.                                                                                                                |
|                      |                 | `{"id": "1199786642791452673"}`                                                                                                   |
| options (default)    | array           | 참조된 투표에서 각 선택지를 설명하는 객체들을 포함합니다.                                                                                                  |
|                      |                 | `{"options": [ { "position": 1, "label": "“C Sharp”", "votes": 795 }, { "position": 2, "label": "“C Hashtag”", "votes": 156 } ]}` |
| duration&#95;minutes | integer         | 이 투표의 총 지속 시간을 지정합니다.                                                                                                             |
|                      |                 | `{"duration_minutes": 1440}`                                                                                                      |
| end&#95;datetime     | date (ISO 8601) | 이 투표가 종료되는 날짜와 시간을 지정합니다.                                                                                                         |
|                      |                 | `{"end_datetime": "2019-11-28T20:26:41.000Z"}`                                                                                    |
| voting&#95;status    | string          | 이 투표가 아직 활성 상태여서 투표를 받을 수 있는지, 아니면 투표가 종료되었는지를 나타냅니다.                                                                             |
|                      |                 | `{"voting_status": "closed"}`                                                                                                     |

**투표 객체 가져오기**

**샘플 요청**

다음 요청에서는 [Tweets lookup](/ko/x-api/posts/lookup/introduction) 엔드포인트에서 트윗에 첨부된 투표 객체의 필드를 요청합니다. 투표는 트윗의 하위(child) 객체이므로 `attachments.poll_id` expansion이 필요합니다. `$BEARER_TOKEN`을 직접 생성한 [Bearer Token](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)으로 반드시 교체하세요.


<div id="direct-message-events">
  ### 다이렉트 메시지 이벤트
</div>

다이렉트 메시지(DM) 대화는 여러 개의 이벤트로 구성됩니다. X API v2는 현재 세 가지 이벤트 유형을 지원합니다: MessageCreate, ParticipantsJoin, ParticipantsLeave.

DM 이벤트 객체는 [Direct Message lookup](/ko/x-api/direct-messages/lookup/introduction) 엔드포인트에서 반환되며, [Manage Direct Messages](/ko/x-api/direct-messages/lookup/introduction) 엔드포인트를 통해 다이렉트 메시지가 성공적으로 생성되면 MessageCreate 이벤트가 생성됩니다.

DM 이벤트를 요청하면 세 가지 기본 이벤트 객체 속성(또는 필드)이 포함됩니다: id, event&#95;type, text. 추가 이벤트 필드를 받으려면 [fields](/ko/x-api/fundamentals/fields) 매개변수인 dm&#95;event.fields를 사용해 다른 필드를 선택하십시오. 사용 가능한 기타 이벤트 필드는 다음과 같습니다: dm&#95;conversation&#95;id, created&#95;at, sender&#95;id, attachments, participant&#95;ids, referenced&#95;tweets.

이 필드 중 일부는 다이렉트 메시지 이벤트와 관련된 다른 X 객체의 ID를 제공합니다.

* sender&#95;id - 메시지를 보낸 계정 또는 그룹 대화에 참가자를 초대한 계정의 ID
* partricipants&#95;ids - 계정 ID 배열입니다. ParticipantsJoin 및 ParticipantsLeave 이벤트의 경우 이 배열에는 이벤트를 생성한 계정의 단일 ID가 포함됩니다.
* attachments - 발신자가 X에 업로드한 콘텐츠에 대한 미디어 ID를 제공합니다.
* referenced&#95;tweets - text 필드에서 트윗 URL이 발견된 경우, 해당 트윗의 ID가 응답에 포함됩니다.

sender&#95;id, participant&#95;ids, referenced&#95;tweets.id, attachments.media&#95;keys [expansions](/ko/x-api/fundamentals/expansions)를 사용해 이러한 X 객체 ID에 해당하는 전체 객체를 확장해 받을 수 있습니다.

|     |     |     |     |
| :--- | :--- | :--- | :--- |
| **Field value** | **Type** | **Description** | **How it can be used** |
| id (default) | string | 이벤트의 고유 식별자입니다.<br /><br />&quot;id&quot;: &quot;1050118621198921728&quot; | 이를 사용해 프로그램을 통해 특정 대화 이벤트를 조회할 수 있습니다(v1.1 엔드포인트에서 사용 가능). |
| event&#95;type (default) | string | 이벤트 유형을 설명합니다. 현재 지원되는 유형은 세 가지입니다:<br /><br />* MessageCreate<br />    <br />* ParticipantsJoin<br />    <br />* ParticipantsLeave<br />    <br /><br />&quot;event&#95;type&quot;: &quot;MessageCreate&quot; | 대화 기록을 조회할 때 메시지가 언제 생성되었는지, 그리고 그룹 대화의 경우에는 참가자가 언제 참여하고 떠났는지 이해하는 데 사용됩니다. 모든 GET 메서드는 event&#95;type= 쿼리 파라미터를 사용해 특정 이벤트 유형으로 필터링을 지원합니다. |
| text (default) | string | 다이렉트 메시지의 실제 UTF-8 텍스트입니다.<br /><br />&quot;text&quot;: &quot;안녕, 지금은 너만!&quot; | 챗봇에서는 이를 사용해 메시지 내용을 분석하고 자동 응답을 결정할 수 있습니다. 대화 검색 기능을 구축하는 데 사용할 수도 있습니다. |
| entities          | object | DM 텍스트에서 파싱된 엔티티입니다.                                       | 해시태그, URL, 멘션 등에 대한 추가 정보를 제공합니다. |

| dm&#95;conversation&#95;id | string | 이벤트가 속해 있는 대화의 고유 식별자입니다.<br /><br />&quot;dm&#95;conversation&#95;id&quot;: &quot;1584988213961031680&quot; | 이를 사용해 프로그램을 통해 특정 대화의 이벤트를 조회하고, 해당 대화에 다이렉트 메시지를 추가할 수 있습니다. |

**다이렉트 메시지 이벤트 객체 가져오기**

**샘플 요청**

이 예제에서는 1:1 대화와 관련된 이벤트를 가져오는 요청을 만듭니다. 이 요청은 기본적인 다이렉트 메시지 이벤트 필드와 함께, 참조된 트윗 및 해당 작성자에 대한 추가 필드를 반환합니다. 다음을 요청하는 쿼리를 만들어 보겠습니다:

* 언제 생성되었는지, 어떤 대화(dm&#95;conversation)의 일부인지와 같은 기본 이벤트 속성
* 다이렉트 메시지를 보낸 사용자의 계정 ID와 설명
* 참조된 트윗의 텍스트와 게시 시각
* 참조된 트윗 작성자의 계정 ID와 설명

이러한 속성을 반환하려면 요청 쿼리에 다음을 포함해야 합니다:

`?dm_event.fields=id,sender_id,text,created_at,dm_conversation_id&expansions=sender_id,referenced_tweets.id&tweet.fields=created_at,text,author_id&user.fields=description`

```bash
    --header 'Authorization: Bearer $BEARER_TOKEN'
```

<div id="community">
  ### Community
</div>

커뮤니티는 X 사용자가 서로 연결하고, 공유하며, 가장 관심 있는 토론에 더 깊이 다가갈 수 있도록 마련된 전용 공간입니다.

커뮤니티의 게시물은 X를 사용하는 누구나 볼 수 있지만, 해당 커뮤니티 구성원만이 상호작용하고 토론에 참여할 수 있습니다.

Community 객체에는 커뮤니티에 대한 관련 메타데이터가 포함됩니다.

| Field value      | Type            | Description                                        |    |
| :--------------- | :-------------- | :------------------------------------------------- | :- |
| created&#95;at   | date (ISO 8601) | 커뮤니티가 생성된 시각.                                      |    |
| id               | string          | 커뮤니티의 고유 식별자.                                      |    |
| name             | string          | 커뮤니티의 이름.                                          |    |
| description      | string          | 제공된 경우, 커뮤니티 설명 텍스트.                               |    |
| access           | string          | 커뮤니티의 액세스 수준. <br /><br /> 다음 중 하나일 수 있음:          |    |
|                  |                 | - `Public`                                         |    |
|                  |                 | - `Closed`                                         |    |
| join&#95;policy  | string          | 커뮤니티에 대한 가입 정책. <br /><br /> 다음 중 하나일 수 있음:        |    |
|                  |                 | - `Open`                                           |    |
|                  |                 | - `RestrictedJoinRequestsDisabled`                 |    |
|                  |                 | - `RestrictedJoinRequestsRequireAdminApproval`     |    |
|                  |                 | - `RestrictedJoinRequestsRequireModeratorApproval` |    |
|                  |                 | - `SuperFollowRequired`                            |    |
| member&#95;count | integer         | 커뮤니티에 가입한 회원 수.                                    |    |

**Community 객체 가져오기**

**샘플 요청**

다음 요청에서는 특정 키워드를 기준으로 커뮤니티 목록을 검색하면서 특정 필드만 요청합니다. `$BEARER_TOKEN`을 직접 발급받은 [Bearer Token](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)으로 반드시 교체해야 합니다.

```bash
curl --location 'https://api.x.com/2/communities/search?query=anime&amp;community.fields=access,created_at,description,id,join_policy,member_count,name' --header 'Authorization: $BEARER_TOKEN'
```

<div id="how-to-use-fields-and-expansions">
  ## fields 및 expansions 사용 방법
</div>

기본적으로 X API v2 데이터 객체는 [fields](/ko/x-api/fundamentals/fields) 또는 [expansions](/ko/x-api/fundamentals/expansions) 파라미터를 사용하지 않고 요청하면 소수의 기본 필드만 포함합니다. 이 가이드는 응답에서 추가 객체와 필드를 받기 위해 요청에 `fields` 및 `expansions` 쿼리 파라미터를 사용하는 방법을 설명합니다.

이 가이드에서는 아래 트윗 스크린샷을 기준으로 여러 필드를 요청해 보겠습니다.
 

![이 이미지는 @X 계정이 게시한 트윗의 스크린샷입니다. 트윗 텍스트, 사용자 이름, 게시 일시, 소스, 공개 메트릭을 확인할 수 있습니다. 또한 동영상도 포함되어 있습니다.](https://cdn.cms-twdigitalassets.com/content/dam/developer-twitter/ScreenShotDataDictionaryGuide.png.twimg.1920.png)

스크린샷에서 볼 수 있듯이, 트윗에는 트윗 작성자, 트윗 메트릭, 생성 타임스탬프, 동영상, 동영상 조회수 등 여러 가지 눈에 보이는 정보가 포함되어 있습니다. 이 밖에도 스크린샷에는 보이지 않지만 여전히 요청할 수 있는 여러 데이터가 존재합니다. 

API에 요청을 보낼 때 기본 응답은 단순하며, 기본 트윗 필드인 `id`와 `text`만 포함합니다. 또한 사용 중인 엔드포인트에서 반환되는 기본 객체만 받을 수 있으며, 기본 객체와 연관된 기타 연관 데이터 객체는 포함되지 않습니다.

이러한 단순한 기본 응답과 fields 및 expansions 파라미터를 함께 사용하면, 사용 사례에 따라 필요한 필드만 선택적으로 요청할 수 있습니다. 
 

<div id="requesting-additional-fields-and-objects">
  #### 추가 필드 및 객체 요청
</div>

먼저 트윗 ID와 [GET /tweets 엔드포인트](/ko/x-api/posts/lookup/introduction)를 사용하여 [트윗 객체](/ko/x-api/fundamentals/data-dictionary#tweet)를 요청합니다.

요청:

```bash
curl --request GET --url 'https://api.x.com/2/tweets?ids=1260294888811347969' \
  --header 'Authorization: Bearer $BEARER_TOKEN'
```

응답:

```
{
    "data": [
        {
            "id": "1260294888811347969",
            "text": "내 트윗에 대한 트윗을 놓치지 마세요. \n\n이제 iOS에서 댓글이 달린 리트윗을 한곳에서 모두 확인할 수 있습니다. https://t.co/oanjZfzC6y"
        }
    ]
}
```

다음 단계별 가이드에서는 스크린샷에 표시된 추가 데이터를 가져오는 방법을 안내합니다.

1. [object model](/ko/x-api/fundamentals/data-dictionary#tweet)을(를) 사용하거나 각 엔드포인트의 API 레퍼런스 페이지에 있는 필드 목록을 검토하여, 요청에 포함할 추가 필드를 식별합니다.

   이 경우 다음과 같은 추가 필드를 요청합니다:
   attachments, author&#95;id, created&#95;at, public&#95;metrics.

2. 위의 필드를 값으로 지정하여 `tweet.fields` 쿼리 파라미터를 쉼표로 구분된 목록으로 구성합니다:
   `?tweet.fields=attachments,author_id,created_at,public_metrics`

3. 이전에 보낸 GET /tweets 요청에 쿼리 파라미터를 추가합니다.

요청:

`curl --request GET --url 'https://api.x.com/2/tweets?ids=1260294888811347969&tweet.fields=attachments,author_id,created_at,public_metrics' \   --header 'Authorization: Bearer $BEARER_TOKEN'`

응답:

```
{
    "data": [
        {
            "id": "1260294888811347969",
            "text": "내 트윗에 대한 트윗을 놓치지 마세요. \n\niOS에서 이제 댓글이 포함된 리트윗을 한곳에서 모두 확인할 수 있습니다. https://t.co/oanjZfzC6y",
            "author_id": "783214",
            "public_metrics": {
                "retweet_count": 5219,
                "reply_count": 1828,
                "like_count": 17141,
                "quote_count": 3255
            },
            "attachments": {
                "media_keys": [
                    "13_1260294804770041858"
                ]
            },
            "created_at": "2020-05-12T19:44:51.000Z"

```

4. 다음으로, 트윗에 포함된 동영상과 관련된 필드를 요청합니다. 이를 위해 `expansions` 매개변수에 `attachments.media_keys` 값을 지정하여 요청에 추가합니다.

?expansions=attachments.media&#95;keys

요청:

```bash
curl --request GET --url 'https://api.x.com/2/tweets?ids=1260294888811347969&tweet.fields=attachments,author_id,created_at,public_metrics&expansions=attachments.media_keys' \
  --header 'Authorization: Bearer $BEARER_TOKEN'
```


?media.fields=public&#95;metrics,duration&#95;ms

요청:
 

`curl --request GET --url 'https://api.x.com/2/tweets?ids=1260294888811347969&tweet.fields=attachments,author_id,created_at,public_metrics&expansions=attachments.media_keys&media.fields=duration_ms,public_metrics' --header 'Authorization: Bearer $BEARER_TOKEN'`

이제 트윗 스크린샷에서 볼 수 있는 모든 데이터를 포함하는 응답은 다음과 같습니다.

총합하면, 이 예제에서 사용한 파라미터는 다음과 같습니다:

* ids=1260294888811347969
* tweet.fields=attachments,author&#95;id,created&#95;at,public&#95;metrics
* expansions=attachments.media&#95;keys
* media.fields=public&#95;metrics,duration&#95;ms
   

이들을 모두 합치면, 전체 쿼리 문자열은 다음과 같습니다:

```
?ids=1260294888811347969&tweet.fields=attachments,author\_id,created\_at,public\_metrics&expansions=attachments.media\_keys&media.fields=public\_metrics,duration\_ms
```
