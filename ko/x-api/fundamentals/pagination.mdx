<div id="introduction">
  ## 소개
</div>

페이지네이션은 단일 응답으로 반환할 수 있는 것보다 더 많은 결과를 제공하는 X API v2 엔드포인트에 있는 기능입니다. 이런 경우 데이터는 일련의 &quot;페이지&quot;로 나누어 반환됩니다. 페이지네이션은 전체 결과 데이터 세트를 가져오기 위해 모든 페이지를 프로그래밍 방식으로 요청하는 방법을 의미합니다. 모든 API 엔드포인트가 페이지네이션을 지원하거나 필요로 하는 것은 아니지만, 결과 집합이 클 때 자주 사용됩니다.

<div id="use-cases-for-pagination">
  ### 페이지네이션의 사용 사례
</div>

**요청에 대한 모든 결과를 가져오기 위해:** 페이지네이션은 특정 요청과 해당 파라미터와 관련된 모든 데이터를 가져오는 데 사용해야 합니다. 요청과 일치하는 결과가 `max_results` 값보다 많은 경우 페이지네이션이 필요합니다. 페이지네이션 토큰을 사용해 요청을 반복해서 보내면 모든 결과를 가져올 수 있습니다. 응답이 `next_token` 값 없이 반환되면, 모든 결과에 대한 페이지네이션이 완료되었다고 볼 수 있습니다. 페이지네이션은 폴링(polling) 용도로 사용해서는 안 됩니다. 이전 요청 이후의 최신 결과를 가져오려면 `since_id` 를 사용한 폴링 방식을 검토하십시오.

**요청 결과를 순회하기 위해:** 페이지네이션은 응답에 포함된 `next_token` 및 `previous_token` 값을 사용하여 요청 결과를 앞뒤로 탐색할 수 있는 옵션을 제공합니다. 이러한 토큰은 다음 요청에서 `pagination_token` 으로 설정하여 다음 또는 이전 결과 페이지로 이동하는 데 사용할 수 있습니다.

<div id="pagination-token-definitions">
  ### 페이지네이션 토큰 정의
</div>

* `next_token` - 페이지네이션을 지원하는 엔드포인트에서 meta 객체 응답 내에 반환되는 불투명한 문자열입니다. 더 많은 결과가 존재함을 나타내며, 다음 요청에서 결과의 다음 페이지를 반환하기 위해 `pagination_token` 파라미터로 사용할 수 있습니다. 마지막 결과 페이지에는 `next_token`이 포함되지 않습니다.
  * `previous_token` - 페이지네이션을 지원하는 엔드포인트에서 meta 객체 응답 내에 반환되는 불투명한 문자열입니다. 이전 결과 페이지가 존재함을 나타내며, 다음 요청에서 결과의 이전 페이지를 반환하기 위해 `pagination_token` 파라미터로 설정할 수 있습니다.
  * `pagination_token` - 페이지네이션 요청에 사용되는 파라미터입니다. 결과의 다음 페이지를 가져오려면 `next_token` 값으로 설정합니다. 결과의 이전 페이지를 가져오려면 `previous_token` 값으로 설정합니다.

<div id="fundamentals-of-pagination">
  ### 페이지네이션의 기초
</div>

* 페이지네이션을 사용하는 엔드포인트는 초기 요청에 대해 첫 번째 페이지의 결과를 반환하고, 추가 결과 페이지가 있는 경우 JSON 응답의 meta 객체 안에 `next_token`을 제공합니다. 모든 결과를 받으려면 응답에 더 이상 `next_token`이 포함되지 않을 때까지 이 과정을 반복합니다.
  * 결과는 최신순(시간 내림차순)으로 제공됩니다. 이는 개별 페이지 내에서도, 여러 페이지 전체에서도 동일하게 적용됩니다.
    * 첫 번째 응답의 첫 번째 게시물은 가장 최근 게시물입니다.
    * 마지막 응답의 마지막 게시물은 가장 오래된 게시물입니다.
  * `max_results` 요청 파라미터를 사용하면 각 응답 페이지마다 반환되는 게시물 수를 설정할 수 있습니다. `max_results`에는 기본값과 최대값이 있습니다.
  * 모든 페이지네이션 구현에는 응답 페이로드에서 토큰을 파싱하고, 이후 요청에서 이를 사용하는 과정이 포함됩니다.
  * 경우에 따라 한 페이지당 `max_results`보다 적은 결과를 받을 수 있습니다. 페이지당 결과 수가 항상 `max_results` 파라미터 값과 같을 것이라고 가정하지 마십시오.
  * 전체 결과를 얻기 위해 페이지네이션을 사용하는 가장 좋은 방법은 통합 코드에서 루프 로직을 사용하거나, X API v2를 지원하는 [라이브러리](/ko/x-api/tools-and-libraries/overview)를 사용하는 것입니다.

<div id="pagination-example">
  ### 페이지네이션 예시
</div>

여기서는 `max_results`가 `100`으로 설정되어 있고, 2019년 1월 1일 17:00:00 UTC부터 2019년 12월 12일 00:00:00 UTC 사이에 사용자 ID `2244994945`(@XDevelopers)가 생성한 게시물이 약 295개이기 때문에 결과는 세 페이지로 구성됩니다. 첫 번째 페이지의 첫 번째 게시물(`1337498609819021312`)이 가장 최근 게시물이고, 세 번째 페이지의 마지막 게시물(`1082718487011885056`)이 가장 오래된 게시물입니다.

<div id="initial-request">
  #### 최초 요청
</div>

```bash
      https://api.x.com/2/users/2244994945/tweets?tweet.fields=created_at&max_results=100&start_time=2019-01-01T17:00:00Z&end_time=2020-12-12T01:00:00Z
```
