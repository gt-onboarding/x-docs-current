<p>X API v2와 함께, 개발자가 X의 퍼블릭 API에 어떤 변경이 언제 발생하는지, 그리고 언제 새로운 버전으로 마이그레이션해야 하는지를 더 잘 이해할 수 있도록 하는 새로운 버전 관리 전략을 도입했습니다. </p>
<p>개발자는 개발 로드맵에 이러한 변경 사항을 적절히 반영해 계획할 수 있도록, 당사의 <a href="https://developer.x.com/en/updates/stay-informed">커뮤니케이션 채널</a>을 통해 X API의 사용 중단, 서비스 종료, 변경, 추가 사항 등에 대한 알림을 받게 됩니다. API의 모든 변경 사항은 변경 로그에 기록됩니다.</p>
<p>현재 X API에는 세 가지 버전이 있습니다. 여러분의 사용 사례에 필요한 기능이 아직 v2에 제공되지 않은 경우를 제외하고, 통합을 계획할 때 X API v2를 사용하시기를 강력히 권장합니다. </p>
<p>각 버전에 대해 자세히 알아보려면 다음 페이지를 방문하세요.</p>

</ul>

<div id="versioning-strategy">
  ## 버전 관리 전략
</div>

<p>X API의 버전 관리는 엔드포인트 라우트 경로에 포함된 버전 번호로 표시됩니다:</p>
<p>`https://api.x.com/2/tweets`</p>
<p>우리는 필요할 때 X API의 주요 버전(메이저 버전)을 출시하되, 12개월보다 더 짧은 주기로는 출시하지 않는 것을 목표로 합니다. 주요 버전은 API에 호환성을 깨는 변경 사항이 도입될 때 출시됩니다. 새로운 주요 버전을 출시할 때는 개발자가 새 버전으로 마이그레이션하는 데 도움이 되도록 마이그레이션 가이드를 제공합니다. </p>
<p>호환성을 깨는 변경은 개발자가 자신들의 앱에서 기존 기능을 유지하기 위해 코드를 수정해야 하는 변경을 의미합니다. 호환성을 깨지 않는 변경은 추가되는 형태로 이루어지며, 준비가 되는 대로 최신 버전에 배포됩니다. 이 경우, 새로운 기능을 활용하고자 하지 않는 한 개발자가 별도로 작업할 필요는 없습니다.</p>
<p>보안 또는 프라이버시와 같은 이유로 주기 중간에 호환성을 깨는 변경을 반드시 배포해야 하는 경우, 이 변경은 최신 버전에 적용됩니다.</p>

<div id="breaking-changes">
  ### 호환성 파괴 변경 사항
</div>

<p>이러한 변경이 이루어지면 개발자는 애플리케이션의 기존 기능을 유지하기 위해 코드를 수정해야 합니다.</p>

<ul>
  <li>새 필수 파라미터 추가</li>
  <li>기존 엔드포인트 제거</li>
  <li>응답에서 어떤 필드든 제거(필수 또는 선택 필드 모두 포함)</li>
  <li>쿼리 파라미터 제거</li>
  <li>입력 또는 출력 형식의 구조 변경(예: 최상위 필드를 하위 필드로 변경하거나, 오류 위치를 인라인으로 바꾸는 경우)</li>
  <li>기존 입력 파라미터나 출력 값의 이름 또는 데이터 유형 변경</li>
  <li>필드 이름 변경</li>
  <li>리소스 이름 변경</li>
  <li>응답 코드 변경</li>
  <li>오류 유형 변경</li>
  <li>기존 권한 스코프 변경</li>
</ul>

<div id="non-breaking-changes">
  ### 비호환성을 일으키지 않는 변경 사항
</div>

<ul>
  <li>새 엔드포인트 추가</li>
  <li>새 선택적 매개변수 추가</li>
  <li>새 응답 필드 추가</li>
  <li>오류 메시지 텍스트 변경</li>
  <li>새 스코프 제공</li>
  <li>필드 &quot;널링&quot;(필드를 완전히 제거하는 대신 개인정보 보호/보안상의 이유로 필드 값을 null로 설정하는 것)</li>
</ul>

<div id="deprecation-and-retirement">
  ### 서비스 중단 및 폐기
</div>

<p>먼저, X API에서 서비스 중단(Deprecation)과 폐기가 무엇을 의미하는지에 대한 정의는 다음과 같습니다.</p>

<ul>
  <li><b>서비스 중단(Deprecation):</b> 해당 기능은 더 이상 팀에서 지원하지 않습니다. 이 기능에는 더 이상 새로운 기능이 추가되지 않으며, 제품에 버그나 문제가 있더라도 이를 수정할 가능성은 매우 낮습니다. </li>
  <li><b>폐기(Retired):</b> 해당 기능은 더 이상 사용할 수 없습니다.</li>
</ul>

<p>대부분의 경우, 새 버전이 출시되는 즉시 이전 버전은 서비스 중단 상태로 표시됩니다. 버전은 일정 기간 동안 서비스 중단 상태를 유지한 후 폐기됩니다. </p>
<p>앞으로 예정된 서비스 중단 및 폐기에 대해 자세히 알아보려면 <a href="https://developer.x.com/en/updates/stay-informed">최신 정보를 확인</a>하시기 바랍니다.</p>
