<div id="overview">
  ## 개요
</div>

`metrics` 필드는 개발자가 게시물 및 미디어 객체에 대한 공개 및 비공개 참여 메트릭을 조회할 수 있도록 합니다. 공개 메트릭은 개발자 계정을 가진 누구나 조회할 수 있지만, 비공개 메트릭은 소유/승인된 계정(아래 정의 참조)에서만 조회할 수 있습니다. 메트릭에는 요청에서 지정한 각 게시물에 대해 노출 수, 리트윗, 인용 트윗, 좋아요, 답글, 동영상 조회 수, 동영상 조회 구간(쿼터일), URL 및 프로필 링크 클릭 수의 총 개수가 포함됩니다. 또한 게시물이 [Ad](https://ads.x.com/)로 집행된 경우, 자연 노출(organic) 및 프로모션(promoted) 컨텍스트에서 획득한 메트릭을 구분해 확인할 수 있는 옵션도 제공됩니다.

공개 메트릭은 [App-only Token](/ko/resources/fundamentals/authentication#oauth-2-0) 인증으로 요청할 수 있습니다. 비공개 메트릭은 소유/승인된 게시물에 대해서만 요청할 수 있으며, 이를 위해 개발자는 [OAuth 2.0](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) 또는 [OAuth 1.0a](/ko/resources/fundamentals/authentication#oauth-1-0a-2) 사용자 컨텍스트 인가를 사용해 인증해야 합니다.

**비공개, 자연 노출, 프로모션 메트릭은 최근 30일 이내에 생성된 게시물에 대해서만 제공됩니다.**

<div id="terminology">
  ### 용어
</div>

* **Authorized account**: 해당 계정에 대한 접근을 허용해 사용자의 [X developer app](/ko/resources/fundamentals/developer-apps)에 권한을 부여한 X 계정(어떤 [app permission level](/ko/resources/fundamentals/developer-apps#app-permissions)이든 게시물 메트릭에 대한 접근을 허용합니다).
* **Owned account**: 사용자의 [X developer app](/ko/resources/fundamentals/developer-apps)에 연결된 X 계정.
* **Public metrics**: 좋아요 개수, 리트윗 개수처럼 [X](https://x.com/home)에서 누구나 볼 수 있는 집계 메트릭.
* **Non-public metrics**: 노출 수, 동영상 조회 사분위수처럼 [X](https://x.com/home)에서 공개적으로 볼 수 없는 집계 메트릭. [OAuth 2.0](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) 또는 [OAuth 1.0a](/ko/resources/fundamentals/authentication#oauth-1-0a-2) 사용자 컨텍스트 인증이 필요합니다.
* **Organic metrics**: 일반적인 방식으로 게시·조회된, 유기적(organic) 컨텍스트에 귀속되는 공개 및 비공개 메트릭의 그룹. [OAuth 2.0](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) 또는 [OAuth 1.0a](/ko/resources/fundamentals/authentication#oauth-1-0a-2) 사용자 컨텍스트 인증이 필요합니다.
* **Promoted metrics**: 광고 캠페인의 일부로 게시되거나 조회되는 등 프로모션(광고) 컨텍스트에 귀속되는 공개 및 비공개 메트릭의 그룹. [OAuth 2.0](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) 또는 [OAuth 1.0a](/ko/resources/fundamentals/authentication#oauth-1-0a-2) 사용자 컨텍스트 인증이 필요하며, 해당 게시물이 광고 캠페인에서 프로모션된 게시물이어야 합니다.

<div id="requesting-metrics">
  ## 메트릭 요청하기
</div>

<div id="public-metrics">
  ### 공개 메트릭
</div>

다음 요청에서는 게시물과 첨부된 동영상에 대해 아래 필드와 `expansions` 파라미터를 사용해 공개 메트릭을 요청합니다. `$BEARER_TOKEN`을 직접 발급받은 Bearer Token 값으로 반드시 교체하세요.

* `tweet.fields=public_metrics`
* `expansions=attachments.media_keys&media.fields=public_metrics`

<div id="sample-request">
  #### 예시 요청
</div>

```bash
curl 'https://api.x.com/2/tweets?ids=1204084171334832128&tweet.fields=public_metrics&expansions=attachments.media_keys&media.fields=public_metrics' --header 'Authorization: Bearer $BEARER_TOKEN'
```

<div id="private-metrics-non-public-organic-metrics">
  ### 비공개 메트릭(Non-public, Organic 메트릭)
</div>

다음 요청에서는 게시물과 첨부된 동영상에 대한 비공개 메트릭과 오가닉 메트릭의 추가 세부 정보를 요청합니다. 이러한 필드는 비공개이며 X에서 공개적으로 조회할 수 없으므로, 요청에는 [OAuth 2.0](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) 또는 [OAuth 1.0a](/ko/resources/fundamentals/authentication#oauth-1-0a-2) 사용자 컨텍스트 인증이 필요합니다. 아래 요청에 필요한 OAuth 1.0a 서명을 생성하는 방법은 [가이드](/ko/resources/fundamentals/authentication#creating-a-signature)를 참조하십시오.

* `tweet.fields=non_public_metrics,organic_metrics`
* `expansions=attachments.media_keys&media.fields=non_public_metrics,organic_metrics`

#### 요청 예시

```bash
curl 'https://api.x.com/2/tweets/1204084171334832128?tweet.fields=non_public_metrics,organic_metrics&media.fields=non_public_metrics,organic_metrics&expansions=attachments.media_keys' --header 'authorization: OAuth oauth_consumer_key="CONSUMER_API_KEY", oauth_nonce="OAUTH_NONCE", oauth_signature="OAUTH_SIGNATURE", oauth_signature_method="HMAC-SHA1", oauth_timestamp="OAUTH_TIMESTAMP", oauth_token="ACCESS_TOKEN", oauth_version="1.0"'
```
