X는 X API로 빌드하는 개발자 커뮤니티를 위해 최선을 다하고 있습니다. 이러한 노력의 일환으로, 우리는 API를 개발자에게 개방적이고 공정하게 제공하는 동시에 X를 사용하는 사람들에게는 안전하게, 그리고 X 플랫폼 전체에 도움이 되도록 하는 것을 목표로 합니다. 오프라인으로 X 콘텐츠를 저장하는 모든 개발자는 해당 데이터가 사용자 의도와 X 상의 콘텐츠의 최신 상태를 반영하도록 하는 것이 매우 중요합니다. 예를 들어, X에서 누군가가 게시물이나 계정을 삭제하거나, 게시물을 보호 처리하거나, 게시물에서 위치(geo) 정보를 제거하는 경우, 그 사람의 기대와 의도를 X와 개발자 모두가 존중하는 것이 매우 중요합니다. 배치 컴플라이언스 엔드포인트는 개발자가 [X 개발자 계약 및 정책](https://developer.x.com/en/developer-terms/policy)을 준수하여 X 데이터를 유지 관리하는 데 도움이 되는 간편한 도구를 제공합니다. 

이 배치 컴플라이언스 엔드포인트를 사용하면 대규모 게시물 또는 사용자 ID 데이터셋을 업로드하여 컴플라이언스 상태를 조회하고, 그 결과를 바탕으로 어떤 데이터에 조치가 필요한지 파악하여 데이터셋을 규정 준수 상태로 맞출 수 있습니다. 배치 컴플라이언스 엔드포인트의 사용은 앞서 언급된 사용 사례로 제한되며, 그 외 다른 목적의 사용은 금지되며 제재 조치로 이어질 수 있다는 점에 유의하십시오.

일반적으로 이 엔드포인트를 사용할 때에는 다음의 4단계가 포함됩니다:

1. **컴플라이언스 작업 생성**
   업로드하려는 데이터셋에 게시물 ID가 있는지, 사용자 ID가 있는지를 나타내기 위해 작업 유형을 지정할 수 있습니다(값은 tweets 또는 users). 작업 유형별로 동시에 실행할 수 있는 작업은 항상 하나뿐입니다.
2. **upload&#95;url로 데이터셋 업로드**
   다음으로, 제공된 upload&#95;url에 데이터셋을 일반 텍스트 파일로 업로드합니다. 파일의 각 줄에는 하나의 게시물 ID 또는 사용자 ID가 포함되어 있어야 합니다. upload&#95;url은 15분 후 만료됩니다.
3. **(선택 사항) 작업 상태 확인**
   컴플라이언스 작업의 상태를 확인하여 생성됨(created), 진행 중(in&#95;progress), 실패함(failed), 완료됨(complete) 상태인지 확인할 수 있습니다.
4. **결과 다운로드**
   작업이 완료되면 download&#95;url을 사용해 결과를 다운로드할 수 있습니다. download&#95;url은 작업이 생성된 시점으로부터 1주일 후에 만료됩니다.

   이 결과에는 JSON 객체 여러 개가 포함되며(줄당 하나의 객체), 각 객체에는 게시물 ID, 게시물 작성 날짜(날짜별로 정렬된 게시물을 찾는 데 유용), 필요한 조치, 컴플라이언스 조치가 필요한 이유, 사용자가 정지된 날짜가 포함됩니다.

결과에서는 다음과 같은 컴플라이언스 이벤트 유형을 받게 됩니다:

* deleted - 게시물 또는 사용자 계정이 삭제되었음을 나타냅니다.
* deactivated - 게시물 또는 사용자 계정이 비활성화되었음을 나타냅니다.
* scrub&#95;geo - 게시물 또는 사용자와 연결된 위치(geo) 정보가 제거되었음을 나타냅니다.
* protected - 게시물을 작성한 계정이 비공개로 전환되었음을 나타냅니다.
* suspended - 게시물을 작성한 계정이 정지되었음을 나타냅니다.

**계정 설정**

이 엔드포인트에 액세스하려면 다음이 필요합니다:

* 승인된 [개발자 계정](https://developer.x.com/en/portal/petition/essential/basic-info)
* [프로젝트](/ko/resources/fundamentals/projects) 내에 있는 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용한 인증

X API v2 엔드포인트에 액세스하는 방법은 [시작 가이드](/ko/x-api/getting-started/getting-access)에서 자세히 알아볼 수 있습니다.
