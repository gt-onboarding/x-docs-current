<div id="introduction">
  ## 소개
</div>

소프트웨어 개발 키트(SDK)는 특정 플랫폼이나 API에 맞게 제작된 소프트웨어 도구와 프로그램의 집합입니다. SDK의 목적은 개발자가 애플리케이션 내에서 활용할 수 있는 라이브러리나 코드베이스를 제공하여, 최소한의 코딩으로 효율적으로 애플리케이션을 개발하거나 기능을 확장할 수 있게 하는 데 있습니다. 이는 개발 프로세스를 크게 가속화하여 시간, 비용, 노력을 절감합니다.

> X Developer Platform은 이제 TypeScript/JavaScript와 Java로 개발하는 사용자를 위해 2개의 공식 SDK를 제공합니다. 이를 통해 개발자는 X API v2를 둘러싼 복잡한 부분을 직접 구현할 필요 없이, 사용 가능한 모든 v2 엔드포인트에 대한 사전 구현된 함수와 간소화된 인증 프로세스를 활용하여 더 효율적으로 애플리케이션을 구축할 수 있습니다. 이 SDK들은 Developer Platform 팀이 직접 개발하고 유지 관리하므로, X API v2의 향후 릴리스에 맞춰 항상 최신 상태가 유지됩니다.

이 SDK들은 X API를 래핑하므로, [프로젝트](/ko/resources/fundamentals/projects) 내에 위치한 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 자격 증명을 사용해 요청을 인증하려면 [개발자 계정](https://developer.x.com/en/portal/dashboard)이 있어야 합니다.

  #### 설치
</div>

<Tabs>
  <Tab title="Java">
    Java 패키지를 설치하는 방법은 여러 가지가 있습니다(Java 1.8+ 필요).

    * Maven 사용자: 프로젝트의 POM 파일에 다음 의존성을 추가하세요.

    ```xml
    <dependency>
      <groupId>com.twitter</groupId>
      <artifactId>twitter-api-java-sdk</artifactId>
      <version>1.1.4</version>
    </dependency>
    ```

    * Gradle 사용자: 프로젝트의 빌드 파일에 다음 의존성을 추가하세요.
      `implementation "com.twitter:twitter-api-java-sdk:1.1.4"`

    * 그 외의 경우: 먼저 다음 명령을 실행하여 JAR을 생성하세요.
      `mvn clean package`
      그런 다음 다음 JAR들을 수동으로 설치하세요.

    `target/twitter-api-java-sdk-1.1.4.jar`

    `target/lib/*.jar`
  </Tab>

  <Tab title="TypeScript">
    이 패키지는 Node.js 14+를 지원합니다. 설치하려면 Node 프로젝트 디렉터리에서 다음 명령을 실행하세요.
    `npm install twitter-api-sdk`
  </Tab>
</Tabs>

<div id="client-basics">
  #### Client 기본 사항
</div>

작업 파일 상단에서 클래스(Java)와 패키지(TypeScript)를 import하여 인증 및 라이브러리 Client에 접근할 수 있습니다. 라이브러리 Client의 메서드를 사용하려면 인증 자격 증명을 전달해야 하며, 이는 Bearer Token(앱 전용(App-only))이거나 OAuth 2.0 사용자 컨텍스트로 인증하는 경우 client id/client secret일 수 있습니다.

다음은 예시입니다:

<Tabs>
  <Tab title="Java">
    ```java
    // 클래스 import:
    import com.twitter.clientlib.ApiClient;
    import com.twitter.clientlib.ApiException;
    import com.twitter.clientlib.Configuration;
    import com.twitter.clientlib.auth.*;
    import com.twitter.clientlib.model.*;
    import com.twitter.clientlib.TwitterCredentialsBearer;
    import com.twitter.clientlib.api.TwitterApi;

    // 라이브러리 Client 인스턴스화
    TwitterApi apiInstance = new TwitterApi();

    // 인증 자격 증명 인스턴스화 (앱 전용(App-only) 예시)
    TwitterCredentialsBearer credentials = new TwitterCredentialsBearer(System.getenv("APP-ONLY-ACCESS-TOKEN"));

    // 자격 증명을 라이브러리 Client에 전달
    apiInstance.setTwitterCredentials(credentials);
    ```
  </Tab>

  <Tab title="TypeScript">
    ```typescript
    // 패키지 import
    import { Client, auth } from "twitter-api-sdk";

    // 먼저 auth Client를 초기화
    const authClient = new auth.OAuth2User({
     client_id: process.env.CLIENT_ID as string,
     client_secret: process.env.CLIENT_SECRET as string,
     callback: "YOUR-CALLBACK",
     scopes: ["tweet.read", "users.read", "offline.access"],
    });

    // auth 자격 증명을 라이브러리 Client에 전달
    const twitterClient = new Client(authClient);
    ```
  </Tab>
</Tabs>

<div id="authentication-flow">
  #### 인증 플로우
</div>

SDK를 인증할 때 애플리케이션 전용 옵션을 사용하는 경우, 토큰만 제공하면 라이브러리 클라이언트가 즉시 엔드포인트 메서드를 사용할 준비가 됩니다. 단, 애플리케이션 전용 토큰은 사용자 컨텍스트 인증이 필요한 엔드포인트에서는 사용할 수 없다는 점을 유의하세요.

OAuth 2.0 사용자 컨텍스트 인증은 auth 클라이언트를 생성한 이후에 몇 가지 추가 단계가 필요합니다.

* authorization URL 생성
* authorization URL에서 애플리케이션 승인
* callback으로 리디렉션(Developer Portal의 auth 설정 페이지에 설정된 callback URL과 일치해야 합니다)
* code verifier를 파싱하여 액세스 토큰으로 교환

SDK는 auth 클라이언트에 이 단계들을 단순화해 주는 메서드를 제공합니다. OAuth 2.0 사용자 컨텍스트로 인증하여 요청을 수행하는 전체 예제는 GitHub 리포지토리를 참고하세요.

* [TypeScript](https://github.com/xdevplatform/twitter-api-typescript-sdk/blob/main/examples/oauth2-callback_pkce_s256.ts)

* [Java](https://github.com/xdevplatform/twitter-api-java-sdk/blob/main/examples/src/main/java/com/twitter/clientlib/auth/OAuth20GetAccessToken.java)

<div id="endpoint-methods">
  #### 엔드포인트 메서드
</div>

라이브러리 Client에서 제공하는 메서드는 각 엔드포인트에 대응하도록 명확하게 이름이 지정되어 있으며, 모든 매개변수는 인수로 전달됩니다.
다음은 ID로 게시물을 조회하는 예시입니다.

<Tabs>
  <Tab title="Java">
    ```java
    String id = "1511757922354663425"; // String | 단일 트윗 ID.
    Set<String> expansions = new HashSet<>(Arrays.asList("author_id")); // Set<String> | 확장할 필드를 쉼표로 구분한 목록.
    Set<String> tweetFields = new HashSet<>(Arrays.asList("created_at", "lang", "context_annotations")); // Set<String> | 표시할 트윗 필드를 쉼표로 구분한 목록.
    Set<String> userFields = new HashSet<>(Arrays.asList("created_at", "description", "name")); // Set<String> | 표시할 사용자 필드를 쉼표로 구분한 목록.

    try {
     SingleTweetLookupResponse result = apiInstance.tweets().findTweetById(id, expansions, tweetFields, userFields, null, null, null);
     System.out.println(result);
    } catch (ApiException e) {
     System.err.println("TweetsApi#findTweetById 호출 시 예외 발생");
     System.err.println("상태 코드: " + e.getCode());
     System.err.println("이유: " + e.getResponseBody());
     System.err.println("응답 헤더: " + e.getResponseHeaders());
     e.printStackTrace();
    }

    ```
  </Tab>

  <Tab title="TypeScript">
    ```typescript
    const lookupTweetById = await client.tweets.findTweetById(
      // 트윗 ID
      "1511757922354663425",
      {
        // 선택적 매개변수
        expansions: ["author_id"],
        "user.fields": ["created_at", "description", "name"],
      }
    );
    ```
  </Tab>
</Tabs>
