이 가이드는 첫 요청을 보내기 위해 따라갈 수 있는 몇 가지 단계를 안내합니다. X 계정에 가입한 후 참고하기에 좋은 자료입니다.

샘플 코드, 더 기술적인 가이드, 또는 Postman 같은 그래픽 도구를 사용하고 싶다면, 첫 요청을 보내는 데 아래 가이드들을 활용해 보세요.

* [Post 엔드포인트에 첫 요청을 보내는 빠른 시작 가이드](/ko/x-api/posts/manage-tweets/quickstart)
* [Postman 시작하기](/ko/tutorials/postman-getting-started) - 초보자에게 가장 적합
* [X API v2 코드 샘플](https://github.com/xdevplatform/Twitter-API-v2-sample-code)
* [Post 조회 API 레퍼런스](/ko/x-api/posts/lookup/introduction)

이 가이드는 여러분이 이미 [API 키 및 시크릿](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret), [사용자 액세스 토큰 및 시크릿](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens), [앱 액세스 토큰](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)을 발급받아 안전한 위치에 보관했다고 가정합니다. 이러한 정보를 얻는 방법은 [X API 액세스 권한 얻기](/ko/x-api/getting-started/getting-access) 가이드의 단계를 따라 학습할 수 있습니다.

<div id="step-1-identify-which-endpoint-you-would-like-to-use">
  ### 1단계. 사용할 endpoint 선택하기
</div>

X API를 사용하면 X 웹사이트나 모바일 애플리케이션에서 할 수 있는 다양한 작업을 코드를 통해 수행할 수 있습니다.

API를 통해 사용할 수 있는 endpoint의 전체 목록은 [API Reference Index](/ko/x-api/introduction)에 정리되어 있지만, 우선은 다음 중 하나의 endpoint부터 사용하는 것을 권장합니다.

* [게시물 관리 &gt; 트윗 게시하기](/ko/x-api/posts/manage-tweets/introduction)
* [게시물 검색](/ko/x-api/posts/search/introduction)
* [사용자 조회](/ko/x-api/users/lookup/introduction)

<div id="step-2-choose-a-tool-to-make-your-request">
  ### 2단계. 요청을 만들 도구 선택하기
</div>

일부 요청은 비교적 간단하지만, 다른 요청은 구성하기 까다로울 수 있습니다. 그래서 뛰어난 개발자 커뮤니티에서 이러한 복잡성을 일부 덜어 주는 도구들을 만들어 두었습니다.

아래는 권장 도구들과 각 도구를 사용하는 방법에 대한 자세한 내용입니다:

<div id="postman">
  #### Postman
</div>

Postman은 REST 엔드포인트에 요청을 전송할 수 있는 그래픽 사용자 인터페이스(GUI) 도구입니다. X API를 통해 제공되는 다양한 엔드포인트를 시작해 보고 탐색하는 데 도움이 되도록 Postman 관련 유용한 자료를 제공하고 있습니다.

[&quot;Getting started with Postman&quot; 튜토리얼](/ko/tutorials/postman-getting-started)을 읽고, 키와 토큰을 추가하는 방법과 첫 번째 요청을 보내는 방법을 익혀 보시기 바랍니다.

또한 대부분 Postman을 사용하는 각 X API v2 엔드포인트용 빠른 시작 가이드도 제공합니다. 각 엔드포인트 섹션에서 해당 가이드를 확인하실 수 있으며, 일부 가이드는 아래 링크에서도 확인할 수 있습니다.

* 빠른 시작: 트윗 게시
* 빠른 시작: 게시물 검색
* 빠른 시작: 사용자 조회

Postman으로는 스트리밍 엔드포인트에 요청을 보낼 수 없습니다. 해당 엔드포인트를 사용하는 방법을 알아보려면 [filtered stream](/ko/x-api/posts/filtered-stream/quickstart) 또는 [1% sampled stream](/ko/x-api/posts/sample-stream) 빠른 시작 가이드를 참고하세요.

보다 단순한 그래픽 도구를 선호하신다면 [Insomnia](https://insomnia.rest) 사용도 고려해 볼 수 있습니다.

<div id="sample-code">
  #### 샘플 코드
</div>

간단한 코드를 사용해 요청을 보내고 싶다면, 각 X API v2 엔드포인트마다 여러 언어로 준비해 둔 샘플 코드가 있습니다.

샘플 코드는 GitHub 리포지토리 [X-API-v2-sample-code](https://github.com/xdevplatform/Twitter-API-v2-sample-code)에서 확인할 수 있으며, 여기에는 요청이 정상적으로 동작하도록 자격 증명을 설정하는 방법을 알아보는 데 활용할 수 있는 README 파일도 포함되어 있습니다.

예를 들어, 아래는 user lookup 엔드포인트에 대한 cURL 예시입니다. 이 요청을 사용하려면 `$ACCESS_TOKEN`과 `$USERNAME`을 각각 [App Access Token](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)과 X 핸들로 바꾸기만 하면 됩니다. 그런 다음 이 코드를 명령줄 도구에 복사해 붙여넣고 &quot;Return&quot; 또는 &quot;Enter&quot; 키를 누르십시오.

```bash
curl "https://api.x.com/2/users/by/username/$USERNAME" -H "Authorization: Bearer $ACCESS_TOKEN"
```

<div id="libraries">
  #### 라이브러리
</div>

멋진 X 개발자 커뮤니티에서는 X API에 요청을 보내는 데 사용할 수 있는, 다양한 프로그래밍 언어로 된 라이브러리도 만들어 두었습니다.

커뮤니티 라이브러리를 모두 모아 정리한 [&quot;도구 및 라이브러리&quot; 페이지](/ko/x-api/tools-and-libraries/overview)를 제공하고 있습니다. 각 라이브러리에는 보통 README 파일이 포함되어 있으며, 이를 통해 로컬 환경에 리포지토리를 설정하고 첫 번째 요청을 보내는 방법을 배울 수 있습니다.

> **참고:** 문제가 발생하는 경우, 요청을 보내려는 엔드포인트에 대한 개발자 문서와 [지원 섹션](https://developer.x.com/en/support)을 확인하거나, 포럼에서 커뮤니티에 도움을 요청해 보세요. 성공적으로 요청을 보내실 수 있도록 도와드리겠습니다!

<div id="step-3-review-the-response">
  ### 3단계. 응답 검토하기
</div>

요청이 성공적으로 처리되면 해당 요청과 관련된 메타데이터가 포함된 페이로드를 받게 됩니다.

GET HTTP 메서드를 사용하는 엔드포인트를 호출한 경우, 요청한 리소스(게시물, 사용자, 리스트, Space 등)와 관련된 메타데이터를 JSON 형식으로 받게 됩니다. 반환된 다양한 필드를 검토하고, 요청한 정보가 X의 어떤 콘텐츠에 해당하는지 확인해 보세요.

POST, PUT, DELETE HTTP 메서드를 사용하는 엔드포인트를 호출한 경우, X에서 어떤 동작을 수행한 것입니다. X.com 또는 모바일 앱으로 이동하여 해당 동작이 실제로 반영되었는지 확인해 보세요.

<div id="step-4-adjust-your-request-using-parameters">
  ### 4단계. 파라미터를 사용해 요청 조정하기
</div>

각 엔드포인트에는 요청을 조정하는 데 사용할 수 있는 서로 다른 파라미터 집합이 있습니다. 예를 들어, `fields` 및 `expansions` 파라미터와 함께 GET 엔드포인트를 사용할 때 추가 메타데이터 필드를 요청할 수 있습니다. 또한 [search Posts](/ko/x-api/posts/search/introduction), [Post counts](/ko/x-api/posts/counts/introduction), [filtered stream](/ko/x-api/posts/filtered-stream/introduction)과 같은 엔드포인트에서 제공하는 다양한 필터링 도구를 활용해, 수신하는 데이터를 관심 있는 게시물만으로 좁힐 수 있습니다.

요청 파라미터와 필드 전체 목록은 작업 중인 엔드포인트의 API Reference에서 확인할 수 있으며, 그 밖에도 통합 가이드와 Fundamentals(기초) 페이지에서 다양한 유용한 통합 관련 정보를 찾아볼 수 있습니다.

[Important resources](/ko/x-api/getting-started/important-resources) 가이드를 통해 제공하는 모든 교육 자료에 대해 더 자세히 알아볼 수 있습니다.
