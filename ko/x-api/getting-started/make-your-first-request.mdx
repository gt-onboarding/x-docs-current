이 가이드는 첫 요청을 보내기 위해 따라야 하는 몇 가지 단계를 안내합니다. X 계정에 가입한 후 참고하기에 좋은 자료입니다.

샘플 코드, 보다 기술적인 가이드, 또는 Postman과 같은 그래픽 도구를 사용하고 싶다면, 첫 요청을 보내는 데 아래 가이드들을 참고해 보세요.

* [게시물 엔드포인트에 첫 요청을 보내는 빠른 시작 가이드](/ko/x-api/posts/manage-tweets/quickstart)
* [Postman 시작하기](/ko/tutorials/postman-getting-started) - 초보자에게 가장 적합
* [X API v2 code samples](https://github.com/xdevplatform/Twitter-API-v2-sample-code)
* [Post 조회 API 레퍼런스](/ko/x-api/posts/lookup/introduction)

이 가이드는 여러분이 이미 [API key and secret](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret), [user Access Token and Secret](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens), [App Access Token](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)을 발급받아 안전한 위치에 저장해 두었다고 가정합니다. 이 작업을 수행하는 방법은 [X API 액세스 권한 받기](/ko/x-api/getting-started/getting-access) 가이드의 단계를 따라 확인할 수 있습니다.

<div id="step-1-identify-which-endpoint-you-would-like-to-use">
  ### 1단계. 사용하려는 엔드포인트를 선택합니다
</div>

X API를 사용하면 X 웹사이트나 모바일 애플리케이션에서 할 수 있는 다양한 작업을 코드로 수행할 수 있습니다.

API를 통해 사용할 수 있는 엔드포인트의 전체 목록은 [API Reference Index](/ko/x-api/introduction)에 정리되어 있지만, 처음에는 단순하게 시작할 수 있도록 아래 엔드포인트 중 하나를 사용할 것을 권장합니다.

* [게시물 관리 &gt; 트윗 작성](/ko/x-api/posts/manage-tweets/introduction)
* [게시물 검색](/ko/x-api/posts/search/introduction)
* [사용자 조회](/ko/x-api/users/lookup/introduction)

<div id="step-2-choose-a-tool-to-make-your-request">
  ### 2단계. 요청을 보내는 데 사용할 도구 선택하기
</div>

일부 요청은 비교적 간단하지만, 다른 요청은 구성하기 까다로울 수 있습니다. 그래서 훌륭한 개발자 커뮤니티가 이러한 복잡한 부분을 대신 처리해 주는 도구들을 만들어 왔습니다.

아래는 권장하는 도구와 사용 방법에 대한 안내입니다.

<div id="postman">
  #### Postman
</div>

Postman은 REST 엔드포인트에 요청을 보내는 데 사용할 수 있는 시각적 도구입니다. X API를 통해 제공되는 다양한 엔드포인트에 입문하고 살펴보는 데 도움이 되도록 Postman과 관련된 유용한 자료를 준비해 두었습니다.

[&quot;Postman 시작하기&quot; 튜토리얼](/ko/tutorials/postman-getting-started)을 읽어 보면서 키와 토큰을 추가하고 첫 번째 요청을 보내는 방법을 익혀 보시기를 권장합니다.

또한 대부분 Postman을 사용하는 X API v2 각 엔드포인트용 빠른 시작 가이드를 제공하고 있습니다. 이러한 가이드는 각 엔드포인트 섹션에서 찾을 수 있으며, 그중 일부 링크는 다음과 같습니다.

* 빠른 시작: 트윗 게시하기
* 빠른 시작: 게시물 검색
* 빠른 시작: 사용자 조회

Postman을 사용해서는 스트리밍 엔드포인트에 요청을 보낼 수 없다는 점에 유의하세요. 해당 엔드포인트를 사용하는 방법을 알아보려면 [filtered stream](/ko/x-api/posts/filtered-stream/quickstart) 또는 [1% sampled stream](/ko/x-api/posts/sample-stream) 빠른 시작 가이드를 방문하세요.

더 단순한 그래픽 도구를 선호한다면 [Insomnia](https://insomnia.rest)를 사용하는 것도 고려해 보세요.

<div id="sample-code">
  #### 샘플 코드
</div>

요청을 보내는 데 간단한 코드를 사용하고 싶다면, 각 X API v2 엔드포인트마다 여러 언어로 준비된 샘플 코드를 사용할 수 있습니다.

GitHub 리포지토리 [X-API-v2-sample-code](https://github.com/xdevplatform/Twitter-API-v2-sample-code)에서 코드 샘플을 확인할 수 있으며, 요청이 올바르게 동작하도록 자격 증명을 설정하는 방법을 익히는 데 사용할 수 있는 README 파일도 포함되어 있습니다.

예를 들어, 다음은 user lookup 엔드포인트를 위한 cURL 예제입니다. 이 요청을 사용하려면 `$ACCESS_TOKEN`과 `$USERNAME`을 각각 [App Access Token](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)과 X 핸들로 바꾸기만 하면 됩니다. 그런 다음 이 코드를 명령줄 도구에 복사해 붙여넣고 &quot;Return&quot; 또는 &quot;Enter&quot; 키를 누르세요.

```bash
curl "https://api.x.com/2/users/by/username/$USERNAME" -H "Authorization: Bearer $ACCESS_TOKEN"
```

<div id="libraries">
  #### 라이브러리
</div>

활발한 X 개발자 커뮤니티에서는 다양한 프로그래밍 언어로 작성된 라이브러리를 구축해 두었으며, 이를 사용해 X API에 요청을 보낼 수 있습니다.

우리는 [&quot;도구 및 라이브러리&quot; 페이지](/ko/x-api/tools-and-libraries/overview)를 제공하며, 여기에는 현재 우리가 파악하고 있는 모든 커뮤니티 라이브러리가 정리되어 있습니다. 각 라이브러리에는 일반적으로 README 파일이 포함되어 있으며, 이를 통해 로컬 환경에 리포지토리를 설정하고 첫 번째 요청을 보내는 방법을 배울 수 있습니다.

> **참고:** 문제가 발생한 경우, 요청을 보내고 있는 엔드포인트에 대한 개발자 문서를 먼저 확인하고, [지원 섹션](https://developer.x.com/en/support)을 살펴보거나, 포럼에서 커뮤니티에 도움을 요청해 보세요. 성공적인 요청을 완료할 수 있도록 도와드리겠습니다.

<div id="step-3-review-the-response">
  ### 3단계. 응답 검토하기
</div>

요청을 성공적으로 보내면 해당 요청과 관련된 메타데이터가 포함된 페이로드를 받게 됩니다.

GET HTTP 메서드를 사용하는 엔드포인트를 사용한 경우, 요청을 보낸 리소스(게시물, 사용자, 리스트, Space 등)와 관련된 메타데이터를 JSON 형식으로 받게 됩니다. 반환된 다양한 필드를 검토하고, 요청한 정보가 X의 콘텐츠와 어떻게 매핑되는지 확인해 보세요.

POST, PUT 또는 DELETE HTTP 메서드를 사용하는 엔드포인트를 사용한 경우, X에서 어떤 작업을 수행한 것입니다. X.com 또는 모바일 앱으로 이동하여 해당 작업을 실제로 확인해 보세요.

<div id="step-4-adjust-your-request-using-parameters">
  ### 4단계. 파라미터를 사용해 요청 조정하기
</div>

각 엔드포인트마다 요청을 조정하는 데 사용할 수 있는 고유한 파라미터 집합이 있습니다. 예를 들어, `fields` 및 `expansions` 파라미터와 함께 GET 엔드포인트를 사용할 때는 추가 메타데이터 필드를 요청할 수 있습니다. 또한 [search Posts](/ko/x-api/posts/search/introduction), [Post counts](/ko/x-api/posts/counts/introduction), [filtered stream](/ko/x-api/posts/filtered-stream/introduction)과 같은 엔드포인트에서 다양한 필터링 도구를 사용해 보면서, 받게 되는 데이터를 관심 있는 게시물만 포함하도록 좁힐 수 있습니다.

사용 중인 엔드포인트에 대한 API Reference에서 요청 파라미터와 필드의 전체 목록을 확인할 수 있으며, 통합 가이드와 Fundamentals 페이지에서 그 밖의 다양한 유용한 통합 관련 정보도 확인할 수 있습니다.

[Important resources](/ko/x-api/getting-started/important-resources) 가이드를 통해 제공되는 모든 교육 자료에 대해 더 자세히 알아볼 수 있습니다.
