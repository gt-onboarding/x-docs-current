<div id="step-one-sign-up-for-a-developer-account">
  ### 1단계: 개발자 계정에 가입하기
</div>

개발자 계정 가입은 빠르고 간단합니다. 아래 버튼을 클릭하고 몇 가지 질문에 답하면, [Basic access](https://developer.x.com/en/portal/products/basic)를 사용해 X API v2를 살펴보고, 이를 기반으로 통합을 구현하기 시작할 수 있습니다.

다음으로 온보딩 과정에서 프로젝트와 이에 연결된 개발자 앱을 생성하게 되며, 이를 통해 API에 대한 모든 요청을 인증하는 데 사용할 자격 증명 세트가 제공됩니다.

[가입하기](https://developer.x.com/en/portal/products/basic)

<div id="step-two-save-your-apps-key-and-tokens-and-keep-them-secure">
  ### 2단계: 앱의 키와 토큰을 저장하고 안전하게 보관하기
</div>

액세스 권한을 얻고 프로젝트와 앱을 생성한 후에는, 개발자 앱 내에서 다음과 같은 자격 증명을 확인하거나 생성할 수 있습니다:

* **[API Key and Secret](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret):** 기본적으로 앱의 사용자 이름과 비밀번호에 해당합니다. 이는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication)가 필요한 요청을 인증하거나, 사용자 액세스 토큰 또는 앱 액세스 토큰과 같은 다른 토큰을 생성하는 데 사용됩니다.

* **[Access Token and Secret](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens):** 일반적으로 액세스 토큰은 여러분이 요청을 대신 수행하는 사용자를 나타냅니다. Developer Portal에서 생성할 수 있는 액세스 토큰은 해당 앱을 소유한 사용자를 나타냅니다. 이는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication)가 필요한 요청을 인증하는 데 사용됩니다. 다른 사용자를 대신해 요청을 수행하려면, 해당 사용자가 여러분의 앱을 승인할 수 있도록 3-legged OAuth 플로우를 사용해야 합니다.

* **[Client ID and Client Secret](/ko/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce):** 이 자격 증명은 OAuth 2.0 인증을 사용하여 사용자 액세스 토큰을 얻는 데 사용됩니다. OAuth 1.0a와 유사하게, 사용자 액세스 토큰은 비공개 사용자 계정 정보를 제공하거나 다른 계정을 대신해 작업을 수행하는 요청을 인증하는 데 사용되지만, 보다 세밀한 범위(scope)를 제공하여 클라이언트 애플리케이션이 사용자에 대해 가지는 액세스 권한을 더 정밀하게 제어할 수 있습니다.

* **[App only Access Token](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only):** X에서 공개적으로 제공되는 정보를 반환하는 엔드포인트에 요청할 때 이 토큰을 사용합니다.

이러한 키와 토큰은 다시 생성하지 않는 한 만료되지 않으므로, 자격 증명을 받은 후에는 비밀번호 관리자와 같이 안전한 위치에 저장할 것을 권장합니다.

> **참고: 키와 토큰은 [developer portal](/ko/resources/fundamentals/developer-portal)에 한 번만 표시되므로, 이를 생성하자마자 비밀번호 관리 시스템에 저장하는 것이 중요합니다.**
>
> 키와 토큰을 분실하거나 기억하지 못하는 경우, 새 자격 증명을 생성하고 기존 자격 증명을 무효화하도록 다시 생성해야 합니다. 이는 이전 자격 증명을 사용하여 설정해 둔 모든 연동을 새 자격 증명으로 업데이트해야 함을 의미합니다.
>
> [인증 모범 사례](/ko/resources/fundamentals/authentication#authentication-best-practices)에 대해 더 알아보세요.

  ### 3단계: 첫 요청 보내기

시작하는 데 도움이 되는 가이드, 튜토리얼, 도구, 그리고 코드 예제가 준비되어 있습니다. 다음 페이지는 시작하기에 아주 좋은 출발점이지만, 전체 문서를 더 잘 살펴볼 수 있도록 중요한 리소스만 모아 둔 페이지도 따로 마련해 두었습니다.
