<div id="step-one-sign-up-for-a-developer-account">
  ### 1단계: 개발자 계정에 가입하기
</div>

개발자 계정은 빠르고 간편하게 신청할 수 있습니다. 아래 버튼을 클릭하고 몇 가지 질문에만 답하면, [Basic access](https://developer.x.com/en/portal/products/basic)로 X API v2를 살펴보고 이를 기반으로 개발을 시작할 수 있습니다.

이후 온보딩 과정에서 프로젝트와 해당 프로젝트에 연결된 개발자 앱을 생성하게 되며, 여기에서 API에 대한 모든 요청을 인증하는 데 사용할 자격 증명 세트를 발급받게 됩니다.

[가입하기](https://developer.x.com/en/portal/products/basic)

<div id="step-two-save-your-apps-key-and-tokens-and-keep-them-secure">
  ### 2단계: 앱의 키와 토큰을 저장하고 안전하게 보호하기
</div>

액세스 권한을 얻고 프로젝트와 앱을 생성한 후에는 개발자 앱에서 다음과 같은 자격 증명을 찾거나 생성할 수 있습니다.

* **[API Key and Secret](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret):** 사실상 앱의 사용자 이름과 비밀번호와 같습니다. 이는 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication)가 필요한 요청을 인증하거나, 사용자 액세스 토큰 또는 앱 액세스 토큰과 같은 다른 토큰을 생성하는 데 사용됩니다.

* **[Access Token and Secret](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens):** 일반적으로 액세스 토큰은 요청을 보낼 때 해당 요청을 대신 수행하는 사용자를 나타냅니다. developer portal에서 생성할 수 있는 액세스 토큰은 앱 소유자를 나타냅니다. 이는 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication)가 필요한 요청을 인증하는 데 사용됩니다. 다른 사용자를 대신하여 요청하려면, 해당 사용자가 권한을 부여할 수 있도록 3-legged OAuth 플로우를 사용해야 합니다.

* **[Client ID and Client Secret](/ko/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce):** 이 자격 증명은 OAuth 2.0 인증을 사용하여 사용자 액세스 토큰을 얻는 데 사용됩니다. OAuth 1.0a와 마찬가지로, 사용자 액세스 토큰은 비공개 사용자 계정 정보를 제공하거나 다른 계정을 대신해 작업을 수행하는 요청을 인증하는 데 사용되지만, 클라이언트 애플리케이션이 사용자 계정에 대해 갖는 액세스 권한을 더 세밀한 스코프로 제어할 수 있습니다.

* **[App only Access Token](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only):** X에서 공개적으로 제공되는 정보를 반환하는 엔드포인트에 요청을 보낼 때 이 토큰을 사용합니다.

이러한 키와 토큰은 재생성하지 않는 한 만료되지 않으므로, 자격 증명을 받은 후 비밀번호 관리자와 같은 안전한 위치에 저장할 것을 권장합니다.

> **주의: [developer portal](/ko/resources/fundamentals/developer-portal)에서는 키와 토큰이 한 번만 표시되므로, 생성하는 즉시 비밀번호 관리 시스템에 이 자격 증명을 저장하는 것이 중요합니다.**
>
> 키와 토큰을 분실하거나 기억하지 못하는 경우, 이를 재생성해야 하며, 이때 새 자격 증명이 생성되고 기존 자격 증명은 무효화됩니다. 이는 이전 자격 증명을 사용하여 설정해 둔 모든 통합을 업데이트해야 함을 의미합니다.
>
> [인증 모범 사례](/ko/resources/fundamentals/authentication#authentication-best-practices)에 대해 자세히 알아보세요.
