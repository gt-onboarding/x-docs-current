<div id="what-is-the-x-api-v2">
  ## X API v2란 무엇인가요?
</div>

X API v2는 이제 주요 X API이며, 제품 개발 투자와 혁신이 집중되는 곳입니다. 우리는 다양한 개발자 커뮤니티를 더 잘 지원하기 위해 차세대 X API를 구축하는 과정에서 개발자들과 협력해 왔습니다. 개발자 피드백을 바탕으로 더 다양한 요구를 충족하고, 새로운 기능과 엔드포인트를 도입하며, 개발자 경험을 개선할 수 있도록 API를 다시 설계했습니다.

X API v2는 이제 주요 X API이며, 제품 개발 투자와 혁신이 집중되는 곳입니다. 지난 몇 년 동안 우리는 개발자들과 협력하여 더 다양한 요구를 충족하고, 새로운 기능과 엔드포인트를 도입하며, 개발자 경험을 개선할 수 있도록 API를 재구축했습니다. 우리는 계속해서 개방형 개발자 플랫폼을 구축할 것을 약속하며, 여러분이 X API v2로 무엇을 만들어 나갈지 기대하고 있습니다.

<div id="why-migrate">
  ## 왜 마이그레이션해야 하나요?
</div>

X API v2는 현대적이고 더 지속 가능한 기반 위에 구축되었으며, 표준 v1.1 및 Enterprise 제품을 대체하는 개선된 엔드포인트뿐만 아니라 완전히 새로운 기능도 제공합니다. 레거시 API(v1.1 및 Enterprise) 고객은 v2로의 마이그레이션을 시작할 것을 강력히 권장하며, 이들 레거시 API는 결국 사용 중단할 예정입니다. X API를 사용해 공개 대화를 수집·분석하고, X에서 사람들과 소통하며, 혁신을 이어가십시오.

이 섹션에서는 엔드포인트와 기능에 대해 설명합니다.

다음 가이드를 통해 v2 endpoint 전체 목록과 각 endpoint에 대응하는 v2 이전 버전 endpoint를 확인할 수 있습니다.

<Button href="/ko/x-api/migrate/x-api-endpoint-map">
  X API Endpoint Map

X API v2의 대부분 endpoint는 기존 endpoint를 대체하지만, 여러 새로운 endpoint도 추가되었습니다. 다음은 v2에서 새로 제공되는 endpoint의 몇 가지 예시입니다:

* [Spaces endpoints](/ko/x-api/spaces/introduction)는 사람들이 X Spaces를 더 잘 활용할 수 있도록 돕고, 개발자가 오디오 대화의 미래를 함께 만들어 갈 수 있도록 합니다.
* [Hide replies](/ko/x-api/posts/hide-replies/introduction)는 학대적이거나, 산만하거나, 오해를 불러일으킬 수 있는 답글의 영향을 대규모로 줄이는 도구를 구축할 수 있게 해줍니다.
* [리스트를 고정 및 고정 해제](/ko/x-api/lists/pinned-lists/introduction)하거나 특정 사용자의 고정된 리스트를 조회할 수 있게 해주는 새로운 Lists endpoints.
* 저장해 둔 사용자 및 트윗 데이터가 규정을 준수하고 있는지 확인할 수 있게 해주는 새로운 [batch compliance endpoints](/ko/x-api/compliance/batch-compliance/introduction).

<div id="discover-new-and-updated-response-objects">
  ### 새로운 응답 객체와 업데이트된 응답 객체 알아보기
</div>

다음 여섯 가지 데이터 객체를 v2 엔드포인트에서 사용할 수 있습니다:

| Object | Description |
|:--------|:-------------|
| [Tweet](/ko/x-api/fundamentals/data-dictionary#tweet) | 트윗 객체에는 `id`, `text`, `created_at`과 같은 루트 수준 필드가 다수 포함되어 있습니다. 트윗 객체는 `user`, `media`, `poll`, `place` 등을 포함한 여러 자식 객체의 부모 객체이기도 합니다. |
| [User](/ko/x-api/fundamentals/data-dictionary#user) | user 객체에는 참조된 사용자를 설명하는 X 사용자 계정 메타데이터가 포함됩니다. |
| [Spaces](/ko/x-api/fundamentals/data-dictionary#space) | Space 객체는 `state`, `host_id`, `is_ticketed`, `lang` 등의 필드로 구성됩니다. |
| [Lists](/ko/x-api/fundamentals/data-dictionary#list) | List 객체에는 `description`, `member_count`, `owner_id` 등을 비롯해 요청된 리스트에 대한 기본 정보가 포함됩니다. |
| [Media](/ko/x-api/fundamentals/data-dictionary#media) | 트윗에 이미지와 같은 미디어가 포함된 경우 `media.fields` 파라미터를 사용해 media 객체를 요청할 수 있으며, `media_key`, `type`, `url`, `preview_image_url` 등의 필드를 포함합니다. |
| [Poll](/ko/x-api/fundamentals/data-dictionary#poll) | 트윗에 포함된 투표는 어느 엔드포인트에서도 기본(1차) 객체는 아니지만, 트윗 객체에서 찾아서 확장해 확인할 수 있습니다. |
| [Place](/ko/x-api/fundamentals/data-dictionary#place) | place 객체에는 `place_id`, `geo` 객체, `country_code` 등의 필드가 포함됩니다. 이 정보는 트윗을 식별하고 위치별로 트윗을 분석하는 데 사용할 수 있습니다. |

<div id="flexibility-to-choose-which-objects-and-fields-you-receive">
  ### 수신할 오브젝트와 필드를 선택할 수 있는 유연성
</div>

GET 엔드포인트에 요청을 보내면 해당 엔드포인트와 연관된 기본 데이터 오브젝트를 수신하며, 여기에 기본 필드 세트가 포함됩니다. 예를 들어, Tweet 오브젝트는 기본적으로 `id`와 `text` 필드를 제공합니다.

요청에서 추가 필드를 함께 받고 싶다면 [`fields`](/ko/x-api/fundamentals/fields) 및 [`expansions`](/ko/x-api/fundamentals/expansions) 파라미터를 사용해야 합니다. `expansions` 파라미터를 사용하면 사용자의 고정 트윗이나 미디어 오브젝트와 같은 연관 데이터 오브젝트를 가져올 수 있으며, 필드 연산자를 사용하면 기본 제공 필드 외에 반환된 오브젝트 내의 특정 필드를 선택해서 요청할 수 있습니다.

다음은 다양한 X API v2 엔드포인트에서 요청할 수 있는 expansions의 전체 목록입니다:

| Object / Resource | Available Expansions |
|:-------------------|:----------------------|
| Tweets            | `author_id`, `edit_history_tweet_ids`, `entities.mentions.username`, `in_reply_to_user_id`, `referenced_tweets.id`, `referenced_tweets.id.author_id`, `attachments.poll_ids`, `attachments.media_keys`, `geo.place_id` |
| Users             | `pinned_tweet_id` |
| Spaces            | `invited_user_ids`, `speaker_ids`, `creator_id`, `host_ids`, `topic_ids` |

[fields와 expansions 사용 방법](/ko/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)에 대해 더 알아보세요.

<div id="edit-tweets">
  ## 트윗 편집
</div>

X API v2 엔드포인트는 편집된 트윗과 관련된 메타데이터를 제공합니다. *트윗 편집* 기능은 2022년 9월 1일 X 직원들을 대상으로 한 테스트에서 처음 도입되었습니다. 해당 날짜부터 자격을 갖춘 트윗은 30분 동안 최대 5번까지 편집할 수 있습니다. [트윗 편집](/ko/x-api/fundamentals/edit-posts)에 대해 더 알아보세요.
X API v2를 사용하면 개발자는 다음을 확인할 수 있습니다:

* 트윗이 생성 시점에 편집 가능했는지 여부. 설문조사가 포함된 트윗이나 예약된 트윗과 같이 일부 트윗은 편집할 수 없습니다.
* 트윗은 30분 동안 편집 가능하며, 최대 5번까지 편집할 수 있습니다. 편집 가능한 트윗의 경우 남은 편집 가능 시간과 추가로 몇 번 더 편집할 수 있는지 확인할 수 있습니다.
* 현재 보고 있는 트윗이 편집된 버전인지 여부(대부분의 경우, 특정 과거 버전을 트윗 ID로 요청하지 않는 한 API는 트윗의 최신 버전을 반환합니다).
* 해당 트윗의 전체 편집 내역.
* 트윗의 각 버전에 귀속된 참여도(engagement).

<div id="track-threaded-conversations">
  ## 스레드 대화 추적
</div>

새로운 트윗 필드는 트윗이 어떤 대화 스레드에 속하는지 식별하는 데 도움이 됩니다. Conversation ID는 해당 대화를 시작한 트윗의 트윗 ID입니다. 자세한 내용은 [대화 추적](/ko/x-api/fundamentals/conversation-id)을 참고하세요.

<div id="ready-to-migrate">
  ## 마이그레이션할 준비가 되었나요?
</div>

v2 엔드포인트를 사용하려면 다음이 필요합니다.

* [개발자 계정](https://developer.x.com/en/portal/petition/essential/basic-info)
* [프로젝트](resources/fundamentals/projects) 내에 생성된 [개발자 App](https://developer.x.com/en/apps)
* 해당 프로젝트의 개발자 App에서 발급된 [키와 토큰](resources/fundamentals/authentication)

프로젝트 내 App의 키와 토큰을 사용하는 것이 중요합니다. 프로젝트 외부의 App에서 발급된 키와 토큰을 사용하는 경우, v2 엔드포인트로 요청을 보낼 수 없습니다.

개발자 계정을 보유하고 있다면, [developer portal](https://developer.x.com/en/portal/petition/essential/basic-info)에서 위의 모든 항목을 설정할 수 있습니다.

<div id="authentication">
  ### 인증
</div>

새로운 X API에서는 서로 다른 엔드포인트에 접근하기 위해 OAuth 1.0a 사용자 컨텍스트와 OAuth 2.0 Bearer Token이라는 두 가지 인증 방식을 사용합니다. 각 방식은 엔드포인트에 요청을 보낼 때 서로 다른 용도로 사용됩니다.

OAuth 1.0a 사용자 컨텍스트는 특정 X 사용자를 대신하여 요청을 보낼 때 필요합니다.\
OAuth 2.0 Bearer Token은 개발자 앱을 대신하여 요청을 보낼 때 필요합니다.

<div id="tools-and-code">
  ## 도구 및 코드
</div>

새로운 엔드포인트와 기능에 익숙해지고 작업을 시작할 수 있도록, 몇 가지 시작 옵션을 제공하고 있습니다.

* Postman 클라이언트를 사용하여 개별 엔드포인트에 요청을 보내고 연결할 수 있도록 하는 Twitter [Postman 컬렉션](https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400)을 제공합니다. 이는 인증을 테스트하고 엔드포인트를 시험해 볼 수 있는 간편한 방법입니다.
* 또한 Ruby, Python, Node, Java 등 여러 언어에 대해 Twitter에서 지원하는 라이브러리와 서드파티 라이브러리 목록도 제공하고 있습니다. 자세한 내용은 [도구 및 라이브러리 페이지](/ko/x-api/tools-and-libraries/overview)를 참고하세요.

<div id="migrating-to-the-new-data-format">
  ## 새로운 데이터 형식으로 마이그레이션하기
</div>

v1.1 또는 Enterprise에서 v2로 마이그레이션할 때는, 데이터가 전달되는 형식이 크게 변경되었다는 점을 이해하는 것이 중요합니다. 새 필드를 추가하고, 필드 순서를 변경했으며, 일부 요소는 완전히 제거했습니다.

이러한 변경 사항을 보다 잘 이해할 수 있도록, v2 이전 데이터 형식의 필드를 새 필드에 매핑하는 방법과 이 새 필드를 요청하는 방법을 설명하는 일련의 가이드를 제공하고 있습니다.

자세한 내용은 이 마이그레이션 허브의 [데이터 형식 마이그레이션](/ko/x-api/migrate/data-format-migration) 섹션을 참조하거나, 아래의 개별 데이터 형식 가이드를 참고하십시오.

* [Native format에서 X API v2로 (표준 v1.1)](/ko/x-api/migrate/data-format-migration#migrating-from-standard-v1-1s-data-format-to-v2)
* [Native Enriched에서 X API v2로 (Enterprise)](/ko/x-api/migrate/data-format-migration#migrating-from-native-enriched-data-format-to-v2)
* [Activity Streams에서 X API v2로 (Enterprise)](/ko/x-api/migrate/data-format-migration#migrating-from-activity-streams-data-format-to-v2)
