<div id="introduction">
  ## Introduction
</div>

X API v2가 출시되면서 새로운 데이터 응답 형식과 객체 및 필드를 요청하는 새로운 방식을 도입했으며, 이를 간단히 X API v2 형식이라고 부르고 있습니다. 

‘General differences’ 섹션에서는 Standard 및 Enterprise 사용자 모두와 관련된 몇 가지 변경 사항을 확인할 수 있습니다. 또한 [standard v1.1 Native format](https://developer.x.com/en/docs/x-api/v1/data-dictionary/overview), Enterprise [Native Enriched format](/ko/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#native-enriched-tweet-object), Enterprise [Activity Streams format](/ko/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#activity-object)에 대한 별도의 가이드를 제공하며, 여기에서는 필드 매핑 방법과 새로운 v2 필드를 요청하기 위해 어떤 필드와 expansions를 사용해야 하는지 설명합니다. 

* [Native format에서 X API v2로 (standard v1.1)](/ko/x-api/migrate/data-format-migration#migrating-from-standard-v1-1s-data-format-to-v2) 
* [Native Enriched에서 X API v2로 (enterprise)](/ko/x-api/migrate/data-format-migration#migrating-from-native-enriched-data-format-to-v2)
* [Activity Streams에서 X API v2로 (enterprise)](/ko/x-api/migrate/data-format-migration#migrating-from-activity-streams-data-format-to-v2)

[X API v1.1 데이터 형식](https://developer.x.com/en/docs/x-api/v1/data-dictionary/overview)과 [X API v2 형식](/ko/x-api/introduction) 간의 차이를 빠르게 확인하는 데 도움이 되는 [시각적 데이터 형식 마이그레이션 도구](/ko/x-api/migrate/data-format-migration#visual-data-format-migration-tool)도 참고하실 수 있습니다.

<div id="general-differences">
  ### 전반적인 차이점
</div>

<div id="requesting-objects-and-fields">
  #### 객체 및 필드 요청
</div>

v2 이전 엔드포인트와 v2 간의 가장 큰 변화 중 하나는, 새 버전(v2)은 기본적으로 소수의 필드만 반환하는 반면 standard, premium, Enterprise 엔드포인트는 대부분의 필드를 기본적으로 제공한다는 점입니다. 새 버전에서는 [fields](/ko/x-api/fundamentals/fields) 및 [expansions](/ko/x-api/fundamentals/expansions) 파라미터를 사용해 기본값을 넘어서는 추가 데이터를 명시적으로 요청합니다. 이를 통해 중요하지 않은 필드를 모두 수집할 필요 없이 필요한 데이터만 요청할 수 있습니다. 

요청한 필드가 기본 데이터 객체와 관련된 경우, 기본값과 함께 해당 기본 데이터 객체 안에 반환됩니다. 그러나 expansions 파라미터를 사용해 확장된 객체를 요청하면, 보조 객체는 새 includes 객체에 반환됩니다. includes 객체에 있는 확장된 객체는 양쪽에 함께 반환되는 id 필드를 사용해 기본 객체와 매칭할 수 있습니다.

예를 들어 v2 [Post lookup](/ko/x-api/posts/lookup/introduction) 엔드포인트를 사용하면서 요청에 expansions=author&#95;id 파라미터를 포함하면, 기본 Post 객체 안에서 author&#95;id 필드를 받게 되며, 동시에 includes 객체 안에서 Post당 하나의 user 객체를 받게 됩니다. 각각의 user 객체에는 Post 객체와 다시 매칭하는 데 사용할 수 있는 기본 id 필드가 포함됩니다. 다음은 그 예시입니다:

```
{
  "data": [
    {
      "author_id": "2244994945",
      "id": "1397568983931392004",
      "text": "The Twitter Developer Platform. Ooh la la! https://t.co/iGTdPXBfOv https://t.co/Ze8z8EODdg"
    }
  ],
  "includes": {
    "users": [
      {
        "id": "2244994945",
        "name": "Twitter Dev",
        "username": "TwitterDev"
      }
    ]
  }
}
```

<div id="updated-json-design">
  #### 업데이트된 JSON 디자인
</div>

특정 필드를 요청하는 방식의 변경 사항 외에도, X API v2는 API가 반환하는 객체(예: [Post](/ko/x-api/fundamentals/data-dictionary#tweet) 및 [user](/ko/x-api/fundamentals/data-dictionary#user) 객체)에 대해 새로운 JSON 디자인을 도입하고 있습니다.

* JSON 루트 레벨에서 기존 표준 엔드포인트는 Post 객체를 **statuses** 배열로 반환하는 반면, X API v2는 이를 **data** 배열로 반환합니다.
* 리트윗 및 인용을 “statuses”로 지칭하는 대신, X API v2 JSON은 리트윗 및 인용된 트윗이라고 표현합니다. **contributors** 및 **user.translator&#95;type**와 같은 많은 레거시 및 사용 중단 필드는 제거되고 있습니다.
* **favorites**(Post 객체 내)와 **favourites**(user 객체 내)를 모두 사용하는 대신, X API v2는 용어를 like로 통일합니다.
* X는 값이 없는 JSON 값(예: null)은 페이로드에 기록하지 않는 규칙을 채택하고 있습니다. Post 및 user 속성은 null이 아닌 값을 가질 때에만 포함됩니다. 
   

<div id="new-v2-fields">
  #### 새로운 v2 필드
</div>

또한 [Post object](/ko/x-api/fundamentals/data-dictionary#tweet)에 다음을 포함한 새로운 필드 집합을 도입했습니다.

* [conversation&#95;id](/ko/x-api/fundamentals/conversation-id) 필드
* context 및 entities를 포함하는 두 개의 새로운 [annotations](/ko/x-api/fundamentals/post-annotations) 필드
* 여러 개의 새로운 [metrics](/ko/x-api/fundamentals/metrics) 필드
* 특정 게시물에 누가 답글을 달 수 있는지 보여주는 새로운 reply&#95;setting 필드

<div id="migrating-from-standard-v11s-data-format-to-v2">
  ### 표준 v1.1 데이터 포맷에서 v2로 마이그레이션하기
</div>

아직 읽어 보지 않았다면, 먼저 [데이터 포맷 마이그레이션](/ko/x-api/migrate/data-format-migration) 소개 문서를 읽을 것을 권장합니다. 또한 [시각적 데이터 포맷 마이그레이션 도구](/ko/x-api/migrate/data-format-migration#visual-data-format-migration-tool)를 사용하면 [X API v1.1 데이터 포맷](https://developer.x.com/en/docs/x-api/v1/data-dictionary/overview)과 [X API v2 포맷](/ko/x-api/fundamentals/data-dictionary) 간의 차이를 빠르게 확인할 수 있습니다.

네이티브 포맷이라고도 불리는 표준 v1.1 데이터 포맷은 [standard v1.1](https://developer.x.com/en/docs/twitter-api/v1) 엔드포인트에서 사용되는 기본 데이터 포맷입니다.

Premium 제품을 사용 중이라면 [native enriched 가이드](/ko/x-api/migrate/data-format-migration#migrating-from-native-enriched-data-format-to-v2)를 참조하세요. Enterprise 고객은 Gnip 콘솔에서의 설정 방식에 따라 native enriched 또는 activity streams 포맷을 사용하고 있을 수 있습니다.

Native Enriched 데이터 포맷은 당사의 [enterprise](/ko/x-api/enterprise-gnip-2.0/enterprise-gnip) 제품에서 사용됩니다.

Native Enriched 데이터 포맷은 편집된 트윗 메타데이터를 제공하도록 업데이트되었습니다. Edit Tweet 메타데이터에 대해 더 알아보려면 [Edit Tweets fundamentals](/ko/x-api/enterprise-gnip-2.0/fundamentals/edit-tweets) 페이지를 참고하세요.

표준 v1.1 엔드포인트를 사용 중이라면 [standard v1.1 to v2 guide](/ko/x-api/migrate/data-format-migration#migrating-from-standard-v1-1s-data-format-to-v2)를 참고하세요. Activity Streams를 사용하는 enterprise 제품을 사용 중이라면 [Activity Streams to v2](/ko/x-api/migrate/data-format-migration#migrating-from-activity-streams-data-format-to-v2) 가이드도 제공하고 있습니다.

X API v2는 [트윗](/ko/x-api/fundamentals/data-dictionary#tweet) 및 [사용자](/ko/x-api/fundamentals/data-dictionary#user) 오브젝트에 대해 새로운 JSON 설계를 도입했습니다.

* JSON 루트 레벨에서 Native Enriched 포맷은 트윗 오브젝트를 results 배열로 반환하는 반면, X API v2는 data 배열로 반환합니다. 
* Tweet 오브젝트의 favorites 및 user 오브젝트의 favourites를 모두 사용하는 대신, X API v2는 like라는 용어를 사용합니다. 
* X는 값이 없는 JSON 값(예: null)은 페이로드에 쓰지 않는 규칙을 채택하고 있습니다. 트윗과 사용자 속성은 null이 아닌 값을 가질 때에만 포함됩니다. 
* v2의 모든 id 필드는 문자열 형식입니다.
   

새로운 JSON 포맷으로 변경된 내용 외에도, 트윗 오브젝트에 다음을 포함한 새로운 필드 집합이 도입되었습니다:

* conversation&#95;id
* reply&#95;settings
* 미디어의 alt&#95;text
* context 및 entities를 포함하는 두 개의 새로운 [annotations](/ko/x-api/fundamentals/post-annotations) 필드
* 여러 개의 새로운 [metrics](/ko/x-api/fundamentals/metrics) 필드
* 여러 개의 새로운 [polls](/ko/x-api/fundamentals/data-dictionary#poll) 필드
   

많은 레거시 및 사용 중단(deprecated) 필드는 제거됩니다:

* contributors
* 특정 entities.media 및 extended&#95;entities.media 필드
* filter&#95;level
* timestamp&#95;ms
* truncated

Activity Streams 데이터 포맷은 당사의 [enterprise](/ko/x-api/enterprise-gnip-2.0/enterprise-gnip) 제품에서 사용할 수 있습니다.

Activity Streams 데이터 포맷은 편집된 트윗 메타데이터를 제공하도록 업데이트되었습니다. Edit Tweet 메타데이터에 대해 더 알아보려면 [Edit Tweets 기본 사항](/ko/x-api/enterprise-gnip-2.0/fundamentals/edit-tweets) 페이지를 참고하세요.

표준 v1.1 엔드포인트를 사용하는 경우 [표준 v1.1에서 v2로 마이그레이션 가이드](/ko/x-api/migrate/overview)를 참고하세요. Premium 엔드포인트 또는 Enterprise용 Native Enriched 포맷을 사용하는 경우 [Native Enriched에서 v2로 마이그레이션 가이드](/ko/x-api/migrate/data-format-migration#migrating-from-native-enriched-data-format-to-v2)를 참고하세요.

X API v2는 [게시물](/ko/x-api/fundamentals/data-dictionary#tweet) 및 [user](/ko/x-api/fundamentals/data-dictionary#user) 객체에 대한 새로운 JSON 구조를 도입합니다.

* JSON 루트 레벨에서 Activity Streams 포맷은 results 배열에 트윗 객체를 반환하는 반면, X API v2는 data 배열을 반환합니다. 
* Retweeted 및 Quoted &quot;activities&quot;를 참조하는 대신, X API v2 JSON은 리트윗(Retweeted) 및 인용(Quoted)된 트윗을 참조합니다. 
* Tweet 객체에서 favorites, user 객체에서 favourites를 모두 사용하는 대신, X API v2는 like라는 용어를 사용합니다. 
* Twitter는 값이 없는 JSON 값(예: null)은 페이로드에 기록하지 않는 규칙을 채택하고 있습니다. 트윗 및 user 속성은 null이 아닌 값을 가진 경우에만 포함됩니다. 
* v2의 모든 id 필드는 문자열(string) 형식입니다.
   

새 JSON 포맷에 적용된 변경 사항 외에도, Tweet 객체에 다음과 같은 새로운 필드 세트를 도입했습니다.

* conversation&#95;id
* reply&#95;settings
* 미디어의 alt&#95;text
* context 및 entities를 포함한 두 개의 새로운 [주석](/ko/x-api/fundamentals/post-annotations) 필드
* 여러 개의 새로운 [메트릭](/ko/x-api/fundamentals/metrics) 필드
* 여러 개의 새로운 [polls](/ko/x-api/fundamentals/data-dictionary#poll) 필드
   

많은 레거시 및 사용 중단(deprecated) 필드가 제거되거나 대체됩니다.

* display&#95;text&#95;range
* generator
* gnip
* link
* objectType
* provider
* twitter&#95;entities.symbols는 data.entities.cashtags로 대체됩니다.
* 일부 twitter&#95;extended&#95;entities.media 및 twitter&#95;entities.media 필드
* twitter&#95;filter&#95;level
* twitterTimeZone
* verb
