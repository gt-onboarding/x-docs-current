이 가이드는 X Activity API 엔드포인트를 사용하여 이벤트를 구독하고 수신하는 방법을 설명합니다. 일반적으로 다음 세 단계로 진행합니다:

1. 필터링하려는 이벤트가 발생하는 User의 User ID를 확인합니다.
2. 해당 User에 대해 필터링하려는 이벤트 유형에 대한 구독을 생성합니다.
3. webhook 또는 지속적인 HTTP 스트림 연결을 사용하여 이벤트를 수신합니다.

<div id="getting-user-ids">
  ## 사용자 ID 가져오기
</div>

구독을 생성하기 전에 필터를 적용하려는 계정의 사용자 ID를 알고 있어야 합니다. 이 예제에서는 XDevelopers 핸들을 사용하겠습니다. 사용자 ID를 조회하는 방법에는 다음과 같이 여러 가지가 있습니다:

**사용자 이름으로 사용자 ID 조회:**

```bash
curl -H "Authorization: Bearer YOUR_BEARER_TOKEN" "https://api.x.com/2/users/by/username/xdevelopers"
```

**내 사용자 ID 가져오기:**

```bash
curl -H "Authorization: Bearer YOUR_BEARER_TOKEN" https://api.x.com/2/users/me
```

두 엔드포인트 모두 `id` 필드를 포함한 사용자 정보를 반환하며, 이 `id`를 구독 필터에서 사용할 수 있습니다. 아래는 예시 JSON 응답입니다.

```json
{
    "data": {
        "id": "2244994945",
        "name": "Developers",
        "username": "XDevelopers"
    }
}
```


<div id="creating-a-subscription">
  ## 구독 생성하기
</div>

다음 단계는 구독을 생성하는 것입니다. 이 예제에서는 XDevelopers의 프로필 소개(bio) 업데이트에 구독합니다. 이를 위해 JSON 본문에 `user_id`와 `event_type`을 전달합니다. 이 경우 `event_type`은 `ProfileBioUpdate`입니다.

X Developer의 사용자 ID `2244994945`와 선택적 태그를 함께 포함합니다.

```json
{
  "event_type": "ProfileBioUpdate",
  "filter": {
    "user_id": "2244994945"
  },
  "tag": "Xdevelopers' bio updates"
}
```

XAA와 관련된 모든 엔드포인트 호출은 developer portal에서 발급받은 [bearer token](https://docs.x.com/fundamentals/authentication/oauth-2-0/overview#bearer-token-also-known-as-app-only)으로 인가를 수행합니다.

```bash
curl -H "Authorization: Bearer YOUR_BEARER_TOKEN" \
  https://api.x.com/2/activity/subscriptions \
  -X POST \
  -d '{
    "event_type": "ProfileBioUpdate",
    "filter": {
      "user_id": "2244994945"
    },
    "tag": "Xdevelopers' bio updates"
  }'
```

요청이 성공적으로 처리되면 구독이 생성됩니다.

```json
{
  "data":[
    {
      "created_at":"2025-10-09T16:35:08.000Z",
      "event_type":"ProfileBioUpdate",
      "filter":{
        "user_id":"2244994945"
      },
      "subscription_id":"1976325569252868096",
      "tag": "X개발자 약력 업데이트",
      "updated_at":"2025-10-09T16:35:08.000Z"
    }
  ],
  "meta": {
    "total_subscriptions": 1
  }
}
```


<div id="getting-the-events">
  ## 이벤트 수신
</div>

구독을 생성한 후에는 [webhooks](https://docs.x.com/x-api/webhooks/introduction) 또는 지속적인 HTTP 스트림을 통해 이벤트를 수신할 수 있습니다. 이 예제에서는 지속적인 HTTP 스트림을 엽니다:

```bash
curl -H "Authorization: Bearer YOUR_BEARER_TOKEN" https://api.x.com/2/activity/stream
```

Xdevelopers 계정에서 프로필 소개(바이오) 정보를 업데이트하면, 해당 이벤트가 스트림으로 전송됩니다:

```json
{
  "data": {
    "filter": {
      "user_id": "2244994945"
    },
    "event_type": "ProfileBioUpdate",
    "tag": "Xdevelopers의 약력 업데이트",
    "payload": {
      "before": "Mars & Cars",
      "after": "Mars, Cars & AI"
    }
  }
}
```


<div id="subscription-management">
  ## 구독 관리
</div>

X Activity API는 표준 CRUD 연산을 통해 구독을 관리하기 위한 엔드포인트를 제공합니다.

<div id="create-subscription">
  ### 구독 생성
</div>

이벤트를 수신하기 위한 새 구독을 생성합니다.

```bash
curl -H "Authorization: Bearer YOUR_BEARER_TOKEN" \
  -X POST \
  https://api.x.com/2/activity/subscriptions \
  -d '{
    "event_type": "ProfileBioUpdate",
    "filter": {
      "user_id": "123456789"
    },
    "tag": "my bio updates",
    "webhook_id": "1976325569252868099"
  }'
```

**참고:**

* `tag` 필드는 선택 사항입니다. 이 필드는 이벤트가 전달될 때 식별하는 데 사용할 수 있습니다.
* `webhook_id` 필드도 선택 사항입니다. webhook 설정 방법은 [webhook 문서](https://docs.x.com/x-api/webhooks/introduction)를 참고하세요. `webhook_id`가 지정된 경우, 스트림이 열려 있다면 이벤트는 해당 스트림뿐만 아니라 지정된 webhook으로도 전달됩니다.

**응답:**

```json
{
  "data": {
    "subscription_id": "1976325569252868096",
    "event_type": "ProfileBioUpdate",
    "filter": {
      "user_id": "123456789"
    },
    "created_at": "2025-10-09T16:35:08.000Z",
    "updated_at": "2025-10-09T16:35:08.000Z",
    "tag": "my bio updates",
    "webhook_id": "1976325569252868099"
  }
}
```


<div id="list-subscriptions">

**응답:**



<div id="delete-subscription">
  ### 구독 삭제
</div>

구독을 삭제합니다:

```bash
curl -H "Authorization: Bearer YOUR_BEARER_TOKEN" \
  -X DELETE \
  https://api.x.com/2/activity/subscriptions/1976325569252868096
```

**응답:**

```json
{
  "data": {
    "deleted": true
  },
  "meta": {
    "total_subscriptions": 0
  }
}
```

`total_subscriptions`는 삭제 작업 이후 앱과 연결된 구독의 남은 수를 표시합니다.


<div id="update-subscription">
  ### 구독 업데이트
</div>

PUT 엔드포인트를 사용하면 구독의 전송 방식 또는 태그를 업데이트할 수 있습니다.

`filter` 또는 `event_type`를 변경하려면 기존 구독을 삭제하고 새로 추가해야 합니다.

```
curl -H "Authorization: Bearer YOUR_BEARER_TOKEN" \
  -X PUT \
  https://api.x.com/2/activity/subscriptions/1976325569252868096 \
  -d '{
    "tag": "my new tag",
    "webhook_id": "192846273860294839"
  }'
```

**응답:**

```json
{
  "data": {
    "subscription_id": "1976325569252868096",
    "event_type": "ProfileBioUpdate",
    "filter": {
      "user_id": "123456789"
    },
    "created_at": "2025-10-09T16:35:08.000Z",
    "updated_at": "2025-10-10T17:10:58.000Z",
    "tag": "my new tag",
    "webhook_id": "192846273860294839"
  },
  "meta": {
    "total_subscriptions": 1
  }
}
```
