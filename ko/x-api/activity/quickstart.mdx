이 가이드는 X Activity API 엔드포인트를 사용하여 이벤트를 구독하고 수신하는 방법을 설명합니다. 일반적으로 다음 세 단계를 거칩니다:

1. 필터링하려는 이벤트의 대상이 되는 사용자의 사용자 id를 확인합니다.
2. 해당 사용자에 대해 필터링하려는 이벤트 유형에 대한 구독을 생성합니다.
3. webhook 또는 지속적인 HTTP 스트리밍 연결을 사용하여 이벤트를 수신합니다.

<div id="getting-user-ids">
  ## 사용자 ID 가져오기
</div>

구독을 생성하기 전에 필터링할 계정의 사용자 ID를 알고 있어야 합니다. 이 예제에서는 XDevelopers 핸들을 사용합니다. 사용자 ID는 다음과 같은 여러 방법으로 조회할 수 있습니다:

**사용자 이름으로 사용자 ID 조회:**

```bash
curl -H "Authorization: Bearer YOUR_BEARER_TOKEN" "https://api.x.com/2/users/by/username/xdevelopers"
```

**내 사용자 id 가져오기:**

```bash
curl -H "Authorization: Bearer YOUR_BEARER_TOKEN" https://api.x.com/2/users/me
```

두 엔드포인트 모두 구독 필터에서 사용할 수 있는 `id` 필드를 포함한 사용자 정보를 반환합니다. 아래는 예시 JSON 응답입니다.

```json
{
    "data": {
        "id": "2244994945",
        "name": "Developers",
        "username": "XDevelopers"
    }
}
```


<div id="creating-a-subscription">
  ## Subscription 생성
</div>

다음 단계는 Subscription을 생성하는 것입니다. 이 예제에서는 XDevelopers 계정의 bio 업데이트를 구독합니다. 이를 위해 JSON body에 `user_id`와 `event_type`을 포함해 전달합니다. 이 경우 `event_type`은 `ProfileBioUpdate`입니다.

X Developers의 user ID인 `2244994945`와 선택적인 tag를 전달하겠습니다:

```json
{
  "event_type": "ProfileBioUpdate",
  "filter": {
    "user_id": "2244994945"
  },
  "tag": "Xdevelopers' bio updates"
}
```

XAA 관련 모든 엔드포인트에 대한 인가를 위해 developer portal에서 발급받은 [Bearer Token](https://docs.x.com/fundamentals/authentication/oauth-2-0/overview#bearer-token-also-known-as-app-only)을 사용합니다.

```bash
curl -H "Authorization: Bearer YOUR_BEARER_TOKEN" \
  https://api.x.com/2/activity/subscriptions \
  -X POST \
  -d '{
    "event_type": "ProfileBioUpdate",
    "filter": {
      "user_id": "2244994945"
    },
    "tag": "Xdevelopers' bio updates"
  }'
```

요청이 성공적으로 처리되면 구독이 생성됩니다:

```json
{
  "data":[
    {
      "created_at":"2025-10-09T16:35:08.000Z",
      "event_type":"ProfileBioUpdate",
      "filter":{
        "user_id":"2244994945"
      },
      "subscription_id":"1976325569252868096",
      "tag": "X개발자 약력 업데이트",
      "updated_at":"2025-10-09T16:35:08.000Z"
    }
  ],
  "meta": {
    "total_subscriptions": 1
  }
}
```


<div id="getting-the-events">
  ## 이벤트 가져오기
</div>

구독을 생성한 후에는 [webhooks](https://docs.x.com/x-api/webhooks/introduction) 또는 지속적인 HTTP 스트림을 통해 이벤트를 수신할 수 있습니다. 이 예제에서는 지속적인 HTTP 스트림을 엽니다:

```bash
curl -H "Authorization: Bearer YOUR_BEARER_TOKEN" https://api.x.com/2/activity/stream
```

Xdevelopers 계정이 프로필 소개를 업데이트하면, 이 이벤트는 스트림을 통해 전달됩니다:

```json
{
  "data": {
    "filter": {
      "user_id": "2244994945"
    },
    "event_type": "ProfileBioUpdate",
    "tag": "Xdevelopers의 약력 업데이트",
    "payload": {
      "before": "Mars & Cars",
      "after": "Mars, Cars & AI"
    }
  }
}
```


<div id="subscription-management">
  ## 구독 관리
</div>

X Activity API는 구독을 표준 CRUD 작업으로 관리할 수 있는 엔드포인트를 제공합니다.

<div id="create-subscription">
  ### 구독 생성
</div>

이벤트를 수신하기 위한 새 구독을 생성합니다.

```bash
curl -H "Authorization: Bearer YOUR_BEARER_TOKEN" \
  -X POST \
  https://api.x.com/2/activity/subscriptions \
  -d '{
    "event_type": "ProfileBioUpdate",
    "filter": {
      "user_id": "123456789"
    },
    "tag": "my bio updates",
    "webhook_id": "1976325569252868099"
  }'
```

**참고:**

* `tag` 필드는 선택 사항입니다. 이 필드는 이벤트가 전달될 때 식별하는 데 도움이 되도록 사용할 수 있습니다.
* `webhook_id` 필드도 선택 사항입니다. webhook 설정 방법은 [webhook 문서](https://docs.x.com/x-api/webhooks/introduction)를 참고하세요. `webhook_id`가 지정된 경우, 이벤트는 스트림이 열려 있는 경우 스트림뿐만 아니라 지정된 webhook으로도 전달됩니다.

**응답:**

```json
{
  "data": {
    "subscription_id": "1976325569252868096",
    "event_type": "ProfileBioUpdate",
    "filter": {
      "user_id": "123456789"
    },
    "created_at": "2025-10-09T16:35:08.000Z",
    "updated_at": "2025-10-09T16:35:08.000Z",
    "tag": "my bio updates",
    "webhook_id": "1976325569252868099"
  }
}
```

<div id="delete-subscription">
  ### 구독 삭제
</div>

구독을 삭제합니다.

```bash
curl -H "Authorization: Bearer YOUR_BEARER_TOKEN" \
  -X DELETE \
  https://api.x.com/2/activity/subscriptions/1976325569252868096
```

**응답:**

```json
{
  "data": {
    "deleted": true
  },
  "meta": {
    "total_subscriptions": 0
  }
}
```

`total_subscriptions` 는 삭제 작업을 수행한 후 앱에 연결된 남은 구독 수를 나타냅니다.


<div id="update-subscription">
  ### 구독 업데이트
</div>

PUT 엔드포인트를 사용하면 구독의 전달 방식 또는 태그를 업데이트할 수 있습니다.

`filter` 또는 `event_type`를 업데이트하려면 기존 구독을 삭제한 후 새 구독을 추가해야 합니다.

```
curl -H "Authorization: Bearer YOUR_BEARER_TOKEN" \
  -X PUT \
  https://api.x.com/2/activity/subscriptions/1976325569252868096 \
  -d '{
    "tag": "my new tag",
    "webhook_id": "192846273860294839"
  }'
```

**응답:**

```json
{
  "data": {
    "subscription_id": "1976325569252868096",
    "event_type": "ProfileBioUpdate",
    "filter": {
      "user_id": "123456789"
    },
    "created_at": "2025-10-09T16:35:08.000Z",
    "updated_at": "2025-10-10T17:10:58.000Z",
    "tag": "my new tag",
    "webhook_id": "192846273860294839"
  },
  "meta": {
    "total_subscriptions": 1
  }
}
```
