<div id="getting-started-with-the-manage-lists-endpoint-group">
  ## List 관리 엔드포인트 그룹 시작하기
</div>

이 간단한 개요에서는 [Postman](/ko/tutorials/postman-getting-started)을 사용하여 List 관리 엔드포인트에 첫 번째 요청을 보내는 방법을 설명합니다.

여러 언어로 된 샘플 코드를 보고 싶다면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요. 

**참고:** 이 예제에서는 *Create a List* 엔드포인트에 요청을 보내겠지만, 여기에서 배운 내용을 다른 List 관리 요청에도 동일하게 적용할 수 있습니다.

<Note>
  **사전 준비 사항**

  이 가이드를 완료하려면 요청을 인증하는 데 사용할 [keys and tokens](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따르면 이러한 키와 토큰을 생성할 수 있습니다:

  * [개발자 계정에 가입](https://developer.x.com/en/apply-for-access)하고 승인을 받습니다.
  * 개발자 포털에서 [프로젝트](/ko/resources/fundamentals/projects)와 연결된 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성합니다.
  * 앱의 “Keys and tokens” 페이지로 이동하여 필요한 자격 증명을 생성합니다. 모든 자격 증명을 안전한 위치에 저장했는지 확인하세요.
</Note>

<div id="steps-to-build-a-manage-list-request">
  ### Manage List 요청을 구성하는 단계
</div>

**1단계: 도구 또는 라이브러리 선택**

이 엔드포인트에 요청을 보내는 데 사용할 수 있는 다양한 도구, 코드 예제, 라이브러리가 있지만, 여기에서는 프로세스를 단순화하기 위해 Postman 도구를 사용합니다.

X API v2 Postman 컬렉션을 환경에 불러오려면 아래 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2를 Postman에 추가
</Button>

Postman에 X API v2 컬렉션을 불러온 후, “List” 폴더로 이동하여 “Manage List” 폴더를 선택한 다음 &quot;Create a List&quot;를 선택합니다.

**2단계: 요청 인증하기**

X API에 올바르게 요청을 보내려면, 해당 작업을 수행할 권한이 있는지 확인해야 합니다. 게시물 관리(manage Posts) 엔드포인트에서 이를 수행하려면, [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용하여 요청을 인증해야 합니다.

이 예시에서는 OAuth 1.0a User Context를 사용합니다.

Postman에 키와 토큰(특히 API Key, API Secret Key, OAuth 1.0a 사용자 Access Token, OAuth 1.0a 사용자 Access Token Secret)을 추가해야 합니다. 이를 위해 Postman 우측 상단에서 “X API v2”라는 이름의 환경을 선택한 뒤, 환경 드롭다운 옆의 눈 아이콘을 클릭하여 &quot;initial value&quot; 및 &quot;current value&quot; 필드에 키와 토큰을 추가합니다.

이 작업을 올바르게 완료했다면, 이 변수들은 요청의 authorization 탭으로 자동으로 불러와집니다.
 
**3단계: 새 List의 이름 지정**

이 엔드포인트로 새 List를 생성할 때, List 이름은 필수 body 파라미터입니다. 선택적으로 설명을 추가하고 List의 비공개 여부를 지정할 수 있습니다.

Postman에서 “Body” 탭으로 이동한 후, name 파라미터의 값으로 List 이름을 입력합니다. 추가로, List에 대한 설명을 추가하려면 name과 동일한 방식으로 description이라는 새 키를 추가한 다음, 값으로 List 설명을 입력하면 됩니다. List를 비공개로 설정하는 경우에도 동일한 패턴을 따르지만, 이 파라미터에는 true 또는 false 값만 허용됩니다. 

|             |                    |                    |
| :---------- | :----------------- | :----------------- |
| **Key**     | **Value**          | **Parameter type** |
| `name`      | 리스트 이름 (필수)        | body               |
| description | 리스트 설명 (선택)        | body               |
| private     | true 또는 false (선택) | body               |

이제 &quot;Send&quot; 버튼 옆에 다음과 비슷한 URL이 표시되는 것을 볼 수 있습니다:

`https://api.x.com/2/lists`

**4단계: 요청을 전송하고 응답 검토하기**

모든 설정을 마쳤다면, &quot;Send&quot; 버튼을 클릭하면 아래 예시 응답과 유사한 응답을 받게 됩니다:

```
{
  "data": {
    "id": "1441162269824405510",
    "name": "New list created from Postman"
  }
}
```

반환된 응답 객체에 id와 리스트 이름이 포함되어 있다면 리스트가 성공적으로 생성된 것입니다. 

리스트를 삭제하려면 Postman에 로드된 X API v2 컬렉션의 “Lists” 폴더에 있는 “Delete a List” 요청을 선택합니다. 이 엔드포인트에는 삭제하려는 리스트의 ID가 필요합니다. “Params” 탭에서 삭제하려는 리스트의 ID를 id 열의 값으로 입력합니다. 

삭제 요청이 성공적으로 처리되면 다음 예시와 유사한 응답을 받게 됩니다.

```
{
  "data": {
    "deleted": true
  }
}
```
