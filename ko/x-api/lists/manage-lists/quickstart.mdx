<div id="getting-started-with-the-manage-lists-endpoint-group">
  ## List 관리 endpoint 그룹 시작하기
</div>

이 간단한 개요에서는 [Postman](/ko/tutorials/postman-getting-started)을 사용해 manage List endpoints에 첫 번째 요청을 보내는 방법을 알아봅니다.

여러 언어로 된 샘플 코드를 보고 싶다면 GitHub의 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) 저장소를 방문하세요. 

**Note:** 이 예제에서는 *Create a List* endpoint에 요청을 보내지만, 여기서 익힌 내용은 다른 List 관리용 요청에도 그대로 적용할 수 있습니다.

<Note>
  **사전 준비 사항**

  이 가이드를 완료하려면 요청을 인증하기 위한 [keys and tokens](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따라 이러한 키와 토큰을 생성할 수 있습니다:

  * [개발자 계정에 가입](https://developer.x.com/en/apply-for-access)하고 승인을 받으세요.
  * developer portal에서 [Project](/ko/resources/fundamentals/projects)와 연결된 [developer App](/ko/resources/fundamentals/developer-apps)을 생성하세요.
  * App의 “Keys and tokens” 페이지로 이동해 필요한 자격 증명을 생성하세요. 생성한 모든 자격 증명은 안전한 위치에 보관하세요.
</Note>

<div id="steps-to-build-a-manage-list-request">
  ### manage List 요청을 구성하는 단계
</div>

**1단계: 도구 또는 라이브러리 선택**

이 엔드포인트에 요청을 보내기 위해 사용할 수 있는 다양한 도구, 코드 예제, 라이브러리가 있지만, 여기서는 과정을 단순화하기 위해 Postman 도구를 사용하겠습니다.

X API v2 Postman 컬렉션을 Postman 환경으로 가져오려면, 다음 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2를 Postman에 추가
</Button>

Postman에 X API v2 컬렉션을 불러온 후, “List” 폴더로 이동해 “Manage List” 폴더를 선택한 다음, &quot;Create a List&quot;를 선택합니다.

**2단계: 요청 인증**

X API에 적절하게 요청을 보내려면, 해당 요청을 보낼 권한이 있는지 검증해야 합니다. manage Posts 엔드포인트에서 이를 수행하려면, [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용해 요청을 인증해야 합니다.

이 예제에서는 OAuth 1.0a User Context를 사용하겠습니다.

Postman에 키와 토큰(특히 API Key, API Secret Key, OAuth 1.0a 사용자 액세스 토큰, OAuth 1.0a 사용자 액세스 토큰 시크릿)을 추가해야 합니다. 이를 위해 Postman의 오른쪽 상단에서 “X API v2”라는 이름의 환경을 선택한 다음, 환경 드롭다운 옆의 눈 모양 아이콘을 클릭해 &quot;initial value&quot; 및 &quot;current value&quot; 필드에 키와 토큰을 추가합니다.

이 작업을 올바르게 수행했다면, 이 변수들은 요청의 Authorization 탭에 자동으로 반영됩니다.
 
**3단계: 새 List의 이름 지정**

이 엔드포인트로 새 List를 생성할 때, List의 이름은 필수 본문(body) 파라미터입니다. 선택적으로 설명을 제공하고, List를 비공개로 할지 여부를 지정할 수 있습니다.

Postman에서 “Body” 탭으로 이동한 후, name 파라미터의 값으로 List의 이름을 입력합니다. 추가로 List에 대한 설명을 추가하고 싶다면, name과 같은 방식으로 description이라는 새 key를 추가한 뒤, 그 값으로 List의 설명을 입력하면 됩니다. List를 비공개로 설정하는 것도 동일한 패턴을 따르지만, 이 파라미터에는 true 또는 false 값만 허용됩니다. 

|             |                       |                    |
| :---------- | :-------------------- | :----------------- |
| **Key**     | **Value**             | **Parameter type** |
| `name`      | 리스트의 이름(필수)           | body               |
| description | 리스트에 대한 설명(선택 사항)     | body               |
| private     | true 또는 false (선택 사항) | body               |

이제 &quot;Send&quot; 버튼 옆에 다음과 비슷한 URL이 표시되는 것을 볼 수 있습니다:

`https://api.x.com/2/lists`

**4단계: 요청 보내기 및 응답 확인**

모든 설정을 마쳤다면, &quot;Send&quot; 버튼을 클릭하세요. 그러면 다음 예시 응답과 유사한 응답을 받게 됩니다:

```
{
  "data": {
    "id": "1441162269824405510",
    "name": "New list created from Postman"
  }
}
```

반환된 response 객체에 id와 리스트 이름이 포함되어 있다면, 리스트가 성공적으로 생성된 것입니다. 

리스트를 삭제하려면 Postman에 로드된 X API v2 컬렉션의 “Lists” 폴더에 있는 “Delete a List” 요청을 선택합니다. 이 엔드포인트에는 삭제하려는 리스트의 ID가 필요합니다. “Params” 탭에서 삭제하려는 리스트의 ID를 id 열의 값으로 입력합니다. 

삭제 요청이 성공하면, 다음 예시와 유사한 response를 받습니다.

```
{
  "data": {
    "deleted": true
  }
}
```
