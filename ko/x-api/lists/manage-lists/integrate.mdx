<div id="postman">

</div>

Postman은 엔드포인트를 테스트해 볼 수 있는 유용한 도구입니다. 각 Postman 요청에는 사용 가능한 항목을 빠르게 파악할 수 있도록 모든 경로 및 본문(body) 매개변수가 포함되어 있습니다. Postman 컬렉션에 대해 더 알아보려면 [&quot;Postman 사용하기&quot;](/ko/tutorials/postman-getting-started) 페이지를 방문하세요.

<div id="authentication">
  #### 인증
</div>

모든 X API v2 엔드포인트에 요청을 보내려면 키와 토큰이라고도 하는 자격 증명 세트를 사용해 인증해야 합니다. 

이 특정 엔드포인트들은 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication)를 사용해야 하며, 이는 요청을 성공적으로 보내기 위해 API 키 세트와 사용자 액세스 토큰을 함께 사용해야 함을 의미합니다. 액세스 토큰은 요청을 대신 수행하려는 해당 사용자와 연결되어 있어야 합니다. 다른 사용자의 액세스 토큰 세트를 생성하려는 경우, 그 사용자는 [3-legged OAuth 플로우](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)를 통해 귀하의 앱을 승인하거나 인증해야 합니다.

OAuth 1.0a는 사용하기 까다로울 수 있다는 점에 유의하십시오. 이 인증 방식에 익숙하지 않다면, [라이브러리](/ko/x-api/tools-and-libraries/overview) 또는 Postman 같은 도구를 사용해 요청을 올바르게 인증할 것을 권장합니다.
 

<div id="developer-portal-projects-and-developer-apps">
  #### 개발자 포털, 프로젝트, 그리고 개발자 앱
</div>

X API v2 엔드포인트에서 사용할 인증 자격 증명 세트를 얻으려면, 먼저 [개발자 계정에 가입](https://developer.x.com/en/portal/petition/essential/basic-info)하고 해당 계정 내에 [프로젝트](/ko/resources/fundamentals/projects)를 설정한 다음, 그 프로젝트 안에 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성해야 합니다. 그런 다음 개발자 앱 내에서 키와 토큰을 확인할 수 있습니다.
 

<div id="rate-limits">
  #### 요청 한도
</div>

매일 수천 명이 넘는 개발자들이 X API에 요청을 보냅니다. 이러한 요청의 방대한 양을 관리하기 위해 각 endpoint에는 [요청 한도](/ko/x-api/fundamentals/rate-limits)가 설정되어 있으며, 이는 앱 또는 인증된 사용자를 대신해 보낼 수 있는 요청 수를 제한합니다. 

이 endpoint에는 사용자 단위의 요청 한도가 적용됩니다. 즉, 어떤 개발자 앱을 사용하더라도, 대신 요청을 보내는 인증된 사용자는 해당 endpoint를 일정 횟수까지만 호출할 수 있습니다. 

아래 표는 각 endpoint에 대한 요청 한도를 보여줍니다.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **HTTP method** | **요청 한도** |
| /2/lists | POST | 15분당 300회 요청 |
| /2/lists/:id | DELETE / PUT | 15분당 300회 요청 |
