<div id="postman">
  #### Postman
</div>

Postman은 엔드포인트를 테스트해 볼 수 있는 매우 유용한 도구입니다. 각 Postman 요청에는 사용 가능한 모든 path 및 body 파라미터가 포함되어 있어, 어떤 항목이 제공되는지 빠르게 파악하는 데 도움이 됩니다. Postman 컬렉션에 대한 자세한 내용은 [&quot;Postman 사용 방법&quot;](/ko/tutorials/postman-getting-started) 페이지를 방문하세요.

<div id="authentication">
  #### 인증
</div>

모든 X API v2 엔드포인트는 키와 토큰이라고도 하는 자격 증명 세트를 사용해 요청을 인증해야 합니다. 

이 엔드포인트들은 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication)를 사용해야 하며, 이는 성공적인 요청을 위해 API 키와 사용자 액세스 토큰 세트를 사용해야 함을 의미합니다. 액세스 토큰은 요청을 대신하려는 대상 사용자의 계정과 연결되어 있어야 합니다. 다른 사용자의 액세스 토큰 세트를 생성하려면, 해당 사용자가 [3-legged OAuth 플로우](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)를 통해 앱을 승인하거나 인증해야 합니다.

OAuth 1.0a는 사용하기 까다로울 수 있습니다. 이 인증 방식에 익숙하지 않다면, [라이브러리](/ko/x-api/tools-and-libraries/overview)나 Postman 같은 도구를 사용해 요청을 올바르게 인증할 것을 권장합니다.
 

<div id="developer-portal-projects-and-developer-apps">
  #### developer portal, 프로젝트, 그리고 개발자 앱
</div>

X API v2 엔드포인트와 함께 사용할 수 있는 인증 자격 증명 세트를 얻으려면 먼저 [개발자 계정에 가입](https://developer.x.com/en/portal/petition/essential/basic-info)하고, 해당 계정 내에 [프로젝트](/ko/resources/fundamentals/projects)를 설정한 뒤, 그 프로젝트 안에 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성해야 합니다. 그런 다음 해당 개발자 앱에서 키와 토큰을 확인할 수 있습니다.
 

<div id="rate-limits">
  #### 요청 한도
</div>

매일 수천 명의 개발자가 X API에 요청을 보냅니다. 이렇게 많은 요청량을 관리하기 위해 각 엔드포인트에는 [요청 한도](/ko/x-api/fundamentals/rate-limits)가 설정되어 있어, 앱 또는 인증된 사용자를 대신해 보낼 수 있는 요청 수를 제한합니다. 

이 엔드포인트들은 사용자 단위로 요청 한도가 적용됩니다. 이는 여러분이 요청을 대신 보내는 인증된 사용자가 어떤 개발자 앱을 사용하더라도, 해당 엔드포인트를 호출할 수 있는 횟수에 공통으로 한도가 적용된다는 의미입니다. 

아래 표는 각 엔드포인트의 요청 한도를 보여줍니다.

|     |     |     |
| :--- | :--- | :--- |
| **엔드포인트** | **HTTP 메서드** | **요청 한도** |
| /2/lists | POST | 15분당 300회 요청 |
| /2/lists/:id | DELETE / PUT | 15분당 300회 요청 |
