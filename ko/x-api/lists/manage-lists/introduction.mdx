[X Lists](https://help.x.com/en/using-twitter/twitter-lists)를 사용하면 사용자가 타임라인에서 보게 되는 게시물을 맞춤 설정하고, 정리하며, 우선순위를 지정할 수 있습니다. Lists 엔드포인트를 사용하면 사용자가 선호도, 관심사, 그룹 또는 주제에 따라 게시물을 선별하고 구성할 수 있도록 하는 솔루션을 구축할 수 있습니다.

리스트 관리 엔드포인트로 사용자를 대신해 요청을 보내는 경우, 이 엔드포인트는 반드시 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)로 인증해야 하며, 여러분의 앱을 승인한 사용자와 연결된 사용자 액세스 토큰을 사용해야 합니다. 이 토큰은 [3-legged OAuth flow](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) 또는 [Authorization Code with PKCE grant flow](/ko/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce-2) (OAuth 2.0)를 사용해 생성할 수 있습니다.

<div id="manage-lists">
  ### 리스트 관리
</div>

리스트 관리 엔드포인트를 사용하면 인증된 사용자를 대신해 리스트를 생성, 삭제 및 업데이트할 수 있습니다. 이 엔드포인트에서는 POST, DELETE, PUT 세 가지 메서드를 사용할 수 있습니다. POST 메서드는 리스트를 생성하는 데 사용되며, DELETE 메서드는 리스트를 삭제하는 데, PUT 메서드는 리스트의 메타데이터를 업데이트하는 데 사용됩니다. 세 엔드포인트 모두에 대해 사용자별 15분당 최대 300회까지 요청할 수 있습니다.

참고로, 계정당 최대 1000개의 리스트를 생성할 수 있습니다.

**계정 설정**

이 엔드포인트에 액세스하려면 다음이 필요합니다.

* 승인된 [개발자 계정](https://developer.x.com/en/portal/petition/essential/basic-info)
* [프로젝트](/ko/resources/fundamentals/projects) 내에 있는 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용한 인증

X API v2 엔드포인트에 액세스하는 방법에 대한 자세한 내용은 [시작 가이드](/ko/x-api/getting-started/getting-access)를 참조하세요.
