[X 리스트](https://help.x.com/en/using-twitter/twitter-lists)는 사용자가 타임라인에 표시되는 게시물을 맞춤 설정하고, 정리하고, 우선순위를 지정할 수 있게 해 줍니다. Lists 엔드포인트를 사용하면 사람들이 선호도, 관심사, 그룹 또는 토픽을 기반으로 게시물을 큐레이션하고 정리할 수 있도록 하는 솔루션을 구축할 수 있습니다.

manage List 엔드포인트로 사용자를 대신하여 요청을 보내는 것이므로, 이 엔드포인트는 반드시 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용해 인증해야 하며, 여러분의 앱을 승인한 사용자와 연결된 액세스 토큰을 사용해야 합니다. 이 토큰은 [3-legged OAuth 플로우](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) 또는 [Authorization Code with PKCE grant flow](/ko/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)) (OAuth 2.0)을 사용해 생성할 수 있습니다.

<div id="manage-lists">
  ### 리스트 관리
</div>

리스트 관리 엔드포인트를 사용하면 인증된 사용자를 대신해 리스트를 생성, 삭제 및 업데이트할 수 있습니다. 이 엔드포인트에서는 POST, DELETE, PUT 세 가지 메서드를 사용할 수 있습니다. POST 메서드는 리스트를 생성할 때, DELETE 메서드는 리스트를 삭제할 때, PUT 메서드는 리스트의 메타데이터를 업데이트할 때 사용합니다. 이 세 엔드포인트 모두에 대해 사용자당 15분에 300회 요청 한도가 적용됩니다.

참고로, 계정당 최대 1000개의 리스트를 생성할 수 있습니다.

**계정 설정**

이 엔드포인트에 액세스하려면 다음이 필요합니다.

* 승인된 [개발자 계정](https://developer.x.com/en/portal/petition/essential/basic-info)
* [프로젝트](/ko/resources/fundamentals/projects) 내에 있는 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용한 인증

X API v2 엔드포인트에 대한 액세스 권한을 얻는 방법에 대해 더 알아보려면 [시작하기 가이드](/ko/x-api/getting-started/getting-access)를 참고하세요.
