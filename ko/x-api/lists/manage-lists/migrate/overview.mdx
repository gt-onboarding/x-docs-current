<div id="comparing-x-apis-lists-endpoints">
  ## X API Lists 엔드포인트 비교
</div>

v2 List 관리 엔드포인트는 최종적으로 [POST lists/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-create), [POST lists/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-destroy), [POST lists/update](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-update)를 대체하게 됩니다. 이전 버전의 이러한 엔드포인트를 사용하는 코드, 앱 또는 도구가 있고, 새로운 X API v2로의 마이그레이션을 고려하고 있다면 이 가이드를 참고하십시오.

다음 표는 표준 v1.1과 X API v2 List 엔드포인트를 비교합니다:

**List 생성**

| 설명 | 표준 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 지원 HTTP 메서드 | POST | POST |
| 호스트 도메인 | https://api.x.com | https://api.x.com |
| 엔드포인트 경로 | /1.1/lists/create.json | /2/lists |
| [인증](/ko/resources/fundamentals/authentication) | OAuth 1.0a 사용자 컨텍스트 | OAuth 1.0a 사용자 컨텍스트 |
| 기본 [요청 한도](/ko/resources/fundamentals/rate-limits) | 없음 | 15분당 300개 요청 (사용자당) |

**List 삭제**

| 설명 | 표준 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 지원 HTTP 메서드 | POST | DELETE |
| 호스트 도메인 | https://api.x.com | https://api.x.com |
| 엔드포인트 경로 | /1.1/lists/destroy.json | /2/lists/:id |
| [인증](/ko/resources/fundamentals/authentication) | OAuth 1.0a 사용자 컨텍스트 | OAuth 1.0a 사용자 컨텍스트 |
| 기본 [요청 한도](/ko/resources/fundamentals/rate-limits) | 없음 | 15분당 300개 요청 (사용자당) |

**List 업데이트**

| 설명 | 표준 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 지원 HTTP 메서드 | POST | PUT |
| 호스트 도메인 | https://api.x.com | https://api.x.com |
| 엔드포인트 경로 | /1.1/lists/update.json | /2/lists/:id |
| [인증](/ko/resources/fundamentals/authentication) | OAuth 1.0a 사용자 컨텍스트 | OAuth 1.0a 사용자 컨텍스트 |
| 기본 [요청 한도](/ko/resources/fundamentals/rate-limits) | 없음 | 15분당 300개 요청 (사용자당) |

X API v2 엔드포인트에 액세스하려면 [개발자 계정에 가입](https://developer.x.com/en/portal/petition/essential/basic-info)해야 합니다. 인증 시에는 [프로젝트](/ko/resources/fundamentals/projects) 내에 있는 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용해야 합니다.

X API v2 엔드포인트에 대한 액세스 권한을 얻는 방법에 대해 더 알아보려면 [시작하기](/ko/x-api/getting-started/getting-access) 페이지를 참조하십시오.
