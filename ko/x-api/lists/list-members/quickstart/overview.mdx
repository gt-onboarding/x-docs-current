<div id="getting-started-with-the-list-members-endpoint-group">
  ## List 멤버 엔드포인트 그룹 시작하기
</div>

이 간단한 개요는 [Postman](/ko/tutorials/postman-getting-started)을 사용하여 List 멤버 엔드포인트 그룹에 첫 요청을 보내는 데 도움이 됩니다.

다양한 언어의 샘플 코드를 보고 싶다면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요.

<Note>
  **사전 준비 사항**

  이 가이드를 완료하려면 요청을 인증할 [키와 토큰](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따라 이러한 키와 토큰을 생성할 수 있습니다.

  * [개발자 계정에 가입](https://developer.x.com/en/apply-for-access)하고 승인을 받으세요.
  * 개발자 포털에서 [Project](/ko/resources/fundamentals/projects)와 연결된 [developer App](/ko/resources/fundamentals/developer-apps)을 생성하세요.
  * App의 “Keys and tokens” 페이지로 이동하여 필요한 자격 증명을 생성하세요. 모든 자격 증명을 안전한 위치에 저장했는지 확인하세요.
</Note>

<div id="authenticate-your-request">
  #### 요청 인증하기
</div>

**조회(lookup)** 엔드포인트를 성공적으로 호출하려면 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication#oauth-1-0a-2), [앱 전용(App only)](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only), 또는 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 중 하나를 사용할 수 있습니다. 반면 **관리(manage)** 엔드포인트의 경우 OAuth 1.0a 사용자 컨텍스트 또는 OAuth 2.0 Authorization Code with PKCE로만 인증할 수 있습니다.

Postman을 사용하는 경우, 환경을 올바르게 설정했다면 기본 인증 키와 토큰이 요청에 자동으로 설정됩니다.

이를 위해 Postman 우측 상단에서 “X API v2”라는 이름의 환경을 선택한 다음, 환경 드롭다운 옆의 눈 모양 아이콘을 클릭해 &quot;initial value&quot;와 &quot;current value&quot; 필드에 키와 토큰을 입력합니다. 여기에 포함되는 키는 다음과 같습니다:

* API Key (Consumer Key라고도 함)
* API Secret Key (Consumer Secret이라고도 함)
* OAuth 1.0a 사용자 Access Token
* OAuth 1.0a 사용자 Access Token Secret
* App only Access Token
* OAuth 2.0 Client Key (App 설정에서 OAuth 2.0 User Authentication 설정을 구성한 경우에만 제공됨)
* OAuth 2.0 Client Secret (App 설정에서 OAuth 2.0 User Authentication 설정을 구성한 경우에만 제공됨)
