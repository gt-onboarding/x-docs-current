<div id="steps-to-build-a-list-members-lookup-request">
  ### List 멤버 조회 요청을 구성하는 단계
</div>

**1단계: Postman에서 List 엔드포인트 컬렉션 선택**

Postman에 X API v2 컬렉션을 불러온 후 “List” 폴더로 이동한 다음, 그 안의 “List members” 폴더를 선택하고 &quot;Members lookup&quot;을 선택합니다.
 

**2단계: 멤버를 조회하려는 List를 식별하고 지정**

멤버를 가져오려는 List를 지정해야 합니다. x.com으로 이동해 원하는 List를 클릭한 후, URL에서 List ID를 확인할 수 있습니다. 예를 들어, 다음 URL의 List ID는 84839422입니다.

https://x.com/i/lists/84839422

대상 ID로는 유효한 아무 List ID나 사용할 수 있습니다. Postman에서 &quot;Params&quot; 탭으로 이동한 뒤, id 경로 변수의 &quot;Value&quot; 열에 ID를 입력합니다. ID 앞이나 뒤에 공백이 포함되지 않도록 주의하세요.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 84839422 (List ID) |

**3단계: 조회하려는 필드를 식별하고 지정**

3단계를 완료한 뒤 &quot;Send&quot; 버튼을 클릭하면, 응답에는 기본 [user object](/ko/x-api/fundamentals/data-dictionary#user) 필드인 id, name, username이 포함되어 반환됩니다.

id, name, username 외에 추가 필드를 받으려면, 요청에 [field](/ko/x-api/fundamentals/fields) 및/또는 [expansion](/ko/x-api/fundamentals/expansions) 파라미터를 사용해 해당 필드를 명시해야 합니다.

이 예제에서는 서로 다른 오브젝트에서 다음 세 가지 추가 필드 세트를 요청합니다.

* 기본 user object에서 추가 user.created&#95;at 필드
* expansion 파라미터를 사용한 전체 [Post object](/ko/x-api/fundamentals/data-dictionary#tweet)
* 연관된 Post object에서 추가 tweet.created&#95;at 필드

Postman에서 &quot;Params&quot; 탭으로 이동한 뒤, &quot;Query Params&quot; 테이블에 다음 key:value 쌍을 추가합니다:

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| user.fields | created&#95;at | `created_at` |
| expansions | pinned&#95;tweet&#95;id | `includes.tweets.id,  <br/>includes.tweets.text  <br/>` |
| tweet.fields | created&#95;at | `includes.tweets.created_at` |

이제 “Send” 버튼 옆에 다음과 유사한 URL이 표시되는 것을 볼 수 있습니다:

`https://api.x.com/2/lists/84839422/members?user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=created_at`
