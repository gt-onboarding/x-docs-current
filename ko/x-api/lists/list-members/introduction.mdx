<div id="list-members-lookup">
  ### 리스트 멤버 조회
</div>

멤버 조회 그룹에는 두 개의 엔드포인트가 있습니다. 지정된 리스트의 멤버에 대한 세부 정보를 조회하거나, 특정 사용자가 멤버로 포함된 리스트를 확인할 수 있습니다. 이 엔드포인트들은 멤버십 정보를 기반으로 사용자가 새로운 리스트를 큐레이션하고 구성할 수 있도록 하는 데 활용할 수 있습니다.

멤버 세부 정보를 조회할 때는 15분당 900회 요청 제한이 있으며, 사용자 멤버십을 조회할 때는 15분당 75회 요청 제한이 있습니다.

이 엔드포인트에 대한 요청을 인증하기 위해 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication), [App only](https://developer.x.com/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), 또는 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용할 수 있습니다. 
 

<div id="manage-list-members">
  ### 리스트 멤버 관리
</div>

리스트 멤버 관리 엔드포인트를 사용하면 인증된 사용자를 대신하여 리스트에 멤버를 추가하고 제거할 수 있습니다. 이 엔드포인트에서는 POST와 DELETE 두 가지 메서드를 사용할 수 있습니다. POST 메서드는 리스트에 멤버를 추가할 때 사용하며, DELETE 메서드는 리스트에서 멤버를 제거할 때 사용합니다. 두 엔드포인트 모두 15분당 300개의 요청이라는 사용자 기준 레이트 리밋이 적용됩니다.

리스트는 멤버가 5,000명을 초과할 수 없습니다.

이러한 엔드포인트로 사용자를 대신하여 요청을 수행하므로, [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용하여 사용자를 인증해야 하며, 앱을 승인한 사용자와 연결된 사용자 액세스 토큰을 사용해야 합니다. 이 토큰은 [3-legged OAuth 플로우](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) 또는 [Authorization Code with PKCE 그랜트 플로우](/ko/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0)를 사용해 생성할 수 있습니다.

<Note>
  **계정 설정**

  이 엔드포인트들에 액세스하려면 다음이 필요합니다:

  * 승인된 [개발자 계정](https://developer.x.com/en/portal/petition/essential/basic-info).
  * [프로젝트](/ko/resources/fundamentals/projects) 내에 위치한 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용해 인증할 수 있어야 합니다. 

  X API v2 엔드포인트에 대한 액세스 권한을 얻는 방법에 대해 자세히 알아보려면 [시작하기 가이드](/ko/x-api/getting-started/getting-access)를 참고하세요.
</Note>
