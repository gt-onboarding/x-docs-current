<div id="list-members-lookup">
  ### 리스트 멤버 조회
</div>

멤버 조회 그룹에는 사용할 수 있는 엔드포인트가 두 개 제공됩니다. 지정된 리스트의 멤버에 대한 상세 정보를 조회하고, 사용자가 어떤 리스트의 멤버인지 확인할 수 있습니다. 이 엔드포인트들을 통해 멤버십 정보를 바탕으로 사용자가 새로운 리스트를 선별하고 구성할 수 있습니다.

멤버 상세 정보를 조회할 때는 15분당 900회의 요청 한도가 있으며, 사용자 멤버십을 조회할 때는 15분당 75회의 요청 한도가 있습니다.

이 엔드포인트에 대한 요청을 인증하기 위해 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication), [App only](https://developer.x.com/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), 또는 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용할 수 있습니다. 
 

<div id="manage-list-members">
  ### 리스트 멤버 관리
</div>

리스트 멤버 관리 엔드포인트를 사용하면 인증된 사용자를 대신하여 리스트에 멤버를 추가하거나 제거할 수 있습니다. 이 엔드포인트에는 POST와 DELETE, 두 가지 메서드가 제공됩니다. POST 메서드는 리스트에 멤버를 추가할 때 사용하고, DELETE 메서드는 리스트에서 멤버를 제거할 때 사용합니다. 두 엔드포인트 모두 사용자 기준으로 15분당 300회의 요청 제한이 있습니다.

리스트는 멤버를 5,000명보다 많이 보유할 수 없다는 점에 유의하세요.

이 엔드포인트들을 사용해 사용자를 대신하여 요청을 수행하므로, [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 중 하나로 사용자를 인증해야 하며, 귀하의 앱을 승인한 사용자와 연결된 사용자 액세스 토큰을 사용해야 합니다. 이러한 토큰은 [3-legged OAuth flow](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) 또는 [Authorization Code with PKCE grant flow](/ko/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0)를 사용해 생성할 수 있습니다.

<Note>
  **계정 설정**

  이 엔드포인트들에 액세스하려면 다음이 필요합니다.

  * 승인된 [개발자 계정](https://developer.x.com/en/portal/petition/essential/basic-info)
  * [프로젝트](/ko/resources/fundamentals/projects) 내에 위치한 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용한 인증

  X API v2 엔드포인트에 대한 액세스 권한을 얻는 방법은 [시작하기 가이드](/ko/x-api/getting-started/getting-access)에서 자세히 알아보세요.
</Note>
