<div id="postman">
  #### Postman
</div>

Postman은 엔드포인트를 테스트해 볼 수 있는 매우 유용한 도구입니다. 각 Postman 요청에는 모든 path 및 body 파라미터가 포함되어 있어, 어떤 기능을 사용할 수 있는지 빠르게 파악하는 데 도움이 됩니다. Postman 컬렉션에 대해 더 알아보려면 [&quot;Using Postman&quot;](/ko/tutorials/postman-getting-started) 페이지를 방문하세요.

<div id="authentication">
  #### 인증
</div>

모든 X API v2 엔드포인트에 요청을 보내려면, `키와 토큰(keys and tokens)`이라고도 불리는 자격 증명 세트로 요청을 인증해야 합니다. Lists **lookup** 엔드포인트에 대한 요청을 인증할 때는 OAuth 1.0a 사용자 컨텍스트, OAuth 2.0 Authorization Code with PKCE, 또는 App only 중 하나를 사용할 수 있습니다. 그러나 **manage** Lists 엔드포인트에는 반드시 OAuth 1.0a 사용자 컨텍스트 또는 OAuth 2.0으로 인증해야 합니다. 

[OAuth 1.0a User Context](/ko/resources/fundamentals/authentication)를 사용하는 경우, 요청을 성공적으로 보내기 위해서는 API Key 세트와 사용자 액세스 토큰을 사용해야 합니다. 액세스 토큰은 요청을 대신해 작업을 수행하려는 해당 사용자와 연관되어 있어야 합니다. 다른 사용자를 위한 액세스 토큰 세트를 생성하려면, 그 사용자가 [3-legged OAuth flow](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)를 사용해 귀하의 앱을 승인해야 합니다.

OAuth 1.0a는 사용하기 어려울 수 있다는 점에 유의하세요. 이 인증 방법에 익숙하지 않다면, [library](/ko/x-api/tools-and-libraries/overview)를 사용하거나 Postman과 같은 도구를 사용하거나, OAuth 2.0 또는 App only를 사용해 요청을 인증할 것을 권장합니다.

[OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)는 애플리케이션의 스코프와 여러 디바이스에 걸친 인가 플로우를 보다 세밀하게 제어할 수 있도록 해줍니다. OAuth 2.0을 사용하면 사용자 대신 특정 권한을 부여하는 세분화된 스코프를 선택할 수 있습니다. 

앱에서 OAuth 2.0을 사용하려면, developer portal의 App 설정 섹션에 있는 해당 App의 인증 설정에서 이를 활성화해야 합니다.

[App only](/ko/resources/fundamentals/authentication#oauth-2-0)는 요청에 [App only Access Token](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)을 포함하기만 하면 됩니다. App only Access Token은 개발자 앱 내에서 직접 생성하거나, [POST oauth2/token](/ko/resources/fundamentals/authentication#post-oauth2-token) 엔드포인트를 사용해 생성할 수 있습니다.

[App only](/ko/resources/fundamentals/authentication#oauth-2-0)는 요청에 [App only Access Token](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)을 포함하기만 하면 됩니다. App only Access Token은 개발자 앱 내에서 직접 생성하거나, [POST oauth2/token](/ko/resources/fundamentals/authentication#post-oauth2-token) 엔드포인트를 사용해 생성할 수 있습니다.

<div id="developer-portal-projects-and-developer-apps">
  #### Developer portal, 프로젝트, 개발자 앱
</div>

X API v2 엔드포인트에서 사용할 수 있는 인증 자격 증명 세트를 발급받으려면 [개발자 계정에 가입](https://developer.x.com/en/portal/petition/essential/basic-info)하고, 해당 계정 내에 [프로젝트](/ko/resources/fundamentals/projects)를 설정한 뒤, 그 프로젝트 내에 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성해야 합니다. 그런 다음 개발자 앱 내에서 키와 토큰을 확인할 수 있습니다.
 

<div id="rate-limits">
  #### 요청 한도
</div>

매일 수천 명의 개발자가 X API에 요청을 보냅니다. 이러한 요청의 방대한 양을 관리하기 위해, [요청 한도](/ko/x-api/fundamentals/rate-limits)가 각 엔드포인트에 설정되어 있으며, 이는 앱 또는 인증된 사용자를 대신해 보낼 수 있는 요청 수를 제한합니다. 

조회(GET) 엔드포인트에는 앱 수준과 사용자 수준 두 가지 요청 한도가 적용되며, 관리(POST/DELETE) 엔드포인트에는 사용자 수준 요청 한도만 적용됩니다. 앱 요청 한도는 개발자인 여러분이 특정 기간 동안 특정 앱(일반적으로 API Key 및 API Secret Key, 또는 앱 전용(App-only) 액세스 토큰을 사용하는 앱)에서 이 엔드포인트로 보낼 수 있는 요청 수에 제한이 있음을 의미합니다. 사용자 요청 한도는 여러분이 요청을 대신 보내는 인증된 사용자가, 어떤 개발자 앱을 사용하든 List 조회를 수행할 수 있는 횟수에 제한이 있음을 의미합니다.

아래 표는 각 엔드포인트에 대한 요청 한도를 보여줍니다.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **HTTP method** | **Rate limit** |
| /2/lists/:id/members | GET | 15분당 900개 요청 |
| /2/users/:id/list&#95;memberships | GET | 15분당 75개 요청 |
| /2/lists/:id/members | POST | 15분당 300개 요청 |
| /2/lists/:id/members/:user&#95;id | DELETE | 15분당 300개 요청 |

<div id="fields-and-expansions">
  #### 필드와 expansions
</div>

X API v2 GET 엔드포인트를 사용하면 `fields` 및 `expansions`라는 도구 세트를 사용해 API에서 어떤 데이터를 반환받을지 정확히 선택할 수 있습니다. `expansions` 파라미터를 사용하면 페이로드에서 참조된 객체를 확장하여 포함할 수 있습니다. 예를 들어, 리스트 멤버 조회를 수행하면 다음과 같은 [expansions](/ko/x-api/fundamentals/expansions)을 요청할 수 있습니다:

* `pinned_tweet_id`

`fields` 파라미터를 사용하면 서로 다른 데이터 객체에서 어떤 [fields](/ko/x-api/fundamentals/fields)를 받을지 정확히 선택할 수 있습니다. 리스트 멤버 조회는 주로 user 객체를 반환합니다. 기본적으로 user 객체는 id, name, username 필드를 반환합니다. `user.created_at` 또는 `user.description`과 같은 추가 필드를 받으려면 user.fields 파라미터를 사용해 해당 필드들을 명시적으로 요청해야 합니다. 

[fields and expansions](/ko/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) 사용 방법에 대한 가이드를 제공했습니다.

아래 표는 각 lookup 엔드포인트에서 사용 가능한 필드와 expansions를 보여 줍니다:

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Fields** | **Expansions** |
| /2/lists/:id/members | `user.fields`<br /><br />`tweet.fields` | `pinned_tweet_id` |
| /2/users/:id/list&#95;memberships | list.fields<br /><br />user.fields | owner&#95;id |

<div id="pagination">
  #### 페이지네이션
</div>

멤버십/멤버를 조회하면 매우 많은 데이터가 반환될 수 있습니다. 언제나 일관되고 성능이 뛰어난 결과를 제공하기 위해 페이지네이션을 사용합니다. 페이지네이션은 단일 응답에 담을 수 있는 양보다 더 많은 결과를 반환하는 X API v2 엔드포인트에서 제공되는 기능입니다. 이런 경우 데이터는 여러 개의 “페이지”로 나누어 반환됩니다. [페이지네이션을 사용해 결과를 조회하는 방법](/ko/x-api/fundamentals/pagination)에 대해 자세히 알아보세요.
