<div id="authentication">
  #### 인증
</div>

모든 X API v2 엔드포인트는 키와 토큰으로도 알려진 자격 증명을 사용해 요청을 인증해야 합니다. Lists **lookup** 엔드포인트의 경우 OAuth 1.0a 사용자 컨텍스트, OAuth 2.0 Authorization Code with PKCE 또는 App only 중 하나를 사용해 요청을 인증할 수 있습니다. 그러나 Lists **manage** 엔드포인트에는 반드시 OAuth 1.0a 사용자 컨텍스트 또는 OAuth 2.0으로 인증해야 합니다. 

[OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication)는 요청을 성공적으로 수행하기 위해 API 키와 사용자 액세스 토큰 세트를 사용해야 함을 의미합니다. 액세스 토큰은 요청을 대신해 작업하려는 사용자와 연관되어 있어야 합니다. 다른 사용자를 위한 액세스 토큰 세트를 생성하려면, 해당 사용자가 [3-legged OAuth 플로우](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)를 사용해 App을 승인해야 합니다.

OAuth 1.0a는 사용하기 까다로울 수 있습니다. 이 인증 방식에 익숙하지 않다면, [라이브러리](/ko/x-api/tools-and-libraries/overview)를 사용하거나 Postman과 같은 도구를 사용하거나, OAuth 2.0 또는 App only를 사용해 요청을 인증할 것을 권장합니다.

[OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)는 애플리케이션의 스코프와 여러 기기에 걸친 인가 플로우를 더 세밀하게 제어할 수 있게 해줍니다. OAuth 2.0을 사용하면 사용자 대신 특정 권한을 부여하는 세분화된 스코프를 선택할 수 있습니다. 

App에서 OAuth 2.0을 사용하려면, developer portal의 App 설정 섹션에 있는 App의 인증 설정에서 OAuth 2.0을 활성화해야 합니다.

[App only](/ko/resources/fundamentals/authentication#oauth-2-0)는 요청과 함께 [App only 액세스 토큰](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)만 전달하면 됩니다. developer App 내에서 직접 App only 액세스 토큰을 생성하거나, [POST oauth2/token](/ko/resources/fundamentals/authentication#post-oauth2-token) 엔드포인트를 사용해 생성할 수 있습니다.

[App only](/ko/resources/fundamentals/authentication#oauth-2-0)는 요청과 함께 [App only 액세스 토큰](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)만 전달하면 됩니다. developer App 내에서 직접 App only 액세스 토큰을 생성하거나, [POST oauth2/token](/ko/resources/fundamentals/authentication#post-oauth2-token) 엔드포인트를 사용해 생성할 수 있습니다.

<div id="developer-portal-projects-and-developer-apps">
  #### developer portal, 프로젝트, 그리고 개발자 앱
</div>

X API v2 엔드포인트에서 사용할 수 있는 인증 자격 증명 세트를 발급받으려면, 먼저 [개발자 계정에 가입](https://developer.x.com/en/portal/petition/essential/basic-info)하고, 그 계정 내에 [프로젝트](/ko/resources/fundamentals/projects)를 만든 뒤, 해당 프로젝트 내에 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성해야 합니다. 이후 개발자 앱에서 키와 토큰을 확인할 수 있습니다.
 

<div id="rate-limits">
  #### 요청 한도
</div>

매일 수많은 개발자들이 X API에 요청을 보냅니다. 이러한 요청의 방대한 양을 관리하기 위해, 각 엔드포인트에는 앱 또는 인증된 사용자를 대신해 보낼 수 있는 요청 수를 제한하는 [요청 한도](/ko/x-api/fundamentals/rate-limits)가 설정되어 있습니다. 

조회(GET) 엔드포인트에는 앱 수준과 사용자 수준 두 가지 요청 한도가 적용되는 반면, 관리(POST/DELETE) 엔드포인트에는 사용자 수준 요청 한도만 적용됩니다. 앱 요청 한도는 개발자인 여러분이 특정 기간 동안 어떤 앱(해당 앱의 API Key 및 API Secret Key 또는 App only Access Token을 사용하는 것으로 간주)에서 이 엔드포인트로 보낼 수 있는 요청 수에 제한이 있음을 의미합니다. 사용자 요청 한도는 여러분이 요청을 대신 보내는 인증된 사용자가, 어떤 개발자 앱을 사용하든 상관없이 리스트 조회를 수행할 수 있는 횟수에 제한이 있음을 의미합니다.

아래 표는 각 엔드포인트의 요청 한도를 보여줍니다.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **HTTP method** | **Rate limit** |
| /2/lists/:id/members | GET | 15분당 900건의 요청 |
| /2/users/:id/list&#95;memberships | GET | 15분당 75건의 요청 |
| /2/lists/:id/members | POST | 15분당 300건의 요청 |
| /2/lists/:id/members/:user&#95;id | DELETE | 15분당 300건의 요청 |

<div id="fields-and-expansions">
  #### 필드와 expansions
</div>

X API v2 GET 엔드포인트에서는 `fields` 및 `expansions` 도구를 사용해 API에서 어떤 데이터를 반환할지 정확하게 선택할 수 있습니다. `expansions` 파라미터를 사용하면 페이로드에서 참조된 객체를 확장할 수 있습니다. 예를 들어, List 멤버 조회를 수행하면 다음 [expansions](/ko/x-api/fundamentals/expansions)을 가져올 수 있습니다:

* `pinned_tweet_id`

`fields` 파라미터를 사용하면 서로 다른 데이터 객체에서 어떤 [fields](/ko/x-api/fundamentals/fields)를 받을지 정확히 선택할 수 있습니다. List 멤버 조회는 주로 사용자 객체를 반환합니다. 기본적으로 user 객체는 id, name, username 필드를 반환합니다. `user.created_at` 또는 `user.description`과 같은 추가 필드를 받으려면 user.fields 파라미터를 사용해 명시적으로 요청해야 합니다. 

[fields and expansions](/ko/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)를 사용하는 방법에 대한 가이드를 제공하고 있습니다.

아래 표는 각 조회 엔드포인트에서 사용 가능한 필드와 expansions를 보여줍니다:

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Fields** | **Expansions** |
| /2/lists/:id/members | `user.fields`<br /><br />`tweet.fields` | `pinned_tweet_id` |
| /2/users/:id/list&#95;memberships | list.fields<br /><br />user.fields | owner&#95;id |

<div id="pagination">
  #### 페이지네이션
</div>

membership/members를 조회하면 매우 많은 데이터를 반환할 수 있습니다. 언제든지 일관되고 높은 성능의 결과를 제공하기 위해 페이지네이션을 사용합니다. 페이지네이션은 한 번의 응답으로 모두 반환할 수 없을 만큼 많은 결과를 반환하는 X API v2 엔드포인트에서 제공하는 기능입니다. 이런 경우 데이터는 일련의 “페이지”로 나누어 반환됩니다. 결과를 [페이지네이션하는 방법](/ko/x-api/fundamentals/pagination)에 대해 더 자세히 알아보세요.
