<div id="getting-started-with-the-list-lookup-endpoint">
  ## 리스트 조회 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 Postman을 사용해 리스트 조회 엔드포인트로 처음 요청을 보내는 방법을 설명합니다.

여러 언어로 된 샘플 코드를 보고 싶다면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요.

**참고:** 이 예제에서는 *List lookup by ID* 엔드포인트에 요청을 보내지만, 이 빠른 시작 가이드에서 배운 내용은 다른 조회 요청에도 동일하게 적용할 수 있습니다.

<Note>
  ### 사전 준비 사항

  이 가이드를 완료하려면 요청을 인증하기 위한 [키와 토큰](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따라 이러한 키와 토큰을 생성할 수 있습니다.

  * [개발자 계정에 가입](https://developer.x.com/en/apply-for-access)하고 승인을 받습니다.
  * developer portal에서 [프로젝트](/ko/resources/fundamentals/projects)와 연결된 [developer App](/ko/resources/fundamentals/developer-apps)을 생성합니다.
  * 앱의 “Keys and tokens” 페이지로 이동하여 필요한 자격 증명을 생성합니다. 모든 자격 증명은 안전한 위치에 보관해야 합니다.
</Note>

<div id="steps-to-build-a-list-lookup-request">
  ### List 조회 요청을 구성하는 단계
</div>

**1단계: 도구 또는 라이브러리 선택**

이 endpoint에 요청을 보내기 위해 사용할 수 있는 도구, 코드 예제, 라이브러리가 여러 가지 있지만, 여기서는 절차를 단순화하기 위해 Postman 도구를 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 불러오려면 다음 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2를 Postman에 추가
</Button>

Postman에 X API v2 컬렉션이 로드되면 “List” 폴더로 이동한 뒤, “List lookup” 폴더를 선택하고 &quot;List by ID&quot;를 선택합니다.
 

**2단계: 요청 인증**

X API에 올바르게 요청을 보내려면, 권한이 있는지 확인해야 합니다. 이 endpoint에서 이를 수행하려면 [App only](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), 또는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 인증 방법 중 하나로 요청을 인증해야 합니다.

이 예제에서는 단순화를 위해 App only를 사용하겠지만, 비공개 [metrics](/ko/x-api/fundamentals/metrics) 또는 List를 요청하려면 다른 인증 방법 중 하나를 사용해야 합니다. 

App only를 사용하려면, Postman에서 “X API v2”라는 이름의 환경(Postman 우측 상단)을 선택한 다음, 거기에 키와 토큰(특히 [App only Access Token](/ko/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token))을 &quot;initial value&quot; 및 &quot;current value&quot; 필드에 추가해야 합니다(환경 드롭다운 옆에 있는 눈 아이콘을 클릭하여 입력).

올바르게 설정했다면, 이러한 변수들이 요청의 authorization 탭에 자동으로 적용됩니다.
 

**3단계: 조회하려는 List 지정**

요청에서 조회하려는 List를 지정해야 합니다. x.com에서 특정 List를 클릭한 후 URL을 확인하여 List ID를 찾을 수 있습니다. 예를 들어, 아래 URL의 List ID는 84839422입니다.

https://x.com/i/lists/84839422

대상 ID는 유효한 어떤 List ID라도 사용할 수 있습니다. Postman에서 &quot;Params&quot; 탭으로 이동한 후, id 경로 변수의 &quot;Value&quot; 열에 ID 값을 입력하세요. ID 앞뒤에 공백이 포함되지 않도록 주의하세요.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| id  | 84839422 (해당 List의 ID) |

**4단계: 조회하려는 fields 지정**

3단계까지 완료한 뒤 &quot;Send&quot; 버튼을 클릭하면, 응답에서 기본 [List object](/ko/x-api/fundamentals/data-dictionary#list) 필드인 id, name을 받게 됩니다.

추가 필드를 받고 싶다면, 요청에 list.fields 및/또는 [expansions](/ko/x-api/fundamentals/expansions) 파라미터를 사용해 해당 필드를 지정해야 합니다.

이 연습에서는 서로 다른 객체에서 세 개의 추가 필드 세트를 요청하겠습니다.

* 기본 Lists 객체에서 created&#95;at 필드 하나를 추가합니다.
* expansions 파라미터를 사용해 전체 [user object](/ko/x-api/fundamentals/data-dictionary#user)를 요청합니다.
* 연관된 user 객체에서 user.created&#95;at 필드를 추가로 요청합니다.

Postman에서 &quot;Params&quot; 탭으로 이동한 다음, &quot;Query Params&quot; 표에 다음 key:value 쌍을 추가합니다:

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| list.fields | created&#95;at | created&#95;at |
| expansions | owner&#95;id | includes.users.id,  <br />includes.users.name,  <br />includes.users.username |
| user.fields | created&#95;at | includes.users.created&#95;at |

이제 “Send” 버튼 옆에 다음과 유사한 URL이 표시됩니다:

`https://api.x.com/2/lists/84839422?list.fields=owner_id&expansions=owner_id&user.fields=created_at`

**5단계: 요청 전송 및 응답 확인**

모든 설정을 마쳤다면 &quot;Send&quot; 버튼을 클릭하세요. 그러면 다음 예시와 유사한 응답을 받게 됩니다.

```
{
  "data": {
    "id": "84839422",
    "name": "Official Twitter Accounts",
    "owner_id": "783214"
  },
  "includes": {
    "users": [
      {
        "name": "Twitter",
        "created_at": "2007-02-20T14:35:54.000Z",
        "username": "Twitter",
        "id": "783214"
      }
    ]
  }
}
```
