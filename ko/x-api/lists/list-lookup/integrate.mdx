<div id="authentication">
  #### 인증
</div>

모든 X API v2 엔드포인트는 요청을 인증하기 위해 자격 증명 세트(‘키와 토큰’이라고도 함)가 필요합니다. 이러한 엔드포인트에 대한 요청을 인증할 때는 OAuth 1.0a 사용자 컨텍스트, 앱 전용(App only), 또는 OAuth 2.0 Authorization Code with PKCE 중 하나를 사용할 수 있습니다. 

[OAuth 1.0a User Context](/ko/resources/fundamentals/authentication#oauth-1-0a-user-context)는 요청을 성공적으로 보내기 위해 API 키와 사용자 액세스 토큰 세트를 사용해야 함을 의미합니다. 액세스 토큰은 요청을 대신 수행하려는 사용자와 연관되어 있어야 합니다. 다른 사용자를 위한 액세스 토큰 세트를 생성하려면, 해당 사용자가 [3-legged OAuth 플로우](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)를 통해 귀하의 앱을 승인해야 합니다.

OAuth 1.0a는 사용하기 까다로울 수 있습니다. 이 인증 방식에 익숙하지 않다면, [라이브러리](/ko/x-api/tools-and-libraries/overview)를 사용하거나, Postman과 같은 도구를 사용하거나, OAuth 2.0을 사용해 요청을 인증할 것을 권장합니다.

[OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)는 애플리케이션의 권한 범위(scope)와 여러 기기에 걸친 인가 플로우를 보다 세밀하게 제어할 수 있게 해줍니다. OAuth 2.0을 사용하면 사용자 대신 수행할 수 있는 권한을 세밀하게 제어하는, 보다 세분화된 특정 스코프를 선택할 수 있습니다. 

앱에서 OAuth 2.0을 활성화하려면, developer portal의 App 설정 섹션에 있는 해당 App의 인증 설정에서 OAuth 2.0을 활성화해야 합니다.

[App only](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)는 요청에 [App only Access Token](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)을 포함하기만 하면 됩니다. App only 토큰은 개발자 앱 내에서 직접 생성하거나, [POST oauth2/token](/ko/resources/fundamentals/authentication#post-oauth2-token) 엔드포인트를 사용해 생성할 수 있습니다.

<div id="developer-portal-projects-and-developer-apps">
  #### 개발자 포털, 프로젝트 및 개발자 앱
</div>

X API v2 엔드포인트에서 사용할 인증 자격 증명 세트를 발급받으려면 [개발자 계정에 가입](https://developer.x.com/en/portal/petition/essential/basic-info)하고, 해당 계정 내에 [프로젝트](/ko/resources/fundamentals/projects)를 설정한 뒤, 그 프로젝트 안에 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성해야 합니다. 그런 다음 개발자 앱에서 키와 토큰을 확인할 수 있습니다.
 

<div id="rate-limits">
  #### 요청 한도
</div>

매일 수많은 개발자들이 X API에 요청을 보냅니다. 이렇게 방대한 양의 요청을 관리하기 위해 각 엔드포인트에는 [요청 한도](/ko/x-api/fundamentals/rate-limits)가 설정되어 있으며, 이를 통해 앱이나 인증된 사용자를 대신하여 보낼 수 있는 요청 수가 제한됩니다. 

이 엔드포인트에는 앱 수준(App-level)과 사용자 수준(user-level) 모두에 요청 한도가 적용됩니다. 앱 요청 한도는 개발자인 여러분이 특정 기간 동안 특정 앱(일반적으로 API Key 및 API Secret Key, 또는 App-only 액세스 토큰을 사용하는 것으로 간주)에서 이 엔드포인트로 보낼 수 있는 최대 요청 수를 의미합니다. 사용자 요청 한도는 여러분이 요청을 대신 보내는 인증된 사용자가, 어떤 개발자 앱을 사용하든 상관없이 List 조회를 수행할 수 있는 최대 횟수를 의미합니다.

아래 표는 각 엔드포인트별 요청 한도를 보여줍니다.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **HTTP method** | **Rate limit** |
| /2/lists/:id | GET | 15분당 75회 요청 |
| /2/users/:id/owned&#95;lists | GET | 15분당 15회 요청 |

Fields와 expansions

X API v2 GET 엔드포인트를 사용하면 `fields` 및 `expansions`라고 불리는 도구 세트를 사용하여 API에서 반환받고자 하는 정확한 데이터를 선택할 수 있습니다. `expansions` 파라미터를 사용하면 페이로드에서 참조된 객체를 확장할 수 있습니다. 예를 들어, ID로 List를 조회할 때 다음 [expansions](/ko/x-api/fundamentals/expansions)을 요청할 수 있습니다:

* `owner_id`

`fields` 파라미터를 사용하면 수신하고자 하는 서로 다른 데이터 객체 내에서 정확히 어떤 [fields](/ko/x-api/fundamentals/fields)를 받을지 선택할 수 있습니다. 이 엔드포인트들은 주로 List 객체를 반환합니다. 기본적으로 List 객체는 `id` 및 `name` 필드를 반환합니다. `list.created_at` 또는 `list.follower_count`와 같은 추가 필드를 받으려면 list.fields 파라미터를 사용해 해당 필드들을 명시적으로 요청해야 합니다. 

[X API v2 데이터 사전](/ko/x-api/fundamentals/data-dictionary)에 [fields와 expansions](/ko/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)를 함께 사용하는 방법에 대한 가이드를 추가해 두었습니다.

아래 표는 이 엔드포인트 그룹에서 사용 가능한 fields와 expansions를 보여줍니다:

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Fields** | **Expansions** |
| /2/lists/:id | `list.fields`<br /><br />`user.fields` | `owner_id` |
| /2/users/:id/owned&#95;lists | list.fields<br /><br />user.fields | owner&#95;id |

<div id="pagination">
  #### Pagination
</div>

사용자가 소유한 리스트를 조회하면 매우 많은 양의 데이터가 반환될 수 있습니다. 언제나 일관되고 성능이 안정적인 결과를 제공하기 위해 페이지네이션을 사용합니다. 페이지네이션은 단일 응답에 담을 수 있는 것보다 더 많은 결과를 반환하는 X API v2 엔드포인트의 기능입니다. 이 경우 데이터는 여러 개의 ‘페이지’로 나누어 반환됩니다. 결과를 [페이지네이션으로 조회하는 방법](/ko/x-api/fundamentals/pagination)에 대해 더 알아보세요.
