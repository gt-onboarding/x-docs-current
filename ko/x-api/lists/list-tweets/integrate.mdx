<div id="authentication">
  #### 인증
</div>

모든 X API v2 엔드포인트는 `키와 토큰`이라고도 불리는 자격 증명 세트를 사용해 요청을 인증해야 합니다. 이 엔드포인트에 대한 요청을 인증할 때는 OAuth 1.0a 사용자 컨텍스트, App only, 또는 OAuth 2.0 Authorization Code with PKCE 중 하나를 사용할 수 있습니다. 

[OAuth 1.0a User Context](/ko/resources/fundamentals/authentication#oauth-1-0a-2)를 사용하는 경우, 성공적인 요청을 위해 API 키와 사용자 액세스 토큰 세트를 사용해야 합니다. 액세스 토큰은 요청을 대신해 작업하려는 해당 사용자와 연관되어 있어야 합니다. 다른 사용자를 위한 액세스 토큰 세트를 생성하려면, 그 사용자가 [3-legged OAuth 플로우](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)를 통해 귀하의 App을 승인해야 합니다.

OAuth 1.0a는 사용하기 까다로울 수 있다는 점에 유의하세요. 이 인증 방식에 익숙하지 않다면, [라이브러리](/ko/x-api/tools-and-libraries/overview)를 사용하거나 Postman과 같은 도구를 사용하거나, OAuth 2.0 또는 App only 방법을 사용해 요청을 인증할 것을 권장합니다.

[OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)는 애플리케이션의 스코프를 보다 정교하게 제어하고, 여러 기기에 걸친 인가 플로우를 지원합니다. OAuth 2.0을 사용하면 사용자 대신 수행할 수 있는 작업에 대해 세밀하게 정의된 스코프를 선택해, 특정 권한만 부여할 수 있습니다. 

App에서 OAuth 2.0을 사용하려면, developer portal의 App 설정 섹션에 있는 해당 App의 인증 설정에서 OAuth 2.0을 활성화해야 합니다.

[App only](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)는 요청에 [App only Access Token](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)을 포함해 전달하기만 하면 됩니다. App only 액세스 토큰은 개발자 앱 내에서 직접 생성하거나, [POST oauth2/token](/ko/resources/fundamentals/authentication#post-oauth2-token) 엔드포인트를 사용해 생성할 수 있습니다.

<div id="developer-portal-projects-and-developer-apps">
  #### developer portal, 프로젝트, 개발자 앱
</div>

X API v2 endpoint에서 사용할 수 있는 인증 자격 증명 세트를 발급받으려면, 먼저 [개발자 계정에 가입](https://developer.x.com/en/portal/petition/essential/basic-info)하고, 해당 계정 내에 [프로젝트](/ko/resources/fundamentals/projects)를 설정한 뒤, 그 프로젝트 안에 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성해야 합니다. 그런 다음 개발자 앱 내에서 키와 토큰을 확인할 수 있습니다.
 

<div id="rate-limits">
  #### Rate limits
</div>

매일 수많은 개발자가 X API에 요청을 보냅니다. 이러한 요청의 방대한 양을 관리하기 위해 각 엔드포인트에는 [요청 한도](https://developer.x.com/x-api/fundamentals/rate-limits)가 설정되어 있으며, 이는 앱 또는 인증된 사용자를 대신하여 보낼 수 있는 요청 수를 제한합니다. 

이 엔드포인트에는 App 수준과 사용자 수준의 요청 한도가 모두 적용됩니다. 앱 요청 한도는 개발자인 여러분이 특정 기간 동안 특정 App(API Key 및 API Secret Key, 또는 Bearer Token을 사용해 식별됨)에서 이 엔드포인트로 보낼 수 있는 요청 수가 제한된다는 의미입니다. 사용자 요청 한도는 여러분이 대신 요청을 수행하는 인증된 사용자가 모든 개발자 앱을 통틀어 List 게시물 조회를 수행할 수 있는 횟수가 제한된다는 의미입니다.

아래 표는 각 엔드포인트의 요청 한도를 보여줍니다.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **HTTP method** | **Rate limit** |
| /2/lists/:id/tweets | GET | 15분당 900회 요청 |

Fields and expansions

X API v2 GET 엔드포인트를 사용하면 `fields` 및 `expansions`라는 도구 세트를 사용해 API에서 반환받고자 하는 정확한 데이터를 선택할 수 있습니다. `expansions` 파라미터를 사용하면 페이로드에서 참조되는 객체를 확장할 수 있습니다. 예를 들어 List 게시물을 조회할 때는 다음과 같은 [expansions](/ko/x-api/fundamentals/expansions)을 가져올 수 있습니다:

* `author_id`

`fields` 파라미터를 사용하면 서로 다른 데이터 객체 내에서 수신하고자 하는 정확한 [필드](/ko/x-api/fundamentals/fields)를 선택할 수 있습니다. 이 엔드포인트는 주로 Post 객체를 반환합니다. 기본적으로 Post 객체는 `id` 및 `text` 필드를 반환합니다. `tweet.created_at` 또는 `tweet.lang` 같은 추가 필드를 받으려면 해당 필드를 fields 파라미터를 사용해 명시적으로 요청해야 합니다. 

[X API v2 데이터 사전](/ko/x-api/fundamentals/data-dictionary)에는 [fields 및 expansions 사용 방법](/ko/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)에 대한 가이드를 제공하고 있습니다.

아래 표는 lookup 엔드포인트에서 사용 가능한 fields 및 expansions를 보여줍니다:

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Fields** | **Expansions** |
| /2/lists/:id/tweets | `tweet.fields`<br /><br />`user.fields` | `author_id` |

Pagination

List 게시물을 조회하면 매우 많은 데이터가 반환될 수 있습니다. 언제나 일관되고 높은 성능의 결과를 반환하기 위해 X API v2 엔드포인트에서는 페이지네이션을 사용합니다. 페이지네이션은 단일 응답에 담을 수 있는 양보다 더 많은 결과를 반환하는 X API v2 엔드포인트에서 사용되는 기능입니다. 이 경우 데이터는 여러 ‘페이지’로 나누어 반환됩니다. 결과를 [페이지네이션을 통해 조회하는 방법](/ko/x-api/fundamentals/pagination)에 대해 더 알아보세요.
