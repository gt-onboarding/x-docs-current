<div id="getting-started-with-the-list-posts-lookup-endpoint">
  ## 리스트 게시물 조회 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 Postman을 사용해 List 게시물 조회 엔드포인트에 첫 요청을 보내는 방법을 안내합니다.

다양한 언어의 샘플 코드를 확인하려면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요.

<Note>
  ### 사전 준비 사항

  이 가이드를 완료하려면 요청을 인증하기 위한 [키와 토큰](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따라 이러한 키와 토큰을 생성할 수 있습니다.

  * [개발자 계정 등록](https://developer.x.com/en/apply-for-access)을 완료하고 승인을 받습니다.
  * developer portal에서 [프로젝트](/ko/resources/fundamentals/projects)를 생성하고 연결된 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 만듭니다.
  * 앱의 “Keys and tokens” 페이지로 이동하여 필요한 자격 증명을 생성합니다. 모든 자격 증명을 안전한 위치에 저장했는지 확인하세요.
</Note>

<div id="steps-to-build-a-list-posts-lookup-request">
  ### 리스트 게시물 조회 요청을 구성하는 단계
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### 1단계: 도구 또는 라이브러리 선택
</div>

이 엔드포인트에 요청을 보내는 데 사용할 수 있는 다양한 도구, 코드 예제, 라이브러리가 있지만 여기서는 과정을 간소화하기 위해 Postman 도구를 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 불러오려면 아래 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2를 Postman에 추가
</Button>

Postman에서 X API v2 컬렉션을 불러온 후 &quot;List&quot; 폴더로 이동한 다음 &quot;List Posts&quot; 폴더를 선택하고, 마지막으로 &quot;List Posts lookup&quot;을 선택하세요.
 

<div id="step-two-authenticate-your-request">
  #### 2단계: 요청 인증하기
</div>

X API에 요청을 올바르게 보내려면, 해당 요청에 대한 권한이 있는지 확인해야 합니다. 이 엔드포인트의 경우, [App only](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), 또는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 인증 방식을 사용해 요청을 인증해야 합니다.

간단히 하기 위해 이 예제에서는 App only를 사용하지만, 비공개 [metrics](/ko/x-api/fundamentals/metrics)나 비공개 게시물을 요청하려면 다른 인증 방식 중 하나를 사용해야 합니다. 

App only를 사용하려면, Postman에서 우측 상단에 있는 “X API v2” 환경을 선택한 다음, 키와 토큰(특히 [App only Access Token](/ko/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token), 즉 App only Bearer Token)을 &quot;initial value&quot;와 &quot;current value&quot; 필드에 추가해야 합니다(환경 드롭다운 옆의 눈 모양 아이콘을 클릭하여 입력).

올바르게 설정했다면, 이 변수 값들은 요청의 Authorization 탭에 자동으로 반영됩니다.
 

<div id="step-three-identify-and-specify-which-list-you-would-like-to-retrieve-posts-from">
  #### 3단계: 어떤 리스트에서 게시물을 가져올지 식별하고 지정하기
</div>

요청에서 조회하려는 리스트를 지정해야 합니다. x.com으로 이동해 원하는 리스트를 클릭한 다음 URL을 보면 List ID를 찾을 수 있습니다. 예를 들어, 다음 URL의 List ID는 84839422입니다.

https://x.com/i/lists/84839422

대상 ID는 어떤 유효한 List ID라도 될 수 있습니다. Postman에서 &quot;Params&quot; 탭으로 이동한 다음, id 경로 변수의 &quot;Value&quot; 열에 해당 ID를 입력하세요. ID 앞이나 뒤에 공백을 포함하지 않도록 주의하세요.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| id  | 84839422 (리스트 ID) |

<div id="step-four-identify-and-specify-which-fields-you-would-like-to-retrieve">
  #### 4단계: 가져오려는 필드를 식별하고 지정하기
</div>

3단계까지 완료한 후 &quot;Send&quot; 버튼을 클릭하면, 응답에 기본 [Post object](/ko/x-api/fundamentals/data-dictionary#tweet) 필드인 id와 text가 포함됩니다.

추가 필드를 받고 싶다면, 요청에서 tweet.fields 및/또는 [expansion](/ko/x-api/fundamentals/expansions) 파라미터를 사용해 해당 필드를 지정해야 합니다.

이 예제에서는 서로 다른 객체에서 세 개의 추가 필드 집합을 요청합니다:

* 기본 List 객체의 추가 created&#95;at 필드

* expansion 파라미터를 사용한 전체 [user object](/ko/x-api/fundamentals/data-dictionary#user)

* 연관된 user 객체의 추가 user.created&#95;at 필드

Postman에서 &quot;Params&quot; 탭으로 이동한 다음 &quot;Query Params&quot; 테이블에 다음 key:value 쌍을 추가합니다:

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| tweet.fields | created&#95;at | created&#95;at |
| expansions | author&#95;id | includes.users.id,  <br />includes.users.name,  <br />includes.users.username |
| user.fields | created&#95;at | includes.users.created&#95;at |

이제 “Send” 버튼 옆에 다음과 비슷한 URL이 표시될 것입니다:

`https://api.x.com/2/lists/84839422/tweets?expansions=author_id&user.fields=created_at&max_results=1`

<div id="step-five-make-your-request-and-review-your-response">
  #### 5단계: 요청을 보내고 응답을 검토합니다
</div>

모든 설정을 완료했다면 &quot;Send&quot; 버튼을 눌러 요청을 보냅니다. 그러면 아래 예시 응답과 유사한 응답을 받게 됩니다:

```
{
  "data": [
    {
      "author_id": "4172587277",
      "id": "1458172421115101189",
      "text": "A Alemanha registrou nesta semana um recorde de novos casos de Covid-19. Segundo o governo e especialistas em Saúde, pessoas não vacinadas são responsáveis pela situação \nhttps://t.co/4POyaPwMLu"
    }
  ],
  "includes": {
    "users": [
      {
        "username": "MomentsBrasil",
        "name": "Twitter Moments Brasil",
        "created_at": "2015-11-12T16:46:02.000Z",
        "id": "4172587277"
      }
    ]
  },
  "meta": {
    "result_count": 1,
    "next_token": "7140dibdnow9c7btw3z2vwioavpvutgzrzm9icis4ndix"
  }
}
```

**참고:** 이 엔드포인트의 응답을 통해 지정된 리스트에 대해 최신 게시물을 최대 800개까지 조회할 수 있습니다
