<div id="authentication">
  #### 인증
</div>

모든 X API v2 엔드포인트는 요청을 인증하기 위해 키와 토큰이라고도 부르는 자격 증명 세트를 요구합니다. 이 엔드포인트에 대한 요청을 인증하려면 OAuth 1.0a 사용자 컨텍스트를 사용할 수 있습니다. 

[OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication)를 사용한다는 것은, 요청을 성공적으로 보내기 위해 반드시 API 키와 사용자 액세스 토큰 세트를 사용해야 함을 의미합니다. 액세스 토큰은 여러분이 대신하여 요청을 보내려는 사용자와 연결되어 있어야 합니다. 다른 사용자를 위한 액세스 토큰 세트를 생성하려는 경우, 해당 사용자는 [3-legged OAuth 플로우](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)를 사용하여 여러분의 앱을 승인해야 합니다.

OAuth 1.0a는 사용하기 어려울 수 있다는 점에 유의하세요. 이 인증 방식에 익숙하지 않다면, [라이브러리](/ko/x-api/tools-and-libraries/overview)나 Postman과 같은 도구를 사용할 것을 권장합니다.

<div id="developer-portal-projects-and-developer-apps">
  #### Developer portal, 프로젝트, 그리고 개발자 앱
</div>

X API v2 엔드포인트에서 사용할 인증용 자격 증명 세트를 발급받으려면 먼저 [개발자 계정에 가입](https://developer.x.com/en/portal/petition/essential/basic-info)하고, 해당 계정 내에 [프로젝트](/ko/resources/fundamentals/projects)를 설정한 뒤, 그 프로젝트 내에 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성해야 합니다. 그런 다음 개발자 앱에서 키와 토큰을 확인할 수 있습니다.
 

<div id="rate-limits">
  #### 요청 한도
</div>

매일 수천 명이 넘는 개발자가 X API에 요청을 보냅니다. 이러한 요청의 방대한 양을 관리하기 위해 각 엔드포인트에는 [요청 한도](/ko/x-api/fundamentals/rate-limits)가 설정되어 있으며, 이를 통해 앱 또는 인증된 사용자를 대신해 보낼 수 있는 요청 수가 제한됩니다. 

이 엔드포인트들에는 사용자 단위의 요청 한도가 적용됩니다. 즉, 요청을 대신 보내는 인증된 사용자는 어떤 개발자 앱을 사용하더라도 해당 엔드포인트를 호출할 수 있는 횟수가 정해져 있습니다. 

아래 표는 각 엔드포인트에 대한 요청 한도를 보여 줍니다.

|     |     |     |
| :--- | :--- | :--- |
| **엔드포인트** | **HTTP 메서드** | **요청 한도** |
| /2/users/:id/pinned&#95;lists | POST | 15분당 50회 요청 |
| /2/users/:id/pinned&#95;lists/:list&#95;id | DELETE | 15분당 50회 요청 |
| /2/users/:id/pinned&#95;lists | GET | 15분당 15회 요청 |

<div id="fields-and-expansions">
  #### 필드와 expansions
</div>

X API v2 GET 엔드포인트를 사용하면 `fields` 및 `expansions`라는 도구 집합을 통해 API에서 어떤 데이터를 반환받을지 정확하게 선택할 수 있습니다. `expansions` 파라미터를 사용하면 페이로드에서 참조되는 객체를 확장하여 포함할 수 있습니다. 예를 들어, 고정된 리스트(pinned Lists)를 조회하면 다음과 같은 [expansions](/ko/x-api/fundamentals/expansions)을 가져올 수 있습니다:

* `owner_id`

`fields` 파라미터를 사용하면 서로 다른 데이터 객체에서 어떤 [fields](/ko/x-api/fundamentals/fields)를 받을지 정확하게 선택할 수 있습니다. 이 엔드포인트는 주로 사용자 객체를 반환합니다. 기본적으로 List 객체는 `id` 및 `name` 필드를 반환합니다. `list.created_at` 또는 `list.description`과 같은 추가 필드를 받으려면, `fields` 파라미터를 사용하여 해당 필드를 명시적으로 요청해야 합니다. 

[X API v2 데이터 딕셔너리](/ko/x-api/fundamentals/data-dictionary)에 [fields 및 expansions](/ko/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)를 함께 사용하는 방법에 대한 가이드를 추가했습니다.

 아래 표는 lookup 엔드포인트에서 사용 가능한 필드와 expansions를 보여줍니다:

|     |     |     |
| :--- | :--- | :--- |
| **엔드포인트** | **필드** | **expansions** |
| /2/users/:id/pinned&#95;lists | `list.fields`<br /><br />`user.fields` | `owner_id` |
