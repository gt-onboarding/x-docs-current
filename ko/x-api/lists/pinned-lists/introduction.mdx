리스트를 고정하면 X 사용자가 자신이 구독했거나 직접 만든 리스트에 더 쉽게 액세스할 수 있습니다.

이 엔드포인트들을 통해 사용자를 대신해 요청을 보내므로, 반드시 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)로 인증해야 하며, 앱을 승인한 사용자의 사용자 액세스 토큰을 사용해야 합니다. 이러한 액세스 토큰은 [3-legged OAuth 플로우](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) 또는 [Authorization Code with PKCE grant flow](/ko/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0)를 사용해 생성할 수 있습니다.

<div id="pinned-list-lookup">
  ### 고정 리스트 조회
</div>

고정 리스트 조회에는 인증된 사용자의 고정 리스트를 조회할 수 있는 하나의 엔드포인트가 있습니다. 이 엔드포인트의 rate limit은 15분당 15회 요청입니다.
 

<div id="manage-pinned-lists">
  ### 고정된 리스트 관리
</div>

고정된 리스트 관리 엔드포인트를 사용하면 인증된 사용자의 계정에서 리스트를 고정하거나 고정 해제할 수 있습니다. 이 엔드포인트는 POST와 DELETE 두 가지 메서드를 제공합니다. POST 메서드는 리스트를 고정하는 데 사용되고, DELETE 메서드는 리스트의 고정을 해제하는 데 사용됩니다. 두 엔드포인트 모두 사용자 기준으로 15분당 50회의 요청 제한이 적용됩니다.

<Note>
  **계정 설정**

  이 엔드포인트에 액세스하려면 다음이 필요합니다:

  * 승인된 [개발자 계정](https://developer.x.com/en/portal/petition/essential/basic-info).
  * [프로젝트](/ko/resources/fundamentals/projects) 내에 있는 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용해 인증해야 합니다. 

  X API v2 엔드포인트에 대한 액세스 권한을 얻는 방법은 [시작하기 가이드](/ko/x-api/getting-started/getting-access)에서 자세히 알아보세요.
</Note>
