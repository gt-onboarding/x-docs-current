<div id="steps-to-build-a-manage-pinned-list-request">
  ### 고정된 리스트 관리 요청을 구성하는 단계
</div>

**1단계: Postman에서 List 엔드포인트 컬렉션 선택**

Postman에 X API v2 컬렉션을 로드한 후 “List” 폴더로 이동하여 “Pinned Lists” 폴더를 선택한 다음 &quot;Pin a List&quot;를 선택합니다.
 

**2단계: 고정할 리스트 지정**

고정된 리스트 관리 엔드포인트에는 두 개의 ID가 필요합니다. 하나는 사용자 ID(리스트를 고정할 인증된 사용자용)이고, 다른 하나는 대상 리스트 ID(고정될 리스트용)입니다. 사용자 ID는 인증에 사용하는 사용자 ID와 일치해야 하므로, 요청을 인증할 때 해당 사용자 ID와 연결된 액세스 토큰을 전달해야 합니다. 이 경우, 자신의 계정에 해당하는 ID를 지정하면 됩니다. ID는 다음 두 가지 방법으로 찾을 수 있습니다.

1. [users lookup](/ko/x-api/users/lookup/introduction) by username 엔드포인트를 사용하면, username을 전달하여 `id` 필드를 받을 수 있습니다.
2. 액세스 토큰을 확인하면, 숫자로 이루어진 부분이 사용자 ID입니다.

대상 리스트 ID는 유효한 리스트라면 무엇이든 사용할 수 있습니다. Postman에서 &quot;Params&quot; 탭으로 이동하여 id 경로 변수의 &quot;Value&quot; 열에 사용자 ID를 입력합니다. 그런 다음 “Body” 탭으로 이동하여, 고정하려는 리스트의 ID를 list&#95;id 파라미터의 값으로 입력합니다. 어떤 ID의 앞이나 뒤에도 공백이 포함되지 않도록 주의하십시오.

|             |               |                    |
| :---------- | :------------ | :----------------- |
| **Key**     | **Value**     | **Parameter type** |
| `id`        | 사용자 ID        | path               |
| list&#95;id | 고정할 대상 리스트 ID | body               |

이제 &quot;Send&quot; 버튼 옆에 이와 비슷한 URL이 표시되어야 합니다.

```
      https://api.x.com/2/users/2244994945/pinned_lists
```

3단계: 요청을 보내고 응답을 확인합니다

모든 설정을 마쳤다면 &quot;Send&quot; 버튼을 클릭하고, 아래 예시와 유사한 응답을 받게 됩니다:

```
{
  "data": {
    "pinned": true
  }
}
```

반환된 응답 객체의 `pinned` 키 값이 `true`이면 대상 리스트가 정상적으로 고정된 것입니다. 

리스트 고정을 해제하려면 Postman에 로드된 X API v2 컬렉션의 “Lists” 폴더에 있는 “Unpin List” 요청을 선택합니다. 이 엔드포인트에는 인증된 사용자 ID와 고정을 해제할 리스트의 ID가 필요합니다. “Params” 탭에서 `id` 열의 값으로 사용자 ID를 입력하고, `list_id` 열의 값으로는 고정을 해제할 리스트의 ID를 입력합니다. 

삭제 요청이 성공적으로 처리되면 다음 예시와 유사한 응답을 받게 됩니다:

```
{
  "data": {
    "pinned": false
  }
}
```
