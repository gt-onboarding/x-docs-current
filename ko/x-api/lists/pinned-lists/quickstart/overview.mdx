  ## 고정된 리스트 엔드포인트 그룹 시작하기
</div>

이 간단한 안내에서는 [Postman](/ko/tutorials/postman-getting-started)을 사용하여 고정된 리스트 엔드포인트에 첫 요청을 보내는 방법을 설명합니다.

여러 언어로 작성된 샘플 코드를 보고 싶다면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문해 보세요.

<Note>
  ### 사전 준비 사항

  이 가이드를 완료하려면 요청을 인증하는 데 사용할 [keys and tokens](/ko/resources/fundamentals/authentication) 일련이 필요합니다. 다음 단계를 따르면 이러한 키와 토큰을 생성할 수 있습니다:

  * [개발자 계정을 신청](https://developer.x.com/en/apply-for-access)하고 승인을 받습니다.
  * developer portal에서 [Project](/ko/resources/fundamentals/projects)와 연결된 [developer App](/ko/resources/fundamentals/developer-apps)을 생성합니다.
  * App의 “Keys and tokens” 페이지로 이동해 필요한 자격 증명을 생성합니다. 생성한 자격 증명은 모두 안전한 위치에 보관하세요.
</Note>

<div id="authenticate-your-request">
  #### 요청 인증
</div>

**lookup** 엔드포인트에 요청을 성공적으로 보내기 위해서는 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication), [OAuth 2.0 앱 전용(App-Only)](/ko/resources/fundamentals/authentication#oauth-2-0), 또는 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 중 하나를 사용할 수 있습니다. 그러나 **manage** 엔드포인트의 경우에는 OAuth 1.0a 사용자 컨텍스트 또는 OAuth 2.0 Authorization Code with PKCE로만 인증할 수 있습니다.

Postman을 사용할 때 환경을 올바르게 설정해 두었다면, 기본 인증용 키와 토큰이 요청에 자동으로 채워집니다.

이를 위해 Postman의 오른쪽 상단에서 “X API v2”라는 이름의 환경을 선택한 후, 환경 드롭다운 옆의 눈 모양 아이콘을 클릭하여 &quot;initial value&quot;와 &quot;current value&quot; 필드에 키와 토큰을 추가하면 됩니다. 여기에 포함되는 키는 다음과 같습니다.

* API Key (Consumer Key라고도 함)
* API Secret Key (Consumer Secret이라고도 함)
* OAuth 1.0a 사용자 액세스 토큰
* OAuth 1.0a 사용자 액세스 토큰 시크릿
* OAuth 2.0 앱 액세스 토큰
* OAuth 2.0 Client Key (앱 설정에서 OAuth 2.0 User Authentication 설정을 구성한 경우에만 사용 가능)
* OAuth 2.0 Client Secret (앱 설정에서 OAuth 2.0 User Authentication 설정을 구성한 경우에만 사용 가능)
