  ## 고정 리스트 엔드포인트 그룹 시작하기
</div>

이 간단한 개요는 [Postman](/ko/tutorials/postman-getting-started)을 사용해 고정 리스트 엔드포인트에 첫 번째 요청을 보내는 데 도움이 됩니다.

여러 언어로 된 샘플 코드를 보고 싶다면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요.

<Note>
  ### 사전 준비 사항

  이 가이드를 완료하려면 요청을 인증할 수 있는 [keys and tokens](/ko/resources/fundamentals/authentication)이 필요합니다. 다음 단계를 따라 이러한 키와 토큰을 생성할 수 있습니다.

  * [개발자 계정](https://developer.x.com/en/apply-for-access)에 가입하고 승인을 받습니다.
  * Developer Portal에서 [Project](/ko/resources/fundamentals/projects)를 생성하고, 연결된 [developer App](/ko/resources/fundamentals/developer-apps)을 만듭니다.
  * App의 &quot;Keys and tokens&quot; 페이지로 이동하여 필요한 자격 증명을 생성합니다. 모든 자격 증명을 안전한 위치에 저장했는지 확인하세요.
</Note>

<div id="authenticate-your-request">
  #### 요청 인증하기
</div>

**lookup** 엔드포인트에 성공적으로 요청하려면 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication), [OAuth 2.0 앱 전용(App-Only)](/ko/resources/fundamentals/authentication#oauth-2-0), 또는 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 중 하나를 사용할 수 있습니다. 그러나 **manage** 엔드포인트의 경우에는 OAuth 1.0a 사용자 컨텍스트 또는 OAuth 2.0 Authorization Code with PKCE로만 인증할 수 있습니다.

Postman을 사용할 때는 환경만 올바르게 설정되어 있으면 기본 인증 키와 토큰이 요청에 자동으로 입력됩니다.

이를 위해 Postman의 오른쪽 상단에서 “X API v2”라는 이름의 환경을 선택한 다음, 환경 드롭다운 옆의 눈 모양 아이콘을 클릭해 &quot;initial value&quot; 및 &quot;current value&quot; 필드에 키와 토큰을 추가하면 됩니다. 여기에는 다음 키들이 포함됩니다:

* API Key (Consumer Key라고도 함)
* API Secret Key (Consumer Secret이라고도 함)
* OAuth 1.0a user Access Token
* OAuth 1.0a user Access Token Secret
* OAuth 2.0 App Access Token
* OAuth 2.0 Client Key (앱 설정에서 OAuth 2.0 User Authentication 설정을 구성한 경우에만 사용 가능)
* OAuth 2.0 Client Secret (앱 설정에서 OAuth 2.0 User Authentication 설정을 구성한 경우에만 사용 가능)
