<div id="getting-started-with-the-mutes-lookup-endpoint">
  ## mutes 조회 엔드포인트 사용 시작하기
</div>

이 빠른 시작 가이드는 Postman을 사용해 mutes 조회 엔드포인트에 첫 번째 요청을 보내는 방법을 안내합니다.

여러 언어로 된 샘플 코드를 보고 싶다면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요.

<Note>
  **사전 준비 사항**

  이 가이드를 완료하려면 요청을 인증하기 위한 [keys and tokens](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따라 이러한 키와 토큰을 생성할 수 있습니다:

  * [개발자 계정에 가입](https://developer.x.com/en/apply-for-access)하고 승인을 받으세요.
  * 개발자 포털에서 [프로젝트](/ko/resources/fundamentals/projects)와 연결된 [developer App](/ko/resources/fundamentals/developer-apps)을 생성하세요.
  * 앱의 &quot;Keys and tokens&quot; 페이지로 이동해 필요한 자격 증명을 생성하세요. 모든 자격 증명은 안전한 위치에 보관해야 합니다.
</Note>

<div id="steps-to-build-a-mutes-lookup-request">
  ### 뮤트 조회 요청을 만드는 단계
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### 1단계: 도구 또는 라이브러리 선택
</div>

이 엔드포인트에 요청을 보내기 위해 사용할 수 있는 도구, 코드 예제, 라이브러리가 여러 가지 있지만, 여기서는 과정을 단순화하기 위해 Postman 도구를 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 불러오려면 아래 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2를 Postman에 추가
</Button>

Postman에 X API v2 컬렉션을 불러온 후, &quot;Mutes&quot; 폴더로 이동하여 &quot;Mutes lookup&quot;을 선택합니다.
 

<div id="step-two-authenticate-your-request">
  #### 2단계: 요청 인증하기
</div>

X API에 요청을 올바르게 보내려면 먼저 권한이 있는지 확인해야 합니다. 이 엔드포인트에서는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용해 요청을 인증해야 합니다.

이 예제에서는 OAuth 1.0a User Context를 사용합니다.

Postman에 키와 토큰, 구체적으로 API Key, API Secret Key, OAuth 1.0a 사용자 액세스 토큰, OAuth 1.0a 사용자 액세스 토큰 시크릿을 추가해야 합니다. 이를 위해 Postman 오른쪽 상단에서 “X API v2”라는 이름의 환경(environment)을 선택한 뒤, 환경 드롭다운 옆의 눈 모양 아이콘을 클릭하여 &quot;initial value&quot; 및 &quot;current value&quot; 필드에 키와 토큰을 추가합니다.

이 변숫값들이 올바르게 설정되면 요청의 Authorization 탭에 자동으로 반영됩니다.
 

<div id="step-three-specify-a-user">
  #### 3단계: 사용자 지정하기
</div>

이 엔드포인트를 사용할 때는, 자신이 누구를 음소거했는지 또는 인증된 사용자가 누구를 음소거했는지 확인하기 위해, 본인의 사용자 ID 또는 인증된 사용자의 ID를 지정해야 합니다.

Postman에서 &quot;Params&quot; 탭으로 이동한 뒤, 섹션 맨 아래에 있는 “Path Variables”의 id 행에서 &quot;Value&quot; 열에 인증된 사용자 ID를 입력합니다. 이때 ID 앞뒤에 공백이 포함되지 않도록 주의하세요.

“Path Variables” 섹션 위에는 추가할 수 있는 선택적 “Query Params”가 있습니다. 이 예시에서는 max&#95;results 변수를 선택하고 값으로 5를 입력합니다.

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Parameter Type** |
| `id` | (본인 사용자 ID) | Path |
| max&#95;results | 5   | Query |

<div id="step-four-identify-and-specify-which-fields-you-would-like-to-retrieve">
  #### Step four: 가져올 필드를 식별하고 지정하기
</div>

3단계까지 완료한 후 &quot;Send&quot; 버튼을 클릭하면, 응답에서 기본 [user object](/ko/x-api/fundamentals/data-dictionary#user) 필드인 id, name, username을 받게 됩니다.

id, name, username 외에 추가 필드를 받고 싶다면, 요청에서 [fields](/ko/x-api/fundamentals/fields) 및/또는 [expansions](/ko/x-api/fundamentals/expansions) 파라미터로 해당 필드를 지정해야 합니다.

이 연습에서는 서로 다른 객체에서 세 가지 추가 필드 집합을 요청합니다:

1. 기본 user 객체의 추가 user.created&#95;at 필드
2. 반환된 사용자에 대해 연관된 고정 게시물(Post) 객체의 기본 필드: id 및 text
3. 연관된 게시물(Post) 객체의 추가 tweet.created&#95;at 필드

Postman에서 &quot;Params&quot; 탭으로 이동한 다음, &quot;Query Params&quot; 표에 다음 key:value 쌍을 추가합니다:

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| user.fields | created&#95;at | user.created&#95;at |
| expansions | pinned&#95;tweet&#95;id | tweet.id, tweet.text |
| tweet.fields | created&#95;at | includes.tweets.created&#95;at |

이제 &quot;Send&quot; 버튼 옆에 표시되는 예시 ID 대신, 여러분의 사용자 ID가 포함된 유사한 URL이 보일 것입니다:

`https://api.x.com/2/users/1324848235714736129/muting?user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=created_at&max_results=5`

<div id="step-five-make-your-request-and-review-your-response">
  #### 다섯 번째 단계: 요청을 보내고 응답을 확인하기
</div>

모든 설정을 마쳤다면 &quot;Send&quot; 버튼을 눌러 요청을 보내세요. 그러면 아래 예시와 비슷한 응답을 받게 됩니다:

```
{
  "data": [
    {
      "username": "TwitterDev",
      "created_at": "2013-12-14T04:35:55.000Z",
      "id": "2244994945",
      "name": "Twitter Dev",
      "pinned_tweet_id": "1430984356139470849"
    }
  ],
  "includes": {
    "tweets": [
      {
        "created_at": "2021-08-26T20:03:51.000Z",
        "id": "1430984356139470849",
        "text": "Help us build a better Twitter Developer Platform!\n \nTake the annual developer survey &gt;&gt;&gt; https://t.co/9yTbEKlJHH https://t.co/fYIwKPzqua"
      }
    ]
  },
  "meta": {
    "result_count": 1
  }
}
```

<div id="step-six-paginate-through-your-results">
  #### 6단계: 결과에 페이지네이션 적용하기
</div>

응답의 하단에 `meta` 객체가 있는 것을 확인할 수 있습니다. `next_token`을 받은 경우, 이는 추가로 가져올 수 있는 결과의 다음 페이지가 있다는 신호입니다. 다음 페이지의 결과를 가져오려면 `next_token` 필드의 값을 가져와, 추가 `pagination_token` 쿼리 매개변수의 값으로 요청에 포함하면 됩니다.
 

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| pagination&#95;token | 1710819323648428707 |

이 추가 매개변수를 포함해 요청을 보내면, 3단계에서 `max_results`를 5로 지정했기 때문에 이후 페이로드에 다음 5개의 결과가 전달됩니다. 모든 결과를 받을 때까지 이 과정을 반복할 수 있지만, `max_results` 매개변수를 사용해 한 번의 요청으로 최대 1000명의 사용자를 요청할 수도 있으므로 결과에 대해 자주 페이지네이션할 필요는 없습니다.
