<div id="getting-started-with-the-manage-mutes-endpoints">
  ## 뮤트 관리 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 [Postman](/ko/tutorials/postman-getting-started)을 사용해 뮤트 관리 엔드포인트에 첫 요청을 보내는 방법을 안내합니다.

다양한 언어로 된 샘플 코드를 보고 싶다면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요.

<Note>
  **사전 준비 사항**

  이 가이드를 완료하려면 요청을 인증하기 위한 [keys and tokens](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따라 이러한 키와 토큰을 생성할 수 있습니다.

  * [개발자 계정에 가입](https://developer.x.com/en/apply-for-access)하고 승인을 받습니다.
  * 개발자 포털에서 [프로젝트](/ko/resources/fundamentals/projects)와 연결된 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성합니다.
  * 앱의 &quot;Keys and tokens&quot; 페이지로 이동하여 필요한 자격 증명을 생성합니다. 모든 자격 증명은 안전한 위치에 저장해야 합니다.
</Note>

<div id="steps-to-build-a-manage-mutes-request">
  ### Manage Mutes 요청을 구성하는 단계
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### 1단계: 도구 또는 라이브러리부터 시작하기
</div>

이 엔드포인트에 요청을 보내기 위해 사용할 수 있는 도구, 코드 예제, 라이브러리는 여러 가지가 있지만, 여기서는 과정을 단순화하기 위해 Postman 도구를 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 불러오려면, 다음 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2를 Postman에 추가
</Button>

Postman에 X API v2 컬렉션을 불러온 후, “Mutes” 폴더로 이동하여 “Mute a user’s ID”를 선택합니다.
 

<div id="step-two-authenticate-your-request">
  #### 2단계: 요청 인증하기
</div>

X API에 올바르게 요청을 보내려면 먼저 요청할 권한이 있는지를 확인해야 합니다. 이 엔드포인트에서는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용해 요청을 인증해야 합니다.

이 예제에서는 OAuth 1.0a User Context를 사용합니다.

Postman에 키와 토큰, 구체적으로 API Key, API Secret Key, OAuth 1.0a 사용자 Access Token, OAuth 1.0a 사용자 Access Token Secret을 추가해야 합니다. 이렇게 하려면 Postman 오른쪽 상단의 &quot;X API v2&quot;라는 이름의 환경(environment)을 선택한 다음, 환경 드롭다운 옆의 눈 아이콘을 클릭해 표시되는 &quot;initial value&quot;와 &quot;current value&quot; 필드에 키와 토큰을 추가하면 됩니다.

올바르게 설정했다면, 이 변수들은 요청의 Authorization 탭에 자동으로 불러와집니다.
 

<div id="step-three-specify-who-is-going-to-mute-whom">
  #### 3단계: 누가 누구를 뮤트할지 지정하기
</div>

Manage mutes 엔드포인트에는 두 개의 ID가 필요합니다. 하나는 사용자(다른 사용자를 뮤트하거나 뮤트 해제하려는 사용자)용이고, 다른 하나는 대상 사용자(뮤트되거나 뮤트 해제될 사용자)용입니다. 사용자의 ID는 인증을 수행하는 사용자의 ID와 일치해야 합니다. 즉, 요청을 인증할 때 해당 사용자 ID와 연결된 액세스 토큰을 전달해야 합니다. 이 경우, 자신의 사용자 ID를 지정할 수 있습니다. 자신의 ID는 다음 두 가지 방법으로 확인할 수 있습니다:

1. [user lookup](/ko/x-api/users/lookup/introduction) by username 엔드포인트를 사용하면, 사용자 이름(username)을 전달하고 id 필드를 받을 수 있습니다. 
2. 자신의 액세스 토큰을 보면, 숫자 부분이 자신의 사용자 ID입니다.
    

대상 ID는 유효한 어떤 사용자 ID라도 될 수 있습니다. Postman에서 &quot;Params&quot; 탭으로 이동한 뒤, `id` 경로 변수의 &quot;Value&quot; 열에 자신의 ID를 입력합니다. 그런 다음 “Body” 탭으로 이동해, 뮤트하려는 사용자의 ID를 target&#95;user&#95;id 파라미터의 값으로 입력합니다. 어떤 ID 앞뒤에도 공백이 포함되지 않았는지 확인하세요.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 인증된 사용자 ID |
| target&#95;user&#95;id | 뮤트하려는 사용자 ID |

<div id="step-four-make-your-request-and-review-your-response">
  #### 4단계: 요청을 보내고 응답을 검토합니다
</div>

모든 설정을 마쳤다면 &quot;Send&quot; 버튼을 클릭하세요. 그러면 아래 예시와 유사한 응답을 받게 됩니다.

```
      { "data": { "muting": true } }
```

&quot;muting&quot;: true 값을 받았다면, 해당 id가 target&#95;user&#95;id를 성공적으로 뮤트한 것입니다.

같은 사용자의 뮤트를 해제하려면 Postman에 로드된 X API v2 컬렉션의 “Mutes” 폴더에 있는 “Unmute a user ID” 요청을 사용할 수 있습니다. source&#95;user&#95;id에는 자신의 사용자 ID를, target&#95;user&#95;id에는 뮤트를 해제할 사용자 ID를 넣어야 합니다. 이 값들은 JSON body로 추가할 필요가 없으므로, source&#95;user&#95;id와 target&#95;user&#95;id에 대한 필요한 query param을 추가했는지 반드시 확인해야 합니다.

뮤트 해제가 성공하면, 다음 예시와 유사한 응답을 받게 됩니다:

```
      { "data": { "muting": false } }
```
