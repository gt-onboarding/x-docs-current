  ### 음소거 관리: 표준 v1.1과 X API v2 비교
</div>

표준 v1.1 [POST mutes/users/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-create) 및 [POST mutes/users/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-destroy) 엔드포인트를 사용해 왔다면, 이 가이드는 표준 v1.1과 X API v2의 음소거 관리 엔드포인트 간 유사점과 차이점을 이해하는 데 도움을 줍니다.

* **유사점**
  * OAuth 1.0a 사용자 컨텍스트
* **차이점**
  * 엔드포인트 URL
  * 앱 및 프로젝트 요구 사항
  * HTTP 메서드
  * 요청 파라미터
     

<div id="similarities">
  #### 유사점
</div>

**OAuth 1.0a 사용자 컨텍스트 인증 방식**

두 엔드포인트 버전 모두 [OAuth 1.0a 사용자 컨텍스트](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication)를 지원합니다. 따라서 이전에 표준 v1.1 뮤트 관리 엔드포인트 중 하나를 사용했다면, X API v2 버전으로 마이그레이션하더라도 동일한 인증 방식을 계속 사용할 수 있습니다.

<div id="differences">
  #### 차이점
</div>

**엔드포인트 URL**

* Standard v1.1 엔드포인트:
  * POST https://api.x.com/1.1/mutes/users/create.json
    (사용자를 뮤트)
  * POST https://api.x.com/1.1/mutes/users/destroy.json
    (사용자 뮤트 해제)
* X API v2 엔드포인트:
  * POST https://api.x.com/2/users/:id/muting
    (사용자를 뮤트)
  * DELETE https://api.x.com/2/users/:source&#95;user&#95;id/muting/:target&#95;user&#95;id
    (사용자 뮤트 해제) 

**App 및 Project 요구 사항**

X API v2 엔드포인트는 요청을 인증할 때 [Project](/ko/resources/fundamentals/projects)에 연결된 [developer App](/ko/resources/fundamentals/developer-apps)의 자격 증명을 사용해야 합니다. 모든 X API v1.1 엔드포인트는 독립 실행형 App 또는 Project에 연결된 App의 자격 증명을 사용할 수 있습니다.

**요청 파라미터**

다음 Standard v1.1 요청 파라미터는 X API v2에서 다음과 같이 대응됩니다:

| Standard v1.1 | X API v2 |
| :--- | :--- |
| user&#95;id | target&#95;user&#95;id |
| screen&#95;name | 해당 없음 |

Standard v1.1 파라미터는 쿼리 파라미터로 전달되는 반면, X API v2 파라미터는 본문 파라미터(POST 엔드포인트의 경우) 또는 경로 파라미터(DELETE 엔드포인트의 경우)로 전달된다는 점에 유의하세요.

또한 Standard v1.1 엔드포인트를 사용할 때는, [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication)와 함께 전달된 액세스 토큰을 통해 어떤 사용자가 뮤트/뮤트 해제를 시작했는지 추론하므로, 대상 사용자를 뮤트하는 사용자의 id는 별도로 지정할 필요가 없습니다.
