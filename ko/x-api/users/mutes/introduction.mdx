계정을 뮤트하면 해당 계정을 언팔로우하거나 차단하지 않고도 그 계정의 게시물을 타임라인에서 숨길 수 있습니다. 뮤트된 계정은 본인이 뮤트되었다는 사실을 알 수 없으며, 언제든지 뮤트를 해제할 수 있습니다. manage mutes 엔드포인트를 사용하면 개발자는 X에서 사람들에게 더 안전한 사용 경험을 제공할 수 있습니다. manage mutes를 활용해 구축할 수 있는 예로, 특정 기간 동안 특정 주제에 대해 게시물을 올릴 가능성이 있는 계정을 뮤트하도록 해주는 애플리케이션이 있습니다. mutes lookup 엔드포인트를 사용하면, 여러분 또는 인증된 사용자가 어떤 계정을 뮤트했는지 확인할 수 있습니다. 이는 뮤트된 계정과 어떻게 상호작용하는지 파악하는 데 유용할 수 있습니다. 

mute lookup를 사용해 비공개 정보에 대한 요청을 보내고, manage mutes를 통해 사용자를 대신해 요청을 보내는 것이므로, 해당 엔드포인트들은 반드시 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용해 인증해야 하며, 여러분의 앱을 승인한 사용자와 연결된 액세스 토큰을 사용해야 합니다. 이 액세스 토큰은 [3-legged OAuth flow](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) 또는 [Authorization Code with PKCE grant flow](/ko/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0)를 사용해 생성할 수 있습니다.

<div id="mutes-lookup">
  ### 뮤트 조회
</div>

뮤트 조회 엔드포인트를 사용하면 인증된 사용자가 어떤 계정을 뮤트했는지 확인할 수 있습니다. 이 엔드포인트의 rate limit은 사용자당 15분마다 15개의 요청입니다.

<div id="manage-mutes">
  ### 음소거 관리
</div>

음소거 관리 엔드포인트를 사용하면 인증된 사용자를 대신해 지정된 계정을 음소거하거나 음소거 해제할 수 있습니다. 이 엔드포인트에는 POST와 DELETE 두 가지 메서드가 있습니다. POST 메서드는 계정을 음소거할 때 사용하고, DELETE 메서드는 계정의 음소거를 해제할 때 사용합니다. POST 및 DELETE 엔드포인트 모두에 대해 사용자 기준 15분당 요청 한도는 50회입니다.

**참고:** 사용자가 [X](https://x.com/)에서 직접 음소거할 경우, 15분당 200회의 요청 한도가 적용됩니다.

**계정 설정**

이 엔드포인트에 액세스하려면 다음이 필요합니다.

* 승인된 [개발자 계정](https://developer.x.com/en/portal/petition/essential/basic-info)
* [프로젝트](/ko/resources/fundamentals/projects) 내에 있는 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용한 인증

X API v2 엔드포인트에 대한 액세스 권한을 얻는 방법은 [시작하기 가이드](/ko/x-api/getting-started/getting-access)를 참고하세요.
