계정을 뮤트하면 해당 계정을 언팔로우하거나 차단하지 않고도 그 계정의 게시물을 타임라인에서 숨길 수 있습니다. 뮤트된 계정은 여러분이 해당 계정을 뮤트했다는 사실을 알 수 없으며, 언제든지 뮤트를 해제할 수 있습니다. manage mutes 엔드포인트를 사용하면 개발자는 X에서 사람들을 위한 더 안전한 사용 경험을 만들 수 있습니다. 예를 들어, 특정 주제에 대해 게시물을 올릴 가능성이 있는 계정을 지정한 기간 동안 뮤트할 수 있게 해 주는 앱을 구축하는 방식이 있습니다. mutes lookup 엔드포인트를 사용하면 여러분 또는 인증된 사용자가 누구를 뮤트했는지 확인할 수 있습니다. 이는 뮤트된 계정과 어떻게 상호작용하는지 파악하는 데 유용할 수 있습니다. 

mute lookup로 프라이빗한 정보에 대한 요청을 하고, manage mutes로 사용자를 대신해 요청을 수행하므로, 이 엔드포인트들은 반드시 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)로 인증해야 하며, 여러분의 앱을 승인한 사용자 계정과 연결된 사용자 액세스 토큰을 사용해야 합니다. 이러한 토큰은 [3-legged OAuth flow](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) 또는 [Authorization Code with PKCE grant flow](/ko/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0)을 사용해 생성할 수 있습니다.

<div id="mutes-lookup">
  ### 뮤트 조회
</div>

뮤트 조회 엔드포인트를 사용하면 인증된 사용자가 뮤트한 계정을 확인할 수 있습니다. 이 엔드포인트는 사용자당 15분마다 최대 15회의 요청으로 제한됩니다.

<div id="manage-mutes">
  ### 음소거 관리
</div>

음소거 관리 엔드포인트를 사용하면 인증된 사용자를 대신해 지정된 계정을 음소거하거나 음소거 해제할 수 있습니다. 이 엔드포인트에서는 POST와 DELETE 두 가지 메서드를 사용할 수 있습니다. POST 메서드는 계정을 음소거하는 데 사용되며, DELETE 메서드는 계정의 음소거를 해제하는 데 사용됩니다. POST와 DELETE 엔드포인트 모두에 대해 사용자 기준 15분당 50회의 요청 제한이 있습니다.

**주의:** 사용자가 [X](https://x.com/)에서 직접 계정을 음소거하는 경우, 15분당 200회의 요청 제한이 있습니다.

**계정 설정**

이 엔드포인트에 액세스하려면 다음이 필요합니다:

* 승인된 [개발자 계정](https://developer.x.com/en/portal/petition/essential/basic-info)
* [프로젝트](/ko/resources/fundamentals/projects) 내에 위치한 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용한 인증

X API v2 엔드포인트에 대한 접근 권한을 얻는 방법은 [시작하기 가이드](/ko/x-api/getting-started/getting-access)를 참조하세요.
