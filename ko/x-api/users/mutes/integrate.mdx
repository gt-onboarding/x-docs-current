<div id="authentication">
  #### 인증
</div>

모든 X API v2 엔드포인트는 키와 토큰이라고도 하는 일련의 자격 증명을 사용해 요청을 인증해야 합니다. 이러한 엔드포인트에 대한 요청을 인증하기 위해 OAuth 1.0a 사용자 컨텍스트 또는 OAuth 2.0 Authorization Code with PKCE 중 하나를 사용할 수 있습니다. 

[OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication)에서는 API 키, 사용자 액세스 토큰, 그리고 몇 가지 다른 매개변수를 사용해 [Authorization 헤더를 생성](/ko/resources/fundamentals/authentication#authorizing-a-request)한 다음, 이를 요청과 함께 전송해야 합니다. 액세스 토큰은 요청을 대신 수행하려는 해당 사용자와 연관되어 있어야 합니다. 다른 사용자를 위한 액세스 토큰 일련을 생성하려면, 해당 사용자가 [3-legged OAuth 플로우](https://developer-staging.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)를 사용해 App을 승인해야 합니다. 

OAuth 1.0a는 사용하기 까다로울 수 있습니다. 이 인증 방식에 익숙하지 않다면 [라이브러리](/ko/x-api/tools-and-libraries/overview)를 사용하거나, Postman과 같은 도구를 사용하거나, OAuth 2.0을 사용해 요청을 인증할 것을 권장합니다.

[OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)는 애플리케이션의 스코프와 여러 기기에 걸친 인가 플로우를 보다 정교하게 제어할 수 있도록 해 줍니다. OAuth 2.0을 사용하면 사용자 대신 수행할 수 있는 권한을 정의하는 세밀한 스코프를 구체적으로 선택할 수 있습니다. 

App에서 OAuth 2.0을 사용하려면, developer portal의 App settings 섹션에 있는 App의 authentication settings에서 이를 활성화해야 합니다.

<div id="developer-portal-projects-and-developer-apps">
  #### 개발자 포털, 프로젝트, 개발자 앱
</div>

X API v2 엔드포인트에서 사용할 수 있는 인증 자격 증명을 발급받으려면, 먼저 [개발자 계정에 가입](https://developer.x.com/en/portal/petition/essential/basic-info)하고, 해당 계정 내에 [프로젝트](/ko/resources/fundamentals/projects)를 설정한 다음, 그 프로젝트 내에 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성해야 합니다. 그런 다음 개발자 앱 내에서 키와 토큰을 확인할 수 있습니다.
 

<div id="rate-limits">
  #### 요청 한도
</div>

매일 수천 명이 넘는 개발자들이 X API에 요청을 보냅니다. 이러한 요청의 방대한 양을 관리하기 위해 각 엔드포인트에는 [요청 한도](/ko/x-api/fundamentals/rate-limits)가 설정되어 있어, 앱 또는 인증된 사용자를 대신해 보낼 수 있는 요청 수를 제한합니다. 

이 엔드포인트들은 사용자 단위로 요청 한도가 적용됩니다. 이는 요청을 보내는 대상인 인증된 사용자가, 어떤 개발자 앱을 사용하든 해당 엔드포인트를 호출할 수 있는 횟수가 정해져 있다는 의미입니다.

POST와 DELETE 메서드 모두에 대해, 엔드포인트별로 15분당 50건의 사용자 요청 한도가 있습니다. 그러나 GET 메서드의 경우 요청 한도는 15분당 15건으로 더 낮습니다.
 

<div id="fields-and-expansions">
  #### Fields and expansions
</div>

X API v2 GET 엔드포인트를 사용하면 `fields` 및 `expansions`라는 도구 세트를 통해 API에서 어떤 데이터를 반환할지 정확히 선택할 수 있습니다. `expansions` 파라미터를 사용하면 페이로드에서 참조된 객체를 확장해 전체 객체를 가져올 수 있습니다. 예를 들어, 이 엔드포인트를 사용하면 다음과 같은 [expansions](/ko/x-api/fundamentals/expansions)를 가져올 수 있습니다:

* pinned&#95;tweet&#95;id

**fields** 파라미터를 사용하면 서로 다른 데이터 객체에서 어떤 [fields](/ko/x-api/fundamentals/fields)를 받을지 정확히 선택할 수 있습니다. 이 엔드포인트는 주로 User 객체를 반환합니다. 기본적으로 User 객체는 **id**, **name**, **username** 필드를 반환합니다. **user.created&#95;at** 또는 **user.entities**와 같은 추가 필드를 받으려면 **fields** 파라미터를 사용해 해당 필드를 명시적으로 요청해야 합니다. 

[X API v2 데이터 사전](/ko/x-api/fundamentals/data-dictionary)에 [fields and expansions](/ko/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)를 함께 사용하는 방법에 대한 가이드를 포함해 두었습니다.

<div id="pagination">
  #### 페이지네이션
</div>

뮤트 조회는 많은 데이터를 반환할 수 있습니다. 언제나 일관되고 고성능의 결과를 제공하기 위해 페이지네이션을 사용합니다. 페이지네이션은 단일 응답으로 반환할 수 있는 양을 초과하는 결과를 제공하는 X API v2 엔드포인트에서 제공하는 기능입니다. 이런 경우, 데이터는 여러 ‘페이지’의 연속으로 반환됩니다. [페이지네이션을 통해 결과를 조회하는 방법](/ko/x-api/fundamentals/pagination)에 대해 더 알아보세요.

<Note>
  **참고:** 사용자가 [X](https://x.com/)에서 뮤트하는 경우, 요청 수는 15분당 최대 200개로 제한됩니다.
</Note>
