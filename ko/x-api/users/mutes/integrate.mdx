<div id="authentication">
  #### 인증
</div>

모든 X API v2 엔드포인트는 키와 토큰이라고도 불리는 자격 증명 세트를 사용해 요청을 인증해야 합니다. 이러한 엔드포인트에 대한 요청을 인증하기 위해 OAuth 1.0a 사용자 컨텍스트 또는 OAuth 2.0 Authorization Code with PKCE 중 하나를 사용할 수 있습니다. 

[OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication)는 API 키, 사용자 액세스 토큰, 그리고 몇 가지 다른 매개변수를 사용하여 요청과 함께 전달할 [Authorization 헤더를 생성](/ko/resources/fundamentals/authentication#authorizing-a-request)하도록 요구합니다. 액세스 토큰은 요청을 대신 수행할 사용자의 계정과 연결되어 있어야 합니다. 다른 사용자를 위한 액세스 토큰 세트를 생성하려면 해당 사용자가 [3-legged OAuth 플로우](https://developer-staging.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)를 사용해 앱을 승인해야 합니다. 

OAuth 1.0a는 사용이 어려울 수 있다는 점에 유의해야 합니다. 이 인증 방식에 익숙하지 않다면 [라이브러리](/ko/x-api/tools-and-libraries/overview)를 사용하거나, Postman과 같은 도구를 사용하거나, OAuth 2.0을 사용해 요청을 인증할 것을 권장합니다.

[OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)는 애플리케이션의 스코프와 여러 기기에 걸친 인증 플로우를 더 세밀하게 제어할 수 있게 해 줍니다. OAuth 2.0을 사용하면 사용자 대신 수행할 작업에 대해 세분화된 스코프를 선택하여 특정 권한만 부여할 수 있습니다. 

앱에서 OAuth 2.0을 사용하려면 developer portal의 App settings 섹션에 있는 앱의 인증 설정에서 이를 활성화해야 합니다.

  #### developer portal, 프로젝트, 그리고 개발자 앱

X API v2 엔드포인트에서 사용할 수 있는 인증 자격 증명 세트를 발급받으려면, 먼저 [개발자 계정에 가입](https://developer.x.com/en/portal/petition/essential/basic-info)하고, 해당 계정 내에 [프로젝트](/ko/resources/fundamentals/projects)를 설정한 뒤, 그 프로젝트 내에 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성해야 합니다. 그런 다음 해당 개발자 앱에서 키와 토큰을 확인할 수 있습니다.
 

<div id="rate-limits">
  #### 요청 한도
</div>

매일 수천 명의 개발자들이 X API에 요청을 보냅니다. 이러한 요청의 막대한 양을 관리하기 위해 각 엔드포인트에는 [요청 한도](/ko/x-api/fundamentals/rate-limits)가 설정되어 있으며, 이를 통해 앱 또는 인증된 사용자를 대신해 보낼 수 있는 요청 수가 제한됩니다. 

이러한 엔드포인트에는 사용자 수준의 요청 한도가 적용됩니다. 즉, 여러분이 요청을 대신 보내는 인증된 사용자는 어떤 개발자 앱을 사용하든 해당 엔드포인트를 호출할 수 있는 횟수가 정해져 있습니다.

POST 및 DELETE 메서드가 있는 각 엔드포인트에는 사용자 기준으로 15분당 50회의 요청 한도가 있습니다. 그러나 GET 메서드의 경우 엔드포인트당 15분당 15회로 제한됩니다.
 

<div id="fields-and-expansions">
  #### 필드와 expansions
</div>

X API v2 GET 엔드포인트를 사용하면 `fields` 및 `expansions`라고 불리는 도구 세트를 통해 API에서 어떤 데이터를 반환할지 정확히 선택할 수 있습니다. `expansions` 파라미터를 사용하면 페이로드에서 참조되는 객체를 확장할 수 있습니다. 예를 들어, 이 엔드포인트를 사용하면 다음과 같은 [expansions](/ko/x-api/fundamentals/expansions)을(를) 요청할 수 있습니다:

* pinned&#95;tweet&#95;id

**fields** 파라미터를 사용하면 수신하려는 서로 다른 데이터 객체 내에서 정확히 어떤 [fields](/ko/x-api/fundamentals/fields)를 포함할지 선택할 수 있습니다. 이 엔드포인트는 주로 User 객체를 반환합니다. 기본적으로 User 객체는 **id**, **name**, **username** 필드를 반환합니다. **user.created&#95;at** 또는 **user.entities**와 같은 추가 필드를 받으려면 **fields** 파라미터를 사용해 해당 필드를 명시적으로 요청해야 합니다. 

[X API v2 데이터 사전](/ko/x-api/fundamentals/data-dictionary)에 `fields`와 `expansions`를 함께 사용하는 방법에 대한 가이드([fields and expansions](/ko/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions))를 추가해 두었습니다.

<div id="pagination">
  #### 페이지네이션
</div>

뮤트 조회는 매우 많은 데이터를 반환할 수 있습니다. 항상 일관되고 성능이 좋은 결과를 제공하기 위해 페이지네이션을 사용합니다. 페이지네이션은 단일 응답에 담을 수 있는 양보다 더 많은 결과를 반환하는 X API v2 엔드포인트에서 사용하는 기능입니다. 이런 경우, 데이터는 여러 ‘페이지’로 나누어 반환됩니다. 결과에 대해 [페이지네이션을 수행하는 방법](/ko/x-api/fundamentals/pagination)에 대해 자세히 알아보세요.

<Note>
  **참고:** 사용자가 [X](https://x.com/)에서 뮤트하는 경우, 15분당 요청 한도는 200건입니다.
</Note>
