<div id="authentication">
  #### 인증
</div>

모든 X API v2 엔드포인트는 키와 토큰이라고도 하는 자격 증명 세트를 사용해 요청을 인증해야 합니다. 이 엔드포인트에 대한 요청을 인증하기 위해 OAuth 1.0a 사용자 컨텍스트 또는 OAuth 2.0 Authorization Code with PKCE를 사용할 수 있습니다. 

[OAuth 1.0a User Context](/ko/resources/fundamentals/authentication)를 사용하려면 API 키, 사용자 액세스 토큰, 그리고 몇 가지 추가 파라미터를 사용해 요청과 함께 전송할 [authorization 헤더를 생성](/ko/resources/fundamentals/authentication#authorizing-a-request)해야 합니다. 액세스 토큰은 요청을 대신 수행하려는 사용자와 연결되어 있어야 합니다. 다른 사용자를 위한 액세스 토큰 세트를 생성하려면, 해당 사용자가 [3-legged OAuth 플로우](https://developer-staging.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)를 사용해 앱을 승인해야 합니다. 

OAuth 1.0a는 사용하기 까다로울 수 있습니다. 이 인증 방식에 익숙하지 않은 경우, [라이브러리](/ko/x-api/tools-and-libraries/overview)를 사용하거나 Postman과 같은 도구를 사용하거나, OAuth 2.0을 사용해 요청을 인증할 것을 권장합니다.

[OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3)는 애플리케이션의 스코프와 여러 디바이스에 걸친 인증 플로우를 보다 세밀하게 제어할 수 있게 해줍니다. OAuth 2.0을 사용하면 사용자 대신 특정 권한을 부여하는 세분화된 스코프를 선택할 수 있습니다. 

앱에서 OAuth 2.0을 사용하려면, developer portal의 App settings 섹션에 있는 앱의 authentication settings에서 이를 활성화해야 합니다.

<div id="developer-portal-projects-and-developer-apps">
  #### 개발자 포털, 프로젝트 및 개발자 앱
</div>

X API v2 엔드포인트에서 사용할 인증 자격 증명 세트를 발급받으려면 [개발자 계정에 가입](https://developer.x.com/en/portal/petition/essential/basic-info)하고, 해당 계정 내에 [프로젝트](/ko/resources/fundamentals/projects)를 설정한 뒤, 그 프로젝트 내에 [developer App](/ko/resources/fundamentals/developer-apps)을 생성해야 합니다. 그런 다음 개발자 앱 내에서 키와 토큰을 확인할 수 있습니다.
 

<div id="rate-limits">
  #### 요청 한도
</div>

매일 수천 명의 개발자가 X API에 요청을 보냅니다. 이러한 요청의 방대한 양을 관리하기 위해, [요청 한도](/ko/x-api/fundamentals/rate-limits)가 각 엔드포인트에 설정되어 있으며, 이는 여러분의 앱 또는 인증된 사용자를 대신해 보낼 수 있는 요청 수를 제한합니다. 

이러한 엔드포인트에는 사용자 단위로 요청 한도가 적용됩니다. 즉, 여러분이 요청을 보내는 대상이 되는 인증된 사용자는 어느 개발자 앱에서든 각 엔드포인트를 호출할 수 있는 횟수가 정해져 있습니다. POST 및 DELETE 메서드에는 엔드포인트당 15분당 50건의 사용자 요청 한도가 있습니다. 그러나 GET 메서드의 경우 요청 한도는 15분당 15건입니다.
 

<div id="fields-and-expansions">
  #### 필드와 expansions
</div>

X API v2 GET 엔드포인트를 사용하면 **fields**와 **expansions**라고 하는 도구 세트를 통해 API에서 어떤 데이터를 반환받을지 정확하게 선택할 수 있습니다. `expansions` 파라미터를 사용하면 페이로드에서 참조된 객체를 확장해 가져올 수 있습니다. 예를 들어, 이 엔드포인트를 사용하면 다음과 같은 [expansions](/ko/x-api/fundamentals/expansions)을(를) 요청할 수 있습니다:

* pinned&#95;tweet&#95;id

**fields** 파라미터를 사용하면 여러 데이터 객체에서 어떤 [fields](/ko/x-api/fundamentals/fields)를 받을지 정확하게 선택할 수 있습니다. 이 엔드포인트들은 주로 Post 객체를 반환합니다. 기본적으로 Post 객체는 **id**와 **text** 필드를 반환합니다. **tweet.created&#95;at** 또는 **tweet.entities**와 같은 추가 필드를 받으려면 **fields** 파라미터를 사용해 해당 필드를 명시적으로 요청해야 합니다. 통합 시 활용을 고려할 수 있는 몇 가지 중요한 필드로는 투표 데이터, 메트릭, 게시물 애노테이션, 그리고 대화 ID 필드가 있습니다.

[X API v2 데이터 사전](/ko/x-api/fundamentals/data-dictionary)에 [fields 및 expansions 사용 방법](/ko/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)을 함께 다루는 가이드를 제공하고 있습니다.
