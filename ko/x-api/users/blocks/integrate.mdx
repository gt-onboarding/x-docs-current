<div id="authentication">
  #### 인증
</div>

모든 X API v2 엔드포인트에 요청을 보내려면, 요청을 인증하기 위한 자격 증명 세트(일명 키와 토큰)가 필요합니다. 이러한 엔드포인트에 대한 요청을 인증할 때는 OAuth 1.0a 사용자 컨텍스트 또는 OAuth 2.0 Authorization Code with PKCE 중 하나를 사용할 수 있습니다. 

[OAuth 1.0a User Context](/ko/resources/fundamentals/authentication)를 사용하려면 API 키, 사용자 액세스 토큰, 그리고 몇 가지 추가 파라미터를 사용해 [Authorization 헤더를 생성](/ko/resources/fundamentals/authentication#authorizing-a-request)한 뒤, 이를 요청과 함께 전달해야 합니다. 액세스 토큰은 요청을 대신 수행하려는 대상 사용자와 연결되어 있어야 합니다. 다른 사용자를 위한 액세스 토큰 세트를 생성하려면, 해당 사용자가 [3-legged OAuth 플로우](https://developer-staging.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)를 통해 앱을 승인해야 합니다. 

OAuth 1.0a는 사용하기 까다로운 편이라는 점에 유의해야 합니다. 이 인증 방식에 익숙하지 않다면, [라이브러리](/ko/x-api/tools-and-libraries/overview)를 사용하거나 Postman과 같은 도구를 사용하거나, OAuth 2.0을 사용해 요청을 인증할 것을 권장합니다.

[OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3)는 애플리케이션의 권한 범위(scope)에 대한 더 세밀한 제어와 여러 기기에 걸친 인증 플로우를 제공합니다. OAuth 2.0을 사용하면 사용자 대신 수행할 수 있는 권한을 정의하는, 보다 세분화된 특정 권한 범위(scope)를 선택할 수 있습니다. 

앱에서 OAuth 2.0을 사용하려면, developer portal 의 App settings 섹션에 있는 앱의 Authentication 설정에서 OAuth 2.0을 활성화해야 합니다.

X API v2 엔드포인트를 사용하기 위한 인증 자격 증명을 발급받으려면, 먼저 [개발자 계정에 가입](https://developer.x.com/en/portal/petition/essential/basic-info)하고, 해당 계정 내에 [프로젝트](/ko/resources/fundamentals/projects)를 생성한 뒤, 그 프로젝트 내에 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 만들어야 합니다. 그런 다음 개발자 앱에서 키와 토큰을 확인할 수 있습니다.
 

<div id="rate-limits">
  #### 요청 한도
</div>

매일 수천 명이 넘는 개발자가 X API에 요청을 보냅니다. 이러한 대규모 요청을 관리하기 위해 각 엔드포인트에는 [요청 한도](/ko/x-api/fundamentals/rate-limits)가 적용되며, 이를 통해 앱 또는 인증된 사용자를 대신해 보낼 수 있는 요청 수가 제한됩니다. 

이 엔드포인트에는 사용자 단위로 요청 한도가 적용됩니다. 이는 여러분이 요청을 수행하는 인증된 사용자가 어떤 개발자 앱을 사용하든, 해당 엔드포인트를 호출할 수 있는 횟수가 전체 개발자 앱을 통틀어 제한된다는 의미입니다. POST 및 DELETE 메서드에는 엔드포인트별로 15분당 50회의 사용자 요청 한도가 있습니다. 그러나 GET 메서드의 경우 요청 한도는 15분당 15회에 불과합니다.
 

<div id="fields-and-expansions">
  #### 필드와 expansions
</div>

X API v2 GET 엔드포인트를 사용하면 `fields`와 `expansions`라고 불리는 도구 모음을 통해 API에서 어떤 데이터를 응답으로 받을지 정확히 선택할 수 있습니다. `expansions` 파라미터를 사용하면 페이로드에서 참조된 객체를 확장할 수 있습니다. 예를 들어, 이 엔드포인트에서는 다음과 같은 [expansions](/ko/x-api/fundamentals/expansions)를 가져올 수 있습니다:

* pinned&#95;tweet&#95;id

**fields** 파라미터를 사용하면 서로 다른 데이터 객체 내에서 어떤 [fields](/ko/x-api/fundamentals/fields)를 받을지 정확히 선택할 수 있습니다. 이 엔드포인트는 주로 게시물(Post) 객체를 제공합니다. 기본적으로 게시물 객체는 **id**와 **text** 필드를 반환합니다. **tweet.created&#95;at** 또는 **tweet.entities**와 같은 추가 필드를 받으려면 **fields** 파라미터를 사용해 해당 필드를 명시적으로 요청해야 합니다. 통합 시 활용을 고려해볼 만한 중요한 필드로는 투표 데이터, 메트릭, 게시물 주석(annotations), 그리고 대화 ID 필드 등이 있습니다.

[X API v2 데이터 사전](/ko/x-api/fundamentals/data-dictionary)에 [fields와 expansions를 함께 사용하는 방법](/ko/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)에 대한 가이드를 추가해 두었습니다.
