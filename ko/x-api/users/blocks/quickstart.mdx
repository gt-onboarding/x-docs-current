<div id="getting-started-with-the-blocks-lookup-endpoint">
  ## blocks 조회 엔드포인트 사용 시작하기
</div>

이 빠른 시작 가이드는 [Postman](/ko/tutorials/postman-getting-started)을 사용하여 blocks 조회 엔드포인트에 첫 번째 요청을 보내는 방법을 설명합니다.

여러 언어로 된 샘플 코드를 보고 싶다면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요.

<Note>
  ### 사전 준비 사항

  이 가이드를 완료하려면 요청을 인증하기 위한 [keys and tokens](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따라 이러한 키와 토큰을 생성할 수 있습니다:

  * [개발자 계정에 가입](https://developer.x.com/en/apply-for-access)하여 승인을 받습니다.
  * 개발자 포털에서 [Project](/ko/resources/fundamentals/projects)를 생성하고, 연결된 [developer App](/ko/resources/fundamentals/developer-apps)을 만듭니다.
  * 앱의 “Keys and tokens” 페이지로 이동하여 필요한 자격 증명을 생성합니다. 모든 자격 증명을 안전한 위치에 저장해야 합니다.
</Note>

<div id="steps-to-build-a-blocks-lookup-request">
  ### 차단 조회 요청을 만드는 단계
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### 1단계: 도구 또는 라이브러리 선택
</div>

이 엔드포인트에 요청을 보내기 위해 사용할 수 있는 도구, 코드 예제, 라이브러리가 여러 가지 있지만, 여기서는 과정을 단순화하기 위해 Postman 도구를 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 불러오려면 다음 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2를 Postman에 추가
</Button>

Postman에 X API v2 컬렉션을 불러온 후, &quot;Blocks&quot; 폴더로 이동하여 &quot;Blocks Lookup&quot;을 선택합니다.
 

<div id="step-two-authenticate-your-request">
  #### 2단계: 요청 인증하기
</div>

X API에 정상적으로 요청을 보내려면, 해당 권한이 있는지 확인해야 합니다. 이 엔드포인트에서 이를 수행하려면 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용하여 요청을 인증해야 합니다.

이 예제에서는 OAuth 1.0a User Context를 사용합니다.

Postman에 키와 토큰, 구체적으로 API Key, API Secret Key, OAuth 1.0a 사용자 액세스 토큰, OAuth 1.0a 사용자 액세스 토큰 시크릿을 추가해야 합니다. Postman 오른쪽 상단에서 &quot;X API v2&quot;라는 이름의 환경(environment)을 선택한 다음, 환경 드롭다운 옆의 눈 모양 아이콘을 클릭해 &quot;initial value&quot;와 &quot;current value&quot; 필드에 키와 토큰을 추가하면 됩니다.

이 변수들은 올바르게 설정했다면 요청의 Authorization 탭에 자동으로 불러와집니다.
 

<div id="step-three-specify-a-user">
  #### 3단계: 사용자 지정
</div>

이 엔드포인트를 사용할 때는 차단한 사용자를 확인하기 위해 자신의 사용자 ID 또는 인증된 사용자의 사용자 ID를 지정해야 합니다.

Postman에서 &quot;Params&quot; 탭으로 이동한 후, 섹션의 맨 아래에 있는 id 경로 변수의 &quot;Value&quot; 열에 해당 사용자 ID를 입력합니다. 이때 사용자 이름 앞뒤에 공백이 포함되지 않도록 주의합니다.
 

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | (your user ID) |
| max&#95;results | 5   |

<div id="step-four-identify-and-specify-which-fields-you-would-like-to-retrieve">
  #### 4단계: 가져오려는 필드를 선택하고 지정하기
</div>

3단계까지 완료한 후 &quot;Send&quot; 버튼을 클릭하면, 응답에서 기본 [user object](/ko/x-api/fundamentals/data-dictionary#user) 필드인 id, name, username을 받게 됩니다.

id, name, username 외에 추가 필드를 받고 싶다면, 요청에서 [fields](/ko/x-api/fundamentals/fields) 및/또는 [expansions](/ko/x-api/fundamentals/expansions) 파라미터를 사용해 해당 필드를 지정해야 합니다.

이 연습에서는 서로 다른 객체에서 세 가지 추가 필드 묶음을 요청합니다:

1. 기본 user 객체의 추가 user.created&#95;at 필드
2. 반환된 사용자에 연관된 고정된 게시물 객체의 기본 필드: id 및 text
3. 연관된 게시물 객체의 추가 tweet.created&#95;at 필드

Postman에서 &quot;Params&quot; 탭으로 이동한 다음, &quot;Query Params&quot; 표에 다음 key:value 쌍을 추가합니다:

|              |                         |                                |
| :----------- | :---------------------- | :----------------------------- |
| **Key**      | **Value**               | **Returned fields**            |
| user.fields  | created&#95;at          | user.created&#95;at            |
| expansions   | pinned&#95;tweet&#95;id | tweet.id, tweet.text           |
| tweet.fields | created&#95;at          | includes.tweets.created&#95;at |

이제 &quot;Send&quot; 버튼 옆에 표시되는 URL에서 TwitterDev의 ID 대신 자신의 사용자 ID가 포함된, 이와 비슷한 URL이 보일 것입니다.

```
      https://api.x.com/2/users/2244994945/blocking?user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=created_at
```

<div id="step-six-paginate-through-your-results">
  #### 6단계: 결과에 페이지네이션 적용하기
</div>

응답 하단에 `meta` 객체가 포함되어 있는 것을 확인할 수 있습니다. `next_token`을 받은 경우, 가져올 수 있는 다음 페이지의 결과가 있다는 의미입니다. 다음 페이지의 결과를 가져오려면 `next_token` 필드의 값을 가져와 요청에 추가하고, 이를 추가적인 `pagination_token` 파라미터의 값으로 사용하면 됩니다.
 

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| pagination&#95;token | 1D3PU6DRII9HEZZZ |

이 추가 파라미터를 포함하여 요청을 보내면, 3단계에서 `max_results`를 5로 지정했기 때문에 다음 다섯 개의 결과가 다음 응답 페이로드에 포함되어 전달됩니다. 이 과정을 모든 결과를 받을 때까지 계속 반복할 수 있지만, `max_results` 파라미터를 사용해 한 번의 요청으로 최대 1000명의 사용자를 요청할 수도 있으므로, 그렇게 하면 페이지네이션을 자주 수행할 필요가 없습니다.
