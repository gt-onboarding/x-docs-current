인증된 사용자 조회를 제외한 모든 사용자 조회 엔드포인트에 대한 요청은 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication), [App only](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), 또는 [OAuth 2.0 Authorization code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용해 인증할 수 있습니다. 그러나 비공개 메트릭이나 비공개 사용자의 데이터에 액세스하려면 OAuth 1.0a User Context 또는 OAuth 2.0 Authorization Code with PKCE를 사용해야 하며, 요청에 인증된 사용자의 액세스 토큰을 함께 전달해야 합니다.    

이 엔드포인트는 사용자에 대한 최신 정보를 가져오거나, 사용자의 존재 여부를 확인하거나, 컴플라이언스 이벤트 이후에 저장된 사용자 정보를 업데이트하는 데 주로 사용됩니다.

<div id="authenticated-user-lookup">
  ### 인증된 사용자 조회
</div>

다른 사용자를 대신해 인증하는 경우, 어떤 사용자를 대신해 요청을 보낼 수 있는지 파악하는 것이 매우 중요합니다.

이 엔드포인트는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용해야 합니다. 이 엔드포인트는 인가된 사용자, 즉 요청에 포함해 전달하는 사용자 액세스 토큰과 연결된 사용자에 대한 정보를 반환합니다.

이 엔드포인트는 GET 메서드를 통해 호출할 수 있습니다. 이 엔드포인트에는 사용자 기준으로 15분당 75회의 요청 한도가 적용됩니다.

<Note>
  **계정 설정**

  이 엔드포인트에 접근하려면 다음이 필요합니다:

  * 승인된 [개발자 계정](https://developer.x.com/en/portal/petition/essential/basic-info)
  * [프로젝트](/ko/resources/fundamentals/projects) 내에 위치한 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용한 인증

  X API v2 엔드포인트에 대한 액세스 권한을 얻는 방법에 대해 더 알아보려면 [시작하기 가이드](/ko/x-api/getting-started/getting-access)를 참고하세요.
</Note>
