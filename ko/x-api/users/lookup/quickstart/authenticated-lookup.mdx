<div id="getting-started-with-the-authenticated-user-lookup-endpoint">
  ## Authenticated User Lookup 엔드포인트 시작하기
</div>

이 빠른 시작 안내서는 Postman을 사용하여 Authenticated User Lookup 엔드포인트에 처음으로 요청을 보내는 방법을 안내합니다.

여러 언어로 작성된 샘플 코드를 보고 싶다면 당사의 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지터리를 방문하세요.

<Note>
  ### 사전 준비 사항

  이 가이드를 완료하려면 요청을 인증할 때 사용할 [keys and tokens](/ko/resources/fundamentals/authentication)이 필요합니다. 다음 단계를 따라 키와 토큰을 생성할 수 있습니다:

  * [개발자 계정에 가입](https://developer.x.com/en/apply-for-access)하고 승인을 받으세요.
  * developer portal 내에서 [Project](/ko/resources/fundamentals/projects)와 연결된 [developer App](/ko/resources/fundamentals/developer-apps)을 생성하세요.
  * 앱의 &quot;Keys and tokens&quot; 페이지로 이동하여 필요한 자격 증명을 생성하세요. 모든 자격 증명은 반드시 안전한 위치에 보관해 두세요.
</Note>

<div id="steps-to-build-an-authenticated-user-lookup-request">
  ### 인증된 사용자 조회 요청을 구성하는 단계
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### 1단계: 도구 또는 라이브러리로 시작하기
</div>

이 엔드포인트에 요청을 보내기 위해 사용할 수 있는 도구, 코드 예제, 라이브러리가 여러 가지 있지만, 여기서는 과정을 단순화하기 위해 Postman 도구를 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 불러오려면 아래 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2를 Postman에 추가
</Button>

Postman에 X API v2 컬렉션을 불러온 후, &quot;Authenticated User Lookup&quot; 폴더로 이동한 다음 &quot;Lookup an Authenticated User&quot;를 선택합니다.

<div id="step-two-authenticate-your-request">
  #### 2단계: 요청 인증하기
</div>

X API에 요청을 제대로 보내려면, 먼저 해당 권한이 있는지 확인해야 합니다. 이 엔드포인트에서는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용해 요청을 인증해야 합니다.

이 예제에서는 OAuth 1.0a User Context를 사용합니다.

Postman에 키와 토큰, 구체적으로 API Key, API Secret Key, OAuth 1.0a 사용자 액세스 토큰, OAuth 1.0a 사용자 액세스 토큰 시크릿을 추가해야 합니다. 이를 위해 Postman 우측 상단에서 “X API v2”라는 이름의 환경(environment)을 선택하고, 환경 드롭다운 옆의 눈 아이콘을 클릭해 &quot;initial value&quot; 및 &quot;current value&quot; 필드에 키와 토큰을 추가합니다.

이 설정을 올바르게 완료했다면, 이 변수들은 요청의 Authorization 탭에 자동으로 반영됩니다.
 

<div id="step-three-determine-which-user-fields-you-want-to-retrieve">
  ##### 3단계: 가져오려는 user 필드를 결정합니다
</div>

3단계까지 완료한 후 &quot;Send&quot; 버튼을 클릭하면, 응답에서 기본 [user object](/ko/x-api/fundamentals/data-dictionary#user) 필드인 id, name, username을 받게 됩니다.

id, name, username 외의 추가 필드를 받고 싶다면, 요청에서 [field](/ko/x-api/fundamentals/fields) 및/또는 [expansion](/ko/x-api/fundamentals/expansions) 파라미터로 해당 필드를 지정해야 합니다.

이 연습에서는 서로 다른 객체에서 세 가지 추가 필드 집합을 요청합니다:

1. 기본 user 객체의 추가 user.created&#95;at 필드

2. 반환된 사용자에 연관된 고정된 게시물(pinned Posts) 객체의 기본 필드: id 및 text

3. 연관된 게시물(Post) 객체의 추가 tweet.created&#95;at 필드

Postman에서 &quot;Params&quot; 탭으로 이동한 다음 &quot;Query Params&quot; 테이블에 다음 key:value 쌍을 추가합니다:

|              |                               |                                                                   |
| :----------- | :---------------------------- | :---------------------------------------------------------------- |
| **Key**      | **Value**                     | **Returned fields**                                               |
| user.fields  | created&#95;at                | user.created&#95;at                                               |
| expansions   | pinned&#95;tweet&#95;id       | includes.tweets.id,  <br />includes.tweets.text                   |
| tweet.fields | created&#95;at, author&#95;id | includes.tweets.created&#95;at, includes&#95;tweets.author&#95;id |

이제 &quot;Send&quot; 버튼 옆에 이와 비슷한 URL이 표시되어야 합니다:

```
      https://api.x.com/2/users/me?user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=author_id,created_at
```
