<div id="getting-started-with-the-users-lookup-endpoints">
  ## users lookup 엔드포인트 사용 시작하기
</div>

이 빠른 시작 가이드는 [Postman](/ko/tutorials/postman-getting-started)을 사용해 지정한 필드 세트와 함께 users lookup 엔드포인트로 첫 번째 요청을 보내는 방법을 안내합니다.

여러 언어로 작성된 샘플 코드를 확인하려면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요.

<Note>
  ### 사전 준비 사항

  이 가이드를 완료하려면 요청을 인증하기 위한 [키와 토큰](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 아래 단계에 따라 이러한 키와 토큰을 생성할 수 있습니다.

  * [개발자 계정에 가입](https://developer.x.com/en/apply-for-access)하고 승인을 받습니다.
  * developer portal에서 [Project](/ko/resources/fundamentals/projects)와 연결된 [developer App](/ko/resources/fundamentals/developer-apps)을 생성합니다.
  * 앱의 &quot;Keys and tokens&quot; 페이지로 이동하여 필요한 자격 증명을 생성합니다. 모든 자격 증명은 안전한 위치에 저장해야 합니다.
</Note>

<div id="steps-to-build-a-users-lookup-request">
  ### Users lookup 요청을 만드는 단계
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### 1단계: 도구 또는 라이브러리부터 시작하기
</div>

이 엔드포인트에 요청을 보내는 데 사용할 수 있는 도구, 코드 예제, 라이브러리 등이 여러 가지 있지만, 여기서는 과정을 단순화하기 위해 Postman 도구를 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 불러오려면 다음 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2를 Postman에 추가
</Button>

Postman에 X API v2 컬렉션을 불러온 후, GET /users/by 엔드포인트로 이동하세요.
 

<div id="step-two-authenticate-your-request">
  #### 2단계: 요청 인증하기
</div>

X API에 올바르게 요청을 보내려면 먼저 해당 요청에 대한 권한이 있는지 확인해야 합니다. 이를 위해 이 엔드포인트는 [App only](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), 또는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 인증 방식을 사용해 요청을 인증할 것을 요구합니다.

설명을 단순하게 하기 위해 여기서는 이 요청에 App only를 사용하겠지만, 비공개 [메트릭](/ko/x-api/fundamentals/metrics)이나 비공개 사용자 데이터를 요청하려면 다른 인증 방식 중 하나를 사용해야 합니다. 

App only를 사용하려면 Postman에 키와 토큰, 특히 [App Access Token](/ko/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token) (App only Bearer Token이라고도 함)을 추가해야 합니다. 이를 위해 Postman 우측 상단에서 “X API v2”라는 이름의 environment를 선택한 뒤, 환경 드롭다운 옆의 눈 모양 아이콘을 클릭해 &quot;initial value&quot;와 &quot;current value&quot; 필드에 키와 토큰을 추가하면 됩니다.

이 작업을 올바르게 완료했다면, 이 변수들은 요청의 Authorization 탭에 자동으로 반영됩니다.
 

<div id="step-three-identify-and-specify-which-users-you-would-like-to-retrieve">
  #### 3단계: 조회하려는 사용자를 식별하고 지정하기
</div>

요청에서 조회하려는 사용자 또는 사용자 집합을 반드시 지정해야 합니다. 어떤 사용자 엔드포인트를 사용하느냐에 따라 user ID 또는 username을 전달할 수 있습니다. 이 예시에서는 하나의 요청으로 여러 username을 전달할 수 있는 [GET /users/by 엔드포인트](/ko/x-api/users/user-lookup-by-usernames)를 사용합니다. 이 엔드포인트를 사용하면 usernames 쿼리 매개변수를 통해 username 집합을 전달할 수 있으며, 이는 [single-ID](/ko/x-api/users/user-lookup-by-id), [multi-ID](/ko/x-api/users/user-lookup-by-ids), [single-username](/ko/x-api/users/user-lookup-by-username) 엔드포인트와는 다릅니다.

Username은 계정 프로필 URL에서 확인할 수 있는 계정 핸들을 의미합니다. 예를 들어, 다음 계정의 username은 XDevelopers입니다.

`https://x.com/XDevelopers`

Postman에서 &quot;Params&quot; 탭으로 이동한 후, 이 username 또는 쉼표로 구분한 username 문자열을 username 매개변수의 &quot;Value&quot; 열에 입력합니다. 이때 username과 쉼표 사이에 공백이 들어가지 않도록 주의하세요. 

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `username` | XDeveloeprs,API,adsapi |

<div id="step-four-identify-and-specify-which-fields-you-would-like-to-retrieve">
  #### 4단계: 조회하려는 필드를 식별하고 지정하기
</div>

3단계까지 완료한 뒤 &quot;Send&quot; 버튼을 클릭하면, 응답에서 기본 [user object](/ko/x-api/fundamentals/data-dictionary#user) 필드인 id, name, username을 받게 됩니다.

id, name, username 외에 추가 필드를 받고 싶다면, 요청에서 [field](/ko/x-api/fundamentals/fields) 및/또는 [expansion](/ko/x-api/fundamentals/expansions) 파라미터를 사용해 해당 필드를 지정해야 합니다.

이 연습에서는 서로 다른 오브젝트에서 세 개의 추가 필드 세트를 요청해 보겠습니다.

1. 기본 user 오브젝트의 추가 user.created&#95;at 필드
2. 반환된 사용자에 연결된 고정 게시물(pinned Posts) 오브젝트의 기본 필드: id와 text
3. 연결된 게시물(Post) 오브젝트의 추가 tweet.created&#95;at 필드
    

Postman에서 &quot;Params&quot; 탭으로 이동한 후 &quot;Query Params&quot; 테이블에 다음 key:value 쌍을 추가합니다:

|                |              |                             |
| :------------- | :----------- | :-------------------------- |
| **Key**        | **Value**    | **반환되는 필드**                 |
| `user.fields`  | `created_at` | `user.created_at`           |
| `expansions`   | `author_id`  | tweet.id, tweet.text        |
| `tweet.fields` | `created_at` | `includes.users.created_at` |

이제 &quot;Send&quot; 버튼 옆에 다음 URL이 표시되는 것을 볼 수 있습니다:

```
      https://api.x.com/2/users/by?usernames=twitterdev,twitterapi,adsapi&user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=author_id,created_at
```
