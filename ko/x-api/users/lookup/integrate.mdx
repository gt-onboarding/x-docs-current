<div id="postman">

Postman은 엔드포인트를 시험해 볼 수 있는 매우 유용한 도구입니다. 각 Postman 요청에는 모든 path 및 body 매개변수가 포함되어 있어, 사용할 수 있는 항목을 빠르게 파악하는 데 도움이 됩니다. Postman 컬렉션에 대해 더 자세히 알아보려면 [&quot;Using Postman&quot;](/ko/tutorials/postman-getting-started) 페이지를 방문하세요.

<div id="authentication">
  #### Authentication
</div>

모든 X API v2 엔드포인트는 키와 토큰이라고도 하는 자격 증명 세트를 사용해 [인증](/ko/resources/fundamentals/authentication)된 요청을 필요로 합니다. 이러한 엔드포인트에 대한 요청을 인증하려면 OAuth 1.0a 사용자 컨텍스트, App only, 또는 OAuth 2.0 Authorization Code with PKCE 중 하나를 사용할 수 있습니다. 

[OAuth 1.0a User Context](/ko/resources/fundamentals/authentication#oauth-1-0a-2)를 사용하려면 요청에 함께 전달할 [Authorization 헤더를 생성](/ko/resources/fundamentals/authentication#authorizing-a-request)하기 위해 API 키, 사용자 액세스 토큰, 그리고 몇 가지 다른 파라미터를 사용해야 합니다. 액세스 토큰은 요청을 대신 수행하려는 사용자를 기준으로 발급되어야 합니다. 다른 사용자를 위한 액세스 토큰 세트를 생성하려면 해당 사용자가 [3-legged OAuth 플로우](https://developer-staging.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)를 사용해 귀하의 App을 승인해야 합니다. 

OAuth 1.0a는 사용하기 어려울 수 있다는 점에 유의하세요. 이 인증 방식에 익숙하지 않다면 [라이브러리](/ko/x-api/tools-and-libraries/overview)를 사용하거나, Postman 같은 도구를 사용하거나, OAuth 2.0을 사용해 요청을 인증할 것을 권장합니다. 이러한 엔드포인트에서 게시물이나 비공개 메트릭을 요청하려는 경우, 콘텐츠를 소유한 사용자로부터 적절한 권한을 확보하기 위해 OAuth 1.0a User Context 또는 OAuth 2.0 Authorization Code with PKCE 중 하나를 사용해야 합니다.
 

[App only](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)는 요청에 [App only Access Token](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)만 함께 전달하면 됩니다. App only Access Token은 개발자 앱 내에서 직접 생성하거나 [POST oauth2/token](/ko/resources/fundamentals/authentication#post-oauth2-token) 엔드포인트를 사용해 생성할 수 있습니다.

[OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)는 애플리케이션의 스코프와 여러 기기에 걸친 인증 플로우를 보다 세밀하게 제어할 수 있게 해 줍니다. OAuth 2.0을 사용하면 사용자 대신 수행할 수 있는 구체적인 권한을 제공하는 세분화된 스코프를 선택할 수 있습니다. 

앱에서 OAuth 2.0을 활성화하려면 developer portal의 App settings 섹션에 있는 앱의 authentication settings에서 이를 켜야 합니다.

**참고**

다음 필드를 요청하는 경우 OAuth 1.0a User Context 또는 OAuth 2.0 Authorization Code가 필요합니다: 

* tweet.fields.non&#95;public&#95;metrics
* tweet.fields.promoted&#95;metrics
* tweet.fields.organic&#95;metrics

X API v2 엔드포인트에서 사용할 수 있는 인증 자격 증명 세트를 발급받으려면 [개발자 계정에 가입](https://developer.x.com/en/portal/petition/essential/basic-info)하고, 해당 계정 내에 [프로젝트](/ko/resources/fundamentals/projects)를 설정한 뒤, 그 프로젝트 내에 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성해야 합니다. 이후 개발자 앱에서 키와 토큰을 확인할 수 있습니다. 
 

<div id="rate-limits">
  #### 요청 한도
</div>

매일 수천 명이 넘는 개발자가 X API에 요청을 보냅니다. 이러한 요청의 방대한 양을 관리하기 위해 각 엔드포인트에는 [요청 한도](/ko/x-api/fundamentals/rate-limits)가 설정되어 있어, 앱을 대신하거나 인증된 사용자를 대신해 보낼 수 있는 요청 수가 제한됩니다. 

사용자 조회(user lookup) 엔드포인트에는 앱 수준과 사용자 수준 두 가지 요청 한도가 적용됩니다. 그러나 인증된 사용자 조회(authenticated user lookup) 엔드포인트는 사용자 수준에서만 요청 한도가 적용됩니다.

앱 수준 요청 한도는 개발자인 여러분이, 특정 기간 동안 여러분이 사용 중인 키와 토큰으로 식별되는 어떤 앱에서든 이 엔드포인트로 보낼 수 있는 요청 수에 제한이 있음을 의미합니다. 사용자 수준 요청 한도는 여러분이 요청을 대신해서 보내는 인증된 사용자가, 어떤 개발자 앱에서든 수행할 수 있는 요청 횟수에 제한이 있음을 의미합니다.

아래 표는 각 엔드포인트에 대한 요청 한도를 보여줍니다.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **HTTP method** | **Rate limit / Level** |
| /2/users | GET | 15분당 900회 요청 / App 및 User |
| /2/users/:id | GET | 15분당 900회 요청 / App 및 User |
| /2/users/by | GET | 15분당 900회 요청 / App 및 User |
| /2/users/by/username/:username | GET | 15분당 900회 요청 / App 및 User |
| /2/users/me | GET | 15분당 75회 요청 / User |

<div id="fields-and-expansions">
  #### 필드와 expansions
</div>

X API v2를 사용하면 `fields` 및 `expansions`라고 하는 도구 세트를 통해 API에서 어떤 데이터를 반환받을지 정확히 선택할 수 있습니다. `expansions` 파라미터를 사용하면 페이로드에서 참조되는 객체를 전체 객체로 확장해 포함시킬 수 있습니다. 예를 들어, 이 엔드포인트에서는 `pinned_tweet_id` expansion을 사용할 수 있습니다.

`fields` 파라미터를 사용하면 서로 다른 데이터 객체 내에서 어떤 [fields](/ko/x-api/fundamentals/fields)를 받을지 정확히 선택할 수 있습니다. 이 엔드포인트들은 주로 user 객체를 반환합니다. 기본적으로 user 객체는 `id`, `name`, `username` 필드를 반환합니다. `user.created_at` 또는 `user.location`과 같은 추가 필드를 받으려면 `fields` 파라미터를 사용해 해당 필드를 명시적으로 요청해야 합니다. 통합 시 고려할 만한 중요한 필드로는 게시물 투표 데이터, 메트릭, 주석(annotations), 대화 ID 필드 등이 있습니다.

[X API v2 데이터 사전](/ko/x-api/fundamentals/data-dictionary)에 [fields와 expansions 사용 방법](/ko/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)에 대한 가이드를 제공하고 있습니다.
