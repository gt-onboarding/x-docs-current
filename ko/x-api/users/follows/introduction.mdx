팔로우 관계를 조회, 생성, 삭제할 수 있도록 `follows lookup`과 `manage follows`라는 두 가지 엔드포인트 그룹을 제공합니다.

<div id="follows-lookup">
  ### 팔로우 조회
</div>

[Enterprise 액세스](/ko/x-api/getting-started/about-x-api)가 있는 경우에만 사용할 수 있습니다.

팔로우 조회 엔드포인트를 사용하면 일명 네트워크 분석이라고도 하는 사용자 간 관계를 탐색하고 분석할 수 있습니다. 구체적으로, 지정한 사용자가 누구를 팔로잉하는지 또는 누가 지정한 사용자를 팔로잉하는지를 나타내는 [user objects](/ko/x-api/fundamentals/data-dictionary#user)를 반환하는 두 개의 REST 엔드포인트가 있습니다.

이 엔드포인트는 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication), [App only](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), 또는 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 중 하나를 사용해 인증할 수 있습니다. 요청당 최대 1,000명의 사용자를 요청할 수 있으며, 대량의 결과 세트를 페이지 단위로 탐색할 수 있도록 페이지네이션 토큰이 제공됩니다.
 

<div id="manage-follows">
  ### 팔로우 관리
</div>

팔로우 관리 엔드포인트를 사용하면 사용자를 팔로우하거나 언팔로우할 수 있습니다.

사용자를 대신해 요청을 보내는 것이므로, 이 엔드포인트는 반드시 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용해 인증해야 하며, 요청을 수행하는 대상 사용자와 연결된 사용자 액세스 토큰을 사용해야 합니다. 이 사용자 액세스 토큰은 [3-legged OAuth 플로우](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) 또는 [Authorization Code with PKCE grant flow](/ko/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)) (OAuth 2.0)를 사용하여 생성할 수 있습니다.

각 인증된 사용자를 대신해 하루에 수행할 수 있는 팔로우 작업은 400회로 제한되며, 모든 인증된 사용자를 통틀어 앱당 하루 1,000회로 제한됩니다. 예를 들어, 인증된 사용자가 다섯 명 있는 경우, 그중 두 명의 사용자로 하루에 각각 400명씩(사용자별 한도) 총 800명을 팔로우할 수 있으며, 나머지 세 명의 사용자에게 남은 200회(앱당 한도)를 나누어 사용해야 합니다. 이 한도는 언팔로우 엔드포인트에는 적용되지 않으며, 언팔로우 엔드포인트에는 앱당 하루 500회의 별도 한도가 적용됩니다.

<Note>
  **계정 설정**

  이 엔드포인트에 액세스하려면 다음이 필요합니다.

  * 승인된 [개발자 계정](https://developer.x.com/en/portal/petition/essential/basic-info)
  * [프로젝트](/ko/resources/fundamentals/projects) 내에 있는 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용해 인증할 수 있어야 합니다. 
</Note>
