  ## X API 팔로우 관련 엔드포인트 비교
</div>

<div id="follows-lookup">
  ### 팔로우 조회
</div>

v2 팔로우 조회 엔드포인트는 기존 v1.1 [followers/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-followers-ids), v1.1 [followers/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-followers-list), v1.1 [friends/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-friends-ids), v1.1 [friends/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-friends-list) 엔드포인트를 대체합니다.

다음 표는 다양한 유형의 팔로우 조회 엔드포인트를 비교합니다:

|     |     |     |
| :--- | :--- | :--- |
| **설명** | **표준 v1.1** | **X API v2** |
| 지원되는 HTTP 메서드 | `GET` | `GET` |
| 호스트 도메인 | `https://api.x.com` | `https://api.x.com` |
| 엔드포인트 경로 | /1.1/friends/ids.json<br /><br />/1.1/friends/list.json<br /><br />/1.1/followers/ids.json<br /><br />/1.1/followers/list.json | /2/users/:id/following<br /><br />/2/users/:id/followers |
| [인증](/ko/resources/fundamentals/authentication) | OAuth 1.0a 사용자 컨텍스트<br /><br />앱 전용 | OAuth 1.0a 사용자 컨텍스트<br /><br />OAuth 2.0 Authorization Code with PKCE<br /><br />앱 전용 |
| 기본 요청 [요청 한도](/ko/resources/fundamentals/rate-limits) | 15분당 15회 요청(사용자당)<br /><br />15분당 15회 요청(앱당) | 15분당 15회 요청(사용자당)<br /><br />15분당 15회 요청(앱당) |
| 응답당 최대 사용자 수 | GET friends/ids 및 GET followers/ids는 페이지당 최대 5000개의 사용자 ID를 반환합니다.<br /><br />  <br /><br />GET friends/list 및 GET followers/list는 페이지당 최대 200개의 사용자 객체를 반환합니다. | 페이지당 1000개의 사용자 객체 |
| 페이지네이션 | 토큰은 next&#95;cursor 필드로 반환되며, 이후 cursor 파라미터의 값으로 전달하여 다음 페이지 결과를 반환할 수 있습니다. | 토큰은 next&#95;token 필드로 반환되며, 이후 token 파라미터의 값으로 전달하여 다음 페이지 결과를 반환할 수 있습니다.<br /><br />v2 페이로드에는 previous&#95;token 필드도 포함되며, 이를 pagination&#95;token 파라미터와 함께 전달하여 이전 페이지 결과를 반환할 수 있습니다. |
| JSON 포맷 | 표준 v1.1 포맷 | [X API v2 포맷](/ko/x-api/fundamentals/data-dictionary) (fields 및 expansions 요청 파라미터에 의해 결정되며, v1.1 포맷과는 하위 호환되지 않음)<br /><br />표준 v1.1 포맷에서 X API v2 포맷으로 마이그레이션하는 방법에 대해 더 알아보려면 [데이터 포맷 마이그레이션 가이드](/ko/x-api/migrate/data-format-migration)를 방문하세요. |
| 페이로드에 어떤 [fields](/ko/x-api/fundamentals/data-dictionary)를 포함해 반환할지 선택 가능 |     | ✔   |
| 게시물 [annotations](/ko/x-api/fundamentals/post-annotations) 필드 지원 |     | ✔   |
| 새로운 [metrics](/ko/x-api/fundamentals/metrics) 필드 요청 지원 |     | ✔   |
| [conversation&#95;id](/ko/x-api/fundamentals/conversation-id) 필드 지원 |     | ✔   |
| [프로젝트](/ko/resources/fundamentals/projects)와 연결된 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 자격 증명 사용 필요 |     | ✔   |

<div id="manage-follows">
  #### 팔로우 관리
</div>

v2 팔로우 관리 엔드포인트는 기존 표준 v1.1 [POST friendships/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-create) 및 [POST friendships/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-destroy) 엔드포인트를 대체합니다.

다음 표는 표준 v1.1 엔드포인트와 X API v2의 팔로우 생성 엔드포인트를 비교한 것입니다:

<div id="follow-a-user">
  #### 사용자를 팔로우하기
</div>

| 설명 | 표준 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 지원 HTTP 메서드 | POST | POST |
| 호스트 도메인 | https://api.x.com | https://api.x.com |
| 엔드포인트 경로 | /1.1/friendships/create.json | /2/users/:id/following |
| [인증](/ko/resources/fundamentals/authentication) | OAuth 1.0a 사용자 컨텍스트 | OAuth 1.0a 사용자 컨텍스트<br /><br />OAuth 2.0 Authorization Code with PKCE |
| 기본 요청 [요청 한도](/ko/resources/fundamentals/rate-limits) | 15분당 50회 요청 | 15분당 50회 요청 |
| 사용자당 일일 최대 작업 수 | 400 | 400 |
| 앱당 일일 최대 작업 수 | 1000 | 1000 |
| [프로젝트](/ko/resources/fundamentals/projects)와 연결된 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 자격 증명 사용 필요 |     | ✔️  |

<div id="unfollow-a-user">
  #### 사용자 언팔로우
</div>

아래 표는 Standard v1.1과 X API v2의 팔로우 삭제 엔드포인트를 비교합니다:

| 설명 | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| 지원되는 HTTP 메서드 | POST | DELETE |
| 호스트 도메인 | https://api.x.com | https://api.x.com |
| 엔드포인트 경로 | /1.1/friendships/destroy.json | /2/users/:source&#95;user&#95;id/following/:target&#95;user&#95;id |
| [인증](/ko/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code with PKCE |
| 기본 요청 [요청 한도](/ko/resources/fundamentals/rate-limits) | 15분당 15개 요청(사용자 기준) | 15분당 50개 요청(사용자 기준) |
| 앱당 하루 최대 작업 수 | 없음 | 500 |
| [프로젝트](/ko/resources/fundamentals/projects)와 연결된 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 자격 증명 사용 필요 |     | ✔️  |
