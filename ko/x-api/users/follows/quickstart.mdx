<div id="getting-started-with-the-manage-follows-endpoints">
  ## 팔로잉 관리 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 [Postman](/ko/tutorials/postman-getting-started)을 사용해 처음으로 팔로잉 관리 엔드포인트에 요청을 보내는 방법을 안내합니다.

여러 언어로 된 샘플 코드를 보고 싶다면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요.

<Note>
  ### 사전 준비 사항

  이 가이드를 완료하려면 요청을 인증할 때 사용할 [키와 토큰](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따라 이러한 키와 토큰을 생성할 수 있습니다:

  * [개발자 계정에 가입](https://developer.x.com/en/apply-for-access)하고 승인을 받으세요.
  * developer portal에서 [프로젝트](/ko/resources/fundamentals/projects)와 이에 연결된 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성하세요.
  * 앱의 “Keys and tokens” 페이지로 이동하여 필요한 자격 증명을 생성하세요. 모든 자격 증명은 안전한 위치에 저장해야 합니다.
</Note>

<div id="steps-to-build-a-manage-follows-request">
  ### manage follows 요청을 구성하는 단계
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### 1단계: 도구 또는 라이브러리로 시작하기
</div>

이 endpoint에 요청을 보내는 데 사용할 수 있는 도구, 코드 예제, 라이브러리는 여러 가지가 있지만 여기서는 과정을 단순화하기 위해 Postman 도구를 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 불러오려면 다음 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2를 Postman에 추가
</Button>

Postman에 X API v2 컬렉션을 불러온 후 &quot;Follows&quot; 폴더로 이동한 다음 &quot;Follow a user ID&quot;를 선택합니다.
 

<div id="step-two-authenticate-your-request">
  #### 2단계: 요청에 인증 추가하기
</div>

X API에 정상적으로 요청하려면, 해당 권한이 있는지 먼저 인증해야 합니다. 이 엔드포인트를 사용할 때에는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용해 요청에 인증을 추가해야 합니다.

이 예제에서는 OAuth 1.0a User Context를 사용합니다.

Postman에 키와 토큰, 구체적으로 API Key, API Secret Key, OAuth 1.0a 사용자 액세스 토큰, OAuth 1.0a 사용자 액세스 토큰 시크릿을 추가해야 합니다. 이를 위해 Postman 오른쪽 상단에서 &quot;X API v2&quot;라는 이름의 environment를 선택한 뒤, 환경 드롭다운 옆에 있는 눈 아이콘을 클릭하여 표시되는 &quot;initial value&quot; 및 &quot;current value&quot; 필드에 키와 토큰을 추가합니다.

이렇게 설정하면, 이 변수들은 요청의 Authorization 탭에 자동으로 채워집니다.
 

<div id="step-three-specify-who-is-going-to-follow-whom">
  #### 3단계: 누가 누구를 팔로우할지 지정하기
</div>

Manage follows 엔드포인트는 두 개의 ID를 사용합니다. 하나는 소스 사용자(다른 사용자를 팔로우하거나 언팔로우하려는 사용자)용이고, 다른 하나는 타깃 사용자(팔로우되거나 언팔로우될 사용자)용입니다. 소스 사용자의 ID는 인증한 사용자의 user ID와 일치해야 합니다. 이 경우 본인 계정의 ID를 지정하면 됩니다. 자신의 ID는 다음 두 가지 방법으로 찾을 수 있습니다.

1. [username으로 사용자 조회](/ko/x-api/users/user-lookup-by-id) 엔드포인트를 사용하여 username을 전달하면 `id` 필드를 받을 수 있습니다. 
2. 자신의 액세스 토큰을 보면, 숫자로 된 부분이 자신의 user ID입니다.
    

타깃 ID는 어떤 유효한 user ID여도 됩니다. 예를 들어 @XDevelopers의 user ID는 2244994945입니다.

Postman에서 &quot;Params&quot; 탭으로 이동하여, id 경로 변수의 &quot;Value&quot; 열에 자신의 ID를 입력합니다. 그런 다음 “Body” 탭으로 이동하여 target&#95;user&#95;id 파라미터의 값으로 2244994945(@XDevelopers의 user ID)를 입력합니다. 이때 어떤 ID 앞뒤에도 공백이 포함되지 않도록 하세요.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | (your user ID) |
| target&#95;user&#95;id | 2244994945 |

&quot;Send&quot; 버튼을 클릭하면 관계 상태를 포함하는 응답 객체를 받게 됩니다.

* &quot;following&quot;: true를 받으면, 해당 id가 target&#95;user&#95;id를 성공적으로 팔로잉하고 있는 것입니다.
* &quot;pending&quot;: true를 받으면, target&#95;user&#95;id가 보호된 계정이므로 팔로우 요청을 수락해야 합니다.

####

4단계: 요청 보내기 및 응답 검토

모든 준비가 끝났다면 “Send” 버튼을 눌러 요청을 보내고, 다음과 같은 응답을 받게 됩니다.

```{
    "data": {
        "following": true,
        "pending_follow": false
    }
}
```

마찬가지로, 사용자를 언팔로우하려면 동일한 Postman 컬렉션 내에서 &quot;Unfollow a user ID&quot; 요청을 사용하면 됩니다. 그러나 `source_user_id` 및 `target_user_id` 매개변수는 언팔로우 엔드포인트를 사용할 때 경로 변수로 전달해야 합니다.
