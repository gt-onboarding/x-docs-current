<div id="getting-started-with-the-manage-follows-endpoints">
  ## 팔로우 관리 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 [Postman](/ko/tutorials/postman-getting-started)을 사용하여 팔로우 관리 엔드포인트에 첫 요청을 보내는 방법을 안내합니다.

다른 언어로 된 샘플 코드를 보고 싶다면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요.

<Note>
  ### 사전 준비 사항

  이 가이드를 완료하려면 요청을 인증할 [keys and tokens](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따르면 이러한 키와 토큰을 생성할 수 있습니다:

  * [개발자 계정에 등록](https://developer.x.com/en/apply-for-access)하고 승인을 받으세요.
  * developer portal에서 [Project](/ko/resources/fundamentals/projects)와 연결된 [developer App](/ko/resources/fundamentals/developer-apps)을 생성하세요.
  * App의 “Keys and tokens” 페이지로 이동하여 필요한 자격 증명을 생성하세요. 모든 자격 증명은 안전한 위치에 저장해야 합니다.
</Note>

<div id="steps-to-build-a-manage-follows-request">
  ### manage follows 요청을 만드는 단계
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### 1단계: 도구 또는 라이브러리 선택
</div>

이 엔드포인트에 요청을 보내는 데 사용할 수 있는 도구, 코드 예제, 라이브러리가 여러 가지 있지만, 여기서는 과정을 단순화하기 위해 Postman을 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 불러오려면 아래 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2를 Postman에 추가
</Button>

Postman에 X API v2 컬렉션을 불러온 다음, &quot;Follows&quot; 폴더로 이동해 &quot;Follow a user ID&quot;를 선택하세요.
 

<div id="step-two-authenticate-your-request">
  #### 2단계: 요청 인증하기
</div>

X API에 정상적으로 요청을 보내려면 먼저 해당 권한이 있는지 확인해야 합니다. 이 엔드포인트에서는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용해 요청을 인증해야 합니다.

이 예제에서는 OAuth 1.0a User Context를 사용합니다.

Postman에 키와 토큰, 즉 API Key, API Secret Key, OAuth 1.0a 사용자 액세스 토큰, OAuth 1.0a 사용자 액세스 토큰 시크릿을 추가해야 합니다. 이를 위해 Postman 오른쪽 상단에서 “X API v2”라는 이름의 environment(환경)를 선택한 뒤, 해당 environment 드롭다운 옆의 눈 모양 아이콘을 클릭하여 &quot;initial value&quot;와 &quot;current value&quot; 필드에 키와 토큰을 추가합니다.

올바르게 설정했다면, 이 변수들은 요청의 Authorization 탭에 자동으로 불러와집니다.
 

<div id="step-three-specify-who-is-going-to-follow-whom">
  #### 3단계: 누가 누구를 팔로우할지 지정하기
</div>

Manage follows 엔드포인트는 두 개의 ID를 사용합니다. 하나는 source 사용자(다른 사용자를 팔로우하거나 언팔로우하려는 사용자)용이고, 다른 하나는 target 사용자(팔로우 또는 언팔로우 대상이 되는 사용자)용입니다. source 사용자의 ID는 인증된 사용자의 사용자 ID와 일치해야 합니다. 이 경우에는 자신의 계정에 속한 ID를 지정하면 됩니다. 자신의 ID는 다음 두 가지 방법으로 확인할 수 있습니다:

1. [user lookup by username](/ko/x-api/users/user-lookup-by-id) 엔드포인트를 사용하여 username을 전달하면 `id` 필드를 받을 수 있습니다. 
2. 자신의 액세스 토큰을 보면, 숫자 부분이 자신의 사용자 ID입니다.
    

target ID는 유효한 어떤 사용자 ID든 사용할 수 있습니다. 예를 들어 @XDevelopers의 사용자 ID는 2244994945입니다.

Postman에서 &quot;Params&quot; 탭으로 이동한 다음, `id` path 변수의 &quot;Value&quot; 열에 자신의 ID를 입력합니다. “Body” 탭으로 이동하여 target&#95;user&#95;id 파라미터의 값으로 2244994945(@XDevelopers의 사용자 ID)를 입력합니다. 어떤 ID 앞이나 뒤에도 공백이 포함되지 않도록 주의합니다.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | (your user ID) |
| target&#95;user&#95;id | 2244994945 |

&quot;Send&quot; 버튼을 클릭하면 관계 상태를 포함한 응답 객체를 받게 됩니다:

* &quot;following&quot;: true 를 받으면, 해당 `id`가 `target_user_id`를 성공적으로 팔로잉하고 있다는 의미입니다.
* &quot;pending&quot;: true 를 받으면, `target_user_id`가 보호된 계정이며 팔로우 요청을 승인해야 한다는 의미입니다.

####

4단계: 요청을 보내고 응답을 확인합니다

모든 설정을 완료했으면 &quot;Send&quot; 버튼을 눌러 요청을 전송하세요. 그러면 다음과 같은 응답을 받게 됩니다:

```{
    "data": {
        "following": true,
        "pending_follow": false
    }
}
```

마찬가지로, 사용자를 언팔로우하려는 경우에는 동일한 Postman 컬렉션 내의 &quot;Unfollow a user ID&quot; 요청을 사용하면 됩니다. 다만 언팔로우 endpoint를 사용할 때는 source&#95;user&#95;id와 target&#95;user&#95;id 파라미터를 모두 경로 변수로 전달해야 합니다.
