[`POST /2/media/upload`](/ko/x-api/media/media-upload) 엔드포인트를 사용할 때 이해해야 할 몇 가지 중요한 개념이 있습니다. OAuth로 미디어를 업로드하는 것은 다소 까다로울 수 있으므로, 이 문서에서는 염두에 두어야 할 사항들과 이 엔드포인트 사용 방법을 보여 주는 실제 예제를 함께 제공합니다.

<div id="keep-in-mind">
  ## 기억해야 할 사항
</div>

* 하나의 게시물에 최대 사진 4장, 애니메이션 GIF 1개 또는 동영상 1개까지 첨부할 수 있습니다.
* 전달되는 이미지는 원본 바이너리 데이터이거나 바이너리 base64 인코딩이어야 하며, `Content-Type`이 적절하게 설정되어 있는 한 별도로 내용을 인코딩하거나 이스케이프할 필요는 없습니다(확실하지 않다면 `application/octet-stream`을 사용하세요).
* base64로 인코딩된 이미지를 게시할 때는 메시지의 이미지 부분에 `Content-Transfer-Encoding: base64`를 반드시 설정해야 합니다.
* 멀티파트 메시지의 경계(boundary)는 각각 독립된 줄에 있어야 하며 CRLF로 종료되어야 합니다.
* 이 엔드포인트를 사용해 POST하는 실제 예시는 [xurl](https://github.com/xdevplatform/xurl)로 테스트해 보기를 권장합니다. 또한, 사용 가능한 [X Libraries](/ko/resources/tools-and-libraries)도 살펴보세요.
* JavaScript 및 long integer를 정확하게 표현할 수 없는 기타 언어에서는 API 응답에 제공되는 `media_id_string`을 사용하세요.

<div id="media-categories">
  ## 미디어 카테고리
</div>

Media Category 매개변수는 업로드할 미디어 파일의 사용 용도를 정의하며, 업로드 시 적용되는 파일 크기 제한이나 기타 제약 조건에 영향을 줄 수 있습니다. 이후 미디어를 사용할 때 문제가 발생하지 않도록, 미디어를 업로드할 때 올바른 media category를 사용하는 것이 중요합니다. 이 값은 업로드 플로우의 일부로 INIT 요청에 포함되어 전달되는 선택적 값입니다. media category가 지정되지 않은 경우, 업로드된 미디어는 콘텐츠 타입에 따라 게시물용 미디어(`tweet_image`, `tweet_video`, 또는 `tweet_gif`)로 간주됩니다.

가장 일반적인 media category는 다음과 같습니다:

* `tweet_image`
* `tweet_video`
* `tweet_gif`
* `dm_image`
* `dm_video`
* `dm_gif`
* `subtitles`

Ads API 파트너인 경우, 프로모션 동영상에 권장되는 media category에 대한 자세한 내용은 [이 문서](/ko/x-ads-api/creatives#promoted-video)를 참고하세요.

<div id="image-specifications-and-recommendations">
  ## 이미지 사양 및 권장사항
</div>

이미지 파일은 다음 기준을 모두 충족해야 합니다.

* **지원되는 이미지 미디어 타입**: `JPG`, `PNG`, `GIF`, `WEBP`
* **이미지 크기**: `<= 5 MB`
* **애니메이션 GIF 크기**: `<= 15 MB`

위의 파일 크기 제한은 미디어 업로드 엔드포인트에서 적용됩니다. 추가로, `media_id`와 함께 게시물 생성(또는 유사한) 엔드포인트를 호출할 때 적용되는 제품 엔터티별 별도 파일 크기 제한이 있습니다. 파일 크기 제한 및 기타 제약 조건은 `media_category` 매개변수에 따라 달라질 수 있습니다.

<div id="animated-gif-recommendations">
  ## 애니메이션 GIF 권장사항
</div>

GIF는 파일 크기 제한을 충족하더라도 게시물 생성 중에 실패할 수 있습니다. 성공률을 높이려면 다음 제약사항을 준수하세요.

* **해상도**: `<= 1280x1080` (`width` x `height`)
* **프레임 수**: `<= 350`
* **픽셀 수**: `<= 300 million` (`width` * `height` * `num_frames`)
* **파일 크기**: `<= 15MB`

더 큰 GIF를 처리하려면 `media_category` 매개변수와 함께 [chunked upload](/ko/x-api/media/quickstart/media-upload-chunked) 엔드포인트를 사용하세요. 이렇게 하면 서버가 GIF 파일을 비동기적으로 처리할 수 있으며, 이는 더 큰 파일을 처리하기 위한 필수 조건입니다. 애니메이션 GIF가 포함된 게시물에 대해 비동기 업로드 방식을 사용하려면 `media_category=tweet_gif`를 지정하세요.

<div id="video-specifications-and-recommendations">
  ## 동영상 사양 및 권장사항
</div>

미디어 업로드에는 Async Path를 사용하세요.

<div id="recommended">
  ### 권장 사양
</div>

* **비디오 코덱**: `H264 High Profile`
* **프레임 레이트**: `30 FPS`, `60 FPS`
* **비디오 해상도**: `1280x720` (가로), `720x1280` (세로), `720x720` (정사각형). 구독 중인 사용자는 1080p 비디오를 업로드하여 1080p로 재생할 수 있습니다. 구독하지 않은 사용자는 720p 비디오를 업로드하여 720p로 재생할 수 있습니다.
* **최소 비디오 비트레이트**: `5,000 kbps`
* **최소 오디오 비트레이트**: `128 kbps`
* **오디오 코덱**: `AAC LC`
* **종횡비(가로:세로 비율)**: `16:9` (가로 또는 세로), `1:1` (정사각형)

<div id="advanced">
  ### 고급
</div>

* **프레임 레이트**: `60 FPS` 이하여야 합니다
* **해상도**: `32x32`에서 `1280x1024` 사이여야 합니다
* **파일 크기**: `512 MB`를 초과해서는 안 됩니다
* **재생 시간**: `0.5초`에서 `140초` 사이여야 합니다
* **종횡비(가로세로 비율)**: `1:3`에서 `3:1` 사이여야 합니다
* **[픽셀 종횡비](https://en.wikipedia.org/wiki/Pixel_aspect_ratio)**: `1:1`이어야 합니다
* **픽셀 포맷**: [YUV](https://en.wikipedia.org/wiki/YUV) 4:2:0만 지원됩니다
* 오디오는 [`Low Complexity` 프로파일의 `AAC`](https://en.wikipedia.org/wiki/Advanced_Audio_Coding#Modular_encoding)여야 합니다. (`High-Efficiency` `AAC`는 지원되지 않습니다)
* 오디오는 `mono` 또는 `stereo`여야 하며, 5.1 이상은 허용되지 않습니다
* [`open GOP`](https://en.wikipedia.org/wiki/Group_of_pictures)를 사용하면 안 됩니다
* [`progressive scan`](https://en.wikipedia.org/wiki/Progressive_scan)을 사용해야 합니다
