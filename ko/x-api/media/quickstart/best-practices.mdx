[`POST /2/media/upload`](/ko/x-api/media/media-upload) 엔드포인트를 사용할 때 이해해 두어야 할 몇 가지 중요한 개념이 있습니다. OAuth를 사용해 미디어를 업로드하는 작업은 다소 까다로울 수 있으므로, 이 엔드포인트를 사용할 때 유의해야 할 사항과 이 엔드포인트 사용 방법을 보여 주는 실행 가능한 예제를 여기에서 정리해 두었습니다.

<div id="keep-in-mind">
  ## 기억해야 할 사항
</div>

* 하나의 게시물에 최대 사진 4장, 애니메이션 GIF 1개 또는 동영상 1개를 첨부할 수 있습니다.
* 전달하는 이미지는 이미지의 원시 바이너리 데이터이거나 base64로 인코딩된 바이너리여야 하며, `Content-Type`이 적절하게 설정되어 있는 한 별도의 추가 인코딩이나 이스케이프는 필요 없습니다(확신이 서지 않을 경우: `application/octet-stream`).
* base64로 인코딩된 이미지를 게시할 때는 메시지에서 이미지 파트에 `Content-Transfer-Encoding: base64` 헤더가 설정되어 있는지 확인해야 합니다.
* 멀티파트 메시지의 경계(boundary)는 반드시 단독으로 한 줄에 위치해야 하며, CRLF로 종료되어야 합니다.
* 이 엔드포인트를 사용해 POST 요청을 보내는 실동 예시는 [xurl](https://github.com/xdevplatform/xurl)로 테스트해 볼 것을 권장합니다. 또한 제공되는 [X Libraries](/ko/resources/tools-and-libraries)도 참고하십시오.
* JavaScript 및 긴 정수를 정확히 표현할 수 없는 다른 언어의 경우 API 응답에서 제공되는 `media_id_string`을 사용하십시오.

<div id="media-categories">
  ## 미디어 카테고리
</div>

Media Category 매개변수는 업로드할 미디어 파일의 사용 용도를 정의하며, 미디어 업로드에 적용되는 파일 크기 제한이나 기타 제약 조건에 영향을 줄 수 있습니다. 이후에 미디어를 사용할 때 문제가 발생하지 않도록, 미디어를 업로드할 때 올바른 미디어 카테고리를 사용하는 것이 중요합니다. 이는 업로드 플로우에서 INIT 요청 시 전달하는 선택적 값입니다. 미디어 카테고리가 지정되지 않으면, 콘텐츠 타입에 따라 업로드된 미디어는 게시물용 미디어(`tweet_image`, `tweet_video`, 또는 `tweet_gif`)로 간주됩니다.

가장 일반적으로 사용되는 미디어 카테고리는 다음과 같습니다:

* `tweet_image`
* `tweet_video`
* `tweet_gif`
* `dm_image`
* `dm_video`
* `dm_gif`
* `subtitles`

Ads API 파트너인 경우 프로모션용 동영상에 권장되는 미디어 카테고리에 대한 자세한 내용은 [이 문서](/ko/x-ads-api/creatives#promoted-video)를 참조하세요.

<div id="image-specifications-and-recommendations">
  ## 이미지 규격 및 권장 사항
</div>

이미지 파일은 아래 모든 기준을 충족해야 합니다:

* **지원되는 이미지 미디어 유형**: `JPG`, `PNG`, `GIF`, `WEBP`
* **이미지 크기**: `<= 5 MB`
* **애니메이션 GIF 크기**: `<= 15 MB`

위의 파일 크기 제한은 미디어 업로드 엔드포인트에서 적용됩니다. 또한, `media_id`와 함께 Post 생성(또는 유사한) 엔드포인트를 호출할 때 적용되는, 제품 엔터티별 별도의 파일 크기 제한이 있습니다. 파일 크기 제한과 기타 제약 조건은 `media_category` 매개변수에 따라 달라질 수 있습니다.

<div id="animated-gif-recommendations">
  ## 애니메이션 GIF 권장사항
</div>

파일 크기 제한 이내의 GIF라도 게시물 생성 중에 실패할 수 있습니다. 성공률을 높이려면 다음 기준을 준수하세요.

* **해상도**: `<= 1280x1080` (`width` x `height`)
* **프레임 수**: `<= 350`
* **픽셀 수**: `<= 300 million` (`width` * `height` * `num_frames`)
* **파일 크기**: `<= 15Mb`

더 큰 GIF를 처리하려면 `media_category` 파라미터와 함께 [chunked upload](/ko/x-api/media/quickstart/media-upload-chunked) 엔드포인트를 사용하세요. 이렇게 하면 서버가 GIF 파일을 비동기적으로 처리할 수 있으며, 이는 더 큰 파일을 처리하기 위한 필수 조건입니다. 애니메이션 GIF가 포함된 게시물에 대해 비동기 업로드 동작을 활성화하려면 `media_category=tweet_gif`를 설정하세요.

<div id="video-specifications-and-recommendations">
  ## 비디오 사양 및 권장사항
</div>

미디어 업로드에는 Async Path를 사용하십시오.

<div id="recommended">
  ### 권장 사양
</div>

* **비디오 코덱**: `H264 High Profile`
* **프레임 레이트**: `30 FPS`, `60 FPS`
* **비디오 해상도**: `1280x720` (가로), `720x1280` (세로), `720x720` (정사각형). 구독 중인 사용자는 1080p 비디오를 업로드하고 1080p로 재생할 수 있습니다. 구독 중이 아닌 사용자는 720p 비디오를 업로드하고 720p로 재생할 수 있습니다.
* **최소 비디오 비트레이트**: `5,000 kbps`
* **최소 오디오 비트레이트**: `128 kbps`
* **오디오 코덱**: `AAC LC`
* **가로세로 비율**: `16:9` (가로 또는 세로), `1:1` (정사각형)

<div id="advanced">
  ### 고급
</div>

* **프레임 레이트**: `60 FPS` 이하여야 합니다
* **치수**: `32x32` 이상 `1280x1024` 이하 범위여야 합니다
* **파일 크기**: `512 mb`를 초과해서는 안 됩니다
* **재생 시간**: `0.5초` 이상 `140초` 이하여야 합니다
* **종횡비**: `1:3`에서 `3:1` 사이여야 합니다
* **[픽셀 종횡비](https://en.wikipedia.org/wiki/Pixel_aspect_ratio)**: `1:1`이어야 합니다
* **픽셀 포맷**: [YUV](https://en.wikipedia.org/wiki/YUV) 4:2:0만 지원됩니다
* 오디오는 [`AAC` Low Complexity 프로파일](https://en.wikipedia.org/wiki/Advanced_Audio_Coding#Modular_encoding)을 사용해야 합니다. (High-Efficiency `AAC`는 지원되지 않습니다.)
* 오디오는 `mono` 또는 `stereo`여야 하며, 5.1 이상은 허용되지 않습니다
* [`open GOP`](https://en.wikipedia.org/wiki/Group_of_pictures)를 사용해서는 안 됩니다
* [`progressive scan`](https://en.wikipedia.org/wiki/Progressive_scan)을 사용해야 합니다

<div id="additional-information">
  ### 추가 정보
</div>

아래 표에서 각 행은 업로드 권장 사항을 나타내지만, 필수 조건은 아닙니다. 모든 업로드는 여러 플랫폼에서 최적화되도록 처리됩니다.

| Orientation | Width | Height | Video Bitrate | Audio Bitrate |
| :--- | :--- | :--- | :--- | :--- |
| Landscape | 1280 | 720 | 2048K | 128K |
| Landscape | 640 | 360 | 768K | 64K |
| Landscape | 320 | 180 | 256K | 64K |
| Portrait | 720 | 1280 | 2048K | 128K |
| Portrait | 360 | 640 | 768K | 64K |
| Portrait | 180 | 320 | 256K | 64K |
| Square | 720 | 720 | 2048K | 128K |
| Square | 480 | 480 | 768K | 64K |
| Square | 240 | 240 | 256K | 32K |

미디어를 업로드하는 방법의 예시는 [청크 단위 미디어 업로드 문서](/ko/x-api/media/quickstart/media-upload-chunked)를 참고하세요.

<div id="troubleshooting">
  ### 문제 해결
</div>

Media API 관련 문제가 발생한 경우, 개발자 포럼의 [Media API 카테고리](https://devcommunity.x.com/c/x-api/media-apis)를 살펴보고 해결 방법을 확인하세요.
