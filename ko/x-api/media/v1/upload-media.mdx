<div id="overview">
  ## 개요
</div>

미디어 객체는 하나의 사진, 비디오 또는 애니메이티드 GIF를 나타냅니다. 미디어 객체는 Twitter API 내의 많은 엔드포인트에서 사용되며, 트윗, 다이렉트 메시지, 사용자 프로필, 광고 크리에이티브 등 다양한 곳에 포함될 수 있습니다. 각 미디어 객체는 해상도나 형식이 서로 다른 여러 개의 표시 또는 재생용 버전을 가질 수 있습니다.

<div id="media-types-size-restrictions">
  ## 미디어 유형 및 크기 제한
</div>

API를 통한 업로드 시 크기 제한 

- 이미지 최대 5 MB
- GIF 최대 15 MB
- 비디오 최대 512 MB (media_category=amplify 사용 시)

<div id="creation">
  ## 생성
</div>

트윗, 다이렉트 메시지, 사용자 프로필 사진, 호스팅된 Ads 카드 등과 같은 객체에는 하나 이상의 미디어 객체가 포함될 수 있습니다. 이러한 최상위 객체를 통칭하여 엔티티라고 합니다. 관련 엔티티 생성 API(예: POST statuses/update)는 고유한 media_id를 사용해 하나 이상의 미디어 객체를 전달할 수 있습니다.

미디어 객체를 포함하는 엔티티는 다음 단계를 따라 생성할 수 있습니다.

1. 권장되는 청크 업로드(이미지/GIF/비디오) 또는 기존의 단순 업로드 방식(이미지만)을 사용하여 미디어 파일을 업로드합니다.
2. 1단계에서 media_id를 받습니다. 엔티티가 여러 개의 media_id 파라미터를 전달할 수 있도록 되어 있는 경우, 이 단계를 서로 다른 미디어에 대해 여러 번 반복할 수 있습니다.
3. media_id 및 기타 필수 파라미터를 포함하여 적절한 endpoint를 호출해 엔티티를 생성합니다. 예를 들어, POST statuses/update endpoint를 사용해 트윗에 media_id를 첨부할 수 있습니다.
