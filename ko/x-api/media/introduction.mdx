미디어 객체는 사진, 동영상 또는 애니메이션 GIF 한 개를 나타냅니다. 미디어 객체는 X API 내의 여러 엔드포인트에서 사용되며, 게시물, 다이렉트 메시지, 사용자 프로필, 광고 크리에이티브 등을 비롯한 다양한 위치에 포함될 수 있습니다. 각 미디어 객체는 해상도나 형식이 서로 다른 여러 표시 또는 재생용 버전을 가질 수 있습니다.

<div id="media-types-size-restrictions">
  ## 미디어 유형 및 크기 제한
</div>

API를 통한 업로드 시 크기 제한

* **Image**: `5 MB`
* **GIF**: `15 MB`
* **Video**: `512 MB` (`media_category=amplify_video`를 사용하는 경우)

<div id="creation">
  ## 생성
</div>

게시물, 다이렉트 메시지, 사용자 프로필 사진, 호스팅된 Ads 카드 등의 객체는 하나 이상의 미디어 객체를 포함할 수 있습니다. 이러한 상위 수준 객체를 통틀어 엔터티라고 합니다. 관련 엔터티 생성 API(예: [`POST /2/tweets`](/ko/x-api/posts/creation-of-a-post))에는 고유한 `media_id`를 사용해 하나 이상의 미디어 객체를 전달할 수 있습니다.

미디어 객체를 포함하는 엔터티는 다음 단계를 따라 생성할 수 있습니다.

1. 권장되는 [chunked](/ko/x-api-/media/quickstart/media-upload-chunked) 업로드(이미지/GIF/동영상) 또는 이전 방식인 [simple](/ko/x-api/media-upload/media-upload) 업로드(이미지 전용)를 사용해 미디어 파일을 업로드합니다.
2. 1단계에서 `media_id`를 받습니다. 엔터티가 여러 `media_id` 파라미터 전달을 허용하는 경우, 서로 다른 미디어에 대해 이 단계를 여러 번 반복할 수 있습니다.
3. 적절한 엔드포인트를 호출하여 `media_id` 및 기타 필수 파라미터를 포함해 엔터티를 생성합니다. 예를 들어, [`POST /2/tweets`](/ko/x-api/posts/creation-of-a-post) 엔드포인트를 사용해 게시물에 `media_id`를 첨부할 수 있습니다.
