미디어 객체는 하나의 사진, 비디오 또는 애니메이션 GIF를 나타냅니다. 미디어 객체는 X API의 여러 엔드포인트에서 사용되며, 게시물, 다이렉트 메시지, 사용자 프로필, 광고 크리에이티브 등 다양한 위치에 포함될 수 있습니다. 각 미디어 객체는 해상도나 형식이 서로 다른 여러 표시/재생용 버전을 가질 수 있습니다.

<div id="media-types-size-restrictions">
  ## 미디어 유형 및 크기 제한
</div>

API 업로드 시 크기 제한

* **Image**: `5 MB`
* **GIF**: `15 MB`
* **Video**: `512 MB` (`media_category=amplify_video`를 사용할 때)

<div id="creation">
  ## 생성
</div>

게시물, 다이렉트 메시지, 사용자 프로필 사진, 호스티드 Ads 카드 등의 객체에는 하나 이상의 미디어 객체가 포함될 수 있습니다. 이러한 최상위 객체를 통칭해 엔티티라고 합니다. 관련 엔티티 생성 API(예: [`POST /2/tweets`](/ko/x-api/posts/creation-of-a-post))에는 고유한 `media_id`를 사용해 하나 이상의 미디어 객체를 전달할 수 있습니다.

미디어 객체를 포함하는 엔티티는 다음 단계를 따라 생성할 수 있습니다:

1. 권장되는 [chunked](/ko/x-api-/media/quickstart/media-upload-chunked) 업로드(이미지/GIF/동영상) 또는 기존 [simple](/ko/x-api/media-upload/media-upload) 업로드(이미지만)를 사용해 미디어 파일을 업로드합니다.
2. 1단계에서 `media_id`를 받습니다. 엔티티에서 여러 개의 `media_id` 파라미터 전달을 허용하는 경우, 이 단계는 서로 다른 미디어로 여러 번 반복할 수 있습니다.
3. 적절한 엔드포인트를 호출할 때 `media_id` 및 기타 필수 파라미터를 포함해 엔티티를 생성합니다. 예를 들어, [`POST /2/tweets`](/ko/x-api/posts/creation-of-a-post) 엔드포인트를 사용해 게시물에 `media_id`를 첨부할 수 있습니다.
