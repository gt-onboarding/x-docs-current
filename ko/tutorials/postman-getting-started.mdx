<div id="introduction">
  ## 소개
</div>

Postman은 그래픽 사용자 인터페이스를 통해 API에 요청을 보낼 수 있는 데스크톱 및 웹 애플리케이션입니다. X API, X Ads API, Labs 엔드포인트의 기능을 살펴볼 때뿐만 아니라 애플리케이션에서 발생하는 문제를 해결할 때에도 Postman 사용을 권장합니다. 

현재 다음 두 가지 Postman 컬렉션을 제공하고 있습니다.

<div id="prerequisites">
  ### 사전 준비 사항
</div>

X의 Postman 컬렉션을 사용하기 전에 사용하려는 X 개발자 플랫폼 도구에 대해 적절한 액세스 권한과 자격 증명이 있는지 확인해야 합니다. 액세스 권한을 얻는 방법은 [“시작하기” 페이지](/ko/resources/platform-overview)에서 자세히 확인할 수 있습니다. 

[개발자 계정](https://developer.x.com/en/portal/petition/essential/basic-info)을 보유하고, [개발자 앱](/ko/resources/fundamentals/developer-apps)을 설정했으며, [인증](/ko/resources/fundamentals/authentication) 키와 토큰을 발급받고, 사용하려는 특정 API에 요청을 보낼 수 있도록 환경을 올바르게 구성했다면, 아래 단계에 따라 시작할 수 있습니다.
 

<div id="getting-started-with-xs-postman-collections">
  ## X의 Postman 컬렉션 시작하기
</div>

<div id="step-one-add-one-of-the-x-postman-collections-to-your-account">
  ### 1단계: X Postman 컬렉션 중 하나를 계정에 추가하기
</div>

Postman에서 사용하려는 엔드포인트를 직접 하나씩 구성할 수도 있지만, 이 수고로운 작업을 대신해 관련 API들로 구성된, 바로 사용할 수 있는 컬렉션을 미리 준비해 두었습니다. 앞의 “Postman collections” 섹션에 있는 링크 중 하나를 클릭하면, 선택한 API에 연결된 모든 엔드포인트가 포함된 컬렉션이 Postman 앱에 추가됩니다. 이 컬렉션들은 [Postman API network](https://explore.postman.com/)에서도 사용할 수 있습니다. 각 엔드포인트에는 사용 가능한 매개변수, 예시 응답, 그리고 인증 유형이 이미 설정되어 있으므로, 자격 증명과 매개변수 값만 추가하면 곧바로 기능을 탐색해 볼 수 있습니다.

이 예제에서는 X [API v2 collection](https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400)을 사용해 보겠습니다. 
 

<div id="step-two-add-your-keys-and-tokens-as-environmental-variables">
  ### Step two: 환경 변수로 키와 토큰 추가하기
</div>

컬렉션을 Postman 인스턴스에 추가하면 자동으로 ‘X API v2’라는 환경이 생성되며, 여기에 키와 토큰을 추가해야 합니다. 이 단계에서는 개발자 앱에서 생성한 키와 토큰을 ‘X API v2’ 환경에 추가하는 방법을 안내합니다. 

“X API v2” 환경에 키와 토큰을 추가하려면 Postman 우측 상단에 있는 “manage environments” 버튼을 클릭하세요. 필요한 경우, 우측 상단의 설정 버튼을 먼저 클릭해야 할 수도 있습니다. 
 

<Frame>
  <img src="/images/using-postman-1.png.twimg.1920.png" alt="이 이미지는 Postman 콘솔에서 &#x22;manage environments&#x22; 버튼이 강조 표시된 모습을 보여줍니다." />
</Frame>

환경 목록에서 “X API v2”를 클릭합니다. 

다음으로, Apps 대시보드에서 생성한 각 키와 토큰에 대해 변수를 추가하여 아래 예시와 같이 표에 입력합니다.

| VARIABLE | INITIAL VALUE | CURRENT VALUE |
| :--- | :--- | :--- |
| consumer&#95;key | `QAktM6W6DF6F7XXXXXX` | `QAktM6W6DF6F7XXXXXX` |
| consumer&#95;secret | `AJX560A2Omgwyjr6Mml2esedujnZLHXXXXXX` | `AJX560A2Omgwyjr6Mml2esedujnZLHXXXXXX` |
| access&#95;token | `1995XXXXX-0NGqVhk3s96IX6SgT3H2bbjOPjcyQXXXXXXX` | `1995XXXXX-0NGqVhk3s96IX6SgT3H2bbjOPjcyQXXXXXXX` |
| token&#95;secret | `rHVuh7dgDuJCOGeoe4tndtjKwWiDjBZHLaZXXXXXX` | `rHVuh7dgDuJCOGeoe4tndtjKwWiDjBZHLaZXXXXXX` |
| bearer&#95;token | `AAAAAAAAAAAAAAAAAAAAAL9v6AAAAAAA99t03huuqRYg0mpYAAFRbPR3XXXXXXX` | `AAAAAAAAAAAAAAAAAAAAAL9v6AAAAAAA99t03huuqRYg0mpYAAFRbPR3XXXXXXX` |

위 표에 사용된 키와 토큰은 예시용으로 임의로 만든 값이므로, 실제 요청에 사용해도 동작하지 않습니다. 

자격 증명을 변수로 추가하고 X API v2 환경이 선택되어 있는 것을 확인했다면, 이제 X API v2 컬렉션에 요청을 보낼 준비가 된 것입니다. 이는 각 엔드포인트의 Authorization 탭이 이 환경에서 정의한 변수를 자동으로 사용하도록 설정되어 있기 때문입니다. 

Postman을 사용자 액세스 토큰과 함께 사용하는 방법을 확인하려면 XXXX로 이동하여 추가 세부 정보를 참고하세요.

<div id="step-four-add-values-to-the-params-tab">
  #### 4단계: Params 탭에 값 추가하기
</div>

다음 단계에서는 Params 탭으로 이동합니다. 여기에서 비활성화된 파라미터들과 각 파라미터의 역할을 설명하는 텍스트, 그리고 요청에 함께 전달할 수 있는 모든 가능한 값 목록을 확인할 수 있습니다. 

이 예제에서는 `expansions`와 `tweet.fields` 쿼리 파라미터를 활성화하고 다음 값을 추가합니다:

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `tweet.fields` | `created_at,attachments` |
| expansions | author&#95;id |

쿼리 파라미터를 추가하는 것 외에도, 필수 Path Variable인 `id`를 추가해야 합니다. 이 엔드포인트는 게시물을 반환하므로, 값으로 유효한 게시물 ID를 추가해야 합니다.

게시물 ID는 x.com으로 이동해 게시물을 클릭한 뒤, URL에서 확인할 수 있습니다. 예를 들어, 다음 URL의 게시물 ID는 `1228393702244134912`입니다:

`https://x.com/XDevelopers/status/1228393702244134912`

Params 탭에서 모든 쿼리 파라미터 영역을 지나 아래로 스크롤하면 “Path Variables” 섹션이 표시됩니다. 여기에서 사용하려는 게시물 ID를 `id` 키의 값으로 추가합니다.

이 단계의 내용을 모두 정확히 입력했다면, Params 탭은 다음과 같이 보일 것입니다:

<div id="step-five-send-your-request-and-review-your-response">
  #### 5단계: 요청 보내기 및 응답 확인하기
</div>

이제 요청에 필요한 모든 설정을 마쳤으므로 “Send” 버튼을 클릭합니다. 

모든 설정이 올바르게 구성되었다면, 다음과 같은 페이로드를 받게 됩니다:

```json
{
    "data": {
        "author_id": "2244994945",
        "text": "What did the developer write in their Valentine’s card?\n  \nwhile(true) {\n    I = Love(You);  \n}",
        "id": "1228393702244134912",
        "created_at": "2020-02-14T19:00:55.000Z"
    },
    "includes": {
        "users": [
            {
                "username": "XDevelopers",
                "name": "개발자",
                "id": "2244994945"
            }
        ]
    }
}
```

<div id="generating-a-user-access-token-with-postman">
  ### Postman을 사용하여 사용자 액세스 토큰 생성하기:
</div>

<div id="using-oauth-10a-to-generate-a-user-access-token">
  #### OAuth 1.0a를 사용하여 사용자 액세스 토큰 생성하기
</div>

[OAuth1.0a flow test collection](https://www.postman.com/xapidevelopers/workspace/twitter-s-public-workspace/collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400?ctx=documentation)에서 사용하는 세 단계 절차를 살펴보세요.

<div id="using-oauth-20-to-generate-a-user-access-token">
  #### OAuth 2.0를 사용해 사용자 액세스 토큰 생성하기
</div>

Postman에서 OAuth 2.0 사용자 액세스 토큰을 생성하려는 경우, X [API v2 Postman collection](https://www.postman.com/xapidevelopers/workspace/twitter-s-public-workspace/collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400?ctx=documentation)과 함께 사용할 OAuth 2.0 액세스 토큰을 생성할 수 있습니다. 

워크스페이스에서 컬렉션을 클릭한 다음 “Auth” 탭으로 이동해 유형을 “OAuth 2.0”으로 선택합니다. 그런 다음 “Configure New Token”이라는 제목 아래의 “Configuration Options”로 이동합니다. 여기에서 “Grant Type”을 “Authorization Code (With PKCE)”로 변경할 수 있습니다.

Callback URL을 애플리케이션에 연결된 callback URL과 일치하도록 변경해야 합니다. 또한 다음 매개변수를 업데이트해야 합니다:

* Auth URL -  https://x.com/i/oauth2/authorize
* Access Token URL -  https://api.x.com/2/oauth2/token
* Client ID - Dev Portal에서 발급받은 OAuth 2.0 client ID
* Client Secret - 기밀 클라이언트를 사용하는 경우
* Scope - 연결하려는 엔드포인트에 맞는 스코프. 예: “tweet.read users.read” 
* Callback URL (redirect URL이라고도 함). 앱의 인증 설정에 구성된 값과 반드시 일치해야 합니다.
* State - state

준비가 되면 “Get New Access Token”을 클릭해 액세스 토큰을 생성할 수 있습니다. 문제가 발생했다는 내용의 대화 상자가 표시되면, 로그인하기 위해 뒤로 가기 버튼을 눌러야 할 수 있습니다. 대화 상자에서 “Authorize app”이라고 표시된 곳을 클릭해, 앱이 계정에 액세스할 수 있도록 권한을 부여해야 합니다.

앱을 승인한 후에는 Postman으로 다시 리디렉션되며, 여기에서 토큰을 확인하고 “Use Token” 버튼을 선택해 인증된 사용자를 대신해 요청을 보내기 시작할 수 있습니다.

이제 Postman 컬렉션을 사용할 준비가 되었습니다.
