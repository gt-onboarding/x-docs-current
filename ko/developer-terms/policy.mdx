<div id="x-developers">
  ## X + 개발자
</div>

X는 개발자를 소중히 생각합니다. 이 커뮤니티가 X 데이터의 강력함을 활용해 만들어 내는 다양한 도구와 서비스에 늘 기쁘고 놀라고 있습니다. 이 커뮤니티에 대한 우리의 약속의 일환으로, 우리는 개발자에게는 개방적이고 공정한 데이터 접근을 제공하고, X 이용자에게는 안전하며, X 플랫폼 전체에도 이로운 방식의 제공을 목표로 합니다. 이러한 목표를 달성하기 위해, 우리는 API 및 X 콘텐츠의 적절한 사용에 관한 규칙과 기대치를 이해하는 데 도움이 되는 안내서로서 Developer Policy를 마련했습니다.

이 Developer Policy(“**Policy**”)는 X의 애플리케이션, 서비스, 웹사이트, 웹 페이지 및 콘텐츠 생태계와 상호작용하는 개발자를 위한 규칙과 가이드라인을 제공합니다. 이는 X API 및 X 콘텐츠에 대한 접근 및 사용을 규율하는 X와의 계약(Developer Agreement 또는 X와 체결한 기타 서면 계약)의 일부를 이룹니다. Policy를 위반하는 행위는 귀하의 계약 위반으로 간주됩니다. 이 Policy는 별도의 통지 없이 수시로 변경될 수 있습니다. 이 Policy에서 사용되지만 정의되지 않은 대문자 용어는 [Developer Agreement](/ko/developer-terms/agreement) 또는 Master License Agreement에서 부여된 의미를 갖습니다.

<div id="using-this-policy">
  ## 이 정책 활용 방법
</div>

이 정책은 최대한 이해하고 준수하기 쉽도록 구성되어 있습니다. X API 및 X 콘텐츠를 사용할 때 아래 정책 섹션의 내용을 항상 염두에 두시기 바랍니다:

1. **성공을 위한 준비** - 모든 X 정책을 준수할 책임은 여러분에게 있습니다. X API 및 X 콘텐츠에 접근하기 전에, 이 정책과 이 문서에서 링크된 정책들을 검토하고 이해하는 것이 중요합니다. 정책을 검토하는 데 쓰는 시간이 나중에 수시간에 달하는 재작업을 줄여 줄 수 있습니다.

2. **프라이버시와 통제권은 필수입니다** - X에서 사람들의 프라이버시를 보호하고 수호하는 것은 회사의 핵심 DNA에 깊이 뿌리내리고 있습니다. 따라서 사람들의 합리적인 프라이버시 기대에 반하는 방식으로 X 데이터가 사용되는 행위는 금지됩니다. 여러분이 X API를 기반으로 빌드하거나 X 콘텐츠에 접근하는 순간부터, 이러한 약속을 지키는 데 있어 특별한 역할을 맡게 되며, 특히 사람들의 프라이버시를 존중하고, 그들의 데이터가 어떻게 사용되는지에 대해 투명성과 통제권을 제공해야 합니다.

3. **플랫폼 사용 가이드라인을 준수하세요** - X API 및 X 콘텐츠에 대한 접근 승인을 받는 것은 첫 단계에 불과합니다. X 플랫폼을 어떻게 사용해야 정책을 준수할 수 있는지에 대해 궁금한 점이 있을 때마다, 플랫폼 사용 가이드라인을 가장 먼저 참고해야 합니다.

아래에서 이 세 가지 핵심 섹션 각각이 의미하는 바를 더 자세히 설명합니다. X API 및 X 콘텐츠 사용이 우리의 정책과 일치하는지 확인할 수 있도록, 반드시 주의 깊게 검토해 주시기 바랍니다.

당사가 여러분이 이 정책(또는 다른 X 정책)을 위반하고 있다고 판단하는 경우, X API 및 X 콘텐츠에 대한 여러분의 접근을 일시 중단하거나 영구적으로 철회할 수 있습니다.

마지막으로, X는 X 애플리케이션을 개선하고 상업적 사용을 검토하며, 승인된 사용 사례 및 본 정책 준수 여부를 확인하기 위해 여러분의 X API 사용을 모니터링할 수 있다는 점에 유의해 주시기 바랍니다.

읽어 주셔서 감사합니다. 그리고 우리와 함께 서비스를 구축해 주셔서 감사합니다! 여러분이 무엇을 만들어낼지 기대하고 있습니다!

<div id="set-yourself-up-for-success">
  ## 성공을 위한 기반을 마련하세요
</div>

X API를 사용할 때 처음부터 올바른 방식으로 서비스를 구축하면, 많은 잠재적인 문제를 피할 수 있습니다. 본 Developer Policy의 이 섹션에는 X API 또는 X Content를 사용하기 전에 모든 개발자가 반드시 따라야 하는 규칙이 포함되어 있습니다.

Free, Basic, Pro 요금제([developer.x.com/en](http://developer.x.com/en)에 설명되어 있음)는 취미 개발, 상업적 프로토타이핑, 초기 단계 X 제품 연동, 그리고 최종 사용자 수가 제한적인 애플리케이션을 지원하기 위해 설계되었습니다. 이 범위를 넘어 X API 및 X Content를 사용하는 경우, Enterprise 요금제([developer.x.com/en](http://developer.x.com/en)에 설명되어 있음)를 신청하거나 이미 구독하고 있어야 합니다.

**당사는 X 개발자 플랫폼의 모든 예정된 사용 방식이 정책을 준수하는지 검토합니다. 따라서 X API 및 X Content의 예정된 사용 방식(및 해당되는 경우 변경 사항)을 공개해야 하며, 이를 통해 접근 권한을 부여받고 유지하게 됩니다.** 모든 신규 개발자는 X API에 접근하기 위해 [개발자 계정을 신청](https://developer.x.com/portal/petition/essential/basic-info)해야 합니다. 승인된 개발자 계정이 없는 기존 개발자는 X의 안내에 따라 계정을 신청해야 합니다. 이 과정의 일부로, X API 및 X Content를 어떻게 사용할 계획인지에 대한 서면 설명을 제공해야 합니다.

사용 사례 설명은 귀하에게 법적 구속력을 가지며, 이에서 실질적으로 벗어나는 행위는 당사 규칙 위반이 될 수 있고, 그에 따른 조치가 이루어질 수 있습니다. 사용 사례에 실질적인 변경이 있는 경우 반드시 당사에 통지하고, 해당 목적을 위해 X Content를 사용하기 전에 승인을 받아야 합니다. 이를 이행하지 않으면 API 및 데이터 접근이 정지되거나 종료될 수 있습니다.

**X API를 기반으로 구축하거나 X Content에 접근하는 경우, 모든 X 정책을 준수해야 합니다.** 여기에는 본 [Developer Policy](/ko/developer-terms/policy), [Automation Rules](https://help.x.com/rules-and-policies/x-automation), [Display Requirements](https://developer.x.com/developer-terms/display-requirements.html), [API Restricted Uses Rules](/ko/developer-terms/restricted-use-cases), [X Rules](https://help.x.com/rules-and-policies/x-rules), [X Brand Resources](https://about.x.com/who-we-are/brand-toolkit)뿐만 아니라, X API 또는 X Content 사용과 관련하여 X와 체결하는 모든 기타 계약(이에 한정되지 않고 [Developer Agreement](/ko/developer-terms/agreement) 또는 Master Licensing Agreement 또는 Order(해당되는 경우)를 포함)이 포함됩니다. 또한 이러한 정책에 대한 모든 수정 사항과 X에서 새로 도입하는 정책도 준수해야 합니다. 귀하의 서비스 사용을 모니터링하고, 귀하의 서비스를 설계하여 이용자가 X 정책을 위반하지 않도록 하는 것은 전적으로 귀하의 책임입니다. 이를 이행하지 않으면 API 및 X Content 접근의 정지 또는 종료로 이어질 수 있습니다.

**단일 사용 사례 또는 실질적으로 유사하거나 중복되는 사용 사례에 대해 여러 개의 애플리케이션을 등록할 수 없습니다.** 여기서 “사용 사례(use case)”란 애플리케이션을 통해 수행되는 분석, 표시 또는 작업의 일관된 집합을 의미합니다. “화이트 라벨(white label)” 형태의 도구나 서비스 버전은 허용되지 않습니다. 애플리케이션의 “화이트 라벨” 버전을 계획하고 있는 경우, 반드시 당사에 통지하고 별도의 승인을 받아야 합니다.

*이 규칙에 대한 단 하나의 예외로,* 동일한 서비스의 development, staging, production 인스턴스를 위해 최대 3개의 애플리케이션을 생성하고 사용할 수 있습니다. 이러한 앱은 단일 계정에 등록되어야 하며, 이름과 설명에서 단일 서비스의 dev, staging, prod 인스턴스임이 명확히 식별되어야 합니다. **development 또는 staging 애플리케이션을 운영(프로덕션) 용도로 사용할 수 없습니다.**

**모든 API 키 및 기타 접근 자격 증명은 비공개로 유지해야 합니다.** 귀하는 다른 사람이 소유한 API 키 또는 기타 접근 자격 증명을 사용할 수 없으며, 다른 사람이 이를 사용하도록 조장하거나 이를 용이하게 해서도 안 됩니다.

**X와의 라이선스 계약은 X API 및 X Content 사용 범위를 제한합니다. 특히, X API에는 공정한 데이터 사용을 보장하고 플랫폼의 스팸을 방지하기 위한 요청 한도가 있습니다.** 귀하는 본 정책 또는 X와의 계약에 명시되어 있거나, [Developer Site](https://developer.x.com/)에 게시되어 있거나, X로부터 통지받은 요청 한도 및 기타 모든 제한이나 제약을 초과하거나 이를 우회할 수 없습니다.

**X API를 통해 수신한 X Content에 포함된 독점적 고지나 표식을 제거하거나 변경할 수 없습니다.** 이는 사람들이 X Content가 어디에서 왔는지, 그리고 누구의 소유인지 알 수 있도록 하기 위한 것입니다.

**데이터 무결성과 플랫폼의 안정성 및 건전성을 위해, X API 또는 X 서비스의 어떤 기능도 방해, 가로채기, 중단 또는 비활성화해서는 안 됩니다.** 다시 말해, [developer.x.com](https://developer.x.com/)에 문서화된 대로 의도된 방식으로 API를 사용해야 합니다. 허용되는 사용 방식에 대한 자세한 내용은 [HackerOne guidelines](https://hackerone.com/x)를 참고하세요.

<div id="privacy-and-control-are-essential">
  ## 프라이버시와 통제는 필수적입니다
</div>

X는 프라이버시를 매우 중요하게 여기며, X 콘텐츠와 X API를 사용하는 모든 분들도 동일한 기준을 지켜 주시기를 기대합니다. X 개발자 플랫폼, X API 또는 X 콘텐츠를 사람들의 합리적인 프라이버시에 대한 기대와 부합하지 않는 방식으로 사용하는 경우, API 및 X 콘텐츠 접근의 일시 중지 또는 종료를 포함한 제재를 받을 수 있습니다.

프라이버시와 통제에 대한 여러분의 책임은 X 콘텐츠의 모든 사용과, 당사의 API를 사용해 여러분이 구축하는 서비스의 모든 측면에까지 확장되어야 합니다. 이를 위해, 여러분의 서비스를 사용하는 사람들이 여러분이 그들의 데이터를 어떻게 사용하는지, 그리고 그들을 대신해 X에 어떻게 접근하는지 이해하고 이에 동의해야 합니다. 이는 사람들에게 명확하고 포괄적이며 투명한 개인정보 처리방침을 제공하고, 그들을 대신해 어떤 조치를 수행하기 전에 서비스 사용자 각각으로부터 명시적이고 충분한 설명에 근거한 동의를 받음으로써 달성할 수 있습니다. **여러분의 서비스에 사용자가 인증을 거쳤다는 사실만으로는 동의가 성립되지 않는다는 점에 유의하십시오.**

<div id="consent-permissions">
  ## 동의 및 권한
</div>

**특히, 아래의 작업 중 어느 하나라도 수행하기 전에 사람들로부터 명시적이고 충분한 설명에 따른 동의를 받아야 합니다.**

* 그들을 대신해 *어떠한* 행동이라도 수행하는 것. 여기에는 다음이 포함되지만 이에 한정되지 않습니다.
  * X에 콘텐츠 게시
  * 계정 팔로우/언팔로우
  * 프로필 또는 계정 정보 수정
  * 게시물에 해시태그 또는 기타 콘텐츠 추가

* X API 또는 기타 X 도구 이외의 수단을 통해 접근한 콘텐츠를 재게시하는 것

* 타인의 X 콘텐츠를 상품 또는 서비스 홍보에 사용하는 것

* 다이렉트 메시지(DM)와 같은 비공개 콘텐츠 또는 기타 개인적이거나 기밀 정보를 저장하는 것

* 보호된 콘텐츠 또는 기타 개인적이거나 기밀 정보를 공유하거나 게시하는 것

**서비스에서 사람들이 X에 콘텐츠를 게시할 수 있게 하는 경우, 게시 전에 다음을 수행해야 합니다.**

* 어떤 내용이 게시될지 정확히 보여주기
* 귀하의 서비스를 사용하는 사람들에게 콘텐츠에 어떤 위치 정보(있는 경우)가 추가될지 명확히 알리기

**서비스에서 사람들이 귀하의 서비스와 X 모두에 콘텐츠를 게시할 수 있게 하는 경우, 게시 전에 다음을 수행해야 합니다.**

* 해당 콘텐츠를 게시할 권한을 얻기
* 어디에 콘텐츠를 게시할 것인지 설명하기

**모든 X 콘텐츠의 보호 및 차단 상태를 존중해야 합니다. 한 사람의 인증 토큰을 사용해 획득한 콘텐츠를, 그 콘텐츠를 볼 권한이 없는 다른 사람에게 제공할 수 없습니다.**

* **보호된 계정:** [보호된 계정](https://help.x.com/safety-and-security/public-and-protected-posts)의 콘텐츠는 계정 소유자가 팔로우를 승인한 사람에게만 제공됩니다. 따라서 보호된 계정에 접근하는 서비스를 운영하는 경우, 해당 콘텐츠를 볼 권한이 있는 특정 사람들에게만 그러한 콘텐츠를 제공하는 목적으로 접근해야 합니다.
* **차단된 계정:** X에서 사람들은 원하는 이유로 자신의 계정 접근을 [차단](https://help.x.com/using-x/blocking-and-unblocking-accounts)할 수 있습니다. 이 선택을 우회하기 위해 토큰(또는 기타 API 기반 동작)으로 얻은 정보를 결합하거나 뒤섞어 사용하는 것은 허용되지 않습니다.

**다이렉트 메시지(DM)는 본질적으로 비공개이므로, DM 기능을 제공하는 서비스는 개인 정보 보호를 위해 추가적인 조치를 취해야 합니다. DM 콘텐츠를 볼 권한이 없는 사람에게 DM 콘텐츠를 제공할 수 없습니다. 서비스에서 DM 기능을 제공하는 경우 다음도 수행해야 합니다.**

* DM에 대해 읽음 확인 이벤트를 전송하는 경우, 사람들에게 이를 알립니다. 이는 서비스 내에서 직접 고지를 제공하거나, 대화 중 다른 참여자의 읽음 확인을 표시하는 방식으로 할 수 있습니다.
* DM에서 전송되는 미디어를 &quot;공유됨&quot;(즉, 여러 DM에서 재사용 가능)으로 설정하기 전에 동의를 얻습니다. DM의 미디어를 “공유됨”으로 허용하는 경우, 이 콘텐츠가 해당 미디어의 URL을 가진 누구에게나 접근 가능하다는 점을 명확하게 고지해야 합니다.

<div id="content-compliance">
  ## 콘텐츠 규정 준수
</div>

**X 콘텐츠를 오프라인으로 저장하는 경우, 해당 콘텐츠를 X에서의 현재 상태와 일치하도록 최신 상태로 유지해야 합니다.** 구체적으로, X에서 콘텐츠가 삭제되거나 수정된 경우, 귀하가 보유한 해당 콘텐츠도 삭제하거나 수정해야 합니다. 이는 합리적으로 가능한 한 신속하게, 또는 X나 해당 X 계정 소유자가 이를 요청한 시점으로부터 24시간 이내에, 혹은 X와의 계약이나 관련 법에서 달리 요구하는 바에 따라 이행되어야 합니다. 관련 법률에서 이를 금지하는 경우를 제외하고는 이 의무를 이행해야 하며, 그러한 경우에도 X의 명시적인 서면 허가가 있는 경우에만 예외가 인정됩니다.

수정된 콘텐츠는 다양한 형태를 취할 수 있습니다. 여기에는 다음이 포함되지만 이에 한정되지는 않습니다:

* 비공개로 전환되었거나 보호 상태가 된 콘텐츠
* 플랫폼에서 정지된 콘텐츠
* 지오태그가 제거된 콘텐츠
* X에서 보류되었거나 제거된 콘텐츠

<div id="off-x-matching">
  ## X 외부 매칭
</div>

**당사는 여러분이 X 상의 개인을 X 외부에서 획득하거나 보관 중인 정보와 매칭할 수 있는 상황을 제한합니다.** X 외부 매칭이란 X 콘텐츠(예: X @handle 또는 user ID)를 개인, 가구, 기기, 브라우저 또는 기타 X 외부 식별자와 연관시키는 것을 말합니다. 여러분은 이러한 연관을 시도하기 전에 그 개인으로부터 명시적 옵트인 동의를 받았거나, 아래에 설명된 경우에만 이를 수행할 수 있습니다.

개인의 X 상의 신원(X identity)을 X 외부 식별자와 연결하기 위한 그 개인의 명시적 옵트인 동의가 없는 상황에서는, 여러분이 형성하는 모든 연결이 합리적인 사람이 그러한 목적으로 사용될 것이라고 예상할 수 있는 정보에만 근거해야 합니다. 또한 개인의 명시적 옵트인 동의가 없는 경우, 여러분은 다음에 기반해서만 특정 개인에 대해 여러분이 보유한 기록을 X 상의 신원(X identity)과 매칭하려고 시도할 수 있습니다.

* **해당 개인이 여러분에게 직접 제공한 정보.** 여러분과 사전에 아무런 관계가 없는 개인에 대한 기록(제3자로부터 취득한 개인 데이터 포함)은 이 기준을 충족하지 않는다는 점에 유의해야 합니다. 및/또는
* **공개 데이터.** 이 문맥에서 “공개 데이터”는 다음을 의미합니다.
  * 여러분이 공개적으로 이용 가능한 출처(예: 직능 단체 회원 명부)에서 취득한 개인 관련 정보
  * 다음을 포함하여, X 상에서 개인에 대해 공개적으로 이용 가능한 정보:
    * 게시물
    * 계정 소개(bio) 및 공개적으로 표시된 위치를 포함한 프로필 정보
    * 표시 이름 및 @handle

<div id="using-geo-data">
  ## 지리 데이터 사용
</div>

**지리 데이터는 민감한 정보이므로 사용에 추가적인 제한이 적용됩니다.** 귀하의 서비스가 게시물에 위치 정보를 추가하는 경우, 이용자에게 다음 사항을 반드시 고지해야 합니다.

* 위치 정보를 언제 추가하는지
* 위치 정보를 지오태그로 추가하는지, 아니면 주석 데이터로 추가하는지
* 위치 정보가 장소로 표시되는지, 아니면 지리 좌표로 표시되는지

귀하의 애플리케이션이 이용자가 자신의 위치와 함께 게시물을 올릴 수 있도록 허용하는 경우, X의 [지리 가이드라인](/ko/developer-terms/geo-guidelines)을 전면적으로 준수해야 합니다.

**위치 데이터나 지리 정보를 단독으로 사용하는 것은 금지됩니다.** 귀하는 (그리고 다른 사람이 그렇게 하도록 허용해서도) 게시물의 일부인 경우를 제외하고, X 콘텐츠에 포함된 위치 데이터 및 기타 지리 정보를 저장, 집계 또는 캐시할 수 없습니다. 예를 들어, 시간 경과에 따라 개인이 어디에 있었는지를 보여주기 위해 게시물에서 위치 데이터나 지리 정보를 분리해서는 안 됩니다. 도시 내 해시태그 사용 인원 수와 같이 집계된 지리 활동을 보여주는 히트맵 및 관련 도구는 허용됩니다.

X의 플랫폼 사용 가이드라인은 서비스의 전체 라이프사이클에 걸쳐 X 콘텐츠 사용이 서비스 시작 첫날부터 계속해서 규정을 준수하도록 필요한 지원을 제공합니다. 연동 기능이 X를 사용하는 사람들과 X 플랫폼 전체에 안전하고 유익한 방식으로 작동하고 있는지 확인하기 위해 이러한 규칙을 정기적으로 검토할 것을 권장합니다.

<div id="spam-bots-and-automation">
  ## 스팸, 봇, 자동화
</div>

**스팸을 생성하거나 어떤 형태로든 플랫폼 조작을 하기 위해 X API 및 개발자 제품을 사용하는 것은 금지됩니다.** [플랫폼 조작 및 스팸](https://help.x.com/rules-and-policies/platform-manipulation)에 관한 X 규칙을 검토하고, 귀하의 서비스가 당사의 정책을 위반하지 않으며, 다른 사람들이 이를 위반하도록 하거나 가능하게 만들지 않도록 반드시 확인해야 합니다.

**게시물 작성, 계정 팔로잉, 다이렉트 메시지 발송 등 쓰기 작업을 수행하는 서비스는 [Automation Rules](https://help.x.com/rules-and-policies/x-automation)를 따라야 합니다.** 특히 다음 사항을 준수해야 합니다.

* 사람에게 [자동화된 답글 또는 다이렉트 메시지](https://twittercommunity.com/t/policy-clarification-automated-replies-and-mentions/94444)를 보내기 전에 항상 명시적인 동의를 받을 것
* 더 이상 연락하지 말아 달라는(opt-out) 요청이 있을 경우 즉시 존중하고 따를 것
* [대량 팔로우](https://twittercommunity.com/t/policy-clarification-aggressive-following-and-inorganic-following-behavior/92769)를 포함해 대량, 공격적이거나 스팸성인 행동을 절대 수행하지 않을 것
* [여러 계정](https://blog.twitter.com/developer/en_us/topics/tips/2018/automation-and-the-use-of-multiple-accounts.html)에 동일하거나 실질적으로 유사한 콘텐츠를 절대 게시하지 않을 것

**API 기반 봇 계정을 운영하는 경우, 해당 계정이 어떤 계정인지와 누가 책임을 지는지 명확하게 표시해야 합니다.** 계정이 봇인지 아닌지에 대해 사람들을 오도하거나 혼란스럽게 해서는 안 됩니다. 이를 위한 좋은 방법은 프로필 소개에 해당 계정이 봇이라는 문구를 포함하는 것입니다.

<div id="public-display-of-posts">

**여러분이 공개적으로 또는 여러분의 서비스를 이용하는 사람들에게 표시하는 모든 X 콘텐츠의 무결성을 유지해야 합니다.** 콘텐츠를 표시할 때 [X for Websites](https://developer.x.com/docs/twitter-for-websites/overview)를 사용하지 않는 경우, X API를 사용해 표시 가능한 최신 버전을 가져와야 합니다. 표시된 콘텐츠가 X API를 통해 더 이상 제공되지 않는 경우, 합리적으로 가능한 한 빠른 시점에, 또는 X나 해당 X 계정 소유자로부터 삭제 요청을 받은 후 24시간 이내에, 또는 관련 법률에 따라 요구되는 경우 그에 따라, 해당 콘텐츠를 여러분의 서비스에서 삭제해야 합니다.

**게시물을 임베드하거나 표시하는 경우, 사이트의 일일 노출 수가 1,000만 회를 초과하면 X API 액세스와 관련해 당사에 연락해야 합니다.** X는 X API 사용의 조건으로 추가 약관을 요구할 권리를 보유합니다. X for Websites 개발자 사용에 대한 추가 제한은 다음과 같습니다.

  * [여기](https://help.x.com/x-for-websites-ads-info-and-privacy)에 설명된 대로, 사람들이 X의 관심 기반 광고 및 개인화를 옵트아웃할 수 있는 방법에 대해 법적으로 충분한 안내를 제공해야 합니다.

* **X for Websites 위젯**
  * 법률상 이러한 정보 제공 및 동의 획득이 요구되는 경우, X의 [쿠키 사용](https://help.x.com/rules-and-policies/x-cookies)에 설명된 대로, 사람들에게 그들의 기기에 저장되거나 그 기기에서 액세스되는 쿠키 또는 기타 정보에 대해 명확하고 포괄적인 정보를 제공하고 이에 대한 동의를 얻어야 합니다.

* **13세 미만 아동을 대상으로 하는 서비스**
  * 13세 미만 아동을 대상으로 하는 서비스는, [여기](https://developer.x.com/docs/twitter-for-websites/privacy)에 설명된 대로 옵트아웃 매개변수를 ‘true’로 설정하여, 임베드된 게시물 및/또는 임베드된 타임라인에서 X의 맞춤화를 옵트아웃해야 합니다.

<div id="content-redistribution">
  ## 콘텐츠 재배포
</div>

**X 콘텐츠를 얻을 수 있는 가장 적절한 출처는 X 자체입니다. 따라서 X는 X 콘텐츠의 제3자 재배포를 제한합니다.** 다운로드 가능한 데이터 세트나 API 등을 통해 제3자에게 X 콘텐츠를 제공하는 경우, 아래에서 설명한 예외를 제외하고 게시물 ID, 다이렉트 메시지 ID 및/또는 사용자 ID만 배포할 수 있습니다.

전체적으로, X로부터 서면 허가를 받지 않는 한, 임의의 30일 기간 동안 어떤 단체(단일 법인에 소속된 여러 개인을 포함)를 기준으로도 1,500,000개를 초과하는 게시물 ID를 배포할 수 없습니다. 또한 비자동화된 수단(예: 스프레드시트 또는 PDF 다운로드)을 통해 제공하는 경우, 개발자는 귀하의 서비스를 이용하는 각 개인에게 하루 최대 500개의 공개 게시물 객체 및/또는 사용자 객체를 제공할 수 있습니다.

학술 연구자는 학술 기관을 대신하여, X로부터 서면 승인을 받은 비영리 연구이거나 해당 연구의 동료 평가 또는 검증 목적으로만 게시물 ID 및/또는 사용자 ID를 배포할 수 있습니다. 그러한 연구를 위해 필요하며 X가 승인한 범위 내에서만 게시물 ID 또는 사용자 ID를 사용할 수 있습니다.

제3자에게 제공되는 모든 X 콘텐츠는 본 정책의 적용을 받으며, 그러한 제3자는 해당 다운로드를 받기 전에 X [서비스 이용 약관](https://x.com/tos), [개인정보 처리방침](https://x.com/privacy), [Developer Agreement](/ko/developer-terms/agreement), 및 [Developer Policy](/ko/developer-terms/policy)에 동의해야 합니다. 귀하는 이 [정책](/ko/developer-terms/policy), [Developer Agreement](/ko/developer-terms/agreement) 또는 X와의 기타 계약에 포함된 X 콘텐츠 배포에 관한 기타 제한이나 한계를 어떠한 주체도 우회할 수 있도록 해서는 안 됩니다.

**참고:** 본 조항은 EU Digital Services Act (2022) (“**DSA**”) 제40조에 따라 X API 액세스를 보유한 연구자에게는 적용되지 않으며, 이들 연구자는 대신 DSA 및 Developer Agreement에 규정된 절차와 제한의 적용을 받습니다.

<div id="advertising-on-x">
  ## X에서의 광고 집행
</div>

**X 콘텐츠 주변에서 어디에서, 그리고 어떤 방식으로 광고를 집행할 수 있는지에 대해 제한이 있습니다.** 먼저, 귀하의 광고는 게시물과 유사해 보이거나 사람들이 게시물로 합리적으로 오인할 수 있어서는 안 됩니다. 광고에 대한 기타 규칙은 다음과 같습니다.

* **X 콘텐츠와 귀하의 광고 사이에는 명확한 구분이 있어야 합니다.** X Ads 이외에는 X 타임라인 내에 어떤 광고도 게재할 수 없습니다.
* **X는 X API를 통해 광고를 제공할 권리를 보유합니다.** 우리가 API를 통해 X Ads 제공을 시작한 이후 귀하가 X Ads 집행을 선택하는 경우, 관련 약관에 따라 광고 수익의 일부를 귀하와 공유합니다.
* **귀하는 X 콘텐츠 또는 X API를 통해 획득한 정보를 사용하여 X 플랫폼 외부에서 사람들을 대상으로 광고를 타게팅할 수 없습니다.**

아래에 나열된 X 서비스 또는 기능을 사용하는 모든 경우에는 다음의 추가 규칙이 적용됩니다.

<div id="x-login">
  ## X 로그인
</div>

**예를 들어 OAuth 프로토콜을 통해, 사람들이 X에 로그인하고 로그아웃할 수 있는 옵션을 쉽게 찾을 수 있도록 반드시 제공해야 합니다.** *Sign in with X* 옵션은 귀하의 서비스에 있는 다른 어떤 가입 또는 로그인 기능만큼 눈에 잘 띄게 표시되어야 합니다. 또한 X 계정이 없는 사람들에게는 X를 통해 계정을 생성할 수 있는 기회도 제공해야 합니다.

**귀하의 서비스에서 누군가가 *Sign in with X*를 통해 인증을 완료하면, 해당 사용자의 X 아이덴티티를 명확하게 표시해야 합니다.** X 아이덴티티에는 해당 사용자의 현재 X @handle, 아바타, 그리고 X 로고가 포함됩니다. 귀하의 서비스에서 누군가의 X 팔로워를 표시하는 경우, 해당 팔로우 관계가 X와 연관된 것임을 명확하게 보여주어야 합니다.

<div id="x-cards">
  ## X Cards
</div>

**양질의 사용 경험을 제공하려면, Cards가 표시되는 모든 플랫폼에서 올바르게 렌더링되도록 Card를 개발해야 합니다.** Cards를 사용할 때 따라야 하는 추가 규칙은 다음과 같습니다.

* Card 내에 해당 미디어를 표시할 계획이라면, 게시물을 민감한 미디어에 대해 ‘true’로 표시해야 합니다.
* Card 내의 모든 애셋을 호스팅할 때는 HTTPS를 사용해야 합니다. Card에서 활성 혼합 콘텐츠와 관련된 브라우저 경고가 발생해서는 안 됩니다.
* 오디오 및 비디오 콘텐츠에는 정지 또는 일시정지 컨트롤이 포함되어야 하며, 자동 재생되는 비디오는 기본 설정이 ‘소리 끔’ 상태여야 합니다.

**Cards에 대해 X가 부과한 제한(해당 Card의 의도된 사용을 포함)을 초과하거나 이를 우회해서는 안 됩니다.** Cards 사용에 대한 추가 제한은 다음과 같습니다.

* X의 승인을 받지 않은 제3자 후원(스폰서드) 콘텐츠를 Cards 내에 게재할 수 없습니다.
* Card 내에서 또는 Card를 통해 X에서 금전적 인센티브(가상 화폐 포함)를 제공하거나 연결할 수 없습니다.
* URL 및 미디어와 같이 오해의 소지가 있거나 문맥상 관련성이 없는 콘텐츠 또는 동작을 Card에 포함할 수 없습니다.
* 사용자가 게시물에서 해당 앱을 명시적으로 홍보하거나 언급하는 경우에만 게시물에 App Card를 첨부할 수 있습니다.
