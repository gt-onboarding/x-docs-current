<div id="x-developers">
  ## X + 개발자
</div>

X는 개발자를 소중히 여깁니다. X 데이터의 강력한 기능을 활용해 이 커뮤니티가 만들어 내는 다양한 도구와 서비스에 늘 감탄하고 있습니다. 이 커뮤니티에 대한 우리의 약속의 일환으로, 우리는 개발자에게는 개방적이고 공정하며, X를 사용하는 사람들에게는 안전하고, X 플랫폼 전체에도 도움이 되는 방식으로 데이터 액세스를 제공하는 것을 목표로 합니다. 이러한 목표를 달성하기 위해, 우리는 API 및 X Content의 적절한 사용과 관련된 우리의 규칙과 기대를 이해하는 데 도움이 되도록 안내서로서 Developer Policy를 마련했습니다.

본 Developer Policy(“**Policy**”)는 X의 애플리케이션, 서비스, 웹사이트, 웹 페이지 및 콘텐츠로 구성된 생태계와 상호작용하는 개발자를 위한 규칙과 가이드라인을 제공합니다. 이는 X API와 X Content에 대한 액세스 및 사용을 규율하는 X와의 계약(Developer Agreement 또는 X와의 기타 서면 계약)의 일부를 구성합니다. Policy 위반은 귀하의 계약 위반으로 간주됩니다. 이 Policy는 별도의 고지 없이 수시로 변경될 수 있습니다. 이 Policy에서 사용되었으나 정의되지 않은 대문자로 표기된 용어는 [Developer Agreement](/ko/developer-terms/agreement) 또는 Master License Agreement에서 부여된 각각의 의미를 갖습니다.

<div id="using-this-policy">
  ## 이 정책 사용 방법
</div>

우리는 이 정책을 가능한 한 이해하기 쉽게 구성했습니다. X API 및 X 콘텐츠를 사용할 때 다음 정책 섹션의 내용을 항상 염두에 두시기 바랍니다:

1. **성공을 위한 준비** - 모든 X 정책을 준수할 책임은 귀하에게 있습니다. X API 및 X 콘텐츠에 액세스하기 전에, 이 정책과 이 문서에서 링크하는 다른 정책들을 검토하고 이해하는 것이 중요합니다. 미리 정책을 검토하는 데 들이는 시간이, 나중에 수 시간에 걸친 재작업을 막아 줄 수 있습니다.

2. **개인정보 보호와 통제는 필수적입니다** - X를 이용하는 사람들의 개인정보를 보호하고 방어하는 것은 우리 회사의 핵심 DNA에 내재해 있습니다. 따라서 우리는 사람들의 합리적인 개인정보 보호 기대 수준에 부합하지 않는 방식으로 X 데이터를 사용하는 것을 금지합니다. X API를 기반으로 서비스를 구축하거나 X 콘텐츠에 액세스하는 경우, 특히 사람들의 개인정보를 존중하고 그들의 데이터가 어떻게 사용되는지에 대해 투명성을 제공하며 통제권을 부여함으로써, 이 약속을 지키는 데 있어 귀하는 중요한 역할을 맡게 됩니다.

3. **플랫폼 사용 가이드라인을 따르세요** - X API 및 X 콘텐츠에 대한 액세스 승인을 받는 것은 첫 단계에 불과합니다. 계획 중인 X 플랫폼 사용이 정책을 준수하는지 궁금한 점이 있을 때마다, Platform Usage Guidelines를 가장 먼저 참고해야 합니다.

아래에서 이 세 가지 핵심 섹션 각각의 의미에 대해 훨씬 더 자세히 설명합니다. X API 및 X 콘텐츠 사용이 우리의 정책과 일치하는지 확인하기 위해, 해당 내용을 꼼꼼히 검토해 주시기 바랍니다.

당사가 귀하가 이 정책(또는 다른 X 정책)을 위반하고 있다고 판단하는 경우, X API 및 X 콘텐츠에 대한 귀하의 액세스를 일시 중단하거나 영구적으로 철회할 수 있습니다.

마지막으로, X는 X 애플리케이션을 개선하고, 상업적 사용을 검토하며, 승인된 사용 사례 및 이 정책에 대한 귀하의 준수 여부를 확인하기 위해 X API 사용을 모니터링할 수 있다는 점에 유의해 주시기 바랍니다.

읽어 주셔서 감사합니다. 그리고 우리와 함께 만들어 주셔서 감사합니다! 여러분이 무엇을 만들어 내실지 기대하겠습니다!

  ## 성공을 위한 기반을 마련하세요
</div>

첫날부터 올바른 방식으로 서비스를 구축하면 X API 사용 중 발생할 수 있는 많은 잠재적인 문제를 피할 수 있습니다. 이 Developer Policy 섹션에는 X API 또는 X Content를 사용하기 전에 모든 개발자가 따라야 하는 규칙이 포함되어 있습니다.

Free, Basic, Pro 플랜([developer.x.com/en](http://developer.x.com/en)에 설명됨)은 취미 개발, 상업적 프로토타이핑, 초기 단계 X 제품 통합, 최종 사용자가 제한적인 애플리케이션을 위해 설계되었습니다. X API 및 X Content를 이 범위를 넘어 사용하는 경우, Enterprise 플랜([developer.x.com/en](http://developer.x.com/en)에 설명됨)에 신청하거나 이미 가입해 있어야 합니다.

**우리는 X developer 플랫폼의 모든 예정된 사용 방식을 검토하여 정책 준수 여부를 확인하므로, X API 및 X Content 사용 승인을 받고 이를 유지하기 위해, 계획된 사용 목적을 반드시 공개하고 필요에 따라 업데이트해야 합니다.** 모든 신규 개발자는 X API에 접근하기 위해 [개발자 계정을 신청](https://developer.x.com/portal/petition/essential/basic-info)해야 합니다. 승인된 개발자 계정이 없는 기존 개발자는 X의 안내에 따라 계정을 신청해야 합니다. 이 과정의 일환으로, X API 및 X Content의 의도된 사용 방식에 대한 서면 설명을 제공해야 합니다.

사용 사례 설명은 귀하에게 구속력을 가지며, 그로부터의 어떠한 실질적인 일탈도 우리의 규칙 위반이 될 수 있고 제재 조치로 이어질 수 있습니다. 사용 사례를 실질적으로 변경하는 경우, 해당 변경 사항을 우리에게 통지하고, 그 새로운 목적을 위해 X Content 사용을 시작하기 전에 승인을 받아야 합니다. 이를 이행하지 않으면 API 및 데이터 접근의 정지 또는 해지로 이어질 수 있습니다.

**X API를 기반으로 구축하거나 X Content에 접근하는 경우, 모든 X 정책을 준수해야 합니다.** 여기에는 이 [Developer Policy](/ko/developer-terms/policy), [Automation Rules](https://help.x.com/rules-and-policies/x-automation), [Display Requirements](https://developer.x.com/developer-terms/display-requirements.html), [API Restricted Uses Rules](/ko/developer-terms/restricted-use-cases), [X Rules](https://help.x.com/rules-and-policies/x-rules), [X Brand Resources](https://about.x.com/who-we-are/brand-toolkit) 및 X API 또는 X Content 사용과 관련하여 X와 체결하는 기타 모든 계약(여기에는 [Developer Agreement](/ko/developer-terms/agreement) 또는 Master Licensing Agreement 또는 Order(해당되는 경우)가 포함되지만 이에 제한되지 않음)이 포함됩니다. 또한 이러한 정책의 수정 사항과 X에서 새로 도입하는 모든 정책을 준수해야 합니다. 귀하의 서비스 사용을 모니터링하고, 사용자가 X 정책을 위반하지 않도록 서비스를 설계하는 것은 귀하의 책임입니다. 이를 이행하지 않으면 API 및 X Content 접근의 정지 또는 해지로 이어질 수 있습니다.

**단일 사용 사례 또는 실질적으로 유사하거나 중복되는 사용 사례에 대해 여러 애플리케이션을 등록할 수 없습니다.** 이 문맥에서 “사용 사례(use case)”란 애플리케이션을 통해 수행되는 일관된 분석, 표시 또는 행동의 집합을 의미합니다. 도구나 서비스의 &quot;화이트 라벨(white label)&quot; 버전은 허용되지 않습니다. 애플리케이션의 “화이트 라벨” 버전을 제공하려는 경우, 반드시 우리에게 이를 통지하고 별도의 승인을 받아야 합니다.

*이러한 규칙에 대한 단 하나의 예외로,* 동일한 서비스의 development, staging, production 인스턴스를 위해 최대 3개의 애플리케이션을 생성하고 사용할 수 있습니다. 이 앱들은 단일 계정에 등록되어야 하며, 이름과 설명에서 각각 단일 서비스의 dev, staging, prod 인스턴스로 명확히 식별되어야 합니다. **development 또는 staging 애플리케이션을 production 용도로 사용할 수 없습니다.**

**모든 API 키 및 기타 접근 자격 증명은 비공개로 유지해야 합니다.** 타인이 소유한 API 키 또는 기타 접근 자격 증명을 사용하거나, 다른 사람이 이를 사용하도록 권장하거나 조장할 수 없습니다.

**X와의 라이선스 계약은 X API 및 X Content 사용을 제한합니다. 그 외에도, X API에는 공정한 데이터 사용을 보장하고 플랫폼의 스팸을 방지하기 위한 요청 한도가 있습니다.** 이 Policy 또는 X와의 계약에 설명되어 있거나, [Developer Site](https://developer.x.com/)에 기재되어 있거나, X가 귀하에게 통지한 요청 한도 또는 그 밖의 제한이나 제약을 초과하거나 이를 우회할 수 없습니다.

**X API를 통해 수신된 X Content에 포함된 모든 독점 고지 또는 표시를 제거하거나 변경할 수 없습니다.** 이는 사람들이 X Content가 어디에서 오는지, 누구의 소유인지 알 수 있도록 하기 위한 것입니다.

**데이터 무결성과 플랫폼의 건전성을 위해, X API 또는 X 서비스의 어떤 기능도 방해, 가로채기, 중단 또는 비활성화할 수 없습니다.** 다시 말해, [developer.x.com](https://developer.x.com/)에 문서화된 대로 의도된 방식으로 API를 사용해야 합니다. 허용 가능한 사용에 대한 자세한 내용은 [HackerOne 가이드라인](https://hackerone.com/x)을 참조하세요.

<div id="privacy-and-control-are-essential">
  ## 프라이버시와 통제는 필수적입니다
</div>

X는 프라이버시를 매우 중요하게 여기며, X 콘텐츠와 X API를 사용하는 모든 분들도 동일하게 이를 준수하시기를 기대합니다. 사람들의 합리적인 프라이버시 기대에 반하는 방식으로 X 개발자 플랫폼, X API 또는 X 콘텐츠를 사용하는 경우, API 및 X 콘텐츠에 대한 접근의 일시 중단이나 해지를 포함한 제재 조치의 대상이 될 수 있습니다.

프라이버시와 통제에 대한 귀하의 책임은 X 콘텐츠의 모든 활용과, 당사 API를 이용해 구축하는 서비스의 모든 측면에까지 확대되어야 합니다. 이를 위해, 귀하의 서비스를 이용하는 사람들은 귀하가 이들의 데이터를 어떻게 사용하고, 이들을 대신해 X에 어떻게 접근하는지 이해하고 이에 동의해야 합니다. 이는 이용자에게 명확하고 포괄적이며 투명한 개인정보 처리방침을 제공하고, 이들을 대신해 어떠한 조치를 취하기 전에 귀하의 서비스를 이용하는 각 사람으로부터 명시적이고 충분한 설명에 기반한 동의를 받음으로써 달성할 수 있습니다. **귀하의 서비스에 인증을 거쳐 로그인하는 행위만으로는 동의가 성립되지 않는다는 점에 유의하시기 바랍니다.**

<div id="consent-permissions">
  ## 동의 및 권한
</div>

**특히, 다음과 같은 작업을 수행하기 전에 사람들로부터 명시적이고 충분한 설명에 근거한 동의를 받아야 합니다.**

* 다른 사람을 대신하여 *어떠한* 행동이라도 하는 것. 여기에는 (이에 한정되지 않지만) 다음이 포함됩니다:
  * X에 콘텐츠 게시
  * 계정 팔로우/언팔로우
  * 프로필 또는 계정 정보 수정
  * 게시물에 해시태그나 기타 콘텐츠 추가

* X API 또는 기타 X 도구 이외의 수단으로 접근한 콘텐츠를 재게시하는 행위

* 누군가의 X 콘텐츠를 사용해 상품이나 서비스를 홍보하는 행위

* 다이렉트 메시지(DM)와 같은 비공개 콘텐츠나 그 밖의 사적 또는 기밀 정보를 저장하는 행위

* 보호된 콘텐츠나 그 밖의 사적 또는 기밀 정보를 공유하거나 게시하는 행위

**서비스에서 사람들이 X에 콘텐츠를 게시할 수 있도록 허용하는 경우, 게시 전에 다음을 수행해야 합니다.**

* 무엇이 게시될지 정확히 보여주기
* 서비스 이용자에게 어떤 위치 정보(geo 정보)(있을 경우)가 콘텐츠에 추가되는지 명확히 알리기

**서비스에서 사람들이 귀하의 서비스와 X 둘 다에 콘텐츠를 게시할 수 있도록 허용하는 경우, 게시 전에 다음을 수행해야 합니다.**

* 콘텐츠 게시 권한을 얻기
* 콘텐츠를 어디에 게시할 것인지 설명하기

**모든 X 콘텐츠의 보호 및 차단 상태를 존중해야 합니다. 한 사람의 인증 토큰을 사용해 얻은 콘텐츠를, 해당 콘텐츠를 볼 권한이 없는 다른 사람에게 제공해서는 안 됩니다.**

* **보호된 계정:** [보호된 계정](https://help.x.com/safety-and-security/public-and-protected-posts)의 콘텐츠는 계정 소유자가 해당 계정을 팔로우하도록 승인한 사람만 열람할 수 있습니다. 따라서 보호된 계정에 접근하는 서비스를 운영하는 경우, 해당 콘텐츠를 볼 권한이 있는 특정인에게 그러한 콘텐츠를 제공하는 목적에 한해서만 접근해야 합니다.
* **차단된 계정:** X 이용자는 자신이 선택한 어떤 이유로든 계정에 대한 접근을 [차단](https://help.x.com/using-x/blocking-and-unblocking-accounts)할 수 있습니다. 이 선택을 우회하기 위해 토큰(또는 기타 API 기반 작업)으로 얻은 정보를 뒤섞어 사용하는 행위는 허용되지 않습니다.

**다이렉트 메시지(DM)는 비공개 특성을 가지므로, DM 기능을 제공하는 서비스는 개인정보 보호를 위해 추가적인 조치를 취해야 합니다. DM 콘텐츠를 볼 권한이 없는 사람에게 해당 콘텐츠를 제공해서는 안 됩니다. 서비스에서 DM 기능을 제공하는 경우 다음 사항도 준수해야 합니다.**

* DM에 대한 읽음 확인(read receipt) 이벤트를 전송하는 경우 이를 사람들에게 알려야 합니다. 이는 서비스 내에서 직접 공지를 제공하거나, 대화의 다른 참여자의 읽음 확인을 표시하는 방식으로 수행할 수 있습니다.
* DM에서 전송되는 미디어를 &quot;공유&quot; 상태(즉, 여러 DM에서 재사용 가능)로 설정하기 전에 동의를 받아야 합니다. DM 내 미디어를 &quot;공유&quot; 가능하게 허용하는 경우, 이 콘텐츠가 미디어 URL을 가진 누구에게나 접근 가능하다는 점을 명확한 공지로 알려야 합니다.

<div id="content-compliance">
  ## 콘텐츠 컴플라이언스
</div>

**X 콘텐츠를 오프라인으로 저장하는 경우, X에서의 현재 상태와 일치하도록 항상 최신 상태로 유지해야 합니다.** 구체적으로, X에서 삭제되거나 수정된 콘텐츠가 있다면, 여러분이 보관하고 있는 해당 콘텐츠도 삭제하거나 수정해야 합니다. 이는 합리적으로 가능한 한 신속하게, 또는 X나 해당 X 계정 소유자로부터 그러한 요청을 받은 후 24시간 이내에, 또는 X와의 계약이나 관련 법률에서 정한 바에 따라 이행되어야 합니다. 법률에 의해 금지되지 않는 한 반드시 이를 준수해야 하며, 그러한 금지가 있는 경우에도 X의 명시적인 서면 허가가 있는 경우에만 예외가 허용됩니다.

수정된 콘텐츠는 여러 형태를 취할 수 있습니다. 여기에는 (이에 국한되지 않고) 다음이 포함됩니다:

* 비공개로 전환되었거나 보호 상태로 설정된 콘텐츠
* 플랫폼에서 정지된 콘텐츠
* 지오태그가 제거된 콘텐츠
* X에서 차단되었거나 삭제된 콘텐츠

<div id="off-x-matching">

</div>

**X에서의 사람을 X 외부에서 획득하거나 보관 중인 정보와 매칭할 수 있는 상황을 우리는 제한합니다.** X 외부 매칭이란, X @handle 또는 사용자 ID를 포함한 X 콘텐츠를 사람, 가구, 기기, 브라우저 또는 기타 X 외부 식별자와 연관시키는 것을 말합니다. 이러한 연관은, 아래에 설명된 경우를 제외하고, 해당 사람으로부터 명시적인 옵트인 동의를 먼저 받은 경우에만 할 수 있습니다.

사람의 X 아이덴티티를 X 외부 식별자와 연결하기 위한 명시적인 옵트인 동의가 없는 상황에서는, 여러분이 만드는 모든 연결은 합리적인 사람이면 그 목적을 위해 사용될 것이라고 예상할 수 있는 정보에만 기반해야 합니다. 또한, 사람의 명시적인 옵트인 동의가 없는 경우, 여러분은 다음에 기반해서만 특정인에 대한 여러분의 기록을 X 아이덴티티와 매칭하려고 시도할 수 있습니다:

* **해당 사람이 여러분에게 직접 제공한 정보.** 여러분과 사전 관계가 없는 개인에 대한 기록(제3자로부터 획득한 개인 데이터 포함)은 이 기준을 충족하지 못한다는 점에 유의해야 합니다. 그리고/또는
* **공개 데이터.** 이 맥락에서 &quot;공개 데이터&quot;란 다음을 의미합니다:
  * 여러분이 공적이고 일반적으로 이용 가능한 리소스(예: 직능 단체 회원 디렉터리)로부터 획득한 개인에 대한 정보
  * X 상에서 특정인에 관해 공개적으로 이용 가능한 정보로, 다음을 포함합니다:
    * 게시물
    * 계정 소개(bio) 및 공개적으로 명시된 위치를 포함한 프로필 정보
    * 표시 이름(display name) 및 @handle

<div id="using-geo-data">
  ## 지리 데이터 사용
</div>

**이 정보의 민감한 특성으로 인해 지리 데이터 사용에는 추가적인 제한이 적용됩니다.** 서비스가 게시물에 위치 정보를 추가하는 경우, 사용자에게 다음 사항을 고지해야 합니다.

* 언제 위치 정보를 추가하는지
* 위치 정보를 지오태그(geotag)로 추가하는지, 아니면 annotations 데이터로 추가하는지
* 위치 정보가 장소(place)로 표시되는지, 아니면 지리 좌표로 표시되는지

애플리케이션이 사용자가 자신의 위치와 함께 게시물을 올릴 수 있도록 허용하는 경우, X의 [지리 가이드라인](/ko/developer-terms/geo-guidelines)을 전면적으로 준수해야 합니다.

**위치 데이터 또는 지리 정보의 단독 사용은 금지됩니다.** 귀하는 (또는 다른 사람이 그렇게 할 수 있도록 허용해서도) 게시물의 일부인 경우를 제외하고, X 콘텐츠에 포함된 위치 데이터 및 기타 지리 정보를 저장, 집계하거나 캐시할 수 없습니다. 예를 들어, 개인이 시간이 지남에 따라 어디에 있었는지를 보여주기 위해 게시물에서 위치 데이터 또는 지리 정보를 분리해서는 안 됩니다. 집계된 지리 활동(예: 특정 도시에서 해시태그를 사용하는 사람 수)을 보여주는 히트맵 및 관련 도구는 허용됩니다.

  ## 플랫폼 사용 가이드라인
</div>

X의 정책을 검토하고 API 액세스 권한을 올바르게 설정했나요? 서비스가 X의 개인정보 보호 및 제어 가이드라인을 따르고 있나요? 이 두 가지 질문에 모두 예라고 답할 수 있다면, 이제 X API와 X 콘텐츠를 사용할 준비가 된 것입니다. X의 플랫폼 사용 가이드라인은 서비스 수명 주기 전반에 걸쳐 X 콘텐츠 사용이 서비스 시작 첫날부터 규정을 준수할 수 있도록 필요한 도움을 제공합니다. 통합 기능이 X를 사용하는 사람들과 X 플랫폼 전체에 안전하고 유익한 방식으로 운영되고 있는지 확인하기 위해, 이러한 규칙을 정기적으로 검토할 것을 권장합니다.

<div id="spam-bots-and-automation">
  ## 스팸, 봇, 그리고 자동화
</div>

**스팸을 생성하거나 어떤 형태로든 플랫폼 조작에 관여하기 위해 X API 및 개발자 제품을 사용하는 것은 금지됩니다.** [플랫폼 조작 및 스팸](https://help.x.com/rules-and-policies/platform-manipulation)에 관한 X 규칙을 검토하고, 귀하의 서비스가 당사의 정책을 위반하지 않도록 하며, 다른 사람이 이를 위반할 수 있도록 하거나 돕지 않도록 해야 합니다.

**게시물 작성, 계정 팔로잉, 다이렉트 메시지 전송 등 쓰기 작업을 수행하는 서비스는 [자동화 규칙](https://help.x.com/rules-and-policies/x-automation)을 따라야 합니다.** 특히, 다음을 준수해야 합니다:

* 사용자에게 [자동화된 답글 또는 다이렉트 메시지](https://twittercommunity.com/t/policy-clarification-automated-replies-and-mentions/94444)를 보내기 전에 항상 명시적인 동의를 받습니다
* 귀하로부터 더 이상 연락받지 않기를 요청하는 경우 즉시 이를 반영합니다
* [대량 팔로잉](https://twittercommunity.com/t/policy-clarification-aggressive-following-and-inorganic-following-behavior/92769)을 포함해 대량·공격적·스팸성 행위를 절대 수행하지 않습니다
* [여러 계정](https://blog.twitter.com/developer/en_us/topics/tips/2018/automation-and-the-use-of-multiple-accounts.html)에 걸쳐 동일하거나 실질적으로 유사한 콘텐츠를 절대 게시하지 않습니다

**API 기반 봇 계정을 운영하는 경우, 해당 계정이 어떤 계정이며 누가 책임을 지는지 명확히 표시해야 합니다.** 귀하의 계정이 봇인지 여부에 대해 사람들을 절대 오도하거나 혼란스럽게 해서는 안 됩니다. 이를 명확히 하는 한 가지 좋은 방법은 프로필 소개에 해당 계정이 봇이라는 문구를 포함하는 것입니다.

<div id="public-display-of-posts">
  ## 게시물의 공개 표시
</div>

**여러분이 공개적으로 또는 여러분의 서비스를 이용하는 사람들에게 표시하는 모든 X 콘텐츠의 무결성을 유지해야 합니다.** 콘텐츠를 표시하기 위해 [X for Websites](https://developer.x.com/docs/twitter-for-websites/overview)를 사용하지 않는 경우, X API를 사용하여 표시 가능한 가장 최신 버전을 조회해야 합니다. 표시 중인 콘텐츠가 X API를 통해 더 이상 제공되지 않게 되면, 합리적으로 가능한 한 신속히, 또는 X나 해당 X 계정 소유자로부터 삭제 요청을 받은 후 24시간 이내에, 또는 관련 법률에 따라 요구되는 경우, 여러분의 서비스에서 해당 콘텐츠를 삭제해야 합니다.

**X 콘텐츠를 오프라인에서 표시하는 경우 따라야 하는 구체적인 규칙이 있습니다.** 게시물을 오프라인에서 표시하는 경우, 방송에서 게시물을 사용할 때 준수해야 할 [가이드라인](https://developer.x.com/developer-terms/display-requirements)을 따르십시오.

**게시물을 임베드하거나 표시하는 경우, 사이트의 일일 노출 수가 1,000만 회를 초과하면 여러분의 X API 액세스와 관련해 저희에게 연락해야 합니다.** X는 X API 사용의 조건으로 추가 약관을 요구할 권리를 보유합니다. X for Websites를 사용하는 개발자에 대한 추가 제한 사항은 다음과 같습니다.

<div id="content-redistribution">
  ## 콘텐츠 재배포
</div>

**X 콘텐츠를 얻는 가장 좋은 방법은 X로부터 직접 제공받는 것입니다. 따라서 우리는 X 콘텐츠의 제3자 재배포를 제한합니다.** 다운로드 가능한 데이터 세트나 API 제공을 포함해 제3자에게 X 콘텐츠를 제공하는 경우, 아래에 설명된 경우를 제외하고는 게시물 ID, 다이렉트 메시지 ID 및/또는 사용자 ID만 배포할 수 있습니다.

총합 기준으로, 어떤 30일 기간에도 X로부터 서면 허가를 받지 않은 이상 하나의 단체(해당 단체에 소속된 여러 개인을 포함)에 1,500,000개를 초과하는 게시물 ID를 배포할 수 없습니다. 또한, 자동화되지 않은 방식(예: 스프레드시트 또는 PDF 다운로드)을 통해 이루어지는 경우에 한해, 개발자는 귀하의 서비스를 사용하는 각 사람에게 매일 최대 500개의 공개 게시물 객체 및/또는 사용자 객체를 제공할 수 있습니다.

학술 연구자는 학술 기관을 대신하여 수행하는 비상업적 연구 목적, 그리고 그러한 연구에 대한 동료 심사 또는 검증 목적을 위해, X로부터 서면 승인을 받은 경우에 한해 게시물 ID 및/또는 사용자 ID를 배포할 수 있습니다. 그러한 연구에 필요하고 X가 승인한 범위 내에서만 게시물 ID 또는 사용자 ID를 사용할 수 있습니다.

제3자에게 제공되는 모든 X 콘텐츠는 계속해서 본 정책의 적용을 받으며, 해당 제3자는 그러한 다운로드를 받기 전에 X의 [이용약관](https://x.com/tos), [개인정보 처리방침](https://x.com/privacy), [Developer Agreement](/ko/developer-terms/agreement), [Developer Policy](/ko/developer-terms/policy)에 동의해야 합니다. 귀하는 이 [Policy](/ko/developer-terms/policy), [Developer Agreement](/ko/developer-terms/agreement) 또는 X와 체결한 그 밖의 어떠한 계약에 포함된 X 콘텐츠 배포에 관한 다른 한도나 제한을 어떤 단체도 우회하도록 허용해서는 안 됩니다.

**참고:** 본 섹션은 EU Digital Services Act (2022) (“**DSA**”) 제40조에 따라 X API에 접근하는 연구자에게는 적용되지 않으며, 이들은 대신 DSA 및 Developer Agreement에 규정된 절차와 제한의 적용을 받습니다.

<div id="x-login">
  ## X 로그인
</div>

**예를 들어 OAuth 프로토콜을 통해, 사람들이 X에 로그인하고 로그아웃할 수 있는 옵션을 쉽게 찾을 수 있도록 제공해야 합니다.** *Sign in with X* 옵션은 귀하의 서비스에서 다른 어떤 회원가입 또는 로그인 기능만큼이나 눈에 잘 띄게 표시되어야 합니다. 또한 X 계정이 없는 사람에게는 X를 통해 새 계정을 만들 수 있는 기회도 제공해야 합니다.

**누군가가 귀하의 서비스에서 *Sign in with X*를 통해 인증을 완료하면, 해당 사용자의 X 아이덴티티를 명확하게 표시해야 합니다.** X 아이덴티티에는 사용자의 현재 X @handle, 아바타, 그리고 X 로고가 포함됩니다. 귀하의 서비스에서 사용자의 X 팔로워를 표시하는 경우, 그 관계가 X와 연관되어 있음을 명확하게 보여 주어야 합니다.

</div>

**양질의 이용 경험을 보장하기 위해, Card가 표시되는 모든 플랫폼에서 올바르게 렌더링되도록 Card를 개발해야 합니다.** Cards를 사용할 때 따라야 하는 추가 규칙은 다음과 같습니다:

* Card 내에 민감한 미디어를 표시할 계획이 있는 경우, 해당 게시물을 민감한 미디어에 대해 ‘true’로 표시해야 합니다
* Card 내의 모든 애셋을 호스팅할 때는 HTTPS를 사용해야 합니다. Card로 인해 활성 혼합 콘텐츠와 관련된 브라우저 경고가 발생해서는 안 됩니다
* 오디오 및 비디오 콘텐츠에는 정지 또는 일시 정지 컨트롤이 포함되어야 하며, 자동 재생되는 비디오의 경우 기본값은 ‘음소거’ 상태여야 합니다

**Card 및 Card의 의도된 사용에 대해 X가 설정한 제한을 초과하거나 이를 우회해서는 안 됩니다.** Cards 사용에 대한 추가 제한 사항은 다음과 같습니다:

* X의 승인을 받지 않고 제3자 스폰서드 콘텐츠를 Cards 내에 게재해서는 안 됩니다
* Card 내에서나 Card를 통해 X에서 금전적 인센티브(가상화폐 포함)를 제공해서는 안 됩니다
* Card 내에 URL 및 미디어와 같이 오해를 불러일으키거나 맥락상 관련성이 없는 콘텐츠나 동작을 포함해서는 안 됩니다
* 사용자가 게시물에서 해당 앱을 명시적으로 홍보하거나 언급하는 경우에만 App Card를 게시물에 첨부할 수 있습니다
