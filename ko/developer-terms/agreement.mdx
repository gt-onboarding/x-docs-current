“Accept &amp; Subscribe”를 클릭하거나, 유료 서비스에 대한 정기 구독료를 계속 지불하거나, 그 밖의 방식으로 어떤 라이선스 자료를 이용하거나 접근하는 경우, 귀하는 본 계약의 조건에 동의하는 것입니다. 구독은 아래에 설명된 바와 같이 취소될 때까지 자동으로 갱신됩니다. 구독하려면 인증된 전화번호가 필요합니다. 다른 플랫폼을 통해 구독한 경우, 해당 플랫폼에서 구독을 관리해야 합니다.

본 X 개발자 계약(“**본 계약**”)은 귀하(본 계약에서 “**귀하**”라고 합니다)와 X 간에 체결되는 법적 구속력이 있는 계약으로서, 아래에 정의된 라이선스 자료(아래에 정의된 모든 유료 서비스를 포함함)에 대한 귀하의 접근 및 이용을 규율합니다.

귀하가 어떤 라이선스 자료(아래에 정의됨)에 접근하거나 이를 이용하거나, 본 계약의 조건 또는 유료 서비스에 대한 정기 구독 결제를 수락하는 버튼을 클릭하는 경우, 귀하는 본 계약의 조건에 구속되는 것에 동의하는 것입니다. 본 계약의 조건을 이해하지 못하거나 그 중 일부라도 수락하지 않는 경우, 어떤 라이선스 자료에도 접근하거나 이를 이용할 수 없습니다. 귀하는 X와 법적 구속력이 있는 계약을 체결할 수 있는 법정 연령에 도달하지 않았거나, 적용 법률에 따라 라이선스 자료의 이용 또는 수령이 금지된 경우, 라이선스 자료를 이용하거나 본 계약을 수락할 수 없습니다. 유료 서비스를 구매하고 이용하려면, 귀하는 (i) 만 18세 이상이거나 귀하가 거주하는 관할 지역의 법률에서 정한 성년이어야 하거나, (ii) 해당 유료 서비스를 구매하고 이용하는 것에 대해 부모 또는 보호자의 명시적인 동의를 받아야 합니다. 귀하가 부모 또는 법적 보호자이고 귀하의 자녀(또는 귀하가 보호자인 아동)가 유료 서비스를 구매하거나 이용하도록 허용하는 경우, 귀하는 본 계약의 조건이 귀하에게도 적용됨에 동의하고, 본 계약을 준수하며, 유료 서비스에서의 해당 아동의 활동에 대해 책임을 지고, 아동 또한 본 계약의 조건을 준수하도록 할 책임이 있음을 인정합니다. 어떠한 경우든, [**X 서비스 약관**](https://x.com/tos)의 “서비스를 이용할 수 있는 사람” 섹션에 명시된 바와 같이, X 서비스 약관에서 정의한 “서비스”를 이용하려면 최소 만 13세 이상이어야 합니다. 귀하가 회사, 조직, 정부 또는 기타 법적 실체를 대신하여 본 계약을 수락하거나 라이선스 자료를 이용하는 경우, 귀하는 그러한 회사, 조직, 정부 또는 기타 법적 실체를 본 계약에 구속시킬 권한을 보유하고 있음을 진술하고 보증하는 것이며, 이 경우 본 계약에서 사용되는 “귀하” 및 “귀하의”라는 용어는 해당 실체를 가리키는 것으로 봅니다. 귀하가 본 계약의 조건에 동의하지 않는 경우, 귀하는 (그리고 귀하의 어떤 인력도) 라이선스 자료에 접근하거나 이를 이용할 수 없습니다.****

4. “**Licensed Material**”은 개별적으로 또는 총칭하여 X API 및 X Content를 의미합니다.

**다이렉트 메시지(DMs)는 본질적으로 비공개이므로, DM 기능을 제공하는 서비스는 개인 정보 보호를 위해 추가적인 조치를 취해야 합니다. DM 콘텐츠를 볼 권한이 없는 사람들에게 DM 콘텐츠를 제공해서는 안 됩니다. 귀하의 서비스가 DM 기능을 제공하는 경우 다음도 수행해야 합니다.**

**X 콘텐츠를 오프라인으로 저장하는 경우, X에 게시된 해당 콘텐츠의 현재 상태와 일치하도록 항상 최신 상태로 유지해야 합니다.** 구체적으로, X에서 삭제되거나 수정된 콘텐츠가 있다면, 여러분이 보관 중인 해당 콘텐츠도 삭제하거나 수정해야 합니다. 이는 합리적으로 가능한 한 신속하게, 그리고 X 또는 해당 X 계정 소유자로부터 그러한 조치를 취하라는 요청을 받은 때로부터 24시간 이내에, 또는 X와의 계약이나 적용 법률에서 정한 기간 내에 수행되어야 합니다. 또한 법률에 의해 달리 금지되지 않는 한 반드시 이행해야 하며, 그러한 금지가 있는 경우에도 X의 명시적인 서면 허가가 있는 때에만 예외가 인정됩니다.

수정된 콘텐츠는 여러 형태가 있을 수 있습니다. 여기에는 (이에 한정되지 않고) 다음과 같은 경우가 포함됩니다.

* 비공개로 전환되었거나 보호 상태가 된 콘텐츠
* 플랫폼에서 정지된 콘텐츠
* 지오태그가 제거된 콘텐츠
* X에서 보류되었거나 제거된 콘텐츠

* **해당 당사자가 귀하에게 직접 제공한 정보.** 귀하와 사전 관계가 없는 개인에 대한 기록, 예를 들어 제3자로부터 취득한 개인 데이터는 이 기준에 부합하지 않는다는 점에 유의해야 합니다. 그리고/또는

</div>

**지리 데이터는 그 정보의 민감한 특성으로 인해 추가적인 제한이 적용됩니다.** 귀하의 서비스가 게시물에 위치 정보를 추가하는 경우, 사용자에게 다음 사항을 반드시 고지해야 합니다:

* 언제 위치 정보를 추가하는지
* 위치 정보를 지오태그로 추가하는지, 아니면 주석(annotation) 데이터로 추가하는지
* 위치 정보가 장소로 표시되는지, 아니면 지리적 좌표로 표시되는지

귀하의 애플리케이션이 사용자가 자신의 위치와 함께 게시할 수 있도록 허용하는 경우, X의 [지리 가이드라인](/ko/developer-terms/geo-guidelines)을 전면적으로 준수해야 합니다.

**위치 데이터나 지리 정보의 단독 사용은 어떠한 형태로든 금지됩니다.** 귀하는 (그리고 다른 사람이 그렇게 할 수 있도록 허용해서도) 게시물의 일부인 경우를 제외하고, X 콘텐츠에 포함된 위치 데이터 및 기타 지리 정보를 저장, 집계 또는 캐시할 수 없습니다. 예를 들어, 개인이 시간이 지나면서 어디에 있었는지를 보여주기 위해 게시물로부터 위치 데이터나 지리 정보를 분리하는 행위는 허용되지 않습니다. 집계된 지리 활동(예: 한 도시에서 특정 해시태그를 사용하는 사람 수)을 보여주는 히트맵 및 관련 도구는 허용됩니다.

X의 정책을 충분히 검토하고 API 액세스를 올바르게 설정했나요? 서비스가 X의 프라이버시 및 제어(controls) 가이드라인을 준수하고 있나요? 이 두 질문에 모두 “예”라고 답할 수 있다면, 이제 X API와 X 콘텐츠를 사용할 준비가 된 것입니다. <div id="spam-bots-and-automation">

</div>

**스팸을 생성하거나 어떤 형태로든 플랫폼 조작에 가담하기 위해 X API 및 개발자 제품을 사용하는 것은 금지됩니다.** [플랫폼 조작 및 스팸](https://help.x.com/rules-and-policies/platform-manipulation)에 관한 X 규정을 검토하고, 귀하의 서비스가 당사 정책을 위반하지 않을 뿐 아니라 다른 사람들이 이를 위반하도록 유도하거나 가능하게 하지 않도록 해야 합니다.

**게시물 작성, 계정 팔로우, Direct Message 발송 등 쓰기 작업을 수행하는 서비스는 [Automation Rules](https://help.x.com/rules-and-policies/x-automation)를 따라야 합니다.** 특히 다음 사항을 준수해야 합니다:

* 사람들에게 [자동 응답 또는 Direct Message](https://twittercommunity.com/t/policy-clarification-automated-replies-and-mentions/94444)를 보내기 전에 항상 명시적인 동의를 받아야 합니다.
* 귀하에게 연락하지 말 것을 요청하는 경우, 이를 즉시 존중하고 더 이상 연락하지 않아야 합니다.
* [대량 팔로우](https://twittercommunity.com/t/policy-clarification-aggressive-following-and-inorganic-following-behavior/92769)를 포함하여 대량·공격적·스팸성 행위를 절대 수행해서는 안 됩니다.
* [여러 계정](https://blog.twitter.com/developer/en_us/topics/tips/2018/automation-and-the-use-of-multiple-accounts.html)에 동일하거나 실질적으로 유사한 콘텐츠를 게시해서는 안 됩니다.

**API 기반 봇 계정을 운영하는 경우, 해당 계정이 어떤 계정인지와 누가 이를 책임지는지 명확히 밝혀야 합니다.** 계정이 봇인지 여부와 관련해 사람들을 절대 오도하거나 혼란스럽게 해서는 안 됩니다. 이를 위한 좋은 방법은 프로필 소개란에 해당 계정이 봇임을 명시하는 문구를 포함하는 것입니다.
