“동의 및 구독”을 클릭하거나 유료 서비스에 대한 정기 구독료를 계속 지불하거나, 기타 어떠한 방식으로든 라이선스된 자료를 액세스하거나 사용함으로써, 귀하는 본 계약의 약관에 동의하는 것입니다. 구독은 아래에 설명된 대로 취소될 때까지 자동으로 갱신됩니다. 구독하려면 인증된 전화번호가 필요합니다. 다른 플랫폼을 통해 구독한 경우, 해당 플랫폼에서 구독을 관리해야 합니다.

본 X 개발자 계약(“**계약**”)은 귀하(본 계약에서 “**귀하**”로 지칭됨)와 X 간에 체결되는 법적 구속력이 있는 계약으로, 라이선스된 자료(아래에 정의함), 및 그 안에 포함되는 유료 서비스(아래에 정의함)에 대한 귀하의 액세스 및 사용을 규율합니다.

귀하가 라이선스된 자료(아래에 정의함)에 액세스하거나 이를 사용하거나, 본 계약의 약관 또는 유료 서비스에 대한 반복적인 구독 결제에 동의하는 버튼을 클릭함으로써, 귀하는 본 계약의 약관에 구속되는 것에 동의합니다. 본 계약의 약관을 이해하지 못하거나 그 일부라도 동의하지 않는 경우, 귀하는 어떠한 라이선스된 자료에도 액세스하거나 이를 사용할 수 없습니다. 귀하는 X와 법적 구속력이 있는 계약을 체결할 수 있는 법적 연령에 해당하지 않거나, 적용 법률상 라이선스된 자료를 사용하거나 수령하는 것이 금지된 경우 라이선스된 자료를 사용하거나 본 계약에 동의할 수 없습니다. 유료 서비스를 구매하고 사용하려면 (i) 귀하가 거주하는 관할 지역의 법률에 따라 정해진 성년이거나 최소 18세 이상이거나, (ii) 해당 유료 서비스를 구매하고 사용하기 위해 부모 또는 보호자의 명시적 동의를 받아야 합니다. 귀하가 부모 또는 법적 보호자이며 귀하의 자녀(또는 귀하가 보호자인 아동)가 유료 서비스를 구매하거나 사용하도록 허용하는 경우, 귀하는 본 계약의 약관이 귀하에게 적용되는 것에 동의하며, 귀하는 본 계약을 준수하고, 유료 서비스에서 해당 아동의 활동에 책임을 지며, 아동 또한 본 계약의 약관을 준수하도록 할 책임이 있습니다. 어떠한 경우든, [**X 서비스 약관**](https://x.com/tos)의 “서비스를 사용할 수 있는 사람” 섹션에 명시된 바와 같이, X 서비스 약관에서 정의된 “서비스”를 사용하려면 최소 13세 이상이어야 합니다. 귀하가 회사, 조직, 정부 또는 기타 법인을 대신하여 본 계약에 동의하거나 라이선스된 자료를 사용하는 경우, 귀하는 해당 법인을 본 계약에 구속시킬 권한이 있음을 진술 및 보증하며, 이 경우 본 계약에서 사용되는 “귀하” 및 “귀하의”는 해당 법인을 의미합니다. 귀하가 본 계약의 약관에 동의하지 않는 경우, 귀하는 (또한 귀하의 어느 인력에게도) 라이선스된 자료에 액세스하거나 이를 사용하도록 허용할 수 없습니다.****

**I. 정의**

본 계약에서 다음의 정의가 적용됩니다.

1. “**Developer Site(개발자 사이트)**”는 X의 개발자 사이트인 [**https://developer.x.com/**](https://developer.x.com/)를 의미합니다.

2. “**Direct Message(다이렉트 메시지)**”는 X의 다이렉트 메시지 기능을 사용하여 한 사용자가 하나 이상의 특정 다른 사용자에게 X 애플리케이션에서 보내는 비공개 메시지를 의미합니다.

3. “**Intellectual Property Rights(지식 재산권)**”는 전 세계적으로 등록 여부와 관계없이 모든 저작권, 인격권, 특허권, 상표권 및 기타 지식 재산권 또는 이와 유사한 권리를 의미합니다.

4. “**Licensed Material(라이선스 대상 자료)**”은 개별적으로 또는 총칭하여 X API 및 X 콘텐츠를 의미합니다.

5. &quot;**Paid Service(s)(유료 서비스)**”는 이 계약 및 관련 기능 또는 기능성에 적용 가능한 정기 요금을 지불하는 대가로 귀하가 액세스하는 Licensed Material의 기능 또는 기능성을 의미합니다. 본 계약에 달리 명시되지 않는 한, “Licensed Material”에 대한 모든 언급에는 Paid Service(s)가 포함되는 것을 의도합니다.

6. “**Payment Portal(결제 포털)**”은 귀하가 Licensed Material 사용을 선택(옵트인)하고, 유료 서비스에 대한 구독을 관리하며, 유료 서비스의 결제 수단을 관리하고, 계정 정보를 업데이트할 수 있도록 귀하에게 제공되는 온라인 포털을 의미합니다.

7. “**Services(서비스)**”는 X 콘텐츠를 표시하거나 그 밖에 Licensed Material을 사용하는 귀하의 서비스, 웹사이트, 애플리케이션 및 기타 제공물(연구를 포함)을 의미합니다.

8. “**Post(게시물)**”는 X 애플리케이션을 통해 배포되는 단문 텍스트 및 멀티미디어 기반 메시지를 의미합니다.

9. “**X**”는 (a) 귀하의 주된 영업소가 유럽연합, EFTA 국가 및 영국 이외에 있는 경우 X Corp.(865 FM 1209, Building 2, Bastrop, TX 78602, USA)를 의미하고, (b) 귀하의 주된 영업소가 유럽연합, EFTA 국가 또는 영국에 있는 경우 X Internet Unlimited Company(One Cumberland Place, Fenian Street, Dublin 2, D02 AX07, Ireland)를 의미합니다.

10. “**X API**”는 X 애플리케이션 프로그래밍 인터페이스(각각, **“API”**), 소프트웨어 개발 키트(각각, **“SDK”**), 그리고 Developer Site를 통해 X가 제공하는 관련 도구, 문서, 데이터, 기술, 코드 및 기타 자료를 의미합니다.

11. “**X Applications(X 애플리케이션)**”는 일반적으로 “X”로 지칭되는 X의 실시간 정보 서비스를 의미하며, 여기에는 X가 현재 자사 웹사이트 및 X가 소유, 운영 또는 관리하는 모바일 애플리케이션, 소셜 플러그인 및 애플리케이션 프로그래밍 인터페이스를 통해 제공하는 서비스가 포함됩니다.

13. “**X Marks(X 마크)**”는 본 계약에 따라 사용하는 경우에 한하여, Developer Site를 통해 제공되는 경우를 포함해 X가 귀하에게 제공하는 X의 이름, 상표 및 로고를 의미합니다.

14. “**Users(사용자)**”는 귀하의 서비스 방문자, 사용자 또는 고객을 의미합니다.

**A. X로부터의 라이선스.** 본 계약 및 아래에 정의된 적용 가능한 포함된 개발자 약관을 준수한다는 조건 하에, X는 귀하에게 다음 각 호의 목적에 한정하여 사용할 수 있는 비독점적이고, 로열티가 부과되지 않으며, 양도 불가능하고, 서브라이선스가 불가능하며, 취소 가능한 라이선스를 부여합니다:

1. X가 명시적으로 승인한 대로, 귀하의 서비스에 X 콘텐츠를 통합하거나 X 콘텐츠에 대한 분석을 수행하기 위해 X API를 사용하는 것;

3. 귀하의 서비스에서 표시하기 위한 형식으로 조정하기 위한 목적에 한하여 X 콘텐츠를 수정하는 것; 및

4. 본 계약에 명시된 바에 따라 X 콘텐츠의 출처로서 X 애플리케이션을 표기하기 위해 X 마크를 사용하고 표시하는 것.

프라이버시와 통제에 대한 귀하의 책임은 X 콘텐츠의 모든 사용과 당사의 API를 사용해 구축하는 서비스의 모든 측면에까지 확대되어야 합니다. 이를 위해 귀하의 서비스를 사용하는 사람들이 귀하가 그들의 데이터를 어떻게 사용하는지, 그리고 그들을 대신해 X에 어떻게 접근하는지 이해하고 이에 동의해야 합니다. 이를 달성하려면 사람들에게 명확하고 포괄적이며 투명한 개인정보 처리방침을 제공하고, 그들을 대신해 어떠한 조치를 취하기 전에 귀하의 서비스를 사용하는 각 사람으로부터 명시적이고 충분한 설명에 기반한 동의를 받아야 합니다. **귀하의 서비스에 로그인 또는 인증을 한다는 사실만으로는 동의로 간주되지 않는다는 점에 유의하시기 바랍니다.**

<div id="consent-permissions">
  ### 동의 및 권한
</div>

**특히, 다음과 같은 작업을 수행하기 전에 사람들로부터 명시적이고 충분한 설명을 통해 얻은 동의를 받아야 합니다.**

* 그 사람을 대신해 *어떠한* 행동이라도 수행하는 경우. 여기에는 다음이 포함되지만, 이에 한정되지는 않습니다.
  * X에 콘텐츠 게시
  * 계정 팔로우/언팔로우
  * 프로필 또는 계정 정보 수정
  * 게시물에 해시태그 또는 기타 콘텐츠 추가

* X API 또는 기타 X 도구가 아닌 수단으로 취득한 콘텐츠를 재게시하는 행위

* 타인의 X 콘텐츠를 제품 또는 서비스 홍보에 사용하는 행위

* 다이렉트 메시지(Direct Messages, DMs)와 같은 비공개 콘텐츠 또는 그 밖의 사적이거나 기밀인 정보를 저장하는 행위

* 보호된 콘텐츠 또는 그 밖의 사적이거나 기밀인 정보를 공유하거나 공개하는 행위

**귀하의 서비스가 사람들이 X에 콘텐츠를 게시할 수 있도록 허용하는 경우, 게시 전에 반드시 다음을 수행해야 합니다.**

* 무엇이 게시될지 정확히 보여줄 것
* 귀하의 서비스를 사용하는 사람들에게 어떤 위치 정보(있는 경우)가 콘텐츠에 추가되는지 명확히 알릴 것

**귀하의 서비스가 사람들이 귀하의 서비스와 X 모두에 콘텐츠를 게시할 수 있도록 허용하는 경우, 게시 전에 반드시 다음을 수행해야 합니다.**

* 해당 콘텐츠를 게시할 수 있는 권한을 취득할 것
* 해당 콘텐츠를 어디에 게시할 것인지 설명할 것

**모든 X 콘텐츠의 보호 및 차단 상태를 존중해야 합니다. 한 사람의 인증 토큰을 사용하여 얻은 콘텐츠를, 해당 콘텐츠를 볼 권한이 없는 다른 사람에게 제공해서는 안 됩니다.**

* **보호된 계정:** [보호된 계정](https://help.x.com/safety-and-security/public-and-protected-posts)의 콘텐츠는 계정 소유자가 팔로우를 승인한 사람만 볼 수 있습니다. 따라서 귀하가 보호된 계정에 접근하는 서비스를 운영하는 경우, 해당 콘텐츠를 볼 권한이 있는 특정 사람들에게 그러한 콘텐츠를 제공하기 위한 목적에 한해서만 해당 계정에 접근해야 합니다.
* **차단된 계정:** X 이용자는 원하는 어떤 이유에서든 자신의 계정에 대한 접근을 [차단](https://help.x.com/using-x/blocking-and-unblocking-accounts)할 수 있습니다. 이 선택을 우회하기 위해 토큰(또는 기타 API 기반 작업)으로 얻은 정보를 혼합하거나 조합해 사용하는 행위는 허용되지 않습니다.

**다이렉트 메시지(Direct Messages, DMs)는 비공개적인 성격을 가지므로, DM 기능을 제공하는 서비스는 개인정보를 보호하기 위해 추가적인 조치를 취해야 합니다. 귀하는 해당 콘텐츠를 볼 권한이 없는 사람에게 DM 콘텐츠를 제공해서는 안 됩니다. 귀하의 서비스가 DM 기능을 제공하는 경우 다음도 수행해야 합니다.**

* DM에 대한 읽음 확인 이벤트를 전송하는 경우 사람들에게 이를 알릴 것. 이는 귀하의 서비스 내에서 직접 고지를 제공하거나, 대화의 다른 참여자들로부터 받은 읽음 확인을 표시하는 방식으로 할 수 있습니다.
* DM에서 전송되는 미디어를 &quot;공유됨&quot;(즉, 여러 DM에서 재사용 가능)으로 설정하기 전에 동의를 받을 것. DM의 미디어를 &quot;공유됨&quot; 상태로 허용하는 경우, 이 콘텐츠가 해당 미디어의 URL을 가진 누구에게나 접근 가능하다는 점을 명확하게 고지해야 합니다.

<div id="content-compliance">
  ### 콘텐츠 컴플라이언스
</div>

**X 콘텐츠를 오프라인으로 저장하는 경우, X 상의 해당 콘텐츠의 현재 상태와 일치하도록 유지해야 합니다.** 구체적으로, X에서 삭제되거나 수정된 콘텐츠가 있다면, 귀하가 보유한 해당 콘텐츠도 삭제하거나 수정해야 합니다. 이러한 조치는 합리적으로 가능한 한 신속하게, 또는 X나 해당 X 계정 소유자로부터 그와 같은 요청을 받은 시점으로부터 24시간 이내, 또는 X와의 계약이나 적용되는 법률에서 정한 기한 내에 이행해야 합니다. 이는 법률에 따라 그러한 조치가 금지되는 경우를 제외하고 적용되며, 그와 같은 경우에도 X의 명시적인 서면 허가가 있는 경우에만 예외로 할 수 있습니다.

수정된 콘텐츠는 다양한 형태를 취할 수 있습니다. 여기에는 다음 예시가 포함되나, 이에 한정되지는 않습니다:

* 비공개로 전환되었거나 보호 상태가 된 콘텐츠
* 플랫폼에서 정지된 콘텐츠
* 지오태그가 제거된 콘텐츠
* X에서 제한되거나 삭제된 콘텐츠

<div id="off-x-matching">
  ### X 외부 매칭
</div>

**X 외부에서 획득하거나 보관한 정보와 X 상의 사람을 매칭할 수 있는 상황은 엄격히 제한됩니다.** X 외부 매칭이란 X 콘텐츠(X @handle 또는 사용자 ID 등을 포함)를 개인, 가구, 기기, 브라우저 또는 기타 X 외부 식별자와 연관시키는 것을 의미합니다. 이러한 연관을 수행하기 전에는, 해당 개인으로부터 사전 명시적 옵트인 동의를 받았거나, 아래에 설명된 경우에만 이를 수행할 수 있습니다.

해당 개인의 X 상 정체성(X identity)을 X 외부 식별자와 연결하기 위한 명시적 옵트인 동의가 없는 상황에서는, 귀하가 수행하는 모든 연결이 그 목적을 위해 사용될 것이라고 합리적으로 예상할 수 있는 정보에만 근거해야 합니다. 또한 개인의 명시적 옵트인 동의가 없는 경우, 귀하는 특정인에 대한 귀하의 보유 기록을 X identity와 매칭하려는 시도를 다음에 근거해서만 할 수 있습니다.

* **해당 개인이 귀하에게 직접 제공한 정보.** 귀하와 어떠한 선행 관계도 없는 개인에 대한 기록(제3자로부터 취득한 개인 데이터 포함)은 이 기준을 충족하지 못한다는 점에 유의해야 합니다. 및/또는
* **공개 데이터.** 이 맥락에서 “공개 데이터”는 다음을 의미합니다.
  * 귀하가 공적이고 일반적으로 이용 가능한 자료(예: 직능 단체 회원 명부)에서 취득한 개인에 관한 정보
  * X 상에서 특정인에 대해 공개적으로 이용 가능한 정보로서, 예를 들면 다음과 같습니다.
    * 게시물
    * 계정 소개(bio) 및 공개적으로 표시된 위치를 포함한 프로필 정보
    * 표시 이름 및 @handle

<div id="your-privacy-policy">
  ### 귀하의 개인정보 처리방침
</div>

**귀하는 사용자가 귀하의 서비스를 다운로드, 설치하거나 귀하의 서비스에 가입하기 전에, 귀하 서비스의 개인정보 처리방침을 반드시 제공해야 합니다.** 해당 개인정보 처리방침에는 최소한 다음 정보가 포함되어야 합니다:

* 귀하의 서비스를 이용하는 사용자로부터 수집하는 정보
* 귀하가 그 정보를 어떻게 사용하고 공유하는지(X와의 공유 포함)
* 사용자가 자신의 정보와 관련된 문의 및 요청을 위해 귀하에게 어떻게 연락할 수 있는지

귀하의 개인정보 처리방침은 적용 가능한 모든 법률을 준수해야 하며, [X의 개인정보 처리방침](https://x.com/privacy) 및 [당사의 기타 서비스 및 계열사의 개인정보 처리방침](https://help.x.com/rules-and-policies/x-services-and-corporate-affiliates)이 사용자에게 부여하는 보호 수준보다 낮아서는 안 됩니다. 귀하는 귀하 자신의 개인정보 처리방침 및/또는 X의 개인정보 처리방침을 준수할 수 없는 경우 X API에 대한 접근과 모든 X 콘텐츠의 사용을 중단해야 합니다.

<div id="using-geo-data">
  ### 지리 데이터 사용
</div>

**지리 데이터는 그 민감한 특성으로 인해 추가적인 제한이 적용됩니다.** 귀하의 서비스가 게시물에 위치 정보를 추가하는 경우, 사람들에게 다음 사항을 고지해야 합니다:

* 언제 위치 정보를 추가하는지
* 위치 정보를 지오태그로 추가하는지, 아니면 주석(annotations) 데이터로 추가하는지
* 해당 위치 정보가 장소로 표시되는지, 아니면 지리 좌표로 표시되는지

애플리케이션이 사람들이 자신의 위치 정보를 포함해 게시할 수 있도록 하는 경우, X의 [geo guidelines](/ko/developer-terms/geo-guidelines)를 전적으로 준수해야 합니다.

**위치 데이터 또는 지리 정보를 단독으로 사용하는 행위는 금지됩니다.** 귀하는 X Content에 포함된 위치 데이터 및 기타 지리 정보를 게시물의 일부인 경우를 제외하고는 저장, 집계 또는 캐싱해서는 안 하며, 다른 사람이 그렇게 하도록 허용해서도 안 됩니다. 예를 들어, 시간이 지남에 따라 개인이 어디에 있었는지를 보여주기 위해 게시물에서 위치 데이터 또는 지리 정보를 분리해서는 안 됩니다. 집계된 지리 활동(예: 특정 도시에서 해시태그를 사용하는 사람 수)을 보여주는 히트맵 및 관련 도구는 허용됩니다.

<div id="x-passwords">
  ### X 비밀번호
</div>

**X 비밀번호를 저장하거나, 사람들이 X 비밀번호, 계정 자격 증명, 또는 개발자 앱 정보(consumer key 포함)를 직접 제공하도록 요청해서는 안 됩니다.** 귀하의 서비스와 X 이용자를 연결하는 인증 수단으로 [Sign-in with X](/ko/fundamentals/authentication/guides/log-in-with-x) 사용을 권장합니다.

<div id="platform-usage-guidelines">
  ### 플랫폼 사용 가이드라인
</div>

X의 정책을 충분히 검토하고 API 액세스를 올바르게 설정했습니까? 귀하의 서비스는 X의 개인정보 보호 및 제어 가이드라인을 준수하고 있습니까? 이 두 질문에 모두 &quot;예&quot;라고 답할 수 있다면, 이제 X API와 X 콘텐츠를 사용할 준비가 된 것입니다. X의 플랫폼 사용 가이드라인은 서비스 라이프사이클 전반에 걸쳐 X 콘텐츠 사용이 처음부터 지속적으로 정책을 준수할 수 있도록 필요한 도움을 제공합니다. 연동이 X 이용자와 X 플랫폼 전체에 대해 안전하고 유익한 방식으로 운영되고 있는지 확인하기 위해, 이러한 규칙을 정기적으로 검토할 것을 권장합니다.

<div id="spam-bots-and-automation">
  ### 스팸, 봇 및 자동화
</div>

**스팸을 생성하거나 어떠한 형태로든 플랫폼 조작에 관여하기 위해 X API 및 개발자 제품을 사용하는 것은 금지됩니다.** [플랫폼 조작 및 스팸](https://help.x.com/rules-and-policies/platform-manipulation)에 관한 X 규칙을 검토하고, 귀하의 서비스가 해당 정책을 위반하지 않을 뿐 아니라, 다른 사람들이 이를 위반하도록 만들거나 가능하게 하지 않도록 하세요.

**게시물 게시, 계정 팔로우, 다이렉트 메시지 발송을 포함한 쓰기 작업을 수행하는 서비스는 반드시 [Automation Rules](https://help.x.com/rules-and-policies/x-automation)를 준수해야 합니다.** 특히 다음 사항을 지켜야 합니다.

* 사람들에게 [자동 응답 또는 다이렉트 메시지](https://twittercommunity.com/t/policy-clarification-automated-replies-and-mentions/94444)를 보내기 전에 항상 명시적인 동의를 받으세요.
* 더 이상 연락을 받고 싶지 않다는 요청이 있을 경우 즉시 이를 존중하세요.
