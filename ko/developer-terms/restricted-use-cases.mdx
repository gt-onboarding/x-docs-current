<div id="sensitive-information">
  ## 민감한 정보
</div>

X 데이터를 사용하여 X 사용자의 잠재적으로 민감한 특성을 도출하거나 추론하는 경우에는 각별히 주의해야 합니다. 다음과 같은 X 사용자의 정보를 도출하거나 추론하거나, 그렇게 도출·추론된 정보를 저장해서는 안 됩니다.

* 건강(임신 여부 포함)
* 부정적인 재정 상태 또는 재정적 상황
* 정치적 성향 또는 신념
* 인종 또는 민족적 출신
* 종교적 또는 철학적 성향 또는 신념
* 성생활 또는 성적 지향
* 노동조합 가입 여부
* 범죄의 혐의 또는 실제 범행 여부

어떠한 개인 데이터(예: 사용자 ID, 사용자 이름, 기타 식별자)도 저장하지 않는 X 콘텐츠에 대한 집계 분석은 허용됩니다. 다만, 해당 분석이 관련 법률 및 Developer Agreement and Policy의 모든 조항을 준수해야 합니다.

<div id="off-x-matching">
  ## X 외부 매칭
</div>

X 외부 매칭은 X 콘텐츠(X 사용자 이름 또는 사용자 ID 등)를 특정 개인, 가구, 디바이스, 브라우저 또는 기타 X 외부 식별자와 연계하는 것을 의미합니다. 예를 들어, X 사용자 이름을 기업의 고객 기록과 연계하는 경우가 이에 해당합니다(예: 고객 기록의 “John Doe”가 X의 @johndoe와 매칭되는 경우).

우리는 이용자가 X에서 별도의 정체성을 만들고, 원하는 경우 가명 기반으로 활동할 수 있기를 바랍니다. X 사용자에 대한 어떠한 정보든 X 외부 식별자와 연계하려는 경우, 해당 연계를 수행하기 전에 반드시 그 사용자로부터 명시적이고 옵트인 방식의 동의를 받아야 합니다. 예를 들어, 사용자가 귀하의 서비스 가입 절차 중에 자신의 X 핸들을 직접 제공하는 경우, 그 과정에서 이러한 동의를 얻을 수 있습니다.

사용자의 X 정체성을 X 외부 식별자와 연결하는 것에 대해 사용자의 명시적 옵트인 동의가 없는 상황에서는, 귀하가 만드는 모든 연결이 해당 목적을 위해 사용될 것이라고 사용자가 합리적으로 예상할 수 있는 정보에만 근거해야 합니다. 사용자가 제공한 정보를 이용해 자신의 X 계정을 X 외부의 정체성과 연결하고 있다는 사실을 알게 되었을 때 놀랄 것 같다면, 그렇게 하지 마십시오. 추가로, 개인의 명시적 옵트인 동의가 없는 경우, 다음에 근거하는 경우에만 귀하가 보유한 특정인에 대한 기록을 X 정체성과 매칭하려고 시도할 수 있습니다.

* **사용자가 귀하에게 직접 제공한 정보.** 귀하와 사전 관계가 전혀 없는 개인에 대한 기록(제3자로부터 취득한 개인 데이터 포함)은 이 기준을 충족하지 못합니다. 그리고/또는
* **공개 데이터.** 이 문맥에서 “공개 데이터”란 다음을 의미합니다.
  * 귀하가 공적이고 일반에 공개된 리소스(예: 어떤 직능단체 회원 명부)에서 취득한, 특정 사용자에 대한 정보
  * X에서 특정 사용자에 대해 공개적으로 이용 가능한 정보로, 다음을 포함합니다.
    * 게시물
    * 사용자의 소개(bio) 및 공개된 위치 정보를 포함한 프로필 정보
    * 표시 이름 및 사용자 이름

<div id="redistribution-of-x-content">
  ## X 콘텐츠 재배포
</div>

X API를 통해 획득한 X 콘텐츠를 다른 당사자와 공유해야 하는 경우, 가장 좋은 방법은 게시물 ID, 다이렉트 메시지 ID 및/또는 사용자 ID를 공유하는 것입니다. 그러면 콘텐츠의 최종 사용자가 X API를 사용하여 이를 복원(즉, 전체 게시물, 사용자 또는 다이렉트 메시지 콘텐츠를 요청)할 수 있습니다. 이를 통해 X 콘텐츠의 최종 사용자는 항상 X로부터 직접 최신 정보를 제공받을 수 있습니다.

X는 비자동화된 방식을 통한 복원된 X 콘텐츠의 제한적인 재배포를 허용합니다. 이와 같은 방식으로 복원된 X 콘텐츠를 다른 당사자와 공유하기로 선택하는 경우, 수신인 1명당 하루 기준으로 최대 50,000개의 복원된 공개 게시물 객체 및/또는 사용자 객체만 공유할 수 있으며, 이 데이터를 블로그 게시물의 첨부 파일이나 공개 GitHub 리포지토리와 같이 공개적으로 제공해서는 안 됩니다.

X 콘텐츠를 재배포할 때 염두에 두어야 할 몇 가지 추가 사항은 다음과 같습니다.

* X로부터 사전에 명시적인 서면 허가를 받지 않는 한, 30일 기간 동안 단일 주체에게 최대 1,500,000개의 게시물 ID까지만 배포할 수 있습니다.
* 비상업적 연구만을 목적으로 학술 기관을 대신하여 게시물 ID 및/또는 사용자 ID를 재배포하는 개인은 게시물 ID 및/또는 사용자 ID를 무제한으로 재배포할 수 있습니다.
* 위에서 설명한 대로 X 콘텐츠 공유에 대한 허가를 요청하려면 API 정책 지원 양식을 사용하십시오.

귀하가 제3자에게 X 콘텐츠를 배포할 수 있는 범위 내에서, 이 콘텐츠는 계속해서 Developer Agreement 및 Developer Policy의 적용을 받으며, 그러한 제3자는 X [이용 약관](https://twitter.com/en/tos), [개인정보 처리방침](https://twitter.com/en/privacy), [Developer Agreement](/ko/developer-terms/agreement), [Developer Policy](/ko/developer-terms/policy)에 동의해야만 X 콘텐츠를 수신할 수 있습니다.

<div id="multiple-applications">
  ## 여러 애플리케이션
</div>

단일 사용 사례 또는 실질적으로 유사하거나 중복되는 사용 사례에 대해 여러 애플리케이션을 등록하는 것은 허용되지 않습니다. 이러한 정책에 대해 자세히 알아보려면 [여기](https://twittercommunity.com/t/policy-clarification-multiple-applications-for-the-same-use-case/92080)를 참고하세요.

이 문맥에서 “사용 사례(use case)”란 하나의 애플리케이션을 통해 수행되는 일관된 분석, 표시, 또는 작업의 집합을 의미합니다. 동일한 서비스나 애플리케이션을 서로 다른 최종 사용자에게 제공하는 경우, 이는 하나의 사용 사례로 간주됩니다. 단, 도구나 서비스의 &quot;화이트 라벨(white label)&quot; 버전은 여기에 포함되지 않습니다. 애플리케이션의 &quot;화이트 라벨&quot; 버전을 제공하려는 경우, 반드시 사전에 당사에 통지하고 별도의 승인을 받아야 합니다. 이러한 규칙은 직접 등록한 애플리케이션뿐만 아니라, 귀하의 도구나 서비스를 사용하는 최종 사용자가 등록하는 애플리케이션에도 적용됩니다. 귀하의 도구나 서비스를 사용하기 위한 목적으로 최종 사용자에게 애플리케이션 등록을 요구하는 경우, 귀하와 귀하의 애플리케이션, 귀하의 고객, 그리고/또는 귀하의 도구 또는 서비스의 최종 사용자가 제재 조치의 대상이 될 수 있습니다.

이 규칙의 유일한 예외는 동일한 서비스에 대한 개발(“dev”), 스테이징, 프로덕션(“prod”) 인스턴스를 생성하는 경우입니다. 이러한 애플리케이션이 명확하게 구분되도록 표시되어 있는지(예: 애플리케이션 이름이나 설명에) 확인하고, 운영 목적에 개발 또는 스테이징 애플리케이션을 사용하지 마세요.

<div id="automation-spam-and-auto-responses">
  ## 자동화, 스팸 및 자동 응답
</div>

스팸을 생성하거나 스팸성 행위를 하기 위해 X의 API 및 개발자 제품을 사용하는 것은 금지되어 있습니다. 스팸과 관련된 [X 규정](https://t.co/rules)을 검토하고, 귀하의 앱이 당사의 정책을 위반하지 않을 뿐만 아니라 사용자에게 정책을 위반하도록 하거나 가능하게 하지 않도록 해야 합니다.

귀하의 앱이 게시물 작성, 계정 팔로잉, 다이렉트 메시지 발송 등 X 서비스에서 쓰기 작업을 수행하는 데 사용될 예정이라면, 서비스가 당사의 가이드라인을 준수하는지 확인하기 위해 [자동화 규정](https://support.twitter.com/articles/76915)을 신중하게 검토해야 합니다. 특히 다음 사항을 준수해야 합니다.

* [자동 응답 또는 메시지](https://twittercommunity.com/t/policy-clarification-automated-replies-and-mentions/94444)를 보내기 전에 항상 사용자로부터 명시적인 동의를 받아야 합니다.
* 귀하에게 연락받는 것을 거부하겠다는 사용자의 요청은 즉시 존중해야 합니다.
* [대량 팔로우](https://twittercommunity.com/t/policy-clarification-aggressive-following-and-inorganic-following-behavior/92769)를 포함하여 대량, 공격적, 스팸성 행위를 절대 수행해서는 안 됩니다.
* [여러 계정](https://blog.twitter.com/developer/en_us/topics/tips/2018/automation-and-the-use-of-multiple-accounts.html)에 걸쳐 동일하거나 실질적으로 유사한 콘텐츠를 절대 게시해서는 안 됩니다.

<div id="measuring-the-x-service">
  ## X 서비스 측정
</div>

벤치마킹 또는 경쟁 목적을 위해 X의 가용성, 성능, 기능 또는 사용량을 측정하는 데 X API를 사용해서는 안 됩니다. 예를 들어, X API를 다음과 같은 목적으로 사용해서는 안 됩니다:

* 전체 활성 사용자 수나 계정 수와 같은 X 사용자 메트릭의 총합을 계산하는 것
* 하루 게시물 총 개수, 사용자 참여 수, 계정 참여 수와 같은 X 게시물 메트릭의 총합을 계산하는 것
* [X 규칙](https://help.twitter.com/en/rules-and-policies/twitter-rules)에서 허용하는 경우를 제외하고, X에서 스팸이나 보안을 측정하거나 분석하는 것

<div id="surveillance-privacy-and-user-protection">
  ## 감시, 프라이버시 및 사용자 보호
</div>

X에서는 사용자의 프라이버시를 보호하고 수호하는 것이 당사의 핵심 DNA에 자리잡고 있으며, 당사의 개발자 및 데이터 제품은 이러한 약속을 반영합니다. 우리는 X 데이터가 전 세계에 선한 영향력을 미칠 수 있는 강력한 힘이 될 수 있다고 믿습니다. 자카르타 홍수 당시 인명을 구하는 데서부터 USGS가 지진을 추적하는 것을 돕는 일, UN과 협력해 지속가능발전목표(SDGs)를 달성하는 일에 이르기까지 다양한 사례가 있습니다. 그러나 우리는 어떤 주체든 감시 목적이거나, 그 밖에 사용자의 합리적인 프라이버시 기대 수준에 어긋나는 방식으로 X 데이터와 X API를 사용하는 것을 금지합니다. 예외는 없습니다.

우리는 [Developer Agreement](/ko/developer-terms/agreement)에서 당사 데이터 및 개발자 제품의 금지된 사용을 설명하고 있으며, 여기에는 X 사용자 또는 그들의 콘텐츠를 조사하거나 추적하는 행위, 그리고 민감한 이벤트(시위, 집회, 커뮤니티 조직 모임 등)를 추적·알림·모니터링하는 행위를 금지하는 내용이 포함됩니다.

이 약관에 따라 금지되는 기타 활동 범주에는 다음이 포함되나, 이에 국한되지 않습니다.

* 노동조합이나 행동주의 단체와 같은 민감한 집단 및 조직을 조사하거나 추적하는 행위
* 신원 조회 또는 모든 형태의 과도한 심층 심사(extreme vetting)
* 신용 또는 보험 리스크 분석
* 개인 프로파일링 또는 사이코그래픽(심리 특성 기반) 세분화
* 안면 인식

이 정책은 당사 API의 모든 사용자에게 적용됩니다. 이러한 목적을 위해 X API를 오용하는 경우, 액세스의 일시 정지 및 종료를 포함할 수 있는 제재 조치의 대상이 됩니다.

X 계정에 대한 정보를 요청하는 법 집행 기관을 위한 추가 정보는 [https://t.co/le](https://t.co/le)를 방문하십시오.

추가로, 현재 시점에서 X는 [Grok](https://help.x.com/en/using-x/about-grok)을 예외로 하고, 기반(foundation) 또는 프런티어(frontier) 모델을 미세 조정(fine-tune)하거나 학습시키기 위한 어떠한 X API 및/또는 X Content 사용도 금지합니다.
