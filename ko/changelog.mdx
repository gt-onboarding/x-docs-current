<Update label="Aug 3, 2022" description="X API v2">
  ### 필터링 연산자 별칭 추가

  필터링/쿼리 연산자 ‘별칭’이 Enterprise, Premium, v2 전반에 걸친 search 및 filtered stream 엔드포인트에 추가되었습니다. 많은 경우 연산자는 statuses 대신 게시물을 참조합니다. 예를 들어 &quot;in&#95;reply&#95;to&#95;status&#95;id&quot;의 별칭으로 &quot;in&#95;reply&#95;to&#95;tweet&#95;id&quot;를 사용할 수 있습니다. 다른 경우에는 새로운 연산자 별칭으로 수행되는 매칭 방식을 더 명확하게 확인할 수 있습니다. 예를 들어 게시물에는 하나의 비디오 첨부만 포함될 수 있으므로, &quot;has:videos&quot;에 대해 새로운 &quot;has:video&#95;link&quot; 별칭이 제공됩니다. 더 많은 예시는 [v2 filtered stream 연산자 표](https://developer.x.com/en/docs/twitter-api/tweets/filtered-stream/integrate/build-a-rule#list)를 확인하세요.
</Update>

<Update label="Mar 24, 2022" description="X API v2">

  ### OAuth 2.0 인증 지원

  오늘부로 모든 개발자는 developer portal에서 인증 방식으로 OAuth 2.0을 선택해 OAuth 2.0으로 인증할 수 있습니다. 이번 릴리스의 일환으로, 기밀 클라이언트(confidential client)와 공개 클라이언트(public client)에 대한 지원과 함께, 관련된 모든 v2 엔드포인트가 이 인증 방식을 사용할 수 있도록 지원을 추가했습니다.

  OAuth 2.0은 업계 표준 인가 프로토콜로, 개발자에게 애플리케이션 스코프에 대한 더 많은 제어 권한을 제공하고 여러 기기에 걸친 인가 플로를 개선합니다. 즉, X를 사용하는 사람들을 위한 애플리케이션을 구축하는 개발자는 자신의 앱이 사용자에게 요청하는 정보에 대해 더 세밀하게 제어할 수 있으므로, 최종 사용자에게 꼭 필요한 데이터와 정보만 요청하면 됩니다.

  이 최신 인가 프로토콜을 사용하면, 최종 사용자에게 앱 인가를 위한 동의 플로를 더 간결한 형태로 제공할 수 있으며, 그 과정에서 여러분이 요청한 특정 스코프만 표시됩니다.

  이번 출시와 관련해 더 자세한 내용을 알아보려면 포럼 공지 게시물을 방문해 주세요.

  오늘 X API v2가 X API의 기본 버전이 되었음을 발표합니다. 현재 X API v2에는 X API를 기반으로 구축된 전체 기존 앱의 90% 요구 사항을 충족할 수 있을 만큼 충분한 엔드포인트와 기능이 출시되어 있습니다.

  X API v2의 개발자 경험을 더욱 개선하기 위해 오늘 다음과 같은 변경 사항을 도입합니다.

  * API에 대한 빠르고 무료로 제공되는 Essential 액세스, 그리고 사용 사례가 승인된 개발자를 위한 무료 Elevated 액세스를 제공합니다. 이러한 액세스 레벨에 대한 더 자세한 내용은 About the X API 페이지에서 확인하실 수 있습니다.

  * X의 핵심 기능을 어떻게 구현할 수 있는지 제한하고, 앱을 통해 지원할 수 있는 사용자 수를 제한하던 Developer Policy의 문구를 제거했습니다.

    * 우리는 여러분이 X를 더 좋고 건강하며 공개 대화를 확장하는 데 기여하는 도구와 제품을 구축할 수 있도록 플랫폼을 개방하고자 합니다. [우리가 여러분이 만들어 주기를 바라는 솔루션 목록을 확인해 보세요](https://developer.x.com/en/docs/twitter-api/what-to-build).

    * 구체적으로는, X의 핵심 기능을 포함해 X 경험을 복제하는 것을 제한하던 조항과, 많은 수의 user token을 보유하기 위해 별도의 허가를 요구하던 조항을 제거했습니다.

    * X에서 사람들을 돕는 솔루션을 구축하는 일은, 종종 개발자가 X에서 제공되는 일부 기능을 직접 구축(또는 복제)해야 한다는 의미이기도 합니다. 이번 Developer Policy 변경 사항은 개발자 생태계에 명확성을 제공하고, 개발자가 더 쉽게 구축하고 혁신하며 공개 대화에 영향을 미칠 수 있도록 하는 개방형 API 플랫폼을 제공하는 것을 목표로 합니다.

  ### Academic Research Product Track 소개

  오늘 우리는 새로운 [Academic Research product track](/ko/resources/fundamentals/projects)를 [X API v2](/ko/x-api/introduction)에 도입합니다. 이번 업데이트를 통해 Academic Researcher를 위한 새로운 신청 프로세스가 도입되며, 승인을 받은 경우 Academic Research [Project](/ko/resources/fundamentals/projects)가 부여되어 더 넓은 데이터 접근 권한과 고급 기능을 사용할 수 있게 됩니다.

  새로운 Academic Research product track를 사용하면 다음 기능에 접근할 수 있습니다.

  * 현재 Academic Research product track에만 제공되는 새로운 X API v2 [full-archive search endpoint](/ko/x-api/posts/full-archive-search)

  * [Standard product track](/ko/resources/fundamentals/projects)의 Basic [access level](https://developer.x.com/en/products/x-api/early-access/guide#na_2)에서 제공되는 500,000개에서 1,000만 개로 늘어난 [post cap](/ko/x-api/fundamentals/post-cap)

  * [recent search](/ko/x-api/posts/recent-search), [full-archive search](/ko/x-api/posts/full-archive-search), 그리고 [filtered stream](/ko/x-api/posts/filtered-stream)에 사용할 수 있는 새로운 필터 연산자. 여기에는 `$`(일명 cashtag), `bio`(filtered stream에서만 사용 가능), `bio_name`(filtered stream에서만 사용 가능), `bio_location`(filtered stream에서만 사용 가능), `place`, `place_country`, `point_radius`, `bounding_box`, `-is:nullcast`, `has:cashtags`, `has:geo`가 포함됩니다.

  * filtered stream에 대해 동시 1,000개 규칙까지로 늘어난 규칙 한도(기존에는 Standard product track의 Basic access level에서 25개까지 지원)

  * 쿼리 및 규칙의 문자 수 한도가 512자에서 1,024자로 증가(기존에는 Standard product track의 Basic access level에서 512자까지 지원)

  이번 출시와 관련된 더 자세한 내용은 [포럼 공지](https://devcommunity.x.com/t/introducing-the-new-academic-research-product-track/148632)를 방문해 확인하세요.

<Update label="Dec 18, 2020" description="v2 X API용 대화 답글 설정 post 필드">
  ### Reply Settings 필드 추가

  오늘 우리는 게시물(post) 객체에 `reply_settings`라는 새 필드를 도입합니다. 이 필드는 post lookup, hide replies, recent search, sampled stream, filtered stream, follows lookup, user post timeline, user mention timeline 엔드포인트를 포함해 게시물을 반환하는 모든 X API v2 엔드포인트에서 사용할 수 있으며, 어떤 엔드포인트에서도 게시물 객체를 [확장(expansions)](/ko/x-api/fundamentals/expansions)하여 사용할 수 있습니다.

  이 필드는 게시물 작성자가 자신의 게시물에 누가 답글을 달 수 있는지에 대한 설정을 나타냅니다. 즉, 모두에게 허용했는지, 게시물에서 언급한 사람들에게만 허용했는지, 아니면 작성자가 팔로잉하는 사람들에게만 허용했는지를 나타냅니다. `reply_settings` 필드로 반환되는 값에는 `everyone`, `mentionedUsers`, `following`이 포함됩니다. 게시물에 대해 별도의 대화 제어가 설정되지 않은 경우에도 이 필드는 게시물 객체에 `everyone` 값을 가진 상태로 포함됩니다.
</Update>

  * 사용자에서 `most_recent_tweet_id` 확장 제거

  Recent Search의 핵심 검색, 페이지네이션 및 기타 기능은 동일하게 유지됩니다.

  v1에서 v2로 업데이트할 때 활용하실 수 있도록 [마이그레이션 자료](/ko/x-api/migrate/overview)를 준비해 두었습니다.

  이 연산자에 대해 더 자세히 알고 싶다면 [문서](/ko/x-api/enterprise-gnip-2.0/fundamentals/rules-filtering)를 확인하세요.

<Update label="Nov 29, 2018" description="X API: Enterprise v1.1">
  ### PowerTrack에서 is:reply 연산자 추가

  오늘 관심 있는 대화만 더 좁혀 볼 수 있도록 새로운 연산자인 `is:reply`를 소개합니다. 이제 결과에서 답글을 제외할 수 있습니다(기존에 제공되던 리포스트(`is:retweet`) 및 인용 게시물(`is:quote`) 제외 기능에 추가로 사용할 수 있습니다). 이 새로운 연산자는 오늘부터 [PowerTrack](/ko/x-api/enterprise-gnip-2.0/powertrack-api), [Historical PowerTrack](https://developer.x.com/content/developer-twitter/en/docs/tweets/batch-historical/overview), 그리고 [Replay](/ko/x-api/enterprise-gnip-2.0/powertrack-api#replay-api)에서 사용할 수 있습니다.

  `is:reply` 불리언 연산자는 게시물에 대한 모든 답글만 필터링하거나, 규칙과 일치하는 명시적인 답글을 제외하는 데 사용할 수 있습니다. 이 연산자는 `is:quote` 연산자와 동일한 방식으로 동작합니다.

  이 연산자에 대해 더 알아보려면 [문서](/ko/x-api/enterprise-gnip-2.0/fundamentals/rules-filtering)를 참고하세요.
</Update>

  ### Usage API의 새 버전

  오늘 Usage API의 새 버전을 출시합니다. 이번 업데이트에서는 API를 새로운 시스템으로 이전하여 안정성을 더욱 높이고자 합니다.

  Usage API 내 제품명 중 Historical PowerTrack Subscription이 Historical PowerTrack 2.0으로 변경되어, 다른 사용자 대상 문구와 제품 이름과의 일관성이 유지됩니다. 이번 업데이트 이후에도 고객은 동일한 형식의 정확히 동일한 데이터를 받게 됩니다. 이 변경 사항은 products.type JSON 객체에서 확인할 수 있습니다.

  문의 사항이 있으시면 담당 계정 매니저에게 연락해 주시기 바랍니다.
</Update>

  ### 인용 페이로드 렌더링 변경
