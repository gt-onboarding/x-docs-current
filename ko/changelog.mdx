  ### DM 이벤트 동작 변경

  오늘 X API v2 DM 이벤트 동작을 변경했습니다. 이번 변경으로 `participant_ids` 배열은 ParticipantsJoin 및 ParticipantsLeave 이벤트에만 포함되며, 이벤트 시점에 참여하거나 나간 참가자만 나열합니다. MessageCreate 이벤트에는 더 이상 `participant_ids` 배열이 포함되지 않습니다. 자세한 내용은 [여기](https://devcommunity.x.com/t/upcoming-change-to-x-api-dm-events-endpoints-behavior/241841)를 참조하세요.

<Update label="Aug 3, 2022" description="X API v2">
  ### 필터링 연산자 별칭 추가

  검색 및 필터링 스트림 엔드포인트(Enterprise, Premium, v2 전반)에 필터링/쿼리 연산자 “별칭(alias)”이 추가되었습니다. 많은 경우 연산자는 상태(statuses)가 아닌 게시물을 참조합니다. 예를 들어 `"in_reply_to_status_id"`의 별칭인 `"in_reply_to_tweet_id"`가 이에 해당합니다. 다른 경우에는 새 연산자 별칭이 수행되는 매칭을 더 명확하게 해 줍니다. 예를 들어 게시물에는 비디오 첨부가 하나만 있을 수 있으므로 `"has:videos"`에 대해 `"has:video_link"`라는 새 별칭이 제공됩니다. 더 많은 예시는 [v2 filtered stream operator table](https://developer.x.com/en/docs/twitter-api/tweets/filtered-stream/integrate/build-a-rule#list)을 확인하세요.
</Update>

  ### OAuth 2.0 인증 지원

  오늘부터 모든 개발자는 developer portal에서 인증 방식으로 OAuth 2.0을 선택해 OAuth 2.0으로 인증할 수 있습니다. 이번 릴리스의 일부로, 기밀(Confidential) 및 공개(Public) 클라이언트 모두에 대한 지원과 이 인증 방식을 사용할 수 있는 모든 관련 v2 엔드포인트 지원을 추가했습니다.

  OAuth 2.0은 애플리케이션의 scope를 개발자가 더 세밀하게 제어할 수 있게 하고, 여러 기기에 걸친 인가 플로를 개선해 주는 업계 표준 인가 프로토콜입니다. 다시 말해, X 이용자를 위한 애플리케이션을 구축하는 개발자는 자신의 앱이 사용자에게 요청하는 정보에 대해 더 많은 제어 권한을 가지게 되며, 이를 통해 최종 사용자에게 실제로 필요한 데이터와 정보만 요청하면 됩니다.

  이 최신 인가 프로토콜을 통해, 여러분은 앱 인가를 위한 동의 플로를 더 간소화된 형태로 최종 사용자에게 제공할 수 있으며, 이 플로에는 여러분이 요청한 특정 scope만 표시됩니다.

  **주요 플랫폼 업데이트: X API v2가 기본 버전으로 전환, 새로운 Essential 및 Elevated 액세스, 정책 변경**

  * API에 빠르게 시작할 수 있는 무료 Essential 액세스를 제공하고, 사용 사례 승인을 받은 개발자에게는 무료 Elevated 액세스를 제공합니다. 이러한 액세스 레벨에 대한 더 자세한 내용은 ‘About the X API’ 페이지에서 확인할 수 있습니다.

    * 우리는 여러분이 X를 더 좋고, 더 건강하게 만들며, 공적 대화를 확장하는 도구와 제품을 구축할 수 있도록 플랫폼을 개방하고자 합니다. [우리가 여러분이 만들어 주기를 바라는 솔루션 목록을 확인해 보세요](https://developer.x.com/en/docs/twitter-api/what-to-build).

    * 구체적으로, X의 핵심 기능을 포함해 X 경험 복제를 제한하던 조항과, 많은 수의 사용자 토큰을 보유하기 위해 별도 승인을 요구하던 조항을 제거했습니다.

    * X에서 사람들을 돕는 솔루션을 구축하려면, 개발자가 종종 X에 이미 제공되어 있는 일부 기능을 구축(또는 복제)해야 한다는 것을 잘 알고 있습니다. Developer Policy의 이러한 변경은 개발자 생태계에 명확성을 제공하고, 개발자가 더 쉽게 빌드하고 혁신하며 공적 대화에 영향을 미칠 수 있는 개방형 API 플랫폼을 제공하기 위한 것입니다.

  ### Academic Research 제품 트랙 소개

  오늘 우리는 새로운 [Academic Research 제품 트랙](/ko/resources/fundamentals/projects)을 [X API v2](/ko/x-api/introduction)에 도입합니다. 이 업데이트는 학술 연구자를 위한 새로운 신청 프로세스를 도입하며, 승인을 받은 경우 Academic Research [프로젝트](/ko/resources/fundamentals/projects)가 제공되어 더 높은 수준의 접근 권한과 고급 기능을 사용할 수 있게 됩니다.

  * Standard 제품 트랙의 Basic 액세스 레벨에서 제공되는 512자 한도에서 증가한, 최대 1,024자의 쿼리 및 규칙 문자 한도

<Update label="Dec 18, 2020" description="v2 X API용 대화 답글 설정 게시물 필드">
  ### Reply Settings 필드 추가

  오늘 우리는 게시물 객체에 `reply_settings`라는 새로운 필드를 출시합니다. 이 필드는 게시물 조회, 답글 숨기기, 최근 검색, 샘플 스트림, filtered stream, 팔로우 조회, 사용자 게시물 타임라인, 사용자 언급 타임라인 엔드포인트 등 게시물을 반환하는 모든 X API v2 엔드포인트에서 사용 가능하며, 어떤 엔드포인트에서든 게시물 객체를 [expansions](/ko/x-api/fundamentals/expansions) 매개변수로 확장할 때도 사용할 수 있습니다.

  이 필드는 게시물 작성자가 다른 사람들이 해당 게시물에 어떻게 답글을 달 수 있도록 설정했는지를 나타내며, 모두 허용할지, 게시물에서 언급한 사람만 허용할지, 아니면 자신이 팔로잉하는 계정만 허용할지를 보여줍니다. `reply_settings` 필드에서 반환되는 값에는 `everyone`, `mentionedUsers`, `following`이 포함됩니다. 게시물에 대해 별도의 대화 제어가 설정되지 않은 경우에도 이 필드는 게시물 객체에 `everyone` 값으로 포함됩니다.
</Update>

  ### Labs v2 엔드포인트 출시

  오늘, 게시물 및 사용자, 최신 검색(Recent Search), 답글 숨기기(Hide Replies) 엔드포인트를 Labs v2에서 제공합니다. Labs v1 엔드포인트는 사용 중단되며, 이 초기 릴리스로부터 90일 후 완전히 폐기될 예정입니다. 이 기간 동안 v1 문서도 계속 제공됩니다. 이 버전 관리 전략에 대한 의견은 [피드백 채널](https://twitterdevfeedback.uservoice.com/)을 통해 전달하실 수 있습니다.

  게시물 및 사용자에 대해 다음과 같은 변경 사항을 릴리스합니다:

  * 쿼리 파라미터로 [fields](/ko/x-api/fundamentals/fields) 도입

  * 쿼리 파라미터로 사용되던 [formats](https://developer.x.com/en/docs/labs#payload) 제거

  * 게시물 및 사용자에서 단일 ID [조회](https://developer.x.com/en/docs/labs/overview/versioning/migration-guide#id-lookup-path)를 위한 경로 변수(path variable) 추가

  * GET /users에서 단일 및 다중 사용자 이름 [조회](/ko/x-api/migrate/overview#username-lookup-path)를 위한 경로 및 쿼리 파라미터 조정

  * 필드 이름 `stats` 를 `public_metrics` 로 변경

  기타 변경 사항은 다음과 같습니다:

  * 사용자에서 `most_recent_tweet_id` 확장(expansion) 제거

  * 게시물의 `source` 필드에서 HTML 태그를 제거하여 해당 필드를 더 쉽게 파싱할 수 있도록 함

  최신 검색(Recent Search)의 핵심 검색 기능, 페이지네이션 및 기타 기능은 그대로 유지됩니다.

  v1에서 v2로 업데이트할 때 참고할 수 있는 [마이그레이션 자료](/ko/x-api/migrate/overview)를 준비해 두었습니다.

  이 변경 사항에 대해 더 자세한 내용은 [포럼 공지](https://devcommunity.x.com/t/releasing-a-new-version-of-labs-endpoints/134219)에서 확인하실 수 있습니다.

<Update label="Jan 21, 2020" description="X API: Standard v1.1">
  ### 액세스 토큰 관리 변경

  오늘, [developer.x.com의 애플리케이션 Dashboard](https://developer.x.com/content/developer-twitter/en/apps)에서 액세스 토큰과 액세스 토큰 시크릿이 표시되고 관리되는 방식을 변경합니다.

  Search API에서는 모든 ‘is:’ 및 ‘has:’ 연산자를 키워드나 해시태그와 같은 단독 검색 연산자와 함께 사용해야 한다는 점을 기억하십시오.

  이 연산자에 대해서는 [문서](/ko/x-api/enterprise-gnip-2.0/fundamentals/rules-filtering)에서 더 자세히 알아볼 수 있습니다.

  ### 인용 페이로드 렌더링 변경 사항

  오늘 인용 페이로드에서 URL이 렌더링되는 방식을 변경합니다.

  인용 게시물을 참조할 수 있도록 &quot;quoted&#95;status&#95;permalink&quot;라는 새 엔티티를 추가하고, 인용 게시물의 &quot;text&quot; 필드에서 t.co 링크를 제거할 예정입니다.

  자세한 내용은 [포럼 글](https://devcommunity.x.com/t/updating-how-urls-are-rendered-in-the-quote-tweet-payload/105473)과 [업데이트된 문서](https://devcommunity.x.com/t/upcoming-changes-to-the-developer-platform/104603)를 읽어보세요.
