<div id="using-and-generating-an-app-only-bearer-token">
  ### 앱 전용 Bearer Token 사용 및 생성
</div>

Bearer Token은 개발자가 X API를 보다 안전하게 사용할 수 있는 진입점을 제공하며, OAuth 2.0의 핵심 기능 중 하나입니다. 

Bearer Token을 사용하는 인증은 애플리케이션 전용(application-only) 인증이라고도 합니다. Bearer Token은 `curl` 명령과 같은 스크립트를 사용해 생성하는, 형식이 정해져 있지 않은 바이트 배열입니다. Bearer Token은 또한 developer portal의 App 설정 화면에 있는 키와 토큰 섹션에서 발급받을 수도 있습니다. 이 기능에 대한 더 자세한 내용은 [OAuth의 공식 문서](https://oauth.net/2/bearer-tokens/)에서 확인할 수 있습니다.

<div id="when-are-they-used">
  #### 언제 사용되나요?
</div>

Bearer Token을 사용해야 하는 제품은 다음과 같습니다:

* [Engagement API](/ko/x-api/enterprise-gnip-2.0/fundamentals/engagement-api)
* [Account Activity API](/ko/x-api/enterprise-gnip-2.0/fundamentals/account-activity)
* v2 및 Labs 엔드포인트 등 OAuth 2.0 Bearer Token 인증을 사용하는 기타 API

<div id="prerequisites">
  #### 사전 준비 사항
</div>

[개발자 계정](https://developer.x.com/en/portal/petition/essential/basic-info)에 가입하고 [X App](/ko/resources/fundamentals/developer-apps)을 생성해야 합니다. 이를 완료한 후에는 [developer portal](/ko/resources/fundamentals/developer-portal)에서 API 키를 발급받아야 합니다. 아래 단계를 따르세요.

1. developer.x.com에서 X 계정으로 로그인합니다.
2. [X App dashboard](https://developer.x.com/content/developer-twitter/en/apps)로 이동한 후 액세스 토큰을 생성하려는 X App을 엽니다.
3. &quot;keys and tokens&quot; 페이지로 이동합니다.
4. 이 페이지에서 API 키, 사용자 액세스 토큰, 그리고 Bearer Token을 확인할 수 있습니다.

<div id="how-to-generate-a-bearer-token">
  ### Bearer Token을 생성하는 방법
</div>

앱의 Bearer Token은 나머지 &quot;키와 토큰(Keys and Tokens)&quot; 정보와 함께 확인할 수 있습니다.

[X 앱](/ko/resources/fundamentals/developer-apps)에서 미리 발급받은 다음 consumer API 키 값으로 변경한 후, 아래 cURL 요청을 명령줄에 복사해 실행하십시오. 이 페이지에서 사용된 consumer API 키는 사용이 중지되었으며 실제 요청에는 사용할 수 없습니다.

* **API key** `<API key>` 예: `xvz1evFS4wEEPTGEFPHBog`
* **API secret key** `<API secret key>` 예: `L8qq9PZyRg6ieKGEKhZolGC0vJWLw8iEJ88DRdyOg`

  ```bash
  curl -u "$API_KEY:$API_SECRET_KEY" \
  --data 'grant_type=client_credentials' \
  'https://api.x.com/oauth2/token'
  ```

다음은 API 키를 입력했을 때 curl 요청이 어떻게 보여야 하는지에 대한 예시입니다.

```bash
   curl -u 'xvz1evFS4wEEPTGEFPHBog:L8qq9PZyRg6ieKGEKhZolGC0vJWLw8iEJ88DRdyOg' \
  --data 'grant_type=client_credentials' \
  'https://api.x.com/oauth2/token'
```

응답은 다음과 같습니다. 참고로 아래 Bearer Token은 사용이 중단된 토큰입니다:

`{"token_type":"bearer","access_token":"AAAAAAAAAAAAAAAAAAAAAMLheAAAAAAA0%2BuSeid%2BULvsea4JtiGRiSDSJSI%3DEUifiRBkKG5E2XzMDjRfl76ZC9Ub0wnz4XsNiRVBChTYbJcE3F"}`

OAuth 2.0로 리소스에 인증할 때 사용되던 Bearer Token은 다음과 같습니다:

`AAAAAAAAAAAAAAAAAAAAAMLheAAAAAAA0%2BuSeid%2BULvsea4JtiGRiSDSJSI%3DEUifiRBkKG5E2XzMDjRfl76ZC9Ub0wnz4XsNiRVBChTYbJcE3F`
