<div id="general">
  ## 일반
</div>

<AccordionGroup>
  <Accordion title="OAuth란 무엇인가요?">
    OAuth는 사용자가 비밀번호를 공유하지 않고도 애플리케이션이 사용자를 대신해 작업하도록 승인할 수 있게 해 주는 인증 프로토콜입니다. 자세한 내용은 [oauth.net](http://oauth.net/)에서 확인할 수 있습니다.
  </Accordion>

  <Accordion title="액세스 토큰은 어떻게 생성하나요?">
    액세스 토큰을 생성하려면 [X 앱](/ko/resources/fundamentals/developer-apps)이 있어야 합니다. 액세스 토큰에 대해 더 알아보려면 [여기](/ko/resources/fundamentals/authentication#oauth-1-0a-2)를 참고하세요. 
  </Accordion>

  <Accordion title="X 앱은 어떻게 생성하나요?">
    [X 앱](/ko/resources/fundamentals/developer-apps)을 생성하려면 먼저 [개발자 계정](/ko/resources/fundamentals/developer-portal)이 있어야 합니다. [여기](https://developer.x.com/en/portal/petition/essential/basic-info)에서 신청할 수 있습니다.
  </Accordion>

  <Accordion title="이미 X 앱이 있는 경우, 해당 앱을 어떻게 보고 편집하나요?">
    developer.x.com에서 X 계정으로 로그인한 상태라면 [X app dashboard](https://developer.x.com/content/developer-twitter/en/apps)에서 앱을 조회하고 수정할 수 있습니다.
  </Accordion>
</AccordionGroup>

<div id="technical">
  ## Technical
</div>

<AccordionGroup>
  <Accordion title="액세스 토큰은 얼마나 오래 유효한가요?">
    액세스 토큰은 명시적으로 만료되지 않습니다. 사용자가 X 계정 설정에서 애플리케이션을 명시적으로 해지(revoke)하거나, X가 애플리케이션을 정지(suspend)하는 경우 액세스 토큰은 무효화됩니다. 애플리케이션이 정지된 경우 [X app](/ko/resources/fundamentals/developer-apps) 대시보드에 정지되었다는 안내가 표시됩니다.
  </Accordion>

  <Accordion title="액세스 토큰이 무효가 되면 어떻게 하나요?">
    사용자의 액세스 토큰은 언제든지 무효가 &#95;될 수 있다&#95;고 가정해야 합니다. 이런 상황이 발생하면, 사용자에게 애플리케이션을 다시 승인하도록 요청하세요. 이 상황을 원활하게 처리하는 것은 좋은 사용자 경험을 제공하는 데 중요합니다.
  </Accordion>

  <Accordion title="애플리케이션 등록 페이지에서 read/write 액세스를 묻습니다. write는 무엇을 의미하나요?">
    많은 사용자는 애플리케이션이 자신의 정보를 읽는 것은 신뢰하지만, 반드시 사용자 프로필 정보를 변경하거나 새 게시물을 올리는 것까지 허용하지는 않습니다. X API를 통해 정보를 업데이트하는 작업(예: 이름, 위치를 변경하거나 새 게시물을 추가하는 것)은 HTTP POST가 필요합니다. HTTP POST가 필요한 모든 API 메서드는 write 메서드로 간주되며, read &amp; write 액세스가 필요합니다.
  </Accordion>
</AccordionGroup>
