<div id="general">
  ## 일반
</div>

<AccordionGroup>
  <Accordion title="OAuth란 무엇인가요?">
    OAuth는 사용자가 비밀번호를 공유하지 않고도 애플리케이션이 자신을 대신해 동작하도록 승인할 수 있게 해 주는 인증 프로토콜입니다. 자세한 내용은 [oauth.net](http://oauth.net/)에서 확인하세요.
  </Accordion>

  <Accordion title="액세스 토큰은 어떻게 생성하나요?">
    액세스 토큰을 생성하려면 [X 앱](/ko/resources/fundamentals/developer-apps)이 있어야 합니다. 액세스 토큰에 대해 더 알아보려면 [여기](/ko/resources/fundamentals/authentication#oauth-1-0a-2)를 참고하세요. 
  </Accordion>

  <Accordion title="X 앱은 어떻게 생성하나요?">
    [X 앱](/ko/resources/fundamentals/developer-apps)을 생성하려면 [개발자 계정](/ko/resources/fundamentals/developer-portal)이 필요합니다. [여기](https://developer.x.com/en/portal/petition/essential/basic-info)에서 가입할 수 있습니다.
  </Accordion>

  <Accordion title="이미 X 앱이 있는 경우, 해당 앱을 어떻게 보고 수정하나요?">
    developer.x.com에서 X 계정으로 로그인되어 있다면, [X 앱 대시보드](https://developer.x.com/content/developer-twitter/en/apps)에서 앱을 확인하고 수정할 수 있습니다.
  </Accordion>
</AccordionGroup>

<div id="technical">
  ## Technical
</div>

<AccordionGroup>
  <Accordion title="액세스 토큰은 얼마나 동안 유효한가요?">
    액세스 토큰은 별도로 만료되도록 설정되어 있지 않습니다. 사용자가 자신의 X 계정 설정에서 애플리케이션을 명시적으로 해지하거나, X가 애플리케이션을 정지시키는 경우 액세스 토큰은 무효화됩니다. 애플리케이션이 정지된 경우 [X 앱](/ko/resources/fundamentals/developer-apps) 대시보드에 정지되었다는 알림이 표시됩니다.
  </Accordion>

  <Accordion title="액세스 토큰이 유효하지 않게 되면 어떻게 하나요?">
    사용자의 액세스 토큰은 언제든지 무효화될 *수* 있다고 가정해야 합니다. 이런 상황이 발생하면 사용자에게 애플리케이션을 다시 승인하도록 요청해야 합니다. 이 상황을 원활하게 처리하는 것은 좋은 사용자 경험을 위해 중요합니다.
  </Accordion>

  <Accordion title="애플리케이션 등록 페이지에서 읽기/쓰기 권한을 묻습니다. 쓰기는 무엇을 의미하나요?">
    많은 사용자는 애플리케이션이 자신의 정보를 읽는 것은 신뢰하지만, 사용자 프로필 정보를 변경하거나 새 게시물을 올리는 것까지는 원하지 않을 수 있습니다. X API를 통해 정보를 업데이트하는 것 — 이름, 위치를 변경하거나 새 게시물을 추가하는 것 — 은 HTTP POST가 필요합니다. HTTP POST가 필요한 모든 API 메서드는 쓰기 메서드로 간주되며, 읽기 및 쓰기 권한이 필요합니다.
  </Accordion>
</AccordionGroup>
