<div id="post-oauthrequest_token">
  ### POST oauth/request&#95;token
</div>

Consumer 애플리케이션이 사용자 인가를 요청하기 위한 OAuth Request Token을 얻을 수 있도록 합니다. 이 메서드는 [OAuth 1.0 인증 플로우](http://oauth.net/core/1.0/#anchor9)의 [섹션 6.1](https://oauth.net/core/1.0/#auth_step1)을 충족합니다.

**모든 OAuth 인가 단계에서 HTTPS 사용이 필수입니다.**

**사용 참고 사항:** `oauth_nonce`에는 ASCII 문자만 허용됩니다.

**리소스 URL[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth/request_token`

**리소스 정보[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Response formats | JSON |
| Requires authentication? | No  |
| Rate limited? | Yes |

**파라미터[](#parameters "Permalink to this headline")**

| Name | Required | Description | Example |
| :--- | :--- | :--- | :--- |
| oauth&#95;callback | required | OAuth 1.0a 준수를 위해 이 파라미터는 **필수**입니다. 여기에서 지정하는 값은 사용자가 애플리케이션에 자신의 계정 접근을 승인했을 때 리디렉션되는 URL로 사용됩니다. out-of-band PIN 모드에서는 이를 `oob`로 설정하십시오. 이는 데스크톱/모바일 애플리케이션에서 사용할 커스텀 콜백을 지정하는 방법이기도 합니다. 사전 등록된 콜백이 있는지와 관계없이, 이 단계에서는 항상 `oauth_callback`을 전송해야 합니다.<br /><br />이 엔드포인트와 함께 사용되는 모든 콜백 URL은 developer.x.com의 앱 설정에서 반드시 구성되어 있어야 합니다* | `http://themattharris.local/auth.php` `twitterclient://callback` |
| x&#95;auth&#95;access&#95;type | optional | 애플리케이션이 사용자 계정에 요청하는 접근 수준을 재정의합니다. 지원되는 값은 `read` 또는 `write`입니다. 이 파라미터는 개발자가 읽기/쓰기 애플리케이션을 등록하되, 필요할 때에는 읽기 전용 접근만 요청할 수 있도록 하기 위한 것입니다. |     |

콜백 URL을 승인하는 방법에 대한 자세한 내용은 [이 페이지](/ko/resources/fundamentals/developer-apps#callback-urls)를 참고하십시오.

**주의 사항** - developer.x.com에서 X 계정으로 로그인한 상태라면, 기존 [X 앱](/ko/resources/fundamentals/developer-apps)을 [X 앱 대시보드](https://developer.x.com/en/apps)를 통해 조회하고 수정할 수 있습니다.

**예시 요청[](#example-request "Permalink to this headline")**

요청 URL: `POST https://api.x.com/oauth/request_token`

요청 POST Body: *N/A*

Authorization 헤더: `OAuth oauth_nonce="K7ny27JTpKVsTgdyLdDfmQQWVLERj2zAK5BslRsqyw", oauth_callback="http%3A%2F%2Fmyapp.com%3A3005%2Ftwitter%2Fprocess_callback", oauth_signature_method="HMAC-SHA1", oauth_timestamp="1300228849", oauth_consumer_key="OqEqJeafRSF11jBMStrZz", oauth_signature="Pc%2BMLdv028fxCErFyi8KXFM%2BddU%3D", oauth_version="1.0"`

응답: `oauth_token=Z6eEdO8MOmk394WozF5oKyuAv855l4Mlqo7hhlSLik&oauth_token_secret=Kd75W4OQfb2oJTV0vzGzeXftVAwgMnEK9MumzYcM&oauth_callback_confirmed=true`

<div id="get-oauthauthorize">
  ### GET oauth/authorize
</div>

Consumer 애플리케이션이 OAuth Request Token을 사용하여 사용자 승인을 요청할 수 있도록 합니다. 이 메서드는 [OAuth 1.0 인증 플로우](http://oauth.net/core/1.0/#anchor9)의 [섹션 6.2](http://oauth.net/core/1.0/#auth_step2)를 충족합니다. 데스크톱 애플리케이션은 반드시 이 메서드를 사용해야 하며, [GET oauth / authenticate](/ko/resources/fundamentals/authentication/api-reference#get-oauth-authenticate)는 사용할 수 없습니다.

**사용 참고:** `oauth_callback`은 이 메서드로 전송되지 않으므로, 대신 [POST oauth / request&#95;token](/ko/resources/fundamentals/authentication/api-reference#post-oauth-request-token)에 제공해야 합니다.

**리소스 URL[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth/authorize`

**리소스 정보[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Response formats | JSON |
| Requires authentication? | 예 |
| Rate limited? | 예 |

**파라미터[](#parameters "Permalink to this headline")**

|     |     |     |     |     |
| :--- | :--- | :--- | :--- | :--- |
| Name | Required | Description | Default Value | Example |
| force&#95;login | optional | 올바른 사용자 계정이 승인되도록 사용자가 자격 증명을 다시 입력하도록 강제합니다. |     |     |
| screen&#95;name | optional | OAuth 로그인 화면의 사용자 이름 입력란을 지정된 값으로 미리 채웁니다. |     |     |

**예시 요청[](#example-request "Permalink to this headline")**

웹 브라우저에서 `oauth/authorize` 단계로 사용자를 보내면서 oauth&#95;token 파라미터를 포함합니다:
`https://api.x.com/oauth/authorize?oauth_token=Z6eEdO8MOmk394WozF5oKyuAv855l4Mlqo7hhlSLik`

<div id="get-oauthauthenticate">
  ### GET oauth/authenticate
</div>

Consumer 앱이 OAuth `request_token`을 사용하여 사용자 인가를 요청할 수 있도록 합니다.

이 메서드는 callback 인증 플로우를 사용하는 애플리케이션에 대해 [OAuth 1.0 authentication flow](http://oauth.net/core/1.0/#anchor9)의 [Section 6.2](http://oauth.net/core/1.0/#auth_step2)를 대체합니다. `force_login` 파라미터가 `true`로 설정되지 않은 경우, 이 메서드는 현재 로그인된 사용자를 액세스 인가 대상 계정으로 사용합니다.

이 메서드는, 사용자가 이미 애플리케이션에 권한을 부여한 경우 사용자가 애플리케이션을 다시 승인할 필요 없이 리디렉션이 자동으로 수행된다는 점에서 [GET oauth / authorize](/ko/resources/fundamentals/authentication/api-reference#get-oauth-authorize)와 다릅니다. 이 동작을 사용하려면 [application record](https://developer.x.com/apps)에서 *Use Sign in with X* 설정을 활성화해야 합니다.

**Resource URL[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth/authenticate`

**Resource Information[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Response formats | JSON |
| Requires authentication? | Yes |
| Rate limited? | Yes |

**Parameters[](#parameters "Permalink to this headline")**

|     |     |     |     |     |
| :--- | :--- | :--- | :--- | :--- |
| Name | Required | Description | Default Value | Example |
| force&#95;login | optional | 올바른 사용자 계정이 인가되도록 사용자가 자격 증명을 반드시 입력하도록 강제합니다. |     | *true* |
| screen&#95;name | optional | OAuth 로그인 화면의 사용자 이름 입력란을 지정한 값으로 미리 채웁니다. |     |     |

**Example request[](#example-request "Permalink to this headline")**

웹 브라우저에서 `oauth/authenticate` 단계로 사용자를 보내면서 `oauth_token` 파라미터를 포함합니다:
`https://api.x.com/oauth/authenticate?oauth_token=Z6eEdO8MOmk394WozF5oKyuAv855l4Mlqo7hhlSLik`

<div id="post-oauthaccess_token">
  ### POST oauth/access&#95;token
</div>

Consumer 앱이 OAuth Request Token을 OAuth Access Token으로 교환할 수 있도록 합니다. 이 메서드는 [OAuth 1.0 인증 플로우](http://oauth.net/core/1.0/#anchor9)의 [섹션 6.3](http://oauth.net/core/1.0/#auth_step3)에 해당합니다.

**Resource URL[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth/access_token`

**Resource Information[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Response formats | JSON |
| Requires authentication? | Yes |
| Rate limited? | Yes |

**Parameters[](#parameters "Permalink to this headline")**

|     |     |     |     |     |
| :--- | :--- | :--- | :--- | :--- |
| Name | Required | Description | Default Value | Example |
| oauth&#95;token | required | 여기의 `oauth_token` 값은 request&#95;token 단계에서 반환된 `oauth_token`과 동일해야 합니다. |     |     |
| oauth&#95;verifier | required | OAuth 웹 플로우를 사용하는 경우 이 매개변수를 콜백 URL에서 반환된 *oauth&#95;verifier* 값으로 설정합니다. out-of-band OAuth를 사용하는 경우 이 값을 핀 코드로 설정합니다. OAuth 1.0a 준수를 위해 이 매개변수는 **필수**입니다. OAuth 1.0a는 엄격하게 적용되며, &#95;oauth&#95;verifier&#95;를 사용하지 않는 애플리케이션은 OAuth 플로우를 완료하지 못합니다. |     |     |

**Example request[](#example-request "Permalink to this headline")**

`POST https://api.x.com/oauth/access_token?oauth_token=qLBVyoAAAAAAx72QAAATZxQWU6P&oauth_verifier=ghLM8lYmAxDbaqL912RZSRjCCEXKDIzx`

PIN 기반 예시: `POST https://api.x.com/oauth/access_token?oauth_token=9Npq8AAAAAAAx72QBRABZ4DAfY9&oauth_verifier=4868795`

**Example response[](#example-response "Permalink to this headline")**

`oauth_token=6253282-eWudHldSbIaelX7swmsiHImEL4KinwaGloHANdrY&oauth_token_secret=2EEfA6BG5ly3sR3XjE0IBSnlQu4ZrUzPiYTmrkVU&user_id=6253282&screen_name=xapi`

<div id="post-oauthinvalidate_token">
  ### POST oauth/invalidate&#95;token
</div>

등록된 애플리케이션이 클라이언트 자격 증명을 제시하여 발급된 OAuth access&#95;token을 취소(무효화)할 수 있도록 합니다. access&#95;token이 무효화되면, 새 토큰을 생성하려는 시도에서는 다른 액세스 토큰이 발급되며, 무효화된 토큰은 더 이상 사용할 수 없습니다.

**리소스 URL[](#resource-url "Permalink to this headline")**

`https://api.x.com/1.1/oauth/invalidate_token`

**리소스 정보[](#resource-information "Permalink to this headline")**

|               |                                |
| :------------ | :----------------------------- |
| 응답 형식         | JSON                           |
| 인증 필요?        | 예 - 무효화하려는 액세스 토큰이 있는 사용자 컨텍스트 |
| 레이트 리밋 적용 여부? | 예                              |

**요청 예시[](#example-request "Permalink to this headline")**

```bash
        curl --request POST
          --url 'https://api.x.com/1.1/oauth/invalidate_token.json'
          --header 'authorization: OAuth oauth_consumer_key="CLIENT_KEY",
         oauth_nonce="AUTO_GENERATED_NONCE", oauth_signature="AUTO_GENERATED_SIGNATURE",
         oauth_signature_method="HMAC-SHA1", oauth_timestamp="AUTO_GENERATED_TIMESTAMP",
         oauth_token="ACCESS_TOKEN", oauth_version="1.0"'
```

**예시 응답[](#example-response "이 제목에 대한 고유 링크")**

```bash
        HTTP/1.1 200 OK
        Content-Type: application/json; charset=utf-8
        Content-Length: 127
        ...

        {"access_token":"ACCESS_TOKEN"}
```

**토큰이 무효화된 후 반환되는 오류 응답 예시[](#example-error-response-after-token-has-been-invalidated "Permalink to this headline")**

```bash
        HTTP/1.1 401 Authorization Required
        ...

        {"errors": [{
          "code": 89,
          "message": "Invalid or expired token."}
        ]}
```

<div id="post-oauth2token">
  ### POST oauth2/token
</div>

등록된 애플리케이션이 OAuth 2 Bearer Token을 발급받을 수 있도록 합니다. 이렇게 발급받은 토큰은 사용자 컨텍스트 없이 애플리케이션 자체를 대신해 API 요청을 보내는 데 사용할 수 있습니다. 이를 [애플리케이션 전용 인증](/ko/resources/fundamentals/authentication/oauth-2-0/application-only)이라고 합니다.

Bearer Token은 oauth2/invalidate&#95;token을 사용해 무효화할 수 있습니다. Bearer Token이 한 번 무효화되면, 새로 생성하려는 시도는 서로 다른 Bearer Token을 반환하며 이전 토큰은 더 이상 사용할 수 없습니다.

하나의 애플리케이션에 대해 동시에 존재할 수 있는 Bearer Token은 하나뿐이며, 이 메서드에 반복해서 요청을 보내면 무효화되기 전까지는 이미 존재하는 동일한 토큰이 반환됩니다.

성공적인 응답에는 발급된 Bearer Token을 나타내는 JSON 구조가 포함됩니다.

이 메서드로 받은 토큰은 캐시해 두어야 합니다. 너무 자주 요청을 시도하면, 요청은 코드 99와 함께 HTTP 403으로 거절됩니다.

**리소스 URL[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth2/token`

**리소스 정보[](#resource-information "Permalink to this headline")**

|                          |                                                                         |
| :----------------------- | :---------------------------------------------------------------------- |
| Response formats         | JSON                                                                    |
| Requires authentication? | Yes - API key를 username으로, API key secret을 password로 사용하는 Basic auth 필요 |
| Rate limited?            | Yes                                                                     |

**파라미터[](#parameters "Permalink to this headline")**

|                |          |                                                                                                                                                                      |               |                          |
| :------------- | :------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :------------ | :----------------------- |
| Name           | Required | Description                                                                                                                                                          | Default Value | Example                  |
| grant&#95;type | required | 애플리케이션이 요청하는 grant 유형을 지정합니다. 현재는 *client&#95;credentials*만 허용됩니다. 자세한 내용은 [애플리케이션 전용 인증](/ko/resources/fundamentals/authentication/oauth-2-0/application-only)을 참고하세요. |               | *client&#95;credentials* |

**요청 예시[](#example-request "Permalink to this headline")**

```bash
    POST /oauth2/token HTTP/1.1
    Host: api.x.com
    User-Agent: My X App v1.0.23
    Authorization: Basic eHZ6MWV2R ... o4OERSZHlPZw==
    Content-Type: application/x-www-form-urlencoded;charset=UTF-8
    Content-Length: 29
    Accept-Encoding: gzip

    grant_type=client_credentials
```

**예시 응답:**

```bash
    HTTP/1.1 200 OK
    Status: 200 OK
    Content-Type: application/json; charset=utf-8
    ...
    Content-Encoding: gzip
    Content-Length: 140

    {"token_type":"bearer","access_token":"AAAA%2FAAA%3DAAAAAAAA"}
```

<div id="post-oauth2invalidate_token">
  ### POST oauth2/invalidate&#95;token
</div>

등록된 애플리케이션이 클라이언트 자격 증명을 제시해 발급된 OAuth 2.0 Bearer Token을 취소할 수 있도록 합니다. Bearer Token이 무효화되면 새로 생성할 때는 다른 Bearer Token이 발급되며, 무효화된 토큰은 더 이상 사용할 수 없습니다.

성공한 응답에는 취소된 Bearer Token을 설명하는 JSON 구조가 포함됩니다.

**리소스 URL[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth2/invalidate_token`

**리소스 정보[](#resource-information "Permalink to this headline")**

|                          |                                                                                                                                   |
| :----------------------- | :-------------------------------------------------------------------------------------------------------------------------------- |
| Response formats         | JSON                                                                                                                              |
| Requires authentication? | 예 - 애플리케이션의 consumer API 키와 애플리케이션 소유자의 액세스 토큰 및 액세스 토큰 시크릿을 사용하는 [OAuth 1.0a](/ko/resources/fundamentals/authentication/oauth-1-0a) |
| Rate limited?            | 예                                                                                                                                 |

**매개변수[](#parameters "Permalink to this headline")**

| Name             | Required | Description                |
| :--------------- | :------- | :------------------------- |
| access&#95;token | required | 무효화하려는 Bearer Token의 값입니다. |

**요청 예시[](#example-request "Permalink to this headline")**

```
        curl --request POST
          --url 'https://api.x.com/oauth2/invalidate_token?access_token=AAAA%2FAAA%3DAAAAAAAA'
          --header 'authorization: OAuth oauth_consumer_key="CLIENT_KEY",
         oauth_nonce="AUTO_GENERATED_NONCE", oauth_signature="AUTO_GENERATED_SIGNATURE",
         oauth_signature_method="HMAC-SHA1", oauth_timestamp="AUTO_GENERATED_TIMESTAMP",
         oauth_token="ACCESS_TOKEN", oauth_version="1.0"'
```

**예시 응답[](#example-response "이 제목에 대한 고유 링크")**

```
         Status: 200 OK
         Content-Type: application/json; charset=utf-8
         Content-Length: 135
         ...
       {
        "access_token": "AAAA%2FAAA%3DAAAAAAAA"
        }
```
