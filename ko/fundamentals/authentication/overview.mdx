X API는 방대한 양의 데이터를 처리합니다. 이 데이터가 개발자와 사용자 모두를 위해 안전하게 보호되도록 인증을 사용합니다. 인증에는 몇 가지 방식이 있으며, 각각 아래에 나열되어 있습니다.

대부분의 개발자는 클라이언트 라이브러리가 이러한 어려운 부분을 자동으로 처리하기 때문에, 인증과 관련된 복잡한 부분을 직접 다룰 필요가 없습니다.

사용 가능한 클라이언트 라이브러리 목록은 [도구 및 라이브러리](/ko/resources/tools-and-libraries) 페이지에서 확인할 수 있습니다.

<Note>
  **참고:**
  앱의 API 키와 앱 전용 액세스 토큰, 그리고 개인 액세스 토큰과 액세스 토큰 시크릿은 [X developer Apps](/ko/resources/fundamentals/developer-apps) 섹션에서 확인할 수 있으며, 이 섹션은 [developer portal](/ko/resources/fundamentals/developer-portal) 내에 있습니다.

  **다른 사용자를 대신해 요청을 전송하려는 경우**, 해당 사용자를 위한 별도의 액세스 토큰 세트를 [3-legged OAuth flow](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)를 사용해 생성한 뒤, 그 사용자의 토큰을 OAuth 1.0a 사용자 컨텍스트 또는 OAuth 2.0 사용자 컨텍스트 요청에 함께 전달해야 합니다.
</Note>
