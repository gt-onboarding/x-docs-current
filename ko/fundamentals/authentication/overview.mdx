X API는 방대한 양의 데이터를 처리합니다. 이 데이터를 개발자와 사용자 모두에게 안전하게 보호하기 위해 인증을 사용합니다. 인증에는 여러 가지 방법이 있으며, 각 방법은 아래에 정리되어 있습니다.

대부분의 개발자는 클라이언트 라이브러리가 이러한 작업을 자동으로 처리하므로, 인증과 관련된 복잡한 부분을 직접 다룰 필요가 없습니다.

사용 가능한 클라이언트 라이브러리 목록은 [도구 및 라이브러리](/ko/resources/tools-and-libraries) 페이지에서 확인할 수 있습니다.

<Note>
  **참고:**
  앱의 API 키 및 App only 액세스 토큰, 그리고 개인 액세스 토큰 및 Access Token Secret은 [developer portal](/ko/resources/fundamentals/developer-portal)에서 확인할 수 있는 [X developer Apps](/ko/resources/fundamentals/developer-apps) 섹션에서 발급받을 수 있습니다.

  **다른 사용자를 대신해 요청을 보내려는 경우**, 해당 사용자를 위한 별도의 액세스 토큰 세트를 [3-legged OAuth 플로우](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)를 사용해 생성한 후, 그 사용자의 토큰을 OAuth 1.0a 사용자 컨텍스트 또는 OAuth 2.0 사용자 컨텍스트 요청에 함께 전달해야 합니다.
</Note>
