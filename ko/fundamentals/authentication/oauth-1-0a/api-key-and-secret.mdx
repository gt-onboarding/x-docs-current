<div id="api-key-and-secret">
  ### API Key and Secret
</div>

API Key와 Secret(Consumer Key와 Secret이라고도 함)은 X API에 액세스하는 데 필요한 가장 기본적인 자격 증명입니다. 이 자격 증명은 X 앱의 사용자 이름과 비밀번호 역할을 하며, X API는 이를 사용해 요청이 어떤 앱에서 오는지 식별합니다. 

이 자격 증명은 [인증 엔드포인트](/ko/resources/fundamentals/authentication/api-reference)를 통해 [사용자 액세스 토큰 및 시크릿](/ko/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret), [Bearer Token](/ko/resources/fundamentals/authentication/oauth-2-0/bearer-tokens)과 같은 추가 자격 증명을 생성하는 데 사용할 수 있습니다. 또한 OAuth 1.0a 사용자 컨텍스트 인증이 필요한 요청을 [인가](/ko/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request)할 때, 액세스 토큰 및 기타 인가 매개변수와 함께 이 자격 증명을 사용해야 합니다. 

<div id="how-to-acquire-an-api-key-and-secret">
  #### API Key 및 Secret 획득 방법
</div>

X API Key 및 Secret을 얻으려면 다음 단계를 따르세요:

1. [X 개발자 계정에 가입합니다](https://developer.x.com/en/apply-for-access)
2. [developer portal](/ko/resources/fundamentals/developer-portal) 내에서 [X App](/ko/resources/fundamentals/developer-apps)을 생성합니다. [X API v2](/ko/x-api/introduction)를 사용하려면 X App을 반드시 [Project](/ko/resources/fundamentals/projects)에 추가해야 합니다.
    

X App을 생성하면 API Key 및 Secret과 함께 Bearer Token이 제공됩니다. 이 자격 증명은 한 번만 표시되므로, 반드시 비밀번호 관리자나 안전한 위치에 저장해 두시기 바랍니다.

[authentication best practices](/ko/resources/fundamentals/authentication/guides/authentication-best-practices) 페이지에서 키와 토큰을 처리하는 추가 권장 사항을 확인할 수 있으며, 여기에는 자격 증명이 유출되었을 때 취해야 할 조치에 대한 상세 내용도 포함되어 있습니다.
 

<div id="how-to-find-and-regenerate-your-api-key-and-secret-after-app-creation">
  #### 앱 생성 후 API Key와 Secret을 찾거나 재발급하는 방법
</div>

이미 앱을 생성했고 API Key와 Secret을 찾아야 하거나 재발급해야 하는 경우, 다음 단계를 따르십시오:

1. developer portal로 이동하십시오
2. 사이드 내비게이션(sidenav)에서 &#39;Projects and Apps&#39; 드롭다운을 펼치십시오
3. 찾거나 재발급하려는 API Key와 Secret이 연결된 앱을 여십시오
4. Keys and tokens 탭으로 이동하십시오

여기에서 앱과 연결된 모든 자격 증명을 확인할 수 있습니다. 
 

<div id="how-to-use-your-api-key-and-secret">
  #### API 키와 Secret을 사용하는 방법
</div>

X Developer Platform을 막 살펴보고 있는 단계라면, 플랫폼에서 어떤 기능을 사용할 수 있는지 확인하기 위해 [도구 또는 라이브러리](/ko/resources/tools-and-libraries)를 사용하는 것을 권장합니다. 이러한 도구들은 인증을 원활하게 처리해 주며, 많은 시간과 불편을 줄여 줄 수 있습니다. 특히 초보 개발자에게는 [Postman 시작하기](/ko/tutorials/postman-getting-started) 또는 [Insomnia](https://insomnia.rest/)를 사용하는 것을 권장합니다. 

요청을 처음부터 직접 구성해 보고 싶다면, [OAuth 1.0a 요청 인증](/ko/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request)에 대한 가이드를 참고하세요.
