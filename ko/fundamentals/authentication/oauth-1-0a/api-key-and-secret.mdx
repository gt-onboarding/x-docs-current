<div id="api-key-and-secret">
  ### API Key and Secret
</div>

API Key와 Secret (Consumer Key와 Secret이라고도 함)은 X API에 접근하기 위해 필요한 가장 기본적인 자격 증명입니다. 이 자격 증명은 X 앱의 사용자 이름과 비밀번호 역할을 하며, X API는 이를 사용해 어떤 앱에서 보낸 요청인지 식별합니다. 

이 자격 증명은 [authentication endpoints](/ko/resources/fundamentals/authentication/api-reference)를 통해 [user Access Tokens and Secrets](/ko/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret), [Bearer Tokens](/ko/resources/fundamentals/authentication/oauth-2-0/bearer-tokens)와 같은 추가 자격 증명을 생성하는 데 사용할 수 있습니다. 또한 OAuth 1.0a 사용자 컨텍스트 인증이 필요한 요청을 [authorize requests](/ko/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request)하기 위해서는 이 자격 증명을 액세스 토큰 및 기타 권한 부여 매개변수와 함께 사용해야 합니다. 

<div id="how-to-acquire-an-api-key-and-secret">
  #### API Key와 Secret을 발급받는 방법
</div>

X API Key와 Secret을 발급받으려면 다음 단계를 따르세요.

1. [X 개발자 계정에 가입하세요](https://developer.x.com/en/apply-for-access)
2. [developer portal](/ko/resources/fundamentals/developer-portal) 내에서 [X App](/ko/resources/fundamentals/developer-apps)을 생성하세요. [X API v2](/ko/x-api/introduction)를 사용하려면, 해당 X App을 반드시 [Project](/ko/resources/fundamentals/projects)에 추가해야 합니다.
    

X App을 생성하면 API Key와 Secret, 그리고 Bearer Token이 함께 발급됩니다. 이 자격 증명은 한 번만 표시되므로 비밀번호 관리자나 안전한 위치에 반드시 저장해 두시기 바랍니다.

키와 토큰 관리 방법, 자격 증명이 유출되었을 때 취해야 할 조치 등에 대한 추가 권장 사항은 [인증 모범 사례(authentication best practices)](/ko/resources/fundamentals/authentication/guides/authentication-best-practices) 페이지에서 확인할 수 있습니다.
 

<div id="how-to-find-and-regenerate-your-api-key-and-secret-after-app-creation">
  #### App 생성 후 API Key와 Secret을 찾고 재생성하는 방법
</div>

이미 App을 생성했고 API Key와 Secret을 다시 확인하거나 재생성해야 하는 경우, 다음 단계를 따르세요:

1. developer portal로 이동합니다.
2. 사이드 내비게이션에서 &#39;Projects and Apps&#39; 드롭다운을 펼칩니다.
3. 찾거나 재생성하려는 API Key와 Secret이 연결된 App을 엽니다.
4. &#39;Keys and tokens&#39; 탭으로 이동합니다.

여기에서 App과 연결된 모든 자격 증명을 확인할 수 있습니다. 
 

<div id="how-to-use-your-api-key-and-secret">
  #### API 키와 Secret 사용 방법
</div>

X Developer Platform을 처음 살펴보고 있다면, 플랫폼에서 어떤 기능을 사용할 수 있는지 확인하기 위해 [도구 또는 라이브러리](/ko/resources/tools-and-libraries)를 사용하는 것을 권장합니다. 이러한 도구는 인증을 편리하게 처리해 주며, 많은 시간과 불필요한 시행착오를 줄여 줄 수 있습니다. 특히 입문 개발자에게는 [Postman 시작하기](/ko/tutorials/postman-getting-started) 또는 [Insomnia](https://insomnia.rest/) 사용을 권장합니다. 

직접 요청을 처음부터 구성해 보고 싶다면, [OAuth 1.0a 요청 인가](/ko/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request) 가이드를 참고해 주세요.
