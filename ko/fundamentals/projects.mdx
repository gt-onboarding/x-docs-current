<div id="introduction">
  ## 소개
</div>

프로젝트를 사용하면 X API 사용 목적에 따라 작업을 체계적으로 구성하여 API 액세스를 효과적으로 관리하고 사용량을 모니터링할 수 있습니다. 각 프로젝트에는 액세스 수준(이 페이지 하단에 설명)에 따라 하나 이상의 [앱](/ko/resources/fundamentals/developer-apps)을 포함할 수 있습니다. 이러한 앱을 통해 [API 키와 시크릿](/ko/resources/fundamentals/authentication#api-key-and-secret), 사용자 [액세스 토큰](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow), [앱 액세스 토큰](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)과 같은 [인증](/ko/resources/fundamentals/authentication) 자격 증명을 생성할 수 있습니다.

모든 앱의 키와 토큰을 사용하여 X API 또는 X Ads API에 액세스할 수 있지만(Ads API 사용을 위해서는 [추가 액세스 신청](/ko/x-ads-api/getting-started/step-by-step-guide)이 필요함), [X API v2](/ko/x-api/introduction) 엔드포인트를 사용하려면 반드시 프로젝트에 연결된 앱의 키와 토큰을 사용해야 합니다.

개발자 계정이 있는 경우 개발자 포털의 [프로젝트 및 앱](https://developer.x.com/en/portal/projects-and-apps) 페이지에서 프로젝트를 확인하고 관리할 수 있습니다. 계정이 없다면 [가입](https://developer.x.com/en/portal/petition/essential/basic-info)하세요.

<div id="projects-and-x-api-access-levels">
  ### 프로젝트와 X API 액세스 레벨
</div>

현재 프로젝트 수준에서 적용되는 액세스 티어는 네 가지가 있습니다:

* Free
* Basic
* Pro
* Enterprise

각 액세스 레벨이 제공하는 기능에 대해 더 알아보려면 [about X API](/ko/x-api/getting-started/about-x-api) 페이지를 방문하세요.

Free, Basic, Pro 중 하나의 액세스 레벨이 적용된 프로젝트는 하나만 가질 수 있습니다.

아래에서 프로젝트별로 다른 몇 가지 사항을 설명합니다:

<div id="free">
  #### Free
</div>

이 액세스 티어는 [가입](https://developer.twitter.com/en/portal/petition/essential/basic-info)을 완료한 모든 개발자 계정 보유자에게 제공됩니다.

해당 프로젝트 내 앱 수: 1\
[트윗 사용 한도](/ko/x-api/fundamentals/post-cap): 월 1,500 트윗

인증 방식:

* [OAuth 2.0 with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
* 앱 전용

<div id="basic">
  #### Basic
</div>

이 액세스 티어는 Developer Portal을 통해 Basic 액세스에 가입한 모든 사용자에게 제공됩니다.

해당 프로젝트 내 앱 수: 2\
[게시물 소비 상한](/ko/x-api/fundamentals/post-cap): 월 10,000개 게시물\
인증 방법:

* [OAuth 2.0 with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
* [OAuth 1.0a](/ko/resources/fundamentals/authentication#oauth-1-0a-2)
* 앱 전용(App-only)

<div id="pro">
  #### Pro
</div>

이 액세스 티어는 developer portal에서 Pro 액세스를 구독한 모든 사용자에게 제공됩니다.

해당 프로젝트 내 앱 수: 3
[Tweet 소비 한도](/ko/x-api/fundamentals/post-cap): 월 100만 트윗

인증 방식:

* [OAuth 2.0 with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
* [OAuth 1.0a](/ko/resources/fundamentals/authentication#oauth-1-0a-2)
* 앱 전용

<div id="enterprise">
  #### Enterprise
</div>

이 액세스 티어는 Enterprise를 신청하여 당사 팀의 승인을 받고, X 어카운트 매니저를 통해 월 자동 갱신되는 계약에 서명한 모든 분께 제공됩니다. [여기에서 신청하세요](https://docs.google.com/forms/d/e/1FAIpQLScO3bczKWO2jFHyVZJUSEGfdyfFaqt2MvmOfl_aJp0KxMqtDA/viewform).

해당 프로젝트 내 앱 수: 3개 이상\
[트윗 소비 한도](/ko/x-api/fundamentals/post-cap): 월 5,000만 개 이상의 트윗

인증 방식:

* [OAuth 2.0 with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
* [OAuth 1.0a](/ko/resources/fundamentals/authentication#oauth-1-0a-2)
* 앱 전용

<div id="tweet-caps-and-projects">
  ### 트윗 한도와 프로젝트
</div>

트윗 사용량 한도는 프로젝트 수준에서 적용되며, 특정 X API v2 엔드포인트에서 한 달 동안 가져올 수 있는 트윗 수를 제한합니다.

[트윗 한도](/ko/x-api/fundamentals/post-cap)에 대해 더 알아보세요.

<div id="configuring-projects">
  ### 프로젝트 설정
</div>

<div id="creating-a-project">
  #### 프로젝트 생성하기
</div>

프로젝트를 생성하려면 [대시보드](https://developer.twitter.com/en/portal/dashboard) 또는 developer portal 내 [Projects &amp; Apps](https://developer.twitter.com/en/portal/projects-and-apps) 페이지에서 &quot;New Project&quot;를 클릭합니다. 이 옵션은 아직 프로젝트를 생성하지 않은 경우에만 표시됩니다. 프로젝트 이름, 설명, 사용 목적을 입력하라는 메시지가 표시됩니다. 또한 새 앱을 생성하거나 기존 독립형 앱을 연결하라는 요청을 받게 됩니다.

<div id="creating-or-connecting-an-app-for-your-project">
  #### 프로젝트용 앱 생성 또는 연결
</div>

프로젝트에 앱이 포함되어 있지 않다면 대시보드에서 프로젝트 이름을 클릭하여 앱을 추가할 수 있습니다. 해당 페이지에서 새 앱을 생성하거나 기존의 독립 실행형 앱을 선택해 프로젝트에 연결할 수 있습니다. 이 앱에서 이 가이드 앞부분에 나열된 인증용 키와 토큰을 생성할 수 있습니다.

<div id="editing-a-project">
  #### 프로젝트 편집
</div>

프로젝트를 편집하려면 developer portal의 대시보드나 Projects &amp; Apps 페이지에서 프로젝트 이름을 클릭하세요. 그러면 해당 프로젝트의 상세 정보가 표시되며, 여기에서 변경을 위해 &quot;edit&quot;를 선택할 수 있습니다.

<div id="standalone-apps">
  ### 독립 실행형 앱
</div>

독립 실행형 앱은 프로젝트 구조와는 별도로 존재하는 앱입니다. 이러한 독립 실행형 앱과 연결된 인증 자격 증명을 사용하면 X의 표준 v1.1 API, premium v1.1, Enterprise, 또는 X Ads API에 성공적으로 요청을 보낼 수 있습니다. 독립 실행형 앱은 프로젝트에 연결하지 않는 한 X API v2 엔드포인트에 요청을 보내면 실패합니다.

2020년 8월 이전에 앱을 생성한 경우, 해당 앱은 개발자 포털의 [Projects &amp; Apps](https://developer.twitter.com/en/portal/projects-and-apps) 아래에 있는 &quot;Standalone Apps&quot; 섹션에서 확인할 수 있습니다. 프로젝트에 연결된 앱을 포함하여 전체 앱 수는 최대 10개로 제한됩니다.

팀 계정에 속해 있는 경우, &quot;Standalone Apps&quot; 아래에서 본인이 소유한 앱을 볼 수 있습니다. 팀원이 프로젝트의 일부인 앱을 소유한 경우, 해당 앱의 이름과 소유자 정보는 볼 수 있지만, 설정을 변경하거나 키와 토큰에 접근하거나 키와 토큰을 재생성할 수는 없습니다. 앱에 변경이 필요하다면 앱 소유자에게 직접 연락해야 합니다.
