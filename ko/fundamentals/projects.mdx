<div id="introduction">
  ## 소개
</div>

프로젝트를 사용하면 X API 활용 목적에 따라 작업을 구성해 API 액세스를 효율적으로 관리하고 사용량을 모니터링할 수 있습니다. 각 프로젝트에는 액세스 수준(이 페이지의 뒷부분에서 설명)을 기준으로 하나 이상의 [Apps](/ko/resources/fundamentals/developer-apps)를 포함할 수 있습니다. 이러한 App을 사용해 [API Keys and Secrets](/ko/resources/fundamentals/authentication#api-key-and-secret), 사용자 [Access Tokens](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow), [App Access Token](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)과 같은 [authentication](/ko/resources/fundamentals/authentication) 자격 증명을 생성합니다.

이러한 키와 토큰은 어떤 App에서 생성된 것이라도 X API 또는 X Ads API에 액세스하는 데 사용할 수 있지만(Ads API를 사용하려면 [추가 액세스를 신청](/ko/x-ads-api/getting-started/step-by-step-guide)해야 함), [X API v2](/ko/x-api/introduction) 엔드포인트를 사용하려면 프로젝트와 연결된 App의 키와 토큰을 사용해야 합니다.

개발자 계정이 있는 경우 developer portal의 [Projects &amp; Apps](https://developer.x.com/en/portal/projects-and-apps) 페이지에서 프로젝트를 확인하고 관리할 수 있습니다. 아직 계정이 없다면 [Sign up](https://developer.x.com/en/portal/petition/essential/basic-info)하여 계정을 생성하세요.

<div id="projects-and-x-api-access-levels">
  ### 프로젝트와 X API 액세스 레벨
</div>

현재 프로젝트 수준에서 적용되는 액세스 레벨은 다음 네 가지입니다:

* Free
* Basic
* Pro
* Enterprise

각 액세스 레벨에서 무엇을 제공하는지 자세히 알아보려면 [about X API](/ko/x-api/getting-started/about-x-api) 페이지를 방문하세요.

Free, Basic, Pro 액세스 레벨로 설정된 프로젝트는 계정당 하나만 만들 수 있습니다.

아래에서는 프로젝트별로 몇 가지 차이점을 설명합니다:

<div id="free">
  #### Free
</div>

이 액세스 티어는 [가입](https://developer.twitter.com/en/portal/petition/essential/basic-info)을 통해 개발자 계정을 생성한 누구에게나 제공됩니다.

해당 프로젝트 내 앱 개수: 1\
[트윗 사용 한도](/ko/x-api/fundamentals/post-cap): 월 1,500 트윗

인증 방식:

* [PKCE를 사용하는 OAuth 2.0](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
* 앱 전용

<div id="basic">
  #### Basic
</div>

이 액세스 티어는 developer portal을 통해 Basic 액세스에 가입한 모든 사용자에게 제공됩니다.

해당 프로젝트에 포함된 앱 수: 2
[Tweet consumption cap](/ko/x-api/fundamentals/post-cap): 월 10,000개의 트윗
인증 방식:

* [OAuth 2.0 with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
* [OAuth 1.0a](/ko/resources/fundamentals/authentication#oauth-1-0a-2)
* 앱 전용 인증

<div id="pro">
  #### Pro
</div>

이 액세스 티어는 developer portal에서 Pro 액세스를 구독한 모든 사용자에게 제공됩니다.

해당 프로젝트 내 앱 수: 3\
[트윗 사용 한도](/ko/x-api/fundamentals/post-cap): 월 100만 개의 트윗

인증 방식:

* [OAuth 2.0 with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
* [OAuth 1.0a](/ko/resources/fundamentals/authentication#oauth-1-0a-2)
* 앱 전용

<div id="enterprise">
  #### Enterprise
</div>

이 액세스 등급은 Enterprise를 신청하고 저희 팀의 승인을 받은 뒤, X 어카운트 매니저를 통해 월 자동 갱신 계약을 체결한 모든 사용자에게 제공됩니다. [여기에서 신청하세요](https://docs.google.com/forms/d/e/1FAIpQLScO3bczKWO2jFHyVZJUSEGfdyfFaqt2MvmOfl_aJp0KxMqtDA/viewform).

해당 프로젝트 내 앱 개수: 3개 이상\
[트윗 사용 한도](/ko/x-api/fundamentals/post-cap): 월 5,000만 개 이상의 트윗

인증 방식:

* [PKCE를 사용하는 OAuth 2.0](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
* [OAuth 1.0a](/ko/resources/fundamentals/authentication#oauth-1-0a-2)
* 앱 전용

<div id="tweet-caps-and-projects">
  ### 트윗 한도와 프로젝트
</div>

트윗 사용 한도는 프로젝트 단위로 적용되며, 특정 X API v2 엔드포인트에서 한 달 동안 가져올 수 있는 트윗 수를 실질적으로 제한합니다.

자세한 내용은 [트윗 한도](/ko/x-api/fundamentals/post-cap)를 참고하세요.

<div id="configuring-projects">
  ### 프로젝트 설정
</div>

<div id="creating-a-project">
  #### 프로젝트 생성하기
</div>

프로젝트를 생성하려면 [dashboard](https://developer.twitter.com/en/portal/dashboard) 또는 developer portal 내 [Projects &amp; Apps](https://developer.twitter.com/en/portal/projects-and-apps) 페이지에서 &quot;New Project&quot;를 클릭합니다. 아직 프로젝트를 생성하지 않았다면 이 옵션이 표시됩니다. 프로젝트 이름, 설명, 사용 목적을 입력하라는 안내가 표시됩니다. 또한 새 앱을 생성하거나 기존 독립 실행형 앱을 연결하라는 요청을 받게 됩니다.

<div id="creating-or-connecting-an-app-for-your-project">
  #### 프로젝트용 앱 생성 또는 연결
</div>

프로젝트에 앱이 포함되어 있지 않은 경우 대시보드에서 프로젝트 이름을 클릭하여 앱을 추가할 수 있습니다. 여기에서 새 앱을 생성하거나 기존의 독립 실행형 앱을 선택해 프로젝트에 연결할 수 있습니다. 이 앱에서 이 가이드의 앞부분에 나열된 인증용 키와 토큰을 생성할 수 있습니다.

<div id="editing-a-project">
  #### 프로젝트 편집하기
</div>

프로젝트를 편집하려면 developer portal의 대시보드 또는 Projects &amp; Apps 페이지에서 프로젝트 이름을 클릭하세요. 그러면 프로젝트의 세부 정보가 표시되며, 여기서 &quot;edit&quot;를 선택해 변경할 수 있습니다.

<div id="standalone-apps">
  ### 독립 실행형 앱
</div>

독립 실행형 앱은 프로젝트 구조 외부에 존재하는 앱입니다. 이러한 독립 실행형 앱에 연결된 인증 자격 증명을 사용하면 X API의 standard v1.1, premium v1.1, Enterprise 또는 X Ads API에 정상적으로 요청을 보낼 수 있습니다. 독립 실행형 앱은 프로젝트에 연결하지 않으면 X API v2 엔드포인트에 대한 요청이 실패합니다.

2020년 8월 이전에 App을 생성했다면, developer portal의 [Projects &amp; Apps](https://developer.twitter.com/en/portal/projects-and-apps) 아래에 있는 &quot;Standalone Apps&quot; 섹션에서 확인할 수 있습니다. 프로젝트에 연결된 App을 포함하여, 보유할 수 있는 App은 총 10개로 제한됩니다.

팀 계정에 속해 있는 경우, Standalone Apps 아래에서 자신이 소유한 App을 확인할 수 있습니다. 팀원이 프로젝트의 일부인 App을 소유하고 있는 경우, 해당 App의 이름과 소유자 정보는 볼 수 있지만, 설정을 변경하거나 키와 토큰에 접근하거나 키와 토큰을 재생성할 수는 없습니다. 해당 App에 변경이 필요하면 App 소유자에게 연락해야 합니다.
