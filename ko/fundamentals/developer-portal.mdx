  ## 개요

<div id="introduction">
  ### 소개
</div>

X 개발자 포털에는 개발자가 X API 및 X Ads API에 대한 접근 권한을 직접 관리하는 데 사용할 수 있는 셀프 서비스 도구 모음이 포함되어 있습니다.

이 포털에서 다음을 수행할 수 있습니다.

* X [Projects](/ko/resources/fundamentals/projects) 및 [Apps](/ko/resources/fundamentals/developer-apps) (및 해당 프로젝트와 앱이 제공하는 인증 키와 토큰)을 생성하고 관리합니다.
* X API 표준 v1.1 및 v2 엔드포인트에 대한 접근 수준과 연동을 관리합니다.
* 제공되는 다양한 엔드포인트와 기능에 대해 더 자세히 알아봅니다.

<div id="ready-to-get-access">
  ### 액세스할 준비가 되셨나요?
</div>

계정을 생성하면 X API를 사용하기 시작할 수 있습니다.

추가 기능이 필요하거나 더 높은 [게시물 한도](/ko/x-api/fundamentals/post-cap)가 필요한 경우 Developer Portal에서 Basic 또는 Pro를 구매할 수 있습니다. Enterprise에 관심이 있는 경우 [여기](https://docs.x.com/enterprise/forms/enterprise-api-interest)에서 신청해 주세요.

<div id="onboarding">
  ### 온보딩
</div>

온보딩 마법사는 처음 [프로젝트](/ko/resources/fundamentals/projects)와 [앱](/ko/resources/fundamentals/developer-apps)을 설정하는 과정을 단계별로 안내합니다. API 요청을 인증하는 데 필요한 자격 증명을 발급받으려면 프로젝트와 앱을 생성해야 합니다. 개발자 계정에 처음 로그인하는 경우 이 마법사가 표시됩니다.

이 과정을 통해 인증에 사용하는 일련의 키와 토큰을 받게 되며, 이에 대해서는 [앱 개요 페이지](/ko/resources/fundamentals/developer-apps#overview)에서 더 자세히 알아볼 수 있습니다. X API로 인증하는 데 필요한 사항에 대해 더 자세히 알고 싶다면 [인증 섹션](/ko/resources/fundamentals/authentication)을 참고하세요.

**주의:** 나중에 다시 사용할 수 있도록 [키와 토큰을 안전한 위치에 저장](/ko/resources/fundamentals/authentication/guides/authentication-best-practices)해야 합니다. 한 번 발급된 자격 증명은 재생성하지 않는 한 다시 조회할 수 있는 방법이 없습니다.

<div id="project-and-app-management">
  ### 프로젝트 및 앱 관리
</div>

개발자 계정의 주요 역할 중 하나는 [프로젝트와 앱을 관리](/ko/resources/fundamentals/developer-apps#app-management)할 수 있도록 하는 것입니다. 개발자는 developer portal 대시보드에서 X 프로젝트와 앱을 생성하고 관리할 수 있습니다. 여기에서 앱 ID를 확인하고, 앱의 설정, 권한, 콜백 URL을 편집하며, 키와 토큰을 생성하거나 폐기할 수 있습니다.

<div id="learn-more-about-whats-available-with-x-api-v2">
  ### X API v2에서 제공되는 기능 자세히 알아보기
</div>

Developer Portal에는 X API의 다양한 버전과 액세스 수준에 대해 자세히 알아볼 수 있는 제품 섹션이 있습니다.

X API v2 제품 섹션에서는 Free, Basic, Pro, Enterprise 액세스 티어에 대한 중요한 정보를 제공합니다. 이 페이지에는 프로젝트 수준 앱 제약 사항, 트윗 상한, 비용뿐만 아니라 엔드포인트별 요청 한도와 특수 속성에 대한 세부 정보가 담겨 있습니다. 또한 서로 다른 액세스 수준을 비교하고, 가능한 경우 추가 액세스를 신청할 수도 있습니다.

[v2 액세스 수준 전체](/ko/x-api/introduction#access-levels)를 확인하세요.

  ## 팀 관리
</div>

<Note>
  **Enterprise v2 계정에서만 사용할 수 있습니다.**
</Note>

[**developer portal의 &quot;team&quot; 페이지를 확인하세요.**](https://developer.twitter.com/en/portal/teams)

<div id="why-use-team-functionality">
  ### 팀 기능을 사용하는 이유
</div>

팀 기능은 X Developer Platform 내에서 프로젝트와 앱을 함께 개발할 수 있도록 지원합니다. 보통 팀에서는 액세스 제어, 결제 및 지불 관리 등 서로 다른 업무를 담당하는 사람이 있고, 팀 기능을 사용하면 이러한 사람들을 초대해 프로젝트에 함께 기여하도록 할 수 있습니다.

<div id="inviting-team-members">
  ### 팀 구성원 초대하기
</div>

팀에 다른 사람을 초대하려면 관리자가 해당 사용자의 X 핸들을 사용해 초대할 수 있습니다. 초대된 사용자는 이메일을 받게 되며, 그 이메일에 포함된 초대 안내를 통해 참여를 수락할 수 있습니다. 초대를 수락하면 Developer Agreement &amp; Policy에 동의해야 하며, 이후 메인 계정의 팀 페이지에 접속할 수 있습니다.

**참고:** 현재 팀 관리는 [App](/ko/resources/fundamentals/developer-apps) 자격 증명에 따라 API 접근 권한을 부여하거나 제한하지 않습니다. 팀 단위로 App 관리를 공유하는 것은 불가능합니다. 소유자가 아닌 사용자는 App(키/토큰)을 수정, 생성 또는 삭제할 수 없습니다.

<div id="team-dashboard">
  ### 팀 대시보드
</div>

팀 대시보드의 &quot;members&quot; 탭에서는 모든 멤버와 각자의 역할을 볼 수 있습니다. 팀의 관리자라면 개발자 접근 권한을 관리하고 각 멤버의 역할을 편집할 수 있습니다.

관리자는 팀 대시보드의 &quot;pending&quot; 탭에도 액세스할 수 있습니다. 이곳에서 발송된 각 초대의 세부 정보를 확인하고 관리할 수 있습니다.

<div id="team-roles">
  ### 팀 역할
</div>

**관리자 역할**

* 팀 프로젝트 및 앱을 관리할 수 있는 권한
* 모든 앱 환경을 관리할 수 있는 권한
* 구독을 선택하거나 업그레이드할 수 있는 권한
* 청구/결제 수단을 변경할 수 있는 권한
* 팀 구성원을 추가/제거할 수 있는 권한
* 팀 구성원의 역할을 변경할 수 있는 권한

**개발자 역할**

* 자신의 프로젝트 및 앱을 관리할 수 있는 권한
* 팀 프로젝트 및 앱에 대한 읽기 전용 접근 권한
* 팀을 떠날 수 있는 권한
