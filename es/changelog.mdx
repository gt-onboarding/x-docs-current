<Update label="Aug 3, 2022" description="X API v2">
  ### Adición de alias de operadores de filtrado

  Se añadieron operadores de filtrado/consulta &quot;aliases&quot; a los endpoints de búsqueda y flujo filtrado, tanto en Enterprise como en Premium y v2. En muchos casos, el operador se refiere a Publicaciones en lugar de statuses, por ejemplo, &quot;in&#95;reply&#95;to&#95;tweet&#95;id&quot; como alias de &quot;in&#95;reply&#95;to&#95;status&#95;id&quot;. En otros casos, el nuevo alias de operador aporta mayor claridad al tipo de coincidencia que se realiza. Por ejemplo, dado que las Publicaciones solo pueden tener un video adjunto, se proporciona un nuevo alias &quot;has:video&#95;link&quot; para &quot;has:videos&quot;. Consulta la [tabla de operadores de flujo filtrado de v2](https://developer.x.com/en/docs/twitter-api/tweets/filtered-stream/integrate/build-a-rule#list) para ver más ejemplos.
</Update>

<Update label="14 de dic de 2021" description="X API v2: Autenticación">
  ### Compatibilidad con la autenticación OAuth 2.0

  A partir de hoy, todos los desarrolladores pueden autenticarse con OAuth 2.0 seleccionando OAuth 2.0 como método de autenticación en el portal para desarrolladores. Hemos añadido compatibilidad para clientes confidenciales y públicos, y hemos habilitado que todos los endpoints v2 relevantes utilicen este método de autenticación como parte de este lanzamiento.

  OAuth 2.0 es un protocolo de autorización estándar del sector que proporciona a los desarrolladores más control sobre los scopes (permisos) de una Aplicación y mejora los flujos de autorización en múltiples dispositivos. En otras palabras, los desarrolladores que crean Aplicaciones para las personas en X tendrán más control sobre la información que su Aplicación solicita a sus usuarios, de modo que solo tengan que pedir a sus usuarios finales los datos e información que realmente necesitan.

  Este protocolo de autorización moderno te permitirá presentar a tus usuarios finales un flujo de consentimiento más sencillo para autorizar tu Aplicación, que solo muestra los scopes específicos que les has solicitado.

  Para obtener más información sobre este lanzamiento, visita nuestro anuncio en el foro.
</Update>

<Update label="15 de nov de 2021" description="X API v2: Acceso anticipado">
  ### Actualizaciones importantes de la plataforma para X API v2

  **Actualizaciones importantes de la plataforma: X API v2 es ahora la versión principal, nuevos niveles de acceso Essential y Elevated, y cambios en las Políticas**

  Hoy anunciamos que X API v2 es ahora la versión principal de la X API. Hemos lanzado suficientes endpoints y funcionalidades en X API v2 para satisfacer las necesidades del 90 % de todas las Aplicaciones existentes creadas sobre la X API.

  Hoy lanzamos los siguientes cambios para mejorar aún más la experiencia de los desarrolladores con X API v2:

  * Acceso Essential inmediato y gratuito a la API y acceso Elevated gratuito para los desarrolladores cuyos casos de uso hayan sido aprobados. Tenemos muchos más detalles sobre estos niveles de acceso disponibles en nuestra página Acerca de la X API.

  * Eliminamos redacción en nuestra Política para desarrolladores que restringía cómo puedes crear con las funciones principales de X y limitaba la cantidad de usuarios que puedes admitir a través de tu Aplicación.

    * Estamos abriendo nuestra plataforma para animarte a crear herramientas y productos que hagan X mejor, más saludable y que amplíen la conversación pública. [Consulta una lista de soluciones que nos encantaría que crearas](https://developer.x.com/en/docs/twitter-api/what-to-build).

    * En concreto, hemos eliminado términos que restringían la replicación de la experiencia de X, incluidas las funciones principales de X, así como términos que requerían permiso para tener grandes cantidades de tokens de usuario.

    * Sabemos que crear soluciones que ayuden a las personas en X a menudo significa que un desarrollador tiene que crear (o replicar) algunas de las cosas que están disponibles en X. Estos cambios en nuestra Política para desarrolladores están pensados para aportar claridad al ecosistema de desarrolladores y proporcionar una plataforma de API abierta que facilite a los desarrolladores crear, innovar y tener un impacto en la conversación pública.

  Profundizamos mucho más en este lanzamiento y en los lanzamientos de X API que nos han llevado hasta este momento en nuestra [publicación en el foro](https://devcommunity.x.com/t/ushering-in-a-new-era-for-the-twitter-developer-platform-with-the-twitter-api-v2/162087).

  Además de explorar las entradas anteriores del registro de cambios, hemos reunido algunos recursos clave que puedes revisar para entender todo lo que se ha lanzado en X API v2 hasta la fecha:

  * [Mapa de endpoints de X API](/es/x-api/migrate/x-api-endpoint-map)

  * [Acerca de la X API](/es/x-api/getting-started/about-x-api)
</Update>

<Update label="May 10, 2018" description="X API: Standard v1.1">
  ### Cambios en el renderizado del contenido de la cita

  Hoy añadimos un cambio en la forma en que se renderizan las URL en el contenido de la cita.

  Estamos agregando una nueva entidad llamada &quot;quoted&#95;status&#95;permalink&quot; para garantizar que se pueda hacer referencia a la publicación citada y eliminaremos el enlace t.co del campo &quot;text&quot; de la publicación citada.

  Con el nuevo formato, el enlace t.co ya no se agrega al campo &quot;text&quot; de la publicación citada en el objeto &quot;quoted&#95;status&quot;.

  Además, ya no colocaremos la URL t.co, expanded&#95;url y display&#95;url de la publicación citada en la entidad &quot;urls&quot;. En su lugar, se agregarán a un nuevo objeto &quot;quoted&#95;status&#95;permalink&quot; con los atributos de URL &quot;url&quot;, &quot;expanded&quot; y &quot;display&quot;. Esto significa que el objeto &quot;quoted&#95;status&#95;permalink&quot; quedará reservado para el enlace t.co que apunta a la publicación citada, mientras que el array &quot;entities.urls&quot; se puede usar para identificar cualquier enlace compartido dentro de la publicación original o la cita.

  Consulta nuestra [publicación en el foro](https://devcommunity.x.com/t/updating-how-urls-are-rendered-in-the-quote-tweet-payload/105473) y la [documentación actualizada](https://devcommunity.x.com/t/upcoming-changes-to-the-developer-platform/104603) para obtener más detalles.
