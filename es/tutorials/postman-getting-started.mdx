<div id="introduction">
  ## Introducción
</div>

Postman es una aplicación de escritorio y web que te permite realizar solicitudes a una API desde una interfaz gráfica de usuario. Recomendamos usar Postman con la X API, X Ads API y los endpoints de Labs cuando explores la funcionalidad de las APIs, así como cuando estés solucionando problemas con tu aplicación. 

Actualmente contamos con dos colecciones de Postman disponibles:

<div id="prerequisites">
  ### Requisitos previos
</div>

Antes de comenzar a usar las colecciones de Postman de X, deberás asegurarte de que tienes el acceso y las credenciales adecuados para la herramienta de la plataforma para desarrolladores de X que elijas. Puedes obtener más información sobre cómo obtener acceso en nuestra [página “Primeros pasos”](/es/resources/platform-overview). 

Una vez que tengas una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info), hayas configurado una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) y dispongas de un conjunto de [credenciales de autenticación](/es/resources/fundamentals/authentication) (claves y tokens), y hayas configurado correctamente tu entorno para realizar solicitudes a la API específica que planeas usar, puedes seguir los pasos que se indican a continuación para comenzar.
 

<div id="getting-started-with-xs-postman-collections">
  ## Primeros pasos con las colecciones de Postman de X
</div>

<div id="step-one-add-one-of-the-x-postman-collections-to-your-account">
  ### Paso uno: Agrega una de las colecciones de Postman de X a tu cuenta
</div>

Aunque podrías armar manualmente los endpoints específicos que quieras usar dentro de Postman, nosotros ya hicimos todo el trabajo pesado por ti y creamos una colección lista para usar con las APIs relevantes. Solo haz clic en uno de los enlaces de la sección anterior «Postman collections» y se agregará a tu aplicación de Postman una colección con todos los endpoints asociados con la API seleccionada. Estas colecciones también están disponibles en la [Postman API network](https://explore.postman.com/). Cada endpoint incluirá automáticamente los parámetros disponibles, respuestas de ejemplo y el tipo de autenticación configurado, por lo que solo necesitas agregar tus credenciales y los valores de los parámetros para empezar a explorar la funcionalidad.

En este ejemplo, vamos a trabajar con la [API v2 collection](https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400) de X. 
 

<div id="step-two-add-your-keys-and-tokens-as-environmental-variables">
  ### Paso dos: Añade tus claves y tokens como variables de entorno
</div>

Cuando añadas la colección a tu instancia de Postman, se añadirá automáticamente un entorno llamado ‘X API v2’, al que tendrás que añadir tus claves y tokens. En este paso, te guiaremos por el proceso de añadir tus claves y tokens desde tu aplicación de desarrollador al entorno ‘X API v2’. 

Para añadir tus claves y tokens al entorno “X API v2”, haz clic en el botón “Manage Environments” (configuración) en la esquina superior derecha de Postman. 
 

<Frame>
  <img src="/images/using-postman-1.png.twimg.1920.png" alt="Esta imagen muestra el botón &#x22;Manage Environments&#x22; resaltado en la consola de Postman." />
</Frame>

En la lista de entornos, haz clic en “X API v2”. 

A continuación, añade una variable para cada una de las diferentes claves y tokens que hayas generado a través del panel de Apps en la tabla. Aquí tienes un ejemplo de cómo debería verse tu tabla:

| VARIABLE | VALOR INICIAL | VALOR ACTUAL |
| :--- | :--- | :--- |
| consumer&#95;key | `QAktM6W6DF6F7XXXXXX` | `QAktM6W6DF6F7XXXXXX` |
| consumer&#95;secret | `AJX560A2Omgwyjr6Mml2esedujnZLHXXXXXX` | `AJX560A2Omgwyjr6Mml2esedujnZLHXXXXXX` |
| access&#95;token | `1995XXXXX-0NGqVhk3s96IX6SgT3H2bbjOPjcyQXXXXXXX` | `1995XXXXX-0NGqVhk3s96IX6SgT3H2bbjOPjcyQXXXXXXX` |
| token&#95;secret | `rHVuh7dgDuJCOGeoe4tndtjKwWiDjBZHLaZXXXXXX` | `rHVuh7dgDuJCOGeoe4tndtjKwWiDjBZHLaZXXXXXX` |
| bearer&#95;token | `AAAAAAAAAAAAAAAAAAAAAL9v6AAAAAAA99t03huuqRYg0mpYAAFRbPR3XXXXXXX` | `AAAAAAAAAAAAAAAAAAAAAL9v6AAAAAAA99t03huuqRYg0mpYAAFRbPR3XXXXXXX` |

Ten en cuenta que las claves y tokens usados en la tabla anterior son ficticios y no funcionarán si se usan en una solicitud. 

Una vez que hayas añadido tus credenciales como variables y te hayas asegurado de que el entorno X API v2 está seleccionado, estarás listo para hacer solicitudes a la colección X API v2. Esto se debe a que cada una de las pestañas de autorización de los endpoints está configurada para heredar automáticamente las variables de este entorno. 

Para usar Postman con tokens de acceso de usuario, salta XXXX a los detalles adicionales sobre cómo hacerlo.

<div id="step-three-select-an-endpoint">
  ### Paso tres: selecciona un endpoint
</div>

El siguiente paso es elegir un endpoint de la colección y comenzar a crear tu solicitud. Puedes seleccionar un endpoint desde la barra de navegación de la derecha. Así es como se ve:

<Frame>
  <img src="/images/using-postman-2.png.twimg.1920.png" alt="Esta imagen muestra la solicitud &#x22;Single Posts&#x22; seleccionada en el menú desplegable &#x22;Post Lookup&#x22; en la sección &#x22;X API v2&#x22;." />
</Frame>

En este ejemplo, vamos a utilizar el endpoint X API v2 &gt; Post Lookup &gt; Single Post. 

<div id="step-four-add-values-to-the-params-tab">
  #### Paso cuatro: agrega valores a la pestaña Params
</div>

En este paso, deberás ir a la pestaña Params. Deberías ver un conjunto de parámetros inactivos con descripciones que explican qué hace cada parámetro y una lista de todos los valores posibles que puedes pasar en tu solicitud.

En este ejemplo, vamos a activar los parámetros de consulta expansions y tweet.fields y agregar los siguientes valores:

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `tweet.fields` | `created_at,attachments` |
| expansions | author&#95;id |

Además de agregar los parámetros de consulta, debemos agregar la variable de ruta (Path Variable) obligatoria, id. Dado que este endpoint devuelve Publicaciones, debemos agregar un Post ID válido como valor.

Puedes encontrar el Post ID navegando a x.com, haciendo clic en una Publicación y luego consultando la URL. Por ejemplo, el Post ID de la siguiente URL es `1228393702244134912`:

`https://x.com/XDevelopers/status/1228393702244134912`

En la pestaña Params, desplázate hacia abajo, por debajo de todos los parámetros de consulta, hasta que se muestre la sección “Path Variables”. Agregaremos el Post ID que quieras usar como valor de la clave id.

Si ingresaste correctamente todo en este paso, la pestaña Params debería verse como se muestra a continuación:

</Frame>

<div id="step-five-send-your-request-and-review-your-response">
  #### Paso cinco: Envía la solicitud y revisa la respuesta
</div>

Ahora que todo está configurado en tu solicitud, puedes hacer clic en el botón &quot;Send&quot;. 

Si todo se configuró correctamente, deberías recibir el siguiente cuerpo de respuesta:

```json
{

        "author_id": "2244994945",
        "text": "¿Qué escribió el desarrollador en su tarjeta de San Valentín?\n  \nwhile(true) {\n    I = Love(You);  \n}",

    "includes": {
        "users": [

                "username": "XDevelopers",
```

<div id="generating-a-user-access-token-with-postman">
  ### Generar un token de acceso de usuario con Postman:
</div>

<div id="using-oauth-10a-to-generate-a-user-access-token">
  #### Uso de OAuth 1.0a para generar un token de acceso de usuario
</div>

Consulta el proceso de tres pasos utilizado en la [colección de prueba del flujo de OAuth 1.0a](https://www.postman.com/xapidevelopers/workspace/twitter-s-public-workspace/collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400?ctx=documentation).

<div id="using-oauth-20-to-generate-a-user-access-token">
  #### Uso de OAuth 2.0 para generar un token de acceso de usuario
</div>

Si quieres generar un token de acceso de usuario de OAuth 2.0 en Postman, puedes hacerlo para utilizarlo junto con la [colección de Postman de X API v2](https://www.postman.com/xapidevelopers/workspace/twitter-s-public-workspace/collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400?ctx=documentation). 

Haz clic en la colección en tu espacio de trabajo, ve a la pestaña titulada “Auth” y selecciona el tipo “OAuth 2.0”. Desde ahí, bajo el encabezado “Configure New Token”, ve a donde dice “Configuration Options”. Ahí puedes actualizar el “Grant Type” a “Authorization Code (With PKCE)”.

Deberás actualizar tu callback URL para que coincida con la callback URL asociada con tu Aplicación. Además, deberás actualizar los siguientes parámetros:

* Auth URL -  https://x.com/i/oauth2/authorize
* Access Token URL -  https://api.x.com/2/oauth2/token
* Client ID - Tu client ID de OAuth 2.0 del Dev Portal
* Client Secret - Si estás usando un cliente confidencial
* Update Scope - Scopes que coincidan con los endpoints a los que quieres conectarte. Ejemplo “tweet.read users.read” 
* Tu callback URL (también conocida como redirect URL). Esta debe coincidir con lo que tienes en la configuración de autenticación de tu Aplicación.
* State - state

Cuando estés listo, puedes hacer clic en “Get New Access Token” para generar un token de acceso. Si ves un cuadro de diálogo que indica que algo salió mal, es posible que necesites presionar el botón Atrás para iniciar sesión. Necesitarás autorizar que tu Aplicación tenga acceso a tu cuenta haciendo clic donde dice “Authorize app” en el cuadro de diálogo.

Después de autorizar tu Aplicación, serás dirigido de vuelta a Postman, donde podrás ver tu token y seleccionar el botón &quot;Use Token&quot; para comenzar a realizar solicitudes en nombre de un usuario autorizado.

Ahora ya estás listo para usar la colección de Postman.
