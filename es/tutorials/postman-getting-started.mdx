<div id="introduction">
  ## Introducción
</div>

Postman es una aplicación de escritorio y web que te permite realizar solicitudes a una API desde una interfaz gráfica de usuario. Recomendamos usar Postman con la X API, la X Ads API y los endpoints de Labs cuando explores la funcionalidad de las APIs, así como cuando soluciones problemas con tu aplicación. 

Actualmente contamos con dos colecciones de Postman disponibles:

<div id="prerequisites">
  ### Requisitos previos
</div>

Antes de comenzar a usar las colecciones de Postman de X, tendrás que asegurarte de que tienes el acceso y las credenciales adecuadas para la herramienta de la plataforma para desarrolladores de X que elijas. Puedes obtener más información sobre cómo obtener acceso en nuestra [página “Primeros pasos”](/es/resources/platform-overview). 

Una vez que tengas una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info), hayas configurado una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) y dispongas de un conjunto de [autenticación](/es/resources/fundamentals/authentication) claves y tokens, y hayas configurado correctamente tu entorno para hacer solicitudes a la API específica que planeas usar, puedes seguir los pasos que se indican a continuación para comenzar.
 

<div id="getting-started-with-xs-postman-collections">
  ## Primeros pasos con las colecciones de Postman de X
</div>

<div id="step-one-add-one-of-the-x-postman-collections-to-your-account">
  ### Paso uno: añade una de las colecciones de Postman de X a tu cuenta
</div>

Si bien podrías configurar manualmente los endpoints específicos que quieres usar dentro de Postman, nosotros hicimos la parte más compleja por ti y creamos una colección lista para usar con las APIs pertinentes. Solo haz clic en uno de los enlaces de la sección anterior “Postman collections” y se agregará a tu aplicación de Postman una colección con todos los endpoints asociados con la API seleccionada. Estas colecciones también están disponibles en la [red de API de Postman](https://explore.postman.com/). Cada endpoint incluirá automáticamente los parámetros disponibles, respuestas de ejemplo y el tipo de autenticación configurado, por lo que solo necesitas agregar tus credenciales y los valores de los parámetros para comenzar a explorar la funcionalidad.

En este ejemplo, vamos a trabajar con la [colección de la API v2 de X](https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400). 
 

<div id="step-two-add-your-keys-and-tokens-as-environmental-variables">
  ### Paso dos: Añade tus claves y tokens como variables de entorno
</div>

Cuando agregues la colección a tu instancia de Postman, se añadirá automáticamente un entorno llamado “X API v2”, al cual deberás agregar tus claves y tokens. En este paso, te guiaremos por el proceso de agregar tus claves y tokens desde tu aplicación de desarrollador al entorno “X API v2”. 

Para agregar tus claves y tokens al entorno “X API v2”, haz clic en el botón “manage environments” en la esquina superior derecha de Postman, es decir, en el botón de configuración situado en la esquina superior derecha. 
 

<Frame>
  <img src="/images/using-postman-1.png.twimg.1920.png" alt="Esta imagen muestra el botón &#x22;manage environments&#x22; resaltado en la consola de Postman." />
</Frame>

En la lista de entornos, haz clic en “X API v2”. 

A continuación, agrega una variable en la tabla para cada una de las diferentes claves y tokens que hayas generado desde el panel de Apps. Aquí tienes un ejemplo de cómo debería verse tu tabla:

| VARIABLE | VALOR INICIAL | VALOR ACTUAL |
| :--- | :--- | :--- |
| consumer&#95;key | `QAktM6W6DF6F7XXXXXX` | `QAktM6W6DF6F7XXXXXX` |
| consumer&#95;secret | `AJX560A2Omgwyjr6Mml2esedujnZLHXXXXXX` | `AJX560A2Omgwyjr6Mml2esedujnZLHXXXXXX` |
| access&#95;token | `1995XXXXX-0NGqVhk3s96IX6SgT3H2bbjOPjcyQXXXXXXX` | `1995XXXXX-0NGqVhk3s96IX6SgT3H2bbjOPjcyQXXXXXXX` |
| token&#95;secret | `rHVuh7dgDuJCOGeoe4tndtjKwWiDjBZHLaZXXXXXX` | `rHVuh7dgDuJCOGeoe4tndtjKwWiDjBZHLaZXXXXXX` |
| bearer&#95;token | `AAAAAAAAAAAAAAAAAAAAAL9v6AAAAAAA99t03huuqRYg0mpYAAFRbPR3XXXXXXX` | `AAAAAAAAAAAAAAAAAAAAAL9v6AAAAAAA99t03huuqRYg0mpYAAFRbPR3XXXXXXX` |

Ten en cuenta que las claves y tokens usados en la tabla anterior son inventados y no funcionarán si se usan en una solicitud. 

Una vez que hayas agregado tus credenciales como variables y te hayas asegurado de que el entorno “X API v2” está seleccionado, estarás listo para realizar solicitudes a la colección X API v2. Esto se debe a que las pestañas de autorización de cada endpoint están configuradas para heredar automáticamente las variables de este entorno. 

Para usar Postman con Tokens de acceso de usuario, sáltate XXXX para ir a los detalles adicionales sobre cómo hacerlo.

<div id="step-three-select-an-endpoint">
  ### Paso tres: Seleccionar un endpoint
</div>

El siguiente paso es elegir un endpoint de la colección y empezar a construir tu solicitud. Puedes seleccionar un endpoint desde la barra de navegación de la derecha. Así es como se ve:

<Frame>
  <img src="/images/using-postman-2.png.twimg.1920.png" alt="Esta imagen muestra la solicitud &#x22;Single Posts&#x22; seleccionada en el menú desplegable &#x22;Post Lookup&#x22; dentro de la sección &#x22;X API v2&#x22;." />
</Frame>

Para este ejemplo, usaremos el endpoint X API v2 &gt; Post Lookup &gt; Single Post. 

<div id="step-four-add-values-to-the-params-tab">
  #### Paso cuatro: Agregar valores a la pestaña Params
</div>

El siguiente paso es ir a la pestaña Params. Deberías ver un conjunto de parámetros inactivos con descripciones que explican qué hace cada parámetro y una lista de todos los valores posibles que puedes pasar con tu solicitud. 

En este ejemplo, vamos a activar los parámetros de consulta expansions y tweet.fields y agregar los siguientes valores:

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `tweet.fields` | `created_at,attachments` |
| expansions | author&#95;id |

Además de agregar los parámetros de consulta, necesitamos añadir la variable de ruta obligatoria, id. Dado que este endpoint devuelve Publicaciones, tenemos que agregar un ID de Publicación válido como valor.

Puedes encontrar el ID de la Publicación navegando a x.com, haciendo clic en una Publicación y luego mirando la URL. Por ejemplo, el ID de Publicación de la siguiente URL es `1228393702244134912`:

`https://x.com/XDevelopers/status/1228393702244134912`

En la pestaña Params, desplázate hacia abajo pasando todos los parámetros de consulta hasta que se muestre la sección “Path Variables”. Agregaremos el ID de Publicación que te gustaría usar como valor para la clave id.

Si has introducido todo lo de este paso correctamente, la pestaña Params debería verse como sigue:

<div id="step-five-send-your-request-and-review-your-response">
  #### Paso cinco: Envía tu solicitud y revisa tu respuesta
</div>

Ahora que todo está configurado en tu solicitud, haz clic en el botón “Send”.

Si todo se configuró correctamente, deberías recibir el siguiente cuerpo de respuesta:


<div id="generating-a-user-access-token-with-postman">
  ### Generar un token de acceso de usuario con Postman:
</div>

<div id="using-oauth-10a-to-generate-a-user-access-token">
  #### Uso de OAuth 1.0a para generar un token de acceso de usuario
</div>

Consulta el proceso de tres pasos que se utiliza en la [colección de prueba del flujo OAuth 1.0a](https://www.postman.com/xapidevelopers/workspace/twitter-s-public-workspace/collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400?ctx=documentation).

<div id="using-oauth-20-to-generate-a-user-access-token">
  #### Uso de OAuth 2.0 para generar un token de acceso de usuario
</div>

Si quieres generar un token de acceso de usuario de OAuth 2.0 en Postman, puedes generar tokens de acceso de OAuth 2.0 para usarlos junto con la [colección de Postman de X API v2](https://www.postman.com/xapidevelopers/workspace/twitter-s-public-workspace/collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400?ctx=documentation). 

Haz clic en la colección de tu espacio de trabajo, ve a la pestaña titulada “Auth” y selecciona el tipo “OAuth 2.0”. Desde ahí, bajo el encabezado “Configure New Token”, ve a donde dice “Configuration Options”. Puedes actualizar el “Grant Type” a “Authorization Code (With PKCE)”.

Debes actualizar tu Callback URL para que coincida con la callback URL asociada con tu Aplicación. Además, debes actualizar los siguientes parámetros:

* Auth URL -  https://x.com/i/oauth2/authorize
* Access Token URL -  https://api.x.com/2/oauth2/token
* Client ID - Tu OAuth 2.0 client ID del Dev Portal
* Client Secret - Si estás usando un confidential client
* Update Scope - Scopes que coincidan con los endpoints a los que quieres conectarte. Ejemplo: “tweet.read users.read” 
* Tu Callback URL (también conocida como redirect URL). Esta debe coincidir con lo que tienes en la configuración de autenticación de tu App.
* State - state

Cuando estés listo, haz clic en “Get New Access Token” para generar un Token de acceso. Si ves un cuadro de diálogo que indica que algo salió mal, puede que necesites presionar el botón Atrás para iniciar sesión. Tendrás que autorizar que tu app tenga acceso a tu cuenta haciendo clic donde dice “Authorize app” en el cuadro de diálogo.

Después de autorizar tu app, serás dirigido de vuelta a Postman, donde podrás ver tu token y seleccionar el botón &quot;Use Token&quot; para empezar a realizar solicitudes en nombre de un usuario autorizado.

Ahora estás listo para usar la colección de Postman.
