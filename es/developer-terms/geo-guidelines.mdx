<div id="introduction">
  ## Introducción
</div>

Posting With Location es la función de geolocalización en la X API. Esta función ayuda a ofrecer una experiencia más relevante para los usuarios al hacer que las publicaciones sean más contextuales. Por ejemplo, “¡Fuegos artificiales!” es mucho más relevante cuando se asocia con una ubicación: “¡Fuegos artificiales!” desde el barrio de SoMa en San Francisco, California. Esperamos colaborar con el ecosistema de desarrolladores para crear experiencias excelentes en torno a esta funcionalidad.

<div id="user-privacy-and-user-control-of-their-location-information">
  ## Privacidad del usuario y control del usuario sobre su información de ubicación
</div>

Para garantizar una experiencia segura y satisfactoria, los desarrolladores deben ser respetuosos con la privacidad de los usuarios al gestionar su información de ubicación, así como proporcionar una coherencia básica en la experiencia de «Publicar con ubicación». A continuación se indican algunos puntos clave que se deben tener en cuenta y que ayudan a proteger la privacidad del usuario y a garantizar que los usuarios comprendan el impacto de sus acciones:

<div id="general-guidelines">
  ## Directrices generales
</div>

* Los usuarios deben dar su consentimiento para usar la función de Publicar con ubicación (activar la ubicación).
* Los usuarios deben dar permiso explícito para que su ubicación exacta se muestre junto con sus Publicaciones. Si la función de ubicación en tu cliente es “persistente”, es decir, una vez que se activa la ubicación siempre acompaña a una Publicación, quizá quieras mostrar solo nombres de lugares (`place_ids`) y permitir que los usuarios decidan además compartir la latitud y la longitud exactas para cada Publicación o como un ajuste alternativo específico.
* Cuando accedas a los datos de ubicación de tus usuarios para publicar una Publicación, no debes almacenar esa información por separado de la Publicación sin el permiso explícito del usuario.
* Debe quedar claro para los usuarios qué nivel de información de ubicación, si la hubiera, se mostrará en asociación con su Publicación. Una forma de hacerlo es mostrando un mapa de cómo se presentará esta información antes de que la Publicación se publique. Si la latitud y la longitud exactas forman parte de la información (o son la única información) que el usuario publica en el cliente, debe quedar claro para el usuario que se publicarán sus coordenadas exactas.
* Los usuarios deben poder activar y desactivar su ubicación cada vez que redacten una Publicación.
* Debes cumplir con los términos de la [Política para desarrolladores](/es/developer-terms/policy.html) de X, incluido el requisito de que no puedes agregar, almacenar en caché ni guardar datos de ubicación u otra información geográfica procedente de la X API, salvo como parte de una Publicación, y no puedes usar dichos datos de ubicación o información geográfica de forma independiente.

<div id="specific-examples">
  ## Ejemplos específicos
</div>

Ejemplo 1: *Sin añadir latitud y longitud explícitas (esta es la interfaz de usuario actual del X Web Client)*

1. Mostrar un enlace a “Añadir tu ubicación” cuando un usuario redacta una Publicación;
2. En segundo plano, utilizar la llamada a la API [reverse&#95;geocode](https://developer.x.com/en/docs/geo/places-near-location/api-reference/get-geo-reverse_geocode.html) de X y pasar la latitud, la longitud, la precisión de la posición y la granularidad preferida por el usuario (que, de forma predeterminada, es “vecindario”);
3. Mostrar el nombre del lugar predeterminado (X intentará ordenarlos de una forma que tenga sentido: el orden es tal que un cliente de X puede simplemente mostrar la lista sin necesidad de modificarla);
4. Permitir que el usuario seleccione una ubicación diferente de la lista (por ejemplo, cambiar de “SoMa, San Francisco” a “South Beach, San Francisco”).
5. Importante: sé transparente con el usuario si vas a mostrar información de ubicación exacta. Asegúrate de que el usuario sepa si se mostrará información de ubicación exacta.
6. Permitir que el usuario elija activar o desactivar la ubicación cada vez que redacte una Publicación.

Ejemplo 2: *Implementaciones de geotagging existentes*

Varias aplicaciones cliente de X tienen implementada la funcionalidad original de geotagging (que solo transmitía lat/lon sin proporcionar place&#95;IDs). El comportamiento predeterminado de la API imita este comportamiento; es decir, como desarrollador no necesitas realizar ningún cambio en tu Aplicación existente para mantener el comportamiento actual. De hecho, si simplemente pasas una coordenada lat/lon a X, X realizará automáticamente la geocodificación inversa de esa latitud y longitud y mostrará el place&#95;ID además de la lat/lon en las áreas donde tengamos datos disponibles.

Asegúrate de que sea transparente para tus usuarios que sus coordenadas exactas se están utilizando y mostrando en X. Una práctica recomendada para las Aplicaciones existentes sería permitir que los usuarios activen o desactiven su ubicación por Publicación. También puedes permitir que tus usuarios compartan solo place&#95;IDs en lugar de coordenadas exactas como una configuración predeterminada “persistente”.

<div id="x-storage-of-user-location-data">
  ## Almacenamiento por parte de X de los datos de ubicación del usuario
</div>

X almacenará los datos enviados por el cliente de la siguiente manera:

* X guardará todos los datos que un usuario decida mostrar públicamente a sus seguidores. Esto es similar a cómo X almacena la marca de tiempo que indica cuándo se hizo la Publicación; es decir, si un usuario publica incluyendo sus coordenadas exactas, X almacenará esta ubicación exacta junto con la Publicación durante todo el tiempo que la Publicación exista. Los usuarios pueden eliminar los datos de ubicación de sus Publicaciones haciendo clic en el botón “Clear my location history” en la página de configuración, tal como se describe [aquí](https://support.twitter.com/articles/122236-how-to-tweet-with-your-location).

<div id="summary">
  ## Resumen
</div>

Añadir una ubicación a una Publicación aporta una enorme cantidad de contexto y valor desde la perspectiva del usuario. Sin embargo, las implementaciones de ubicación en tu cliente deben prestar especial atención a la privacidad y a la transparencia hacia el usuario, considerándolas elementos clave al diseñar un producto centrado en la ubicación.

Si tienes alguna pregunta o comentario sobre privacidad, ubicación u otras funciones de geolocalización, [contáctanos.](https://support.twitter.com/forms/platform)
