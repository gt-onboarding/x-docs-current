<div id="x-developers">
  ## X + Desarrolladores
</div>

En X nos encantan los desarrolladores. Nos entusiasman y sorprenden las herramientas y los servicios que esta comunidad crea aprovechando la potencia de los datos de X. Como parte de nuestro compromiso con esta comunidad, nuestro objetivo es proporcionar un acceso a los datos que sea abierto y justo para los desarrolladores, seguro para las personas en X y beneficioso para la plataforma X en su conjunto. Para cumplir estos objetivos, hemos elaborado la Política para Desarrolladores como una guía para ayudar a las personas a entender nuestras reglas y expectativas sobre el uso adecuado de la API y del Contenido de X.

Esta Política para Desarrolladores (“**Política**”) establece reglas y pautas para los desarrolladores que interactúan con el ecosistema de aplicaciones, servicios, sitio web, páginas web y contenido de X. Forma parte de tu contrato con X, que rige el acceso y el uso de la X API y del Contenido de X (ya sea como parte del Acuerdo para Desarrolladores u otro acuerdo escrito con X). Las infracciones de la Política se consideran infracciones de tu acuerdo. Esta Política puede modificarse periódicamente sin previo aviso. Los términos en mayúsculas utilizados en esta Política que no se definan en esta Política tendrán los significados que se les atribuyen en el [Acuerdo para Desarrolladores](/es/developer-terms/agreement) o en el Acuerdo Marco de Licencia.

<div id="using-this-policy">
  ## Uso de esta política
</div>

Hemos estructurado esta política para que sea lo más fácil de seguir posible. Ten presente la información de las siguientes secciones de esta política mientras utilizas la X API y el Contenido de X:

1. **Prepárate para tener éxito** - Eres responsable de cumplir todas las políticas de X. Es importante que revises y entiendas esta Política, así como las políticas a las que enlazamos en este documento, antes de acceder a la X API y al Contenido de X. El tiempo que dediques a revisar nuestras políticas puede ahorrarte horas de retrabajo en el futuro.

2. **La privacidad y el control son esenciales** - Proteger y defender la privacidad de las personas en X forma parte del ADN mismo de nuestra empresa. Por ello, prohibimos el uso de datos de X de cualquier manera que sea incompatible con las expectativas razonables de privacidad de las personas. Al desarrollar con la X API o acceder al Contenido de X, tienes un papel especial en la protección de este compromiso, sobre todo respetando la privacidad de las personas y ofreciéndoles transparencia y control sobre cómo se utilizan sus datos.

3. **Sigue las Directrices de Uso de la Plataforma** - Que se apruebe tu acceso a la X API y al Contenido de X es solo el primer paso. Nuestras Directrices de Uso de la Plataforma deben ser tu primer recurso siempre que tengas preguntas sobre cómo garantizar el cumplimiento de las políticas para tu uso previsto de la plataforma de X.

A continuación, proporcionamos más detalles sobre lo que significa cada una de estas tres secciones clave. Revísalas detenidamente para asegurarte de que tu uso de la X API y el Contenido de X sea coherente con nuestras políticas.

Si creemos que estás infringiendo esta Política (o cualquier otra política de X), podemos suspender o revocar permanentemente tu acceso a la X API y al Contenido de X.

Por último, ten en cuenta que X puede supervisar tu uso de la X API para mejorar las Aplicaciones de X, examinar cualquier uso comercial y garantizar que cumples con tu caso de uso aprobado y con esta Política.

Gracias por leer y por crear con nosotros. ¡Estamos deseando ver lo que creas!

<div id="set-yourself-up-for-success">
  ## Prepárese para el éxito
</div>

Puede evitar muchos posibles inconvenientes al usar la X API si se asegura de que su servicio haya sido desarrollado de la manera correcta desde el día 1. Esta sección de la Política para Desarrolladores contiene reglas que todos los desarrolladores deben seguir antes de usar la X API o el Contenido de X.

Los planes Free, Basic y Pro (según se describen en [developer.x.com/en](http://developer.x.com/en)) están diseñados para aficionados, creación de prototipos comerciales, integraciones de productos de X en etapas iniciales y para aplicaciones que tienen un número limitado de usuarios finales. Si utiliza la X API y el Contenido de X más allá de este alcance, entonces debe solicitar (o ya estar suscrito a) un plan Enterprise (según se describe en [developer.x.com/en](http://developer.x.com/en)).

**Revisamos todos los usos propuestos de la plataforma para desarrolladores de X para verificar el cumplimiento de las políticas, por lo que se le exige revelar (y actualizar, según corresponda) su uso previsto de la X API y el Contenido de X para que se le conceda y mantenga el acceso.** Todos los desarrolladores nuevos deben [solicitar una cuenta de desarrollador](https://developer.x.com/portal/petition/essential/basic-info) para acceder a la X API. Los desarrolladores actuales sin una cuenta de desarrollador aprobada deben solicitar una, según las instrucciones que reciban de X. Como parte de este proceso, deberá proporcionarnos una descripción por escrito de los usos previstos de la X API y el Contenido de X.

La descripción de su caso de uso es vinculante para usted, y cualquier desviación sustancial de ella puede constituir una infracción de nuestras reglas y dar lugar a medidas de cumplimiento. Debe notificarnos cualquier modificación sustancial de su caso de uso y recibir nuestra aprobación antes de empezar a usar Contenido de X para ese nuevo propósito. El incumplimiento de esta obligación puede dar lugar a la suspensión y terminación de su acceso a la API y a los datos.

**Al desarrollar sobre la X API o acceder al Contenido de X, debe cumplir con TODAS las políticas de X.** Esto incluye esta [Política para Desarrolladores](/es/developer-terms/policy), las [Reglas de Automatización](https://help.x.com/rules-and-policies/x-automation), los [Requisitos de Visualización](https://developer.x.com/developer-terms/display-requirements.html), las [Reglas de Usos Restringidos de la API](/es/developer-terms/restricted-use-cases), las [Reglas de X](https://help.x.com/rules-and-policies/x-rules) y los [Recursos de Marca de X](https://about.x.com/who-we-are/brand-toolkit), así como cualquier otro acuerdo que celebre con X relacionado con su uso de la X API o el Contenido de X, incluidos, entre otros, el [Acuerdo para Desarrolladores](/es/developer-terms/agreement) o un Acuerdo Maestro de Licencia o Pedido (según corresponda). También debe cumplir con cualquier modificación de estas políticas y con cualquier política nueva que X lance. Es su responsabilidad supervisar el uso de su servicio y diseñar su servicio para prevenir infracciones de las políticas de X por parte de las personas que lo utilizan. El incumplimiento de esta obligación puede dar lugar a la suspensión o terminación de su acceso a la API y al Contenido de X.

**No puede registrar múltiples aplicaciones para un solo caso de uso ni para casos de uso sustancialmente similares o superpuestos.** En este contexto, un “caso de uso” es un conjunto coherente de análisis, visualizaciones o acciones ejecutadas a través de una aplicación. Las versiones de una herramienta o servicio de tipo &quot;white label&quot; no están permitidas. Si planea ofrecer versiones &quot;white label&quot; de su aplicación, debe notificárnoslo y recibir nuestra aprobación por separado.

*Como única excepción a estas reglas,* puede crear y usar un máximo de 3 aplicaciones para las instancias de desarrollo, staging y producción del mismo servicio. Estas aplicaciones deben estar registradas en una única cuenta y deben estar claramente identificadas (en el nombre y la descripción) como instancias de dev, staging y prod de un único servicio. **No puede usar aplicaciones de desarrollo o staging con fines de producción.**

**Debe mantener privadas todas las claves de API y otras credenciales de acceso.** No puede usar, ni alentar o facilitar que otros usen, claves de API u otras credenciales de acceso que pertenezcan a terceros.

**Su acuerdo de licencia con X limita su uso de la X API y el Contenido de X. Entre otras cosas, la X API tiene límites de frecuencia que ayudan a garantizar un uso equitativo de los datos y a combatir el spam en la plataforma.** No puede exceder ni eludir los límites de frecuencia, ni ninguna otra limitación o restricción descrita en esta Política o en su acuerdo con X, que aparezcan en el [Sitio para Desarrolladores](https://developer.x.com/) o que le sean comunicadas por X.

**No puede eliminar ni alterar ningún aviso o marca de propiedad en el Contenido de X recibido a través de la X API.** Esto ayuda a garantizar que las personas sepan de dónde proviene el Contenido de X y a quién pertenece.

**Por motivos de integridad de los datos y salud de la plataforma, no puede interferir, interceptar, interrumpir ni deshabilitar ninguna función de la X API o del servicio de X.** En otras palabras, use las APIs tal como se pretende y según se documenta en [developer.x.com](https://developer.x.com/). Consulte nuestras [directrices de HackerOne](https://hackerone.com/x) para obtener más detalles sobre el uso aceptable.

<div id="privacy-and-control-are-essential">
  ## La privacidad y el control son esenciales
</div>

X se toma la privacidad en serio y esperamos que todas las personas que usan X Content y X API hagan lo mismo. Cualquier uso de la plataforma para desarrolladores de X, de X API o de X Content de forma incompatible con las expectativas razonables de privacidad de las personas puede estar sujeto a acciones de cumplimiento, que pueden incluir la suspensión y la revocación del acceso a la API y a X Content.

Tu compromiso con la privacidad y el control debe extenderse a todos los usos de X Content y a todos los aspectos del servicio que crees utilizando nuestra API. Para ello, las personas que usen tu servicio deben comprender y consentir cómo utilizas sus datos y cómo accedes a X en su nombre. Esto puede lograrse proporcionando a las personas una política de privacidad clara, completa y transparente, así como garantizando que obtengas el consentimiento expreso e informado de cada persona que use tu servicio antes de realizar cualquier acción en su nombre. **Ten en cuenta que el hecho de que una persona se autentique en tu servicio no constituye por sí solo consentimiento.**

<div id="consent-permissions">
  ## Consentimiento y permisos
</div>

**En particular, debes obtener el consentimiento expreso e informado de las personas antes de realizar cualquiera de las siguientes acciones:**

* Realizar *cualquier* acción en su nombre. Esto incluye (entre otras cosas):
  * Publicar contenido en X
  * Seguir o dejar de seguir cuentas
  * Modificar la información del perfil o de la cuenta
  * Agregar hashtags o cualquier otro contenido a las Publicaciones

* Republicar contenido al que se accede por medios distintos de la X API u otras herramientas de X

* Usar el Contenido de X de alguien para promocionar un producto o servicio

* Almacenar contenido no público como Mensajes Directos (DMs) o cualquier otra información privada o confidencial

* Compartir o publicar contenido protegido, o cualquier otra información privada o confidencial

**Si tu servicio permite que las personas publiquen contenido en X, debes hacer lo siguiente antes de publicar:**

* Mostrar exactamente qué se va a publicar
* Dejar claro a las personas que usan tu servicio qué información geográfica (si la hubiera) se agregará al contenido

**Si tu servicio permite que las personas publiquen contenido tanto en tu servicio como en X, debes hacer lo siguiente antes de publicar:**

* Obtener permiso para publicar el contenido
* Explicar dónde publicarás el contenido

**Debes respetar el estado protegido y bloqueado de todo el Contenido de X. No puedes mostrar contenido obtenido usando el token de autenticación de una persona a otra persona que no esté autorizada a ver ese contenido.**

* **Cuentas protegidas:** El contenido de una [cuenta protegida](https://help.x.com/safety-and-security/public-and-protected-posts) solo está disponible para las personas que el propietario haya aprobado para seguir esa cuenta. Por lo tanto, si gestionas un servicio que accede a cuentas protegidas, solo puedes hacerlo para mostrar dicho contenido a las personas específicas que tengan permiso para verlo.
* **Cuentas bloqueadas:** Las personas en X pueden [bloquear](https://help.x.com/using-x/blocking-and-unblocking-accounts) el acceso a sus cuentas por cualquier motivo que elijan. No está permitido combinar información obtenida de tokens (o de cualquier otra acción basada en la API) para eludir esta elección.

**Dado que los Mensajes Directos (DMs) son de naturaleza no pública, los servicios que proporcionan funciones de DM deben tomar medidas adicionales para proteger la privacidad personal. No puedes mostrar contenido de DMs a personas que no estén autorizadas a ver ese contenido. Si tu servicio ofrece funcionalidades de DM, también debes:**

* Notificar a las personas si envías eventos de confirmación de lectura para DMs. Puedes hacerlo proporcionando un aviso directamente en tu servicio o mostrando las confirmaciones de lectura de otros participantes en una conversación.
* Obtener consentimiento antes de configurar contenido multimedia para que se envíe en un DM como &quot;compartido&quot; (es decir, reutilizable en múltiples DMs). Si permites que contenido multimedia en un DM se “comparta”, debes proporcionar un aviso claro de que este contenido será accesible para cualquier persona que tenga la URL del contenido multimedia.

<div id="content-compliance">
  ## Cumplimiento de contenido
</div>

**Si almacenas Contenido de X sin conexión, debes mantenerlo actualizado con el estado actual de dicho contenido en X.** En concreto, debes eliminar o modificar cualquier contenido que tengas si se elimina o modifica en X. Esto debe hacerse tan pronto como sea razonablemente posible, o dentro de un plazo de 24 horas tras recibir una solicitud en ese sentido por parte de X o del titular de la cuenta de X correspondiente, o según lo exija tu acuerdo con X o la legislación aplicable. Esto deberá hacerse salvo que la ley lo prohíba, y solo entonces con el permiso expreso y por escrito de X.

El contenido modificado puede adoptar varias formas. Esto incluye (entre otras):

* Contenido que se ha hecho privado o ha pasado a tener estado protegido
* Contenido que ha sido suspendido en la plataforma
* Contenido al que se le han eliminado las etiquetas de geolocalización
* Contenido que ha sido retenido o eliminado de X

<div id="off-x-matching">
  ## Vinculación con datos fuera de X
</div>

**Limitamos las circunstancias en las que puedes vincular a una persona en X con información obtenida o almacenada fuera de X.** La vinculación fuera de X implica asociar Contenido de X, incluido un @handle de X o un ID de usuario, con una persona, hogar, dispositivo, navegador u otro identificador ajeno a X. Solo puedes hacer esto si cuentas con el consentimiento expreso y voluntario de la persona antes de realizar la asociación, o según se describe a continuación.

En situaciones en las que no cuentes con el consentimiento expreso y voluntario de una persona para vincular su identidad en X con un identificador fuera de X, exigimos que cualquier conexión que establezcas se base únicamente en información que una persona razonablemente esperaría que se utilizara para ese fin. Además, en ausencia del consentimiento expreso y voluntario de una persona, solo puedes intentar vincular tus registros sobre alguien con una identidad en X basándote en:

* **Información proporcionada directamente por la persona.** Ten en cuenta que los registros sobre personas con las que no tienes una relación previa, incluidos los datos sobre personas obtenidos de terceros, no cumplen esta norma; y/o
* **Datos públicos.** En este contexto, &quot;datos públicos&quot; se refiere a:
  * Información sobre una persona que obtuviste de un recurso público y de acceso general (como un directorio de miembros de una asociación profesional)
  * Información en X sobre una persona que sea de acceso público, incluyendo:
    * Publicaciones
    * Información de perfil, incluida la biografía de la cuenta y la ubicación declarada públicamente
    * Nombre para mostrar y @handle

<div id="your-privacy-policy">
  ## Tu política de privacidad
</div>

**Debes presentar la política de privacidad de tu servicio a las personas antes de permitirles descargar, instalar o registrarse en tu servicio.** Esta debe incluir, como mínimo, la siguiente información:

* La información que recopilas de las personas que usan tu servicio
* Cómo utilizas y compartes esa información (incluida con X)
* Cómo las personas pueden ponerse en contacto contigo para realizar consultas y solicitudes relacionadas con su información

Tu política de privacidad debe ser coherente con todas las leyes aplicables y no debe ofrecer una protección inferior a la que ofrecen a las personas la [Política de privacidad de X](https://x.com/privacy) y la [política de privacidad de nuestros otros servicios y afiliadas corporativas](https://help.x.com/rules-and-policies/x-services-and-corporate-affiliates). Debes interrumpir tu acceso a la X API y dejar de usar todo el Contenido de X si no puedes cumplir con tu Política de privacidad y/o con la Política de privacidad de X.

<div id="using-geo-data">
  ## Uso de datos geográficos
</div>

**El uso de datos geográficos conlleva restricciones adicionales debido a la naturaleza sensible de esta información.** Si tu servicio añade información de ubicación a Publicaciones, debes informar a las personas de lo siguiente:

* Cuándo añades información de ubicación
* Si añades información de ubicación como una etiqueta geográfica (geotag) o como datos de anotación
* Si la información de ubicación se presenta como un lugar o como coordenadas geográficas

Si tu Aplicación permite que las personas publiquen con su ubicación, debes cumplir íntegramente con las [directrices geográficas](/es/developer-terms/geo-guidelines) de X.

**Cualquier uso de datos de ubicación o información geográfica de forma independiente está prohibido.** No puedes (ni puedes permitir que otros lo hagan) almacenar, agregar o poner en caché datos de ubicación ni otra información geográfica contenida en Contenido de X, salvo como parte de una Publicación. Por ejemplo, no puedes separar los datos de ubicación o la información geográfica de las Publicaciones para mostrar dónde han estado las personas a lo largo del tiempo. Se permiten los mapas de calor y las herramientas relacionadas que muestran actividad geográfica agregada (por ejemplo, el número de personas en una ciudad que usan un hashtag).

<div id="x-passwords">
  ## Contraseñas de X
</div>

**No debes almacenar contraseñas de X ni solicitar que las personas te proporcionen directamente su contraseña de X, credenciales de cuenta o información de su aplicación de desarrollador (incluida la consumer key).** Recomendamos utilizar [Iniciar sesión con X](/es/fundamentals/authentication/guides/log-in-with-x) como herramienta de autenticación para vincular tu servicio con las personas en X.

<div id="platform-usage-guidelines">
  ## Directrices de uso de la plataforma
</div>

¿Te has asegurado de revisar las políticas de X y de configurar correctamente tu acceso a la API? ¿Tu servicio cumple las directrices de privacidad y control de X? Si puedes responder que sí a estas dos preguntas, entonces estás listo para empezar a usar la X API y el Contenido de X. Las Directrices de uso de la plataforma de X proporcionan la ayuda necesaria para garantizar que tu uso del Contenido de X cumpla las normas desde el día 1 y durante todo el ciclo de vida de tu servicio. Te recomendamos revisar estas reglas periódicamente para asegurarte de que tu integración funcione de una manera segura y beneficiosa para las personas en X y para la plataforma de X en su conjunto.

<div id="spam-bots-and-automation">
  ## Spam, bots y automatización
</div>

**Está prohibido usar la X API y los productos para desarrolladores de X para crear spam o participar en cualquier forma de manipulación de la plataforma.** Debes revisar las Reglas de X sobre [manipulación de la plataforma y spam](https://help.x.com/rules-and-policies/platform-manipulation) y asegurarte de que tu servicio no infrinja nuestras políticas ni permita que otras personas lo hagan.

**Los servicios que realizan acciones de escritura, incluyendo publicar Publicaciones, seguir cuentas o enviar Mensajes Directos, deben cumplir con las [Reglas de Automatización](https://help.x.com/rules-and-policies/x-automation).** En particular, debes:

* Obtener siempre el consentimiento explícito antes de enviar a las personas [respuestas automatizadas o Mensajes Directos](https://twittercommunity.com/t/policy-clarification-automated-replies-and-mentions/94444)
* Respetar de inmediato las solicitudes de dejar de ser contactados por ti
* Nunca realizar acciones masivas, agresivas o de tipo spam, incluyendo el [seguimiento masivo](https://twittercommunity.com/t/policy-clarification-aggressive-following-and-inorganic-following-behavior/92769)
* Nunca publicar contenido idéntico o sustancialmente similar en [múltiples cuentas](https://blog.twitter.com/developer/en_us/topics/tips/2018/automation-and-the-use-of-multiple-accounts.html)

**Si operas una cuenta bot basada en API, debes indicar claramente cuál es la naturaleza de la cuenta y quién es responsable de ella.** Nunca debes engañar ni confundir a las personas sobre si tu cuenta es o no un bot. Una buena manera de hacerlo es incluir en la biografía del perfil una declaración de que la cuenta es un bot.

<div id="x-performance-benchmarking">
  ## Evaluación comparativa del rendimiento de X
</div>

**No puedes usar la X API para medir la disponibilidad, el rendimiento, la funcionalidad o el uso de X con fines de evaluación comparativa, competitivos o comerciales.** Por ejemplo, nunca debes usar la X API para:

* Calcular métricas agregadas de X, como el número total de usuarios activos mensuales (MA) o usuarios activos diarios (DA)
* Calcular métricas agregadas de Publicaciones de X, como el número total de Publicaciones realizadas por día o el número de interacciones con la cuenta
* Medir o analizar la capacidad de respuesta de X
* Medir o analizar el spam o la seguridad en X, salvo en los casos permitidos a continuación

**Apoyamos la investigación que ayuda a mejorar la salud de las conversaciones en X.** No puedes divulgar públicamente ninguna investigación ni hallazgos relacionados con, ni desarrollar, crear u ofrecer servicios que utilicen, la X API o Contenido de X que midan, analicen o intenten identificar comportamientos o contenido que infrinjan las políticas de X sin el permiso expreso y por escrito de X.

**Investigadores DSA:** Si necesitas ponerte en contacto con X en relación con el acceso conforme al art. 40 de la Ley de Servicios Digitales (Digital Services Act), comunícate con [**EU-Questions@X.com**](mailto:EU-Questions@X.com). Si deseas solicitar acceso como investigador, envía una [solicitud](https://forms.gle/btDwenPF7M3hgSvw7).

<div id="public-display-of-posts">
  ## Visualización pública de Publicaciones
</div>

**Debes mantener la integridad de todo el Contenido de X que muestres públicamente o a las personas que usan tu servicio.** Si no utilizas [X for Websites](https://developer.x.com/docs/twitter-for-websites/overview) para mostrar contenido, debes usar la X API para recuperar la versión más reciente disponible para su visualización. Si el contenido mostrado deja de estar disponible a través de la X API, debes eliminarlo de tu servicio tan pronto como sea razonablemente posible, o dentro de las 24 horas posteriores a la recepción de una solicitud de eliminación por parte de X, del titular de la cuenta de X correspondiente, o según lo exija la legislación aplicable.

**Hay reglas específicas que debes seguir si muestras Contenido de X sin conexión.** Sigue las [directrices](https://developer.x.com/developer-terms/display-requirements) para usar Publicaciones en emisiones si muestras Publicaciones sin conexión.

**Si incrustas o muestras Publicaciones, debes ponerte en contacto con nosotros sobre tu acceso a la X API si tu sitio supera los 10 millones de impresiones diarias.** X se reserva el derecho de exigir términos adicionales como condición para tu uso de la X API. Las restricciones adicionales sobre el uso para desarrolladores de X for Websites incluyen:

* **Publicaciones incrustadas y/o cronologías incrustadas**
  * Debes proporcionar a las personas un aviso legalmente suficiente que revele de forma completa la recopilación y el uso por parte de X de datos sobre las actividades de navegación en tu sitio web, incluida la publicidad basada en intereses y la personalización. También debes obtener el consentimiento legalmente suficiente de las personas para dicha recopilación y uso
  * Debes proporcionar instrucciones legalmente suficientes sobre cómo las personas pueden inhabilitar la publicidad basada en intereses y la personalización de X, según se describe [aquí](https://help.x.com/x-for-websites-ads-info-and-privacy)

* **Widgets de X for Websites**
  * Debes garantizar que las personas reciban información clara y completa sobre el almacenamiento y el acceso a cookies u otra información en sus dispositivos y que presten su consentimiento a ello, según se describe en el documento de [uso de cookies](https://help.x.com/rules-and-policies/x-cookies) de X, cuando la ley exija proporcionar dicha información y obtener dicho consentimiento

* **Servicios dirigidos a menores de 13 años**
  * Los servicios dirigidos a menores de 13 años deben inhabilitar la personalización de X en cualquier Publicación incrustada y/o cronologías incrustadas estableciendo el parámetro de exclusión voluntaria en ‘true’, según se describe [aquí](https://developer.x.com/docs/twitter-for-websites/privacy)

<div id="content-redistribution">
  ## Redistribución de contenido
