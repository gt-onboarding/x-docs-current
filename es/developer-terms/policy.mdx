<div id="x-developers">
  ## X + Desarrolladores
</div>

X valora a los desarrolladores. Nos entusiasman y sorprenden las herramientas y servicios que esta comunidad crea aprovechando la potencia de los datos de X. Como parte de nuestro compromiso con esta comunidad, nuestro objetivo es proporcionar un acceso a datos que sea abierto y equitativo para los desarrolladores, seguro para las personas en X y beneficioso para la plataforma X en su conjunto. Para promover estos objetivos, hemos elaborado la Política para Desarrolladores como una guía para ayudar a las personas a comprender nuestras reglas y expectativas sobre el uso adecuado de la API y del Contenido de X.

Esta Política para Desarrolladores (“**Política**”) proporciona reglas y directrices para los desarrolladores que interactúan con el ecosistema de aplicaciones, servicios, sitio web, páginas web y contenido de X. Forma parte de tu contrato con X que rige el acceso y uso de X API y del Contenido de X (ya sea como parte del Acuerdo para Desarrolladores u otro acuerdo escrito con X). Las infracciones de la Política se consideran infracciones de tu acuerdo. Esta Política puede modificarse periódicamente sin previo aviso. Los términos definidos con mayúscula inicial utilizados en esta Política que no se definan en ella tendrán los significados respectivos que se les atribuyen en el [Acuerdo para Desarrolladores](/es/developer-terms/agreement) o en el Master License Agreement.

<div id="using-this-policy">
  ## Uso de esta política
</div>

Hemos estructurado esta política para que sea lo más fácil de seguir posible. Ten siempre presente la información de las siguientes secciones de la política mientras utilizas la X API y el Contenido de X:

1. **Prepárate para tener éxito** - Eres responsable de cumplir con todas las políticas de X. Es importante que revises y entiendas esta Política, así como las políticas a las que enlazamos en este documento, antes de acceder a la X API y al Contenido de X. El tiempo que dediques a revisar nuestras políticas puede ahorrarte horas de trabajo adicional más adelante.

2. **La privacidad y el control son esenciales** - La protección y defensa de la privacidad de las personas en X forma parte del ADN central de nuestra empresa. Por ello, prohibimos el uso de datos de X de cualquier manera que sea incompatible con las expectativas razonables de privacidad de las personas. Al desarrollar con la X API o acceder al Contenido de X, desempeñas un papel especial en la protección de este compromiso, sobre todo respetando la privacidad de las personas y proporcionándoles transparencia y control sobre cómo se utilizan sus datos.

3. **Sigue las Directrices de Uso de la Plataforma** - Obtener la aprobación para acceder a la X API y al Contenido de X es solo el primer paso. Nuestras Directrices de Uso de la Plataforma deberían ser tu primera referencia siempre que tengas preguntas sobre cómo garantizar el cumplimiento de las políticas para el uso que planeas hacer de la plataforma de X.

A continuación, encontrarás muchos más detalles sobre lo que significa cada una de estas tres secciones clave. Por favor, revísalas detenidamente para asegurarte de que tu uso de la X API y del Contenido de X sea coherente con nuestras políticas.

Si consideramos que estás infringiendo esta Política (o cualquier otra política de X), podremos suspender o revocar permanentemente tu acceso a la X API y al Contenido de X.

Por último, ten en cuenta que X puede supervisar tu uso de la X API para mejorar las Aplicaciones de X, examinar cualquier uso comercial y garantizar que cumples con tu caso de uso aprobado y con esta Política.

Gracias por leer y gracias por crear con nosotros. ¡Tenemos muchas ganas de ver lo que creas!

<div id="set-yourself-up-for-success">
  ## Prepárate para el éxito
</div>

Puedes evitar muchos posibles inconvenientes al usar la X API si te aseguras de que tu servicio se haya creado de la manera correcta desde el día 1. Esta sección de la Política para Desarrolladores contiene reglas que todos los desarrolladores deben seguir antes de usar la X API o el Contenido de X.

Los planes Free, Basic y Pro (según se describen en [developer.x.com/en](http://developer.x.com/en)) están diseñados para aficionados, creación de prototipos comerciales, integraciones de productos de X en etapas iniciales y para aplicaciones que dan soporte a un número limitado de usuarios finales. Si usas la X API y el Contenido de X más allá de este alcance, entonces debes solicitar (o ya estar suscrito a) un plan Enterprise (según se describe en [developer.x.com/en](http://developer.x.com/en)).

**Revisamos todos los usos propuestos de la plataforma para desarrolladores de X para verificar el cumplimiento de las políticas, por lo que debes revelar (y actualizar, según corresponda) tu uso previsto de la X API y el Contenido de X para que se te conceda y mantenga el acceso.** Todos los nuevos desarrolladores deben [solicitar una cuenta de desarrollador](https://developer.x.com/portal/petition/essential/basic-info) para acceder a la X API. Los desarrolladores actuales sin una cuenta de desarrollador aprobada deben solicitar una, según lo indique X. Como parte de este proceso, deberás proporcionarnos una descripción escrita de tus usos previstos de la X API y el Contenido de X.

La descripción de tu caso de uso es vinculante para ti, y cualquier desviación sustantiva de esta puede constituir una infracción de nuestras reglas y dar lugar a medidas de cumplimiento. Debes notificarnos cualquier modificación sustantiva de tu caso de uso y recibir aprobación antes de comenzar a usar Contenido de X para ese nuevo propósito. El incumplimiento de esta obligación puede dar lugar a la suspensión y terminación de tu acceso a la API y a los datos.

**Al crear con la X API o acceder al Contenido de X, debes cumplir con TODAS las políticas de X.** Estas incluyen esta [Política para Desarrolladores](/es/developer-terms/policy), las [Reglas de automatización](https://help.x.com/rules-and-policies/x-automation), los [Requisitos de visualización](https://developer.x.com/developer-terms/display-requirements.html), las [Reglas de usos restringidos de la API](/es/developer-terms/restricted-use-cases), las [Reglas de X](https://help.x.com/rules-and-policies/x-rules) y los [Recursos de marca de X](https://about.x.com/who-we-are/brand-toolkit), así como cualquier otro acuerdo que celebres con X relacionado con tu uso de la X API o el Contenido de X, incluidos, entre otros, el [Acuerdo para Desarrolladores](/es/developer-terms/agreement) o un Acuerdo o Pedido Maestro de Licencias (según corresponda). También debes cumplir con cualquier modificación de estas políticas y con cualquier política nueva que lance X. Es tu responsabilidad supervisar el uso de tu servicio y diseñar tu servicio para prevenir infracciones de la política de X por parte de las personas que lo utilizan. El incumplimiento de esta obligación puede dar lugar a la suspensión o terminación de tu acceso a la API y al Contenido de X.

**No puedes registrar múltiples aplicaciones para un único caso de uso ni para casos de uso sustancialmente similares o superpuestos.** En este contexto, un “caso de uso” es un conjunto coherente de análisis, visualizaciones o acciones realizadas mediante una aplicación. Las versiones &quot;white label&quot; de una herramienta o servicio no están permitidas. Si planeas crear versiones &quot;white label&quot; de tu aplicación, debes notificarnos y recibir una aprobación independiente de nuestra parte.

*Como única excepción a estas reglas,* puedes crear y usar un máximo de 3 aplicaciones para los entornos de desarrollo, staging y producción del mismo servicio. Estas aplicaciones deben estar registradas en una única cuenta y deben identificarse claramente (en el nombre y la descripción) como instancias de dev, staging y prod de un único servicio. **No puedes utilizar aplicaciones de desarrollo o staging con fines de producción.**

**Debes mantener todas las claves de la API u otras credenciales de acceso en privado.** No puedes usar ni fomentar ni facilitar que otros usen claves de la API u otras credenciales de acceso que pertenezcan a terceros.

**Tu acuerdo de licencia con X limita tu uso de la X API y el Contenido de X. Entre otras cosas, la X API tiene límites de frecuencia que ayudan a garantizar un uso justo de los datos y a combatir el spam en la plataforma.** No puedes exceder ni eludir los límites de frecuencia ni ninguna otra limitación o restricción descrita en esta Política o en tu acuerdo con X, que se enumeren en el [Sitio para Desarrolladores](https://developer.x.com/) o que X te comunique.

**No puedes eliminar ni alterar ningún aviso o marca de propiedad en el Contenido de X recibido a través de la X API.** Esto ayuda a garantizar que las personas sepan de dónde proviene el Contenido de X y a quién pertenece.

**Por motivos de integridad de los datos y de salud de la plataforma, no puedes interferir, interceptar, interrumpir ni desactivar ninguna función de la X API o del servicio de X.** En otras palabras, utiliza las API según lo previsto y documentado en [developer.x.com](https://developer.x.com/). Consulta nuestras [directrices de HackerOne](https://hackerone.com/x) para obtener más detalles sobre el uso aceptable.

<div id="privacy-and-control-are-essential">
  ## La privacidad y el control son esenciales
</div>

X se toma la privacidad en serio y esperamos que todas las personas que utilicen Contenido de X y la X API hagan lo mismo. Cualquier uso de la plataforma para desarrolladores de X, la X API o el Contenido de X de una manera que sea inconsistente con las expectativas razonables de privacidad de las personas puede estar sujeto a medidas de cumplimiento, que pueden incluir la suspensión y la terminación del acceso a la API y al Contenido de X.

Tu compromiso con la privacidad y el control debe abarcar todos los usos del Contenido de X y todos los aspectos del servicio que crees utilizando nuestra API. Con ese fin, las personas que usen tu servicio deben comprender y aceptar cómo utilizas sus datos y cómo accedes a X en su nombre. Esto puede lograrse proporcionando a las personas una política de privacidad clara, completa y transparente, así como garantizando que obtengas un consentimiento expreso e informado de cada persona que use tu servicio antes de realizar cualquier acción en su nombre. **Ten en cuenta que el hecho de que una persona se autentique en tu servicio no constituye por sí solo consentimiento.**

<div id="consent-permissions">
  ## Consentimiento y permisos
</div>

**En particular, debes obtener el consentimiento expreso e informado de las personas antes de realizar cualquiera de las siguientes acciones:**

* Realizar *cualquier* acción en su nombre. Esto incluye (entre otras):
  * Publicar contenido en X
  * Seguir o dejar de seguir cuentas
  * Modificar la información del perfil o de la cuenta
  * Agregar hashtags o cualquier otro contenido a las Publicaciones

* Volver a publicar contenido al que se haya accedido por medios distintos de la X API u otras herramientas de X

* Usar el Contenido de X de alguien para promocionar un producto o servicio

* Almacenar contenido no público, como Mensajes Directos (DMs), o cualquier otra información privada o confidencial

* Compartir o publicar contenido protegido, o cualquier otra información privada o confidencial

**Si tu servicio permite que las personas publiquen contenido en X, debes hacer lo siguiente antes de publicar:**

* Mostrar exactamente qué se va a publicar
* Dejar claro a las personas que usan tu servicio qué información de geolocalización (si la hay) se añadirá al contenido

**Si tu servicio permite que las personas publiquen contenido tanto en tu servicio como en X, debes hacer lo siguiente antes de publicar:**

* Obtener permiso para publicar el contenido
* Explicar dónde publicarás el contenido

**Debes respetar el estado de protección y bloqueo de todo el Contenido de X. No puedes ofrecer contenido obtenido usando el token de autenticación de una persona a otra persona que no esté autorizada a ver ese contenido.**

* **Cuentas protegidas:** El contenido de una [cuenta protegida](https://help.x.com/safety-and-security/public-and-protected-posts) solo está disponible para las personas que hayan sido aprobadas por el propietario para seguir esa cuenta. Por lo tanto, si administras un servicio que accede a cuentas protegidas, solo puedes hacerlo para ofrecer dicho contenido a las personas específicas que tengan permiso para verlo.
* **Cuentas bloqueadas:** Las personas en X pueden [bloquear](https://help.x.com/using-x/blocking-and-unblocking-accounts) el acceso a sus cuentas por cualquier motivo que elijan. Combinar información obtenida a partir de tokens (o cualquier otra acción basada en la API) para eludir esta decisión no está permitido.

**Dado que los Mensajes Directos (DMs) son de naturaleza no pública, los servicios que proporcionan funciones de DM deben tomar medidas adicionales para salvaguardar la privacidad personal. No puedes ofrecer contenido de DMs a personas que no estén autorizadas a ver ese contenido. Si tu servicio proporciona funcionalidad de DM, también debes:**

* Notificar a las personas si envías eventos de confirmación de lectura para DMs. Puedes hacer esto proporcionando un aviso directamente en tu servicio o mostrando las confirmaciones de lectura de otros participantes en una conversación.
* Obtener consentimiento antes de configurar contenido multimedia para que se envíe en un DM como «compartido» (es decir, reutilizable en varios DMs). Si permites que el contenido multimedia en un DM se «comparta», debes proporcionar un aviso claro de que este contenido será accesible para cualquiera que tenga la URL del contenido multimedia.

<div id="content-compliance">
  ## Cumplimiento de contenido
</div>

**Si almacenas Contenido de X sin conexión, debes mantenerlo actualizado con el estado actual de ese contenido en X.** En concreto, debes eliminar o modificar cualquier contenido que tengas si se elimina o modifica en X. Esto debe hacerse tan pronto como sea razonablemente posible, o dentro de las 24 horas siguientes a la recepción de una solicitud para hacerlo por parte de X o del titular de la cuenta de X correspondiente, o según lo exija tu acuerdo con X o la legislación aplicable. Esto debe hacerse, salvo que la ley lo prohíba, y en ese caso solo con el permiso expreso por escrito de X.

El contenido modificado puede adoptar diversas formas. Esto incluye (entre otras cosas, sin limitarse a ellas):

* Contenido que se ha hecho privado o que ha pasado a tener estado protegido
* Contenido que ha sido suspendido de la plataforma
* Contenido al que se le han eliminado las etiquetas de geolocalización
* Contenido que se ha retenido o eliminado en X

<div id="off-x-matching">
  ## Coincidencias fuera de X
</div>

**Limitamos las circunstancias en las que puedes hacer coincidir a una persona en X con información obtenida o almacenada fuera de X.** La coincidencia fuera de X implica asociar Contenido de X, incluido un @handle de X o un ID de usuario de X, con una persona, un hogar, un dispositivo, un navegador u otro identificador fuera de X. Solo puedes hacerlo si cuentas con el consentimiento expreso (opt-in) de la persona antes de realizar la asociación, o según se describe a continuación.

En situaciones en las que no tengas el consentimiento expreso (opt-in) de una persona para vincular su identidad en X con un identificador fuera de X, exigimos que cualquier conexión que establezcas se base únicamente en información que alguien razonablemente esperaría que se utilizara para ese fin. Además, si no cuentas con el consentimiento expreso (opt-in) de la persona, solo puedes intentar hacer coincidir tus registros sobre alguien con una identidad en X basándote en:

* **Información proporcionada directamente por la persona.** Ten en cuenta que los registros sobre personas con las que no tienes una relación previa, incluidos los datos sobre personas obtenidos de terceros, no cumplen esta condición; y/o
* **Datos públicos.** En este contexto, “datos públicos” se refiere a:
  * Información sobre una persona que hayas obtenido de un recurso público y de acceso general (como un directorio de miembros de una asociación profesional)
  * Información en X sobre una persona que sea de acceso público, incluida:
    * Publicaciones
    * Información de perfil, que incluye la biografía de la cuenta y la ubicación declarada públicamente
    * Nombre para mostrar y @handle

<div id="your-privacy-policy">
  ## Tu política de privacidad
</div>

**Debes mostrar la política de privacidad de tu servicio a las personas antes de que se les permita descargar, instalar o registrarse en tu servicio.** Debe incluir, como mínimo, la siguiente información:

* La información que recopilas de las personas que usan tu servicio
* Cómo usas y compartes esa información (incluida la que compartes con X)
* Cómo pueden ponerse en contacto contigo las personas para realizar consultas y solicitudes relacionadas con su información

Tu política de privacidad debe ser coherente con todas las leyes aplicables y no debe ser menos protectora de las personas que la [Política de privacidad de X](https://x.com/privacy) y la [política de privacidad de nuestros otros servicios y empresas afiliadas](https://help.x.com/rules-and-policies/x-services-and-corporate-affiliates). Debes interrumpir tu acceso a la X API y dejar de utilizar todo el Contenido de X si no puedes cumplir con tu Política de privacidad y/o con la de X.

<div id="using-geo-data">
  ## Uso de datos de ubicación geográfica
</div>

**El uso de datos de ubicación geográfica conlleva restricciones adicionales debido a la naturaleza sensible de esta información.** Si tu servicio añade información de ubicación a las Publicaciones, debes informar a las personas de lo siguiente:

* Cuándo añades información de ubicación
* Si añades información de ubicación como geotag o como datos de anotaciones
* Si tu información de ubicación se muestra como un lugar o como coordenadas geográficas

Si tu aplicación permite que las personas publiquen con su ubicación, debes cumplir íntegramente las [directrices de geolocalización](/es/developer-terms/geo-guidelines) de X.

**Está prohibido cualquier uso de datos de ubicación o información geográfica de forma independiente.** No puedes (ni puedes permitir que otros lo hagan) almacenar, agrupar o guardar en caché datos de ubicación u otra información geográfica contenida en Contenido de X, excepto como parte de una Publicación. Por ejemplo, no puedes separar los datos de ubicación o la información geográfica de las Publicaciones para mostrar dónde han estado las personas a lo largo del tiempo. Se permiten los mapas de calor y las herramientas relacionadas que muestren actividad geográfica agregada (por ejemplo, la cantidad de personas en una ciudad que usan un hashtag).

<div id="x-passwords">
  ## Contraseñas de X
</div>

**No debes almacenar contraseñas de X ni solicitar que las personas te proporcionen directamente su contraseña de X, las credenciales de su cuenta o información de su aplicación de desarrollador (incluida la consumer key).** Sugerimos el uso de [Sign-in with X](/es/fundamentals/authentication/guides/log-in-with-x) como herramienta de autenticación para vincular tu servicio con las personas en X.

<div id="platform-usage-guidelines">
  ## Directrices de uso de la plataforma

Las Directrices de uso de la plataforma de X ofrecen la orientación necesaria para garantizar que el uso que haces del Contenido de X cumpla las normas desde el primer día y durante todo el ciclo de vida de tu servicio. <div id="spam-bots-and-automation">
  ## Spam, bots y automatización
</div>

**Está prohibido usar la X API y los productos para desarrolladores para crear spam o participar en cualquier forma de manipulación de la plataforma.** Debes revisar las Reglas de X sobre [manipulación de la plataforma y spam](https://help.x.com/rules-and-policies/platform-manipulation) y asegurarte de que tu servicio no infrinja nuestras políticas ni permita que otras personas lo hagan.

**Los servicios que realizan acciones de escritura, incluyendo publicar Publicaciones, seguir cuentas o enviar Mensajes Directos, deben cumplir con las [Reglas de automatización](https://help.x.com/rules-and-policies/x-automation).** En particular, debes:

* Obtener siempre un consentimiento explícito antes de enviar a las personas [respuestas automatizadas o Mensajes Directos](https://twittercommunity.com/t/policy-clarification-automated-replies-and-mentions/94444)
* Respetar de inmediato las solicitudes de dejar de recibir comunicaciones tuyas
* Nunca realizar acciones masivas, agresivas o similares al spam, incluyendo el [seguimiento masivo](https://twittercommunity.com/t/policy-clarification-aggressive-following-and-inorganic-following-behavior/92769)
* Nunca publicar contenido idéntico o sustancialmente similar a través de [múltiples cuentas](https://blog.twitter.com/developer/en_us/topics/tips/2018/automation-and-the-use-of-multiple-accounts.html)

**Si operas una cuenta bot basada en API, debes indicar claramente de qué tipo de cuenta se trata y quién es responsable de ella.** Nunca debes engañar ni confundir a las personas sobre si tu cuenta es un bot o no. Una buena forma de hacerlo es incluir en la biografía del perfil una declaración de que la cuenta es un bot.

<div id="x-performance-benchmarking">
  ## Evaluación comparativa del rendimiento de X
</div>

**No puedes usar la X API para medir la disponibilidad, el rendimiento, la funcionalidad o el uso de X con fines de evaluación comparativa, competitivos o comerciales.** Por ejemplo, nunca debes usar la X API para:

* Calcular métricas agregadas de X, como el número total de usuarios activos mensuales (MA) o usuarios activos diarios (DA)
* Calcular métricas agregadas de Publicaciones en X, como el número total de Publicaciones publicadas por día o el número de interacciones con cuentas
* Medir o analizar la capacidad de respuesta de X
* Medir o analizar el spam o la seguridad en X, salvo en los casos permitidos a continuación

**Apoyamos la investigación que ayuda a mejorar la salud de las conversaciones en X.** No puedes divulgar públicamente ninguna investigación ni resultados relacionados con el uso de la X API o Contenido de X, ni desarrollar, crear u ofrecer servicios que usen la X API o Contenido de X para medir, analizar o intentar identificar comportamientos o contenido que infrinjan las políticas de X sin el permiso expreso por escrito de X.

**Investigadores DSA:** Si necesitas ponerte en contacto con X en relación con el acceso en virtud del Art. 40 de la Ley de Servicios Digitales, comunícate con [**EU-Questions@X.com**](mailto:EU-Questions@X.com). Si deseas solicitar acceso como investigador, envía una [solicitud](https://forms.gle/btDwenPF7M3hgSvw7).

<div id="public-display-of-posts">
  ## Visualización pública de Publicaciones
</div>

**Debe mantener la integridad de todo el Contenido de X que muestre públicamente o a las personas que usan su servicio.** Si no utiliza [X for Websites](https://developer.x.com/docs/twitter-for-websites/overview) para mostrar contenido, entonces debe usar la X API para recuperar la versión más reciente disponible para su visualización. Si el contenido mostrado deja de estar disponible a través de la X API, debe retirarlo de su servicio tan pronto como sea razonablemente posible, o dentro de las 24 horas posteriores a la recepción de una solicitud de eliminación por parte de X, del titular de la cuenta de X correspondiente, o según lo exija la legislación aplicable.

**Existen reglas específicas que debe seguir si muestra Contenido de X sin conexión.** Siga las [directrices](https://developer.x.com/developer-terms/display-requirements) para usar Publicaciones en emisiones si muestra Publicaciones sin conexión.

**Si incrusta o muestra Publicaciones, debe ponerse en contacto con nosotros acerca de su acceso a la X API si su sitio supera los 10 millones de impresiones diarias.** X se reserva el derecho de exigir condiciones adicionales como requisito para su uso de la X API. Las restricciones adicionales aplicables al uso de X for Websites por parte de desarrolladores incluyen:

* **Publicaciones incrustadas y/o cronologías incrustadas**
  * Debe proporcionar a las personas un aviso legalmente suficiente que revele por completo la recopilación y el uso por parte de X de datos sobre las actividades de navegación en su sitio web, incluso para publicidad basada en intereses y personalización. También debe obtener el consentimiento legalmente suficiente de las personas para dicha recopilación y uso.
  * Debe proporcionar instrucciones legalmente suficientes sobre cómo las personas pueden excluirse de la publicidad basada en intereses y la personalización de X, según se describe [aquí](https://help.x.com/x-for-websites-ads-info-and-privacy).

* **Widgets de X for Websites**
  * Debe asegurarse de que las personas reciban información clara y completa sobre el almacenamiento y el acceso a cookies u otra información en sus dispositivos, y de que presten su consentimiento para ello, según se describe en el documento de X sobre [uso de cookies](https://help.x.com/rules-and-policies/x-cookies), cuando la ley exija proporcionar dicha información y obtener dicho consentimiento.

* **Servicios dirigidos a menores de 13 años**
  * Los servicios dirigidos a menores de 13 años deben excluir la personalización de X en cualquier Publicación incrustada y/o cronologías incrustadas, estableciendo el parámetro de exclusión en ‘true’, según se describe [aquí](https://developer.x.com/docs/twitter-for-websites/privacy).

<div id="content-redistribution">
  ## Redistribución de contenido
</div>

**El mejor lugar para obtener Contenido de X es directamente de X. En consecuencia, restringimos la redistribución de Contenido de X a terceros.** Si proporcionas Contenido de X a terceros, incluidos conjuntos de datos descargables o a través de una API, solo puedes distribuir IDs de Publicación, IDs de Mensajes Directos y/o IDs de Usuarios (excepto según se describe a continuación).

En total, no puedes distribuir más de 1.500.000 IDs de Publicación a ninguna entidad (incluidas varias personas asociadas con una misma entidad) dentro de cualquier período de 30 días, a menos que hayas recibido permiso por escrito de X. Además, los desarrolladores pueden proporcionar hasta 500 objetos de Publicación públicos y/o objetos de Usuario a cada persona que utilice tu servicio diariamente si esto se hace por medios no automatizados (por ejemplo, descarga de hojas de cálculo o archivos PDF).

A los investigadores académicos se les permite distribuir IDs de Publicación y/o IDs de Usuario únicamente con fines de investigación no comercial en nombre de una institución académica, y siempre que dicha investigación haya sido aprobada por X por escrito, o para la revisión por pares o validación de dicha investigación. Solo podrán utilizarse la cantidad de IDs de Publicación o IDs de Usuario que sea necesaria para dicha investigación y que X haya aprobado.

Todo Contenido de X proporcionado a terceros sigue estando sujeto a esta Política, y dichos terceros deben aceptar los [Términos de servicio](https://x.com/tos) de X, la [Política de privacidad](https://x.com/privacy), el [Acuerdo para Desarrolladores](/es/developer-terms/agreement) y la [Política para Desarrolladores](/es/developer-terms/policy) antes de recibir dichas descargas. No puedes permitir que ninguna entidad eluda cualesquiera otras limitaciones o restricciones sobre la distribución de Contenido de X contenidas en esta [Política](/es/developer-terms/policy), en el [Acuerdo para Desarrolladores](/es/developer-terms/agreement) o en cualquier otro acuerdo con X.

**Nota:** Esta Sección no se aplica a los investigadores con acceso a X API conforme al Art. 40 de la Ley de Servicios Digitales de la UE (2022) (“**DSA**”), quienes en su lugar están sujetos a los procedimientos y restricciones establecidos en la DSA y en el Acuerdo para Desarrolladores.

<div id="pay-to-engage">
  ## Pagar para interactuar
</div>

**Tu servicio no debe compensar a las personas por realizar acciones en X, ya que eso genera interacciones no auténticas que degradan la salud de la plataforma.** Al usar la X API, no puedes vender ni recibir compensación monetaria o virtual por ninguna acción en X. Esto incluye, entre otras, Publicaciones, seguir, dejar de seguir, reposts, Me gusta, comentarios y respuestas.

<div id="service-authenticity">
  ## Autenticidad del servicio
</div>

**Debes identificar claramente tu servicio para que las personas puedan entender su origen y propósito.** No uses nombres, logotipos o URL que oculten la identidad y las características de tu servicio, o que impliquen falsamente una afiliación con X o con terceros. Ten en cuenta que está prohibido crear aplicaciones con el fin de vender nombres o impedir que otros los utilicen.

**No puedes usar ninguna URL (incluidas las URL acortadas) de tu servicio que dirija a las personas a:**

* Un sitio que no esté relacionado con tu servicio
* Un sitio de spam o malware
* Un sitio que incite a las personas a infringir las políticas de X

<div id="x-name-logo-and-likeness">
  ## Nombre, logotipo y apariencia de X
</div>

**Solo puedes usar y mostrar el nombre y el logotipo de X para identificar a X como la fuente de Contenido de X.** Nunca debes usar el nombre y el logotipo de X, la insignia del Programa de Socios Oficiales de X ni otras marcas o nombres similares de una manera que genere una falsa impresión de respaldo, patrocinio o asociación con X. Los [recursos de marca](https://about.x.com/who-we-are/brand-toolkit) de X contienen información detallada para ayudarte a usar la marca X de la manera correcta.

**Solo puedes usar la insignia de Cuenta Verificada de X y cualquier otra clasificación mejorada de cuentas según te la comunique X a través de la API.** Esto ayuda a las personas a saber que el contenido que muestra tu servicio es equivalente al que se muestra en X.

<div id="advertising-on-x">
  ## Publicidad en X
</div>

**Existen restricciones sobre cómo y dónde puedes hacer publicidad en relación con el Contenido de X.** Para empezar, tus anuncios no pueden parecerse a una Publicación ni ser razonablemente confundidos con una. Otras reglas sobre publicidad incluyen:

* **Debe haber una separación clara entre el Contenido de X y tus anuncios.** No puedes colocar ningún anuncio dentro del timeline de X que no sea un X Ads.
* **X se reserva el derecho de mostrar publicidad a través de la X API.** Si decides mostrar X Ads una vez que empecemos a entregarlos mediante la API, compartiremos contigo una parte de los ingresos publicitarios de acuerdo con los términos y condiciones correspondientes.
* **No puedes usar Contenido de X ni información obtenida de la X API para dirigir publicidad a personas fuera de la plataforma de X.**

  ## Inicio de sesión con X
</div>

**Debes ofrecer a las personas opciones fáciles de encontrar para iniciar y cerrar sesión en X, por ejemplo, mediante el protocolo OAuth.** La opción *Iniciar sesión con X* debe mostrarse con al menos la misma relevancia que cualquier otra función de registro o inicio de sesión en tu servicio. <div id="x-cards">
  ## X Cards
</div>

**Para garantizar una experiencia de calidad, debes desarrollar tu Card para que se muestre correctamente en todas las plataformas donde se presenten Cards.** Las reglas adicionales que debes seguir al usar Cards incluyen:

* Debes marcar tu Publicación como ‘true’ para contenido multimedia sensible si planeas mostrar dicho contenido multimedia dentro de una Card
* Debes usar HTTPS para alojar todos los recursos dentro de tu Card. Tu Card nunca debe generar advertencias del navegador sobre contenido mixto activo
* El contenido de audio y vídeo debe incluir controles de detener o pausar y, en el caso de los vídeos que reproduzcan contenido automáticamente, debe tener el sonido desactivado de forma predeterminada

**No puedes exceder ni eludir las limitaciones que X impone a cualquier Card, incluido el uso previsto de la Card.** Las restricciones adicionales sobre el uso de Cards incluyen:

* No puedes incluir contenido patrocinado de terceros dentro de las Cards sin la aprobación de X
* No puedes adjuntar incentivos monetarios (incluida moneda virtual) dentro de tu Card ni en X desde tu Card
* No puedes incluir contenido o acciones dentro de tu Card que sean engañosos o no pertinentes para el contexto, como URL y contenido multimedia
* Solo puedes adjuntar una App Card a una Publicación cuando alguien esté promocionando o mencionando explícitamente la aplicación en la Publicación
