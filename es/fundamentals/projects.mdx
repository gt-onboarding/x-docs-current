<div id="introduction">
  ## Introducción
</div>

Los Proyectos te permiten organizar tu trabajo según cómo tengas previsto usar la X API, de modo que puedas gestionar eficazmente tu acceso a la API y supervisar tu uso. Cada Proyecto puede contener una o varias [Apps](/es/resources/fundamentals/developer-apps), según tu nivel de acceso (descrito más adelante en esta página). Utilizarás estas Apps para generar credenciales de [autenticación](/es/resources/fundamentals/authentication), como [API Keys and Secrets](/es/resources/fundamentals/authentication#api-key-and-secret), [Tokens de acceso](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) de usuario y [Token de acceso de la aplicación](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only).

Si bien puedes usar estas claves y tokens de cualquier App para acceder a la X API o a la X Ads API (debes [solicitar acceso adicional](/es/x-ads-api/getting-started/step-by-step-guide) para usar la Ads API), debes usar claves y tokens de una App asociada a un Proyecto para poder usar los endpoints de [X API v2](/es/x-api/introduction).

Si tienes una cuenta de desarrollador, puedes ver y gestionar tus Proyectos en la página [Projects &amp; Apps](https://developer.x.com/en/portal/projects-and-apps) dentro del portal para desarrolladores. [Regístrate](https://developer.x.com/en/portal/petition/essential/basic-info) para obtener una cuenta si aún no tienes una.

<div id="projects-and-x-api-access-levels">
  ### Proyectos y niveles de acceso a X API
</div>

Actualmente, hay 4 niveles de acceso diferentes que se aplican a nivel de Proyecto:

* Free
* Basic
* Pro
* Enterprise

Para obtener más información sobre lo que ofrece cada uno de estos niveles de acceso, visita la página [acerca de X API](/es/x-api/getting-started/about-x-api).

Solo puedes tener un Proyecto con uno de los niveles Free, Basic o Pro.

A continuación describimos algunas diferencias específicas del Proyecto:

<div id="free">
  #### Gratuito
</div>

Este nivel de acceso está disponible para cualquier persona que se haya [registrado](https://developer.twitter.com/en/portal/petition/essential/basic-info) para obtener una cuenta de desarrollador.

Número de aplicaciones en ese Proyecto: 1\
[Límite de consumo de publicaciones](/es/x-api/fundamentals/post-cap): 1,500 publicaciones al mes

Métodos de autenticación:

* [OAuth 2.0 con PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
* Solo aplicación

<div id="basic">
  #### Basic
</div>

Este nivel de acceso está disponible para cualquier persona que se haya suscrito al nivel Basic a través del portal para desarrolladores.

Número de Aplicaciones dentro de ese Proyecto: 2
[Límite de consumo de Publicaciones](/es/x-api/fundamentals/post-cap): 10.000 Publicaciones por mes
Métodos de autenticación:

* [OAuth 2.0 with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
* [OAuth 1.0a](/es/resources/fundamentals/authentication#oauth-1-0a-2)
* Solo de aplicación

<div id="pro">
  #### Pro
</div>

Este nivel de acceso está disponible para cualquier persona que se haya suscrito al nivel Pro a través del portal para desarrolladores.

Número de aplicaciones dentro de ese Proyecto: 3\
[Límite de consumo de Publicaciones](/es/x-api/fundamentals/post-cap): 1 millón de Publicaciones por mes

Métodos de autenticación:

* [OAuth 2.0 with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
* [OAuth 1.0a](/es/resources/fundamentals/authentication#oauth-1-0a-2)
* Solo aplicación

<div id="enterprise">
  #### Enterprise
</div>

Este nivel de acceso se proporciona a cualquier persona que haya solicitado Enterprise, haya sido aprobada por nuestro equipo y haya firmado un contrato mensual con renovación automática a través de un responsable de cuenta de X. [Solicita acceso aquí](https://docs.google.com/forms/d/e/1FAIpQLScO3bczKWO2jFHyVZJUSEGfdyfFaqt2MvmOfl_aJp0KxMqtDA/viewform).

Número de aplicaciones dentro de ese Proyecto: 3+
[Límite de consumo de Publicaciones](/es/x-api/fundamentals/post-cap): más de 50 millones de Publicaciones al mes

Métodos de autenticación:

* [OAuth 2.0 with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
* [OAuth 1.0a](/es/resources/fundamentals/authentication#oauth-1-0a-2)
* Solo aplicación (app-only)

<div id="tweet-caps-and-projects">
  ### Límites de Publicaciones y Proyectos
</div>

Los límites de consumo de Publicaciones se aplican a nivel de Proyecto y, en la práctica, limitan el volumen de publicaciones que puedes recuperar de determinados endpoints de X API v2 en un mes dado.

Obtén más información sobre los [límites de Publicaciones](/es/x-api/fundamentals/post-cap).

<div id="configuring-projects">
  ### Configuración de Proyectos
</div>

<div id="creating-a-project">
  #### Creación de un Proyecto
</div>

Para crear un Proyecto, haz clic en &quot;New Project&quot; en tu [dashboard](https://developer.twitter.com/en/portal/dashboard) o en la página [Projects &amp; Apps](https://developer.twitter.com/en/portal/projects-and-apps) dentro del portal para desarrolladores. Solo podrás ver esta opción si aún no has creado un Proyecto. Se te pedirá que indiques un nombre de Proyecto, una descripción y un caso de uso. También se te pedirá que crees una nueva Aplicación o conectes una Aplicación independiente ya existente.

<div id="creating-or-connecting-an-app-for-your-project">
  #### Creación o conexión de una Aplicación para tu Proyecto
</div>

Si tu Proyecto no tiene una Aplicación asociada, puedes añadir una haciendo clic en el nombre del Proyecto en el panel de control. Desde allí, puedes crear una nueva Aplicación o seleccionar una Aplicación independiente existente para conectarla a tu Proyecto. La Aplicación es donde puedes generar las claves y tokens de autenticación indicados al comienzo de esta guía.

<div id="editing-a-project">
  #### Edición de un Proyecto
</div>

Para editar un Proyecto, haz clic en el nombre de tu Proyecto desde el panel o desde la página de Projects &amp; Apps dentro del portal para desarrolladores. Desde ahí verás los detalles de tu Proyecto y podrás seleccionar &quot;edit&quot; para realizar cambios.

<div id="standalone-apps">
  ### Aplicaciones independientes
</div>

Las aplicaciones independientes son aplicaciones que existen fuera de la estructura de Proyecto. Las credenciales de autenticación asociadas con estas aplicaciones independientes pueden realizar solicitudes satisfactorias a las API estándar v1.1, premium v1.1, Enterprise de X o a la X Ads API. Las aplicaciones independientes no podrán realizar solicitudes a los endpoints de la X API v2 a menos que las conectes a un Proyecto.

Si creaste una aplicación antes de agosto de 2020, será visible en la sección &quot;Standalone Apps&quot; del portal para desarrolladores, en [Projects &amp; Apps](https://developer.twitter.com/en/portal/projects-and-apps). Estarás limitado a un total de diez aplicaciones, incluidas aquellas que están conectadas a tu Proyecto.

Si eres parte de una cuenta de equipo, verás las aplicaciones que te pertenecen en Standalone Apps. Si un compañero de equipo es propietario de una aplicación que forma parte de un Proyecto, podrás ver el nombre de la aplicación y la información del propietario, pero no podrás cambiar su configuración, acceder a sus claves y tokens ni regenerar sus claves y tokens. Deberás ponerte en contacto con el propietario de la aplicación para realizar cualquier cambio en su aplicación.
