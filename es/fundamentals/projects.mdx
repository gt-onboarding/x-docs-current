<div id="introduction">
  ## Introducción
</div>

Los Proyectos te permiten organizar tu trabajo en función de cómo tengas previsto usar la X API, para que puedas administrar eficazmente tu acceso a la API y supervisar tu uso. Cada Proyecto puede contener una o varias [Aplicaciones](/es/resources/fundamentals/developer-apps), según tu nivel de acceso (descrito más adelante en esta página). Utilizarás estas Aplicaciones para generar credenciales de [autenticación](/es/resources/fundamentals/authentication), como [API Keys and Secrets](/es/resources/fundamentals/authentication#api-key-and-secret), [Access Tokens](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) de usuario y [App Access Token](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only).

Si bien puedes usar estas claves y tokens de cualquier Aplicación para acceder a la X API o a la X Ads API (debes [solicitar acceso adicional](/es/x-ads-api/getting-started/step-by-step-guide) para poder usar la Ads API), debes usar claves y tokens de una Aplicación asociada con un Proyecto para poder usar los endpoints de [X API v2](/es/x-api/introduction).

Si tienes una cuenta de desarrollador, puedes ver y administrar tus Proyectos en la página [Projects &amp; Apps](https://developer.x.com/en/portal/projects-and-apps) dentro del portal para desarrolladores. [Regístrate](https://developer.x.com/en/portal/petition/essential/basic-info) para obtener una cuenta si aún no tienes una.

<div id="projects-and-x-api-access-levels">
  ### Proyectos y niveles de acceso de X API
</div>

Actualmente, hay 4 niveles de acceso diferentes que se aplican a nivel de Proyecto:

* Free
* Basic
* Pro
* Enterprise

Para obtener más información sobre lo que ofrece cada uno de estos niveles de acceso, visita la página [acerca de X API](/es/x-api/getting-started/about-x-api).

Solo puedes tener un Proyecto con uno de los niveles Free, Basic o Pro.

Aquí describimos algunas diferencias específicas de los Proyectos:

<div id="free">
  #### Gratuito
</div>

Este nivel de acceso está disponible para cualquier persona que se haya [registrado](https://developer.twitter.com/en/portal/petition/essential/basic-info) para obtener una cuenta de desarrollador.

Número de aplicaciones en ese proyecto: 1\
[Límite de consumo de publicaciones](/es/x-api/fundamentals/post-cap): 1.500 publicaciones al mes

Métodos de autenticación:

* [OAuth 2.0 with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
* Solo aplicación

<div id="basic">
  #### Básico
</div>

Este nivel de acceso se proporciona a cualquier persona que se haya suscrito al nivel de acceso Básico a través del portal para desarrolladores.

Número de aplicaciones dentro de ese Proyecto: 2\
[Límite de consumo de publicaciones](/es/x-api/fundamentals/post-cap): 10.000 publicaciones por mes\
Métodos de autenticación:

* [OAuth 2.0 con PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
* [OAuth 1.0a](/es/resources/fundamentals/authentication#oauth-1-0a-2)
* Solo aplicación

<div id="pro">
  #### Pro
</div>

Este nivel de acceso se ofrece a cualquier persona que se haya suscrito al nivel Pro a través del portal para desarrolladores.

Número de aplicaciones en ese Proyecto: 3
[Límite de consumo de Publicaciones](/es/x-api/fundamentals/post-cap): 1 millón de Publicaciones al mes

Métodos de autenticación:

* [OAuth 2.0 with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
* [OAuth 1.0a](/es/resources/fundamentals/authentication#oauth-1-0a-2)
* Solo aplicación

<div id="enterprise">
  #### Enterprise
</div>

Este nivel de acceso se ofrece a cualquier persona que haya solicitado Enterprise, haya sido aprobada por nuestro equipo y haya firmado un contrato de renovación automática mensual a través de un account manager de X. [Solicita el acceso aquí](https://docs.google.com/forms/d/e/1FAIpQLScO3bczKWO2jFHyVZJUSEGfdyfFaqt2MvmOfl_aJp0KxMqtDA/viewform).

Número de aplicaciones dentro de ese Proyecto: 3+
[Límite de consumo de Publicaciones](/es/x-api/fundamentals/post-cap): más de 50 millones de Publicaciones al mes

Métodos de autenticación:

* [OAuth 2.0 con PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
* [OAuth 1.0a](/es/resources/fundamentals/authentication#oauth-1-0a-2)
* Solo aplicación

<div id="tweet-caps-and-projects">
  ### Límites de publicaciones y Proyectos
</div>

Los límites de consumo de publicaciones se aplican a nivel de Proyecto, lo que en la práctica limita el volumen de publicaciones que puedes recuperar desde ciertos endpoints de X API v2 durante un mes determinado.

Obtén más información sobre los [límites de publicaciones](/es/x-api/fundamentals/post-cap).

<div id="configuring-projects">
  ### Configuración de Proyectos
</div>

<div id="creating-a-project">
  #### Creación de un Proyecto
</div>

Para crear un Proyecto, haz clic en &quot;New Project&quot; en tu [panel](https://developer.twitter.com/en/portal/dashboard) o en la página de [Projects &amp; Apps](https://developer.twitter.com/en/portal/projects-and-apps) dentro del portal para desarrolladores. Solo podrás ver esta opción si aún no has creado un Proyecto. Se te pedirá que indiques un nombre de Proyecto, una descripción y un caso de uso. También se te pedirá que crees una nueva Aplicación o conectes una Aplicación independiente existente.

<div id="creating-or-connecting-an-app-for-your-project">
  #### Crear o conectar una Aplicación para tu Proyecto
</div>

Si tu Proyecto todavía no tiene una Aplicación, puedes añadir una haciendo clic en el nombre del Proyecto en el panel de control. Desde allí, puedes crear una nueva Aplicación o seleccionar una Aplicación independiente existente para conectarla a tu Proyecto. La Aplicación es donde puedes generar las claves y tokens de autenticación indicados al principio de esta guía.

<div id="editing-a-project">
  #### Edición de un Proyecto
</div>

Para editar un Proyecto, haz clic en el nombre de tu Proyecto desde el panel de control o la página Projects &amp; Apps dentro del portal para desarrolladores. Desde ahí verás los detalles de tu Proyecto y podrás seleccionar &quot;Edit&quot; para realizar cambios.

<div id="standalone-apps">
  ### Aplicaciones independientes
</div>

Las Aplicaciones independientes son Aplicaciones que existen fuera de la estructura de Proyecto. Las credenciales de autenticación asociadas a estas Aplicaciones independientes pueden realizar solicitudes satisfactorias a las APIs estándar v1.1, premium v1.1, Enterprise de X o a la X Ads API. Las Aplicaciones independientes no podrán realizar solicitudes a los endpoints de la X API v2 a menos que las conectes a un Proyecto.

Si creaste una Aplicación antes de agosto de 2020, la verás en la sección &quot;Standalone Apps&quot; del portal para desarrolladores, en [Projects &amp; Apps](https://developer.twitter.com/en/portal/projects-and-apps). Estarás limitado a un total de diez Aplicaciones, incluidas aquellas que estén conectadas a tu Proyecto.

Si formas parte de una cuenta de equipo, verás en Standalone Apps las Aplicaciones de las que eres propietario. Si un compañero de equipo es propietario de una Aplicación que forma parte de un Proyecto, podrás ver el nombre de la Aplicación y la información del propietario, pero no podrás cambiar su configuración, acceder a sus claves y tokens ni regenerar sus claves y tokens. Deberás ponerte en contacto con el propietario de la Aplicación para realizar cualquier cambio en su Aplicación.
