<div id="api-key-and-secret">
  ### API Key and Secret
</div>

La API Key y Secret (también conocidas como Consumer Key y Secret) son las credenciales más básicas necesarias para acceder a la X API. Estas credenciales funcionan como el nombre de usuario y la contraseña de tu aplicación de X y son utilizadas por la X API para entender de qué aplicación provienen las solicitudes. 

Estas credenciales pueden ser utilizadas por los [endpoints de autenticación](/es/resources/fundamentals/authentication/api-reference) para generar credenciales adicionales, como [Tokens de acceso de usuario y Secrets](/es/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret), y [Bearer Tokens](/es/resources/fundamentals/authentication/oauth-2-0/bearer-tokens). También necesitas usar estas credenciales junto con tokens de acceso y otros parámetros de autorización para [autorizar solicitudes](/es/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request) que requieren autenticación con Contexto de usuario de OAuth 1.0a. 

<div id="how-to-acquire-an-api-key-and-secret">
  #### Cómo obtener una API Key y Secret
</div>

Para obtener una X API Key y Secret, sigue estos pasos:

1. [Regístrate para obtener una cuenta de desarrollador en X](https://developer.x.com/en/apply-for-access)
2. Crea una [X App](/es/resources/fundamentals/developer-apps) dentro del [portal para desarrolladores](/es/resources/fundamentals/developer-portal). Ten en cuenta que, si deseas usar [X API v2](/es/x-api/introduction), debes añadir tu X App a un [Proyecto](/es/resources/fundamentals/projects).
    

Cuando crees tu X App, se te mostrarán tu API Key y Secret, junto con un Bearer Token. Ten en cuenta que solo mostramos estas credenciales una vez, así que asegúrate de guardarlas en tu gestor de contraseñas o en algún lugar seguro.

Encontrarás más recomendaciones sobre cómo manejar tus claves y tokens en nuestra página de [mejores prácticas de autenticación](/es/resources/fundamentals/authentication/guides/authentication-best-practices), incluidos detalles sobre qué hacer si tus credenciales se han visto comprometidas.
 

<div id="how-to-find-and-regenerate-your-api-key-and-secret-after-app-creation">
  #### Cómo encontrar y regenerar tu API Key and Secret después de crear una Aplicación
</div>

Si ya has creado una Aplicación y necesitas encontrar o regenerar tu API Key and Secret, sigue estos pasos:

1. Ve al portal para desarrolladores
2. Despliega el menú &quot;Projects and Apps&quot; en la barra lateral
3. Abre la Aplicación asociada con la API Key and Secret que deseas encontrar o regenerar
4. Accede a la pestaña Keys and tokens

Desde ahí podrás ver todas las credenciales asociadas con tu Aplicación. 
 

<div id="how-to-use-your-api-key-and-secret">
  #### Cómo usar tu API Key y Secret
</div>

Si solo estás explorando la plataforma para desarrolladores de X, te recomendamos que uses una [herramienta o biblioteca](/es/resources/tools-and-libraries) para ver qué está disponible en la plataforma. Estas herramientas gestionan la autenticación de forma sencilla y pueden ahorrarte mucho tiempo y frustración. En concreto, recomendamos [empezar con Postman](/es/tutorials/postman-getting-started) o [Insomnia](https://insomnia.rest/) para desarrolladores que están comenzando. 

Si te interesa crear una solicitud desde cero, consulta nuestra guía sobre [cómo autorizar una solicitud OAuth 1.0a](/es/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request).
