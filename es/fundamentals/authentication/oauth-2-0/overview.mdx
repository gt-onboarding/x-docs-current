<div id="bearer-token-also-known-as-app-only">
  ### Bearer Token (también conocido como app-only)
</div>

OAuth 2.0 Bearer Token autentica las solicitudes en nombre de tu [aplicación de desarrollador](/es/resources/fundamentals/developer-apps). Como este método es específico de la aplicación, no implica a ningún usuario. Este método se utiliza normalmente para desarrolladores que necesitan acceso de solo lectura a información pública. 

Este método de autenticación requiere que incluyas un Bearer Token en tu solicitud, que puedes generar en la sección Claves y tokens de tus aplicaciones de desarrollador. Aquí tienes un ejemplo de cómo se ve una solicitud con un Bearer Token de ejemplo:

```json
  curl "https://api.x.com/2/tweets?ids=1261326399320715264,1278347468690915330" \
  -H "Authorization: Bearer AAAAAAAAAAAAAAAAAAAAAFnz2wAAAAAAxTmQbp%2BIHDtAhTBbyNJon%2BA72K4%3DeIaigY0QBrv6Rp8KZQQLOTpo9ubw5Jt?WRE8avbi"
```

Las llamadas a la API que usan autenticación de solo aplicación están [limitadas por frecuencia](/es/resources/fundamentals/rate-limits) por endpoint a nivel de la Aplicación.

Para usar este método, necesitarás un Bearer Token, que puedes generar pasando tu API Key y Secret al endpoint [POST oauth2/token](/es/resources/fundamentals/authentication/api-reference#post-oauth2-token), o generándolo en la sección &quot;keys and tokens&quot; de la configuración de tu Aplicación en el [portal para desarrolladores](/es/resources/fundamentals/developer-portal).

Si quieres revocar un Bearer Token, puedes usar el endpoint [POST oauth2/invalidate&#95;token](/es/resources/fundamentals/authentication/api-reference#post-oauth2-invalidate-token), o hacer clic en &quot;revoke&quot; junto al Bearer Token en la sección &quot;keys and tokens&quot; de la configuración de tu Aplicación.

<div id="oauth-20-authorization-code-flow-with-pkce">
  ### Flujo de código de autorización OAuth 2.0 con PKCE
</div>

El flujo de código de autorización OAuth 2.0 con PKCE te permite autenticarte en nombre de otro usuario y te ofrece un mayor control sobre los permisos (*scopes*) de una Aplicación, además de mejorar los flujos de autorización en múltiples dispositivos. En otras palabras, los desarrolladores que crean Aplicaciones para personas que usan X tendrán más control sobre la información que su Aplicación solicita a sus usuarios, de modo que solo tengas que pedir a tus usuarios finales los datos y la información que necesitas.

Este protocolo de autorización moderno te permitirá presentar a tus usuarios finales un flujo de consentimiento más sencillo para autorizar tu Aplicación, que muestra únicamente los permisos específicos que has solicitado. Esto no solo reduce tu carga de datos, sino que también puede generar una mayor confianza por parte de los usuarios finales.
