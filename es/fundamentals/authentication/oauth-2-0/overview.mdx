<div id="bearer-token-also-known-as-app-only">
  ### Bearer Token (también conocido como app-only)
</div>

OAuth 2.0 Bearer Token autentica solicitudes en nombre de tu [aplicación de desarrollador](/es/resources/fundamentals/developer-apps). Como este método es específico de la aplicación, no involucra a ningún usuario. Este método suele utilizarse cuando los desarrolladores necesitan acceso de solo lectura a información pública. 

Este método de autenticación requiere que envíes un Bearer Token con tu solicitud, el cual puedes generar en la sección Claves y tokens de tus aplicaciones de desarrollador. Aquí tienes un ejemplo de cómo se ve una solicitud con un Bearer Token falso:

```json
  curl "https://api.x.com/2/tweets?ids=1261326399320715264,1278347468690915330" \
  -H "Authorization: Bearer AAAAAAAAAAAAAAAAAAAAAFnz2wAAAAAAxTmQbp%2BIHDtAhTBbyNJon%2BA72K4%3DeIaigY0QBrv6Rp8KZQQLOTpo9ubw5Jt?WRE8avbi"
```

Las llamadas a la API que utilizan autenticación solo de App están [limitadas por frecuencia](/es/resources/fundamentals/rate-limits) por endpoint a nivel de la App.

Para usar este método, necesitarás un Bearer Token, que puedes generar pasando tu API Key y Secret al endpoint [POST oauth2/token](/es/resources/fundamentals/authentication/api-reference#post-oauth2-token), o generándolo en la sección «keys and token» de la configuración de tu App en el [portal para desarrolladores](/es/resources/fundamentals/developer-portal).

Si deseas revocar un Bearer Token, puedes usar el endpoint [POST oauth2/invalidate&#95;token](/es/resources/fundamentals/authentication/api-reference#post-oauth2-invalidate-token), o hacer clic donde dice «revoke» junto al Bearer Token en la sección «keys and tokens» de la configuración de tu App.

<div id="oauth-20-authorization-code-flow-with-pkce">
  ### OAuth 2.0 Authorization Code Flow with PKCE
</div>

OAuth 2.0 Authorization Code Flow with PKCE te permite autenticarte en nombre de otro usuario y tener un mayor control sobre los scopes de una Aplicación, además de mejorar los flujos de autorización en múltiples dispositivos. En otras palabras, los desarrolladores que crean aplicaciones para personas en X tendrán más control sobre la información que su Aplicación solicita a sus usuarios, de modo que solo tengas que pedir a tus usuarios finales los datos y la información que realmente necesitas.

Este protocolo de autorización moderno te permitirá presentar a tus usuarios finales un flujo de consentimiento más simple y optimizado para autorizar tu Aplicación, que solo muestra los scopes específicos que les has solicitado. Esto no solo reduce la carga de datos que gestionas, sino que también puede generar una mayor confianza por parte de los usuarios finales.
