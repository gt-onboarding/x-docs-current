<div id="using-and-generating-an-app-only-bearer-token">
  ### Uso y generación de un Bearer Token de solo aplicación
</div>

Un Bearer Token permite a los desarrolladores disponer de un punto de entrada más seguro para usar las APIs de X y es una de las características fundamentales de OAuth 2.0. 

La autenticación que utiliza un Bearer Token también se conoce como autenticación de solo aplicación. Un Bearer Token es una secuencia de bytes de formato no especificado que generas usando un script, como un comando curl. También puedes obtener un Bearer Token en el portal para desarrolladores, dentro de la sección de claves y tokens de la configuración de tu aplicación. Puedes encontrar más información sobre esta funcionalidad en la [documentación oficial de OAuth](https://oauth.net/2/bearer-tokens/).

<div id="when-are-they-used">
  #### ¿Cuándo se usan?
</div>

Los productos que requieren el uso de un Bearer Token son los siguientes:

* [Engagement API](/es/x-api/enterprise-gnip-2.0/fundamentals/engagement-api)
* [Account Activity API](/es/x-api/enterprise-gnip-2.0/fundamentals/account-activity)
* Otras API que utilizan la autenticación OAuth 2.0 con Bearer Token, como los endpoints de v2 y Labs.

<div id="prerequisites">
  #### Requisitos previos
</div>

Necesitarás [registrarte para obtener una cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) y haber creado una [X App](/es/resources/fundamentals/developer-apps). Una vez que los tengas, también necesitarás obtener las claves de la API que se encuentran en el [portal para desarrolladores](/es/resources/fundamentals/developer-portal). Sigue los pasos a continuación:

1. Inicia sesión en tu cuenta de X en developer.x.com.
2. Ve al [panel de X App](https://developer.x.com/content/developer-twitter/en/apps) y abre la X App para la cual deseas generar tokens de acceso.
3. Ve a la página &quot;keys and tokens&quot;.
4. Encontrarás las claves de la API, los tokens de acceso de usuario y el Bearer Token en esta página.

<div id="how-to-generate-a-bearer-token">
  ### Cómo generar un Bearer Token
</div>

Puedes encontrar el Bearer Token de tu aplicación junto con el resto de tus &quot;claves y tokens&quot;.

Copia la siguiente solicitud cURL en tu línea de comandos después de sustituir las siguientes claves de API de consumidor obtenidas previamente de tu [X App](/es/resources/fundamentals/developer-apps). Ten en cuenta que las claves de API de consumidor utilizadas en esta página se han dado de baja y no funcionarán para solicitudes reales.

* **API key** `<API key>` p. ej. `xvz1evFS4wEEPTGEFPHBog`
* **API secret key** `<API secret key>` p. ej. `L8qq9PZyRg6ieKGEKhZolGC0vJWLw8iEJ88DRdyOg`

  ```bash
  curl -u "$API_KEY:$API_SECRET_KEY" \
  --data 'grant_type=client_credentials' \
  'https://api.x.com/oauth2/token'
  ```

Aquí tienes un ejemplo de cómo debería verse la solicitud cURL con tus claves de API introducidas:

```bash
   curl -u 'xvz1evFS4wEEPTGEFPHBog:L8qq9PZyRg6ieKGEKhZolGC0vJWLw8iEJ88DRdyOg' \
  --data 'grant_type=client_credentials' \
  'https://api.x.com/oauth2/token'
```

Así se vería la respuesta. Ten en cuenta que se trata de un Bearer Token deshabilitado:

`{"token_type":"bearer","access_token":"AAAAAAAAAAAAAAAAAAAAAMLheAAAAAAA0%2BuSeid%2BULvsea4JtiGRiSDSJSI%3DEUifiRBkKG5E2XzMDjRfl76ZC9Ub0wnz4XsNiRVBChTYbJcE3F"}`

Nuestro Bearer Token que usaríamos para autenticarnos en los recursos con OAuth 2.0 sería:

`AAAAAAAAAAAAAAAAAAAAAMLheAAAAAAA0%2BuSeid%2BULvsea4JtiGRiSDSJSI%3DEUifiRBkKG5E2XzMDjRfl76ZC9Ub0wnz4XsNiRVBChTYbJcE3F`
