<div id="general">
  ## General
</div>

<AccordionGroup>
  <Accordion title="¿Qué es OAuth?">
    OAuth es un protocolo de autenticación que permite a los usuarios aprobar que una aplicación actúe en su nombre sin compartir su contraseña. Puedes encontrar más información en [oauth.net](http://oauth.net/).
  </Accordion>

  <Accordion title="¿Cómo genero tokens de acceso?">
    Necesitas una [aplicación de X](/es/resources/fundamentals/developer-apps) para generar tokens de acceso. Obtén más información sobre los tokens de acceso [aquí](/es/resources/fundamentals/authentication#oauth-1-0a-2). 
  </Accordion>

  <Accordion title="¿Cómo creo una aplicación de X?">
    Necesitas una [cuenta de desarrollador](/es/resources/fundamentals/developer-portal) para crear una [aplicación de X](/es/resources/fundamentals/developer-apps). Puedes registrarte para obtener una [aquí](https://developer.x.com/en/portal/petition/essential/basic-info).
  </Accordion>

  <Accordion title="Si ya tengo una aplicación de X, ¿cómo puedo verla y editarla?">
    Puedes ver y editar tu aplicación desde el [panel de aplicaciones de X](https://developer.x.com/content/developer-twitter/en/apps) si has iniciado sesión en tu cuenta de X en developer.x.com.
  </Accordion>
</AccordionGroup>

<div id="technical">
  ## Técnico
</div>

<AccordionGroup>
  <Accordion title="¿Cuánto tiempo dura un token de acceso?">
    Los tokens de acceso no tienen una fecha de caducidad explícita. Un token de acceso se invalidará si un usuario revoca explícitamente una aplicación en la configuración de su cuenta de X, o si X suspende una aplicación. Si una aplicación es suspendida, habrá una nota en el panel de [X app](/es/resources/fundamentals/developer-apps) que indicará que ha sido suspendida.
  </Accordion>

  <Accordion title="¿Qué ocurre si un token de acceso se vuelve inválido?">
    Supón que el token de acceso de un usuario *puede* volverse inválido en cualquier momento. Si esto ocurre, solicita al usuario que vuelva a autorizar la aplicación. Es importante gestionar esta situación de manera adecuada para ofrecer una buena experiencia de usuario.
  </Accordion>

  <Accordion title="La página de registro de la aplicación pregunta sobre acceso de lectura/escritura. ¿Qué se considera una operación de escritura?">
    Muchos usuarios confían en que una aplicación lea su información, pero no necesariamente que cambie la información de su perfil de usuario o publique nuevas publicaciones. Actualizar información a través de la X API, ya sea el nombre, la ubicación o agregar una nueva publicación, requiere una solicitud HTTP POST. Cualquier método de la API que requiera una solicitud HTTP POST se considera un método de escritura y requiere acceso de lectura y escritura.
  </Accordion>
</AccordionGroup>
