La ordenación está disponible para la mayoría de los endpoints de colección de Ads API (que devuelven una lista de objetos). Dependiendo del endpoint, hay varios parámetros habilitados para la ordenación. La mayoría de los campos devueltos por estos parámetros (excepto los IDs y ENUMs) se podrán ordenar.

| Colección de la API | Parámetros ordenables |
| :--- | :--- |
| [Accounts](/es/x-ads-api/campaign-management#accounts) | `created_at`, `updated_at`, `deleted`, `name` |
| [Funding Instruments](/es/x-ads-api/campaign-management#funding-instruments) | `created_at`, `updated_at`, `deleted`, `funded_amount_local_micro`, `start_time`, `end_time` |
| [Campaigns](/es/x-ads-api/campaign-management#campaigns) | `created_at`, `updated_at`, `deleted`, `name`, `start_time`, `end_time`, `daily_budget_amount_local_micro`, `total_budget_amount_local_micro`, `standard_delivery` |
| [Line Items](/es/x-ads-api/campaign-management#line-items) | `created_at`, `updated_at`, `deleted`, `bid_amount_local_micro` |
| [Cards](https://developer.x.com/en/docs/ads/creatives/api-reference/image-app-download) | `created_at`, `updated_at`, `deleted`, `name` |
| [Promoted Accounts](/es/x-ads-api/campaign-management#advertiser-api) | `created_at`, `updated_at`, `deleted`, `paused` |
| [Promoted Tweets](/es/x-ads-api/campaign-management#promoted-tweets) | `created_at`, `updated_at`, `deleted`, `paused` |
| [App Event Tags](/es/x-ads-api/measurement/mobile-conversions#app-event-tags) | `created_at`, `updated_at`, `deleted`, `post_view_attribution_window`, `post_engagement_attribution_window`, `assisted_conversion`, `provider_app_event_name` |

<div id="getting-started">
  ## Primeros pasos
</div>

En tu solicitud de API, simplemente añade `sort_by=[attribute name]-[asc/desc]`, donde el atributo es un atributo válido que se devuelve en el JSON de la respuesta a tu solicitud GET.

Por ejemplo, puedes usar `?name-asc` para ordenar por nombre alfabéticamente o `?name-desc` para ordenar en orden inverso.
