Para tener éxito con la Ads API, es importante comprender cómo se relacionan entre sí las entidades en X Ads. Este tutorial repasa los conceptos básicos de la jerarquía de objetos de X Ads.

![Jerarquía de la Ads API de X](/images/adsapi-heirarchy.png.twimg.1920.png)

<div id="ad-group-line-items">
  ## Grupo de anuncios (line items)
</div>

Los grupos de anuncios, conocidos como [line items](https://developer.x.com/content/developer-twitter/en/docs/tutorials/ads-api-hierarchy-terminology#line-items) en la Ads API, existen dentro de las campañas y se utilizan para segmentar y pujar por un conjunto de usuarios de X. Los anunciantes promocionan Publicaciones o contenido multimedia (por ejemplo, vídeos que se promocionan como anuncios in-stream) asociándolos con un line item. Consulta también Grupo de anuncios.

<div id="account-media">
  ## Contenido multimedia de la cuenta
</div>

Una colección de creatividades que se han cargado en una cuenta de anuncios como [account media](/es/x-ads-api/creatives#account-media) y que puede incluir [recursos de pre-roll](/es/x-ads-api/campaign-management#video-views-preroll-objective) de video promocionado (también conocidos como anuncios in-stream) e imágenes que se pueden promocionar en la [X Audience Platform](https://developer.x.com/content/developer-twitter/en/docs/ads/measurement/overview/twitter-audience-platform).

<div id="ads-accounts">
  ## Cuentas de anuncios
</div>

En el nivel superior de la jerarquía de X Ads se encuentran las cuentas de anuncios. Una cuenta de anuncios puede ser administrada por varios usuarios de X. @Furni, nuestra tienda de muebles de demostración, sería un ejemplo de un usuario que administra una cuenta de anuncios. Identificamos las cuentas de anuncios mediante un account&#95;id, que es una cadena alfanumérica en [base36](https://en.wikipedia.org/wiki/Base36).

En cada cuenta, almacenamos varios tipos de entidades que se pueden reutilizar en distintas campañas, como audiencias personalizadas que se pueden usar para la segmentación, contenido multimedia de la cuenta, es decir, videos o imágenes, y tarjetas (cards), que son creativos ampliados que puedes agregar a una Publicación.

<div id="amplify-video">
  ## Amplify Video
</div>

Al cargar un activo de vídeo, la entidad que realiza la carga puede definir una categoría para el vídeo (por ejemplo, Vídeo de Publicación, GIF de Publicación, Imagen de Publicación y Amplify Video). La categoría Amplify Video es obligatoria cuando se intenta cargar un vídeo que se mostrará como un anuncio in-stream. Estos vídeos requieren un procesamiento especial para garantizar que se puedan mostrar y, por lo tanto, se requiere la categoría de contenido multimedia Amplify Video.

<div id="analytics">
  ## Analytics
</div>

Las métricas de Analytics ayudan a socios y anunciantes a comprender el rendimiento del contenido que promocionan en X. Esto incluye información como impresiones, clics, visualizaciones de video y gasto publicitario. Además, los socios y anunciantes pueden obtener métricas detalladas para varios segmentos de las audiencias a las que llegan.

La Ads API admite dos formas de obtener métricas detalladas de rendimiento de campañas: de forma síncrona y asíncrona.

Con las solicitudes de [analytics síncronos](/es/x-ads-api/analytics#synchronous-analytics), las métricas solicitadas se devuelven en la respuesta. El endpoint síncrono admite intervalos de tiempo cortos y es ideal para consultar el rendimiento de la campaña en tiempo real.

Con los endpoints de [analytics asíncronos](/es/x-ads-api/analytics#asynchronous-analytics), las métricas solicitadas están disponibles en un archivo de resultados descargable después de que la tarea asociada haya terminado de procesarse. Los endpoints asíncronos admiten la solicitud de métricas en un intervalo de tiempo mucho más largo, permiten devolver datos segmentados y están pensados para recuperar un volumen de datos mucho mayor, lo que es ideal para generar informes o completar datos históricos.

<div id="campaigns">
  ## Campañas
</div>

Cada campaña que ejecuta el anunciante debe crearse bajo un instrumento de financiación existente. Una campaña te permite definir cosas como el nombre, la programación y el presupuesto de una campaña publicitaria. Siguiendo con el ejemplo de Furni, podríamos crear una campaña llamada “Furni home decor”.

Cabe señalar que una cuenta de anuncios está limitada a como máximo 200 Campañas activas; este límite puede aumentarse por el gestor de cuenta del anunciante. Una Campaña se considera activa hasta que se alcanza su fecha de finalización o se agota su presupuesto. Una campaña en pausa sigue considerándose una campaña activa a efectos del límite de campañas.

<div id="cards">
  ## Tarjetas
</div>

Las tarjetas ofrecen una forma de extender visualmente una Publicación para incluir una imagen, un vídeo o elementos de interfaz de usuario (UI). La Ads API admite varios tipos de tarjetas que se pueden usar en Publicaciones y luego promocionar en campañas.

<div id="creatives">
  ## Creativos
</div>

Los anunciantes pueden promocionar Publicaciones en campañas que incluyan creativos, como imágenes, GIFs y vídeos. Estos creativos se pueden asociar con X cards, que permiten adjuntar fotos, vídeos y experiencias de contenido multimedia enriquecidas a las Publicaciones. Los creativos también se pueden adjuntar directamente a las Publicaciones. Por ejemplo, puedes incluir texto y un vídeo (o hasta 3 imágenes) y crear una Publicación promocionada.

Cada elemento de línea debe tener al menos un creativo asociado para poder activarse. Hay varias opciones de creativos disponibles, incluidas Cuentas promocionadas para conseguir nuevos seguidores, creativos de contenido multimedia (como una imagen o un vídeo) para elementos de línea en toda TX Audience Platform, o Publicaciones promocionadas.

<div id="funding-instruments">
  ## Instrumentos de financiación
</div>

También dentro de la cuenta de anuncios se encuentran los instrumentos de financiación, que son las diferentes fuentes de financiación que el anunciante ha configurado para usar en X Ads. Estos abarcan desde órdenes de inserción que haya firmado con X hasta una tarjeta de crédito asociada a la cuenta.

<div id="line-items">
  ## Elementos de línea
</div>

Dentro de una campaña, puedes tener uno o más grupos de anuncios, también conocidos como elementos de línea en la Ads API. Los elementos de línea pueden considerarse como diferentes segmentos de una campaña, cada uno con su propio conjunto de segmentación y su propio creativo. Las campañas que contienen múltiples elementos de línea suelen denominarse “campañas de grupos de anuncios”.

Uno de los beneficios de tener múltiples elementos de línea dentro de una sola campaña es que puedes crear una mayor variedad de combinaciones de segmentación y creativos.

En nuestra campaña “Furni home decor”, podríamos crear dos elementos de línea: uno podría ser “Compradores de vivienda en San Francisco” y el otro podría ser “Compradores de vivienda en Londres”.

Se permiten hasta 100 elementos de línea por campaña, y una cuenta de anuncios está limitada a 4.000 elementos de línea activos en total en todas las campañas.

<div id="mobile-app-promotion-map">
  ## Promoción de aplicaciones móviles (MAP)
</div>

La promoción de aplicaciones móviles (MAP) es un conjunto de productos que permite a los anunciantes promocionar apps móviles a través de X. Entre ellos se incluyen instalaciones de apps, conversiones y re-engagement.

<div id="mobile-app-conversion-tracking-mact">
  ## Seguimiento de conversiones de aplicaciones móviles (MACT)
</div>

La medición de promociones de aplicaciones móviles en X permite a los anunciantes medir el éxito de las campañas publicitarias en X diseñadas para impulsar instalaciones u otras conversiones dentro de la aplicación. Un socio de medición móvil de X permite a un anunciante gestionar qué conversiones quiere medir de las aplicaciones que promociona en X. Descripción general de MACT

<div id="media-library">
  ## Biblioteca de contenido multimedia
</div>

Los endpoints de la Biblioteca de contenido multimedia permiten gestionar imágenes, GIFs y videos para cuentas de X Ads. Los recursos multimedia de la biblioteca se pueden usar en Publicaciones y para crear tarjetas. También se pueden reutilizar en múltiples creativos, lo que elimina la necesidad de subir el mismo recurso varias veces.

<div id="promoted-accounts">
  ## Cuentas promocionadas
</div>

Las Cuentas promocionadas (campañas de seguidores) forman parte de A quién seguir, que sugiere cuentas que las personas no siguen actualmente y que pueden resultarles interesantes. Las Cuentas promocionadas ayudan a presentar una variedad aún más amplia de cuentas que pueden ser de su interés.

Las Cuentas promocionadas permiten a los anunciantes promocionar su cuenta y hacer crecer su audiencia en X. Las campañas de seguidores se muestran en múltiples ubicaciones de la plataforma de X —incluidas las cronologías de inicio, A quién seguir y los resultados de búsqueda— y esta sugerencia se marca como Promocionado para distinguirla de otras cuentas recomendadas.

<div id="promotable-users">
  ## Usuarios promocionables
</div>

Cada cuenta de X Ads puede tener uno o más usuarios promocionables, que son usuarios que podemos promocionar a través de la cuenta de anuncios. El usuario @Furni de Furni sería un ejemplo de un usuario promocionable COMPLETO en la cuenta de anuncios de Furni, lo que significa que podemos ejecutar todo tipo de campañas para @Furni y sus Publicaciones. Solo puede haber un usuario promocionable COMPLETO por cuenta de anuncios.

Ten en cuenta que también puede haber otros tipos de usuarios promocionables en una cuenta de anuncios. Por ejemplo, si quisiéramos promocionar las Publicaciones de @Jack sobre productos de Furni, se convertiría en un usuario promocionable solo para retweets en la cuenta, lo cual se indica con RETWEETS&#95;ONLY. Los anunciantes deben ponerse en contacto con su representante de cuenta de X para obtener información sobre cómo agregar usuarios de retweet a su cuenta de anuncios.

<div id="promoted-video">
  ## Video promocionado
</div>

Los videos promocionados son videos dentro de Publicaciones promocionadas por los que los anunciantes pagan. Estos aparecen en la cronología de Inicio, en la parte superior de los resultados de búsqueda en X y en otros lugares de la plataforma, y están claramente marcados como “Promocionado”.

<div id="scheduled-tweets">
  ## Publicaciones programadas
</div>

Las publicaciones programadas permiten que anunciantes o usuarios creen una Publicación y la programen para que se publique en una fecha posterior. Además de poder crear y administrar estas Publicaciones, la API permite asociarlas con un line item para que se promocionen una vez que la Publicación se publique. Esto permite a los anunciantes preparar Publicaciones nativas y planificar con antelación las creatividades de sus campañas antes de cualquier iniciativa clave. Por ejemplo, preparar la creatividad de una Publicación para que se publique inmediatamente tras el anuncio de un nuevo producto.

<div id="segmentation">
  ## Segmentación
</div>

La segmentación permite que socios y anunciantes obtengan métricas desglosadas por valores de segmentación específicos —como edad, dispositivos o idiomas— y solo está disponible a través de nuestros endpoints de analítica asíncrona. Para solicitar métricas segmentadas, usa el parámetro de solicitud segmentation&#95;type. Para obtener más detalles sobre las opciones de segmentación, consulta Métricas y segmentación.

<div id="tailored-audience-list">
  ## Audiencia personalizada - Lista
</div>

Las audiencias personalizadas a partir de listas son uno de los tres tipos de Tailored Audiences, creadas cargando un archivo que contiene tus propios datos de usuarios y clientes. Se comparan tus listas con los usuarios de X para que puedas dirigirte a ellos directamente en tus campañas. Consulta estas restricciones.

<div id="tailored-audience-web">
  ## Audiencia personalizada - Web
</div>

La Audiencia personalizada desde la web, uno de los tres tipos de Audiencias personalizadas, es un conjunto de usuarios de X que han visitado tu sitio web. Para usar Audiencias personalizadas desde la web, debes tener instalado nuestro píxel de X en tu sitio web o utilizar uno de nuestros socios de gestión de etiquetas admitidos. Consulta estas restricciones.

<div id="targeting-criteria">
  ## Criterios de segmentación
</div>

Como mencionamos anteriormente, los criterios de segmentación están asociados a un line item. Contamos con una variedad de tipos de segmentación que puedes usar, desde ubicación, palabras clave y género hasta atributos más granulares, como comportamientos e intereses de los usuarios. Para el line item “Home Buyers in San Francisco”, podríamos segmentar el comportamiento “home buying” junto con la ubicación “San Francisco”.

Para obtener información sobre los diferentes tipos de segmentación disponibles, visita la página Targeting Overview en nuestro sitio para desarrolladores.

  ## Twitter Object Nest (TON)
</div>

Twitter Object Nest (TON) almacena contenido multimedia y otros recursos subidos mediante los endpoints de Media Upload para usarlos como Creatives.
