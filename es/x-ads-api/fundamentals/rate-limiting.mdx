La Advertiser API tiene límites de frecuencia similares a los de la API REST v1.1, tal como se documenta aquí: [Limitación de frecuencia de la API REST en v1.1](https://developer.x.com/content/developer-twitter/en/docs/basics/rate-limits). A diferencia de la API REST v1.1, no existe un índice programático de los límites por endpoint. Los límites de frecuencia de cada endpoint y las ventanas de restablecimiento se comunican mediante encabezados de respuesta HTTP. Toda la limitación de frecuencia en Ads API utiliza OAuth 1.0A.

<div id="user-level-and-ad-account-level-limits">
  ## Límites a nivel de usuario y a nivel de cuenta de anuncios
</div>

Hay dos tipos de límites de frecuencia: a nivel de token de usuario y a nivel de cuenta de anuncios. Un subconjunto de endpoints está configurado para utilizar la limitación de frecuencia a nivel de cuenta de anuncios. Un token de usuario es el [token de acceso OAuth](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) que utilizas para autenticarte y llamar a la Ads API. Cada token de usuario puede tener acceso a una o más cuentas de anuncios.

Los desarrolladores deben utilizar el límite de frecuencia a nivel de cuenta de anuncios cuando se devuelva en los encabezados de la respuesta y solo utilizar el límite a nivel de usuario cuando no se encuentre el límite de cuenta de anuncios.

Los límites de frecuencia a nivel de usuario se expresan mediante estos encabezados: **x-rate-limit-limit**, **x-rate-limit-remaining** y **x-rate-limit-reset**

Para los endpoints con limitación de frecuencia a nivel de cuenta de anuncios habilitada, los límites de frecuencia se expresan mediante estos encabezados: **x-account-rate-limit-limit**, **x-account-rate-limit-remaining** y **x-account-rate-limit-reset**.

Los límites de frecuencia a nivel de cuenta de anuncios se proporcionan solo para endpoints de tipo GET para permitir que las aplicaciones sincronicen datos de entidades (como objetos de campaña o de línea de pedido) desde un único token de usuario que accede a varias cuentas de anuncios. No se garantiza que las acciones de escritura utilicen los mismos límites de frecuencia a nivel de cuenta de anuncios.

Para los endpoints limitados a nivel de cuenta de anuncios, el límite de frecuencia a nivel de usuario se establece en un valor alto que representa una cuota global para toda tu aplicación. Cuando esté disponible, el límite de frecuencia a nivel de cuenta de anuncios debe tener prioridad para controlar tu volumen de solicitudes.

<div id="best-practices">
  ## Mejores prácticas
</div>

1. Guarda en tu base de datos una marca de tiempo con la última sincronización y, al solicitar datos, cuando corresponda, pide los datos con la opción `sort_by=updated_at-desc` para que puedas detener tu proceso de sincronización una vez que llegues a datos anteriores a tu última marca de tiempo sincronizada. Esto evitará sincronizar los mismos datos de forma redundante.

2. Solicita múltiples entidades en una sola solicitud: Algunos endpoints te permiten especificar una lista de valores separados por comas para recuperar múltiples elementos de datos similares. Esto puede reducir el número total de llamadas que realizas y, por lo tanto, aprovechar el límite de frecuencia de manera más eficiente.

3. Solicita el valor máximo de `count` en tus solicitudes: Se recomienda encarecidamente llamar a algunos endpoints, como [GET accounts/:account&#95;id/targeting&#95;criteria](/es/x-ads-api/campaign-management#targeting-criteria), con el valor máximo de `count` para devolver 1000 objetos en lugar del valor predeterminado de 200.

<div id="faqs">
  ## Preguntas frecuentes
</div>

*¿Es posible aumentar los límites de frecuencia para una cuenta publicitaria específica o para nuestra aplicación?*

Por lo general, no podemos aumentar los límites de frecuencia y están configurados para admitir incluso las cuentas publicitarias de mayor tamaño. Implementa primero las prácticas recomendadas enumeradas en este documento y, si los límites de frecuencia siguen afectando tu capacidad para escalar o alcanzar tus objetivos comerciales, ponte en contacto con tus representantes de X Ads API con todos los detalles sobre el caso de uso y las solicitudes asociadas.
