La Advertiser API está sujeta a límites de frecuencia de forma similar a la REST API v1.1, tal como se describe aquí: [REST API Rate Limiting in v1.1](https://developer.x.com/content/developer-twitter/en/docs/basics/rate-limits). A diferencia de la REST API v1.1, no existe un índice programático de los límites por endpoint. Los límites de frecuencia por endpoint y las ventanas de restablecimiento se comunican mediante encabezados de respuesta HTTP. Todos los límites de frecuencia de la Ads API utilizan OAuth 1.0A.

<div id="user-level-and-ad-account-level-limits">
  ## Límites a nivel de usuario y a nivel de cuenta de anuncios
</div>

Hay dos tipos de límites de frecuencia: a nivel de token de usuario y a nivel de cuenta de anuncios. Un subconjunto de endpoints tiene habilitada la limitación de frecuencia a nivel de cuenta de anuncios. Un token de usuario es el [OAuth access token](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) que utilizas para autenticarte y llamar a la Ads API. Cada token de usuario puede tener acceso a una o más cuentas de anuncios.

Los desarrolladores deben utilizar el límite de frecuencia a nivel de cuenta de anuncios cuando se devuelva en los encabezados de respuesta y utilizar únicamente el límite a nivel de usuario cuando no se encuentre el límite de la cuenta de anuncios.

Los límites de frecuencia a nivel de usuario se expresan mediante estos encabezados: **x-rate-limit-limit**, **x-rate-limit-remaining** y **x-rate-limit-reset**.

Para los endpoints con la limitación de frecuencia a nivel de cuenta de anuncios habilitada, los límites de frecuencia se expresan mediante estos encabezados: **x-account-rate-limit-limit**, **x-account-rate-limit-remaining** y **x-account-rate-limit-reset**.

Los límites de frecuencia a nivel de cuenta de anuncios se proporcionan solo para endpoints GET para permitir que las aplicaciones sincronicen datos de entidades (como objetos de campaña o de línea de pedido) desde un único token de usuario que accede a varias cuentas de anuncios. No se garantiza que las acciones de escritura utilicen los mismos límites de frecuencia a nivel de cuenta de anuncios.

Para los endpoints con límites de frecuencia a nivel de cuenta de anuncios, el límite de frecuencia a nivel de usuario se establece en un valor alto que representa una cuota global para toda tu aplicación. Cuando esté disponible, el límite de frecuencia a nivel de cuenta de anuncios debe tener prioridad para controlar tu volumen de solicitudes.

<div id="best-practices">
  ## Mejores prácticas
</div>

1. Guarda en tu base de datos una marca de tiempo con la última sincronización y, al solicitar datos, cuando corresponda, pide los datos con la opción `sort_by=updated_at-desc` para poder detener tu proceso de sincronización cuando llegues a datos anteriores a tu última marca de tiempo sincronizada. Esto evitará sincronizar los mismos datos de forma redundante.

2. Solicita varias entidades en una sola solicitud: algunos endpoints te permiten especificar una lista de valores separados por comas para recuperar múltiples conjuntos de datos similares. Esto puede reducir la cantidad total de llamadas que realizas y, por lo tanto, aprovechar el límite de frecuencia de forma más eficiente.

3. Solicita el valor máximo de `count` en tus solicitudes: en algunos endpoints, como [GET accounts/:account&#95;id/targeting&#95;criteria](/es/x-ads-api/campaign-management#targeting-criteria), se recomienda encarecidamente llamarlos con el valor máximo de `count` para devolver 1000 objetos en lugar del valor predeterminado de 200.

<div id="faqs">
  ## Preguntas frecuentes
</div>

*¿Es posible aumentar los límites de frecuencia para una cuenta publicitaria en particular o para nuestra aplicación?*

Por lo general, no podemos aumentar los límites de frecuencia y estos se establecen para admitir a las cuentas publicitarias más grandes. Implementa primero las prácticas recomendadas indicadas en este documento y, si los límites de frecuencia siguen afectando tu capacidad de escalar o de alcanzar tus objetivos comerciales, ponte en contacto con tus contactos de X Ads API con todos los detalles sobre el caso de uso y las solicitudes correspondientes.
