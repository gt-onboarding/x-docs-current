<div id="how-to-get-access-to-the-ads-api">
  ## Cómo obtener acceso a la Ads API
</div>

1. Regístrate para obtener una [cuenta de desarrollador](https://developer.x.com/en/apply-for-access).
2. Crea una [aplicación de desarrollador](https://developer.x.com/app) y protege tu token.
3. Visita [ads.x.com/help](https://ads.x.com/help) para solicitar acceso a la Ads API para cada una de tus aplicaciones de desarrollador.

<Note>**Nota:** Si ya estás desarrollando en la plataforma para desarrolladores de X y tienes una cuenta de desarrollador, pasa al paso tres.</Note>

<div id="step-one-signup-for-a-developer-account">
  ### Paso uno: Regístrate para obtener una cuenta de desarrollador
</div>

Para realizar solicitudes a cualquiera de los productos de API de X, primero debes registrarte para obtener una cuenta de desarrollador. Dentro del portal para desarrolladores, crea un Proyecto y una aplicación de desarrollador. Esto te proporcionará un conjunto de credenciales que utilizarás para autenticar todas las solicitudes a la API.

<Button href="https://developer.x.com/en/apply-for-access">
  Regístrate para obtener una cuenta de desarrollador
</Button>

<div id="step-two-save-your-apps-key-and-tokens-and-keep-them-secure">
  ### Paso dos: Guarda las claves y tokens de tu aplicación y mantenlos seguros
</div>

Dentro de tu aplicación de desarrollador, se te proporcionará un conjunto de claves de API (API Keys, también conocidas como Consumer Keys). También tendrás la posibilidad de generar un conjunto de Tokens de acceso que pueden utilizarse para hacer solicitudes en nombre de tu cuenta personal de X, y un Bearer Token que puede utilizarse para autenticar endpoints que requieren OAuth 2.0 Bearer Token. Como estas claves y tokens no caducan a menos que se regeneren, sugerimos crear variables de entorno o usar un gestor de contraseñas seguro.

Toma también nota de tu App ID, que se puede encontrar en la URL de la página de detalles de tu aplicación. Esto facilitará solicitar acceso a la Ads API en el siguiente paso.

<Note>
**Nota:** Tus claves y tokens solo se mostrarán una vez en el portal para desarrolladores, por lo que es importante que almacenes estas credenciales en tu gestor de contraseñas tan pronto como las generes.

Si pierdes u olvidas las claves y tokens, tendrás que regenerarlos, lo que crea nuevas claves y tokens e invalida las anteriores. Esto significa que tendrás que actualizar cualquier integración que puedas haber configurado con tus credenciales anteriores.

Obtén más información sobre nuestras [mejores prácticas de autenticación](https://developer.x.com/en/docs/authentication/guides/authentication-best-practices).
</Note>

<div id="step-three-apply-for-access-to-the-ads-api">
  ### Paso tres: solicita acceso a la Ads API
</div>

En este punto ya tendrás [acceso básico a la X API](https://developer.x.com/en/docs/twitter-api/v1), pero no a la funcionalidad específica de la X Ads API. A continuación, deberás solicitar acceso y obtener aprobación para usar la Ads API. Visita [ads.x.com/help](https://ads.x.com/help) para enviar una solicitud de acceso a la Ads API para cada una de tus aplicaciones de desarrollador.

<div id="access-tiers">
  ### Niveles de acceso
</div>

Como parte del proceso de solicitud, deberás indicar qué nivel de acceso necesitas. Obtén más información sobre [permisos a nivel de Aplicación y a nivel de Cuenta de anuncios](https://developer.x.com/en/docs/twitter-ads-api/obtaining-ads-account-access#levels-of-access).

<div id="conversion-only">
  #### Solo conversiones
</div>

Acceso a los endpoints de conversiones móviles y web con permisos de lectura y escritura.

<div id="standard-access">
  #### Acceso estándar
</div>

Acceso a los endpoints de Analytics, Campaign Management, Creatives, Custom Audiences y Conversion con permisos de lectura y escritura.

<Note>**Nota:** Después de que tu aplicación sea aprobada para el acceso a la Ads API, deberás [regenerar cualquier token de acceso de usuario](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens) para poder [enviar correctamente solicitudes autenticadas a la Ads API](https://developer.x.com/en/docs/twitter-ads-api/making-authenticated-requests).</Note>

<Note>**Nota:** Los desarrolladores de la Ads API que solicitaron acceso antes de julio de 2023 pueden tener distintos niveles de acceso y permisos, y pueden estar limitados a cinco tokens OAuth. Consulta nuestra guía sobre [cómo aumentar el acceso](https://developer.x.com/en/docs/twitter-ads-api/increasing-access) para obtener acceso a endpoints adicionales o levantar los límites de tokens para las aplicaciones existentes.</Note>

<div id="make-your-first-request">
  ## Realiza tu primera solicitud
</div>

Para probar tu acceso a la Ads API, realiza una solicitud al endpoint [GET accounts](https://developer.x.com/en/docs/twitter-ads-api/campaign-management/api-reference/accounts). Este endpoint devolverá las cuentas publicitarias a las que tiene acceso el usuario actualmente autorizado. Usa los ID de las cuentas publicitarias de esta solicitud en solicitudes posteriores a la API para leer y escribir datos de una cuenta publicitaria específica. Usando [Twurl](https://developer.x.com/en/docs/tutorials/using-twurl) en la línea de comandos, la solicitud sería similar a la siguiente.

**Solicitud de ejemplo**

```bash
twurl -H ads-api.x.com "/11/accounts"
```

**Ejemplo de respuesta**
