<div id="mact-overview">
  ## Descripción general de MACT
</div>

La medición de [promoción de aplicaciones móviles](https://biz.x.com/mobile-app-promotion) de X permite a los anunciantes hacer un seguimiento del éxito de las campañas publicitarias en X que están diseñadas para generar instalaciones u otras conversiones dentro de la aplicación. Un socio de medición móvil de X permite al anunciante gestionar qué conversiones quiere medir en las aplicaciones que está promocionando en X. Una vez que el anunciante haya configurado el tipo y las ventanas temporales de los eventos que desea rastrear, el socio de medición móvil enviará todos esos eventos a X y recibirá inmediatamente datos de atribución que indican la campaña y el creativo con los que interactuó el usuario y que generaron la conversión.

<div id="conversion-events">
  ### Eventos de conversión
</div>

Para poder reportar un [evento de conversión](https://developer.x.com/en/docs/x-ads-api/measurement/api-reference/conversion-event) a X y obtener datos de atribución, el socio debe enviar el ID de la aplicación móvil, el tipo y la marca de tiempo de la conversión, así como un valor con hash del ID de publicidad/dispositivo. Proporcionar estos datos registrará el evento de conversión en la plataforma de X y también devolverá la atribución según corresponda. Además, el anunciante puede optar por proporcionar metadatos adicionales sobre el evento de conversión, como el valor de una compra, el ID de un producto o un método de registro.

<div id="post-conversion_event">
  #### POST conversion&#95;event
</div>

Registra un evento de conversión de medición móvil. La respuesta indicará la atribución de X o de X Audience Platform (TAP).

Este endpoint está relacionado con el endpoint [GET conversion&#95;attribution](https://developer.x.com/en/docs/x-ads-api/measurement/api-reference/conversion-attribution#get-conversion-attribution).

En la respuesta se indicará atribución de X, de TAP o ninguna atribución. El nodo `twitter_attribution` siempre estará presente y tendrá un valor de `null` cuando no haya atribución de X (consulta el ejemplo de respuesta más abajo). Si se asigna atribución a TAP, el nodo `tpn_attribution` estará presente y se completará en consecuencia. Consulta la [descripción general de TAP](https://developer.x.com/en/docs/x-ads-api/measurement/overview/twitter-audience-platform) para obtener más información.

Hay varios parámetros opcionales disponibles para establecer metadatos asociados a cada evento de conversión. Estos metadatos no afectan a los cálculos de atribución.

<div id="resource-url">
  ##### URL del recurso
</div>

`https://ads-api.x.com/12/conversion_event`

<div id="parameters">
  ##### Parameters
</div>

| Name | Description |
| :--- | :--- |
| app&#95;id  <br />*required* | El identificador único en la tienda de aplicaciones correspondiente.<br /><br />Type: int, string<br /><br />Example: `333903271`, `com.vine.android` |
| conversion&#95;time  <br />*required* | El momento del evento de conversión en formato de marca de tiempo ISO-8601, con milisegundos añadidos.<br /><br />Type: string<br /><br />Example: `2014-05-22T02:38:28.103Z` |
| conversion&#95;type  <br />*required* | El tipo de evento de conversión.<br /><br />Type: enum<br /><br />Possible values: `PURCHASE`, `SIGN_UP`, `INSTALL`, `RE_ENGAGE`, `UPDATE`, `TUTORIAL_COMPLETE`, `RESERVATION`, `ADD_TO_CART`, `ADD_TO_WISHLIST`, `LOGIN`, `CHECKOUT_INITIATED`, `SEARCH`, `LEVEL_ACHIEVED`, `ACHIEVEMENT_UNLOCKED`, `CONTENT_VIEW`, `SHARE`, `INVITE`, `ADDED_PAYMENT_INFO`, `SPENT_CREDITS`, `RATED` |
| hashed&#95;device&#95;id  <br />*required* | El IDFA o AdID con hash HMAC&#95;SHA-256.<br /><br />Type: string<br /><br />Example: `ABCD1234XYZ` |
| os&#95;type  <br />*required* | El tipo de sistema operativo de la aplicación.<br /><br />Type: enum<br /><br />Possible values: `IOS`, `ANDROID` |
| click&#95;window  <br />*optional* | La ventana de atribución por clic para este evento, en días.<br /><br />Type: int<br /><br />**Note**: `click_window` debe ser mayor o igual que view&#95;through&#95;window<br /><br />Default: `14`  <br />Possible values: `1`, `7`, `14`, `30` |
| device&#95;ip&#95;address  <br />*optional* | Dirección IPv4 o IPv6 del dispositivo cuando ocurrió el evento de conversión.<br /><br />Type: string<br /><br />Example: `192.133.78.1` |
| level  <br />*optional* | Un nivel asociado con este evento.<br /><br />Type: int<br /><br />Example: `2` |
| non&#95;twitter&#95;engagement&#95;time  <br />*optional* | La hora de la última interacción ajena a Twitter anterior a la conversión.<br /><br />Type: string<br /><br />Example: `2014-05-22T02:38:28.103Z` |
| non&#95;twitter&#95;engagement&#95;type  <br />*optional* | El tipo de interacción ajena a Twitter previa al evento de conversión.<br /><br />Type: enum<br /><br />Possible values: `CLICK`, `VIEW` |
| number&#95;items  <br />*optional* | Número de elementos asociados con este evento.<br /><br />Type: int<br /><br />Example: `2` |
| price&#95;currency  <br />*optional* | Se espera que sea un código ISO 4217 que indique la moneda asociada con este evento.<br /><br />Type: String<br /><br />Examples: `EUR`,`USD`,`JPY` |
| price&#95;micro  <br />*optional* | Un importe de precio asociado a este evento en micros.<br /><br />Type: int<br /><br />Example: `123450000` |
| user&#95;payment&#95;info  <br />*optional* | Un valor booleano que indica si la información de pago del usuario se almacena en la aplicación asociada con este evento.<br /><br />Type: bool<br /><br />Possible values: `true` or `false` |
| view&#95;through&#95;window  <br />*optional* | La ventana de atribución por visualización para este evento, en días.<br /><br />**Note**:<br /><br />`click_window` debe ser mayor o igual que view&#95;through&#95;window.<br /><br />Type: int<br /><br />Default: `1`  <br />Possible values: `0`, `1`, `7`, `14`, `30` |

</div>

Realiza una consulta a X para comprobar la atribución de conversiones sin registrar un evento de conversión. La respuesta indicará la atribución de X.

Esto está relacionado con el endpoint [POST conversion&#95;event](https://developer.x.com/en/docs/x-ads-api/measurement/api-reference/conversion-event#post-conversion-event).

En la respuesta se indicará atribución de X, de TAP o ninguna atribución. El nodo `twitter_attribution` siempre estará presente y tendrá un valor de `null` cuando no haya atribución de X, y se completará como se muestra en el ejemplo de respuesta a continuación. Si se indica atribución de TAP, habrá un nodo `tpn_attribution` presente y se completará en consecuencia. Consulta la [descripción general de TAP](https://developer.x.com/en/docs/twitter-ads-api/measurement/twitter-audience-platform/overview) para obtener más información.
