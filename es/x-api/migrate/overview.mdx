<div id="what-is-the-x-api-v2">
  ## ¿Qué es la X API v2?
</div>

La X API v2 es ahora la X API principal y es donde se concentran la inversión en producto y la innovación. Hemos colaborado con desarrolladores para crear la próxima generación de la X API y servir mejor a nuestra diversa comunidad de desarrolladores. Basándonos en los comentarios de los desarrolladores, hemos reconstruido la API para satisfacer mejor un conjunto más amplio de necesidades, hemos introducido nuevas funciones y endpoints, y hemos mejorado la experiencia para desarrolladores.

La X API v2 es ahora la X API principal y es donde se concentran la inversión en producto y la innovación. En los últimos años, hemos colaborado con desarrolladores y hemos reconstruido la API para satisfacer mejor un conjunto más amplio de necesidades, introducir nuevas funciones y endpoints, y mejorar la experiencia para desarrolladores. Estamos comprometidos a seguir construyendo una plataforma abierta para desarrolladores y nos entusiasma ver lo que construyas con la X API v2.

<div id="why-migrate">
  ## ¿Por qué migrar?
</div>

La X API v2 se basa en una infraestructura moderna y más sostenible, e incluye tanto endpoints de reemplazo mejorados para la versión estándar v1.1 y los productos Enterprise como también funcionalidad totalmente nueva. Recomendamos encarecidamente que los clientes de las APIs heredadas (v1.1 y Enterprise) comiencen a migrar a v2, ya que tenemos la intención de retirarlas eventualmente. Usa la X API para escuchar y analizar la conversación pública, interactuar con las personas en X e innovar.

En esta sección, hablaremos sobre los endpoints y la funcionalidad.

Algunas de las mejoras de la API incluyen:

* [Un diseño coherente en todos los endpoints](/es/x-api/fundamentals/consistency)

* [Recibir y filtrar datos con nueva información contextual generada a partir de anotaciones de Publicaciones](/es/x-api/fundamentals/post-annotations)
* [Acceso a nuevas métricas](/es/x-api/fundamentals/metrics)
* [Identificar y filtrar fácilmente las conversaciones que pertenecen a un hilo de respuesta](/es/x-api/fundamentals/conversation-id)
* [Funcionalidad avanzada y mayor acceso a datos para investigadores académicos](https://developer.x.com/content/developer-twitter/en/products/twitter-api/academic-research)
* [Funcionalidad de recuperación y redundancia para endpoints de streaming](/es/x-api/posts/filtered-stream/integrate/recovery-and-redundancy-features)

* Especificación OpenAPI para crear nuevas bibliotecas y hacer un seguimiento de los cambios de forma más transparente

<div id="discover-new-and-updated-response-objects">
  ### Descubre objetos de respuesta nuevos y actualizados
</div>

Los siguientes seis objetos de datos están disponibles con los endpoints de la v2:

| Objeto | Descripción |
|:--------|:-------------|
| [Tweet](/es/x-api/fundamentals/data-dictionary#tweet) | El objeto Publicación tiene una larga lista de campos de nivel raíz, como `id`, `text` y `created_at`. Los objetos Publicación también son el objeto padre de varios objetos secundarios, incluidos `user`, `media`, `poll` y `place`. |
| [User](/es/x-api/fundamentals/data-dictionary#user) | El objeto User contiene metadatos de la cuenta de usuario de X que describen al usuario de referencia. |
| [Spaces](/es/x-api/fundamentals/data-dictionary#space) | El objeto Space consta de campos como `state`, `host_id`, `is_ticketed` e incluso `lang`. |
| [Lists](/es/x-api/fundamentals/data-dictionary#list) | El objeto List contiene información básica sobre la lista solicitada, incluidos `description`, `member_count` y `owner_id`. |
| [Media](/es/x-api/fundamentals/data-dictionary#media) | Si una Publicación contiene contenido multimedia (como imágenes), el objeto Media se puede solicitar usando el parámetro `media.fields` e incluye campos como `media_key`, `type`, `url`, `preview_image_url` y más. |
| [Poll](/es/x-api/fundamentals/data-dictionary#poll) | Una encuesta incluida en una Publicación no es un objeto primario en ningún endpoint, pero se puede encontrar y expandir en el objeto Publicación. |
| [Place](/es/x-api/fundamentals/data-dictionary#place) | El objeto Place consta de campos como `place_id`, el objeto `geo`, `country_code` y más. Esta información se puede usar para identificar Publicaciones y analizarlas por ubicación. |

Obtén más información sobre [cómo usar campos y expansiones](/es/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions).

<div id="flexibility-to-choose-which-objects-and-fields-you-receive">
  ### Flexibilidad para elegir qué objetos y campos recibes
</div>

Cuando haces una solicitud a un endpoint GET, recibirás el objeto de datos principal relacionado con ese endpoint, que incluirá un conjunto de campos predeterminados. Por ejemplo, el objeto Publicación entrega los campos `id` y `text` como valores predeterminados.

Si deseas recuperar campos adicionales con tu solicitud, tendrás que usar los parámetros [fields](/es/x-api/fundamentals/fields) y [expansions](/es/x-api/fundamentals/expansions). El parámetro expansions te permite recuperar objetos de datos relacionados, como la Publicación fijada de un usuario o un objeto de contenido multimedia, mientras que los operadores de campos te permiten solicitar campos específicos dentro de los objetos devueltos más allá de los predeterminados.

A continuación se muestra una lista completa de expansiones que puedes solicitar con los diferentes endpoints de X API v2:

| Objeto / Recurso | Expansiones disponibles |
|:-------------------|:----------------------|
| Publicaciones     | `author_id`, `edit_history_tweet_ids`, `entities.mentions.username`, `in_reply_to_user_id`, `referenced_tweets.id`, `referenced_tweets.id.author_id`, `attachments.poll_ids`, `attachments.media_keys`, `geo.place_id` |
| Usuarios          | `pinned_tweet_id` |
| Espacios          | `invited_user_ids`, `speaker_ids`, `creator_id`, `host_ids`, `topic_ids` |

Obtén más información sobre [cómo usar fields y expansions](/es/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions).

<div id="edit-tweets">
  ## Editar Publicaciones
</div>

Los endpoints de X API v2 proporcionan metadatos sobre Publicaciones editadas. La función *Edit Tweet* se introdujo por primera vez para pruebas internas entre empleados de X el 1 de septiembre de 2022. A partir de esa fecha, las Publicaciones que cumplan los requisitos se pueden editar durante 30 minutos y hasta 5 veces. Obtén más información sobre [Edit Tweets](/es/x-api/fundamentals/edit-posts).
Usando X API v2, un desarrollador puede averiguar:

* Si una Publicación era apta para edición en el momento de su creación. Algunas Publicaciones, como aquellas con encuestas o Publicaciones programadas, no se pueden editar.
* Que las Publicaciones se pueden editar durante 30 minutos y hasta 5 veces. Para las Publicaciones editables es posible ver si aún queda tiempo para editar y cuántas ediciones adicionales son posibles.
* Si se está viendo una versión editada de una Publicación (en la mayoría de los casos, la API devolverá la versión más reciente de una Publicación, a menos que se solicite una versión anterior específica por id de Publicación).
* Todo el historial de edición de la Publicación.
* La interacción atribuida a cada versión de la Publicación.

<div id="track-threaded-conversations">
  ## Seguir conversaciones en hilo
</div>

Un nuevo campo de la Publicación ayuda a identificar a qué hilo de conversación pertenece una Publicación. El id de conversación es el id de la Publicación que inició la conversación. Obtén más información sobre el [seguimiento de conversaciones](/es/x-api/fundamentals/conversation-id).

<div id="ready-to-migrate">
  ## Listo para migrar
</div>

Para poder usar los endpoints de v2, necesitarás lo siguiente:

* [Una cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info)
* [Una aplicación de desarrollador](https://developer.x.com/en/apps) creada dentro de un [Proyecto](resources/fundamentals/projects)
* [Claves y tokens](resources/fundamentals/authentication) de la aplicación de desarrollador de ese Proyecto

Es importante que utilices claves y tokens de una aplicación dentro de un Proyecto. Si utilizas claves y tokens de una aplicación fuera de un Proyecto, no podrás realizar solicitudes a los endpoints de v2.

Una vez que tengas una cuenta de desarrollador, puedes configurar todo lo anterior en el [portal para desarrolladores](https://developer.x.com/en/portal/petition/essential/basic-info).

<div id="authentication">
  ### Autenticación
</div>

Con la nueva API de Twitter, usarás dos patrones de autenticación diferentes, Contexto de usuario de OAuth 1.0a y Bearer Token de OAuth 2.0, para acceder a distintos endpoints. Cada uno cumple un propósito diferente al hacer solicitudes a los endpoints:

Contexto de usuario de OAuth 1.0a es necesario cuando haces una solicitud en nombre de un usuario de Twitter.
Bearer Token de OAuth 2.0 es necesario para hacer solicitudes en nombre de tu aplicación de desarrollador.

<div id="migrating-to-the-new-data-format">
  ## Migración al nuevo formato de datos
</div>

A medida que migras de v1.1 o Enterprise a v2, es importante entender que el formato en el que se entregan los datos ha cambiado significativamente. Hemos añadido nuevos campos, modificado el orden de los campos y, en algunos casos, eliminado elementos por completo.

Para obtener más información sobre estos cambios, estamos elaborando una serie de guías que te ayudarán a establecer la correspondencia entre los campos del formato de datos anterior a v2 y los campos más recientes, y explicarán cómo solicitar estos nuevos campos.

Puedes obtener más información visitando nuestra sección de [migración de formatos de datos](/es/x-api/migrate/data-format-migration) en este hub de migración, o consultando nuestras guías específicas sobre formatos de datos:

* [Formato nativo a X API v2 (v1.1 estándar)](/es/x-api/migrate/data-format-migration#migrating-from-standard-v1-1s-data-format-to-v2)
* [Native Enriched a X API v2 (enterprise)](/es/x-api/migrate/data-format-migration#migrating-from-native-enriched-data-format-to-v2)
