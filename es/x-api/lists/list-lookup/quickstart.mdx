<div id="getting-started-with-the-list-lookup-endpoint">
  ## Primeros pasos con el endpoint de consulta de Listas
</div>

Esta guía de inicio rápido te ayudará a realizar tu primera solicitud al endpoint de consulta de Listas usando Postman.

Visita nuestro repositorio de GitHub de [código de ejemplo de X API v2](https://github.com/xdevplatform/Twitter-API-v2-sample-code) si quieres ver código de ejemplo en diferentes lenguajes de programación.

**Nota:** En este ejemplo, realizaremos una solicitud al endpoint *List lookup by ID*, pero puedes aplicar lo que aprendas en esta guía de inicio rápido a otras solicitudes de consulta también.

<Note>
  ### Requisitos previos

  Para completar esta guía, necesitarás un conjunto de [claves y tokens](/es/resources/fundamentals/authentication) para autenticar tu solicitud. Puedes generar estas claves y tokens siguiendo estos pasos:

  * [Regístrate para obtener una cuenta de desarrollador](https://developer.x.com/en/apply-for-access) y espera la aprobación.
  * Crea un [Proyecto](/es/resources/fundamentals/projects) y una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) asociada en el portal para desarrolladores.
  * Navega a la página “Keys and tokens” de tu App para generar las credenciales necesarias. Asegúrate de guardar todas las credenciales en una ubicación segura.
</Note>

<div id="steps-to-build-a-list-lookup-request">
  ### Pasos para crear una solicitud de búsqueda de Lista
</div>

**Paso uno: Comienza con una herramienta o biblioteca**

Hay varias herramientas, ejemplos de código y bibliotecas que puedes usar para hacer una solicitud a este endpoint, pero aquí usaremos la herramienta Postman para simplificar el proceso.

Para cargar la colección de Postman de X API v2 en tu entorno, haz clic en el siguiente botón:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Agregar X API v2 a Postman
</Button>

Una vez que tengas la colección de X API v2 cargada en Postman, navega a la carpeta “List”, selecciona la carpeta “List lookup” y luego elige &quot;List by ID&quot;.
 

**Paso dos: Autentica tu solicitud**

Para hacer correctamente una solicitud a la X API, debes verificar que tienes permiso. Para hacerlo con este endpoint, debes autenticar tu solicitud con alguno de los siguientes métodos de autenticación: [App only](/es/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) o [OAuth 1.0a User Context](/es/resources/fundamentals/authentication).

Para simplificar las cosas, vamos a utilizar App only con esta solicitud, pero si quieres solicitar [métricas](/es/x-api/fundamentals/metrics) privadas o Listas, deberás usar uno de los otros métodos de autenticación. 

Para utilizar App only, debes agregar tus claves y tokens (específicamente el [App only Access Token](/es/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)) a Postman seleccionando el entorno llamado “X API v2” (en la esquina superior derecha de Postman) y agregando tus claves y tokens en los campos &quot;initial value&quot; y &quot;current value&quot; (haciendo clic en el ícono de ojo junto al menú desplegable de entornos).

Si lo has hecho correctamente, estas variables se rellenarán automáticamente en la pestaña de autorización de la solicitud.
 

**Paso tres: Identifica y especifica qué Lista quieres recuperar**

Debes especificar una Lista que quieras recibir dentro de la solicitud. Puedes encontrar el List ID navegando a x.com, haciendo clic en una Lista y luego mirando la URL. Por ejemplo, el List ID de la siguiente URL es 84839422.

https://x.com/i/lists/84839422

El ID de destino puede ser cualquier List ID válido. En Postman, navega a la pestaña &quot;Params&quot; e introduce tu ID en la columna &quot;Value&quot; de la variable de ruta id. Asegúrate de no incluir espacios antes o después de ningún ID.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| id  | 84839422 (el List ID) |

**Paso cuatro: Identifica y especifica qué campos quieres recuperar**

Si haces clic en el botón &quot;Send&quot; después del paso tres, recibirás en tu respuesta los campos predeterminados del [List object](/es/x-api/fundamentals/data-dictionary#list): id, name.

Si quieres recibir campos adicionales, tendrás que especificar esos campos en tu solicitud con los parámetros list.fields y/o [expansions](/es/x-api/fundamentals/expansions).

Para este ejercicio, solicitaremos tres conjuntos adicionales de campos de distintos objetos:

* El campo adicional created&#95;at en el objeto principal de Listas.

* El [user object](/es/x-api/fundamentals/data-dictionary#user) completo usando el parámetro de expansión.

* El campo adicional user.created&#95;at en el objeto de usuario asociado.

En Postman, navega a la pestaña &quot;Params&quot; y agrega el siguiente par clave: valor a la tabla &quot;Query Params&quot;:

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| list.fields | created&#95;at | created&#95;at |
| expansions | owner&#95;id | includes.users.id,  <br />includes.users.name,  <br />includes.users.username |
| user.fields | created&#95;at | includes.users.created&#95;at |

Ahora deberías ver una URL similar junto al botón “Send”:

`https://api.x.com/2/lists/84839422?list.fields=owner_id&expansions=owner_id&user.fields=created_at`

**Paso cinco: Haz tu solicitud y revisa tu respuesta**

Una vez que tengas todo configurado, haz clic en el botón &quot;Send&quot; y recibirás una respuesta similar al siguiente ejemplo de respuesta:

```
{
  "data": {
    "id": "84839422",
    "name": "Official Twitter Accounts",
    "owner_id": "783214"
  },
  "includes": {
    "users": [
      {
        "name": "Twitter",
        "created_at": "2007-02-20T14:35:54.000Z",
        "username": "Twitter",
        "id": "783214"
      }
    ]
  }
}
```
