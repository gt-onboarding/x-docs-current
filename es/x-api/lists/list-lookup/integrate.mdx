<div id="authentication">
  #### Autenticación
</div>

Todos los endpoints de X API v2 requieren que autentiques tus solicitudes con un conjunto de credenciales, también conocidas como claves y tokens. Puedes usar Contexto de usuario de OAuth 1.0a, solo aplicación o OAuth 2.0 Authorization Code with PKCE para autenticar tus solicitudes en estos endpoints. 

Con [OAuth 1.0a User Context](/es/resources/fundamentals/authentication#oauth-1-0a-user-context), debes usar un conjunto de claves de API y tokens de acceso de usuario para realizar una solicitud correcta. Los tokens de acceso deben estar asociados con el usuario en cuyo nombre estás realizando la solicitud. Si deseas generar un conjunto de tokens de acceso para otro usuario, ese usuario debe autorizar tu aplicación usando el [flujo OAuth de tres patas](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow).

Ten en cuenta que OAuth 1.0a puede ser difícil de usar. Si no estás familiarizado con este método de autenticación, te recomendamos que uses una [biblioteca](/es/x-api/tools-and-libraries/overview), una herramienta como Postman o OAuth 2.0 para autenticar tus solicitudes.

[OAuth 2.0 Authorization Code with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) permite un mayor control sobre el alcance (scope) de una aplicación y sobre los flujos de autorización entre múltiples dispositivos. OAuth 2.0 te permite elegir ámbitos específicos y granulares que te otorgan permisos concretos en nombre de un usuario. 

Para habilitar OAuth 2.0 en tu aplicación, debes activarlo en la configuración de autenticación de tu aplicación, ubicada en la sección de configuración de la aplicación del portal para desarrolladores.

[App only](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) solo requiere que incluyas un [App only Access Token](/es/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) en tu solicitud. Puedes generar un token de solo aplicación directamente desde una aplicación de desarrollador o generarlo usando el endpoint [POST oauth2/token](/es/resources/fundamentals/authentication#post-oauth2-token).

<div id="developer-portal-projects-and-developer-apps">
  #### Portal para desarrolladores, Proyectos y aplicaciones de desarrollador
</div>

Para obtener un conjunto de credenciales de autenticación que funcionen con los endpoints de X API v2, debes [registrarte y obtener una cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info), configurar un [Proyecto](/es/resources/fundamentals/projects) en esa cuenta y crear una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) dentro de ese Proyecto. Luego podrás encontrar tus claves y tokens en tu aplicación de desarrollador.
 

<div id="rate-limits">
  #### Límites de frecuencia
</div>

Cada día, muchos miles de desarrolladores realizan solicitudes a la X API. Para ayudar a gestionar el enorme volumen de estas solicitudes, se aplican [límites de frecuencia](/es/x-api/fundamentals/rate-limits) a cada endpoint que limitan la cantidad de solicitudes que puedes hacer en nombre de tu aplicación o en nombre de un usuario autenticado. 

Estos endpoints tienen límites de frecuencia tanto a nivel de aplicación como a nivel de usuario. El límite de frecuencia de la aplicación significa que tú, como desarrollador, solo puedes hacer una cierta cantidad de solicitudes a este endpoint durante un período de tiempo determinado desde cualquier aplicación determinada (ya sea usando la API Key y la API Secret Key, o el Token de acceso de solo aplicación). El límite de frecuencia del usuario significa que el usuario autenticado en cuyo nombre haces la solicitud solo puede realizar una búsqueda de Lista una cierta cantidad de veces en cualquier aplicación de desarrollador.

La siguiente tabla muestra los límites de frecuencia para cada endpoint.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Método HTTP** | **Límite de frecuencia** |
| /2/lists/:id | GET | 75 solicitudes cada 15 minutos |
| /2/users/:id/owned&#95;lists | GET | 15 solicitudes cada 15 minutos |

Campos y expansiones

El endpoint GET de la X API v2 permite a los usuarios seleccionar exactamente qué datos quieren que se devuelvan de la API utilizando un conjunto de herramientas llamadas `fields` y `expansions`. El parámetro `expansions` te permite expandir los objetos referenciados en el payload. Por ejemplo, buscar una Lista por ID te permite obtener las siguientes [expansiones](/es/x-api/fundamentals/expansions):

* `owner_id`

El parámetro `fields` te permite seleccionar exactamente qué [campos](/es/x-api/fundamentals/fields) dentro de los distintos objetos de datos deseas recibir. Estos endpoints entregan principalmente objetos de Lista. De forma predeterminada, el objeto de Lista devuelve los campos `id` y `name`. Para recibir campos adicionales como `list.created_at` o `list.follower_count`, tendrás que solicitarlos específicamente usando un parámetro list.fields. 

Hemos añadido a nuestro [diccionario de datos de X API v2](/es/x-api/fundamentals/data-dictionary) una guía sobre cómo usar [campos y expansiones](/es/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) juntos.

 La siguiente tabla muestra los campos y expansiones disponibles para este grupo de endpoints:

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Campos** | **Expansiones** |
| /2/lists/:id | `list.fields`<br /><br />`user.fields` | `owner_id` |
| /2/users/:id/owned&#95;lists | `list.fields`<br /><br />`user.fields` | `owner_id` |

<div id="pagination">
  #### Paginación
</div>

Consultar las Listas de un usuario puede devolver una gran cantidad de datos. Para garantizar que ofrezcamos resultados coherentes y de alto rendimiento en todo momento, usamos la paginación. La paginación es una característica de los endpoints de X API v2 que devuelven más resultados de los que se pueden incluir en una sola respuesta. Cuando eso sucede, los datos se devuelven en una serie de «páginas». Obtén más información sobre cómo [paginar los resultados.](/es/x-api/fundamentals/pagination)
