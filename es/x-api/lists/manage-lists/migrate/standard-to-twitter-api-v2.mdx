<div id="manage-lists-standard-v11-compared-to-x-api-v2">
  ### Administrar listas: estándar v1.1 frente a X API v2
</div>

Si has estado trabajando con los endpoints estándar v1.1 [POST lists/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-create), [POST lists/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-destroy) y [POST lists/update](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-update), el objetivo de esta guía es ayudarte a comprender las similitudes y diferencias entre los endpoints estándar v1.1 para administrar listas y los endpoints de X API v2 para administrar listas.

* **Similitudes**
  * Autenticación
* **Diferencias**
  * URLs de endpoints
  * Requisitos de Aplicación y Proyecto
  * Métodos HTTP
  * Límites de frecuencia
  * Parámetros de la solicitud

<div id="similarities">
  #### Similitudes
</div>

**Autenticación**

Ambas versiones de este endpoint admiten [Contexto de usuario de OAuth 1.0a](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Por lo tanto, si anteriormente usabas uno de los endpoints estándar v1.1 para administrar Listas, puedes seguir utilizando el mismo método de autenticación si migras a la versión de X API v2.

<div id="differences">
  #### Diferencias
</div>

**URL de endpoints**

* Endpoints estándar v1.1:
  * POST https://api.x.com/1.1/lists/create.json
    (Crea una Lista)
  * POST https://api.x.com/1.1/lists/destroy.json
    (Elimina una Lista)
  * POST https://api.x.com/1.1/lists/update.json
    (Actualiza una Lista)
* Endpoint de X API v2:
  * POST https://api.x.com/2/lists
    (Crea una Lista)

  * DELETE https://api.x.com/2/lists/:id
    (Elimina una Lista)

  * PUT https://api.x.com/2/lists/:id
    (Actualiza una Lista)

**Límites de frecuencia**

| **Estándar v1.1** | **X API v2** |
| :--- | :--- |
| /1.1/lists/create.json<br /><br />ninguno | /2/lists<br /><br />300 solicitudes por ventana de 15 minutos con Contexto de usuario de OAuth 1.0a |
| /1.1/lists/destroy.json<br /><br />ninguno | /2/lists/:id<br /><br />300 solicitudes por ventana de 15 minutos con Contexto de usuario de OAuth 1.0a |
| /1.1/lists/update.json<br /><br />ninguno | /2/lists/:id<br /><br />300 solicitudes por ventana de 15 minutos con Contexto de usuario de OAuth 1.0a |

**Requisitos de Aplicación y Proyecto**

Los endpoints de X API v2 requieren que uses credenciales de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) asociada con un [Proyecto](/es/resources/fundamentals/projects) al autenticar tus solicitudes. Todos los endpoints de X API v1.1 pueden usar credenciales de Aplicaciones independientes o de Aplicaciones relacionadas con un Proyecto.

**Parámetros de la solicitud**

Los siguientes parámetros de solicitud estándar v1.1 tienen equivalentes en X API v2:

**Crear una Lista**

| **Estándar** | **X API v2** |
| :--- | :--- |
| name | name |
| mode | private |
| description | description |

**Eliminar/actualizar una Lista**

| **Estándar** | **X API v2** |
| :--- | :--- |
| owner&#95;screen&#95;name | Sin equivalente |
| owner&#95;id | Sin equivalente |
| list&#95;id | id  |
| slug | Sin equivalente |

<Note>
  **Ten en cuenta:** los parámetros estándar v1.1 se envían como parámetros de consulta, mientras que los parámetros de X API v2 se envían como parámetros del cuerpo (para el endpoint POST) o parámetros de ruta (para los endpoints DELETE y PUT).
</Note>
