<div id="getting-started-with-the-manage-lists-endpoint-group">
  ## Introducción al grupo de endpoints para gestionar Listas
</div>

Esta breve descripción general te ayudará a realizar tu primera solicitud a los endpoints para gestionar Listas usando [Postman](/es/tutorials/postman-getting-started).

Si deseas ver código de ejemplo en distintos lenguajes de programación, visita nuestro repositorio de GitHub [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code). 

**Nota: **Para este ejemplo, realizaremos una solicitud al endpoint *Create a List*, pero también puedes aplicar lo que aprendas en esta guía rápida a otras solicitudes de gestión.

<Note>
  **Requisitos previos**

  Para completar esta guía, necesitarás un conjunto de [keys and tokens](/es/resources/fundamentals/authentication) para autenticar tu solicitud. Puedes generar estas claves y tokens siguiendo estos pasos:

  * [Regístrate para obtener una cuenta de desarrollador](https://developer.x.com/en/apply-for-access) y recibe la aprobación.
  * Crea un [Proyecto](/es/resources/fundamentals/projects) y una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) asociada en el portal para desarrolladores.
  * Accede a la página “Keys and tokens” de tu App para generar las credenciales necesarias. Asegúrate de guardar todas las credenciales en un lugar seguro.
</Note>

<div id="steps-to-build-a-manage-list-request">
  ### Pasos para crear una solicitud de administración de Lista
</div>

**Paso uno: Comienza con una herramienta o biblioteca**

Existen varias herramientas, ejemplos de código y bibliotecas que puedes usar para hacer una solicitud a este endpoint, pero aquí usaremos la herramienta Postman para simplificar el proceso.

Para cargar la colección de Postman de X API v2 en tu entorno, haz clic en el siguiente botón:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Agregar X API v2 a Postman
</Button>

Una vez que tengas la colección de X API v2 cargada en Postman, navega a la carpeta “List”, selecciona otra carpeta “Manage List” y luego elige &quot;Create a List&quot;.

**Paso dos: Autentica tu solicitud**

Para hacer correctamente una solicitud a la X API, debes verificar que tienes permiso para hacerlo. Para ello, con los endpoints de administración de Listas, debes autenticar tu solicitud usando ya sea [Contexto de usuario de OAuth 1.0a](/es/resources/fundamentals/authentication) o [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

En este ejemplo, vamos a usar Contexto de usuario de OAuth 1.0a.

Debes agregar tus claves y tokens (y específicamente tu API Key, API Secret Key, OAuth 1.0a user Access Token y OAuth 1.0a user Access Token Secret) a Postman. Puedes hacer esto seleccionando el entorno llamado “X API v2” (en la esquina superior derecha de Postman) y agregando tus claves y tokens a los campos &quot;initial value&quot; y &quot;current value&quot; (haciendo clic en el ícono de ojo junto al menú desplegable de entornos).

Si has hecho esto correctamente, estas variables se insertarán automáticamente en la pestaña de autorización de la solicitud.
 
**Paso tres: Especifica el nombre de la nueva Lista**

Al crear una nueva Lista con este endpoint, el nombre de la Lista es un parámetro obligatorio en el cuerpo. Opcionalmente, puedes proporcionar una descripción y especificar si la Lista es privada.

En Postman, navega a la pestaña “Body” e introduce el nombre de la Lista como valor para el parámetro name. Además, si deseas agregar una descripción para la Lista, simplemente agrega una nueva clave etiquetada description de la misma forma que name, seguida de la descripción de la Lista como valor. Hacer que una Lista sea privada seguirá el mismo patrón, pero solo se aceptan valores true o false para este parámetro. 

|             |                                    |                    |
| :---------- | :--------------------------------- | :----------------- |
| **Key**     | **Value**                          | **Parameter type** |
| `name`      | Nombre de la lista (obligatorio)   | body               |
| description | Descripción de la lista (opcional) | body               |
| private     | true o false (opcional)            | body               |

Ahora deberías ver una URL similar junto al botón &quot;Send&quot;:

`https://api.x.com/2/lists`

Paso cuatro: Haz tu solicitud y revisa tu respuesta

Una vez que tengas todo configurado, haz clic en el botón &quot;Send&quot; y recibirás una respuesta similar al siguiente ejemplo de respuesta:

```
{
  "data": {
    "id": "1441162269824405510",
    "name": "New list created from Postman"
  }
}
```

Si el objeto de respuesta devuelto contiene un id y el nombre de tu Lista, has creado la Lista correctamente. 

Para eliminar una Lista, selecciona la solicitud “Delete a List”, que también se encuentra en la carpeta “Lists” de la colección de X API v2 cargada en Postman. Este endpoint requiere el id de la Lista que deseas eliminar. En la pestaña “Params”, introduce el id de la Lista que deseas eliminar como valor para la columna id. 

Si la solicitud de eliminación se completa correctamente, recibirás una respuesta similar al siguiente ejemplo:

```
{
  "data": {
    "deleted": true
  }
}
```
