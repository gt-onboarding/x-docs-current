<div id="authentication">
  #### Autenticación
</div>

Todos los endpoints de X API v2 requieren que autentiques tus solicitudes con un conjunto de credenciales, también conocidas como claves y tokens. Puedes usar Contexto de usuario de OAuth 1.0a, solo aplicación o OAuth 2.0 Authorization Code with PKCE para autenticar tus solicitudes a este endpoint. 

[Contexto de usuario de OAuth 1.0a](/es/resources/fundamentals/authentication#oauth-1-0a-2), lo que significa que debes usar un conjunto de claves de API y Tokens de acceso de usuario para realizar una solicitud correcta. Los tokens de acceso deben estar asociados con el usuario en cuyo nombre estás realizando la solicitud. Si deseas generar un conjunto de Tokens de acceso para otro usuario, este debe autorizar tu aplicación utilizando el [flujo OAuth de tres patas](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow).

Ten en cuenta que OAuth 1.0a puede ser difícil de usar. Si no estás familiarizado con este método de autenticación, te recomendamos que uses una [biblioteca](/es/x-api/tools-and-libraries/overview), una herramienta como Postman o bien OAuth 2.0 o solo aplicación para autenticar tus solicitudes.

[OAuth 2.0 Authorization Code with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) permite un mayor control sobre el alcance de una aplicación y flujos de autorización en múltiples dispositivos. OAuth 2.0 te permite elegir ámbitos granulares específicos que te otorgan permisos específicos en nombre de un usuario. 

Para habilitar OAuth 2.0 en tu aplicación, debes activarlo en la configuración de autenticación de tu aplicación, que se encuentra en la sección de configuración de la aplicación del portal para desarrolladores.

[solo aplicación](/es/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) solo requiere que envíes un [Token de acceso de solo aplicación](/es/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) con tu solicitud. Puedes generar un Token de acceso de solo aplicación directamente dentro de una aplicación de desarrollador o generar uno usando el endpoint [POST oauth2/token](/es/resources/fundamentals/authentication#post-oauth2-token).

<div id="rate-limits">
  #### Límites de frecuencia
</div>

Cada día, muchos miles de desarrolladores hacen solicitudes a la X API. Para ayudar a gestionar el enorme volumen de estas solicitudes, se aplican [límites de frecuencia](https://developer.x.com/x-api/fundamentals/rate-limits) a cada endpoint, que limitan el número de solicitudes que puedes hacer en nombre de tu aplicación o en nombre de un usuario autenticado. 

Este endpoint tiene límites de frecuencia tanto a nivel de aplicación como a nivel de usuario. El límite de frecuencia de la aplicación significa que tú, como desarrollador, solo puedes hacer una cierta cantidad de solicitudes a este endpoint durante un periodo de tiempo determinado desde cualquier aplicación concreta (suponiendo que uses la API Key y la API Secret Key, o el Bearer Token). El límite de frecuencia del usuario significa que el usuario autenticado en cuyo nombre estás haciendo la solicitud solo puede realizar una búsqueda de Publicaciones de una Lista un número determinado de veces en cualquier aplicación de desarrollador.

| :--- | :--- | :--- |
| **Endpoint** | **Método HTTP** | **Límite de frecuencia** |

Fields y expansions

El endpoint GET de X API v2 permite a los usuarios seleccionar exactamente qué datos quieren obtener de la API mediante un conjunto de herramientas denominado `fields` y `expansions`. El parámetro `expansions` te permite expandir objetos referenciados en el payload. Por ejemplo, consultar Publicaciones de una Lista te permite obtener las siguientes [expansions](/es/x-api/fundamentals/expansions):

* `author_id`

El parámetro `fields` te permite seleccionar exactamente qué [fields](/es/x-api/fundamentals/fields) dentro de los diferentes objetos de datos quieres recibir. Este endpoint entrega principalmente objetos de Publicación. De forma predeterminada, el objeto de Publicación devuelve los campos `id` y `text`. Para recibir campos adicionales como `tweet.created_at` o `tweet.lang`, tendrás que solicitar específicamente esos campos usando un parámetro `fields`. 

Hemos añadido a nuestro [diccionario de datos de X API v2](/es/x-api/fundamentals/data-dictionary) una guía sobre cómo usar conjuntamente [fields y expansions](/es/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions).

La siguiente tabla muestra los fields y expansions disponibles para el endpoint de búsqueda:

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Fields** | **Expansions** |
| /2/lists/:id/tweets | `tweet.fields`<br /><br />`user.fields` | `author_id` |

Paginación

Consultar Publicaciones de una Lista puede devolver muchos datos. Para asegurarnos de que devolvemos resultados coherentes y de alto rendimiento en cualquier momento, usamos paginación. La paginación es una funcionalidad de los endpoints de X API v2 que devuelven más resultados de los que pueden incluirse en una sola respuesta. Cuando eso ocurre, los datos se devuelven en una serie de &quot;páginas&quot;. Obtén más información sobre cómo [paginar los resultados.](/es/x-api/fundamentals/pagination)
