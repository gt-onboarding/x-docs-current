<div id="comparing-x-apis-list-members-endpoints">
  ## Comparación de los endpoints de miembros de List de X API
</div>

El grupo de endpoints de miembros de List de v2 sustituirá a los endpoints estándar de v1.1 [GET lists/members](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-members), [GET lists/memberships](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-memberships), [POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create) y [POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy). Si tienes código, aplicaciones o herramientas que usan alguna de estas versiones de los endpoints de miembros de List y estás considerando migrar al nuevo endpoint de X API v2, entonces esta serie de guías es para ti. 

<div id="list-members-lookup">
  ### Búsqueda de miembros de Lista
</div>

Los endpoints de búsqueda de miembros de Lista de v2 sustituirán a los endpoints estándar [GET lists/members](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-members), [GET lists/memberships](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-memberships).

Las siguientes tablas comparan los endpoints estándar de Lista de v1.1 y X API v2:

**Búsqueda de miembros de Lista**

|     |     |     |
| :--- | :--- | :--- |
| Descripción | Estándar v1.1 | X API v2 |
| Métodos HTTP compatibles | `GET` | `GET` |
| Dominio del host | `https://api.x.com` | `https://api.x.com` |
| Ruta del endpoint | `/1.1/lists/members.json` | `/2/lists/:id/members` |
| [Autenticación](/es/resources/fundamentals/authentication) | Contexto de usuario de OAuth 1.0a<br /><br />Solo aplicación | Contexto de usuario de OAuth 1.0a<br /><br />OAuth 2.0 Authorization Code with PKCE<br /><br />Solo aplicación |
| [Límites de frecuencia](/es/resources/fundamentals/rate-limits) de solicitudes predeterminados | 900 solicitudes cada 15 min con OAuth 1.0a<br /><br />75 solicitudes cada 15 min con solo aplicación | 900 solicitudes cada 15 min con OAuth 1.0a<br /><br />900 solicitudes cada 15 min con OAuth 2.0<br /><br />900 solicitudes cada 15 min con solo aplicación |

**Búsqueda de membresías en Listas**

|     |     |     |
| :--- | :--- | :--- |
| Descripción | Estándar v1.1 | X API v2 |
| Métodos HTTP compatibles | `GET` | `GET` |
| Dominio del host | `https://api.x.com` | `https://api.x.com` |
| Ruta del endpoint | `/1.1/lists/memberships.json` | `/2/users/:id/list_memberships` |
| [Autenticación](/es/resources/fundamentals/authentication) | Contexto de usuario de OAuth 1.0a<br /><br />Solo aplicación | Contexto de usuario de OAuth 1.0a<br /><br />OAuth 2.0 Authorization Code with PKCE<br /><br />Solo aplicación |
| [Límites de frecuencia](/es/resources/fundamentals/rate-limits) de solicitudes predeterminados | 75 solicitudes cada 15 min con OAuth 1.0a<br /><br />75 solicitudes cada 15 min con solo aplicación | 75 solicitudes cada 15 min con OAuth 1.0a<br /><br />75 solicitudes cada 15 min con OAuth 2.0<br /><br />75 solicitudes cada 15 min con solo aplicación |

<div id="manage-list-members">
  ### Administrar miembros de Listas
</div>

Los endpoints de la X API v2 para administrar miembros de Listas reemplazarán los endpoints estándar [POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create), [POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy).

Las siguientes tablas comparan los endpoints estándar de List de v1.1 y los de X API v2:

**Agregar miembro**

| Descripción | v1.1 estándar | X API v2 |
| :--- | :--- | :--- |
| Métodos HTTP admitidos | POST | POST |
| Dominio de host | https://api.x.com | https://api.x.com |
| Ruta del endpoint | /1.1/lists/members/create.json | /2/lists/:id/members |
| [Autenticación](/es/resources/fundamentals/authentication) | Contexto de usuario de OAuth 1.0a | Contexto de usuario de OAuth 1.0a<br /><br />OAuth 2.0 Authorization Code with PKCE |
| [Límites de frecuencia](/es/resources/fundamentals/rate-limits) de solicitudes predeterminados | Ninguno | 300 solicitudes por 15 min (por usuario) |

**Eliminar miembro**

| Descripción | v1.1 estándar | X API v2 |
| :--- | :--- | :--- |
| Métodos HTTP admitidos | POST | DELETE |
| Dominio de host | https://api.x.com | https://api.x.com |
| Ruta del endpoint | /1.1/lists/members/destroy.json | /2/lists/:id/:user&#95;id |
| [Autenticación](/es/resources/fundamentals/authentication) | Contexto de usuario de OAuth 1.0a | Contexto de usuario de OAuth 1.0a<br /><br />OAuth 2.0 Authorization Code with PKCE |
| [Límites de frecuencia](/es/resources/fundamentals/rate-limits) de solicitudes predeterminados | Ninguno | 300 solicitudes por 15 min (por usuario) |

Para acceder a los endpoints de X API v2, debes [registrarte para obtener una cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info). Al autenticarte, debes usar claves y tokens de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) que forme parte de un [Proyecto](/es/resources/fundamentals/projects). 

Obtén más información sobre cómo acceder a los endpoints de X API v2 en nuestra página de [primeros pasos](/es/x-api/getting-started/getting-access).
