<div id="manage-list-members-standard-v11-compared-to-x-api-v2">
  ### Administrar miembros de listas: Standard v1.1 en comparación con X API v2
</div>

Si has estado usando los endpoints estándar v1.1 [POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create) y [POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy), el objetivo de esta guía es ayudarte a comprender las similitudes y diferencias entre los endpoints estándar v1.1 y X API v2 para administrar miembros de listas.

* **Similitudes**
  * Autenticación
* **Diferencias**
  * URL de los endpoints
  * Requisitos de la Aplicación y el Proyecto
  * Métodos HTTP
  * Límites de frecuencia
  * Parámetros de solicitud

<div id="similarities">
  #### Similitudes
</div>

**Autenticación**

Ambas versiones de este endpoint son compatibles con el [Contexto de usuario de OAuth 1.0a](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Por lo tanto, si anteriormente utilizabas uno de los endpoints estándar de v1.1 para administrar los miembros de una Lista, puedes seguir usando el mismo método de autenticación si migras a la versión de X API v2.

<div id="differences">
  #### Diferencias
</div>

**URLs de endpoints**

* Endpoints estándar v1.1:
  * POST https://api.x.com/1.1/lists/members/create.json
    (Agrega un miembro a una Lista específica)
  * POST https://api.x.com/1.1/lists/members/destroy.json
    (Elimina un miembro de una Lista específica)
* Endpoint de X API v2:
  * POST https://api.x.com/2/lists/:id/members
    (Agrega un miembro a una Lista específica)

  * DELETE https://api.x.com/2/lists/:id/members/:user&#95;id
    (Elimina un miembro de una Lista específica)

**Límites de frecuencia**

| **Estándar v1.1** | **X API v2** |
| :--- | :--- |
| /1.1/lists/members/create.json<br /><br />ninguno | /2/lists/:id/members<br /><br />300 solicitudes por ventana de 15 minutos con Contexto de usuario de OAuth 1.0a<br /><br />300 solicitudes por ventana de 15 minutos con OAuth 2.0 Authorization Code with PKCE |
| /1.1/lists/members/destroy.json<br /><br />ninguno | /2/lists/:id/members/:user&#95;id<br /><br />300 solicitudes por ventana de 15 minutos con Contexto de usuario de OAuth 1.0a<br /><br />300 solicitudes por ventana de 15 minutos con OAuth 2.0 Authorization Code with PKCE |

**Requisitos de Aplicación y Proyecto**

Los endpoints de X API v2 requieren que uses credenciales de una [developer App](/es/resources/fundamentals/developer-apps) que esté asociada con un [Project](/es/resources/fundamentals/projects) al autenticar tus solicitudes. Todos los endpoints de X API v1.1 pueden usar credenciales de aplicaciones independientes o de aplicaciones relacionadas con un proyecto.

**Parámetros de la solicitud**

Los siguientes parámetros de solicitud estándar v1.1 tienen equivalentes en X API v2:

| **Estándar v1.1** | **X API v2** |
| :--- | :--- |
| list&#95;id | id  |
| slug | Sin equivalente |
| screen&#95;name | Sin equivalente |
| owner&#95;screen&#95;name | Sin equivalente |
| owner&#95;id | Sin equivalente |
