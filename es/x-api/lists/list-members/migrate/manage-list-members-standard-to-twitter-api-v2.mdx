<div id="manage-list-members-standard-v11-compared-to-x-api-v2">
  ### Administrar miembros de List: v1.1 estándar en comparación con X API v2
</div>

Si has estado trabajando con los endpoints estándar v1.1 [POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create) y [POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy), el objetivo de esta guía es ayudarte a entender las similitudes y diferencias entre los endpoints estándar v1.1 y los de X API v2 para administrar miembros de List.

* **Similitudes**
  * Autenticación
* **Diferencias**
  * URLs de endpoints
  * Requisitos de la aplicación y el Proyecto
  * Métodos HTTP
  * Límites de frecuencia
  * Parámetros de la solicitud

<div id="similarities">
  #### Similitudes
</div>

**Autenticación**

Ambas versiones del endpoint son compatibles con [Contexto de usuario de OAuth 1.0a](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Por lo tanto, si antes utilizabas uno de los endpoints estándar de v1.1 para administrar los miembros de una Lista, puedes seguir utilizando el mismo método de autenticación al migrar a la versión de X API v2.

<div id="differences">
  #### Diferencias
</div>

**URLs de endpoints**

* Endpoints estándar v1.1:
  * POST https://api.x.com/1.1/lists/members/create.json
    (Agrega un miembro a una Lista especificada)
  * POST https://api.x.com/1.1/lists/members/destroy.json
    (Elimina un miembro de una Lista especificada)
* Endpoint de X API v2:
  * POST https://api.x.com/2/lists/:id/members
    (Agrega un miembro a una Lista especificada)

  * DELETE https://api.x.com/2/lists/:id/members/:user&#95;id
    (Elimina un miembro de una Lista especificada)

**Límites de frecuencia**

| **v1.1 estándar** | **X API v2** |
| :--- | :--- |
| /1.1/lists/members/create.json<br /><br />ninguno | /2/lists/:id/members<br /><br />300 solicitudes por ventana de 15 minutos con Contexto de usuario de OAuth 1.0a<br /><br />300 solicitudes por ventana de 15 minutos con OAuth 2.0 Authorization Code with PKCE |
| /1.1/lists/members/destroy.json<br /><br />ninguno | /2/lists/:id/members/:user&#95;id<br /><br />300 solicitudes por ventana de 15 minutos con Contexto de usuario de OAuth 1.0a<br /><br />300 solicitudes por ventana de 15 minutos con OAuth 2.0 Authorization Code with PKCE |

**Requisitos de Aplicación y Proyecto**

Los endpoints de X API v2 requieren que utilices credenciales de una [developer App](/es/resources/fundamentals/developer-apps) que esté asociada con un [Project](/es/resources/fundamentals/projects) al autenticar tus solicitudes. Todos los endpoints de X API v1.1 pueden usar credenciales de Apps independientes o de Apps relacionadas con un Proyecto.

**Parámetros de la solicitud**

Los siguientes parámetros de solicitud estándar v1.1 tienen equivalentes en X API v2:

| **v1.1 estándar** | **X API v2** |
| :--- | :--- |
| list&#95;id | id  |
| slug | Sin equivalente |
| screen&#95;name | Sin equivalente |
| owner&#95;screen&#95;name | Sin equivalente |
| owner&#95;id | Sin equivalente |
