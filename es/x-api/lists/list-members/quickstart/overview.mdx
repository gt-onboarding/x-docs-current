<div id="getting-started-with-the-list-members-endpoint-group">
  ## Introducción al grupo de endpoints de miembros de List
</div>

Este breve resumen te ayudará a realizar tu primera solicitud a los endpoints de miembros de List utilizando [Postman](/es/tutorials/postman-getting-started).

Si deseas ver código de ejemplo en diferentes lenguajes de programación, visita nuestro repositorio de [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) en GitHub. 

<Note>
  **Requisitos previos**

  Para completar esta guía, necesitas contar con un conjunto de [keys and tokens](/es/resources/fundamentals/authentication) para autenticar tu solicitud. Puedes generar estas claves y tokens siguiendo estos pasos:

  * [Regístrate para obtener una cuenta de desarrollador](https://developer.x.com/en/apply-for-access) y recibe la aprobación.
  * Crea un [Proyecto](/es/resources/fundamentals/projects) y una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) asociada en el portal para desarrolladores.
  * Navega a la página “Keys and tokens” de tu App para generar las credenciales necesarias. Asegúrate de guardar todas las credenciales en un lugar seguro.
</Note>

<div id="authenticate-your-request">
  #### Autentica tu solicitud
</div>

Para realizar una solicitud correcta a los endpoints de **lookup**, puedes usar [OAuth 1.0a User Context](/es/resources/fundamentals/authentication#oauth-1-0a-2), [App only](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) o [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). Sin embargo, con los endpoints de **manage**, solo puedes autenticarte con OAuth 1.0a User Context o OAuth 2.0 Authorization Code with PKCE.

En cualquier caso, cuando uses Postman, las claves y tokens de autenticación predeterminados se rellenarán automáticamente en tus solicitudes siempre que hayas configurado correctamente tu entorno.

Puedes hacerlo seleccionando el entorno llamado &quot;X API v2&quot; (en la esquina superior derecha de Postman) y añadiendo tus claves y tokens a los campos &quot;initial value&quot; y &quot;current value&quot; (haciendo clic en el ícono de ojo junto al menú desplegable de entornos). Estas claves incluyen lo siguiente:

* API Key (también conocida como Consumer Key)
* API Secret Key (también conocida como Consumer Secret)
* OAuth 1.0a user Access Token
* OAuth 1.0a user Access Token Secret
* App only Access Token
* OAuth 2.0 Client Key (solo disponible si has configurado OAuth 2.0 User Authentication en la configuración de tu Aplicación)
* OAuth 2.0 Client Secret (solo disponible si has configurado OAuth 2.0 User Authentication en la configuración de tu Aplicación)
