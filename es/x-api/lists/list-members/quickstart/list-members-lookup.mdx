<div id="steps-to-build-a-list-members-lookup-request">
  ### Pasos para crear una solicitud de búsqueda de miembros de una Lista
</div>

**Paso uno: Elige la colección de endpoints de List en Postman**

Una vez que tengas la colección de X API v2 cargada en Postman, navega a la carpeta “List”, selecciona otra carpeta “List members” y luego elige &quot;Members lookup&quot;.
 

**Paso dos: Identifica y especifica de qué Lista quieres recuperar miembros**

Debes especificar una Lista de la que quieres recibir miembros. Puedes encontrar el ID de la Lista navegando a x.com, haciendo clic en una Lista y luego mirando la URL. Por ejemplo, el ID de Lista de la siguiente URL es 84839422.

https://x.com/i/lists/84839422

El ID de destino puede ser cualquier ID de Lista válido. En Postman, navega a la pestaña &quot;Params&quot; e ingresa tu ID en la columna &quot;Value&quot; de la variable de ruta id. Asegúrate de no incluir espacios antes o después de ningún ID.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 84839422 (List ID) |

**Paso tres: Identifica y especifica qué campos quieres recuperar**

Si haces clic en el botón &quot;Send&quot; después del paso tres, recibirás en tu respuesta los campos predeterminados del [objeto de usuario](/es/x-api/fundamentals/data-dictionary#user): id, name y username.

Si quieres recibir campos adicionales además de id, name y username, tendrás que especificar esos campos en tu solicitud con los parámetros [field](/es/x-api/fundamentals/fields) y/o [expansion](/es/x-api/fundamentals/expansions).

Para este ejercicio, solicitaremos tres conjuntos adicionales de campos de distintos objetos:

* El campo adicional user.created&#95;at en los objetos de usuario principales.
* El [objeto de Publicación](/es/x-api/fundamentals/data-dictionary#tweet) completo usando el parámetro expansion.
* El campo adicional tweet.created&#95;at en los objetos de Publicación asociados.

En Postman, navega a la pestaña &quot;Params&quot; y agrega el siguiente par clave:valor a la tabla &quot;Query Params&quot;:

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| user.fields | created&#95;at | `created_at` |
| expansions | pinned&#95;tweet&#95;id | `includes.tweets.id,  <br/>includes.tweets.text  <br/>` |
| tweet.fields | created&#95;at | `includes.tweets.created_at` |

Ahora deberías ver una URL similar junto al botón “Send”:

`https://api.x.com/2/lists/84839422/members?user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=created_at`
