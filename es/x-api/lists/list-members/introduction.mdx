<div id="list-members-lookup">
  ### Búsqueda de miembros de la Lista
</div>

El grupo de búsqueda de miembros tiene dos endpoints disponibles. Puedes obtener detalles sobre los miembros de una Lista específica y ver de qué Listas es miembro un usuario. Estos endpoints se pueden usar para permitir que las personas seleccionen y organicen nuevas Listas basadas en la información sobre la pertenencia.

Hay un límite de frecuencia de 900 solicitudes cada 15 minutos al buscar detalles de miembros y un límite de 75 solicitudes cada 15 minutos al buscar membresías de usuario.

Puedes usar [Contexto de usuario de OAuth 1.0a](/es/resources/fundamentals/authentication), [solo aplicación](https://developer.x.com/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), o [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) para autenticar tus solicitudes a este endpoint. 
 

<div id="manage-list-members">
  ### Administrar miembros de una Lista
</div>

Los endpoints para administrar miembros de una Lista te permiten agregar y eliminar miembros de una Lista en nombre de un usuario autenticado. Para estos endpoints, hay dos métodos disponibles: POST y DELETE. El método POST te permite agregar un miembro a una Lista, y el método DELETE te permite eliminar un miembro de una Lista. Existe un límite de frecuencia por usuario de 300 solicitudes cada 15 minutos para ambos endpoints.

Ten en cuenta que las Listas no pueden tener más de 5,000 miembros.

Dado que estás realizando solicitudes en nombre de un usuario con estos endpoints, debes autenticarlo usando [OAuth 1.0a User Context](/es/resources/fundamentals/authentication) o [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), y usar los tokens de acceso de usuario asociados con un usuario que haya autorizado tu aplicación, los cuales se pueden generar usando el [flujo OAuth de tres patas](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) o el [flujo de concesión Authorization Code with PKCE](/es/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0).

<Note>
  **Configuración de la cuenta**

  Para acceder a estos endpoints, necesitarás:

  * Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
  * Autenticarte usando las claves y tokens de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) que se encuentre dentro de un [Proyecto](/es/resources/fundamentals/projects).

  Obtén más información sobre cómo acceder a los endpoints de X API v2 en nuestra [guía de primeros pasos](/es/x-api/getting-started/getting-access).
</Note>
