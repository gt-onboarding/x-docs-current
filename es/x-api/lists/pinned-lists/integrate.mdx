<div id="authentication">
  #### Autenticación
</div>

Todos los endpoints de X API v2 requieren que autentiques tus solicitudes con un conjunto de credenciales, también conocidas como claves y tokens. Puedes usar el Contexto de usuario de OAuth 1.0a para autenticar tus solicitudes a este endpoint. 

[Contexto de usuario de OAuth 1.0a](/es/resources/fundamentals/authentication), lo que significa que debes usar un conjunto de claves de API y tokens de acceso de usuario para realizar una solicitud correcta. Los tokens de acceso deben estar asociados con el usuario en cuyo nombre realizas la solicitud. Si deseas generar un conjunto de tokens de acceso para otro usuario, este debe autorizar tu Aplicación usando el [flujo OAuth de tres patas](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens).

Ten en cuenta que OAuth 1.0a puede ser difícil de usar. Si no estás familiarizado con este método de autenticación, te recomendamos que uses una [biblioteca](/es/x-api/tools-and-libraries/overview) o una herramienta como Postman.

<div id="developer-portal-projects-and-developer-apps">
  #### Portal para desarrolladores, Proyectos y aplicaciones de desarrollador
</div>

Para obtener credenciales de autenticación que funcionen con los endpoints de X API v2, debes [registrarte para obtener una cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info), configurar un [Proyecto](/es/resources/fundamentals/projects) dentro de esa cuenta y crear una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) dentro de ese Proyecto. Luego podrás encontrar tus claves y tokens en tu aplicación de desarrollador.
 

<div id="rate-limits">
  #### Límites de frecuencia
</div>

Cada día, muchos miles de desarrolladores realizan solicitudes a la X API. Para ayudar a gestionar el gran volumen de estas solicitudes, se establecen [límites de frecuencia](/es/x-api/fundamentals/rate-limits) en cada endpoint, que limitan la cantidad de solicitudes que puedes hacer en nombre de tu Aplicación o en nombre de un usuario autenticado. 

Estos endpoints tienen límites de frecuencia a nivel de usuario, lo que significa que el usuario autenticado en cuyo nombre realizas la solicitud solo puede invocar el endpoint un número determinado de veces, independientemente de la aplicación de desarrollador utilizada. 

La tabla siguiente muestra los límites de frecuencia para cada endpoint.

| :--- | :--- | :--- |
| **Endpoint** | **Método HTTP** | **Límite de frecuencia** |
| /2/users/:id/pinned&#95;lists | POST | 50 solicitudes por 15 minutos |
| /2/users/:id/pinned&#95;lists/:list&#95;id | DELETE | 50 solicitudes por 15 minutos |
| /2/users/:id/pinned&#95;lists | GET | 15 solicitudes por 15 minutos |

<div id="fields-and-expansions">
  #### Campos y expansiones
</div>

El endpoint GET de X API v2 permite a los usuarios seleccionar exactamente qué datos quieren que devuelva la API utilizando un conjunto de parámetros llamados `fields` y `expansions`. El parámetro `expansions` te permite expandir los objetos referenciados en el payload. Por ejemplo, consultar Listas fijadas te permite obtener las siguientes [expansions](/es/x-api/fundamentals/expansions):

* `owner_id`

El parámetro `fields` te permite seleccionar exactamente qué [fields](/es/x-api/fundamentals/fields) dentro de los diferentes objetos de datos quieres recibir. Este endpoint entrega principalmente objetos de usuario. De forma predeterminada, el objeto List devuelve los campos `id` y `name`. Para recibir campos adicionales como `list.created_at` o `list.description`, tendrás que solicitar específicamente esos campos usando un parámetro `fields`. 

Hemos añadido a nuestro [diccionario de datos de X API v2](/es/x-api/fundamentals/data-dictionary) una guía sobre cómo usar juntos [fields y expansions](/es/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions).
