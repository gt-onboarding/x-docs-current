<div id="getting-started-with-the-pinned-list-endpoint-group">
  ## Introducción al grupo de endpoints de listas fijadas
</div>

Esta breve descripción general te ayudará a realizar tu primera solicitud a los endpoints de listas fijadas usando [Postman](/es/tutorials/postman-getting-started).

Visita nuestro repositorio de [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) en GitHub si quieres ver código de ejemplo en distintos lenguajes de programación.

<Note>
  ### Requisitos previos

  Para completar esta guía, necesitarás contar con un conjunto de [claves y tokens](/es/resources/fundamentals/authentication) para autenticar tu solicitud. Puedes generar estas claves y tokens siguiendo estos pasos:

  * [Regístrate para obtener una cuenta de desarrollador](https://developer.x.com/en/apply-for-access) y recibe la aprobación.
  * Crea un [Proyecto](/es/resources/fundamentals/projects) y una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) asociada en el portal para desarrolladores.
  * Navega a la página “Keys and tokens” de tu App para generar las credenciales necesarias. Asegúrate de guardar todas las credenciales en un lugar seguro.
</Note>

Cargar la colección de Postman de X API v2

Para cargar la colección de Postman de X API v2 en tu espacio de trabajo, haz clic en el siguiente botón:

<div id="authenticate-your-request">
  #### Autentica tu solicitud
</div>

Para realizar una solicitud exitosa a los endpoints de **lookup**, puedes usar [OAuth 1.0a User Context](/es/resources/fundamentals/authentication), [OAuth 2.0 App-Only](/es/resources/fundamentals/authentication#oauth-2-0) o [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). Sin embargo, con los endpoints de **manage**, solo puedes autenticarte con OAuth 1.0a User Context o OAuth 2.0 Authorization Code with PKCE.

En cualquier caso, cuando uses Postman, las claves y tokens de autenticación predeterminados se completarán automáticamente en tus solicitudes siempre que hayas configurado correctamente tu entorno.

Puedes hacer esto seleccionando el entorno llamado “X API v2” (en la esquina superior derecha de Postman) y añadiendo tus claves y tokens a los campos &quot;initial value&quot; y &quot;current value&quot; (haciendo clic en el icono con forma de ojo junto al menú desplegable de entornos). Estas claves incluyen lo siguiente:

* API Key (también conocida como Consumer Key)
* API Secret Key (también conocida como Consumer Secret)
* OAuth 1.0a user Access Token
* OAuth 1.0a user Access Token Secret
* OAuth 2.0 App Access Token
* OAuth 2.0 Client Key (solo disponible si has configurado OAuth 2.0 User Authentication en la configuración de tu Aplicación)
* OAuth 2.0 Client Secret (solo disponible si has configurado OAuth 2.0 User Authentication en la configuración de tu Aplicación)
