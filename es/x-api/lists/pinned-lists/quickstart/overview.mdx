<div id="getting-started-with-the-pinned-list-endpoint-group">
  ## Introducción al grupo de endpoints de Listas fijadas
</div>

Esta breve descripción general te ayudará a realizar tu primera solicitud a los endpoints de Listas fijadas usando [Postman](/es/tutorials/postman-getting-started).

Visita nuestro repositorio de GitHub [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) si quieres ver código de ejemplo en diferentes lenguajes de programación.

<Note>
  ### Requisitos previos

  Para completar esta guía, necesitarás contar con un conjunto de [claves y tokens](/es/resources/fundamentals/authentication) para autenticar tu solicitud. Puedes generar estas claves y tokens siguiendo estos pasos:

  * [Regístrate para obtener una cuenta de desarrollador](https://developer.x.com/en/apply-for-access) y recibe la aprobación.
  * Crea un [Proyecto](/es/resources/fundamentals/projects) y una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) asociada en el portal para desarrolladores.
  * Navega a la página &quot;Keys and tokens&quot; de tu App para generar las credenciales necesarias. Asegúrate de guardar todas las credenciales en una ubicación segura.
</Note>

Cargar la colección de Postman de X API v2

Para cargar la colección de Postman de X API v2 en tu espacio de trabajo, haz clic en el siguiente botón:

  Agregar X API v2 a Postman
</Button>

 

<div id="authenticate-your-request">
  #### Autentica tu solicitud
</div>

Para realizar correctamente una solicitud a los endpoints de **lookup**, puedes usar [OAuth 1.0a User Context](/es/resources/fundamentals/authentication), [OAuth 2.0 App-Only](/es/resources/fundamentals/authentication#oauth-2-0) o [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). Sin embargo, con los endpoints de **manage** solo puedes autenticarte con OAuth 1.0a User Context o OAuth 2.0 Authorization Code with PKCE.

En cualquier caso, cuando utilices Postman, las claves y tokens de autenticación predeterminados se rellenarán automáticamente en tus solicitudes siempre que hayas configurado correctamente tu entorno.

Puedes hacerlo seleccionando el entorno llamado “X API v2” (en la esquina superior derecha de Postman) y añadiendo tus claves y tokens a los campos &quot;initial value&quot; y &quot;current value&quot; (haciendo clic en el icono del ojo junto al menú desplegable de entornos). Estas claves incluyen lo siguiente:

* API Key (también conocida como Consumer Key)
* API Secret Key (también conocida como Consumer Secret)
* OAuth 1.0a user Access Token
* OAuth 1.0a user Access Token Secret
* OAuth 2.0 App Access Token
* OAuth 2.0 Client Key (solo disponible si has configurado &quot;OAuth 2.0 User Authentication&quot; en la configuración de tu App)
* OAuth 2.0 Client Secret (solo disponible si has configurado &quot;OAuth 2.0 User Authentication&quot; en la configuración de tu App)
