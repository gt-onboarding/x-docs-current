Anclar listas permite a los usuarios de X acceder más fácilmente a las listas a las que se han suscrito o que han creado ellos mismos.

Dado que realizas solicitudes en nombre de un usuario con estos endpoints, debes autenticarlo con [OAuth 1.0a User Context](/es/resources/fundamentals/authentication) o [OAuth 2.0 Authorization Code with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), y usar los tokens de acceso de usuario asociados con un usuario que haya autorizado tu Aplicación, que puedes generar mediante el [flujo OAuth de tres patas](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) o el [Authorization Code with PKCE grant flow](/es/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0).

<div id="pinned-list-lookup">
  ### Consulta de Listas fijadas
</div>

La consulta de Listas fijadas tiene un único endpoint disponible que te permite recuperar las Listas fijadas del usuario autenticado. Existe un límite de 15 solicitudes cada 15 minutos para este endpoint.
 

<div id="manage-pinned-lists">
  ### Administrar Listas fijadas
</div>

Los endpoints para administrar Listas fijadas te permiten fijar y dejar de fijar una Lista en nombre de un usuario autenticado. Para estos endpoints hay dos métodos disponibles: POST y DELETE. El método POST te permite fijar una Lista y el método DELETE te permite dejar de fijar una Lista. Hay un límite de tasa por usuario de 50 solicitudes cada 15 minutos para ambos endpoints.

<Note>
  **Configuración de la cuenta**

  Para acceder a estos endpoints, necesitarás:

  * Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
  * Autenticarte usando las claves y tokens de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) que forme parte de un [Proyecto](/es/resources/fundamentals/projects). 

  Obtén más información sobre cómo obtener acceso a los endpoints de X API v2 en nuestra [guía de introducción](/es/x-api/getting-started/getting-access).
</Note>
