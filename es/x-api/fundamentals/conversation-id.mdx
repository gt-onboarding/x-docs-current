<div id="conversation-threads-using-the-x-api">
  ### Hilos de conversación mediante la X API
</div>

Si observas cómo evolucionan las conversaciones en X, una sola Publicación puede generar varios hilos de conversación, cada uno de los cuales puede crecer en longitud y complejidad a medida que más personas participan. Identificar las relaciones entre Publicaciones y comprender los hilos de conversación es una característica del payload y las capacidades de búsqueda de la X API v2. Cuando se publican Publicaciones en respuesta a una Publicación (conocida como respuesta), o en respuesta a una respuesta, ahora existe un `conversation_id` definido en cada respuesta, que coincide con el Post ID de la Publicación original que inició la conversación.

Las respuestas a una Publicación determinada, así como las respuestas a esas respuestas, se incluyen todas en la conversación que se origina a partir de una única Publicación original. Independientemente de la cantidad de hilos de respuesta que se generen, todos compartirán el mismo `conversation_id` de la Publicación original que desencadenó la conversación. Con la X API v2, puedes recuperar y reconstruir un hilo de conversación completo, lo que te permite comprender qué se está diciendo y cómo evolucionan las conversaciones y las ideas.

Recuperar `conversation_id` como parámetro de `tweet.fields`
Para solicitar el `conversation_id` de todas las Publicaciones devueltas en un endpoint v2, agrega el campo tweet.fields=conversation&#95;id a los parámetros de la solicitud. El campo conversation&#95;id siempre corresponde al Post ID de la Publicación original en el hilo de respuestas de la conversación. Todas las Publicaciones dentro del mismo hilo de respuestas, incluidas las que se crean a partir de hilos de respuestas anteriores, mostrarán el mismo conversation&#95;id.

<div id="request-with-conversation_id-parameter">
  ### Solicitud con el parámetro `conversation_id`
</div>

```
curl --request GET \
  --url 'https://api.x.com/2/tweets?ids=1225917697675886593&tweet.fields=author_id,conversation_id,created_at,in_reply_to_user_id,referenced_tweets&expansions=author_id,in_reply_to_user_id,referenced_tweets.id&user.fields=name,username' \
  --header 'Authorization: Bearer $ACCESS_TOKEN'
```

<div id="using-conversation_id-as-a-filter-operator">
  ### Usar `conversation_id` como operador de filtro
</div>

El `conversation_id` se puede usar como parámetro de consulta en Recent search o como operador dentro de una regla para [filtered stream](/es/x-api/posts/filtered-stream/introduction). Esto devuelve todas las Publicaciones del hilo de conversación, ya sea en tiempo real a través de filtered stream o en orden cronológico inverso desde [search Tweets](/es/x-api/posts/search/introduction). También puedes obtener el recuento de Publicaciones en una conversación usando este operador con [Posts counts](/es/x-api/posts/counts/introduction).

<div id="request-to-query-by-conversation_id">
  ## Solicitud para realizar una consulta por `conversation_id`
</div>

```
curl --request GET \
  --url 'https://api.x.com/2/tweets/search/recent?query=conversation_id:1279940000004973111&tweet.fields=in_reply_to_user_id,author_id,created_at,conversation_id' \
  --header 'Authorization: Bearer $ACCESS_TOKEN'
```
