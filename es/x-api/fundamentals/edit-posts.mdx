<div id="introduction">
  # Introduction
</div>

Los endpoints de X API v2 proporcionan metadatos de Publicaciones editadas. La función *Edit Posts* se introdujo por primera vez para pruebas entre empleados de X el 1 de septiembre de 2022. A partir de esa fecha, las Publicaciones elegibles pueden editarse durante 30 minutos y hasta 5 veces. *Todos los objetos de Publicaciones creadas desde el 29 de septiembre de 2022* incluyen metadatos de edición de la Publicación, incluso si la Publicación nunca se editó. Cada vez que se edita una Publicación, se crea una nueva id de la Publicación. El historial de edición de una Publicación se puede describir encadenando estas ids, comenzando con la id original. Además, si se elimina cualquier Publicación en la cadena de edición, todas las Publicaciones de esa cadena también se eliminan.

Usando X API v2, un desarrollador puede averiguar:

* Si una Publicación era elegible para edición en el momento de su creación. Algunas Publicaciones, como aquellas con encuestas o Publicaciones programadas, no se pueden editar.
* Que las Publicaciones pueden editarse durante 30 minutos y hasta 5 veces. Para las Publicaciones editables, puedes ver si aún queda tiempo para editar y cuántas ediciones adicionales son posibles.
* Si estás viendo una versión editada de una Publicación (en la mayoría de los casos, la API devolverá la versión más reciente de una Publicación, a menos que se solicite una versión anterior específica mediante la id de la Publicación).
* Todo el historial de edición de la Publicación.
* La interacción atribuida a cada versión de la Publicación.

Hay tres componentes en el historial de edición de una Publicación:

1. De forma predeterminada, el payload de la Publicación contendrá un arreglo de ids de Publicaciones que forman parte del historial de edición de una Publicación. Esta información se especifica mediante `edit_history_tweet_ids`, que es un campo predeterminado en el payload de la Publicación. Este arreglo contendrá al menos una id: la id de la Publicación original y sin editar. Cuando solo hay una id, eso significa que la Publicación no tiene historial de edición. 
2. Puedes obtener información, como si una Publicación era editable en el momento en que se creó, cuánto tiempo, si lo hay, queda para que se edite una Publicación y cuántas ediciones quedan, especificando `edit_controls` en tu parámetro [`tweet.fields`](/es/x-api/fundamentals/fields).
3. Finalmente, puedes obtener los objetos de Publicación para cada Publicación en el historial de edición de una Publicación especificando `edit_history_tweet_ids` mediante el parámetro [expansion](/es/x-api/fundamentals/expansions).

La mayoría de las Publicaciones son elegibles para edición. Sin embargo, los siguientes tipos de Publicaciones no lo son: 

* La Publicación es promocionada
* La Publicación tiene una encuesta
* La Publicación es una respuesta en un hilo que no es propio
* La Publicación es un Retweet (ten en cuenta que las Publicaciones citadas (Quote Tweets) son elegibles para edición)
* La Publicación es nullcast
* Publicación de comunidad
* Publicación de Superfollow
* Publicación colaborativa

Ejemplos

***

Los ejemplos a continuación muestran cómo un desarrollador puede solicitar metadatos de Publicaciones editadas usando X API v2. 

**Nota:** Los ejemplos a continuación usan el endpoint User Post Timeline, pero puedes solicitar estos metadatos usando los mismos parámetros (con campos y expansiones) para todos los endpoints que devuelven Publicaciones (por ejemplo, búsqueda de Publicaciones, búsqueda, flujo filtrado, etc.).

<div id="default-behavior">
  ### Comportamiento predeterminado
</div>

De forma predeterminada, cuando realizas una solicitud a cualquier endpoint que devuelve objetos Post en la X API v2, recibirás:

* El ID de la Publicación
* El texto de la Publicación
* Una lista de ID de Publicación que forman parte del historial de edición de una Publicación. Si solo se proporciona un ID, significa que la Publicación no se ha editado.

**Solicitud:**

`curl --request GET 'https://api.x.com/2/users/:id/tweets' --header 'Authorization: Bearer $BEARER_TOKEN'`

**Respuesta de ejemplo:**

```{
  "data": [
    {
      "id": "1514991667853602823",
      "text": "we have an edit button",
      "edit_history_tweet_ids": ["1514991667853602822", "1514991667853602823"]
    }
  ]
}
```

<div id="getting-additional-data-with-edit_controls">
  ### Obtener datos adicionales con edit&#95;controls
</div>

Si deseas metadatos adicionales de la Publicación editada, como si una Publicación era elegible para ser editada cuando se creó y cuánto tiempo queda para que una Publicación siga siendo editable, puedes solicitar esta información usando el parámetro tweet.fields y estableciéndolo en edit&#95;control.

**Solicitud:**

`curl --request GET 'https://api.x.com/2/users/:id/tweets?tweet.fields=edit_control' --header 'Authorization: Bearer $BEARER_TOKEN'`

**Respuesta de ejemplo:**

```
  {
  "data": [
    {
      "id": "1514991667853602823",
      "text": "we have an edit button",
      "edit_history_tweet_ids": ["1514991667853602822", "1514991667853602823"],
      "edit_controls": {
        "is_edit_eligible": true,
        "editable_until": "2022-04-21 09:35:20.311",
        "edits_remaining": 4
      }
    }
  ]
}
```

<div id="getting-post-objects-for-all-posts-in-a-posts-edit-history">
  ### Obtener objetos de Publicación para todas las Publicaciones del historial de edición de una Publicación
</div>

Los ejemplos anteriores proporcionan una matriz de id de Publicación en el historial de edición de una Publicación. Si quieres el objeto de Publicación para cada uno de estos id de Publicación, puedes usar el parámetro de expansión y establecerlo en `edit_history_tweet_ids`. Los objetos de Publicación que componen el historial de edición se proporcionarán en el objeto «includes». 

**Solicitud:**

```bash
      curl --location --request GET 'https://api.x.com/2/users/:id/tweets?tweet.fields=edit_control&expansions=edit_history_tweet_ids' --header 'Authorization: Bearer $BEARER_TOKEN'`
```

**Ejemplo de respuesta:**

```
  {
  "data": [
    {
      "id": "1514991667853602823",
      "text": "we have an edit button",
      "edit_history_tweet_ids": ["1514991667853602822", "1514991667853602823"],
      "edit_controls": {
        "is_edit_eligible": true,
        "editable_until": "2022-04-21 09:35:20.311",
        "edits_remaining": 4
      }
    }
  ],
  "includes": {
    "tweets": [
      {
        "id": "1514991667853602822",
        "text": "we need an eidt button",
        "edit_history_tweet_ids": [
          "1514991667853602822",
          "1514991667853602823"
        ],
        "edit_controls": {
          "is_edit_eligible": true,
          "editable_until": "2022-04-21 09:35:20.311",
          "edits_remaining": 4
        }
      }
    ]
  }
}
```
