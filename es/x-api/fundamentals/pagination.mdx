<div id="introduction">
  ## Introducción
</div>

La paginación es una característica de los endpoints de X API v2 que devuelven más resultados de los que se pueden incluir en una sola respuesta. Cuando esto sucede, los datos se devuelven en una serie de &quot;páginas&quot;. La paginación se refiere a los métodos para solicitar de forma programática todas las páginas con el fin de recuperar el conjunto completo de resultados. No todos los endpoints de la API admiten o requieren paginación, pero se utiliza con frecuencia cuando los conjuntos de resultados son grandes.

<div id="use-cases-for-pagination">
  ### Casos de uso de la paginación
</div>

**Para recuperar todos los resultados de una solicitud:** Se debe usar la paginación para obtener todos los datos relevantes relacionados con una solicitud y parámetros específicos. La paginación es necesaria en los casos en los que haya más resultados coincidentes que el valor de `max_results` para una solicitud. Realizar solicitudes en bucle con tokens de paginación te permite recuperar todos los resultados. Una vez que se devuelve una respuesta sin un valor de `next_token`, se puede asumir que se han paginado todos los resultados. No se debe usar la paginación con fines de sondeo. Para obtener los resultados más recientes desde una solicitud previa, consulta el sondeo con `since_id`.

**Para recorrer los resultados de una solicitud:** La paginación proporciona opciones direccionales para navegar por los resultados de una solicitud, utilizando los valores de `next_token` y `previous_token` de las respuestas. Estos tokens se pueden establecer como `pagination_token` en la solicitud siguiente para ir a la página siguiente o anterior de resultados.

<div id="pagination-token-definitions">
  ### Definiciones de tokens de paginación
</div>

* `next_token` - Cadena opaca devuelta dentro del objeto meta de la respuesta en los endpoints que admiten paginación. Indica que hay más resultados disponibles y se puede usar como parámetro `pagination_token` en la siguiente solicitud para devolver la página siguiente de resultados. La última página de resultados no incluirá un `next_token`.
  * `previous_token` - Cadena opaca devuelta dentro del objeto meta de la respuesta en los endpoints que admiten paginación. Indica que hay una página anterior de resultados disponible y se puede establecer como parámetro `pagination_token` en la siguiente solicitud para devolver la página anterior de resultados.
  * `pagination_token` - Parámetro utilizado en las solicitudes de paginación. Establécelo en el valor de `next_token` para la página siguiente de resultados. Establécelo en el valor de `previous_token` para la página anterior de resultados.

<div id="fundamentals-of-pagination">
  ### Fundamentos de la paginación
</div>

* Los endpoints que usan paginación responderán a una solicitud inicial con la primera página de resultados y proporcionarán un `next_token` dentro del objeto meta en la respuesta JSON si hay páginas adicionales de resultados disponibles. Para recibir todos los resultados, repite este proceso hasta que no se incluya ningún `next_token` en la respuesta.
  * Los resultados se entregan en orden cronológico inverso. Esto es válido dentro de las páginas individuales y también entre varias páginas:
    * La primera Publicación en la primera respuesta será la más reciente.
    * La última Publicación en la última respuesta será la más antigua.
  * El parámetro de solicitud `max_results` te permite configurar la cantidad de Publicaciones devueltas por página de respuesta. Hay un `max_results` predeterminado y un `max_results` máximo.
  * Toda implementación de paginación conllevará extraer tokens del payload de la respuesta, que pueden usarse en solicitudes posteriores.
  * En algunas circunstancias, puedes recibir menos resultados que el valor de `max_results` por página. No confíes en que los resultados por página siempre sean iguales al valor del parámetro `max_results`.
  * Las mejores formas de usar la paginación para obtener resultados completos son utilizar lógica de bucle dentro del código de integración o usar una [biblioteca](/es/x-api/tools-and-libraries/overview) que sea compatible con X API v2.

La primera Publicación en la primera página (`1337498609819021312`) es la más reciente, y la última Publicación en la tercera página de resultados (`1082718487011885056`) es la más antigua.
