<div id="introduction">
  ## Introducción
</div>

La paginación es una característica de los endpoints de X API v2 que devuelven más resultados de los que se pueden incluir en una sola respuesta. Cuando eso ocurre, los datos se devuelven en una serie de &quot;páginas&quot;. La paginación se refiere a los métodos programáticos para solicitar todas las páginas y así recuperar el conjunto completo de datos de resultados. No todos los endpoints de la API admiten o requieren paginación, pero suele utilizarse cuando los conjuntos de resultados son grandes.

<div id="use-cases-for-pagination">
  ### Casos de uso de la paginación
</div>

**Para recuperar todos los resultados de una solicitud:** Se debe usar la paginación para recibir todos los datos relevantes relacionados con una solicitud específica y sus parámetros. La paginación es obligatoria en los casos en que haya más resultados coincidentes que el valor de `max_results` de una solicitud. Hacer solicitudes en bucle usando tokens de paginación te permite recuperar todos los resultados. Una vez que se recibe una respuesta sin un valor de `next_token`, se puede asumir que se han paginado todos los resultados. No se debe usar la paginación para realizar sondeos (polling). Para obtener los resultados más recientes a partir de una solicitud anterior, revisa el uso de sondeo con `since_id`.

**Para recorrer los resultados de una solicitud:** La paginación proporciona opciones direccionales para navegar por los resultados de una solicitud, usando los valores `next_token` y `previous_token` de las respuestas. Estos tokens se pueden establecer como el `pagination_token` en la siguiente solicitud para ir a la página siguiente o anterior de resultados.

<div id="pagination-token-definitions">
  ### Definiciones de tokens de paginación
</div>

* `next_token` - Cadena opaca devuelta dentro del objeto meta de la respuesta en endpoints que admiten paginación. Indica que hay más resultados disponibles y puede usarse como el parámetro `pagination_token` en la siguiente solicitud para devolver la página siguiente de resultados. La última página de resultados no incluirá un `next_token`.
  * `previous_token` - Cadena opaca devuelta dentro del objeto meta de la respuesta en endpoints que admiten paginación. Indica que hay una página anterior de resultados disponible y puede establecerse como el parámetro `pagination_token` en la siguiente solicitud para devolver la página anterior de resultados.
  * `pagination_token` - Parámetro usado en solicitudes de paginación. Establécelo con el valor de `next_token` para la página siguiente de resultados. Establécelo con el valor de `previous_token` para la página anterior de resultados.

<div id="fundamentals-of-pagination">
  ### Fundamentos de la paginación
</div>

* Los endpoints que usan paginación responderán a una solicitud inicial con la primera página de resultados y proporcionarán un `next_token` dentro del objeto `meta` en la respuesta JSON si hay páginas adicionales de resultados disponibles. Para recibir todos los resultados, repite este proceso hasta que no se incluya ningún `next_token` en la respuesta.
  * Los resultados se entregan en orden cronológico inverso. Esto es cierto dentro de las páginas individuales y también a través de varias páginas:
    * La primera Publicación en la primera respuesta será la más reciente.
    * La última Publicación en la última respuesta será la más antigua.
  * El parámetro de solicitud `max_results` te permite configurar el número de Publicaciones devueltas por página de respuesta. Hay un valor predeterminado y un valor máximo para `max_results`.
  * Toda implementación de paginación requerirá extraer los tokens del cuerpo (payload) de la respuesta, que se pueden usar en solicitudes posteriores.
  * En algunas circunstancias, puedes recibir menos elementos por página que el valor de `max_results`. No confíes en que los resultados por página siempre sean iguales al valor del parámetro `max_results`.
  * La mejor manera de usar la paginación para obtener resultados completos es mediante lógica de bucle dentro del código de integración o utilizando una [biblioteca](/es/x-api/tools-and-libraries/overview) que sea compatible con X API v2.

La primera Publicación de la primera página (`1337498609819021312`) es la más reciente y la última Publicación de la tercera página de resultados (`1082718487011885056`) es la más antigua.
