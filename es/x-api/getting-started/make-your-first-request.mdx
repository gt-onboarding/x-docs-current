Esta guía te mostrará algunos pasos que puedes seguir para realizar tu primera solicitud. Es un excelente recurso que te ayudará una vez que hayas creado una cuenta en X.

Si te interesa usar ejemplos de código, guías más técnicas o una herramienta gráfica como Postman, te recomendamos usar las siguientes guías para realizar tu primera solicitud:

* [Guía de inicio rápido para realizar tu primera solicitud al endpoint de Publicaciones](/es/x-api/posts/manage-tweets/quickstart)
* [Primeros pasos con Postman](/es/tutorials/postman-getting-started) - ideal para principiantes
* [Código de ejemplo de la X API v2](https://github.com/xdevplatform/Twitter-API-v2-sample-code)
* [Referencia de la API de consulta de Publicaciones](/es/x-api/posts/lookup/introduction)

Esta guía asume que ya has obtenido tu [API key y secret](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret), tu [Token de acceso de usuario y Secret](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens), tu [Token de acceso de la Aplicación](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) y que los has almacenado en un lugar seguro. Puedes aprender cómo hacerlo siguiendo los pasos de la guía [obtener acceso a la X API](/es/x-api/getting-started/getting-access).

<div id="step-1-identify-which-endpoint-you-would-like-to-use">
  ### Paso 1. Identifica qué endpoint te gustaría usar
</div>

La X API te permite realizar, mediante código, muchas de las acciones que también puedes llevar a cabo en el sitio web o la aplicación móvil de X.

Tenemos una lista completa de los endpoints que están disponibles a través de la API en nuestro [Índice de referencia de la API](/es/x-api/introduction), pero recomendamos ceñirte a uno de los siguientes para mayor simplicidad:

* [Administrar publicaciones &gt; Publicar una publicación](/es/x-api/posts/manage-tweets/introduction)
* [Buscar publicaciones](/es/x-api/posts/search/introduction)
* [Búsqueda de usuarios](/es/x-api/users/lookup/introduction)

<div id="step-2-choose-a-tool-to-make-your-request">
  ### Paso 2. Elige una herramienta para realizar tu solicitud
</div>

Aunque algunas solicitudes pueden ser sencillas, otras pueden ser más complejas de armar. Por eso la increíble comunidad de desarrolladores ha creado herramientas que abstraen parte de esa complejidad.

A continuación se muestran algunas herramientas recomendadas y detalles sobre cómo usarlas:

<div id="postman">
  #### Postman
</div>

Postman es una herramienta visual que puedes usar para realizar solicitudes a endpoints REST. Hemos creado material muy útil sobre Postman para ayudarte a comenzar y a explorar los diferentes endpoints disponibles a través de la X API.

Te recomendamos leer nuestro tutorial [&quot;Introducción a Postman&quot;](/es/tutorials/postman-getting-started) para aprender cómo agregar tus claves y tokens y realizar tu primera solicitud.

También hemos elaborado una guía de inicio rápido para cada uno de nuestros endpoints de la X API v2, la mayoría de los cuales usan Postman. Puedes encontrar estas guías en la sección correspondiente de cada endpoint, pero aquí tienes algunos enlaces:

* Inicio rápido: Publicar una Publicación
* Inicio rápido: Buscar Publicaciones
* Inicio rápido: Consultar un usuario

Ten en cuenta que no puedes realizar solicitudes a endpoints de streaming usando Postman. Visita la guía de inicio rápido de [filtered stream](/es/x-api/posts/filtered-stream/quickstart) o de [1% sampled stream](/es/x-api/posts/sample-stream) para aprender cómo trabajar con esos endpoints.

Si prefieres una herramienta gráfica más simple, también deberías considerar usar [Insomnia](https://insomnia.rest).

<div id="sample-code">
  #### Código de ejemplo
</div>

Si te interesa usar código sencillo para hacer tu solicitud, hemos preparado código de ejemplo en varios lenguajes distintos para cada uno de nuestros endpoints de X API v2.

Puedes encontrar los ejemplos de código en nuestro repositorio de GitHub, [X-API-v2-sample-code](https://github.com/xdevplatform/Twitter-API-v2-sample-code), que también contiene un archivo README que puedes usar para aprender cómo configurar tus credenciales para trabajar correctamente con las solicitudes.

Por ejemplo, aquí tienes un ejemplo en cURL para el endpoint de búsqueda de usuario. Todo lo que tienes que hacer para usar esta solicitud es reemplazar $ACCESS&#95;TOKEN y $USERNAME por tu [Token de acceso de la Aplicación](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) y tu handle de X. Luego, copia y pega este código en tu herramienta de línea de comandos y presiona “Return” o “Enter”.

```bash
curl "https://api.x.com/2/users/by/username/$USERNAME" -H "Authorization: Bearer $ACCESS_TOKEN"
```

  #### Bibliotecas

La increíble comunidad de desarrolladores de X también ha creado bibliotecas en diversos lenguajes de programación que se pueden usar para hacer solicitudes a la X API.

<div id="step-3-review-the-response">
  ### Paso 3. Revisa la respuesta
</div>

Una vez que hayas realizado una solicitud correctamente, recibirás una carga útil con metadatos relacionados con la solicitud.

Si usaste un endpoint que utiliza el método HTTP GET, recibirás metadatos relacionados con el recurso (Publicación, usuario, Lista, Space, etc.) al que hiciste la solicitud en formato JSON. Revisa los diferentes campos que se devolvieron y verifica si puedes relacionar la información que solicitaste con el contenido de X.

Si usaste un endpoint que utiliza un método HTTP POST, PUT o DELETE, realizaste una acción en X. Ve a X.com o a la aplicación móvil y verifica si puedes localizar esa acción.

<div id="step-4-adjust-your-request-using-parameters">
  ### Paso 4. Ajusta tu solicitud usando parámetros
</div>

Cada endpoint tiene un conjunto diferente de parámetros que puedes usar para modificar tu solicitud. Por ejemplo, puedes solicitar campos de metadatos adicionales cuando uses endpoints GET con los parámetros fields y expansions. También puedes experimentar con una variedad de herramientas de filtrado con endpoints como [search Posts](/es/x-api/posts/search/introduction), [Post counts](/es/x-api/posts/counts/introduction) y [filtered stream](/es/x-api/posts/filtered-stream/introduction) para ayudarte a reducir los datos que recibes únicamente a las Publicaciones que te interesan.

Puedes encontrar una lista completa de los parámetros de solicitud y campos en la referencia de la API para el endpoint con el que estés trabajando, así como mucha otra información útil para la integración en nuestras guías de integración y páginas de fundamentos.

Puedes obtener más información sobre todos los materiales educativos que hemos puesto a tu disposición en nuestra guía de [Recursos importantes](/es/x-api/getting-started/important-resources).
