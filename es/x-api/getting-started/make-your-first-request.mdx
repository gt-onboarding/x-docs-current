Esta guía te guiará a través de algunos pasos que puedes seguir para realizar tu primera solicitud. Es un excelente recurso que te ayudará una vez que hayas creado una cuenta en X.

Si te interesa usar ejemplos de código, guías más técnicas o una herramienta gráfica como Postman, considera utilizar las siguientes guías para realizar tu primera solicitud:

* [Guía de inicio rápido para realizar tu primera solicitud al endpoint de publicaciones](/es/x-api/posts/manage-tweets/quickstart)
* [Primeros pasos con Postman](/es/tutorials/postman-getting-started) - ideal para principiantes
* [Ejemplos de código para X API v2](https://github.com/xdevplatform/Twitter-API-v2-sample-code)
* [Referencia de la API de búsqueda de publicaciones](/es/x-api/posts/lookup/introduction)

Esta guía asume que ya has obtenido tu [clave y secreto de API](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret), [token de acceso de usuario y secreto](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens), [token de acceso de la aplicación](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only), y que los has almacenado en un lugar seguro. Puedes aprender cómo hacerlo siguiendo los pasos de la guía [obtener acceso a la X API](/es/x-api/getting-started/getting-access).

<div id="step-1-identify-which-endpoint-you-would-like-to-use">
  ### Paso 1. Identifica qué endpoint quieres usar
</div>

La X API te permite realizar una variedad de acciones mediante código, de forma similar a como podrías hacerlo en el sitio web o la aplicación móvil de X.

Tenemos una lista completa de los endpoints que están disponibles a través de la API en nuestro [Índice de referencia de la API](/es/x-api/introduction), pero recomendamos ceñirte a uno de los siguientes para mayor simplicidad:

* [Gestionar publicaciones &gt; Publicar una publicación](/es/x-api/posts/manage-tweets/introduction)
* [Buscar publicaciones](/es/x-api/posts/search/introduction)
* [Búsqueda de usuarios](/es/x-api/users/lookup/introduction)

<div id="step-2-choose-a-tool-to-make-your-request">
  ### Paso 2. Elige una herramienta para realizar tu solicitud
</div>

Aunque algunas solicitudes pueden ser sencillas, otras pueden ser más complicadas de armar. Por eso la increíble comunidad de desarrolladores ha creado herramientas para abstraer parte de esa complejidad.

A continuación se presentan algunas herramientas recomendadas y detalles sobre cómo usarlas:

<div id="postman">
  #### Postman
</div>

Postman es una herramienta visual que puedes usar para realizar solicitudes a endpoints REST. Hemos creado material muy útil sobre Postman para ayudarte a comenzar y a explorar los diferentes endpoints disponibles a través de la X API.

Te recomendamos leer nuestro tutorial [&quot;Introducción a Postman&quot;](/es/tutorials/postman-getting-started) para aprender cómo agregar tus claves y tokens y realizar tu primera solicitud.

También hemos elaborado una guía de inicio rápido para cada uno de nuestros endpoints de X API v2, la mayoría de los cuales usan Postman. Puedes encontrar estas guías en la sección correspondiente a cada endpoint, pero aquí tienes un enlace a algunas:

* Inicio rápido: Publicar una Publicación
* Inicio rápido: Buscar Publicaciones
* Inicio rápido: Buscar un usuario

Ten en cuenta que no puedes realizar solicitudes a endpoints de streaming usando Postman. Visita la guía de inicio rápido de [flujo filtrado](/es/x-api/posts/filtered-stream/quickstart) o de [flujo muestreado del 1%](/es/x-api/posts/sample-stream) para aprender cómo trabajar con esos endpoints.

Si prefieres una herramienta gráfica más simple, también deberías considerar usar [Insomnia](https://insomnia.rest).

<div id="sample-code">
  #### Código de ejemplo
</div>

Si te interesa usar código sencillo para hacer tu solicitud, hemos preparado código de ejemplo en varios lenguajes de programación diferentes para cada uno de nuestros endpoints de X API v2.

Puedes encontrar estos ejemplos de código en nuestro repositorio de GitHub, [X-API-v2-sample-code](https://github.com/xdevplatform/Twitter-API-v2-sample-code), que también contiene un archivo README que puedes usar para aprender a configurar tus credenciales para que las solicitudes funcionen correctamente.

Por ejemplo, aquí tienes un ejemplo en cURL para el endpoint de consulta de usuarios. Todo lo que tienes que hacer para usar esta solicitud es reemplazar $ACCESS&#95;TOKEN y $USERNAME por tu [Token de acceso de App](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) y tu handle de X. Luego, copia y pega este código en tu herramienta de línea de comandos y presiona &#39;Return&#39; o &#39;Enter&#39;.

```bash
curl "https://api.x.com/2/users/by/username/$USERNAME" -H "Authorization: Bearer $ACCESS_TOKEN"
```

La increíble comunidad de desarrolladores de X también ha creado bibliotecas en una variedad de lenguajes de programación que puedes usar para realizar solicitudes a la X API.

> **Ten en cuenta:** Si encuentras algún problema, revisa nuestra documentación para desarrolladores del endpoint al que estás realizando la solicitud, nuestra [sección de soporte](https://developer.x.com/en/support) o ponte en contacto con la comunidad en nuestros foros para obtener ayuda. ¡Te ayudaremos a lograr una solicitud exitosa!

<div id="step-3-review-the-response">
  ### Paso 3. Revisa la respuesta
</div>

Una vez que hayas realizado una solicitud exitosa, recibirás una carga útil con metadatos relacionados con la solicitud.

Si utilizaste un endpoint que usa el método HTTP GET, recibirás metadatos relacionados con el recurso (Publicación, usuario, Lista, Space, etc.) al que hiciste la solicitud en formato JSON. Revisa los diferentes campos devueltos y verifica si puedes vincular la información que solicitaste con el contenido en X.

Si utilizaste un endpoint que usa un método HTTP POST, PUT o DELETE, realizaste una acción en X. Ve a X.com o a la aplicación móvil y verifica si puedes localizar esa acción.

<div id="step-4-adjust-your-request-using-parameters">
  ### Paso 4. Ajusta tu solicitud usando parámetros
</div>

Cada endpoint tiene un conjunto diferente de parámetros que puedes usar para modificar tu solicitud. Por ejemplo, puedes solicitar campos de metadatos adicionales cuando uses endpoints GET con los parámetros `fields` y `expansions`. También puedes experimentar con una variedad de herramientas de filtrado con endpoints como [búsqueda de Publicaciones](/es/x-api/posts/search/introduction), [recuento de Publicaciones](/es/x-api/posts/counts/introduction) y [flujo filtrado](/es/x-api/posts/filtered-stream/introduction) para ayudarte a acotar los datos que recibes solo a aquellas Publicaciones que te interesan.

Puedes encontrar una lista completa de los parámetros de la solicitud y los campos en la Referencia de la API del endpoint con el que estés trabajando, así como mucha otra información útil sobre la integración en nuestras guías de integración y páginas de fundamentos.

Puedes obtener más información sobre todos los materiales educativos que hemos puesto a tu disposición en nuestra guía de [Recursos importantes](/es/x-api/getting-started/important-resources).
