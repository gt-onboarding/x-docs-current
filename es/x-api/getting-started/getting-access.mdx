<div id="step-one-sign-up-for-a-developer-account">
  ### Paso uno: Regístrate para obtener una cuenta de desarrollador
</div>

Registrarte para obtener una cuenta de desarrollador es rápido y sencillo. Solo haz clic en el botón de abajo, responde algunas preguntas y podrás comenzar a explorar y desarrollar con la X API v2 usando [Basic access](https://developer.x.com/en/portal/products/basic).

A continuación, crearás un Proyecto y una aplicación de desarrollador asociada durante el proceso de onboarding, lo que te proporcionará un conjunto de credenciales que usarás para autenticar todas las solicitudes a la API.

[Regístrate](https://developer.x.com/en/portal/products/basic)

<div id="step-two-save-your-apps-key-and-tokens-and-keep-them-secure">
  ### Paso dos: guarda las claves y tokens de tu Aplicación y mantenlos seguros
</div>

Una vez que tengas acceso y hayas creado un Proyecto y una Aplicación, podrás encontrar o generar las siguientes credenciales dentro de tu aplicación de desarrollador:

* **[API Key and Secret](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret):** Básicamente, el nombre de usuario y la contraseña de tu Aplicación. Las usarás para autenticar solicitudes que requieran [Contexto de usuario de OAuth 1.0a](/es/resources/fundamentals/authentication), o para generar otros tokens como tokens de acceso de usuario o el token de acceso de la App.

* **[Access Token and Secret](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens):** En general, los tokens de acceso representan al usuario en cuyo nombre realizas la solicitud. Los que puedes generar a través del portal para desarrolladores representan al usuario que es propietario de la Aplicación. Los usarás para autenticar solicitudes que requieran [Contexto de usuario de OAuth 1.0a](/es/resources/fundamentals/authentication). Si deseas realizar solicitudes en nombre de otro usuario, tendrás que usar el flujo OAuth de tres patas para que ese usuario autorice tu aplicación.

* **[Client ID and Client Secret](/es/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce):** Estas credenciales se usan para obtener un token de acceso de usuario con autenticación OAuth 2.0. De forma similar a OAuth 1.0a, los tokens de acceso de usuario se utilizan para autenticar solicitudes que proporcionan información privada de la cuenta de usuario o que realizan acciones en nombre de otra cuenta, pero con un alcance de permisos granular para un mayor control sobre a qué tiene acceso la aplicación cliente en la cuenta del usuario.

* **[App only Access Token](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only):** Usarás este token cuando hagas solicitudes a endpoints que responden con información disponible públicamente en X.

Dado que estas claves y tokens no caducan a menos que se vuelvan a generar, te sugerimos que las guardes en un lugar seguro, como un gestor de contraseñas, una vez que hayas recibido tus credenciales.

> **Ten en cuenta: tus claves y tokens solo se mostrarán una vez en el [portal para desarrolladores](/es/resources/fundamentals/developer-portal), por lo que es importante que almacenes estas credenciales en tu sistema de gestión de contraseñas tan pronto como las generes.**
>
> Si pierdes u olvidas las claves y tokens, tendrás que volver a generarlas, lo que crea nuevas credenciales e invalida las anteriores. Esto significa que tendrás que actualizar cualquier integración que puedas haber configurado con tus credenciales anteriores.
>
> Obtén más información sobre nuestras [mejores prácticas de autenticación](/es/resources/fundamentals/authentication#authentication-best-practices).
