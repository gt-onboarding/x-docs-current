<div id="step-one-sign-up-for-a-developer-account">
  ### Paso uno: Regístrate para obtener una cuenta de desarrollador
</div>

Registrarte para obtener una cuenta de desarrollador es rápido y sencillo. Solo haz clic en el botón de abajo, responde algunas preguntas y podrás comenzar a explorar y desarrollar con la X API v2 usando [Basic access](https://developer.x.com/en/portal/products/basic).

Luego, durante el proceso de configuración inicial, crearás un Proyecto y una aplicación de desarrollador asociada, lo que te proporcionará un conjunto de credenciales que usarás para autenticar todas las solicitudes que hagas a la API.

[Regístrate](https://developer.x.com/en/portal/products/basic)

<div id="step-two-save-your-apps-key-and-tokens-and-keep-them-secure">
  ### Paso dos: Guarda la clave y los tokens de tu App y mantenlos seguros
</div>

Una vez que tengas acceso y hayas creado un Proyecto y una App, podrás encontrar o generar las siguientes credenciales dentro de tu aplicación de desarrollador:

* **[Clave y secreto de API](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret):** Son esencialmente el nombre de usuario y la contraseña de tu App. Los usarás para autenticar solicitudes que requieren [OAuth 1.0a User Context](/es/resources/fundamentals/authentication) o para generar otros tokens, como Tokens de acceso de usuario o Token de acceso de App.

* **[Token de acceso y secreto](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens):** En general, los Tokens de acceso representan al usuario en cuyo nombre estás realizando la solicitud. Los que puedes generar a través del portal para desarrolladores representan al usuario que es propietario de la App. Los usarás para autenticar solicitudes que requieren [OAuth 1.0a User Context](/es/resources/fundamentals/authentication). Si deseas realizar solicitudes en nombre de otro usuario, deberás usar el flujo OAuth de tres patas para que te autorice.

* **[Client ID y Client Secret](/es/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce):** Estas credenciales se utilizan para obtener un Token de acceso de usuario con autenticación OAuth 2.0. De forma similar a OAuth 1.0a, los Tokens de acceso de usuario se utilizan para autenticar solicitudes que proporcionan información privada de la cuenta de usuario o realizan acciones en nombre de otra cuenta, pero con un alcance de permisos más granular para ofrecer un mayor control sobre el acceso que tiene la aplicación cliente al usuario.

* **[Token de acceso solo de App](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only):** Usarás este token al realizar solicitudes a endpoints que responden con información disponible públicamente en X.

Dado que estas claves y tokens no caducan a menos que se regeneren, te sugerimos que los guardes en una ubicación segura, como un administrador de contraseñas, una vez que hayas recibido tus credenciales.

> **Ten en cuenta: Tus claves y tokens solo se mostrarán una vez en el [developer portal](/es/resources/fundamentals/developer-portal), por lo que es importante que almacenes estas credenciales en tu sistema de gestión de contraseñas tan pronto como las generes.**
>
> Si pierdes u olvidas las claves y tokens, deberás regenerarlos, lo que crea nuevas credenciales e invalida las antiguas. Esto significa que tendrás que actualizar cualquier integración que hayas configurado con tus credenciales anteriores.
>
> Obtén más información sobre nuestras [prácticas recomendadas de autenticación](/es/resources/fundamentals/authentication#authentication-best-practices).
