<div id="overview">
  ## Descripción general
</div>

El grupo de endpoints filtered stream permite a los desarrolladores filtrar un flujo de Publicaciones públicas. La funcionalidad de este grupo de endpoints incluye varios endpoints que te permiten crear y administrar reglas y aplicar esas reglas para filtrar un flujo de Publicaciones que devolverá las Publicaciones públicas que coincidan. Este grupo de endpoints permite a los usuarios escuchar temas y eventos específicos, monitorear la conversación en torno a competiciones, entender cómo se desarrollan las tendencias y mucho más.

Los desarrolladores pueden usar el endpoint REST de [reglas](/es/x-api/posts/filtered-stream#post-2-tweets-search-stream-rules) para agregar y eliminar reglas con el fin de filtrar las Publicaciones que coincidan con un criterio definido. Estas [reglas](/es/x-api/posts/filtered-stream#building-rules-for-filtered-stream) se pueden crear con operadores que hagan coincidir atributos de la Publicación, como palabras clave del mensaje, hashtags y URL. Los operadores y las cláusulas de regla se pueden combinar con lógica booleana y paréntesis para ayudar a refinar el comportamiento de coincidencia del filtro. 

Una vez que hayas agregado un conjunto de reglas, puedes registrar tu webhook, donde X comenzará a entregar [objetos de Publicación](/es/x-api/fundamentals/data-dictionary#tweet) en formato JSON. Solo recibirás en tu webhook contenido que coincida con tus reglas.

Este endpoint admite Publicaciones editadas. Tu webhook recibirá Publicaciones editadas que coincidan con uno o más de tus filtros, junto con su historial de ediciones, que incluye un array de id de Publicación. Para Publicaciones sin historial de ediciones, este array contendrá un único id. Para Publicaciones que hayan sido editadas, este array contiene varios id, ordenados de forma ascendente, lo que refleja el orden de las ediciones, con la versión más reciente en la última posición del array. Para obtener más información sobre cómo funcionan las ediciones de Publicaciones, consulta la página [Fundamentos de las ediciones de Publicaciones](/es/x-api/fundamentals/edit-posts). 

**Nota:** Este grupo de endpoints es similar al [endpoint v2 filtered stream](/es/x-api/posts/filtered-stream/introduction) para configurar tus reglas. La única diferencia son los mecanismos de entrega para las Publicaciones que coincidan con tus reglas.

* En el caso del endpoint v2 filtered stream, debes establecer una conexión persistente y escuchar las Publicaciones que coincidan con tus reglas.
* En el caso de este endpoint webhook de filtered stream, registras tu webhook y X entrega las Publicaciones que coincidan con tus reglas a tu webhook.

Esto elimina la necesidad de manejar desconexiones derivadas de caídas del lado del cliente o del lado del servidor por reinicios normales de nuestro servicio. Dado que probablemente estaremos entregando cientos de eventos por segundo, es necesario que tu
servidor de webhook pueda manejar estos intentos de conexión simultáneos.

Actualmente, este endpoint está disponible para desarrolladores de Enterprise y admite la siguiente configuración:

* Más de 25,000 reglas por Proyecto
* Puedes usar todos los operadores al crear tu regla
* Puedes crear reglas de hasta 2048 caracteres de longitud
* Solicita acceso Enterprise [aquí](https://developer.x.com/en/products/x-api/enterprise/enterprise-api-interest-form)

Las Publicaciones devueltas desde el flujo filtrado cuentan para el [límite de Publicaciones](/es/x-api/fundamentals/post-cap) mensual.
