<div id="overview">
  ## Overview
</div>

El grupo de endpoints filtered stream permite a los desarrolladores filtrar un flujo de Publicaciones públicas. La funcionalidad de este grupo de endpoints incluye múltiples endpoints que te permiten crear y administrar reglas, y aplicar esas reglas para filtrar un flujo de Publicaciones que devolverá Publicaciones públicas coincidentes. Este grupo de endpoints permite a los usuarios escuchar temas y eventos específicos, monitorear la conversación en torno a competiciones, entender cómo se desarrollan las tendencias y mucho más.

Los desarrolladores pueden usar el endpoint de reglas de REST ([rules endpoint](/es/x-api/posts/filtered-stream#post-2-tweets-search-stream-rules)) para agregar y eliminar reglas y así filtrar Publicaciones que coincidan con criterios definidos. Estas [rules](/es/x-api/posts/filtered-stream#building-rules-for-filtered-stream) se pueden crear con operadores que coinciden con atributos de la Publicación, como palabras clave del texto, hashtags y URL. Los operadores y las cláusulas de reglas se pueden combinar con lógica booleana y paréntesis para ayudar a refinar el comportamiento de coincidencia del filtro. 

Una vez que hayas agregado un conjunto de reglas, puedes registrar tu webhook, donde X empezará a entregar [objetos Post](/es/x-api/fundamentals/data-dictionary#tweet) en formato JSON. Solo recibirás en tu webhook contenido que coincida con tus reglas.

Este endpoint es compatible con Publicaciones editadas. Tu webhook recibirá Publicaciones editadas que coincidan con uno o más de tus filtros, junto con su historial de ediciones, incluido un array de ids de Publicación. Para Publicaciones sin historial de ediciones, este array contendrá un único id. Para Publicaciones que hayan sido editadas, este array contiene múltiples ids, ordenados de forma ascendente, lo que refleja el orden de las ediciones, con la versión más reciente en la última posición del array. Para obtener más información sobre cómo funcionan las ediciones de Publicaciones, consulta la página [Posts edits fundamentals](/es/x-api/fundamentals/edit-posts). 

**Nota:** Este grupo de endpoints es similar al [v2 filtered stream endpoint](/es/x-api/posts/filtered-stream/introduction) para configurar tus reglas. La única diferencia son los mecanismos de entrega de las Publicaciones que coinciden con tus reglas.

* En el caso del v2 filtered stream endpoint, debes establecer una conexión persistente y escuchar las Publicaciones que coincidan con tus reglas.
* En el caso de este filtered stream webhook endpoint, registras tu webhook y X entrega a tu webhook las Publicaciones que coincidan con tus reglas.

Esto elimina la necesidad de gestionar desconexiones derivadas de caídas del cliente o caídas del lado del servidor por reinicios normales de nuestro servicio. Dado que probablemente estemos entregando cientos de eventos por segundo, es necesario que tu servidor de webhook sea capaz de manejar estos intentos de conexión simultáneos.

Actualmente, este endpoint está disponible para desarrolladores Enterprise y admite la siguiente configuración:

* 25.000+ reglas por proyecto
* Se pueden usar todos los operadores al crear tu regla
* Se pueden crear reglas de hasta 2048 caracteres de longitud
* Solicita [aquí](https://developer.x.com/en/products/x-api/enterprise/enterprise-api-interest-form) acceso a Enterprise

Las Publicaciones devueltas desde filtered stream cuentan para el [límite mensual de Publicaciones](/es/x-api/fundamentals/post-cap).
