Esta versión inicial de la función de consulta de Mensajes Directos incluye tres métodos GET:

* **GET /2/dm&#95;conversations/with/:participant&#95;id/dm&#95;events** - Recupera eventos de Mensajes Directos asociados con una conversación uno a uno. El parámetro de ruta :participant&#95;id es el ID de usuario de la cuenta que mantiene la conversación con el usuario autenticado que realiza esta solicitud. 
* **GET /2/dm&#95;conversations/:dm&#95;conversation&#95;id/dm&#95;events** - Recupera eventos de Mensajes Directos asociados con un ID de conversación específico, según se indica mediante el parámetro de ruta :dm&#95;conversation&#95;id. 
* **GET /2/dm&#95;events** - Recupera eventos de Mensajes Directos asociados con un usuario, incluidas conversaciones tanto uno a uno como de grupo. Se pueden obtener eventos de hasta 30 días atrás.  

Ten en cuenta que los IDs de eventos de Mensajes Directos son comunes en v1.1 y v2 (así como en la X App), por lo que el método de v1.1 para listar un solo evento se puede usar junto con estos nuevos endpoints de v2. Ten en cuenta también que las APIs de Account Activity Enterprise y Premium son compatibles con mensajes v2 uno a uno, pero todavía no admiten conversaciones de grupo.   

Con esta versión se admiten tres tipos de eventos, y estos endpoints permiten usar parámetros de consulta para filtrarlos:

* **MessageCreate** - Se ha creado un mensaje. 
* **ParticipantsJoin** - Un nuevo participante se ha unido a una conversación. 
* **ParticipantsLeave** - Un participante ha abandonado una conversación. 

Existe un límite de frecuencia por usuario de 300 solicitudes cada 15 minutos para los métodos GET. Este límite de frecuencia se comparte entre estos endpoints GET.

Dado que estás haciendo solicitudes en nombre de un usuario con endpoints de Mensajes Directos v2, debes autenticarte con [OAuth 1.0a User Context](/es/resources/fundamentals/authentication) o con [OAuth 2.0 Authorization Code with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3), utilizando tokens de acceso asociados con usuarios que hayan autorizado tu X App. Para generar estos tokens de acceso con OAuth 1.0a, puedes usar el [flujo OAuth de tres patas](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow). Para generar tokens de acceso de usuario con OAuth 2.0, puedes usar el [flujo de concesión Authorization Code with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

**Configuración de la cuenta**

Para acceder a estos endpoints, necesitarás:

* Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
* Autenticarte usando las claves y tokens de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) que se encuentre dentro de un [Proyecto](/es/resources/fundamentals/projects). 

Obtén más información sobre cómo conseguir acceso a los endpoints de la X API v2 en nuestra [guía de introducción](/es/x-api/getting-started/getting-access).
