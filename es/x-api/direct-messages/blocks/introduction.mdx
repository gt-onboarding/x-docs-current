Los endpoints para gestionar bloqueos de MD te permiten bloquear o desbloquear una cuenta determinada en nombre de un usuario autenticado. Para estos endpoints, hay dos métodos POST disponibles:

* **/2/users/:id/dm/block**: Te permite bloquear una cuenta
* **/2/users/:id/dm/unblock**: Te permite desbloquear una cuenta

<div id="getting-started">
  ### Primeros pasos
</div>

<div id="authentication">
  ### Autenticación
</div>

Como realizas solicitudes en nombre de un usuario, debes autenticar estos endpoints con [Contexto de usuario de OAuth 1.0a](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) o con [OAuth 2.0 Authorization Code with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), y utilizar los tokens de acceso de usuario asociados al usuario en cuyo nombre realizas la solicitud. Puedes generar este token de acceso de usuario utilizando el [flujo OAuth de tres patas](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens) (OAuth 1.0a) o utilizando el [flujo de concesión Authorization Code with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) (OAuth 2.0).

<div id="making-a-request">
  ### Realizar una solicitud
</div>

Block

Una vez que un usuario se haya autenticado en tu aplicación, puedes llamar al endpoint Block en nombre de ese usuario, como se muestra a continuación:

```
      curl --request POST 'https://api.x.com/2/users/:id/dm/block' --header 'Authorization: ••••••'
```

Si la solicitud se realiza correctamente, deberías ver la respuesta JSON como se muestra a continuación:

```
{
   "data": {
       "blocked": true
   }
}
```

**Unblock**

Una vez que un usuario se haya autenticado en tu aplicación, puedes llamar al endpoint Unblock en nombre del usuario como se muestra a continuación:

```
      curl --request POST 'https://api.x.com/2/users/:id/dm/unblock' --header 'Authorization: ••••••'
```

Si la solicitud se completa correctamente, deberías ver la respuesta JSON que se muestra a continuación:

```
{
   "data": {
       "blocked": false
   }
}
```
