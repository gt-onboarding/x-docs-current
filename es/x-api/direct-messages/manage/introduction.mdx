Los Mensajes Directos permiten conversaciones privadas en X. Los Mensajes Directos son una de las funciones más populares de X, con una amplia variedad de casos de uso. Estos casos de uso van desde chats grupales entre amigos hasta la atención al cliente para marcas de todo el mundo. Las nuevas versiones v2 de los endpoints de Mensajes Directos se irán introduciendo por etapas, y esta primera etapa incluye endpoints fundamentales para crear Mensajes Directos y listar eventos de conversaciones de Mensajes Directos. Por primera vez, la X API v2 admite conversaciones de *grupo*.

Esta versión inicial de Manage Direct Messages incluye tres métodos POST para crear Mensajes Directos:

* **POST /2/dm&#95;conversations/with/:participant&#95;id/messages** - Crea un Mensaje Directo uno a uno. Este método crea una nueva conversación 1-1 o recupera la conversación actual y le agrega el Mensaje Directo. El parámetro de ruta :participant&#95;id es el ID de usuario de la cuenta que recibe el mensaje. 
* **POST /2/dm&#95;conversations** - Crea una nueva conversación de grupo y agrega un Mensaje Directo a ella. Estas solicitudes requieren una lista de participantes de la conversación. Ten en cuenta que puedes crear varias conversaciones con la misma lista de participantes. Estas solicitudes siempre devolverán un nuevo ID de conversación. 
* **POST /2/dm&#95;conversations/:dm&#95;conversation&#95;id/messages** - Crea un Mensaje Directo y lo agrega a una conversación existente. El parámetro de ruta :dm&#95;conversation&#95;id es el ID de la conversación a la que se agregará el mensaje. 

Ten en cuenta que los ID de eventos de Mensajes Directos son comunes en v1.1 y v2 (así como en la X App), por lo que los métodos de v1.1 para ocultar/eliminar Mensajes Directos se pueden usar junto con este nuevo endpoint de v2. También ten en cuenta que las APIs de Account Activity Enterprise y Premium admiten mensajes uno a uno de v2, pero aún no admiten conversaciones de grupo.   

Existe un límite de frecuencia por usuario de 200 solicitudes cada 15 minutos para el método POST. También hay un límite de frecuencia de 1000 solicitudes por usuario cada 24 horas. Además, hay un límite de frecuencia de 15000 solicitudes por 24 horas. Ten en cuenta que estos límites de frecuencia se comparten entre estos endpoints POST.

Como estás realizando solicitudes en nombre de un usuario con los endpoints de administración de Publicaciones, debes autenticarte con [OAuth 1.0a User Context](https://developer-staging.x.com/resources/fundamentals/authentication) o [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), y usar un Token de acceso de usuario asociado con un usuario que haya autorizado tu aplicación. Para generar este Token de acceso de usuario con OAuth 1.0a, puedes usar el [flujo OAuth de tres patas](/es/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token). Para generar un Token de acceso de usuario con OAuth 2.0, puedes usar el [flujo de concesión Authorization Code with PKCE](/es/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)).

<Note>
  **Configuración de la cuenta**

  Para acceder a estos endpoints, necesitarás:

  * Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
  * Autenticarte usando las claves y tokens de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) que se encuentre dentro de un [Proyecto](/es/resources/fundamentals/projects). 

  Obtén más información sobre cómo acceder a los endpoints de X API v2 en nuestra [guía de primeros pasos](/es/x-api/getting-started/getting-access).
</Note>
