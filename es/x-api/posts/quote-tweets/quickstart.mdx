<div id="getting-started-with-the-quote-tweets-lookup-endpoint">
  ## Introducción al endpoint de consulta de publicaciones citadas
</div>

Esta guía de inicio rápido te ayudará a realizar tu primera solicitud a los endpoints de consulta de publicaciones citadas con un conjunto de campos específicos usando [Postman](/es/tutorials/postman-getting-started).

Si deseas ver código de ejemplo en diferentes lenguajes de programación, visita nuestro repositorio de [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) en GitHub. 

<div id="prerequisites">
  #### Requisitos previos
</div>

Para completar esta guía, necesitarás tener un conjunto de [claves y tokens](/es/resources/fundamentals/authentication) para autenticar tu solicitud. Puedes generar estas claves y tokens siguiendo estos pasos:

* [Regístrate para obtener una cuenta de desarrollador](https://developer.x.com/en/apply-for-access) y espera la aprobación.
* Crea un [Proyecto](/es/resources/fundamentals/projects) y una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) asociada en el portal para desarrolladores.
* Ve a la página “Keys and tokens” de tu App para generar las credenciales necesarias. Asegúrate de guardar todas las credenciales en una ubicación segura.

**Pasos para crear una solicitud de búsqueda de Quote Tweets**

**Paso uno: Comienza con una herramienta o biblioteca**

Hay varias herramientas, ejemplos de código y bibliotecas que puedes usar para hacer una solicitud a este endpoint, pero aquí usaremos la herramienta Postman para simplificar el proceso.

Para cargar la colección de Postman de X API v2 en tu entorno, haz clic en el siguiente botón:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Agregar X API v2 a Postman
</Button>

Una vez que tengas la colección de X API v2 cargada en Postman, navega a la carpeta timeline y busca la solicitud &quot;Quote Tweets by ID&quot;.

**Paso dos: Autentica tu solicitud**

Para realizar correctamente una solicitud a la X API, debes verificar que tienes los permisos necesarios. Para hacerlo con este endpoint, debes autenticar tu solicitud con uno de los siguientes métodos de autenticación: [OAuth 2.0 solo para la aplicación](/es/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) o [Contexto de usuario de OAuth 1.0a](/es/resources/fundamentals/authentication).

Para simplificar, utilizaremos OAuth 2.0 solo para la aplicación con esta solicitud, pero deberás usar uno de los otros métodos de autenticación si quieres solicitar [métricas](/es/x-api/fundamentals/metrics) privadas o Publicaciones. 

Debes agregar tus claves y tokens, específicamente el [Token de acceso de la aplicación](/es/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token) (también conocido como el Bearer Token solo para la aplicación) en Postman. Puedes hacerlo seleccionando el entorno llamado “X API v2” en la esquina superior derecha de Postman y agregando tus claves y tokens en los campos &quot;initial value&quot; y &quot;current value&quot; (haciendo clic en el ícono de ojo junto al menú desplegable de entornos).

Esta variable se incorporará automáticamente en la pestaña de autorización de la solicitud si lo has hecho correctamente.

**Paso tres: Identifica y especifica la Publicación para la que deseas recuperar los Quote Tweets**

Debes especificar una Publicación para la que deseas recuperar los Quote Tweets dentro de la solicitud pasando el id de la Publicación.

En este ejemplo, el id de la Publicación que usaremos es 1409931481552543749.

En Postman, navega a la pestaña &quot;Params&quot; e introduce este id de la Publicación en la columna &quot;Value&quot; del parámetro id.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 1409931481552543749 |

**Paso cuatro: Identifica y especifica qué campos deseas recuperar**

Si haces clic en el botón &quot;Send&quot; después del paso tres, recibirás en tu respuesta los campos predeterminados del [objeto Post](/es/x-api/fundamentals/data-dictionary#tweet): id,  text y edit&#95;history&#95;tweet&#95;ids.

Si deseas recibir campos adicionales, tendrás que especificar esos campos en tu solicitud con los parámetros [field](/es/x-api/fundamentals/fields) y/o [expansion](/es/x-api/fundamentals/expansions).

Para este ejercicio, solicitaremos tres conjuntos adicionales de campos pertenecientes a distintos objetos:

1. El campo adicional tweet.created&#95;at en los objetos de Publicación principales.

2. Los campos predeterminados de los objetos de usuario de los autores asociados para las Publicaciones devueltas: id, name y username.

3. El campo adicional user.created&#95;at en los objetos de usuario asociados.
    

En Postman, navega a la pestaña &quot;Params&quot; y agrega el siguiente par clave:valor a la tabla &quot;Query Params&quot;:

|     |     |     |     |
| :--- | :--- | :--- | :--- |
| **Clave** | **Valor** | **Campos devueltos** |
| `tweet.fields` | `created_at` | `tweets.created_at` |
| `expansions` | `author_id` | `includes.users.id`, `includes.users.name`, `includes.users.username` |
| `user.fields` | `created_at` | `includes.users.created_at` |     |

Ahora deberías ver la siguiente URL junto al botón &quot;Send&quot;:

`https://api.x.com/2/tweets/:id/quote_tweets?expansions=author_id&tweet.fields=created_at&user.fields=created_at`

**Ten en cuenta lo siguiente:**

En Postman, el parámetro de ruta :id en el campo de la URL **no** se actualizará automáticamente al valor que introduzcas en el campo de parámetros `id`, por lo que la URL anterior incluye `:id` y no 1409931481552543749.

<div id="step-six-paginate-through-your-results">
  #### Paso seis: pagina tus resultados
</div>

En la respuesta anterior, encontrarás un objeto de metadatos en la parte inferior que incluye los siguientes campos:

* results&#95;count
* next&#95;token

En el paso cuatro, como no especificamos un parámetro max&#95;results, solo obtendremos 10 Publicaciones de forma predeterminada. Para acceder a las páginas adicionales de datos, tomaremos el valor del campo next&#95;token de nuestros últimos resultados y agregaremos esa cadena como valor del parámetro pagination&#95;token en la pestaña Params de Postman, manteniendo todo lo demás igual. 

|     |     |
| :--- | :--- |
| **Clave** | **Valor** |
| `pagination_token` | avdjwk0udyx6 |

Una vez que todo esté configurado, puedes hacer clic en &quot;Send&quot; de nuevo y deberías recibir la siguiente página de resultados. 

Hemos preparado una guía sobre [paginación](/es/x-api/fundamentals/pagination) para explicar más a fondo este concepto.
