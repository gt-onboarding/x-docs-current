<div id="getting-started-with-the-manage-bookmarks-endpoints">
  ## Introducción a los endpoints para gestionar marcadores
</div>

Esta guía de inicio rápido te ayudará a realizar tu primera solicitud a los endpoints para gestionar marcadores usando [Postman](/es/tutorials/postman-getting-started).

Si quieres ver código de ejemplo en diferentes lenguajes de programación, visita nuestro repositorio de [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) en GitHub.

<Note>
  **Requisitos previos**

  Para completar esta guía, necesitarás un conjunto de [claves y tokens](/es/resources/fundamentals/authentication) para autenticar tu solicitud. Puedes generar estas claves y tokens siguiendo estos pasos:

  * [Regístrate para obtener una cuenta de desarrollador](https://developer.x.com/en/apply-for-access) y obtén la aprobación.
  * Crea un [Proyecto](/es/resources/fundamentals/projects) y una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) asociada en el portal para desarrolladores.
  * Accede a la página “Keys and tokens” de tu App para generar las credenciales necesarias. Asegúrate de guardar todas las credenciales en un lugar seguro.
</Note>

<div id="steps-to-build-a-manage-bookmarks-request">
  ### Pasos para crear una solicitud para administrar Marcadores
</div>

**Paso uno: Comienza con una herramienta o biblioteca**

Hay varias herramientas, ejemplos de código y bibliotecas que puedes utilizar para realizar una solicitud a este endpoint, pero aquí vamos a usar la herramienta Postman para simplificar el proceso.

Para cargar la colección de Postman de X API v2 en tu entorno, haz clic en el siguiente botón:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Agregar X API v2 a Postman
</Button>

Una vez que tengas la colección de X API v2 cargada en Postman, navega a la carpeta “Bookmarks” y selecciona “Create a Bookmark”.

**Paso dos: Autentica tu solicitud**

Para realizar una solicitud correcta a este endpoint, deberás usar [OAuth 2.0 Authorization Code Flow with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3). Puedes generar un Token de acceso dentro de Postman. 

Ve a la pestaña titulada “Authorization” y selecciona “OAuth 2.0”.

En esta pestaña, asegúrate de seguir estos pasos:

1. Nombra tu token

2. Selecciona el Grant Type como Authorization Code (with PKCE)

3. Actualiza los parámetros:

   **Callback URL** - [https://www.example.com](https://www.example.com/)

   Esta debe coincidir con la callback URL que configuraste en la página de ajustes de autenticación en el portal para desarrolladores.

   **Auth URL** - [https://x.com/i/oauth2/authorize](https://x.com/i/oauth2/authorize)

   **Access Token URL** - [https://api.x.com/2/oauth2/token](https://api.x.com/2/oauth2/token)

   **Client ID** - Copia y pega el OAuth 2.0 Client ID desde el portal para desarrolladores

   **Client Secret **- Copia y pega el OAuth 2.0 Client Secret desde el portal para desarrolladores. Solo necesitarás esto si estás usando un tipo de Aplicación que sea un cliente confidencial.

4. Actualiza los scopes con los siguientes valores: post.read users.read bookmark.write

5. Rellena el campo state con “State”

6. Haz clic donde dice “Generate Token”

7. Presiona el ícono de guardar para guardar los cambios de la carpeta.

Es posible que recibas un mensaje de que no has iniciado sesión en X. Si recibes este error, deberás iniciar sesión en la cuenta de X, dentro de Postman, en cuyo nombre intentas publicar.

**Paso tres: Especifica un usuario**

Con este endpoint, debes especificar el ID de usuario cuyos Marcadores te gustaría recibir en la respuesta. Por ejemplo, el ID de usuario de @XDevelopers es `2244994945`.

En Postman, navega a la pestaña “Params” e introduce tu ID o el de un usuario autenticado como valor para el parámetro id.

|    |     |              |
| :- | :-- | :----------- |
|    | Key | Value        |
|    | id  | `2244994945` |

**Paso cuatro: Especifica una Publicación que quieras agregar a Marcadores**

Deberás navegar a la pestaña “Body” y asegurarte de que el ID de la Publicación esté configurado con el de la Publicación que quieres guardar en tus Marcadores. La carga útil JSON debería ser similar a la que se muestra a continuación.

```
      {"tweet_id": "1460323737035677698"}
```

**Paso cinco: realiza tu solicitud y revisa tu respuesta**

Una vez que tengas todo configurado, haz clic en el botón «Send» y recibirás una respuesta similar a la del siguiente ejemplo:

```
{
   "data": {
       "bookmarked": true
   }
}
```

Para eliminar una Publicación, selecciona la solicitud “Remove a Bookmark”, que también se encuentra en la carpeta “Bookmarks” de la colección X API v2 cargada en Postman. Primero deberás especificar el id del usuario en cuyo nombre estás realizando la solicitud como valor para la columna “`id`”. Este endpoint también requiere el id de la Publicación que deseas eliminar. Luego, en la pestaña “Params”, introduce el id de la Publicación que deseas eliminar como valor para la columna “`tweet_id`”. 

Cuando la solicitud de eliminación se haya realizado correctamente, recibirás una respuesta similar al siguiente ejemplo:

```
{
   "data": {
       "bookmarked": false
   }
}
```
