<div id="authentication">
  #### Autenticación
</div>

Todos los endpoints de X API v2 requieren que autentiques tus solicitudes con un conjunto de credenciales, también conocidas como claves y tokens. 

Estos endpoints específicos requieren el uso del [flujo de código de autorización de OAuth 2.0 con PKCE](/es/resources/fundamentals/authentication/oauth-2-0/authorization-code), lo que significa que debes usar un conjunto de claves y Tokens de acceso de usuario para realizar una solicitud correctamente. Los Tokens de acceso deben estar asociados con el usuario en cuyo nombre estás realizando la solicitud. Si quieres generar un conjunto de Tokens de acceso para otro usuario, este debe autorizar o autenticar tu aplicación usando una [URL de autorización](/es/resources/fundamentals/authentication#authorize-url).

Ten en cuenta que OAuth 2.0 puede ser complejo de usar. Si no estás familiarizado con este método de autenticación, te recomendamos usar una [biblioteca](/es/x-api/tools-and-libraries/overview) o una herramienta como Postman para autenticar correctamente tus solicitudes.

<div id="developer-portal-projects-and-developer-apps">
  #### Portal para desarrolladores, Proyectos y aplicaciones de desarrollador
</div>

Para obtener un conjunto de credenciales de autenticación compatibles con los endpoints de X API v2, debes tener una [cuenta de desarrollador](/es/resources/fundamentals/developer-portal), configurar un [Proyecto](/es/resources/fundamentals/projects) dentro de esa cuenta y crear una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) dentro de ese Proyecto. Luego podrás encontrar tus claves y tokens en tu aplicación de desarrollador. 

<div id="rate-limits">
  #### Límites de frecuencia
</div>

Cada día, muchos miles de desarrolladores realizan solicitudes a la X API. Para ayudar a gestionar el gran volumen de estas solicitudes, se aplican [límites de frecuencia](/es/resources/fundamentals/rate-limits) a cada endpoint, los cuales limitan la cantidad de solicitudes que puedes hacer en nombre de tu aplicación o en nombre de un usuario autenticado. 

Estos endpoints tienen límites de frecuencia a nivel de usuario, lo que significa que el usuario autenticado en cuyo nombre realizas la solicitud solo puede llamar al endpoint un número determinado de veces a través de cualquier aplicación de desarrollador. Hay un límite de frecuencia por usuario de 180 solicitudes por ventana de 15 minutos para el método GET. Con el método GET del endpoint de Bookmarks lookup obtendrás las 800 Publicaciones más recientes que has añadido a marcadores. Además, hay un límite de frecuencia por usuario de 50 solicitudes por ventana de 15 minutos para los métodos POST y DELETE.
