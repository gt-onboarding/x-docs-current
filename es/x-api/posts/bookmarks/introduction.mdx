<div id="manage-bookmarks">
  ### Administrar marcadores
</div>

Tenemos dos métodos disponibles para administrar marcadores: POST y DELETE. El método POST te permite crear marcadores. Del mismo modo, el método DELETE te permite eliminar un marcador específico. 

Hay un límite de uso por usuario de 50 solicitudes cada 15 minutos para los métodos POST y DELETE.

Dado que estás realizando solicitudes en nombre de un usuario con los endpoints para administrar marcadores, debes autenticarte generando un token de acceso de usuario con OAuth 2.0. Puedes usar el [flujo de concesión Authorization Code with PKCE](/es/resources/fundamentals/authentication/oauth-2-0/user-access-token) para hacerlo. Para usar este endpoint debes enviar los scopes `tweet.read`, `users.read` y `bookmark.write`.

<div id="bookmarks-lookup">
  ### Consulta de Marcadores
</div>

El endpoint de consulta de marcadores tiene un método disponible, GET. Este método te permite recuperar marcadores de tu propia cuenta o de una cuenta autenticada. Se proporcionarán tokens de paginación para recorrer conjuntos grandes de resultados en este endpoint.

Hay un límite de frecuencia por usuario de 180 solicitudes por ventana de 15 minutos para el método GET. Con este endpoint obtendrás 800 de tus Publicaciones marcadas más recientes.

Dado que estás realizando solicitudes en nombre de un usuario con los endpoints de consulta de marcadores, debes autenticarte generando un Token de acceso de usuario con OAuth 2.0. Puedes usar el [flujo de concesión Authorization Code con PKCE](/es/resources/fundamentals/authentication/oauth-2-0/user-access-token) para hacerlo. Para usar este endpoint debes incluir los permisos `tweet.read`, `users.read` y `bookmark.read`.

**Configuración de la cuenta**

Para acceder a estos endpoints, necesitarás:

* Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
* Autenticarte utilizando las claves y tokens de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) que se encuentre dentro de un [Proyecto](/es/resources/fundamentals/projects).

Obtén más información sobre cómo acceder a los endpoints de X API v2 en nuestra [guía de primeros pasos](/es/x-api/getting-started/getting-access).
