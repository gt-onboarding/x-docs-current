<div id="standard-v11-compared-to-x-api-v2">
  ## v1.1 estándar frente a X API v2
</div>

Si has estado trabajando con los endpoints estándar v1.1 [POST statuses/update](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-update) y [POST statuses/destroy/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-destroy-id), el objetivo de esta guía es ayudarte a comprender las similitudes y diferencias entre los endpoints estándar y los endpoints de X API v2 para la gestión de Publicaciones.

* **Similitudes**
  * Autenticación
* **Diferencias**
  * URL de los endpoints

  * Requisitos de Aplicación y Proyecto

  * Parámetros de solicitud

<div id="similarities">
  ### Similitudes
</div>

**Autenticación**

Tanto los endpoints estándar de la v1.1 como los de X API v2 que gestionan Publicaciones ([POST statuses/update](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-update) y [POST statuses/destroy/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-destroy-id)) usan [Contexto de usuario de OAuth 1.0a](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Por lo tanto, si anteriormente utilizabas uno de los endpoints estándar de la v1.1, puedes seguir utilizando el mismo método de autenticación si migras a la versión de X API v2. 

<div id="differences">
  ### Diferencias
</div>

**URLs de endpoints**

* Endpoints estándar v1.1:
  * https://api.x.com/1.1/statuses/update.json
    (Crea una Publicación)
  * `https://api.x.com/1.1/statuses/destroy/:id.json`
    (Elimina una Publicación)
* Endpoint de X API v2:
  * https://api.x.com/2/tweets
    (Crea una Publicación)
  * https://api.x.com/2/tweets/:id
    (Elimina una Publicación específica)

<div id="app-and-project-requirements">
  ### Requisitos de Aplicación y Proyecto
</div>

Los endpoints de X API v2 requieren que uses credenciales de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) que esté asociada con un [Proyecto](/es/resources/fundamentals/developer-apps) al autenticar tus solicitudes. Todos los endpoints de X API v1.1 pueden usar credenciales de aplicaciones independientes o de aplicaciones asociadas con un Proyecto.

<div id="request-parameters">
  ### Parámetros de solicitud
</div>

Los siguientes parámetros de solicitud estándar de v1.1 aceptaban dos parámetros de consulta en la URL (user&#95;id o screen&#95;name). X API v2 solo acepta el ID numérico de la Publicación para el endpoint DELETE, y este debe pasarse como parte de la ruta del endpoint.

Para el endpoint POST, será necesario pasar parámetros adicionales en el cuerpo JSON de la solicitud. Puedes obtener más información sobre qué parámetros están disponibles en la [guía de referencia de la API](/es/x-api/posts/manage-tweets/introduction).
