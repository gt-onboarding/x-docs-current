<div id="standard-v11-compared-to-x-api-v2">
  ## v1.1 estándar frente a X API v2
</div>

Si has estado trabajando con los endpoints estándar de v1.1 [POST statuses/update](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-update) y [POST statuses/destroy/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-destroy-id), el objetivo de esta guía es ayudarte a entender las similitudes y diferencias entre los endpoints estándar y los endpoints de administración de Publicaciones de X API v2.

* **Similitudes**
  * Autenticación
* **Diferencias**
  * URLs de los endpoints

  * Requisitos de la aplicación y del proyecto

  * Parámetros de solicitud

<div id="similarities">
  ### Similitudes
</div>

**Autenticación**

Tanto los endpoints estándar de v1.1 como los de X API v2 para gestionar Publicaciones ([POST statuses/update](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-update) y [POST statuses/destroy/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-destroy-id)) utilizan [Contexto de usuario de OAuth 1.0a](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Por lo tanto, si ya utilizabas alguno de los endpoints estándar de v1.1, puedes seguir usando el mismo método de autenticación al migrar a la versión de X API v2. 

<div id="differences">
  ### Diferencias
</div>

**URLs de endpoints**

* Endpoints estándar de v1.1:
  * https://api.x.com/1.1/statuses/update.json
    (Crea una Publicación)
  * `https://api.x.com/1.1/statuses/destroy/:id.json`
    (Elimina una Publicación)
* Endpoints de X API v2:
  * https://api.x.com/2/tweets
    (Crea una Publicación)
  * https://api.x.com/2/tweets/:id
    (Elimina una Publicación específica)

<div id="app-and-project-requirements">
  ### Requisitos de Aplicaciones y Proyectos
</div>

Los endpoints de X API v2 requieren que utilices credenciales de una [developer App](/es/resources/fundamentals/developer-apps) que esté asociada con un [Project](/es/resources/fundamentals/developer-apps) cuando autentiques tus solicitudes. Todos los endpoints de X API v1.1 pueden usar credenciales de aplicaciones independientes o de aplicaciones asociadas con un proyecto.

<div id="request-parameters">
  ### Parámetros de la solicitud
</div>

Los siguientes parámetros de solicitud estándar de v1.1 aceptaban dos parámetros de consulta (user&#95;id o screen&#95;name). La X API v2 solo acepta el id numérico de la Publicación para el endpoint DELETE, y debe pasarse como parte de la ruta del endpoint.

Para el endpoint POST, será necesario pasar parámetros adicionales en el cuerpo JSON de la solicitud. Puedes obtener más información sobre qué parámetros están disponibles en la [guía de referencia de la API](/es/x-api/posts/manage-tweets/introduction).
