## Primeros pasos con los endpoints de gestión de Publicaciones**

Esta guía de inicio rápido te ayudará a realizar tu primera solicitud a los endpoints de gestión de Publicaciones con [Postman](/es/tutorials/postman-getting-started).

Visita nuestro repositorio de GitHub [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) si quieres ver código de ejemplo en distintos lenguajes de programación.

<Info>
  **Requisitos previos**

  Para completar esta guía, necesitarás un conjunto de [claves y tokens](/es/resources/fundamentals/authentication) para autenticar tu solicitud. Puedes generar estas claves y tokens siguiendo estos pasos:

  * [Regístrate para obtener una cuenta de desarrollador](https://developer.x.com/en/apply-for-access) y espera la aprobación.
  * Crea un [Proyecto](/es/resources/fundamentals/projects) y una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) asociada en el portal para desarrolladores.
  * Navega a la página “Keys and tokens” de tu App para generar las credenciales necesarias. Asegúrate de guardar todas las credenciales en una ubicación segura.
</Info>

<div id="steps-to-building-manage-posts-requests">
  ### Pasos para crear solicitudes a los endpoints Manage Posts
</div>

**Paso uno: Comienza con una herramienta o biblioteca**

Hay varias herramientas, ejemplos de código y bibliotecas que puedes usar para realizar una solicitud a este endpoint, pero aquí vamos a usar la herramienta Postman para simplificar el proceso.

Para cargar la colección de Postman de X API v2 en tu entorno, haz clic en el siguiente botón:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Agregar X API v2 a Postman
</Button>

Una vez que tengas la colección de X API v2 cargada en Postman, navega a la carpeta “Manage Posts” y selecciona “Create a Post”.
 

**Paso dos: Autentica tu solicitud**

Para realizar correctamente una solicitud a la X API, debes verificar que tienes permiso para hacerlo. Para esto, con los endpoints de Manage Posts, debes autenticar tu solicitud usando [OAuth 1.0a User Context](/es/resources/fundamentals/authentication) o [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

En este ejemplo, vamos a usar OAuth 1.0a User Context.

Debes agregar tus claves y tokens (y específicamente tu API Key, API Secret Key, OAuth 1.0a user Access Token y OAuth 1.0a user Access Token Secret) a Postman. Puedes hacer esto seleccionando el entorno llamado “X API v2” (en la esquina superior derecha de Postman) y agregando tus claves y tokens a los campos &quot;initial value&quot; y &quot;current value&quot; (haciendo clic en el ícono de ojo junto al menú desplegable del entorno).

Si has hecho esto correctamente, estas variables se incorporarán automáticamente en la pestaña de autorización de la solicitud.

**Paso tres: Especifica el texto de la Publicación**

Al crear una nueva Publicación con este endpoint, el texto o el contenido multimedia de la Publicación son los parámetros obligatorios del cuerpo.

En Postman, navega a la pestaña “Body” e introduce el texto de la Publicación como valor del parámetro `text`. Además, si deseas agregar parámetros para elementos como encuestas, responder a una Publicación por id o agregar configuraciones de respuesta, también puedes hacerlo aquí. Puedes obtener más información sobre lo que está disponible en nuestra [guía de referencia de la API](/es/x-api/posts/manage-tweets/introduction).

|           |              |                       |
| :-------- | :----------- | :-------------------- |
| **Clave** | **Valor**    | **Tipo de parámetro** |
| `text`    | Hello world! | body                  |

**Paso cuatro: Identifica y especifica qué campos deseas recuperar**

Una vez que tengas todo configurado, haz clic en el botón &quot;Send&quot; y recibirás una respuesta similar al siguiente ejemplo de respuesta:

```
{
  "data": {
    "id": "1445880548472328192",
    "text": "Hello world!"
  }
}
```

Si el objeto de respuesta devuelto contiene un `id` y el `text` de tu Publicación, significa que has creado correctamente una Publicación.
 

**Paso cinco: Eliminar tu Publicación**

Para eliminar una Publicación, selecciona la solicitud &quot;Delete a Post&quot; que también se encuentra en la carpeta &quot;Manage Posts&quot; de la colección de X API v2 cargada en Postman. Este endpoint requiere el ID de la Publicación que deseas eliminar. Luego, en la pestaña &quot;Params&quot;, introduce el ID de la Publicación que deseas eliminar como valor para la columna `id`. 

Si la solicitud de eliminación se realiza correctamente, recibirás una respuesta similar al siguiente ejemplo:

```
{
   "data": {
       "deleted" : true
   }
}
```
