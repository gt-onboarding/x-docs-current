Crear y eliminar Publicaciones con la X API es esencial para interactuar con la conversación pública. Los nuevos endpoints de gestión de Publicaciones te permiten hacer precisamente eso y mucho más.

Tenemos dos métodos disponibles para gestionar Publicaciones, POST y DELETE. El método POST te permite publicar encuestas, citar Publicaciones, publicar con configuraciones de respuesta, publicar con ubicación, publicar con contenido multimedia y etiquetar usuarios, y publicar para Super Followers, además de otras funcionalidades. Del mismo modo, el método DELETE te permite eliminar una Publicación específica. Para el método POST, puedes pasar [los parámetros](/es/x-api/posts/creation-of-a-post) que necesites para personalizar aún más tu solicitud.

El método &#39;delete Post&#39; se ha actualizado para admitir Publicaciones editadas. Cuando se elimina cualquier Publicación en una cadena de ediciones de Publicaciones, todas las Publicaciones de esa cadena de edición también se eliminan. Para obtener más información sobre los metadatos de edición de Publicaciones, consulta la página [Aspectos básicos sobre Editar Publicaciones](/es/x-api/fundamentals/edit-posts).

Existe un límite de uso por usuario de 200 solicitudes cada 15 minutos para el método POST. El método DELETE tiene un límite de 50 solicitudes cada 15 minutos. Además, hay un límite de 300 solicitudes cada 3 horas, incluidas las Publicaciones creadas con los endpoints de gestión de Publicaciones o de gestión de Retweets.

Dado que estás realizando solicitudes en nombre de un usuario con los endpoints de gestión de Publicaciones, debes autenticarte con [OAuth 1.0a User Context](https://en.docs/authentication/oauth-1-0a) o con [OAuth 2.0 Authorization Code with PKCE](https://en.docs/authentication/oauth-2-0/authorization-code), y utilizar tokens de acceso de usuario asociados con un usuario que haya autorizado tu aplicación. Para generar este token de acceso de usuario con OAuth 1.0a, puedes usar el [flujo OAuth de tres patas](https://en.docs/authentication/oauth-2-0/bearer-tokens). Para generar un token de acceso de usuario con OAuth 2.0, puedes usar el [flujo de autorización Authorization Code with PKCE](https://en.docs/authentication/oauth-2-0/user-access-token).
