Crear y eliminar Publicaciones con la X API es esencial para interactuar con la conversación pública. Los nuevos endpoints de gestión de Publicaciones te permiten hacer justamente eso y mucho más.

Tenemos dos métodos disponibles para gestionar Publicaciones, POST y DELETE. El método POST te permite publicar encuestas, citar Publicaciones, publicar con configuración de las respuestas, publicar con datos de ubicación (geo), publicar con contenido multimedia y etiquetar usuarios, y publicar para Super Followers, además de otras funcionalidades. Del mismo modo, el método DELETE te permite eliminar una Publicación específica. Para el método POST, puedes pasar [los parámetros](/es/x-api/posts/creation-of-a-post) que necesites para personalizar aún más tu solicitud.

El método &#39;delete Post&#39; se ha actualizado para admitir Publicaciones editadas. Cuando se elimina cualquier Publicación en una cadena de ediciones de Publicación, también se eliminan todas las Publicaciones de esa cadena de edición. Para obtener más información sobre los metadatos de Edit Post, consulta la página [fundamentos de Edit Posts](/es/x-api/fundamentals/edit-posts).

Existe un límite de frecuencia por usuario de 200 solicitudes cada 15 minutos para el método POST. El método DELETE tiene un límite de frecuencia de 50 solicitudes cada 15 minutos. Además, hay un límite de 300 solicitudes cada 3 horas, incluidas las Publicaciones creadas tanto con manage Posts como con manage Retweets.

Dado que estás realizando solicitudes en nombre de un usuario con los endpoints de manage Posts, debes autenticarte con [OAuth 1.0a User Context](https://en.docs/authentication/oauth-1-0a) o [OAuth 2.0 Authorization Code with PKCE](https://en.docs/authentication/oauth-2-0/authorization-code), y usar Tokens de acceso de usuario asociados a un usuario que haya autorizado tu Aplicación. Para generar este Token de acceso de usuario con OAuth 1.0a, puedes usar el [flujo OAuth de tres patas](https://en.docs/authentication/oauth-2-0/bearer-tokens). Para generar un Token de acceso de usuario con OAuth 2.0, puedes usar el [flujo de concesión Authorization Code with PKCE](https://en.docs/authentication/oauth-2-0/user-access-token).
