El grupo de endpoints de flujo filtrado permite a los desarrolladores filtrar un flujo de Publicaciones públicas. La funcionalidad de este grupo de endpoints incluye varios endpoints que le permiten crear y administrar reglas, y aplicar esas reglas para filtrar un flujo de Publicaciones que devolverá Publicaciones públicas coincidentes. Este grupo de endpoints permite a los usuarios escuchar temas y eventos específicos, monitorear la conversación en torno a competiciones, entender cómo se desarrollan las tendencias y mucho más.

Los desarrolladores pueden usar el [endpoint de reglas](/es/x-api/posts/filtered-stream#post-2-tweets-search-stream-rules) REST para agregar y eliminar reglas en una conexión de flujo persistente sin necesidad de desconectarse. Estas [reglas](/es/x-api/posts/filtered-stream#building-rules-for-filtered-stream) se pueden crear con operadores que hagan coincidencias sobre atributos de la Publicación, como palabras clave del mensaje, hashtags y URL. Los operadores y las cláusulas de reglas se pueden combinar con lógica booleana y paréntesis para ayudar a refinar el comportamiento de coincidencia del filtro.

Una vez que haya agregado un conjunto de reglas, puede [establecer una conexión de streaming](/es/x-api/posts/filtered-stream#get-2-tweets-search-stream) que empezará a entregar [objetos Post](/es/x-api/fundamentals/data-dictionary#tweet) en formato JSON a través de una conexión de streaming HTTP persistente. Solo recibirá contenido que coincida con sus reglas mientras esté conectado al flujo.

De forma alternativa, puede vincular Filtered Stream a un webhook para la entrega como eventos de webhook. Para obtener más información sobre el uso de webhooks con Filtered Stream, consulte la [introducción a Filtered Stream Webhooks](/es/x-api/webhooks/stream/introduction).

El endpoint de búsqueda filtrada es compatible con Publicaciones editadas. Este endpoint entregará Publicaciones editadas que coincidan con uno o más de sus filtros, junto con su historial de ediciones, incluido un arreglo de id de Publicación. Para las Publicaciones sin historial de ediciones, este arreglo contendrá un único id. Para las Publicaciones que hayan sido editadas, este arreglo contiene varios id, ordenados de forma ascendente para reflejar el orden de las ediciones, con la versión más reciente en la última posición del arreglo. Para obtener más información sobre cómo funcionan las ediciones de Publicaciones, consulte la página [Fundamentos de edición de Publicaciones](/es/x-api/fundamentals/edit-posts).

Ciertos aspectos del endpoint de flujo filtrado están limitados por el [nivel de acceso](/es/x-api/getting-started/about-x-api):

**Acceso Pro**

* 1000 reglas por Proyecto
* 100 solicitudes cada 15 minutos cuando se usa el endpoint POST /2/tweets/search/stream/rules para agregar reglas
* Puede usar todos los operadores al crear su regla
* Puede crear reglas de hasta 1024 caracteres de longitud

**Acceso Enterprise**

* 25,000+ reglas por Proyecto
* Puede usar todos los operadores al crear su regla
* Puede crear reglas de hasta 2048 caracteres de longitud
* Solicite acceso Enterprise [aquí](https://developer.x.com/en/products/x-api/enterprise/enterprise-api-interest-form)

Las Publicaciones devueltas por el flujo filtrado cuentan para el [límite mensual de Publicaciones](/es/x-api/fundamentals/post-cap).

<Note>
  **Configuración de la cuenta**

  Para acceder a estos endpoints, necesitará:

  * Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
  * Autenticarse usando las claves y tokens de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) que esté ubicada dentro de un [Proyecto](/es/resources/fundamentals/projects).

  Obtenga más información sobre cómo acceder a los endpoints de X API v2 en nuestra [guía de introducción](/es/x-api/getting-started/getting-access).
</Note>
