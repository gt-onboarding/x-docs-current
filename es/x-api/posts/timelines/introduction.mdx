Estos tres endpoints de timelines admiten Publicaciones editadas. Estos endpoints siempre devolverán la edición más reciente, junto con el historial de ediciones. Cualquier Publicación obtenida después de su ventana de edición de 30 minutos representará su versión final. Los metadatos de edición incluyen un array de ID de todas las Publicaciones en su historial. Para las Publicaciones sin historial de edición, este array contendrá un único ID. Para las Publicaciones que han sido editadas, este array contiene múltiples ID, ordenados de forma ascendente que refleja el orden de las ediciones, con la versión más reciente en la última posición del array. Para obtener más información sobre cómo funcionan las ediciones de Publicaciones, consulta la página [Conceptos básicos sobre la edición de Publicaciones](/es/x-api/fundamentals/edit-posts).

<div id="reverse-chronological-home-timeline">
  ## Cronología de inicio en orden cronológico inverso
</div>

Este endpoint te permite recuperar las Publicaciones, Retweets y respuestas más recientes publicadas por el usuario autenticado y las cuentas que sigue. 

Dado que estás realizando solicitudes en nombre de un usuario, debes autenticar estos endpoints usando un [flujo de código de autorización de OAuth 2.0 con PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) o [Contexto de usuario de OAuth 1.0a](/es/resources/fundamentals/authentication). Este endpoint tiene un límite de uso por usuario de 180 solicitudes por ventana de 15 minutos. Este endpoint puede devolver todas las Publicaciones creadas en la cronología durante los últimos 7 días, así como las 800 más recientes, independientemente de su fecha de creación.

<div id="user-post-timeline">
  ## Cronología de Publicaciones de un usuario
</div>

El endpoint de cronología de Publicaciones de un usuario ofrece acceso a las Publicaciones realizadas por una cuenta específica de X. Recuperar las Publicaciones de un usuario te permite crear experiencias como mostrar una cronología en una interfaz de usuario, analizar las Publicaciones de un usuario para comprender mejor su contenido o crear flujos de trabajo de interacción con sus Publicaciones de forma programática. Este endpoint te da acceso a las Publicaciones, Retweets, respuestas y Publicaciones citadas más recientes de una sola cuenta de X, similar a lo que se puede ver en la cronología del perfil de un usuario.

Esta es una cronología de usuario para @XDevelopers:

El endpoint de cronología de Publicaciones de un usuario es un endpoint REST que recibe un único parámetro de ruta para indicar el usuario deseado (por ID de usuario). El endpoint puede devolver las 3.200 Publicaciones, Retweets, respuestas y Publicaciones citadas más recientes publicadas por el usuario.

Las Publicaciones se entregan en orden cronológico inverso, comenzando por la más reciente. Los resultados se [paginarán](/es/x-api/fundamentals/pagination) hasta 100 Publicaciones por página. Se proporcionan tokens de paginación para recorrer conjuntos grandes de Publicaciones. Los ID de Publicación de las Publicaciones más reciente y más antigua incluidas en una página determinada también se proporcionan como metadatos, que también pueden utilizarse para consultar cronologías en busca de Publicaciones recientes. La cronología de Publicaciones de un usuario también admite la opción de especificar los parámetros start&#95;time y end&#95;time para recibir Publicaciones que se hayan creado dentro de una determinada ventana de tiempo. 

El endpoint de cronología de Publicaciones de un usuario admite los parámetros [fields](/es/x-api/fundamentals/fields) y [expansions](/es/x-api/fundamentals/expansions), y devuelve el [nuevo formato de datos JSON](/es/x-api/fundamentals/data-dictionary).

Para realizar correctamente una solicitud a este endpoint, deberás autorizar tu solicitud con los métodos de autenticación [Contexto de usuario de OAuth 1.0a](/es/resources/fundamentals/authentication), [OAuth 2.0 Authorization Code with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) o [OAuth 2.0 solo para la aplicación](/es/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token). Debes utilizar Contexto de usuario de OAuth 1.0a u OAuth 2.0 Authorization Code with PKCE cuando solicites métricas no públicas, métricas promocionadas o la cronología de un usuario protegido. 

El endpoint de cronología de Publicaciones de un usuario está diseñado para admitir dos patrones de uso comunes: 

* &quot;Obtener las Publicaciones históricas de un usuario&quot;: Solicitudes realizadas al endpoint de cronología de Publicaciones de un usuario para recibir las Publicaciones creadas por el usuario de interés en orden cronológico durante un periodo reciente específico. El periodo se puede establecer utilizando start&#95;time y end&#95;time y paginando por todos los resultados. En algunos casos, se puede recuperar todo el historial de Publicaciones de un usuario si el usuario solo ha creado hasta 3.200 Publicaciones en su cuenta. Las Publicaciones incluidas dependerán de la disponibilidad pública y de la autenticación que se utilice para las solicitudes.

* &quot;Sondeo de nuevas Publicaciones&quot;: Solicitudes realizadas al endpoint de cronología de Publicaciones de un usuario de forma continua, para recuperar nuevas Publicaciones creadas por un usuario específico. El último ID de Publicación recibido puede establecerse como parámetro para cualquier solicitud nueva desde la última Publicación.

<div id="user-mention-timeline">
  ## Cronología de menciones de usuario
</div>

El endpoint de cronología de menciones de usuario te permite solicitar Publicaciones que mencionan a un usuario específico de X, por ejemplo, si una cuenta de X menciona a @XDevelopers en una Publicación. Esto también incluirá las respuestas a las Publicaciones del usuario solicitado. Recuperar las menciones de un usuario te permite crear experiencias como descubrir rápidamente quién está respondiendo a las Publicaciones de un usuario, quién lo está mencionando o crear flujos de trabajo de interacción con sus Publicaciones de forma programática. El endpoint te permite solicitar las menciones y respuestas más recientes de un único usuario, de forma similar a lo que puede verse en las [notificaciones de menciones](https://x.com/notifications/mentions) de un usuario en X.

La cronología de menciones de usuario es un endpoint REST que recibe un único parámetro de ruta para indicar el usuario deseado (por su id de usuario). El endpoint puede devolver las 800 menciones más recientes para ese usuario.

Las Publicaciones se devuelven en orden cronológico inverso, comenzando por la más reciente. Los resultados se [paginan](/es/x-api/fundamentals/pagination) en hasta 100 Publicaciones por página. Se proporcionan tokens de paginación para avanzar por conjuntos grandes de Publicaciones. Los id de Publicación de las Publicaciones más nuevas y más antiguas incluidas en una página determinada también se proporcionan como metadatos, que también se pueden usar para consultar cronologías en busca de Publicaciones recientes o para navegar por la cronología de forma similar al endpoint v1.1 [mentions&#95;timeline](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-mentions_timeline). El endpoint también admite la posibilidad de especificar los parámetros start&#95;time y end&#95;time para recibir Publicaciones que se crearon dentro de una determinada ventana de tiempo. 

Para realizar correctamente una solicitud a este endpoint, deberás autorizar tu solicitud con los métodos de autenticación [OAuth 1.0a User Context](/es/resources/fundamentals/authentication), [OAuth 2.0 Authorization Code with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) o [OAuth 2.0 App-Only](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only). Debes usar OAuth 1.0a User Context u OAuth 2.0 Authorization Code with PKCE cuando solicites métricas no públicas, métricas promocionadas o la cronología de un usuario protegido.

El endpoint de cronología de menciones de usuario admite los parámetros [fields](/es/x-api/fundamentals/fields) y [expansions](/es/x-api/fundamentals/expansions), y devuelve el [nuevo formato de datos JSON](/es/x-api/fundamentals/data-dictionary).
