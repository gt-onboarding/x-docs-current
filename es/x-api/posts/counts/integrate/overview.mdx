<div id="key-concepts">
  ### Conceptos clave
</div>

<div id="authentication">
  #### Autenticación
</div>

Todos los endpoints de la X API v2 requieren que las solicitudes estén [autenticadas](/es/resources/fundamentals/authentication) con un conjunto de credenciales, también conocidas como claves y tokens. Este endpoint específico requiere el uso de [OAuth 2.0 Bearer Token](/es/resources/fundamentals/authentication#oauth-2-0), lo que significa que debes enviar un [Bearer Token](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) para realizar una solicitud satisfactoria. Puedes generar un Bearer Token directamente en una aplicación de desarrollador o generar uno usando el endpoint [POST oauth2/token](/es/resources/fundamentals/authentication#post-oauth2-token).

<div id="developer-portal-projects-and-developer-apps">
  #### Portal para desarrolladores, Proyectos y aplicaciones de desarrollador
</div>

Para trabajar con cualquier endpoint de X API v2, debes tener una [cuenta de desarrollador](/es/resources/fundamentals/developer-portal), configurar un [Proyecto](/es/resources/fundamentals/projects) dentro de esa cuenta y crear una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) dentro de ese Proyecto. Tus claves y tokens dentro de esa aplicación de desarrollador funcionarán con los endpoints de conteo de publicaciones recientes. Si deseas usar el endpoint de conteo de publicaciones del archivo completo, o utilizar los operadores avanzados y una consulta de mayor longitud, deberás haber sido aprobado para acceso a Enterprise.

Visita nuestra sección sobre acceso a Enterprise para obtener más información.

<div id="rate-limits">
  #### Límites de frecuencia
</div>

Cada día, muchos miles de desarrolladores realizan solicitudes a la X API. Para ayudar a gestionar el volumen, se aplican [límites de frecuencia](/es/x-api/fundamentals/rate-limits) a cada endpoint, que limitan la cantidad de solicitudes que cada desarrollador puede realizar en nombre de una aplicación o en nombre de un usuario autenticado.

Este endpoint tiene límites de frecuencia a nivel de aplicación, lo que significa que tú, como desarrollador, solo puedes realizar una cierta cantidad de solicitudes a este endpoint durante un período de tiempo determinado desde cualquier aplicación (identificada por las credenciales que estés utilizando). 

<div id="building-queries">
  #### Creación de consultas
</div>

La característica central de estos endpoints es el uso de una única consulta para filtrar las Publicaciones y obtener los recuentos que se te entregan. Estas consultas se componen de operadores que se aplican a atributos de la Publicación y del usuario, como palabras clave del mensaje, hashtags y URL. Los operadores se pueden combinar en consultas con lógica booleana y paréntesis para ayudar a refinar cómo la consulta realiza las coincidencias.

Puedes consultar nuestra guía sobre [cómo crear una consulta](/es/x-api/posts/counts/integrate/build-a-query) para obtener más información.

<div id="pagination">
  #### Paginación
</div>

Para recuentos de Publicaciones recientes, no se devuelve ningún next&#95;token, lo que significa que, independientemente de la granularidad, obtendrás el volumen de Publicaciones de los últimos 7 días en una sola llamada a la API.

Para recuentos de Publicaciones de archivo completo, obtendrás datos de los últimos 30 días. Para datos de más de 30 días, obtendrás un next&#95;token que luego podrás usar para paginar y obtener datos adicionales. 

<div id="building-queries-for-post-counts">
  ### Creación de consultas para las cantidades de Publicaciones
</div>

Los endpoints de cantidad de Publicaciones aceptan una única consulta mediante una solicitud GET y devuelven un conjunto de cantidades históricas de Publicaciones que coinciden con la consulta. Las consultas se componen de operadores que se utilizan para hacer coincidir diversos atributos de las Publicaciones.
