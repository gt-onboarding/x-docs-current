</div>

</div>

Postman es una excelente herramienta que puedes usar para probar estos endpoints. Cada solicitud de Postman incluye todos los parámetros del endpoint correspondiente para ayudarte a comprender rápidamente qué opciones tienes disponibles. Para obtener más información sobre nuestras colecciones de Postman, visita nuestra página [Uso de Postman](/es/tutorials/postman-getting-started).
 

<div id="key-concepts">
  ### Conceptos clave
</div>

<div id="authentication">
  #### Autenticación
</div>

Todos los endpoints de X API v2 requieren que las solicitudes se [autentiquen](/es/resources/fundamentals/authentication) con un conjunto de credenciales, también conocidas como claves y tokens. Este endpoint específico requiere el uso de [OAuth 2.0 Bearer Token](/es/resources/fundamentals/authentication#oauth-2-0), lo que significa que debes enviar un [Bearer Token](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) para realizar una solicitud satisfactoria. Puedes generar un Bearer Token directamente desde una aplicación de desarrollador o generar uno usando el endpoint [POST oauth2/token](/es/resources/fundamentals/authentication#post-oauth2-token).

<div id="developer-portal-projects-and-developer-apps">
  #### Portal para desarrolladores, Proyectos y aplicaciones de desarrollador
</div>

Para trabajar con cualquier endpoint de X API v2, debes tener una [cuenta de desarrollador](/es/resources/fundamentals/developer-portal), configurar un [Proyecto](/es/resources/fundamentals/projects) dentro de esa cuenta y crear una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) dentro de ese Proyecto. Tus claves y tokens dentro de esa aplicación de desarrollador funcionarán para los endpoints de conteo de Publicaciones recientes. Si deseas usar el endpoint de conteo de Publicaciones de archivo completo, o utilizar los operadores avanzados y una longitud de consulta mayor, deberás haber obtenido acceso Enterprise.

Visita nuestra sección sobre acceso Enterprise para obtener más información.

<div id="rate-limits">
  #### Límites de frecuencia
</div>

Cada día, muchos miles de desarrolladores realizan solicitudes a X API. Para ayudar a gestionar el volumen, se aplican [límites de frecuencia](/es/x-api/fundamentals/rate-limits) a cada endpoint que limitan la cantidad de solicitudes que cada desarrollador puede hacer en nombre de una aplicación o en nombre de un usuario autenticado.

Este endpoint tiene límites de frecuencia a nivel de aplicación, lo que significa que tú, como desarrollador, solo puedes hacer una cierta cantidad de solicitudes a este endpoint durante un período de tiempo determinado desde cualquier aplicación dada (según las credenciales que estés usando). 

<div id="building-queries">
  #### Creación de consultas
</div>

La característica central de estos endpoints es que utilizan una única consulta para filtrar las Publicaciones en los recuentos que recibes. Estas consultas se componen de operadores que se aplican a atributos de las Publicaciones y de los usuarios, como palabras clave del mensaje, hashtags y URL. Los operadores se pueden combinar en consultas con lógica booleana y paréntesis para ayudarte a refinar el comportamiento de coincidencia de la consulta.

Puedes usar nuestra guía sobre [cómo crear una consulta](/es/x-api/posts/counts/integrate/build-a-query) para obtener más información.

<div id="pagination">
  #### Paginación
</div>

Para los conteos de Publicaciones recientes, no se devuelve un next&#95;token, lo que significa que, independientemente de la granularidad, obtendrás el volumen de Publicaciones de los últimos 7 días en una única llamada a la API.

Para los conteos de Publicaciones de archivo completo, obtendrás datos de los últimos 30 días. Para datos de más de 30 días, obtendrás un next&#95;token que luego puedes usar para paginar y obtener datos adicionales. 

<div id="building-queries-for-post-counts">
  ### Creación de consultas para los recuentos de Publicaciones
</div>

Los endpoints de recuentos de Publicaciones aceptan una única consulta en una solicitud GET y devuelven un conjunto de recuentos históricos de Publicaciones que coinciden con la consulta. Las consultas se componen de operadores que se utilizan para hacer coincidir una variedad de atributos de Publicaciones.
