<div id="getting-started-with-the-full-archive-post-counts-endpoint">
  ### Introducción al endpoint de conteo de Publicaciones del archivo completo
</div>

Esta guía de inicio rápido te ayudará a hacer tu primera solicitud al endpoint de conteo de Publicaciones del archivo completo usando Postman, una herramienta gráfica que te permite realizar solicitudes HTTP.

Si deseas ver código de ejemplo en diferentes lenguajes de programación, visita nuestro repositorio de GitHub [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code). 

<Note>
  ### Requisitos previos

  Actualmente, el endpoint de conteo de Publicaciones del archivo completo solo está disponible para quienes tienen acceso Pro o Enterprise. Para utilizar este endpoint, debes solicitar acceso Pro o [acceso Enterprise](https://docs.google.com/forms/d/e/1FAIpQLScO3bczKWO2jFHyVZJUSEGfdyfFaqt2MvmOfl_aJp0KxMqtDA/viewform). 

  Además de obtener la aprobación para el acceso, necesitarás un conjunto de [claves y tokens](/es/resources/fundamentals/authentication) para autenticar tu solicitud. Puedes generar estas claves y tokens siguiendo estos pasos:

  * Ve a tu [Proyecto](/es/resources/fundamentals/projects) con acceso Enterprise en el portal para desarrolladores y asegúrate de tener una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) asociada dentro de ese Proyecto.

  * Ve a la página “Keys and tokens” de tu aplicación para generar las credenciales necesarias. Asegúrate de guardar todas las credenciales en una ubicación segura.
</Note>

<div id="steps-to-build-a-full-archive-post-counts-request">
  #### Pasos para crear una solicitud de recuento de Publicaciones de archivo completo
</div>

**Paso uno: Comienza con una herramienta o biblioteca**

Hay varias herramientas, ejemplos de código y bibliotecas que puedes usar para realizar una solicitud a este endpoint, pero aquí usaremos la herramienta Postman para simplificar el proceso.

Para cargar la colección de Postman de X API v2 en tu entorno, haz clic en el siguiente botón:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Agregar X API v2 a Postman
</Button>

Una vez que tengas la colección de X API v2 cargada en Postman, navega a la solicitud Post counts &gt; Full-archive Post counts.

**Paso dos: Autentica tu solicitud**

Para poder realizar correctamente una solicitud a la X API, debes verificar que tienes permiso. Para hacerlo con este endpoint, debes autenticar tu solicitud con los métodos de autenticación [OAuth 2.0 App-Only](/es/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token).

Debes agregar tus claves y tokens, específicamente el [App Access Token](/es/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token) (también conocido como el Bearer Token solo para la aplicación) a Postman. Puedes hacerlo seleccionando el entorno llamado “X API v2” en la esquina superior derecha de Postman y agregando tus claves y tokens a los campos &quot;initial value&quot; y &quot;current value&quot; (haciendo clic en el ícono de ojo junto al menú desplegable de entornos).

Esta variable se incorporará automáticamente en la pestaña de autorización de la solicitud si lo has hecho correctamente.
 

**Paso tres: Crea una consulta**

Cada solicitud de recuento de Publicaciones de archivo completo requiere una única [query](/es/x-api/posts/counts/integrate/build-a-query). Para este ejemplo, vamos a usar una query que coincida con las Publicaciones publicadas por la cuenta @XDevelopers. Para esta query usamos el operador from y lo configuramos en XDevelopers (no distingue entre mayúsculas y minúsculas):

`from:XDevelopers`

En Postman, navega a la pestaña &quot;Params&quot; e ingresa este ID, o una cadena de IDs de Publicaciones separadas por comas, en la columna &quot;Value&quot; del parámetro `ids`.
 

| Key | Value | Description |
| :--- | :--- | :--- |
| `query` | `from:XDevelopers` | Consulta que se enviará al endpoint de recuento de Publicaciones de archivo completo |

Paso cuatro (opcional): Especifica la granularidad y el período de tiempo

Si haces clic en el botón ‘Send’ después del paso tres, obtendrás el recuento de Publicaciones de archivo completo predeterminado: por hora durante los últimos 30 días. Si deseas obtener recuentos de Publicaciones de archivo completo por día, deberás agregar el parámetro granularity con un valor de day. Si deseas recuentos de Publicaciones de hace más de 30 días, deberás especificar los parámetros start&#95;time y end&#95;time con los valores deseados. 

En Postman, navega a la pestaña &quot;Params&quot; y agrega el siguiente par clave:valor a la tabla &quot;Query Params&quot;:

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Description** |
| granularity | day | La granularidad de los resultados de recuento de Publicaciones. Los valores posibles son day, hour o minute |
| start&#95;time | 2021-05-01T00:00:00Z | La marca de tiempo UTC más antigua a partir de la cual se proporcionarán las Publicaciones |
| end&#95;time | 2021-06-01T00:00:00Z | La marca de tiempo UTC más antigua a partir de la cual se proporcionarán las Publicaciones. |

Ahora deberías ver la siguiente URL junto al botón &quot;Send&quot;:

`https://api.x.com/2/tweets/counts/all?query=from%3AXDevelopers&start_time=2021-05-01T00:00:00Z&end_time=2021-06-01T00:00:00Z&granularity=day`

**Paso seis: pagina por tus resultados**

Si el objeto `meta` en tu respuesta también contiene `next_token`, puedes pasar su valor al parámetro de consulta `next_token`.

| Key | Value | Description |
| :--- | :--- | :--- |
| next&#95;token | Agregarás el `next_token` que obtuviste del objeto `meta` de tu solicitud anterior y lo colocarás aquí. | Si tu solicitud más reciente no devuelve el resto de los resultados, recibirás un `next_token` en el objeto `meta`. Tomarás el valor de ese campo y lo usarás como valor del parámetro `next_token` en tu siguiente solicitud, manteniendo constantes todos los demás parámetros de la solicitud. |

Una vez que tengas configurado el valor correcto para `next_token`, haz clic en el botón &quot;Send&quot; y recibirás la siguiente página de resultados.
