<div id="getting-started-with-the-recent-post-counts-endpoint">
  ### Introducción al endpoint de conteo de Publicaciones recientes
</div>

Esta guía de inicio rápido te ayudará a realizar tu primera solicitud al endpoint de conteo de Publicaciones recientes usando Postman, una herramienta gráfica que te permite enviar solicitudes HTTP.

Si quieres ver código de ejemplo en distintos lenguajes de programación, visita nuestro repositorio de GitHub [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code). 

<Note>
  ### Requisitos previos

  Para completar esta guía, necesitarás un conjunto de [keys and tokens](/es/resources/fundamentals/authentication) para autenticar tu solicitud. Puedes generar estas claves y tokens siguiendo estos pasos:

  * [Regístrate para obtener una cuenta de desarrollador](https://developer.x.com/en/apply-for-access) y obtén la aprobación.
  * Crea un [Proyecto](/es/resources/fundamentals/projects) y una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) asociada en el portal para desarrolladores.
  * Navega a la página “Keys and tokens” de tu App para generar las credenciales necesarias. Asegúrate de guardar todas las credenciales en un lugar seguro.
</Note>

<div id="steps-to-build-a-recent-post-counts-request">
  #### Pasos para crear una solicitud de conteos de Publicaciones recientes
</div>

**Paso uno: Comience con una herramienta o biblioteca**

Hay varias herramientas, ejemplos de código y bibliotecas que puede usar para realizar una solicitud a este endpoint, pero aquí vamos a usar la herramienta Postman para simplificar el proceso.

Para cargar la colección de Postman de X API v2 en su entorno, haga clic en el siguiente botón:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Agregar X API v2 a Postman
</Button>

Una vez que tenga la colección de X API v2 cargada en Postman, navegue a la solicitud Post counts &gt; Recent Post counts.

**Paso dos: Autentique su solicitud**

Para realizar correctamente una solicitud a X API, debe verificar que tenga permiso. Para hacerlo con este endpoint, debe autenticar su solicitud con los métodos de autenticación [OAuth 2.0 App-Only](/es/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token).

Debe agregar sus claves y tokens, específicamente el [App Access Token](/es/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token) (también conocido como el Bearer Token de solo aplicación) a Postman. Puede hacer esto seleccionando el entorno llamado &quot;X API v2&quot; en la esquina superior derecha de Postman y agregando sus claves y tokens en los campos &quot;initial value&quot; y &quot;current value&quot; (haciendo clic en el ícono de ojo junto al menú desplegable de entornos).

Esta variable se insertará automáticamente en la pestaña de autorización de la solicitud si lo ha hecho correctamente.
 

**Paso tres: Cree una consulta**

Cada solicitud de conteos de Publicaciones recientes requiere una única [query](/es/x-api/posts/counts/integrate/build-a-query). Para este ejemplo, vamos a usar una query que coincida con las Publicaciones de la cuenta @XDevelopers. Para esta query usamos el operador from: y lo configuramos en XDevelopers (no distingue mayúsculas y minúsculas):

`from:XDevelopers`

En Postman, navegue a la pestaña &quot;Params&quot; e ingrese este id, o una cadena de id de Publicaciones separadas por una coma, en la columna &quot;Value&quot; del parámetro `ids`.

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Description** |
| `query` | from:XDevelopers | Consulta que se enviará al endpoint de conteos de Publicaciones recientes |

**Paso cuatro (opcional): Especifique la granularidad de la solicitud**

Si hace clic en el botón &quot;Send&quot; después del paso tres, obtendrá los conteos de Publicaciones recientes predeterminados: por hora durante los últimos siete días. Si desea obtener los conteos de Publicaciones recientes por día, tendrá que agregar el parámetro granularity con un valor de day.

En Postman, navegue a la pestaña &quot;Params&quot; e ingrese day en la columna &quot;Value&quot; del parámetro granularity.

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Description** |
| granularity | day | La granularidad de los resultados de conteos de Publicaciones. Los valores posibles son day, hour o minute |

Ahora debería ver la siguiente URL junto al botón &quot;Send&quot;:

`https://api.x.com/2/tweets/counts/recent?query=from%3AXDevelopers&granularity=day`
