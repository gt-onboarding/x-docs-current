Los endpoints de recuento de Publicaciones de la v2 permiten a los desarrolladores comprender y obtener el volumen de datos para una consulta determinada. Esto puede ser beneficioso por varias razones, entre ellas:

* Entender el volumen de Publicaciones para una palabra clave con el fin de crear visualizaciones, como líneas de tendencia.

* Entender el período de tiempo en el que se produjo un evento o conversación, para asegurarte de que tu consulta capture los datos relevantes.

* Entender cuántas Publicaciones devolverá una consulta de búsqueda, con el fin de refinar tu consulta antes de usar los endpoints de búsqueda reciente o de búsqueda en todo el archivo.
  **Ten en cuenta:** los recuentos no siempre coincidirán con el resultado que se devolverá desde los endpoints de búsqueda, porque los endpoints de búsqueda se someten a controles de cumplimiento adicionales por los que los endpoints de recuento no pasan.

* Entender el tamaño de la conversación en torno a un tema, sin tener que extraer realmente los datos sin procesar y aplicar las Publicaciones a tu [límite de Publicaciones](/es/x-api/fundamentals/post-cap) mensual. 

Al desarrollar una consulta, estarás limitado a una longitud de consulta determinada y a operadores específicos en función de tu [nivel de acceso](/es/x-api/getting-started/about-x-api). 

* Si usas un Proyecto con acceso Pro, puedes usar todos los operadores disponibles y formular consultas de hasta 1024 caracteres de longitud. 

* Si usas un Proyecto con acceso Enterprise, puedes usar todos los operadores disponibles y formular consultas de hasta 4096 caracteres de longitud. 

También puedes especificar la granularidad (que puede ser día, hora o minuto), así como el período de tiempo para el que necesitas los recuentos de Publicaciones (usando los parámetros `start_time` y `end_time`). La granularidad de tiempo predeterminada que usa este endpoint es de una hora, lo que significa que, si no especificas el parámetro de granularidad, el endpoint te proporcionará los recuentos de Publicaciones por hora para los últimos 7 días.

<Note>
  **Configuración de la cuenta**

  Para acceder a estos endpoints, necesitarás:

  * Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
  * Autenticarte usando las claves y tokens de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) que se encuentra dentro de un [Proyecto](/es/resources/fundamentals/projects). 

  Obtén más información sobre cómo obtener acceso a los endpoints de X API v2 en nuestra [guía de primeros pasos](/es/x-api/getting-started/getting-access).
</Note>

<div id="recent-post-counts">
  ### Conteos de Publicaciones recientes
</div>

El endpoint de conteo de Publicaciones recientes te permite recuperar de forma programática el número de Publicaciones para una consulta en los últimos siete días. Este endpoint está disponible para cualquiera que utilice claves y tokens asociados con una [App](/es/resources/fundamentals/developer-apps) dentro de un [Project](/es/resources/fundamentals/projects) y utiliza [OAuth 2.0 solo para la aplicación](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) para la autenticación.

<div id="full-archive-post-counts">
  ### Conteos de Publicaciones del archivo completo
</div>

El endpoint de conteos de Publicaciones del archivo completo te permite recuperar de forma programática el número de Publicaciones que coinciden con una consulta, a partir de todo el archivo de Publicaciones públicas. Actualmente, este endpoint solo está disponible para quienes hayan sido aprobados para [Academic Research or Enterprise access](/es/x-api/getting-started/about-x-api) y usen [OAuth 2.0 solo para la aplicación](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) para la autenticación.

Un ejemplo: podrías usar el endpoint de conteos de Publicaciones del archivo completo para ver la cantidad de Publicaciones con el hashtag #SOSHurricaneHarvey por día entre agosto y septiembre de 2017.

**Ten en cuenta:** El endpoint de conteos pagina en intervalos de 31 días por respuesta. Por ejemplo, al configurar una granularidad de day, se devolverá la cantidad de resultados por día durante 31 días por página. Al configurar una granularidad de hour, se devolverá la cantidad de resultados por hora durante 744 horas (31 días x 24 horas) por página. Si no especificas la granularidad y el período de tiempo, este endpoint te dará los conteos de Publicaciones por hora para una consulta, durante los últimos 30 días.
