Los endpoints de recuento de Publicaciones de la v2 permiten a los desarrolladores comprender y obtener el volumen de datos para una consulta determinada. Esto puede ser útil por varios motivos, entre ellos:

* Entender el volumen de Publicaciones para una palabra clave con el fin de crear visualizaciones, como líneas de tendencia.

* Entender el período de tiempo en el que ocurrió un evento o conversación, para asegurarte de que tu consulta capture los datos relevantes.

* Entender cuántas Publicaciones devolverá una consulta de búsqueda, con el fin de refinar tu consulta antes de usar los endpoints de búsqueda reciente o de búsqueda en el archivo completo.
  **Ten en cuenta:** Los recuentos no siempre coincidirán con el resultado que devolverán los endpoints de búsqueda porque los endpoints de búsqueda pasan por procesos de cumplimiento adicionales por los que no pasan los endpoints de recuentos.

* Entender el tamaño de la conversación en torno a un tema, sin tener que extraer directamente los datos sin procesar y aplicar esas Publicaciones a tu [límite de Publicaciones](/es/x-api/fundamentals/post-cap) mensual.

Al desarrollar una consulta, estarás limitado a una cierta longitud de consulta y a operadores específicos según tu [nivel de acceso](/es/x-api/getting-started/about-x-api).

* Si estás usando un Proyecto con acceso Pro, puedes usar todos los operadores disponibles y consultas de hasta 1024 caracteres de longitud.

* Si estás usando un Proyecto con acceso Enterprise, puedes usar todos los operadores disponibles y consultas de hasta 4096 caracteres de longitud.

También puedes especificar la granularidad (que puede ser día, hora o minuto), así como el período de tiempo para el que necesitas los recuentos de Publicaciones (usando los parámetros start&#95;time y end&#95;time). La granularidad de tiempo predeterminada que usa este endpoint es la hora, lo que significa que, si no especificas el parámetro de granularidad, el endpoint te dará los recuentos de Publicaciones por hora de los últimos 7 días.

<Note>
  **Configuración de la cuenta**

  Para acceder a estos endpoints, necesitarás:

  * Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
  * Autenticarte usando las claves y tokens de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) que se encuentre dentro de un [Proyecto](/es/resources/fundamentals/projects).

  Obtén más información sobre cómo acceder a los endpoints de X API v2 en nuestra [guía de primeros pasos](/es/x-api/getting-started/getting-access).
</Note>

<div id="recent-post-counts">
  ### Conteos de Publicaciones recientes
</div>

El endpoint de conteo de Publicaciones recientes te permite recuperar de forma programática el número de Publicaciones para una consulta en los últimos siete días. Este endpoint está disponible para cualquier persona que use claves y tokens asociados con una [App](/es/resources/fundamentals/developer-apps) dentro de un [Project](/es/resources/fundamentals/projects) y utiliza [OAuth 2.0 solo para la aplicación](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) para la autenticación.

<div id="full-archive-post-counts">
  ### Conteos de Publicaciones del archivo completo
</div>

El endpoint de conteos de Publicaciones del archivo completo te permite obtener de forma programática el número de Publicaciones para una consulta a partir de todo el archivo de Publicaciones públicas. Actualmente, este endpoint solo está disponible para quienes hayan sido aprobados para el [acceso a Academic Research o Enterprise](/es/x-api/getting-started/about-x-api) y utilicen [OAuth 2.0 solo para la aplicación](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) para la autenticación.

Por ejemplo, podrías usar el endpoint de conteos de Publicaciones del archivo completo para ver el número de Publicaciones con el hashtag #SOSHurricaneHarvey por día entre agosto y septiembre de 2017.

**Ten en cuenta:** El endpoint de conteos pagina en intervalos de 31 días por respuesta. Por ejemplo, si configuras una granularidad de day, devolverá la cantidad de resultados por día para 31 días por página. Si configuras una granularidad de hour, devolverá la cantidad de resultados por hora para 744 horas (31 días x 24 horas) por página. Si no especificas la granularidad ni el período de tiempo, este endpoint te dará conteos de Publicaciones para una consulta por hora durante los últimos 30 días.
