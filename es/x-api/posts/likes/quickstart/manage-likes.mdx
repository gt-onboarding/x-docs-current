<div id="getting-started-with-the-manage-likes-endpoints">
  ### Introducción al uso de los endpoints para administrar Me gusta
</div>

Esta guía de inicio rápido te ayudará a realizar tu primera solicitud a los endpoints para administrar Me gusta usando [Postman](/es/tutorials/postman-getting-started).

Si deseas ver código de ejemplo en diferentes idiomas, visita nuestro repositorio de GitHub de [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code). 

<div id="prerequisites">
  #### Requisitos previos
</div>

Para poder completar esta guía, necesitarás tener un conjunto de [claves y tokens](/es/resources/fundamentals/authentication), que puedes generar siguiendo estos pasos:

1. [Solicita una cuenta de desarrollador](https://developer.x.com/en/apply-for-access) y recibe la aprobación.

2. Crea un [Proyecto](/es/resources/fundamentals/projects) y una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) asociada en el portal para desarrolladores.

3. Accede a la página “Keys and tokens” de tu aplicación y guarda tus API keys, tokens de acceso y Bearer Token en tu gestor de contraseñas.

<div id="steps-to-build-a-manage-likes-request">
  #### Pasos para crear una solicitud de administración de Me gusta
</div>

**Primer paso: Comienza con una herramienta o biblioteca**

Existen varias herramientas, ejemplos de código y bibliotecas que puedes usar para hacer una solicitud a este endpoint, pero aquí vamos a usar la herramienta Postman para simplificar el proceso.

Para cargar la colección de Postman de X API v2 en tu entorno, haz clic en el siguiente botón:

[![](https://cdn.cms-twdigitalassets.com/content/dam/developer-twitter/m1_vnext/carat.svg)<br />Agregar X API v2 a Postman](https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400)

Una vez que tengas la colección de X API v2 cargada en Postman, navega a la carpeta “Likes” y selecciona “Like a Post”.

**Segundo paso: Autentica tu solicitud**

Para realizar correctamente una solicitud a este endpoint, deberás usar [OAuth 1.0a User Context](/es/resources/fundamentals/authentication). Para hacerlo, debes agregar las siguientes claves y tokens a Postman seleccionando el entorno llamado “X API v2” y agregando las siguientes variables a los campos de valor inicial y valor actual:

* consumer&#95;key con tu API Key
* consumer&#95;secret con tu API Key Secret
* access&#95;token con tu Token de acceso
* token&#95;secret con tu Secreto del Token de acceso
   

**Tercer paso: Especifica qué Publicación vas a indicar que te gusta**

Los endpoints de administración de Me gusta requieren dos IDs: uno para el usuario (el usuario que desea indicar que le gusta una Publicación) y otro que representa el ID de la Publicación que el usuario intenta indicar que le gusta o a la que intenta quitarle el Me gusta. 

El ID del usuario debe corresponder al ID del usuario autenticado, lo que significa que debes pasar los [Access Tokens](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) asociados con el ID de usuario al autenticar tu solicitud. En este caso, puedes especificar el ID que pertenece a tu propio usuario. Puedes encontrar tu ID de dos maneras:

1. Usando el endpoint [user lookup by username](/es/x-api/users/user-lookup-by-id), puedes pasar un nombre de usuario y recibir el campo id. 
2. Al revisar tu Token de acceso, verás que la parte numérica es tu ID de usuario.
    

También debes especificar una Publicación a la que quieras indicar que te gusta. Puedes encontrar el ID de la Publicación navegando a x.com, haciendo clic en una Publicación y luego revisando la URL. Por ejemplo, el ID de la Publicación de la siguiente URL es 1228393702244134912.

[https://x.com/TwitterDev/status/1228393702244134912](https://x.com/TwitterDev/status/1228393702244134912)

En Postman, navega a la pestaña &quot;Params&quot; e introduce tu ID en la columna &quot;Value&quot; de la variable de ruta id. Navega a la pestaña “Body” e introduce el ID de la Publicación a la que deseas indicar que te gusta como valor del parámetro tweet&#95;id. Asegúrate de no incluir espacios antes o después de ningún ID.

|              |                                                            |
| :----------- | :--------------------------------------------------------- |
| **Key**      | **Value**                                                  |
| `id`         | (tu ID de usuario)                                         |
| tweet&#95;id | (el ID de la Publicación que quieres indicar que te gusta) |

Si haces clic en el botón &quot;Send&quot;, recibirás un objeto de respuesta que contiene el estado de la relación:

* Si recibes un &quot;liked&quot;: true, entonces el id está indicando correctamente que le gusta el tweet&#95;id.
   

**Cuarto paso: Haz tu solicitud y revisa tu respuesta**

Una vez que tengas todo configurado, haz clic en el botón &quot;Send&quot; y recibirás la siguiente respuesta:

```
{
    "data": {
        "liked": true
    }
}
```

Si deseas quitar el Me gusta de la misma Publicación, puedes usar la solicitud titulada “Unlike a Post”, que también se encuentra en la carpeta “Likes” de la colección de X API v2 en Postman. El id y el tweet&#95;id se deben usar de la misma manera que en el ejemplo anterior. Para quitar el Me gusta de una Publicación, no tendrás que añadir esto en el cuerpo JSON, por lo que deberás asegurarte de añadir los parámetros de consulta necesarios para id y tweet&#95;id.
