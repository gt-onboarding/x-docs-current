<div id="likes-lookup">
  ### Consulta de Me gusta
</div>

Con los endpoints del grupo de consulta de Me gusta, puedes recuperar una lista de cuentas a las que les ha gustado una Publicación o una lista de Publicaciones que le han gustado a una cuenta. Estos endpoints incluyen:

* Publicaciones que le han gustado a un usuario - GET /2/users/:id/liked&#95;tweets
* Usuarios a los que les ha gustado una Publicación - GET /2/tweets/:id/liking&#95;users

Puedes autenticar estos endpoints con [Contexto de usuario de OAuth 1.0a](/es/resources/fundamentals/authentication) o con [OAuth 2.0 Bearer Token](/es/resources/fundamentals/authentication#oauth-2-0). Para los endpoints de Publicaciones con Me gusta, se proporcionarán tokens de paginación para recorrer conjuntos grandes de resultados.

El endpoint de usuarios a los que les ha gustado una Publicación limita el total a 100 cuentas con Me gusta por publicación a lo largo de todo el tiempo. Además, el endpoint de Publicaciones con Me gusta también está sujeto al límite mensual de [Publicaciones](/es/x-api/fundamentals/post-cap) aplicado a nivel de Proyecto.
 

<div id="manage-likes">
  ### Administrar Me gusta
</div>

Los endpoints para administrar Me gusta te permiten marcar con Me gusta o quitar el Me gusta de una Publicación específica en nombre de una cuenta autenticada. Para este grupo de endpoints, hay dos métodos disponibles: POST y DELETE. El método POST te permite marcar con Me gusta una Publicación y el método DELETE te permite quitar el Me gusta de una Publicación.

Dado que estás realizando solicitudes en nombre de un usuario, debes autenticar estos endpoints con [Contexto de usuario de OAuth 1.0a](/es/resources/fundamentals/authentication) y usar los tokens de acceso asociados con el usuario, que se pueden generar mediante el [flujo OAuth de tres patas](/es/resources/fundamentals/authentication#oauth-1-0a-2)/obtaining-user-access-tokens). Puedes indicar Me gusta en una Publicación desde tu cuenta o desde la cuenta de un usuario autenticado. En ambos endpoints, hay un límite de frecuencia por usuario de 50 solicitudes cada 15 minutos por endpoint. 
 

Para acceder a estos endpoints, debes tener una [cuenta de desarrollador](/es/resources/fundamentals/developer-portal) aprobada. Al autenticarte, debes usar las claves y tokens de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) que esté ubicada dentro de un [Proyecto](/es/resources/fundamentals/projects). 

Obtén más información sobre cómo acceder a los endpoints de X API v2 en nuestra página de [primeros pasos](/es/x-api/getting-started/about-x-api).
