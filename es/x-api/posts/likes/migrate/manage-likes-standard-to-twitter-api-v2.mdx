<div id="manage-likes-standard-v11-compared-to-x-api-v2">
  ### Administrar Me gusta: standard v1.1 en comparación con X API v2
</div>

Si has estado trabajando con los endpoints standard v1.1 [POST favorites/create](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-create) y [POST favorites/destroy](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-destroy), el objetivo de esta guía es ayudarte a comprender las similitudes y diferencias entre los endpoints para administrar Me gusta de standard v1.1 y de X API v2.

* **Similitudes**
  * Contexto de usuario de OAuth 1.0a
* **Diferencias**
  * URLs de los endpoints y métodos HTTP
  * Requisitos de App y Proyecto
  * Parámetros de la solicitud

<div id="similarities">
  #### Similitudes
</div>

**Método de autenticación de Contexto de usuario de OAuth 1.0a**

Ambas versiones del endpoint son compatibles con el [Contexto de usuario de OAuth 1.0a](/es/resources/fundamentals/authentication#oauth-1-0a-2). Por lo tanto, si antes usabas uno de los endpoints estándar de v1.1 para gestionar los Me gusta, puedes seguir usando el mismo método de autenticación si migras a la versión de X API v2.

<div id="differences">
  #### Diferencias
</div>

**URLs de endpoints y métodos HTTP**

* Endpoints de Standard v1.1:
  * POST https://api.x.com/1.1/favorites/create.json
    (indicar que te gusta una Publicación)
  * POST https://api.x.com/1.1/favorites/destroy.json
    (quitar el me gusta de una Publicación)
* Endpoint de X API v2:
  * POST https://api.x.com/2/tweets/:id/likes
    (indicar que te gusta una Publicación)
  * DELETE https://api.x.com/2/tweets/:id/likes/:tweet&#95;id
    (quitar el me gusta de una Publicación) 

**Requisitos de la Aplicación y el Proyecto**

Los endpoints de X API v2 requieren que uses credenciales de una [developer App](/es/resources/fundamentals/developer-apps) que esté asociada a un [Project](/es/resources/fundamentals/projects) al autenticar tus solicitudes. Todos los endpoints de X API v1.1 pueden usar credenciales de Apps independientes o de Apps asociadas a un Proyecto.

**Parámetros de la solicitud**

Los siguientes parámetros de solicitud de Standard v1.1 tienen equivalentes en X API v2:

| Standard v1.1 | X API v2 |
| :--- | :--- |
| id  | id  |
| includes&#95;entities | Sin equivalente |

Ten en cuenta que los parámetros de Standard v1.1 se pasan como parámetros de consulta, mientras que los parámetros de X API v2 se pasan como parámetros de cuerpo para el endpoint POST o parámetros de ruta para el endpoint DELETE.

Además, no se requiere el id del usuario que indica que le gusta una Publicación cuando se usan los endpoints de Standard v1.1, ya que a partir de los [Access Tokens](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) enviados con [OAuth 1.0a User Context](/es/resources/fundamentals/authentication) se infiere qué usuario está iniciando la acción de indicar o quitar el me gusta.
