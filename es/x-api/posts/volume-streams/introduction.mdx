* Flujo muestreado al 1 %.
* Flujo muestreado al 10 %. Comúnmente denominado «Decahose».

Se conocen como «flujos de volumen» porque ambos entregan grandes volúmenes de datos. Incluso el flujo del 1 % puede emitir muchas docenas de Publicaciones cada segundo. Con estos flujos, puedes identificar y seguir tendencias, monitorear el sentimiento general, monitorear eventos globales y mucho más. 

Estos endpoints de streaming entregan [objetos Post](/es/x-api/fundamentals/data-dictionary#tweet) a través de una conexión HTTP GET persistente y usan autenticación [OAuth 2.0 solo para la aplicación](/es/resources/fundamentals/authentication#oauth-2-0). Con acceso Essential, puedes tener solo una conexión a la vez. Con todos los niveles de acceso, se pueden realizar solicitudes de conexión hasta 50 veces por ventana de 15 minutos.

Estos endpoints de flujo de volumen admiten Publicaciones editadas. Estos endpoints entregarán Publicaciones editadas, junto con su historial de ediciones, incluido un array de ids de Publicaciones. Para las Publicaciones sin historial de ediciones, este array contendrá un solo id. Para las Publicaciones que se han editado, este array contiene varios ids, ordenados de forma ascendente según el orden de las ediciones, con la versión más reciente en la última posición del array. Para obtener más información sobre cómo funcionan las ediciones de Publicaciones, consulta la página [Conceptos básicos sobre ediciones de Publicaciones](/es/x-api/fundamentals/edit-posts). 

*Para usar estas API, primero debes configurar una cuenta con nuestro equipo de ventas para clientes empresariales.*

<Note>
  **Configuración de la cuenta**

  Para acceder a estos endpoints, necesitarás:

  * Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
  * Autenticarte usando las claves y tokens de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) que esté ubicada dentro de un [Proyecto](/es/resources/fundamentals/projects). 

  Obtén más información sobre cómo acceder a los endpoints de X API v2 en nuestra [guía de primeros pasos](/es/x-api/getting-started/getting-access).
</Note>
