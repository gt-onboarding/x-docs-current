* Flujo muestreado al 1 %.
* Flujo muestreado al 10 %. Comúnmente conocido como &quot;Decahose&quot;.

A estos se les llama &quot;flujos de volumen&quot; porque ambos entregan grandes volúmenes de datos. Incluso el flujo del 1 % puede emitir muchas docenas de Publicaciones cada segundo. Con estos flujos, puedes identificar y seguir tendencias, monitorear el sentimiento general, supervisar eventos globales y mucho más. 

Estos endpoints de streaming entregan [objetos Post](/es/x-api/fundamentals/data-dictionary#tweet) a través de una conexión HTTP GET persistente y usan autenticación [OAuth 2.0 App-Only](/es/resources/fundamentals/authentication#oauth-2-0). Con acceso Essential, puedes tener una conexión a la vez. Con todos los niveles de acceso, se pueden realizar solicitudes de conexión hasta 50 veces por ventana de 15 minutos.

Estos endpoints de flujos de volumen admiten Publicaciones editadas. Estos endpoints entregarán Publicaciones editadas, junto con su historial de ediciones, incluida una matriz de id de Publicación. Para las Publicaciones sin historial de edición, esta matriz contendrá un único id. Para las Publicaciones que han sido editadas, esta matriz contiene varios id, organizados en orden ascendente que refleja el orden de las ediciones, con la versión más reciente en la última posición de la matriz. Para obtener más información sobre cómo funcionan las ediciones de Publicaciones, consulta la página de [fundamentos de ediciones de Publicaciones](/es/x-api/fundamentals/edit-posts). 

*Para usar estas API, primero debes configurar una cuenta con nuestro equipo de ventas de Enterprise.*

<Note>
  **Configuración de la cuenta**

  Para acceder a estos endpoints, necesitarás:

  * Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
  * Autenticarte usando las claves y tokens de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) que se encuentre dentro de un [Proyecto](/es/resources/fundamentals/projects). 

  Obtén más información sobre cómo conseguir acceso a los endpoints de X API v2 en nuestra [guía de primeros pasos](/es/x-api/getting-started/getting-access).
</Note>
