<div id="authentication">
  ### Autenticación
</div>

Todos los endpoints de X API v2 requieren solicitudes autenticadas. Puedes autenticarte con:

* [Contexto de usuario de OAuth 1.0a](/es/resources/fundamentals/authentication) usando claves de API, Tokens de acceso y parámetros adicionales para [crear un encabezado de autorización](/es/resources/fundamentals/authentication#authorizing-a-request).
* [OAuth 2.0 solo para la aplicación](/es/resources/fundamentals/authentication#oauth-2-0) pasando un [Token de acceso de la App](/es/resources/fundamentals/authentication) en tu solicitud.
* [OAuth 2.0 Authorization Code with PKCE](/es/resources/fundamentals/authentication) para un mayor control sobre el alcance de la aplicación y la autorización en varios dispositivos.

<Note>
  OAuth 1.0a puede ser difícil de implementar. Si no estás familiarizado con este método, considera usar una [biblioteca](/es/resources/tools-and-libraries) u OAuth 2.0 para tus solicitudes. Para métricas privadas o Publicaciones, utiliza Contexto de usuario de OAuth 1.0a u OAuth 2.0 Authorization Code with PKCE.
</Note>

<Tip>
  **Ten en cuenta**

  > Si solicitas los siguientes campos, se requiere Contexto de usuario de OAuth 1.0a u OAuth 2.0 Authorization Code:
  >
  > * `tweet.fields.non_public_metrics`
  > * `tweet.fields.promoted_metrics`
  > * `tweet.fields.organic_metrics`
  > * `media.fields.non_public_metrics`
  > * `media.fields.promoted_metrics`
  > * `media.fields.organic_metrics`
</Tip>

<div id="developer-portal-projects-and-apps">
  ### Portal para desarrolladores, Proyectos y Aplicaciones
</div>

Para obtener credenciales de X API v2, necesitas:

1. Una [cuenta de desarrollador](/es/resources/fundamentals/developer-portal) aprobada.
2. Un [Proyecto](/es/resources/fundamentals/projects) dentro de la cuenta de desarrollador.
3. Una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) dentro de ese Proyecto, donde se pueden encontrar las claves y tokens.

<div id="rate-limits">
  ### Límites de frecuencia
</div>

Las solicitudes a la X API están sujetas a [límites de frecuencia](/es/resources/fundamentals/rate-limits) para gestionar el volumen. Los límites se aplican tanto a nivel de aplicación como de usuario:

* **A nivel de aplicación**: Limita la cantidad de solicitudes que cualquier aplicación puede realizar por periodo.
* **A nivel de usuario**: Limita la frecuencia con la que un usuario autenticado puede realizar consultas de Publicaciones entre aplicaciones de desarrollador.

<div id="fields-and-expansions">
  ### Campos y expansiones
</div>

La X API v2 permite seleccionar campos de datos específicos mediante los parámetros `fields` y `expansions`:

* **Expansions**: Permiten recuperar objetos relacionados adicionales. Las expansiones compatibles incluyen:
  * `edit_history_tweet_ids`
  * `attachments.poll_ids`
  * `attachments.media_keys`
  * `author_id`
  * `entities.mentions.username`
  * `geo.place_id`
  * `in_reply_to_user_id`
  * `referenced_tweets.id`
  * `referenced_tweets.id.author_id`

* **Fields**: Especifican los campos de datos dentro de los objetos para incluir datos adicionales en la respuesta. El objeto Post devuelve de forma predeterminada `id`, `text` y `edit_history_tweet_ids`. Otras opciones, como `tweet.created_at` y `tweet.entities`, deben solicitarse explícitamente.

Para obtener más información, consulta la guía sobre campos y expansiones en el [diccionario de datos de X API v2](/es/x-api/fundamentals/data-dictionary).

<div id="post-edits">
  ### Ediciones de Publicaciones
</div>

Las Publicaciones que cumplan los requisitos se pueden editar hasta cinco veces durante los 30 minutos posteriores a su publicación. El endpoint de consulta de Publicaciones siempre proporciona la versión más reciente de la Publicación. Para casos de uso casi en tiempo real, ten en cuenta esta ventana de tiempo. Para más detalles, consulta [Fundamentos de edición de Publicaciones](/es/x-api/fundamentals/edit-posts).

<div id="edge-cases">
  ### Casos límite
</div>

* **Promoted metrics**: Solicitar métricas promocionadas para Publicaciones no promocionadas devuelve una respuesta vacía.
* **Truncated text**: El texto de la Publicación se trunca en los Retweets. Para obtener el texto completo, expande la Publicación referenciada.
