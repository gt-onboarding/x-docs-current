<div id="getting-started-with-the-posts-lookup-endpoints">
  ## Primeros pasos con los endpoints de consulta de Publicaciones
</div>

Esta guía de inicio rápido te ayudará a realizar tu primera solicitud a los endpoints de consulta de Publicaciones, especificando un conjunto de campos, usando Postman.

Si deseas ver código de ejemplo en diferentes lenguajes de programación, visita nuestro repositorio de GitHub [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code).

<Note>
  ### Requisitos previos

  Para completar esta guía, necesitarás contar con un conjunto de [claves y tokens](/es/resources/fundamentals/authentication) para autenticar tu solicitud. Puedes generar estas claves y tokens siguiendo estos pasos:

  * [Regístrate para obtener una cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) y recibe la aprobación.
  * Crea un [Proyecto](/es/resources/fundamentals/projects) y una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) asociada en el portal para desarrolladores.
  * Ve a la página “Keys and tokens” de tu App para generar las credenciales necesarias. Asegúrate de guardar todas las credenciales en un lugar seguro.
</Note>

<div id="steps-to-build-a-get-tweets-request">
  ### Pasos para construir una solicitud GET /tweets
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### Paso uno: comienza con una herramienta o biblioteca
</div>

Hay varias herramientas, ejemplos de código y bibliotecas que puedes usar para realizar una solicitud a este endpoint, pero aquí vamos a usar la herramienta Postman para simplificar el proceso.

Para cargar la colección de Postman de X API v2 en tu entorno, haz clic en el botón de abajo:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Agregar X API v2 a Postman
</Button>

Una vez que tengas la colección de X API v2 cargada en Postman, navega a la solicitud &quot;Post Lookup &gt; Multiple Posts&quot;.

<div id="step-two-authenticate-your-request">
  #### Paso dos: Autenticar tu solicitud
</div>

Para realizar correctamente una solicitud a la X API, debes asegurarte de que tienes permiso. Puedes autenticar tu solicitud con:

* [OAuth 2.0 solo para la aplicación](resources/fundamentals/authentication)
* [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication)
* [Contexto de usuario de OAuth 1.0a](resources/fundamentals/authentication)

Para simplificar, vamos a utilizar OAuth 2.0 solo para la aplicación con esta solicitud, pero si deseas solicitar [métricas](/es/x-api/fundamentals/metrics) privadas o Publicaciones, tendrás que usar uno de los otros métodos de autenticación.

Para utilizar OAuth 2.0 solo para la aplicación, debes agregar tus claves y tokens (y específicamente el Token de acceso de la aplicación, también conocido como el Bearer Token solo para la aplicación) a Postman, seleccionando el entorno llamado “X API v2” (en la esquina superior derecha de Postman) y agregando tus claves y tokens a los campos &quot;initial value&quot; y &quot;current value&quot; (haciendo clic en el ícono de ojo junto al menú desplegable de entornos).

Si has hecho esto correctamente, estas variables se incorporarán automáticamente en la pestaña de autorización de la solicitud.

<div id="step-three-identify-and-specify-which-posts-you-would-like-to-retrieve">
  #### Paso tres: identifica y especifica qué Publicaciones deseas recuperar
</div>

Debes especificar una Publicación o un conjunto de Publicaciones que deseas recibir en la solicitud. Puedes encontrar el ID de la Publicación navegando a X.com, haciendo clic en una Publicación y luego buscando en la URL. Por ejemplo, el ID de Publicación de la siguiente URL es `1228393702244134912`.

`https://x.com/TwitterDev/status/1228393702244134912`

En Postman, navega a la pestaña &quot;Params&quot; e ingresa este ID, o una lista de IDs de Publicaciones separados por comas, en la columna &quot;Value&quot; del parámetro ids.

| Key  | Value |
| :---- | :----- |
| `ids` | `1228393702244134912,1227640996038684673,1199786642791452673` |

***

<div id="step-four-identify-and-specify-which-fields-you-would-like-to-retrieve">
  #### Paso cuatro: Identifica y especifica qué campos deseas recuperar
</div>

Si haces clic en el botón &quot;Send&quot; después del paso tres, recibirás los [campos predeterminados del objeto Post](/es/x-api/fundamentals/data-dictionary) en tu respuesta: `id`, `text` y `edit_history_tweet_ids`.

Para este ejercicio, solicitaremos tres conjuntos adicionales y diferentes de campos de distintos objetos:

* El campo adicional `tweet.created_at` en los objetos de usuario principales.
* Los campos predeterminados del objeto de usuario de los autores asociados para las Publicaciones devueltas: `id`, `name` y `username`.
* El campo adicional `user.created_at` en los objetos de usuario asociados.

En Postman, navega a la pestaña &quot;Params&quot; y agrega el siguiente par clave-valor (`key:value`) a la tabla &quot;Query Params&quot;:

| Key            | Value        | Returned fields                                                       |
| :------------- | :----------- | :-------------------------------------------------------------------- |
| `tweet.fields` | `created_at` | `tweets.created_at`                                                   |
| `expansions`   | `author_id`  | `includes.users.id`, `includes.users.name`, `includes.users.username` |
| `user.fields`  | `created_at` | `includes.users.created_at`                                           |

Deberías ver esta URL junto al botón &quot;Send&quot;:

```
https://api.x.com/2/tweets?ids=1228393702244134912,1227640996038684673,1199786642791452673&tweet.fields=created_at&expansions=author_id&user.fields=created_at
```
