<div id="manage-replies-in-realtime">
  ### Gestionar respuestas en tiempo real
</div>

Con el endpoint `hide replies`, puedes crear un flujo de trabajo que ayude a tus usuarios a ocultar respuestas que tienen una probabilidad muy alta de ser irrelevantes.

Las aplicaciones útiles suelen combinar tecnologías para poder aportar valor a sus usuarios. Esta página muestra cómo ocultar respuestas utilizando la [Perspective API](https://www.perspectiveapi.com/). Esta API es una inteligencia artificial entrenada por [Jigsaw](https://jigsaw.google.com/), una unidad de Google, para detectar comentarios tóxicos. La lógica de la aplicación funcionará de la siguiente manera:

1.  Pide permiso al usuario para leer sus Publicaciones y ocultar o mostrar sus respuestas.
2.  Utiliza la [Account Activity API](https://developer.x.com/en/docs/x-api/v1/accounts-and-users/subscribe-account-activity/overview) para detectar respuestas entrantes.
3.  Pide a la Perspective API que proporcione una "puntuación" (un número entre 0 y 1) que indique el nivel de confianza de su algoritmo en que un comentario es similar a comentarios tóxicos que ha visto en el pasado. (Perspective no almacena el texto enviado al servicio).
4.  Llama a `hide replies` si la puntuación del algoritmo es muy alta.

<div id="strive-for-transparency">
  #### Procura mantener la transparencia
</div>

Debido a que Perspective no se ha entrenado con Publicaciones reales, ciertos matices del lenguaje pueden hacer que esta aplicación oculte una respuesta que un usuario quiera mantener visible. Independientemente de la tecnología o del enfoque que uses al diseñar tu aplicación, haz siempre todo lo posible por garantizar que tus usuarios entiendan qué ha ocultado tu aplicación y puedan hacer cambios.

* La mejor opción es confiar siempre en el usuario y darle control total sobre sus decisiones. Esto significa que tu experiencia de usuario debe incluir controles para deshacer cualquier acción que tu aplicación haya realizado en nombre del usuario.
* Cuando uses inteligencia artificial, tu aplicación debe emplear un umbral de confianza muy alto para detectar y ocultar Publicaciones.
* No todas las personas usan las mismas palabras y tu aplicación debe estar diseñada para evitar cualquier sesgo. Ten en cuenta las palabras reapropiadas y la jerga que pueden generar falsos positivos.
* Si estás entrenando una inteligencia artificial, considera adoptar un modelo que refleje de cerca el lenguaje que se usa con frecuencia en X.
