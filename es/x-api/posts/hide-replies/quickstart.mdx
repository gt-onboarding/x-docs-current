<div id="getting-started-with-the-hide-replies-endpoint">
  ## Introducción al endpoint para ocultar respuestas
</div>

Esta guía de inicio rápido te ayudará a realizar tu primera solicitud al endpoint para ocultar respuestas usando [Postman](/es/tutorials/postman-getting-started).

Si quieres ver algunos fragmentos de código en distintos lenguajes de programación, visita la [página de referencia de la API de ocultar respuestas](/es/x-api/posts/hide-replies#put-2-tweets-id-hidden). 

<Note>
  ### Requisitos previos

  Para completar esta guía, necesitarás un conjunto de [claves y tokens](/es/resources/fundamentals/authentication) para autenticar tu solicitud. Puedes generar estas claves y tokens siguiendo estos pasos:

  * [Regístrate para obtener una cuenta de desarrollador](https://developer.x.com/en/apply-for-access) y recibir la aprobación.
  * Crea un [Proyecto](/es/resources/fundamentals/projects) y una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) asociada en el portal para desarrolladores.
  * Ve a la página «Claves y tokens» de tu aplicación para generar las credenciales necesarias. Asegúrate de guardar todas las credenciales en un lugar seguro.
</Note>

<div id="steps-to-build-a-put-tweetsidhidden-request">
  ### Pasos para construir una solicitud PUT a /tweets/:id/hidden
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### Paso uno: Comienza con una herramienta o biblioteca
</div>

Hay varias herramientas, ejemplos de código y bibliotecas que puedes usar para realizar una solicitud a este endpoint, pero en este caso usaremos la herramienta Postman para simplificar el proceso.

Para cargar la colección de X API v2 para Postman en tu entorno, haz clic en el siguiente botón:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Agregar X API v2 a Postman
</Button>

Una vez que tengas la colección de X API v2 cargada en Postman, navega al endpoint para ocultar respuestas.

<div id="step-two-authenticate-your-request">
  #### Paso dos: Autentica tu solicitud
</div>

Para realizar correctamente una solicitud a la X API, debes verificar que tienes permiso. Para hacerlo con este endpoint, debes autenticar tu solicitud utilizando ya sea [Contexto de usuario de OAuth 1.0a](/es/resources/fundamentals/authentication) u [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

En este ejemplo, vamos a usar el Contexto de usuario de OAuth 1.0a.

Debes agregar tus claves y tokens – específicamente tu API Key, API Secret Key, OAuth 1.0a user Access Token y OAuth 1.0a user Access Token Secret – a Postman. Puedes hacer esto seleccionando el entorno llamado &quot;X API v2&quot; en la esquina superior derecha de Postman y agregando tus claves y tokens a los campos &quot;initial value&quot; y &quot;current value&quot; (haciendo clic en el ícono de ojo junto al menú desplegable de entornos).

Si lo has hecho correctamente, estas variables se rellenarán automáticamente en la pestaña de autorización de la solicitud.
 

<div id="step-three-find-a-post-id-to-hide">
  #### Paso tres: Encontrar un ID de Publicación para ocultar
</div>

El endpoint hide replies puede ocultar o dejar de ocultar respuestas en nombre de un usuario autorizado. Como en este ejemplo estamos usando los tokens de acceso asociados a tu perfil de usuario, podrás ocultar respuestas de usuarios que participen en una conversación iniciada por ti. De manera similar, si estuvieras usando tokens de acceso que pertenecen a otro usuario que autorizó tu aplicación, podrías moderar las respuestas a cualquier conversación iniciada por esa cuenta.

Pídele a un amigo que responda a una Publicación (indícale que estás probando hide replies) o responde a cualquiera de tus Publicaciones desde una cuenta de prueba. Haz clic en esa respuesta y luego copia la parte numérica de su URL. Ese será el ID de la Publicación que ocultaremos.

En este caso, usaremos la siguiente Publicación, que tiene el ID `1232720193182412800`:

`https://x.com/TwitterDev/status/1232720193182412800`

<div id="step-four-hide-the-post">
  #### Paso cuatro: Ocultar la Publicación
</div>

En Postman, abre la carpeta Hide replies y selecciona Hide a reply. En la pestaña Params, pega el ID de la Publicación junto al campo id (no será necesario reemplazar :id en la URL). Haz clic en «Send» y verás una respuesta satisfactoria.

`{"hidden":true}`

<div id="step-five-unhide-the-post">
  #### Paso cinco: mostrar la Publicación
</div>

Las Publicaciones ocultas se mueven a una pestaña independiente en la aplicación X. Para dejar de ocultar una Publicación en Postman, abre la carpeta Hide replies y selecciona Unhide a reply. En la pestaña Params, pega el mismo ID de la Publicación usado en el paso anterior en el campo id. Haz clic en &quot;Send&quot; y verás una respuesta satisfactoria.

`{"hidden":false}`

El campo hidden representa el estado de oculto de la Publicación. Un estado de oculto true significa que la Publicación está oculta. De forma similar, false significa que la Publicación no está oculta.
