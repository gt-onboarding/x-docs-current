<div id="getting-started-with-the-hide-replies-endpoint">
  ## Introducción al endpoint para ocultar respuestas
</div>

Esta guía de inicio rápido te ayudará a realizar tu primera solicitud al endpoint para ocultar respuestas usando [Postman](/es/tutorials/postman-getting-started).

Si deseas ver algunos fragmentos de código en distintos lenguajes de programación, visita la [página de referencia de la API para ocultar respuestas](/es/x-api/posts/hide-replies#put-2-tweets-id-hidden). 

<Note>
  ### Requisitos previos

  Para completar esta guía, necesitarás un conjunto de [keys and tokens](/es/resources/fundamentals/authentication) para autenticar tu solicitud. Puedes generar estas claves y tokens siguiendo estos pasos:

  * [Regístrate para obtener una cuenta de desarrollador](https://developer.x.com/en/apply-for-access) y obtén la aprobación.
  * Crea un [Proyecto](/es/resources/fundamentals/projects) y una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) asociada en el portal para desarrolladores.
  * Navega a la página “Keys and tokens” de tu App para generar las credenciales requeridas. Asegúrate de guardar todas las credenciales en una ubicación segura.
</Note>

<div id="steps-to-build-a-put-tweetsidhidden-request">
  ### Pasos para crear una solicitud PUT a /tweets/:id/hidden
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### Paso uno: Comienza con una herramienta o biblioteca
</div>

Hay varias herramientas, ejemplos de código y bibliotecas diferentes que puedes usar para hacer una solicitud a este endpoint, pero aquí vamos a usar la herramienta Postman para simplificar el proceso.

Para cargar la colección de X API v2 de Postman en tu entorno, haz clic en el siguiente botón:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Agregar X API v2 a Postman
</Button>

Una vez que tengas la colección de X API v2 cargada en Postman, navega al endpoint de ocultar respuestas.

<div id="step-two-authenticate-your-request">
  #### Paso dos: Autenticar tu solicitud
</div>

Para hacer correctamente una solicitud a X API, debes comprobar que tienes permiso. Para hacerlo con este endpoint, debes autenticar tu solicitud usando [OAuth 1.0a User Context](/es/resources/fundamentals/authentication) o [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

En este ejemplo, vamos a usar OAuth 1.0a User Context.

Debes agregar tus claves y tokens —específicamente tu API Key, API Secret Key, Token de acceso de usuario de OAuth 1.0a y Token de acceso secreto de usuario de OAuth 1.0a— a Postman. Puedes hacer esto seleccionando el entorno llamado “X API v2” en la esquina superior derecha de Postman y agregando tus claves y tokens a los campos &quot;initial value&quot; y &quot;current value&quot; (haciendo clic en el ícono de ojo junto al menú desplegable del entorno).

Estas variables se rellenarán automáticamente en la pestaña de autorización de la solicitud si lo configuraste correctamente.
 

<div id="step-three-find-a-post-id-to-hide">
  #### Paso tres: Encontrar el ID de la Publicación que vamos a ocultar
</div>

El endpoint de ocultar respuestas puede ocultar o mostrar respuestas en nombre de un usuario autorizado. Como en este ejemplo estamos usando los Tokens de acceso relacionados con tu perfil de usuario, podrás ocultar respuestas de usuarios que participen en una conversación iniciada por ti. De manera similar, si estuvieras usando Tokens de acceso que pertenecen a otro usuario que autorizó tu aplicación, podrías moderar las respuestas de cualquier conversación iniciada por esa cuenta.

Pídele a un amigo que responda a una Publicación (indícale que estás probando la función de ocultar respuestas) o responde a cualquiera de tus Publicaciones desde una cuenta de prueba. Haz clic en esa respuesta y luego copia la parte numérica de su URL. Ese será el ID de la Publicación que ocultaremos.

En este caso, usaremos la siguiente Publicación, que tiene el ID `1232720193182412800`:

`https://x.com/TwitterDev/status/1232720193182412800`

<div id="step-four-hide-the-post">
  #### Paso cuatro: Ocultar la Publicación
</div>

En Postman, abre la carpeta &quot;Hide replies&quot; y selecciona &quot;Hide a reply&quot;. En la pestaña Params, pega el ID de la Publicación junto al campo id (no será necesario reemplazar :id en la URL). Haz clic en &quot;Send&quot; y verás una respuesta exitosa.

`{"hidden":true}`

<div id="step-five-unhide-the-post">
  #### Paso cinco: Mostrar la Publicación
</div>

Las Publicaciones ocultas se trasladan a una pestaña separada en la aplicación de X. Para mostrar una Publicación en Postman, abre la carpeta Hide replies y selecciona Unhide a reply. En la pestaña Params, pega el mismo ID de la Publicación usado en el paso anterior en el campo id. Haz clic en &quot;Send&quot; y verás una respuesta satisfactoria.

`{"hidden":false}`

El campo hidden representa el estado de ocultamiento de la Publicación. Un estado de ocultamiento igual a true significa que la Publicación está oculta. De manera similar, false significa que la Publicación no está oculta.
