<div id="comparing-x-apis-retweets-endpoints">
  ## Comparación de los endpoints de Retweets de X API
</div>

**Búsqueda de Retweets**

El endpoint de búsqueda de Retweets de v2 reemplazará los endpoints estándar [v1.1 GET statuses/retweets/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-retweets-id) y [v1.1 GET statuses/retweets/:ids](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-retweeters-ids).

Las siguientes tablas comparan los endpoints estándar v1.1 y X API v2 de Retweets:

| Descripción | Estándar v1.1 | X API v2 |
| :--- | :--- | :--- |
| Métodos HTTP admitidos | GET | GET |
| Dominio de host | https://api.x.com | https://api.x.com |
| Ruta del endpoint | /1.1/retweeters/id.json<br /><br />`/1.1/retweets/ids.json` | /2/users/:id/retweeted&#95;by |
| [Autenticación](/es/resources/fundamentals/authentication) | Contexto de usuario de OAuth 1.0a | Bearer Token de OAuth 2.0<br /><br />Contexto de usuario de OAuth 1.0a |
| [Límites de frecuencia](/es/resources/fundamentals/rate-limits) de solicitud predeterminados | 75 solicitudes por 15 min | 75 solicitudes por 15 min (por Aplicación)<br /><br />75 solicitudes por 15 min (por usuario) |
| Formato de datos | Formato estándar v1.1 | [Formato X API v2](/es/x-api/fundamentals/data-dictionary) (determinado por los parámetros de solicitud fields y expansions, no es retrocompatible con los formatos de v1.1)<br /><br />Para obtener más información sobre cómo migrar del formato estándar v1.1 al formato X API v2, consulta nuestra [guía de migración de formatos de datos](/es/x-api/migrate/data-format-migration). |
| Requiere el uso de credenciales de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) que esté asociada a un [Proyecto](/es/resources/fundamentals/projects) |     | ✔️  |

<div id="manage-retweets">
  ### Administrar Retweets
</div>

Las siguientes tablas comparan el endpoint estándar de v1.1 y de X API v2 para deshacer un Retweet:

**Hacer Retweet de una Publicación**

| Description | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| HTTP methods supported | POST | POST |
| Host domain | https://api.x.com | https://api.x.com |
| Endpoint path | /1.1/statuses/retweet/:id.json | /2/users/:id/retweets |
| [Authentication](/es/resources/fundamentals/authentication) | Contexto de usuario de OAuth 1.0a | Contexto de usuario de OAuth 1.0a |
| Default request [rate limits](/es/resources/fundamentals/rate-limits) | None<br /><br />300 requests per 3-hour window (per user, per app). This is shared with the POST Tweet endpoint | 50 requests per 15 min (per user)<br /><br />300 requests per 3-hour window (per user, per app). This is shared with the POST Tweet endpoint for manage Posts. |
| Requires the use of credentials from a [developer App](/es/resources/fundamentals/developer-apps) that is associated with a [Project](/es/resources/fundamentals/projects) |     | ✔️  |

<div id="undo-a-retweet">
  #### Deshacer un Retweet

La siguiente tabla compara el endpoint para deshacer un Retweet en Standard v1.1 y en X API v2:

| Descripción | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| Métodos HTTP admitidos | POST | DELETE |
| Dominio del host | https://api.x.com | https://api.x.com |
| Ruta del endpoint | /1.1/statuses/unretweet/:id.json | /2/users/:id/retweets/:source&#95;tweet&#95;id |
