<div id="comparing-x-apis-retweets-endpoints">
  ## Comparación de los endpoints de Retweets de la X API
</div>

**Búsqueda de Retweets**

El endpoint de búsqueda de Retweets de v2 reemplazará los endpoints estándar [v1.1 GET statuses/retweets/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-retweets-id) y [v1.1 GET statuses/retweets/:ids](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-retweeters-ids).

La siguiente tabla compara los endpoints estándar de Retweets de v1.1 y de X API v2:

| Descripción | v1.1 estándar | X API v2 |
| :--- | :--- | :--- |
| Métodos HTTP compatibles | GET | GET |
| Dominio de host | https://api.x.com | https://api.x.com |
| Ruta del endpoint | /1.1/retweeters/id.json<br /><br />`/1.1/retweets/ids.json` | /2/users/:id/retweeted&#95;by |
| [Autenticación](/es/resources/fundamentals/authentication) | Contexto de usuario de OAuth 1.0a | OAuth 2.0 Bearer Token<br /><br />Contexto de usuario de OAuth 1.0a |
| [Límites de frecuencia](/es/resources/fundamentals/rate-limits) de solicitudes predeterminados | 75 solicitudes por 15 min | 75 solicitudes por 15 min (por aplicación)<br /><br />75 solicitudes por 15 min (por usuario) |
| Formato de datos | Formato estándar v1.1 | [Formato de X API v2](/es/x-api/fundamentals/data-dictionary) (determinado por los parámetros de solicitud fields y expansions, no es retrocompatible con los formatos de v1.1)<br /><br />Para obtener más información sobre cómo migrar del formato estándar v1.1 al formato de X API v2, visita nuestra [guía de migración de formatos de datos](/es/x-api/migrate/data-format-migration). |
| Requiere el uso de credenciales de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) que esté asociada con un [Proyecto](/es/resources/fundamentals/projects) |     | ✔️  |

<div id="manage-retweets">
  ### Administrar Retweets
</div>

Las siguientes tablas comparan el endpoint estándar de v1.1 y el endpoint de X API v2 para deshacer un Retweet:

**Hacer Retweet de una Publicación**

| Descripción | v1.1 estándar | X API v2 |
| :--- | :--- | :--- |
| Métodos HTTP compatibles | POST | POST |
| Dominio del host | https://api.x.com | https://api.x.com |
| Ruta del endpoint | /1.1/statuses/retweet/:id.json | /2/users/:id/retweets |
| [Autenticación](/es/resources/fundamentals/authentication) | Contexto de usuario de OAuth 1.0a | Contexto de usuario de OAuth 1.0a |
| [Límites de frecuencia](/es/resources/fundamentals/rate-limits) de solicitud predeterminados | Ninguno<br /><br />300 solicitudes por ventana de 3 horas (por usuario, por aplicación). Se comparte con el endpoint POST Tweet | 50 solicitudes cada 15 min (por usuario)<br /><br />300 solicitudes por ventana de 3 horas (por usuario, por aplicación). Se comparte con el endpoint POST Tweet para administrar publicaciones. |
| Requiere el uso de credenciales de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) que esté asociada con un [Proyecto](/es/resources/fundamentals/projects) |     | ✔️  |
