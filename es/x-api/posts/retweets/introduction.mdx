<Note>
  **Configuración de la cuenta**

  Para acceder a estos endpoints, necesitarás:

  * Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
  * Autenticarte usando las claves y tokens de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) que se encuentre dentro de un [Proyecto](/es/resources/fundamentals/projects). 

  Obtén más información sobre cómo acceder a los endpoints de la X API v2 en nuestra [guía de primeros pasos](/es/x-api/getting-started/getting-access).

<div id="retweets-lookup">
  ### Consulta de Retweets
</div>

Con el endpoint de consulta de Retweets, puedes obtener una lista de cuentas que han hecho Retweet de una Publicación. Para este endpoint, se proporcionarán tokens de paginación para recorrer grandes conjuntos de resultados en lotes de hasta 100 usuarios. 

Puedes autenticar estos endpoints con [Contexto de usuario de OAuth 1.0a](/es/resources/fundamentals/authentication), [OAuth 2.0 solo para la aplicación](https://developer.x.com(/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) o [OAuth 2.0 Authorization Code with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3). 

<div id="retweets-of-me">
  ### Reposts de mí
</div>

La consulta Reposts de mí te permite ver cuáles de tus propias Publicaciones han sido compartidas por otras personas.

Esto ofrece a los usuarios información sobre qué Publicaciones suyas se están compartiendo y con cuáles está conectando su audiencia, lo que puede ayudar a propiciar interacciones más dirigidas y ajustes en la estrategia de contenido.

Puedes autenticar estos endpoints con cualquiera de los dos métodos: [OAuth 1.0a User Context](/es/resources/fundamentals/authentication) o [OAuth 2.0 Authorization Code with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

<div id="manage-retweets">
  #### Administrar Retweets
</div>

Los endpoints para administrar Retweets te permiten hacer Retweet o deshacer el Retweet de una Publicación específica en nombre de una cuenta autenticada. Para este grupo de endpoints, hay dos métodos disponibles: POST y DELETE. El método POST te permite hacer Retweet de una Publicación y el método DELETE te permite deshacer el Retweet de una Publicación determinada.

Como estás haciendo solicitudes en nombre de un usuario, debes autenticar estos endpoints con [Contexto de usuario de OAuth 1.0a](/es/resources/fundamentals/authentication) o [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), y utilizar los tokens de acceso de usuario asociados al usuario en cuyo nombre realizas la solicitud. Puedes generar este token de acceso de usuario mediante el [flujo OAuth de tres patas](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) o mediante el [flujo de concesión Authorization Code with PKCE](/es/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0). Puedes hacer Retweet de una Publicación desde tu cuenta o desde la cuenta de un usuario autenticado. Con ambos endpoints, hay un límite de uso por usuario de 50 solicitudes cada 15 minutos por endpoint.
