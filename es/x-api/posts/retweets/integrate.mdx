<div id="authentication">
  #### Autenticación
</div>

Todos los endpoints de X API v2 requieren que autentiques tus solicitudes con un conjunto de credenciales, también conocidas como claves y tokens.

Puedes usar Contexto de usuario de OAuth 1.0a o Bearer Token de OAuth 2.0 para autenticar tus solicitudes al endpoint de consulta de Retweets. 

Los endpoints para gestionar Retweets requieren el uso de Contexto de usuario de OAuth 1.0a, lo que significa que debes usar un conjunto de claves de API y tokens de acceso de usuario para realizar una solicitud correctamente. Los tokens de acceso deben estar asociados con el usuario en cuyo nombre estás realizando la solicitud. Si deseas generar un conjunto de tokens de acceso para otro usuario, este debe autorizar o autenticar tu aplicación usando el flujo OAuth de tres patas.

Ten en cuenta que OAuth 1.0a puede resultar complicado de usar. Si no estás familiarizado con este método de autenticación, te recomendamos que uses una [biblioteca](/es/x-api/tools-and-libraries/overview) para autenticar tus solicitudes correctamente.

**Ten en cuenta**

Si estás solicitando los siguientes campos, se requiere Contexto de usuario de OAuth 1.0a: 

* tweet.fields.non&#95;public&#95;metrics
* tweet.fields.promoted&#95;metrics
* tweet.fields.organic&#95;metrics
* media.fields.non&#95;public&#95;metrics
* media.fields.promoted&#95;metrics
* media.fields.organic&#95;metrics

<div id="developer-portal-projects-and-developer-apps">
  #### Portal para desarrolladores, Proyectos y aplicaciones de desarrollador
</div>

Para obtener un conjunto de credenciales de autenticación que funcionen con los endpoints de X API v2, debes [registrarte para obtener una cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info), configurar un [Proyecto](/es/resources/fundamentals/projects) dentro de esa cuenta y crear una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) dentro de ese Proyecto. Luego podrás encontrar tus claves y tokens dentro de tu aplicación de desarrollador. 
 

<div id="rate-limits">
  #### Límites de frecuencia
</div>

Cada día, muchos miles de desarrolladores realizan solicitudes a la X API. Para ayudar a gestionar el gran volumen de estas solicitudes, se aplican [límites de frecuencia](/es/x-api/fundamentals/rate-limits) a cada endpoint, que restringen el número de solicitudes que puedes realizar en nombre de tu Aplicación o en nombre de un usuario autenticado.

Los endpoints para gestionar Retweets están limitados a 50 solicitudes por cada 15 minutos (por usuario). Además, para el endpoint POST, tienes un límite de 300 solicitudes por intervalo de 3 horas (por usuario, por Aplicación). 

Con el endpoint de consulta de Retweets, tienes un límite de 75 solicitudes por intervalo de 15 minutos. Además, este endpoint solo devolverá a los 100 usuarios más recientes que hayan hecho Retweet.

<div id="fields-and-expansions">
  #### Campos y expansiones
</div>

La X API v2 permite a los usuarios seleccionar exactamente qué datos quieren obtener de la API usando un conjunto de herramientas llamadas campos y expansiones. El parámetro expansions te permite expandir los objetos referenciados en el payload. Por ejemplo, este endpoint te permite obtener las siguientes [expansiones](/es/x-api/fundamentals/expansions):

* attachments.poll&#95;ids
* attachments.media&#95;keys
* author&#95;id, entities.mentions.username
* geo.place&#95;id
* in&#95;reply&#95;to&#95;user&#95;id,
* referenced&#95;tweets.id,
* referenced&#95;tweets.id.author&#95;id

El parámetro fields te permite seleccionar exactamente qué [campos](/es/x-api/fundamentals/fields) dentro de los distintos objetos de datos quieres recibir. Estos endpoints devuelven principalmente objetos de tipo Publicación. De forma predeterminada, el objeto de Publicación devuelve los campos id y text. Para recibir campos adicionales como tweet.created&#95;at o tweet.entities, tendrás que solicitarlos específicamente usando un parámetro fields. Algunos campos importantes que quizá quieras considerar en tu integración son nuestros datos de encuestas, métricas, anotaciones de Publicación y campos de id de conversación.

Hemos añadido una guía sobre cómo [usar campos y expansiones](/es/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) juntos en nuestro [diccionario de datos de X API v2](/es/x-api/fundamentals/data-dictionary).
