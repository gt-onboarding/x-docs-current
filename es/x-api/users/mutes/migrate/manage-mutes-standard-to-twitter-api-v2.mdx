<div id="manage-mutes-standard-v11-compared-to-x-api-v2">
  ### Administrar silencios: estándar v1.1 en comparación con X API v2
</div>

Si has estado trabajando con los endpoints estándar v1.1 [POST mutes/users/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-create) y [POST mutes/users/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-destroy), el objetivo de esta guía es ayudarte a comprender las similitudes y diferencias entre los endpoints para administrar silencios de la versión estándar v1.1 y los de X API v2.

* **Similitudes**
  * Contexto de usuario de OAuth 1.0a
* **Diferencias**
  * URLs de los endpoints
  * Requisitos de aplicación y Proyecto
  * Métodos HTTP
  * Parámetros de la solicitud
     

<div id="similarities">
  #### Similitudes
</div>

**Método de autenticación con contexto de usuario de OAuth 1.0a**

Ambas versiones del endpoint son compatibles con el [Contexto de usuario de OAuth 1.0a](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Por lo tanto, si antes utilizabas uno de los endpoints estándar de v1.1 para gestionar silencios, puedes seguir usando el mismo método de autenticación si migras a la versión de X API v2 de este endpoint.

<div id="differences">
  #### Diferencias
</div>

**URLs de endpoints**

* Endpoints de Standard v1.1:
  * POST https://api.x.com/1.1/mutes/users/create.json
    (silenciar a un usuario)
  * POST https://api.x.com/1.1/mutes/users/destroy.json
    (dejar de silenciar a un usuario)
* Endpoint de X API v2:
  * POST https://api.x.com/2/users/:id/muting
    (silenciar a un usuario)
  * DELETE https://api.x.com/2/users/:source&#95;user&#95;id/muting/:target&#95;user&#95;id
    (dejar de silenciar a un usuario) 

**Requisitos de Aplicación y Proyecto**

Los endpoints de X API v2 requieren que uses credenciales de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) que esté asociada con un [Proyecto](/es/resources/fundamentals/projects) al autenticar tus solicitudes. Todos los endpoints de X API v1.1 pueden usar credenciales de aplicaciones independientes o de aplicaciones asociadas a un proyecto.

**Parámetros de la solicitud**

Los siguientes parámetros de solicitud de Standard v1.1 tienen equivalentes en X API v2:

| Standard v1.1 | X API v2 |
| :--- | :--- |
| user&#95;id | target&#95;user&#95;id |
| screen&#95;name | Sin equivalente |

Ten en cuenta que los parámetros de Standard v1.1 se envían como parámetros de consulta, mientras que los parámetros de X API v2 se envían como parámetros del cuerpo (para el endpoint POST) o parámetros de ruta (para el endpoint DELETE).

Además, no es necesario indicar el id del usuario que silencia al usuario de destino cuando se usan los endpoints de Standard v1.1, ya que los Tokens de acceso enviados con [Contexto de usuario de OAuth 1.0a](/es/resources/fundamentals/authentication) permitían inferir qué usuario estaba iniciando la acción de silenciar o dejar de silenciar.
