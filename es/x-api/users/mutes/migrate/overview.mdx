  ## Comparación de los endpoints de silenciamiento de X API
</div>

<div id="mutes-lookup">
  #### Búsqueda de cuentas silenciadas
</div>

El endpoint de búsqueda de cuentas silenciadas de la v2 reemplazará los endpoints estándar [v1.1 GET mutes/users/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/get-mutes-users-ids) y [GET mutes/users/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/get-mutes-users-list).

La siguiente tabla compara los endpoints estándar de silencios de v1.1 y X API v2:
 

| Descripción | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| Métodos HTTP compatibles | ******GET****** | ******GET****** |
| Dominio del host | ******https://api.x.com****** | ******https://api.x.com****** |
| Ruta del endpoint | ******/1.1/mutes/users/ids.json******<br /><br />/1.1/mutes/users/list.json | ******/2/users/:id/muting****** |
| [Autenticación](/es/resources/fundamentals/authentication) | Contexto de usuario de OAuth 1.0a | Contexto de usuario de OAuth 1.0a<br /><br />OAuth 2.0 Authorization Code with PKCE |
| [Límites de frecuencia](/es/resources/fundamentals/rate-limits) de solicitud predeterminados | 15 solicitudes cada 15 min (por usuario) | 15 solicitudes cada 15 min (por usuario) |
| Formatos de datos | Formato estándar v1.1 | [Formato de X API v2](/es/x-api/fundamentals/data-dictionary) (determinado por los parámetros de solicitud fields y expansions; no es compatible con versiones anteriores de los formatos de v1.1)<br /><br />Para obtener más información sobre cómo migrar del formato estándar v1.1 al formato de X API v2, visita nuestra [guía de migración de formatos de datos](/es/x-api/migrate/data-format-migration). |
| Requiere el uso de credenciales de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) asociada con un [Proyecto](/es/resources/fundamentals/projects) |     | ✔️  |

<div id="manage-mutes">
  #### Administrar silencios
</div>

Los endpoints de v2 para administrar silencios reemplazarán los endpoints estándar de v1.1 [POST mutes/users/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-create) y [POST mutes/users/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-destroy).

Las siguientes tablas comparan los endpoints estándar de v1.1 y X API v2 para silenciar:

**Silenciar a un usuario**

| Descripción | v1.1 estándar | X API v2 |
| :--- | :--- | :--- |
| Métodos HTTP admitidos | POST | POST |
| Dominio del host | https://api.x.com | https://api.x.com |
| Ruta del endpoint | /1.1/mutes/users/create.json | /2/users/:id/muting |
| [Autenticación](/es/resources/fundamentals/authentication) | Contexto de usuario de OAuth 1.0a | Contexto de usuario de OAuth 1.0a<br /><br />OAuth 2.0 Authorization Code with PKCE |
| [Límites de frecuencia](/es/resources/fundamentals/rate-limits) de solicitud predeterminados | 50 solicitudes por 15 min | 50 solicitudes por 15 min |
| Requiere el uso de credenciales de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) que esté asociada con un [Proyecto](/es/resources/fundamentals/projects) |     | ✔️  |

**Dejar de silenciar a un usuario**

Las siguientes tablas comparan los endpoints estándar de v1.1 y X API v2 para dejar de silenciar:

| Descripción | v1.1 estándar | X API v2 |
| :--- | :--- | :--- |
| Métodos HTTP admitidos | POST | DELETE |
| Dominio del host | https://api.x.com | https://api.x.com |
| Ruta del endpoint | /1.1/mutes/users/destroy.json | /2/users/:source&#95;user&#95;id/muting/:target&#95;user&#95;id |
| [Autenticación](/es/resources/fundamentals/authentication) | Contexto de usuario de OAuth 1.0a | Contexto de usuario de OAuth 1.0a<br /><br />OAuth 2.0 Authorization Code with PKCE |
| [Límites de frecuencia](/es/resources/fundamentals/rate-limits) de solicitud predeterminados | 50 solicitudes por 15 min | 50 solicitudes por 15 min |
| Requiere el uso de credenciales de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) que esté asociada con un [Proyecto](/es/resources/fundamentals/projects) |     | ✔️  |
