Silenciar una cuenta te permite quitar las Publicaciones de esa cuenta de tu cronología sin dejar de seguirla ni bloquearla. Las cuentas silenciadas no sabrán que las has silenciado y puedes dejar de silenciarlas en cualquier momento. Con los endpoints de gestión de silencios, los desarrolladores pueden crear experiencias más seguras para las personas en X. Un ejemplo de cómo usar la gestión de silencios es una aplicación que te permite silenciar cuentas que puedan publicar sobre temas específicos durante un período de tiempo determinado. Con el endpoint de búsqueda de silencios, puedes ver a quién has silenciado tú o un usuario autenticado. Esto puede ser útil para determinar cómo interactúas con las cuentas silenciadas. 

Dado que estás realizando solicitudes de información privada con la búsqueda de silencios y actuando en nombre de un usuario con la gestión de silencios, debes autenticar estos endpoints con [Contexto de usuario de OAuth 1.0a](/es/resources/fundamentals/authentication) o [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), y usar los tokens de acceso de usuario asociados con un usuario que haya autorizado tu aplicación, los cuales se pueden generar usando el [flujo OAuth de tres patas](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) o el [Authorization Code with PKCE grant flow](/es/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0).

<div id="mutes-lookup">
  ### Consulta de cuentas silenciadas
</div>

El endpoint de consulta de cuentas silenciadas te permite ver qué cuentas ha silenciado el usuario autenticado. Este endpoint tiene un límite de 15 solicitudes por usuario cada 15 minutos.

<div id="manage-mutes">
  ### Gestionar silenciamientos
</div>

Los endpoints para gestionar silenciamientos te permiten silenciar o dejar de silenciar una cuenta específica en nombre de un usuario autenticado. Para estos endpoints hay disponibles dos métodos: POST y DELETE. El método POST te permite silenciar una cuenta y el método DELETE te permite dejar de silenciar una cuenta. Existe un límite de uso por usuario de 50 solicitudes cada 15 minutos para los endpoints POST y DELETE.

**Ten en cuenta:** Si un usuario silencia desde [X](https://x.com/), hay un límite de 200 solicitudes cada 15 minutos.

**Configuración de la cuenta**

Para acceder a estos endpoints, necesitarás:

* Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
* Autenticarte usando las claves y tokens de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) que se encuentre dentro de un [Proyecto](/es/resources/fundamentals/projects).

Obtén más información sobre cómo acceder a los endpoints de X API v2 en nuestra [guía de introducción](/es/x-api/getting-started/getting-access).
