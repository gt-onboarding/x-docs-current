<div id="authentication">
  #### Autenticación
</div>

Todos los endpoints de X API v2 requieren que autentiques tus solicitudes con un conjunto de credenciales, también conocidas como claves y tokens. Puedes usar Contexto de usuario de OAuth 1.0a o OAuth 2.0 Authorization Code with PKCE para autenticar tus solicitudes a estos endpoints.

[Contexto de usuario de OAuth 1.0a](/es/resources/fundamentals/authentication) requiere que utilices tus claves de API, los tokens de acceso de usuario y algunos otros parámetros para [crear un encabezado de autorización](/es/resources/fundamentals/authentication#authorizing-a-request), que luego enviarás con tu solicitud. Los tokens de acceso deben estar asociados con el usuario en cuyo nombre estás realizando la solicitud. Si deseas generar un conjunto de tokens de acceso para otro usuario, ese usuario debe autorizar tu Aplicación utilizando el [flujo OAuth de tres patas](https://developer-staging.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens).

Ten en cuenta que OAuth 1.0a puede ser difícil de usar. Si no estás familiarizado con este método de autenticación, te recomendamos que utilices una [librería](/es/x-api/tools-and-libraries/overview), una herramienta como Postman o que utilices OAuth 2.0 para autenticar tus solicitudes.

[OAuth 2.0 Authorization Code with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) permite un mayor control sobre el alcance de una Aplicación y sobre los flujos de autorización en múltiples dispositivos. OAuth 2.0 te permite elegir ámbitos específicos y granulares que te otorgan permisos concretos en nombre de un usuario.

Para habilitar OAuth 2.0 en tu Aplicación, debes activarlo en la configuración de autenticación de tu Aplicación, que se encuentra en la sección de configuración de la Aplicación del portal para desarrolladores.

<div id="developer-portal-projects-and-developer-apps">
  #### Portal para desarrolladores, Proyectos y aplicaciones de desarrollador
</div>

Para obtener un conjunto de credenciales de autenticación que funcionen con los endpoints de X API v2, debes [registrarte para obtener una cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info), configurar un [Proyecto](/es/resources/fundamentals/projects) dentro de esa cuenta y crear una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) dentro de ese Proyecto. Luego podrás encontrar tus claves y tokens dentro de tu aplicación de desarrollador.
 

<div id="rate-limits">
  #### Límites de frecuencia
</div>

Cada día, muchos miles de desarrolladores realizan solicitudes a la X API. Para ayudar a gestionar el gran volumen de estas solicitudes, se aplican [límites de frecuencia](/es/x-api/fundamentals/rate-limits) a cada endpoint, lo que limita el número de solicitudes que puedes hacer en nombre de tu aplicación o en nombre de un usuario autenticado. 

Estos endpoints tienen límites de frecuencia a nivel de usuario, lo que significa que el usuario autenticado en cuyo nombre realizas la solicitud solo puede llamar al endpoint un cierto número de veces, independientemente de la aplicación de desarrollador que se utilice.

Hay un límite de frecuencia por usuario de 50 solicitudes por 15 minutos y por endpoint para los métodos POST y DELETE. Sin embargo, con el método GET, el límite de frecuencia es de solo 15 solicitudes por 15 minutos.
 

<div id="fields-and-expansions">
  #### Campos y expansiones
</div>

El endpoint GET de la X API v2 permite a los usuarios seleccionar exactamente qué datos quieren que se devuelvan por la API utilizando un conjunto de herramientas llamadas campos y expansiones. El parámetro expansions te permite expandir los objetos referenciados en el payload. Por ejemplo, este endpoint te permite obtener las siguientes [expansiones](/es/x-api/fundamentals/expansions):

* pinned&#95;tweet&#95;id

El parámetro **fields** te permite seleccionar exactamente qué [campos](/es/x-api/fundamentals/fields) dentro de los diferentes objetos de datos deseas recibir. Este endpoint devuelve principalmente objetos User. De forma predeterminada, el objeto User devuelve los campos **id**, **name** y **username**. Para recibir campos adicionales como **user.created&#95;at** o **user.entities**, tendrás que solicitar específicamente esos campos usando un parámetro **fields**. 

Hemos añadido una guía sobre cómo usar [campos y expansiones](/es/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) conjuntamente en nuestro [diccionario de datos de X API v2](/es/x-api/fundamentals/data-dictionary).

<div id="pagination">
  #### Paginación
</div>

La consulta de cuentas silenciadas puede devolver una gran cantidad de datos. Para asegurarnos de ofrecer resultados coherentes y de alto rendimiento en todo momento, utilizamos la paginación. La paginación es una funcionalidad de los endpoints de X API v2 que devuelven más resultados de los que se pueden incluir en una sola respuesta. Cuando eso sucede, los datos se devuelven en una serie de «páginas». Obtén más información sobre cómo [paginar los resultados.](/es/x-api/fundamentals/pagination)

<Note>
  **Ten en cuenta:** Si un usuario silencia a otro desde [X](https://x.com/), hay un límite de 200 solicitudes cada 15 minutos.
</Note>
