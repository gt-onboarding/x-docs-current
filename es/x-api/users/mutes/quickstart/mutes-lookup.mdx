<div id="getting-started-with-the-mutes-lookup-endpoint">
  ## Introducción al endpoint de consulta de silenciamientos
</div>

Esta guía de inicio rápido te ayudará a realizar tu primera solicitud al endpoint de consulta de silenciamientos con Postman.

Visita nuestro repositorio de GitHub de [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) si quieres ver código de ejemplo en distintos lenguajes.

<Note>
  **Requisitos previos**

  Para completar esta guía, necesitarás un conjunto de [claves y tokens](/es/resources/fundamentals/authentication) para autenticar tu solicitud. Puedes generar estas claves y tokens siguiendo estos pasos:

  * [Regístrate para obtener una cuenta de desarrollador](https://developer.x.com/en/apply-for-access) y obtén la aprobación.
  * Crea un [Project](/es/resources/fundamentals/projects) y una [developer App](/es/resources/fundamentals/developer-apps) asociada en el portal para desarrolladores.
  * Accede a la página “Keys and tokens” de tu App para generar las credenciales necesarias. Asegúrate de guardar todas las credenciales en un lugar seguro.
</Note>

<div id="steps-to-build-a-mutes-lookup-request">
  ### Pasos para crear una solicitud de búsqueda de cuentas silenciadas
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### Paso uno: Comienza con una herramienta o biblioteca
</div>

Existen varias herramientas, ejemplos de código y bibliotecas que puedes usar para realizar una solicitud a este endpoint, pero aquí usaremos la herramienta Postman para simplificar el proceso.

Para cargar la colección de X API v2 para Postman en tu entorno, haz clic en el siguiente botón:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Agregar X API v2 a Postman
</Button>

Una vez que tengas la colección de X API v2 cargada en Postman, navega a la carpeta “Mutes” y selecciona “Mutes lookup”.
 

<div id="step-two-authenticate-your-request">
  #### Paso dos: autentica tu solicitud
</div>

Para realizar correctamente una solicitud a la X API, debes verificar que tienes permiso. Para hacerlo con este endpoint, debes autenticar tu solicitud utilizando ya sea [OAuth 1.0a User Context](/es/resources/fundamentals/authentication) u [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

En este ejemplo, vamos a usar OAuth 1.0a User Context.

Debes añadir tus claves y tokens (específicamente tu API Key, API Secret Key, Token de acceso de usuario OAuth 1.0a y Secret del token de acceso de usuario OAuth 1.0a) a Postman. Puedes hacer esto seleccionando el entorno llamado &quot;X API v2&quot; en la esquina superior derecha de Postman y añadiendo tus claves y tokens a los campos &quot;initial value&quot; y &quot;current value&quot; (haciendo clic en el icono de ojo junto al menú desplegable del entorno).

Estas variables se completarán automáticamente en la pestaña de autorización de la solicitud si lo has hecho correctamente.
 

<div id="step-three-specify-a-user">
  #### Paso tres: Especificar un usuario
</div>

Con este endpoint, debes especificar tu ID de usuario o el ID de un usuario autenticado para ver a quién has silenciado tú o el usuario autenticado.

En Postman, ve a la pestaña &quot;Params&quot; e introduce el ID de usuario autenticado en la columna &quot;Value&quot; del id en “Path Variables” (al final de la sección), asegurándote de no incluir espacios antes o después del ID.

Encima de la sección “Path Variables”, verás que hay “Query Params” opcionales que puedes añadir. Para este ejemplo, marcaremos la variable max&#95;results y añadiremos un valor de 5.

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Parameter Type** |
| `id` | (tu ID de usuario) | Path |
| max&#95;results | 5   | Query |

<div id="step-four-identify-and-specify-which-fields-you-would-like-to-retrieve">
  #### Paso cuatro: Identificar y especificar qué campos te gustaría obtener
</div>

Si haces clic en el botón &quot;Send&quot; después del paso tres, recibirás en tu respuesta los campos predeterminados del [objeto de usuario](/es/x-api/fundamentals/data-dictionary#user): id, name y username.

Si quieres recibir campos adicionales además de id, name y username, tendrás que especificar esos campos en tu solicitud con los parámetros [fields](/es/x-api/fundamentals/fields) y/o [expansions](/es/x-api/fundamentals/expansions).

Para este ejercicio, solicitaremos tres conjuntos adicionales de campos de distintos objetos:

1. El campo adicional user.created&#95;at en los objetos de usuario principales.
2. Los campos predeterminados del objeto de Publicaciones fijadas asociadas de los usuarios devueltos: id y text.
3. El campo adicional tweet.created&#95;at en los objetos de Publicaciones asociadas.

En Postman, ve a la pestaña &quot;Params&quot; y agrega el siguiente par clave:valor a la tabla &quot;Query Params&quot;:

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| user.fields | created&#95;at | user.created&#95;at |
| expansions | pinned&#95;tweet&#95;id | tweet.id, tweet.text |
| tweet.fields | created&#95;at | includes.tweets.created&#95;at |

Ahora deberías ver una URL similar con tu propio id de usuario en lugar del id de ejemplo junto al botón &quot;Send&quot;:

`https://api.x.com/2/users/1324848235714736129/muting?user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=created_at&max_results=5`

<div id="step-five-make-your-request-and-review-your-response">
  #### Paso cinco: Realiza tu solicitud y revisa tu respuesta
</div>

Una vez que tengas todo listo, haz clic en el botón &quot;Send&quot; y recibirás una respuesta similar a la siguiente respuesta de ejemplo:

```
{
  "data": [
    {
      "username": "TwitterDev",
      "created_at": "2013-12-14T04:35:55.000Z",
      "id": "2244994945",
      "name": "Twitter Dev",
      "pinned_tweet_id": "1430984356139470849"
    }
  ],
  "includes": {
    "tweets": [
      {
        "created_at": "2021-08-26T20:03:51.000Z",
        "id": "1430984356139470849",
        "text": "Help us build a better Twitter Developer Platform!\n \nTake the annual developer survey &gt;&gt;&gt; https://t.co/9yTbEKlJHH https://t.co/fYIwKPzqua"
      }

  },
  "meta": {
    "result_count": 1
  }

```

<div id="step-six-paginate-through-your-results">
  #### Paso seis: pagina tus resultados
</div>

Puede que notes que hay un objeto meta ubicado al final de la respuesta. Si recibiste un next&#95;token, esto indica que hay otra página de resultados que podemos recuperar. Para obtener la siguiente página de resultados, deberás tomar el valor del campo next&#95;token y añadirlo a la solicitud como el valor de un parámetro de consulta pagination&#95;token adicional.
 

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| pagination&#95;token | 1710819323648428707 |

Si envías la solicitud después de añadir este parámetro adicional, los siguientes cinco resultados se entregarán con la siguiente carga útil, ya que especificamos max&#95;results como 5 en el paso tres. Puedes seguir repitiendo este proceso hasta que se hayan devuelto todos los resultados, pero también puedes usar el parámetro max&#95;results para solicitar hasta 1000 usuarios por solicitud, de modo que no tengas que paginar tantas veces.
