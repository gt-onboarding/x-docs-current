Ofrecemos dos conjuntos de endpoints para ayudarte a consultar, crear y eliminar relaciones de seguimiento: follows lookup y manage follows.

<div id="follows-lookup">
  ### Consulta de seguimientos
</div>

Disponible solo para quienes tengan [acceso Enterprise](/es/x-api/getting-started/about-x-api)

Los endpoints de consulta de seguimientos te permiten explorar y analizar las relaciones entre usuarios, lo que a veces se denomina análisis de red. En concreto, hay dos endpoints REST que devuelven [objetos de usuario](/es/x-api/fundamentals/data-dictionary#user) que representan a quién sigue un usuario específico o quién sigue a un usuario específico.

Puedes autenticar este endpoint con [Contexto de usuario de OAuth 1.0a](/es/resources/fundamentals/authentication), [solo aplicación](/es/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) o [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). Puedes solicitar hasta 1,000 usuarios por solicitud, y se proporcionarán tokens de paginación para recorrer conjuntos grandes de resultados.
 

<div id="manage-follows">
  ### Administrar seguimientos
</div>

Los endpoints de gestión de seguimientos te permiten seguir o dejar de seguir a usuarios.

Dado que estás realizando solicitudes en nombre de un usuario, debes autenticar estos endpoints con [OAuth 1.0a User Context](/es/resources/fundamentals/authentication) o [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), y utilizar los tokens de acceso de usuario asociados con el usuario en cuyo nombre estás realizando la solicitud. Puedes generar este token de acceso de usuario usando el [flujo OAuth de tres patas](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) o usando el [Authorization Code with PKCE grant flow](/es/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0).

Tienes un límite de 400 acciones de seguimiento por día en nombre de cada usuario autenticado y un límite de 1.000 acciones por día por aplicación en todos tus usuarios autenticados. Por ejemplo, si tienes cinco usuarios autenticados, puedes seguir a 400 usuarios por día (límite por usuario) con dos de esos usuarios para un total de 800 acciones, y tendrás que dividir las 200 acciones restantes (por aplicación) entre los otros tres usuarios. Este límite no se aplica al endpoint de dejar de seguir, que tiene un límite independiente de 500 acciones por día (por aplicación).

<Note>
  **Configuración de la cuenta**

  Para acceder a estos endpoints, necesitarás:

  * Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
  * Autenticarte utilizando las claves y tokens de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) que se encuentre dentro de un [Proyecto](/es/resources/fundamentals/projects). 
</Note>

Obtén más información sobre cómo obtener acceso a los endpoints de X API v2 en nuestra [guía de introducción](/es/x-api/getting-started/getting-access).
