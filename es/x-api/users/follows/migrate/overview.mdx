  ## Comparación de los endpoints de relaciones de seguimiento de la X API
</div>

<div id="follows-lookup">
  ### Consulta de seguimientos
</div>

Los endpoints de consulta de seguimientos de v2 reemplazarán los endpoints estándar de v1.1 [followers/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-followers-ids), v1.1 [followers/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-followers-list), v1.1 [friends/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-friends-ids) y v1.1 [friends/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-friends-list).

Las siguientes tablas comparan los diversos tipos de endpoints de consulta de seguimientos:

|     |     |     |
| :--- | :--- | :--- |
| **Descripción** | **Estándar v1.1** | **X API v2** |
| Métodos HTTP admitidos | `GET` | `GET` |
| Dominio del host | `https://api.x.com` | `https://api.x.com` |
| Ruta del endpoint | /1.1/friends/ids.json<br /><br />/1.1/friends/list.json<br /><br />/1.1/followers/ids.json<br /><br />/1.1/followers/list.json | /2/users/:id/following<br /><br />/2/users/:id/followers |
| [Autenticación](/es/resources/fundamentals/authentication) | Contexto de usuario de OAuth 1.0a<br /><br />Solo aplicación | Contexto de usuario de OAuth 1.0a<br /><br />Código de autorización de OAuth 2.0 con PKCE<br /><br />Solo aplicación |
| [Límites de frecuencia](/es/resources/fundamentals/rate-limits) de solicitud predeterminados | 15 solicitudes por 15 min (por usuario)<br /><br />15 solicitudes por 15 min (por aplicación) | 15 solicitudes por 15 min (por usuario)<br /><br />15 solicitudes por 15 min (por aplicación) |
| Máximo de usuarios por respuesta | GET friends/id y GET followers/id devuelven un máximo de 5000 IDs de usuarios por página.<br /><br />  <br /><br />GET friends/list y GET followers/list devuelven un máximo de 200 objetos de usuario por página. | 1000 objetos de usuario por página |
| Paginación | El token se devuelve en un campo next&#95;cursor, que luego puede pasarse como valor del parámetro cursor para devolver la siguiente página de resultados. | El token se devuelve en un campo next&#95;token, que luego puede pasarse como valor del parámetro token para devolver la siguiente página de resultados.<br /><br />La carga útil de v2 también proporciona un campo previous&#95;token, que también puede pasarse con el parámetro pagination&#95;token para devolver la página anterior de resultados. |
| Formato JSON | Formato estándar v1.1 | [Formato de X API v2](/es/x-api/fundamentals/data-dictionary) (determinado por los parámetros de solicitud fields y expansions, no es compatible con versiones anteriores de los formatos v1.1)<br /><br />Para obtener más información sobre cómo migrar del formato estándar v1.1 al formato de X API v2, visite nuestra [guía de migración de formatos de datos](/es/x-api/migrate/data-format-migration). |
| Admite la selección de qué [campos](/es/x-api/fundamentals/data-dictionary) se devuelven en la carga útil |     | ✔   |
| Admite los campos de [anotaciones](/es/x-api/fundamentals/post-annotations) de publicaciones |     | ✔   |
| Admite la solicitud de nuevos campos de [métricas](/es/x-api/fundamentals/metrics) |     | ✔   |
| Admite el campo [conversation&#95;id](/es/x-api/fundamentals/conversation-id) |     | ✔   |
| Requiere el uso de credenciales de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) asociada con un [Proyecto](/es/resources/fundamentals/projects) |     | ✔   |

<div id="manage-follows">
  #### Gestionar seguimientos
</div>

Los endpoints de gestión de seguimientos de v2 reemplazarán los endpoints estándar de v1.1 [POST friendships/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-create) y [POST friendships/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-destroy).

Las siguientes tablas comparan los endpoints estándar de v1.1 y de X API v2 para seguir a un usuario:

<div id="follow-a-user">
  #### Seguir a un usuario
</div>

| Descripción | Estándar v1.1 | X API v2 |
| :--- | :--- | :--- |
| Métodos HTTP compatibles | POST | POST |
| Dominio host | https://api.x.com | https://api.x.com |
| Ruta del endpoint | /1.1/friendships/create.json | /2/users/:id/following |
| [Autenticación](/es/resources/fundamentals/authentication) | Contexto de usuario de OAuth 1.0a | Contexto de usuario de OAuth 1.0a<br /><br />OAuth 2.0 Authorization Code con PKCE |
| [Límites de frecuencia](/es/resources/fundamentals/rate-limits) de solicitudes predeterminados | 50 solicitudes cada 15 minutos | 50 solicitudes cada 15 minutos |
| Número máximo de operaciones diarias por usuario | 400 | 400 |
| Número máximo de operaciones diarias por aplicación | 1000 | 1000 |
| Requiere el uso de credenciales de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) asociada a un [Proyecto](/es/resources/fundamentals/projects) |     | ✔️  |

<div id="unfollow-a-user">
  #### Dejar de seguir a un usuario
</div>

Las siguientes tablas comparan los endpoints de eliminación de seguimientos estándar de v1.1 y de X API v2:

| Descripción | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| Métodos HTTP compatibles | POST | DELETE |
| Dominio de host | https://api.x.com | https://api.x.com |
| Ruta del endpoint | /1.1/friendships/destroy.json | /2/users/:source&#95;user&#95;id/following/:target&#95;user&#95;id |
| [Autenticación](/es/resources/fundamentals/authentication) | Contexto de usuario de OAuth 1.0a | Contexto de usuario de OAuth 1.0a<br /><br />OAuth 2.0 Authorization Code with PKCE |
| [Límites de frecuencia](/es/resources/fundamentals/rate-limits) de solicitudes predeterminados | 15 solicitudes por 15 min (por usuario) | 50 solicitudes por 15 min (por usuario) |
| Operaciones diarias máximas por aplicación | Sin límite | 500 |
| Requiere el uso de credenciales de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) que esté asociada con un [Proyecto](/es/resources/fundamentals/projects) |     | ✔️  |
