<div id="getting-started-with-the-authenticated-user-lookup-endpoint">
  ## Introducción al endpoint de consulta de usuario autenticado
</div>

Esta guía de inicio rápido te ayudará a hacer tu primera solicitud al endpoint de consulta de usuario autenticado usando Postman.

Visita nuestro repositorio de GitHub de [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) si quieres ver código de ejemplo en distintos lenguajes de programación.

<Note>
  ### Requisitos previos

  Para completar esta guía, necesitarás contar con un conjunto de [keys and tokens](/es/resources/fundamentals/authentication) para autenticar tu solicitud. Puedes generar estas claves y tokens siguiendo estos pasos:

  * [Regístrate para obtener una cuenta de desarrollador](https://developer.x.com/en/apply-for-access) y obtén la aprobación.
  * Crea un [Project](/es/resources/fundamentals/projects) y una [developer App](/es/resources/fundamentals/developer-apps) asociada en el portal para desarrolladores.
  * Navega a la página «Keys and tokens» de tu App para generar las credenciales necesarias. Asegúrate de guardar todas las credenciales en una ubicación segura.
</Note>

<div id="steps-to-build-an-authenticated-user-lookup-request">
  ### Pasos para crear una solicitud de búsqueda del usuario autenticado
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### Paso uno: Comienza con una herramienta o biblioteca
</div>

Hay varias herramientas, ejemplos de código y bibliotecas que puedes usar para realizar una solicitud a este endpoint, pero aquí usaremos la herramienta Postman para simplificar el proceso.

Para cargar la colección de Postman de X API v2 en tu entorno, haz clic en el siguiente botón:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Agregar X API v2 a Postman
</Button>

Una vez que tengas la colección de X API v2 cargada en Postman, ve a la carpeta “Authenticated User Lookup” y selecciona “Lookup an Authenticated User”.

<div id="step-two-authenticate-your-request">
  #### Paso dos: Autentica tu solicitud
</div>

Para realizar correctamente una solicitud a la X API, debes verificar que tienes permiso. Para hacerlo con este endpoint, debes autenticar tu solicitud utilizando [OAuth 1.0a User Context](/es/resources/fundamentals/authentication) o [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

En este ejemplo, vamos a usar OAuth 1.0a User Context.

Debes agregar tus claves y tokens (específicamente tu API Key, API Secret Key, Token de acceso de usuario de OAuth 1.0a y Token de acceso secreto de usuario de OAuth 1.0a) a Postman. Puedes hacerlo seleccionando el entorno llamado “X API v2” en la esquina superior derecha de Postman y agregando tus claves y tokens a los campos &quot;initial value&quot; y &quot;current value&quot; (haciendo clic en el ícono de ojo junto al menú desplegable del entorno).

Estas variables se insertarán automáticamente en la pestaña de autorización de la solicitud si has realizado esto correctamente.
 

<div id="step-three-determine-which-user-fields-you-want-to-retrieve">
  ##### Paso tres: determina qué campos de usuario quieres recuperar
</div>

Si haces clic en el botón &quot;Send&quot; después del paso tres, recibirás en tu respuesta los campos predeterminados del [objeto de usuario](/es/x-api/fundamentals/data-dictionary#user): id, name y username.

Si deseas recibir campos adicionales más allá de id, name y username, tendrás que especificar esos campos en tu solicitud con los parámetros [field](/es/x-api/fundamentals/fields) y/o [expansion](/es/x-api/fundamentals/expansions).

Para este ejercicio, solicitaremos tres conjuntos adicionales de campos de diferentes objetos:

1. El campo adicional user.created&#95;at en los objetos de usuario principales.

2. Los campos predeterminados del objeto de Publicación fijada asociada para los usuarios devueltos: id y text.

3. El campo adicional tweet.created&#95;at en los objetos de Publicación asociados.

En Postman, navega a la pestaña &quot;Params&quot; y agrega el siguiente par clave-valor a la tabla &quot;Query Params&quot;:

|              |                               |                                                                   |
| :----------- | :---------------------------- | :---------------------------------------------------------------- |
| **Key**      | **Value**                     | **Returned fields**                                               |
| user.fields  | created&#95;at                | user.created&#95;at                                               |
| expansions   | pinned&#95;tweet&#95;id       | includes.tweets.id,  <br />includes.tweets.text                   |
| tweet.fields | created&#95;at, author&#95;id | includes.tweets.created&#95;at, includes&#95;tweets.author&#95;id |

Ahora deberías ver una URL similar junto al botón &quot;Send&quot;:

```
      https://api.x.com/2/users/me?user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=author_id,created_at
```
