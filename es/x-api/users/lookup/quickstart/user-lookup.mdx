<div id="getting-started-with-the-users-lookup-endpoints">
  ## Introducción a los endpoints de consulta de usuarios
</div>

Esta guía de inicio rápido te ayudará a realizar tu primera solicitud a los endpoints de consulta de usuarios con un conjunto de campos especificados usando [Postman](/es/tutorials/postman-getting-started).

Si deseas ver código de ejemplo en distintos lenguajes de programación, visita nuestro repositorio de [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) en GitHub. 

<Note>
  ### Requisitos previos

  Para completar esta guía, necesitarás contar con un conjunto de [claves y tokens](/es/resources/fundamentals/authentication) para autenticar tu solicitud. Puedes generar estas claves y tokens siguiendo estos pasos:

  * [Regístrate para obtener una cuenta de desarrollador](https://developer.x.com/en/apply-for-access) y recibe la aprobación.
  * Crea un [Proyecto](/es/resources/fundamentals/projects) y una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) asociada en el portal para desarrolladores.
  * Ve a la página “Keys and tokens” de tu App para generar las credenciales requeridas. Asegúrate de guardar todas las credenciales en una ubicación segura.
</Note>

<div id="steps-to-build-a-users-lookup-request">
  ### Pasos para crear una solicitud de consulta de usuarios
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### Paso uno: comienza con una herramienta o biblioteca
</div>

Hay varias herramientas, ejemplos de código y bibliotecas que puedes usar para realizar una solicitud a este endpoint, pero aquí vamos a usar la herramienta Postman para simplificar el proceso.

Para cargar la colección de X API v2 para Postman en tu entorno, haz clic en el siguiente botón:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Agregar X API v2 a Postman
</Button>

Una vez que tengas la colección de X API v2 cargada en Postman, navega al endpoint GET /users/by.
 

<div id="step-two-authenticate-your-request">
  #### Paso dos: autentica tu solicitud
</div>

Para realizar correctamente una solicitud a la X API, debes verificar que tienes permiso. Para ello, este endpoint requiere que autentiques tu solicitud con los métodos de autenticación [App only](/es/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) o [Contexto de usuario de OAuth 1.0a](/es/resources/fundamentals/authentication).

Para simplificar, utilizaremos App only con esta solicitud, pero deberás usar uno de los otros métodos de autenticación si deseas solicitar [métricas](/es/x-api/fundamentals/metrics) privadas o datos de usuarios privados.

Para utilizar App only, debes agregar tus claves y tokens, específicamente el [App Access Token](/es/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token) (también conocido como el App only Bearer Token) a Postman. Puedes hacerlo seleccionando el entorno llamado “X API v2” en la esquina superior derecha de Postman y agregando tus claves y tokens a los campos &quot;initial value&quot; y &quot;current value&quot; (haciendo clic en el ícono de ojo junto al menú desplegable de entornos).

Si lo has hecho correctamente, estas variables se incorporarán automáticamente en la pestaña &quot;Authorization&quot; de la solicitud.
 

<div id="step-three-identify-and-specify-which-users-you-would-like-to-retrieve">
  #### Paso tres: identifica y especifica qué usuario(s) quieres recuperar
</div>

Debes especificar un usuario o un conjunto de usuarios que quieres recibir en la solicitud. Según el endpoint de usuario que uses, puedes pasar un ID de usuario o un nombre de usuario. En esta situación, vamos a usar el [endpoint GET /users/by](/es/x-api/users/user-lookup-by-usernames), que te permite pasar múltiples nombres de usuario en una sola solicitud (en lugar de los endpoints [single-ID](/es/x-api/users/user-lookup-by-id), [multi-ID](/es/x-api/users/user-lookup-by-ids) y [single-username](/es/x-api/users/user-lookup-by-username)) y pasar un conjunto de nombres de usuario usando el parámetro de consulta usernames.

Los nombres de usuario son simplemente el identificador de la cuenta que puedes encontrar en la URL del perfil de una cuenta. Por ejemplo, el nombre de usuario de la siguiente cuenta es XDevelopers.

`https://x.com/XDevelopers`

En Postman, ve a la pestaña &quot;Params&quot; e introduce este nombre de usuario, o una cadena de nombres de usuario separados por una coma, en la columna &quot;Value&quot; del parámetro username, asegurándote de no incluir espacios entre los nombres de usuario y las comas. 

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `username` | XDeveloeprs,API,adsapi |

<div id="step-four-identify-and-specify-which-fields-you-would-like-to-retrieve">
  #### Paso cuatro: Identifica y especifica qué campos deseas recuperar
</div>

Si haces clic en el botón &quot;Send&quot; después del paso tres, recibirás en tu respuesta los campos predeterminados del [objeto de usuario](/es/x-api/fundamentals/data-dictionary#user): id, name y username.

Si deseas recibir campos adicionales más allá de id, name y username, tendrás que especificar esos campos en tu solicitud con los parámetros [fields](/es/x-api/fundamentals/fields) y/o [expansions](/es/x-api/fundamentals/expansions).

Para este ejercicio, solicitaremos tres conjuntos adicionales de campos de distintos objetos:

1. El campo adicional user.created&#95;at en los objetos de usuario principales.
2. Los campos predeterminados del objeto de Publicación fijada asociado para los usuarios devueltos: id y text.
3. El campo adicional tweet.created&#95;at en los objetos de Publicación asociados.
    

En Postman, ve a la pestaña &quot;Params&quot; y añade el siguiente par clave-valor a la tabla &quot;Query Params&quot;:

|                |              |                             |
| :------------- | :----------- | :-------------------------- |
| **Key**        | **Value**    | **Campos devueltos**        |
| `user.fields`  | `created_at` | `user.created_at`           |
| `expansions`   | `author_id`  | tweet.id, tweet.text        |
| `tweet.fields` | `created_at` | `includes.users.created_at` |

Ahora deberías ver la siguiente URL junto al botón &quot;Send&quot;:

```
      https://api.x.com/2/users/by?usernames=twitterdev,twitterapi,adsapi&user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=author_id,created_at
```
