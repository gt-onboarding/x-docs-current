  ## Comparación de los endpoints de búsqueda de usuarios de X API
</div>

Los endpoints de búsqueda de usuarios de X API v2 reemplazarán los endpoints estándar de v1.1 [GET users/lookup](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-users-lookup.html) y [GET users/show](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-users-show.html). Si tienes código, Aplicaciones o herramientas que usan una de estas versiones de los endpoints de búsqueda de usuarios y estás considerando migrar al endpoint más reciente de X API v2, entonces este conjunto de guías es para ti. 
 

Las siguientes tablas comparan los distintos tipos de endpoints de búsqueda de usuarios:
 

|     |     |     |
| :--- | :--- | :--- |
| **Descripción** | **Standard v1.1** | **X API v2** |
| Métodos HTTP admitidos | `GET` | `GET` |
| Dominio del host | `https://api.x.com` | `https://api.x.com` |
| Ruta del endpoint | `/1.1/users/show.json` `/1.1/users/lookup.json` | `/2/users`<br /><br />`/2/users/:id`<br /><br />`/2/users/by`<br /><br />`/2/users/by/:username` |
| [Autenticación](/es/resources/fundamentals/authentication) | Contexto de usuario de OAuth 1.0a | Contexto de usuario de OAuth 1.0a<br /><br />Solo aplicación<br /><br />OAuth 2.0 Authorization Code with PKCE |
| [Límites de frecuencia](/es/resources/fundamentals/rate-limits) predeterminados por solicitud | 900 solicitudes cada 15 min (por usuario)<br /><br />/show: 900 solicitudes cada 15 min (por aplicación)  <br />/lookup: 300 solicitudes cada 15 min (por aplicación) | 900 solicitudes cada 15 min (por usuario)<br /><br />300 solicitudes cada 15 min (por aplicación) |
| Máximo de usuarios por respuesta | /show: 1<br /><br />/lookup: 100 | 100 |
| Formato del objeto de respuesta JSON | Formato Standard v1.1 | [Formato X API v2](/es/x-api/fundamentals/data-dictionary) (determinado por los parámetros de solicitud fields y expansions, no es compatible de forma retroactiva con los formatos de v1.1)<br /><br />Para obtener más información sobre cómo migrar del formato Standard v1.1 al formato de X API v2, visita nuestra [guía de migración de formatos de datos](/es/x-api/migrate/data-format-migration). |
| Permite seleccionar qué [fields](/es/x-api/fundamentals/data-dictionary) se devuelven en el payload |     | ✔   |
| Admite los campos [annotations](/es/x-api/fundamentals/post-annotations) (en la Publicación fijada) |     | ✔   |
| Admite solicitar nuevos campos de [metrics](/es/x-api/fundamentals/metrics) (en la Publicación fijada) |     | ✔   |
| Admite el campo [conversation&#95;id](/es/x-api/fundamentals/conversation-id) (en la Publicación fijada) |     | ✔   |
| Requiere el uso de credenciales de una [developer App](/es/resources/fundamentals/developer-apps) asociada a un [Project](/es/resources/fundamentals/projects) |     | ✔   |
