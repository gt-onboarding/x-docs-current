<div id="authentication">
  #### Autenticación
</div>

Todos los endpoints de X API v2 requieren que las solicitudes estén [autenticadas](/es/resources/fundamentals/authentication) con un conjunto de credenciales, también conocidas como claves y tokens. Puedes usar Contexto de usuario de OAuth 1.0a, solo aplicación u OAuth 2.0 Authorization Code with PKCE para autenticar las solicitudes a estos endpoints. 

[Contexto de usuario de OAuth 1.0a](/es/resources/fundamentals/authentication#oauth-1-0a-2) requiere que utilices tus claves de API, Tokens de acceso de usuario y algunos otros parámetros para [crear un encabezado de autorización](/es/resources/fundamentals/authentication#authorizing-a-request), que luego enviarás junto con tu solicitud. Los Tokens de acceso deben estar asociados con el usuario en cuyo nombre estás realizando la solicitud. Si deseas generar un conjunto de Tokens de acceso para otro usuario, ese usuario debe autorizar tu Aplicación usando el [flujo OAuth de tres patas](https://developer-staging.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens). 

Ten en cuenta que OAuth 1.0a puede ser difícil de usar. Si no estás familiarizado con este método de autenticación, te recomendamos que uses una [biblioteca](/es/x-api/tools-and-libraries/overview), que utilices una herramienta como Postman o que uses OAuth 2.0 para autenticar tus solicitudes. Si deseas solicitar una Publicación o métricas privadas desde estos endpoints, tendrás que usar Contexto de usuario de OAuth 1.0a u OAuth 2.0 Authorization Code with PKCE, lo que garantizará que tengas los permisos adecuados del usuario que es propietario de ese contenido.
 

[Solo aplicación](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) solo requiere que envíes un [Token de acceso de solo aplicación](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) con tu solicitud. Puedes generar un Token de acceso de solo aplicación directamente desde una aplicación de desarrollador o generar uno usando el endpoint [POST oauth2/token](/es/resources/fundamentals/authentication#post-oauth2-token).

[OAuth 2.0 Authorization Code with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) permite un mayor control sobre el alcance de una aplicación y flujos de autorización a través de múltiples dispositivos. OAuth 2.0 te permite elegir scopes específicos y granulares que te otorgan permisos concretos en nombre de un usuario. 

Para habilitar OAuth 2.0 en tu Aplicación, debes habilitarlo en la configuración de autenticación de tu Aplicación, ubicada en la sección de configuración de la App en el portal para desarrolladores.

**Ten en cuenta**

Si estás solicitando los siguientes campos, se requiere Contexto de usuario de OAuth 1.0a u OAuth 2.0 Authorization Code: 

* tweet.fields.non&#95;public&#95;metrics
* tweet.fields.promoted&#95;metrics
* tweet.fields.organic&#95;metrics

<div id="developer-portal-projects-and-developer-apps">
  #### Portal para desarrolladores, Proyectos y aplicaciones de desarrollador
</div>

Para obtener un conjunto de credenciales de autenticación que funcionen con los endpoints de X API v2, debes [registrarte para obtener una cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info), configurar un [Proyecto](/es/resources/fundamentals/projects) dentro de esa cuenta y crear una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) dentro de ese Proyecto. Luego podrás encontrar tus claves y tokens dentro de tu aplicación de desarrollador. 
 

<div id="rate-limits">
  #### Límites de frecuencia
</div>

Cada día, muchos miles de desarrolladores realizan solicitudes a la X API. Para ayudar a gestionar el gran volumen de estas solicitudes, se aplican [límites de frecuencia](/es/x-api/fundamentals/rate-limits) a cada endpoint, que limitan la cantidad de solicitudes que puedes realizar en nombre de tu aplicación o en nombre de un usuario autenticado.

Los endpoints de búsqueda de usuarios tienen límites de frecuencia tanto a nivel de aplicación como a nivel de usuario. Sin embargo, el endpoint de búsqueda de usuario autenticado tiene límites de frecuencia únicamente a nivel de usuario.

El límite de frecuencia a nivel de aplicación significa que tú, como desarrollador, solo puedes realizar una cierta cantidad de solicitudes a este endpoint durante un período de tiempo determinado desde cualquier App (identificada por las claves y tokens que estés utilizando). El límite de frecuencia a nivel de usuario significa que el usuario autenticado en cuyo nombre estás realizando la solicitud solo puede realizar esa operación un número determinado de veces en cualquier aplicación de desarrollador.

La tabla siguiente muestra los límites de frecuencia para cada endpoint.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Método HTTP** | **Límite de frecuencia / Nivel** |
| /2/users | GET | 900 solicitudes cada 15 minutos / App y Usuario |
| /2/users/:id | GET | 900 solicitudes cada 15 minutos / App y Usuario |
| /2/users/by | GET | 900 solicitudes cada 15 minutos / App y Usuario |
| /2/users/by/username/:username | GET | 900 solicitudes cada 15 minutos / App y Usuario |
| /2/users/me | GET | 75 solicitudes cada 15 minutos / Usuario |

<div id="fields-and-expansions">
  #### Campos y expansiones
</div>

La X API v2 permite a los usuarios seleccionar exactamente qué datos quieren que se devuelvan de la API utilizando un conjunto de herramientas llamado campos y expansiones. El parámetro expansion te permite expandir objetos referenciados en el payload. Por ejemplo, este endpoint te permite usar la expansión pinned&#95;tweet&#95;id.

El parámetro fields te permite seleccionar exactamente qué [campos](/es/x-api/fundamentals/fields) dentro de los diferentes objetos de datos quieres recibir. Estos endpoints devuelven principalmente objetos de usuario. De forma predeterminada, el objeto de usuario devuelve los campos id, name y username. Para recibir campos adicionales como user.created&#95;at o user.location, tendrás que solicitarlos específicamente usando un parámetro fields. Algunos campos importantes que podrías considerar usar en tu integración son nuestros datos de encuestas de Publicaciones, métricas, anotaciones y los campos de ID de conversación.

Hemos añadido una guía sobre cómo [usar campos y expansiones](/es/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) en nuestro [diccionario de datos de X API v2](/es/x-api/fundamentals/data-dictionary).
