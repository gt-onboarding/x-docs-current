<div id="authentication">
  #### Autenticación
</div>

Todos los endpoints de X API v2 requieren que las solicitudes estén [autenticadas](/es/resources/fundamentals/authentication) con un conjunto de credenciales, también conocidas como claves y tokens. Puedes usar Contexto de usuario de OAuth 1.0a, App only o OAuth 2.0 Authorization Code with PKCE para autenticar las solicitudes a estos endpoints. 

[OAuth 1.0a User Context](/es/resources/fundamentals/authentication#oauth-1-0a-2) requiere que utilices tus claves de API, tokens de acceso de usuario y varios otros parámetros para [crear un encabezado de autorización](/es/resources/fundamentals/authentication#authorizing-a-request), que luego pasarás junto con tu solicitud. Los tokens de acceso deben estar asociados con el usuario en cuyo nombre estás realizando la solicitud. Si deseas generar un conjunto de tokens de acceso para otro usuario, este debe autorizar tu App mediante el [flujo OAuth de tres patas](https://developer-staging.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens). 

Ten en cuenta que OAuth 1.0a puede ser difícil de usar. Si no estás familiarizado con este método de autenticación, te recomendamos que utilices una [biblioteca](/es/x-api/tools-and-libraries/overview), uses una herramienta como Postman o utilices OAuth 2.0 para autenticar tus solicitudes. Si deseas solicitar una Publicación o métricas privadas en estos endpoints, tendrás que usar Contexto de usuario de OAuth 1.0a u OAuth 2.0 Authorization Code with PKCE, lo que garantizará que tengas los permisos adecuados del usuario propietario de ese contenido.
 

[App only](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) solo requiere que pases un [App only Access Token](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) con tu solicitud. Puedes generar un App only Access Token directamente dentro de una aplicación de desarrollador o generar uno utilizando el endpoint [POST oauth2/token](/es/resources/fundamentals/authentication#post-oauth2-token).

[OAuth 2.0 Authorization Code with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) permite un mayor control sobre el alcance de una aplicación y sobre los flujos de autorización en múltiples dispositivos. OAuth 2.0 te permite elegir ámbitos específicos y detallados que te otorgan permisos concretos en nombre de un usuario. 

Para habilitar OAuth 2.0 en tu App, debes activarlo en la configuración de autenticación de tu App que se encuentra en la sección de configuración de la App en el portal para desarrolladores.

**Ten en cuenta lo siguiente**

Si estás solicitando los siguientes campos, se requiere OAuth 1.0a User Context u OAuth 2.0 Authorization Code: 

* tweet.fields.non&#95;public&#95;metrics
* tweet.fields.promoted&#95;metrics
* tweet.fields.organic&#95;metrics

<div id="developer-portal-projects-and-developer-apps">
  #### Portal para desarrolladores, Proyectos y aplicaciones de desarrollador
</div>

Para obtener un conjunto de credenciales de autenticación que funcionen con los endpoints de X API v2, debes [registrarte para obtener una cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info), configurar un [Proyecto](/es/resources/fundamentals/projects) dentro de esa cuenta y crear una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) dentro de ese Proyecto. Luego podrás encontrar tus claves y tokens dentro de tu aplicación de desarrollador. 
 

<div id="rate-limits">
  #### Límites de frecuencia
</div>

Todos los días, muchos miles de desarrolladores hacen solicitudes a la X API. Para ayudar a gestionar el gran volumen de estas solicitudes, se aplican [límites de frecuencia](/es/x-api/fundamentals/rate-limits) a cada endpoint que limitan el número de solicitudes que puedes hacer en nombre de tu aplicación o en nombre de un usuario autenticado. 

Los endpoints de búsqueda de usuarios tienen límites de frecuencia tanto a nivel de App como a nivel de usuario. Sin embargo, el endpoint de búsqueda de usuario autenticado tiene límite de frecuencia únicamente a nivel de usuario.

El límite de frecuencia a nivel de App significa que tú, como desarrollador, solo puedes hacer una cantidad determinada de solicitudes a este endpoint durante un período de tiempo dado desde cualquier App (según las claves y tokens que estés usando). El límite de frecuencia a nivel de usuario significa que el usuario autenticado en cuyo nombre realizas la solicitud solo puede realizar esta operación una cantidad determinada de veces en cualquier aplicación de desarrollador.

La siguiente tabla muestra los límites de frecuencia para cada endpoint.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Método HTTP** | **Límite de frecuencia / Nivel** |
| /2/users | GET | 900 solicitudes cada 15 minutos / App y usuario |
| /2/users/:id | GET | 900 solicitudes cada 15 minutos / App y usuario |
| /2/users/by | GET | 900 solicitudes cada 15 minutos / App y usuario |
| /2/users/by/username/:username | GET | 900 solicitudes cada 15 minutos / App y usuario |
| /2/users/me | GET | 75 solicitudes cada 15 minutos / Usuario |

<div id="fields-and-expansions">
  #### Campos y expansiones
</div>

La X API v2 permite a los usuarios seleccionar exactamente qué datos quieren que devuelva la API mediante un conjunto de herramientas llamadas campos y expansiones. El parámetro `expansions` te permite expandir los objetos referenciados en el payload. Por ejemplo, este endpoint te permite usar la expansión `pinned_tweet_id`.

El parámetro `fields` te permite seleccionar exactamente qué [campos](/es/x-api/fundamentals/fields) dentro de los diferentes objetos de datos deseas recibir. Estos endpoints devuelven principalmente objetos de usuario. De forma predeterminada, el objeto de usuario devuelve los campos `id`, `name` y `username`. Para recibir campos adicionales como `user.created_at` o `user.location`, tendrás que solicitarlos específicamente usando un parámetro `fields`. Algunos campos importantes que quizá quieras considerar en tu integración son nuestros datos de encuestas de Publicaciones, métricas, anotaciones y los campos de ID de conversación.

Hemos añadido una guía sobre cómo [usar campos y expansiones](/es/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) en nuestro [diccionario de datos de X API v2](/es/x-api/fundamentals/data-dictionary).
