Puedes autenticar tu solicitud a todos los endpoints de consulta de usuarios, excepto al endpoint de consulta del usuario autenticado, con [OAuth 1.0a User Context](/es/resources/fundamentals/authentication), [App only](/es/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) o [OAuth 2.0 Authorization code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). Sin embargo, si quieres acceder a métricas privadas o a datos de usuarios privados, deberás utilizar OAuth 1.0a User Context u OAuth 2.0 Authorization Code with PKCE y pasar el Token de acceso de los usuarios autenticados con tu solicitud.    

Este endpoint se utiliza comúnmente para obtener detalles actualizados de un usuario, verificar que un usuario existe o actualizar los detalles que tienes almacenados después de un evento de cumplimiento.

<div id="authenticated-user-lookup">
  ### Búsqueda de usuario autenticado
</div>

Si te autenticas en nombre de otros usuarios, es fundamental poder ver por quién puedes realizar una solicitud.

Este endpoint requiere que uses [OAuth 1.0a User Context](/es/resources/fundamentals/authentication) o [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). Devuelve información sobre el usuario autorizado, es decir, el usuario asociado con los tokens de acceso de usuario que envías con la solicitud.

Puedes acceder a este endpoint mediante el método GET. Hay un límite de frecuencia por usuario de 75 solicitudes cada 15 minutos para este endpoint.

<Note>
  **Configuración de la cuenta**

  Para acceder a estos endpoints, necesitarás:

  * Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
  * Autenticarte usando las claves y tokens de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) que se encuentre dentro de un [Proyecto](/es/resources/fundamentals/projects). 

  Obtén más información sobre cómo obtener acceso a los endpoints de X API v2 en nuestra [guía de primeros pasos](/es/x-api/getting-started/getting-access).
</Note>
