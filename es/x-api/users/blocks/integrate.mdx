<div id="authentication">
  #### Autenticación
</div>

Todos los endpoints de X API v2 requieren que autentiques tus solicitudes con un conjunto de credenciales, también conocidas como claves y tokens. Puedes usar Contexto de usuario de OAuth 1.0a u OAuth 2.0 Authorization Code with PKCE para autenticar tus solicitudes a estos endpoints. 

[Contexto de usuario de OAuth 1.0a](/es/resources/fundamentals/authentication) requiere que utilices tus claves de API, tokens de acceso de usuario y un conjunto adicional de parámetros para [crear un encabezado de autorización](/es/resources/fundamentals/authentication#authorizing-a-request), que luego enviarás con tu solicitud. Los tokens de acceso deben estar asociados con el usuario en cuyo nombre estás realizando la solicitud. Si deseas generar un conjunto de tokens de acceso para otro usuario, este debe autorizar tu Aplicación utilizando el [flujo OAuth de tres patas](https://developer-staging.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens). 

Ten en cuenta que OAuth 1.0a puede ser difícil de usar. Si no estás familiarizado con este método de autenticación, te recomendamos que uses una [biblioteca](/es/x-api/tools-and-libraries/overview), utilices una herramienta como Postman o uses OAuth 2.0 para autenticar tus solicitudes.

[OAuth 2.0 Authorization Code with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) permite un mayor control sobre el alcance de una Aplicación y sobre los flujos de autorización en múltiples dispositivos. OAuth 2.0 te permite elegir ámbitos específicos y granulares que te otorgan permisos concretos en nombre de un usuario. 

Para habilitar OAuth 2.0 en tu Aplicación, debes activarlo en la configuración de autenticación de tu Aplicación, que se encuentra en la sección App settings del portal para desarrolladores.

<div id="developer-portal-projects-and-developer-apps">
  #### Portal para desarrolladores, Proyectos y aplicaciones de desarrollador
</div>

Para obtener un conjunto de credenciales de autenticación que funcionen con los endpoints de X API v2, debes [registrar una cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info), configurar un [Proyecto](/es/resources/fundamentals/projects) dentro de esa cuenta y crear una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) dentro de ese Proyecto. Luego podrás encontrar tus claves y tokens dentro de tu aplicación de desarrollador.
 

<div id="rate-limits">
  #### Límites de frecuencia
</div>

Cada día, muchos miles de desarrolladores realizan solicitudes a la X API. Para ayudar a gestionar el enorme volumen de estas solicitudes, se aplican [límites de frecuencia](/es/x-api/fundamentals/rate-limits) a cada endpoint, que limitan la cantidad de solicitudes que puedes realizar en nombre de tu aplicación o en nombre de un usuario autenticado. 

Estos endpoints tienen límites de frecuencia a nivel de usuario, lo que significa que el usuario autenticado en cuyo nombre realizas la solicitud solo puede invocar el endpoint un determinado número de veces en cualquier aplicación de desarrollador. Existe un límite de frecuencia por usuario de 50 solicitudes por 15 minutos por endpoint con los métodos POST y DELETE. Sin embargo, con el método GET, el límite de frecuencia es de solo 15 solicitudes por 15 minutos.
 

<div id="fields-and-expansions">
  #### Campos y expansiones
</div>

Los endpoints GET de X API v2 permiten a los usuarios seleccionar exactamente qué datos quieren que se devuelvan de la API utilizando un conjunto de herramientas llamadas campos y expansiones. El parámetro expansions te permite expandir los objetos referenciados en el payload. Por ejemplo, este endpoint te permite obtener las siguientes [expansiones](/es/x-api/fundamentals/expansions):

* pinned&#95;tweet&#95;id

El parámetro **fields** te permite seleccionar exactamente qué [campos](/es/x-api/fundamentals/fields) dentro de los diferentes objetos de datos deseas recibir. Estos endpoints devuelven principalmente objetos de Publicación. De forma predeterminada, el objeto de Publicación devuelve los campos **id** y **text**. Para recibir campos adicionales como **tweet.created&#95;at** o **tweet.entities**, tendrás que solicitarlos específicamente usando un parámetro **fields**. Algunos campos importantes que quizá quieras considerar usar en tu integración son nuestros datos de encuestas, métricas, anotaciones de Publicación y campos de id de conversación.

Hemos añadido una guía sobre cómo [usar campos y expansiones](/es/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) en conjunto en nuestro [diccionario de datos de X API v2](/es/x-api/fundamentals/data-dictionary).
