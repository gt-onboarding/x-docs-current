<div id="introduction">
  ## Introducción
</div>

Un kit de desarrollo de software (SDK) es un conjunto de herramientas y programas de software adaptados a una plataforma o API específica. El propósito de un SDK es crear o ampliar la funcionalidad de las aplicaciones proporcionando bibliotecas o bases de código que los desarrolladores pueden utilizar dentro de sus aplicaciones, de forma eficiente y con una cantidad mínima de código. Esto acelera significativamente el proceso de desarrollo, ahorrando tiempo, dinero y esfuerzo.

> La Plataforma para Desarrolladores de X ahora ofrece dos SDK oficiales para quienes desarrollan en TypeScript/JavaScript y Java. Estos permitirán a los desarrolladores crear de forma más eficaz al eliminar la necesidad de programar manualmente las complejidades relacionadas con la X API v2, utilizando las funciones predefinidas para todos los endpoints v2 disponibles, además de simplificar el proceso de autenticación. Como estos SDK son creados y mantenidos por el equipo de Developer Platform, siempre estarán actualizados con las futuras versiones de la X API v2.

Dado que estos SDK encapsulan la X API, debes tener una [cuenta de desarrollador](https://developer.x.com/en/portal/dashboard) para autenticar las solicitudes utilizando las credenciales de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps), ubicada dentro de un [Proyecto](/es/resources/fundamentals/projects).

<div id="client-basics">
  #### Conceptos básicos del cliente
</div>

Importa las clases (Java) y el paquete (TypeScript) al inicio de un archivo de trabajo para obtener acceso a los clientes de autenticación y de biblioteca. Para poder usar los métodos del cliente de biblioteca, se deben pasar las credenciales de autenticación; estas pueden ser un Bearer Token (solo de Aplicación) o `client id`/`client secret` si te autenticas con contexto de usuario de OAuth 2.0.

Aquí tienes ejemplos de cómo se ve esto:

<Tabs>
  <Tab title="Java">
    ```java
    // Importa las clases:
    import com.twitter.clientlib.ApiClient;
    import com.twitter.clientlib.ApiException;
    import com.twitter.clientlib.Configuration;
    import com.twitter.clientlib.auth.*;
    import com.twitter.clientlib.model.*;
    import com.twitter.clientlib.TwitterCredentialsBearer;
    import com.twitter.clientlib.api.TwitterApi;

    // Instancia el cliente de biblioteca
    TwitterApi apiInstance = new TwitterApi();

    // Instancia las credenciales de autenticación (ejemplo solo de Aplicación)
    TwitterCredentialsBearer credentials = new TwitterCredentialsBearer(System.getenv("APP-ONLY-ACCESS-TOKEN"));

    // Pasa las credenciales al cliente de biblioteca
    apiInstance.setTwitterCredentials(credentials);
    ```
  </Tab>

  <Tab title="TypeScript">
    ```typescript
    // Importa el paquete
    import { Client, auth } from "twitter-api-sdk";

    // Inicializa primero el cliente de autenticación
    const authClient = new auth.OAuth2User({
     client_id: process.env.CLIENT_ID as string,
     client_secret: process.env.CLIENT_SECRET as string,
     callback: "YOUR-CALLBACK",
     scopes: ["tweet.read", "users.read", "offline.access"],
    });

    // Pasa las credenciales de autenticación al cliente de biblioteca
    const twitterClient = new Client(authClient);
    ```
  </Tab>
</Tabs>

<div id="authentication-flow">
  #### Flujo de autenticación
</div>

Si utilizas la opción de solo aplicación para autenticar los SDKs, solo necesitarás proporcionar el token y el cliente de la librería estará listo para usar de inmediato los métodos del endpoint. Ten en cuenta que los tokens de solo aplicación no se pueden usar en endpoints que requieren autenticación con contexto de usuario.

La autenticación en contexto de usuario con OAuth 2.0 requiere algunos pasos adicionales después de crear el cliente de autenticación.

* Generar la URL de autorización
* Autorizar la aplicación desde la URL de autorización
* Redirigir a la URL de callback (esta debe coincidir con la URL de callback configurada en la página de configuración de autenticación en el portal para desarrolladores).
* Analizar el code verifier para intercambiarlo por un token de acceso

Los SDKs proporcionan métodos en el cliente de autenticación que simplifican estos pasos. Para ver un ejemplo completo de cómo hacer una solicitud autenticándote con contexto de usuario de OAuth 2.0, consulta los repositorios de GitHub.

* [TypeScript](https://github.com/xdevplatform/twitter-api-typescript-sdk/blob/main/examples/oauth2-callback_pkce_s256.ts)

* [Java](https://github.com/xdevplatform/twitter-api-java-sdk/blob/main/examples/src/main/java/com/twitter/clientlib/auth/OAuth20GetAccessToken.java)

<div id="endpoint-methods">
  #### Métodos del endpoint
</div>

Los métodos proporcionados dentro del cliente de la librería tienen nombres claros que corresponden a cada endpoint y todos los parámetros se pasan como argumentos.
A continuación se muestra un ejemplo de búsqueda de una Publicación por id:

<Tabs>
  <Tab title="Java">
    ```java
    String id = "1511757922354663425"; // String | Un único id de Publicación.
    Set<String> expansions = new HashSet<>(Arrays.asList("author_id")); // Set<String> | Una lista de valores separados por comas de campos para expandir.
    Set<String> tweetFields = new HashSet<>(Arrays.asList("created_at", "lang", "context_annotations")); // Set<String> | Una lista de valores separados por comas de campos de Publicación para mostrar.
    Set<String> userFields = new HashSet<>(Arrays.asList("created_at", "description", "name")); // Set<String> | Una lista de valores separados por comas de campos de Usuario para mostrar.

    try {
     SingleTweetLookupResponse result = apiInstance.tweets().findTweetById(id, expansions, tweetFields, userFields, null, null, null);
     System.out.println(result);
    } catch (ApiException e) {
     System.err.println("Exception when calling TweetsApi#findTweetById");
     System.err.println("Status code: " + e.getCode());
     System.err.println("Reason: " + e.getResponseBody());
     System.err.println("Response headers: " + e.getResponseHeaders());
     e.printStackTrace();
    }

    ```
  </Tab>

  <Tab title="TypeScript">
    ```typescript
    const lookupTweetById = await client.tweets.findTweetById(
      // id de la Publicación
      "1511757922354663425",
      {
        // Parámetros opcionales
        expansions: ["author_id"],
        "user.fields": ["created_at", "description", "name"],
      }
    );
    ```
  </Tab>
