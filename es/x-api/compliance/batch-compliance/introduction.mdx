X está comprometida con nuestra comunidad de desarrolladores que crean con la X API. Como parte de este compromiso, nuestro objetivo es que nuestra API sea abierta y justa para los desarrolladores, segura para las personas en X y beneficiosa para la plataforma X en su conjunto. Es fundamental que cualquier desarrollador que almacene contenido de X sin conexión se asegure de que los datos reflejen la intención del usuario y el estado actual del contenido en X. Por ejemplo, cuando alguien en X elimina una Publicación o su cuenta, protege sus Publicaciones o elimina la información geográfica de sus Publicaciones, es fundamental tanto para X como para nuestros desarrolladores respetar las expectativas e intención de esa persona. Los endpoints de batch compliance proporcionan a los desarrolladores una herramienta sencilla para ayudar a mantener los datos de X en cumplimiento con el [X Developer Agreement and Policy](https://developer.x.com/en/developer-terms/policy). 

Estos endpoints de batch compliance te permiten cargar grandes conjuntos de datos de IDs de Publicaciones o de usuarios para recuperar su estado de cumplimiento con el fin de determinar qué datos requieren acción para que tus conjuntos de datos cumplan con las normas. Ten en cuenta que el uso de los endpoints de batch compliance está restringido a los casos de uso antes mencionados, y cualquier otro propósito está prohibido y puede dar lugar a medidas de aplicación.

Normalmente, hay 4 pasos implicados al trabajar con este endpoint:

1. **Create a compliance job**
   Puedes especificar el tipo de job (con el valor tweets o users) para indicar si el conjunto de datos que quieres cargar contiene IDs de Publicaciones o IDs de usuario. Puedes tener un job concurrente por tipo de job en cualquier momento.
2. **Upload your dataset to the upload&#95;url**
   A continuación, cargas tu conjunto de datos como un archivo de texto plano en la upload&#95;url proporcionada, donde cada línea del archivo contiene un solo ID de Publicación o ID de usuario. La upload&#95;url caduca a los 15 minutos.
3. **(Optional) Check the job status**
   Puedes comprobar el estado de tu compliance job para ver si está en estado created, in&#95;progress, failed o complete.
4. **Download the results**
   Cuando tu job se complete, podrás descargar los resultados utilizando la download&#95;url. La download&#95;url caduca después de una semana (desde que se creó el job).

   Este resultado contendrá un conjunto de objetos JSON (un objeto por línea). Cada objeto contendrá un ID de Publicación, la fecha de creación de la Publicación (útil para localizar Publicaciones organizadas por fecha), la acción requerida, el motivo de la acción de cumplimiento y la fecha en que el usuario fue suspendido.

Recibirás los siguientes tipos de eventos de cumplimiento en tus resultados:

* deleted: indica que la Publicación o la cuenta de usuario se eliminó
* deactivated: indica que la Publicación o la cuenta de usuario se desactivó
* scrub&#95;geo: indica que se eliminó la información geográfica asociada con la Publicación o el usuario
* protected: indica que la cuenta que hizo la Publicación se volvió privada
* suspended: indica que la cuenta que hizo la Publicación fue suspendida

**Configuración de la cuenta**

Para acceder a estos endpoints, necesitarás:

* Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
* Autenticarte utilizando las claves y tokens de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) que se encuentra dentro de un [Proyecto](/es/resources/fundamentals/projects). 

Obtén más información sobre cómo acceder a los endpoints de X API v2 en nuestra [guía de primeros pasos](/es/x-api/getting-started/getting-access).
