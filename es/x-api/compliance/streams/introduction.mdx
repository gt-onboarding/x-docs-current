X se compromete con nuestra comunidad de desarrolladores que desarrollan con la X API. Como parte de este compromiso, nuestro objetivo es que nuestra API sea abierta y justa para los desarrolladores, segura para las personas en X y beneficiosa para la plataforma X en su conjunto. Es crucial que cualquier desarrollador que almacene contenido de X sin conexión se asegure de que los datos reflejen la intención del usuario y el estado actual del contenido en X. Por ejemplo, cuando alguien en X elimina una Publicación o su cuenta, protege sus Publicaciones o edita una Publicación, es fundamental tanto para X como para nuestros desarrolladores respetar las expectativas e intención de esa persona.

Los streams en tiempo real de eventos de cumplimiento proporcionan a los desarrolladores las herramientas para mantener los datos de X en cumplimiento con el [Acuerdo y Política para Desarrolladores de X](https://developer.x.com/en/developer-terms/policy). 

Hay dos streams de c&#95;ompliance event&#95;: uno para eventos de cumplimiento de Post&#95; y otro para eventos de *cumplimiento de Usuario*. Estos streams están disponibles con acceso Enterprise y están diseñados para ayudar a los socios que ingieren grandes volúmenes de datos a “escuchar” eventos de cumplimiento, como los eventos de edición de Publicaciones.

Estos streams proporcionan los siguientes eventos: 

**Post compliance stream: **

* delete - indica que la Publicación fue eliminada.

* tweet&#95;edit - indica que una Publicación ha sido editada y proporciona el ID de la Publicación actualizada. 

* withheld - indica que la Publicación ha sido restringida en uno o más países. 

* drop - indica que la Publicación debe eliminarse de la vista pública.

* undrop - indica que la Publicación puede mostrarse de nuevo y tratarse como pública.

**User compliance stream: **

* **user&#95;delete** - indica que la cuenta de Usuario fue eliminada

* **user&#95;undelete** - indica que la cuenta de Usuario fue restaurada

* **user&#95;protect** - indica que la cuenta de Usuario pasó a ser privada

* **user&#95;unprotect** - indica que la cuenta de Usuario pasó a ser pública

* **user&#95;withheld** - indica que la cuenta de Usuario ha sido restringida en uno o más países. 

* **user&#95;suspend** - indica que la cuenta de Usuario fue suspendida

* **user&#95;unsuspend** - indica que la cuenta de Usuario fue restablecida

* **user&#95;profile&#95;modification** - indica que el perfil de Usuario se ha actualizado. Esto incluye una descripción, nombre, ubicación y URL actualizados.

**Configuración de la cuenta**

Para acceder a estos endpoints, necesitarás:

* Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
* Autenticarte usando las claves y tokens de una [aplicación de desarrollador](/es/resources/fundamentals/developer-apps) que se encuentre dentro de un [Proyecto](/es/resources/fundamentals/projects). 

Obtén más información sobre cómo acceder a los endpoints de X API v2 en nuestra [guía de primeros pasos](/es/x-api/getting-started/getting-access).
