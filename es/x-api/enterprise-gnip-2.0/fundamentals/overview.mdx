## Descripción general

> La consola Enterprise (en console.gnip.com) está disponible para clientes de datos Enterprise con acceso contratado a las API Enterprise. Si actualmente no eres un cliente de datos Enterprise pero deseas más información, puedes obtener más detalles sobre los [datos Enterprise](https://developer.x.com/en/products/x-api/enterprise) aquí.

Esta es una descripción general del panel de la consola Enterprise de X. Las secciones que se tratan corresponden a cada área específica de la consola, para ver detalles de productos, uso y administración general de la cuenta. Si tienes alguna pregunta sobre tu cuenta específica, comunícate con tu administrador de cuenta asignado o revisa nuestra [documentación técnica para productos Enterprise](/es/x-api/enterprise-gnip-2.0/enterprise-gnip).

Como parte del proceso de prueba Enterprise e incorporación con tu administrador de cuenta, los clientes Enterprise recibirán credenciales de acceso a [console.gnip.com](http://console.gnip.com), que es la interfaz de usuario donde se puede revisar y configurar el acceso a los productos Enterprise. El acceso inicial a la consola para el administrador del cliente lo configura X como parte del proceso de incorporación Enterprise, y luego los administradores pueden agregar usuarios adicionales. La consola Enterprise te permite gestionar y acceder a más detalles relacionados con tus productos Enterprise.

<div id="products-tab">
  ## Pestaña Products
</div>

Al iniciar sesión en tu cuenta en [console.gnip.com](http://console.gnip.com), accederás a la pestaña Products del panel. Esta página incluye una vista general de los productos Enterprise que están actualmente disponibles en tu cuenta.

Para productos que usan entrega mediante streaming, como PowerTrack o Decahose, esta página muestra el nombre del producto y la etiqueta del stream, el número actual de conexiones activas, el número de reglas actualmente activas para cada uno (cuando corresponda) y el recuento bruto de actividades (por ejemplo, Publicaciones) entregadas en las últimas 24 horas.

Para productos con entrega REST, como Search API, esta página muestra el nombre del producto, las etiquetas (también mostradas como &quot;streams&quot;), las actividades actuales (por ejemplo, Publicaciones) devueltas a través de estos endpoints y varios recuentos de solicitudes por endpoint.

Ten en cuenta que la [Usage API](/es/x-api/enterprise-gnip-2.0/fundamentals/usage) entrega muchos de estos datos de manera programática.

Los detalles específicos por stream están disponibles haciendo clic en el nombre o en el botón de configuración.

<div id="overview">
  ### Descripción general
</div>

Al hacer clic en uno de los streams en el panel principal, accederás a una página de descripción general de ese stream.

Para los productos de entrega en streaming, esta página incluye lo siguiente:

1. Un gráfico de volumen con la cantidad de actividades que se te entregan a través de cada conexión de stream específica 

2. Detalles (id de conexión y dirección IP) sobre las conexiones actualmente activas en el stream 

3. Un registro de eventos recientes de conexión, desconexión y actualización de reglas de tu stream

Ten en cuenta que la escala del gráfico se puede ajustar con los enlaces en la esquina superior derecha. La visibilidad de conexiones y desconexiones individuales se puede activar o desactivar haciendo clic en el elemento correspondiente en la leyenda directamente debajo del gráfico.

<div id="connections">
  ### Conexiones
</div>

La página Conexiones muestra detalles sobre los eventos de conexión recientes en tu stream. Esto incluye la hora de inicio y de finalización de cada conexión (en formato de 24 horas, UTC), la duración de cada conexión, la dirección IP del servidor que realizó la conexión, un ID de conexión único para referencia y el estado de la conexión actual. El estado corresponde al evento más reciente para la conexión especificada, es decir, Client Connected o una desconexión, con el tipo de desconexión indicado. Para obtener más detalles sobre la depuración de conexiones, visita la [guía sobre desconexiones explicadas](/es/x-api/enterprise-gnip-2.0/powertrack-api#integrating-with-powertrack).

</div>

Además, incluye comandos curl de ejemplo e instrucciones para conectarte al endpoint del stream y para agregar, eliminar y listar reglas de forma programática desde el endpoint de la Rules API de tu stream.

### Rules

La pestaña Rules está disponible para los streams de PowerTrack y proporciona una forma rápida de comenzar introduciendo manualmente reglas en texto plano a través de una interfaz de usuario. Ten en cuenta que la interfaz solo permite agregar hasta 1000 reglas mediante este método manual y debe utilizarse únicamente para las pruebas iniciales. Recomendamos gestionar las reglas de forma programática mediante la [Rules API](/es/x-api/enterprise-gnip-2.0/powertrack-api#api-reference-index) en cualquier entorno de producción.

<div id="settings">
  ### Configuración
</div>

La pestaña Configuración te permite cambiar el formato de salida de los datos de tu stream, para el que se admiten varias opciones de formato. Para cambiar el formato, solo usa los botones de opción que indican las diferentes opciones. El cambio entrará en vigor cuando te vuelvas a conectar al stream. Ten en cuenta que la actualización de esta configuración se aplicará de inmediato en la siguiente solicitud o conexión y que el nuevo formato puede hacer que tu analizador deje de funcionar correctamente. 

**Ten en cuenta:** La configuración recomendada para obtener la mayor cantidad de datos es &quot;Leave data in the original format&quot;, que devolverá los datos en el formato nativo enriquecido descrito [aquí](/es/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#post-object). El formato de los activity streams se describe [aquí](/es/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#activity-object).

<div id="usage-tab">
  ## Pestaña Uso
</div>

La pestaña Uso ofrece información sobre el uso de tus streams en distintos períodos de tiempo. Para acceder de forma programática a la información sobre el uso, consulta la [Usage API](/es/x-api/enterprise-gnip-2.0/fundamentals/usage).

<div id="monthly">
  ### Mensual
</div>

La página de uso mensual muestra el uso de tu stream desglosado por producto. Por ejemplo, los productos de cobertura (como PowerTrack, Historical PowerTrack y PowerTrack Replay) para una fuente de datos determinada se agruparán para proporcionar datos por separado, así como un recuento total combinado. Los recuentos incluyen el acumulado del mes actual hasta la fecha, una estimación de fin de mes (basada en el uso de este mes hasta ahora y el tiempo restante del mes) y los recuentos de los dos meses anteriores.

Cabe destacar que estos recuentos eliminan los duplicados para cada producto y stream. Si recibiste la misma Publicación a través de múltiples conexiones al mismo stream de PowerTrack, esa Publicación se contará solo una vez para estos fines. Los recuentos se actualizarán cada 24 horas a las 00:00 UTC.

<div id="daily">
  ### Diario
</div>

La página de uso diario muestra recuentos diarios sin duplicados para cada día del mes en curso, desglosados por producto. Estos recuentos se actualizan cada 24 horas a las 00:00 UTC.

<div id="rules">
  ### Reglas
</div>

Los límites de reglas se basan en tu paquete de reglas de PowerTrack contratado y se aplican a todos los streams de un mismo tipo. Los recuentos se actualizan cada vez que se añade o elimina una nueva regla.

<div id="account-tab">
  ## Pestaña «Cuenta»
</div>

<div id="my-profile">
  ### Mi perfil
</div>

Aquí puedes ver los detalles de tu perfil de usuario individual y cambiar tu contraseña. Además, puedes configurar notificaciones de umbral de uso específicas para tu cuenta. Las notificaciones sobre el estado de la API están disponibles en la página de estado en https://api.twitterstat.us.

<div id="account-settings">
  ### Configuración de la cuenta
</div>

Puede agregar, eliminar y editar usuarios, y configurar notificaciones por correo electrónico para usuarios individuales y alertas de umbral de uso.

<Frame>
  <img src="/images/account-settings.png.twimg.1920.png" />
</Frame>

Tenga en cuenta que hay tres tipos de usuarios: Administrador de cuenta, Usuario y Solo correo electrónico.

* Los administradores de cuenta pueden crear, eliminar y editar otros usuarios, y pueden usar autenticación básica con nombre de usuario (correo electrónico) y contraseña para conectarse a las API de Enterprise.

* Los usuarios no pueden crear ni modificar otros usuarios, pero pueden usar autenticación básica con nombre de usuario (correo electrónico) y contraseña para conectarse a las API de Enterprise.

* Los usuarios de solo correo electrónico no tienen acceso al panel de control, no están autorizados para conectarse a las API de Enterprise y solo reciben notificaciones si están configurados para recibirlas en la sección Notificaciones.

<div id="usage-thresholds">
  ### Umbrales de uso
</div>

Configura los umbrales de volumen para tus productos. Estos activarán alertas por correo electrónico para los usuarios que hayan configurado dichas notificaciones en sus perfiles, tanto para el umbral de advertencia como para el umbral crítico de cada producto.

<Frame>
  <img src="/images/usage-thresholds.png.twimg.1920.png" />
</Frame>

Ten en cuenta que estos umbrales se evalúan una vez al día, a las 19:30 (UTC), y no se evalúan en tiempo real.
