Los endpoints de Enterprise se han actualizado para proporcionar metadatos de Publicaciones editadas. La funcionalidad &#95;Edit Posts&#95; se introdujo por primera vez para pruebas entre empleados de X el 1 de septiembre de 2022. A partir de esa fecha, las Publicaciones elegibles podían editarse durante 30 minutos y hasta 5 veces. *Todos los objetos de Publicaciones creadas desde el 29 de septiembre de 2022* incluyen metadatos de edición de la Publicación, incluso si la Publicación nunca se editó. Cada vez que se edita una Publicación, se crea un nuevo ID de Publicación. El historial de edición de una Publicación puede describirse encadenando estos IDs, comenzando con el ID original. Además, si alguna Publicación de la cadena de edición se elimina, todas las Publicaciones de esa cadena también se eliminan.

Estos metadatos se incluyen automáticamente. No se necesitan parámetros en la solicitud para incluir el historial de edición disponible como parte del objeto de la Publicación.

Con estos nuevos metadatos, un desarrollador puede averiguar:

* Si una Publicación era elegible para edición en el momento de su creación. Algunas Publicaciones, como aquellas con encuestas o Publicaciones programadas, no se pueden editar.
* Que las Publicaciones son editables durante 30 minutos y se pueden editar hasta 5 veces. Para las Publicaciones editables, puedes ver si aún queda tiempo para editar y cuántas ediciones más son posibles.
* Si estás viendo una versión editada de una Publicación. En la mayoría de los casos, la API devolverá la versión más reciente de una Publicación, a menos que se solicite una versión anterior específica mediante el ID de Publicación.

Se han añadido tres nuevos atributos de la Publicación en el nivel raíz:

* **edit&#95;history** - Proporciona todos los IDs de Publicaciones asociados con el historial de edición de la Publicación. El atributo &quot;initial&#95;tweet&#95;id&quot; indica la Publicación original y el atributo &quot;edit&#95;tweet&#95;ids&quot; es un array que proporciona todos los IDs asociados con su historial de edición. Si la Publicación no ha sido editada, este array contendrá un único ID.

```
"edit_history": {
    "initial\_tweet\_id": "1283764123"
    "edit\_tweet\_ids": \["1283764123"\]
  }
```

* **edit&#95;controls** - Proporciona atributos que indican cuándo termina la ventana de edición de 30 minutos y cuántas ediciones potenciales restan.

```
"edit_controls": {  
     "editable\_until\_ms": 1660155761384
     "edits_remaining": 3   
  }
```

* **editable** - Indica si una Publicación era apta para ser editada cuando se creó. 

&quot;editable&quot;: true

La mayoría de las Publicaciones son aptas. Sin embargo, los siguientes tipos de Publicaciones no lo son: 

* La Publicación está promocionada
* La Publicación tiene una encuesta
* La Publicación es una respuesta en un hilo de otra persona
* La Publicación es un Retweet (ten en cuenta que las Publicaciones citadas sí son aptas para edición)
* La Publicación es nullcast
* Publicación de Comunidad
* Publicación de Superfollow
* Publicación colaborativa

**Ejemplo de atributos para una Publicación no editada**

El JSON siguiente resalta los metadatos de edición que se incluyen para una Publicación publicada después de que se añadiera la función de edición de Publicaciones. Este ejemplo es para una Publicación que no tiene historial de edición. 

Ten en cuenta que el array `"edit_tweet_ids"` tiene un solo id.

```
{
  "created_at": "Wed Aug 16 18:29:02 +0000 2022",
  "id": 1557433858676740098,
  "id_str": "1557433858676740098",
  "text": "I wonder if I will every use teh edit button",
  "edit_history": {
    "initial_tweet_id": "1557433858676740098",
    "edit_tweet_ids": ["1557433858676740098"]
  },
  "edit_controls": {
    "editable_until_ms": 1660155761384,
    "edits_remaining": 5
  },
  "editable": true
}
```

**Atributos de ejemplo para una Publicación editada**

El siguiente JSON destaca los metadatos de edición que se incluyen para una Publicación creada después de que se añadió la función de editar Publicaciones. Este ejemplo corresponde a una Publicación que solo se ha editado una vez. 

Ten en cuenta que el array `"edit_tweet_ids"` contiene dos ids, uno para la Publicación original y otro para la versión editada.

```
{
  "created_at": "Wed Aug 16 18:35:42 +0000 2022",
  "id": 1557445923210514432,
  "id_str": "1557445923210514432",
  "text": "I wonder if I will ever use the edit button",
  "edit_history": {
    "initial_tweet_id": "1557433858676740098",
    "edit_tweet_ids": ["1557433858676740098", "1557445923210514432"]
  },
  "edit_controls": {
    "editable_until_ms": 1660155761384,
    "edits_remaining": 4
  },
  "editable": true
}
```

**Compatibilidad con el cumplimiento normativo**

El [Compliance Firehose](/es/x-api/enterprise-gnip-2.0/fundamentals/firehouse) y el [endpoint de cumplimiento por lotes](/es/x-api/compliance/batch-compliance/introduction) de v2 se han actualizado para admitir la edición de Publicaciones: 

Se ha añadido un nuevo tipo de evento &quot;tweet&#95;edit&quot; al Compliance Firehose.

```
{
  "tweet_edit": {
    "id": <tweetId>,
    "initial_tweet_id": <tweetId>,
    "edit_tweet_ids": [<tweetId1>, <tweetId2>, <tweetId3> ...],
    "timestamp_ms": "<timestampMsStr>"
  }
}
```
