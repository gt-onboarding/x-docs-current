Un objeto de contenido multimedia representa una única foto, video o GIF animado. Los objetos de contenido multimedia son utilizados por muchos endpoints en la X API y pueden incluirse en Publicaciones, Mensajes Directos, perfiles de usuario, creativos publicitarios y otros contextos. Cada objeto de contenido multimedia puede tener múltiples variantes de visualización o reproducción, con diferentes resoluciones o formatos.

<div id="media-types-size-restrictions">
  ## Tipos de contenido multimedia y restricciones de tamaño
</div>

Restricciones de tamaño para subir contenido mediante la API

* **Imagen**: `5 MB`
* **GIF**: `15 MB`
* **Vídeo**: `512 MB` (al usar `media_category=amplify_video`)

<div id="creation">
  ## Creación
</div>

Objetos como Publicaciones, Mensajes Directos, fotos de perfil de usuario, tarjetas de Ads alojadas, etc. pueden contener uno o más objetos de contenido multimedia. Estos objetos de nivel superior se conocen colectivamente como entidades. A la API de creación de entidades correspondiente (por ejemplo, [`POST /2/tweets`](/es/x-api/posts/creation-of-a-post)) se le pueden pasar uno o más objetos de contenido multimedia usando un `media_id` único.

Una entidad que contiene uno o más objetos de contenido multimedia se puede crear siguiendo estos pasos:

1. Sube los archivos de contenido multimedia utilizando la carga [segmentada](/es/x-api-/media/quickstart/media-upload-chunked) recomendada (imágenes/GIF/video) o la carga [simple](/es/x-api/media-upload/media-upload) anterior (solo imágenes).
2. Obtén un `media_id` del paso 1. Este paso se puede repetir varias veces con diferentes contenidos multimedia si la entidad permite pasar varios parámetros `media_id`.
3. Crea la entidad llamando al endpoint correspondiente, incluyendo el `media_id` y otros parámetros requeridos. Por ejemplo, adjunta un `media_id` a una Publicación usando el endpoint [`POST /2/tweets`](/es/x-api/posts/creation-of-a-post).
