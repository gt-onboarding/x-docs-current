Un objeto de contenido multimedia representa una única foto, video o GIF animado. Los objetos de contenido multimedia se utilizan en muchos endpoints dentro de la X API y pueden incluirse en Publicaciones, Mensajes Directos, perfiles de usuario, elementos creativos publicitarios y otros contextos. Cada objeto de contenido multimedia puede tener múltiples variantes de visualización o reproducción, con diferentes resoluciones o formatos.

<div id="media-types-size-restrictions">
  ## Tipos de contenido multimedia y restricciones de tamaño
</div>

Restricciones de tamaño para cargar mediante la API

* **Imagen**: `5 MB`
* **GIF**: `15 MB`
* **Video**: `512 MB` (al usar `media_category=amplify_video`)

<div id="creation">
  ## Creación
</div>

Objetos como Publicaciones, Mensajes Directos, fotos de perfil de usuario, tarjetas de anuncios alojadas, etc. pueden contener uno o más objetos de contenido multimedia. Estos objetos de nivel superior se conocen colectivamente como entidades. A la API de creación de entidades correspondiente (por ejemplo, [`POST /2/tweets`](/es/x-api/posts/creation-of-a-post)) se le pueden pasar uno o más objetos de contenido multimedia mediante un `media_id` único.

Para crear una entidad que contenga uno o más objetos de contenido multimedia, sigue estos pasos:

1. Carga el/los archivo(s) de contenido multimedia utilizando la carga [fragmentada](/es/x-api-/media/quickstart/media-upload-chunked) recomendada (imágenes/GIF/video) o la carga [simple](/es/x-api/media-upload/media-upload) más antigua (solo imágenes).
2. Obtén un `media_id` del paso 1. Este paso se puede repetir varias veces con contenido multimedia diferente si la entidad permite que se pasen varios parámetros `media_id`.
3. Crea la entidad llamando al endpoint correspondiente e incluyendo el `media_id` y otros parámetros obligatorios. Por ejemplo, adjunta un `media_id` a una Publicación usando el endpoint [`POST /2/tweets`](/es/x-api/posts/creation-of-a-post).
