<div id="overview">
  ## Descripción general
</div>

Un objeto de contenido multimedia representa una única foto, un vídeo o un GIF animado. Los objetos de contenido multimedia se utilizan en muchos endpoints de la API de Twitter y pueden incluirse en Publicaciones, Mensajes Directos, perfiles de usuario, creatividades publicitarias y otros contextos. Cada objeto de contenido multimedia puede tener varias variantes de visualización o reproducción, con diferentes resoluciones o formatos.

<div id="media-types-size-restrictions">
  ## Tipos de contenido multimedia y restricciones de tamaño
</div>

Restricciones de tamaño para subir mediante la API 

- Imagen: 5 MB
- GIF: 15 MB
- Vídeo: 512 MB (cuando se usa media_category=amplify)

<div id="creation">
  ## Creación
</div>

Objetos como Publicaciones, Mensajes Directos, fotos de perfil de usuario, tarjetas de Ads alojadas, etc. pueden contener uno o más objetos de contenido multimedia. Estos objetos de nivel superior se conocen colectivamente como entidades. A la API de creación de entidades correspondiente (por ejemplo, POST statuses/update) se le pueden pasar uno o más objetos de contenido multimedia usando un media_id único.

Se puede crear una entidad que contenga uno o más objetos de contenido multimedia siguiendo estos pasos:

1. Sube uno o varios archivos de contenido multimedia utilizando la carga fragmentada recomendada (imágenes/GIF/video) o el método de carga simple anterior (solo imágenes).
2. Recibe un media_id del paso 1. Este paso se puede repetir varias veces con contenido multimedia diferente si la entidad permite que se pasen varios parámetros media_id.
3. Crea la entidad llamando al endpoint correspondiente, incluyendo el media_id y otros parámetros obligatorios. Por ejemplo, adjunta un media_id a una Publicación utilizando el endpoint POST statuses/update.

  ## Recuperar
</div>
