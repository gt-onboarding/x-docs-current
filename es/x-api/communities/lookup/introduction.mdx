<Note>
  **Configuración de la cuenta**

  Para acceder a estos endpoints, necesitarás:

  * Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
  * Autenticarte usando las claves y tokens de una [developer App](/es/resources/fundamentals/developer-apps) que esté contenida en un [Proyecto](/es/resources/fundamentals/projects). 

  Obtén más información sobre cómo acceder a los endpoints de la X API v2 en nuestra [guía de introducción](/es/x-api/getting-started/getting-access).
</Note>

**Introducción**

<div id="authentication">
  ### Autenticación
</div>

Puedes autenticar las solicitudes a este endpoint con [Contexto de usuario de OAuth 1.0a](/es/resources/fundamentals/authentication), [OAuth 2.0 solo para la aplicación](https://developer.x.com(/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), o [OAuth 2.0 Authorization Code with PKCE](resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).
 

<div id="making-a-request">
  ### Realizar una solicitud
</div>

Puedes llamar al endpoint Communities Lookup proporcionando un id, como se muestra a continuación (usando el id `1758747817642700922`):

```
      curl --location 'https://api.x.com/2/communities/1758747817642700922' --header 'Authorization: ••••••'
```

Si la solicitud se realiza con éxito, deberías ver la respuesta JSON que se muestra a continuación:

```
{
    "data": {
        "access": "Public",
        "description": "Welcome to the Anime Community! Where anime fans gather to share their favorite shows and discuss everything anime-related.",
        "join_policy": "Open",
        "name": "Anime Community"
    }
}
```

<div id="community-fields">
  ### Campos de la comunidad
</div>

Puedes obtener datos adicionales asociados a una comunidad especificando los campos de comunidad.

Los siguientes campos de comunidad están disponibles:

| Valor del campo | Tipo | Descripción |     |
| :--- | :--- | :--- | :--- |
| created&#95;at | date (ISO 8601) | Fecha y hora de creación de la comunidad. |     |
| id  | string | El identificador único de la comunidad. |     |
| name | string | El nombre de la comunidad. |     |
| description | string | El texto de la descripción de la comunidad, si se ha proporcionado. |     |
| access | string | El nivel de acceso de la comunidad.  <br />  <br />Puede tener uno de los siguientes valores:<br /><br />* `Public`<br />* `Closed` |     |
| join&#95;policy | string | La política para unirse a la comunidad.  <br />  <br />Puede tener uno de los siguientes valores:<br /><br />* `Open`<br />* `RestrictedJoinRequestsDisabled`<br />* `RestrictedJoinRequestsRequireAdminApproval`<br />* `RestrictedJoinRequestsRequireModeratorApproval`<br />* `SuperFollowRequired` |     |
| member&#95;count | integer | El número de miembros que se han unido a la comunidad. |     |

<div id="example-request">
  #### Ejemplo de solicitud
</div>

Puedes llamar al endpoint Communities Lookup con campos de comunidad adicionales, como se muestra a continuación (usando el ID `1758747817642700922`):

```
curl --location 'https://api.x.com/2/communities/1758747817642700922?community.fields=access,created_at,description,id,join_policy,member_count,name' --header 'Authorization: ••••••'
```

**Respuesta de ejemplo**

Si la solicitud se completa correctamente, verás la respuesta JSON que se muestra a continuación:

```
{
    "data": {
        "description": "Welcome to the Anime Community! Where anime fans gather to share their favorite shows and discuss everything anime-related.",
        "access": "Public",
        "member_count": 40369,
        "created_at": "2024-02-17T06:58:50.000Z",
        "join_policy": "Open",
        "name": "Anime Community",
        "id": "Q29tbXVuaXR5OjE3NTg3NDc4MTc2NDI3MDA5MjI="
    }
}
```
